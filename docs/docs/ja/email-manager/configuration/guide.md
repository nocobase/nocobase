# 設定の流れ

## 概要
メール**プラグイン**を有効にした後、一般ユーザーがメールアカウントをNocoBaseに接続できるようにするには、管理者がまず関連する設定を完了する必要があります。（現在、OutlookおよびGmailアカウントの認証ログインのみをサポートしており、MicrosoftアカウントやGoogleアカウントの直接接続はまだサポートしていません。）

設定の核となるのは、メールサービスプロバイダーのAPI呼び出しにおける認証設定です。**プラグイン**が正常に機能するために、管理者は以下の手順を完了する必要があります。

1.  **サービスプロバイダーから認証情報を取得する**
    -   メールサービスプロバイダーのデベロッパーコンソール（例：Google Cloud ConsoleやMicrosoft Azure Portal）にログインします。
    -   新しいアプリケーションまたはプロジェクトを作成し、GmailまたはOutlookメールAPIサービスを有効にします。
    -   対応するクライアントID（Client ID）とクライアントシークレット（Client Secret）を取得します。
    -   リダイレクトURIを設定し、NocoBaseの**プラグイン**コールバックアドレスと一致していることを確認します。

2.  **メールサービスプロバイダーの設定**
    -   メール**プラグイン**の設定ページに移動します。
    -   必要なAPI認証情報（クライアントID、クライアントシークレットなど）を提供し、メールサービスプロバイダーとの認証連携が正常に行われるようにします。

3.  **認証ログイン**
    -   ユーザーはOAuthプロトコルを通じてメールアカウントにログインします。
    -   **プラグイン**は、その後のAPI呼び出しやメール操作のために、ユーザーの認証トークンを自動的に生成・保存します。

4.  **メールアカウントの接続**
    -   認証が成功すると、ユーザーのメールアカウントはNocoBaseに接続されます。
    -   **プラグイン**はユーザーのメールデータを同期し、メールの管理、送受信機能を提供します。

5.  **メール機能の利用**
    -   ユーザーはプラットフォーム内で直接、メールの閲覧、管理、送信などを行うことができます。
    -   すべての操作はメールサービスプロバイダーのAPI呼び出しを通じて完了し、リアルタイム同期と効率的な転送を保証します。

上記のプロセスを通じて、NocoBaseのメール**プラグイン**は、ユーザーに効率的で安全なメール管理サービスを提供します。設定中に問題が発生した場合は、関連ドキュメントを参照するか、テクニカルサポートチームにお問い合わせください。

## **プラグイン**設定

### メール**プラグイン**の有効化

1.  **プラグイン**管理ページに移動します
2.  「Email manager」**プラグイン**を見つけて有効にします

### メールサービスプロバイダーの設定

メール**プラグイン**を有効にした後、メールサービスプロバイダーの設定を行うことができます。現在、GoogleとMicrosoftの2種類のメールサービスをサポートしています。上部の「設定」->「メール設定」をクリックして、設定ページに移動します。
