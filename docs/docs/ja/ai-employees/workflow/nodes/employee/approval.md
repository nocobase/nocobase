# ノードの承認

## 通知

ワークフロー内の AI 従業員ノードがトリガーされると、ノードに設定された承認者はホームページの AI 従業員入口に赤いドットの通知が表示されます。

![20260420171416](https://static-docs.nocobase.com/20260420171416.png)

AI 従業員のセッション一覧に入ると、承認待ちのワークフロータスクが表示されます。

![20260420171642](https://static-docs.nocobase.com/20260420171642.png)

## 承認

ワークフロータスクをクリックしてセッションウィンドウを開くと、AI 従業員の過去のメッセージが表示されます。AI 従業員が最後に送信したカードが、現在のノードで承認が必要な内容です。

![20260420180710](https://static-docs.nocobase.com/20260420180710.png)

承認待ちの内容に対しては、3 つの操作があります：Approve（承認）、Revise（修正）、Reject（拒否）。

「Approve」をクリックすると、AI が生成した内容の使用が承認され、ワークフローが続行されます。セッションは読み取り専用状態になり、AI 従業員との会話を続けることはできません。

![20260420180632](https://static-docs.nocobase.com/20260420180632.png)

「Revise」をクリックすると、AI に修正意見を提出します。送信後、AI が新しい承認カードで返信します。

![20260420180908](https://static-docs.nocobase.com/20260420180908.png)

「Reject」をクリックすると、実行を拒否し、ワークフローが停止します。セッションは読み取り専用状態になり、AI 従業員との会話を続けることはできません。

![20260420181003](https://static-docs.nocobase.com/20260420181003.png)
