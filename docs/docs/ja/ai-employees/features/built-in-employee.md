# 組み込み AI 従業員

NocoBase には、特定のシナリオに特化した複数の AI 従業員がプリセットされています。

LLM サービスを設定して対応する従業員を有効にするだけで、すぐに業務を開始できます。モデルは会話の中で必要に応じて切り替えることが可能です。


## 紹介

![clipboard-image-1766653060](https://static-docs.nocobase.com/clipboard-image-1766653060.png)

| 従業員名 | ロール | コア能力 |
| :--- | :--- | :--- |
| **Cole** | NocoBase アシスタント | 製品の使用方法に関する Q&A、ドキュメント検索 |
| **Ellis** | メールエキスパート | メールの作成、要約の生成、返信案の提案 |
| **Dex** | データ整理エキスパート | フィールドの翻訳、フォーマット、情報の抽出 |
| **Viz** | インサイトアナリスト | データインサイト、トレンド分析、主要指標の解釈 |
| **Lexi** | 翻訳アシスタント | 多言語翻訳、コミュニケーション補助 |
| **Vera** | リサーチアナリスト | ネット検索、情報の集計、詳細な調査 |
| **Dara** | データ視覚化エキスパート | チャート設定、視覚化レポートの生成 |
| **Orin** | データモデリングエキスパート | コレクション構造の設計補助、フィールドの提案 |
| **Nathan** | フロントエンドエンジニア | フロントエンドのコードスニペット作成補助、スタイル調整 |


アプリケーション画面の右下にある **AI フローティングボタン**をクリックし、必要な従業員を選択することで、コラボレーションを開始できます。


## 特定シナリオ専用の AI 従業員

一部の組み込み AI 従業員（構築系）は、右下の AI 従業員リストには表示されません。これらには以下のような専用の作業シナリオがあります：

* **Orin**: データソース設定ページにのみ表示されます。
* **Dara**: チャート設定ページにのみ表示されます。
* **Nathan**: JS エディタ上にのみ表示されます。



---

以下に、AI 従業員の代表的な活用シナリオをいくつか挙げます。これらがあなたのインスピレーションの一助となれば幸いです。さらなる可能性については、実際の業務の中でぜひ探索してみてください。


## Viz：インサイトアナリスト

### 紹介

> ワンクリックでチャートとインサイトを生成し、データに自ら語らせる。

**Viz** は、組み込みの **AI インサイトアナリスト**です。
彼は現在のページのデータ（リード、案件、顧客など）を読み取り、トレンドチャート、比較グラフ、KPI カード、および簡潔な結論を自動的に生成します。これにより、ビジネス分析が簡単かつ直感的になります。

> 「最近、なぜ売上が下がっているのか？」を知りたいですか？
> Viz に一言伝えるだけで、どの部分で減少しているのか、考えられる原因は何か、そして次に何をすべきかを教えてくれます。

### 活用シナリオ

月次の経営レビュー、チャネル ROI、販売ファネルなど、あらゆる場面で Viz に分析、チャート生成、結果の解釈を任せることができます。

| シナリオ | 知りたい内容 | Viz のアウトプット |
| -------- | ------------ | ------------------- |
| **月次レビュー** | 今月は先月と比べてどこが良かったか？ | KPI カード + トレンドチャート + 3つの改善提案 |
| **成長要因の分解** | 売上増は数量によるものか、単価によるものか？ | 要因分解図 + 比較表 |
| **チャネル分析** | どのチャネルに継続して投資すべきか？ | ROI チャート + 継続率曲線 + 提案 |
| **ファネル分析** | トラフィックはどのステップで滞留しているか？ | ファネルチャート + ボトルネックの説明 |
| **顧客維持** | どの顧客が最も価値が高いか？ | RFM セグメンテーション図 + 継続率曲線 |
| **プロモーション評価** | 大規模キャンペーンの効果はどうだったか？ | 比較チャート + 価格弾力性分析 |

### 使用方法

**ページ入り口**

* **右上のボタン（推奨）**
  
  リード、案件、顧客などのページの右上にある **Viz アイコン**をクリックすると、以下のようなプリセットタスクを選択できます：

  * フェーズ転換とトレンド
  * 流入チャネルの比較
  * 月次レビュー分析

  ![clipboard-image-1771913319](https://static-docs.nocobase.com/clipboard-image-1771913319.png)

* **右下のグローバルパネル**
  
  どのページからでも、グローバル AI パネルを呼び出して Viz に直接話しかけることができます：

  ```
  直近90日間の売上変化を分析して
  ```

  Viz は、現在表示しているページのデータコンテキストを自動的に取り込みます。

**インタラクション形式**

Viz は自然言語での質問をサポートしており、多段階の追質問も理解します。
例：

```
Hi Viz、今月のリードのトレンドを生成して。
```

```
サードパーティチャネルのパフォーマンスだけを見せて。
```

```
じゃあ、どの地域の成長が一番早いの？
```

追質問のたびに、前回の分析結果に基づいてさらに深く掘り下げていくため、データ条件を繰り返し入力する必要はありません。

### Viz と対話する際のコツ

| 方法 | 効果 |
| ---------- | ------------------- |
| 時間範囲を指定する | 「直近30日間」「先月 vs 今月」など、より正確になります |
| ディメンションを指定する | 「地域別/チャネル別/製品別に見る」ことで視点を合わせやすくなります |
| 詳細よりもトレンドに注目する | Viz は変化の方向性と主要な原因を見つけるのが得意です |
| 自然言語で話す | 命令的な構文は不要です。チャットするように質問してください |

## Dex：データ整理エキスパート

### 紹介

> フォームを素早く抽出・入力し、乱雑な情報を構造化データに変える。

`Dex` はデータ整理の専門家です。非構造化データやファイルから必要な情報を抽出して構造化された情報に整理し、ツールを呼び出してその情報をフォームに入力することができます。

### 使用方法

フォームページで `Dex` を呼び出し、対話ウィンドウを開きます。

![20251022155316](https://static-docs.nocobase.com/20251022155316.png)

入力ボックスの `Add work context` をクリックし、`Pick block` を選択します。ページがブロック選択状態になります。

![20251022155414](https://static-docs.nocobase.com/20251022155414.png)

ページ上のフォームブロックを選択します。

![20251022155523](https://static-docs.nocobase.com/20251022155523.png)

ダイアログボックスに `Dex` に整理させたいデータを入力します。

![20251022155555](https://static-docs.nocobase.com/20251022155555.png)

送信後、`Dex` はデータを構造化し、スキルを使用して選択されたフォームにデータを更新します。

![20251022155733](https://static-docs.nocobase.com/20251022155733.png)

## Orin：データモデリングエキスパート

### 紹介

> データテーブルをインテリジェントに設計し、データベース構造を最適化する。

`Orin` はデータモデリングの専門家です。メインデータソースの設定ページで、`Orin` にデータテーブル（コレクション）の作成や変更を依頼できます。

![20251022160628](https://static-docs.nocobase.com/20251022160628.png)

### 使用方法

データソース管理プラグインに入り、メインデータソースの設定を選択します。

![20251022161146](https://static-docs.nocobase.com/20251022161146.png)

右上の `Orin` のアバターをクリックし、AI 従業員ダイアログを開きます。

![20251022161641](https://static-docs.nocobase.com/20251022161641.png)

`Orin` にモデリングの要件を伝え、送信して返信を待ちます。

`Orin` が要件を確認すると、スキルを使用してデータモデリングのプレビューを返信します。

プレビューを確認後、`Finish review and apply` ボタンをクリックすると、`Orin` のモデリングに従ってデータテーブルが作成されます。

![20251022162142](https://static-docs.nocobase.com/20251022162142.png)

## Nathan：フロントエンドエンジニア

### 紹介

> フロントエンドコードの記述と最適化を支援し、複雑なインタラクションロジックを実現する。

`Nathan` は NocoBase 内のフロントエンド開発エキスパートです。`JSBlock`, `JSField`, `JSItem`, `JSColumn`, `JSAction`, `Event Flow`, `Linkage` など、JavaScript の記述が必要なシーンにおいて、コードエディタの右上に `Nathan` のアバターが表示されます。彼にコードの作成や修正を依頼することができます。

![20251022163405](https://static-docs.nocobase.com/20251022163405.png)

### 使用方法

コードエディタで `Nathan` をクリックして AI 従業員ダイアログを開くと、エディタ内のコードが自動的に入力ボックスに添付され、アプリケーションコンテキストとして `Nathan` に送信されます。

![20251022163935](https://static-docs.nocobase.com/20251022163935.png)

コーディングの要件を入力して `Nathan` に送信し、返信を待ちます。

![20251022164041](https://static-docs.nocobase.com/20251022164041.png)

`Nathan` が返信したコードブロックにある `Apply to editor` ボタンをクリックすると、彼のコードがエディタ内のコードに上書きされます。

![20251022164323](https://static-docs.nocobase.com/20251022164323.png)

コードエディタの `Run` ボタンをクリックすると、リアルタイムで効果を確認できます。

![20251022164436](https://static-docs.nocobase.com/20251022164436.png)

### コード履歴

`Nathan` のダイアログ右上にある「コマンドライン」アイコンをクリックすると、現在のセッションで送信したコードスニペットと `Nathan` が返信したコードスニペットの履歴を確認できます。

![20251022164644](https://static-docs.nocobase.com/20251022164644.png)
