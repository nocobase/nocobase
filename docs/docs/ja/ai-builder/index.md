# AI ビルダー クイックスタート

AI ビルダーは NocoBase が提供する AI アシスト構築機能です。自然言語で要件を記述するだけで、AI がデータモデリング、ページ設定、権限設定などの操作を自動的に行います。より効率的で、よりモダンな構築体験を提供します。

## クイックスタート

すでに [NocoBase CLI](../ai/quick-start.md) をインストール済みの場合は、このステップをスキップできます。

### ワンクリック AI インストール

以下のプロンプトを AI アシスタント（Claude Code、Codex、Cursor、Trae など）にコピーすると、インストールと設定が自動的に完了します：

```
NocoBase CLI をインストールして初期化してください：https://docs.nocobase.com/cn/ai/ai-quick-start.md （リンク先の内容を直接参照してください）
```

### 手動インストール

```bash
npm install -g @nocobase/cli@beta
nb init --ui
```

ブラウザが自動的にビジュアル設定ページを開き、NocoBase Skills のインストール、データベースの設定、アプリケーションの起動をガイドします。詳細な手順は[クイックスタート](../ai/quick-start.md)を参照してください。

## 会話で手動設定を置き換える

NocoBase CLI のインストールが完了したら、AI アシスタントで自然言語を使って NocoBase を操作できます。以下はいくつかの実際のシナリオです。テーブルの作成からシステム全体の構築まで、AI ビルダーの能力を体験してみてください。

### 業務要件を伝えると、AI がテーブルとリレーションを設計

どんなシステムを作りたいか AI に伝えるだけで、データテーブル、フィールドタイプ、リレーションを自動的に設計します。ER 図を自分で描く必要はありません。

```
CRM を構築しています。データモデルを設計して構築してください
```

![AI が CRM データモデルを設計](https://static-docs.nocobase.com/202604162126729.png)

AI が顧客、連絡先、商談、注文などのデータテーブルと、それらの間のリレーションを自動生成しました：

![CRM データモデルの結果](https://static-docs.nocobase.com/202604162201867.png)

データモデリングの詳しい使い方は、[データモデリング](./data-modeling)を参照してください。

### 業務用語でページを記述すると、AI が構築

設定ルールを学ぶ必要はありません。検索ボックス、テーブル、フィルター条件など、欲しいページを直接伝えるだけで完成します。

```
顧客管理ページを作成してください。名前検索ボックスと顧客テーブルを含め、テーブルには名前、電話番号、メール、作成日時を表示してください
```

![顧客管理ページ](https://static-docs.nocobase.com/20260420100608.png)

UI 設定の詳しい使い方は、[UI 設定](./ui-builder)を参照してください。

### 一言で自動化ワークフローを構築

業務フローのトリガー条件と処理ロジックを記述すると、AI がトリガーとノードチェーンを自動的に作成します。

```
注文作成後に商品在庫を自動的に減らすワークフローを構築してください
```

![注文在庫減少ワークフロー](https://static-docs.nocobase.com/20260419234303.png)

ワークフローの詳しい使い方は、[ワークフロー管理](./workflow)を参照してください。

### データテーブル、ページ、ダッシュボードを一括構築

:::warning 注意

ソリューション機能は現在テスト中であり、安定性に制限があります。お試し用としてご利用ください。

:::

業務シナリオを一言で記述するだけで、AI がデータテーブル、管理ページ、ダッシュボード、チャートをすべて構築します。

```
nocobase-dsl-reconciler skill を使って、ダッシュボード、チケット一覧、ユーザー管理、SLA 設定を含むチケット管理システムを構築してください
```

AI がまず設計案を出力し、確認後に一括で構築を完了します：

![チケットシステム設計案](https://static-docs.nocobase.com/20260420100420.png)

![チケットシステム構築結果](https://static-docs.nocobase.com/20260420100450.png)

システム全体の構築について詳しくは、[ソリューション](./dsl-reconciler)を参照してください。

## セキュリティと監査

AI Agent に NocoBase を操作させる前に、認証方式、権限制御、操作監査について理解しておくことをお勧めします。AI が適切な操作のみを行い、すべてのステップが記録されるようにしましょう。[セキュリティと監査](./security)を参照してください。

## NocoBase Skills

[NocoBase Skills](https://github.com/nocobase/skills) は AI Agent にインストール可能なドメイン知識パッケージで、AI が NocoBase の設定体系を理解できるようにします。NocoBase は 8 つの Skills を提供し、構築プロセス全体をカバーしています：

- [環境管理](./env-bootstrap) — 環境チェック、インストール、アップグレード、トラブルシューティング
- [データモデリング](./data-modeling) — データテーブル、フィールド、リレーションの作成と管理
- [UI 設定](./ui-builder) — ページ、ブロック、ポップアップ、インタラクション連動の作成と編集
- [ワークフロー管理](./workflow) — ワークフローの作成、編集、有効化、診断
- [権限設定](./acl) — ロール、権限ポリシー、ユーザー割り当て、リスク評価の管理
- [ソリューション](./dsl-reconciler) — YAML による業務システムの一括構築
- [プラグイン管理](./plugin-manage) — プラグインの一覧表示、有効化、無効化
- [リリース管理](./publish) — クロス環境リリース、バックアップ復元、マイグレーション

:::tip ヒント

NocoBase CLI は初期化プロセス（`nb init`）で Skills を自動的にインストールするため、手動でのインストールは不要です。

:::
