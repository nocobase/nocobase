# 権限設定

:::tip 前提条件

このページを読む前に、[AI ビルダー クイックスタート](./index.md)に従って NocoBase CLI のインストールと初期化が完了していることを確認してください。

:::

## 概要

権限設定 Skill は、自然言語で NocoBase のロール、権限ポリシー、ユーザー割り当て、ACL リスク評価を管理するためのツールです。業務目標を記述すると、適切なコマンドとパラメータが自動的に選択されます。


## 対応機能

- 新しいロールの作成
- グローバルロールモードの切り替え（独立モード / 統合モード）
- データテーブルのアクション権限とデータスコープの一括設定
- ユーザーとロールの関連付け解除
- ロールレベル、ユーザーレベル、システムレベルのリスク評価レポートの出力

## プロンプト例

### シナリオ A：ユーザーの一括割り当て
:::tip 前提条件
現在の環境に Member ロールと複数のユーザーが存在すること
:::

```
これらの新しいユーザーに Member ロールを割り当ててください：James、Emma、Michael
```

![ユーザーの一括割り当て](https://static-docs.nocobase.com/20260422202343.png)

### シナリオ B：ページ権限の一括設定
:::tip 前提条件
現在の環境に Member ロールと複数のページが存在すること
:::
```
Member ロールにこれらのページの権限を設定してください：Product、Order、Stock
```

![ページ権限の一括設定](https://static-docs.nocobase.com/20260422202949.png)

### シナリオ C：複数データテーブルの権限一括設定
:::tip 前提条件
現在の環境に Member ロールと複数のデータテーブルが存在すること
:::

```
Member ロールにこれらのデータテーブルの独立読み取り専用権限を追加してください：order、product、stock
```

![データテーブル独立権限の一括設定](https://static-docs.nocobase.com/20260422205341.png)

![データテーブル独立権限の一括設定2](https://static-docs.nocobase.com/20260422205430.png)

### シナリオ D：複数ロール・複数データテーブルの権限設定
:::tip 前提条件
現在の環境に複数のロールと複数のデータテーブルが存在すること
:::

```
Member、Sales ロールにこれらのデータテーブルの独立読み書き権限を追加してください：order、product、stock
```

![複数ロール・複数データテーブルの設定](https://static-docs.nocobase.com/20260422213524.png)

### シナリオ E：リスク評価

```
Member ロールの権限リスクを評価してください
```

リスクスコア、影響範囲の説明、改善提案が出力されます。

## よくある質問

**権限を設定したのに反映されない場合は？**

まず、グローバルロールモードが正しいか確認してください。ユーザーが複数のロールを持っている場合、統合モードと独立モードでは動作が大きく異なります。現在のモードを確認することで問題を特定できます。
