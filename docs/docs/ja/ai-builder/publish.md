# リリース管理

:::tip 前提条件

- このページを読む前に、[AI ビルダー クイックスタート](./index.md)に従って NocoBase CLI のインストールと初期化が完了していることを確認してください。
- プロフェッショナル版以上のライセンスが必要です。[NocoBase 商用版](https://www.nocobase.com/cn/commercial)をご覧ください。
- バックアップ管理とマイグレーション管理の 2 つのプラグインが有効化され、最新バージョンにアップグレードされていることを確認してください。

:::

:::warning 注意
リリース管理関連の CLI は現在も開発中であり、一時的に使用できません。
:::
## 概要

リリース管理 Skill は、複数環境間でリリース操作を実行するためのツールです。バックアップ復元とマイグレーションの 2 つのリリース方式に対応しています。


## 対応機能

- 単一環境のバックアップ復元：バックアップパッケージを使用してローカルデータをフルリストア
- クロス環境のバックアップ復元：バックアップパッケージを使用してターゲット環境のデータをフルリストア
- クロス環境マイグレーション：新規マイグレーションパッケージを使用してターゲット環境のデータを差分更新

## プロンプト例

### シナリオ A：単一環境のバックアップ復元
:::tip 前提条件

現在の環境にバックアップパッケージが存在するか、先にバックアップを作成してから復元する必要があります

:::

プロンプトモード
```
<file-name> を使ってバックアップ復元してください
```
コマンドラインモード
```
// 利用可能なバックアップパッケージを確認。なければ nb backup <file-name> を実行
nb backup list 
nb restore <file-name> 
```
![バックアップ復元](https://static-docs.nocobase.com/20260417150854.png)

### シナリオ B：クロス環境のバックアップ復元

:::tip 前提条件

2 つの環境を準備する必要があります。例えば、ローカルの dev 環境とリモートの test 環境、またはローカルに 2 つの環境をインストールします。

:::

プロンプトモード
```
dev を test に復元してください
```
コマンドラインモード
```
// 利用可能なバックアップパッケージを確認。なければ nb backup <file-name> --env dev を実行
nb backup list --env dev
// バックアップパッケージを使用して復元
nb restore <file-name> --env test
```
![バックアップ復元](https://static-docs.nocobase.com/20260417150854.png)

### シナリオ C：クロス環境マイグレーション

:::tip 前提条件

シナリオ B と同様に、2 つの環境を準備する必要があります。例えば、ローカルの dev 環境とリモートの test 環境、またはローカルに 2 つの環境をインストールします。

:::

プロンプトモード
```
dev を test にマイグレーションしてください
```
コマンドラインモード
```
// マイグレーションルールを新規作成。新しい ruleId が生成されます。または nb migration rule list --env dev で過去の ruleId を取得
nb migration rule add --env dev 
// ruleId を使用してマイグレーションパッケージを生成
nb migration generate <ruleId> --env dev 
// マイグレーションパッケージを使用してマイグレーションを実行
nb migration run <file-name> --env test
```
![マイグレーションリリース](https://static-docs.nocobase.com/20260417151022.png)

## よくある質問

**バックアップ復元とマイグレーションのどちらを選ぶべきですか？**

利用可能なバックアップパッケージがある場合は、バックアップ復元を選んでください。どのデータを同期するかをポリシーで制御したい場合（例：構造のみ同期してデータは同期しない）は、マイグレーションを選んでください。

**マイグレーションプラグインがない場合は？**

マイグレーション管理プラグインにはプロフェッショナル版以上が必要です。詳しくは [NocoBase 商用版](https://www.nocobase.com/cn/commercial)をご覧ください。
