# ブロック

ブロックはデータやコンテンツを格納するもので、ページ（Page）、モーダル（Modal）、ドロワー（Drawer）内に配置できます。複数のブロックは自由にドラッグ＆ドロップして配置を調整できます。

## ブロックの種類

![ブロックの種類](https://static-docs.nocobase.com/f71af45b5cd914ea0558f760ddbbba58.png)

- データブロック：データソースから取得したデータをインターフェース上に表示するために使用します。
- フィルターブロック：データソースのデータをフィルター条件として使用し、他のデータブロックをフィルターするために使用します。
- その他のブロック：ワークフローのToDo、監査ログ、Markdownなどの特定の、または独立したコンテンツを格納するために使用します。

## ブロックの追加

ブロックは、ページ（Page）、モーダル（Modal）、またはドロワー（Drawer）内に配置できます。

### ページ内のブロック

現在、ページ内に配置できるブロックの種類は、データブロック、フィルターブロック、その他のブロックです。

![20251023222441](https://static-docs.nocobase.com/20251023222441.png)

### ポップアップ（モーダルまたはドロワー）内のブロック

ポップアップにはモーダルとドロワーの2種類があり、ページと同様にブロックを追加できます。ただし、ポップアップ内のフォームブロックは通常、単一レコードの追加、編集、表示などに使用され、ブロックの種類はデータブロックとその他のブロックが含まれます。

![20251023222613](https://static-docs.nocobase.com/20251023222613.png)

## ブロックのデザイナー

各ブロックの右上には3つの小さなアイコンがあり、左から順に以下の機能を表しています。

1. ドラッグ＆ドロップによるレイアウト調整
2. [イベントフロー](/interface-builder/event-flow)
3. ブロック設定

![20251023224032](https://static-docs.nocobase.com/20251023224032.png)

シンプルなブロックの場合、すべての設定項目は「ブロック設定」に集約されています。例えば、JSブロックなどです。

![20251023224903](https://static-docs.nocobase.com/20251023224903.png)

複雑なデータタイプのブロックでは、独立した組み込みの「フィールド設定」と「アクション設定」も提供されます。

![20251023225141](https://static-docs.nocobase.com/20251023225141.png)

さらに、チャートブロックのように、より多くのネスト（入れ子）の可能性を提供し、自由にカスタマイズすることもできます。

![](https://static-docs.nocobase.com/07588190b3f41ae3060e71d8b76b4447.png)
