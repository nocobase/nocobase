# データスコープの設定

## はじめに

リレーションフィールドのデータスコープ設定は、ブロックのデータスコープ設定と同様に、リレーションデータにデフォルトのフィルター条件を設定します。

## 使用方法

![20251028211328](https://static-docs.nocobase.com/20251028211328.png)

### 静的値

例：削除されていない商品のみを関連付けとして選択できます。

> フィールドリストには、リレーションフィールドのターゲットコレクションのフィールドが含まれます。

![20251028211434](https://static-docs.nocobase.com/20251028211434.png)

### 変数値

例：サービス日が注文日より後の商品のみを関連付けとして選択できます。

![20251028211727](https://static-docs.nocobase.com/20251028211727.png)

変数に関する詳細は、[変数](/interface-builder/variables)を参照してください。

### リレーションフィールドの連携

リレーションフィールド間の連携は、データスコープを設定することで実現されます。

例：注文コレクションには、一対多リレーションフィールド「商談商品」と多対一リレーションフィールド「商談」があります。商談商品コレクションには、多対一リレーションフィールド「商談」があります。注文フォームブロックでは、「商談商品」の選択可能なデータは、現在のフォームで選択されている「商談」に関連付けられた商談商品にフィルターされます。
