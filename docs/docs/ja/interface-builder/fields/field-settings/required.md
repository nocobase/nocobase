# 必須

## はじめに

必須項目は、フォームの入力検証でよく使われるルールです。フィールド設定で直接有効にできるほか、フォームの連携ルールを使ってフィールドを動的に必須に設定することも可能です。

## フィールドを必須に設定する方法

### コレクションのフィールド設定

コレクションのフィールドで必須設定を行うと、バックエンドの検証がトリガーされ、フロントエンドでもデフォルトで必須項目として表示されます（変更不可）。
![20251025175418](https://static-docs.nocobase.com/20251025175418.png)

### フィールド設定

フィールドを直接必須項目として設定します。これは、ユーザー名やパスワードなど、常にユーザーに入力してもらう必要があるフィールドに適しています。

![20251028222818](https://static-docs.nocobase.com/20251028222818.png)

### 連携ルール

フォームブロックのフィールド連携ルールを使って、条件に基づいて必須設定を行います。

例：注文日が空でない場合、注文番号は必須です。
