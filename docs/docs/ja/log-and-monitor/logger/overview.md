# サーバーログ、監査ログ、履歴

## サーバーログ

### システム実行ログ

> [システムログ](./index.md#システムログ) を参照してください。

- アプリケーションシステムの実行情報を記録し、コードの実行チェーンを追跡し、実行エラーなどの例外情報を遡及します。
- ログレベルが存在し、機能モジュールごとに分類されています。
- ターミナル出力またはファイル形式で保存されます。
- 主にシステム稼働中に発生した異常事態の調査に使用されます。

### リクエストログ

> [リクエストログ](./index.md#リクエストログ) を参照してください。

- HTTP API のリクエストとレスポンス情報を記録します。リクエスト ID、API パス、リクエストヘッダー、レスポンスステータスコード、所要時間などの記録に重点を置いています。
- ターミナル出力またはファイル形式で保存されます。
- 主に API の呼び出しと実行状況の追跡に使用されます。

## 監査ログ

> [監査ログ](/security/audit-logger/index.md) を参照してください。

- ユーザー（API）によるシステムリソースへの操作動作を記録します。リソースタイプ、リソースオブジェクト、操作タイプ、ユーザー情報、操作ステータスなどの記録に重点を置いています。
- ユーザー操作の具体的な内容と結果をより良く追跡するために、リクエストパラメータとレスポンスがメタデータ情報として記録されます。この情報の一部はリクエストログと重複しますが、完全に一致するわけではありません。例えば、既存のリクエストログでは通常、完全なリクエストボディは記録されません。
- リクエストパラメータとレスポンスはリソースのスナップショットと同等ではありません。パラメータとコードロジックを通じてどのような変更が行われたかを知ることはできますが、変更前のデータレコードの内容を正確に把握することはできず、バージョン管理や誤操作後のデータ復旧を実現するものではありません。
- ファイルおよびデータテーブル形式で保存されます。

## 履歴

> [履歴](/record-history/index.md) を参照してください。

- データ内容の変更履歴を記録します。
- 主な記録内容は、リソースタイプ、リソースオブジェクト、操作タイプ、変更されたフィールド、変更前後の値です。
- データの比較に使用できます。
- データテーブル形式で保存されます。
