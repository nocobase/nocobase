# サーバーログ、監査ログ、変更履歴

## サーバーログ

### システムログ

> 「[システムログ](./logger/index.md#システムログ)」をご参照ください。

- アプリケーションシステムの稼働状況を記録し、コードロジックの実行経路を追跡し、コード実行時のエラーやその他の異常情報を特定します。
- ログは重要度レベルや機能モジュールごとに分類されます。
- ターミナルに出力されるか、ファイルとして保存されます。
- 主に、システム稼働中に発生する異常の診断やトラブルシューティングに利用されます。

### リクエストログ

> 「[リクエストログ](./logger/index.md#リクエストログ)」をご参照ください。

- HTTP API のリクエストとレスポンスの詳細を記録し、リクエストID、APIパス、ヘッダー、レスポンスステータスコード、処理時間などの情報に焦点を当てています。
- ターミナルに出力されるか、ファイルとして保存されます。
- 主に、API の呼び出しと実行状況を追跡するために利用されます。

## 監査ログ

> 「[監査ログ](../security/audit-logger/index.md)」をご参照ください。

- ユーザー (またはAPI) によるシステムリソースへの操作を記録し、リソースタイプ、対象オブジェクト、操作タイプ、ユーザー情報、操作ステータスなどの情報に焦点を当てています。
- ユーザー操作の具体的な内容と結果をより詳細に追跡するため、リクエストパラメータとレスポンスはメタデータとして記録されます。この情報はリクエストログと部分的に重複しますが、完全に同一ではありません。例えば、通常のリクエストログには完全なリクエストボディは含まれません。
- リクエストパラメータとレスポンスは、リソースの**スナップショットとは異なります**。パラメータとコードロジックから、どのような操作が行われたかは分かりますが、変更前のデータテーブルの正確な内容は分かりません。そのため、バージョン管理や誤操作後のデータ復元には利用できません。
- ファイルとデータベーステーブルの両方で保存されます。

## 変更履歴

> 「[変更履歴](/record-history/index.md)」をご参照ください。

- データコンテンツの**変更履歴**を記録します。
- 主に、リソースタイプ、リソースオブジェクト、操作タイプ、変更されたフィールド、変更前後の値を追跡します。
- **データ比較や監査**に役立ちます。
- データベーステーブルに保存されます。
