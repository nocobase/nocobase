# 権限の設定

## 一般的な権限設定

![](https://static-docs.nocobase.com/119a9650259f9be71210121d0d3a435d.png)

### 設定権限

1.  **インターフェースの設定を許可**: この権限は、ユーザーがインターフェースを設定できるかどうかを制御します。有効にすると、UI設定ボタンが表示されるようになります。「admin」ロールでは、この権限はデフォルトで有効です。
2.  **プラグインのインストール、有効化、無効化を許可**: この権限は、ユーザーが**プラグイン**を有効または無効にできるかどうかを制御します。有効にすると、ユーザーは**プラグイン**マネージャーインターフェースにアクセスできるようになります。「admin」ロールでは、この権限はデフォルトで有効です。
3.  **プラグインの設定を許可**: この権限は、ユーザーが**プラグイン**のパラメーターを設定したり、**プラグイン**のバックエンドデータを管理したりできるかどうかを制御します。「admin」ロールでは、この権限はデフォルトで有効です。
4.  **キャッシュのクリア、アプリケーションの再起動を許可**: この権限は、キャッシュのクリアやアプリケーションの再起動といったシステムメンテナンス作業に関連しています。有効にすると、関連する操作ボタンが個人センターに表示されますが、この権限はデフォルトでは無効です。
5.  **新規メニュー項目のデフォルトアクセスを許可**: 新しく作成されたメニューはデフォルトでアクセス可能であり、この設定はデフォルトで有効になっています。

### グローバル操作権限

グローバル操作権限は、すべての**コレクション**に普遍的に適用され、操作タイプによって分類されます。これらの権限は、データ範囲の次元に基づいて設定できます。具体的には、「すべてのデータ」と「自分のデータ」です。前者は**コレクション**全体に対して操作を実行することを許可し、後者はユーザー自身に関連するデータのみに操作を制限します。

## コレクション操作権限

![](https://static-docs.nocobase.com/6a6e0281391cecdea5b5218e6173c5d7.png)

![](https://static-docs.nocobase.com/9814140434ff9e1bf028a6c282a5a165.png)

**コレクション**操作権限は、グローバル操作権限をさらに細かく調整するもので、各**コレクション**内のリソースアクセスに対して個別の権限設定が可能です。これらの権限は、次の2つの側面から構成されます。

1.  **操作権限**: 操作権限には、追加、表示、編集、削除、エクスポート、インポートの操作が含まれます。これらの権限は、データ範囲の次元に基づいて設定されます。
    -   **すべてのデータ**: ユーザーが**コレクション**内のすべてのレコードに対して操作を実行する権限を付与します。
    -   **自分のデータ**: ユーザーが自身で作成したデータレコードに対してのみ操作を実行するように制限します。

2.  **フィールド権限**: フィールド権限を使用すると、異なる操作中に各フィールドに対して特定の権限を設定できます。例えば、特定のフィールドは編集権限なしで表示のみを許可するように設定できます。

## メニューアクセス権限

メニューアクセス権限は、メニュー項目を次元としてアクセスを制御します。

![](https://static-docs.nocobase.com/28eddfc843d27641162d9129e3b6e33f.png)

## プラグイン設定権限

**プラグイン**設定権限は、特定の**プラグイン**パラメーターの設定権限を制御します。この権限が有効になると、管理センターに対応する**プラグイン**管理インターフェースが表示されます。
