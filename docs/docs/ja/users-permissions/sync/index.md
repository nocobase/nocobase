# ユーザーデータ同期

## はじめに

ユーザーデータ同期元を登録・管理する機能です。デフォルトではHTTP APIが提供されていますが、プラグインを通じて他のデータソースを拡張することも可能です。デフォルトでは、**ユーザー** コレクションと**部門** コレクションへのデータ同期をサポートしており、プラグインを使って他の同期ターゲットリソースに拡張することもできます。

## データソースの管理とデータ同期

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
ユーザーデータ同期元を提供するプラグインがインストールされていない場合、HTTP API を使用してユーザーデータを同期できます。詳細は [データソース - HTTP API](./sources/api.md) を参照してください。
:::

## データソースの追加

ユーザーデータ同期元を提供するプラグインをインストールすると、該当するデータソースを追加できるようになります。有効化されたデータソースのみ、「同期」ボタンと「タスク」ボタンが表示されます。

> 例：WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## データの同期

「同期」ボタンをクリックすると、データ同期が開始されます。

![](https://static-docs.nocobase.com/202412041055022.png)

「タスク」ボタンをクリックすると、同期ステータスを確認できます。同期が成功した後、ユーザーリストと部門リストでデータを確認できます。

![](https://static-docs.nocobase.com/202412041202337.png)

同期に失敗したタスクは、「再試行」をクリックできます。

![](https://static-docs.nocobase.com/202412041058337.png)

同期が失敗した場合は、システムログで原因を特定できます。また、アプリケーションのログディレクトリにある `user-data-sync` ディレクトリには、元のデータ同期記録が保存されています。

![](https://static-docs.nocobase.com/202412041205655.png)
