# データの削除

特定の条件を満たすコレクションのデータを削除するために使用します。

削除ノードの基本的な使い方は更新ノードと似ていますが、削除ノードではフィールドに値を割り当てる必要がなく、コレクションとフィルター条件を選択するだけで済みます。削除ノードの結果は、削除に成功したデータの行数を返しますが、これは実行履歴でのみ確認でき、後続のノードで変数として使用することはできません。

:::info{title=注意}
現在、削除ノードは行ごとの削除をサポートしておらず、すべて一括削除となります。そのため、個々のデータ削除に対する他のイベントはトリガーされません。
:::

## ノードの作成

ワークフロー設定画面で、フロー内のプラス（「+」）ボタンをクリックして、「データの削除」ノードを追加します。

![削除データノードの作成](https://static-docs.nocobase.com/e1d6b8728251fcdbed6c7f50e5570da2.png)

## ノード設定

![削除ノード_ノード設定](https://static-docs.nocobase.com/580600c2b13ef4e01dfa48b23539648e.png)

### コレクション

データを削除するコレクションを選択します。

### フィルター条件

通常のコレクションクエリにおけるフィルター条件と同様に、ワークフローのコンテキスト変数を活用できます。

## 例

例えば、キャンセルされた無効な過去の注文データを定期的にクリーンアップする場合、削除ノードを使用して実現できます。

![削除ノード_例_ノード設定](https://static-docs.nocobase.com/b94b7507417252f8523c3f13ce5f320.png)

ワークフローは定期的にトリガーされ、キャンセルされたすべての無効な過去の注文データを削除します。
