# ディレイ

## はじめに

ディレイノードを使うと、ワークフローに遅延を追加できます。遅延が終了した後、設定に応じて、遅延終了後のノードの実行を続行するか、ワークフローを途中で終了するかを選択できます。

通常、並列分岐ノードと組み合わせて使用します。いずれかの分岐にディレイノードを追加することで、タイムアウト後の処理を行うことができます。例えば、並列分岐の一方に手動処理、もう一方にディレイノードを設定するケースです。手動処理がタイムアウトした場合、タイムアウト時に「失敗」と設定すると、手動処理は制限時間内に完了する必要があります。一方、タイムアウト時に「続行」と設定すると、時間経過後にその手動処理を無視できます。

## インストール

組み込みのプラグインなので、インストールは不要です。

## ノードの作成

ワークフロー設定画面で、ワークフロー内のプラス（「+」）ボタンをクリックして「ディレイ」ノードを追加します。

## ノード設定

![ディレイノード_ノード設定](https://static-docs.nocobase.com/5fe8a36535f20a087a0148ffa1cd2aea.png)

### 遅延時間

遅延時間には数値を入力し、時間単位を選択できます。サポートされている時間単位は、秒、分、時間、日、週です。

### タイムアウト時のステータス

タイムアウト時のステータスとして、「通過して続行」または「失敗して終了」を選択できます。「通過して続行」は、遅延終了後、ワークフローが後続のノードの実行を続けることを意味します。「失敗して終了」は、遅延終了後、ワークフローが失敗ステータスで途中で終了することを意味します。

## 例

ワークオーダーが発行された後、制限時間内に返信が必要なシナリオを例に挙げます。並列する2つの分岐のうち、一方に手動ノードを、もう一方にディレイノードを追加します。手動処理が10分以内に返信されなかった場合、ワークオーダーのステータスを「タイムアウト未処理」に更新します。
