# ワークフローの終了

このノードが実行されると、現在実行中のワークフローは直ちに終了し、ノードで設定されたステータスで完了します。これは通常、特定のロジックに基づくフロー制御に使用されます。特定の条件が満たされた場合、現在のワークフローから抜け出し、後続の処理は実行されません。プログラミング言語の `return` 命令に似ており、現在実行中の関数を終了するために使用されます。

## ノードの追加

ワークフロー設定画面で、フロー内のプラス（「+」）ボタンをクリックして、「ワークフローを終了する」ノードを追加します。

![ワークフローを終了_追加](https://static-docs.nocobase.com/672186ab4c8f7313dd3cf9c880b524b8.png)

## ノードの設定

![ワークフローを終了_ノード設定](https://static-docs.nocobase.com/bb6a597f25e9afb72836a14a0fe0683e.png)

### 終了ステータス

終了ステータスは、このワークフロー実行プランの最終ステータスに影響を与えます。「成功」または「失敗」に設定できます。フローがこのノードに到達すると、設定されたステータスで直ちに終了します。

:::info{title=補足}
「操作前イベント」タイプのワークフローで使用する場合、操作を開始したリクエストをインターセプトします。詳細については、[「操作前イベント」の使用方法](../triggers/pre-action)をご参照ください。

また、操作を開始したリクエストをインターセプトするだけでなく、終了ステータスの設定は、このタイプのワークフローにおける「応答メッセージ」で返される情報のステータスにも影響を与えます。
:::
