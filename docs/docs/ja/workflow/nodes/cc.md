# CC <Badge>v1.8.2+</Badge>

## はじめに

CCノードは、ワークフローの実行中に生成される特定のコンテキスト情報を、情報共有や確認のために指定したユーザーに送信する際に使用します。たとえば、承認プロセスやその他のワークフローにおいて、関連情報を他の参加者にCCとして送ることで、彼らが作業の進捗状況をタイムリーに把握できるようにします。

ワークフローには複数のCCノードを設定できます。ワークフローがそのノードに到達すると、関連情報が指定された受信者に送信されます。

CCされた内容は、「タスクセンター」の「自分宛のCC」メニューに表示され、ユーザーはここで自分宛にCCされたすべての内容を確認できます。未読の状態に基づいて、まだ確認していないCC内容がユーザーに通知されます。ユーザーは内容を確認した後、手動で既読にマークできます。

## ノードの作成

ワークフローの設定画面で、フロー内のプラス（「+」）ボタンをクリックして、「CC」ノードを追加します。

![抄送_添加](https://static-docs.nocobase.com/20250710222842.png)

## ノードの設定

![节点配置](https://static-docs.nocobase.com/20250710224041.png)

ノードの設定画面では、以下のパラメーターを設定できます。

### 受信者

受信者は、CCの対象となるユーザーの集合です。1人または複数のユーザーを指定できます。選択肢のソースとしては、ユーザーリストから選択する静的な値、変数で指定する動的な値、またはユーザーコレクションに対するクエリ結果を指定できます。

![接收者配置](https://static-docs.nocobase.com/20250710224421.png)

### ユーザーインターフェース

受信者は、「タスクセンター」の「自分宛のCC」メニューでCC内容を確認する必要があります。ワークフローのコンテキストにあるトリガーや任意のノードの結果をコンテンツブロックとして設定できます。

![用户界面](https://static-docs.nocobase.com/20250710225400.png)

### タスクタイトル

タスクタイトルは、「タスクセンター」に表示されるタイトルです。ワークフローのコンテキストにある変数を使用して、タイトルを動的に生成できます。

![任务标题](https://static-docs.nocobase.com/20250710225603.png)

## タスクセンター

ユーザーはタスクセンターで、自分宛にCCされたすべての内容を確認・管理できます。また、既読ステータスに基づいてフィルタリングして表示することも可能です。

![20250710232932](https://static-docs.nocobase.com/20250710232932.png)

![20250710233032](https://static-docs.nocobase.com/20250710233032.png)

内容を確認した後、既読としてマークできます。これにより、未読の件数が減少します。

![20250710233102](https://static-docs.nocobase.com/20250710233102.png)
