# 概要

ワークフローの組み込み機能だけでは、すべてのシナリオをカバーしきれません。たとえば、組み込みのノードタイプでは、あらゆるビジネスシナリオにおけるすべての操作を網羅することはできません。そこで、ワークフローを拡張するための設計も提供しており、トリガーやノードタイプの拡張などが可能です。組み込み機能では対応できないビジネスシナリオでは、ローコードの手法で拡張して解決できます。

拡張は主に以下の2つの部分に分けられます。

- [トリガータイプの拡張](./trigger.md)
- [ノードタイプの拡張](./node.md)
