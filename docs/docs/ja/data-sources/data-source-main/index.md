# メインデータベース

## 概要

[デプロイした NocoBase](/ai/install-nocobase-app) で設定されたデータベースで、NocoBase のシステムテーブルデータを保存します。また、ユーザーの業務テーブルデータの保存にも対応しています。

メインデータベースがサポートするデータベースのバージョンおよび商用エディションは以下のとおりです。

| データベース | 対応バージョン | コミュニティ版 | スタンダード版 | プロフェッショナル版 | エンタープライズ版 |
| --- | --- | --- | --- | --- | --- |
| MySQL | >= 8.0.17 | ✅ | ✅ | ✅ | ✅ |
| PostgreSQL | >= 10 | ✅ | ✅ | ✅ | ✅ |
| MariaDB | >= 10.9 | ✅ | ✅ | ✅ | ✅ |
| KingbaseES | >=V9 | ❌ | ❌ | ✅ | ✅ |
| OceanBase | >=4.3 | ❌ | ❌ | ❌ | ✅ |

:::tip ヒント

KingbaseES は PostgreSQL 互換モードのみ、OceanBase は MySQL 互換モードのみサポートしています。

:::

## プラグインのインストール

| データベース | 対応プラグイン | インストール方法 |
| --- | --- | --- |
| MySQL | なし | 組み込みプラグインのため、個別にインストールする必要はありません。 |
| PostgreSQL | なし | 組み込みプラグインのため、個別にインストールする必要はありません。 |
| MariaDB | なし | 組み込みプラグインのため、個別にインストールする必要はありません。 |
| KingbaseES | `@nocobase/plugin-data-source-kingbase` | 商用ライセンスが必要です。インストール後、プラグインはデフォルトで有効になります。 |
| OceanBase | `@nocobase/plugin-data-source-oceanbase` | 商用ライセンスが必要です。インストール後、プラグインはデフォルトで有効になります。 |

## メインデータソースへのアクセス

1. システム機能のデータソースメニューをクリックして、データソースのホームページにアクセスします。
2. データソース一覧から **Main** データソースを選択し、**設定**操作をクリックしてメインデータベースにアクセスし、管理します。

![configure_main_datasource](https://static-docs.nocobase.com/configure_main_datasource.png)

## メインデータソースの管理

メインデータベースでは、データテーブルの管理機能を提供しています。データテーブルの検索、作成、変更、削除、およびデータベースに既に存在するデータテーブルのフィールドの同期が可能です。また、データテーブルのフィールドの作成、変更、削除にも対応しています。
- **フィルター**：NocoBase メインデータベースで管理しているデータテーブルを検索します
- **データテーブルを作成**：業務データテーブルを新規作成します
- **編集**：業務データテーブルを変更します
- **削除**：業務データテーブルを削除します
- **データベースから同期**：データベースに既に存在するデータテーブルの構造を同期します
- **フィールドを設定**：データテーブルのフィールドを作成、変更、削除します
-  **+** ：タブページの **+** からデータテーブルを分類管理し、分類の作成、変更、削除を行います
![main_datasource_management](https://static-docs.nocobase.com/main_datasource_management.png)

### データベースに既に存在するテーブルを同期する

![nocobase_doc-2025-10-29-19-46-34](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-46-34.png)

メインデータソースの重要な機能の 1 つは、データベースに既に存在するテーブルを NocoBase に同期して管理できることです。これにより、次のことが可能になります。

- **既存の投資を保護**：データベースに大量の業務テーブルが既に存在する場合、再作成する必要はなく、そのまま同期して利用できます
- **柔軟な統合**：他のツール（SQL スクリプト、データベース管理ツールなど）で作成したテーブルを NocoBase の管理対象に追加できます
- **段階的な移行**：既存のシステムを一度に再構築するのではなく、段階的に NocoBase へ移行できます

「データベースから読み込む」機能を使用すると、次の操作ができます。
1. データベース内のすべてのテーブルを確認する
2. 同期するテーブルを選択する
3. テーブル構造とフィールドタイプを自動的に識別する
4. ワンクリックで NocoBase にインポートして管理する

### 複数のテーブル構造タイプをサポート

![nocobase_doc-2025-10-29-19-47-14](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-47-14.png)

NocoBase では、複数のタイプのデータテーブルを作成および管理できます。
- **通常テーブル**：一般的に使用されるシステムフィールドが組み込まれています；
- **継承テーブル**：親テーブルを作成し、その親テーブルから子テーブルを派生させることができます。子テーブルは親テーブルの構造を継承しながら、独自の列も定義できます。
- **ツリーテーブル**：ツリー構造のテーブルです。現在は隣接リストモデルのみをサポートしています；
- **カレンダーテーブル**：カレンダー関連のイベントテーブルを作成するために使用します；
- **ファイルテーブル**：ファイルストレージを管理するために使用します；
- **SQL テーブル**：実際のデータベーステーブルではなく、SQL クエリをすばやく構造化して表示します；
- **ビュー テーブル**：既存のデータベースビューに接続します；

### 豊富なフィールドタイプを提供

![nocobase_doc-2025-10-29-19-48-51](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-48-51.png)

#### 柔軟なフィールドタイプ変換

NocoBase では、同じデータベースタイプを基盤として、フィールドタイプを柔軟に変換できます。

**例：String タイプフィールドの変換オプション**

データベース内のフィールドが String タイプの場合、NocoBase では以下のいずれかの形式に変換できます。

- **基本タイプ**：単一行テキスト、複数行テキスト、電話番号、メールアドレス、URL、パスワード、カラー、アイコン
- **選択タイプ**：ドロップダウン（単一選択）、ラジオボタン
- **リッチメディアタイプ**：Markdown、Markdown (Vditor)、リッチテキスト、添付ファイル（URL）
- **日時タイプ**：日時（タイムゾーンあり）、日時（タイムゾーンなし）
- **高度なタイプ**：自動採番、データテーブルセレクター、暗号化

この柔軟な変換メカニズムにより、次のことが可能になります。
- **データベース構造を変更する必要がない**：フィールドの基盤となる保存タイプは変わらず、NocoBase 上での表示形式のみが変わります
- **業務の変化に対応**：業務要件の変化に応じて、フィールドの表示方法や操作方法を迅速に調整できます
- **データの安全性**：変換プロセスが既存データの完全性に影響を与えることはありません

### フィールドレベルでの柔軟な同期

NocoBase はテーブル全体の同期だけでなく、フィールドレベルでのきめ細かな同期管理にも対応しています。

![nocobase_doc-2025-10-29-19-49-56](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-49-56.png)

#### フィールド同期の特徴：

1. **リアルタイム同期**：データベースのテーブル構造に変更があった場合、追加されたフィールドをいつでも同期できます
2. **選択的同期**：すべてのフィールドではなく、必要なフィールドだけを選択して同期できます
3. **タイプの自動識別**：データベースのフィールドタイプを自動的に識別し、NocoBase のフィールドタイプにマッピングします
4. **データの完全性を維持**：同期プロセスが既存データに影響を与えることはありません

#### 使用シーン：

- **データベース構造の発展**：業務要件の変化に伴ってデータベースに新しいフィールドを追加した場合、すぐに NocoBase へ同期できます
- **チームコラボレーション**：他のチームメンバーや DBA がデータベースにフィールドを追加した場合、速やかに同期できます
- **ハイブリッド管理モード**：一部のフィールドを NocoBase で管理し、別のフィールドを従来の方法で管理するなど、柔軟に組み合わせられます

この柔軟な同期メカニズムにより、NocoBase は既存の技術アーキテクチャにスムーズに統合できます。従来のデータベース管理方法を変更する必要がない一方で、NocoBase がもたらすローコード開発の利便性を享受できます。

詳しくは「[データテーブルフィールド / 概要](../data-modeling/collection-fields/index.md)」の章をご覧ください。
