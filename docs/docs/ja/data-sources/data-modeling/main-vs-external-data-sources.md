# 主データベースと外部データベースの比較

NocoBase における主データベースと外部データベースの違いは、主に次の 4 つの点にあります：データベースタイプのサポート、データテーブルタイプのサポート、フィールドタイプのサポート、バックアップ・復元・移行。

## 一、データベースタイプのサポート

詳しくは、[データソース管理](https://docs.nocobase.com/data-sources/data-source-manager)を参照してください。

### データベースタイプ

| データベースタイプ | 主データベースのサポート | 外部データベースのサポート |
|-----------|-------------|--------------|
| PostgreSQL | ✅ | ✅ |
| MySQL | ✅ | ✅ |
| MariaDB | ✅ | ✅ |
| KingbaseES | ✅ | ✅ |
| MSSQL | ❌ | ✅ |
| Oracle | ❌ | ✅ |

### データテーブル管理

| データテーブル管理 | 主データベースのサポート | 外部データベースのサポート |
|-----------|-------------|--------------|
| 基本管理 | ✅ | ✅ |
| ビジュアル管理 | ✅ | ❌ |

## 二、データテーブルタイプのサポート

詳しくは、[データテーブル](https://docs.nocobase.com/data-sources/data-modeling/collection)を参照してください。

| データテーブルタイプ | 主データベース | 外部データベース | 説明 |
|-----------|---------|-----------|------|
| 通常テーブル | ✅ | ✅ | 基本的なデータテーブル |
| ビューテーブル | ✅ | ✅ | データソースのビュー |
| 継承テーブル | ✅ | ❌ | データモデルの継承をサポート。主データソースのみ対応 |
| ファイルテーブル | ✅ | ❌ | ファイルのアップロードをサポート。主データソースのみ対応 |
| コメントテーブル | ✅ | ❌ | 組み込みコメントシステム。主データソースのみ対応 |
| カレンダーテーブル | ✅ | ❌ | カレンダービューで使用するデータテーブル |
| 式テーブル | ✅ | ❌ | 数式による計算をサポート |
| ツリーテーブル | ✅ | ❌ | ツリー構造データのモデリングに使用 |
| SQL テーブル | ✅ | ❌ | SQL で定義できるデータテーブル |
| 外部データテーブル接続 | ✅ | ❌ | 外部データソースへの接続テーブル。機能に制限あり |

## 四、バックアップと移行のサポート比較

| 機能 | 主データベース | 外部データベース |
|-----|---------|-----------|
| バックアップと復元 | ✅ | ❌（ユーザー自身で対応） |
| 移行管理 | ✅ | ❌（ユーザー自身で対応） |

:::info
NocoBase は、主データベースのバックアップ、復元、および構造移行機能を提供しています。外部データベースについては、これらの操作をユーザーが自身のデータベース環境に応じて独自に実施する必要があり、NocoBase は組み込みのサポートを提供していません。
:::

## 比較まとめ

| 比較項目 | 主データベース | 外部データベース |
|-------|---------|-----------|
| データベースタイプ | PostgreSQL、MySQL、MariaDB、KingbaseES | PostgreSQL、MySQL、MariaDB、MSSQL、Oracle、KingbaseES |
| テーブルタイプのサポート | すべてのテーブルタイプ | 通常テーブルとビューテーブルのみサポート |
| フィールドタイプのサポート | すべてのフィールドタイプ | 添付ファイルフィールド以外のすべてのフィールドタイプ |
| バックアップと移行 | 組み込みサポート | ユーザー自身で対応 |

## 推奨事項

- **NocoBase を使って新しい業務システムを構築する場合**は、**主データベース**を使用してください。NocoBase のすべての機能を利用できます。
- **NocoBase を使って他のシステムのデータベースに接続し、基本的なデータの追加・削除・更新・参照を行う場合**は、**外部データベース**を使用してください。
