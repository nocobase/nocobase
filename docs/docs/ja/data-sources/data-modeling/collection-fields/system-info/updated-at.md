# 更新日時

## 概要

NocoBase では、**更新日時（Updated at）** を使用して、レコードが最後に更新された日時を自動的に記録します。

更新日時は通常、プリセットフィールドによって生成されます。監査、同期の判定、並べ替え、絞り込み、ワークフローの条件などに適しています。

業務上の完了日時や承認日時などを保存する必要がある場合は、[日時](../datetime/datetime.md)を使用してください。

## 適用シーン

更新日時は、次のような業務シーンに適しています。

- 最終更新日時を確認する
- 最近更新されたデータを絞り込む
- データが長期間メンテナンスされていないかを判断する
- 外部システムとの同期時に更新日時を比較する

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「更新日時」を選択すると、更新日時フィールドを作成できます。

![20240512174826](https://static-docs.nocobase.com/20240512174826.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。更新日時に対応する `updatedAt` を指定し、ページ上での入力方法や表示方法を決定します。 |
| Field display name | インターフェース上で表示されるフィールド名。たとえば「更新日時」「最終更新日時」などです。業務担当者がすぐに理解できる名称を使用することを推奨します。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字でなければなりません。 |
| Field type | データレイヤーにおけるフィールドのタイプ。更新日時には通常 `date` を使用します。 |
| Default value | デフォルト値。レコードの新規作成時にユーザーが入力しなかった場合、自動的にデフォルト値を設定できます。 |
| Validation rules | システムによって自動的に管理され、通常は手動で検証する必要はありません。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整の負担を避けてください。

:::

## フィールドの特性

更新日時フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `updatedAt`。 |
| デフォルトの Field type | `date`。 |
| 選択可能な Field type | `date`。 |
| ページコンポーネント | システムが自動的に書き込み、ページ上では通常読み取り専用で表示されます。 |
| 絞り込み | 時刻や期間による絞り込みに対応しています。 |
| 並べ替え | 時刻による並べ替えに対応しています。 |
| 検証 | システムが自動的に書き込みます。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、更新日時フィールドの設定を編集できます。フィールドの編集は、主に NocoBase 上でのフィールドの表示方法や使用方法を調整するために行います。たとえば、表示名、説明、デフォルト値、検証ルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | インターフェース上のフィールド表示名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は、作成後通常、編集フォームから変更できません。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上での入力、表示、検証の方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | レコードの新規作成時に使用するデフォルト値を調整します。 |
| Validation rules | はい | フィールドの検証ルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、表示名を単に変更することとは異なります。フィールドの保存方法、入力コンポーネント、検証ルール、絞り込み条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、事前にデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、更新日時フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成した更新日時フィールドを削除すると、通常、データベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースから同期したフィールドや外部データソースからマッピングしたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの出所によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、絞り込み、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での使用

更新日時フィールドは、一覧、詳細、絞り込み、同期の判定などで使用するのに適しています。
![20260710153223](https://static-docs.nocobase.com/20260710153223.png)

| シーン | 用途 |
| --- | --- |
| テーブルブロック | 最終更新日時の表示や並べ替えに使用します。 |
| フィルターブロック | 最近更新されたレコードや、長期間更新されていないレコードを絞り込みます。 |
| 詳細ブロック | 最終更新日時を確認します。 |
| ワークフロー | 時間条件として判定に使用します。 |
