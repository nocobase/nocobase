# 作成者

## 概要

NocoBase では、**作成者（Created by）** を使用して、レコードの作成者を自動的に記録できます。

作成者は通常、あらかじめ用意されたフィールドによって生成されます。権限制御、責任の追跡、絞り込み、監査などに適しています。

業務上の責任者、担当者、承認者を表す場合は、作成者フィールドと混用せず、ユーザー関係フィールドを別途作成することをおすすめします。

## 適用シーン

作成者は、次のような業務シーンに適しています。

- 自分が作成したデータのみを表示する
- 作成者でレコードを絞り込む
- レコード作成の責任を監査する
- ワークフローでレコードの作成者を判定する

## 作成設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「作成者」を選択すると、作成者フィールドを作成できます。

![index-2025-11-01-00-50-59](https://static-docs.nocobase.com/index-2025-11-01-00-50-59.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。作成者には `createdBy` が対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | インターフェース上に表示されるフィールド名。たとえば「作成者」などです。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、関係フィールド、権限、ワークフローなどで内部的に参照するフィールド識別名。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字にする必要があります。 |
| Field type | データ層におけるフィールドのタイプ。作成者は通常、ユーザーテーブルを参照する `belongsTo` 関係フィールドです。 |
| Default value | デフォルト値。レコードの新規作成時にユーザーが入力しなかった場合、デフォルト値を自動的に設定できます。 |
| Validation rules | システムによって自動的に管理されるため、通常は手動で検証する必要はありません。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、API から参照されます。後から変更すると設定の調整コストが発生するため、作成前に命名を確認してください。

:::

## フィールドの特性

作成者フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `createdBy`。 |
| デフォルトの Field type | `belongsTo`。 |
| 選択可能な Field type | `belongsTo`。 |
| ページコンポーネント | システムによって自動的に書き込まれます。ページ上では通常、ユーザー選択またはユーザー表示コンポーネントとして表示されます。 |
| 絞り込み | ユーザーで絞り込むことができます。 |
| 並べ替え | 通常、作成者では並べ替えません。 |
| 検証 | システムによって自動的に書き込まれます。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、作成者フィールドの設定を編集できます。フィールドの編集は、主に NocoBase でのフィールドの表示方法や使用方法を調整するために行います。たとえば、表示名、説明、デフォルト値、検証ルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。これは、データベースのフィールドを NocoBase の Field type と Field interface にマッピングする作業です。

| 設定 | 編集可能 | 説明 |
| --- | --- | --- |
| Field display name | はい | インターフェース上に表示されるフィールド名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は、作成後に編集フォームから変更できないのが通常です。 |
| Field interface | 条件付きで可能 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整すると、ページ上での入力、表示、検証方法に影響します。 |
| Field type | 条件付きで可能 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整前に、既存のデータを新しいタイプで使用できるか確認する必要があります。 |
| Default value | はい | レコードの新規作成時に使用するデフォルト値を調整します。 |
| Validation rules | はい | フィールドの検証ルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、検証ルール、絞り込み条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、作成者フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成した作成者フィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されているデータも同時に削除されます。データベースとの同期や外部データソースからマッピングされたフィールドを削除する場合の影響範囲は、対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、絞り込み、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除する前に、そのフィールドが業務上の設定で引き続き参照されていないか確認してください。

:::

## ページ設定での使用

作成者フィールドは、権限、絞り込み、監査、ワークフローで使用するのに適しています。
![20260710153223](https://static-docs.nocobase.com/20260710153223.png)

| シーン | 用途 |
| --- | --- |
| テーブルブロック | 作成者を表示します。 |
| 絞り込みブロック | 作成者でレコードを絞り込みます。 |
| 権限 | 「自分が作成したデータのみを表示する」などのルールを設定します。 |
| ワークフロー | 作成者を取得して通知を送信したり、条件を設定したりします。 |
