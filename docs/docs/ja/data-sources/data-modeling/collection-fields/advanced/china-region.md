# 中国行政区

<PluginInfo name="field-china-region"></PluginInfo>

## 概要

NocoBase では、**中国行政区（China region）** は、省、市、区県などの中国の行政区画情報を保存するために使用します。

中国行政区フィールドは、組み込みの `chinaRegions` 行政区画データテーブルを基にしており、ページではカスケードセレクターを使用して入力します。ユーザーは階層に沿って省、市、区を順番に選択でき、表示時には階層ごとに連結されて完全なパスとして表示されます。

番地や建物名などの詳細な住所を保存する場合は、[単一行テキスト](../basic/input.md)または[複数行テキスト](../basic/textarea.md)フィールドと組み合わせて使用できます。

## 適用シーン

中国行政区は、次のような業務シーンに適しています。

- 顧客、連絡先、店舗、プロジェクトの所在地
- 戸籍地、出身地、配送先地域などの住所に関する基本情報
- サービス地域、販売地域、プロジェクト実施地域
- 省・市・区での絞り込みや集計が必要なデータ

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「中国行政区」を選択すると、中国行政区フィールドを作成できます。

![](https://static-docs.nocobase.com/data-source-manager-main-NocoBase-04-29-2026_04_52_PM.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。中国行政区では `chinaRegion` に対応し、ページ上での入力・表示方法を決定します。 |
| Field display name | 画面上に表示されるフィールド名。例：「所在地」「サービス地域」「配送先地域」など。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名。通常、作成後の変更は行わず、使用できるのは英字、数字、アンダースコアのみで、先頭は英字である必要があります。 |
| Field type | データ層におけるフィールドのタイプ。中国行政区は通常、関連レコードまたは構造化された値として保存されます。具体的な内容はフィールド設定に準じます。 |
| 選択階層 | 選択できる最下層を指定します。現在は「省」「市」「区」に対応しており、デフォルトは「区」です。 |
| 最下層まで選択必須 | 有効にすると、ユーザーは設定された最下層まで選択しないと送信できません。無効の場合は、途中の階層で選択を完了できます。 |
| Validation rules | バリデーションルール。通常は必須入力や選択階層を設定します。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整のコストを避けてください。

:::

## フィールドの特性

中国行政区フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `chinaRegion`。 |
| データソース | 組み込みの `chinaRegions` 行政区画データテーブル。 |
| ページコンポーネント | 編集モードではカスケードセレクターを使用します。 |
| 選択階層 | 現在は省、市、区の3階層に対応しています。 |
| 表示方法 | 閲覧モードでは、階層に応じて `省 / 市 / 区` として表示されます。 |
| 絞り込み | 保存済みの地域値による絞り込みに対応しています。具体的な機能は、フィールド設定とページブロックによって異なります。 |
| 複数選択 | 複数選択には対応していません。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、中国行政区フィールドの設定を編集できます。フィールドの編集は、NocoBase での表示方法や使用方法を調整するために行います。たとえば、表示名、説明、バリデーションルール、選択階層、または最下層まで選択必須にするかどうかを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上のフィールド表示名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は通常、編集フォームから変更できません。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整後は、ページ上での入力、表示、バリデーション方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| 選択階層 | はい | フィールドで選択できる範囲を、省、市、区のいずれかに調整します。 |
| 最下層まで選択必須 | はい | 送信時に設定された最下層まで選択必須にするかどうかを指定します。 |
| Validation rules | はい | 必須入力などのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

中国行政区フィールドは、プラグインが提供する `chinaRegions` データテーブルに依存します。使用前に「中国行政区画」フィールドプラグインが有効になっていることを確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、中国行政区フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成した中国行政区フィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースとの同期や外部データソースからのマッピングによって作成されたフィールドを削除する場合の影響範囲は、対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、絞り込み、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での使用

中国行政区フィールドは、住所、地域、集計などのシーンで使用するのに適しています。

| シーン | 用途 |
| --- | --- |
| フォームブロック | カスケードセレクターで省、市、区県を選択します。 |
| 詳細ブロック | 行政区画のパスを表示します。 |
| テーブルブロック | レコードの所属地域を表示します。 |
| 絞り込みブロック | 地域でレコードを絞り込みます。 |
| グラフブロック | 省、市、区ごとに業務データを集計します。 |

### 編集モード

編集モードでは、中国行政区フィールドはカスケードセレクターとして表示されます。

![](https://static-docs.nocobase.com/%E7%9C%81%E5%B8%82%E5%8C%BA-04-29-2026_04_54_PM.png)

### 閲覧モード

閲覧モードでは、中国行政区フィールドはテキストパスとして表示されます。例：

```text
北京市 / 市辖区 / 东城区
```
