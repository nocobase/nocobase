# 数値

## 概要

NocoBase では、**数値（Number）** を小数を含む数値の保存に使用します。

数値フィールドは、金額、重量、面積、評価、単価などの業務データに適しています。フィルター、並べ替え、集計、数式、ワークフローの条件に利用できます。

値を整数に限定する場合は、[整数](./integer.md)を選ぶ方が適しています。比率やパーセンテージとして表示する場合は、[パーセンテージ](./percent.md)を選択してください。

## 利用シーン

数値は、次のような業務シーンに適しています。

- 注文金額、契約金額、単価
- 重量、面積、体積、距離
- 評価、係数、割引前の数値
- 集計や数式計算に使用する小数

## 作成時の設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「数値」を選択すると、数値フィールドを作成できます。

![20240512175752](https://static-docs.nocobase.com/20240512175752.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。数値の場合は `number` に対応し、画面上での入力方法と表示方法を決定します。 |
| Field display name | 画面上に表示されるフィールド名。例：「注文金額」「評価」「重量」。業務担当者が直感的に理解できる名称を推奨します。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字である必要があります。 |
| Field type | データ層におけるフィールドのタイプ。数値フィールドのデフォルトは `double` です。金額など、正確な小数が必要な場合は `decimal` を選択できます。 |
| Default value | デフォルト値。レコードの新規作成時にユーザーが入力しなかった場合、自動的に設定できます。 |
| Validation rules | バリデーションルール。最小値、最大値、精度、必須かどうかなどを制限できます。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、API から参照されます。後から変更すると設定の調整コストが発生するため、作成前に命名を確認してください。

:::

## フィールドの特性

数値フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `number`。 |
| デフォルトの Field type | `double`。 |
| 選択可能な Field type | `float`、`double`、`decimal`。 |
| ページコンポーネント | 編集モードでは数値入力ボックスを使用します。 |
| フィルター | 等しい、等しくない、より大きい、より小さい、範囲、空、空でないなどの数値フィルターに対応しています。 |
| 並べ替え | テーブルブロックでの並べ替えに対応しています。 |
| バリデーション | 数値範囲、必須などのバリデーションに対応しています。 |

## 編集時の設定

作成後、フィールド右側の「Edit」をクリックすると、数値フィールドの設定を編集できます。フィールドの編集では、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定など、NocoBase におけるフィールドの表示方法や使用方法を調整します。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上のフィールド表示名を変更します。フィールド識別名は変更しません。 |
| Field name | いいえ | フィールド識別名は、作成後、通常は編集フォームで変更できません。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上の入力、表示、バリデーション方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | レコードの新規作成時に使用するデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更するだけの操作ではありません。フィールドの保存方式、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、事前にデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、数値フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新しく作成した数値フィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されているデータも同時に削除されます。データベースから同期したフィールドや外部データソースからマッピングしたフィールドを削除した場合の影響範囲は、対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが業務上の設定から参照されていないか確認してください。

:::

## ページ設定での利用

数値フィールドは、入力、集計、グラフ、ワークフローの判定などに適しています。
![20260709225103](https://static-docs.nocobase.com/20260709225103.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | 金額、評価、重量などの数値を入力します。 |
| テーブルブロック | 数値の表示、並べ替え、フィルタリングを行います。 |
| グラフブロック | 数値フィールドによる集計、合計、平均の算出を行います。 |
| 数式フィールド | 数式計算の入力フィールドとして使用します。 |
