# カラー

## 概要

NocoBase では、**カラー（Color）** を使用して色の値を保存します。

カラー項目は、分類、タグ、ステータス、グラフ、表示設定などに色を保存する場合に適しています。通常は、16進数のカラー値またはコンポーネントが対応している形式で保存されます。

色が選択肢項目の一部にすぎない場合は、選択肢項目内で直接色を設定できるため、必ずしも個別にカラー項目を作成する必要はありません。

## 利用シーン

カラーは、次のような業務シーンに適しています。

- 顧客ランクの色、ステータスの色
- タグの色、分類の色
- グラフの系列の色
- ページやカードの表示設定

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「カラー」を選択すると、カラー項目を作成できます。

![20240512175956](https://static-docs.nocobase.com/20240512175956.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | 項目のインターフェースタイプ。カラーの場合は `color` に対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | 画面上に表示される項目名。たとえば「ステータスの色」「タグの色」「グラフの色」などです。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーション項目、権限、ワークフローなどから内部参照するための項目識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字にする必要があります。 |
| Field type | データ層における項目のタイプ。カラー項目のデフォルトは `string` です。 |
| Default value | デフォルト値。新しいレコードを追加する際、ユーザーが入力しなかった場合に自動的に設定できます。 |
| Validation rules | 検証ルール。通常は必須に設定します。 |
| Description | 項目の説明。項目の意味、入力要件、データソース、管理者などを記載するのに適しています。 |

:::warning 注意

項目名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整のコストを避けてください。

:::

## 項目の特性

カラー項目のデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `color`。 |
| デフォルトの Field type | `string`。 |
| 選択可能な Field type | `string`。 |
| ページコンポーネント | 編集モードではカラーピッカーコンポーネントを使用します。 |
| フィルター | 色の値で絞り込めますが、通常は主要な検索条件にはなりません。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| 検証 | 必須などの基本的な検証に対応しています。 |

## 設定の編集

作成後、項目の右側にある「Edit」をクリックすると、カラー項目の設定を編集できます。項目の編集は、主に NocoBase での項目の表示方法や利用方法を調整するために使用します。たとえば、表示名、説明、デフォルト値、検証ルール、項目固有の設定などを変更できます。

項目がメインデータベース内ですでに同期されているテーブルに由来する場合、編集時には通常、項目マッピングを行います。これは、データベース項目を NocoBase の Field type と Field interface にマッピングする作業です。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上に表示される項目名を変更します。項目識別名は変更しません。 |
| Field name | いいえ | 項目識別名は、作成後通常は編集フォームから変更できません。 |
| Field interface | 条件付きで可能 | メインデータベースの項目または同期された項目は、項目マッピング時に調整できます。変更すると、ページ上の入力方法、表示方法、検証方法に影響します。 |
| Field type | 条件付きで可能 | メインデータベースの項目または同期された項目は、項目マッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | 新しいレコードを追加する際のデフォルト値を調整します。 |
| Validation rules | はい | 項目の検証ルールを調整します。 |
| Description | はい | 項目の意味、入力要件、データソース、管理者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。項目の保存方式、入力コンポーネント、検証ルール、フィルター条件、ワークフロー変数の利用方法に影響します。既存データが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## 項目の削除

項目の右側にある「Delete」をクリックすると、カラー項目を削除できます。メインデータベースでは、複数の項目を選択して一括削除することもできます。

メインデータベースで新しく作成したカラー項目を削除すると、通常はデータベース内の実際の列と、その列に保存されているデータも同時に削除されます。データベースとの同期または外部データソースからマッピングされた項目を削除する場合の影響範囲は、対応するデータソースと項目の取得元によって異なります。

:::danger 警告

項目を削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポートとエクスポート、既存データに影響する可能性があります。削除前に、その項目が現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での利用

カラー項目は、ページの表示や設定に関するシーンに適しています。
![20260709225444](https://static-docs.nocobase.com/20260709225444.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | 色の値を選択します。 |
| 詳細ブロック | 色を表示します。 |
| リストまたはカード | ステータス、タグ、分類の視覚的な識別子として使用します。 |
| グラフ | 色設定の参照元として使用します。 |
