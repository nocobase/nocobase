# フィールド

## 概要

NocoBase において、**Field（フィールド）** は [Collection（データテーブル）](../collection.md) 内のデータを表す業務属性です。1 つのレコードにどのような情報を保存できるか、またそれらの情報をページ上でどのように入力、表示、絞り込み、業務ロジックに利用するかを定義します。

| 定義 | 説明 |
| --- | --- |
| どのようなデータを保存するか | テキスト、数値、日付、ファイル、ステータス、リレーション、JSON など。 |
| ページ上でどのように使用するか | 入力ボックス、日付選択、ドロップダウンメニュー、添付ファイルアップロード、リレーション選択などを使って入力・表示します。 |
| どのように業務機能に関わるか | フィールドは、ページブロック、絞り込み、並べ替え、権限、ワークフロー、API、データのインポート・エクスポートなどの機能で使用されます。 |

フィールドは、次のようなものに対応できます。
- メインデータベース内の実際のデータベース列
- 外部データベース内の既存のデータベース列
- データベースビュー内のフィールド
- SQL クエリ結果内のフィールド
- REST API レスポンス内のフィールド
- データテーブル内のリレーションフィールド、システムフィールド、または仮想フィールド

「業務オブジェクトの 1 つの属性」と考えるとわかりやすいでしょう。たとえば、次のようになります。

| 業務オブジェクト | 対応する Field |
| --- | --- |
| 顧客 | 顧客名、携帯電話番号、顧客ランク、担当者 |
| 注文 | 注文番号、注文金額、注文ステータス、顧客 |
| 契約 | 契約名、契約締結日、契約添付ファイル、承認ステータス |
| タスク | タスクタイトル、期限、優先度、実行担当者 |
| ファイル | ファイル名、サイズ、MIME タイプ、URL |

## 使用シーン

ここでは、フィールドの分類ごとに一般的な使用シーンを整理します。まずはどの種類のフィールドを選択すればよいかを判断する際に役立ててください。具体的な設定方法、タイプマッピング、注意事項については、各分類のドキュメントで詳しく確認できます。

| フィールド分類 | 使用シーン |
| --- | --- |
| [テキストフィールド](./basic/input.md) | 名前、番号、説明、連絡先、リンク先などの内容の保存に適しています。 |
| [リッチテキストフィールド](./media/rich-text.md) | 本文、説明ドキュメント、対応方法、コードスニペットなど、より複雑な内容の保存に適しています。 |
| [数値フィールド](./basic/number.md) | 数量、金額、評価、割合などの数値の保存に適しています。 |
| [日付・時刻フィールド](./datetime/index.md) | 時点、日付、時刻、外部システムのタイムスタンプなどの保存に適しています。 |
| [ステータス・オプションフィールド](./choices/select.md) | 有効・無効、注文ステータス、顧客ランク、顧客タグなど、一定の範囲内の値の保存に適しています。 |
| [添付ファイルフィールド](./media/field-attachment.md) | ファイルのアップロード、または外部ファイルの URL の保存に適しています。 |
| [リレーションフィールド](./associations/index.md) | 注文が顧客に属する、顧客が注文を所有する、ユーザーがロールに関連付けられているなど、異なるデータテーブル間の関係の表現に適しています。 |
| [識別子・コードフィールド](./advanced/uuid.md) | 内部主キー、外部同期 ID、公開アクセス用識別子、業務番号などの保存に適しています。 |
| [図形フィールド](./geometric/point.md) | 店舗の位置、配送ルート、サービス範囲など、空間情報や地理情報の保存に適しています。 |
| [システムフィールド](./system-info/created-at.md) | ID、作成日時、作成者、更新日時など、NocoBase やデータベースが管理するシステム情報の保存に適しています。 |
| [その他のフィールド](./advanced/json.md) | 並べ替え、数式、JSON など、他の分類に直接当てはまらないフィールド要件への対応に適しています。 |

## フィールドのデータタイプ

各 Field Interface にはデフォルトのデータタイプがあります。たとえば、Interface が数値（Number）のフィールドでは、データタイプのデフォルトは double ですが、float や decimal などを指定することもできます。現在、次のデータタイプをサポートしています。

![20240512103733](https://static-docs.nocobase.com/20240512103733.png)

## フィールドタイプのマッピング

メインデータベースでフィールドを追加する手順は次のとおりです。

1. Interface タイプを選択する
2. 現在の Interface で選択可能なデータタイプを設定する

![20240512172416](https://static-docs.nocobase.com/20240512172416.png)

外部データソースでのフィールドマッピング手順は次のとおりです。

1. 外部データベースのフィールドタイプに基づいて、対応するデータタイプ（Field type）と UI タイプ（Field Interface）を自動的にマッピングします。
2. 必要に応じて、より適切なデータタイプと Interface タイプに変更します

![20240512172759](https://static-docs.nocobase.com/20240512172759.png)
