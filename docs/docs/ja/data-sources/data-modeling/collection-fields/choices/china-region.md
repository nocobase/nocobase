# 中国行政区画（廃止）

## 概要

:::warning 注意

中国行政区画フィールドは廃止されています。代わりに、関係フィールドを使用してツリーテーブルと関連付けることをおすすめします。

:::

NocoBase の **中国行政区画（China region）** は、中国の省、市、区・県などの行政区画を選択するために使用します。

中国行政区画フィールドは、顧客住所、店舗住所、サービス提供地域など、地域を構造化して選択する必要がある場面に適しています。手入力で住所を入力するよりも、絞り込みや集計が容易になります。

完全な詳細住所を保存する必要がある場合は、[単行テキスト](../basic/input.md)または[複数行テキスト](../basic/textarea.md)と組み合わせて、街区や番地を保存できます。

## 適用シーン

中国行政区画は、次のような業務シーンに適しています。

- 顧客の所在する省・市・区
- 店舗のサービス提供地域
- プロジェクトの実施地域
- 配送先住所の行政区画

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「中国行政区画」を選択すると、中国行政区画フィールドを作成できます。

![20240512180305](https://static-docs.nocobase.com/20240512180305.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。中国行政区画には `chinaRegion` が対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | 画面上に表示されるフィールド名。「所在地域」「サービス提供地域」「配送先地域」などが使用できます。業務担当者が直接理解できる名称をおすすめします。 |
| Field name | API、関係フィールド、権限、ワークフローなどで内部的に参照するフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字である必要があります。 |
| Field type | データ層におけるフィールドの型です。行政区画フィールドは通常、構造化された値として保存されます。具体的な Field type はフィールド設定に従います。 |
| Default value | デフォルト値。レコードを新規作成する際にユーザーが入力しなかった場合、自動的にデフォルト値を設定できます。 |
| Validation rules | バリデーションルール。通常は必須入力や選択階層を設定します。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整のコストを避けてください。

:::

## フィールドの特性

中国行政区画フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `chinaRegion`。 |
| デフォルトの Field type | `json`。 |
| 選択可能な Field type | `json`、`string`。実際のフィールド設定に従います。 |
| ページコンポーネント | 編集モードでは行政区画選択コンポーネントを使用します。 |
| 絞り込み | 地域の値によるレコードの絞り込みに対応します。具体的な機能はフィールド設定によって異なります。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| バリデーション | 必須入力などの基本的なバリデーションに対応します。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、中国行政区画フィールドの設定を編集できます。フィールドの編集は、NocoBaseでのフィールドの表示方法や利用方法を調整するために行います。たとえば、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上に表示されるフィールド名を変更します。フィールド識別名は変更しません。 |
| Field name | いいえ | フィールド識別名は、作成後に編集フォームから変更できないのが通常です。 |
| Field interface | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上の入力、表示、バリデーション方法に影響します。 |
| Field type | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しい型で利用できるか確認する必要があります。 |
| Default value | はい | レコード新規作成時のデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、表示名を変更するだけの単純な操作ではありません。フィールドの保存方法、入力コンポーネント、バリデーションルール、絞り込み条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、あらかじめデータ形式が一致するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、中国行政区画フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成した中国行政区画フィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースから同期されたフィールドや外部データソースからマッピングされたフィールドを削除する場合の影響範囲は、対応するデータソースとフィールドの出所によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、絞り込み、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での使用

中国行政区画フィールドは、住所、地域、集計に関するシーンでの使用に適しています。

| シーン | 用途 |
| --- | --- |
| フォームブロック | 省、市、区・県を選択します。 |
| 詳細ブロック | 行政区画を表示します。 |
| 絞り込みブロック | 地域でレコードを絞り込みます。 |
| グラフブロック | 地域別に業務データを集計します。 |
