# ドロップダウン単一選択

## 概要

NocoBase では、**ドロップダウン単一選択（Select）** を使用して、選択肢の中から1つの値を選択できます。

ドロップダウン単一選択は、ステータス、レベル、タイプ、ソースなど、値の範囲が固定された業務フィールドに適しています。選択肢には、表示名、選択肢の値、色を設定できます。

複数の値を選択する必要がある場合は、[ドロップダウン複数選択](./multiple-select.md)を選択してください。「はい」または「いいえ」だけの場合は、[チェックボックス](./checkbox.md)を選択してください。

## 適用シーン

ドロップダウン単一選択は、次のような業務シーンに適しています。

- 注文ステータス、チケットステータス、承認ステータス
- 顧客レベル、リードソース、優先度
- プロジェクトタイプ、資産カテゴリ、契約タイプ
- 固定された範囲内から1つの値だけを選択するフィールド

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「ドロップダウン単一選択」を選択すると、ドロップダウン単一選択フィールドを作成できます。

![20240512180203](https://static-docs.nocobase.com/20240512180203.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。ドロップダウン単一選択は `select` に対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | インターフェース上で表示されるフィールド名。たとえば「注文ステータス」「顧客レベル」「優先度」などです。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字にする必要があります。 |
| Field type | データ層におけるフィールドのタイプ。ドロップダウン単一選択のデフォルトは `string` で、選択した選択肢の値を保存します。 |
| Default value | デフォルト値。レコードを新規作成する際、ユーザーが入力しなかった場合に自動的に設定できます。 |
| Validation rules | バリデーションルール。通常は必須設定や選択肢の値を設定します。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後から変更することによる設定変更の負担を避けてください。

:::

## フィールドの特性

ドロップダウン単一選択フィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `select`。 |
| デフォルトの Field type | `string`。 |
| 選択可能な Field type | `string`。 |
| ページコンポーネント | 編集モードではドロップダウンセレクターを使用します。 |
| フィルター | 選択肢によるフィルタリングに対応しています。 |
| 並べ替え | 選択肢の値による並べ替えに対応しています。 |
| バリデーション | 必須設定と選択肢の範囲制約に対応しています。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、ドロップダウン単一選択フィールドの設定を編集できます。フィールドの編集では、主に NocoBase でのフィールドの表示方法や使用方法を調整します。たとえば、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | インターフェース上で表示されるフィールド名を変更します。フィールド識別名は変更しません。 |
| Field name | いいえ | フィールド識別名は、作成後通常、編集フォームでは変更できません。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整すると、ページ上の入力方法、表示方法、バリデーション方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整前に、既存のデータを新しいタイプで使用できるか確認する必要があります。 |
| Default value | はい | レコードを新規作成する際のデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存のデータが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、ドロップダウン単一選択フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成したドロップダウン単一選択フィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースとの同期や外部データソースからマッピングされたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での使用

ドロップダウン単一選択フィールドは、フォーム、テーブル、カンバン、フィルターでの使用に適しています。
![20260709225912](https://static-docs.nocobase.com/20260709225912.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | ドロップダウンの選択肢から1つの値を選択します。 |
| テーブルブロック | 選択肢をラベルまたはテキストとして表示します。 |
| カンバンブロック | ステータスやステージなどの選択肢ごとにグループ化します。 |
| フィルターブロック | 選択肢によってレコードをすばやく絞り込みます。 |
