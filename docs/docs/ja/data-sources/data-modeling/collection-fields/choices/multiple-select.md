# ドロップダウン複数選択

## 概要

NocoBase では、**ドロップダウン複数選択（Multiple select）** を使用して、選択肢の中から複数の値を選択できます。

ドロップダウン複数選択は、タグ、能力、好み、適用範囲などのフィールドに適しています。複数の選択肢の値を保存し、ページ上では通常タグ形式で表示されます。

1つの値しか選択できない場合は、[ドロップダウン単一選択](./select.md)または[ラジオボタングループ](./radio-group.md)を選択してください。

## 適用シーン

ドロップダウン複数選択は、次のような業務シーンに適しています。

- 顧客タグ、ユーザーの好み
- 製品の適用シーン、デバイスの能力
- プロジェクトのリスク要因、問題の分類
- 複数の固定値を選択できるフィールド

## 作成設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「ドロップダウン複数選択」を選択すると、ドロップダウン複数選択フィールドを作成できます。

![20240512180236](https://static-docs.nocobase.com/20240512180236.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。ドロップダウン複数選択は `multipleSelect` に対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | インターフェース上で表示されるフィールド名。たとえば「顧客タグ」「適用シーン」「問題の分類」などです。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字でなければなりません。 |
| Field type | データ層におけるフィールドのタイプです。ドロップダウン複数選択は通常、配列または JSON として保存されます。具体的な形式は、フィールド設定とデータソースの対応状況によって異なります。 |
| Default value | デフォルト値。新しいレコードを追加するときに、ユーザーが入力しなかった場合はデフォルト値が自動的に入力されます。 |
| Validation rules | バリデーションルール。通常は必須入力と選択肢の範囲を設定します。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、API から参照されます。作成前に命名を確認し、後からの変更による設定調整のコストを避けてください。

:::

## フィールドの特性

ドロップダウン複数選択フィールドのデフォルト動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `multipleSelect`。 |
| デフォルトの Field type | `array`。 |
| 選択可能な Field type | `array`、`json`。実際のフィールドマッピングに準じます。 |
| ページコンポーネント | 編集モードでは複数選択ドロップダウンセレクターを使用します。 |
| フィルター | 特定の選択肢を含むレコードでフィルタリングできます。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| バリデーション | 必須入力と選択肢の範囲を制約できます。 |

## 編集設定

作成後、フィールド右側の「Edit」をクリックすると、ドロップダウン複数選択フィールドの設定を編集できます。フィールドの編集は主に、NocoBase におけるフィールドの表示方法や使用方法を調整するために行います。たとえば、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | インターフェース上で表示されるフィールド名を変更します。フィールド識別名は変更しません。 |
| Field name | いいえ | フィールド識別名は、作成後通常、編集フォームでは変更できません。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整すると、ページ上での入力、表示、バリデーションの方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整前に、既存データを新しいタイプで使用できるか確認する必要があります。 |
| Default value | はい | 新しいレコードを追加するときのデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type または Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、ドロップダウン複数選択フィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新しく作成したドロップダウン複数選択フィールドを削除すると、通常、データベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースとの同期や外部データソースからのマッピングによって作成されたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの出所によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが業務設定で参照されていないか確認してください。

:::

## ページ設定での使用

ドロップダウン複数選択フィールドは、複数のタグや固定された複数の選択肢を表すのに適しています。
![20260709230017](https://static-docs.nocobase.com/20260709230017.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | 選択肢から複数の値を選択します。 |
| テーブルブロック | 選択肢を複数のタグとして表示します。 |
| フィルターブロック | 特定のタグを含むレコードでフィルタリングします。 |
| ワークフローと権限 | 含む、含まないなどの条件判定に使用します。 |
