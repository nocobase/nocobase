# ラジオボタングループ

## 紹介

NocoBase では、**ラジオボタングループ（Radio group）**を使用して、複数の選択肢から1つの値を選択し、選択肢をフォーム内に直接表示できます。

ラジオボタングループは、選択肢の数が少なく、ユーザーがすべての選択肢を一目で確認する必要がある場面に適しています。ドロップダウン形式の単一選択と似ていますが、主な違いは画面上の操作方法です。

選択肢が多い場合は、[ドロップダウン単一選択](./select.md)のほうが省スペースです。複数選択する場合は、[チェックボックスグループ](./checkbox-group.md)を選択してください。

## 適用シーン

ラジオボタングループは、次のような業務シーンに適しています。

- 優先度：低、中、高
- 性別、タイプ、はい／いいえ形式の拡張選択肢
- 承認結果：承認、却下
- 少数の固定された選択肢からすばやく選択する場合

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「ラジオボタングループ」を選択すると、ラジオボタングループフィールドを作成できます。
![20260709231205](https://static-docs.nocobase.com/20260709231205.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプです。ラジオボタングループには `radioGroup` が対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | 「優先度」「承認結果」「タイプ」など、画面上に表示されるフィールド名です。業務担当者が直接理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部的に参照するフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字で始める必要があります。 |
| Field type | データ層におけるフィールドのタイプです。ラジオボタングループのデフォルトは `string` で、選択した選択肢の値を保存します。 |
| Default value | デフォルト値です。レコードの新規作成時にユーザーが入力しなかった場合、自動的にデフォルト値を設定できます。 |
| Validation rules | バリデーションルールです。通常は必須設定や選択肢の値を設定します。 |
| Description | フィールドの説明です。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整の負担を避けてください。

:::

## フィールドの特性

ラジオボタングループフィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `radioGroup`。 |
| デフォルトの Field type | `string`。 |
| 選択可能な Field type | `string`。 |
| ページコンポーネント | 編集モードではラジオボタングループを使用します。 |
| フィルター | 選択肢によるフィルタリングに対応します。 |
| 並べ替え | 選択肢の値による並べ替えに対応します。 |
| バリデーション | 必須設定と選択肢の範囲制約に対応します。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、ラジオボタングループフィールドの設定を編集できます。フィールドの編集は主に、NocoBase におけるフィールドの表示方法や利用方法を調整するために使用します。たとえば、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上に表示されるフィールド名を変更します。フィールド識別名は変更しません。 |
| Field name | いいえ | フィールド識別名は、作成後、通常は編集フォームで変更できません。 |
| Field interface | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上の入力、表示、バリデーション方法に影響します。 |
| Field type | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | レコードの新規作成時に設定されるデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、データ形式が適合するか事前に確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、ラジオボタングループフィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成したラジオボタングループフィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースから同期したフィールドや外部データソースからマッピングしたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート／エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが業務設定で引き続き参照されていないか確認してください。

:::

## ページ設定での使用

ラジオボタングループは、フォーム内で少数の選択肢を並べて表示するのに適しています。
![20260709230347](https://static-docs.nocobase.com/20260709230347.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | すべての選択肢を直接表示し、その中から1つを選択します。 |
| 詳細ブロック | 選択された選択肢を表示します。 |
| フィルターブロック | 選択肢に基づいてレコードを絞り込みます。 |
| ワークフローと権限 | ステータスやタイプなどの条件として判定に使用します。 |
