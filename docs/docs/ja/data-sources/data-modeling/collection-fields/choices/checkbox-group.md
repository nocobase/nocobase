# チェックボックスグループ

## 概要

NocoBase の **チェックボックスグループ（Checkbox group）** は、複数の選択肢から複数の値を選択し、その選択肢をフォーム内に直接表示するために使用します。

チェックボックスグループは、選択肢が少なく、複数選択が必要な場面に適しています。下拉式の複数選択と似ていますが、主な違いはページ上での操作方法です。

選択肢が多い場合は、[下拉式複数選択](./multiple-select.md)のほうが省スペースです。1つだけ選択する場合は、[ラジオボタングループ](./radio-group.md)を使用します。

## 適用シーン

チェックボックスグループは、次のような業務シーンに適しています。

- サービス範囲、適用チャネル
- 機能権限のチェック項目
- 顧客ニーズのタグ
- 少数の固定された複数選択項目

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「チェックボックスグループ」を選択すると、チェックボックスグループフィールドを作成できます。
![20260709231244](https://static-docs.nocobase.com/20260709231244.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。チェックボックスグループには `checkboxGroup` が対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | 画面上に表示されるフィールド名。たとえば「サービス範囲」「適用チャネル」「ニーズタグ」などです。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、先頭は英字にする必要があります。 |
| Field type | データ層におけるフィールドのタイプ。チェックボックスグループは通常、配列または JSON として保存されます。具体的な形式は、フィールド設定とデータソースの機能によって異なります。 |
| Default value | デフォルト値。新しいレコードを追加する際に、ユーザーが入力しなかった場合は自動的にデフォルト値が入力されます。 |
| Validation rules | バリデーションルール。通常は必須入力や選択肢の範囲を設定します。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、API から参照されます。作成前に命名を確認し、後から変更することで設定の調整が必要になるのを避けてください。

:::

## フィールドの特性

チェックボックスグループフィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `checkboxGroup`。 |
| デフォルトの Field type | `array`。 |
| 選択可能な Field type | `array`、`json`。実際のフィールドマッピングに準じます。 |
| ページコンポーネント | 編集モードではチェックボックスグループを使用します。 |
| フィルター | 特定の選択肢を含むレコードでフィルタリングできます。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| バリデーション | 必須入力と選択肢の範囲制約に対応しています。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、チェックボックスグループフィールドの設定を編集できます。フィールドの編集は主に、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定など、NocoBase におけるフィールドの表示方法や使用方法を調整するために行います。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースのフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上に表示されるフィールド名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は、作成後に編集フォームから変更できないのが通常です。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上での入力、表示、バリデーションの方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | 新しいレコードを追加する際のデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、担当者などを補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存形式、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、チェックボックスグループフィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新しく作成したチェックボックスグループフィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースとの同期や外部データソースからマッピングされたフィールドを削除する場合の影響範囲は、対応するデータソースとフィールドの由来によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが現在も業務設定から参照されていないか確認してください。

:::

## ページ設定での使用

チェックボックスグループは、少数の複数選択項目をフォーム上に並べて表示するのに適しています。
![20260709230421](https://static-docs.nocobase.com/20260709230421.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | すべての選択肢を直接表示し、複数選択します。 |
| 詳細ブロック | 複数の選択肢をタグまたはテキストで表示します。 |
| フィルターブロック | 特定の選択肢を含むレコードでフィルタリングします。 |
| ワークフローと権限 | 含む、含まないなどの条件として判定に使用します。 |
