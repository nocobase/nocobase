# リッチテキスト

## 概要

NocoBase では、**リッチテキスト（Rich text）** を使用して書式付きコンテンツを保存します。

リッチテキストフィールドは、告知本文、記事本文、メールテンプレート、説明ドキュメントなどに適しています。見たまま編集できる操作感に近いのが特徴です。

チームで Markdown を使い慣れている場合や、プレーンテキストで書式を細かく制御したい場合は、[Markdown](./markdown.md) または [Markdown Vditor](../../../field-markdown-vditor/index.md) を選択してください。

## 適用シーン

リッチテキストは、次のような業務シーンに適しています。

- 告知本文、記事本文
- メールテンプレート、通知テンプレート
- 製品説明、操作説明
- 画像、リンク、スタイルが必要なコンテンツ

## 作成時の設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「リッチテキスト」を選択すると、リッチテキストフィールドを作成できます。

![20240512181002](https://static-docs.nocobase.com/20240512181002.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。リッチテキストでは `richText` に対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | 画面上に表示されるフィールド名。「告知本文」「メールテンプレート」「製品説明」などが該当します。業務担当者がすぐに理解できる名称を使用することをおすすめします。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名です。作成後は通常変更せず、使用できるのは英字、数字、アンダースコアのみで、先頭は英字にする必要があります。 |
| Field type | データ層におけるフィールドのタイプ。リッチテキストフィールドでは通常、`text` を使用して内容を保存します。 |
| Default value | デフォルト値。レコードの新規作成時にユーザーが入力しなかった場合、自動的にデフォルト値を設定できます。 |
| Validation rules | バリデーションルール。文字数や必須入力を制限できます。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、管理担当者などを記載するのに適しています。 |

:::warning 注意

フィールド名は作成後、ページブロック、権限、ワークフロー、API から参照されます。作成前に命名を確認し、後からの変更による設定調整の負担を避けてください。

:::

## フィールドの特徴

リッチテキストフィールドのデフォルト動作は次のとおりです。

| 特徴 | 説明 |
| --- | --- |
| デフォルトの Field interface | `richText`。 |
| デフォルトの Field type | `text`。 |
| 選択可能な Field type | `text`。 |
| ページコンポーネント | 編集モードではリッチテキストエディターを使用します。 |
| フィルター | 含む、空、空でないなど、テキスト向けのフィルターに対応しています。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| バリデーション | 文字数、必須入力などのテキストバリデーションに対応しています。 |

## 編集時の設定

作成後、フィールド右側の「Edit」をクリックすると、リッチテキストフィールドの設定を編集できます。フィールドの編集では、表示名、説明、デフォルト値、バリデーションルール、フィールド固有の設定など、NocoBase での表示方法や利用方法を調整します。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | 画面上に表示されるフィールド名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は、作成後に編集フォームから変更できないのが一般的です。 |
| Field interface | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更すると、ページ上での入力、表示、バリデーションの方法に影響します。 |
| Field type | 条件付きで可 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。変更前に、既存データを新しいタイプで利用できるか確認する必要があります。 |
| Default value | はい | レコードを新規作成する際のデフォルト値を調整します。 |
| Validation rules | はい | フィールドのバリデーションルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、管理担当者などを補足します。 |

:::warning 注意

Field type または Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、バリデーションルール、フィルター条件、ワークフロー変数の利用方法に影響します。既存データが多い場合は、あらかじめデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、リッチテキストフィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成したリッチテキストフィールドを削除すると、通常はデータベース内の実際の列と、その列に保存されている既存データも同時に削除されます。データベースとの同期や外部データソースからのマッピングによって作成されたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの取得元によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが業務設定から参照されていないか確認してください。

:::

## ページ設定での利用

リッチテキストフィールドは、コンテンツの編集や表示を行うシーンに適しています。
![20260709231418](https://static-docs.nocobase.com/20260709231418.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | リッチテキストコンテンツを編集します。 |
| 詳細ブロック | リッチテキスト形式でコンテンツを表示します。 |
| メールまたは通知テンプレート | テンプレート本文のソースとして使用します。 |
| テーブルブロック | 概要または簡略化した内容を表示します。 |
