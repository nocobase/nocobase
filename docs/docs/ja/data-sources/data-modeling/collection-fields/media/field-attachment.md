# 添付ファイルフィールド

## はじめに

システムには「添付ファイル」タイプのフィールドが組み込まれており、カスタムの**コレクション**でファイルのアップロードをサポートします。

添付ファイルフィールドは、その内部ではシステムに組み込まれているファイル**コレクション**「Attachments」（`attachments`）を指す多対多のリレーションフィールドです。任意の**コレクション**に添付ファイルフィールドを作成すると、多対多のリレーション用の中間テーブルが自動的に生成されます。アップロードされたファイルのメタデータは「Attachments」**コレクション**に保存され、**コレクション**内で参照されるファイル情報は、この中間テーブルを介して関連付けられます。

## フィールド設定

![20240512180916](https://static-docs.nocobase.com/20251031000729.png)

### MIMEタイプ制限

アップロードを許可するファイルの種類を制限するために使用します。[MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types)構文で形式を記述します。例えば、`image/*` は画像ファイルを表します。複数のタイプはカンマ（,）で区切ることができ、`image/*,application/pdf` のように記述すると、画像タイプとPDFタイプの両方のファイルを許可します。

### ストレージエンジン

アップロードされたファイルを保存するためのストレージエンジンを選択します。空欄の場合、システムのデフォルトストレージエンジンが使用されます。
