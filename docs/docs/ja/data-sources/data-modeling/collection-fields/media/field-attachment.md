# 添付ファイルフィールド

## 概要

システムには「添付ファイル」タイプのフィールドが組み込まれており、カスタムデータテーブルでユーザーがファイルをアップロードできるようにします。

添付ファイルフィールドは、多対多リレーションフィールドとして実装されており、システムに組み込まれた「添付ファイル」ファイルテーブル（`attachments`）を参照します。任意のデータテーブルに添付ファイルフィールドを作成すると、添付ファイルテーブルとの多対多リレーション用中間テーブルが自動的に生成されます。アップロードされたファイルのメタデータは「添付ファイル」テーブルに保存され、データテーブルで参照されるファイル情報は、この中間テーブルを介して関連付けられます。

## フィールド設定

![20240512180916](https://static-docs.nocobase.com/20251031000729.png)

### MIMEタイプの制限

アップロード可能なファイルの種類を制限するために使用します。形式の記述には [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types) 構文を使用します。たとえば、`image/*` は画像ファイルを表します。複数のタイプは英語のカンマで区切って指定できます。たとえば、`image/*,application/pdf` は画像ファイルとPDFファイルを許可することを表します。

### ストレージエンジン

アップロードされたファイルの保存に使用するストレージエンジンを選択します。指定しない場合は、システムのデフォルトストレージエンジンが使用されます。
