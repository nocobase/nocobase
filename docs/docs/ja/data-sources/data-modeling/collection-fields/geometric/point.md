# ポイント

## 概要

NocoBase では、**ポイント（Point）**を使用して単一の位置座標を保存します。

ポイントフィールドは、店舗の位置、顧客の位置、デバイスの位置などの空間データに適しています。地図ブロックと組み合わせることで、レコードを地図上に表示できます。

ルートを保存する場合は[ライン](./line.md)を選択してください。エリアを保存する場合は[ポリゴン](./polygon.md)または[円](./circle.md)を選択してください。

## 適用シーン

ポイントは、次のような業務シーンに適しています。

- 店舗の位置、倉庫の位置
- 顧客住所の座標
- デバイスの設置位置
- 巡回点検のチェックイン位置

## 作成と設定

データテーブルの「Configure fields」ページで「Add field」をクリックし、「ポイント」を選択すると、ポイントフィールドを作成できます。

![20240512181420](https://static-docs.nocobase.com/20240512181420.png)

| 設定 | 説明 |
| --- | --- |
| Field interface | フィールドのインターフェースタイプ。ポイントの場合は `point` に対応し、ページ上での入力方法と表示方法を決定します。 |
| Field display name | インターフェース上で表示されるフィールド名。たとえば「店舗の位置」「デバイス座標」「顧客の位置」などです。業務担当者が直接理解できる名称を使用することを推奨します。 |
| Field name | API、リレーションフィールド、権限、ワークフローなどで内部参照するためのフィールド識別名です。作成後は通常変更せず、英字、数字、アンダースコアのみ使用でき、英字で始める必要があります。 |
| Field type | データ層におけるフィールドのタイプ。ポイントフィールドのデフォルトは `point` です。 |
| Default value | デフォルト値。新規レコードの追加時にユーザーが入力しなかった場合、自動的にデフォルト値を設定できます。 |
| Validation rules | 検証ルール。通常は必須項目に設定すれば十分です。 |
| Description | フィールドの説明。フィールドの意味、入力要件、データソース、または管理担当者を記載するのに適しています。 |

:::warning 注意

フィールド名は、作成後にページブロック、権限、ワークフロー、APIから参照されます。作成前に命名を確認し、後からの変更による設定調整のコストを避けてください。

:::

## フィールドの特性

ポイントフィールドのデフォルトの動作は次のとおりです。

| 特性 | 説明 |
| --- | --- |
| デフォルトの Field interface | `point`。 |
| デフォルトの Field type | `point`。 |
| 選択可能な Field type | `point`。 |
| ページコンポーネント | 編集モードでは、地図または座標選択コンポーネントを使用します。 |
| フィルター | 空間フィルターの機能は、地図プラグインとデータソースの機能に依存します。 |
| 並べ替え | 通常、並べ替えには使用しません。 |
| 検証 | 必須項目などの基本的な検証に対応しています。 |

## 設定の編集

作成後、フィールド右側の「Edit」をクリックすると、ポイントフィールドの設定を編集できます。フィールドの編集は、主に NocoBase でのフィールドの表示方法や使用方法を調整するために使用します。たとえば、表示名、説明、デフォルト値、検証ルール、フィールド固有の設定などを変更できます。

フィールドがメインデータベースですでに同期されているテーブルに由来する場合、編集時には通常、フィールドマッピングを行います。つまり、データベースフィールドを NocoBase の Field type と Field interface にマッピングします。

| 設定 | 編集可否 | 説明 |
| --- | --- | --- |
| Field display name | はい | インターフェース上の表示名を変更します。フィールド識別名は変更されません。 |
| Field name | いいえ | フィールド識別名は、作成後通常、編集フォームでは変更できません。 |
| Field interface | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整後は、ページ上での入力、表示、検証の方法に影響します。 |
| Field type | 条件付きで対応 | メインデータベースのフィールドまたは同期フィールドは、フィールドマッピング時に調整できます。調整前に、既存データを新しいタイプで使用できるか確認する必要があります。 |
| Default value | はい | 新規レコード追加時のデフォルト値を調整します。 |
| Validation rules | はい | フィールドの検証ルールを調整します。 |
| Description | はい | フィールドの意味、入力要件、データソース、または管理担当者に関する情報を補足します。 |

:::warning 注意

Field type や Field interface の切り替えは、単に表示名を変更することとは異なります。フィールドの保存方法、入力コンポーネント、検証ルール、フィルター条件、ワークフロー変数の使用方法に影響します。既存データが多い場合は、事前にデータ形式が適合するか確認してください。

:::

## フィールドの削除

フィールド右側の「Delete」をクリックすると、ポイントフィールドを削除できます。メインデータベースでは、複数のフィールドを選択して一括削除することもできます。

メインデータベースで新規作成したポイントフィールドを削除すると、通常、データベース内の実際の列と、その列に保存されているデータも同時に削除されます。データベースから同期したフィールドや外部データソースからマッピングしたフィールドを削除する場合、影響範囲は対応するデータソースとフィールドの出所によって異なります。

:::danger 警告

フィールドを削除すると、ページブロック、フォーム、フィルター、権限、ワークフロー、API、インポート・エクスポート、既存データに影響する可能性があります。削除前に、そのフィールドが業務設定で参照されていないか確認してください。

:::

## ページ設定での使用

ポイントフィールドは、地図や位置管理のシーンで使用するのに適しています。
![20260710144034](https://static-docs.nocobase.com/20260710144034.png)

| シーン | 用途 |
| --- | --- |
| フォームブロック | 位置を1つ選択または入力します。 |
| 詳細ブロック | 位置座標または地図上のポイントを表示します。 |
| 地図ブロック | 地図上にポイントを表示します。 |
| ワークフロー | 位置に関連する業務条件の入力として使用します。 |
