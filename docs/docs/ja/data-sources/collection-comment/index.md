# コメントコレクション

## はじめに

コメントコレクションは、ユーザーのコメントやフィードバックを保存するために特化したデータテーブルテンプレートです。このコメント機能を使えば、あらゆるデータテーブルにコメント機能を追加でき、ユーザーは特定のレコードについて議論したり、フィードバックを提供したり、注釈を付けたりすることができます。コメントコレクションはリッチテキスト編集に対応しており、柔軟なコンテンツ作成が可能です。

![comment-collection-2025-11-01-00-39-01](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-39-01.png)

## 機能

- **リッチテキスト編集**: デフォルトでMarkdown（vditor）エディタが含まれており、リッチテキストコンテンツの作成をサポートしています。
- **任意のデータテーブルへの関連付け**: リレーションフィールドを通じて、コメントを任意のデータテーブルのレコードに関連付けることができます。
- **多階層コメント**: コメントへの返信をサポートし、コメントツリー構造を構築できます。
- **ユーザー追跡**: コメントの作成者と作成時間を自動的に記録します。

## 利用ガイド

### コメントコレクションの作成

![comment-collection-2025-11-01-00-37-10](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-37-10.png)

1. データテーブル管理ページに移動します。
2. 「新規コレクション作成」ボタンをクリックします。
3. 「コメントコレクション」テンプレートを選択します。
4. テーブル名（例：「タスクコメント」、「記事コメント」など）を入力します。
5. システムは、以下のデフォルトフィールドを含むコメントコレクションを自動的に作成します。
   - コメント内容（Markdown vditor タイプ）
   - 作成者（ユーザーテーブルにリンク）
   - 作成日時（日付時刻タイプ）

### 関連関係の設定

コメントをターゲットのデータテーブルに関連付けるには、リレーションフィールドを設定する必要があります。

![](https://static-docs.nocobase.com/Solution/demoE3v1-19N.gif)

1. コメントコレクションに「多対一」のリレーションフィールドを追加します。
2. 関連付けたいターゲットのデータテーブル（例：タスクテーブル、記事テーブルなど）を選択します。
3. フィールド名（例：「所属タスク」、「所属記事」など）を設定します。

### ページでのコメントブロックの使用

![Enable Comments Collection](https://static-docs.nocobase.com/Solution/demoE3v1-20.gif)

1. コメント機能を追加したいページに移動します。
2. ターゲットレコードの詳細またはポップアップにブロックを追加します。
3. 「コメント」ブロックタイプを選択します。
4. 先ほど作成したコメントコレクションを選択します。

### 典型的な利用シーン

- **タスク管理システム**: チームメンバーがタスクについて議論し、フィードバックを提供します。
- **コンテンツ管理システム**: 読者が記事にコメントしたり、交流したりします。
- **承認ワークフロー**: 承認者が申請書に注釈を付けたり、意見を述べたりします。
- **顧客フィードバック**: 製品やサービスに対する顧客の評価を収集します。

## 注意点

- コメントコレクションは商用プラグイン機能であり、利用するにはコメントプラグインを有効にする必要があります。
- コメントコレクションには適切な権限を設定し、誰がコメントを閲覧、作成、削除できるかを制御することをお勧めします。
- 大量のコメントがあるシナリオでは、パフォーマンス向上のためにページネーションを有効にすることをお勧めします。
