# 概要

## 組み込みエンジン

現在、NocoBase に組み込まれているエンジンの種類は次のとおりです。

- [ローカルストレージ](./local.md)
- [Alibaba Cloud OSS](./aliyun-oss.md)
- [Amazon S3](./amazon-s3.md)
- [Tencent Cloud COS](./tencent-cos.md)

システムのインストール時に、ローカルストレージエンジンが自動的に追加され、そのまま使用できます。新しいエンジンを追加したり、既存のエンジンのパラメータを編集したりすることもできます。

## エンジン共通パラメータ

各エンジン固有のパラメータ以外に、以下は共通パラメータです（ローカルストレージを例に説明します）。

![ファイルストレージエンジンの設定例](https://static-docs.nocobase.com/20240529115151.png)

### タイトル

ストレージエンジンの名称。識別しやすくするために使用します。

### システム名

ストレージエンジンのシステム名。システムによる識別に使用します。システム内で一意である必要があり、入力しない場合はシステムによって自動的にランダム生成されます。

### アクセス URL のベース

ファイルに外部からアクセスするための URL アドレスのプレフィックス部分です。CDN のアクセス URL のベースを指定することもできます。例：「`https://cdn.nocobase.com/app`」（末尾の「`/`」は不要）。

### パス

ファイルの保存時に使用する相対パスです。アクセス時には、この部分も最終的な URL に自動的に連結されます。例：「`user/avatar`」（先頭と末尾の「`/`」は不要）。

### ファイルサイズの上限

このストレージエンジンにファイルをアップロードする際のサイズ上限です。設定したサイズを超えるファイルはアップロードできません。システムのデフォルト上限は 20MB で、最大 1GB まで変更できます。

### ファイルタイプ

アップロードするファイルのタイプを制限できます。[MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types) 構文で形式を指定します。例：`image/*` は画像ファイルを表します。複数のタイプは半角カンマで区切ります。例：`image/*, application/pdf` は画像ファイルと PDF ファイルを許可することを表します。

### デフォルトストレージエンジン

チェックすると、システムのデフォルトストレージエンジンに設定されます。添付ファイルフィールドまたはファイルテーブルでストレージエンジンが指定されていない場合、アップロードされたファイルはすべてデフォルトストレージエンジンに保存されます。デフォルトストレージエンジンは削除できません。

### レコード削除時にファイルを保持

チェックすると、添付ファイルテーブルまたはファイルテーブルのデータレコードを削除しても、ストレージエンジンにアップロード済みのファイルは保持されます。デフォルトではチェックされていないため、レコードを削除するとストレージエンジン内のファイルも同時に削除されます。

:::info{title=ヒント}
ファイルのアップロード後、最終的なアクセスパスは複数の部分を連結して構成されます。

```
<访问 URL 基础>/<路径>/<文件名><后缀名>
```

例：`https://cdn.nocobase.com/app/user/avatar/20240529115151.png`。
:::
