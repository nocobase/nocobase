# ファイルマネージャー

## 概要

ファイルマネージャー**プラグイン**は、ファイル管理を効率的に行うための**コレクション**、添付ファイルフィールド、およびファイルストレージエンジンを提供します。ファイルとは、ファイルのメタ情報を保存し、ファイルマネージャーで管理できる、特殊な構造を持つデータテーブルのレコード（**ファイルコレクション**）です。添付ファイルフィールドは、このファイルコレクションに関連付けられた特定の関連フィールドとなります。ファイルは複数のストレージ方法に対応しており、現在サポートされているファイルストレージエンジンには、ローカルストレージ、Alibaba Cloud OSS、Amazon S3、Tencent Cloud COS などがあります。

## ユーザーマニュアル

### ファイルコレクション

すべての添付ファイルフィールドに関連付けられたファイルを保存するための `attachments` コレクションが組み込まれています。これとは別に、特定のファイルを保存するための新しいファイルコレクションを作成することも可能です。

詳細については、[ファイルコレクションの紹介ドキュメント](/data-sources/file-manager/file-collection)をご覧ください。

### 添付ファイルフィールド

添付ファイルフィールドは、ファイルコレクションに関連付けられた特定の関連フィールドです。「添付ファイルタイプ」フィールドとして作成することも、「関連」フィールドとして設定することもできます。

詳細については、[添付ファイルフィールドの紹介ドキュメント](/data-sources/file-manager/field-attachment)をご覧ください。

### ファイルストレージエンジン

ファイルストレージエンジンは、ファイルを特定のサービスに保存するために使用されます。これには、ローカルストレージ（サーバーのハードドライブへの保存）やクラウドストレージなどが含まれます。

詳細については、[ファイルストレージエンジンの紹介ドキュメント](./storage/index.md)をご覧ください。

### HTTP API

ファイルのアップロードは HTTP API を介して処理できます。詳細については、[HTTP API](./http-api.md) を参照してください。
