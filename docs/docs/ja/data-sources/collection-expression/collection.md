# 式テーブル

## 「式」テンプレートテーブルを作成する

ワークフローで動的な式の計算ノードを使用する前に、データテーブル管理ツールで「式」テンプレートテーブルを作成し、さまざまな式を保存できるようにします。

![式テンプレートテーブルの作成](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## 式データを入力する

次に、テーブルブロックを作成してこのテンプレートテーブルに数件の数式データを追加します。「式」テンプレートテーブルの各行のデータは、特定のデータテーブルモデルを対象とした計算ルールとして扱うことができます。各行の数式データでは、異なるデータテーブルモデルのフィールド値を変数として使用し、さまざまな式を計算ルールとして記述できます。もちろん、異なる計算エンジンを使用することも可能です。

![式データの入力](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=ヒント}
数式を作成した後は、業務データと数式を関連付ける必要があります。各業務データを数式データの各行に直接関連付けるのは手間がかかるため、通常は、分類のようなメタデータテーブルと数式テーブルを多対一（または一対一）で関連付け、さらに業務データと分類メタデータを多対一で関連付けます。こうすることで、業務データを作成する際には特定の分類メタデータを指定するだけで、後からこの関連付けの経路を通じて対応する数式データを見つけて使用できます。
:::

## ワークフローで対応するデータを読み込む

データテーブルイベントを例に、注文が作成されたときにトリガーされ、注文に関連する商品データと商品の式データを事前に読み込むワークフローを作成します。

![データテーブルイベント_トリガー設定](https://static-docs.nocobase.com/f181f75b10007afd5de068f3458d2e04.png)
