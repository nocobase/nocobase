# 通知：アプリ内メッセージ

## はじめに

NocoBaseアプリケーション内で、ユーザーがリアルタイムにメッセージ通知を受け取れるようにします。

## インストール

このプラグインは組み込みプラグインのため、インストールは不要です。

## アプリ内メッセージチャネルの追加

通知管理を開き、[追加]ボタンをクリックして、[アプリ内メッセージ]を選択します。
![2024-11-08-08-33-26-20241108083326](https://static-docs.nocobase.com/2024-11-08-08-33-26-20241108083326.png)

チャネル名と説明を入力後、[送信]をクリックします。
![2024-11-08-08-34-32-20241108083431](https://static-docs.nocobase.com/2024-11-08-08-34-32-20241108083431.png)

リストに新しいチャネルが追加されます。

![2024-11-08-08-34-52-20241108083452](https://static-docs.nocobase.com/2024-11-08-08-34-52-20241108083452.png)

## ユースケース例

アプリ内メッセージの利用方法をより深くご理解いただくために、「マーケティングリードのフォローアップ」に関する例をご紹介します。

あなたのチームが、潜在顧客からのフィードバックやニーズを追跡する重要なマーケティングキャンペーンを実施しているとします。アプリ内メッセージを使用すると、次のことができます。

**通知チャネルの作成**：まず、通知管理で「Marketing Clue」という名前のアプリ内メッセージチャネルを設定し、チームメンバーがそのチャネルの用途を明確に識別できるようにします。

![2024-11-08-08-34-32-20241108083431](https://static-docs.nocobase.com/2024-11-08-08-34-32-20241108083431.png)

**ワークフローの構成**：新しいマーケティングリードが発生した際に、自動的に通知をトリガーするワークフローを作成します。ワークフローに通知ノードを追加し、以前作成した「Marketing Clue」チャネルを選択して、実際の要件に応じてメッセージ内容を設定できます。例：

![image-1-2024-10-27-14-07-17](https://static-docs.nocobase.com/image-1-2024-10-27-14-07-17.png)

**リアルタイム通知の受信**：ワークフローがトリガーされると、関連するすべての担当者がリアルタイムで通知を受け取り、チームが迅速に対応・行動できるようになります。

![image-2-2024-10-27-14-07-22](https://static-docs.nocobase.com/image-2-2024-10-27-14-07-22.png)

**メッセージの管理と追跡**：アプリ内メッセージは、送信チャネル名に基づいてグループ化されます。メッセージの既読・未読ステータスでフィルタリングし、重要な情報を素早く確認できます。[表示]ボタンをクリックすると、設定されたリンクページに移動し、関連する事項をさらに処理できます。

![20241027140648-2024-10-27-14-06-51-2024-10-29-13-26-41](https://static-docs.nocobase.com/20241027140648-2024-10-27-14-06-51-2024-10-29-13-26-41.png)
