# インストール方法

> 現在のバージョンは **バックアップ復元** 方式でデプロイします。今後のバージョンでは、既存の NocoBase 環境への組み込みを容易にする **差分マイグレーション** 方式に切り替える可能性があります。

一体型業務管理システムは **CRM(顧客管理)、販売管理、ヘルプデスク、プロジェクト管理、固定資産管理、HR 人事管理** の 6 モジュールを網羅しています。本ソリューションをお使いの NocoBase 環境にスムーズに導入していただけるよう、2 種類の復元方法を用意しています。ご利用中のエディションや技術背景に合わせて適した方法を選んでください。

開始前にご確認ください:

- 基本的な NocoBase 実行環境が準備できていること。本体のインストールについては [公式インストールガイド](https://docs-cn.nocobase.com/welcome/getting-started/installation) を参照してください。
- NocoBase のバージョンが **v2.1.0-alpha.34 以降** であること。
- 一体型ソリューションの対応ファイルを以下からダウンロード済みであること:
  - **バックアップファイル**:[nocobase_all_in_one_backup_260521.nbdata](https://static-docs.nocobase.com/nocobase_all_in_one_backup_260521.nbdata) — 方法 1 で使用
  - **SQL ファイル**:[nocobase_all_in_one_sql_260521.zip](https://static-docs.nocobase.com/nocobase_all_in_one_sql_260521.zip) — 方法 2 で使用

**重要事項**:

- 本ソリューションは **PostgreSQL 16** で作成しています。お使いの環境も PostgreSQL 16 であることを確認してください。
- **DB_UNDERSCORED は true にしない**:`docker-compose.yml` で `DB_UNDERSCORED` 環境変数が `true` に設定されていないことを確認してください。設定されていると、バックアップの設定と競合し、復元に失敗します。

## 方法 1:バックアップマネージャーで復元(プロフェッショナル / エンタープライズ版向け推奨)

NocoBase 内蔵の「[バックアップマネージャー](https://docs-cn.nocobase.com/handbook/backups)」(プロフェッショナル / エンタープライズ版)プラグインでワンクリック復元する方法です。操作は最も簡単ですが、環境とエディションに一定の要件があります。

### 特徴

* **メリット**:
  1. **操作が手軽**:UI 上で完結し、プラグインを含むすべての設定を完全に復元できます。
  2. **完全復元**:**すべてのシステムファイルを復元** できます。テンプレート印刷ファイル、レコードのファイル添付、AI 社員のアバターなどを含みます。
* **制約**:
  1. **プロフェッショナル / エンタープライズ版限定**:「バックアップマネージャー」はエンタープライズ向けプラグインで、プロフェッショナル / エンタープライズ版でのみ利用できます。
  2. **環境要件が厳しい**:お使いのデータベース環境(バージョン、大文字小文字の感度など)が、バックアップ作成時の環境と高い互換性を持っている必要があります。
  3. **プラグイン依存**:ローカル環境に存在しない商用プラグインが含まれている場合、復元に失敗します。

### 操作手順

**Step 1:【強く推奨】`full` イメージでアプリを起動**

データベースクライアントの欠落による復元失敗を避けるため、Docker イメージは `full` 版の利用を強く推奨します。`full` には必要な周辺コンポーネントがすべて含まれており、追加設定は不要です。

イメージ取得コマンドの例:

```bash
docker pull nocobase/nocobase:alpha-full
```

このイメージを使って NocoBase を起動してください。

> **注意**:`full` イメージを使わない場合、コンテナ内に `pg_dump` クライアントを手動でインストールする必要があり、手順が煩雑かつ不安定になりがちです。

**Step 2:「バックアップマネージャー」プラグインを有効化**

1. NocoBase システムにログインします。
2. **`プラグイン管理`** に入ります。
3. **`バックアップマネージャー`** プラグインを見つけて有効化します。

**Step 3:ローカルバックアップファイルから復元**

1. プラグインを有効化したらページを更新します。
2. 左メニューの **`システム管理`** → **`バックアップマネージャー`** に進みます。
3. 右上の **`ローカルバックアップから復元`** ボタンをクリックします。
4. ダウンロードした `nocobase_all_in_one_backup_260521.nbdata` ファイルをアップロード領域にドラッグします。
5. **`送信`** をクリックし、復元完了までしばらく待ちます(数十秒から数分かかることがあります)。

### 注意事項

* **データベース互換性**:本方法で最も重要な点です。PostgreSQL の **バージョン、文字セット、大文字小文字の感度設定** がバックアップ元と一致している必要があり、特に `schema` 名は完全に一致させてください。
* **商用プラグインの整合性**:本ソリューションが必要とする商用プラグインをすべて保有・有効化していることを確認してください。不足があると復元が中断します。一体型ソリューションが利用する商用プラグインには、バックアップマネージャー、メール管理、監査ログ、AI 社員などがあります。

## 方法 2:SQL ファイルを直接インポート(汎用、コミュニティ版に適している)

データベースを直接操作して復元する方法で、「バックアップマネージャー」プラグインに依存しないため、プロフェッショナル / エンタープライズ版でなくても利用できます。

### 特徴

* **メリット**:
  1. **エディション制限なし**:コミュニティ版を含むすべての NocoBase ユーザーで利用可能です。
  2. **高い互換性**:アプリ内の `dump` ツールに依存せず、データベースに接続できれば操作できます。
  3. **高い耐障害性**:お使いの環境にない商用プラグインが含まれていても、該当機能は有効化されないだけで他の機能には影響せず、アプリは正常に起動できます。
* **制約**:
  1. **データベース操作のスキルが必要**:`.sql` ファイルの実行など、基本的なデータベース操作ができる必要があります。
  2. **システムファイルが失われる**:**本方法ではすべてのシステムファイルが失われます**。テンプレート印刷ファイル、レコードのファイル添付、AI 社員のアバターなどです。

### 操作手順

**Step 1:クリーンなデータベースを準備**

これからインポートするデータ用に、新規の空データベース(PostgreSQL 16)を用意します。

**Step 2:`.sql` ファイルをデータベースにインポート**

ダウンロードした `nocobase_all_in_one_sql_260521.zip` を解凍し、得られた `.sql` ファイルを Step 1 のデータベースにインポートします。インポート方法は環境によって複数あります。

* **オプション A:サーバーコマンドライン経由(Docker を例に)**

  NocoBase とデータベースを Docker で稼働させている場合、`.sql` ファイルをサーバーにアップロードし、`docker exec` でインポートできます。PostgreSQL コンテナ名が `my-nocobase-db` だと仮定します:

  ```bash
  # sql ファイルをコンテナにコピー
  docker cp nocobase_all_in_one_sql_260521.sql my-nocobase-db:/tmp/
  # コンテナに入ってインポートを実行
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_all_in_one_sql_260521.sql
  ```

* **オプション B:リモートデータベースクライアント(Navicat など)経由**

  データベースがポートを公開している場合、Navicat、DBeaver、pgAdmin などの GUI クライアントから接続して操作できます:

  1. 対象データベースを右クリック
  2. 「SQL ファイルを実行」または「SQL スクリプトを実行」を選択
  3. ダウンロードした `.sql` ファイルを選んで実行

**Step 3:データベースに接続してアプリを起動**

NocoBase の起動パラメータ(環境変数 `DB_HOST`、`DB_PORT`、`DB_DATABASE`、`DB_USER`、`DB_PASSWORD` など)を、データをインポートしたばかりのデータベースに向くように設定し、NocoBase サービスを通常通り起動します。

### 注意事項

* **データベース権限**:本方法では、データベースを直接操作できるアカウントとパスワードが必要です。
* **プラグインの状態**:インポート成功後、商用プラグインのデータはデータベース上に存在しますが、対応プラグインがローカルでインストール・有効化されていなければ該当機能は表示も利用もできません。ただしアプリ自体がクラッシュすることはありません。

## よくある質問

### プロフェッショナル版でも使えますか?エラーは出ますか?

そのまま使え、エラーは出ません。Demo ではエンタープライズ版プラグイン(メール管理、監査ログ、AI 社員など)を一部使用しています。プロフェッショナル版にこれらが無い場合、該当機能の入口は表示されませんが、**システムの他のモジュールには影響しません**。例えば監査ログの入口は表示されなくなりますが、CRM、販売、チケット、プロジェクト、資産、HR などのコアモジュールは問題なく利用できます。

### 復元後はどのバージョンを選ぶべきですか?

最新の `alpha-full` イメージ(例:`nocobase/nocobase:alpha-full`)を推奨します。`full` にはデータベースクライアントなどの依存が同梱されており、ツール不足による復元失敗を防げます。

### 復元後にロゴが表示されません

公式 Demo のロゴはドメイン制限が設定されており、ローカルドメインでは読み込めません。**システム設定** から自分のロゴをアップロードしてください。

### ファイルアップロードでエラー(OSS Key エラー)が出ます

SQL 方式でインストールするとファイルアップロードで OSS 関連のエラーが出ることがあります。解決方法:**プラグイン管理 → ファイルマネージャー** に進み、**Local Storage(ローカルストレージ)** をデフォルトに設定して保存します。

### 多言語の切り替えは?

一体型ソリューションは 20 以上の言語にローカライズ済みです(`nb_demo` 名前空間)。復元後はデフォルトで中国語表示となるため、他言語に切り替えるには **システム設定 → 対象言語を有効化** してください(`ar-SA` は現在 NocoBase の表示に不具合があるため有効化を避けてください)。

### 差分アップグレードはどうすればよいですか?

現バージョンのアップグレードは全置換であり、独自カスタマイズは上書きされます。アップグレード前のバックアップを必ず取得してください。差分マイグレーション方式は計画中で、まずプロフェッショナル / エンタープライズ版から対応する予定です。コミュニティ版はマイグレーション管理プラグインがないため、現時点では対応が困難です。

ご不明な点があれば、いつでもお気軽にお問い合わせください。
