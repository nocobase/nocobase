# 添付ファイルフィールド

## はじめに

NocoBaseには、カスタムコレクションでファイルをアップロードできるように、「添付ファイル」タイプのフィールドが標準で組み込まれています。

添付ファイルフィールドは、実体としては多対多のリレーションシップフィールドであり、システムに組み込まれている「Attachments」（`attachments`）コレクションを指します。任意のコレクションで添付ファイルフィールドを作成すると、添付ファイルコレクションとの多対多リレーションシップのための中間テーブルが自動的に生成されます。アップロードされたファイルのメタデータは「Attachments」コレクションに保存され、コレクション内で参照されるファイル情報は、この中間テーブルを介して関連付けられます。

## フィールド設定

![20240512180916](https://static-docs.nocobase.com/20251031000729.png)

### MIMEタイプ制限

アップロードを許可するファイルの種類を制限するために使用します。[MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types)構文形式で記述してください。例えば、`image/*` は画像ファイルを表します。複数のタイプはカンマ（,）で区切ることができます。例として、`image/*,application/pdf` は画像タイプとPDFタイプの両方のファイルを許可します。

### ストレージエンジン

アップロードされたファイルを保存するためのストレージエンジンを選択します。空欄の場合、システムのデフォルトストレージエンジンが使用されます。
