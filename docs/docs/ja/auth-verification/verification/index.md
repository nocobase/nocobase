# 検証

:::info{title=注記}
`1.6.0-alpha.30` 以降、従来の**認証コード**機能は**検証管理**にアップグレードされました。これにより、さまざまなユーザー本人確認方法の管理と連携が可能になります。ユーザーは、対応する検証方法を紐付けることで、必要な場面で本人確認を行うことができます。この機能は、バージョン `1.7.0` から安定版としてサポートされる予定です。
:::



## はじめに

**検証管理センターは、さまざまなユーザー本人確認方法の管理と連携をサポートしています。** 例えば、以下のようなものがあります。

- SMS認証コード – 検証プラグインによってデフォルトで提供されます。参照：[検証：SMS](./sms)
- TOTPオーセンティケーター – 参照：[検証：TOTPオーセンティケーター](../verification-totp/)

開発者は、プラグインとして他の検証タイプを拡張することも可能です。参照：[検証タイプの拡張](./dev/type)

**ユーザーは、対応する検証方法を紐付けることで、必要な場面で本人確認を行うことができます。** 例えば、以下のようなものがあります。

- SMS認証コードによるログイン – 参照：[認証：SMS](../auth-sms/index.md)
- 二要素認証 (2FA) – 参照：[二要素認証 (2FA)](../2fa/)
- リスク操作時の二次検証 – 将来的にサポート予定

開発者は、拡張プラグインとして、本人確認を他の必要なシナリオに連携させることも可能です。参照：[検証シナリオの拡張](./dev/scene)

**検証モジュールとユーザー認証モジュールの違いと関連性：** ユーザー認証モジュールは、主にユーザーログイン時の認証を担当します。SMSログインや二要素認証などのプロセスは、検証モジュールが提供する検証機能に依存します。一方、検証モジュールは、さまざまなリスク操作における本人確認を担当し、ユーザーログインもそのリスク操作シナリオの一つです。
