## 新規登録の許可

新規登録が許可されている場合、ログインページにアカウント作成のリンクが表示され、登録ページへ移動できます。

![](https://static-docs.nocobase.com/78903930d4b47aaf75cf94c55dd3596e.png)

登録ページ

![](https://static-docs.nocobase.com/ac3c3ab42df28cb7c6dc70b24e99e7f7.png)

新規登録が許可されていない場合、ログインページにアカウント作成のリンクは表示されません。

![](https://static-docs.nocobase.com/8d5e3b6df9991bfc1c2e095a3745121.png)

新規登録が許可されていない場合、登録ページにアクセスすることはできません。

![](https://static-docs.nocobase.com/09325c4b07e09f88f80a14dff8430556.png)

## パスワードを忘れた場合<Badge>v1.8.0+</Badge>

「パスワードを忘れた場合」機能を利用すると、ユーザーはパスワードを忘れてしまった際に、メール認証を通じてパスワードを再設定できます。

### 管理者設定

1.  **パスワード忘れ機能を有効にする**

    「設定」>「認証」>「パスワードを忘れた場合」タブで、「パスワード忘れ機能を有効にする」チェックボックスにチェックを入れます。

    ![20250423071957_rec_](https://static-docs.nocobase.com/20250423071957_rec_.gif)

2.  **通知チャネルを設定する**

    メール通知チャネルを1つ選択します（現在、メールのみをサポートしています）。利用可能な通知チャネルがない場合は、まず追加してください。

    ![20250423072225_rec_](https://static-docs.nocobase.com/20250423072225_rec_.gif)

3.  **パスワードリセットメールを設定する**

    メールの件名と内容をカスタマイズします。HTMLまたはプレーンテキスト形式をサポートしています。以下の変数が利用できます。
    - 現在のユーザー
    - システム設定
    - パスワードリセットリンク
    - リセットリンクの有効期限（分）

    ![20250427170047](https://static-docs.nocobase.com/20250427170047.png)

4.  **リセットリンクの有効期限を設定する**

    リセットリンクの有効期間を分単位で設定します。デフォルトは120分です。

    ![20250423073557](https://static-docs.nocobase.com/20250423073557.png)

### ユーザーの操作手順

1.  **パスワードリセットをリクエストする**

    ログインページで「パスワードを忘れた場合」リンクをクリックすると（この機能は管理者によって事前に有効化されている必要があります）、パスワード忘れページに移動します。

    ![20250421103458_rec_](https://static-docs.nocobase.com/20250421103458_rec_.gif)

    登録済みのメールアドレスを入力し、「リセットメールを送信」ボタンをクリックします。

    ![20250421113442_rec_](https://static-docs.nocobase.com/20250421113442_rec_.gif)

2.  **パスワードをリセットする**

    ユーザーはリセットリンクを含むメールを受信します。リンクをクリックすると、新しいパスワードを設定するページが開きます。

    ![20250421113748](https://static-docs.nocobase.com/20250421113748.png)

    設定が完了すると、ユーザーは新しいパスワードでシステムにログインできます。

### 注意事項

-   リセットリンクには時間制限があり、デフォルトでは生成後120分間有効です（管理者によって設定可能です）。
-   リンクは一度しか使用できず、使用後はすぐに無効になります。
-   ユーザーがリセットメールを受信しない場合は、メールアドレスが正しいか確認するか、迷惑メールフォルダをご確認ください。
-   管理者は、リセットメールが確実に送信されるよう、メールサーバーの設定が正しいことを確認する必要があります。
