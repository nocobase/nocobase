# توسيع نوع المصادقة

## نظرة عامة

يدعم NocoBase توسيع أنواع مصادقة المستخدم حسب الحاجة. تنقسم مصادقة المستخدم بشكل عام إلى نوعين: الأول هو تحديد هوية المستخدم داخل تطبيق NocoBase نفسه، مثل تسجيل الدخول بكلمة المرور وتسجيل الدخول عبر SMS وما إلى ذلك؛ والثاني هو أن تقوم خدمات طرف ثالث بتحديد هوية المستخدم وإخطار تطبيق NocoBase بالنتيجة من خلال ردود الاتصال، مثل OIDC وSAML وطرق المصادقة الأخرى. عملية المصادقة لهذين النوعين المختلفين من طرق المصادقة في NocoBase هي أساسًا كما يلي:

### لا تتطلب ردود اتصال من طرف ثالث

1. يستخدم العميل NocoBase SDK لاستدعاء واجهة تسجيل الدخول `api.auth.signIn()`، وطلب واجهة تسجيل الدخول `auth:signIn`، مع حمل معرف أداة المصادقة الحالية من خلال ترويسة الطلب `X-Authenticator` إلى الخلفية.
2. تقوم واجهة `auth:signIn` بالتوجيه إلى نوع المصادقة المقابل بناءً على معرف أداة المصادقة في ترويسة الطلب، وتقوم طريقة `validate` في فئة المصادقة المسجلة لنوع المصادقة هذا بالمعالجة المنطقية المقابلة.
3. يسترد العميل معلومات المستخدم ورمز المصادقة من استجابة واجهة `auth:signIn`، ويحفظ الرمز في Local Storage، ويكمل تسجيل الدخول. يتم التعامل مع هذه الخطوة تلقائيًا داخليًا بواسطة SDK.

<img src="https://static-docs.nocobase.com/202404211852848.png"/>

### تعتمد على ردود اتصال من طرف ثالث

1. يحصل العميل على رابط تسجيل الدخول للطرف الثالث من خلال واجهته المسجلة الخاصة (مثل `auth:getAuthUrl`)، ويحمل معلومات مثل اسم التطبيق ومعرف أداة المصادقة وفقًا للبروتوكول.
2. إعادة التوجيه إلى رابط الطرف الثالث لإتمام تسجيل الدخول. تستدعي خدمة الطرف الثالث واجهة رد الاتصال لتطبيق NocoBase (التي تحتاج إلى تسجيلها بنفسك، مثل `auth:redirect`)، وتعيد نتيجة المصادقة، وتعيد معلومات مثل اسم التطبيق ومعرف أداة المصادقة.
3. في طريقة واجهة رد الاتصال، قم بتحليل المعاملات للحصول على معرف أداة المصادقة، واحصل على فئة المصادقة المقابلة من خلال `AuthManager`، واستدع طريقة `auth.signIn()` بنشاط. ستستدعي طريقة `auth.signIn()` طريقة `validate()` للتعامل مع منطق المصادقة.
4. بعد أن تحصل طريقة رد الاتصال على رمز المصادقة، تعيد التوجيه إلى صفحة الواجهة الأمامية برمز حالة 302، وتحمل `token` ومعرف أداة المصادقة في معاملات URL، `?authenticator=xxx&token=yyy`.

<img src="https://static-docs.nocobase.com/202404211852377.png"/>

بعد ذلك، سنناقش كيفية تسجيل واجهات الخادم وواجهات المستخدم من جهة العميل.

## الخادم

### واجهة المصادقة

توفر نواة NocoBase التسجيل والإدارة لتوسيع أنواع المصادقة. تتطلب معالجة المنطق الأساسي لإضافة تسجيل الدخول الموسعة وراثة الفئة المجردة `Auth` من النواة وتنفيذ الواجهات القياسية المقابلة.
للاطلاع على API الكامل، راجع [Auth](/api/auth/auth).

```typescript
import { Auth } from '@nocobase/auth';

class CustomAuth extends Auth {
  set user(user) {}
  get user() {}

  async check() {}
  async signIn() {}
}
```

تسجل النواة أيضًا عمليات الموارد الأساسية المتعلقة بمصادقة المستخدم.

| API            | الوصف                    |
| -------------- | ------------------------ |
| `auth:check`   | التحقق من تسجيل دخول المستخدم |
| `auth:signIn`  | تسجيل الدخول             |
| `auth:signUp`  | التسجيل                  |
| `auth:signOut` | تسجيل الخروج             |

في معظم الحالات، يمكن لنوع مصادقة المستخدم الموسع أيضًا استخدام منطق مصادقة JWT الموجود لتوليد بيانات الاعتماد للمستخدم للوصول إلى API. قامت فئة `BaseAuth` في النواة بالتنفيذ الأساسي للفئة المجردة `Auth`، راجع [BaseAuth](../../../api/auth/base-auth.md). يمكن للإضافات أن ترث مباشرةً من فئة `BaseAuth` لإعادة استخدام جزء من كود المنطق وتقليل تكاليف التطوير.

```javascript
import { BaseAuth } from '@nocobase/auth';

class CustomAuth extends BaseAuth {
  constructor(config: AuthConfig) {
    // تعيين مجموعة المستخدمين
    const userCollection = config.ctx.db.getCollection('users');
    super({ ...config, userCollection });
  }

  // تنفيذ منطق مصادقة المستخدم
  async validate() {}
}
```

### بيانات المستخدم

عند تنفيذ منطق مصادقة المستخدم، يتضمن ذلك عادةً التعامل مع بيانات المستخدم. في تطبيق NocoBase، يتم تعريف المجموعات ذات الصلة افتراضيًا على النحو التالي:

| المجموعات | الوصف | الإضافة |
| --------- | ----- | ------- |
| `users` | تخزين معلومات المستخدم، مثل البريد الإلكتروني والاسم المستعار وكلمة المرور | [إضافة المستخدمين (`@nocobase/plugin-users`)](/users-permissions/user) |
| `authenticators` | تخزين معلومات أداة المصادقة (كيان نوع المصادقة)، المقابلة لنوع المصادقة والتهيئة | إضافة مصادقة المستخدم (`@nocobase/plugin-auth`) |
| `usersAuthenticators` | ربط المستخدمين وأدوات المصادقة، وحفظ معلومات المستخدم تحت أداة المصادقة المقابلة | إضافة مصادقة المستخدم (`@nocobase/plugin-auth`) |

بشكل عام، تستخدم طرق تسجيل الدخول الموسعة `users` و`usersAuthenticators` لتخزين بيانات المستخدم المقابلة. فقط في حالات خاصة تحتاج إلى إضافة مجموعة جديدة بنفسك.

الحقول الرئيسية لـ `usersAuthenticators` هي:

| الحقل | الوصف |
| ----- | ----- |
| `uuid` | معرف فريد لهذا النوع من المصادقة، مثل رقم الهاتف أو معرف مستخدم خدمة طرف ثالث |
| `meta` | حقل JSON، معلومات أخرى يجب حفظها |
| `userId` | معرف المستخدم |
| `authenticator` | اسم أداة المصادقة (المعرف الفريد) |

لعمليات استعلام المستخدم وإنشائه، يغلف نموذج بيانات `authenticators` المسمى `AuthModel` أيضًا عدة طرق يمكن استخدامها في فئة `CustomAuth` عبر `this.authenticator[methodName]`. للاطلاع على API الكامل، راجع [AuthModel](./api#authmodel).

```ts
import { AuthModel } from '@nocobase/plugin-auth';

class CustomAuth extends BaseAuth {
  async validate() {
    // ...
    const authenticator = this.authenticator as AuthModel;
    this.authenticator.findUser(); // استعلام المستخدم
    this.authenticator.newUser(); // إنشاء مستخدم جديد
    this.authenticator.findOrCreateUser(); // استعلام أو إنشاء مستخدم جديد
    // ...
  }
}
```

### تسجيل نوع المصادقة

يجب تسجيل طريقة المصادقة الموسعة مع وحدة إدارة المصادقة.

```javascript
class CustomAuthPlugin extends Plugin {
  async load() {
    this.app.authManager.registerTypes('custom-auth-type', {
      auth: CustomAuth,
    });
  }
}
```

## العميل

يتم تسجيل واجهة مستخدم العميل من خلال الواجهة `registerType` التي توفرها إضافة مصادقة المستخدم من جهة العميل:

```ts
import AuthPlugin from '@nocobase/plugin-auth/client';

class CustomAuthPlugin extends Plugin {
  async load() {
    const auth = this.app.pm.get(AuthPlugin);
    auth.registerType('custom-auth-type', {
      components: {
        SignInForm,      // نموذج تسجيل الدخول
        SignInButton,    // زر تسجيل الدخول (طرف ثالث)، بديل لنموذج تسجيل الدخول
        SignUpForm,      // نموذج التسجيل
        AdminSettingsForm, // نموذج إعدادات المسؤول
      },
    });
  }
}
```

### طلب واجهات API

لبدء طلبات لواجهات متعلقة بمصادقة المستخدم من جهة العميل، يمكنك استخدام SDK الذي يوفره NocoBase.

```ts
import { useAPIClient } from '@nocobase/client';

// الاستخدام في المكوّن
const api = useAPIClient();
api.auth.signIn(data, authenticator);
```

للاطلاع على مراجع API التفصيلية، راجع [@nocobase/sdk - Auth](/api/sdk/auth).
