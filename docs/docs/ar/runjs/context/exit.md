# ctx.exit()

ينهي تنفيذ تدفق الأحداث (event flow) الحالي؛ ولن يتم تشغيل الخطوات اللاحقة. يُستخدم عادةً عندما لا تتحقق شروط العمل، أو عند إلغاء المستخدم للعملية، أو عند حدوث خطأ غير قابل للاسترداد.

## حالات الاستخدام

تُستخدم `ctx.exit()` بشكل عام في السياقات التالية التي يمكن فيها تنفيذ كود JS:

| السيناريو | الوصف |
|------|------|
| **تدفق الأحداث** | في تدفقات الأحداث الناتجة عن تقديم النماذج، أو نقرات الأزرار، وما إلى ذلك، حيث يتم إنهاء الخطوات اللاحقة عندما لا تتحقق الشروط. |
| **قواعد الربط** | في ربط الحقول، وربط التصفية، وما إلى ذلك، حيث يتم إنهاء تدفق الأحداث الحالي عند فشل التحقق من الصحة أو عند الحاجة إلى تخطي التنفيذ. |
| **أحداث الإجراءات** | في الإجراءات المخصصة (مثل تأكيد الحذف، والتحقق المسبق قبل الحفظ)، حيث يتم الخروج عند إلغاء المستخدم أو فشل التحقق من الصحة. |

> الفرق عن `ctx.exitAll()`: تنهي `ctx.exit()` تدفق الأحداث الحالي فقط؛ ولا تتأثر تدفقات الأحداث الأخرى ضمن نفس الحدث. بينما تنهي `ctx.exitAll()` تدفق الأحداث الحالي بالإضافة إلى أي تدفقات أحداث لاحقة ضمن نفس الحدث لم يتم تنفيذها بعد.

## تعريف النوع

```ts
exit(): never;
```

يؤدي استدعاء `ctx.exit()` إلى إلقاء استثناء داخلي `FlowExitException` يتم التقاطه بواسطة محرك التدفق (FlowEngine) لإيقاف تنفيذ تدفق الأحداث الحالي. بمجرد استدعائه، لن يتم تنفيذ البيانات المتبقية في كود JS الحالي.

## المقارنة مع ctx.exitAll()

| الطريقة | نطاق التأثير |
|------|----------|
| `ctx.exit()` | ينهي تدفق الأحداث الحالي فقط؛ ولا تتأثر تدفقات الأحداث اللاحقة. |
| `ctx.exitAll()` | ينهي تدفق الأحداث الحالي ويوقف تدفقات الأحداث اللاحقة ضمن نفس الحدث والمعدة **للتنفيذ المتسلسل**. |

## أمثلة

### الخروج عند إلغاء المستخدم

```ts
// في نافذة التأكيد المنبثقة، إذا نقر المستخدم على إلغاء، يتم إنهاء تدفق الأحداث
if (!confirmed) {
  ctx.message.info('تم إلغاء العملية');
  ctx.exit();
}
```

### الخروج عند فشل التحقق من صحة المعلمات

```ts
// عرض تنبيه والإنهاء عند فشل التحقق
if (!params.value || params.value.length < 3) {
  ctx.message.error('المعلمات غير صالحة، يجب أن يكون الطول 3 على الأقل');
  ctx.exit();
}
```

### الخروج عندما لا تتحقق شروط العمل

```ts
// الإنهاء إذا لم تتحقق الشروط؛ لن يتم تنفيذ الخطوات اللاحقة
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'يمكن تقديم المسودات فقط' });
  ctx.exit();
}
```

### الاختيار بين ctx.exit() و ctx.exitAll()

```ts
// الحاجة للخروج من تدفق الأحداث الحالي فقط ← استخدم ctx.exit()
if (!params.valid) {
  ctx.message.error('المعلمات غير صالحة');
  ctx.exit();  // لا تتأثر تدفقات الأحداث الأخرى
}

// الحاجة لإنهاء جميع تدفقات الأحداث اللاحقة ضمن الحدث الحالي ← استخدم ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'صلاحيات غير كافية' });
  ctx.exitAll();  // يتم إنهاء تدفق الأحداث الحالي وتدفقات الأحداث اللاحقة معاً
}
```

### الخروج بناءً على اختيار المستخدم بعد تأكيد النافذة المنبثقة

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'تأكيد الحذف',
  content: 'لا يمكن التراجع عن هذا الإجراء بعد التنفيذ، هل تريد الاستمرار؟',
});
if (!ok) {
  ctx.message.info('تم الإلغاء');
  ctx.exit();
}
```

## ملاحظات

- بعد استدعاء `ctx.exit()`، لن يتم تنفيذ الكود اللاحق في ملف JS الحالي؛ يُنصح بتوضيح السبب للمستخدم عبر `ctx.message` أو `ctx.notification` أو نافذة منبثقة قبل الاستدعاء.
- عادةً لا توجد حاجة لالتقاط `FlowExitException` في كود العمل؛ اترك محرك التدفق (FlowEngine) يتعامل مع ذلك.
- إذا كنت بحاجة إلى إنهاء جميع تدفقات الأحداث اللاحقة ضمن الحدث الحالي، استخدم `ctx.exitAll()`.
