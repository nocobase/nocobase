# ctx.logger

غلاف للسجلات (logging wrapper) يعتمد على [pino](https://github.com/pinojs/pino)، يوفر سجلات JSON مهيكلة وعالية الأداء. يوصى باستخدام `ctx.logger` بدلاً من `console` لتسهيل جمع السجلات وتحليلها.

## حالات الاستخدام

يمكن استخدام `ctx.logger` في جميع سيناريوهات RunJS لأغراض تصحيح الأخطاء (debugging)، وتتبع الأخطاء، وتحليل الأداء، وما إلى ذلك.

## تعريف النوع

```ts
logger: pino.Logger;
```

`ctx.logger` هو عبارة عن `engine.logger.child({ module: 'flow-engine' })`، أي أنه سجل فرعي (child logger) من pino مع سياق `module`.

## مستويات السجل

يدعم pino المستويات التالية (من الأعلى إلى الأدنى):

| المستوى | الطريقة | الوصف |
|------|------|------|
| `fatal` | `ctx.logger.fatal()` | خطأ فادح، يؤدي عادةً إلى إنهاء العملية |
| `error` | `ctx.logger.error()` | خطأ، يشير إلى فشل الطلب أو العملية |
| `warn` | `ctx.logger.warn()` | تحذير، يشير إلى مخاطر محتملة أو حالات غير طبيعية |
| `info` | `ctx.logger.info()` | معلومات التشغيل العامة |
| `debug` | `ctx.logger.debug()` | معلومات تصحيح الأخطاء، تُستخدم أثناء التطوير |
| `trace` | `ctx.logger.trace()` | تتبع تفصيلي، يُستخدم للتشخيص العميق |

## الطريقة الموصى بها

يوصى باستخدام صيغة `level(msg, meta)`: الرسالة أولاً، ثم كائن البيانات الوصفية (metadata) الاختياري.

```ts
ctx.logger.info('تم تحميل الكتلة بنجاح');
ctx.logger.info('تمت العملية بنجاح', { recordId: 456 });
ctx.logger.warn('تحذير في الأداء', { duration: 5000 });
ctx.logger.error('فشلت العملية', { userId: 123, action: 'create' });
ctx.logger.error('فشل الطلب', { err });
```

يدعم pino أيضاً `level(meta, msg)` (الكائن أولاً) أو `level({ msg, ...meta })` (كائن واحد)، ويمكن استخدامها حسب الحاجة.

## أمثلة

### الاستخدام الأساسي

```ts
ctx.logger.info('تم تحميل الكتلة بنجاح');
ctx.logger.warn('فشل الطلب، يتم استخدام التخزين المؤقت', { err });
ctx.logger.debug('جاري الحفظ...', { recordId: ctx.record?.id });
```

### إنشاء سجل فرعي باستخدام child()

```ts
// إنشاء سجل فرعي مع سياق للمنطق الحالي
const log = ctx.logger.child({ scope: 'myBlock' });
log.info('تنفيذ الخطوة 1');
log.debug('تنفيذ الخطوة 2', { step: 2 });
```

### العلاقة مع console

يوصى باستخدام `ctx.logger` مباشرة للحصول على سجلات JSON مهيكلة. إذا كنت معتاداً على استخدام `console`، فإن المقابلات هي: `console.log` ← `ctx.logger.info`، و `console.error` ← `ctx.logger.error`، و `console.warn` ← `ctx.logger.warn`.

## تنسيق السجل

يخرج pino بتنسيق JSON مهيكل، حيث يحتوي كل سجل على:

- `level`: مستوى السجل (رقمي)
- `time`: الطابع الزمني (بالملي ثانية)
- `msg`: رسالة السجل
- `module`: ثابت كـ `flow-engine`
- حقول مخصصة أخرى (يتم تمريرها عبر الكائنات)

## ملاحظات

- السجلات هي JSON مهيكل، مما يسهل جمعها والبحث فيها وتحليلها.
- السجلات الفرعية التي يتم إنشاؤها عبر `child()` تتبع أيضاً توصية `level(msg, meta)`.
- قد تستخدم بعض بيئات التشغيل (مثل سير العمل) طرقاً مختلفة لإخراج السجلات.
