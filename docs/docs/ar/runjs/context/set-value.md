# ctx.setValue()

يقوم بتعيين قيمة الحقل الحالي في سيناريوهات الحقول القابلة للتحرير مثل JSField و JSItem. وبالتكامل مع `ctx.getValue()`، فإنه يتيح الربط ثنائي الاتجاه (two-way binding) مع النموذج.

## حالات الاستخدام

| السيناريو | الوصف |
|------|------|
| **JSField** | كتابة القيم التي يختارها المستخدم أو القيم المحسوبة في الحقول المخصصة القابلة للتحرير. |
| **JSItem** | تحديث قيمة الخلية الحالية في العناصر القابلة للتحرير ضمن الجداول أو الجداول الفرعية. |
| **JSColumn** | تحديث قيمة الحقل للصف المقابل بناءً على المنطق البرمجي أثناء رندر (rendering) أعمدة الجدول. |

> **ملاحظة**: تتوفر `ctx.setValue(v)` فقط في سياقات RunJS المرتبطة بنموذج (form binding)؛ ولا تتوفر هذه الطريقة في السيناريوهات التي لا تحتوي على ربط للحقول مثل سير العمل (Workflow)، أو قواعد الربط (linkage rules)، أو JSBlock. يُنصح باستخدام التسلسل الاختياري (optional chaining) قبل الاستخدام: `ctx.setValue?.(value)`.

## تعريف النوع

```ts
setValue<T = any>(value: T): void;
```

- **المعاملات**: `value` هي قيمة الحقل المراد كتابتها، ويتم تحديد النوع حسب نوع عنصر النموذج الخاص بالحقل.

## السلوك

- تقوم `ctx.setValue(v)` بتحديث قيمة الحقل الحالي في Ant Design Form، وتُحفز منطق الربط والتحقق (validation) المتعلق بالنموذج.
- إذا لم ينتهِ رندر (rendering) النموذج بعد أو لم يتم تسجيل الحقل، فقد يكون الاستدعاء غير فعال. يُنصح باستخدام `ctx.getValue()` للتأكد من نتيجة الكتابة.

## أمثلة

### الربط ثنائي الاتجاه مع getValue

```tsx
const { Input } = ctx.libs.antd;

const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```

### تعيين القيم الافتراضية بناءً على الشروط

```ts
const status = ctx.getValue();
if (status == null || status === '') {
  ctx.setValue?.('draft');
}
```

### الكتابة في الحقل الحالي عند الارتباط بحقول أخرى

```ts
// تحديث الحقل الحالي بشكل متزامن عند تغيير حقل آخر
const otherValue = ctx.form?.getFieldValue('type');
if (otherValue === 'custom') {
  ctx.setValue?.({ label: 'مخصص', value: 'custom' });
}
```

## ملاحظات

- في الحقول غير القابلة للتحرير (مثل وضع التفاصيل في JSField أو JSBlock)، قد تكون `ctx.setValue` غير معرفة (`undefined`)؛ لذا يُنصح باستخدام `ctx.setValue?.(value)` لتجنب الأخطاء.
- عند تعيين قيم لحقول الارتباط (M2O، O2M، إلخ)، يجب تمرير هيكل يتوافق مع نوع الحقل (مثل `{ id, [titleField]: label }`)، وذلك حسب إعدادات الحقل المحددة.
