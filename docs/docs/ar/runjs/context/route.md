# ctx.route

معلومات مطابقة المسار الحالية، والتي تتوافق مع مفهوم `route` في React Router. تُستخدم للحصول على تكوين المسار المطابق الحالي، والمعلمات (parameters)، وما إلى ذلك. تُستخدم عادةً بالاقتران مع `ctx.router` و `ctx.location`.

## حالات الاستخدام

| السيناريو | الوصف |
|------|------|
| **JSBlock / JSField** | تنفيذ التصيير الشرطي أو عرض معرف الصفحة الحالي بناءً على `route.pathname` أو `route.params`. |
| **قواعد الربط / محرك سير العمل** | قراءة معلمات المسار (مثل `params.name`) لتنفيذ تفرع منطقي أو تمريرها إلى المكونات الفرعية. |
| **التنقل في العرض** | مقارنة `ctx.route.pathname` داخلياً مع المسار المستهدف لتحديد ما إذا كان سيتم تشغيل `ctx.router.navigate`. |

> **ملاحظة:** `ctx.route` متاح فقط في بيئات RunJS التي تحتوي على سياق توجيه (مثل JSBlock داخل الصفحة، صفحات سير العمل، إلخ)؛ قد يكون فارغاً في السياقات الخلفية البحتة أو التي لا تحتوي على توجيه (مثل سير العمل في الخلفية).

## تعريف النوع

```ts
type RouteOptions = {
  name?: string;   // المعرف الفريد للمسار
  path?: string;   // قالب المسار (مثل /admin/:name)
  params?: Record<string, any>;  // معلمات المسار (مثل { name: 'users' })
  pathname?: string;  // المسار الكامل للمسار الحالي (مثل /admin/users)
};
```

## الحقول الشائعة

| الحقل | النوع | الوصف |
|------|------|------|
| `pathname` | `string` | المسار الكامل للمسار الحالي، وهو متوافق مع `ctx.location.pathname`. |
| `params` | `Record<string, any>` | المعلمات الديناميكية التي تم تحليلها من قالب المسار، مثل `{ name: 'users' }`. |
| `path` | `string` | قالب المسار، مثل `/admin/:name`. |
| `name` | `string` | المعرف الفريد للمسار، يُستخدم غالباً في سيناريوهات علامات التبويب المتعددة أو طرق العرض المتعددة. |

## العلاقة مع ctx.router و ctx.location

| الغرض | الاستخدام الموصى به |
|------|----------|
| **قراءة المسار الحالي** | `ctx.route.pathname` أو `ctx.location.pathname`؛ كلاهما متطابق عند المطابقة. |
| **قراءة معلمات المسار** | `ctx.route.params`؛ مثل `params.name` الذي يمثل معرف الصفحة الحالي (UID). |
| **التنقل (Navigation)** | `ctx.router.navigate(path)` |
| **قراءة معلمات الاستعلام والحالة** | `ctx.location.search` و `ctx.location.state` |

يركز `ctx.route` على "تكوين المسار المطابق"، بينما يركز `ctx.location` على "موقع URL الحالي". يعمل الاثنان معاً لوصف حالة التوجيه الحالية بشكل كامل.

## أمثلة

### قراءة pathname

```ts
// عرض المسار الحالي
ctx.message.info('الصفحة الحالية: ' + ctx.route.pathname);
```

### التفرع بناءً على params

```ts
// params.name عادة ما يكون معرف الصفحة الحالي (مثل معرف صفحة سير العمل)
if (ctx.route.params?.name === 'users') {
  // تنفيذ منطق محدد في صفحة إدارة المستخدمين
}
```

### العرض في صفحة سير العمل (Flow)

```tsx
<div>
  <h1>الصفحة الحالية - {ctx.route.pathname}</h1>
  <p>معرف المسار: {ctx.route.params?.name}</p>
</div>
```

- [ctx.router](./router.md): التنقل عبر المسارات. عند تغيير المسار باستخدام `ctx.router.navigate()`، سيتم تحديث `ctx.route` تبعاً لذلك.
- [ctx.location](./location.md): موقع URL الحالي (pathname، search، hash، state)، يُستخدم بالاقتران مع `ctx.route`.
