# ctx.getValue()

في سيناريوهات الحقول القابلة للتحرير مثل JSField و JSItem، يُستخدم هذا التابع للحصول على أحدث قيمة للحقل الحالي. وبالاقتران مع `ctx.setValue(v)`، فإنه يتيح الربط ثنائي الاتجاه (two-way binding) مع النموذج.

## سيناريوهات الاستخدام

| السيناريو | الوصف |
|------|------|
| **JSField** | قراءة مدخلات المستخدم أو قيمة النموذج الحالية في الحقول المخصصة القابلة للتحرير. |
| **JSItem** | قراءة قيمة الخلية الحالية في العناصر القابلة للتحرير في الجداول أو الجداول الفرعية. |
| **JSColumn** | قراءة قيمة الحقل للصف المقابل أثناء تصيير (rendering) أعمدة الجدول. |

> **ملاحظة**: `ctx.getValue()` متاح فقط في سياقات RunJS المرتبطة بنموذج؛ ولا يتوفر هذا التابع في السيناريوهات التي لا تحتوي على ربط للحقول، مثل سير العمل (workflows) أو قواعد الربط.

## تعريف النوع

```ts
getValue<T = any>(): T | undefined;
```

- **القيمة المرجعة**: قيمة الحقل الحالية، ويتم تحديد نوعها حسب نوع عنصر النموذج الخاص بالحقل؛ قد تكون `undefined` إذا لم يتم تسجيل الحقل أو لم يتم ملؤه.

## ترتيب استرداد القيم

يسترد `ctx.getValue()` القيم بالترتيب التالي:

1. **حالة النموذج**: الأولوية للقراءة من الحالة الحالية لـ Ant Design Form.
2. **القيمة الاحتياطية (Fallback)**: إذا لم يكن الحقل موجوداً في النموذج، فإنه يعود إلى القيمة الأولية للحقل أو الخصائص (props).

> إذا لم ينتهِ تصيير النموذج بعد أو لم يتم تسجيل الحقل، فقد يتم إرجاع `undefined`.

## أمثلة

### التصيير بناءً على القيمة الحالية

```ts
const current = ctx.getValue();
if (current == null || current === '') {
  ctx.render(<span>يرجى إدخال المحتوى أولاً</span>);
} else {
  ctx.render(<span>القيمة الحالية: {current}</span>);
}
```

### الربط ثنائي الاتجاه مع setValue

```tsx
const { Input } = ctx.libs.antd;

// قراءة القيمة الحالية كقيمة افتراضية
const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```
