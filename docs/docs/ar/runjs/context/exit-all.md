# ctx.exitAll()

يُنهي تدفق الحدث الحالي وجميع تدفقات الأحداث اللاحقة التي تم إطلاقها في نفس عملية جدولة الأحداث. يُستخدم عادةً عند الحاجة إلى إيقاف جميع تدفقات الأحداث تحت الحدث الحالي فوراً بسبب خطأ عام أو فشل في التحقق من الصلاحيات.

## حالات الاستخدام

يُستخدم `ctx.exitAll()` بشكل عام في سياقات تنفيذ JS عندما يكون من الضروري **إيقاف تدفق الحدث الحالي وتدفقات الأحداث اللاحقة الناتجة عن هذا الحدث في آن واحد**:

| السيناريو | الوصف |
|------|------|
| **سير العمل** | فشل التحقق من تدفق الحدث الرئيسي (مثل عدم كفاية الصلاحيات)، مما يتطلب إنهاء التدفق الرئيسي وأي تدفقات لاحقة لم يتم تنفيذها بعد تحت نفس الحدث. |
| **قواعد الارتباط** | عند فشل التحقق من الارتباط، يجب إنهاء الارتباط الحالي والارتباطات اللاحقة الناتجة عن نفس الحدث. |
| **أحداث العمليات** | فشل التحقق المسبق للعملية (مثل التحقق من الصلاحيات قبل الحذف)، مما يتطلب منع العملية الرئيسية والخطوات اللاحقة. |

> الفرق عن `ctx.exit()`: يقوم `ctx.exit()` بإنهاء تدفق الحدث الحالي فقط؛ بينما يقوم `ctx.exitAll()` بإنهاء تدفق الحدث الحالي وأي تدفقات أحداث لاحقة **لم يتم تنفيذها** بعد في نفس جدولة الأحداث.

## تعريف النوع

```ts
exitAll(): never;
```

يؤدي استدعاء `ctx.exitAll()` إلى إلقاء استثناء داخلي `FlowExitAllException` يلتقطه محرك سير العمل لإيقاف مثيل تدفق الحدث الحالي وتدفقات الأحداث اللاحقة تحت نفس الحدث. بمجرد استدعائه، لن يتم تنفيذ العبارات المتبقية في كود JS الحالي.

## المقارنة مع ctx.exit()

| الطريقة | نطاق التأثير |
|------|----------|
| `ctx.exit()` | ينهي تدفق الحدث الحالي فقط، ولا تتأثر تدفقات الأحداث اللاحقة. |
| `ctx.exitAll()` | ينهي تدفق الحدث الحالي ويوقف تدفقات الأحداث اللاحقة التي يتم تنفيذها **بالتتابع** تحت نفس الحدث. |

## شرح وضع التنفيذ

- **التنفيذ المتسلسل (sequential)**: يتم تنفيذ تدفقات الأحداث تحت نفس الحدث بالترتيب؛ بعد استدعاء `ctx.exitAll()` في أي تدفق، لن يتم تنفيذ التدفقات اللاحقة.
- **التنفيذ المتوازي (parallel)**: يتم تنفيذ تدفقات الأحداث تحت نفس الحدث بشكل متوازٍ؛ استدعاء `ctx.exitAll()` في أحد التدفقات لن يقطع التدفقات الأخرى التي بدأت بالفعل (لأنها مستقلة).

## أمثلة

### إنهاء جميع تدفقات الأحداث عند فشل التحقق من الصلاحيات

```ts
// إيقاف تدفق الحدث الرئيسي والتدفقات اللاحقة عند عدم كفاية الصلاحيات
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'لا توجد صلاحية للعملية' });
  ctx.exitAll();
}
```

### الإنهاء عند فشل التحقق المسبق العام

```ts
// مثال: إذا تبين أن البيانات المرتبطة غير قابلة للحذف قبل عملية الحذف، يتم منع تدفق الحدث الرئيسي والعمليات اللاحقة
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('لا يمكن الحذف: توجد بيانات مرتبطة');
  ctx.exitAll();
}
```

### الاختيار بين ctx.exit() و ctx.exitAll()

```ts
// الحاجة للخروج من تدفق الحدث الحالي فقط -> استخدم ctx.exit()
if (!params.valid) {
  ctx.message.error('المعلمات غير صالحة');
  ctx.exit();  // لن تتأثر تدفقات الأحداث اللاحقة
}

// الحاجة لإنهاء جميع تدفقات الأحداث اللاحقة تحت الحدث الحالي -> استخدم ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'صلاحيات غير كافية' });
  ctx.exitAll();  // سيتم إنهاء تدفق الحدث الرئيسي والتدفقات اللاحقة تحت نفس الحدث معاً
}
```

### التنبيه قبل الإنهاء

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('يرجى تصحيح الأخطاء في النموذج أولاً');
  ctx.exitAll();
}
```

## ملاحظات

- بعد استدعاء `ctx.exitAll()`، لن يتم تنفيذ الكود اللاحق في JS الحالي؛ يُنصح بتوضيح السبب للمستخدم عبر `ctx.message` أو `ctx.notification` أو نافذة منبثقة قبل الاستدعاء.
- عادةً لا يحتاج كود الأعمال إلى التقاط `FlowExitAllException`؛ اترك محرك سير العمل يتعامل معه.
- إذا كنت بحاجة فقط لإيقاف تدفق الحدث الحالي دون التأثير على التدفقات اللاحقة، استخدم `ctx.exit()`.
- في الوضع المتوازي، ينهي `ctx.exitAll()` تدفق الحدث الحالي فقط ولا يقطع التدفقات المتوازية الأخرى التي بدأت بالفعل.
