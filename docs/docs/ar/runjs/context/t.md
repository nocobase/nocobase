# ctx.t()

دالة مختصرة للتدويل (i18n) تُستخدم في RunJS لترجمة النصوص بناءً على إعدادات اللغة في السياق الحالي. وهي مناسبة لتدويل النصوص المضمنة مثل الأزرار، والعناوين، والتنبيهات.

## حالات الاستخدام

يمكن استخدام `ctx.t()` في جميع بيئات تنفيذ RunJS.

## تعريف النوع

```ts
t(key: string, options?: Record<string, any>): string
```

## المعاملات

| المعامل | النوع | الوصف |
|-----------|------|-------------|
| `key` | `string` | مفتاح الترجمة أو قالب يحتوي على عناصر نائبة (مثل `Hello {{name}}` أو `{{count}} rows`). |
| `options` | `object` | اختياري. متغيرات الإقحام (مثل `{ name: 'John', count: 5 }`)، أو خيارات i18n (مثل `defaultValue` أو `ns`). |

## القيمة المرجعة

- تُرجع النص المترجم؛ وفي حال عدم وجود ترجمة للمفتاح وعدم توفير `defaultValue` (قيمة افتراضية)، قد تُرجع المفتاح نفسه أو النص بعد عملية الإقحام.

## مساحة الاسم (ns)

**مساحة الاسم الافتراضية لبيئة RunJS هي `runjs`**. عند عدم تحديد `ns` (مساحة الاسم)، ستبحث `ctx.t(key)` عن المفتاح ضمن مساحة الاسم `runjs`.

```ts
// البحث عن المفتاح في مساحة الاسم 'runjs' افتراضياً
ctx.t('Submit'); // يعادل ctx.t('Submit', { ns: 'runjs' })

// البحث عن المفتاح في مساحة اسم محددة
ctx.t('Submit', { ns: 'myModule' });

// البحث في مساحات اسم متعددة بالتتابع (أولاً 'runjs'، ثم 'common')
ctx.t('Save', { ns: ['runjs', 'common'] });
```

## أمثلة

### مفتاح بسيط

```ts
ctx.t('Submit');
ctx.t('No data');
```

### مع متغيرات الإقحام

```ts
const text = ctx.t('Hello {{name}}', { name: ctx.user?.nickname || 'Guest' });
ctx.render(`<div>${text}</div>`);
```

```ts
ctx.message.success(ctx.t('Processed {{count}} rows', { count: rows.length }));
```

### نصوص ديناميكية (مثل الوقت النسبي)

```ts
if (minutes < 60) return ctx.t('{{count}} minutes ago', { count: minutes });
if (hours < 24) return ctx.t('{{count}} hours ago', { count: hours });
```

### تحديد مساحة الاسم

```ts
ctx.t('Hello {{name}}', { name: 'Guest', ns: 'myModule' });
```

## ملاحظات

- **إضافة التوطين**: لترجمة النصوص، يجب تفعيل إضافة التوطين أولاً. سيتم استخراج مفاتيح الترجمة المفقودة تلقائياً إلى قائمة إدارة التوطين لتسهيل صيانتها وترجمتها بشكل موحد.
- يدعم الإقحام بأسلوب i18next: استخدم `{{variableName}}` في المفتاح ومرر المتغير المقابل في `options` لاستبداله.
- يتم تحديد اللغة من خلال السياق الحالي (مثل `ctx.i18n.language` أو الإعدادات المحلية للمستخدم).
