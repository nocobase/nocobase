# ctx.router

مثيل للموجه (router) يعتمد على React Router، يُستخدم للتنقل عبر الكود البرمجي في RunJS. يُستخدم عادةً بالتزامن مع `ctx.route` و `ctx.location`.

## حالات الاستخدام

| السيناريو | الوصف |
|------|------|
| **JSBlock / JSField** | الانتقال إلى صفحات التفاصيل، أو صفحات القوائم، أو الروابط الخارجية بعد النقر على زر. |
| **قواعد الربط / تدفق الأحداث** | تنفيذ `navigate` إلى قائمة أو تفاصيل بعد نجاح الإرسال، أو تمرير `state` إلى الصفحة الهدف. |
| **JSAction / معالجة الأحداث** | تنفيذ التنقل بين المسارات ضمن منطق معالجة إرسال النماذج أو النقر على الروابط. |
| **التنقل بين العروض** | تحديث عنوان URL عبر `navigate` أثناء التبديل بين مكدس العروض الداخلية. |

> ملاحظة: `ctx.router` متاح فقط في بيئات RunJS التي تحتوي على سياق توجيه (مثل JSBlock داخل الصفحة، صفحات Flow، تدفقات الأحداث، إلخ)؛ وقد يكون فارغاً في السياقات الخلفية (backend) البحتة أو السياقات التي لا تدعم التوجيه (مثل سير العمل).

## تعريف النوع

```typescript
router: Router
```

يأتي `Router` من `@remix-run/router`. في RunJS، يتم تنفيذ عمليات التنقل مثل الانتقال، والرجوع، والتحديث عبر `ctx.router.navigate()`.

## الطرق (Methods)

### ctx.router.navigate()

الانتقال إلى مسار محدد، أو تنفيذ عملية الرجوع/التحديث.

**التوقيع:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**المعلمات:**

- `to`: المسار الهدف (string)، أو الموقع النسبي في السجل (number، مثل `-1` للرجوع)، أو `null` (لتحديث الصفحة الحالية).
- `options`: تكوينات اختيارية.
  - `replace?: boolean`: ما إذا كان سيتم استبدال السجل الحالي (الافتراضي هو `false` ، أي دفع سجل جديد).
  - `state?: any`: الحالة (state) المراد تمريرها إلى المسار الهدف. لا تظهر هذه البيانات في عنوان URL، ويمكن الوصول إليها عبر `ctx.location.state` في الصفحة الهدف، وهي مناسبة للمعلومات الحساسة، أو البيانات المؤقتة، أو المعلومات التي لا يفضل وضعها في عنوان URL.

## أمثلة

### الانتقال الأساسي

```ts
// الانتقال إلى قائمة المستخدمين (دفع سجل جديد، يمكن الرجوع)
ctx.router.navigate('/admin/users');

// الانتقال إلى صفحة التفاصيل
ctx.router.navigate(`/admin/users/${recordId}`);
```

### استبدال السجل (بدون إضافة سجل جديد)

```ts
// إعادة التوجيه إلى الصفحة الرئيسية بعد تسجيل الدخول؛ لن يعود المستخدم لصفحة تسجيل الدخول عند الرجوع
ctx.router.navigate('/admin', { replace: true });

// استبدال الصفحة الحالية بصفحة التفاصيل بعد نجاح إرسال النموذج
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### تمرير الحالة (state)

```ts
// حمل البيانات أثناء الانتقال؛ تسترجعها الصفحة الهدف عبر ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### الرجوع والتحديث

```ts
// الرجوع صفحة واحدة للوراء
ctx.router.navigate(-1);

// الرجوع صفحتين للوراء
ctx.router.navigate(-2);

// تحديث الصفحة الحالية
ctx.router.navigate(null);
```

## العلاقة مع ctx.route و ctx.location

| الغرض | الاستخدام الموصى به |
|------|----------|
| **التنقل / الانتقال** | `ctx.router.navigate(path)` |
| **قراءة المسار الحالي** | `ctx.route.pathname` أو `ctx.location.pathname` |
| **قراءة الحالة الممررة أثناء التنقل** | `ctx.location.state` |
| **قراءة معلمات المسار (params)** | `ctx.route.params` |

يتولى `ctx.router` مسؤولية "إجراءات التنقل"، بينما يتولى `ctx.route` و `ctx.location` مسؤولية "حالة المسار الحالية".

## ملاحظات

- يقوم `navigate(path)` افتراضياً بدفع (push) سجل تاريخ جديد، مما يسمح للمستخدم بالعودة عبر زر الرجوع في المتصفح.
- الخيار `replace: true` يستبدل السجل الحالي دون إضافة سجل جديد، وهو مناسب لسيناريوهات مثل إعادة التوجيه بعد تسجيل الدخول أو الانتقال بعد نجاح الإرسال.
- **بخصوص المعلمة `state`**:
  - البيانات الممررة عبر `state` لا تظهر في عنوان URL، مما يجعلها مناسبة للبيانات الحساسة أو المؤقتة.
  - يمكن الوصول إليها عبر `ctx.location.state` في الصفحة الهدف.
  - يتم حفظ `state` في سجل المتصفح وتظل متاحة عند التنقل للأمام أو للخلف.
  - ستُفقد الـ `state` عند تحديث الصفحة بشكل كامل.
