# إجراءات الصيانة

## التشغيل الأولي للتطبيق

عند تشغيل التطبيق للمرة الأولى، ابدأ تشغيل عقدة واحدة أولاً. انتظر حتى يتم تثبيت الإضافات وتمكينها، ثم ابدأ تشغيل العقد الأخرى.

## ترقية الإصدار

عند الحاجة إلى ترقية إصدار NocoBase، اتبع هذا الإجراء.

:::warning{title=ملاحظة}
في بيئة إنتاجية **للمجموعات**، يجب استخدام ميزات مثل إدارة الإضافات وترقية الإصدارات بحذر أو حظرها.

لا يدعم NocoBase حاليًا الترقية عبر الإنترنت لإصدارات المجموعات. لضمان اتساق البيانات، يجب إيقاف الخدمات الخارجية مؤقتًا أثناء عملية الترقية.
:::

خطوات:

1.  إيقاف الخدمة الحالية

    أوقف جميع مثيلات تطبيق NocoBase، وأعد توجيه حركة مرور موازن التحميل إلى صفحة حالة 503.

2.  نسخ البيانات احتياطيًا

    قبل الترقية، يوصى بشدة بنسخ بيانات قاعدة البيانات احتياطيًا لمنع حدوث أي مشكلات أثناء عملية الترقية.

3.  تحديث الإصدار

    ارجع إلى [ترقية Docker](../get-started/upgrading/docker) لتحديث إصدار صورة تطبيق NocoBase.

4.  بدء الخدمة

    1.  ابدأ تشغيل عقدة واحدة في المجموعة وانتظر حتى يكتمل التحديث وتبدأ العقدة بنجاح.
    2.  تحقق من صحة الوظائف. إذا كانت هناك أي مشكلات لا يمكن حلها من خلال استكشاف الأخطاء وإصلاحها، يمكنك التراجع إلى الإصدار السابق.
    3.  ابدأ تشغيل العقد الأخرى.
    4.  أعد توجيه حركة مرور موازن التحميل إلى مجموعة التطبيقات.

## الصيانة داخل التطبيق

تشير الصيانة داخل التطبيق إلى تنفيذ عمليات متعلقة بالصيانة أثناء تشغيل التطبيق، وتشمل:

*   إدارة الإضافات (تثبيت الإضافات، تمكينها، تعطيلها، وما إلى ذلك)
*   النسخ الاحتياطي والاستعادة
*   إدارة ترحيل البيئة

خطوات:

1.  تقليص عدد العقد

    قلص عدد عقد التطبيق العاملة في المجموعة إلى عقدة واحدة، وأوقف الخدمة على العقد الأخرى.

2.  تنفيذ عمليات الصيانة داخل التطبيق، مثل تثبيت الإضافات وتمكينها، ونسخ البيانات احتياطيًا، وما إلى ذلك.

3.  استعادة العقد

    بعد اكتمال عمليات الصيانة والتحقق من صحة الوظائف، ابدأ تشغيل العقد الأخرى. بمجرد بدء تشغيل العقد بنجاح، استعد حالة تشغيل المجموعة.
