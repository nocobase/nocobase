# تطبيقات متعددة (Multi-app)


## مقدمة

تسمح إضافة **التطبيقات المتعددة (Multi-app)** بإنشاء وإدارة تطبيقات مستقلة متعددة ديناميكيًا دون الحاجة إلى نشر (Deployment) منفصل لكل منها. كل تطبيق فرعي هو نسخة مستقلة تمامًا، لها قاعدة بياناتها الخاصة، وإضافاتها، وإعداداتها.

#### حالات الاستخدام
- **تعدد المستأجرين (Multi-tenancy)**: توفير نسخ تطبيقات مستقلة حيث يمتلك كل عميل بياناته الخاصة، وإعدادات الإضافات، ونظام الصلاحيات.
- **الأنظمة الرئيسية والفرعية لمجالات عمل مختلفة**: نظام كبير يتكون من عدة تطبيقات صغيرة يتم نشرها بشكل مستقل.


:::warning
لا توفر إضافة التطبيقات المتعددة بحد ذاتها ميزة مشاركة المستخدمين.  
لتمكين ربط المستخدمين بين التطبيقات المختلفة، يمكن استخدامها مع **[إضافة المصادقة (Authentication Plugin)](/auth-verification)**.
:::


## التثبيت

ابحث عن إضافة **التطبيقات المتعددة (Multi-app)** في مدير الإضافات وقم بتفعيلها.

![](https://static-docs.nocobase.com/multi-app/Plugin-manager-NocoBase-10-16-2025_03_07_PM.png)


## دليل الاستخدام


### إنشاء تطبيق فرعي

انقر على "تطبيقات متعددة" في قائمة إعدادات النظام للدخول إلى صفحة إدارة التطبيقات المتعددة:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

انقر على زر "إضافة جديد" لإنشاء تطبيق فرعي جديد:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### وصف حقول النموذج

* **الاسم**: معرف التطبيق الفرعي، يجب أن يكون فريدًا على مستوى النظام.
* **اسم العرض**: الاسم الذي يظهر للتطبيق الفرعي في الواجهة.
* **وضع التشغيل**:
  * **التشغيل عند أول وصول**: يبدأ التطبيق الفرعي بالعمل فقط عندما يصل إليه المستخدم عبر الرابط (URL) لأول مرة.
  * **التشغيل مع التطبيق الرئيسي**: يبدأ التطبيق الفرعي بالعمل بالتزامن مع التطبيق الرئيسي (مما يزيد من وقت تشغيل التطبيق الرئيسي).
* **المنفذ**: رقم المنفذ (Port) الذي يستخدمه التطبيق الفرعي أثناء التشغيل.
* **نطاق مخصص**: إعداد نطاق فرعي (Subdomain) مستقل للتطبيق الفرعي.
* **التثبيت في القائمة**: تثبيت مدخل التطبيق الفرعي في الجانب الأيسر من شريط التنقل العلوي.
* **اتصال قاعدة البيانات**: يستخدم لتكوين مصدر البيانات للتطبيق الفرعي، ويدعم ثلاث طرق:
  * **قاعدة بيانات جديدة**: إعادة استخدام خدمة البيانات الحالية لإنشاء قاعدة بيانات مستقلة.
  * **اتصال بيانات جديد**: إعداد خدمة قاعدة بيانات جديدة تمامًا.
  * **وضع المخطط (Schema)**: إنشاء Schema مستقل للتطبيق الفرعي في PostgreSQL.
* **ترقية**: إذا كانت قاعدة البيانات المتصلة تحتوي على هيكل بيانات NocoBase من إصدار قديم، فسيتم ترقيتها تلقائيًا إلى الإصدار الحالي.


### تشغيل وإيقاف التطبيقات الفرعية

انقر على زر **بدء التشغيل** لتشغيل التطبيق الفرعي.  
> إذا تم تحديد *"التشغيل عند أول وصول"* أثناء الإنشاء، فسيتم التشغيل تلقائيًا عند الزيارة الأولى.  

انقر على زر **عرض** لفتح التطبيق الفرعي في علامة تبويب جديدة.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)


### حالة التشغيل والسجلات

يمكنك عرض استهلاك الذاكرة والمعالج (CPU) لكل تطبيق في القائمة.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

انقر على زر **السجلات (Logs)** لعرض سجلات تشغيل التطبيق الفرعي.  
> إذا تعذر الوصول إلى التطبيق الفرعي بعد تشغيله (بسبب تلف قاعدة البيانات مثلاً)، يمكنك استكشاف الأخطاء وإصلاحها عبر السجلات.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)


### حذف تطبيق فرعي

انقر على زر **حذف** لإزالة التطبيق الفرعي.  
> عند الحذف، يمكنك اختيار ما إذا كنت تريد حذف قاعدة البيانات أيضًا. يرجى الحذر، فهذا الإجراء لا يمكن التراجع عنه.


### الوصول إلى التطبيقات الفرعية
بشكل افتراضي، استخدم `/_app/:appName/admin/` للوصول إلى التطبيقات الفرعية، على سبيل المثال:
```
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
بالإضافة إلى ذلك، يمكنك إعداد نطاقات فرعية مستقلة للتطبيقات الفرعية. تحتاج إلى توجيه النطاق (DNS) إلى عنوان IP الحالي، وإذا كنت تستخدم Nginx، فيجب إضافة النطاق في إعدادات Nginx أيضًا.


### إدارة التطبيقات الفرعية عبر سطر الأوامر

في المجلد الرئيسي للمشروع، يمكنك استخدام سطر الأوامر لإدارة نسخ التطبيقات الفرعية عبر **PM2**:

```bash
yarn nocobase pm2 list              # عرض قائمة النسخ التي تعمل حاليًا
yarn nocobase pm2 stop [appname]    # إيقاف عملية تطبيق فرعي معين
yarn nocobase pm2 delete [appname]  # حذف عملية تطبيق فرعي معين
yarn nocobase pm2 kill              # إنهاء جميع العمليات التي تم تشغيلها قسريًا (قد تشمل نسخة التطبيق الرئيسي)
```

### ترحيل بيانات التطبيقات المتعددة القديمة

انتقل إلى صفحة إدارة التطبيقات المتعددة القديمة وانقر على زر **ترحيل البيانات إلى التطبيقات المتعددة الجديدة** لإجراء عملية الترحيل.

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)


## الأسئلة الشائعة

#### 1. إدارة الإضافات
يمكن للتطبيقات الفرعية استخدام نفس الإضافات الموجودة في التطبيق الرئيسي (بما في ذلك الإصدارات)، ولكن يمكن تهيئة واستخدام الإضافات بشكل مستقل في كل تطبيق.

#### 2. عزلة قاعدة البيانات
يمكن تهيئة التطبيقات الفرعية بقواعد بيانات مستقلة. إذا كنت ترغب في مشاركة البيانات بين التطبيقات، يمكن تحقيق ذلك عبر مصادر بيانات خارجية.

#### 3. النسخ الاحتياطي والترحيل للبيانات
حالياً، لا يشمل النسخ الاحتياطي للبيانات في التطبيق الرئيسي بيانات التطبيقات الفرعية (يشمل فقط المعلومات الأساسية عنها). يجب إجراء النسخ الاحتياطي والترحيل يدويًا داخل كل تطبيق فرعي.

#### 4. النشر والتحديثات
ستتبع إصدارات التطبيقات الفرعية ترقيات التطبيق الرئيسي تلقائيًا، مما يضمن توافق الإصدارات بين التطبيق الرئيسي والفرعي.

#### 5. إدارة الموارد
استهلاك الموارد لكل تطبيق فرعي هو نفسه تقريبًا للتطبيق الرئيسي. حاليًا، يبلغ استهلاك الذاكرة للتطبيق الواحد حوالي 500-600 ميجابايت.
