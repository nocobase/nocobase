# مجموعة SQL

## مقدمة

توفر مجموعة SQL طريقة قوية لاسترداد البيانات باستخدام استعلامات SQL. من خلال استخراج حقول البيانات عبر استعلامات SQL وتكوين البيانات الوصفية للحقول المرتبطة بها، يمكن للمستخدمين استخدام هذه الحقول كما لو كانوا يتعاملون مع جدول قياسي. هذه الميزة مفيدة بشكل خاص للسيناريوهات التي تتضمن استعلامات الربط المعقدة، والتحليل الإحصائي، والمزيد.

## دليل المستخدم

### إنشاء مجموعة SQL جديدة

<img src="https://static-docs.nocobase.com/202405191452918.png"/>

1.  أدخل استعلام SQL الخاص بك في مربع الإدخال المخصص وانقر على تنفيذ (Execute). سيقوم النظام بتحليل الاستعلام لتحديد الجداول والحقول المتضمنة، ويستخرج تلقائيًا البيانات الوصفية للحقول ذات الصلة من الجداول المصدر.

<img src="https://static-docs.nocobase.com/202405191453556.png"/>

2.  إذا كان تحليل النظام للجداول والحقول المصدر غير صحيح، يمكنك تحديد الجداول والحقول المناسبة يدويًا لضمان استخدام البيانات الوصفية الصحيحة. ابدأ بتحديد الجدول المصدر، ثم اختر الحقول المقابلة في قسم مصدر الحقل أدناه.

<img src="https://static-docs.nocobase.com/202405191453579.png"/>

3.  بالنسبة للحقول التي ليس لها مصدر مباشر، سيستنتج النظام نوع الحقل بناءً على نوع البيانات. إذا كان هذا الاستنتاج غير صحيح، يمكنك تحديد نوع الحقل الصحيح يدويًا.

<img src="https://static-docs.nocobase.com/202405191454703.png"/>

4.  أثناء تكوين كل حقل، يمكنك معاينة عرضه في منطقة المعاينة، مما يتيح لك رؤية التأثير الفوري لإعداداتك.

<img src="https://static-docs.nocobase.com/202405191455439.png"/>

5.  بعد الانتهاء من التكوين والتأكد من صحة كل شيء، انقر على زر تأكيد (Confirm) أسفل مربع إدخال SQL لإتمام عملية الإرسال.

<img src="https://static-docs.nocobase.com/202405191455302.png"/>

### التحرير

1.  إذا كنت بحاجة إلى تعديل استعلام SQL، انقر على زر التعديل (Edit) لتغيير عبارة SQL مباشرة وإعادة تكوين الحقول حسب الحاجة.

2.  لتعديل البيانات الوصفية للحقول، استخدم خيار تكوين الحقول (Configure fields)، والذي يسمح لك بتحديث إعدادات الحقول تمامًا كما تفعل مع جدول عادي.

### المزامنة

إذا ظل استعلام SQL دون تغيير ولكن تم تعديل هيكل جدول قاعدة البيانات الأساسي، يمكنك مزامنة الحقول وإعادة تكوينها عن طريق تحديد تكوين الحقول (Configure fields) - المزامنة من قاعدة البيانات (Sync from database).

<img src="https://static-docs.nocobase.com/202405191456216.png"/>

### مجموعة SQL مقابل طرق عرض قاعدة البيانات المرتبطة

| نوع القالب             | أفضل ما يناسب                                                                                             | طريقة التنفيذ | دعم عمليات CRUD |
| :--------------------- | :-------------------------------------------------------------------------------------------------------- | :------------ | :-------------- |
| SQL                    | نماذج بسيطة، حالات استخدام خفيفة الوزن<br />تفاعل محدود مع قاعدة البيانات<br />تجنب صيانة طرق العرض<br />تفضيل العمليات التي تعتمد على واجهة المستخدم | استعلام فرعي SQL | غير مدعوم       |
| الاتصال بعرض قاعدة البيانات | نماذج معقدة<br />يتطلب تفاعل قاعدة البيانات<br />الحاجة إلى تعديل البيانات<br />يتطلب دعمًا أقوى وأكثر استقرارًا لقاعدة البيانات | عرض قاعدة البيانات | مدعوم جزئيًا    |

:::warning
عند استخدام مجموعة SQL، تأكد من تحديد الجداول التي يمكن إدارتها ضمن NocoBase. قد يؤدي استخدام جداول من نفس قاعدة البيانات غير المتصلة بـ NocoBase إلى تحليل غير دقيق لاستعلام SQL. إذا كان هذا مصدر قلق، ففكر في إنشاء عرض وربطه.
:::
