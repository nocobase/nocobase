# نظرة عامة

نمذجة البيانات خطوة أساسية عند تصميم قواعد البيانات، وتتضمن عملية تحليل وتجريد عميق لأنواع البيانات المختلفة وعلاقاتها المتبادلة في العالم الحقيقي. خلال هذه العملية، نسعى للكشف عن الروابط الجوهرية بين البيانات وصياغتها رسميًا كنماذج بيانات، مما يضع الأساس لهيكل قاعدة بيانات نظام المعلومات. NocoBase هي منصة تعتمد على نماذج البيانات، وتتميز بما يلي:

## يدعم الوصول إلى البيانات من مصادر متنوعة

يمكن أن تكون مصادر البيانات في NocoBase من أصول متنوعة، بما في ذلك قواعد البيانات الشائعة ومنصات API (SDK) والملفات.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

يوفر NocoBase [إضافة إدارة مصادر البيانات](/data-sources/data-source-manager) لإدارة مصادر البيانات المختلفة و`مجموعاتها`. توفر إضافة إدارة مصادر البيانات واجهة إدارة لجميع مصادر البيانات فقط، ولا توفر القدرة على الوصول المباشر إلى مصادر البيانات. يجب استخدامها بالاقتران مع إضافات مصادر البيانات المختلفة. تشمل مصادر البيانات المدعومة حاليًا ما يلي:

- [قاعدة البيانات الرئيسية](/data-sources/data-source-main): قاعدة بيانات NocoBase الرئيسية، تدعم قواعد البيانات العلائقية مثل MySQL و PostgreSQL و MariaDB.
- [KingbaseES](/data-sources/data-source-kingbase): استخدام قاعدة بيانات KingbaseES ك`مصدر بيانات`، يمكن استخدامها كقاعدة بيانات رئيسية أو خارجية.
- [MySQL خارجي](/data-sources/data-source-external-mysql): استخدام قاعدة بيانات MySQL خارجية ك`مصدر بيانات`.
- [MariaDB خارجي](/data-sources/data-source-external-mariadb): استخدام قاعدة بيانات MariaDB خارجية ك`مصدر بيانات`.
- [PostgreSQL خارجي](/data-sources/data-source-external-postgres): استخدام قاعدة بيانات PostgreSQL خارجية ك`مصدر بيانات`.
- [MSSQL خارجي](/data-sources/data-source-external-mssql): استخدام قاعدة بيانات MSSQL (SQL Server) خارجية ك`مصدر بيانات`.
- [Oracle خارجي](/data-sources/data-source-external-oracle): استخدام قاعدة بيانات Oracle خارجية ك`مصدر بيانات`.

![20240512083651](https://static-docs.nocobase.com/20240512083651.png)

## يوفر مجموعة متنوعة من أدوات نمذجة البيانات

**واجهة إدارة `المجموعات` المبسطة**: تُستخدم لإنشاء نماذج (`مجموعات`) متنوعة أو الاتصال بالنماذج الموجودة.

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**واجهة مرئية بنمط مخططات الكيانات والعلاقات (ERD)**: تُستخدم لاستخلاص الكيانات وعلاقاتها من متطلبات المستخدم والأعمال. توفر طريقة بديهية وسهلة الفهم لوصف نماذج البيانات، ومن خلال مخططات ERD، يمكنك فهم كيانات البيانات الرئيسية في النظام وعلاقاتها بشكل أوضح.

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## يدعم أنواعًا متنوعة من `المجموعات`

| المجموعة | الوصف |
| - | - |
| [مجموعة عامة](/data-sources/data-source-main/general-collection) | تتضمن حقول النظام الشائعة المدمجة. |
| [مجموعة تقويم](/data-sources/calendar/calendar-collection) | تُستخدم لإنشاء `مجموعات` أحداث متعلقة بالتقويم. |
| مجموعة التعليقات | تُستخدم لتخزين التعليقات أو الملاحظات حول البيانات. |
| [مجموعة شجرية](/data-sources/collection-tree) | `مجموعة` ذات هيكل شجري، تدعم حاليًا نموذج قائمة التجاور فقط. |
| [مجموعة الملفات](/data-sources/file-manager/file-collection) | تُستخدم لإدارة تخزين الملفات. |
| [مجموعة SQL](/data-sources/collection-sql) | ليست `مجموعة` قاعدة بيانات فعلية، بل تعرض استعلامات SQL بشكل منظم وسريع. |
| [الاتصال بعرض قاعدة بيانات](/data-sources/collection-view) | تتصل بعروض قواعد البيانات الموجودة. |
| مجموعة التعبيرات | تُستخدم لسيناريوهات التعبيرات الديناميكية في `سير العمل`. |
| [الاتصال بالبيانات الخارجية](/data-sources/collection-fdw) | تسمح لنظام قاعدة البيانات بالوصول المباشر إلى البيانات والاستعلام عنها في مصادر البيانات الخارجية بناءً على تقنية FDW. |
