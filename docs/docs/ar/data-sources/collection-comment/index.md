# مجموعة التعليقات

## مقدمة

مجموعة التعليقات هي قالب جدول بيانات متخصص مصمم لتخزين تعليقات المستخدمين وملاحظاتهم. من خلال ميزة التعليقات، يمكنك إضافة إمكانات التعليق إلى أي جدول بيانات، مما يسمح للمستخدمين بمناقشة سجلات معينة أو تقديم ملاحظات عليها أو إضافة شروحات لها. تدعم مجموعة التعليقات التحرير بالنص المنسق، مما يوفر قدرات مرنة لإنشاء المحتوى.

![comment-collection-2025-11-01-00-39-01](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-39-01.png)

## الميزات

-   **تحرير النص المنسق**: يتضمن محرر Markdown (vditor) افتراضيًا، ويدعم إنشاء محتوى نص منسق.
-   **الربط بأي جدول بيانات**: يمكن ربط التعليقات بالسجلات في أي جدول بيانات من خلال حقول العلاقة.
-   **تعليقات متعددة المستويات**: يدعم الرد على التعليقات، وبناء هيكل شجري للتعليقات.
-   **تتبع المستخدمين**: يسجل تلقائيًا منشئ التعليق ووقت الإنشاء.

## دليل المستخدم

### إنشاء مجموعة تعليقات

![comment-collection-2025-11-01-00-37-10](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-37-10.png)

1.  انتقل إلى صفحة إدارة جداول البيانات.
2.  انقر على زر "إنشاء مجموعة" (Create Collection).
3.  اختر قالب "مجموعة التعليقات" (Comment Collection).
4.  أدخل اسم الجدول (على سبيل المثال: "تعليقات المهام"، "تعليقات المقالات"، إلخ).
5.  سيقوم النظام تلقائيًا بإنشاء جدول تعليقات يتضمن الحقول الافتراضية التالية:
    -   محتوى التعليق (نوع Markdown vditor)
    -   تم الإنشاء بواسطة (مرتبط بجدول المستخدمين)
    -   وقت الإنشاء (نوع التاريخ والوقت)

### إعداد العلاقات

لربط التعليقات بجدول بيانات مستهدف، تحتاج إلى إعداد حقول العلاقة:

![](https://static-docs.nocobase.com/Solution/demoE3v1-19N.gif)

1.  أضف حقل علاقة "متعدد إلى واحد" (Many-to-One) في جدول التعليقات.
2.  اختر جدول البيانات المستهدف للربط به (على سبيل المثال: جدول المهام، جدول المقالات، إلخ).
3.  عيّن اسم الحقل (على سبيل المثال: "ينتمي إلى مهمة"، "ينتمي إلى مقال"، إلخ).

### استخدام كتل التعليقات في الصفحات

![Enable Comments Collection](https://static-docs.nocobase.com/Solution/demoE3v1-20.gif)

1.  انتقل إلى الصفحة التي ترغب في إضافة وظيفة التعليقات إليها.
2.  أضف كتلة في تفاصيل السجل المستهدف أو في النافذة المنبثقة.
3.  اختر نوع الكتلة "تعليقات" (Comments).
4.  اختر مجموعة التعليقات التي أنشأتها للتو.

### حالات الاستخدام النموذجية

-   **أنظمة إدارة المهام**: يناقش أعضاء الفريق المهام ويقدمون ملاحظات عليها.
-   **أنظمة إدارة المحتوى**: يعلق القراء على المقالات ويتفاعلون معها.
-   **سير عمل الموافقات**: يضيف الموافقون شروحات وآراء على نماذج الطلبات.
-   **ملاحظات العملاء**: جمع تقييمات العملاء للمنتجات أو الخدمات.

## ملاحظات

-   مجموعة التعليقات هي ميزة إضافة تجارية وتتطلب تمكين إضافة التعليقات لاستخدامها.
-   يوصى بتعيين أذونات مناسبة لجدول التعليقات للتحكم في من يمكنه عرض التعليقات وإنشائها وحذفها.
-   في السيناريوهات التي تحتوي على عدد كبير من التعليقات، يوصى بتمكين التحميل المقسم (pagination) لتحسين الأداء.
