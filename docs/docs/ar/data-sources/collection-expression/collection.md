# مجموعة التعبيرات

## إنشاء مجموعة قوالب "تعبير"

قبل استخدام عقد عمليات التعبير الديناميكية ضمن سير العمل، يجب أولاً إنشاء مجموعة قالب "تعبير" في أداة إدارة المجموعات. ستكون هذه المجموعة بمثابة مستودع للتعبيرات المختلفة:

![إنشاء مجموعة تعبيرات](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## إدخال بيانات التعبير

بعد ذلك، يمكنك إعداد كتلة جدول وإدخال عدة إدخالات صيغ في مجموعة القالب هذه. يمكن اعتبار كل صف في مجموعة قالب "التعبير" قاعدة حساب مصممة لنموذج بيانات مجموعة معين. يمكنك استخدام قيم الحقول من نماذج بيانات مجموعات مختلفة كمتغيرات، وصياغة تعبيرات فريدة كقواعد حساب. وبالطبع، يمكنك أيضًا استخدام محركات حساب مختلفة.

![إدخال بيانات التعبير](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=تلميح}
بمجرد إنشاء الصيغ، يجب ربطها ببيانات العمل. قد يكون ربط كل صف من بيانات العمل مباشرة ببيانات الصيغة أمرًا شاقًا. لذا، فإن النهج الشائع هو استخدام مجموعة بيانات وصفية، مشابهة لمجموعة تصنيف، لإنشاء علاقة متعدد إلى واحد (أو واحد إلى واحد) مع مجموعة الصيغ. ثم يتم ربط بيانات العمل بالبيانات الوصفية المصنفة في علاقة متعدد إلى واحد. يتيح هذا النهج لك ببساطة تحديد البيانات الوصفية المصنفة ذات الصلة عند إنشاء بيانات العمل، مما يسهل تحديد موقع بيانات الصيغة المقابلة واستخدامها عبر مسار الربط المحدد.
:::
