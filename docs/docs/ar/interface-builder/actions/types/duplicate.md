# نسخ

## مقدمة

تتيح عملية النسخ للمستخدمين إنشاء سجلات جديدة بسرعة بناءً على البيانات الموجودة. وهي تدعم طريقتين للنسخ: **النسخ المباشر** و**النسخ إلى نموذج والمتابعة**.

## التثبيت

إضافة مدمجة، لا تتطلب تثبيتاً منفصلاً.

## وضع النسخ

![20260209224344](https://static-docs.nocobase.com/20260209224344.png)

### النسخ المباشر

![20260209224506](https://static-docs.nocobase.com/20260209224506.png)

- يتم التنفيذ افتراضياً بوضع "النسخ المباشر"؛
- **حقول القالب**: حدد الحقول المراد نسخها، يمكن اختيار الكل، وهو إعداد مطلوب.

![20260209225910](https://static-docs.nocobase.com/20260209225910.gif)

بعد إكمال الإعداد، انقر فوق الزر لنسخ البيانات.

### النسخ إلى نموذج والمتابعة

سيتم ملء حقول القالب التي تم تكوينها كـ **قيم افتراضية** في النموذج، ويمكن للمستخدم تعديل هذه القيم قبل الإرسال لإتمام عملية النسخ.

![20260209224704](https://static-docs.nocobase.com/20260209224704.png)

**تكوين حقول القالب**: سيتم جلب الحقول المحددة فقط واستخدامها كقيم افتراضية.

![20260209225148](https://static-docs.nocobase.com/20260209225148.png)

#### مزامنة حقول النموذج

- تحليل الحقول التي تم تكوينها بالفعل في كتلة النموذج الحالية تلقائياً كحقول للقالب؛
- إذا تم تعديل حقول كتلة النموذج لاحقاً (مثل تعديل مكونات حقول العلاقة)، فيجب فتح إعدادات القالب مرة أخرى والنقر فوق **مزامنة حقول النموذج** لضمان الاتساق.

![20260209225450](https://static-docs.nocobase.com/20260209225450.gif)

سيتم ملء بيانات القالب كقيم افتراضية للنموذج، ويمكن للمستخدم تعديلها ثم الإرسال لإتمام النسخ.


### ملاحظات إضافية

#### النسخ، المرجعية، والتحميل المسبق

تختلف منطق المعالجة باختلاف أنواع الحقول (أنواع العلاقات): **النسخ / المرجعية / التحميل المسبق**. كما يؤثر **مكون الحقل** الخاص بحقل العلاقة على هذا المنطق:

- Select / Record picker: يستخدم لـ **المرجعية**
- Sub-form / Sub-table: يستخدم لـ **النسخ**

**النسخ**

- الحقول العادية يتم نسخها؛
- `hasOne` / `hasMany` يمكن نسخها فقط (لا ينبغي استخدام مكونات الاختيار مثل القائمة المنسدلة أو اختيار السجل لهذه العلاقات؛ بل يجب استخدام مكونات النموذج الفرعي أو الجدول الفرعي)؛
- تغيير المكون لـ `hasOne` / `hasMany` **لن** يغير منطق المعالجة (يظل نسخاً)؛
- بالنسبة لحقول العلاقة المنسوخة، يمكن اختيار جميع الحقول الفرعية.

**المرجعية**

- `belongsTo` / `belongsToMany` تُعامل كمرجعية؛
- إذا تم تغيير مكون الحقل من "قائمة منسدلة" إلى "نموذج فرعي"، ستتغير العلاقة من **مرجعية إلى نسخ** (وبمجرد أن تصبح نسخاً، تصبح جميع الحقول الفرعية قابلة للاختيار).

**التحميل المسبق**

- حقول العلاقة الموجودة تحت حقل مرجعي تُعامل كتحميل مسبق؛
- قد تتحول حقول التحميل المسبق إلى مرجعية أو نسخ بعد تغيير المكون.

#### اختيار الكل

- سيتم تحديد جميع **حقول النسخ** و**حقول المرجعية**.

#### سيتم تصفية الحقول التالية من السجل المختار كقالب بيانات:

- يتم تصفية المفاتيح الأساسية لبيانات العلاقة المنسوخة؛ بينما لا يتم تصفية المفاتيح الأساسية للمرجعية والتحميل المسبق؛
- المفاتيح الخارجية؛
- الحقول التي لا تسمح بالتكرار (فريدة)؛
- حقول الترتيب؛
- حقول الترقيم التلقائي؛
- كلمة المرور؛
- المنشئ، تاريخ الإنشاء؛
- آخر تحديث بواسطة، تاريخ آخر تحديث.

#### مزامنة حقول النموذج

- تحليل الحقول المكونة في كتلة النموذج الحالية تلقائياً إلى حقول قالب؛
- بعد تعديل حقول كتلة النموذج (مثل تعديل مكونات حقول العلاقة)، يجب المزامنة مرة أخرى لضمان الاتساق.
