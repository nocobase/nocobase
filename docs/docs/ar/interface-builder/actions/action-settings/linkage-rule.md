# قواعد ربط الإجراءات

## مقدمة

تتيح قواعد ربط الإجراءات للمستخدمين التحكم ديناميكيًا في حالة الإجراء (مثل العرض، التمكين، الإخفاء، التعطيل، إلخ.) بناءً على شروط محددة. من خلال إعداد هذه القواعد، يمكن للمستخدمين ربط سلوك أزرار الإجراءات بالسجل الحالي، أو دور المستخدم، أو بيانات السياق.

![20251029150224](https://static-docs.nocobase.com/20251029150224.png)

## كيفية الاستخدام

عند استيفاء الشرط (يتم تمريره افتراضيًا إذا لم يتم تعيين أي شرط)، يتم تشغيل تنفيذ إعدادات الخصائص أو JavaScript. يتم دعم استخدام الثوابت والمتغيرات في تحديد الشروط.

![20251030224601](https://static-docs.nocobase.com/20251030224601.png)

تدعم القاعدة تعديل خصائص الأزرار.

![20251029150452](https://static-docs.nocobase.com/20251029150452.png)

## الثوابت

مثال: لا يمكن تعديل الطلبات المدفوعة.

![20251029150638](https://static-docs.nocobase.com/20251029150638.png)

## المتغيرات

### متغيرات النظام

![20251029150014](https://static-docs.nocobase.com/20251029150014.png)

مثال 1: التحكم في عرض زر بناءً على نوع الجهاز الحالي.

![20251029151057](https://static-docs.nocobase.com/20251029151057.png)

مثال 2: زر التحديث المجمع في رأس جدول كتلة الطلبات متاح فقط لدور المسؤول؛ لا يمكن للأدوار الأخرى تنفيذ هذا الإجراء.

![20251029151209](https://static-docs.nocobase.com/20251029151209.png)

### متغيرات السياق

مثال: يتم تمكين زر الإضافة في فرص الطلبات (كتلة الارتباط) فقط عندما تكون حالة الطلب "قيد الدفع" أو "مسودة". في الحالات الأخرى، سيتم تعطيل الزر.

![20251029151520](https://static-docs.nocobase.com/20251029151520.png)
