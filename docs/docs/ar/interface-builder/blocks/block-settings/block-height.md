# ارتفاع الكتلة

## مقدمة

يدعم ارتفاع الكتلة ثلاثة أنماط: **الارتفاع الافتراضي**، **الارتفاع المحدد**، و**الارتفاع الكامل**. تدعم معظم الكتل إعدادات الارتفاع.

![20260211154020](https://static-docs.nocobase.com/20260211154020.png)

## أنماط الارتفاع

### الارتفاع الافتراضي

تختلف استراتيجية الارتفاع الافتراضي باختلاف أنواع الكتل. على سبيل المثال، تتكيف كتل الجداول والنماذج تلقائياً مع المحتوى، ولا تظهر أشرطة تمرير داخل الكتلة.

### الارتفاع المحدد

يمكنك تحديد الارتفاع الإجمالي للإطار الخارجي للكتلة يدوياً. ستقوم الكتلة تلقائياً بحساب وتوزيع الارتفاع المتاح داخلياً.

![20260211154043](https://static-docs.nocobase.com/20260211154043.gif)

### الارتفاع الكامل

يشبه نمط الارتفاع الكامل الارتفاع المحدد، ولكن يتم حساب ارتفاع الكتلة بناءً على **منطقة العرض (viewport)** الحالية للمتصفح لتحقيق أقصى ارتفاع لملء الشاشة. لن تظهر أشرطة تمرير في صفحة المتصفح؛ بل ستظهر أشرطة التمرير فقط داخل الكتلة.

يختلف التعامل مع التمرير الداخلي في نمط الارتفاع الكامل قليلاً بين الكتل:

- **الجدول**: تمرير داخلي ضمن `tbody`؛
- **النموذج / التفاصيل**: تمرير داخلي ضمن الشبكة (Grid) (تمرير المحتوى باستثناء منطقة الإجراءات)؛
- **القائمة / بطاقة الشبكة**: تمرير داخلي ضمن الشبكة (Grid) (تمرير المحتوى باستثناء منطقة الإجراءات وشريط الترقيم)؛
- **الخريطة / التقويم**: ارتفاع متكيف بشكل عام، بدون أشرطة تمرير؛
- **Iframe / Markdown**: تقييد الارتفاع الإجمالي لإطار الكتلة، مع ظهور أشرطة التمرير داخل الكتلة.
