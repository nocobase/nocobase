# الترتيب بالسحب والإفلات

## مقدمة

يعتمد الترتيب بالسحب والإفلات على حقل ترتيب (sort field) لإعادة ترتيب سجلات الكتلة (block) يدوياً.


:::info{title=تلميح}
* عند استخدام نفس حقل الترتيب للترتيب بالسحب والإفلات عبر كتل متعددة، قد يؤدي ذلك إلى تعارض في الترتيب الحالي.
* عند استخدام الترتيب بالسحب والإفلات في جدول، لا يمكن تعيين قواعد تجميع (grouping rules) لحقل الترتيب.
* تدعم جداول الشجرة (Tree tables) ترتيب العقد ضمن نفس المستوى فقط.
:::


## إعداد السحب والإفلات

أضف حقلاً من نوع "ترتيب" (Sort). لم تعد حقول الترتيب تُنشأ تلقائياً عند إنشاء مجموعة (Collection)؛ بل يجب إنشاؤها يدوياً.

## شرح قواعد الترتيب

لنفترض أن الترتيب الحالي هو:

```
[1,2,3,4,5,6,7,8,9]
```

عندما يتم نقل عنصر (مثلاً 5) للأمام إلى موضع العنصر 3، ستتغير قيم الترتيب للعناصر 3 و4 و5 فقط: يأخذ العنصر 5 موضع العنصر 3، وينتقل كل من 3 و4 للخلف بمقدار موضع واحد.

```
[1,2,5,3,4,6,7,8,9]
```

إذا قمت بعد ذلك بنقل العنصر 6 للخلف إلى موضع العنصر 8، فسيأخذ العنصر 6 موضع العنصر 8، وينتقل كل من 7 و8 للأمام بمقدار موضع واحد.

```
[1,2,5,3,4,7,8,6,9]
```
