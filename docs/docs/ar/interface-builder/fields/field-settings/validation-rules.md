# إعداد قواعد التحقق

## مقدمة

تُستخدم قواعد التحقق لضمان أن البيانات التي يدخلها المستخدم تتوافق مع التوقعات.

## أين يمكن إعداد قواعد التحقق للحقول

### إعداد قواعد التحقق لحقول المجموعة

تدعم معظم الحقول إمكانية إعداد قواعد التحقق. بعد إعداد قواعد التحقق لحقل ما، يتم تشغيل التحقق من جانب الخادم (backend validation) عند إرسال البيانات. تدعم أنواع الحقول المختلفة قواعد تحقق متنوعة.

- **حقل التاريخ**

  ![20251028225946](https://static-docs.nocobase.com/20251028225946.png)

- **حقل الأرقام**

  ![20251028230418](https://static-docs.nocobase.com/20251028230418.png)

- **حقل النص**

  بالإضافة إلى إمكانية تحديد طول النص، تدعم حقول النص أيضًا التعبيرات النمطية المخصصة (regular expressions) لإجراء تحقق أكثر دقة.

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### التحقق من جانب العميل (Frontend) في إعدادات الحقل

قواعد التحقق التي يتم إعدادها في تهيئة الحقل ستؤدي إلى تشغيل التحقق من جانب العميل، مما يضمن توافق مدخلات المستخدم مع المتطلبات.

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

![20251028230255](https://static-docs.nocobase.com/20251028230255.png)

تدعم **حقول النص** أيضًا التحقق باستخدام التعبيرات النمطية المخصصة لتلبية متطلبات تنسيق محددة.

![20251028230903](https://static-docs.nocobase.com/20251028230903.png)
