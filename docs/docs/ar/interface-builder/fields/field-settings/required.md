# مطلوب

## مقدمة

"مطلوب" هي قاعدة شائعة للتحقق من صحة النماذج. يمكنك تمكينها مباشرة في إعدادات الحقل، أو تعيين حقل كمطلوب ديناميكيًا عبر قواعد الربط الخاصة بالنموذج.

## أين يمكن تعيين حقل كمطلوب

### إعدادات حقل المجموعة

عندما يتم تعيين حقل في المجموعة كمطلوب، فإنه يؤدي إلى تشغيل التحقق من صحة البيانات في الواجهة الخلفية (backend)، وتظهر الواجهة الأمامية (frontend) الحقل كمطلوب افتراضيًا (لا يمكن تعديله).
![20251025175418](https://static-docs.nocobase.com/20251025175418.png)

### إعدادات الحقل

قم بتعيين الحقل كمطلوب مباشرة. هذا مناسب للحقول التي يحتاج المستخدم دائمًا إلى تعبئتها، مثل اسم المستخدم وكلمة المرور وما إلى ذلك.

![20251028222818](https://static-docs.nocobase.com/20251028222818.png)

### قواعد الربط

قم بتعيين حقل كمطلوب بناءً على شروط محددة من خلال قواعد ربط الحقول في كتلة النموذج.

مثال: رقم الطلب مطلوب عندما لا يكون تاريخ الطلب فارغًا.
