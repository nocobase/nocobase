# الجدول الفرعي (تحرير في نافذة منبثقة)

## مقدمة

يُستخدم الجدول الفرعي (تحرير في نافذة منبثقة) لإدارة بيانات الارتباط المتعددة (مثل واحد إلى متعدد O2M أو متعدد إلى متعدد M2M) داخل النموذج. يعرض الجدول السجلات المرتبطة حالياً فقط، وتتم عمليات الإضافة أو التحرير داخل نافذة منبثقة، حيث تُحفظ البيانات في قاعدة البيانات بشكل موحد عند إرسال النموذج الرئيسي.

## تعليمات الاستخدام

![20260212152204](https://static-docs.nocobase.com/20260212152204.png)

**حالات الاستخدام:**

- حقول الارتباط: o2m / m2m / mbm
- الاستخدامات الشائعة: تفاصيل الطلبات، قوائم العناصر الفرعية، الوسوم/الأعضاء المرتبطين، إلخ.

## خيارات تكوين الحقل

### السماح باختيار البيانات الموجودة (افتراضي: مفعل)

يدعم اختيار الارتباطات من البيانات الموجودة مسبقاً.

![20260212152312](https://static-docs.nocobase.com/20260212152312.png)

![20260212152343](https://static-docs.nocobase.com/20260212152343.gif)

### مكون الحقل

[مكون الحقل](/interface-builder/fields/association-field): التبديل إلى مكونات حقول ارتباط أخرى، مثل الاختيار المنسدل، أو منتقي المجموعة، إلخ.

### السماح بإلغاء ربط البيانات الموجودة (افتراضي: مفعل)

> يتحكم في ما إذا كان يُسمح بإلغاء ربط البيانات المرتبطة مسبقاً في نموذج التحرير، بينما يُسمح دائماً بإزالة البيانات المضافة حديثاً.

![20260212165752](https://static-docs.nocobase.com/20260212165752.gif)

### السماح بالإضافة (افتراضي: مفعل)

يتحكم في عرض زر الإضافة. عندما لا يمتلك المستخدم صلاحية `create` للمجموعة المستهدفة، سيتم تعطيل الزر مع ظهور تلميح يفيد بعدم وجود صلاحية.

### السماح بالتحرير السريع (افتراضي: معطل)

عند تفعيله، سيظهر رمز التحرير عند تمرير الماوس فوق الخلية، مما يسمح بتحرير محتوى الخلية بسرعة.

يمكن تفعيل التحرير السريع لجميع الحقول من خلال إعدادات مكون حقل الارتباط.

![20260212165102](https://static-docs.nocobase.com/20260212165102.png)

كما يمكن تفعيله لكل حقل عمود بشكل فردي.

![20260212165025](https://static-docs.nocobase.com/20260212165025.png)

![20260212165201](https://static-docs.nocobase.com/20260212165201.gif)

### حجم الصفحة (افتراضي: 10)

يحدد عدد السجلات المعروضة في كل صفحة داخل الجدول الفرعي.

## ملاحظات حول السلوك

- عند اختيار سجلات موجودة، يتم إلغاء التكرار بناءً على المفتاح الأساسي لتجنب تكرار ارتباط نفس السجل.
- يتم ملء السجلات المضافة حديثاً مباشرة في الجدول الفرعي، وينتقل العرض تلقائياً إلى الصفحة التي تحتوي على السجل الجديد.
- التحرير السطري (Inline edit) يعدل بيانات الصف الحالي فقط.
- تؤدي الإزالة فقط إلى إلغاء الارتباط في النموذج الحالي، ولا يتم حذف البيانات الأصلية من المصدر.
- يتم حفظ البيانات في قاعدة البيانات بشكل موحد فقط عند إرسال النموذج الرئيسي.
