# جدول فرعي

## مقدمة

يُستخدم الجدول الفرعي للتعامل مع حقول العلاقات من نوع "واحد إلى متعدد". وهو يدعم إنشاء بيانات جديدة في الـ `مجموعة` الهدف وربطها بشكل مجمع، أو اختيار وربط البيانات الموجودة.

## تعليمات الاستخدام

![20251027223350](https://static-docs.nocobase.com/20251027223350.png)

تُعرض أنواع مختلفة من الحقول في الجدول الفرعي باستخدام مكونات حقول مختلفة. أما الحقول الكبيرة (مثل النص المنسق، JSON، النص متعدد الأسطر، وغيرها) فيتم تعديلها عبر نافذة منبثقة عائمة.

![20251027223426](https://static-docs.nocobase.com/20251027223426.png)

حقول العلاقات في الجدول الفرعي.

الطلبات (واحد إلى متعدد) > منتجات الطلب (واحد إلى واحد) > الفرصة

![20251027223530](https://static-docs.nocobase.com/20251027223530.png)

المكون الافتراضي لحقل العلاقة هو القائمة المنسدلة (يدعم القائمة المنسدلة / منتقي البيانات).

![20251027223754](https://static-docs.nocobase.com/20251027223754.png)

## خيارات تهيئة الحقل

### السماح باختيار البيانات الموجودة (معطل افتراضيًا)

يدعم اختيار وربط البيانات الموجودة.
![20251027224008](https://static-docs.nocobase.com/20251027224008.png)

![20251027224023](https://static-docs.nocobase.com/20251027224023.gif)

### مكون الحقل

[مكون الحقل](/interface-builder/fields/association-field): التبديل إلى مكونات حقول علاقة أخرى، مثل القائمة المنسدلة، منتقي البيانات، وغيرها؛

### السماح بفك ارتباط البيانات الموجودة

> هل يُسمح بفك ارتباط البيانات الموجودة لحقل العلاقة في نموذج التعديل؟
