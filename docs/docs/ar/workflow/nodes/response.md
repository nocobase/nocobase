# استجابة HTTP

## مقدمة

تُستخدم هذه العقدة فقط في سير عمل Webhook المتزامن، وهي مخصصة لإرجاع استجابة إلى نظام خارجي. على سبيل المثال، أثناء معالجة استدعاء دفع (payment callback)، إذا كانت هناك نتيجة غير متوقعة في عملية الأعمال (مثل خطأ أو فشل)، يمكنك استخدام عقدة الاستجابة لإرجاع استجابة تعبر عن الخطأ إلى النظام الخارجي، مما يسمح لبعض الأنظمة الخارجية بإعادة المحاولة لاحقًا بناءً على الحالة.

بالإضافة إلى ذلك، يؤدي تنفيذ عقدة الاستجابة إلى إنهاء تنفيذ سير العمل، ولن يتم تنفيذ العقد اللاحقة. إذا لم يتم تكوين عقدة استجابة في سير العمل بأكمله، فسيقوم النظام بالاستجابة تلقائيًا بناءً على حالة تنفيذ سير العمل: حيث يعيد `200` عند التنفيذ الناجح، و `500` عند التنفيذ الفاشل.
