# التكوين المتقدم

## وضع التنفيذ

تُنفذ سير العمل إما بشكل "غير متزامن" أو "متزامن"، وذلك بناءً على نوع المشغل الذي يتم اختياره عند الإنشاء. في الوضع غير المتزامن، بعد تشغيل حدث معين، يدخل سير العمل قائمة الانتظار ليتم تنفيذه واحدًا تلو الآخر بواسطة جدولة الخلفية. أما الوضع المتزامن، فلا يدخل سير العمل قائمة الانتظار بعد تشغيله؛ بل يبدأ التنفيذ مباشرة ويقدم استجابة فورية عند الانتهاء.

تُنفذ أحداث المجموعة، وأحداث ما بعد الإجراء، وأحداث الإجراءات المخصصة، وأحداث المهام المجدولة، وأحداث الموافقة بشكل غير متزامن افتراضيًا. بينما تُنفذ أحداث ما قبل الإجراء بشكل متزامن افتراضيًا. تدعم أحداث المجموعة وأحداث النموذج كلا الوضعين، ويمكن اختيار الوضع عند إنشاء سير العمل:

![وضع المتزامن_إنشاء سير عمل متزامن](https://static-docs.nocobase.com/39bc0821f50c1bde4729c531c6236795.png)

:::info{title=ملاحظة}
بسبب طبيعتها، لا يمكن لسير العمل المتزامنة استخدام العقد التي تنتج حالة "انتظار"، مثل "المعالجة اليدوية".
:::

## الحذف التلقائي لسجل التنفيذ

عندما يتم تشغيل سير العمل بشكل متكرر، يمكنك تكوين الحذف التلقائي لسجل التنفيذ لتقليل التداخل وتخفيف ضغط التخزين على قاعدة البيانات.

يمكنك أيضًا تكوين ما إذا كان سيتم حذف سجل التنفيذ لسير العمل تلقائيًا في مربعات حوار الإنشاء والتحرير الخاصة به:

![تكوين الحذف التلقائي لسجل التنفيذ](https://static-docs.nocobase.com/b2e4c08e7a01e213069912fe04baa7bd.png)

يمكن تكوين الحذف التلقائي بناءً على حالة نتيجة التنفيذ. في معظم الحالات، يُوصى بتحديد حالة "مكتمل" فقط، وذلك للاحتفاظ بسجلات عمليات التنفيذ الفاشلة لتسهيل استكشاف الأخطاء وإصلاحها لاحقًا.

يُنصح بعدم تمكين الحذف التلقائي لسجل التنفيذ عند تصحيح أخطاء سير العمل، وذلك لتتمكن من استخدام السجل للتحقق مما إذا كان منطق تنفيذ سير العمل يطابق التوقعات.

:::info{title=ملاحظة}
لا يؤدي حذف سجل سير العمل إلى تقليل عدد مرات تنفيذه.
:::
