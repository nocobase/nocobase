# نظرة عامة

توفر إضافة تصور البيانات في NocoBase إمكانية استعلام البيانات المرئي ومجموعة غنية من مكونات الرسوم البيانية. من خلال إعدادات بسيطة، يمكن للمستخدمين إنشاء لوحات معلومات بسرعة، وعرض الرؤى، ودعم التحليل والعرض متعدد الأبعاد للبيانات.

![clipboard-image-1761749573](https://static-docs.nocobase.com/clipboard-image-1761749573.png)

## المفاهيم الأساسية
- كتلة الرسم البياني: مكون رسم بياني قابل للتكوين في الصفحة يدعم استعلام البيانات، وخيارات الرسم البياني، وأحداث التفاعل.
- استعلام البيانات (Builder / SQL): الحصول على البيانات عن طريق التكوين الرسومي باستخدام Builder، أو بكتابة استعلام SQL.
- المقاييس (Measures) والأبعاد (Dimensions): تُستخدم المقاييس للتجميع العددي، بينما تُستخدم الأبعاد للتجميع (مثل التاريخ، الفئة، المنطقة).
- تعيين الحقول: ربط أعمدة نتائج الاستعلام بحقول الرسم البياني الأساسية، مثل `xField`، `yField`، `seriesField`، أو `Category / Value`.
- خيارات الرسم البياني (Basic / Custom): تقوم Basic بتكوين الخصائص الشائعة بشكل رسومي؛ بينما تُرجع Custom كائن `option` كامل من ECharts عبر JavaScript.
- تشغيل الاستعلام: تشغيل طلب الاستعلام لجلب البيانات في لوحة التكوين، ويمكن التبديل بين `Table` / `JSON` لعرض البيانات المُرجعة.
- المعاينة والحفظ: المعاينة هي تأثير مؤقت؛ بعد النقر على "حفظ"، يتم كتابة التكوين في قاعدة البيانات ويصبح ساري المفعول رسميًا.
- متغيرات السياق: إعادة استخدام معلومات السياق للصفحة والمستخدم وعوامل التصفية (مثل `{{ ctx.user.id }}`) لاستخدامها في الاستعلامات وتكوين الرسم البياني.
- عوامل التصفية والربط على مستوى الصفحة: تقوم "كتل عوامل التصفية" على مستوى الصفحة بجمع شروط الإدخال الموحدة، ودمجها تلقائيًا في استعلامات الرسم البياني وتحديث الرسوم البيانية المرتبطة.
- أحداث التفاعل: تسجيل الأحداث عبر `chart.on` لتحقيق سلوكيات مثل التمييز، الانتقال، والتفصيل (drill-down).

## التثبيت
تصور البيانات هي إضافة مدمجة في NocoBase، جاهزة للاستخدام فورًا ولا تتطلب تثبيتًا منفصلاً.
