# سجلات الخادم، سجلات التدقيق، وسجل التغييرات

## سجلات الخادم

### سجلات النظام

> راجع [سجلات النظام](#)

- تسجل معلومات تشغيل نظام التطبيق، وتتبع مسارات تنفيذ منطق الكود، وتتعقب الأخطاء والاستثناءات في تشغيل الكود.
- يتم تصنيف السجلات حسب مستويات الخطورة والوحدات الوظيفية.
- تُخرج عبر الطرفية أو تُخزن كملفات.
- تُستخدم بشكل أساسي لتشخيص واستكشاف أخطاء النظام أثناء التشغيل.

### سجلات الطلبات

> راجع [سجلات الطلبات](#)

- تسجل تفاصيل طلبات واستجابات HTTP API، مع التركيز على معرف الطلب (Request ID)، مسار API (API Path)، رؤوس الطلب (Headers)، رمز حالة الاستجابة (Response Status Code)، والمدة الزمنية (Duration).
- تُخرج عبر الطرفية أو تُخزن كملفات.
- تُستخدم بشكل أساسي لتتبع استدعاءات API وأداء تنفيذها.

## سجلات التدقيق

> راجع [سجلات التدقيق](../security/audit-logger/index.md)

- تسجل إجراءات المستخدمين (أو API) على موارد النظام، مع التركيز على نوع المورد، الكائن المستهدف، نوع العملية، معلومات المستخدم، وحالة العملية.
- لتتبع ما فعله المستخدمون والنتائج التي تم إنتاجها بشكل أفضل، يتم تخزين معلمات الطلب والاستجابات كبيانات وصفية (Metadata). يتداخل هذا جزئيًا مع سجلات الطلبات ولكنه ليس متطابقًا تمامًا—على سبيل المثال، لا تتضمن سجلات الطلبات عادةً نصوص الطلبات الكاملة.
- معلمات الطلب والاستجابات **ليست مكافئة** لقطات البيانات. يمكنها الكشف عن نوع العمليات التي حدثت ولكن ليس البيانات الدقيقة قبل التعديل، وبالتالي لا يمكن استخدامها للتحكم في الإصدارات أو استعادة البيانات بعد العمليات الخاطئة.
- تُخزن كملفات وجداول قاعدة بيانات.

## سجل التغييرات

> راجع [سجل التغييرات](/record-history/index.md)

- يسجل **تاريخ التغييرات** لمحتوى البيانات.
- يتتبع نوع المورد، الكائن المستهدف، نوع العملية، الحقول المتغيرة، والقيم قبل وبعد التغيير.
- مفيد **لمقارنة البيانات والتدقيق**.
- يُخزن في جداول قاعدة البيانات.
