# سجلات الخادم، سجلات التدقيق، وسجل التغييرات

## سجلات الخادم

### سجلات النظام

> راجع [سجلات النظام](#)

- تسجل معلومات تشغيل نظام التطبيق، وتتبع سلاسل تنفيذ منطق التعليمات البرمجية، وتتعقب الأخطاء أو الاستثناءات أثناء تشغيل التعليمات البرمجية.
- تُصنف السجلات حسب مستويات الخطورة والوحدات الوظيفية.
- تُعرض عبر الطرفية أو تُخزن كملفات.
- تُستخدم بشكل أساسي لتشخيص واستكشاف أخطاء النظام وإصلاحها أثناء التشغيل.

### سجلات الطلبات

> راجع [سجلات الطلبات](#)

- تسجل تفاصيل طلبات واستجابات HTTP API، مع التركيز على معرف الطلب (Request ID)، مسار الـ API (API Path)، الرؤوس (Headers)، رمز حالة الاستجابة (Response Status Code)، والمدة الزمنية (Duration)، وغيرها من المعلومات.
- تُعرض عبر الطرفية أو تُخزن كملفات.
- تُستخدم بشكل أساسي لتتبع استدعاءات الـ API وأداء تنفيذها.

## سجلات التدقيق

> راجع [سجلات التدقيق](/security/audit-logger/index.md)

- تسجل إجراءات المستخدمين (أو الـ API) على موارد النظام، مع التركيز على نوع المورد، الكائن المستهدف، نوع العملية، معلومات المستخدم، وحالة العملية، وغيرها.
- لتتبع محتوى ونتائج عمليات المستخدمين بشكل أفضل، تُسجل معلمات الطلب والاستجابات كمعلومات MetaData. تتداخل هذه المعلومات جزئيًا مع سجلات الطلبات ولكنها ليست متطابقة تمامًا؛ على سبيل المثال، لا تتضمن سجلات الطلبات الحالية عادةً نص الطلب الكامل (full request body).
- معلمات الطلب والاستجابات **ليست مكافئة** لقطات البيانات (data snapshots). يمكنها الكشف عن نوع العمليات التي حدثت ولكنها لا تستطيع تحديد البيانات الدقيقة قبل التعديل، وبالتالي لا يمكن استخدامها للتحكم في الإصدارات (version control) أو استعادة البيانات بعد العمليات الخاطئة.
- تُخزن كملفات وجداول قواعد بيانات.

## سجل التغييرات

> راجع [سجل التغييرات](/record-history/index.md)

- يسجل **تاريخ التغييرات** لمحتوى البيانات.
- يتتبع نوع المورد، الكائن المستهدف، نوع العملية، الحقول التي تم تغييرها، والقيم قبل وبعد التغيير.
- مفيد لـ **مقارنة البيانات والتدقيق**.
- يُخزن في جداول قواعد البيانات.
