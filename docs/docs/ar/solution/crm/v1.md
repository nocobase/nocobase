# نظام إدارة علاقات العملاء القديم (CRM v1)

> **للمعاينة فقط**: هذا الإصدار مكتمل الميزات نسبيًا، لكن التصميم العام لجداول البيانات غير مثالي، ويفتقر إلى دعم الترجمة المتعددة اللغات، وهناك تداخل بين مجموعات (Collections) CRM ومجموعات إدارة المشاريع. أدت هذه المشكلات إلى إعادة تصميم CRM 2.0. يُنصح حاليًا باستخدام [إصدار CRM 2.0](./index.md)، الذي يتميز بهيكل بيانات أكثر منطقية، وسهولة في التوسع والتعديل، ودعم كامل للغات المتعددة.

## مميزات الإصدار

يتضمن نظام CRM القديم الوحدات الوظيفية التالية:

- **إدارة العملاء**: معلومات العملاء، جهات الاتصال، وتصنيف العملاء.
- **مسار المبيعات**: إدارة العملاء المحتملين، متابعة الفرص البيعية، وقمع المبيعات.
- **إدارة الطلبات**: عروض الأسعار، الطلبات، والعقود.
- **إدارة المشاريع** (غير مدرجة حاليًا في إصدار v2).
- **التواصل والتعاون**: التواصل عبر البريد الإلكتروني، التعليقات، وعمليات سير العمل للموافقات.
- **تحليل البيانات**: لوحات المعلومات، التقارير، وتصور البيانات.

## كيفية التثبيت

> يتم نشر هذا الإصدار حاليًا عن طريق النسخ الاحتياطي والاستعادة.

قبل البدء، يرجى التأكد مما يلي:

- لديك بيئة تشغيل NocoBase أساسية. لتثبيت النظام الرئيسي، يرجى الرجوع إلى [وثائق التثبيت الرسمية](https://docs-cn.nocobase.com/welcome/getting-started/installation) المفصلة.
- إصدار NocoBase هو **v2.1.0-beta.2 أو أحدث**.
- قمت بتنزيل الملفات الخاصة بـ CRM v1:
  - **ملف النسخ الاحتياطي**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — ينطبق على الطريقة الأولى.
  - **ملف SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — ينطبق على الطريقة الثانية.

**ملاحظات هامة**:
- يعتمد هذا الحل على قاعدة بيانات **PostgreSQL 16**، يرجى التأكد من أن بيئتك تستخدم PostgreSQL 16.
- **يجب ألا تكون قيمة DB_UNDERSCORED هي true**: يرجى التحقق من ملف `docker-compose.yml` والتأكد من عدم تعيين متغير البيئة `DB_UNDERSCORED` إلى `true`؛ وإلا فسيحدث تعارض مع النسخة الاحتياطية مما يؤدي إلى فشل الاستعادة.

## الطريقة الأولى: الاستعادة باستخدام مدير النسخ الاحتياطي (موصى بها لمستخدمي النسخة الاحترافية/المؤسسات)

تتم هذه الطريقة من خلال إضافة "**مدير النسخ الاحتياطي**" (النسخة الاحترافية/المؤسسات) المدمجة في NocoBase للاستعادة بنقرة واحدة، وهي الطريقة الأسهل.

### خطوات العمل

**الخطوة 1: تفعيل إضافة "مدير النسخ الاحتياطي"**

1. قم بتسجيل الدخول إلى نظام NocoBase الخاص بك.
2. انتقل إلى **`إدارة الإضافات`**.
3. ابحث عن إضافة **`مدير النسخ الاحتياطي`** وقم بتفعيلها.

**الخطوة 2: الاستعادة من ملف نسخ احتياطي محلي**

1. بعد تفعيل الإضافة، قم بتحديث الصفحة.
2. انتقل إلى القائمة اليسرى: **`إدارة النظام`** -> **`مدير النسخ الاحتياطي`**.
3. انقر على زر **`استعادة من نسخة احتياطية محلية`** في الزاوية العلوية اليسرى.
4. قم بسحب وإفلات ملف النسخ الاحتياطي الذي قمت بتنزيله إلى منطقة التحميل.
5. انقر على **`إرسال`**، وانتظر بصبر حتى يكمل النظام عملية الاستعادة.

### ملاحظات

* **مقتصر على النسخة الاحترافية/المؤسسات**: "مدير النسخ الاحتياطي" هي إضافة مخصصة للمؤسسات، ومتاحة فقط لمستخدمي النسخة الاحترافية أو نسخة المؤسسات.
* **توافق الإضافات التجارية**: يرجى التأكد من حصولك على الإضافات التجارية المطلوبة للحل وتفعيلها.

## الطريقة الثانية: استيراد ملف SQL مباشرة (عامة)

تتم هذه الطريقة عن طريق التعامل المباشر مع قاعدة البيانات لاستعادة البيانات، وهي صالحة لجميع مستخدمي NocoBase.

### خطوات العمل

**الخطوة 1: إعداد قاعدة بيانات نظيفة**

قم بإعداد قاعدة بيانات جديدة وفارغة للبيانات التي ستستوردها.

**الخطوة 2: استيراد ملف `.sql` إلى قاعدة البيانات**

* **عبر سطر الأوامر (مثال Docker)**:

  ```bash
  # نسخ ملف sql إلى داخل الحاوية
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # الدخول إلى الحاوية وتنفيذ أمر الاستيراد
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **عبر عميل قاعدة البيانات**: استخدم أدوات مثل DBeaver أو Navicat أو pgAdmin للاتصال بقاعدة البيانات وتنفيذ ملف SQL.

**الخطوة 3: ربط قاعدة البيانات وتشغيل التطبيق**

قم بتكوين معاملات تشغيل NocoBase لتشير إلى قاعدة البيانات التي تم استيراد البيانات إليها، ثم ابدأ تشغيل الخدمة.
