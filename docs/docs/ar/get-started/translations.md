# المساهمة في الترجمة

اللغة الافتراضية لـ NocoBase هي الإنجليزية. حاليًا، يدعم التطبيق الرئيسي اللغات الإنجليزية، والإيطالية، والهولندية، والصينية المبسطة، واليابانية. ندعوكم بصدق للمساهمة في ترجمة لغات إضافية، مما يتيح للمستخدمين حول العالم الاستمتاع بتجربة NocoBase أكثر سهولة وسلاسة.

---

## أولاً: تعريب النظام (System Localization)

### 1. ترجمة واجهة النظام والإضافات

#### 1.1 نطاق الترجمة
ينطبق هذا فقط على تعريب واجهة نظام NocoBase والإضافات (Plugins)، ولا يشمل المحتويات المخصصة الأخرى (مثل جداول البيانات أو كتل Markdown).

![bbb6e0b44aeg](https://static-docs.nocobase.com/img_v3_02kh_8d429938-3aca-44b6-a437-bbb6e0b44aeg.jpg)

![20250319220127](https://static-docs.nocobase.com/20250319220127.png)


#### 1.2 نظرة عامة على محتوى التعريب
يستخدم NocoBase نظام Git لإدارة محتوى التعريب. المستودع الرئيسي هو:
https://github.com/nocobase/nocobase/tree/main/locales

يتم تمثيل كل لغة بملف JSON مسمى وفقًا لرمز اللغة (على سبيل المثال: de-DE.json، fr-FR.json). يتم تنظيم هيكل الملف حسب وحدات الإضافات، باستخدام أزواج "المفتاح-القيمة" لتخزين الترجمات. مثال:

```json
{
  // إضافة العميل (Client plugin)
  "@nocobase/client": {
    "(Fields only)": "(Fields only)",
    "12 hour": "12 hour",
    "24 hour": "24 hour"
    // ...أزواج مفتاح-قيمة أخرى
  },
  "@nocobase/plugin-acl": {
    // أزواج مفتاح-قيمة لهذه الإضافة
  }
  // ...وحدات إضافات أخرى
}
```

عند الترجمة، يرجى تحويلها تدريجيًا إلى هيكل مشابه لما يلي:

```json
{
  // إضافة العميل (Client plugin)
  "@nocobase/client": {
    "(Fields only)": "(الحقول فقط - مترجم)",
    "12 hour": "12 ساعة",
    "24 hour": "24 ساعة"
    // ...أزواج مفتاح-قيمة أخرى
  },
  "@nocobase/plugin-acl": {
    // أزواج مفتاح-قيمة لهذه الإضافة
  }
  // ...وحدات إضافات أخرى
}
```

#### 1.3 اختبار الترجمة والمزامنة
- بعد الانتهاء من الترجمة، يرجى اختبار والتحقق من أن جميع النصوص تظهر بشكل صحيح.
لقد أصدرنا أيضًا إضافة للتحقق من الترجمة - ابحث عن `Locale tester` في سوق الإضافات.
![20250422233152](https://static-docs.nocobase.com/20250422233152.png)
بعد التثبيت، انسخ محتوى JSON من ملف التعريب المقابل في مستودع git، والصقه بداخله، ثم انقر فوق "موافق" للتحقق مما إذا كان محتوى الترجمة فعالاً.
![20250422233950](https://static-docs.nocobase.com/20250422233950.png)

- بمجرد التقديم، ستقوم نصوص النظام البرمجية تلقائيًا بمزامنة محتوى التعريب مع مستودع الكود.

## ثانياً: تعريب الوثائق (NocoBase 2.0)

يتم إدارة وثائق NocoBase 2.0 في هيكل جديد. توجد ملفات مصدر الوثائق في مستودع NocoBase الرئيسي:

https://github.com/nocobase/nocobase/tree/next/docs

### 2.1 هيكل الوثائق

تستخدم الوثائق [Rspress](https://rspress.dev/) كمولد للمواقع الثابتة وتدعم 22 لغة. يتم تنظيم الهيكل على النحو التالي:

```
docs/
├── docs/
│   ├── en/                    # الإنجليزية (اللغة المصدر)
│   ├── cn/                    # الصينية المبسطة
│   ├── ja/                    # اليابانية
│   ├── ko/                    # الكورية
│   ├── de/                    # الألمانية
│   ├── fr/                    # الفرنسية
│   ├── es/                    # الإسبانية
│   ├── pt/                    # البرتغالية
│   ├── ru/                    # الروسية
│   ├── it/                    # الإيطالية
│   ├── tr/                    # التركية
│   ├── uk/                    # الأوكرانية
│   ├── vi/                    # الفيتنامية
│   ├── id/                    # الإندونيسية
│   ├── th/                    # التايلاندية
│   ├── pl/                    # البولندية
│   ├── nl/                    # الهولندية
│   ├── cs/                    # التشيكية
│   ├── ar/                    # العربية
│   ├── he/                    # العبرية
│   ├── hi/                    # الهندية
│   ├── sv/                    # السويدية
│   └── public/                # الأصول المشتركة (الصور، إلخ)
├── theme/                     # القالب المخصص
├── rspress.config.ts          # إعدادات Rspress
└── package.json
```

### 2.2 سير عمل الترجمة

1. **المزامنة مع المصدر الإنجليزي**: يجب أن تستند جميع الترجمات إلى الوثائق الإنجليزية (`docs/en/`). عند تحديث الوثائق الإنجليزية، يجب تحديث الترجمات وفقًا لذلك.

2. **استراتيجية الفروع (Branch strategy)**:
   - استخدم فرع `develop` أو `next` كمرجع لأحدث محتوى باللغة الإنجليزية.
   - أنشئ فرع الترجمة الخاص بك من الفرع المستهدف.

3. **هيكل الملفات**: يجب أن يعكس دليل كل لغة هيكل الدليل الإنجليزي. على سبيل المثال:
   ```
   docs/en/get-started/index.md    →    docs/ar/get-started/index.md
   docs/en/api/acl/acl.md          →    docs/ar/api/acl/acl.md
   ```

### 2.3 المساهمة في الترجمات

1. قم بعمل Fork للمستودع: https://github.com/nocobase/nocobase
2. قم بعمل Clone للـ fork الخاص بك وانتقل إلى فرع `develop` أو `next`.
3. انتقل إلى دليل `docs/docs/`.
4. ابحث عن دليل اللغة التي تريد المساهمة فيها (على سبيل المثال، `ar/` للغة العربية).
5. ترجم ملفات markdown، مع الحفاظ على نفس هيكل الملفات كما في الإصدار الإنجليزي.
6. اختبر تغييراتك محليًا:
   ```bash
   cd docs
   yarn install
   yarn dev
   ```
7. أرسل طلب سحب (Pull Request) إلى المستودع الرئيسي.

### 2.4 إرشادات الترجمة

- **الحفاظ على اتساق التنسيق**: حافظ على نفس هيكل markdown، والعناوين، وكتل الكود، والروابط كما في المصدر.
- **الحفاظ على frontmatter**: حافظ على أي YAML frontmatter في الجزء العلوي من الملفات دون تغيير، إلا إذا كان يحتوي على محتوى قابل للترجمة.
- **مراجع الصور**: استخدم نفس مسارات الصور من `docs/public/` - الصور مشتركة بين جميع اللغات.
- **الروابط الداخلية**: قم بتحديث الروابط الداخلية لتشير إلى مسار اللغة الصحيح.
- **أمثلة الكود**: بشكل عام، لا ينبغي ترجمة أمثلة الكود، ولكن يمكن ترجمة التعليقات داخل الكود.

### 2.5 إعدادات التنقل (Navigation)

يتم تحديد هيكل التنقل لكل لغة في ملفات `_nav.json` و `_meta.json` داخل دليل كل لغة. عند إضافة صفحات أو أقسام جديدة، تأكد من تحديث ملفات الإعداد هذه.

### 3.2 هيكل المحتوى وطريقة التعريب

نحن نستخدم نهجًا مختلطًا لإدارة المحتوى. يتم مزامنة المحتوى والموارد باللغات الإنجليزية والصينية واليابانية بانتظام من نظام CMS واستبدالها، بينما يمكن تحرير اللغات الأخرى مباشرة في الملفات المحلية. يتم تخزين المحتوى المحلي في دليل `content` المنظم كما يلي:

```
/content
  /articles        # مقالات المدونة
    /article-slug
      index.md     # المحتوى الإنجليزي (افتراضي)
      index.cn.md  # المحتوى الصيني
      index.ja.md  # المحتوى الياباني
      metadata.json # البيانات الوصفية وخصائص التعريب الأخرى
  /tutorials       # البرامج التعليمية
  /releases        # معلومات الإصدار
  /pages           # بعض الصفحات الثابتة
  /categories      # معلومات الفئات
    /article-categories.json  # قائمة فئات المقالات
    /category-slug            # تفاصيل الفئة الفردية
      /category.json
  /tags            # معلومات الوسوم
    /article-tags.json        # قائمة وسوم المقالات
    /release-tags.json        # قائمة وسوم الإصدارات
    /tag-slug                 # تفاصيل الوسم الفردي
      /tag.json
  /help-center     # محتوى مركز المساعدة
    /help-center-tree.json    # هيكل التنقل في مركز المساعدة
  ....
```

### 3.3 إرشادات ترجمة المحتوى

- **حول ترجمة محتوى Markdown**

1. أنشئ ملف لغة جديدًا بناءً على الملف الافتراضي (على سبيل المثال، من `index.md` إلى `index.ar.md`).
2. أضف الخصائص المترجمة في الحقول المقابلة في ملف JSON.
3. حافظ على الاتساق في هيكل الملفات، والروابط، ومراجع الصور.

- **ترجمة محتوى JSON**
يتم تخزين العديد من البيانات الوصفية للمحتوى في ملفات JSON، والتي تحتوي عادةً على حقول متعددة اللغات:

```json
{
  "id": 123,
  "title": "English Title",       // العنوان الإنجليزي (افتراضي)
  "title_cn": "中文标题",          // العنوان الصيني
  "title_ja": "日本語タイトル",    // العنوان الياباني
  "description": "English description",
  "description_cn": "中文描述",
  "description_ja": "日本語の説明",
  "slug": "article-slug",         // مسار URL (عادة لا يترجم)
  "status": "published",
  "publishedAt": "2025-03-19T12:00:00Z"
}
```

**ملاحظات الترجمة:**

1. **اصطلاح تسمية الحقول**: تستخدم حقول الترجمة عادةً تنسيق `{الحقل_الأصلي}_{رمز_اللغة}`.
   - على سبيل المثال: title_ar (العنوان العربي)، description_de (الوصف الألماني).

2. **عند إضافة لغة جديدة**:
   - أضف نسخة لاحقة للغة المقابلة لكل حقل يحتاج إلى ترجمة.
   - لا تقم بتعديل قيم الحقول الأصلية (مثل title، description، إلخ)، لأنها تعمل كمحتوى اللغة الافتراضية (الإنجليزية).

3. **آلية مزامنة CMS**:
   - يقوم نظام CMS بتحديث المحتوى باللغات الإنجليزية والصينية واليابانية بشكل دوري.
   - سيقوم النظام فقط بتحديث/استبدال المحتوى لهذه اللغات الثلاث (بعض الخصائص في JSON)، ولن يحذف حقول اللغة التي أضافها مساهمون آخرون.
   - على سبيل المثال: إذا أضفت ترجمة عربية (title_ar)، فلن تؤثر مزامنة CMS على هذا الحقل.


### 3.4 إعداد دعم لغة جديدة

لإضافة دعم للغة جديدة، تحتاج إلى تعديل إعداد `SUPPORTED_LANGUAGES` في ملف `src/utils/index.ts`:

```typescript
export const SUPPORTED_LANGUAGES = {
  en: {
    code: 'en',
    locale: 'en-US',
    name: 'English',
    default: true
  },
  cn: {
    code: 'cn',
    locale: 'zh-CN',
    name: 'Chinese'
  },
  ja: {
    code: 'ja',
    locale: 'ja-JP',
    name: 'Japanese'
  },
  // مثال على إضافة لغة جديدة:
  ar: {
    code: 'ar',
    locale: 'ar-SA',
    name: 'Arabic'
  }
};
```

### 3.5 ملفات القوالب (Layout) والأنماط

تحتاج كل لغة إلى ملفات قوالب مقابلة:

1. أنشئ ملف قالب جديد (على سبيل المثال، للغة العربية، أنشئ `src/layouts/BaseAR.astro`).
2. يمكنك نسخ ملف قالب موجود (مثل `BaseEN.astro`) وترجمته.
3. يحتوي ملف القالب على ترجمات للعناصر العالمية مثل قوائم التنقل، والتذييلات، إلخ.
4. تأكد من تحديث إعدادات مبدل اللغة للتبديل بشكل صحيح إلى اللغة المضافة حديثًا.

### 3.6 إنشاء أدلة صفحات اللغة

أنشئ أدلة صفحات مستقلة للغة الجديدة:

1. أنشئ مجلدًا مسمى برمز اللغة في دليل `src` (على سبيل المثال `src/ar/`).
2. انسخ هيكل الصفحات من أدلة اللغات الأخرى (مثل `src/en/`).
3. قم بتحديث محتوى الصفحة، وترجمة العناوين والأوصاف والنصوص إلى اللغة المستهدفة.
4. تأكد من أن الصفحات تستخدم مكون القالب الصحيح (على سبيل المثال، `.layout: '@/layouts/BaseAR.astro'`).

### 3.7 تعريب المكونات

تحتاج بعض المكونات الشائعة أيضًا إلى ترجمة:

1. تحقق من المكونات في دليل `src/components/`.
2. انتبه بشكل خاص للمكونات ذات النصوص الثابتة (مثل أشرطة التنقل، والتذييلات، إلخ).
3. قد تستخدم المكونات العرض الشرطي (conditional rendering) لعرض المحتوى بلغات مختلفة:

```astro
{Astro.url.pathname.startsWith('/en') && <p>English content</p>}
{Astro.url.pathname.startsWith('/ar') && <p>محتوى باللغة العربية</p>}
```

### 3.8 الاختبار والتحقق

بعد الانتهاء من الترجمة، قم بإجراء اختبار شامل:

1. قم بتشغيل الموقع محليًا (عادةً باستخدام `yarn dev`).
2. تحقق من كيفية عرض جميع الصفحات باللغة الجديدة.
3. تأكد من أن وظيفة تبديل اللغة تعمل بشكل صحيح.
4. تأكد من أن جميع الروابط تشير إلى إصدارات صفحات اللغة الصحيحة.
5. تحقق من التصميمات المتجاوبة للتأكد من أن النص المترجم لا يفسد تصميم الصفحة.
