# نظرة عامة

## المقدمة

تُستخدم محرّكات التخزين لحفظ الملفات في خدمات محددة، مثل التخزين المحلي (المحفوظ على القرص الصلب للخادم) أو التخزين السحابي وغيرها.

قبل رفع أي ملفات، يجب إعداد محرك تخزين. يقوم النظام تلقائيًا بإضافة محرك تخزين محلي أثناء التثبيت، ويمكن استخدامه مباشرة. كما يمكنك إضافة محركات جديدة أو تعديل إعدادات المحركات الموجودة.

## المعلمات العامة

بالإضافة إلى المعلمات الخاصة بكل نوع من المحركات، توجد المعلمات العامة التالية (باستخدام التخزين المحلي كمثال):

![مثال على إعداد محرك تخزين الملفات](https://static-docs.nocobase.com/20240529115151.png)

### العنوان (Title)

اسم محرك التخزين، ويُستخدم للتعرّف عليه من قبل المستخدم.

### اسم النظام (System Name)

الاسم الداخلي لمحرك التخزين المستخدم من قبل النظام للتعرّف عليه. يجب أن يكون فريدًا داخل النظام. إذا تُرك فارغًا، سيقوم النظام بإنشاء اسم عشوائي تلقائيًا.

### بادئة الرابط العام (Public URL prefix)

الجزء الأول من رابط URL العام الذي يمكن من خلاله الوصول إلى الملف. يمكن أن يكون رابط CDN، مثل:  
`https://cdn.nocobase.com/app` (بدون "/" في النهاية).

### المسار (Path)

المسار النسبي المستخدم عند تخزين الملفات. يتم أيضًا إضافته تلقائيًا إلى الرابط النهائي عند الوصول. مثال:  
`user/avatar` (بدون "/" في البداية أو النهاية).

### حد حجم الملف (File size limit)

الحد الأقصى لحجم الملفات التي يمكن رفعها إلى هذا المحرك. لا يمكن رفع الملفات التي تتجاوز هذا الحجم. الحد الافتراضي هو 20MB، ويمكن زيادته حتى 1GB كحد أقصى.

### أنواع الملفات (File types)

يمكنك تقييد أنواع الملفات المسموح برفعها باستخدام صيغة [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types).  
على سبيل المثال: `image/*` تمثل جميع ملفات الصور.  
يمكن تحديد عدة أنواع مفصولة بفواصل، مثل: `image/*, application/pdf` للسماح برفع الصور وملفات PDF.

### محرك التخزين الافتراضي (Default storage engine)

عند تفعيله، يتم تعيين هذا المحرك كمحرك التخزين الافتراضي للنظام.  
إذا لم يتم تحديد محرك تخزين لحقل مرفقات أو مجموعة ملفات، سيتم حفظ الملفات في المحرك الافتراضي.  
لا يمكن حذف محرك التخزين الافتراضي.

### الاحتفاظ بالملف عند حذف السجل (Keep file when record is deleted)

عند تفعيله، سيتم الاحتفاظ بالملف داخل محرك التخزين حتى إذا تم حذف السجل المرتبط به.  
بشكل افتراضي، هذا الخيار غير مفعّل، مما يعني أن الملف سيتم حذفه مع السجل.

:::info{title=معلومة}
بعد رفع الملف، يتم تكوين رابط الوصول النهائي عن طريق دمج عدة أجزاء:

```
<Public URL prefix>/<Path>/<Filename><Extension>
```

مثال: `https://cdn.nocobase.com/app/user/avatar/20240529115151.png`.
:::
