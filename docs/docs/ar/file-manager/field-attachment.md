# المرفقات

## المقدمة

يحتوي النظام على نوع حقل مدمج باسم "المرفقات" (Attachment) لدعم رفع الملفات داخل المجموعات المخصصة.

يُعد حقل المرفقات في الأساس حقل علاقة من نوع Many-to-Many يشير إلى مجموعة "Attachments" (`attachments`) المدمجة. عند إنشاء حقل مرفقات داخل أي مجموعة، يتم تلقائيًا إنشاء جدول وسيط (junction table) لإدارة العلاقة متعددة-إلى-متعددة. يتم تخزين بيانات الملفات (metadata) داخل مجموعة "Attachments"، ويتم ربط الملفات مع السجلات داخل المجموعة عبر هذا الجدول الوسيط.

### تقييد نوع MIME

يُستخدم لتحديد أنواع الملفات المسموح برفعها باستخدام صيغة [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types).  
على سبيل المثال: `image/*` تمثل جميع ملفات الصور.  
يمكن تحديد عدة أنواع مفصولة بفاصلة، مثل: `image/*,application/pdf` للسماح برفع الصور وملفات PDF.

### محرك التخزين

اختر محرك التخزين الذي سيتم استخدامه لحفظ الملفات المرفوعة.  
إذا تُرك هذا الحقل فارغًا، سيتم استخدام محرك التخزين الافتراضي للنظام.
