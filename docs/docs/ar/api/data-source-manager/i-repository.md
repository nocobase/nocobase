# `IRepository`

تُعرّف واجهة `Repository` مجموعة من طرق التعامل مع النماذج، وذلك لتكييف عمليات الإنشاء والقراءة والتحديث والحذف (CRUD) الخاصة بـ مصدر البيانات.

## واجهة برمجة التطبيقات (API)

### `find()`

تُرجع قائمة بالنماذج التي تتوافق مع معلمات الاستعلام.

#### التوقيع

- `find(options?: any): Promise<IModel[]>`

### `findOne()`

تُرجع نموذجًا يتوافق مع معلمات الاستعلام. إذا كان هناك عدة نماذج متطابقة، فسيتم إرجاع الأول فقط.

#### التوقيع

- `findOne(options?: any): Promise<IModel>`

### `count()`

تُرجع عدد النماذج التي تتوافق مع معلمات الاستعلام.

#### التوقيع

- `count(options?: any): Promise<Number>`

### `findAndCount()`

تُرجع قائمة النماذج وعددها التي تتوافق مع معلمات الاستعلام.

#### التوقيع

- `findAndCount(options?: any): Promise<[IModel[], Number]>`

### `create()`

تُنشئ كائن بيانات نموذج.

#### التوقيع

- `create(options: any): void`

### `update()`

تُحدّث كائن بيانات نموذج بناءً على شروط الاستعلام.

#### التوقيع

- `update(options: any): void`

### `destroy()`

تحذف كائن بيانات نموذج بناءً على شروط الاستعلام.

#### التوقيع

- `destroy(options: any): void`
