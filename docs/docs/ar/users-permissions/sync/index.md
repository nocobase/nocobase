# مزامنة بيانات المستخدم

## مقدمة

تتيح لك هذه الميزة تسجيل وإدارة مصادر مزامنة بيانات المستخدم. يتم توفير واجهة برمجة تطبيقات HTTP (HTTP API) بشكل افتراضي، ولكن يمكن دعم مصادر بيانات إضافية عبر الإضافات. يدعم بشكل افتراضي مزامنة البيانات مع مجموعتي **المستخدمين** و**الأقسام**، مع إمكانية توسيع المزامنة لتشمل موارد مستهدفة أخرى باستخدام الإضافات.

## إدارة مصادر البيانات والمزامنة

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
إذا لم يتم تثبيت أي إضافات توفر مصادر مزامنة بيانات المستخدم، يمكن مزامنة بيانات المستخدم باستخدام واجهة برمجة تطبيقات HTTP (HTTP API). راجع [مصدر البيانات - HTTP API](./sources/api.md).
:::

## إضافة مصدر بيانات

بمجرد تثبيت إضافة توفر مصدر مزامنة بيانات المستخدم، يمكنك إضافة مصدر البيانات المقابل. فقط مصادر البيانات المُمكّنة هي التي ستعرض زري "مزامنة" و"المهمة".

> مثال: WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## مزامنة البيانات

انقر على زر **مزامنة** لبدء مزامنة البيانات.

![](https://static-docs.nocobase.com/202412041055022.png)

انقر على زر **المهمة** لعرض حالة المزامنة. بعد نجاح المزامنة، يمكنك عرض البيانات في قائمتي المستخدمين والأقسام.

![](https://static-docs.nocobase.com/202412041202337.png)

بالنسبة لمهام المزامنة الفاشلة، يمكنك النقر على **إعادة المحاولة**.

![](https://static-docs.nocobase.com/202412041058337.png)

في حال فشل المزامنة، يمكنك استكشاف المشكلة وإصلاحها عبر سجلات النظام. بالإضافة إلى ذلك، يتم تخزين سجلات المزامنة الأصلية في دليل `user-data-sync` ضمن مجلد سجلات التطبيق.

![](https://static-docs.nocobase.com/202412041205655.png)
