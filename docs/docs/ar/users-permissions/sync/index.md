# مزامنة بيانات المستخدم

## المقدمة

تتيح هذه الميزة تسجيل وإدارة مصادر مزامنة بيانات المستخدم. بشكل افتراضي، يتم توفير واجهة HTTP API، ولكن يمكن دعم مصادر بيانات إضافية من خلال الإضافات (Plugins). تدعم الميزة مزامنة البيانات إلى مجموعتي **Users** و **Departments** بشكل افتراضي، مع إمكانية توسيع المزامنة لتشمل موارد هدف أخرى باستخدام الإضافات.

## إدارة مصدر البيانات والمزامنة


![](https://static-docs.nocobase.com/202412041043465.png)


:::info
إذا لم يتم تثبيت أي إضافات توفر مصادر مزامنة بيانات المستخدم، يمكن مزامنة البيانات باستخدام واجهة HTTP API. راجع [مصدر البيانات - HTTP API](./sources/api.md).
:::

## إضافة مصدر بيانات

بمجرد تثبيت إضافة توفر مصدرًا لمزامنة بيانات المستخدم، يمكنك إضافة مصدر البيانات المقابل. فقط مصادر البيانات المفعّلة ستعرض زري "Sync" و "Task".

> مثال: WeCom


![](https://static-docs.nocobase.com/202412041053785.png)


## مزامنة البيانات

انقر على زر **Sync** لبدء مزامنة البيانات.


![](https://static-docs.nocobase.com/202412041055022.png)


انقر على زر **Task** لعرض حالة المزامنة. بعد نجاح المزامنة، يمكنك عرض البيانات في قوائم Users و Departments.


![](https://static-docs.nocobase.com/202412041202337.png)


بالنسبة لمهام المزامنة التي فشلت، يمكنك النقر على **Retry**.


![](https://static-docs.nocobase.com/202412041058337.png)


في حال فشل المزامنة، يمكنك استكشاف المشكلة وإصلاحها من خلال سجلات النظام. بالإضافة إلى ذلك، يتم تخزين سجلات المزامنة الخام في مجلد `user-data-sync` داخل مجلد سجلات التطبيق.


![](https://static-docs.nocobase.com/202412041205655.png)
