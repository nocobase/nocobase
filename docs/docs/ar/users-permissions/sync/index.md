# مزامنة بيانات المستخدمين

## مقدمة

تتيح لك هذه الميزة تسجيل وإدارة مصادر مزامنة بيانات المستخدمين. يتم توفير واجهة HTTP API افتراضيًا، ولكن يمكن دعم مصادر بيانات إضافية من خلال الإضافات. يدعم مزامنة البيانات إلى مجموعتي **المستخدمين** و**الأقسام** افتراضيًا، مع إمكانية توسيع المزامنة إلى موارد مستهدفة أخرى باستخدام الإضافات.

## إدارة مصدر البيانات والمزامنة

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
إذا لم يتم تثبيت أي إضافات توفر مصادر مزامنة بيانات المستخدمين، يمكن مزامنة بيانات المستخدمين باستخدام HTTP API. راجع [مصدر البيانات - HTTP API](./sources/api.md).
:::

## إضافة مصدر بيانات

بمجرد تثبيت إضافة توفر مصدر مزامنة بيانات المستخدمين، يمكنك إضافة مصدر البيانات المقابل. ستعرض مصادر البيانات المفعّلة فقط زرَّي "مزامنة" و"مهمة".

> مثال: WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## مزامنة البيانات

انقر على زر **مزامنة** لبدء مزامنة البيانات.

![](https://static-docs.nocobase.com/202412041055022.png)

انقر على زر **مهمة** لعرض حالة المزامنة. بعد المزامنة الناجحة، يمكنك عرض البيانات في قائمتي المستخدمين والأقسام.

![](https://static-docs.nocobase.com/202412041202337.png)

بالنسبة لمهام المزامنة الفاشلة، يمكنك النقر على **إعادة المحاولة**.

![](https://static-docs.nocobase.com/202412041058337.png)

في حالة فشل المزامنة، يمكنك استكشاف المشكلة وإصلاحها من خلال سجلات النظام. بالإضافة إلى ذلك، يتم تخزين سجلات المزامنة الخام في مجلد `user-data-sync` ضمن مجلد سجلات التطبيق.

![](https://static-docs.nocobase.com/202412041205655.png)
