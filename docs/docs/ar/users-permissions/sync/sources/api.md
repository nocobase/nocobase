# مزامنة بيانات المستخدمين عبر HTTP API

## الحصول على مفتاح API

راجع [مفاتيح API](/auth-verification/api-keys). تأكد من أن الدور المرتبط بمفتاح API يمتلك الصلاحيات اللازمة لمزامنة بيانات المستخدمين.

## نظرة عامة على API

### مثال

```bash
curl 'https://localhost:13000/api/userData:push' \
  -H 'Authorization: Bearer <token>' \
  --data-raw '{"dataType":"user","records":[]}'
```

### نقطة النهاية

```bash
POST /api/userData:push
```

### تنسيق بيانات المستخدم

#### UserData

| المعامل    | النوع                              | الوصف                                                              |
| ---------- | ---------------------------------- | ------------------------------------------------------------------ |
| `dataType` | `'user' \| 'department'`           | مطلوب. نوع البيانات المُرسلة. استخدم `user` لإرسال بيانات المستخدم. |
| `matchKey` | `'username' \| 'email' \| 'phone'` | اختياري. يُستخدم لمطابقة المستخدمين الموجودين في النظام.           |
| `records`  | `UserRecord[]`                     | مطلوب. مصفوفة سجلات بيانات المستخدم.                              |

#### UserRecord

| المعامل       | النوع      | الوصف                                                                        |
| ------------- | ---------- | ---------------------------------------------------------------------------- |
| `uid`         | `string`   | مطلوب. المعرف الفريد لبيانات المستخدم المصدر. غير قابل للتغيير.             |
| `nickname`    | `string`   | اختياري. اسم المستخدم المستعار.                                              |
| `username`    | `string`   | اختياري. اسم المستخدم.                                                       |
| `email`       | `string`   | اختياري. عنوان البريد الإلكتروني للمستخدم.                                  |
| `phone`       | `string`   | اختياري. رقم هاتف المستخدم.                                                  |
| `departments` | `string[]` | اختياري. مصفوفة من معرفات الأقسام التي ينتمي إليها المستخدم.               |
| `isDeleted`   | `boolean`  | اختياري. يشير إلى ما إذا كان السجل محذوفًا.                                 |
| `<field>`     | `any`      | اختياري. حقول مخصصة في جدول المستخدمين.                                     |

### تنسيق بيانات القسم

:::info
يتطلب إرسال بيانات القسم تثبيت إضافة [الأقسام](../../departments) وتفعيلها.
:::

#### DepartmentData

| المعامل    | النوع                    | الوصف                                                                   |
| ---------- | ------------------------ | ----------------------------------------------------------------------- |
| `dataType` | `'user' \| 'department'` | مطلوب. نوع البيانات المُرسلة. استخدم `department` لبيانات الأقسام.     |
| `records`  | `DepartmentRecord[]`     | مطلوب. مصفوفة سجلات بيانات القسم.                                      |

#### DepartmentRecord

| المعامل     | النوع     | الوصف                                                              |
| ----------- | --------- | ------------------------------------------------------------------ |
| `uid`       | `string`  | مطلوب. المعرف الفريد لبيانات القسم المصدر. غير قابل للتغيير.      |
| `title`     | `string`  | مطلوب. عنوان القسم.                                                |
| `parentUid` | `string`  | اختياري. معرف القسم الأب.                                          |
| `isDeleted` | `boolean` | اختياري. يشير إلى ما إذا كان السجل محذوفًا.                       |
| `<field>`   | `any`     | اختياري. حقول مخصصة في جدول الأقسام.                              |

:::info

1. إرسال البيانات هو عملية متكافئة.
2. إذا لم يكن القسم الأب موجودًا عند إرسال بيانات القسم، لا يمكن إجراء الارتباط. يمكنك إرسال البيانات مرة أخرى بعد إنشاء القسم الأب.
3. إذا لم يكن قسم المستخدم موجودًا عند إرسال بيانات المستخدم، لا يمكن ربط المستخدم بذلك القسم. يمكنك إرسال بيانات المستخدم مرة أخرى بعد إرسال بيانات القسم.

:::
