# مزامنة بيانات المستخدم عبر HTTP API

## الحصول على مفتاح API

راجع [مفاتيح API](/auth-verification/api-keys). تأكد من أن الدور المرتبط بمفتاح API يمتلك الصلاحيات اللازمة لمزامنة بيانات المستخدمين.

---

## نظرة عامة على الـ API

### مثال

```bash
curl 'https://localhost:13000/api/userData:push' \
  -H 'Authorization: Bearer <token>' \
  --data-raw '{"dataType":"user","records":[]}' # راجع تفاصيل جسم الطلب أدناه
```

---

### نقطة النهاية (Endpoint)

```bash
POST /api/userData:push
```

---

## تنسيق بيانات المستخدم

### UserData

| المعامل    | النوع                              | الوصف                                                            |
| ---------- | ---------------------------------- | ---------------------------------------------------------------- |
| `dataType` | `'user' \| 'department'`           | مطلوب. نوع البيانات المرسلة. استخدم `user` لرفع بيانات المستخدم. |
| `matchKey` | `'username' \| 'email' \| 'phone'` | اختياري. يُستخدم لمطابقة المستخدمين الحاليين حسب الحقل المحدد.   |
| `records`  | `UserRecord[]`                     | مطلوب. مصفوفة بيانات المستخدمين.                                 |

---

### UserRecord

| المعامل       | النوع      | الوصف                                                                                                 |
| ------------- | ---------- | ----------------------------------------------------------------------------------------------------- |
| `uid`         | `string`   | مطلوب. معرف فريد لبيانات المصدر، يُستخدم لربط البيانات بالمستخدم داخل النظام. ثابت وغير قابل للتغيير. |
| `nickname`    | `string`   | اختياري. الاسم المستعار للمستخدم.                                                                     |
| `username`    | `string`   | اختياري. اسم المستخدم.                                                                                |
| `email`       | `string`   | اختياري. البريد الإلكتروني.                                                                           |
| `phone`       | `string`   | اختياري. رقم الهاتف.                                                                                  |
| `departments` | `string[]` | اختياري. قائمة معرفات الأقسام التي ينتمي إليها المستخدم.                                              |
| `isDeleted`   | `boolean`  | اختياري. يحدد ما إذا كان السجل محذوفًا.                                                               |
| `<field>`     | `any`      | اختياري. حقول مخصصة داخل جدول المستخدم.                                                               |
