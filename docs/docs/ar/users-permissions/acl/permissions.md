# تهيئة الصلاحيات

## إعدادات الصلاحيات العامة

![](https://static-docs.nocobase.com/119a9650259f9be71210121d0d3a435d.png)

### صلاحيات الإعداد

1. **السماح بتهيئة الواجهة**: تتحكم هذه الصلاحية في ما إذا كان المستخدم يمكنه تهيئة الواجهة. يؤدي تفعيلها إلى إضافة زر تهيئة واجهة المستخدم. يمتلك دور "المسؤول" هذه الصلاحية بشكل افتراضي.
2. **السماح بتثبيت الإضافات وتفعيلها وتعطيلها**: تحدد هذه الصلاحية ما إذا كان المستخدم يمكنه تفعيل الإضافات أو تعطيلها. عند تفعيلها، يحصل المستخدم على وصول إلى واجهة مدير الإضافات. يمتلك دور "المسؤول" هذه الصلاحية بشكل افتراضي.
3. **السماح بتهيئة الإضافات**: تتيح هذه الصلاحية للمستخدم تهيئة معاملات الإضافات أو إدارة بيانات الواجهة الخلفية للإضافات. يمتلك دور "المسؤول" هذه الصلاحية بشكل افتراضي.
4. **السماح بمسح ذاكرة التخزين المؤقت وإعادة تشغيل التطبيق**: ترتبط هذه الصلاحية بمهام صيانة النظام مثل مسح ذاكرة التخزين المؤقت وإعادة تشغيل التطبيق. عند تفعيلها، تظهر أزرار العمليات ذات الصلة في المركز الشخصي. هذه الصلاحية معطلة افتراضيًا.
5. **يُسمح بالوصول إلى عناصر القائمة الجديدة بشكل افتراضي**: القوائم المنشأة حديثًا يمكن الوصول إليها بشكل افتراضي، وهذا الإعداد مفعّل افتراضيًا.

### صلاحيات الإجراءات العامة

تنطبق صلاحيات الإجراءات العامة على جميع المجموعات وتُصنَّف حسب نوع العملية. يمكن تهيئة هذه الصلاحيات بناءً على نطاق البيانات: جميع البيانات أو بيانات المستخدم الخاصة.
