# Starší verze CRM (v1)

> **Pouze pro náhled**: Tato verze je funkčně poměrně kompletní, ale celkový návrh datových tabulek není optimální, chybí podpora pro vícejazyčnou lokalizaci a existuje propojení mezi kolekcemi CRM a kolekcemi pro správu projektů. Tyto problémy vedly k přepracování na CRM 2.0. V současné době doporučujeme používat [verzi CRM 2.0](./index.md), která má racionálnější strukturu dat, snadněji se rozšiřuje a upravuje a má dokončenou vícejazyčnou lokalizaci.

## Vlastnosti verze

Starší verze CRM obsahuje následující funkční moduly:

- **Správa zákazníků**: Informace o zákaznících, kontakty, klasifikace zákazníků.
- **Obchodní proces (Sales Pipeline)**: Správa leadů, sledování obchodních příležitostí, prodejní trychtýř.
- **Správa objednávek**: Cenové nabídky, objednávky, smlouvy.
- **Správa projektů** (zatím není součástí verze v2).
- **Komunikace a spolupráce**: E-mailová komunikace, komentáře, schvalovací pracovní postupy.
- **Analýza dat**: Nástěnky, reporty, vizualizace dat.

## Jak instalovat

> Aktuální verze se nasazuje formou zálohování a obnovení.

Než začnete, ujistěte se, že:

- Máte základní běžící prostředí NocoBase. Ohledně instalace hlavního systému se prosím podívejte na podrobnou [oficiální instalační dokumentaci](https://docs-cn.nocobase.com/welcome/getting-started/installation).
- Verze NocoBase je **v2.1.0-beta.2 nebo vyšší**.
- Máte stažené odpovídající soubory pro CRM v1:
  - **Soubor zálohy**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — Určeno pro Metodu 1.
  - **SQL soubor**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — Určeno pro Metodu 2.

**Důležitá upozornění**:
- Toto řešení je vytvořeno pro databázi **PostgreSQL 16**. Ujistěte se, že vaše prostředí používá PostgreSQL 16.
- **DB_UNDERSCORED nesmí být true**: Zkontrolujte prosím svůj soubor `docker-compose.yml` a ujistěte se, že proměnná prostředí `DB_UNDERSCORED` není nastavena na `true`, jinak dojde ke konfliktu se zálohou a obnovení selže.

## Metoda 1: Obnovení pomocí Správce záloh (doporučeno pro uživatele verzí Pro/Enterprise)

Tento způsob využívá vestavěný plugin "[Správce záloh](https://docs-cn.nocobase.com/handbook/backups)" (verze Pro/Enterprise) pro obnovení jedním kliknutím, což je nejjednodušší postup.

### Postup

**Krok 1: Aktivujte plugin "Správce záloh"**

1. Přihlaste se do svého systému NocoBase.
2. Přejděte do **`Správy pluginů`**.
3. Najděte a aktivujte plugin **`Správce záloh`**.

**Krok 2: Obnovení z lokálního souboru zálohy**

1. Po aktivaci pluginu obnovte stránku.
2. V levém menu přejděte do **`Správa systému`** -> **`Správce záloh`**.
3. Klikněte na tlačítko **`Obnovit z lokální zálohy`** v pravém horním rohu.
4. Přetáhněte stažený soubor zálohy do oblasti pro nahrávání.
5. Klikněte na **`Odeslat`** a vyčkejte, dokud systém nedokončí obnovu.

### Poznámky

* **Pouze pro verze Pro/Enterprise**: "Správce záloh" je plugin na podnikové úrovni a je dostupný pouze uživatelům verzí Pro/Enterprise.
* **Kompatibilita komerčních pluginů**: Ujistěte se, že vlastníte a máte aktivované komerční pluginy vyžadované tímto řešením.

## Metoda 2: Přímý import SQL souboru (univerzální)

Tento způsob obnovuje data přímou operací s databází a je použitelný pro všechny uživatele NocoBase.

### Postup

**Krok 1: Příprava čisté databáze**

Připravte si zcela novou, prázdnou databázi pro data, která se chystáte importovat.

**Krok 2: Import souboru `.sql` do databáze**

* **Přes příkazový řádek (příklad pro Docker)**:

  ```bash
  # Zkopírujte sql soubor do kontejneru
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # Vstupte do kontejneru a proveďte příkaz pro import
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Přes databázového klienta**: Použijte nástroje jako DBeaver, Navicat nebo pgAdmin pro připojení k databázi a spusťte SQL soubor.

**Krok 3: Připojení k databázi a spuštění aplikace**

Nakonfigurujte spouštěcí parametry NocoBase tak, aby ukazovaly na databázi, do které byla importována data, a poté spusťte službu.
