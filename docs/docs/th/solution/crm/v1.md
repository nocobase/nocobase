# CRM รุ่นเก่า (v1)

> **สำหรับดูตัวอย่างเท่านั้น**: เวอร์ชันนี้มีฟังก์ชันการทำงานที่ค่อนข้างครบถ้วน แต่การออกแบบโครงสร้างตารางข้อมูลโดยรวมยังไม่เหมาะสม ขาดการรองรับการแปลภาษา (Localization) และมีการเชื่อมโยงกัน (Coupling) ระหว่างคอลเลกชัน CRM และคอลเลกชันการจัดการโปรเจกต์ ปัญหาเหล่านี้จึงนำไปสู่การออกแบบ CRM 2.0 ใหม่ ในปัจจุบันขอแนะนำให้ใช้ [CRM รุ่น 2.0](./index.md) ซึ่งมีโครงสร้างตารางข้อมูลที่สมเหตุสมผลกว่า ขยายและปรับแต่งได้ง่ายกว่า และรองรับการใช้งานหลายภาษาเรียบร้อยแล้วครับ

## คุณสมบัติของเวอร์ชันนี้

CRM รุ่นเก่าประกอบด้วยโมดูลฟังก์ชันดังต่อไปนี้ครับ:

- **การจัดการลูกค้า**: ข้อมูลลูกค้า, ผู้ติดต่อ และการจัดลำดับความสำคัญของลูกค้า
- **ไปป์ไลน์การขาย**: การจัดการลีด (Lead), การติดตามโอกาสทางการขาย (Opportunity) และกรวยการขาย (Sales Funnel)
- **การจัดการคำสั่งซื้อ**: ใบเสนอราคา, คำสั่งซื้อ และสัญญา
- **การจัดการโปรเจกต์** (ยังไม่รวมอยู่ในรุ่น v2)
- **การสื่อสารและการทำงานร่วมกัน**: การสื่อสารผ่านอีเมล, การแสดงความคิดเห็น และเวิร์กโฟลว์การอนุมัติ
- **การวิเคราะห์ข้อมูล**: แดชบอร์ด, รายงาน และการแสดงข้อมูลด้วยภาพ (Data Visualization)

## วิธีการติดตั้ง

> เวอร์ชันปัจจุบันใช้รูปแบบการ**สำรองและคืนค่าข้อมูล (Backup and Restore)** ในการติดตั้งครับ

ก่อนเริ่มต้น โปรดตรวจสอบให้แน่ใจว่า:

- คุณมีสภาพแวดล้อมการรัน NocoBase พื้นฐานอยู่แล้ว สำหรับการติดตั้งระบบหลัก โปรดอ้างอิงจาก[เอกสารการติดตั้งอย่างเป็นทางการ](https://docs-cn.nocobase.com/welcome/getting-started/installation)
- เวอร์ชัน NocoBase คือ **v2.1.0-beta.2 ขึ้นไป**
- คุณได้ดาวน์โหลดไฟล์ที่เกี่ยวข้องของ CRM v1 เรียบร้อยแล้ว:
  - **ไฟล์สำรองข้อมูล**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — สำหรับวิธีที่ 1
  - **ไฟล์ SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — สำหรับวิธีที่ 2

**หมายเหตุสำคัญ**:
- โซลูชันนี้สร้างขึ้นบนฐานข้อมูล **PostgreSQL 16** โปรดตรวจสอบให้แน่ใจว่าสภาพแวดล้อมของคุณใช้ PostgreSQL 16 ครับ
- **DB_UNDERSCORED ต้องไม่เป็น true**: โปรดตรวจสอบไฟล์ `docker-compose.yml` ของคุณ และตรวจสอบให้แน่ใจว่าตัวแปรสภาพแวดล้อม `DB_UNDERSCORED` ไม่ได้ตั้งค่าเป็น `true` มิฉะนั้นจะเกิดความขัดแย้งกับการสำรองข้อมูลและทำให้การคืนค่าล้มเหลวครับ

## วิธีที่ 1: คืนค่าโดยใช้ตัวจัดการการสำรองข้อมูล (แนะนำสำหรับผู้ใช้รุ่น Pro/Enterprise)

วิธีนี้จะทำการคืนค่าผ่านปลั๊กอิน "[ตัวจัดการการสำรองข้อมูล](https://docs-cn.nocobase.com/handbook/backups)" (รุ่น Pro/Enterprise) ที่มาพร้อมกับ NocoBase ซึ่งเป็นวิธีที่ง่ายที่สุดครับ

### ขั้นตอนการดำเนินการ

**ขั้นตอนที่ 1: เปิดใช้งานปลั๊กอิน "ตัวจัดการการสำรองข้อมูล"**

1. เข้าสู่ระบบ NocoBase ของคุณ
2. ไปที่ **`การจัดการปลั๊กอิน` (Plugin Manager)**
3. ค้นหาและเปิดใช้งานปลั๊กอิน **`ตัวจัดการการสำรองข้อมูล` (Backup Manager)**

**ขั้นตอนที่ 2: คืนค่าจากไฟล์สำรองข้อมูลในเครื่อง**

1. หลังจากเปิดใช้งานปลั๊กอินแล้ว ให้รีเฟรชหน้าเว็บ
2. ไปที่เมนูด้านซ้าย **`การจัดการระบบ` (System Administration)** -> **`ตัวจัดการการสำรองข้อมูล` (Backup Manager)**
3. คลิกปุ่ม **`คืนค่าจากการสำรองข้อมูลในเครื่อง` (Restore from local backup)** ที่มุมขวาบน
4. ลากและวางไฟล์สำรองข้อมูลที่ดาวน์โหลดมาลงในพื้นที่อัปโหลด
5. คลิก **`ส่ง` (Submit)** และรอให้ระบบดำเนินการคืนค่าข้อมูลจนเสร็จสิ้นครับ

### ข้อควรระวัง

* **จำกัดเฉพาะรุ่น Pro/Enterprise**: "ตัวจัดการการสำรองข้อมูล" เป็นปลั๊กอินระดับองค์กร ซึ่งใช้งานได้เฉพาะผู้ใช้รุ่น Pro/Enterprise เท่านั้นครับ
* **การจับคู่ปลั๊กอินเชิงพาณิชย์**: โปรดตรวจสอบให้แน่ใจว่าคุณมีและเปิดใช้งานปลั๊กอินเชิงพาณิชย์ที่จำเป็นสำหรับโซลูชันนี้แล้วครับ

## วิธีที่ 2: นำเข้าไฟล์ SQL โดยตรง (ทั่วไป)

วิธีนี้จะคืนค่าข้อมูลโดยการจัดการฐานข้อมูลโดยตรง ซึ่งเหมาะสำหรับผู้ใช้ NocoBase ทุกคนครับ

### ขั้นตอนการดำเนินการ

**ขั้นตอนที่ 1: เตรียมฐานข้อมูลที่สะอาด**

เตรียมฐานข้อมูลใหม่ที่ว่างเปล่าสำหรับข้อมูลที่คุณกำลังจะนำเข้าครับ

**ขั้นตอนที่ 2: นำเข้าไฟล์ `.sql` เข้าสู่ฐานข้อมูล**

* **ผ่าน Command Line (ตัวอย่าง Docker)**:

  ```bash
  # คัดลอกไฟล์ sql เข้าไปในคอนเทนเนอร์
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # เข้าไปในคอนเทนเนอร์และรันคำสั่งนำเข้าข้อมูล
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **ผ่านโปรแกรมจัดการฐานข้อมูล (Database Client)**: ใช้เครื่องมืออย่าง DBeaver, Navicat หรือ pgAdmin เพื่อเชื่อมต่อกับฐานข้อมูลและรันไฟล์ SQL ครับ

**ขั้นตอนที่ 3: เชื่อมต่อฐานข้อมูลและเริ่มใช้งานแอปพลิเคชัน**

กำหนดค่าพารามิเตอร์การเริ่มต้นของ NocoBase ให้ชี้ไปยังฐานข้อมูลที่นำเข้าข้อมูลแล้ว จากนั้นจึงเริ่มการทำงานของบริการครับ
