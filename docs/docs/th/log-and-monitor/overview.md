# บันทึกเซิร์ฟเวอร์, บันทึกการตรวจสอบ (Audit Logs) และประวัติการเปลี่ยนแปลงข้อมูล

## บันทึกเซิร์ฟเวอร์

### บันทึกระบบ

> ดู [บันทึกระบบ](#)

- บันทึกข้อมูลการทำงานของระบบแอปพลิเคชัน, ติดตามการทำงานของโค้ด, และระบุข้อผิดพลาดหรือข้อยกเว้นที่เกิดขึ้นระหว่างการทำงานครับ/ค่ะ
- มีการจัดระดับความสำคัญของบันทึก (Log Levels) และจัดหมวดหมู่ตามโมดูลฟังก์ชันการทำงานครับ/ค่ะ
- แสดงผลทางเทอร์มินัลหรือจัดเก็บในรูปแบบไฟล์ครับ/ค่ะ
- ใช้เป็นหลักในการวินิจฉัยและแก้ไขปัญหาข้อผิดพลาดของระบบที่เกิดขึ้นระหว่างการทำงานครับ/ค่ะ

### บันทึกคำขอ (Request Logs)

> ดู [บันทึกคำขอ](#)

- บันทึกรายละเอียดคำขอและผลตอบกลับของ HTTP API โดยเน้นที่ Request ID, API Path, ส่วนหัว (Headers), รหัสสถานะการตอบกลับ (Response Status Code) และระยะเวลาที่ใช้ในการดำเนินการครับ/ค่ะ
- แสดงผลทางเทอร์มินัลหรือจัดเก็บในรูปแบบไฟล์ครับ/ค่ะ
- ใช้เป็นหลักในการติดตามการเรียกใช้ API และประสิทธิภาพการทำงานครับ/ค่ะ

## บันทึกการตรวจสอบ (Audit Logs)

> ดู [บันทึกการตรวจสอบ](/security/audit-logger/index.md)

- บันทึกการกระทำของผู้ใช้ (หรือ API) ต่อทรัพยากรของระบบ โดยเน้นที่ประเภททรัพยากร, วัตถุเป้าหมาย, ประเภทการดำเนินการ, ข้อมูลผู้ใช้ และสถานะการดำเนินการครับ/ค่ะ
- เพื่อให้สามารถติดตามสิ่งที่ผู้ใช้ทำและผลลัพธ์ที่เกิดขึ้นได้ดียิ่งขึ้น พารามิเตอร์คำขอ (Request Parameters) และผลตอบกลับ (Responses) จะถูกจัดเก็บเป็นข้อมูลเมตา (Metadata) ครับ/ค่ะ ข้อมูลส่วนนี้อาจทับซ้อนกับบันทึกคำขอ (Request Logs) บางส่วน แต่ไม่เหมือนกันทั้งหมด เช่น บันทึกคำขอโดยทั่วไปจะไม่รวมเนื้อหาคำขอ (Request Body) แบบเต็มครับ/ค่ะ
- พารามิเตอร์คำขอและผลตอบกลับ **ไม่เทียบเท่า** กับสแนปช็อตข้อมูล (Data Snapshots) ครับ/ค่ะ ข้อมูลเหล่านี้สามารถบอกได้ว่ามีการดำเนินการประเภทใดเกิดขึ้น แต่ไม่สามารถระบุข้อมูลที่แท้จริงก่อนการแก้ไขได้ จึงไม่สามารถนำไปใช้สำหรับการควบคุมเวอร์ชัน (Version Control) หรือการกู้คืนข้อมูลหลังจากเกิดข้อผิดพลาดในการดำเนินการได้ครับ/ค่ะ
- จัดเก็บในรูปแบบไฟล์และตารางฐานข้อมูลครับ/ค่ะ

## ประวัติการเปลี่ยนแปลงข้อมูล (Record History)

> ดู [ประวัติการเปลี่ยนแปลงข้อมูล](/record-history/index.md)

- บันทึก**ประวัติการเปลี่ยนแปลง**ของเนื้อหาข้อมูลครับ/ค่ะ
- สิ่งที่บันทึกเป็นหลักคือ ประเภททรัพยากร, วัตถุทรัพยากร, ประเภทการดำเนินการ, ฟิลด์ที่มีการเปลี่ยนแปลง และค่าก่อน/หลังการเปลี่ยนแปลงครับ/ค่ะ
- มีประโยชน์สำหรับการ**เปรียบเทียบข้อมูลและการตรวจสอบ**ครับ/ค่ะ
- จัดเก็บในรูปแบบตารางฐานข้อมูลครับ/ค่ะ
