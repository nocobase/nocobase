# บันทึกของเซิร์ฟเวอร์, บันทึกการตรวจสอบ และประวัติการเปลี่ยนแปลงข้อมูล

## บันทึกของเซิร์ฟเวอร์ (Server Logs)

### บันทึกการทำงานของระบบ (System Logs)

> อ้างอิง [บันทึกการทำงานของระบบ](./index.md#บันทึกการทำงานของระบบ)

- บันทึกข้อมูลการทำงานของระบบแอปพลิเคชัน ติดตามลำดับการทำงานของโค้ด และตรวจสอบข้อผิดพลาดหรือความผิดปกติที่เกิดขึ้นระหว่างการรันโปรแกรมครับ
- มีการแบ่งระดับของบันทึก (Log Levels) และจำแนกตามโมดูลการทำงาน
- แสดงผลผ่านเทอร์มินัลหรือจัดเก็บในรูปแบบไฟล์
- ใช้สำหรับวิเคราะห์และแก้ไขปัญหาความผิดปกติที่เกิดขึ้นระหว่างที่ระบบทำงานเป็นหลักครับ

### บันทึกการร้องขอ (Request Logs)

> อ้างอิง [บันทึกการร้องขอ](./index.md#บันทึกการร้องขอ)

- บันทึกรายละเอียดการร้องขอ (Request) และการตอบกลับ (Response) ของ HTTP API โดยเน้นที่ Request ID, API Path, ส่วนหัว (Headers), รหัสสถานะการตอบกลับ และระยะเวลาที่ใช้ครับ
- แสดงผลผ่านเทอร์มินัลหรือจัดเก็บในรูปแบบไฟล์
- ใช้สำหรับติดตามการเรียกใช้งาน API และตรวจสอบประสิทธิภาพการทำงานเป็นหลักครับ

## บันทึกการตรวจสอบ (Audit Logs)

> อ้างอิง [บันทึกการตรวจสอบ](/security/audit-logger/index.md)

- บันทึกพฤติกรรมการใช้งานของผู้ใช้ (หรือ API) ต่อทรัพยากรต่างๆ ของระบบ โดยเน้นที่ประเภททรัพยากร, ออบเจ็กต์ของทรัพยากร, ประเภทการดำเนินการ, ข้อมูลผู้ใช้ และสถานะการดำเนินการครับ
- เพื่อให้สามารถติดตามเนื้อหาและผลลัพธ์ของการดำเนินการของผู้ใช้ได้ดียิ่งขึ้น พารามิเตอร์การร้องขอและการตอบกลับจะถูกบันทึกไว้เป็นข้อมูลเมตา (Metadata) ซึ่งข้อมูลส่วนนี้อาจมีความซ้ำซ้อนกับบันทึกการร้องขอ (Request Logs) บางส่วน แต่ไม่เหมือนกันทั้งหมดครับ ตัวอย่างเช่น ในบันทึกการร้องขอทั่วไปมักจะไม่ได้บันทึกเนื้อหาการร้องขอ (Request Body) ทั้งหมด
- พารามิเตอร์การร้องขอและการตอบกลับ **ไม่เทียบเท่ากับ** การบันทึกภาพข้อมูล (Data Snapshot) ของทรัพยากร แม้จะทราบได้ว่ามีการแก้ไขอะไรผ่านพารามิเตอร์และตรรกะของโค้ด แต่จะไม่สามารถทราบข้อมูลที่แน่นอนก่อนการแก้ไขได้ ดังนั้นจึงไม่สามารถใช้สำหรับการควบคุมเวอร์ชัน (Version Control) หรือการกู้คืนข้อมูลหลังจากเกิดความผิดพลาดได้ครับ
- จัดเก็บในรูปแบบไฟล์และตารางฐานข้อมูล

## ประวัติการเปลี่ยนแปลงข้อมูล (Record History)

> อ้างอิง [ประวัติการเปลี่ยนแปลงข้อมูล](/record-history/index.md)

- บันทึกประวัติการเปลี่ยนแปลงของเนื้อหาข้อมูลครับ
- บันทึกข้อมูลหลัก ได้แก่ ประเภททรัพยากร, ออบเจ็กต์ของทรัพยากร, ประเภทการดำเนินการ, ฟิลด์ที่มีการเปลี่ยนแปลง และค่าก่อนและหลังการเปลี่ยนแปลง
- ใช้สำหรับการเปรียบเทียบข้อมูลและการตรวจสอบครับ
- จัดเก็บในรูปแบบตารางฐานข้อมูล
