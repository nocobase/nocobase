# การเลือกแบบ Cascading

## บทนำ

ตัวเลือกแบบ Cascading (cascading selector) เหมาะสำหรับฟิลด์ความสัมพันธ์ที่คอลเลกชันเป้าหมายเป็นตารางแบบ Tree (tree table) ครับ/ค่ะ ผู้ใช้สามารถเลือกข้อมูลตามโครงสร้างลำดับชั้นของคอลเลกชันแบบ Tree และยังรองรับการค้นหาแบบคลุมเครือ (fuzzy search) เพื่อช่วยให้ค้นหาข้อมูลได้อย่างรวดเร็วครับ/ค่ะ

## วิธีใช้งาน

- สำหรับความสัมพันธ์แบบ **to-one** ตัวเลือกแบบ Cascading จะเป็นแบบ **เลือกได้รายการเดียว (single-select)** ครับ/ค่ะ

![20251125214656](https://static-docs.nocobase.com/20251125214656.png)

- สำหรับความสัมพันธ์แบบ **to-many** ตัวเลือกแบบ Cascading จะเป็นแบบ **เลือกได้หลายรายการ (multi-select)** ครับ/ค่ะ

![20251125215318](https://static-docs.nocobase.com/20251125215318.png)

## ตัวเลือกการกำหนดค่าฟิลด์

### ฟิลด์ชื่อเรื่อง (Title Field)

ฟิลด์ชื่อเรื่องคือฟิลด์ที่ใช้แสดงเป็นป้ายกำกับ (label) สำหรับแต่ละตัวเลือกครับ/ค่ะ

![20251125214923](https://static-docs.nocobase.com/20251125214923.gif)

> รองรับการค้นหาอย่างรวดเร็วโดยใช้ฟิลด์ชื่อเรื่องครับ/ค่ะ

![20251125215026](https://static-docs.nocobase.com/20251125215026.gif)

ดูรายละเอียดเพิ่มเติมได้ที่: [ฟิลด์ชื่อเรื่อง (Title Field)](/interface-builder/fields/field-settings/title-field)

### การกำหนดขอบเขตข้อมูล (Data Scope)

ใช้ควบคุมขอบเขตข้อมูลของรายการแบบ Tree (tree list) ครับ/ค่ะ (หากเรคคอร์ดลูกตรงตามเงื่อนไข เรคคอร์ดแม่ก็จะถูกรวมอยู่ด้วยเช่นกัน)

![20251125215111](https://static-docs.nocobase.com/20251125215111.png)

ดูรายละเอียดเพิ่มเติมได้ที่: [ขอบเขตข้อมูล (Data Scope)](/interface-builder/fields/field-settings/data-scope)
