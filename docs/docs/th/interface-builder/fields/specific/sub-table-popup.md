# ตารางย่อย (แก้ไขผ่านป๊อปอัป)

## บทนำ

ตารางย่อย (แก้ไขผ่านป๊อปอัป) ใช้สำหรับจัดการข้อมูลความสัมพันธ์แบบกลุ่ม (เช่น One-to-Many หรือ Many-to-Many) ภายในฟอร์ม โดยตารางจะแสดงเฉพาะรายการที่เชื่อมโยงกันอยู่เท่านั้น การเพิ่มหรือแก้ไขข้อมูลจะทำผ่านหน้าต่างป๊อปอัป และข้อมูลจะถูกบันทึกลงฐานข้อมูลพร้อมกันเมื่อมีการส่งฟอร์มหลักครับ

## คำแนะนำการใช้งาน

![20260212152204](https://static-docs.nocobase.com/20260212152204.png)

**สถานการณ์ที่เหมาะสม:**

- ฟิลด์ความสัมพันธ์: O2M / M2M / MBM
- การใช้งานทั่วไป: รายละเอียดคำสั่งซื้อ, รายการย่อย, แท็ก/สมาชิกที่เกี่ยวข้อง เป็นต้นครับ

## การตั้งค่าฟิลด์

### อนุญาตให้เลือกข้อมูลที่มีอยู่ (ค่าเริ่มต้น: เปิดใช้งาน)

รองรับการเลือกความสัมพันธ์จากข้อมูลที่มีอยู่เดิมครับ

![20260212152312](https://static-docs.nocobase.com/20260212152312.png)

![20260212152343](https://static-docs.nocobase.com/20260212152343.gif)

### คอมโพเนนต์ฟิลด์

[คอมโพเนนต์ฟิลด์](/interface-builder/fields/association-field): เปลี่ยนเป็นคอมโพเนนต์ฟิลด์ความสัมพันธ์อื่นๆ เช่น รายการเลือก (Single select), ตัวเลือกคอลเลกชัน (Collection selector) เป็นต้นครับ

### อนุญาตให้ยกเลิกการเชื่อมโยงข้อมูลที่มีอยู่ (ค่าเริ่มต้น: เปิดใช้งาน)

> ควบคุมว่าข้อมูลที่เชื่อมโยงอยู่แล้วในฟอร์มแก้ไขจะได้รับอนุญาตให้ยกเลิกการเชื่อมโยงหรือไม่ ส่วนข้อมูลที่เพิ่มเข้ามาใหม่จะสามารถลบออกได้เสมอครับ

![20260212165752](https://static-docs.nocobase.com/20260212165752.gif)

### อนุญาตให้เพิ่มข้อมูล (ค่าเริ่มต้น: เปิดใช้งาน)

ควบคุมว่าจะแสดงปุ่ม "เพิ่ม" หรือไม่ หากผู้ใช้ไม่มีสิทธิ์ `create` ในคอลเลกชันเป้าหมาย ปุ่มจะถูกปิดใช้งานและแสดงข้อความแจ้งเตือนว่าไม่มีสิทธิ์ครับ

### อนุญาตให้แก้ไขด่วน (ค่าเริ่มต้น: ปิดใช้งาน)

เมื่อเปิดใช้งาน เมื่อวางเมาส์เหนือเซลล์จะปรากฏไอคอนแก้ไข ซึ่งช่วยให้สามารถแก้ไขเนื้อหาในเซลล์ได้อย่างรวดเร็วครับ

รองรับการเปิดใช้งานการแก้ไขด่วนสำหรับทุกฟิลด์ผ่านการตั้งค่าคอมโพเนนต์ฟิลด์ความสัมพันธ์

![20260212165102](https://static-docs.nocobase.com/20260212165102.png)

และยังรองรับการเปิดใช้งานการแก้ไขด่วนสำหรับแต่ละคอลัมน์แยกกันได้ด้วยครับ

![20260212165025](https://static-docs.nocobase.com/20260212165025.png)

![20260212165201](https://static-docs.nocobase.com/20260212165201.gif)

### ขนาดหน้า (ค่าเริ่มต้น: 10)

กำหนดจำนวนรายการที่จะแสดงต่อหนึ่งหน้าในตารางย่อยครับ

## คำอธิบายพฤติกรรมการทำงาน

- เมื่อเลือกรายการที่มีอยู่ ระบบจะลบรายการที่ซ้ำกันโดยอ้างอิงจากคีย์หลัก (Primary Key) เพื่อป้องกันการเชื่อมโยงรายการเดิมซ้ำครับ
- รายการที่เพิ่มใหม่จะถูกเติมกลับเข้าไปในตารางย่อยโดยตรง และระบบจะเปลี่ยนไปยังหน้าที่แสดงรายการใหม่นั้นโดยอัตโนมัติครับ
- การแก้ไขในแถว (Inline editing) จะแก้ไขเฉพาะข้อมูลในแถวนั้นๆ เท่านั้นครับ
- การลบรายการออกจะเป็นเพียงการยกเลิกการเชื่อมโยงภายในฟอร์มปัจจุบันเท่านั้น ไม่ได้เป็นการลบข้อมูลต้นฉบับออกจากฐานข้อมูลครับ
- ข้อมูลจะถูกบันทึกลงฐานข้อมูลพร้อมกันเมื่อมีการส่งฟอร์มหลักครับ
