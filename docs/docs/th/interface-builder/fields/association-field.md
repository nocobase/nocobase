# คอมโพเนนต์ฟิลด์ความสัมพันธ์

## บทนำ

คอมโพเนนต์ฟิลด์ความสัมพันธ์ของ NocoBase ถูกออกแบบมาเพื่อช่วยให้ผู้ใช้สามารถแสดงและจัดการข้อมูลที่เชื่อมโยงกันได้อย่างมีประสิทธิภาพมากขึ้นครับ/ค่ะ ไม่ว่าจะเป็นความสัมพันธ์ประเภทใด คอมโพเนนต์เหล่านี้ก็มีความยืดหยุ่นและหลากหลาย ทำให้ผู้ใช้สามารถเลือกและตั้งค่าให้เหมาะสมกับความต้องการเฉพาะของตนเองได้เลยครับ/ค่ะ

### ดรอปดาวน์

สำหรับฟิลด์ความสัมพันธ์ทั้งหมด ยกเว้นกรณีที่คอลเลกชันเป้าหมายเป็นคอลเลกชันไฟล์ คอมโพเนนต์เริ่มต้นในโหมดแก้ไขจะเป็นดรอปดาวน์ครับ/ค่ะ ตัวเลือกในดรอปดาวน์จะแสดงค่าของฟิลด์ชื่อเรื่อง ซึ่งเหมาะสำหรับสถานการณ์ที่ต้องการเลือกข้อมูลที่เชื่อมโยงกันอย่างรวดเร็ว โดยการแสดงข้อมูลฟิลด์สำคัญเพียงฟิลด์เดียวครับ/ค่ะ

![20240429205659](https://static-docs.nocobase.com/20240429205659.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ดรอปดาวน์](/interface-builder/fields/specific/select)

### ตัวเลือกข้อมูล

ตัวเลือกข้อมูลจะแสดงข้อมูลในรูปแบบหน้าต่างป๊อปอัปครับ/ค่ะ ผู้ใช้สามารถตั้งค่าฟิลด์ที่ต้องการให้แสดงในตัวเลือกข้อมูลได้ (รวมถึงฟิลด์จากความสัมพันธ์ที่ซ้อนกัน) เพื่อให้สามารถเลือกข้อมูลที่เชื่อมโยงกันได้อย่างแม่นยำยิ่งขึ้นครับ/ค่ะ

![20240429210824](https://static-docs.nocobase.com/20240429210824.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ตัวเลือกข้อมูล](/interface-builder/fields/specific/picker)

### ซับฟอร์ม

เมื่อต้องจัดการกับข้อมูลความสัมพันธ์ที่ซับซ้อน การใช้ดรอปดาวน์หรือตัวเลือกข้อมูลอาจไม่สะดวกครับ/ค่ะ ในกรณีเช่นนี้ ผู้ใช้อาจต้องเปิดหน้าต่างป๊อปอัปบ่อยครั้ง เพื่อแก้ปัญหานี้ เราสามารถใช้ซับฟอร์มได้ครับ/ค่ะ ซับฟอร์มช่วยให้ผู้ใช้สามารถจัดการฟิลด์ของคอลเลกชันที่เชื่อมโยงได้โดยตรงบนหน้าปัจจุบัน หรือในบล็อกป๊อปอัปปัจจุบัน โดยไม่ต้องเปิดหน้าต่างป๊อปอัปใหม่ซ้ำๆ ทำให้ขั้นตอนการทำงานราบรื่นยิ่งขึ้นครับ/ค่ะ ความสัมพันธ์หลายระดับจะแสดงในรูปแบบของฟอร์มที่ซ้อนกันครับ/ค่ะ

![20251029122948](https://static-docs.nocobase.com/20251029122948.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ซับฟอร์ม](/interface-builder/fields/specific/sub-form)

### ซับเทเบิล

ซับเทเบิลจะแสดงเรคคอร์ดความสัมพันธ์แบบหนึ่งต่อกลุ่ม (one-to-many) หรือกลุ่มต่อกลุ่ม (many-to-many) ในรูปแบบตารางครับ/ค่ะ ซึ่งเป็นวิธีที่ชัดเจนและมีโครงสร้างในการแสดงและจัดการข้อมูลที่เชื่อมโยงกัน อีกทั้งยังรองรับการสร้างข้อมูลใหม่เป็นจำนวนมาก หรือการเลือกข้อมูลที่มีอยู่เพื่อเชื่อมโยงได้ด้วยครับ/ค่ะ

![20251029123042](https://static-docs.nocobase.com/20251029123042.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ซับเทเบิล](/interface-builder/fields/specific/sub-table)

### ซับดีเทล

ซับดีเทลเป็นคอมโพเนนต์ที่สอดคล้องกับซับฟอร์มในโหมดอ่านอย่างเดียวครับ/ค่ะ ซึ่งรองรับการแสดงข้อมูลที่มีความสัมพันธ์หลายระดับที่ซ้อนกันได้ครับ/ค่ะ

![20251030213050](https://static-docs.nocobase.com/20251030213050.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ซับดีเทล](/interface-builder/fields/specific/sub-detail)

### ตัวจัดการไฟล์

ตัวจัดการไฟล์เป็นคอมโพเนนต์ฟิลด์ความสัมพันธ์ที่ใช้สำหรับจัดการโดยเฉพาะ เมื่อคอลเลกชันเป้าหมายของความสัมพันธ์เป็นคอลเลกชันไฟล์ครับ/ค่ะ

![20240429222753](https://static-docs.nocobase.com/20240429222753.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ตัวจัดการไฟล์](/interface-builder/fields/specific/file-manager)

### ชื่อเรื่อง

คอมโพเนนต์ฟิลด์ชื่อเรื่องเป็นคอมโพเนนต์ฟิลด์ความสัมพันธ์ที่ใช้ในโหมดอ่านอย่างเดียวครับ/ค่ะ โดยการตั้งค่าฟิลด์ชื่อเรื่อง คุณสามารถตั้งค่าคอมโพเนนต์ฟิลด์ที่เกี่ยวข้องได้ครับ/ค่ะ

![20251030213327](https://static-docs.nocobase.com/20251030213327.png)

สำหรับรายละเอียดเพิ่มเติม โปรดดูที่ [ชื่อเรื่อง](/interface-builder/fields/specific/title)
