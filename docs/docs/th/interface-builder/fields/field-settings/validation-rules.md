# ตั้งค่ากฎการตรวจสอบความถูกต้อง

## บทนำ

กฎการตรวจสอบความถูกต้องใช้เพื่อให้แน่ใจว่าข้อมูลที่ผู้ใช้ป้อนเข้ามานั้นเป็นไปตามที่คาดหวังครับ/ค่ะ

## ตั้งค่ากฎการตรวจสอบความถูกต้องของฟิลด์ได้ที่ไหนบ้าง

### การตั้งค่ากฎการตรวจสอบความถูกต้องสำหรับฟิลด์ใน คอลเลกชัน

ฟิลด์ส่วนใหญ่รองรับการตั้งค่ากฎการตรวจสอบความถูกต้องครับ/ค่ะ เมื่อฟิลด์ถูกตั้งค่ากฎการตรวจสอบความถูกต้องแล้ว การตรวจสอบความถูกต้องทางฝั่งเซิร์ฟเวอร์ (backend validation) จะถูกเรียกใช้เมื่อมีการส่งข้อมูลครับ/ค่ะ ฟิลด์แต่ละประเภทจะรองรับกฎการตรวจสอบความถูกต้องที่แตกต่างกันไปครับ/ค่ะ

- **ฟิลด์ตัวเลข**

- **ฟิลด์ข้อความ**

  นอกจากจะจำกัดความยาวของข้อความได้แล้ว ฟิลด์ข้อความยังรองรับการใช้ Regular Expression (Regex) แบบกำหนดเอง เพื่อการตรวจสอบความถูกต้องที่ละเอียดขึ้นครับ/ค่ะ

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### การตรวจสอบความถูกต้องทางฝั่งไคลเอนต์ (Frontend Validation) ในการตั้งค่าฟิลด์

กฎการตรวจสอบความถูกต้องที่ตั้งค่าไว้ในการตั้งค่าฟิลด์ จะเรียกใช้การตรวจสอบความถูกต้องทางฝั่งไคลเอนต์ เพื่อให้แน่ใจว่าข้อมูลที่ผู้ใช้ป้อนเข้ามาเป็นไปตามข้อกำหนดครับ/ค่ะ

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

![20251028230255](https://static-docs.nocobase.com/20251028230255.png)

**ฟิลด์ข้อความ**ยังรองรับการตรวจสอบด้วย Regular Expression (Regex) แบบกำหนดเอง เพื่อให้เป็นไปตามข้อกำหนดรูปแบบเฉพาะครับ/ค่ะ
