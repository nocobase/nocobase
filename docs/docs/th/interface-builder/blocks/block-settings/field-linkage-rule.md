# กฎการเชื่อมโยงฟิลด์

## บทนำ

กฎการเชื่อมโยงฟิลด์ช่วยให้คุณสามารถปรับสถานะของฟิลด์ในบล็อกฟอร์ม (Form Block) หรือบล็อกรายละเอียด (Details Block) ได้อย่างไดนามิกตามการกระทำของผู้ใช้ครับ/ค่ะ ปัจจุบัน บล็อกที่รองรับกฎการเชื่อมโยงฟิลด์ได้แก่:

- [บล็อกฟอร์ม (Form Block)](/interface-builder/blocks/data-blocks/form)
- [บล็อกรายละเอียด (Details Block)](/interface-builder/blocks/data-blocks/details)
- [ซับฟอร์ม (Sub-form)](/interface-builder/fields/specific/sub-form)

## วิธีการใช้งาน

### **บล็อกฟอร์ม (Form Block)**

ในบล็อกฟอร์ม (Form Block) กฎการเชื่อมโยงฟิลด์สามารถปรับเปลี่ยนพฤติกรรมของฟิลด์ได้อย่างไดนามิกตามเงื่อนไขที่กำหนดครับ/ค่ะ:

- **ควบคุมการแสดง/ซ่อนฟิลด์**: กำหนดว่าจะให้แสดงฟิลด์ปัจจุบันหรือไม่ โดยอิงตามค่าของฟิลด์อื่น ๆ ครับ/ค่ะ
- **กำหนดให้ฟิลด์เป็นฟิลด์ที่ต้องกรอก**: สามารถตั้งค่าให้ฟิลด์เป็นฟิลด์ที่ต้องกรอกหรือไม่ต้องกรอกได้แบบไดนามิกภายใต้เงื่อนไขที่เฉพาะเจาะจงครับ/ค่ะ
- **กำหนดค่า**: กำหนดค่าให้กับฟิลด์โดยอัตโนมัติตามเงื่อนไขที่กำหนดครับ/ค่ะ
- **เรียกใช้ JavaScript ที่กำหนด**: เขียน JavaScript ตามความต้องการทางธุรกิจครับ/ค่ะ

### **บล็อกรายละเอียด (Details Block)**

ในบล็อกรายละเอียด (Details Block) กฎการเชื่อมโยงฟิลด์ส่วนใหญ่จะใช้เพื่อควบคุมการแสดงและซ่อนฟิลด์บนบล็อกได้อย่างไดนามิกครับ/ค่ะ

## การเชื่อมโยงคุณสมบัติ

### กำหนดค่า

ตัวอย่าง: เมื่อมีการเลือกคำสั่งซื้อเป็น "คำสั่งซื้อเพิ่มเติม" (Supplementary Order) สถานะคำสั่งซื้อจะถูกกำหนดเป็น "รอการตรวจสอบ" (Pending Review) โดยอัตโนมัติครับ/ค่ะ

![20251029115348](https://static-docs.nocobase.com/20251029115348.png)

### ต้องกรอก

ตัวอย่าง: เมื่อสถานะคำสั่งซื้อเป็น "ชำระเงินแล้ว" (Paid) ฟิลด์ยอดรวมคำสั่งซื้อจะต้องกรอกครับ/ค่ะ

![20251029115031](https://static-docs.nocobase.com/20251029115031.png)

### แสดง/ซ่อน

ตัวอย่าง: บัญชีการชำระเงินและยอดรวมจะแสดงขึ้นเมื่อสถานะคำสั่งซื้อเป็น "รอการชำระเงิน" (Pending Payment) เท่านั้นครับ/ค่ะ
