# กฎการเชื่อมโยงบล็อก

## บทนำ

กฎการเชื่อมโยงบล็อกช่วยให้ผู้ใช้สามารถควบคุมการแสดงผลของบล็อกได้อย่างไดนามิก โดยจัดการการแสดงผลขององค์ประกอบต่าง ๆ ในระดับบล็อกได้โดยรวมครับ/ค่ะ เนื่องจากบล็อกเป็นตัวกลางสำหรับฟิลด์และปุ่มดำเนินการ กฎเหล่านี้จึงช่วยให้ผู้ใช้สามารถควบคุมการแสดงผลของมุมมองทั้งหมดได้อย่างยืดหยุ่นจากมิติของบล็อกครับ/ค่ะ

![20251029112218](https://static-docs.nocobase.com/20251029112218.png)

![20251029112338](https://static-docs.nocobase.com/20251029112338.png)

> **ข้อควรทราบ**: ก่อนที่กฎการเชื่อมโยงบล็อกจะทำงาน การแสดงผลของบล็อกจะต้องผ่าน **การตรวจสอบสิทธิ์ ACL** ก่อนครับ/ค่ะ เฉพาะเมื่อผู้ใช้มีสิทธิ์เข้าถึงที่เกี่ยวข้องเท่านั้น จึงจะเข้าสู่กระบวนการพิจารณาตามตรรกะของกฎการเชื่อมโยงบล็อกครับ/ค่ะ กล่าวอีกนัยหนึ่งคือ กฎการเชื่อมโยงบล็อกจะมีผลก็ต่อเมื่อเป็นไปตามข้อกำหนดสิทธิ์การดูของ ACL แล้วเท่านั้น หากไม่มีกฎการเชื่อมโยงบล็อก บล็อกจะแสดงผลตามค่าเริ่มต้นครับ/ค่ะ

### การควบคุมบล็อกด้วยตัวแปรส่วนกลาง

**กฎการเชื่อมโยงบล็อก** รองรับการใช้ตัวแปรส่วนกลางเพื่อควบคุมเนื้อหาที่แสดงในบล็อกได้อย่างไดนามิก ทำให้ผู้ใช้ที่มีบทบาทและสิทธิ์ที่แตกต่างกันสามารถเห็นและโต้ตอบกับมุมมองข้อมูลที่ปรับแต่งได้ครับ/ค่ะ ตัวอย่างเช่น ในระบบจัดการคำสั่งซื้อ แม้ว่าบทบาทที่แตกต่างกัน (เช่น ผู้ดูแลระบบ, พนักงานขาย, เจ้าหน้าที่การเงิน ฯลฯ) จะมีสิทธิ์ในการดูคำสั่งซื้อทั้งหมด แต่ฟิลด์และปุ่มดำเนินการที่แต่ละบทบาทต้องการอาจแตกต่างกันไปครับ/ค่ะ ด้วยการตั้งค่าตัวแปรส่วนกลาง คุณสามารถปรับฟิลด์ที่แสดง, ปุ่มดำเนินการ, หรือแม้แต่กฎการจัดเรียงและกรองข้อมูลได้อย่างยืดหยุ่นตามบทบาท, สิทธิ์, หรือเงื่อนไขอื่น ๆ ของผู้ใช้ครับ/ค่ะ

#### กรณีการใช้งานเฉพาะ:

-   **การควบคุมบทบาทและสิทธิ์**: ควบคุมการมองเห็นหรือการแก้ไขฟิลด์บางอย่างตามสิทธิ์ของบทบาทที่แตกต่างกันครับ/ค่ะ ตัวอย่างเช่น พนักงานขายสามารถดูได้เฉพาะข้อมูลพื้นฐานของคำสั่งซื้อ ในขณะที่เจ้าหน้าที่การเงินสามารถดูรายละเอียดการชำระเงินได้ครับ/ค่ะ
-   **มุมมองส่วนบุคคล**: ปรับแต่งมุมมองบล็อกที่แตกต่างกันสำหรับแต่ละแผนกหรือทีม เพื่อให้แน่ใจว่าผู้ใช้แต่ละคนจะเห็นเฉพาะเนื้อหาที่เกี่ยวข้องกับงานของตน ซึ่งจะช่วยเพิ่มประสิทธิภาพในการทำงานครับ/ค่ะ
-   **การจัดการสิทธิ์การดำเนินการ**: ควบคุมการแสดงผลของปุ่มดำเนินการโดยใช้ตัวแปรส่วนกลางครับ/ค่ะ ตัวอย่างเช่น บางบทบาทอาจสามารถดูข้อมูลได้อย่างเดียว ในขณะที่บทบาทอื่น ๆ สามารถดำเนินการแก้ไขหรือลบได้ครับ/ค่ะ

### การควบคุมบล็อกด้วยตัวแปรบริบท

บล็อกยังสามารถถูกควบคุมได้ด้วยตัวแปรในบริบทครับ/ค่ะ ตัวอย่างเช่น คุณสามารถใช้ตัวแปรบริบท เช่น "เรคคอร์ดปัจจุบัน", "ฟอร์มปัจจุบัน" และ "เรคคอร์ดในป๊อปอัปปัจจุบัน" เพื่อแสดงหรือซ่อนบล็อกได้อย่างไดนามิกครับ/ค่ะ

ตัวอย่าง: บล็อก "ข้อมูลโอกาสทางธุรกิจของคำสั่งซื้อ" จะแสดงขึ้นก็ต่อเมื่อสถานะของคำสั่งซื้อเป็น "ชำระเงินแล้ว" เท่านั้นครับ/ค่ะ
