# บล็อกอ้างอิง

## บทนำ
บล็อกอ้างอิงช่วยให้คุณสามารถแสดงบล็อกที่ตั้งค่าไว้แล้วบนหน้าปัจจุบันได้ทันที เพียงแค่ระบุ UID ของบล็อกเป้าหมาย โดยไม่ต้องตั้งค่าซ้ำครับ/ค่ะ

## การเปิดใช้งานปลั๊กอิน
ปลั๊กอินนี้เป็นแบบ Built-in แต่จะถูกปิดใช้งานโดยค่าเริ่มต้นครับ/ค่ะ
เปิด 'การจัดการปลั๊กอิน' → ค้นหา 'บล็อก: อ้างอิง' → แล้วคลิก 'เปิดใช้งาน' ครับ/ค่ะ

![Enable Reference block in Plugin Manager](https://static-docs.nocobase.com/block-reference-enable-20251102.png)

## วิธีการเพิ่มบล็อก
1) เพิ่มบล็อก → กลุ่ม 'บล็อกอื่นๆ' → เลือก 'บล็อกอ้างอิง' ครับ/ค่ะ
2) ในการตั้งค่าบล็อกอ้างอิง ให้กำหนดค่าดังนี้ครับ/ค่ะ
   - `Block UID`: UID ของบล็อกเป้าหมาย
   - `Reference mode`: เลือก `อ้างอิง` หรือ `คัดลอก`

![Reference block add and configure demo](https://static-docs.nocobase.com/20251102193949_rec_.gif)

### วิธีการรับ Block UID
- เปิดเมนูการตั้งค่าของบล็อกเป้าหมาย แล้วคลิก `คัดลอก UID` เพื่อคัดลอก UID ของบล็อกนั้นได้เลยครับ/ค่ะ

![Copy UID from block settings](https://static-docs.nocobase.com/block-reference-copy-uid-20251102.png)

## โหมดและลักษณะการทำงาน
- `อ้างอิง` (ค่าเริ่มต้น)
  - ใช้การตั้งค่าเดียวกันกับบล็อกต้นฉบับ หากมีการแก้ไขบล็อกต้นฉบับหรือแก้ไขจากบล็อกอ้างอิงใดๆ การเปลี่ยนแปลงทั้งหมดจะถูกอัปเดตพร้อมกันครับ/ค่ะ

- `คัดลอก`
  - จะสร้างบล็อกอิสระที่เหมือนกับบล็อกต้นฉบับ ณ เวลานั้น การแก้ไขในภายหลังจะไม่ส่งผลกระทบต่อกันและจะไม่ซิงค์กันครับ/ค่ะ

## การตั้งค่า
- บล็อกอ้างอิง:
  - 'การตั้งค่าบล็อกอ้างอิง': ใช้สำหรับระบุ Block UID ของบล็อกเป้าหมาย และเลือกโหมด 'อ้างอิง' หรือ 'คัดลอก' ครับ/ค่ะ
  - พร้อมกันนั้น จะแสดงการตั้งค่าทั้งหมดของ 'บล็อกที่ถูกอ้างอิง' ด้วย (ซึ่งเทียบเท่ากับการตั้งค่าโดยตรงบนบล็อกต้นฉบับ) ครับ/ค่ะ

![Reference block settings](https://static-docs.nocobase.com/block-reference-settings-20251102.png)

- บล็อกที่ถูกคัดลอก:
  - บล็อกที่ได้จากการคัดลอกจะมีประเภทเดียวกับบล็อกต้นฉบับ และจะมีการตั้งค่าเฉพาะของบล็อกนั้นเองครับ/ค่ะ
  - และจะไม่มี 'การตั้งค่าบล็อกอ้างอิง' อีกต่อไปครับ/ค่ะ

## สถานะข้อผิดพลาดและการจัดการ
- เมื่อบล็อกเป้าหมายหายไปหรือไม่ถูกต้อง: จะแสดงข้อความแจ้งเตือนสถานะข้อผิดพลาด คุณสามารถกำหนด Block UID ใหม่ได้ในการตั้งค่าของบล็อกอ้างอิง (การตั้งค่าบล็อกอ้างอิง → Block UID) แล้วบันทึกเพื่อกู้คืนการแสดงผลครับ/ค่ะ

![Error state when target block is invalid](https://static-docs.nocobase.com/block-reference-error-20251102.png)

## ข้อควรทราบและข้อจำกัด
- เป็นฟังก์ชันทดลอง โปรดใช้งานด้วยความระมัดระวังในสภาพแวดล้อมการทำงานจริงครับ/ค่ะ
- เมื่อคัดลอกบล็อก การตั้งค่าบางอย่างที่ขึ้นอยู่กับ Block UID เป้าหมาย อาจจำเป็นต้องตั้งค่าใหม่ครับ/ค่ะ
- การตั้งค่าทั้งหมดของบล็อกอ้างอิงจะถูกซิงค์โดยอัตโนมัติ รวมถึงการตั้งค่า 'ขอบเขตข้อมูล' (data scope) และอื่นๆ ครับ/ค่ะ อย่างไรก็ตาม บล็อกอ้างอิงสามารถมีการตั้งค่า[เวิร์กโฟลว์เหตุการณ์](/interface-builder/event-flow/)ของตัวเองได้ ดังนั้น คุณสามารถใช้เวิร์กโฟลว์เหตุการณ์และการดำเนินการด้วย JavaScript แบบกำหนดเอง เพื่อให้ได้ขอบเขตข้อมูลหรือการตั้งค่าที่แตกต่างกันโดยอ้อมครับ/ค่ะ
