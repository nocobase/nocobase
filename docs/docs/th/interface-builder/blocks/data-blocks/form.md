# ฟอร์มบล็อก

## แนะนำ

ฟอร์มบล็อกเป็นบล็อกสำคัญที่ใช้สำหรับสร้างอินเทอร์เฟซการป้อนข้อมูลและแก้ไขข้อมูลครับ มีความสามารถในการปรับแต่งสูง โดยจะใช้คอมโพเนนต์ที่เหมาะสมเพื่อแสดงฟิลด์ที่ต้องการตามโมเดลข้อมูลครับ ผ่านกระแสเหตุการณ์เช่นกฎการเชื่อมโยง ฟอร์มบล็อกสามารถแสดงฟิลด์แบบไดนามิกได้ครับ นอกจากนี้ ยังสามารถรวมเข้ากับเวิร์กโฟลว์เพื่อเรียกใช้กระบวนการอัตโนมัติและจัดการข้อมูล ซึ่งจะช่วยเพิ่มประสิทธิภาพในการทำงานหรือจัดลำดับการทำงานเชิงตรรกะได้ครับ

## เพิ่มฟอร์มบล็อก

- **แก้ไขฟอร์ม**: ใช้สำหรับแก้ไขข้อมูลที่มีอยู่ครับ
- **เพิ่มฟอร์ม**: ใช้สำหรับสร้างรายการข้อมูลใหม่ครับ

## รายการการตั้งค่าบล็อก

![20251023191448](https://static-docs.nocobase.com/20251023191448.png)

### กฎการเชื่อมโยงบล็อก

ควบคุมพฤติกรรมของบล็อกผ่านกฎการเชื่อมโยง (เช่น การแสดงผลหรือการรัน javaScript) ครับ

![20251023191703](https://static-docs.nocobase.com/20251023191703.png)

ดูรายละเอียดเพิ่มเติมที่ [กฎการเชื่อมโยงบล็อก](/interface-builder/blocks/block-settings/block-linkage-rule)

### กฎการเชื่อมโยงฟิลด์

ควบคุมพฤติกรรมของฟิลด์ในฟอร์มผ่านกฎการเชื่อมโยงครับ

![20251023191849](https://static-docs.nocobase.com/20251023191849.png)

ดูรายละเอียดเพิ่มเติมที่ [กฎการเชื่อมโยงฟิลด์](/interface-builder/blocks/block-settings/field-linkage-rule)

### เลย์เอาต์

ฟอร์มบล็อกรองรับรูปแบบการจัดวางสองแบบ ซึ่งตั้งค่าผ่านคุณสมบัติ `layout` ครับ:

- **horizontal** (เลย์เอาต์แนวนอน): เลย์เอาต์นี้ช่วยให้ป้ายกำกับและเนื้อหาแสดงอยู่ในบรรทัดเดียวกัน ช่วยประหยัดพื้นที่แนวตั้ง เหมาะสำหรับฟอร์มที่เรียบง่ายหรือกรณีที่มีข้อมูลน้อยครับ
- **vertical** (เลย์เอาต์แนวตั้ง) (ค่าเริ่มต้น): ป้ายกำกับจะอยู่เหนือฟิลด์ เลย์เอาต์นี้ช่วยให้ฟอร์มอ่านและกรอกได้ง่ายขึ้น โดยเฉพาะอย่างยิ่งสำหรับฟอร์มที่มีหลายฟิลด์หรือรายการป้อนข้อมูลที่ซับซ้อนครับ

![20251023193638](https://static-docs.nocobase.com/20251023193638.png)

## การกำหนดค่าฟิลด์

### ฟิลด์ของคอลเลกชันนี้

> **หมายเหตุ**: ฟิลด์จากคอลเลกชันที่สืบทอดมา (นั่นคือฟิลด์ของคอลเลกชันหลัก) จะถูกรวมและแสดงในรายการฟิลด์ปัจจุบันโดยอัตโนมัติครับ

![20240416230739](https://static-docs.nocobase.com/20240416230739.png)

### ฟิลด์ของคอลเลกชันความสัมพันธ์

> ฟิลด์ของคอลเลกชันความสัมพันธ์เป็นแบบอ่านอย่างเดียวในฟอร์ม โดยปกติจะใช้ร่วมกับฟิลด์ความสัมพันธ์เพื่อแสดงค่าฟิลด์หลายค่าของข้อมูลที่เกี่ยวข้องครับ

![20260212161035](https://static-docs.nocobase.com/20260212161035.png)

- ปัจจุบันรองรับเฉพาะความสัมพันธ์แบบหนึ่งต่อหนึ่ง (เช่น belongsTo / hasOne เป็นต้น) ครับ
- โดยปกติจะใช้ร่วมกับฟิลด์ความสัมพันธ์ (ที่ใช้สำหรับเลือกบันทึกที่เกี่ยวข้อง): คอมโพเนนต์ฟิลด์ความสัมพันธ์มีหน้าที่ในการเลือก/เปลี่ยนแปลงบันทึกที่เกี่ยวข้อง ส่วนฟิลด์ของคอลเลกชันความสัมพันธ์มีหน้าที่แสดงข้อมูลเพิ่มเติมของบันทึกนั้น (อ่านอย่างเดียว) ครับ

**ตัวอย่าง**: หลังจากเลือก 「ผู้รับผิดชอบ」 แล้ว จะแสดงหมายเลขโทรศัพท์ อีเมล และข้อมูลอื่นๆ ของผู้รับผิดชอบคนนั้นในฟอร์มครับ

> ในฟอร์มแก้ไข หากไม่ได้กำหนดค่าฟิลด์ความสัมพันธ์ 「ผู้รับผิดชอบ」 ก็ยังสามารถแสดงข้อมูลที่เกี่ยวข้องได้ เมื่อมีการกำหนดค่าฟิลด์ความสัมพันธ์ 「ผู้รับผิดชอบ」 แล้ว เมื่อเปลี่ยนผู้รับผิดชอบ ข้อมูลที่เกี่ยวข้องจะถูกอัปเดตเป็นบันทึกที่เกี่ยวข้องครับ

![20260212160748](https://static-docs.nocobase.com/20260212160748.gif)
