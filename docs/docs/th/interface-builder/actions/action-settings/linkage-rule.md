# กฎการเชื่อมโยงการทำงาน

## บทนำ

กฎการเชื่อมโยงการทำงานช่วยให้ผู้ใช้สามารถควบคุมสถานะของการทำงาน (เช่น แสดง, เปิดใช้งาน, ซ่อน, ปิดใช้งาน เป็นต้น) ได้อย่างไดนามิกตามเงื่อนไขที่กำหนดครับ/ค่ะ ด้วยการตั้งค่ากฎเหล่านี้ ผู้ใช้สามารถเชื่อมโยงพฤติกรรมการทำงานของปุ่มเข้ากับเรคคอร์ดปัจจุบัน, บทบาทของผู้ใช้ หรือข้อมูลบริบทได้ครับ/ค่ะ

![20251029150224](https://static-docs.nocobase.com/20251029150224.png)

## วิธีใช้งาน

เมื่อเงื่อนไขเป็นจริง (หากไม่มีการตั้งค่าเงื่อนไข ระบบจะถือว่าผ่านโดยอัตโนมัติ) ระบบจะเรียกใช้การตั้งค่าคุณสมบัติ หรือ JavaScript ครับ/ค่ะ รองรับการใช้ค่าคงที่ (Constants) และตัวแปร (Variables) ในการพิจารณาเงื่อนไขครับ/ค่ะ

![20251030224601](https://static-docs.nocobase.com/20251030224601.png)

กฎนี้รองรับการแก้ไขคุณสมบัติของปุ่มครับ/ค่ะ

![20251029150452](https://static-docs.nocobase.com/20251029150452.png)

## ค่าคงที่ (Constants)

ตัวอย่าง: คำสั่งซื้อที่ชำระเงินแล้ว จะไม่สามารถแก้ไขได้ครับ/ค่ะ

![20251029150638](https://static-docs.nocobase.com/20251029150638.png)

## ตัวแปร (Variables)

### ตัวแปรระบบ (System Variables)

![20251029150014](https://static-docs.nocobase.com/20251029150014.png)

ตัวอย่างที่ 1: ควบคุมการแสดงผลของปุ่มตามประเภทอุปกรณ์ปัจจุบันครับ/ค่ะ

![20251029151057](https://static-docs.nocobase.com/20251029151057.png)

ตัวอย่างที่ 2: ปุ่มอัปเดตจำนวนมาก (Bulk Update) ที่ส่วนหัวของตารางบล็อกคำสั่งซื้อ จะใช้งานได้เฉพาะบทบาทผู้ดูแลระบบ (Admin) เท่านั้นครับ/ค่ะ บทบาทอื่น ๆ จะไม่สามารถดำเนินการนี้ได้ครับ/ค่ะ

![20251029151209](https://static-docs.nocobase.com/20251029151209.png)

### ตัวแปรบริบท (Contextual Variables)

ตัวอย่าง: ปุ่มเพิ่ม (Add) บนโอกาสทางธุรกิจของคำสั่งซื้อ (บล็อกความสัมพันธ์) จะเปิดใช้งานได้เฉพาะเมื่อสถานะคำสั่งซื้อเป็น 'รอชำระเงิน' หรือ 'ฉบับร่าง' เท่านั้นครับ/ค่ะ ในสถานะอื่น ๆ ปุ่มจะถูกปิดใช้งานครับ/ค่ะ

![20251029151520](https://static-docs.nocobase.com/20251029151520.png)

![20251029152200](https://static-docs.nocobase.com/20251029152200.png)
