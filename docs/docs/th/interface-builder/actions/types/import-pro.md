# Import Pro

## บทนำ

ปลั๊กอิน Import Pro มีฟังก์ชันที่ได้รับการปรับปรุงเพิ่มเติมจากฟังก์ชันการนำเข้ามาตรฐานทั่วไปครับ/ค่ะ

## การติดตั้ง

ปลั๊กอินนี้ต้องพึ่งพาปลั๊กอินการจัดการงานแบบ Asynchronous ครับ/ค่ะ ก่อนใช้งาน คุณต้องเปิดใช้งานปลั๊กอินการจัดการงานแบบ Asynchronous ก่อนนะครับ/คะ

## การปรับปรุงฟังก์ชัน

![20251029172052](https://static-docs.nocobase.com/20251029172052.png)

- รองรับการดำเนินการนำเข้าแบบ Asynchronous ซึ่งทำงานในเธรดแยกต่างหาก และรองรับการนำเข้าข้อมูลจำนวนมากครับ/ค่ะ

- รองรับตัวเลือกการนำเข้าขั้นสูงครับ/ค่ะ

## คู่มือการใช้งาน

### การนำเข้าแบบ Asynchronous

หลังจากที่คุณดำเนินการนำเข้าแล้ว กระบวนการนำเข้าจะทำงานในเธรดเบื้องหลังที่แยกต่างหาก โดยไม่ต้องให้ผู้ใช้กำหนดค่าด้วยตนเองครับ/ค่ะ ในส่วนติดต่อผู้ใช้ (UI) หลังจากเริ่มการนำเข้า งานนำเข้าที่กำลังทำงานอยู่จะแสดงที่มุมขวาบน พร้อมทั้งแสดงความคืบหน้าของงานแบบเรียลไทม์ครับ/ค่ะ

![index-2024-12-30-09-21-05](https://static-docs.nocobase.com/index-2024-12-30-09-21-05.png)

เมื่อการนำเข้าเสร็จสมบูรณ์ คุณสามารถดูผลลัพธ์ได้ในส่วนของงานนำเข้าครับ/ค่ะ

จากผลการทดสอบประสิทธิภาพข้างต้นและการออกแบบที่มีอยู่ มีคำอธิบายและข้อเสนอแนะเกี่ยวกับปัจจัยที่มีผลกระทบดังต่อไปนี้ครับ/ค่ะ

1.  **กลไกการจัดการรายการซ้ำ**: เมื่อคุณเลือกตัวเลือก **อัปเดตรายการซ้ำ** หรือ **อัปเดตเฉพาะรายการซ้ำ** ระบบจะดำเนินการค้นหาและอัปเดตทีละรายการ ซึ่งจะลดประสิทธิภาพการนำเข้าลงอย่างมากครับ/ค่ะ หากไฟล์ Excel ของคุณมีข้อมูลซ้ำที่ไม่จำเป็น จะส่งผลกระทบต่อความเร็วในการนำเข้าอย่างมากยิ่งขึ้นครับ/ค่ะ ขอแนะนำให้ล้างข้อมูลซ้ำที่ไม่จำเป็นในไฟล์ Excel (เช่น ใช้เครื่องมือเฉพาะทางในการลบข้อมูลซ้ำ) ก่อนนำเข้าสู่ระบบ เพื่อหลีกเลี่ยงการเสียเวลาโดยไม่จำเป็นครับ/ค่ะ

2.  **ประสิทธิภาพการประมวลผลฟิลด์ความสัมพันธ์**: ระบบประมวลผลฟิลด์ความสัมพันธ์โดยการสอบถามความสัมพันธ์ทีละรายการ ซึ่งอาจกลายเป็นคอขวดด้านประสิทธิภาพในสถานการณ์ที่มีข้อมูลจำนวนมากครับ/ค่ะ สำหรับโครงสร้างความสัมพันธ์ที่เรียบง่าย (เช่น ความสัมพันธ์แบบหนึ่งต่อหลายระหว่างสองคอลเลกชัน) ขอแนะนำกลยุทธ์การนำเข้าแบบหลายขั้นตอน โดยนำเข้าข้อมูลพื้นฐานของคอลเลกชันหลักก่อน แล้วจึงสร้างความสัมพันธ์ระหว่างคอลเลกชันหลังจากเสร็จสิ้นครับ/ค่ะ หากความต้องการทางธุรกิจจำเป็นต้องนำเข้าข้อมูลความสัมพันธ์พร้อมกัน โปรดอ้างอิงผลการทดสอบประสิทธิภาพในตารางด้านบนเพื่อวางแผนเวลาการนำเข้าอย่างเหมาะสมนะครับ/คะ

3.  **กลไกการประมวลผลเวิร์กโฟลว์**: ไม่แนะนำให้เปิดใช้งานการทริกเกอร์เวิร์กโฟลว์ในสถานการณ์การนำเข้าข้อมูลขนาดใหญ่ครับ/ค่ะ โดยหลักแล้วมีสองเหตุผลดังนี้:
    -   แม้ว่าสถานะของงานนำเข้าจะแสดง 100% แต่ก็ยังไม่สิ้นสุดทันทีครับ/ค่ะ ระบบยังคงต้องการเวลาเพิ่มเติมในการสร้างแผนการดำเนินการเวิร์กโฟลว์ ในขั้นตอนนี้ ระบบจะสร้างแผนการดำเนินการเวิร์กโฟลว์ที่สอดคล้องกันสำหรับข้อมูลที่นำเข้าแต่ละรายการ ซึ่งจะใช้เธรดการนำเข้า แต่ไม่ส่งผลกระทบต่อการใช้งานข้อมูลที่นำเข้าแล้วครับ/ค่ะ
    -   หลังจากงานนำเข้าเสร็จสมบูรณ์โดยสมบูรณ์ การดำเนินการเวิร์กโฟลว์จำนวนมากพร้อมกันอาจทำให้ทรัพยากรระบบตึงเครียด ซึ่งส่งผลกระทบต่อความเร็วในการตอบสนองของระบบโดยรวมและประสบการณ์ของผู้ใช้ครับ/ค่ะ

ปัจจัยที่มีผลกระทบทั้ง 3 ข้อข้างต้น จะได้รับการพิจารณาเพื่อปรับปรุงให้ดียิ่งขึ้นในอนาคตครับ/ค่ะ

### การกำหนดค่าการนำเข้า

#### ตัวเลือกการนำเข้า - ทริกเกอร์เวิร์กโฟลว์

![20251029172235](https://static-docs.nocobase.com/20251029172235.png)

คุณสามารถเลือกได้ว่าจะทริกเกอร์เวิร์กโฟลว์หรือไม่ในระหว่างการนำเข้าครับ/ค่ะ หากเลือกตัวเลือกนี้และคอลเลกชันนั้นผูกกับเวิร์กโฟลว์ (เหตุการณ์คอลเลกชัน) การนำเข้าจะทริกเกอร์การดำเนินการเวิร์กโฟลว์สำหรับแต่ละแถวครับ/ค่ะ

ตัวเลือกในการกำหนดค่าการนำเข้าจะถูกนำไปใช้เป็นค่าเริ่มต้นครับ/ค่ะ ผู้ดูแลระบบสามารถควบคุมได้ว่าจะอนุญาตให้ผู้ที่อัปโหลดแก้ไขตัวเลือกเหล่านี้หรือไม่ (ยกเว้นตัวเลือกการทริกเกอร์เวิร์กโฟลว์) ครับ/ค่ะ

##### คำอธิบายโหมด

- **ข้ามรายการซ้ำ**: ระบบจะค้นหารายการที่มีอยู่โดยอ้างอิงจากเนื้อหาของ "ฟิลด์ตัวระบุ" ครับ/ค่ะ หากรายการมีอยู่แล้ว จะข้ามแถวนี้ไป แต่หากไม่มีอยู่ จะนำเข้าเป็นรายการใหม่ครับ/ค่ะ
- **อัปเดตรายการซ้ำ**: ระบบจะค้นหารายการที่มีอยู่โดยอ้างอิงจากเนื้อหาของ "ฟิลด์ตัวระบุ" ครับ/ค่ะ หากรายการมีอยู่แล้ว จะอัปเดตรายการนี้ แต่หากไม่มีอยู่ จะนำเข้าเป็นรายการใหม่ครับ/ค่ะ
- **อัปเดตเฉพาะรายการซ้ำ**: ระบบจะค้นหารายการที่มีอยู่โดยอ้างอิงจากเนื้อหาของ "ฟิลด์ตัวระบุ" ครับ/ค่ะ หากรายการมีอยู่แล้ว จะอัปเดตรายการนี้ แต่หากไม่มีอยู่ จะข้ามไปครับ/ค่ะ

##### ฟิลด์ตัวระบุ

ระบบจะระบุว่าแถวนั้นเป็นรายการซ้ำหรือไม่ โดยอ้างอิงจากค่าของฟิลด์นี้ครับ/ค่ะ

- [กฎการเชื่อมโยง](/interface-builder/actions/action-settings/linkage-rule): แสดง/ซ่อนปุ่มแบบไดนามิกครับ/ค่ะ
- [ปุ่มแก้ไข](/interface-builder/actions/action-settings/edit-button): แก้ไขชื่อ, ประเภท และไอคอนของปุ่มครับ/ค่ะ
