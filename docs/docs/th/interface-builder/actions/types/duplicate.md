# การทำซ้ำ (Duplicate)

## บทนำ

การดำเนินการทำซ้ำช่วยให้ผู้ใช้สามารถสร้างบันทึกใหม่ได้อย่างรวดเร็วโดยอ้างอิงจากข้อมูลที่มีอยู่ รองรับรูปแบบการทำซ้ำ 2 วิธี: **การทำซ้ำโดยตรง** และ **การทำซ้ำไปยังฟอร์มและกรอกข้อมูลต่อ** ครับ

## การติดตั้ง

เป็นปลั๊กอินที่ติดตั้งมาในตัว ไม่จำเป็นต้องติดตั้งเพิ่มเติมครับ

## โหมดการทำซ้ำ

![20260209224344](https://static-docs.nocobase.com/20260209224344.png)

### การทำซ้ำโดยตรง

![20260209224506](https://static-docs.nocobase.com/20260209224506.png)

- ดำเนินการในรูปแบบ "การทำซ้ำโดยตรง" เป็นค่าเริ่มต้นครับ
- **ฟิลด์เทมเพลต (Template fields)**: ระบุฟิลด์ที่ต้องการทำซ้ำ สามารถเลือกทั้งหมดได้ และเป็นส่วนที่จำเป็นต้องกำหนดค่าครับ

![20260209225910](https://static-docs.nocobase.com/20260209225910.gif)

เมื่อกำหนดค่าเสร็จสิ้น ให้คลิกปุ่มเพื่อทำซ้ำข้อมูลครับ

### การทำซ้ำไปยังฟอร์มและกรอกข้อมูลต่อ

ฟิลด์เทมเพลตที่กำหนดค่าไว้จะถูกเติมลงในฟอร์มเป็น **ค่าเริ่มต้น (Default values)** ผู้ใช้สามารถแก้ไขค่าเหล่านั้นก่อนส่งฟอร์มเพื่อเสร็จสิ้นการทำซ้ำครับ

![20260209224704](https://static-docs.nocobase.com/20260209224704.png)

**กำหนดค่าฟิลด์เทมเพลต**: เฉพาะฟิลด์ที่เลือกไว้เท่านั้นที่จะถูกดึงข้อมูลมาเป็นค่าเริ่มต้นครับ

![20260209225148](https://static-docs.nocobase.com/20260209225148.png)

#### ซิงค์ฟิลด์ของฟอร์ม

- วิเคราะห์ฟิลด์ที่กำหนดค่าไว้ในบล็อกฟอร์มปัจจุบันโดยอัตโนมัติเพื่อใช้เป็นฟิลด์เทมเพลตครับ
- หากมีการแก้ไขฟิลด์ในบล็อกฟอร์มในภายหลัง (เช่น การปรับเปลี่ยนคอมโพเนนต์ของฟิลด์ความสัมพันธ์) จำเป็นต้องเปิดการกำหนดค่าเทมเพลตอีกครั้งและคลิก **ซิงค์ฟิลด์ของฟอร์ม** เพื่อให้ข้อมูลสอดคล้องกันครับ

![20260209225450](https://static-docs.nocobase.com/20260209225450.gif)

ข้อมูลเทมเพลตจะถูกเติมเป็นค่าเริ่มต้นของฟอร์ม และผู้ใช้สามารถส่งฟอร์มหลังจากแก้ไขข้อมูลเพื่อเสร็จสิ้นการทำซ้ำครับ

### คำอธิบายเพิ่มเติม

#### การทำซ้ำ, การอ้างอิง, การโหลดล่วงหน้า

ฟิลด์ประเภทต่างๆ (ประเภทความสัมพันธ์) จะมีตรรกะการประมวลผลที่แตกต่างกัน: **การทำซ้ำ (Duplicate) / การอ้างอิง (Reference) / การโหลดล่วงหน้า (Preload)** นอกจากนี้ **คอมโพเนนต์ของฟิลด์ (Field component)** ของฟิลด์ความสัมพันธ์ยังมีผลต่อตรรกะนี้ด้วยครับ:

- Select / Record picker: ใช้สำหรับ **การอ้างอิง**
- Sub-form / Sub-table: ใช้สำหรับ **การทำซ้ำ**

**การทำซ้ำ (Duplicate)**

- ฟิลด์ทั่วไปจะเป็นการทำซ้ำครับ
- `hasOne` / `hasMany` สามารถเป็นการทำซ้ำได้เท่านั้น (ความสัมพันธ์ประเภทนี้ไม่ควรใช้คอมโพเนนต์การเลือก เช่น Select หรือ Record picker แต่ควรใช้คอมโพเนนต์ Sub-form หรือ Sub-table แทนครับ)
- การเปลี่ยนคอมโพเนนต์สำหรับ `hasOne` / `hasMany` **จะไม่** เปลี่ยนตรรกะการประมวลผล (ยังคงเป็นการทำซ้ำ)
- สำหรับฟิลด์ความสัมพันธ์ที่ทำซ้ำ ฟิลด์ย่อยทั้งหมดจะสามารถเลือกได้ครับ

**การอ้างอิง (Reference)**

- `belongsTo` / `belongsToMany` จะถือเป็นการอ้างอิงครับ
- หากเปลี่ยนคอมโพเนนต์ของฟิลด์จาก "Select" เป็น "Sub-form" ความสัมพันธ์จะเปลี่ยนจาก **การอ้างอิงเป็นการทำซ้ำ** (เมื่อเปลี่ยนเป็นการทำซ้ำแล้ว ฟิลด์ย่อยทั้งหมดจะสามารถเลือกได้ครับ)

**การโหลดล่วงหน้า (Preload)**

- ฟิลด์ความสัมพันธ์ที่อยู่ภายใต้ฟิลด์การอ้างอิงจะถือเป็นการโหลดล่วงหน้าครับ
- ฟิลด์การโหลดล่วงหน้าอาจเปลี่ยนเป็นการอ้างอิงหรือการทำซ้ำหลังจากมีการเปลี่ยนคอมโพเนนต์ครับ

#### เลือกทั้งหมด

- จะเลือก **ฟิลด์การทำซ้ำ** และ **ฟิลด์การอ้างอิง** ทั้งหมดครับ

#### บันทึกที่ถูกเลือกเป็นเทมเพลตข้อมูลจะถูกกรองฟิลด์ต่อไปนี้ออก:

- คีย์หลัก (Primary keys) ของข้อมูลความสัมพันธ์ที่ทำซ้ำจะถูกกรองออก ส่วนคีย์หลักสำหรับการอ้างอิงและการโหลดล่วงหน้าจะไม่ถูกกรองออกครับ
- คีย์นอก (Foreign keys)
- ฟิลด์ที่ไม่อนุญาตให้ซ้ำ (Unique)
- ฟิลด์การเรียงลำดับ (Sort fields)
- ฟิลด์รหัสอัตโนมัติ (Sequence fields)
- รหัสผ่าน
- ผู้สร้าง, วันที่สร้าง
- ผู้แก้ไขล่าสุด, วันที่แก้ไขล่าสุด

#### ซิงค์ฟิลด์ของฟอร์ม

- วิเคราะห์ฟิลด์ที่กำหนดค่าไว้ในบล็อกฟอร์มปัจจุบันให้เป็นฟิลด์เทมเพลตโดยอัตโนมัติครับ
- หลังจากแก้ไขฟิลด์ในบล็อกฟอร์ม (เช่น การปรับเปลี่ยนคอมโพเนนต์ของฟิลด์ความสัมพันธ์) คุณต้องทำการซิงค์อีกครั้งเพื่อให้แน่ใจว่าข้อมูลสอดคล้องกันครับ
