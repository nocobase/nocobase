# การจัดการการเผยแพร่

## บทนำ

ในการใช้งานจริง เพื่อให้มั่นใจในความปลอดภัยของข้อมูลและการทำงานที่เสถียรของแอปพลิเคชัน เรามักจะต้องติดตั้งใช้งานหลายสภาพแวดล้อม เช่น สภาพแวดล้อมสำหรับการพัฒนา (development environment), สภาพแวดล้อมสำหรับการทดสอบก่อนเผยแพร่ (pre-release environment) และสภาพแวดล้อมสำหรับการใช้งานจริง (production environment) เอกสารนี้จะยกตัวอย่างกระบวนการพัฒนาแบบ No-Code สองแบบที่พบบ่อย และอธิบายรายละเอียดเกี่ยวกับวิธีการจัดการการเผยแพร่ใน NocoBase ครับ/ค่ะ

## การติดตั้ง

ปลั๊กอินสามตัวนี้มีความสำคัญอย่างยิ่งสำหรับการจัดการการเผยแพร่ โปรดตรวจสอบให้แน่ใจว่าได้เปิดใช้งานปลั๊กอินต่อไปนี้แล้วครับ/ค่ะ

### ตัวแปรสภาพแวดล้อม

- เป็นปลั๊กอินในตัวที่ติดตั้งและเปิดใช้งานโดยค่าเริ่มต้นครับ/ค่ะ
- ใช้สำหรับตั้งค่าและจัดการตัวแปรสภาพแวดล้อมและคีย์แบบรวมศูนย์ เพื่อจัดเก็บข้อมูลที่ละเอียดอ่อน, นำข้อมูลการตั้งค่ากลับมาใช้ใหม่, แยกการตั้งค่าตามสภาพแวดล้อม และอื่น ๆ ครับ/ค่ะ ([ดูเอกสาร](#))

### ตัวจัดการการสำรองข้อมูล (Backup Manager)

- ปลั๊กอินนี้ใช้งานได้เฉพาะในเวอร์ชัน Professional ขึ้นไปเท่านั้นครับ/ค่ะ ([ดูรายละเอียดเพิ่มเติม](https://www.nocobase.com/en/commercial))
- มีฟังก์ชันสำรองและกู้คืนข้อมูล รองรับการสำรองข้อมูลตามกำหนดเวลา เพื่อให้มั่นใจในความปลอดภัยของข้อมูลและการกู้คืนที่รวดเร็วครับ/ค่ะ ([ดูเอกสาร](../backup-manager/index.mdx))

### ตัวจัดการการย้ายข้อมูล (Migration Manager)

- ปลั๊กอินนี้ใช้งานได้เฉพาะในเวอร์ชัน Professional ขึ้นไปเท่านั้นครับ/ค่ะ ([ดูรายละเอียดเพิ่มเติม](https://www.nocobase.com/en/commercial))
- ใช้สำหรับย้ายการตั้งค่าแอปพลิเคชันจากสภาพแวดล้อมหนึ่งไปยังอีกสภาพแวดล้อมหนึ่งครับ/ค่ะ ([ดูเอกสาร](../migration-manager/index.md))

## กระบวนการพัฒนาแบบ No-Code ที่พบบ่อย

### สภาพแวดล้อมการพัฒนาเดียว, การเผยแพร่แบบทางเดียว

เหมาะสำหรับกระบวนการพัฒนาที่ไม่ซับซ้อน โดยมีสภาพแวดล้อมสำหรับการพัฒนา, การทดสอบก่อนเผยแพร่ และการใช้งานจริง อย่างละหนึ่งชุด การเปลี่ยนแปลงจะถูกเผยแพร่จากสภาพแวดล้อมการพัฒนาไปยังสภาพแวดล้อมการทดสอบก่อนเผยแพร่ และสุดท้ายจะถูกนำไปใช้งานในสภาพแวดล้อมการใช้งานจริง ในกระบวนการนี้ เฉพาะสภาพแวดล้อมการพัฒนาเท่านั้นที่สามารถแก้ไขการตั้งค่าได้ ส่วนสภาพแวดล้อมการทดสอบก่อนเผยแพร่และสภาพแวดล้อมการใช้งานจริงจะไม่ได้รับอนุญาตให้แก้ไขครับ/ค่ะ

![20250106234710](https://static-docs.nocobase.com/20250106234710.png)

เมื่อตั้งค่ากฎการย้ายข้อมูล สำหรับตารางใน Core และปลั๊กอินที่มาพร้อมระบบ ให้เลือกกฎ **"ครอบทับก่อน" (Overwrite Prioritize)** ส่วนรายการอื่น ๆ หากไม่มีความจำเป็นพิเศษ สามารถใช้ค่าเริ่มต้นได้เลยครับ/ค่ะ

![20250105194845](https://static-docs.nocobase.com/20250105194845.png)

### หลายสภาพแวดล้อมการพัฒนา, การเผยแพร่แบบรวม

เหมาะสำหรับสถานการณ์การทำงานร่วมกันหลายคนหรือโปรเจกต์ที่ซับซ้อน สภาพแวดล้อมการพัฒนาหลายชุดสามารถทำงานได้อย่างอิสระ การเปลี่ยนแปลงทั้งหมดจะถูกรวมเข้าด้วยกันในสภาพแวดล้อมการทดสอบก่อนเผยแพร่เพื่อทำการทดสอบและตรวจสอบ จากนั้นจึงเผยแพร่ไปยังสภาพแวดล้อมการใช้งานจริง ในกระบวนการนี้ เฉพาะสภาพแวดล้อมการพัฒนาเท่านั้นที่สามารถแก้ไขการตั้งค่าได้ ส่วนสภาพแวดล้อมการทดสอบก่อนเผยแพร่และสภาพแวดล้อมการใช้งานจริงจะไม่ได้รับอนุญาตให้แก้ไขครับ/ค่ะ

![20250107103829](https://static-docs.nocobase.com/20250107103829.png)

เมื่อตั้งค่ากฎการย้ายข้อมูล สำหรับตารางใน Core และปลั๊กอินที่มาพร้อมระบบ ให้เลือกกฎ **"แทรกหรืออัปเดตก่อน" (Insert or Update Prioritize)** ส่วนรายการอื่น ๆ หากไม่มีความจำเป็นพิเศษ สามารถใช้ค่าเริ่มต้นได้เลยครับ/ค่ะ

![20250105194942](https://static-docs.nocobase.com/20250105194942.png)

## การย้อนกลับ (Rollback)

ก่อนที่จะดำเนินการย้ายข้อมูล ระบบจะทำการสำรองข้อมูลแอปพลิเคชันปัจจุบันโดยอัตโนมัติ หากการย้ายข้อมูลล้มเหลวหรือไม่เป็นไปตามที่คาดหวัง คุณสามารถย้อนกลับและกู้คืนได้ผ่าน [ตัวจัดการการสำรองข้อมูล](../backup-manager/index.mdx) ครับ/ค่ะ
