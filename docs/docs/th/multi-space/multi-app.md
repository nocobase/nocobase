# มัลติแอป (Multi-app)


## บทนำ

**ปลั๊กอินมัลติแอป (Multi-app)** ช่วยให้สามารถสร้างและจัดการแอปพลิเคชันอิสระหลายรายการได้แบบไดนามิกโดยไม่ต้องแยกการติดตั้ง (Deployment) แต่ละแอปย่อย (Sub-app) จะเป็นอินสแตนซ์ที่เป็นอิสระต่อกันอย่างสมบูรณ์ โดยมีฐานข้อมูล ปลั๊กอิน และการกำหนดค่าเป็นของตัวเองครับ

#### สถานการณ์ที่เหมาะสม
- **มัลติเทแนนซี (Multi-tenancy)**: จัดเตรียมอินสแตนซ์แอปพลิเคชันที่แยกจากกัน โดยที่ลูกค้าแต่ละรายจะมีข้อมูล การกำหนดค่าปลั๊กอิน และระบบสิทธิ์เป็นของตนเอง
- **ระบบหลักและระบบย่อยสำหรับโดเมนธุรกิจที่แตกต่างกัน**: ระบบขนาดใหญ่ที่ประกอบด้วยแอปพลิเคชันขนาดเล็กหลายแอปที่ติดตั้งแยกกัน


:::warning
ปลั๊กอินมัลติแอปไม่ได้ให้ความสามารถในการแชร์ผู้ใช้ในตัว หากต้องการเชื่อมต่อผู้ใช้ระหว่างแอปต่างๆ สามารถใช้งานร่วมกับ **[ปลั๊กอินการยืนยันตัวตน (Authentication Plugin)](/auth-verification)** ได้ครับ
:::

## คู่มือการใช้งาน


### การสร้างแอปย่อย

ในเมนูการตั้งค่าระบบ คลิกที่ "มัลติแอป" เพื่อเข้าสู่หน้าการจัดการมัลติแอป:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

คลิกปุ่ม "เพิ่มใหม่" (Add New) เพื่อสร้างแอปย่อยใหม่:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### คำอธิบายฟิลด์ในแบบฟอร์ม

* **ชื่อ (Name)**: ตัวระบุแอปย่อย ต้องไม่ซ้ำกันในระบบ
* **ชื่อที่แสดง (Display Name)**: ชื่อของแอปย่อยที่จะแสดงในหน้าอินเทอร์เฟซ
* **โหมดการเริ่มต้น (Startup Mode)**:
  * **เริ่มทำงานเมื่อมีการเข้าถึงครั้งแรก**: แอปย่อยจะเริ่มทำงานเมื่อมีผู้ใช้เข้าถึงผ่าน URL เป็นครั้งแรกเท่านั้น
  * **เริ่มทำงานพร้อมกับแอปหลัก**: แอปย่อยจะเริ่มทำงานพร้อมกับแอปหลัก (ซึ่งจะเพิ่มเวลาในการเริ่มต้นของแอปหลัก)
* **พอร์ต (Port)**: หมายเลขพอร์ตที่แอปย่อยใช้ในขณะทำงาน
* **โดเมนที่กำหนดเอง (Custom Domain)**: กำหนดค่าโดเมนย่อย (Subdomain) แยกต่างหากสำหรับแอปย่อย
* **ปักหมุดที่เมนู (Pin to Menu)**: ปักหมุดทางเข้าแอปย่อยไว้ที่ด้านซ้ายของแถบนำทางด้านบน
* **การเชื่อมต่อฐานข้อมูล (Database Connection)**: ใช้สำหรับกำหนดค่าแหล่งข้อมูลสำหรับแอปย่อย โดยรองรับ 3 วิธี:
  * **ฐานข้อมูลใหม่**: ใช้บริการข้อมูลปัจจุบันซ้ำเพื่อสร้างฐานข้อมูลอิสระ
  * **การเชื่อมต่อข้อมูลใหม่**: กำหนดค่าบริการฐานข้อมูลใหม่ทั้งหมด
  * **โหมด Schema**: สร้าง Schema แยกต่างหากสำหรับแอปย่อยใน PostgreSQL
* **อัปเกรด (Upgrade)**: หากฐานข้อมูลที่เชื่อมต่อมีโครงสร้างข้อมูล NocoBase เวอร์ชันเก่า ระบบจะอัปเกรดเป็นเวอร์ชันปัจจุบันโดยอัตโนมัติ


### การเริ่มและหยุดการทำงานของแอปย่อย

คลิกปุ่ม **เริ่มทำงาน (Start)** เพื่อเริ่มแอปย่อย  
> หากเลือก *"เริ่มทำงานเมื่อมีการเข้าถึงครั้งแรก"* ไว้ตอนสร้าง แอปจะเริ่มทำงานโดยอัตโนมัติเมื่อมีการเข้าถึงครั้งแรกครับ

คลิกปุ่ม **ดู (View)** เพื่อเปิดแอปย่อยในแท็บใหม่

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)


### สถานะการทำงานและบันทึก (Logs) ของแอปย่อย

คุณสามารถดูการใช้งานหน่วยความจำ (Memory) และ CPU ของแต่ละแอปพลิเคชันได้ในรายการ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

คลิกปุ่ม **บันทึก (Logs)** เพื่อดูบันทึกการทำงานของแอปย่อย  
> หากแอปย่อยไม่สามารถเข้าถึงได้หลังจากเริ่มทำงาน (เช่น ฐานข้อมูลเสียหาย) คุณสามารถตรวจสอบหาสาเหตุได้จากบันทึกครับ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)


### การลบแอปย่อย

คลิกปุ่ม **ลบ (Delete)** เพื่อลบแอปย่อย  
> เมื่อลบ คุณสามารถเลือกได้ว่าจะลบฐานข้อมูลไปด้วยหรือไม่ โปรดดำเนินการด้วยความระมัดระวัง เนื่องจากการดำเนินการนี้ไม่สามารถย้อนกลับได้ครับ


### การเข้าถึงแอปย่อย
โดยปกติจะใช้ `/_app/:appName/admin/` ในการเข้าถึงแอปย่อย ตัวอย่างเช่น:
```
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
นอกจากนี้ คุณยังสามารถกำหนดค่าโดเมนย่อยอิสระสำหรับแอปย่อยได้ โดยต้องชี้โดเมน (Resolve) มาที่ IP ปัจจุบัน หากใช้ Nginx จะต้องเพิ่มโดเมนในไฟล์กำหนดค่าของ Nginx ด้วยครับ


### การจัดการแอปย่อยผ่านบรรทัดคำสั่ง (CLI)

ในไดเรกทอรีรากของโปรเจกต์ (Project root) คุณสามารถใช้บรรทัดคำสั่งผ่าน **PM2** เพื่อจัดการอินสแตนซ์ของแอปย่อยได้:

```bash
yarn nocobase pm2 list              # ดูรายการอินสแตนซ์ที่กำลังรันอยู่
yarn nocobase pm2 stop [appname]    # หยุดโปรเซสของแอปย่อยที่ระบุ
yarn nocobase pm2 delete [appname]  # ลบโปรเซสของแอปย่อยที่ระบุ
yarn nocobase pm2 kill              # บังคับปิดโปรเซสทั้งหมดที่เริ่มทำงาน (อาจรวมถึงอินสแตนซ์ของแอปหลัก)
```

### การย้ายข้อมูลจากมัลติแอปเวอร์ชันเก่า

เข้าไปที่หน้าจัดการมัลติแอปแบบเก่า แล้วคลิกปุ่ม **ย้ายข้อมูลไปยังมัลติแอปใหม่ (Migrate Data to New Multi-app)** เพื่อดำเนินการย้ายข้อมูลครับ

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)


## คำถามที่พบบ่อย (FAQ)

#### 1. การจัดการปลั๊กอิน
แอปย่อยสามารถใช้ปลั๊กอินเดียวกับแอปหลักได้ (รวมถึงเวอร์ชันเดียวกัน) แต่สามารถกำหนดค่าและใช้งานปลั๊กอินแยกกันได้ครับ

#### 2. การแยกฐานข้อมูล
แอปย่อยสามารถกำหนดค่าฐานข้อมูลอิสระได้ หากต้องการแชร์ข้อมูลระหว่างแอปพลิเคชัน สามารถทำได้ผ่านแหล่งข้อมูลภายนอก (External Data Source) ครับ

#### 3. การสำรองข้อมูลและการย้ายข้อมูล
ในปัจจุบัน การสำรองข้อมูลในแอปหลักจะไม่รวมข้อมูลของแอปย่อย (จะรวมเฉพาะข้อมูลพื้นฐานของแอปย่อยเท่านั้น) การสำรองข้อมูลและการย้ายข้อมูลจะต้องดำเนินการด้วยตนเองภายในแต่ละแอปย่อยครับ

#### 4. การติดตั้งและการอัปเดต
เวอร์ชันของแอปย่อยจะอัปเกรดตามแอปหลักโดยอัตโนมัติ เพื่อให้มั่นใจว่าเวอร์ชันของแอปหลักและแอปย่อยตรงกันเสมอครับ

#### 5. การจัดการทรัพยากร
การใช้ทรัพยากรของแต่ละแอปย่อยจะใกล้เคียงกับแอปหลัก โดยปัจจุบันการใช้งานหน่วยความจำของหนึ่งแอปพลิเคชันจะอยู่ที่ประมาณ 500-600MB ครับ
