### การจัดรูปแบบตัวเลข

#### 1. :formatN(precision)

##### คำอธิบายไวยากรณ์
ฟอร์แมตตัวเลขตามการตั้งค่าภาษาและภูมิภาค (localization) ครับ/ค่ะ
พารามิเตอร์:
- `precision`: จำนวนทศนิยมที่ต้องการแสดง
  สำหรับไฟล์รูปแบบ ODS/XLSX จำนวนทศนิยมที่แสดงจะถูกกำหนดโดยโปรแกรมแก้ไขข้อความ แต่สำหรับรูปแบบอื่น ๆ จะขึ้นอยู่กับพารามิเตอร์นี้ครับ/ค่ะ

##### ตัวอย่าง
```
// สภาพแวดล้อมตัวอย่าง: ตัวเลือก API { "lang": "en-us" }
'10':formatN()         // ผลลัพธ์คือ "10.000"
'1000.456':formatN()   // ผลลัพธ์คือ "1,000.456"
```

##### ผลลัพธ์
ตัวเลขจะถูกแสดงผลตามจำนวนทศนิยมที่ระบุและรูปแบบภาษา/ภูมิภาคที่ตั้งค่าไว้ครับ/ค่ะ

#### 2. :round(precision)

##### คำอธิบายไวยากรณ์
ปัดเศษตัวเลขให้เป็นจำนวนทศนิยมตามที่ระบุครับ/ค่ะ

##### ตัวอย่าง
```
10.05123:round(2)      // ผลลัพธ์คือ 10.05
1.05:round(1)          // ผลลัพธ์คือ 1.1
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นตัวเลขที่ถูกปัดเศษแล้วครับ/ค่ะ

#### 3. :add(value)

##### คำอธิบายไวยากรณ์
บวกค่าที่ระบุเข้ากับตัวเลขปัจจุบันครับ/ค่ะ
พารามิเตอร์:
- `value`: ตัวเลขที่ต้องการนำมาบวก

##### ตัวอย่าง
```
1000.4:add(2)         // ผลลัพธ์คือ 1002.4
'1000.4':add('2')      // ผลลัพธ์คือ 1002.4
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นค่าที่ถูกบวกแล้วครับ/ค่ะ

#### 4. :sub(value)

##### คำอธิบายไวยากรณ์
ลบค่าที่ระบุออกจากตัวเลขปัจจุบันครับ/ค่ะ
พารามิเตอร์:
- `value`: ตัวเลขที่ต้องการนำมาลบ

##### ตัวอย่าง
```
1000.4:sub(2)         // ผลลัพธ์คือ 998.4
'1000.4':sub('2')      // ผลลัพธ์คือ 998.4
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นค่าที่ถูกลบแล้วครับ/ค่ะ

#### 5. :mul(value)

##### คำอธิบายไวยากรณ์
คูณตัวเลขปัจจุบันด้วยค่าที่ระบุครับ/ค่ะ
พารามิเตอร์:
- `value`: ตัวเลขที่ต้องการนำมาคูณ

##### ตัวอย่าง
```
1000.4:mul(2)         // ผลลัพธ์คือ 2000.8
'1000.4':mul('2')      // ผลลัพธ์คือ 2000.8
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นค่าที่ถูกคูณแล้วครับ/ค่ะ

#### 6. :div(value)

##### คำอธิบายไวยากรณ์
หารตัวเลขปัจจุบันด้วยค่าที่ระบุครับ/ค่ะ
พารามิเตอร์:
- `value`: ตัวหาร

##### ตัวอย่าง
```
1000.4:div(2)         // ผลลัพธ์คือ 500.2
'1000.4':div('2')      // ผลลัพธ์คือ 500.2
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นค่าที่ถูกหารแล้วครับ/ค่ะ

#### 7. :mod(value)

##### คำอธิบายไวยากรณ์
คำนวณค่าโมดูลัส (เศษเหลือ) ของตัวเลขปัจจุบันเมื่อหารด้วยค่าที่ระบุครับ/ค่ะ
พารามิเตอร์:
- `value`: ตัวหารสำหรับโมดูลัส

##### ตัวอย่าง
```
4:mod(2)              // ผลลัพธ์คือ 0
3:mod(2)              // ผลลัพธ์คือ 1
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นเศษเหลือจากการดำเนินการโมดูลัสครับ/ค่ะ

#### 8. :abs

##### คำอธิบายไวยากรณ์
ส่งคืนค่าสัมบูรณ์ของตัวเลขครับ/ค่ะ

##### ตัวอย่าง
```
-10:abs()             // ผลลัพธ์คือ 10
-10.54:abs()          // ผลลัพธ์คือ 10.54
10.54:abs()           // ผลลัพธ์คือ 10.54
'-200':abs()          // ผลลัพธ์คือ 200
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นค่าสัมบูรณ์ครับ/ค่ะ

#### 9. :ceil

##### คำอธิบายไวยากรณ์
ปัดเศษขึ้น โดยจะส่งคืนจำนวนเต็มที่น้อยที่สุดที่มากกว่าหรือเท่ากับตัวเลขปัจจุบันครับ/ค่ะ

##### ตัวอย่าง
```
10.05123:ceil()       // ผลลัพธ์คือ 11
1.05:ceil()           // ผลลัพธ์คือ 2
-1.05:ceil()          // ผลลัพธ์คือ -1
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นจำนวนเต็มที่ถูกปัดขึ้นแล้วครับ/ค่ะ

#### 10. :floor

##### คำอธิบายไวยากรณ์
ปัดเศษลง โดยจะส่งคืนจำนวนเต็มที่มากที่สุดที่น้อยกว่าหรือเท่ากับตัวเลขปัจจุบันครับ/ค่ะ

##### ตัวอย่าง
```
10.05123:floor()      // ผลลัพธ์คือ 10
1.05:floor()          // ผลลัพธ์คือ 1
-1.05:floor()         // ผลลัพธ์คือ -2
```

##### ผลลัพธ์
ผลลัพธ์ที่ได้จะเป็นจำนวนเต็มที่ถูกปัดลงแล้วครับ/ค่ะ

#### 11. :int

##### คำอธิบายไวยากรณ์
แปลงตัวเลขให้เป็นจำนวนเต็ม (ไม่แนะนำให้ใช้งานครับ/ค่ะ)

##### ตัวอย่างและผลลัพธ์
ขึ้นอยู่กับกรณีการแปลงที่เฉพาะเจาะจงครับ/ค่ะ

#### 12. :toEN

##### คำอธิบายไวยากรณ์
แปลงตัวเลขเป็นรูปแบบภาษาอังกฤษ (ใช้ `.` เป็นจุดทศนิยม) ไม่แนะนำให้ใช้งานครับ/ค่ะ

##### ตัวอย่างและผลลัพธ์
ขึ้นอยู่กับกรณีการแปลงที่เฉพาะเจาะจงครับ/ค่ะ

#### 13. :toFixed

##### คำอธิบายไวยากรณ์
แปลงตัวเลขเป็นสตริง โดยคงจำนวนทศนิยมที่ระบุไว้เท่านั้น (ไม่แนะนำให้ใช้งานครับ/ค่ะ)

##### ตัวอย่างและผลลัพธ์
ขึ้นอยู่กับกรณีการแปลงที่เฉพาะเจาะจงครับ/ค่ะ

#### 14. :toFR

##### คำอธิบายไวยากรณ์
แปลงตัวเลขเป็นรูปแบบภาษาฝรั่งเศส (ใช้ `,` เป็นตัวคั่นทศนิยม) ไม่แนะนำให้ใช้งานครับ/ค่ะ

##### ตัวอย่างและผลลัพธ์
ขึ้นอยู่กับกรณีการแปลงที่เฉพาะเจาะจงครับ/ค่ะ
