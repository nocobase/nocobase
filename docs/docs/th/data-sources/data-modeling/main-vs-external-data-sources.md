# ฐานข้อมูลหลัก vs. ฐานข้อมูลภายนอก

ความแตกต่างระหว่างฐานข้อมูลหลักและฐานข้อมูลภายนอกใน NocoBase หลักๆ แล้วจะเห็นได้จาก 4 ด้านนี้ครับ/ค่ะ: การรองรับประเภทฐานข้อมูล, การรองรับประเภทคอลเลกชัน, การรองรับประเภทฟิลด์ และความสามารถในการสำรองข้อมูล กู้คืน และย้ายข้อมูล

## 1. การรองรับประเภทฐานข้อมูล

ดูรายละเอียดเพิ่มเติมได้ที่: [การจัดการแหล่งข้อมูล](https://docs.nocobase.com/data-sources/data-source-manager)

### ประเภทฐานข้อมูล

| ประเภทฐานข้อมูล | รองรับในฐานข้อมูลหลัก | รองรับในฐานข้อมูลภายนอก |
|------------------|---------------------------|------------------------------|
| PostgreSQL | ✅ | ✅ |
| MySQL | ✅ | ✅ |
| MariaDB | ✅ | ✅ |
| KingbaseES | ✅ | ✅ |
| MSSQL | ❌ | ✅ |
| Oracle | ❌ | ✅ |

### การจัดการคอลเลกชัน

| การจัดการคอลเลกชัน | รองรับในฐานข้อมูลหลัก | รองรับในฐานข้อมูลภายนอก |
|----------------------|-----------------------------|------------------------------|
| การจัดการพื้นฐาน | ✅ | ✅ |
| การจัดการแบบกราฟิก (Visual) | ✅ | ❌ |

## 2. การรองรับประเภทคอลเลกชัน

ดูรายละเอียดเพิ่มเติมได้ที่: [คอลเลกชัน](https://docs.nocobase.com/data-sources/data-modeling/collection)

| ประเภทคอลเลกชัน | ฐานข้อมูลหลัก | ฐานข้อมูลภายนอก | คำอธิบาย |
|----------------|-------------------|---------------------|-------------|
| คอลเลกชันทั่วไป | ✅ | ✅ | คอลเลกชันพื้นฐาน |
| คอลเลกชันวิว | ✅ | ✅ | วิวของแหล่งข้อมูล |
| คอลเลกชันแบบสืบทอด | ✅ | ❌ | รองรับการสืบทอดโมเดลข้อมูล, เฉพาะแหล่งข้อมูลหลักเท่านั้น |
| คอลเลกชันไฟล์ | ✅ | ❌ | รองรับการอัปโหลดไฟล์, เฉพาะแหล่งข้อมูลหลักเท่านั้น |
| คอลเลกชันคอมเมนต์ | ✅ | ❌ | ระบบคอมเมนต์ในตัว, เฉพาะแหล่งข้อมูลหลักเท่านั้น |
| คอลเลกชันปฏิทิน | ✅ | ❌ | คอลเลกชันสำหรับมุมมองปฏิทิน |
| คอลเลกชันนิพจน์ | ✅ | ❌ | รองรับการคำนวณด้วยสูตร |
| คอลเลกชันแบบ Tree | ✅ | ❌ | สำหรับสร้างโมเดลข้อมูลโครงสร้างแบบ Tree |
| คอลเลกชัน SQL | ✅ | ❌ | คอลเลกชันที่กำหนดผ่าน SQL |
| คอลเลกชันเชื่อมต่อภายนอก | ✅ | ❌ | คอลเลกชันสำหรับเชื่อมต่อแหล่งข้อมูลภายนอก, มีฟังก์ชันการทำงานที่จำกัด |

## 4. การเปรียบเทียบการรองรับการสำรองข้อมูลและการย้ายข้อมูล

| ฟังก์ชัน | ฐานข้อมูลหลัก | ฐานข้อมูลภายนอก |
|-----------|-------------------|---------------------|
| สำรองและกู้คืน | ✅ | ❌ (ต้องจัดการเอง) |
| การจัดการการย้ายข้อมูล | ✅ | ❌ (ต้องจัดการเอง) |

:::info
NocoBase มีความสามารถในการสำรองข้อมูล กู้คืน และย้ายโครงสร้างข้อมูลสำหรับฐานข้อมูลหลักครับ/ค่ะ ส่วนฐานข้อมูลภายนอก ผู้ใช้จะต้องดำเนินการเหล่านี้ด้วยตนเองตามสภาพแวดล้อมฐานข้อมูลของตนเอง โดย NocoBase จะไม่มีการรองรับในตัวครับ/ค่ะ
:::

## สรุปการเปรียบเทียบ

| รายการเปรียบเทียบ | ฐานข้อมูลหลัก | ฐานข้อมูลภายนอก |
|--------------------|-------------------|---------------------|
| ประเภทฐานข้อมูล | PostgreSQL, MySQL, MariaDB, KingbaseES | PostgreSQL, MySQL, MariaDB, MSSQL, Oracle, KingbaseES |
| การรองรับประเภทคอลเลกชัน | คอลเลกชันทุกประเภท | รองรับเฉพาะคอลเลกชันทั่วไปและคอลเลกชันวิวเท่านั้น |
| การรองรับประเภทฟิลด์ | ฟิลด์ทุกประเภท | ฟิลด์ทุกประเภท ยกเว้นฟิลด์ไฟล์แนบ |
| การสำรองและย้ายข้อมูล | รองรับในตัว | ต้องจัดการเอง |

## ข้อแนะนำ

- **หากคุณกำลังใช้ NocoBase เพื่อสร้างระบบธุรกิจใหม่ทั้งหมด** ขอแนะนำให้ใช้ **ฐานข้อมูลหลัก** ครับ/ค่ะ เพื่อให้คุณสามารถใช้งานฟังก์ชันทั้งหมดของ NocoBase ได้อย่างเต็มที่
- **แต่หากคุณต้องการใช้ NocoBase เพื่อเชื่อมต่อกับฐานข้อมูลของระบบอื่น ๆ สำหรับการดำเนินการ CRUD (สร้าง, อ่าน, อัปเดต, ลบ) พื้นฐาน** ขอแนะนำให้ใช้ **ฐานข้อมูลภายนอก** ครับ/ค่ะ
