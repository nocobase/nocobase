# ประเภทฟิลด์วันที่และเวลา

## ประเภทฟิลด์วันที่และเวลา

ประเภทฟิลด์วันที่และเวลามีดังต่อไปนี้ครับ/ค่ะ:

- **วันที่และเวลา (พร้อมโซนเวลา):** ค่าเหล่านี้จะถูกแปลงเป็นเวลา UTC (Coordinated Universal Time) มาตรฐาน และจะมีการปรับเปลี่ยนโซนเวลาเมื่อจำเป็นครับ/ค่ะ
- **วันที่และเวลา (ไม่มีโซนเวลา):** ประเภทนี้จะจัดเก็บข้อมูลวันที่และเวลาโดยไม่มีข้อมูลโซนเวลาครับ/ค่ะ
- **วันที่ (ไม่มีเวลา):** รูปแบบนี้จะจัดเก็บเฉพาะข้อมูลวันที่เท่านั้น โดยไม่รวมส่วนของเวลาครับ/ค่ะ
- **เวลา:** จัดเก็บเฉพาะข้อมูลเวลาเท่านั้น โดยไม่รวมรายละเอียดวันที่ครับ/ค่ะ
- **Unix Timestamp:** ประเภทนี้จะแสดงจำนวนวินาทีที่ผ่านไปตั้งแต่ 1 มกราคม 1970 และจัดเก็บเป็น Unix Timestamp ครับ/ค่ะ

นี่คือตัวอย่างสำหรับแต่ละประเภทฟิลด์ที่เกี่ยวข้องกับวันที่และเวลาครับ/ค่ะ:

| **ประเภทฟิลด์** | **ค่าตัวอย่าง** | **คำอธิบาย** |
|---|---|---|
| วันที่และเวลา (พร้อมโซนเวลา) | 2024-08-24T07:30:00.000Z | แปลงเป็น UTC และสามารถปรับตามโซนเวลาได้ |
| วันที่และเวลา (ไม่มีโซนเวลา) | 2024-08-24 15:30:00 | จัดเก็บวันที่และเวลาโดยไม่มีการพิจารณาโซนเวลา |
| วันที่ (ไม่มีเวลา) | 2024-08-24 | จัดเก็บเฉพาะข้อมูลวันที่ โดยไม่มีข้อมูลเวลา |
| เวลา | 15:30:00 | จัดเก็บเฉพาะข้อมูลเวลา โดยไม่รวมรายละเอียดวันที่ |
| Unix Timestamp | 1724437800 | แสดงจำนวนวินาทีที่ผ่านไปตั้งแต่ 1970-01-01 00:00:00 UTC |

## การเปรียบเทียบแหล่งข้อมูล

ตารางเปรียบเทียบระหว่าง NocoBase, MySQL และ PostgreSQL มีดังนี้ครับ/ค่ะ:

| **ประเภทฟิลด์** | **NocoBase** | **MySQL** | **PostgreSQL** |
|---|---|---|---|
| วันที่และเวลา (พร้อมโซนเวลา) | Datetime with timezone | TIMESTAMP<br/> DATETIME | TIMESTAMP WITH TIME ZONE |
| วันที่และเวลา (ไม่มีโซนเวลา) | Datetime without timezone | DATETIME | TIMESTAMP WITHOUT TIME ZONE |
| วันที่ (ไม่มีเวลา) | Date | DATE | DATE |
| เวลา | Time | TIME | TIME WITHOUT TIME ZONE |
| Unix Timestamp | Unix timestamp | INTEGER<br/>BIGINT | INTEGER<br/>BIGINT |
| เวลา (พร้อมโซนเวลา) | - | - | TIME WITH TIME ZONE |

**หมายเหตุ:**
- ประเภท TIMESTAMP ของ MySQL มีช่วงข้อมูลระหว่าง `1970-01-01 00:00:01 UTC` ถึง `2038-01-19 03:14:07 UTC` ครับ/ค่ะ หากวันที่และเวลาอยู่นอกช่วงนี้ ขอแนะนำให้ใช้ DATETIME หรือ BIGINT เพื่อจัดเก็บ Unix Timestamp ครับ/ค่ะ

## ขั้นตอนการประมวลผลการจัดเก็บวันที่และเวลา

### พร้อมโซนเวลา

ส่วนนี้จะรวมถึง `วันที่และเวลา (พร้อมโซนเวลา)` และ `Unix Timestamp` ครับ/ค่ะ

![20240824191933](https://static-docs.nocobase.com/20240824191933.png)

**หมายเหตุ:**
- เพื่อรองรับช่วงข้อมูลที่กว้างขึ้น NocoBase ใช้ประเภท DATETIME ในฐานข้อมูล MySQL สำหรับฟิลด์วันที่และเวลา (พร้อมโซนเวลา) ครับ/ค่ะ ค่าวันที่ที่จัดเก็บจะถูกแปลงตามตัวแปรสภาพแวดล้อม TZ ของเซิร์ฟเวอร์ ซึ่งหมายความว่าหากตัวแปรสภาพแวดล้อม TZ เปลี่ยนแปลง ค่าวันที่และเวลาที่จัดเก็บไว้ก็จะเปลี่ยนแปลงไปด้วยครับ/ค่ะ
- เนื่องจากมีความแตกต่างของโซนเวลาระหว่าง UTC และเวลาท้องถิ่น การแสดงค่า UTC ดิบโดยตรงอาจทำให้ผู้ใช้สับสนได้ครับ/ค่ะ

## UTC

UTC (Coordinated Universal Time) หรือเวลาสากลเชิงพิกัด เป็นมาตรฐานเวลาสากลที่ใช้ในการประสานงานและทำให้เวลาทั่วโลกเป็นหนึ่งเดียวกันครับ/ค่ะ เป็นมาตรฐานเวลาที่มีความแม่นยำสูง ซึ่งดูแลโดยนาฬิกาอะตอม และซิงโครไนซ์กับการหมุนของโลกครับ/ค่ะ

ความแตกต่างระหว่างเวลา UTC และเวลาท้องถิ่นอาจทำให้เกิดความสับสนเมื่อแสดงค่า UTC ดิบครับ/ค่ะ ตัวอย่างเช่น:

| **โซนเวลา** | **วันที่และเวลา** |
|---|---|
| UTC | 2024-08-24T07:30:00.000Z |
| UTC+8 | 2024-08-24 15:30:00 |
| UTC+5 | 2024-08-24 12:30:00 |
| UTC-5 | 2024-08-24 02:30:00 |
| UTC+0 | 2024-08-24 07:30:00 |
| UTC-6 | 2024-08-23 01:30:00 |

เวลาที่แสดงทั้งหมดนี้คือช่วงเวลาเดียวกัน เพียงแต่แสดงในโซนเวลาที่แตกต่างกันครับ/ค่ะ
