# ภาพรวม

การทำ Data Modeling (การสร้างแบบจำลองข้อมูล) เป็นขั้นตอนสำคัญในการออกแบบฐานข้อมูล ซึ่งเกี่ยวข้องกับการวิเคราะห์เชิงลึกและการสรุปข้อมูลประเภทต่างๆ ในโลกแห่งความเป็นจริง รวมถึงความสัมพันธ์ระหว่างข้อมูลเหล่านั้น ในกระบวนการนี้ เราพยายามที่จะเปิดเผยความเชื่อมโยงภายในระหว่างข้อมูล และจัดทำให้อยู่ในรูปแบบของ Data Model เพื่อวางรากฐานสำหรับโครงสร้างฐานข้อมูลของระบบสารสนเทศ NocoBase เป็นแพลตฟอร์มที่ขับเคลื่อนด้วย Data Model และมีคุณสมบัติดังต่อไปนี้ครับ/ค่ะ:

## รองรับการเชื่อมต่อแหล่งข้อมูลที่หลากหลาย

แหล่งข้อมูลของ NocoBase สามารถเป็นได้ทั้งฐานข้อมูลทั่วไป, แพลตฟอร์ม API (SDK) และไฟล์ต่างๆ ครับ/ค่ะ

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

NocoBase มี [ปลั๊กอินจัดการแหล่งข้อมูล](/data-sources/data-source-manager) สำหรับจัดการแหล่งข้อมูลต่างๆ และคอลเลกชันของแหล่งข้อมูลเหล่านั้นครับ/ค่ะ ปลั๊กอินจัดการแหล่งข้อมูลนี้เป็นเพียงส่วนที่ให้หน้าจอสำหรับจัดการแหล่งข้อมูลทั้งหมดเท่านั้น ไม่ได้ให้ความสามารถในการเชื่อมต่อแหล่งข้อมูลโดยตรง แต่ต้องใช้งานร่วมกับปลั๊กอินแหล่งข้อมูลประเภทต่างๆ ครับ/ค่ะ ปัจจุบันแหล่งข้อมูลที่รองรับมีดังนี้ครับ/ค่ะ:

- [ฐานข้อมูลหลัก](/data-sources/data-source-main): ฐานข้อมูลหลักของ NocoBase ที่รองรับฐานข้อมูลเชิงสัมพันธ์ เช่น MySQL, PostgreSQL และ MariaDB ครับ/ค่ะ
- [KingbaseES](/data-sources/data-source-kingbase): ใช้ฐานข้อมูล KingbaseES เป็นแหล่งข้อมูล ซึ่งสามารถใช้เป็นฐานข้อมูลหลักหรือฐานข้อมูลภายนอกก็ได้ครับ/ค่ะ
- [External MySQL](/data-sources/data-source-external-mysql): ใช้ฐานข้อมูล MySQL ภายนอกเป็นแหล่งข้อมูลครับ/ค่ะ
- [External MariaDB](/data-sources/data-source-external-mariadb): ใช้ฐานข้อมูล MariaDB ภายนอกเป็นแหล่งข้อมูลครับ/ค่ะ
- [External PostgreSQL](/data-sources/data-source-external-postgres): ใช้ฐานข้อมูล PostgreSQL ภายนอกเป็นแหล่งข้อมูลครับ/ค่ะ
- [External MSSQL](/data-sources/data-source-external-mssql): ใช้ฐานข้อมูล MSSQL (SQL Server) ภายนอกเป็นแหล่งข้อมูลครับ/ค่ะ
- [External Oracle](/data-sources/data-source-external-oracle): ใช้ฐานข้อมูล Oracle ภายนอกเป็นแหล่งข้อมูลครับ/ค่ะ

![20240512083651](https://static-docs.nocobase.com/20240512083651.png)

## มีเครื่องมือ Data Modeling ที่หลากหลาย

**หน้าจอจัดการคอลเลกชันที่ใช้งานง่าย**: ใช้สำหรับสร้างโมเดล (คอลเลกชัน) ต่างๆ หรือเชื่อมต่อกับโมเดล (คอลเลกชัน) ที่มีอยู่แล้วครับ/ค่ะ

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**หน้าจอแสดงผลแบบ ER Diagram**: ใช้สำหรับดึง Entity (เอนทิตี) และความสัมพันธ์ระหว่าง Entity เหล่านั้นจากความต้องการของผู้ใช้และธุรกิจ โดยมีวิธีที่เข้าใจง่ายและเป็นธรรมชาติในการอธิบาย Data Model ซึ่ง ER Diagram จะช่วยให้เข้าใจ Entity ข้อมูลหลักและความสัมพันธ์ระหว่าง Entity เหล่านั้นในระบบได้ชัดเจนยิ่งขึ้นครับ/ค่ะ

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## รองรับการสร้างคอลเลกชันประเภทต่างๆ

| คอลเลกชัน | คำอธิบาย |
| - | - |
| [คอลเลกชันทั่วไป](/data-sources/data-source-main/general-collection) | มีฟิลด์ระบบที่ใช้บ่อยในตัว |
| [คอลเลกชันปฏิทิน](/data-sources/calendar/calendar-collection) | ใช้สำหรับสร้างตารางเหตุการณ์ที่เกี่ยวข้องกับปฏิทิน |
| คอลเลกชันความคิดเห็น | ใช้สำหรับจัดเก็บความคิดเห็นหรือข้อเสนอแนะเกี่ยวกับข้อมูล |
| [คอลเลกชันโครงสร้างแบบต้นไม้](/data-sources/collection-tree) | คอลเลกชันโครงสร้างแบบต้นไม้ ซึ่งปัจจุบันรองรับเฉพาะการออกแบบแบบ Adjacency List เท่านั้น |
| [คอลเลกชันไฟล์](/data-sources/file-manager/file-collection) | ใช้สำหรับการจัดการจัดเก็บไฟล์ |
| [คอลเลกชัน SQL](/data-sources/collection-sql) | ไม่ใช่ตารางฐานข้อมูลจริง แต่เป็นการแสดงผลการสืบค้น SQL ในรูปแบบที่มีโครงสร้างอย่างรวดเร็ว |
| [เชื่อมต่อ Database View](/data-sources/collection-view) | เชื่อมต่อกับ Database View ที่มีอยู่แล้ว |
| คอลเลกชัน Expression | ใช้สำหรับสถานการณ์ Expression แบบไดนามิกในเวิร์กโฟลว์ |
| [เชื่อมต่อข้อมูลภายนอก (FDW)](/data-sources/collection-fdw) | การเชื่อมต่อกับตารางข้อมูลระยะไกลที่ใช้เทคโนโลยี FDW ของฐานข้อมูล |

![20240512102212](https://static-docs.nocobase.com/20240512102212.png)

สำหรับข้อมูลเพิ่มเติม โปรดดูที่หัวข้อ 「[คอลเลกชัน / ภาพรวม](/data-sources/data-modeling/collection)」 ครับ/ค่ะ
