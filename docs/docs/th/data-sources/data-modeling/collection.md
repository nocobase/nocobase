# ภาพรวมคอลเลกชัน

NocoBase มี DSL (Domain-Specific Language) เฉพาะตัวสำหรับใช้อธิบายโครงสร้างของข้อมูล ซึ่งเราเรียกว่า "คอลเลกชัน" ครับ/ค่ะ คอลเลกชันนี้จะช่วยรวมโครงสร้างข้อมูลจากแหล่งต่าง ๆ ให้เป็นหนึ่งเดียวกัน เพื่อเป็นรากฐานที่มั่นคงสำหรับการจัดการ การวิเคราะห์ และการนำข้อมูลไปใช้งานต่อไปครับ/ค่ะ

![20240512161522](https://static-docs.nocobase.com/20240512161522.png)

เพื่อให้การใช้งานโมเดลข้อมูลต่าง ๆ เป็นไปอย่างสะดวก NocoBase จึงรองรับการสร้างคอลเลกชันหลากหลายประเภทดังนี้ครับ/ค่ะ

- [คอลเลกชันทั่วไป](/data-sources/data-source-main/general-collection): มีฟิลด์ระบบที่ใช้งานทั่วไปในตัวครับ/ค่ะ
- [คอลเลกชันแบบสืบทอด](/data-sources/data-source-main/inheritance-collection): สามารถสร้างคอลเลกชันหลัก (parent collection) แล้วสร้างคอลเลกชันย่อย (child collection) ที่สืบทอดมาจากคอลเลกชันหลักนั้นได้ครับ/ค่ะ โดยคอลเลกชันย่อยจะสืบทอดโครงสร้างจากคอลเลกชันหลัก และยังสามารถกำหนดคอลัมน์ของตัวเองเพิ่มเติมได้ด้วยครับ/ค่ะ
- [คอลเลกชันแบบต้นไม้](/data-sources/collection-tree): เป็นคอลเลกชันโครงสร้างแบบต้นไม้ ซึ่งปัจจุบันรองรับเฉพาะการออกแบบแบบ Adjacency List ครับ/ค่ะ
- [คอลเลกชันปฏิทิน](/data-sources/calendar/calendar-collection): ใช้สำหรับสร้างคอลเลกชันเหตุการณ์ที่เกี่ยวข้องกับปฏิทินครับ/ค่ะ
- [คอลเลกชันไฟล์](/data-sources/file-manager/file-collection): ใช้สำหรับการจัดการจัดเก็บไฟล์ครับ/ค่ะ
- : ใช้สำหรับสถานการณ์การแสดงออกทางไดนามิก (dynamic expression) ในเวิร์กโฟลว์ครับ/ค่ะ
- [คอลเลกชัน SQL](/data-sources/collection-sql): ไม่ใช่ตารางฐานข้อมูลจริง แต่เป็นการนำผลลัพธ์จากการคิวรี SQL มาแสดงผลในรูปแบบที่มีโครงสร้างอย่างรวดเร็วครับ/ค่ะ
- [คอลเลกชันวิว](/data-sources/collection-view): ใช้เชื่อมต่อกับ Database View ที่มีอยู่แล้วครับ/ค่ะ
- [คอลเลกชันภายนอก](/data-sources/collection-fdw): ช่วยให้ระบบฐานข้อมูลสามารถเข้าถึงและคิวรีข้อมูลในแหล่งข้อมูลภายนอกได้โดยตรง โดยใช้เทคโนโลยี FDW ครับ/ค่ะ
