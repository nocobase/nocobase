# การซิงค์ข้อมูลผู้ใช้

## บทนำ

คุณสามารถลงทะเบียนและจัดการแหล่งข้อมูลสำหรับการซิงค์ข้อมูลผู้ใช้ได้ โดยค่าเริ่มต้น จะมี HTTP API ให้ใช้งาน แต่ก็สามารถขยายเพื่อรองรับแหล่งข้อมูลอื่น ๆ ผ่านปลั๊กอินได้ครับ/ค่ะ โดยค่าเริ่มต้น รองรับการซิงค์ข้อมูลไปยังคอลเลกชัน**ผู้ใช้**และ**แผนก** และยังสามารถขยายการซิงค์ไปยังทรัพยากรเป้าหมายอื่น ๆ ด้วยปลั๊กอินได้อีกด้วยครับ/ค่ะ

## การจัดการแหล่งข้อมูลและการซิงค์ข้อมูล

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
หากยังไม่ได้ติดตั้งปลั๊กอินที่รองรับการซิงค์ข้อมูลผู้ใช้ คุณสามารถซิงค์ข้อมูลผู้ใช้ได้โดยใช้ HTTP API ครับ/ค่ะ โปรดดูรายละเอียดเพิ่มเติมที่ [แหล่งข้อมูล - HTTP API](./sources/api.md) ครับ/ค่ะ
:::

## การเพิ่มแหล่งข้อมูล

หลังจากที่คุณติดตั้งปลั๊กอินที่รองรับการซิงค์ข้อมูลผู้ใช้แล้ว คุณจะสามารถเพิ่มแหล่งข้อมูลที่เกี่ยวข้องได้เลยครับ/ค่ะ เฉพาะแหล่งข้อมูลที่เปิดใช้งานเท่านั้นที่จะแสดงปุ่ม "ซิงค์" และ "งาน" ครับ/ค่ะ

> ตัวอย่าง: WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## การซิงค์ข้อมูล

คลิกปุ่ม "ซิงค์" เพื่อเริ่มการซิงค์ข้อมูลครับ/ค่ะ

![](https://static-docs.nocobase.com/202412041055022.png)

คลิกปุ่ม "งาน" เพื่อดูสถานะการซิงค์ได้ครับ/ค่ะ เมื่อซิงค์ข้อมูลสำเร็จแล้ว คุณสามารถไปที่รายการผู้ใช้และแผนกเพื่อดูข้อมูลได้เลยครับ/ค่ะ

![](https://static-docs.nocobase.com/202412041202337.png)

สำหรับงานซิงค์ที่ล้มเหลว คุณสามารถคลิก "ลองใหม่" เพื่อดำเนินการอีกครั้งได้ครับ/ค่ะ

![](https://static-docs.nocobase.com/202412041058337.png)

หากการซิงค์ล้มเหลว คุณสามารถตรวจสอบหาสาเหตุได้จากบันทึกระบบ (system logs) ครับ/ค่ะ นอกจากนี้ ยังมีบันทึกการซิงค์ข้อมูลดิบ (raw synchronization records) เก็บไว้ในไดเรกทอรี `user-data-sync` ซึ่งอยู่ในโฟลเดอร์บันทึกของแอปพลิเคชันครับ/ค่ะ

![](https://static-docs.nocobase.com/202412041205655.png)
