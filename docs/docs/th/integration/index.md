# การเชื่อมต่อระบบ

## ภาพรวม

NocoBase มีความสามารถในการเชื่อมต่อระบบที่ครอบคลุม ช่วยให้คุณสามารถเชื่อมต่อกับระบบภายนอก, บริการจากภายนอก (third-party services) และแหล่งข้อมูลต่างๆ ได้อย่างราบรื่นครับ/ค่ะ ด้วยวิธีการเชื่อมต่อที่ยืดหยุ่น คุณสามารถขยายขีดความสามารถของ NocoBase เพื่อตอบสนองความต้องการทางธุรกิจที่หลากหลายได้เลยครับ/ค่ะ

## วิธีการเชื่อมต่อระบบ

### การเชื่อมต่อผ่าน API

NocoBase มีความสามารถด้าน API ที่ทรงพลังสำหรับการเชื่อมต่อกับแอปพลิเคชันและบริการภายนอกครับ/ค่ะ:

![20250301003106](https://static-docs.nocobase.com/20250301003106.png)

- **[API Keys](/integration/api-keys/)**: ใช้ API Keys เพื่อยืนยันตัวตนอย่างปลอดภัย และเข้าถึงทรัพยากรของ NocoBase ด้วยการเขียนโปรแกรมครับ/ค่ะ
- **[เอกสาร API](/integration/api-doc/)**: มีเอกสาร API ในตัว (built-in) สำหรับสำรวจและทดสอบ Endpoint ต่างๆ ครับ/ค่ะ

### Single Sign-On (SSO)

เชื่อมต่อกับระบบยืนยันตัวตนขององค์กร เพื่อการยืนยันตัวตนแบบรวมศูนย์ครับ/ค่ะ:

![auth_sso-2025-11-24-08-26-46](https://static-docs.nocobase.com/auth_sso-2025-11-24-08-26-46.png)

- **[การเชื่อมต่อ SSO](/integration/sso/)**: รองรับการยืนยันตัวตนผ่าน SAML, OIDC, CAS, LDAP และแพลตฟอร์มภายนอก (third-party platforms) ครับ/ค่ะ
- การจัดการผู้ใช้และการควบคุมการเข้าถึงแบบรวมศูนย์
- ประสบการณ์การยืนยันตัวตนที่ราบรื่นในทุกระบบ

### การเชื่อมต่อเวิร์กโฟลว์

เชื่อมต่อเวิร์กโฟลว์ของ NocoBase เข้ากับระบบภายนอกครับ/ค่ะ:

![auth_sso-2025-11-24-12-02-01](https://static-docs.nocobase.com/auth_sso-2025-11-24-12-02-01.png)

- **[Webhook ของเวิร์กโฟลว์](/integration/workflow-webhook/)**: รับเหตุการณ์จากระบบภายนอกเพื่อเรียกใช้งานเวิร์กโฟลว์
- **[คำขอ HTTP ของเวิร์กโฟลว์](/integration/workflow-http-request/)**: ส่งคำขอ HTTP ไปยัง API ภายนอกจากเวิร์กโฟลว์
- สร้างระบบอัตโนมัติสำหรับกระบวนการทางธุรกิจข้ามแพลตฟอร์ม

### แหล่งข้อมูลภายนอก

เชื่อมต่อกับฐานข้อมูลและระบบข้อมูลภายนอกครับ/ค่ะ:

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

- **[ฐานข้อมูลภายนอก](/data-sources/)**: เชื่อมต่อโดยตรงกับฐานข้อมูล MySQL, PostgreSQL, MariaDB, MSSQL, Oracle และ KingbaseES
- สามารถระบุโครงสร้างตารางของฐานข้อมูลภายนอก และดำเนินการเพิ่ม, แก้ไข, ลบ, ค้นหา (CRUD operations) ข้อมูลภายนอกได้โดยตรงภายใน NocoBase ครับ/ค่ะ
- อินเทอร์เฟซการจัดการข้อมูลแบบรวมศูนย์

### การฝังเนื้อหา

ฝังเนื้อหาภายนอกลงใน NocoBase ครับ/ค่ะ:

![20240408220322](https://static-docs.nocobase.com/20240408220322.png)

- **[บล็อก Iframe](/integration/block-iframe/)**: ฝังหน้าเว็บและแอปพลิเคชันภายนอก
- **บล็อก JS**: รันโค้ด JavaScript ที่กำหนดเองเพื่อการเชื่อมต่อระบบขั้นสูง

## สถานการณ์การเชื่อมต่อระบบที่พบบ่อย

### การเชื่อมต่อระบบองค์กร

- เชื่อมต่อ NocoBase กับระบบ ERP, CRM หรือระบบองค์กรอื่นๆ
- ซิงค์ข้อมูลแบบสองทาง
- สร้างระบบอัตโนมัติสำหรับเวิร์กโฟลว์ข้ามระบบ

### การเชื่อมต่อบริการภายนอก (Third-Party Services)

- สอบถามสถานะการชำระเงินจาก Payment Gateway, เชื่อมต่อบริการส่งข้อความ หรือแพลตฟอร์มคลาวด์
- ใช้ประโยชน์จาก API ภายนอกเพื่อขยายฟังก์ชันการทำงาน
- สร้างการเชื่อมต่อระบบแบบกำหนดเองโดยใช้ Webhook และคำขอ HTTP

### การเชื่อมต่อข้อมูล

- เชื่อมต่อกับแหล่งข้อมูลหลายแหล่ง
- รวบรวมข้อมูลจากระบบต่างๆ
- สร้างแดชบอร์ดและรายงานแบบรวมศูนย์

## ข้อควรพิจารณาด้านความปลอดภัย

เมื่อเชื่อมต่อ NocoBase กับระบบภายนอก โปรดพิจารณาแนวทางปฏิบัติที่ดีที่สุดด้านความปลอดภัยดังต่อไปนี้ครับ/ค่ะ:

1.  **ใช้ HTTPS**: ควรใช้การเชื่อมต่อที่เข้ารหัสสำหรับการส่งข้อมูลเสมอ
2.  **ปกป้อง API Keys**: จัดเก็บ API Keys อย่างปลอดภัยและหมุนเวียน (rotate) เป็นประจำ
3.  **หลักการสิทธิ์ขั้นต่ำ (Principle of Least Privilege)**: ให้สิทธิ์ที่จำเป็นสำหรับการเชื่อมต่อระบบเท่านั้น
4.  **บันทึกการตรวจสอบ (Audit Logging)**: ตรวจสอบและบันทึกกิจกรรมการเชื่อมต่อระบบ
5.  **การตรวจสอบความถูกต้องของข้อมูล (Data Validation)**: ตรวจสอบความถูกต้องของข้อมูลทั้งหมดที่มาจากแหล่งภายนอก
