# เชื่อมต่อตารางข้อมูลภายนอก (FDW)

## บทนำ

ฟีเจอร์การเชื่อมต่อกับตารางข้อมูลระยะไกล (Remote Data Tables) ที่ทำงานบนพื้นฐานของ Foreign Data Wrapper (FDW) ของฐานข้อมูล ปัจจุบันรองรับฐานข้อมูล MySQL และ PostgreSQL ครับ

:::info{title="การเชื่อมต่อแหล่งข้อมูล vs การเชื่อมต่อตารางข้อมูลภายนอก"}
- **การเชื่อมต่อแหล่งข้อมูล (Connecting Data Sources)** หมายถึงการสร้างการเชื่อมต่อกับฐานข้อมูลหรือบริการ API เฉพาะเจาะจง ซึ่งคุณสามารถใช้งานคุณสมบัติของฐานข้อมูลหรือบริการที่ API มอบให้ได้อย่างเต็มรูปแบบครับ
- **การเชื่อมต่อตารางข้อมูลภายนอก (Connecting External Data Tables)** หมายถึงการดึงข้อมูลจากภายนอกและนำมาแมป (Map) เพื่อใช้งานภายในเครื่อง ในทางฐานข้อมูลจะเรียกว่า FDW (Foreign Data Wrapper) ซึ่งเป็นเทคโนโลยีฐานข้อมูลที่เน้นการใช้งานตารางระยะไกลเสมือนเป็นตารางในเครื่อง และสามารถเชื่อมต่อได้ทีละตารางเท่านั้น เนื่องจากการเข้าถึงข้อมูลเป็นการเข้าถึงจากระยะไกล จึงอาจมีข้อจำกัดและเงื่อนไขต่างๆ ในการใช้งานครับ

ทั้งสองรูปแบบสามารถใช้งานร่วมกันได้ โดยแบบแรกใช้เพื่อสร้างการเชื่อมต่อกับแหล่งข้อมูล และแบบหลังใช้สำหรับการเข้าถึงข้อมูลข้ามแหล่งข้อมูล ตัวอย่างเช่น เมื่อเชื่อมต่อกับแหล่งข้อมูล PostgreSQL หนึ่งๆ แล้ว ตารางบางตารางในแหล่งข้อมูลนั้นอาจเป็นตารางภายนอกที่สร้างขึ้นโดยใช้ FDW ครับ
:::

### MySQL

MySQL ใช้งานผ่านเอนจิน `federated` ซึ่งจำเป็นต้องเปิดใช้งานก่อน รองรับการเชื่อมต่อกับ MySQL ระยะไกลและฐานข้อมูลที่เข้ากันได้กับโปรโตคอล เช่น MariaDB สำหรับรายละเอียดเพิ่มเติม สามารถอ้างอิงได้จากเอกสาร [Federated Storage Engine](https://dev.mysql.com/doc/refman/8.0/en/federated-storage-engine.html) ครับ

### PostgreSQL

ใน PostgreSQL สามารถใช้ส่วนขยาย (Extension) `fdw` ประเภทต่างๆ เพื่อรองรับข้อมูลระยะไกลที่แตกต่างกัน โดยส่วนขยายที่รองรับในปัจจุบัน ได้แก่:

- [postgres_fdw](https://www.postgresql.org/docs/current/postgres-fdw.html): เชื่อมต่อกับฐานข้อมูล PostgreSQL ระยะไกลใน PostgreSQL
- [mysql_fdw](https://github.com/EnterpriseDB/mysql_fdw): เชื่อมต่อกับฐานข้อมูล MySQL ระยะไกลใน PostgreSQL
- สำหรับส่วนขยาย fdw ประเภทอื่นๆ สามารถอ้างอิงได้จาก [PostgreSQL Foreign Data Wrappers](https://wiki.postgresql.org/wiki/Foreign_data_wrappers) ทั้งนี้การนำมาใช้ใน NocoBase จำเป็นต้องมีการพัฒนาอินเทอร์เฟซการปรับแต่ง (Adaptation Interface) ที่เกี่ยวข้องในโค้ดด้วยครับ

## เงื่อนไขก่อนการใช้งาน

- หากฐานข้อมูลหลักของ NocoBase คือ MySQL จำเป็นต้องเปิดใช้งาน `federated` ก่อน โดยอ้างอิงได้จาก [วิธีเปิดใช้งานเอนจิน federated ใน MySQL](./enable-federated)

จากนั้นทำการติดตั้งและเปิดใช้งานปลั๊กอินผ่านตัวจัดการปลั๊กอิน (Plugin Manager)

![ติดตั้งและเปิดใช้งานปลั๊กอิน](https://static-docs.nocobase.com/f84276c5712851fb3ff33af3f1ff0f59.png)
