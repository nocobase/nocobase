# ctx.setValue()

ใช้สำหรับกำหนดค่าของฟิลด์ปัจจุบันในสถานการณ์ที่เป็นฟิลด์ที่แก้ไขได้ เช่น JSField และ JSItem เมื่อใช้งานร่วมกับ `ctx.getValue()` จะช่วยให้สามารถทำ Two-way binding กับฟอร์มได้ครับ

## สถานการณ์ที่ใช้งาน

| สถานการณ์ | คำอธิบาย |
|------|------|
| **JSField** | เขียนค่าที่ผู้ใช้เลือกหรือค่าที่คำนวณได้ลงในฟิลด์ที่กำหนดเอง (Custom Field) ที่แก้ไขได้ |
| **JSItem** | อัปเดตค่าของเซลล์ปัจจุบันในรายการที่แก้ไขได้ของตารางหรือตารางย่อย |
| **JSColumn** | อัปเดตค่าฟิลด์ของแถวที่เกี่ยวข้องตามตรรกะในขณะที่เรนเดอร์คอลัมน์ของตาราง |

> **หมายเหตุ**: `ctx.setValue(v)` จะใช้งานได้เฉพาะในบริบท (Context) ของ RunJS ที่มีการเชื่อมโยงกับฟอร์มเท่านั้น ในสถานการณ์ที่ไม่มีการเชื่อมโยงฟิลด์ เช่น เวิร์กโฟลว์ (Workflow), กฎการเชื่อมโยง (Linkage rules) หรือ JSBlock จะไม่มีเมธอดนี้อยู่ แนะนำให้ใช้ Optional Chaining ก่อนใช้งาน: `ctx.setValue?.(value)` ครับ

## การกำหนดประเภท (Type Definition)

```ts
setValue<T = any>(value: T): void;
```

- **พารามิเตอร์**: `value` คือค่าของฟิลด์ที่ต้องการเขียน โดยประเภทของข้อมูลจะขึ้นอยู่กับประเภทของรายการฟอร์ม (Form item type) ของฟิลด์นั้นๆ ครับ

## คำอธิบายการทำงาน

- `ctx.setValue(v)` จะอัปเดตค่าของฟิลด์ปัจจุบันใน Ant Design Form และทริกเกอร์การเชื่อมโยงฟอร์ม (Form linkage) รวมถึงตรรกะการตรวจสอบความถูกต้อง (Validation) ที่เกี่ยวข้อง
- หากฟอร์มยังเรนเดอร์ไม่เสร็จสิ้นหรือฟิลด์ยังไม่ได้ลงทะเบียน การเรียกใช้งานอาจไม่เกิดผล แนะนำให้ใช้ร่วมกับ `ctx.getValue()` เพื่อยืนยันผลลัพธ์การเขียนข้อมูลครับ

## ตัวอย่าง

### การใช้งานร่วมกับ getValue เพื่อทำ Two-way binding

```tsx
const { Input } = ctx.libs.antd;

const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```

### การกำหนดค่าเริ่มต้นตามเงื่อนไข

```ts
const status = ctx.getValue();
if (status == null || status === '') {
  ctx.setValue?.('draft');
}
```

### การเขียนค่ากลับไปยังฟิลด์ปัจจุบันเมื่อมีการเชื่อมโยงกับฟิลด์อื่น

```ts
// อัปเดตฟิลด์ปัจจุบันให้สอดคล้องเมื่อฟิลด์อื่นมีการเปลี่ยนแปลง
const otherValue = ctx.form?.getFieldValue('type');
if (otherValue === 'custom') {
  ctx.setValue?.({ label: 'กำหนดเอง', value: 'custom' });
}
```

## ข้อควรระวัง

- ในฟิลด์ที่แก้ไขไม่ได้ (เช่น JSField ในโหมดรายละเอียด หรือ JSBlock) `ctx.setValue` อาจเป็น `undefined` แนะนำให้ใช้ `ctx.setValue?.(value)` เพื่อหลีกเลี่ยงข้อผิดพลาดครับ
- เมื่อกำหนดค่าให้กับฟิลด์ความสัมพันธ์ (เช่น M2O, O2M เป็นต้น) จำเป็นต้องส่งโครงสร้างข้อมูลที่ตรงกับประเภทของฟิลด์ (เช่น `{ id, [titleField]: label }`) โดยขึ้นอยู่กับการกำหนดค่าของฟิลด์นั้นๆ ครับ
