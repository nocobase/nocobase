# ctx.router

อินสแตนซ์ของเร้าเตอร์ (Router instance) ที่อิงตาม React Router ใช้สำหรับการนำทาง (navigation) ผ่านโค้ดใน RunJS โดยปกติจะใช้ร่วมกับ `ctx.route` และ `ctx.location` ครับ

## สถาณการณ์ที่ใช้งาน

| สถานการณ์ | คำอธิบาย |
|------|------|
| **JSBlock / JSField** | เปลี่ยนหน้าไปยังหน้ารายละเอียด หน้ารายการ หรือลิงก์ภายนอกหลังจากคลิกปุ่ม |
| **กฎการเชื่อมโยง / Event Flow** | ดำเนินการ `navigate` ไปยังหน้ารายการหรือรายละเอียดหลังจากส่งข้อมูลสำเร็จ หรือส่ง `state` ไปยังหน้าปลายทาง |
| **JSAction / การจัดการเหตุการณ์** | ดำเนินการเปลี่ยนเส้นทางภายในโลจิก เช่น การส่งฟอร์ม หรือการคลิกลิงก์ |
| **การนำทางในมุมมอง** | อัปเดต URL ผ่าน `navigate` เมื่อมีการสลับสแต็กมุมมองภายใน |

> หมายเหตุ: `ctx.router` จะใช้งานได้เฉพาะในสภาพแวดล้อม RunJS ที่มีบริบทของเร้าติ้ง (เช่น JSBlock ภายในหน้าเพจ, หน้า Flow, Event Flow เป็นต้น) แต่อาจเป็นค่าว่างในบริบทที่เป็นหลังบ้านล้วนๆ หรือไม่มีเร้าติ้ง (เช่น เวิร์กโฟลว์) ครับ

## การกำหนดประเภท (Type Definition)

```typescript
router: Router
```

`Router` มาจาก `@remix-run/router` โดยใน RunJS จะใช้ `ctx.router.navigate()` เพื่อดำเนินการนำทาง เช่น การเปลี่ยนหน้า, การย้อนกลับ หรือการรีเฟรชครับ

## เมธอด (Methods)

### ctx.router.navigate()

นำทางไปยังเส้นทางปลายทาง หรือดำเนินการย้อนกลับ/รีเฟรช

**Signature:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**พารามิเตอร์:**

- `to`: เส้นทางปลายทาง (string), ตำแหน่งประวัติสัมพัทธ์ (number เช่น `-1` หมายถึงย้อนกลับ) หรือ `null` (เพื่อรีเฟรชหน้าปัจจุบัน)
- `options`: การตั้งค่าเพิ่มเติม
  - `replace?: boolean`: กำหนดว่าจะแทนที่รายการประวัติปัจจุบันหรือไม่ (ค่าเริ่มต้นคือ `false` ซึ่งจะเป็นการ push รายการใหม่)
  - `state?: any`: ข้อมูลที่จะส่งไปยังเร้าท์ปลายทาง ข้อมูลนี้จะไม่ปรากฏใน URL และสามารถเข้าถึงได้ผ่าน `ctx.location.state` ในหน้าปลายทาง เหมาะสำหรับข้อมูลที่ละเอียดอ่อน ข้อมูลชั่วคราว หรือข้อมูลที่ไม่ควรแสดงใน URL

## ตัวอย่าง

### การนำทางพื้นฐาน

```ts
// ไปยังรายการผู้ใช้ (push ประวัติใหม่ สามารถย้อนกลับได้)
ctx.router.navigate('/admin/users');

// ไปยังหน้ารายละเอียด
ctx.router.navigate(`/admin/users/${recordId}`);
```

### การแทนที่ประวัติ (ไม่มีการเพิ่มรายการใหม่)

```ts
// รีไดเรกต์ไปยังหน้าหลักหลังล็อกอิน เพื่อไม่ให้ผู้ใช้ย้อนกลับมาหน้าล็อกอินได้อีก
ctx.router.navigate('/admin', { replace: true });

// แทนที่หน้าปัจจุบันด้วยหน้ารายละเอียดหลังจากส่งฟอร์มสำเร็จ
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### การส่ง state

```ts
// ส่งข้อมูลไปพร้อมกับการนำทาง หน้าปลายทางจะรับข้อมูลผ่าน ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### การย้อนกลับและรีเฟรช

```ts
// ย้อนกลับ 1 หน้า
ctx.router.navigate(-1);

// ย้อนกลับ 2 หน้า
ctx.router.navigate(-2);

// รีเฟรชหน้าปัจจุบัน
ctx.router.navigate(null);
```

## ความสัมพันธ์กับ ctx.route และ ctx.location

| วัตถุประสงค์ | วิธีการที่แนะนำ |
|------|----------|
| **การนำทาง/เปลี่ยนหน้า** | `ctx.router.navigate(path)` |
| **อ่านเส้นทางปัจจุบัน** | `ctx.route.pathname` หรือ `ctx.location.pathname` |
| **อ่าน state ที่ส่งมาตอนนำทาง** | `ctx.location.state` |
| **อ่านพารามิเตอร์ของเร้าท์** | `ctx.route.params` |

`ctx.router` มีหน้าที่รับผิดชอบ "การดำเนินการนำทาง" ในขณะที่ `ctx.route` และ `ctx.location` มีหน้าที่รับผิดชอบ "สถานะของเร้าท์ปัจจุบัน" ครับ

## ข้อควรระวัง

- `navigate(path)` โดยปกติจะ push รายการประวัติใหม่ ทำให้ผู้ใช้สามารถย้อนกลับผ่านปุ่มของเบราว์เซอร์ได้
- `replace: true` จะแทนที่รายการประวัติปัจจุบันโดยไม่เพิ่มรายการใหม่ เหมาะสำหรับสถานการณ์เช่น การรีไดเรกต์หลังล็อกอิน หรือการเปลี่ยนหน้าหลังส่งข้อมูลสำเร็จ
- **เกี่ยวกับพารามิเตอร์ `state`**:
  - ข้อมูลที่ส่งผ่าน `state` จะไม่ปรากฏใน URL จึงเหมาะสำหรับข้อมูลที่ละเอียดอ่อนหรือข้อมูลชั่วคราว
  - สามารถเข้าถึงได้ผ่าน `ctx.location.state` ในหน้าปลายทาง
  - `state` จะถูกบันทึกไว้ในประวัติของเบราว์เซอร์ และยังคงเข้าถึงได้เมื่อมีการกดไปข้างหน้าหรือย้อนกลับ
  - `state` จะหายไปหากมีการรีเฟรชหน้าเว็บ (Hard refresh)
