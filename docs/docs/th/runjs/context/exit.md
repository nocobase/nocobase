# ctx.exit()

ยุติการทำงานของเวิร์กโฟลว์เหตุการณ์ (event flow) ปัจจุบัน โดยขั้นตอนหลังจากนี้จะไม่ทำงาน มักใช้เมื่อเงื่อนไขทางธุรกิจไม่เป็นไปตามที่กำหนด ผู้ใช้ยกเลิกการดำเนินการ หรือเกิดข้อผิดพลาดที่ไม่สามารถกู้คืนได้ครับ

## สถานการณ์ที่ใช้งาน

`ctx.exit()` โดยทั่วไปจะใช้ในบริบทที่สามารถรัน JS ได้ดังนี้:

| สถานการณ์ | คำอธิบาย |
|------|------|
| **เวิร์กโฟลว์เหตุการณ์** | ในเวิร์กโฟลว์ที่ถูกทริกเกอร์โดยการส่งฟอร์ม การคลิกปุ่ม ฯลฯ เพื่อยุติขั้นตอนถัดไปเมื่อเงื่อนไขไม่เป็นไปตามที่กำหนด |
| **กฎการเชื่อมโยง** | ในการเชื่อมโยงฟิลด์ (field linkage), การเชื่อมโยงตัวกรอง (filter linkage) ฯลฯ เพื่อยุติเวิร์กโฟลว์ปัจจุบันเมื่อการตรวจสอบความถูกต้องล้มเหลวหรือต้องการข้ามการทำงาน |
| **เหตุการณ์การดำเนินการ** | ในการดำเนินการที่กำหนดเอง (เช่น การยืนยันการลบ, การตรวจสอบก่อนบันทึก) เพื่อออกจากการทำงานเมื่อผู้ใช้ยกเลิกหรือการตรวจสอบไม่ผ่าน |

> ข้อแตกต่างจาก `ctx.exitAll()`: `ctx.exit()` จะยุติเฉพาะเวิร์กโฟลว์เหตุการณ์ปัจจุบันเท่านั้น เวิร์กโฟลว์อื่น ๆ ภายใต้เหตุการณ์เดียวกันจะไม่ได้รับผลกระทบ ส่วน `ctx.exitAll()` จะยุติทั้งเวิร์กโฟลว์ปัจจุบันและเวิร์กโฟลว์ถัดไปภายใต้เหตุการณ์เดียวกันที่ยังไม่ได้ทำงานครับ

## การกำหนดประเภท (Type Definition)

```ts
exit(): never;
```

การเรียกใช้ `ctx.exit()` จะโยน (throw) `FlowExitException` ภายใน ซึ่งจะถูกดักจับโดยเอนจินของเวิร์กโฟลว์เพื่อหยุดการทำงานของเวิร์กโฟลว์เหตุการณ์ปัจจุบัน เมื่อเรียกใช้แล้ว โค้ด JS ที่เหลือในบล็อกปัจจุบันจะไม่ถูกดำเนินการครับ

## การเปรียบเทียบกับ ctx.exitAll()

| วิธีการ | ขอบเขตของผลกระทบ |
|------|----------|
| `ctx.exit()` | ยุติเฉพาะเวิร์กโฟลว์เหตุการณ์ปัจจุบัน เวิร์กโฟลว์ถัดไปจะไม่ได้รับผลกระทบ |
| `ctx.exitAll()` | ยุติเวิร์กโฟลว์เหตุการณ์ปัจจุบัน และยกเลิกเวิร์กโฟลว์ถัดไปภายใต้เหตุการณ์เดียวกันที่ตั้งค่าให้ **ทำงานตามลำดับ (execute sequentially)** |

## ตัวอย่าง

### ออกจากการทำงานเมื่อผู้ใช้ยกเลิก

```ts
// ในหน้าต่างยืนยัน หากผู้ใช้คลิกยกเลิก ให้ยุติเวิร์กโฟลว์เหตุการณ์
if (!confirmed) {
  ctx.message.info('ยกเลิกการดำเนินการแล้ว');
  ctx.exit();
}
```

### ออกจากการทำงานเมื่อการตรวจสอบพารามิเตอร์ล้มเหลว

```ts
// แจ้งเตือนและยุติการทำงานเมื่อการตรวจสอบไม่ผ่าน
if (!params.value || params.value.length < 3) {
  ctx.message.error('พารามิเตอร์ไม่ถูกต้อง ความยาวต้องไม่น้อยกว่า 3');
  ctx.exit();
}
```

### ออกจากการทำงานเมื่อเงื่อนไขทางธุรกิจไม่เป็นไปตามที่กำหนด

```ts
// ยุติการทำงานหากเงื่อนไขไม่เป็นไปตามที่กำหนด ขั้นตอนถัดไปจะไม่ทำงาน
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'สามารถส่งได้เฉพาะสถานะฉบับร่างเท่านั้น' });
  ctx.exit();
}
```

### การเลือกใช้ระหว่าง ctx.exit() และ ctx.exitAll()

```ts
// ต้องการให้ออกเฉพาะเวิร์กโฟลว์ปัจจุบันเท่านั้น → ใช้ ctx.exit()
if (!params.valid) {
  ctx.message.error('พารามิเตอร์ไม่ถูกต้อง');
  ctx.exit();  // เวิร์กโฟลว์อื่นจะไม่ได้รับผลกระทบ
}

// ต้องการยุติเวิร์กโฟลว์ถัดไปทั้งหมดภายใต้เหตุการณ์ปัจจุบัน → ใช้ ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'สิทธิ์ไม่เพียงพอ' });
  ctx.exitAll();  // ทั้งเวิร์กโฟลว์หลักและเวิร์กโฟลว์ถัดไปภายใต้เหตุการณ์เดียวกันจะถูกยุติ
}
```

### ออกจากการทำงานตามการเลือกของผู้ใช้หลังจากยืนยันผ่าน Modal

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'ยืนยันการลบ',
  content: 'ลบแล้วไม่สามารถกู้คืนได้ ต้องการดำเนินการต่อหรือไม่?',
});
if (!ok) {
  ctx.message.info('ยกเลิกแล้ว');
  ctx.exit();
}
```

## ข้อควรระวัง

- หลังจากเรียกใช้ `ctx.exit()` โค้ดส่วนที่เหลือใน JS ปัจจุบันจะไม่ทำงาน แนะนำให้แจ้งเหตุผลแก่ผู้ใช้ผ่าน `ctx.message`, `ctx.notification` หรือหน้าต่างป๊อปอัปก่อนเรียกใช้ครับ
- โดยปกติไม่จำเป็นต้องดักจับ (catch) `FlowExitException` ในโค้ดทางธุรกิจ ให้เอนจินของเวิร์กโฟลว์เป็นผู้จัดการเอง
- หากต้องการยุติเวิร์กโฟลว์เหตุการณ์ถัดไปทั้งหมดภายใต้เหตุการณ์ปัจจุบัน ให้ใช้ `ctx.exitAll()`
