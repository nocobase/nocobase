# ctx.message

API สำหรับ message ทั่วโลกของ Ant Design ใช้สำหรับแสดงข้อความแจ้งเตือนชั่วคราว (light prompts) ที่บริเวณกึ่งกลางด้านบนของหน้าจอ ข้อความจะปิดโดยอัตโนมัติหลังจากเวลาผ่านไปช่วงหนึ่ง หรือผู้ใช้สามารถปิดเองได้ครับ

## 适用场景 (สถานการณ์ที่ใช้งาน)

| สถานการณ์ | คำอธิบาย |
|------|------|
| **JSBlock / JSField / JSItem / JSColumn** | การตอบสนองการทำงาน (Operation feedback), การแจ้งเตือนการตรวจสอบ (validation prompts), การคัดลอกสำเร็จ และการแจ้งเตือนแบบเบาอื่น ๆ |
| **การดำเนินการในฟอร์ม / เเวิร์กโฟลว์** | การตอบสนองเมื่อส่งข้อมูลสำเร็จ, บันทึกล้มเหลว, การตรวจสอบข้อมูลไม่ผ่าน ฯลฯ |
| **เหตุการณ์การดำเนินการ (JSAction)** | การตอบสนองทันทีเมื่อคลิก หรือเมื่อการดำเนินการแบบกลุ่ม (batch operation) เสร็จสิ้น |

## 类型定义 (คำจำกัดความประเภท)

```ts
message: MessageInstance;
```

`MessageInstance` คืออินเทอร์เฟซ message ของ Ant Design ซึ่งมีเมธอดดังต่อไปนี้ครับ

## 常用方法 (เมธอดที่ใช้บ่อย)

| เมธอด | คำอธิบาย |
|------|------|
| `success(content, duration?)` | แสดงข้อความแจ้งเตือนเมื่อสำเร็จ |
| `error(content, duration?)` | แสดงข้อความแจ้งเตือนเมื่อเกิดข้อผิดพลาด |
| `warning(content, duration?)` | แสดงข้อความแจ้งเตือนคำเตือน |
| `info(content, duration?)` | แสดงข้อความแจ้งเตือนข้อมูล |
| `loading(content, duration?)` | แสดงข้อความแจ้งเตือนขณะกำลังโหลด (ต้องปิดด้วยตนเอง) |
| `open(config)` | เปิดข้อความโดยใช้การกำหนดค่าแบบกำหนดเอง |
| `destroy()` | ปิดข้อความทั้งหมดที่กำลังแสดงอยู่ |

**พารามิเตอร์:**

- `content` (`string` \| `ConfigOptions`): เนื้อหาของข้อความหรือออบเจ็กต์การกำหนดค่า
- `duration` (`number`, ไม่บังคับ): ระยะเวลาหน่วงก่อนปิดอัตโนมัติ (วินาที) ค่าเริ่มต้นคือ 3 วินาที หากตั้งเป็น 0 จะไม่ปิดอัตโนมัติ

**ConfigOptions** (เมื่อ `content` เป็นออบเจ็กต์):

```ts
interface ConfigOptions {
  content: React.ReactNode;  // เนื้อหาข้อความ
  duration?: number;        // ระยะเวลาหน่วงก่อนปิดอัตโนมัติ (วินาที)
  onClose?: () => void;    // คอลแบ็กเมื่อปิด
  icon?: React.ReactNode;  // ไอคอนแบบกำหนดเอง
}
```

## 示例 (ตัวอย่าง)

### 基础用法 (การใช้งานพื้นฐาน)

```ts
ctx.message.success('ดำเนินการสำเร็จ');
ctx.message.error('ดำเนินการล้มเหลว');
ctx.message.warning('กรุณาเลือกข้อมูลก่อน');
ctx.message.info('กำลังดำเนินการ...');
```

### 与 ctx.t 配合国际化 (ใช้งานร่วมกับ ctx.t สำหรับการรองรับหลายภาษา)

```ts
ctx.message.success(ctx.t('Copied'));
ctx.message.warning(ctx.t('Please select at least one row'));
ctx.message.success(ctx.t('Exported {{count}} records', { count: rows.length }));
```

### loading 与手动关闭 (การใช้ loading และการปิดด้วยตนเอง)

```ts
const hide = ctx.message.loading(ctx.t('Saving...'));
// ดำเนินการแบบ Asynchronous
await saveData();
hide();  // ปิด loading ด้วยตนเอง
ctx.message.success(ctx.t('Saved'));
```

### 使用 open 自定义配置 (ใช้ open เพื่อกำหนดค่าเอง)

```ts
ctx.message.open({
  type: 'success',
  content: 'ข้อความแจ้งเตือนสำเร็จแบบกำหนดเอง',
  duration: 5,
  onClose: () => console.log('message closed'),
});
```

### 关闭所有消息 (ปิดข้อความทั้งหมด)

```ts
ctx.message.destroy();
```

## 与 ctx.notification 的区别 (ความแตกต่างระหว่าง ctx.message และ ctx.notification)

| คุณสมบัติ | ctx.message | ctx.notification |
|------|--------------|------------------|
| **ตำแหน่ง** | กึ่งกลางด้านบนของหน้าจอ | มุมขวาบน |
| **วัตถุประสงค์** | แจ้งเตือนชั่วคราว หายไปเองอัตโนมัติ | แผงการแจ้งเตือน สามารถมีหัวข้อและรายละเอียด เหมาะสำหรับการแสดงผลระยะยาว |
| **สถานการณ์ทั่วไป** | การตอบสนองการทำงาน, การตรวจสอบข้อมูล, คัดลอกสำเร็จ | การแจ้งเตือนเมื่องานเสร็จสิ้น, ข้อความระบบ, เนื้อหาที่ยาวและต้องการให้ผู้ใช้สังเกตเห็น |
