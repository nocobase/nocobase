# ctx.notification

อ้างอิงจาก Ant Design Notification ซึ่งเป็น API สำหรับการแจ้งเตือนแบบโกลบอล (Global Notification API) ใช้สำหรับแสดงแผงการแจ้งเตือนที่ **มุมขวาบน** ของหน้าจอ เมื่อเปรียบเทียบกับ `ctx.message` การแจ้งเตือนนี้สามารถมีทั้งหัวข้อและรายละเอียด จึงเหมาะสำหรับเนื้อหาที่ต้องการแสดงเป็นเวลานานหรือต้องการให้ผู้ใช้สังเกตเห็นครับ

## สถานการณ์ที่เหมาะสม

| สถานการณ์ | คำอธิบาย |
|------|------|
| **JSBlock / เหตุการณ์การดำเนินการ (Action Events)** | การแจ้งเตือนเมื่อเสร็จสิ้นภารกิจ, ผลลัพธ์ของการดำเนินการแบบกลุ่ม (Batch operations), การส่งออกข้อมูลเสร็จสิ้น เป็นต้น |
| **กระแสเหตุการณ์ (Event Stream)** | การแจ้งเตือนระดับระบบหลังจากสิ้นสุดกระบวนการแบบอะซิงโครนัส (Asynchronous) |
| **เนื้อหาที่ต้องการการแสดงผลนานขึ้น** | การแจ้งเตือนที่สมบูรณ์พร้อมหัวข้อ, คำอธิบาย และปุ่มการดำเนินการ |

## การกำหนดประเภท (Type Definition)

```ts
notification: NotificationInstance;
```

`NotificationInstance` คืออินเทอร์เฟซการแจ้งเตือนของ Ant Design ซึ่งมีวิธีการ (Methods) ดังต่อไปนี้ครับ

## วิธีการที่ใช้บ่อย

| วิธีการ | คำอธิบาย |
|------|------|
| `open(config)` | เปิดการแจ้งเตือนด้วยการกำหนดค่าเอง |
| `success(config)` | แสดงการแจ้งเตือนประเภทสำเร็จ |
| `info(config)` | แสดงการแจ้งเตือนประเภทข้อมูล |
| `warning(config)` | แสดงการแจ้งเตือนประเภทคำเตือน |
| `error(config)` | แสดงการแจ้งเตือนประเภทข้อผิดพลาด |
| `destroy(key?)` | ปิดการแจ้งเตือนตาม key ที่ระบุ หากไม่ระบุ key จะปิดการแจ้งเตือนทั้งหมด |

**พารามิเตอร์การกำหนดค่า** (สอดคล้องกับ [Ant Design notification](https://ant.design/components/notification)):

| พารามิเตอร์ | ประเภท | คำอธิบาย |
|------|------|------|
| `message` | `ReactNode` | หัวข้อการแจ้งเตือน |
| `description` | `ReactNode` | รายละเอียดการแจ้งเตือน |
| `duration` | `number` | ระยะเวลาหน่วงก่อนปิดอัตโนมัติ (วินาที) ค่าเริ่มต้นคือ 4.5 วินาที หากตั้งเป็น 0 จะไม่ปิดอัตโนมัติ |
| `key` | `string` | รหัสเฉพาะของการแจ้งเตือน ใช้สำหรับ `destroy(key)` เพื่อปิดการแจ้งเตือนที่ระบุ |
| `onClose` | `() => void` | ฟังก์ชันคอลแบ็กเมื่อปิดการแจ้งเตือน |
| `placement` | `string` | ตำแหน่ง: `topLeft` \| `topRight` \| `bottomLeft` \| `bottomRight` |

## ตัวอย่าง

### การใช้งานพื้นฐาน

```ts
ctx.notification.open({
  message: 'ดำเนินการสำเร็จ',
  description: 'บันทึกข้อมูลลงในเซิร์ฟเวอร์เรียบร้อยแล้ว',
});
```

### การเรียกใช้งานด่วนตามประเภท

```ts
ctx.notification.success({
  message: ctx.t('Export completed'),
  description: ctx.t('Exported {{count}} records', { count: 10 }),
});

ctx.notification.error({
  message: ctx.t('Export failed'),
  description: ctx.t('Please check the console for details'),
});

ctx.notification.warning({
  message: ctx.t('Warning'),
  description: ctx.t('Some data may be incomplete'),
});
```

### การกำหนดระยะเวลาและ key เอง

```ts
ctx.notification.open({
  key: 'task-123',
  message: ctx.t('Task in progress'),
  description: ctx.t('Please wait...'),
  duration: 0,  // ไม่ปิดอัตโนมัติ
});

// ปิดด้วยตนเองหลังจากภารกิจเสร็จสิ้น
ctx.notification.destroy('task-123');
```

### การปิดการแจ้งเตือนทั้งหมด

```ts
ctx.notification.destroy();
```

## ความแตกต่างจาก ctx.message

| คุณสมบัติ | ctx.message | ctx.notification |
|------|--------------|------------------|
| **ตำแหน่ง** | กึ่งกลางด้านบนของหน้าจอ | มุมขวาบน (กำหนดค่าได้) |
| **โครงสร้าง** | ข้อความบรรทัดเดียวแบบเบา | มีหัวข้อ + รายละเอียด |
| **วัตถุประสงค์** | การตอบสนองชั่วคราว หายไปเองอัตโนมัติ | การแจ้งเตือนที่สมบูรณ์ แสดงผลได้นาน |
| **สถานการณ์ทั่วไป** | ดำเนินการสำเร็จ, ตรวจสอบข้อมูลไม่ผ่าน, คัดลอกสำเร็จ | ภารกิจเสร็จสิ้น, ข้อความจากระบบ, เนื้อหายาวที่ต้องการให้ผู้ใช้สนใจ |
