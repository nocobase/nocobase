# ctx.t()

ฟังก์ชันทางลัด i18n สำหรับแปลข้อความใน RunJS โดยอ้างอิงตามการตั้งค่าภาษาของบริบท (Context) ปัจจุบัน เหมาะสำหรับการทำ Internationalization (i18n) สำหรับข้อความในหน้าจอ เช่น ปุ่ม, หัวข้อ และข้อความแจ้งเตือนต่าง ๆ ครับ

## สถานการณ์ที่ใช้งาน

`ctx.t()` สามารถใช้งานได้ในทุกสภาพแวดล้อมการรัน (Execution environment) ของ RunJS ครับ

## คำนิยามประเภท (Type Definition)

```ts
t(key: string, options?: Record<string, any>): string
```

## พารามิเตอร์

| พารามิเตอร์ | ประเภท | คำอธิบาย |
|-----------|------|-------------|
| `key` | `string` | คีย์สำหรับการแปล หรือเทมเพลตที่มีตัวสำรองที่นั่ง (เช่น `Hello {{name}}`, `{{count}} rows`) |
| `options` | `object` | (ไม่บังคับ) ตัวแปรสำหรับแทนที่ค่า (เช่น `{ name: 'John', count: 5 }`) หรือตัวเลือก i18n (เช่น `defaultValue`, `ns`) |

## ค่าที่ส่งกลับ

- ส่งกลับสตริงที่แปลแล้ว หากไม่พบคีย์ที่ตรงกับการแปลและไม่ได้ระบุ `defaultValue` อาจส่งกลับเป็นตัวคีย์เองหรือสตริงที่แทนที่ค่าแล้วครับ

## เนมสเปซ (ns)

**เนมสเปซเริ่มต้นสำหรับสภาพแวดล้อม RunJS คือ `runjs`** เมื่อไม่มีการระบุ `ns` ฟังก์ชัน `ctx.t(key)` จะค้นหาคีย์จากเนมสเปซ `runjs` โดยอัตโนมัติครับ

```ts
// ค้นหาคีย์จากเนมสเปซ 'runjs' เป็นค่าเริ่มต้น
ctx.t('Submit'); // มีค่าเท่ากับ ctx.t('Submit', { ns: 'runjs' })

// ค้นหาคีย์จากเนมสเปซที่ระบุ
ctx.t('Submit', { ns: 'myModule' });

// ค้นหาจากหลายเนมสเปซตามลำดับ (เริ่มจาก 'runjs' แล้วตามด้วย 'common')
ctx.t('Save', { ns: ['runjs', 'common'] });
```

## ตัวอย่าง

### คีย์แบบง่าย

```ts
ctx.t('Submit');
ctx.t('No data');
```

### พร้อมตัวแปรแทนที่ค่า (Interpolation Variables)

```ts
const text = ctx.t('Hello {{name}}', { name: ctx.user?.nickname || 'Guest' });
ctx.render(`<div>${text}</div>`);
```

```ts
ctx.message.success(ctx.t('Processed {{count}} rows', { count: rows.length }));
```

### ข้อความแบบไดนามิก (เช่น เวลาสัมพัทธ์)

```ts
if (minutes < 60) return ctx.t('{{count}} minutes ago', { count: minutes });
if (hours < 24) return ctx.t('{{count}} hours ago', { count: hours });
```

### การระบุเนมสเปซ

```ts
ctx.t('Hello {{name}}', { name: 'Guest', ns: 'myModule' });
```

## ข้อควรระวัง

- **ปลั๊กอินการปรับให้เข้ากับท้องถิ่น (Localization Plugin)**: หากต้องการแปลข้อความ จำเป็นต้องเปิดใช้งานปลั๊กอินการปรับให้เข้ากับท้องถิ่นก่อน คีย์ที่ยังไม่มีคำแปลจะถูกดึงไปยังรายการจัดการการปรับให้เข้ากับท้องถิ่นโดยอัตโนมัติ เพื่อความสะดวกในการดูแลรักษาและแปลภาษาในที่เดียวครับ
- รองรับการแทนที่ค่าในรูปแบบ i18next: โดยการใช้ `{{ชื่อตัวแปร}}` ในคีย์ และส่งตัวแปรที่มีชื่อเดียวกันใน `options` เพื่อแทนที่ค่าครับ
- ภาษาจะถูกกำหนดโดยบริบทปัจจุบัน (เช่น `ctx.i18n.language` หรือ Locale ของผู้ใช้) ครับ
