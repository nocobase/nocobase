# ctx.exitAll()

ยุติเวิร์กโฟลว์ของเหตุการณ์ (event flow) ปัจจุบันและเวิร์กโฟลว์ของเหตุการณ์ที่ตามมาทั้งหมดที่ถูกเรียกใช้ในการจัดส่งเหตุการณ์ (event dispatch) เดียวกัน มักใช้เมื่อจำเป็นต้องยกเลิกเวิร์กโฟลว์ของเหตุการณ์ทั้งหมดภายใต้เหตุการณ์ปัจจุบันทันทีเนื่องจากข้อผิดพลาดระดับโกลบอลหรือการตรวจสอบสิทธิ์ครับ

## สถาณการณ์ที่ใช้งาน

โดยทั่วไป `ctx.exitAll()` จะใช้ในบริบทที่สามารถรัน JS ได้ และเมื่อต้องการ **ยกเลิกทั้งเวิร์กโฟลว์ของเหตุการณ์ปัจจุบันและเวิร์กโฟลว์ที่ตามมาซึ่งถูกเรียกโดยเหตุการณ์นั้นพร้อมกัน**:

| สถานการณ์ | คำอธิบาย |
|------|------|
| **เวิร์กโฟลว์ของเหตุการณ์** | การตรวจสอบเวิร์กโฟลว์หลักล้มเหลว (เช่น สิทธิ์ไม่เพียงพอ) จำเป็นต้องยุติเวิร์กโฟลว์หลักและเวิร์กโฟลว์ที่ตามมาภายใต้เหตุการณ์เดียวกันที่ยังไม่ได้ดำเนินการ |
| **กฎการเชื่อมโยง (Linkage Rules)** | เมื่อการตรวจสอบการเชื่อมโยงไม่ผ่าน จำเป็นต้องยุติการเชื่อมโยงปัจจุบันและการเชื่อมโยงที่ตามมาซึ่งถูกเรียกโดยเหตุการณ์เดียวกัน |
| **เหตุการณ์การดำเนินการ (Action Events)** | การตรวจสอบก่อนการดำเนินการล้มเหลว (เช่น การตรวจสอบสิทธิ์ก่อนลบ) จำเป็นต้องระงับการดำเนินการหลักและขั้นตอนที่ตามมา |

> ข้อแตกต่างจาก `ctx.exit()`: `ctx.exit()` จะยุติเฉพาะเวิร์กโฟลว์ของเหตุการณ์ปัจจุบันเท่านั้น ส่วน `ctx.exitAll()` จะยุติเวิร์กโฟลว์ของเหตุการณ์ปัจจุบันและเวิร์กโฟลว์ที่ตามมาในเหตุการณ์เดียวกันที่ **ยังไม่ได้ดำเนินการ** ครับ

## การกำหนดประเภท (Type Definition)

```ts
exitAll(): never;
```

การเรียก `ctx.exitAll()` จะส่ง (throw) `FlowExitAllException` ภายใน ซึ่งจะถูกดักจับโดยเอนจินของเวิร์กโฟลว์เพื่อหยุดอินสแตนซ์ของเวิร์กโฟลว์ปัจจุบันและเวิร์กโฟลว์ที่ตามมาภายใต้เหตุการณ์เดียวกัน เมื่อเรียกใช้แล้ว คำสั่งที่เหลือในโค้ด JS ปัจจุบันจะไม่ถูกดำเนินการครับ

## การเปรียบเทียบกับ ctx.exit()

| วิธีการ | ขอบเขตการทำงาน |
|------|----------|
| `ctx.exit()` | ยุติเฉพาะเวิร์กโฟลว์ของเหตุการณ์ปัจจุบัน เวิร์กโฟลว์ที่ตามมาจะไม่ได้รับผลกระทบ |
| `ctx.exitAll()` | ยุติเวิร์กโฟลว์ของเหตุการณ์ปัจจุบัน และยกเลิกเวิร์กโฟลว์ที่ตามมาซึ่งถูกดำเนินการ **แบบตามลำดับ (sequential)** ภายใต้เหตุการณ์เดียวกัน |

## คำอธิบายโหมดการดำเนินการ

- **การดำเนินการแบบตามลำดับ (Sequential)**: เวิร์กโฟลว์ภายใต้เหตุการณ์เดียวกันจะทำงานตามลำดับ หากเวิร์กโฟลว์ใดเรียก `ctx.exitAll()` เวิร์กโฟลว์ที่ตามมาจะไม่ถูกดำเนินการ
- **การดำเนินการแบบขนาน (Parallel)**: เวิร์กโฟลว์ภายใต้เหตุการณ์เดียวกันจะทำงานพร้อมกัน การเรียก `ctx.exitAll()` ในเวิร์กโฟลว์หนึ่งจะไม่ขัดจังหวะเวิร์กโฟลว์อื่นที่กำลังทำงานขนานกัน (เนื่องจากแยกเป็นอิสระต่อกัน)

## ตัวอย่าง

### ยุติเวิร์กโฟลว์ทั้งหมดเมื่อการตรวจสอบสิทธิ์ล้มเหลว

```ts
// ยกเลิกเวิร์กโฟลว์หลักและเวิร์กโฟลว์ที่ตามมาเมื่อสิทธิ์ไม่เพียงพอ
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'ไม่มีสิทธิ์ในการดำเนินการ' });
  ctx.exitAll();
}
```

### ยุติเมื่อการตรวจสอบก่อนดำเนินการระดับโกลบอลไม่ผ่าน

```ts
// ตัวอย่าง: หากพบว่าข้อมูลที่เกี่ยวข้องไม่สามารถลบได้ก่อนทำการลบ ให้ระงับเวิร์กโฟลว์หลักและการดำเนินการที่ตามมา
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('ไม่สามารถลบได้: มีข้อมูลที่เกี่ยวข้องอยู่');
  ctx.exitAll();
}
```

### การเลือกใช้ระหว่าง ctx.exit() และ ctx.exitAll()

```ts
// ต้องการให้ออกจากเวิร์กโฟลว์ปัจจุบันเท่านั้น -> ใช้ ctx.exit()
if (!params.valid) {
  ctx.message.error('พารามิเตอร์ไม่ถูกต้อง');
  ctx.exit();  // เวิร์กโฟลว์ที่ตามมาจะไม่ได้รับผลกระทบ
}

// ต้องการยุติเวิร์กโฟลว์ที่ตามมาทั้งหมดภายใต้เหตุการณ์ปัจจุบัน -> ใช้ ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'สิทธิ์ไม่เพียงพอ' });
  ctx.exitAll();  // ทั้งเวิร์กโฟลว์หลักและเวิร์กโฟลว์ที่ตามมาภายใต้เหตุการณ์เดียวกันจะถูกยุติ
}
```

### แสดงข้อความแจ้งเตือนก่อนยุติ

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('โปรดแก้ไขข้อผิดพลาดในฟอร์มก่อนครับ');
  ctx.exitAll();
}
```

## ข้อควรระวัง

- หลังจากเรียก `ctx.exitAll()` โค้ดที่ตามมาใน JS ปัจจุบันจะไม่ทำงาน แนะนำให้แจ้งเหตุผลแก่ผู้ใช้ผ่าน `ctx.message`, `ctx.notification` หรือหน้าต่างป๊อปอัพก่อนเรียกใช้งานครับ
- โดยปกติแล้วโค้ดทางธุรกิจไม่จำเป็นต้องดักจับ `FlowExitAllException` ปล่อยให้เอนจินของเวิร์กโฟลว์จัดการเองได้เลยครับ
- หากต้องการหยุดเฉพาะเวิร์กโฟลว์ปัจจุบันโดยไม่กระทบต่อเวิร์กโฟลว์ที่ตามมา ให้ใช้ `ctx.exit()`
- ในโหมดขนาน `ctx.exitAll()` จะยุติเฉพาะเวิร์กโฟลว์ปัจจุบันเท่านั้น และจะไม่ขัดจังหวะเวิร์กโฟลว์อื่นที่รันขนานกันอยู่
