# โหมดคลัสเตอร์

## บทนำ

NocoBase ตั้งแต่เวอร์ชัน v1.6.0 เป็นต้นไป รองรับการรันแอปพลิเคชันในโหมดคลัสเตอร์ครับ/ค่ะ เมื่อแอปพลิเคชันทำงานในโหมดคลัสเตอร์ จะสามารถเพิ่มประสิทธิภาพในการจัดการการเข้าถึงพร้อมกันได้ โดยใช้หลายอินสแตนซ์และโหมดมัลติคอร์ครับ/ค่ะ

## สถาปัตยกรรมระบบ

![20251031221530](https://static-docs.nocobase.com/20251031221530.png)

*   **คลัสเตอร์แอปพลิเคชัน**: คลัสเตอร์ที่ประกอบด้วยอินสแตนซ์ของแอปพลิเคชัน NocoBase หลายตัว สามารถติดตั้งบนเซิร์ฟเวอร์หลายเครื่อง หรือรันเป็นหลายโปรเซสในโหมดมัลติคอร์บนเซิร์ฟเวอร์เครื่องเดียวได้ครับ/ค่ะ
*   **ฐานข้อมูล**: ใช้จัดเก็บข้อมูลของแอปพลิเคชัน ซึ่งอาจเป็นฐานข้อมูลแบบโหนดเดียว (single-node) หรือฐานข้อมูลแบบกระจาย (distributed database) ก็ได้ครับ/ค่ะ
*   **พื้นที่จัดเก็บข้อมูลแบบแชร์**: ใช้สำหรับจัดเก็บไฟล์และข้อมูลของแอปพลิเคชัน โดยรองรับการเข้าถึงแบบอ่านและเขียนจากหลายอินสแตนซ์ครับ/ค่ะ
*   **มิดเดิลแวร์**: ประกอบด้วยคอมโพเนนต์ต่างๆ เช่น แคช (cache), สัญญาณซิงโครไนซ์ (synchronization signals), คิวข้อความ (message queue) และการล็อกแบบกระจาย (distributed locks) เพื่อรองรับการสื่อสารและการประสานงานภายในคลัสเตอร์แอปพลิเคชันครับ/ค่ะ
*   **โหลดบาลานเซอร์**: มีหน้าที่กระจายคำขอจากไคลเอนต์ไปยังอินสแตนซ์ต่างๆ ในคลัสเตอร์แอปพลิเคชัน รวมถึงทำการตรวจสอบสถานะ (health checks) และการย้ายระบบเมื่อเกิดข้อผิดพลาด (failover) ครับ/ค่ะ

## เรียนรู้เพิ่มเติม
