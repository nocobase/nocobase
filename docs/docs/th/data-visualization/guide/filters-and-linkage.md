# ตัวกรองหน้าและการเชื่อมโยง

ตัวกรองหน้า (Filter Block) ใช้สำหรับกำหนดเงื่อนไขการกรองข้อมูลแบบรวมศูนย์ในระดับหน้าเพจ และนำไปรวมกับคำสั่งคิวรีของแผนภูมิ เพื่อให้แผนภูมิหลายรายการสามารถกรองข้อมูลและเชื่อมโยงกันได้อย่างสอดคล้องกันครับ/ค่ะ

## ภาพรวมคุณสมบัติ
- เพิ่ม "Filter Block" ลงในหน้าเพจ เพื่อเป็นช่องทางในการกรองข้อมูลแบบรวมศูนย์สำหรับแผนภูมิทั้งหมดบนหน้าเพจปัจจุบันครับ/ค่ะ
- ใช้ปุ่ม "กรอง" (Filter), "รีเซ็ต" (Reset) และ "ยุบ" (Collapse) เพื่อดำเนินการกรอง, ล้างข้อมูล และยุบส่วนแสดงผลครับ/ค่ะ
- หากตัวกรองเลือกฟิลด์ที่เชื่อมโยงกับแผนภูมิ ค่าของฟิลด์เหล่านั้นจะถูกรวมเข้ากับคำขอคิวรีของแผนภูมิโดยอัตโนมัติ และกระตุ้นให้แผนภูมิรีเฟรชข้อมูลครับ/ค่ะ
- ตัวกรองยังสามารถสร้างฟิลด์ที่กำหนดเองได้ และลงทะเบียนในตัวแปรบริบท (Context Variables) เพื่อให้สามารถอ้างอิงได้ในบล็อกข้อมูลต่างๆ เช่น แผนภูมิ, ตาราง, ฟอร์ม และอื่นๆ ครับ/ค่ะ

สำหรับข้อมูลเพิ่มเติมเกี่ยวกับการใช้งานตัวกรองหน้า และการเชื่อมโยงกับบล็อกข้อมูลอื่นๆ เช่น แผนภูมิ โปรดดูเอกสารประกอบเรื่อง [ตัวกรองหน้า](#) ครับ/ค่ะ

## การใช้ค่าจากตัวกรองหน้าในคำสั่งคิวรีของแผนภูมิ
- โหมด Builder (แนะนำ)
  - รวมอัตโนมัติ: เมื่อแหล่งข้อมูลและคอลเลกชันตรงกัน ไม่จำเป็นต้องเขียนตัวแปรเพิ่มเติมในคำสั่งคิวรีของแผนภูมิ ตัวกรองหน้าจะถูกรวมด้วย `$and` ครับ/ค่ะ
  - เลือกด้วยตนเอง: คุณยังสามารถเลือกค่าจาก "ฟิลด์ที่กำหนดเอง" ของตัวกรองหน้าได้โดยตรงในเงื่อนไขการกรองของแผนภูมิครับ/ค่ะ

- โหมด SQL (ผ่านการแทรกตัวแปร)
  - ในคำสั่ง SQL ให้ "เลือกตัวแปร" (Choose variable) เพื่อแทรกค่าจาก "ฟิลด์ที่กำหนดเอง" ของตัวกรองหน้าครับ/ค่ะ
