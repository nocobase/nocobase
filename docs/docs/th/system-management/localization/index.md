# การจัดการการแปลภาษา

## บทนำ

ปลั๊กอินการจัดการการแปลภาษาใช้สำหรับจัดการและนำทรัพยากรการแปลภาษาของ NocoBase ไปใช้งานครับ/ค่ะ โดยสามารถแปลเมนูของระบบ, คอลเลกชัน, ฟิลด์ และปลั๊กอินทั้งหมด เพื่อให้เข้ากับภาษาและวัฒนธรรมของแต่ละภูมิภาคได้ครับ/ค่ะ

## การติดตั้ง

ปลั๊กอินนี้เป็นปลั๊กอินในตัว จึงไม่จำเป็นต้องติดตั้งเพิ่มเติมครับ/ค่ะ

### การซิงค์รายการคำแปล

<img src="https://static-docs.nocobase.com/202404202134850.png"/>

ปัจจุบันรองรับการซิงค์เนื้อหาดังต่อไปนี้ครับ/ค่ะ:

- ชุดภาษาท้องถิ่นของระบบและปลั๊กอิน
- ชื่อคอลเลกชัน, ชื่อฟิลด์ และป้ายกำกับตัวเลือกฟิลด์
- ชื่อเมนู

หลังจากซิงค์เสร็จสิ้น ระบบจะแสดงรายการคำแปลที่สามารถแปลได้ทั้งหมดของภาษาปัจจุบันครับ/ค่ะ

<img src="https://static-docs.nocobase.com/202404202136567.png"/>

:::info{title=คำแนะนำ}
โมดูลที่แตกต่างกันอาจมีรายการคำต้นฉบับที่เหมือนกัน ซึ่งจำเป็นต้องแปลแยกกันครับ/ค่ะ
:::

### การสร้างรายการคำแปลโดยอัตโนมัติ

เมื่อแก้ไขหน้ากระดาษ ข้อความที่กำหนดเองในแต่ละบล็อกจะสร้างรายการคำแปลที่เกี่ยวข้องโดยอัตโนมัติ และจะสร้างเนื้อหาคำแปลสำหรับภาษาปัจจุบันไปพร้อมกันครับ/ค่ะ

![](https://static-docs.nocobase.com/Localization-02-12-2026_08_39_AM.png)

![](https://static-docs.nocobase.com/Localization-NocoBase-02-12-2026_08_39_AM.png)

:::info{title=คำแนะนำ}
เมื่อกำหนดข้อความในโค้ด จำเป็นต้องระบุ ns (namespace) ด้วยตนเอง เช่น: `${ctx.i18n.t('My custom js block', { ns: 'lm-flow-engine' })}` ครับ/ค่ะ
:::
