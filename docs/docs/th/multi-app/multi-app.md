# หลายแอป

## บทนำ

ปลั๊กอิน **หลายแอป (Multi-App)** ช่วยให้คุณสามารถสร้างและจัดการแอปพลิเคชันย่อยหลายตัวได้อย่างยืดหยุ่น โดยไม่จำเป็นต้องติดตั้งแยกกัน แอปพลิเคชันย่อยแต่ละตัวจะเป็นอินสแตนซ์ที่ทำงานได้อย่างอิสระ มีฐานข้อมูล, ปลั๊กอิน และการตั้งค่าเป็นของตัวเองครับ/ค่ะ

#### กรณีการใช้งาน
- **Multi-tenancy (หลายผู้เช่า)**: จัดเตรียมอินสแตนซ์แอปพลิเคชันที่เป็นอิสระ โดยลูกค้าแต่ละรายจะมีข้อมูล, การตั้งค่าปลั๊กอิน และระบบสิทธิ์เป็นของตัวเองครับ/ค่ะ
- **ระบบหลักและระบบย่อยสำหรับโดเมนธุรกิจที่แตกต่างกัน**: ระบบขนาดใหญ่ที่ประกอบด้วยแอปพลิเคชันขนาดเล็กหลายตัวซึ่งติดตั้งแยกกันครับ/ค่ะ

:::warning
ปลั๊กอินหลายแอปเองไม่ได้มีความสามารถในการแชร์ผู้ใช้งานนะครับ/คะ
หากต้องการให้ผู้ใช้งานสามารถเข้าถึงแอปพลิเคชันหลายตัวได้ สามารถใช้งานร่วมกับ **[ปลั๊กอินการยืนยันตัวตน](/auth-verification)** ได้ครับ/ค่ะ
:::

## การติดตั้ง

ในหน้าจัดการปลั๊กอิน ให้ค้นหาปลั๊กอิน **หลายแอป (Multi-app)** แล้วเปิดใช้งานได้เลยครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Plugin-manager-NocoBase-10-16-2025_03_07_PM.png)

## คู่มือการใช้งาน

### การสร้างแอปพลิเคชันย่อย

ในเมนูการตั้งค่าระบบ ให้คลิกที่ "หลายแอป" เพื่อเข้าสู่หน้าจัดการหลายแอปครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

คลิกปุ่ม "เพิ่มใหม่" เพื่อสร้างแอปพลิเคชันย่อยตัวใหม่ครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### คำอธิบายฟิลด์ในฟอร์ม

*   **ชื่อ (Name)**: ตัวระบุแอปพลิเคชันย่อย ซึ่งต้องไม่ซ้ำกันทั่วทั้งระบบครับ/ค่ะ
*   **ชื่อที่แสดง (Display Name)**: ชื่อของแอปพลิเคชันย่อยที่จะแสดงในหน้าจอครับ/ค่ะ
*   **โหมดการเริ่มต้น (Startup Mode)**:
    *   **เริ่มเมื่อมีการเข้าถึงครั้งแรก**: แอปพลิเคชันย่อยจะเริ่มทำงานเมื่อผู้ใช้งานเข้าถึงผ่าน URL เป็นครั้งแรกเท่านั้นครับ/ค่ะ
    *   **เริ่มพร้อมกับแอปพลิเคชันหลัก**: แอปพลิเคชันย่อยจะเริ่มทำงานพร้อมกับแอปพลิเคชันหลัก (ซึ่งจะทำให้เวลาในการเริ่มต้นของแอปพลิเคชันหลักเพิ่มขึ้นครับ/ค่ะ)
*   **พอร์ต (Port)**: หมายเลขพอร์ตที่แอปพลิเคชันย่อยใช้ในการทำงานครับ/ค่ะ
*   **โดเมนแบบกำหนดเอง (Custom Domain)**: ตั้งค่าซับโดเมนแยกต่างหากสำหรับแอปพลิเคชันย่อยครับ/ค่ะ
*   **ปักหมุดที่เมนู (Pin to menu)**: ปักหมุดทางเข้าแอปพลิเคชันย่อยให้แสดงที่ด้านซ้ายของแถบนำทางด้านบนครับ/ค่ะ
*   **การเชื่อมต่อฐานข้อมูล (Database Connection)**: ใช้สำหรับตั้งค่า แหล่งข้อมูล ของแอปพลิเคชันย่อย โดยรองรับสามวิธีดังนี้ครับ/ค่ะ
    *   **ฐานข้อมูลใหม่**: ใช้บริการข้อมูลปัจจุบันซ้ำ เพื่อสร้างฐานข้อมูลที่เป็นอิสระครับ/ค่ะ
    *   **การเชื่อมต่อข้อมูลใหม่**: ตั้งค่าบริการฐานข้อมูลใหม่ทั้งหมดครับ/ค่ะ
    *   **โหมด Schema**: สร้าง Schema ที่เป็นอิสระสำหรับแอปพลิเคชันย่อยใน PostgreSQL ครับ/ค่ะ
*   **อัปเกรด (Upgrade)**: หากฐานข้อมูลที่เชื่อมต่อมีโครงสร้างข้อมูล NocoBase เวอร์ชันเก่าอยู่ ระบบจะทำการอัปเกรดเป็นเวอร์ชันปัจจุบันโดยอัตโนมัติครับ/ค่ะ

### การเริ่มและหยุดแอปพลิเคชันย่อย

คลิกปุ่ม **เริ่ม (Start)** เพื่อเริ่มแอปพลิเคชันย่อยได้เลยครับ/ค่ะ
> หากคุณเลือก *“เริ่มเมื่อมีการเข้าถึงครั้งแรก”* ในระหว่างการสร้าง แอปพลิเคชันย่อยจะเริ่มทำงานโดยอัตโนมัติเมื่อมีการเข้าถึงครั้งแรกครับ/ค่ะ

คลิกปุ่ม **ดู (View)** เพื่อเปิดแอปพลิเคชันย่อยในแท็บใหม่ครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)

### สถานะและการบันทึก (Log) ของแอปพลิเคชันย่อย

ในรายการ คุณสามารถดูการใช้งานหน่วยความจำและ CPU ของแต่ละแอปพลิเคชันได้ครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

คลิกปุ่ม **บันทึก (Logs)** เพื่อดูบันทึกการทำงานของแอปพลิเคชันย่อยครับ/ค่ะ
> หากแอปพลิเคชันย่อยไม่สามารถเข้าถึงได้หลังจากเริ่มต้น (เช่น ฐานข้อมูลเสียหาย) คุณสามารถใช้บันทึกเพื่อตรวจสอบและแก้ไขปัญหาได้ครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)

### การลบแอปพลิเคชันย่อย

คลิกปุ่ม **ลบ (Delete)** เพื่อลบแอปพลิเคชันย่อยได้เลยครับ/ค่ะ
> เมื่อทำการลบ คุณสามารถเลือกว่าจะลบฐานข้อมูลไปด้วยหรือไม่ โปรดดำเนินการด้วยความระมัดระวัง เพราะการกระทำนี้ไม่สามารถย้อนกลับได้ครับ/ค่ะ

### การเข้าถึงแอปพลิเคชันย่อย
โดยปกติแล้ว แอปพลิเคชันย่อยจะถูกเข้าถึงโดยใช้ `/_app/:appName/admin/` เช่น
```
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
นอกจากนี้ คุณยังสามารถตั้งค่าซับโดเมนแยกต่างหากสำหรับแอปพลิเคชันย่อยได้ด้วยครับ/ค่ะ โดยจะต้องทำการชี้โดเมนไปยัง IP ปัจจุบัน และหากคุณใช้ Nginx ก็จำเป็นต้องเพิ่มโดเมนดังกล่าวในการตั้งค่า Nginx ด้วยครับ/ค่ะ

### การจัดการแอปพลิเคชันย่อยผ่าน Command Line

ในไดเรกทอรีหลักของโปรเจกต์ คุณสามารถใช้ Command Line เพื่อจัดการอินสแตนซ์ของแอปพลิเคชันย่อยผ่าน **PM2** ได้ครับ/ค่ะ

```bash
yarn nocobase pm2 list              # ดูรายการอินสแตนซ์ที่กำลังทำงานอยู่
yarn nocobase pm2 stop [appname]    # หยุดกระบวนการของแอปพลิเคชันย่อยที่ระบุ
yarn nocobase pm2 delete [appname]  # ลบกระบวนการของแอปพลิเคชันย่อยที่ระบุ
yarn nocobase pm2 kill              # บังคับยุติกระบวนการที่เริ่มต้นทั้งหมด (อาจรวมถึงอินสแตนซ์ของแอปพลิเคชันหลัก)
```

### การย้ายข้อมูลจาก Multi-app เวอร์ชันเก่า

ไปที่หน้าจัดการ Multi-app เวอร์ชันเก่า แล้วคลิกปุ่ม **ย้ายข้อมูลไปยัง Multi-app ใหม่** เพื่อทำการย้ายข้อมูลครับ/ค่ะ

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)

## คำถามที่พบบ่อย

#### 1. การจัดการปลั๊กอิน
แอปพลิเคชันย่อยสามารถใช้ปลั๊กอินได้เหมือนกับแอปพลิเคชันหลัก (รวมถึงเวอร์ชันด้วย) แต่สามารถตั้งค่าและใช้งานปลั๊กอินได้อย่างอิสระครับ/ค่ะ

#### 2. การแยกฐานข้อมูล
แอปพลิเคชันย่อยสามารถตั้งค่าฐานข้อมูลที่เป็นอิสระได้ครับ/ค่ะ หากต้องการแชร์ข้อมูลระหว่างแอปพลิเคชัน สามารถทำได้ผ่าน แหล่งข้อมูล ภายนอกครับ/ค่ะ

#### 3. การสำรองและย้ายข้อมูล
ปัจจุบัน การสำรองข้อมูลบนแอปพลิเคชันหลักไม่รองรับการรวมข้อมูลของแอปพลิเคชันย่อย (จะรวมเฉพาะข้อมูลพื้นฐานของแอปพลิเคชันย่อยเท่านั้น) คุณจำเป็นต้องสำรองและย้ายข้อมูลภายในแอปพลิเคชันย่อยแต่ละตัวด้วยตนเองครับ/ค่ะ

#### 4. การติดตั้งและการอัปเดต
เวอร์ชันของแอปพลิเคชันย่อยจะได้รับการอัปเกรดโดยอัตโนมัติพร้อมกับแอปพลิเคชันหลัก เพื่อให้มั่นใจว่าเวอร์ชันของแอปพลิเคชันหลักและแอปพลิเคชันย่อยมีความสอดคล้องกันครับ/ค่ะ

#### 5. การจัดการทรัพยากร
การใช้ทรัพยากรของแอปพลิเคชันย่อยแต่ละตัวจะใกล้เคียงกับแอปพลิเคชันหลักครับ/ค่ะ ปัจจุบัน แอปพลิเคชันหนึ่งตัวใช้หน่วยความจำประมาณ 500-600MB ครับ/ค่ะ
