# โหมดหน่วยความจำร่วม

## บทนำ

เมื่อผู้ใช้ต้องการแยกธุรกิจในระดับแอปพลิเคชัน แต่ไม่ต้องการนำโครงสร้างการปรับใช้และการดูแลระบบที่ซับซ้อนมาใช้ สามารถใช้โหมดหลายแอปพลิเคชันแบบหน่วยความจำร่วมได้ครับ

ในโหมดนี้ อินสแตนซ์ NocoBase เดียวสามารถรันหลายแอปพลิเคชันได้พร้อมกันครับ แต่ละแอปพลิเคชันมีความเป็นอิสระ สามารถเชื่อมต่อกับฐานข้อมูลแยกกัน สามารถสร้าง เริ่มต้น และหยุดการทำงานได้โดยอิสระ แต่จะใช้กระบวนการ (Process) และพื้นที่หน่วยความจำร่วมกัน ทำให้ผู้ใช้ยังคงต้องดูแลรักษา NocoBase เพียงอินสแตนซ์เดียวเท่านั้นครับ

## คู่มือการใช้งาน

### การตั้งค่าตัวแปรสภาพแวดล้อม

ก่อนใช้งานฟังก์ชันหลายแอปพลิเคชัน โปรดตรวจสอบให้แน่ใจว่าได้ตั้งค่าตัวแปรสภาพแวดล้อมต่อไปนี้เมื่อเริ่มต้น NocoBase ครับ:

```bash
APP_DISCOVERY_ADAPTER=local
APP_PROCESS_ADAPTER=local
```

### การสร้างแอปพลิเคชัน

ในเมนูการตั้งค่าระบบ ให้คลิกที่ "App Supervisor" เพื่อเข้าสู่หน้าการจัดการแอปพลิเคชันครับ

![](https://static-docs.nocobase.com/202512291056215.png)

คลิกที่ปุ่ม "เพิ่มใหม่" เพื่อสร้างแอปพลิเคชันใหม่ครับ

![](https://static-docs.nocobase.com/202512291057696.png)

#### คำอธิบายรายการกำหนดค่า

| รายการกำหนดค่า | คำอธิบาย |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ชื่อแอปพลิเคชัน** | ชื่อของแอปพลิเคชันที่แสดงในอินเทอร์เฟซครับ |
| **ID แอปพลิเคชัน** | ตัวระบุแอปพลิเคชัน ซึ่งต้องไม่ซ้ำกันทั่วโลกครับ |
| **โหมดการเริ่มต้น** | - เริ่มต้นเมื่อเข้าถึงครั้งแรก: เริ่มต้นเมื่อผู้ใช้เข้าถึงแอปพลิเคชันย่อยผ่าน URL เป็นครั้งแรกเท่านั้นครับ<br>- เริ่มต้นพร้อมกับแอปพลิเคชันหลัก: เริ่มต้นแอปพลิเคชันย่อยพร้อมกับการเริ่มต้นแอปพลิเคชันหลัก (จะเพิ่มเวลาในการเริ่มต้นแอปพลิเคชันหลักครับ) |
| **สภาพแวดล้อม** | ในโหมดหน่วยความจำร่วม จะมีเพียงสภาพแวดล้อมท้องถิ่นเท่านั้นที่ใช้งานได้ คือ `local` ครับ |
| **การเชื่อมต่อฐานข้อมูล** | ใช้สำหรับกำหนดค่าแหล่งข้อมูลหลักของแอปพลิเคชัน รองรับสามวิธีดังนี้ครับ:<br>- ฐานข้อมูลใหม่: ใช้บริการฐานข้อมูลปัจจุบันซ้ำเพื่อสร้างฐานข้อมูลอิสระ<br>- การเชื่อมต่อข้อมูลใหม่: เชื่อมต่อกับบริการฐานข้อมูลอื่น<br>- โหมด Schema: เมื่อแหล่งข้อมูลหลักปัจจุบันเป็น PostgreSQL จะสร้าง Schema อิสระสำหรับแอปพลิเคชันครับ |
| **การอัปเกรด** | หากมีข้อมูลแอปพลิเคชัน NocoBase เวอร์ชันเก่าอยู่ในฐานข้อมูลที่เชื่อมต่อ จะอนุญาตให้อัปเกรดเป็นเวอร์ชันปัจจุบันโดยอัตโนมัติหรือไม่ครับ |
| **รหัสลับ JWT** | สร้างรหัสลับ JWT อิสระสำหรับแอปพลิเคชันโดยอัตโนมัติ เพื่อให้แน่ใจว่าเซสชันของแอปพลิเคชันแยกจากแอปพลิเคชันหลักและแอปพลิเคชันอื่นครับ |
| **โดเมนที่กำหนดเอง** | กำหนดค่าโดเมนการเข้าถึงอิสระสำหรับแอปพลิเคชันครับ |

### การเริ่มต้นแอปพลิเคชัน

คลิกที่ปุ่ม **เริ่มต้น** เพื่อเริ่มการทำงานของแอปพลิเคชันย่อยครับ

> หากเลือก _"เริ่มต้นเมื่อเข้าถึงครั้งแรก"_ ขณะสร้าง แอปพลิเคชันจะเริ่มต้นโดยอัตโนมัติเมื่อมีการเข้าถึงครั้งแรกครับ

![](https://static-docs.nocobase.com/202512291121065.png)

### การเข้าถึงแอปพลิเคชัน

คลิกที่ปุ่ม **เข้าชม** เพื่อเปิดแอปพลิเคชันย่อยในแท็บใหม่ครับ

โดยปกติจะใช้ `/apps/:appName/admin/` ในการเข้าถึงแอปพลิเคชันย่อย ตัวอย่างเช่น

```bash
http://localhost:13000/apps/a_7zkxoarusnx/admin/
```

ในขณะเดียวกัน คุณยังสามารถกำหนดค่าโดเมนอิสระสำหรับแอปพลิเคชันย่อยได้ โดยจำเป็นต้องชี้โดเมนไปยัง IP ปัจจุบัน และหากใช้ Nginx จะต้องเพิ่มโดเมนในไฟล์กำหนดค่าของ Nginx ด้วยครับ

### การหยุดแอปพลิเคชัน

คลิกที่ปุ่ม **หยุด** เพื่อหยุดการทำงานของแอปพลิเคชันย่อยครับ

![](https://static-docs.nocobase.com/202512291122113.png)

### สถานะแอปพลิเคชัน

คุณสามารถดูสถานะปัจจุบันของแต่ละแอปพลิเคชันได้ในรายการครับ

![](https://static-docs.nocobase.com/202512291122339.png)

### การลบแอปพลิเคชัน

คลิกที่ปุ่ม **ลบ** เพื่อลบแอปพลิเคชันครับ

![](https://static-docs.nocobase.com/202512291122178.png)

## คำถามที่พบบ่อย

### 1. การจัดการปลั๊กอิน

ปลั๊กอินที่แอปพลิเคชันอื่นสามารถใช้งานได้จะเหมือนกับแอปพลิเคชันหลัก (รวมถึงเวอร์ชัน) แต่สามารถกำหนดค่าและใช้งานปลั๊กอินแยกกันได้ครับ

### 2. การแยกฐานข้อมูล

แอปพลิเคชันอื่นสามารถกำหนดค่าฐานข้อมูลอิสระได้ หากต้องการแชร์ข้อมูลระหว่างแอปพลิเคชัน สามารถทำได้ผ่านแหล่งข้อมูลภายนอกครับ

### 3. การสำรองข้อมูลและการย้ายข้อมูล

ในปัจจุบัน การสำรองข้อมูลในแอปพลิเคชันหลักไม่รองรับการรวมข้อมูลจากแอปพลิเคชันอื่น (รวมเฉพาะข้อมูลพื้นฐานของแอปพลิเคชันเท่านั้น) จำเป็นต้องสำรองข้อมูลและย้ายข้อมูลด้วยตนเองภายในแอปพลิเคชันอื่นครับ

### 4. การปรับใช้และการอัปเดต

ในโหมดหน่วยความจำร่วม เวอร์ชันของแอปพลิเคชันอื่นจะอัปเกรดตามแอปพลิเคชันหลักโดยอัตโนมัติ เพื่อให้แน่ใจว่าเวอร์ชันของแอปพลิเคชันตรงกันโดยอัตโนมัติครับ

### 5. เซสชันแอปพลิเคชัน

- หากแอปพลิเคชันใช้รหัสลับ JWT อิสระ เซสชันของแอปพลิเคชันจะแยกจากแอปพลิเคชันหลักและแอปพลิเคชันอื่นครับ หากเข้าถึงแอปพลิเคชันที่ต่างกันผ่านเส้นทางย่อยของโดเมนเดียวกัน เนื่องจาก TOKEN ของแอปพลิเคชันถูกแคชไว้ใน LocalStorage จึงจำเป็นต้องเข้าสู่ระบบใหม่เมื่อสลับไปมาระหว่างแอปพลิเคชันครับ แนะนำให้กำหนดค่าโดเมนอิสระสำหรับแอปพลิเคชันต่างๆ เพื่อให้มีการแยกเซสชันที่ดีขึ้นครับ
- หากแอปพลิเคชันไม่ได้ใช้รหัสลับ JWT อิสระ จะมีการใช้เซสชันร่วมกับแอปพลิเคชันหลัก เมื่อเข้าถึงแอปพลิเคชันอื่นในเบราว์เซอร์เดียวกันแล้วกลับมาที่แอปพลิเคชันหลักก็ไม่จำเป็นต้องเข้าสู่ระบบใหม่ครับ แต่มีความเสี่ยงด้านความปลอดภัย หาก ID ผู้ใช้ของแอปพลิเคชันต่างๆ ซ้ำกัน อาจทำให้ผู้ใช้สามารถเข้าถึงข้อมูลของแอปพลิเคชันอื่นโดยไม่ได้รับอนุญาตได้ครับ
