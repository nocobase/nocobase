# การยืนยันตัวตน: LDAP

## บทนำ

ปลั๊กอินการยืนยันตัวตน: LDAP เป็นไปตามมาตรฐานโปรโตคอล LDAP (Lightweight Directory Access Protocol) ซึ่งช่วยให้ผู้ใช้สามารถเข้าสู่ระบบ NocoBase โดยใช้บัญชีและรหัสผ่านจากเซิร์ฟเวอร์ LDAP ได้ครับ/ค่ะ

## การตั้งค่า

### การตั้งค่าพื้นฐาน

<img src="https://static-docs.nocobase.com/202405101605728.png"/>

- Sign up automatically when the user does not exist - เมื่อไม่พบบัญชีผู้ใช้เดิมที่ตรงกัน จะสร้างผู้ใช้ใหม่โดยอัตโนมัติหรือไม่ครับ/คะ
- LDAP URL - ที่อยู่เซิร์ฟเวอร์ LDAP
- Bind DN - DN ที่ใช้สำหรับทดสอบการเชื่อมต่อเซิร์ฟเวอร์และค้นหาผู้ใช้
- Bind password - รหัสผ่านของ Bind DN
- Test connection - คลิกปุ่มเพื่อทดสอบการเชื่อมต่อเซิร์ฟเวอร์และตรวจสอบความถูกต้องของ Bind DN ครับ/ค่ะ

### การตั้งค่าการค้นหา

<img src="https://static-docs.nocobase.com/202405101609984.png"/>

- Search DN - DN ที่ใช้สำหรับค้นหาผู้ใช้
- Search filter - เงื่อนไขการกรองสำหรับการค้นหาผู้ใช้ โดยใช้ `{{account}}` แทนบัญชีผู้ใช้ที่ใช้ในการเข้าสู่ระบบ
- Scope - `Base`, `One level`, `Subtree`, ค่าเริ่มต้นคือ `Subtree`
- Size limit - ขนาดการแบ่งหน้าผลการค้นหา

### การแมปแอตทริบิวต์

<img src="https://static-docs.nocobase.com/202405101612814.png"/>

- Use this field to bind the user - ฟิลด์ที่ใช้สำหรับผูกกับผู้ใช้เดิม หากบัญชีที่ใช้เข้าสู่ระบบเป็นชื่อผู้ใช้ ให้เลือกชื่อผู้ใช้ หากเป็นอีเมล ให้เลือกอีเมล ค่าเริ่มต้นคือชื่อผู้ใช้ครับ/ค่ะ
- Attribute map - การแมปแอตทริบิวต์ของผู้ใช้กับฟิลด์ในตารางผู้ใช้ของ NocoBase
