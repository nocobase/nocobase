# ภาพรวม

![clipboard-image-1771905619](https://static-docs.nocobase.com/clipboard-image-1771905619.png)

พนักงาน AI (`AI Employees`) คือความสามารถของเอเจนต์อัจฉริยะที่ผสานรวมเข้ากับระบบธุรกิจของ NocoBase อย่างลึกซึ้งครับ/ค่ะ

นี่ไม่ใช่หุ่นยนต์ที่ "ทำได้แค่แชท" แต่เป็น "เพื่อนร่วมงานดิจิทัล" ที่สามารถเข้าใจบริบทและดำเนินการได้โดยตรงในอินเทอร์เฟซธุรกิจครับ/ค่ะ:

- **เข้าใจบริบททางธุรกิจ**: รับรู้หน้าปัจจุบัน, บล็อก, โครงสร้างข้อมูล และเนื้อหาที่เลือกครับ/ค่ะ
- **สามารถดำเนินการได้โดยตรง**: สามารถเรียกใช้ทักษะเพื่อทำงานต่างๆ เช่น การค้นหา, การวิเคราะห์, การกรอกข้อมูล, การกำหนดค่า และการสร้างครับ/ค่ะ
- **การทำงานร่วมกันตามบทบาท**: กำหนดค่าพนักงานที่แตกต่างกันตามตำแหน่งงาน และสลับโมเดลเพื่อทำงานร่วมกันในการสนทนาครับ/ค่ะ

## เส้นทางการเริ่มต้นใช้งานใน 5 นาที

ดู [เริ่มต้นใช้งานด่วน](/ai-employees/quick-start) ก่อน และกำหนดค่าขั้นต่ำที่ใช้งานได้ตามลำดับต่อไปนี้ครับ/ค่ะ:

1. กำหนดค่า [บริการ LLM](/ai-employees/features/llm-service) อย่างน้อยหนึ่งรายการครับ/ค่ะ
2. เปิดใช้งาน [พนักงาน AI](/ai-employees/features/enable-ai-employee) อย่างน้อยหนึ่งคนครับ/ค่ะ
3. เปิดการสนทนาและเริ่ม [ทำงานร่วมกับพนักงาน AI](/ai-employees/features/collaborate) ครับ/ค่ะ
4. เปิดใช้งาน [การค้นหาเว็บ](/ai-employees/features/web-search) และ [งานด่วน](/ai-employees/features/task) ตามต้องการครับ/ค่ะ

## แผนที่ฟีเจอร์

### A. การกำหนดค่าพื้นฐาน (ผู้ดูแลระบบ)

- [กำหนดค่าบริการ LLM](/ai-employees/features/llm-service): เชื่อมต่อ Provider, กำหนดค่าและจัดการโมเดลที่พร้อมใช้งานครับ/ค่ะ
- [เปิดใช้งานพนักงาน AI](/ai-employees/features/enable-ai-employee): เปิด/ปิดพนักงานในตัว และควบคุมขอบเขตการใช้งานครับ/ค่ะ
- [สร้างพนักงาน AI ใหม่](/ai-employees/features/new-ai-employees): กำหนดบทบาท, บุคลิกภาพ, ข้อความต้อนรับ และขอบเขตความสามารถครับ/ค่ะ
- [การใช้ทักษะ](/ai-employees/features/tool): กำหนดค่าสิทธิ์ของทักษะ (`Ask` / `Allow`) เพื่อควบคุมความเสี่ยงในการดำเนินการครับ/ค่ะ

### B. การทำงานร่วมกันในแต่ละวัน (ผู้ใช้ทางธุรกิจ)

- [ทำงานร่วมกับพนักงาน AI](/ai-employees/features/collaborate): สลับพนักงานและโมเดลภายในการสนทนาเพื่อการทำงานร่วมกันอย่างต่อเนื่องครับ/ค่ะ
- [เพิ่มบริบท - บล็อก](/ai-employees/features/pick-block): ส่งบล็อกของหน้าเป็นบริบทให้กับ AI ครับ/ค่ะ
- [งานด่วน](/ai-employees/features/task): ตั้งค่างานที่ใช้บ่อยไว้ล่วงหน้าบนหน้า/บล็อก และดำเนินการได้ในคลิกเดียวครับ/ค่ะ
- [การค้นหาเว็บ](/ai-employees/features/web-search): เปิดใช้งานการค้นหาเพื่อเสริมคำตอบเมื่อต้องการข้อมูลล่าสุดครับ/ค่ะ

### C. ความสามารถขั้นสูง (ส่วนขยาย)

- [พนักงาน AI ในตัว](/ai-employees/features/built-in-employee): ทำความเข้าใจตำแหน่งและสถานการณ์ที่เหมาะสมของพนักงานที่ตั้งค่าไว้ล่วงหน้าครับ/ค่ะ
- [การควบคุมสิทธิ์](/ai-employees/permission): ควบคุมการเข้าถึงพนักงาน, ทักษะ และข้อมูลตามโมเดลสิทธิ์ขององค์กรครับ/ค่ะ
- [คลังความรู้ AI](/ai-employees/knowledge-base/index): นำความรู้ขององค์กรมาใช้ เพื่อเพิ่มความเสถียรและความสามารถในการตรวจสอบย้อนกลับของคำตอบครับ/ค่ะ
- [โหนด LLM ในเวิร์กโฟลว์](/ai-employees/workflow/nodes/llm/chat): จัดลำดับความสามารถของ AI เข้าสู่กระบวนการอัตโนมัติครับ/ค่ะ

## แนวคิดหลัก (แนะนำให้ทำความเข้าใจให้ตรงกันก่อน)

คำศัพท์ต่อไปนี้สอดคล้องกับอภิธานศัพท์ แนะนำให้ใช้ร่วมกันภายในทีมครับ/ค่ะ:

- **พนักงาน AI (AI Employee)**: เอเจนต์ที่สามารถดำเนินการได้ ซึ่งประกอบด้วยการตั้งค่าบทบาท (Role setting) และทักษะ (Tool / Skill) ครับ/ค่ะ
- **บริการ LLM (LLM Service)**: หน่วยการเข้าถึงโมเดลและการกำหนดค่าความสามารถ ใช้สำหรับจัดการ Provider และรายการโมเดลครับ/ค่ะ
- **ผู้ให้บริการ (Provider)**: ผู้จัดหาโมเดลที่อยู่เบื้องหลังบริการ LLM ครับ/ค่ะ
- **โมเดลที่เปิดใช้งาน (Enabled Models)**: ชุดของโมเดลที่บริการ LLM ปัจจุบันอนุญาตให้เลือกในการสนทนาครับ/ค่ะ
- **ตัวสลับพนักงาน AI (AI Employee Switcher)**: สลับพนักงานที่ทำงานร่วมกันในปัจจุบันภายในการสนทนาครับ/ค่ะ
- **ตัวสลับโมเดล (Model Switcher)**: สลับโมเดลภายในการสนทนา และจดจำความชอบตามรายพนักงานครับ/ค่ะ
- **ทักษะ (Tool / Skill)**: หน่วยความสามารถในการดำเนินการที่ AI สามารถเรียกใช้ได้ครับ/ค่ะ
- **สิทธิ์ของทักษะ (Permission: Ask / Allow)**: การกำหนดว่าต้องมีการยืนยันจากมนุษย์ก่อนเรียกใช้ทักษะหรือไม่ครับ/ค่ะ
- **บริบท (Context)**: ข้อมูลสภาพแวดล้อมทางธุรกิจ เช่น หน้า, บล็อก, โครงสร้างข้อมูล เป็นต้นครับ/ค่ะ
- **การสนทนา (Chat)**: กระบวนการโต้ตอบอย่างต่อเนื่องระหว่างผู้ใช้และพนักงาน AI ครับ/ค่ะ
- **การค้นหาเว็บ (Web Search)**: ความสามารถในการเสริมข้อมูลแบบเรียลไลม์โดยอิงจากการค้นหาภายนอกครับ/ค่ะ
- **คลังความรู้ (Knowledge Base / RAG)**: การนำความรู้ขององค์กรมาใช้ผ่านการสร้างแบบเสริมการค้นหาครับ/ค่ะ
- **การจัดเก็บเวกเตอร์ (Vector Store)**: การจัดเก็บแบบเวกเตอร์ที่ให้ความสามารถในการค้นหาเชิงความหมายสำหรับคลังความรู้ครับ/ค่ะ

## คำแนะนำการติดตั้ง

พนักงาน AI เป็นปลั๊กอินในตัวของ NocoBase (`@nocobase/plugin-ai`) ซึ่งพร้อมใช้งานทันทีโดยไม่ต้องติดตั้งแยกต่างหากครับ/ค่ะ
