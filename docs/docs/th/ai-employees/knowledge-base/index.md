# ภาพรวม

## บทนำ

ปลั๊กอิน AI Knowledge Base ช่วยให้ผู้ช่วย AI มีความสามารถในการดึงข้อมูลแบบ RAG

ความสามารถในการดึงข้อมูลแบบ RAG ช่วยให้ผู้ช่วย AI สามารถให้คำตอบที่แม่นยำ เป็นมืออาชีพ และเกี่ยวข้องกับข้อมูลภายในองค์กรได้ดียิ่งขึ้น เมื่อตอบคำถามของผู้ใช้

ด้วยการใช้เอกสารเฉพาะทางและเอกสารภายในองค์กรจากฐานความรู้ที่ผู้ดูแลระบบดูแลรักษา จะช่วยเพิ่มความแม่นยำและความสามารถในการตรวจสอบย้อนกลับของคำตอบที่ผู้ช่วย AI ให้บริการ

### RAG คืออะไร

RAG (Retrieval Augmented Generation) ย่อมาจาก "Retrieval-Augmented-Generation" ซึ่งหมายถึง "การดึงข้อมูล-เสริม-สร้าง"

- **การดึงข้อมูล (Retrieval):** คำถามของผู้ใช้จะถูกแปลงเป็นเวกเตอร์โดยโมเดล Embedding (เช่น BERT) จากนั้นระบบจะดึงบล็อกข้อความที่เกี่ยวข้อง Top-K จากไลบรารีเวกเตอร์ โดยใช้การดึงข้อมูลแบบหนาแน่น (Semantic Similarity) หรือการดึงข้อมูลแบบกระจัดกระจาย (Keyword Matching)
- **การเสริม (Augmentation):** ผลลัพธ์จากการดึงข้อมูลจะถูกนำมารวมกับคำถามต้นฉบับเพื่อสร้าง Prompt ที่เสริมขึ้นมา (Augmented Prompt) ซึ่งจะถูกส่งเข้าไปใน Context Window ของ LLM
- **การสร้าง (Generation):** LLM จะใช้ Augmented Prompt เพื่อสร้างคำตอบสุดท้าย เพื่อให้มั่นใจในความถูกต้องตามข้อเท็จจริงและความสามารถในการตรวจสอบย้อนกลับ
