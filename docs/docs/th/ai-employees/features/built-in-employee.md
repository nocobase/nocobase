# พนักงาน AI ในตัว

NocoBase มาพร้อมกับพนักงาน AI ในตัวที่ออกแบบมาสำหรับสถานการณ์เฉพาะด้านต่างๆ ครับ

คุณเพียงแค่กำหนดค่าบริการ LLM และเปิดใช้งานพนักงานที่ต้องการ ก็สามารถเริ่มทำงานได้ทันที โดยคุณสามารถสลับโมเดลได้ตามความต้องการภายในเซสชันการสนทนาครับ

## พนักงาน AI สำหรับสถานการณ์เฉพาะ

พนักงาน AI ในตัวบางส่วน (ประเภทนักพัฒนา) จะไม่ปรากฏในรายการพนักงาน AI ที่มุมขวาล่าง เนื่องจากพวกเขามีพื้นที่ทำงานเฉพาะเจาะจง เช่น:

* **Orin** จะปรากฏเฉพาะในหน้าการกำหนดค่าแหล่งข้อมูล (Data Source) เท่านั้น
* **Dara** จะปรากฏเฉพาะในหน้าการกำหนดค่าแผนภูมิเท่านั้น
* **Nathan** จะปรากฏเฉพาะในตัวแก้ไข JS (JS editor) เท่านั้น



---

ด้านล่างนี้คือตัวอย่างสถานการณ์การใช้งานทั่วไปของพนักงาน AI เพื่อเป็นแรงบันดาลใจให้แก่คุณ และยังมีศักยภาพอีกมากมายที่รอให้คุณไปสำรวจในการใช้งานจริงครับ


## Viz: นักวิเคราะห์ข้อมูลเชิงลึก

### แนะนำเบื้องต้น

> สร้างแผนภูมิและข้อมูลเชิงลึกได้ในคลิกเดียว ให้ข้อมูลเป็นผู้พูดแทนคุณ

**Viz** คือ **นักวิเคราะห์ข้อมูลเชิงลึก AI** ในตัวครับ
เขาสามารถอ่านข้อมูลในหน้าปัจจุบันของคุณได้ (เช่น Leads, Opportunities, Accounts) และสร้างแผนภูมิแนวโน้ม, แผนภูมิเปรียบเทียบ, การ์ด KPI และข้อสรุปที่กระชับได้โดยอัตโนมัติ ช่วยให้การวิเคราะห์ธุรกิจกลายเป็นเรื่องง่ายและเห็นภาพชัดเจน

> อยากรู้ว่า "ทำไมยอดขายถึงลดลงในช่วงนี้?"
> เพียงแค่บอก Viz คำเดียว เขาสามารถบอกคุณได้ว่ายอดลดลงที่จุดไหน สาเหตุที่เป็นไปได้คืออะไร และขั้นตอนต่อไปที่ควรทำคืออะไรครับ

### สถานการณ์การใช้งาน

ไม่ว่าจะเป็นการสรุปผลการดำเนินงานรายเดือน, ROI ของช่องทางต่างๆ หรือกรวยการขาย (Sales Funnel) คุณสามารถให้ Viz วิเคราะห์ สร้างแผนภูมิ และตีความผลลัพธ์ได้ครับ

| สถานการณ์ | สิ่งที่คุณต้องการทราบ | ผลลัพธ์จาก Viz |
| -------- | ------------ | ------------------- |
| **สรุปผลรายเดือน** | เดือนนี้ดีกว่าเดือนที่แล้วอย่างไร? | การ์ด KPI + แผนภูมิแนวโน้ม + คำแนะนำการปรับปรุง 3 ข้อ |
| **วิเคราะห์การเติบโต** | รายได้ที่เพิ่มขึ้นมาจากปริมาณหรือราคา? | แผนภูมิแยกองค์ประกอบ + ตารางเปรียบเทียบ |
| **วิเคราะห์ช่องทาง** | ช่องทางไหนคุ้มค่าที่จะลงทุนต่อที่สุด? | แผนภูมิ ROI + กราฟการรักษาลูกค้า (Retention) + คำแนะนำ |
| **วิเคราะห์ Funnel** | ทราฟฟิกติดขัดอยู่ที่ขั้นตอนไหน? | แผนภูมิกรวยการขาย + คำอธิบายจุดที่เป็นคอขวด |
| **การรักษาลูกค้า** | ลูกค้ากลุ่มไหนมีมูลค่าสูงสุด? | แผนภูมิแบ่งกลุ่ม RFM + กราฟการรักษาลูกค้า |
| **ประเมินโปรโมชัน** | ผลลัพธ์ของโปรโมชันใหญ่เป็นอย่างไร? | แผนภูมิเปรียบเทียบ + การวิเคราะห์ความยืดหยุ่นของราคา |

### วิธีการใช้งาน

**จุดเข้าใช้งานในหน้าเพจ**

* **ปุ่มมุมขวาบน (แนะนำ)**
  
  ในหน้าเพจ เช่น Leads, Opportunities และ Accounts ให้คลิกที่ **ไอคอน Viz** ที่มุมขวาบน เพื่อเลือกงานที่ตั้งค่าไว้ล่วงหน้า เช่น:

  * การเปลี่ยนผ่านของขั้นตอนและแนวโน้ม (Stage conversion and trends)
  * การเปรียบเทียบช่องทางที่มา (Source channel comparison)
  * การวิเคราะห์สรุปผลรายเดือน (Monthly review analysis)

  ![clipboard-image-1771913319](https://static-docs.nocobase.com/clipboard-image-1771913319.png)

* **แผงควบคุมส่วนกลางมุมขวาล่าง**
  
  ไม่ว่าคุณจะอยู่ที่หน้าใด คุณสามารถเรียกแผง AI ส่วนกลางและพูดคุยกับ Viz ได้โดยตรง:

  ```
  วิเคราะห์การเปลี่ยนแปลงของยอดขายในช่วง 90 วันที่ผ่านมา
  ```

  Viz จะนำบริบทข้อมูลของหน้าปัจจุบันที่คุณอยู่มาใช้ในการวิเคราะห์โดยอัตโนมัติครับ

**รูปแบบการโต้ตอบ**

Viz รองรับคำถามด้วยภาษาธรรมชาติและเข้าใจคำถามต่อเนื่องหลายขั้นตอน
ตัวอย่างเช่น:

```
สวัสดี Viz ช่วยสร้างแนวโน้มของ Leads ในเดือนนี้ให้หน่อย
```

```
ขอดูเฉพาะผลงานจากช่องทางบุคคลที่สาม (Third-party) เท่านั้น
```

```
แล้วภูมิภาคไหนที่มีการเติบโตเร็วที่สุด?
```

การถามต่อเนื่องแต่ละครั้งจะเจาะลึกลงไปโดยอิงจากผลการวิเคราะห์ก่อนหน้า โดยไม่จำเป็นต้องระบุเงื่อนไขข้อมูลซ้ำครับ

### เคล็ดลับในการคุยกับ Viz

| วิธีการ | ผลลัพธ์ |
| ---------- | ------------------- |
| ระบุช่วงเวลาให้ชัดเจน | "30 วันล่าสุด" หรือ "เดือนที่แล้วเทียบกับเดือนนี้" จะให้ผลที่แม่นยำกว่า |
| ระบุมิติข้อมูล (Dimension) | "ดูตามภูมิภาค/ช่องทาง/ผลิตภัณฑ์" ช่วยให้ได้มุมมองที่ตรงจุด |
| เน้นที่แนวโน้มมากกว่ารายละเอียด | Viz เก่งในการระบุทิศทางการเปลี่ยนแปลงและสาเหตุหลัก |
| ใช้ภาษาธรรมชาติ | ไม่จำเป็นต้องใช้ไวยากรณ์คำสั่งที่ซับซ้อน เพียงแค่ถามเหมือนกำลังแชทคุยกันครับ |

## Dex: ผู้เชี่ยวชาญด้านการจัดการข้อมูล

### แนะนำเบื้องต้น

> สกัดข้อมูลและกรอกฟอร์มอย่างรวดเร็ว เปลี่ยนข้อมูลที่ยุ่งเหยิงให้เป็นระเบียบ

`Dex` คือผู้เชี่ยวชาญด้านการจัดการข้อมูลที่สามารถสกัดข้อมูลที่ต้องการจากข้อมูลที่ไม่มีโครงสร้างหรือจากไฟล์ แล้วจัดระเบียบให้เป็นข้อมูลที่มีโครงสร้าง นอกจากนี้เขายังสามารถเรียกใช้เครื่องมือเพื่อกรอกข้อมูลลงในฟอร์มให้คุณได้ด้วยครับ

### วิธีการใช้งาน

เรียกใช้งาน `Dex` ในหน้าฟอร์มเพื่อเปิดหน้าต่างสนทนา

![20251022155316](https://static-docs.nocobase.com/20251022155316.png)

คลิกที่ `Add work context` ในช่องป้อนข้อมูล แล้วเลือก `Pick block` หน้าเพจจะเข้าสู่โหมดการเลือกบล็อกครับ

![20251022155414](https://static-docs.nocobase.com/20251022155414.png)

เลือกบล็อกฟอร์ม (Form block) บนหน้าเพจ

![20251022155523](https://static-docs.nocobase.com/20251022155523.png)

พิมพ์ข้อมูลที่คุณต้องการให้ `Dex` จัดระเบียบลงในช่องสนทนา

![20251022155555](https://static-docs.nocobase.com/20251022155555.png)

หลังจากส่งข้อมูลแล้ว `Dex` จะจัดโครงสร้างข้อมูลและใช้ความสามารถของเขาในการอัปเดตข้อมูลลงในฟอร์มที่เลือกไว้ครับ

![20251022155733](https://static-docs.nocobase.com/20251022155733.png)

## Orin: ผู้เชี่ยวชาญด้านการสร้างแบบจำลองข้อมูล

### แนะนำเบื้องต้น

> ออกแบบคอลเลกชันอย่างชาญฉลาด ปรับปรุงโครงสร้างฐานข้อมูลให้เหมาะสม

`Orin` คือผู้เชี่ยวชาญด้านการสร้างแบบจำลองข้อมูล ในหน้าการกำหนดค่าแหล่งข้อมูลหลัก คุณสามารถให้ `Orin` ช่วยคุณสร้างหรือแก้ไขคอลเลกชันได้ครับ

![20251022160628](https://static-docs.nocobase.com/20251022160628.png)

### วิธีการใช้งาน

เข้าไปที่ปลั๊กอินจัดการแหล่งข้อมูล (Data Source Manager) และเลือกกำหนดค่าแหล่งข้อมูลหลัก

![20251022161146](https://static-docs.nocobase.com/20251022161146.png)

คลิกที่รูปโปรไฟล์ของ `Orin` ที่มุมขวาบนเพื่อเปิดช่องสนทนา AI

![20251022161641](https://static-docs.nocobase.com/20251022161641.png)

อธิบายความต้องการในการสร้างแบบจำลองของคุณให้ `Orin` ฟัง จากนั้นส่งและรอการตอบกลับ 

เมื่อ `Orin` ยืนยันความต้องการของคุณแล้ว เขาจะดำเนินการและส่งตัวอย่างการสร้างแบบจำลองข้อมูลมาให้คุณดู

หลังจากตรวจสอบตัวอย่างแล้ว ให้คลิกปุ่ม `Finish review and apply` เพื่อสร้างคอลเลกชันตามแบบจำลองที่ `Orin` ออกแบบไว้ครับ

![20251022162142](https://static-docs.nocobase.com/20251022162142.png)

## Nathan: วิศวกรฟรอนต์เอนด์

### แนะนำเบื้องต้น

> ช่วยคุณเขียนและปรับแต่งโค้ดฟรอนต์เอนด์ เพื่อสร้างลอจิกการโต้ตอบที่ซับซ้อน

`Nathan` คือผู้เชี่ยวชาญด้านการพัฒนาฟรอนต์เอนด์ใน NocoBase ในสถานการณ์ที่ต้องเขียน JavaScript เช่น `JSBlock`, `JSField`, `JSItem`, `JSColumn`, `JSAction`, `Event Flow` และ `Linkage` จะมีรูปโปรไฟล์ของ `Nathan` ปรากฏที่มุมขวาบนของตัวแก้ไขโค้ด ซึ่งคุณสามารถขอให้เขาช่วยเขียนหรือแก้ไขโค้ดในตัวแก้ไขได้ครับ

![20251022163405](https://static-docs.nocobase.com/20251022163405.png)

### วิธีการใช้งาน

ในตัวแก้ไขโค้ด ให้คลิกที่ `Nathan` เพื่อเปิดช่องสนทนา AI โค้ดในตัวแก้ไขจะถูกแนบไปในช่องป้อนข้อมูลโดยอัตโนมัติเพื่อเป็นบริบทในการทำงานส่งให้ `Nathan` ครับ

![20251022163935](https://static-docs.nocobase.com/20251022163935.png)

พิมพ์ความต้องการในการเขียนโค้ดของคุณ ส่งให้ `Nathan` และรอการตอบกลับ

![20251022164041](https://static-docs.nocobase.com/20251022164041.png)

คลิกปุ่ม `Apply to editor` บนบล็อกโค้ดที่ `Nathan` ตอบกลับมา เพื่อนำโค้ดของเขาไปเขียนทับในตัวแก้ไขโค้ดครับ

![20251022164323](https://static-docs.nocobase.com/20251022164323.png)

คลิกปุ่ม `Run` ในตัวแก้ไขโค้ดเพื่อดูผลลัพธ์แบบเรียลไทม์
