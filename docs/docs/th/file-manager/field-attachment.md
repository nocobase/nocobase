# ฟิลด์ Attachment

## บทนำ

ระบบมีฟิลด์ประเภท "Attachment" ในตัว เพื่อรองรับการอัปโหลดไฟล์ของผู้ใช้ในคอลเลกชันที่กำหนดเองครับ/ค่ะ

ฟิลด์ Attachment โดยพื้นฐานแล้วเป็นฟิลด์ความสัมพันธ์แบบ Many-to-Many ที่ชี้ไปยังคอลเลกชัน "Attachments" (`attachments`) ที่มาพร้อมระบบครับ/ค่ะ เมื่อคุณสร้างฟิลด์ Attachment ในคอลเลกชันใด ๆ ระบบจะสร้างตารางเชื่อมโยง (junction table) สำหรับความสัมพันธ์แบบ Many-to-Many ขึ้นมาโดยอัตโนมัติครับ/ค่ะ เมตาดาต้าของไฟล์ที่อัปโหลดจะถูกเก็บไว้ในคอลเลกชัน "Attachments" และข้อมูลไฟล์ที่อ้างอิงในคอลเลกชันก็จะเชื่อมโยงกันผ่านตารางเชื่อมโยงนี้ครับ/ค่ะ

### การจำกัดประเภท MIME

ใช้สำหรับจำกัดประเภทไฟล์ที่อนุญาตให้อัปโหลด โดยใช้รูปแบบไวยากรณ์ [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types) ครับ/ค่ะ ตัวอย่างเช่น `image/*` หมายถึงไฟล์รูปภาพ คุณสามารถระบุหลายประเภทได้โดยคั่นด้วยเครื่องหมายจุลภาค (comma) เช่น `image/*,application/pdf` หมายถึงอนุญาตทั้งไฟล์รูปภาพและไฟล์ PDF ครับ/ค่ะ

### Storage Engine

เลือก Storage Engine ที่ใช้สำหรับจัดเก็บไฟล์ที่อัปโหลดครับ/ค่ะ หากคุณเว้นว่างไว้ ระบบจะใช้ Storage Engine เริ่มต้นของระบบครับ/ค่ะ
