# ภาพรวม

ฟังก์ชันในตัวของเวิร์กโฟลว์อาจไม่สามารถครอบคลุมได้ทุกสถานการณ์เสมอไปครับ/ค่ะ ตัวอย่างเช่น ประเภทโหนดในตัวอาจไม่สามารถรองรับการทำงานทุกรูปแบบในทุกสถานการณ์ทางธุรกิจได้อย่างครบถ้วน ด้วยเหตุนี้ เราจึงได้ออกแบบให้เวิร์กโฟลว์สามารถขยายการทำงานได้ ซึ่งรวมถึงการขยายประเภททริกเกอร์และประเภทโหนด เพื่อให้ในสถานการณ์ที่ฟังก์ชันในตัวไม่เพียงพอ คุณสามารถแก้ไขปัญหาได้ด้วยการขยายการทำงานโดยใช้ Low-code ครับ/ค่ะ

การขยายการทำงานหลักๆ จะแบ่งออกเป็นสองส่วนครับ/ค่ะ:

- [การขยายประเภททริกเกอร์](./trigger.md)
- [การขยายประเภทโหนด](./node.md)
