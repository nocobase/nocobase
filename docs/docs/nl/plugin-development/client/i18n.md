# I18n

NocoBase plugins ondersteunen meertalige internationalisering (i18n) voor zowel de frontend als de backend. Dankzij een uniform mechanisme kunt u eenvoudig meertalige inhoud implementeren in uw plugins.

## Meertalig Bestandsbeheer

De meertalige bestanden van plugins worden uniform opgeslagen in de map `src/locale`. Het wordt aanbevolen om ze te benoemen op basis van de taal, bijvoorbeeld:

```bash
|- /plugin-hello
  |- /src
    |- /locale
      |- en-US.json   # Engelse taal
      |- zh-CN.json   # Chinese taal
```

Elk taalbestand exporteert een JSON-object dat alle vertaalitems voor die taal bevat, bijvoorbeeld:

```json
// zh-CN.json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

```json
// en-US.json
{
  "Hello": "Hello",
  "World": "World",
  "Enter your name": "Enter your name",
  "Your name is {{name}}": "Your name is {{name}}"
}
```

> Wanneer u voor het eerst taalbestanden toevoegt, moet u de applicatie opnieuw opstarten om ze van kracht te laten worden. U kunt via de API controleren of de vertaalitems correct zijn toegepast:  
> http://localhost:13000/api/app:getLang?locale=zh-CN
