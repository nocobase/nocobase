## 4. Omgevingsvariabelen instellen

De omgevingsvariabelen die NocoBase nodig heeft, staan in het `.env`-bestand in de hoofdmap. Pas de omgevingsvariabelen aan uw specifieke situatie aan.

```bash
TZ=Asia/Shanghai
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning Beschrijving van omgevingsvariabelen

- `TZ` wordt gebruikt om de tijdzone van de applicatie in te stellen. Deze is standaard de tijdzone van het besturingssysteem.
- `APP_KEY` is de geheime sleutel van de applicatie, die wordt gebruikt voor het genereren van gebruikerstokens en dergelijke (als de `APP_KEY` wordt gewijzigd, worden oude tokens ongeldig). Het kan elke willekeurige tekenreeks zijn. Wijzig deze in uw eigen geheime sleutel en zorg ervoor dat deze niet openbaar wordt gemaakt.
- `DB_*` zijn gerelateerd aan de database. Als u niet de standaard databaseservice uit het voorbeeld gebruikt, pas deze dan aan uw specifieke situatie aan.

:::

## 5. NocoBase installeren

```bash
yarn nocobase install --lang=en-US
```

## 6. NocoBase starten

**Ontwikkelomgeving**

```bash
yarn dev
```

**Productieomgeving (niet aanbevolen)**

Het wordt niet aanbevolen om NocoBase direct vanuit de broncode in een productieomgeving te implementeren. Raadpleeg voor productieomgevingen de handleiding voor [Productie-implementatie](../deployment/production.md).

```bash
yarn build
yarn start
```
