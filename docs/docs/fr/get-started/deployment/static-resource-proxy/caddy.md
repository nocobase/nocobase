# Caddy

Si vous cherchez aujourd'hui à configurer le proxy de production pour une application NocoBase, commencez de préférence par [Reverse proxy en production](../../../nocobase-cli/production/reverse-proxy/index.md) puis par la page [Caddy](../../../nocobase-cli/production/reverse-proxy/caddy.md).

Cette section plus ancienne servait surtout de point d'entrée pour le proxy des ressources statiques. La documentation actuelle a été réorganisée autour de `nb proxy caddy`, qui couvre de manière unifiée la génération de configuration, l'exécution locale ou Docker, l'entrée HTTPS et les routes `uploads`, `dist`, `api`, `ws` et SPA.
