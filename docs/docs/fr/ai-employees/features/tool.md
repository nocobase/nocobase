# Utiliser les compétences

Les compétences (outils) définissent ce qu'un employé IA « peut faire ».

## Structure des compétences

La page des compétences est divisée en trois sections :

1. `General skills` : partagées par tous les employés IA, en lecture seule.
2. `Employee-specific skills` : propres à l'employé actuel, généralement en lecture seule.
3. `Custom skills` : compétences personnalisées pouvant être ajoutées/supprimées et configurées avec des permissions par défaut.

![skills-three-sections-general-specific-custom.png](https://static-docs.nocobase.com/ai-employees/2026-02-14/skills-three-sections-general-specific-custom.png)

## Permissions des compétences

Les permissions des compétences sont unifiées comme suit :

- `Ask` : demande une confirmation avant l'exécution.
- `Allow` : permet l'exécution directe.

Recommandation : utilisez `Ask` par défaut pour les compétences impliquant la modification de données.

![skills-permission-ask-allow-segmented.png](https://static-docs.nocobase.com/ai-employees/2026-02-14/skills-permission-ask-allow-segmented.png)

## Ajout et maintenance

Cliquez sur `Add skill` dans `Custom skills` pour ajouter des compétences et configurer les permissions en fonction des risques métier.
