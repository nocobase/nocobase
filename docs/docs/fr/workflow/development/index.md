# Présentation

Les fonctionnalités intégrées de nos **flux de travail** ne peuvent pas toujours couvrir l'intégralité des scénarios. Par exemple, les types de nœuds fournis ne peuvent pas énumérer chaque opération possible dans tous les contextes métier. C'est pourquoi nous avons prévu une architecture permettant d'étendre les **flux de travail**, notamment en ajoutant de nouveaux types de déclencheurs et de nœuds. Lorsque les fonctionnalités intégrées ne suffisent pas, vous avez la possibilité de les enrichir via des méthodes low-code.

Les extensions se divisent principalement en deux parties :

- [Étendre les types de déclencheurs](./trigger.md)
- [Étendre les types de nœuds](./node.md)
