# CRM Hérité (v1)

> **Aperçu uniquement** : Cette version est relativement complète en termes de fonctionnalités, mais la conception globale de la structure des données n'est pas optimale, elle manque de support pour la localisation multilingue et présente un couplage entre les collections CRM et celles de gestion de projet. Ces problèmes ont conduit à la refonte du CRM 2.0. Il est actuellement recommandé d'utiliser le [CRM 2.0](./index.md), qui bénéficie d'une structure de données plus rationnelle, est plus facile à étendre et à modifier, et dispose d'une localisation multilingue complète.

## Caractéristiques de la version

Le CRM hérité comprend les modules fonctionnels suivants :

- **Gestion des clients** : Informations clients, contacts et classification des clients.
- **Pipeline de vente** : Gestion des prospects, suivi des opportunités et entonnoir de vente.
- **Gestion des commandes** : Devis, commandes et contrats.
- **Gestion de projet** (non incluse pour le moment dans la version v2).
- **Communication et collaboration** : Échanges par e-mail, commentaires et flux de travail d'approbation.
- **Analyse de données** : Tableaux de bord, rapports et visualisation de données.

## Comment installer

> La version actuelle est déployée via un système de sauvegarde et de restauration.

Avant de commencer, veuillez vous assurer que :

- Vous disposez d'un environnement d'exécution NocoBase de base. Pour l'installation du système principal, veuillez vous référer à la [documentation officielle d'installation](https://docs.nocobase.com/welcome/getting-started/installation).
- La version de NocoBase est **v2.1.0-beta.2 ou supérieure**.
- Vous avez téléchargé les fichiers correspondants au CRM v1 :
  - **Fichier de sauvegarde** : [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — Applicable à la Méthode 1.
  - **Fichier SQL** : [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — Applicable à la Méthode 2.

**Remarques importantes** :
- Cette solution est basée sur la base de données **PostgreSQL 16**. Veuillez vous assurer que votre environnement utilise PostgreSQL 16.
- **DB_UNDERSCORED ne doit pas être défini sur true** : Veuillez vérifier votre fichier `docker-compose.yml` et vous assurer que la variable d'environnement `DB_UNDERSCORED` n'est pas définie sur `true`. Sinon, cela entrera en conflit avec la sauvegarde et entraînera l'échec de la restauration.

## Méthode 1 : Restaurer à l'aide du Gestionnaire de sauvegardes (Recommandé pour les utilisateurs Pro/Entreprise)

Cette méthode utilise le plugin intégré "**Gestionnaire de sauvegardes**" (Pro/Entreprise) pour une restauration en un clic, ce qui est l'approche la plus simple.

### Étapes

**Étape 1 : Activer le plugin "Gestionnaire de sauvegardes"**

1. Connectez-vous à votre système NocoBase.
2. Allez dans le **Gestionnaire de plugins**.
3. Recherchez et activez le plugin **Gestionnaire de sauvegardes**.

**Étape 2 : Restaurer à partir du fichier de sauvegarde local**

1. Après avoir activé le plugin, rafraîchissez la page.
2. Allez dans **Administration du système** -> **Gestionnaire de sauvegardes** dans le menu de gauche.
3. Cliquez sur le bouton **Restaurer à partir d'une sauvegarde locale** en haut à droite.
4. Glissez et déposez le fichier de sauvegarde téléchargé dans la zone de téléchargement.
5. Cliquez sur **Soumettre** et attendez que le système termine la restauration.

### Remarques

* **Édition Pro/Entreprise uniquement** : Le "Gestionnaire de sauvegardes" est un plugin de niveau entreprise et n'est disponible que pour les utilisateurs Pro/Entreprise.
* **Compatibilité des plugins commerciaux** : Veuillez vous assurer que vous avez obtenu et activé les plugins commerciaux requis pour cette solution.

## Méthode 2 : Importer directement le fichier SQL (Général)

Cette méthode restaure les données en opérant directement sur la base de données et s'applique à tous les utilisateurs de NocoBase.

### Étapes

**Étape 1 : Préparer une base de données propre**

Préparez une base de données vide et neuve pour les données que vous allez importer.

**Étape 2 : Importer le fichier .sql dans la base de données**

* **Via la ligne de commande (Exemple Docker)** :

  ```bash
  # Copier le fichier sql dans le conteneur
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # Entrer dans le conteneur et exécuter la commande d'importation
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Via un client de base de données** : Utilisez des outils tels que DBeaver, Navicat ou pgAdmin pour vous connecter à la base de données et exécuter le fichier SQL.

**Étape 3 : Connecter la base de données et démarrer l'application**

Configurez les paramètres de démarrage de NocoBase pour pointer vers la base de données où les données ont été importées, puis démarrez le service.
