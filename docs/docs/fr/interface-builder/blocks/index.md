# Blocs

Les blocs sont des conteneurs pour vos données et contenus. Vous pouvez les placer sur une Page, dans une Modale (boîte de dialogue) ou un Tiroir. De plus, vous pouvez organiser plusieurs blocs en les déplaçant librement par glisser-déposer.

## Types de blocs

![Types de blocs](https://static-docs.nocobase.com/f71af45b5cd914ea0558f760ddbbba58.png)

- **Bloc de données :** Permet d'afficher les données d'une [source de données](#) sur l'interface.
- **Bloc de filtre :** Utilise les données d'une [source de données](#) comme critères de filtre pour filtrer d'autres blocs de données.
- **Autres blocs :** Permet d'afficher des contenus spécifiques ou indépendants, tels que les tâches en attente d'un [flux de travail](#), les journaux d'audit, ou du Markdown.

## Ajouter un bloc

Vous pouvez placer des blocs sur une Page, dans une Modale (boîte de dialogue) ou un Tiroir.

### Blocs sur une Page

Actuellement, les types de blocs disponibles sur une page sont : le bloc de données, le bloc de filtre et les autres blocs.

![20251023222441](https://static-docs.nocobase.com/20251023222441.png)

### Blocs dans une fenêtre contextuelle (Modale ou Tiroir)

Les fenêtres contextuelles se présentent sous deux formes : les modales et les tiroirs. Comme pour les pages, vous pouvez y ajouter des blocs. La différence est que les blocs de formulaire dans les fenêtres contextuelles sont généralement utilisés pour ajouter, modifier ou afficher une seule entrée. Les types de blocs incluent le bloc de données et les autres blocs.

![20251023222613](https://static-docs.nocobase.com/20251023222613.png)

## Le concepteur de blocs

Chaque bloc dispose de trois petites icônes dans le coin supérieur droit. De gauche à droite, elles représentent :

1. Disposition par glisser-déposer
2. [Flux d'événements](/interface-builder/event-flow)
3. Paramètres du bloc

![20251023224032](https://static-docs.nocobase.com/20251023224032.png)

Pour les blocs simples, toutes les options de configuration sont centralisées dans les « Paramètres du bloc », comme pour le bloc JS.

![20251023224903](https://static-docs.nocobase.com/20251023224903.png)

Les blocs de données complexes offrent également des options intégrées distinctes pour « Configurer les champs » et « Configurer les actions ».

![20251023225141](https://static-docs.nocobase.com/20251023225141.png)

De plus, vous avez la liberté d'explorer davantage de possibilités d'imbrication, comme le montre le bloc de graphique.

![](https://static-docs.nocobase.com/07588190b3f41ae3060e71d8b76b4447.png)
