# Champ

Dans l'interface utilisateur, un champ est un composant qui sert de support aux données unitaires. Chaque type de donnée est présenté par un composant de champ distinct. Les champs ne peuvent être attachés qu'à des blocs et ne peuvent pas être utilisés indépendamment.

## Les champs dans un bloc

Les champs ne sont généralement pas utilisés seuls, mais existent en tant qu'éléments enfants des blocs de type de données. Les blocs de type de données disposent généralement d'une option « Configurer les champs », et la liste des champs est fournie par la collection actuelle.

![20251029204240](https://static-docs.nocobase.com/20251029204240.png)

## Le concepteur de champ (barre d'outils)

Comme les blocs, le coin supérieur droit d'un composant de champ comporte également trois icônes, qui sont :

- Glisser-déposer pour la disposition
- [Flux d'événements](/interface-builder/event-flow)
- Configuration

![20251025165432](https://static-docs.nocobase.com/20251025165432.png)

## La disposition des champs

Vous pouvez personnaliser la disposition des champs au sein d'un bloc en les glissant-déposant.

![20251025165605](https://static-docs.nocobase.com/20251025165605.gif)

## Options de configuration des champs

Outre les options de configuration générales des champs, différents types de champs prennent en charge des options de configuration spécifiques.
