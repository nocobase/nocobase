# Mode d'affichage

## Introduction

Contrairement aux blocs, les composants de champ disposent de trois modes d'affichage (uniquement pris en charge pour les champs au sein d'un formulaire). Le passage d'un mode d'affichage à l'autre entraîne des options de configuration de champ différentes.

- Modifiable ;
- Lecture seule (non modifiable) ;
- Lecture seule (mode lecture) ;

### Modes d'affichage pour les champs de relation

Le **mode d'affichage d'un champ de relation** détermine la manière dont le champ est présenté dans l'interface, ainsi que les types de composants de champ disponibles.

En **état modifiable**, les champs de relation prennent en charge plusieurs types de composants. Vous pouvez ainsi choisir différents composants de champ de relation pour afficher ou sélectionner les données associées en fonction de vos besoins métier.

#### Composants de champ de relation en état modifiable

![20251028220447](https://static-docs.nocobase.com/20251028220447.png)

Dans cet état, vous pouvez choisir de manière flexible la méthode d'affichage appropriée pour traiter les données plus efficacement.

#### Composants de champ de relation en état de lecture

Lorsque vous passez en **état de lecture**, le système utilise automatiquement par défaut le **composant de champ de titre** pour afficher les données associées. Cela convient aux scénarios où vous avez uniquement besoin de consulter les données sans les modifier.
