# Blöcke

Blöcke sind Träger für Daten und Inhalte. Sie können auf einer Seite (Page), in einem Dialogfeld (Modal) oder in einem Schubladenmenü (Drawer) platziert werden. Mehrere Blöcke lassen sich frei per Drag & Drop anordnen.

## Block-Typen

![Block-Typen](https://static-docs.nocobase.com/f71af45b5cd914ea0558f760ddbbba58.png)

- Datenblock: Dient dazu, Daten aus einer Datenquelle auf der Benutzeroberfläche anzuzeigen.
- Filterblock: Ermöglicht es, Daten aus einer Datenquelle als Filterkriterien zu verwenden, um andere Datenblöcke zu filtern.
- Sonstige Blöcke: Diese Blöcke dienen dazu, spezifische oder unabhängige Inhalte wie Workflow-Aufgaben, Audit-Logs oder Markdown darzustellen.

## Block hinzufügen

Blöcke können auf einer Seite (Page), in einem Dialogfeld (Modal) oder in einem Schubladenmenü (Drawer) platziert werden.

### Blöcke auf einer Seite

Aktuell umfassen die Block-Typen auf einer Seite: Datenblöcke, Filterblöcke und Sonstige Blöcke.

![20251023222441](https://static-docs.nocobase.com/20251023222441.png)

### Blöcke in einem Pop-up (Dialogfeld oder Schubladenmenü)

Pop-ups gibt es in zwei Varianten: als Dialogfeld (Modal) und als Schubladenmenü (Drawer). Ähnlich wie bei Seiten können Sie auch hier Blöcke hinzufügen. Der Unterschied besteht darin, dass Formularblöcke in Pop-ups in der Regel für das Hinzufügen, Bearbeiten oder Anzeigen eines einzelnen Datensatzes verwendet werden. Die Block-Typen umfassen Datenblöcke und Sonstige Blöcke.

![20251023222613](https://static-docs.nocobase.com/20251023222613.png)

## Der Block-Designer

Jeder Block verfügt oben rechts über drei kleine Symbole, die von links nach rechts folgende Funktionen darstellen:

1. Drag & Drop-Layout
2. [Ereignisfluss](/interface-builder/event-flow)
3. Block-Einstellungen

![20251023224032](https://static-docs.nocobase.com/20251023224032.png)

Bei einfachen Blöcken sind alle Konfigurationsoptionen in den „Block-Einstellungen“ zusammengefasst, zum Beispiel beim JS Block.

![20251023224903](https://static-docs.nocobase.com/20251023224903.png)

Komplexe Datenblöcke bieten zusätzlich separate, eingebettete Optionen zum „Konfigurieren von Feldern“ und „Konfigurieren von Aktionen“.

![20251023225141](https://static-docs.nocobase.com/20251023225141.png)

Darüber hinaus gibt es flexible Möglichkeiten für weitere Verschachtelungen, wie zum Beispiel beim Diagramm-Block.
