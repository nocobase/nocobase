# Caddy

Wenn Sie heute den Produktions-Proxy für eine NocoBase-Anwendung einrichten möchten, beginnen Sie am besten mit [Reverse Proxy in Produktion](../../../nocobase-cli/production/reverse-proxy/index.md) und anschließend mit der Seite [Caddy](../../../nocobase-cli/production/reverse-proxy/caddy.md).

Dieser ältere Abschnitt diente vor allem als Einstieg für den Proxy statischer Ressourcen. Die aktuelle Dokumentation wurde rund um `nb proxy caddy` neu organisiert und deckt die Konfigurationsgenerierung, den lokalen oder Docker-Betrieb, den HTTPS-Einstieg sowie die Routen `uploads`, `dist`, `api`, `ws` und SPA einheitlich ab.
