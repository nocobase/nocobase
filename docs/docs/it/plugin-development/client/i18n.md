# I18n Internazionalizzazione

I plugin di NocoBase supportano l'internazionalizzazione (i18n) multilingue sia per il frontend che per il backend. Grazie a un meccanismo unificato, può implementare facilmente contenuti multilingue nei suoi plugin.

## Gestione dei file multilingue

I file multilingue dei plugin sono archiviati in modo uniforme nella directory `src/locale`. Si consiglia di nominarli in base alla lingua, ad esempio:

```bash
|- /plugin-hello
  |- /src
    |- /locale
      |- en-US.json   # Lingua inglese
      |- zh-CN.json   # Lingua cinese
```

Ogni file di lingua esporta un oggetto JSON che contiene tutte le voci di traduzione per quella lingua, ad esempio:

```json
// zh-CN.json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

```json
// en-US.json
{
  "Hello": "Hello",
  "World": "World",
  "Enter your name": "Enter your name",
  "Your name is {{name}}": "Your name is {{name}}"
}
```

> Quando aggiunge i file di lingua per la prima volta, deve riavviare l'applicazione affinché abbiano effetto. Può verificare che le voci di traduzione siano state applicate tramite l'API:  
> http://localhost:13000/api/app:getLang?locale=zh-CN
