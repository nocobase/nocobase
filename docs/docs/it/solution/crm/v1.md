# CRM Legacy (v1)

> **Solo per anteprima**: Questa versione è relativamente completa in termini di funzionalità, ma il design complessivo della struttura dei dati non è ottimale, manca il supporto per la localizzazione multilingue e le collezioni del CRM sono accoppiate con quelle della gestione progetti. Questi problemi hanno portato alla riprogettazione di CRM 2.0. Attualmente si consiglia di utilizzare la [versione CRM 2.0](./index.md), che presenta una struttura dei dati più razionale, è più facile da estendere e modificare ed è già stata localizzata in più lingue.

## Caratteristiche della versione

La versione legacy del CRM include i seguenti moduli funzionali:

- **Gestione Clienti**: informazioni sui clienti, contatti e classificazione dei clienti.
- **Pipeline di Vendita**: gestione dei lead, monitoraggio delle opportunità e funnel di vendita.
- **Gestione Ordini**: preventivi, ordini e contratti.
- **Gestione Progetti** (non ancora inclusa nella versione v2).
- **Comunicazione e Collaborazione**: comunicazioni via email, commenti e flussi di lavoro di approvazione.
- **Analisi dei Dati**: dashboard, report e visualizzazione dei dati.

## Come installare

> L'attuale versione viene distribuita tramite backup e ripristino.

Prima di iniziare, si assicuri che:

- Disponga di un ambiente di runtime NocoBase di base. Per l'installazione del sistema principale, consulti la [Documentazione ufficiale di installazione](https://docs.nocobase.com/welcome/getting-started/installation) dettagliata.
- La versione di NocoBase sia la **v2.1.0-beta.2 o superiore**.
- Abbia scaricato i file corrispondenti per CRM v1:
  - **File di backup**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — Applicabile al Metodo 1.
  - **File SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — Applicabile al Metodo 2.

**Note importanti**:
- Questa soluzione si basa sul database **PostgreSQL 16**. Si assicuri che il Suo ambiente utilizzi PostgreSQL 16.
- **DB_UNDERSCORED non deve essere impostato su true**: Controlli il Suo file `docker-compose.yml` e si assicuri che la variabile d'ambiente `DB_UNDERSCORED` non sia impostata su `true`. In caso contrario, entrerà in conflitto con il backup e causerà il fallimento del ripristino.

## Metodo 1: Ripristino tramite Gestore Backup (consigliato per utenti Pro/Enterprise)

Questo metodo utilizza il plugin integrato "**Gestore Backup**" (Pro/Enterprise) per un ripristino con un solo clic, rappresentando l'approccio più semplice.

### Passaggi

**Passaggio 1: Abilitare il plugin "Gestore Backup"**

1. Acceda al Suo sistema NocoBase.
2. Vada in **`Gestione plugin`**.
3. Trovi e abiliti il plugin **`Gestore Backup`**.

**Passaggio 2: Ripristino da un file di backup locale**

1. Dopo aver abilitato il plugin, aggiorni la pagina.
2. Vada su **`Amministrazione di sistema`** -> **`Gestore Backup`** nel menu a sinistra.
3. Clicchi sul pulsante **`Ripristina da backup locale`** in alto a destra.
4. Trascini e rilasci il file di backup scaricato nell'area di caricamento.
5. Clicchi su **`Invia`** e attenda che il sistema completi il ripristino.

### Note

* **Solo per edizioni Pro/Enterprise**: "Gestore Backup" è un plugin di livello enterprise ed è disponibile solo per gli utenti Pro/Enterprise.
* **Compatibilità dei plugin commerciali**: Si assicuri di aver ottenuto e abilitato i plugin commerciali richiesti per questa soluzione.

## Metodo 2: Importazione diretta del file SQL (Generale)

Questo metodo ripristina i dati operando direttamente sul database ed è applicabile a tutti gli utenti NocoBase.

### Passaggi

**Passaggio 1: Preparare un database pulito**

Prepari un database nuovo e vuoto per i dati che sta per importare.

**Passaggio 2: Importare il file `.sql` nel database**

* **Tramite riga di comando (Esempio Docker)**:

  ```bash
  # Copia il file sql all'interno del container
  docker cp nocobase_crm_v1_sql_260224.sql mio-db-nocobase:/tmp/
  # Entra nel container ed esegue il comando di importazione
  docker exec -it mio-db-nocobase psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Tramite client database**: Utilizzi strumenti come DBeaver, Navicat o pgAdmin per connettersi al database ed eseguire il file SQL.

**Passaggio 3: Connettersi al database e avviare l'applicazione**

Configuri i parametri di avvio di NocoBase in modo che puntino al database in cui sono stati importati i dati, quindi avvii il servizio.
