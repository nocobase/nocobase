# Синхронізація даних користувачів

## Вступ

Ця функція дозволяє реєструвати та керувати джерелами синхронізації даних користувачів. За замовчуванням надається HTTP API, але інші джерела даних можна підтримувати за допомогою **плагінів**. Вона підтримує синхронізацію даних до **колекцій** "Користувачі" та "Відділи" за замовчуванням, з можливістю розширення синхронізації на інші цільові ресурси за допомогою **плагінів**.

## Керування джерелами даних та синхронізація

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
Якщо не встановлено **плагінів**, що надають джерела синхронізації даних користувачів, ви можете синхронізувати дані користувачів за допомогою HTTP API. Дивіться [Джерело даних - HTTP API](./sources/api.md).
:::

## Додавання джерела даних

Після встановлення **плагіна**, який надає джерело синхронізації даних користувачів, ви можете додати відповідне **джерело даних**. Лише увімкнені **джерела даних** відображатимуть кнопки "Синхронізувати" та "Завдання".

> Приклад: WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## Синхронізація даних

Натисніть кнопку **Синхронізувати**, щоб розпочати синхронізацію даних.

![](https://static-docs.nocobase.com/202412041055022.png)

Натисніть кнопку **Завдання**, щоб переглянути статус синхронізації. Після успішної синхронізації ви можете переглянути дані у списках користувачів та відділів.

![](https://static-docs.nocobase.com/202412041202337.png)

Для невдалих завдань синхронізації ви можете натиснути **Повторити**.

![](https://static-docs.nocobase.com/202412041058337.png)

У разі невдалої синхронізації ви можете усунути проблему за допомогою системних журналів. Крім того, вихідні записи синхронізації зберігаються в каталозі `user-data-sync` у папці журналів програми.

![](https://static-docs.nocobase.com/202412041205655.png)
