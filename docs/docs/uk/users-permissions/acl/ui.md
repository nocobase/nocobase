# Застосування в інтерфейсі користувача

## Дозволи для блоків даних

Видимість блоків даних у колекції контролюється дозволами на операцію перегляду. Індивідуальні налаштування мають пріоритет над глобальними.

Наприклад, за глобальних дозволів роль «admin» має повний доступ, але для колекції «Замовлення» можуть бути налаштовані індивідуальні дозволи, що робить її невидимою.

Налаштування глобальних дозволів:

![](https://static-docs.nocobase.com/3d026311739c7cf5fdcd03f710d09bc4.png)

Налаштування індивідуальних дозволів для колекції «Замовлення»:

![](https://static-docs.nocobase.com/a88caba1cad47001c1610bf402a4a2c1.png)

В інтерфейсі користувача всі блоки колекції «Замовлення» не відображаються.

Повний процес налаштування:

![](https://static-docs.nocobase.com/b283c004ffe0b746fddbffcf4f27b1df.gif)

## Дозволи для полів

**Перегляд**: Визначає, чи є певні поля видимими на рівні поля, дозволяючи контролювати, які поля доступні для перегляду певним ролям у колекції «Замовлення».

![](https://static-docs.nocobase.com/30dea84d984d95039e6f7b180955a6cf.png)

В інтерфейсі користувача в блоці колекції «Замовлення» відображаються лише поля з налаштованими дозволами. Системні поля (Id, CreatedAt, LastUpdatedAt) зберігають дозволи на перегляд навіть без спеціальних налаштувань.

![](https://static-docs.nocobase.com/40cc49b517efe701147fd2e799e79dcc.png)

- **Редагування**: Контролює, чи можна редагувати та зберігати (оновлювати) поля.

  Налаштуйте дозволи на редагування для полів колекції «Замовлення» (кількість та пов'язані товари мають дозволи на редагування):

  ![](https://static-docs.nocobase.com/6531ca4122f0887547b5719e2146ba93.png)

  В інтерфейсі користувача в блоці форми операції редагування колекції «Замовлення» відображаються лише поля з дозволами на редагування.

  ![](https://static-docs.nocobase.com/12982450c311ec1bf87eb9dc5fb04650.png)

  Повний процес налаштування:

  ![](https://static-docs.nocobase.com/1dbe559a92c052e194e50edc74a7.gif)

- **Додавання**: Визначає, чи можна додавати (створювати) поля.

  Налаштуйте дозволи на додавання для полів колекції «Замовлення» (номер замовлення, кількість, товари та відправлення мають дозволи на додавання):

  ![](https://static-docs.nocobase.com/3ab1bbe41e61915e920fd257f2e0da7e.png)

  В інтерфейсі користувача в блоці форми операції додавання колекції «Замовлення» відображаються лише поля з дозволами на додавання.

  ![](https://static-docs.nocobase.com/8d0c07893b63771c428974f9e126bf35.png)

- **Експорт**: Контролює, чи можна експортувати поля.
- **Імпорт**: Контролює, чи підтримують поля імпорт.

## Дозволи для операцій

Індивідуально налаштовані дозволи мають найвищий пріоритет. Якщо налаштовані специфічні дозволи, вони замінюють глобальні налаштування; в іншому випадку застосовуються глобальні налаштування.

- **Додавання**: Контролює, чи відображається кнопка операції додавання в блоці.

  Налаштуйте індивідуальні дозволи для операцій колекції «Замовлення», дозволяючи додавання:

  ![](https://static-docs.nocobase.com/2e3123b5dbc72ae78942481360626629.png)

  Коли операція додавання дозволена, кнопка «Додати» з'являється в області дій блоку колекції «Замовлення» в інтерфейсі користувача.

  ![](https://static-docs.nocobase.com/f0458980d450544d94c73160d75ba96c.png)

- **Перегляд**: Визначає, чи є блок даних видимим.

  Налаштування глобальних дозволів (без дозволу на перегляд):

  ![](https://static-docs.nocobase.com/6e4a1e6ea92f50bf84959dedbf1d5683.png)

  Індивідуальні налаштування дозволів для колекції «Замовлення»:

  ![](https://static-docs.nocobase.com/f2dd142a40fe19fb657071fd901b2291.png)

  В інтерфейсі користувача блоки даних для всіх інших колекцій залишаються прихованими, але блок колекції «Замовлення» відображається.

  Повний приклад процесу налаштування:

  ![](https://static-docs.nocobase.com/b92f0edc51a27b52e85cdeb76271b936.gif)

- **Редагування**: Контролює, чи відображається кнопка операції редагування в блоці.

  ![](https://static-docs.nocobase.com/fb1c0290e2a833f1c2b415c761e54c45.gif)

  Дозволи для операцій можна додатково уточнити, встановивши область даних.

  Наприклад, налаштування колекції «Замовлення» таким чином, щоб користувачі могли редагувати лише власні дані:

  ![](https://static-docs.nocobase.com/b082308f62a3a9084cab78a370c14a9f.gif)

- **Видалення**: Контролює, чи відображається кнопка операції видалення в блоці.

  ![](https://static-docs.nocobase.com/021c9e79bcc1ad221b606a9555ff5644.gif)

- **Експорт**: Контролює, чи відображається кнопка операції експорту в блоці.

- **Імпорт**: Контролює, чи відображається кнопка операції імпорту в блоці.

## Дозволи для зв'язків

### Як поле

- Дозволи поля зв'язку контролюються дозволами полів вихідної колекції. Це визначає, чи відображається весь компонент поля зв'язку.

Наприклад, у колекції «Замовлення» поле зв'язку «Клієнт» має лише дозволи на перегляд, імпорт та експорт.

![](https://static-docs.nocobase.com/d0dc797aae73feeabc436af285dd4f59.png)

В інтерфейсі користувача це означає, що поле зв'язку «Клієнт» не відображатиметься в блоках операцій додавання та редагування колекції «Замовлення».

Повний приклад процесу налаштування:

![](https://static-docs.nocobase.com/372f8a4f414feea097c23b2ba326c0ef.gif)

- Дозволи для полів у компоненті поля зв'язку (наприклад, підтаблиці або підформи) визначаються дозволами цільової колекції.

Коли компонент поля зв'язку є підформою:

Як показано нижче, поле зв'язку «Клієнт» у колекції «Замовлення» має всі дозволи, тоді як сама колекція «Клієнти» налаштована як «лише для читання».

Індивідуальні налаштування дозволів для колекції «Замовлення», де поле зв'язку «Клієнт» має всі дозволи для полів:

![](https://static-docs.nocobase.com/3a3ab9722f14a7b3a35361219d67fa40.png)

Індивідуальні налаштування дозволів для колекції «Клієнти», де поля мають дозволи лише на перегляд:

![](https://static-docs.nocobase.com/46704d179b931006a9a22852e6c5089e.png)

В інтерфейсі користувача поле зв'язку «Клієнт» видиме в блоці колекції «Замовлення». Однак, при перемиканні на підформу, поля в підформі видимі в детальному перегляді, але не відображаються в операціях додавання та редагування.

Повний приклад процесу налаштування:

![](https://static-docs.nocobase.com/932dbf6ac46e36ee357ff3e8b9ea1423.gif)

Щоб додатково контролювати дозволи для полів у підформі, ви можете надати дозволи окремим полям.

Як показано, колекція «Клієнти» налаштована з індивідуальними дозволами для полів (Ім'я клієнта невидиме та недоступне для редагування).

![](https://static-docs.nocobase.com/e7b875521cbc4e28640f027f36d0413c.png)

Повний приклад процесу налаштування:

![](https://static-docs.nocobase.com/7a07e68c2fe2a13f0c2cef19be489264.gif)

Коли компонент поля зв'язку є підтаблицею, ситуація відповідає ситуації з підформою:

Як показано, поле зв'язку «Відправлення» в колекції «Замовлення» має всі дозволи, тоді як колекція «Відправлення» налаштована як «лише для читання».

В інтерфейсі користувача це поле зв'язку видиме. Однак, при перемиканні на підтаблицю, поля в підтаблиці видимі в операції перегляду, але не в операціях додавання та редагування.

![](https://static-docs.nocobase.com/fd4b7d81cdd765db789d9c85cf9dc324.gif)

Щоб додатково контролювати дозволи для полів у підтаблиці, ви можете надати дозволи окремим полям:

![](https://static-docs.nocobase.com/51d70a624cb2b0366e421bcdc8abb7fd.gif)

### Як блок

- Видимість блоку зв'язку контролюється дозволами цільової колекції відповідного поля зв'язку і не залежить від дозволів самого поля зв'язку.

Наприклад, відображення блоку зв'язку «Клієнт» контролюється дозволами колекції «Клієнти».

![](https://static-docs.nocobase.com/633ebb301767430b740ecfce11df47b3.gif)

- Поля в блоці зв'язку контролюються дозволами полів у цільовій колекції.

Як показано, ви можете встановити дозволи на перегляд для окремих полів у колекції «Клієнти».

![](https://static-docs.nocobase.com/35af9426c20911323b17f67f81bac8fc.gif)
