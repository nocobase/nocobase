# Підключення зовнішніх таблиць даних (FDW)

## Вступ

Функція підключення до віддалених таблиць даних, реалізована на основі Foreign Data Wrapper (FDW) бази даних. Наразі підтримуються бази даних MySQL та PostgreSQL.

:::info{title="Підключення джерел даних vs Підключення зовнішніх таблиць даних"}
- **Підключення джерел даних** означає встановлення зв'язку з конкретною базою даних або API-сервісом, що дозволяє повноцінно використовувати можливості бази даних або послуги, що надаються API;
- **Підключення зовнішніх таблиць даних** означає отримання даних ззовні та їх відображення для локального використання. У базах даних це називається FDW (Foreign Data Wrapper) — технологія, яка фокусується на використанні віддалених таблиць як локальних і дозволяє підключати лише одну таблицю за раз. Оскільки це віддалений доступ, при його використанні існують певні обмеження та ліміти.

Обидва підходи можна використовувати разом. Перший використовується для встановлення з'єднання з джерелом даних, а другий — для доступу до даних між різними джерелами. Наприклад, якщо підключено певне джерело даних PostgreSQL, і в цьому джерелі є таблиця, створена як зовнішня на основі FDW.
:::

### MySQL

MySQL використовує рушій `federated`, який необхідно активувати. Він підтримує підключення до віддалених баз даних MySQL та сумісних за протоколом баз даних, таких як MariaDB. Детальнішу документацію див. у [Federated Storage Engine](https://dev.mysql.com/doc/refman/8.0/en/federated-storage-engine.html).

### PostgreSQL

У PostgreSQL для підтримки різних типів віддалених даних можна використовувати різні типи розширень `fdw`. Наразі підтримуються такі розширення:

- [postgres_fdw](https://www.postgresql.org/docs/current/postgres-fdw.html): Підключення до віддаленої бази даних PostgreSQL у PostgreSQL.
- [mysql_fdw](https://github.com/EnterpriseDB/mysql_fdw): Підключення до віддаленої бази даних MySQL у PostgreSQL.
- Для інших типів розширень fdw зверніться до [PostgreSQL Foreign Data Wrappers](https://wiki.postgresql.org/wiki/Foreign_data_wrappers). Для інтеграції в NocoBase необхідно реалізувати відповідний інтерфейс адаптації в коді.

![Підключення зовнішніх даних](https://static-docs.nocobase.com/029d946a6d067d1c35a39755219d623c.png)

![Сервіс бази даних](https://static-docs.nocobase.com/766271708a911950a5599d60d6be4a4d.png)
