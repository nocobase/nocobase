# Огляд

## Вступ

Рушії зберігання використовуються для збереження файлів у певних службах, таких як локальне сховище (на жорсткому диску сервера), хмарне сховище тощо.

Перш ніж завантажувати будь-які файли, вам потрібно налаштувати рушій зберігання. Під час встановлення система автоматично додає рушій локального сховища, який можна використовувати одразу. Ви також можете додати нові рушії або редагувати параметри існуючих.

## Типи рушіїв зберігання

Наразі NocoBase має вбудовану підтримку таких типів рушіїв:

- [Локальне сховище](./local)
- [Amazon S3](./amazon-s3)
- [Aliyun OSS](./aliyun-oss)
- [Tencent COS](./tencent-cos)
- [S3 Pro](./s3-pro)

Під час встановлення система автоматично додає рушій локального сховища, який можна використовувати одразу. Ви також можете додати нові рушії або редагувати параметри існуючих.

## Загальні параметри

Окрім специфічних параметрів для різних типів рушіїв, нижче наведено загальні параметри (на прикладі локального сховища):

![Приклад конфігурації рушія зберігання файлів](https://static-docs.nocobase.com/20240529115151.png)

### Назва

Назва рушія зберігання, що використовується для ідентифікації людиною.

### Системна назва

Системна назва рушія зберігання, що використовується для ідентифікації системою. Вона має бути унікальною в межах системи. Якщо поле залишити порожнім, система автоматично згенерує випадкову назву.

### Префікс публічної URL-адреси

Префіксна частина публічно доступної URL-адреси файлу. Це може бути базова URL-адреса CDN, наприклад: "`https://cdn.nocobase.com/app`" (без кінцевого символу "/").

### Шлях

Відносний шлях, що використовується при збереженні файлів. Ця частина також буде автоматично додана до кінцевої URL-адреси під час доступу. Наприклад: "`user/avatar`" (без початкового або кінцевого символу "/").

### Обмеження розміру файлу

Обмеження розміру файлів, що завантажуються до цього рушія зберігання. Файли, що перевищують встановлений розмір, не можуть бути завантажені. Системне обмеження за замовчуванням становить 20 МБ, його можна збільшити до максимум 1 ГБ.

### Типи файлів

Ви можете обмежити типи файлів, які можна завантажувати, використовуючи синтаксис [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types). Наприклад: `image/*` представляє файли зображень. Кілька типів можна розділити комами, наприклад: `image/*, application/pdf` дозволяє файли зображень та PDF.

### Рушій зберігання за замовчуванням

Якщо встановлено, цей рушій буде встановлено як рушій зберігання за замовчуванням для системи. Коли поле вкладення або **колекція** файлів не вказує рушій зберігання, завантажені файли будуть збережені в рушії зберігання за замовчуванням. Рушій зберігання за замовчуванням не можна видалити.

### Зберігати файл при видаленні запису

Якщо встановлено, завантажений файл у рушії зберігання буде збережено, навіть якщо запис даних у таблиці вкладень або **колекції** файлів буде видалено. За замовчуванням цей прапорець не встановлено, що означає, що файл у рушії зберігання буде видалено разом із записом.

:::info{title=Підказка}
Після завантаження файлу кінцевий шлях доступу формується шляхом об'єднання кількох частин:

```
<Префікс публічної URL-адреси>/<Шлях>/<Ім'я файлу><Розширення>
```

Наприклад: `https://cdn.nocobase.com/app/user/avatar/20240529115151.png`.
:::
