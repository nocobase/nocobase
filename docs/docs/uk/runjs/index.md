# Огляд RunJS

RunJS — це середовище виконання JavaScript у NocoBase, яке використовується для таких сценаріїв, як **JS-блоки**, **JS-поля** та **JS-дії**. Код виконується в обмеженій пісочниці, що забезпечує безпечний доступ до `ctx` (Context API) і має наступні можливості:

- Асинхронність на верхньому рівні (Top-level `await`)
- Імпорт зовнішніх модулів
- Рендеринг у контейнері
- Глобальні змінні

## Асинхронність на верхньому рівні (Top-level `await`)

RunJS підтримує `await` на верхньому рівні, тому код не потрібно огортати в IIFE.

**Не рекомендовано**

```jsx
async function test() {}
(async () => {
  await test();
})();
```

**Рекомендовано**

```js
async function test() {}
await test();
```

## Імпорт зовнішніх модулів

- Використовуйте `ctx.importAsync()` для ESM-модулів (рекомендовано)
- Використовуйте `ctx.requireAsync()` для UMD/AMD-модулів

## Рендеринг у контейнері

Використовуйте `ctx.render()` для рендерингу вмісту в поточний контейнер (`ctx.element`). Підтримуються наступні три формати:

### Рендеринг JSX

```jsx
ctx.render(<button>Button</button>);
```

### Рендеринг DOM-вузлів

```js
const div = document.createElement('div');
div.innerHTML = 'Hello World';

ctx.render(div);
```

### Рендеринг HTML-рядків

```js
ctx.render('<h1>Hello World</h1>');
```

## Глобальні змінні

- `window`
- `document`
- `navigator`
- `ctx`
