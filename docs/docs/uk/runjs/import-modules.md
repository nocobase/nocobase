# Імпорт модулів

У RunJS можна використовувати два типи модулів: **вбудовані модулі** (використовуються безпосередньо через `ctx.libs` без необхідності імпорту) та **зовнішні модулі** (завантажуються за запитом через `ctx.importAsync()` або `ctx.requireAsync()`).

---

## Вбудовані модулі — ctx.libs (імпорт не потрібен)

RunJS містить вбудовані популярні бібліотеки, до яких можна отримати доступ безпосередньо через `ctx.libs`. Вам **не потрібно** використовувати `import` або асинхронне завантаження для них.

| Властивість | Опис |
|------|------|
| **ctx.libs.React** | Ядро React, використовується для JSX та Hooks |
| **ctx.libs.ReactDOM** | ReactDOM (можна використовувати для `createRoot` тощо) |
| **ctx.libs.antd** | Бібліотека компонентів Ant Design |
| **ctx.libs.antdIcons** | Іконки Ant Design |
| **ctx.libs.math** | [Math.js](https://mathjs.org/): математичні вирази, операції з матрицями тощо |
| **ctx.libs.formula** | [Formula.js](https://formulajs.github.io/): формули у стилі Excel (SUM, AVERAGE тощо) |

### Приклад: React та antd

```tsx
const { Button } = ctx.libs.antd;

ctx.render(<Button>Натиснути</Button>);
```

### Приклад: ctx.libs.math

```ts
const result = ctx.libs.math.evaluate('2 + 3 * 4');
// result === 14
```

### Приклад: ctx.libs.formula

```ts
const values = [1, 2, 3, 4];
const sum = ctx.libs.formula.SUM(values);
const avg = ctx.libs.formula.AVERAGE(values);
```

## Зовнішні модулі

Якщо вам потрібні сторонні бібліотеки, виберіть метод завантаження залежно від формату модуля:

- **Модулі ESM** → використовуйте `ctx.importAsync()`
- **Модулі UMD/AMD** → використовуйте `ctx.requireAsync()`

---

### Імпорт модулів ESM

Використовуйте **`ctx.importAsync()`** для динамічного завантаження ESM-модулів за URL-адресою. Це підходить для таких сценаріїв, як блоки JS, поля JS та дії JS.

```ts
importAsync<T = any>(url: string): Promise<T>;
```

- **url**: адреса ESM-модуля. Підтримує скорочені формати, такі як `<назва_пакета>@<версія>` або підшляхи, наприклад `<назва_пакета>@<версія>/<шлях_до_файлу>` (наприклад, `vue@3.4.0`, `lodash@4/lodash.js`). До них буде додано префікс налаштованого CDN; також підтримуються повні URL-адреси.
- **Повертає**: об'єкт простору імен розпізнаного модуля.

#### За замовчуванням: https://esm.sh

Якщо конфігурація відсутня, для скорочених форм як префікс CDN використовуватиметься **https://esm.sh**. Наприклад:

```ts
const Vue = await ctx.importAsync('vue@3.4.0');
// Еквівалентно завантаженню з https://esm.sh/vue@3.4.0
```

#### Власний сервіс esm.sh

Якщо вам потрібна внутрішня мережа або власний CDN, ви можете розгорнути сервіс, сумісний із протоколом esm.sh, і вказати його через змінні оточення:

- **ESM_CDN_BASE_URL**: базова URL-адреса для ESM CDN (за замовчуванням `https://esm.sh`)
- **ESM_CDN_SUFFIX**: необов'язковий суфікс (наприклад, `/+esm` для jsDelivr)

Для розгортання власного сервісу зверніться до: [https://github.com/nocobase/esm-server](https://github.com/nocobase/esm-server)

### Імпорт модулів UMD/AMD

Використовуйте **`ctx.requireAsync()`** для асинхронного завантаження модулів UMD/AMD або скриптів, що прикріплюються до глобального об'єкта.

```ts
requireAsync<T = any>(url: string): Promise<T>;
```

- **url**: підтримує дві форми:
  - **Скорочений шлях**: `<назва_пакета>@<версія>/<шлях_до_файлу>`, аналогічно до `ctx.importAsync()`, розпізнається відповідно до поточної конфігурації ESM CDN. При розпізнаванні додається `?raw` для безпосереднього запиту вихідного файлу (зазвичай це збірка UMD). Наприклад, `echarts@5/dist/echarts.min.js` фактично запитує `https://esm.sh/echarts@5/dist/echarts.min.js?raw` (при використанні esm.sh за замовчуванням).
  - **Повний URL**: будь-яка повна адреса CDN (наприклад, `https://cdn.jsdelivr.net/npm/xxx`).
- **Повертає**: об'єкт завантаженої бібліотеки (конкретна форма залежить від того, як бібліотека експортує свій вміст).

Після завантаження багато бібліотек UMD прикріплюють себе до глобального об'єкта (наприклад, `window.xxx`). Ви можете використовувати їх згідно з документацією відповідної бібліотеки.

**Приклад**

```ts
// Скорочений шлях (розпізнається через esm.sh як ...?raw)
const echarts = await ctx.requireAsync('echarts@5/dist/echarts.min.js');

// Повний URL
const dayjs = await ctx.requireAsync('https://cdn.jsdelivr.net/npm/dayjs@1/dayjs.min.js');
```

**Примітка**: якщо бібліотека надає версію ESM, краще використовувати `ctx.importAsync()` для кращої семантики модулів та Tree-shaking.
