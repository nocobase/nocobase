# ctx.message

Глобальний API повідомлень Ant Design, що використовується для відображення тимчасових легких підказок у верхній центральній частині сторінки. Повідомлення закриваються автоматично через певний час або можуть бути закриті користувачем вручну.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSBlock / JSField / JSItem / JSColumn** | Зворотний зв'язок щодо операцій, підказки валідації, успішне копіювання та інші легкі сповіщення |
| **Операції з формами / Робочий процес** | Зворотний зв'язок при успішному відправленні, помилці збереження, невдалій валідації тощо |
| **Події дій (JSAction)** | Миттєвий зворотний зв'язок для кліків, завершення групових операцій тощо |

## Визначення типу

```ts
message: MessageInstance;
```

`MessageInstance` — це інтерфейс повідомлень Ant Design, який надає наступні методи.

## Основні методи

| Метод | Опис |
|------|------|
| `success(content, duration?)` | Відобразити підказку про успіх |
| `error(content, duration?)` | Відобразити підказку про помилку |
| `warning(content, duration?)` | Відобразити попереджувальну підказку |
| `info(content, duration?)` | Відобразити інформаційну підказку |
| `loading(content, duration?)` | Відобразити підказку про завантаження (потрібно закривати вручну) |
| `open(config)` | Відкрити повідомлення з користувацькою конфігурацією |
| `destroy()` | Закрити всі відображені повідомлення |

**Параметри:**

- `content` (`string` | `ConfigOptions`): Вміст повідомлення або об'єкт конфігурації
- `duration` (`number`, опціонально): Затримка автоматичного закриття (в секундах), за замовчуванням 3 секунди; встановіть 0, щоб вимкнути автоматичне закриття

**ConfigOptions** (коли `content` є об'єктом):

```ts
interface ConfigOptions {
  content: React.ReactNode;  // Вміст повідомлення
  duration?: number;        // Затримка автоматичного закриття (секунди)
  onClose?: () => void;    // Зворотний виклик при закритті
  icon?: React.ReactNode;  // Користувацька іконка
}
```

## Приклади

### Базове використання

```ts
ctx.message.success('Операція успішна');
ctx.message.error('Помилка операції');
ctx.message.warning('Будь ласка, спочатку виберіть дані');
ctx.message.info('Обробка...');
```

### Інтернаціоналізація з ctx.t

```ts
ctx.message.success(ctx.t('Copied'));
ctx.message.warning(ctx.t('Please select at least one row'));
ctx.message.success(ctx.t('Exported {{count}} records', { count: rows.length }));
```

### Завантаження та ручне закриття

```ts
const hide = ctx.message.loading(ctx.t('Saving...'));
// Виконання асинхронної операції
await saveData();
hide();  // Закрити loading вручну
ctx.message.success(ctx.t('Saved'));
```

### Використання open з користувацькою конфігурацією

```ts
ctx.message.open({
  type: 'success',
  content: 'Користувацька підказка про успіх',
  duration: 5,
  onClose: () => console.log('message closed'),
});
```

### Закриття всіх повідомлень

```ts
ctx.message.destroy();
```

## Різниця між ctx.message та ctx.notification

| Характеристика | ctx.message | ctx.notification |
|------|--------------|------------------|
| **Позиція** | Верхня центральна частина сторінки | Правий верхній кут |
| **Призначення** | Тимчасова легка підказка, зникає автоматично | Панель сповіщень, може містити заголовок та опис, підходить для тривалого відображення |
| **Типові сценарії** | Зворотний зв'язок щодо операцій, підказки валідації, успішне копіювання | Сповіщення про завершення завдань, системні повідомлення, довший вміст, що потребує уваги користувача |
