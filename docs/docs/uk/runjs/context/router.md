# ctx.router

Екземпляр маршрутизатора на основі React Router, що використовується для навігації за допомогою коду в RunJS. Зазвичай використовується разом із `ctx.route` та `ctx.location`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock / JSField** | Перехід до сторінок деталей, списків або зовнішніх посилань після натискання кнопки. |
| **Правила зв'язку / Потік подій** | Виконання `navigate` до списку або деталей після успішного надсилання, або передача `state` на цільову сторінку. |
| **JSAction / Обробка подій** | Виконання навігації за маршрутом у логіці надсилання форм або натискання посилань. |
| **Навігація переглядами** | Оновлення URL за допомогою `navigate` під час перемикання стека внутрішніх переглядів. |

> Примітка: `ctx.router` доступний лише в середовищах RunJS із контекстом маршрутизації (наприклад, JSBlock на сторінці, сторінки Flow, потоки подій тощо); він може бути порожнім у суто бекенд-контекстах або контекстах без маршрутизації (наприклад, у робочих процесах).

## Визначення типу

```typescript
router: Router
```

`Router` походить від `@remix-run/router`. У RunJS операції навігації, такі як перехід, повернення назад та оновлення, реалізуються через `ctx.router.navigate()`.

## Методи

### ctx.router.navigate()

Перехід за цільовим шляхом або виконання дії повернення/оновлення.

**Сигнатура:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**Параметри:**

- `to`: Цільовий шлях (string), відносна позиція в історії (number, наприклад, `-1` для повернення назад) або `null` (для оновлення поточної сторінки).
- `options`: Додаткова конфігурація.
  - `replace?: boolean`: Чи замінювати поточний запис в історії (за замовчуванням `false`, тобто додається новий запис).
  - `state?: any`: Стан, що передається цільовому маршруту. Ці дані не відображаються в URL і доступні через `ctx.location.state` на цільовій сторінці. Підходить для конфіденційної інформації, тимчасових даних або відомостей, які не варто розміщувати в URL.

## Приклади

### Базова навігація

```ts
// Перехід до списку користувачів (додає новий запис в історію, можна повернутися назад)
ctx.router.navigate('/admin/users');

// Перехід до сторінки деталей
ctx.router.navigate(`/admin/users/${recordId}`);
```

### Заміна історії (без нового запису)

```ts
// Перенаправлення на головну сторінку після входу; користувач не повернеться на сторінку входу при натисканні «назад»
ctx.router.navigate('/admin', { replace: true });

// Заміна поточної сторінки сторінкою деталей після успішного надсилання форми
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### Передача стану (state)

```ts
// Передача даних під час навігації; цільова сторінка отримує їх через ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### Назад та оновлення

```ts
// Повернутися на одну сторінку назад
ctx.router.navigate(-1);

// Повернутися на дві сторінки назад
ctx.router.navigate(-2);

// Оновити поточну сторінку
ctx.router.navigate(null);
```

## Зв'язок із ctx.route та ctx.location

| Призначення | Рекомендоване використання |
|------|----------|
| **Навігація / Перехід** | `ctx.router.navigate(path)` |
| **Читання поточного шляху** | `ctx.route.pathname` або `ctx.location.pathname` |
| **Читання стану, переданого під час навігації** | `ctx.location.state` |
| **Читання параметрів маршруту** | `ctx.route.params` |

`ctx.router` відповідає за «дії навігації», тоді як `ctx.route` та `ctx.location` відповідають за «поточний стан маршруту».

## Примітки

- `navigate(path)` за замовчуванням додає новий запис в історію, дозволяючи користувачам повернутися за допомогою кнопки «назад» у браузері.
- `replace: true` замінює поточний запис в історії без додавання нового, що підходить для таких сценаріїв, як перенаправлення після входу або перехід після успішного надсилання даних.
- **Щодо параметра `state`**:
  - Дані, передані через `state`, не з'являються в URL, що робить їх придатними для конфіденційних або тимчасових даних.
  - Доступ до них можна отримати через `ctx.location.state` на цільовій сторінці.
  - `state` зберігається в історії браузера і залишається доступним під час навігації вперед/назад.
  - `state` буде втрачено після повного оновлення сторінки.
