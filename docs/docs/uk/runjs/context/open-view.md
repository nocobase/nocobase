# ctx.openView()

Програмне відкриття вказаного представлення (шухляда, діалогове вікно, вбудована сторінка тощо). Надається `FlowModelContext`, використовується в `JSBlock`, комірках таблиць, робочих процесах та інших сценаріях для відкриття налаштованих представлень `ChildPage` або `PopupAction`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock** | Відкриття діалогового вікна деталей/редагування після натискання кнопки, передача `filterByTk` поточного рядка. |
| **Комірка таблиці** | Рендеринг кнопки в комірці, натискання якої відкриває діалогове вікно деталей рядка. |
| **Робочий процес / JSAction** | Відкриття наступного представлення або діалогового вікна після успішної операції. |
| **Поле асоціації** | Відкриття діалогового вікна вибору/редагування через `ctx.runAction('openView', params)`. |

> Примітка: `ctx.openView` доступний у середовищі RunJS, де існує контекст FlowModel; якщо модель, що відповідає `uid`, не існує, буде автоматично створено та збережено `PopupActionModel`.

## Сигнатура

```ts
openView(uid: string, options?: OpenViewOptions): Promise<void>
```

## Опис параметрів

### uid

Унікальний ідентифікатор моделі представлення. Якщо він не існує, його буде автоматично створено та збережено. Рекомендується використовувати стабільний UID, наприклад `${ctx.model.uid}-detail`, щоб конфігурацію можна було використовувати повторно при багаторазовому відкритті того самого діалогового вікна.

### Основні поля options

| Поле | Тип | Опис |
|------|------|------|
| `mode` | `drawer` / `dialog` / `embed` | Спосіб відкриття: шухляда, діалогове вікно або вбудоване. За замовчуванням `drawer`. |
| `size` | `small` / `medium` / `large` | Розмір діалогового вікна/шухляди. За замовчуванням `medium`. |
| `title` | `string` | Заголовок представлення. |
| `params` | `Record<string, any>` | Довільні параметри, що передаються у представлення. |
| `filterByTk` | `any` | Значення первинного ключа, використовується для сценаріїв деталей/редагування одного запису. |
| `sourceId` | `string` | ID вихідного запису, використовується в сценаріях асоціацій. |
| `dataSourceKey` | `string` | Джерело даних. |
| `collectionName` | `string` | Назва колекції. |
| `associationName` | `string` | Назва поля асоціації. |
| `navigation` | `boolean` | Чи використовувати навігацію за маршрутом. Якщо передано `defineProperties` або `defineMethods`, примусово встановлюється значення `false`. |
| `preventClose` | `boolean` | Чи запобігати закриттю. |
| `defineProperties` | `Record<string, PropertyOptions>` | Динамічне впровадження властивостей у модель всередині представлення. |
| `defineMethods` | `Record<string, Function>` | Динамічне впровадження методів у модель всередині представлення. |

## Приклади

### Базове використання: відкриття шухляди

```ts
const popupUid = `${ctx.model.uid}-detail`;
await ctx.openView(popupUid, {
  mode: 'drawer',
  size: 'medium',
  title: ctx.t('Деталі'),
});
```

### Передача контексту поточного рядка

```ts
const primaryKey = ctx.collection?.primaryKey || 'id';
await ctx.openView(`${ctx.model.uid}-1`, {
  mode: 'dialog',
  title: ctx.t('Деталі рядка'),
  params: {
    filterByTk: ctx.record?.[primaryKey],
    record: ctx.record,
  },
});
```

### Відкриття через runAction

Коли модель налаштована з дією `openView` (наприклад, поля асоціації або поля, на які можна натиснути), ви можете викликати:

```ts
await ctx.runAction('openView', {
  navigation: false,
  mode: 'dialog',
  collectionName: 'users',
  filterByTk: ctx.record?.id,
});
```

### Впровадження власного контексту

```ts
await ctx.openView(`${ctx.model.uid}-edit`, {
  mode: 'drawer',
  filterByTk: ctx.record?.id,
  defineProperties: {
    onSaved: {
      get: () => () => ctx.resource?.refresh?.(),
      cache: false,
    },
  },
});
```

## Зв'язок із ctx.viewer та ctx.view

| Призначення | Рекомендоване використання |
|------|----------|
| **Відкриття налаштованого представлення процесу** | `ctx.openView(uid, options)` |
| **Відкриття власного вмісту (без процесу)** | `ctx.viewer.dialog()` / `ctx.viewer.drawer()` |
| **Робота з поточним відкритим представленням** | `ctx.view.close()`, `ctx.view.inputArgs` |

`ctx.openView` відкриває `FlowPage` (`ChildPageModel`), всередині якого рендериться повна сторінка процесу; `ctx.viewer` відкриває будь-який вміст React.

## Примітки

- Рекомендується пов'язувати `uid` із `ctx.model.uid` (наприклад, `${ctx.model.uid}-xxx`), щоб уникнути конфліктів між кількома блоками.
- При передачі `defineProperties` / `defineMethods` параметр `navigation` буде примусово встановлено в `false`, щоб запобігти втраті контексту після оновлення.
- Всередині діалогового вікна `ctx.view` вказує на поточний екземпляр представлення, а `ctx.view.inputArgs` дозволяє зчитувати параметри, передані при відкритті.
