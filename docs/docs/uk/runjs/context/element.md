# ctx.element

Екземпляр `ElementProxy`, що вказує на DOM-контейнер пісочниці та слугує ціллю рендерингу за замовчуванням для `ctx.render()`. Він доступний у сценаріях, де існує контейнер рендерингу, таких як `JSBlock`, `JSField`, `JSItem` та `JSColumn`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock** | DOM-контейнер блоку, що використовується для рендерингу користувацького вмісту блоку. |
| **JSField / JSItem / FormJSFieldItem** | Контейнер рендерингу для поля або елемента форми (зазвичай `<span>`). |
| **JSColumn** | DOM-контейнер комірки таблиці, що використовується для рендерингу користувацького вмісту колонки. |

> Примітка: `ctx.element` доступний лише в контекстах RunJS, які мають контейнер рендерингу. У сценаріях без UI (наприклад, суто бекенд-логіка) він може бути `undefined`. Рекомендується перевіряти на пусте значення перед використанням.

## Визначення типу

```typescript
element: ElementProxy | undefined;

// ElementProxy — це проксі для сирого HTMLElement, що надає безпечний API
class ElementProxy {
  __el: HTMLElement;  // Внутрішній сирий DOM-елемент (доступний лише в окремих сценаріях)
  innerHTML: string;  // Очищується за допомогою DOMPurify під час читання/запису
  outerHTML: string; // Те саме, що й вище
  appendChild(child: HTMLElement | string): void;
  // Інші методи HTMLElement прокидаються далі (пряме використання не рекомендується)
}
```

## Вимоги до безпеки

**Рекомендовано: весь рендеринг має виконуватися через `ctx.render()`.** Уникайте прямого використання DOM API `ctx.element` (наприклад, `innerHTML`, `appendChild`, `querySelector` тощо).

### Чому рекомендовано ctx.render()

| Перевага | Опис |
|------|------|
| **Безпека** | Централізований контроль безпеки для запобігання XSS та неналежним операціям з DOM. |
| **Підтримка React** | Повна підтримка JSX, компонентів React та життєвих циклів. |
| **Успадкування контексту** | Автоматично успадковує `ConfigProvider` додатка, теми тощо. |
| **Обробка конфліктів** | Автоматичне керування створенням/видаленням кореня React для уникнення конфліктів між екземплярами. |

### ❌ Не рекомендовано: Пряме маніпулювання ctx.element

```ts
// ❌ Не рекомендовано: пряме використання API ctx.element
ctx.element.innerHTML = '<div>Вміст</div>';
ctx.element.appendChild(node);
ctx.element.querySelector('.class');
```

> `ctx.element.innerHTML` застарів. Будь ласка, використовуйте `ctx.render()` замість нього.

### ✅ Рекомендовано: Використання ctx.render()

```ts
// ✅ Рендеринг React-компонента
const { Button, Card } = ctx.libs.antd;
ctx.render(
  <Card title={ctx.t('Вітаємо')}>
    <Button type="primary">Натиснути</Button>
  </Card>
);

// ✅ Рендеринг HTML-рядка
ctx.render('<div style="padding:16px;">' + ctx.t('Вміст') + '</div>');

// ✅ Рендеринг DOM-вузла
const div = document.createElement('div');
div.textContent = ctx.t('Привіт');
ctx.render(div);
```

## Особливий випадок: як якір для Popover

Коли вам потрібно відкрити Popover, використовуючи поточний елемент як якір, ви можете звернутися до `ctx.element?.__el`, щоб отримати сирий DOM як `target`:

```ts
// ctx.viewer.popover потребує сирий DOM як target
await ctx.viewer.popover({
  target: ctx.element?.__el,
  content: <div>Вміст спливаючого вікна</div>,
});
```

> Використовуйте `__el` лише в таких сценаріях, як «використання поточного контейнера як якоря»; в інших випадках не маніпулюйте DOM безпосередньо.

## Зв'язок із ctx.render

- Якщо `ctx.render(vnode)` викликається без аргументу `container`, за замовчуванням рендеринг відбувається в контейнер `ctx.element`.
- Якщо `ctx.element` відсутній і `container` не передано, буде видано помилку.
- Ви можете явно вказати контейнер: `ctx.render(vnode, customContainer)`.

## Примітки

- `ctx.element` призначений для внутрішнього використання `ctx.render()`. Прямий доступ або зміна його властивостей/методів не рекомендується.
- У контекстах без контейнера рендерингу `ctx.element` буде `undefined`. Перед викликом `ctx.render()` переконайтеся, що контейнер доступний, або передайте `container` вручну.
- Хоча `innerHTML`/`outerHTML` в `ElementProxy` очищуються за допомогою DOMPurify, все одно рекомендується використовувати `ctx.render()` для уніфікованого керування рендерингом.
