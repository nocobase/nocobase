# ctx.setValue()

У сценаріях редагованих полів, таких як JSField та JSItem, цей метод встановлює значення поточного поля. У поєднанні з `ctx.getValue()` він дозволяє реалізувати двосторонню прив'язку (binding) до форми.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSField** | Запис вибраних користувачем або обчислених значень у редаговані користувацькі поля. |
| **JSItem** | Оновлення значення поточної комірки в редагованих елементах таблиць або підтаблиць. |
| **JSColumn** | Оновлення значення поля відповідного рядка на основі логіки під час рендерингу стовпців таблиці. |

> **Примітка**: `ctx.setValue(v)` доступний лише в контекстах RunJS із прив'язкою до форми. Він відсутній у сценаріях без прив'язки до полів, таких як потоки подій, правила зв'язку або JSBlock. Перед використанням рекомендується застосовувати опціональний ланцюжок: `ctx.setValue?.(value)`.

## Визначення типу

```ts
setValue<T = any>(value: T): void;
```

- **Параметри**: `value` — це значення поля для запису. Тип визначається типом елемента форми цього поля.

## Поведінка

- `ctx.setValue(v)` оновлює значення поточного поля у формі Ant Design і запускає відповідну логіку зв'язків форми та валідації.
- Якщо форма ще не завершила рендеринг або поле не зареєстроване, виклик може бути недійсним. Рекомендується використовувати `ctx.getValue()` для підтвердження результату запису.

## Приклади

### Двостороння прив'язка за допомогою getValue

```tsx
const { Input } = ctx.libs.antd;

const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```

### Встановлення значення за замовчуванням на основі умов

```ts
const status = ctx.getValue();
if (status == null || status === '') {
  ctx.setValue?.('draft');
}
```

### Зворотний запис у поточне поле при зв'язку з іншими полями

```ts
// Синхронне оновлення поточного поля при зміні іншого поля
const otherValue = ctx.form?.getFieldValue('type');
if (otherValue === 'custom') {
  ctx.setValue?.({ label: 'Користувацький', value: 'custom' });
}
```

## Зауваження

- У нередагованих полях (наприклад, JSField у режимі лише для читання, JSBlock) `ctx.setValue` може бути `undefined`. Рекомендується використовувати `ctx.setValue?.(value)`, щоб уникнути помилок.
- При встановленні значень для полів зв'язку (m2o, o2m тощо) необхідно передавати структуру, яка відповідає типу поля (наприклад, `{ id, [titleField]: label }`), залежно від конкретної конфігурації поля.

## Пов'язане

- [ctx.getValue()](./get-value.md) — отримання значення поточного поля, використовується разом із setValue для двосторонньої прив'язки.
- [ctx.form](./form.md) — екземпляр форми Ant Design, використовується для читання або запису інших полів.
- `js-field:value-change` — подія контейнера, що виникає при зміні зовнішнього значення, використовується для оновлення відображення.
