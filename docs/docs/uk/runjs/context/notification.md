# ctx.notification

Глобальний API сповіщень на основі Ant Design Notification, який використовується для відображення панелей сповіщень у **верхньому правому куті** сторінки. Порівняно з `ctx.message`, сповіщення можуть містити заголовок і опис, що робить їх придатними для вмісту, який має відображатися довше або потребує особливої уваги користувача.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSBlock / Події дій** | Сповіщення про завершення завдань, результати групових операцій, завершення експорту тощо. |
| **Потоки подій** | Системні сповіщення після завершення асинхронних процесів. |
| **Вміст, що потребує тривалого відображення** | Повні сповіщення із заголовками, описами та кнопками дій. |

## Визначення типу

```ts
notification: NotificationInstance;
```

`NotificationInstance` — це інтерфейс сповіщень Ant Design, що надає наступні методи.

## Основні методи

| Метод | Опис |
|------|------|
| `open(config)` | Відкриває сповіщення з користувацькою конфігурацією |
| `success(config)` | Відображає сповіщення про успіх |
| `info(config)` | Відображає інформаційне сповіщення |
| `warning(config)` | Відображає попередження |
| `error(config)` | Відображає сповіщення про помилку |
| `destroy(key?)` | Закриває сповіщення з вказаним ключем; якщо ключ не вказано, закриває всі сповіщення |

**Параметри конфігурації** (відповідають [Ant Design notification](https://ant.design/components/notification)):

| Параметр | Тип | Опис |
|------|------|------|
| `message` | `ReactNode` | Заголовок сповіщення |
| `description` | `ReactNode` | Опис сповіщення |
| `duration` | `number` | Затримка автоматичного закриття (у секундах). Типово 4.5 сек; встановіть 0, щоб вимкнути автоматичне закриття |
| `key` | `string` | Унікальний ідентифікатор сповіщення, використовується для `destroy(key)`, щоб закрити конкретне сповіщення |
| `onClose` | `() => void` | Функція зворотного виклику при закритті |
| `placement` | `string` | Розташування: `topLeft` \| `topRight` \| `bottomLeft` \| `bottomRight` |

## Приклади

### Базове використання

```ts
ctx.notification.open({
  message: 'Операція успішна',
  description: 'Дані збережено на сервері.',
});
```

### Швидкий виклик за типом

```ts
ctx.notification.success({
  message: ctx.t('Export completed'),
  description: ctx.t('Exported {{count}} records', { count: 10 }),
});

ctx.notification.error({
  message: ctx.t('Export failed'),
  description: ctx.t('Please check the console for details'),
});

ctx.notification.warning({
  message: ctx.t('Warning'),
  description: ctx.t('Some data may be incomplete'),
});
```

### Користувацька тривалість та ключ

```ts
ctx.notification.open({
  key: 'task-123',
  message: ctx.t('Task in progress'),
  description: ctx.t('Please wait...'),
  duration: 0,  // Не закривати автоматично
});

// Закрити вручну після завершення завдання
ctx.notification.destroy('task-123');
```

### Закрити всі сповіщення

```ts
ctx.notification.destroy();
```

## Відмінність від ctx.message

| Характеристика | ctx.message | ctx.notification |
|------|--------------|------------------|
| **Позиція** | Вгорі по центру сторінки | Верхній правий кут (налаштовується) |
| **Структура** | Однорядкова легка підказка | Може містити заголовок + опис |
| **Призначення** | Тимчасовий зворотний зв'язок, зникає автоматично | Повне сповіщення, може відображатися тривалий час |
| **Типові сценарії** | Успіх операції, помилка валідації, успішне копіювання | Завершення завдання, системні повідомлення, довгий вміст, що потребує уваги користувача |
