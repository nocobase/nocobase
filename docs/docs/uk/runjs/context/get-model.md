# ctx.getModel()

Отримує екземпляр моделі (такий як `BlockModel`, `PageModel`, `ActionModel` тощо) з поточного рушія або стека представлень на основі `uid` моделі. Це використовується в RunJS для доступу до інших моделей між блоками, сторінками або спливаючими вікнами.

Якщо вам потрібна лише модель або блок, де знаходиться поточний контекст виконання, надавайте перевагу використанню `ctx.model` або `ctx.blockModel` замість `ctx.getModel`.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSBlock / JSAction** | Отримання моделей інших блоків на основі відомого `uid` для читання або запису їхніх `resource`, `form`, `setProps` тощо. |
| **RunJS у спливаючих вікнах** | Коли у спливаючому вікні потрібно отримати доступ до моделі на сторінці, яка його відкрила, передайте `searchInPreviousEngines: true`. |
| **Спеціальні дії** | Пошук форм або підмоделей у панелі налаштувань за `uid` через стеки представлень для читання їхньої конфігурації або стану. |

## Визначення типу

```ts
getModel<T extends FlowModel = FlowModel>(
  uid: string,
  searchInPreviousEngines?: boolean
): T | undefined
```

## Параметри

| Параметр | Тип | Опис |
|------|------|------|
| `uid` | `string` | Унікальний ідентифікатор цільового екземпляра моделі, вказаний під час конфігурації або створення (наприклад, `ctx.model.uid`). |
| `searchInPreviousEngines` | `boolean` | Необов'язково, за замовчуванням `false`. Якщо `true`, пошук здійснюється в «стеку представлень» від поточного рушія до кореня, що дозволяє отримати доступ до моделей у рушіях вищого рівня (наприклад, сторінки, яка відкрила спливаюче вікно). |

## Значення, що повертається

- Повертає відповідний екземпляр підкласу `FlowModel` (наприклад, `BlockModel`, `FormBlockModel`, `ActionModel`), якщо його знайдено.
- Повертає `undefined`, якщо не знайдено.

## Область пошуку

- **За замовчуванням (`searchInPreviousEngines: false`)**: Пошук за `uid` здійснюється лише в межах **поточного рушія**. У спливаючих вікнах або багаторівневих представленнях кожне представлення має незалежний рушій; за замовчуванням пошук ведеться лише в межах поточного представлення.
- **`searchInPreviousEngines: true`**: Пошук вгору по ланцюжку `previousEngine`, починаючи з поточного рушія, повертаючи перший збіг. Це корисно для доступу до моделі на сторінці, яка відкрила поточне спливаюче вікно.

## Приклади

### Отримання іншого блоку та оновлення

```ts
const block = ctx.getModel('list-block-uid');
if (block?.resource) {
  await block.resource.refresh();
}
```

### Доступ до моделі на сторінці зі спливаючого вікна

```ts
// Доступ до блоку на сторінці, яка відкрила поточне спливаюче вікно
const pageBlock = ctx.getModel('page-block-uid', true);
if (pageBlock) {
  pageBlock.rerender?.();
}
```

### Читання/запис між моделями та запуск перерендерінгу (rerender)

```ts
const target = ctx.getModel('other-block-uid');
if (target) {
  target.setProps({ loading: true });
  target.rerender?.();
}
```

### Перевірка безпеки

```ts
const model = ctx.getModel(someUid);
if (!model) {
  ctx.message.warning('Цільова модель не існує');
  return;
}
```

## Пов'язане

- [ctx.model](./model.md): Модель, у якій знаходиться поточний контекст виконання.
- [ctx.blockModel](./block-model.md): Батьківська модель блоку, у якій знаходиться поточний JS; зазвичай доступна без використання `getModel`.
