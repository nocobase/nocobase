# ctx.form

Екземпляр Ant Design Form у поточному блоці, який використовується для читання/запису полів форми, запуску валідації та відправки. Він еквівалентний `ctx.blockModel?.form` і може використовуватися безпосередньо в блоках, пов'язаних із формами (форма, форма редагування, дочірня форма тощо).

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSField** | Читання/запис інших полів форми для реалізації взаємодії, виконання обчислень або валідації на основі значень інших полів. |
| **JSItem** | Читання/запис полів у тому самому рядку або інших полів у елементах дочірньої таблиці для реалізації взаємодії всередині таблиці. |
| **JSColumn** | Читання значень поточного рядка або асоційованих полів у колонці таблиці для рендерингу. |
| **Дії форми / Потік подій** | Валідація перед відправкою, масове оновлення полів, скидання форми тощо. |

> Примітка: `ctx.form` доступний лише в контекстах RunJS, пов'язаних із блоками форм (Form, EditForm, дочірня форма тощо). У сценаріях, не пов'язаних із формами (наприклад, незалежні блоки JSBlock або блоки таблиць), він може бути відсутнім. Перед використанням рекомендується перевіряти на null: `ctx.form?.getFieldsValue()`.

## Визначення типу

```ts
form: FormInstance<any>;
```

`FormInstance` — це тип екземпляра Ant Design Form. Основні методи наведені нижче.

## Основні методи

### Читання значень форми

```ts
// Читання значень поточних зареєстрованих полів (за замовчуванням лише відрендерені поля)
const values = ctx.form.getFieldsValue();

// Читання значень усіх полів (включаючи зареєстровані, але не відрендерені, наприклад, приховані або в згорнутих секціях)
const allValues = ctx.form.getFieldsValue(true);

// Читання одного поля
const email = ctx.form.getFieldValue('email');

// Читання вкладених полів (наприклад, у дочірній таблиці)
const amount = ctx.form.getFieldValue(['orders', 0, 'amount']);
```

### Запис значень форми

```ts
// Масове оновлення (часто використовується для взаємодії полів)
ctx.form.setFieldsValue({
  status: 'active',
  updatedAt: new Date(),
});

// Оновлення одного поля
ctx.form.setFieldValue('remark', 'Оновлено примітку');
```

### Валідація та відправка

```ts
// Запуск валідації форми
await ctx.form.validateFields();

// Запуск відправки форми
ctx.form.submit();
```

### Скидання

```ts
// Скидання всіх полів
ctx.form.resetFields();

// Скидання лише певних полів
ctx.form.resetFields(['status', 'remark']);
```

## Зв'язок із супутніми контекстами

### ctx.getValue / ctx.setValue

| Сценарій | Рекомендоване використання |
|------|----------|
| **Читання/запис поточного поля** | `ctx.getValue()` / `ctx.setValue(v)` |
| **Читання/запис інших полів** | `ctx.form.getFieldValue(name)` / `ctx.form.setFieldValue(name, v)` |

У межах поточного JS-поля віддавайте пріоритет `getValue`/`setValue` для роботи з самим полем; використовуйте `ctx.form`, коли потрібно отримати доступ до інших полів.

### ctx.blockModel

| Потреба | Рекомендоване використання |
|------|----------|
| **Читання/запис полів форми** | `ctx.form` (еквівалентно `ctx.blockModel?.form`, але зручніше) |
| **Доступ до батьківського блоку** | `ctx.blockModel` (містить `collection`, `resource` тощо) |

### ctx.getVar('ctx.formValues')

Значення форми потрібно отримувати через `await ctx.getVar('ctx.formValues')`, вони не доступні безпосередньо як `ctx.formValues`. У контексті форми краще використовувати `ctx.form.getFieldsValue()` для читання останніх значень у реальному часі.

## Примітки

- `getFieldsValue()` за замовчуванням повертає лише відрендерені поля. Щоб включити невідрендерені поля (наприклад, у згорнутих секціях або приховані за умовами), передайте `true`: `getFieldsValue(true)`.
- Шляхи для вкладених полів, таких як дочірні таблиці, є масивами, наприклад, `['orders', 0, 'amount']`. Ви можете використовувати `ctx.namePath`, щоб отримати шлях поточного поля та побудувати шляхи для інших колонок у тому самому рядку.
- `validateFields()` викидає об'єкт помилки, що містить `errorFields` та іншу інформацію. Якщо валідація не вдалася перед відправкою, ви можете використати `ctx.exit()`, щоб перервати подальші кроки.
- У асинхронних сценаріях, таких як робочі процеси або правила взаємодії, `ctx.form` може бути ще не готовим. Рекомендується використовувати опціональний ланцюжок або перевірку на null.

## Приклади

### Взаємодія полів: відображення різного вмісту залежно від типу

```ts
const type = ctx.form.getFieldValue('type');
if (type === 'vip') {
  ctx.form.setFieldsValue({ discount: 0.8 });
} else {
  ctx.form.setFieldsValue({ discount: 1 });
}
```

### Обчислення поточного поля на основі інших полів

```ts
const quantity = ctx.form.getFieldValue('quantity') ?? 0;
const price = ctx.form.getFieldValue('price') ?? 0;
ctx.setValue(quantity * price);
```

### Читання/запис інших колонок у тому самому рядку дочірньої таблиці

```ts
// ctx.namePath — це шлях поточного поля у формі, наприклад, ['orders', 0, 'amount']
// Читання 'status' у тому самому рядку: ['orders', 0, 'status']
const rowIndex = ctx.namePath?.[1];
const status = ctx.form.getFieldValue(['orders', rowIndex, 'status']);
```

### Валідація перед відправкою

```ts
try {
  await ctx.form.validateFields();
  // Валідація пройдена, продовжуємо логіку відправки
} catch (e) {
  ctx.message.error('Будь ласка, перевірте поля форми');
  ctx.exit();
}
```

### Відправка після підтвердження

```ts
const confirmed = await ctx.modal.confirm({
  title: 'Підтвердження відправки',
  content: 'Ви не зможете змінити це після відправки. Продовжити?',
  okText: 'Підтвердити',
  cancelText: 'Скасувати',
});
if (confirmed) {
  await ctx.form.validateFields();
  ctx.form.submit();
} else {
  ctx.exit(); // Перервати, якщо користувач скасував дію
}
```

## Пов'язане

- [ctx.getValue()](./get-value.md) / [ctx.setValue()](./set-value.md): Читання та запис значення поточного поля.
- [ctx.blockModel](./block-model.md): Модель батьківського блоку; `ctx.form` еквівалентний `ctx.blockModel?.form`.
- [ctx.modal](./modal.md): Діалогові вікна підтвердження, часто використовуються з `ctx.form.validateFields()` та `ctx.form.submit()`.
- [ctx.exit()](./exit.md): Переривання процесу при помилці валідації або скасуванні користувачем.
- `ctx.namePath`: Шлях (масив) поточного поля у формі, використовується для побудови імен для `getFieldValue` / `setFieldValue` у вкладених полях.
