# ctx.exitAll()

Перериває поточний потік подій та всі наступні потоки подій, ініційовані в межах тієї ж диспетчеризації подій. Зазвичай використовується, коли необхідно негайно припинити всі потоки подій для поточної події через глобальну помилку або перевірку прав доступу.

## Сценарії застосування

`ctx.exitAll()` зазвичай використовується в контекстах виконання JS, коли необхідно **одночасно перервати поточний потік подій та наступні потоки подій, ініційовані цією подією**:

| Сценарій | Опис |
|------|------|
| **Потік подій** | Помилка валідації основного потоку подій (наприклад, недостатньо прав), що вимагає припинення основного потоку та будь-яких наступних потоків у межах тієї ж події, які ще не були виконані. |
| **Правила зв'язку** | Коли валідація зв'язку не проходить, поточний зв'язок та наступні зв'язки, ініційовані тією ж подією, мають бути припинені. |
| **Події дій** | Помилка попередньої перевірки дії (наприклад, перевірка прав перед видаленням), що вимагає запобігання основній дії та наступним крокам. |

> Відмінність від `ctx.exit()`: `ctx.exit()` перериває лише поточний потік подій; `ctx.exitAll()` перериває поточний потік подій та будь-які **невиконані** наступні потоки подій у тій самій диспетчеризації подій.

## Визначення типу

```ts
exitAll(): never;
```

Виклик `ctx.exitAll()` генерує внутрішній виняток `FlowExitAllException`, який перехоплюється рушієм потоків подій для зупинки поточного екземпляра потоку подій та наступних потоків подій у межах тієї ж події. Після виклику решта операторів у поточному коді JS не виконуватимуться.

## Порівняння з ctx.exit()

| Метод | Область дії |
|------|----------|
| `ctx.exit()` | Перериває лише поточний потік подій; наступні потоки подій залишаються незмінними. |
| `ctx.exitAll()` | Перериває поточний потік подій та припиняє наступні потоки подій, що виконуються **послідовно** в межах тієї ж події. |

## Опис режиму виконання

- **Послідовне виконання (sequential)**: потоки подій у межах однієї події виконуються по порядку. Після того, як будь-який потік подій викликає `ctx.exitAll()`, наступні потоки подій не виконуватимуться.
- **Паралельне виконання (parallel)**: потоки подій у межах однієї події виконуються паралельно. Виклик `ctx.exitAll()` в одному потоці подій не перерве інші конкурентні потоки подій (оскільки вони незалежні).

## Приклади

### Припинення всіх потоків подій у разі помилки перевірки прав доступу

```ts
// Перервати основний потік подій та наступні потоки подій, якщо прав недостатньо
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'Немає прав для виконання операції' });
  ctx.exitAll();
}
```

### Припинення у разі помилки глобальної попередньої перевірки

```ts
// Приклад: якщо перед видаленням виявлено, що пов'язані дані не можна видалити, запобігти основному потоку подій та наступним діям
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('Неможливо видалити: існують пов’язані дані');
  ctx.exitAll();
}
```

### Вибір між ctx.exit() та ctx.exitAll()

```ts
// Потрібно вийти лише з поточного потоку подій -> Використовуйте ctx.exit()
if (!params.valid) {
  ctx.message.error('Недійсні параметри');
  ctx.exit();  // Наступні потоки подій не зачіпаються
}

// Потрібно припинити всі наступні потоки подій у межах поточної події -> Використовуйте ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'Недостатньо прав' });
  ctx.exitAll();  // Основний потік подій та наступні потоки подій у межах тієї ж події припиняються одночасно
}
```

### Сповіщення перед припиненням

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('Будь ласка, спочатку виправте помилки у формі');
  ctx.exitAll();
}
```

## Примітки

- Після виклику `ctx.exitAll()` наступний код у поточному JS не виконуватиметься. Рекомендується пояснити причину користувачеві за допомогою `ctx.message`, `ctx.notification` або модального вікна перед викликом.
- У бізнес-коді зазвичай не потрібно перехоплювати `FlowExitAllException`; залиште це рушію потоків подій.
- Якщо вам потрібно зупинити лише поточний потік подій, не впливаючи на наступні, використовуйте `ctx.exit()`.
- У паралельному режимі `ctx.exitAll()` перериває лише поточний потік подій і не зупиняє інші конкурентні потоки подій.
