# ctx.model

Екземпляр `FlowModel`, у якому знаходиться поточний контекст виконання RunJS. Він слугує точкою входу за замовчуванням для таких сценаріїв, як JSBlock, JSField та JSAction. Конкретний тип змінюється залежно від контексту: це може бути підклас, такий як `BlockModel`, `ActionModel` або `JSEditableFieldModel`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock** | `ctx.model` є моделлю поточного блоку. Ви можете отримати доступ до `resource`, `collection`, `setProps` тощо. |
| **JSField / JSItem / JSColumn** | `ctx.model` є моделлю поля. Ви можете отримати доступ до `setProps`, `dispatchEvent` тощо. |
| **Події дій / ActionModel** | `ctx.model` є моделлю дії. Ви можете читати/записувати параметри кроків, відправляти події тощо. |

> Порада: Якщо вам потрібно отримати доступ до **батьківського блоку, що містить поточний JS** (наприклад, блок форми або таблиці), використовуйте `ctx.blockModel`; щоб отримати доступ до **інших моделей**, використовуйте `ctx.getModel(uid)`.

## Визначення типу

```ts
model: FlowModel;
```

`FlowModel` є базовим класом. Під час виконання це екземпляр різних підкласів (таких як `BlockModel`, `FormBlockModel`, `TableBlockModel`, `JSEditableFieldModel`, `ActionModel тощо`). Доступні властивості та методи залежать від конкретного типу.

## Загальні властивості

| Властивість | Тип | Опис |
|------|------|------|
| `uid` | `string` | Унікальний ідентифікатор моделі. Може використовуватися для `ctx.getModel(uid)` або прив'язки UID спливаючого вікна. |
| `collection` | `Collection` | Колекція, прив'язана до поточної моделі (існує, коли блок/поле прив'язані до даних). |
| `resource` | `Resource` | Пов'язаний екземпляр ресурсу, що використовується для оновлення, отримання вибраних рядків тощо. |
| `props` | `object` | Конфігурація UI/поведінки моделі. Може бути оновлена за допомогою `setProps`. |
| `subModels` | `Record<string, FlowModel>` | Колекція дочірніх моделей (наприклад, поля у формі, колонки в таблиці). |
| `parent` | `FlowModel` | Батьківська модель (якщо є). |

## Загальні методи

| Метод | Опис |
|------|------|
| `setProps(partialProps: any): void` | Оновлює конфігурацію моделі та ініціює повторний рендеринг (наприклад, `ctx.model.setProps({ loading: true })`). |
| `dispatchEvent(eventName: string, payload?: any, options?: any): Promise<any[]>` | Відправляє подію моделі, запускаючи робочі процеси, налаштовані для цієї моделі, які прослуховують цю назву події. Необов'язковий `payload` передається в обробник робочого процесу; `options.debounce` дозволяє увімкнути усунення брязкоту (debounce). |
| `getStepParams?.(flowKey, stepKey)` | Зчитує параметри кроків потоку конфігурації (використовується в панелях налаштувань, користувацьких діях тощо). |
| `setStepParams?.(flowKey, stepKey, params)` | Записує параметри кроків потоку конфігурації. |

## Зв'язок із ctx.blockModel та ctx.getModel

| Потреба | Рекомендоване використання |
|------|----------|
| **Модель поточного контексту виконання** | `ctx.model` |
| **Батьківський блок поточного JS** | `ctx.blockModel`. Часто використовується для доступу до `resource`, `form` або `collection`. |
| **Отримати будь-яку модель за UID** | `ctx.getModel(uid)` або `ctx.getModel(uid, true)` (пошук по всіх стеках представлень). |

У JSField `ctx.model` — це модель поля, тоді як `ctx.blockModel` — це блок форми або таблиці, що містить це поле.

## Приклади

### Оновлення статусу блоку/дії

```ts
ctx.model.setProps({ loading: true });
await doSomething();
ctx.model.setProps({ loading: false });
```

### Відправка подій моделі

```ts
// Відправити подію, щоб запустити робочий процес, налаштований для цієї моделі, який прослуховує цю назву події
await ctx.model.dispatchEvent('remove');

// Якщо надано payload, він передається в ctx.inputArgs обробника робочого процесу
await ctx.model.dispatchEvent('customEvent', { id: 123 });
```

### Використання UID для прив'язки спливаючих вікон або міжмодельного доступу

```ts
const myUid = ctx.model.uid;
// У конфігурації спливаючого вікна ви можете передати openerUid: myUid для встановлення зв'язку
const other = ctx.getModel('other-block-uid');
if (other) other.rerender?.();
```
