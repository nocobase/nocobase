# ctx.modal

Швидкий API на основі Ant Design Modal, що використовується для активного відкриття модальних вікон (інформаційних повідомлень, вікон підтвердження тощо) у RunJS. Реалізовано через `ctx.viewer` / систему представлення.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock / JSField** | Відображення результатів операцій, повідомлень про помилки або повторного підтвердження після взаємодії з користувачем. |
| **Робочий процес / Події дій** | Спливаюче вікно підтвердження перед відправкою; переривання наступних кроків за допомогою `ctx.exit()`, якщо користувач скасовує дію. |
| **Правила зв'язку** | Спливаючі підказки для користувача у разі невдалої валідації. |

> Примітка: `ctx.modal` доступний у середовищах RunJS із контекстом представлення (наприклад, JSBlock на сторінці, робочі процеси тощо); він може бути відсутнім у бекенді або контекстах без інтерфейсу користувача. При використанні рекомендується перевіряти наявність через опціональний ланцюжок (`ctx.modal?.confirm?.()`).

## Визначення типів

```ts
modal: {
  info: (config: ModalConfig) => Promise<void>;
  success: (config: ModalConfig) => Promise<void>;
  error: (config: ModalConfig) => Promise<void>;
  warning: (config: ModalConfig) => Promise<void>;
  confirm: (config: ModalConfig) => Promise<boolean>;  // Повертає true, якщо користувач натиснув OK, і false, якщо скасував
};
```

`ModalConfig` відповідає конфігурації статичних методів `Modal` в Ant Design.

## Основні методи

| Метод | Значення, що повертається | Опис |
|------|--------|------|
| `info(config)` | `Promise<void>` | Модальне вікно з інформацією |
| `success(config)` | `Promise<void>` | Модальне вікно успіху |
| `error(config)` | `Promise<void>` | Модальне вікно помилки |
| `warning(config)` | `Promise<void>` | Модальне вікно попередження |
| `confirm(config)` | `Promise<boolean>` | Вікно підтвердження; повертає `true`, якщо користувач натиснув OK, і `false`, якщо скасував |

## Параметри конфігурації

Відповідають параметрам `Modal` в Ant Design, основні поля включають:

| Параметр | Тип | Опис |
|------|------|------|
| `title` | `ReactNode` | Заголовок |
| `content` | `ReactNode` | Вміст |
| `okText` | `string` | Текст кнопки підтвердження |
| `cancelText` | `string` | Текст кнопки скасування (тільки для `confirm`) |
| `onOk` | `() => void \| Promise<void>` | Виконується при натисканні OK |
| `onCancel` | `() => void` | Виконується при натисканні "Скасувати" |

## Зв'язок із ctx.message та ctx.openView

| Призначення | Рекомендоване використання |
|------|----------|
| **Легке тимчасове повідомлення** | `ctx.message`, зникає автоматично |
| **Модальне вікно інфо/успіху/помилки/попередження** | `ctx.modal.info` / `success` / `error` / `warning` |
| **Повторне підтвердження (потребує вибору)** | `ctx.modal.confirm`, використовується разом із `ctx.exit()` для керування процесом |
| **Складні взаємодії (форми, списки тощо)** | `ctx.openView` для відкриття кастомного представлення (сторінка/панель/модальне вікно) |

## Приклади

### Просте інформаційне вікно

```ts
ctx.modal.info({
  title: 'Підказка',
  content: 'Операцію завершено',
});
```

### Вікно підтвердження та керування процесом

```ts
const confirmed = await ctx.modal.confirm({
  title: 'Підтвердити видалення',
  content: 'Ви впевнені, що хочете видалити цей запис?',
  okText: 'Підтвердити',
  cancelText: 'Скасувати',
});
if (!confirmed) {
  ctx.exit();  // Переривання наступних кроків, якщо користувач скасував дію
  return;
}
await ctx.runAction('destroy', { filterByTk: ctx.record?.id });
```

### Вікно підтвердження з onOk

```ts
await ctx.modal.confirm({
  title: 'Підтвердити відправку',
  content: 'Після відправки зміни не можна буде редагувати. Продовжити?',
  async onOk() {
    await ctx.form.submit();
  },
});
```

### Повідомлення про помилку

```ts
try {
  await someOperation();
  ctx.modal.success({ title: 'Успішно', content: 'Операцію завершено' });
} catch (e) {
  ctx.modal.error({ title: 'Помилка', content: e.message });
}
```
