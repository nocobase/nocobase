# ctx.blockModel

Модель батьківського блоку (екземпляр BlockModel), у якому розташоване поточне JS-поле / JS-блок. У сценаріях JSField, JSItem, JSColumn тощо `ctx.blockModel` вказує на блок форми або таблиці, що містить поточну JS-логіку; у незалежному JSBlock він може бути `null` або збігатися з `ctx.model`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSField** | Доступ до `form`, `колекція` (collection) та `resource` батьківського блоку форми всередині поля форми для реалізації зв'язків або валідації. |
| **JSItem** | Доступ до ресурсу та інформації про колекцію батьківського блоку таблиці/форми всередині елемента підтаблиці. |
| **JSColumn** | Доступ до `resource` (наприклад, `getSelectedRows`) та `колекція` (collection) батьківського блоку таблиці всередині стовпця таблиці. |
| **Дії форми / Потік подій** | Доступ до `form` для валідації перед відправкою, `resource` для оновлення тощо. |

> Примітка: `ctx.blockModel` доступний лише в контекстах RunJS, де існує батьківський блок; у незалежних JSBlock (без батьківської форми чи таблиці) він може бути `null`, тому перед використанням рекомендується перевірка на пусте значення.

## Визначення типу

```ts
blockModel: BlockModel | FormBlockModel | TableBlockModel | CollectionBlockModel | DataBlockModel | null;
```

Конкретний тип залежить від типу батьківського блоку: блоки форми — це переважно `FormBlockModel` або `EditFormModel`, блоки таблиці — переважно `TableBlockModel`.

## Основні властивості

| Властивість | Тип | Опис |
|------|------|------|
| `uid` | `string` | Унікальний ідентифікатор моделі блоку. |
| `collection` | `Collection` | Колекція, прив'язана до поточного блоку. |
| `resource` | `Resource` | Екземпляр ресурсу, що використовується блоком (`SingleRecordResource` / `MultiRecordResource` тощо). |
| `form` | `FormInstance` | Блок форми: екземпляр форми Ant Design, що підтримує `getFieldsValue`, `validateFields`, `setFieldsValue` тощо. |
| `emitter` | `EventEmitter` | Емітер подій, дозволяє прослуховувати `formValuesChange`, `onFieldReset` тощо. |

## Зв'язок із ctx.model та ctx.form

| Потреба | Рекомендоване використання |
|------|----------|
| **Батьківський блок поточного JS** | `ctx.blockModel` |
| **Читання/запис полів форми** | `ctx.form` (еквівалентно `ctx.blockModel?.form`, зручніше у блоках форми) |
| **Модель поточного контексту виконання** | `ctx.model` (модель поля в JSField, модель блоку в JSBlock) |

У JSField `ctx.model` — це модель поля, а `ctx.blockModel` — блок форми або таблиці, що містить це поле; `ctx.form` зазвичай є `ctx.blockModel.form`.

## Приклади

### Таблиця: Отримання вибраних рядків та їх обробка

```ts
const rows = ctx.blockModel?.resource?.getSelectedRows?.() || [];
if (rows.length === 0) {
  ctx.message.warning('Будь ласка, спочатку виберіть дані');
  return;
}
```

### Сценарій форми: Валідація та оновлення

```ts
if (ctx.blockModel?.form) {
  await ctx.blockModel.form.validateFields();
  await ctx.blockModel.resource?.refresh?.();
}
```

### Прослуховування змін форми

```ts
ctx.blockModel?.emitter?.on?.('formValuesChange', (payload) => {
  // Реалізація зв'язку або повторний рендеринг на основі останніх значень форми
});
```

### Тригер повторного рендерингу блоку

```ts
ctx.blockModel?.rerender?.();
```

## Зауваження

- У **незалежному JSBlock** (без батьківського блоку форми чи таблиці) `ctx.blockModel` може бути `null`. Рекомендується використовувати опціональний ланцюжок при доступі до його властивостей: `ctx.blockModel?.resource?.refresh?.()`.
- У **JSField / JSItem / JSColumn** `ctx.blockModel` вказує на блок форми або таблиці, що містить поточне поле; у **JSBlock** це може бути сам блок або блок вищого рівня, залежно від фактичної ієрархії.
- `resource` існує лише в блоках даних; `form` існує лише в блоках форми, блоки таблиці зазвичай не мають `form`.
