# ctx.on()

Підписка на події контексту (такі як зміна значень полів, зміна властивостей, оновлення ресурсів тощо) у RunJS. Події відображаються на власні DOM-події в `ctx.element` або внутрішню шину подій `ctx.resource` залежно від їхнього типу.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSField / JSEditableField** | Відстеження змін значень полів із зовнішніх джерел (форми, зв'язки тощо) для синхронного оновлення інтерфейсу користувача, забезпечуючи двостороннє зв'язування. |
| **JSBlock / JSItem / JSColumn** | Відстеження власних подій у контейнері для реагування на зміни даних або стану. |
| **Пов'язано з ресурсом (resource)** | Відстеження подій життєвого циклу ресурсів, таких як оновлення або збереження, для виконання логіки після оновлення даних. |

## Визначення типу

```ts
on(eventName: string, handler: (event?: any) => void): void;
```

## Поширені події

| Назва події | Опис | Джерело події |
|--------|------|----------|
| `js-field:value-change` | Значення поля змінено ззовні (наприклад, через зв'язок у формі або оновлення значення за замовчуванням) | CustomEvent у `ctx.element`, де `ev.detail` — це нове значення |
| `resource:refresh` | Дані ресурсу оновлено | Шина подій `ctx.resource` |
| `resource:saved` | Збереження ресурсу завершено | Шина подій `ctx.resource` |

> Правила відображення подій: події з префіксом `resource:` проходять через `ctx.resource.on`, тоді як інші зазвичай проходять через DOM-події в `ctx.element` (якщо він існує).

## Приклади

### Двостороннє зв'язування полів (React useEffect + очищення)

```tsx
React.useEffect(() => {
  const handler = (ev) => setValue(ev?.detail ?? '');
  ctx.on?.('js-field:value-change', handler);
  return () => {
    ctx.off?.('js-field:value-change', handler);
  };
}, []);
```

### Пряме відстеження DOM (альтернатива, коли ctx.on недоступний)

```ts
// Коли ctx.on не надається, ви можете використовувати ctx.element безпосередньо
const handler = (ev) => {
  if (selectEl) selectEl.value = String(ev?.detail ?? '');
};
ctx.element?.addEventListener('js-field:value-change', handler);
// Під час очищення: ctx.element?.removeEventListener('js-field:value-change', handler);
```

### Оновлення інтерфейсу після оновлення ресурсу

```ts
ctx.resource?.on('refresh', () => {
  const data = ctx.resource?.getData?.();
  // Оновлення рендерингу на основі даних
});
```

## Взаємодія з ctx.off

- Слухачі, зареєстровані за допомогою `ctx.on`, мають бути видалені у відповідний час через [ctx.off](./off.md), щоб уникнути витоку пам'яті або повторних спрацьовувань.
- У React `ctx.off` зазвичай викликається у функції очищення `useEffect`.
- `ctx.off` може бути відсутнім; рекомендується використовувати опціональний ланцюжок: `ctx.off?.('eventName', handler)`.

## Примітки

1. **Парне скасування**: кожному `ctx.on(eventName, handler)` має відповідати `ctx.off(eventName, handler)`, причому посилання на `handler` повинно бути ідентичним.
2. **Життєвий цикл**: видаляйте слухачів перед розмонтуванням компонента або знищенням контексту, щоб запобігти витоку пам'яті.
3. **Доступність подій**: різні типи контексту підтримують різні події. Детальнішу інформацію дивіться в документації конкретних компонентів.
