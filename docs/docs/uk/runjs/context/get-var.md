# ctx.getVar()

**Асинхронно** зчитує значення змінної з поточного контексту виконання. Розбір змінних відповідає логіці `{{ctx.xxx}}` у SQL та шаблонах, зазвичай вони походять від поточного користувача, поточного запису, параметрів представлення, контексту спливаючого вікна тощо.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSBlock / JSField** | Отримання інформації про поточний запис, користувача, ресурс тощо для рендерингу або логічних операцій. |
| **Правила зв'язку / Потік подій** | Читання `ctx.record`, `ctx.formValues` тощо для перевірки умов. |
| **Формули / Шаблони** | Використання тих самих правил розбору змінних, що й у `{{ctx.xxx}}`. |

## Визначення типів

```ts
getVar(path: string): Promise<any>;
```

| Параметр | Тип | Опис |
|------|------|------|
| `path` | `string` | Шлях до змінної, **обов'язково має починатися з `ctx.`**. Підтримує доступ через крапку та індекси масивів. |

**Значення, що повертається**: `Promise<any>`, необхідно використовувати `await` для отримання розібраного значення; якщо змінна не існує, повертається `undefined`.

> Якщо передати шлях, який не починається з `ctx.`, буде видано помилку: `ctx.getVar(path) expects an expression starting with "ctx.", got: "..."`.

## Поширені шляхи змінних

| Шлях | Опис |
|------|------|
| `ctx.record` | Поточний запис (доступно, коли форма/деталі прив'язані до запису) |
| `ctx.record.id` | Первинний ключ поточного запису |
| `ctx.formValues` | Поточні значення форми (часто використовується в правилах зв'язку та потоках подій; у сценаріях з формами краще використовувати `ctx.form.getFieldsValue()` для читання в реальному часі) |
| `ctx.user` | Поточний авторизований користувач |
| `ctx.user.id` | ID поточного користувача |
| `ctx.user.nickname` | Нікнейм поточного користувача |
| `ctx.user.roles.name` | Назви ролей поточного користувача (масив) |
| `ctx.popup.record` | Запис у спливаючому вікні |
| `ctx.popup.record.id` | Первинний ключ запису у спливаючому вікні |
| `ctx.urlSearchParams` | Параметри запиту URL (розібрані з `?key=value`) |
| `ctx.token` | Поточний API Token |
| `ctx.role` | Поточна роль |

## ctx.getVarInfos()

Отримує **структурну інформацію** (тип, заголовок, дочірні властивості тощо) змінних, які можна розібрати в поточному контексті. Це полегшує дослідження доступних шляхів. Значення, що повертається, є статичним описом на основі `meta` і не містить фактичних значень під час виконання.

### Визначення типів

```ts
getVarInfos(options?: { path?: string | string[]; maxDepth?: number }): Promise<Record<string, any>>;
```

У значенні, що повертається, кожен ключ — це шлях до змінної, а значення — структурна інформація для цього шляху (включаючи `type`, `title`, `properties` тощо).

### Параметри

| Параметр | Тип | Опис |
|------|------|------|
| `path` | `string \| string[]` | Шлях обрізки; збирає структуру змінних лише за цим шляхом. Підтримує `'record'`, `'record.id'`, `'ctx.record'`, `'{{ ctx.record }}'`; масив означає об'єднання кількох шляхів. |
| `maxDepth` | `number` | Максимальна глибина розгортання, за замовчуванням `3`. Якщо `path` не вказано, властивості верхнього рівня мають `depth=1`; якщо вказано, вузол, що відповідає шляху, має `depth=1`. |

### Приклад

```ts
// Отримання структури змінних для record (розгортання до 3 рівнів)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });

// Отримання структури для popup.record
const vars = await ctx.getVarInfos({ path: 'popup.record', maxDepth: 3 });

// Отримання повної структури змінних верхнього рівня (maxDepth=3 за замовчуванням)
const vars = await ctx.getVarInfos();
```

## Різниця з ctx.getValue

| Метод | Сценарій використання | Опис |
|------|----------|------|
| `ctx.getValue()` | Редаговані поля, такі як JSField або JSItem | Синхронно отримує значення **поточного поля**; потребує прив'язки до форми. |
| `ctx.getVar(path)` | Будь-який контекст RunJS | Асинхронно отримує **будь-яку змінну ctx**; шлях повинен починатися з `ctx.`. |

У JSField використовуйте `getValue`/`setValue` для читання/запису поточного поля; використовуйте `getVar` для доступу до інших змінних контексту (таких як `record`, `user`, `formValues`).

## Примітки

- **Шлях повинен починатися з `ctx.`**: наприклад, `ctx.record.id`, інакше буде видано помилку.
- **Асинхронний метод**: ви повинні використовувати `await` для отримання результату, наприклад, `const id = await ctx.getVar('ctx.record.id')`.
- **Змінна не існує**: повертає `undefined`. Ви можете використовувати `??` після результату, щоб встановити значення за замовчуванням: `(await ctx.getVar('ctx.user.nickname')) ?? 'Гість'`.
- **Значення форми**: `ctx.formValues` потрібно отримувати через `await ctx.getVar('ctx.formValues')`; воно не доступне безпосередньо як `ctx.formValues`. У контексті форми краще використовувати `ctx.form.getFieldsValue()` для читання останніх значень у реальному часі.

## Приклади

### Отримання ID поточного запису

```ts
const recordId = await ctx.getVar('ctx.record.id');
if (recordId) {
  ctx.message.info(`Поточний запис: ${recordId}`);
}
```

### Отримання запису у спливаючому вікні

```ts
const recordId = await ctx.getVar('ctx.popup.record.id');
if (recordId) {
  ctx.message.info(`Запис у спливаючому вікні: ${recordId}`);
}
```

### Читання елементів масиву

```ts
const roleNames = await ctx.getVar('ctx.user.roles.name');
// Повертає масив назв ролей, наприклад, ['admin', 'member']
```

### Встановлення значення за замовчуванням

```ts
// getVar не має параметра defaultValue; використовуйте ?? після результату
const userName = (await ctx.getVar('ctx.user.nickname')) ?? 'Гість';
```

### Читання значень полів форми

```ts
// І ctx.formValues, і ctx.form стосуються сценаріїв з формами; використовуйте getVar для читання вкладених полів
const status = await ctx.getVar('ctx.formValues.status');
if (status === 'draft') {
  // ...
}
```

### Читання параметрів запиту URL

```ts
const id = await ctx.getVar('ctx.urlSearchParams.id'); // Відповідає ?id=xxx
```

### Дослідження доступних змінних

```ts
// Отримання структури змінних для record (розгортання до 3 рівнів)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });
// vars виглядає як { 'record.id': { type: 'string', title: 'id' }, ... }
```
