# ctx.route

Інформація про поточну відповідність маршруту, що відповідає концепції `route` у React Router. Використовується для отримання поточної конфігурації маршруту, параметрів тощо. Зазвичай використовується разом із `ctx.router` та `ctx.location`.

## Сценарії застосування

| Сценарій | Опис |
|------|------|
| **JSBlock / JSField** | Виконання умовного рендерингу або відображення ідентифікатора поточної сторінки на основі `route.pathname` або `route.params`. |
| **Правила зв'язку / Потоки подій** | Зчитування параметрів маршруту (наприклад, `params.name`) для логічного розгалуження або передачі дочірнім компонентам. |
| **Навігація представленнями** | Внутрішнє порівняння `ctx.route.pathname` із цільовим шляхом для прийняття рішення про запуск `ctx.router.navigate`. |

> Примітка: `ctx.route` доступний лише в середовищах RunJS, які містять контекст маршрутизації (наприклад, JSBlock на сторінці, сторінки робочих процесів тощо). Він може бути порожнім у чисто бекендових середовищах або контекстах без маршрутизації (наприклад, у робочих процесах).

## Визначення типу

```ts
type RouteOptions = {
  name?: string;   // Унікальний ідентифікатор маршруту
  path?: string;   // Шаблон маршруту (наприклад, /admin/:name)
  params?: Record<string, any>;  // Параметри маршруту (наприклад, { name: 'users' })
  pathname?: string;  // Повний шлях поточного маршруту (наприклад, /admin/users)
};
```

## Основні поля

| Поле | Тип | Опис |
|------|------|------|
| `pathname` | `string` | Повний шлях поточного маршруту, збігається з `ctx.location.pathname`. |
| `params` | `Record<string, any>` | Динамічні параметри, витягнуті з шаблону маршруту, наприклад `{ name: 'users' }`. |
| `path` | `string` | Шаблон маршруту, наприклад `/admin/:name`. |
| `name` | `string` | Унікальний ідентифікатор маршруту, часто використовується в сценаріях з кількома вкладками або представленнями. |

## Зв'язок із ctx.router та ctx.location

| Призначення | Рекомендоване використання |
|------|----------|
| **Зчитування поточного шляху** | `ctx.route.pathname` або `ctx.location.pathname`; обидва збігаються під час зіставлення. |
| **Зчитування параметрів маршруту** | `ctx.route.params`, наприклад, `params.name` представляє UID поточної сторінки. |
| **Навігація** | `ctx.router.navigate(path)` |
| **Зчитування параметрів запиту, стану (state)** | `ctx.location.search`, `ctx.location.state` |

`ctx.route` зосереджений на «конфігурації знайденого маршруту», тоді як `ctx.location` — на «поточному розташуванні URL». Разом вони забезпечують повний опис стану маршрутизації.

## Приклади

### Зчитування pathname

```ts
// Відображення поточного шляху
ctx.message.info('Поточна сторінка: ' + ctx.route.pathname);
```

### Розгалуження на основі params

```ts
// params.name зазвичай є UID поточної сторінки (наприклад, ідентифікатор сторінки робочого процесу)
if (ctx.route.params?.name === 'users') {
  // Виконати специфічну логіку на сторінці керування користувачами
}
```

### Відображення на сторінці робочого процесу (Flow)

```tsx
<div>
  <h1>Поточна сторінка - {ctx.route.pathname}</h1>
  <p>Ідентифікатор маршруту: {ctx.route.params?.name}</p>
</div>
```

## Пов'язані теми

- [ctx.router](./router.md): Навігація маршрутами. Коли `ctx.router.navigate()` змінює шлях, `ctx.route` оновлюється відповідно.
- [ctx.location](./location.md): Поточне розташування URL (pathname, search, hash, state), використовується разом із `ctx.route`.
