# ctx.off()

Видаляє обробники подій, зареєстровані за допомогою `ctx.on(eventName, handler)`. Часто використовується разом із [ctx.on](./on.md) для скасування підписки у відповідний момент, щоб уникнути витоків пам'яті або повторних спрацьовувань.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **Очищення React useEffect** | Викликається у функції очищення (cleanup) `useEffect` для видалення обробників під час розмонтування компонента. |
| **JSField / JSEditableField** | Скасування підписки на `js-field:value-change` під час двостороннього зв'язування даних для полів. |
| **Робота з ресурсами** | Скасування підписки на такі події, як `refresh` або `saved`, зареєстровані через `ctx.resource.on`. |

## Визначення типу

```ts
off(eventName: string, handler: (event?: any) => void): void;
```

## Приклади

### Парне використання в React useEffect

```tsx
React.useEffect(() => {
  const handler = (ev) => setValue(ev?.detail ?? '');
  ctx.on('js-field:value-change', handler);
  return () => ctx.off('js-field:value-change', handler);
}, []);
```

### Скасування підписки на події ресурсів

```ts
const handler = () => { /* ... */ };
ctx.resource?.on('refresh', handler);
// У відповідний момент
ctx.resource?.off('refresh', handler);
```

## Примітки

1. **Ідентичність посилання на handler**: `handler`, що передається в `ctx.off`, повинен бути тим самим посиланням, що і в `ctx.on`; інакше його не вдасться правильно видалити.
2. **Своєчасне очищення**: викликайте `ctx.off` перед розмонтуванням компонента або знищенням контексту, щоб уникнути витоків пам'яті.
