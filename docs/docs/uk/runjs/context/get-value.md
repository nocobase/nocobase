# ctx.getValue()

У сценаріях редагованих полів, таких як JSField та JSItem, використовуйте цей метод для отримання останнього значення поточного поля. У поєднанні з `ctx.setValue(v)` він забезпечує двостороннє зв'язування з формою.

## Сценарії використання

| Сценарій | Опис |
|------|------|
| **JSField** | Читання вводу користувача або поточного значення форми в редагованих користувацьких полях. |
| **JSItem** | Читання значення поточної комірки в редагованих елементах таблиць/підтаблиць. |
| **JSColumn** | Читання значення поля відповідного рядка під час рендерингу стовпця таблиці. |

> **Примітка**: `ctx.getValue()` доступний лише в контекстах RunJS із прив'язкою до форми; він відсутній у сценаріях без прив'язки до полів, таких як робочі процеси або правила взаємодії.

## Визначення типу

```ts
getValue<T = any>(): T | undefined;
```

- **Значення, що повертається**: Поточне значення поля, тип якого визначається типом елемента форми поля; може бути `undefined`, якщо поле не зареєстроване або не заповнене.

## Порядок отримання значення

`ctx.getValue()` отримує значення в такому порядку:

1. **Стан форми**: Пріоритетне читання з поточного стану Ant Design Form.
2. **Резервне значення**: Якщо поле відсутнє у формі, використовується початкове значення поля або props.

> Якщо форма ще не завершила рендеринг або поле не зареєстроване, може повернутися `undefined`.

## Приклади

### Рендеринг на основі поточного значення

```ts
const current = ctx.getValue();
if (current == null || current === '') {
  ctx.render(<span>Будь ласка, спочатку введіть вміст</span>);
} else {
  ctx.render(<span>Поточне значення: {current}</span>);
}
```

### Двостороннє зв'язування з використанням setValue

```tsx
const { Input } = ctx.libs.antd;

// Читання поточного значення як значення за замовчуванням
const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```
