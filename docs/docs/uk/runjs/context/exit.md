# ctx.exit()

Завершує виконання поточного потоку подій; наступні кроки не будуть виконані. Зазвичай використовується, коли бізнес-умови не виконуються, користувач скасовує дію або виникає невиправна помилка.

## Сценарії використання

`ctx.exit()` зазвичай використовується в наступних контекстах, де можна виконувати JS:

| Сценарій | Опис |
|------|------|
| **Потік подій** | У потоках подій, що ініціюються надсиланням форм, натисканням кнопок тощо, завершує наступні кроки, якщо умови не виконуються. |
| **Правила зв'язку** | У зв'язках полів, фільтрів тощо, завершує поточний потік подій, якщо валідація не пройдена або потрібно пропустити виконання. |
| **Події дій** | У користувацьких діях (наприклад, підтвердження видалення, попередня валідація перед збереженням), вихід відбувається, коли користувач скасовує дію або валідація не пройдена. |

> Відмінність від `ctx.exitAll()`: `ctx.exit()` завершує лише поточний потік подій; інші потоки подій у межах тієї ж події залишаються незмінними. `ctx.exitAll()` завершує поточний потік подій, а також усі наступні потоки подій у межах тієї ж події, які ще не були виконані.

## Визначення типу

```ts
exit(): never;
```

Виклик `ctx.exit()` генерує внутрішній виняток `FlowExitException`, який перехоплюється рушієм потоку подій для зупинки виконання поточного потоку. Після виклику решта інструкцій у поточному коді JS не виконуватиметься.

## Порівняння з ctx.exitAll()

| Метод | Область дії |
|------|----------|
| `ctx.exit()` | Завершує лише поточний потік подій; наступні потоки подій не зачіпаються. |
| `ctx.exitAll()` | Завершує поточний потік подій та перериває наступні потоки подій у межах тієї ж події, які налаштовані на **послідовне виконання**. |

## Приклади

### Вихід при скасуванні користувачем

```ts
// У вікні підтвердження, якщо користувач натискає «Скасувати», потік подій завершується
if (!confirmed) {
  ctx.message.info('Операцію скасовано');
  ctx.exit();
}
```

### Вихід при помилці валідації параметрів

```ts
// Виведення повідомлення та завершення, якщо валідація не пройдена
if (!params.value || params.value.length < 3) {
  ctx.message.error('Некоректні параметри, довжина має бути не менше 3');
  ctx.exit();
}
```

### Вихід, якщо бізнес-умови не виконуються

```ts
// Завершення, якщо умови не виконуються; наступні кроки не будуть виконані
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'Можна надсилати лише чернетки' });
  ctx.exit();
}
```

### Вибір між ctx.exit() та ctx.exitAll()

```ts
// Потрібно вийти лише з поточного потоку подій → використовуйте ctx.exit()
if (!params.valid) {
  ctx.message.error('Некоректні параметри');
  ctx.exit();  // Інші потоки подій не зачіпаються
}

// Потрібно завершити всі наступні потоки подій у межах поточної події → використовуйте ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'Недостатньо прав' });
  ctx.exitAll();  // Основний потік подій та наступні потоки тієї ж події завершуються одночасно
}
```

### Вихід на основі вибору користувача після підтвердження в модальному вікні

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'Підтвердити видалення',
  content: 'Цю дію неможливо скасувати. Продовжити?',
});
if (!ok) {
  ctx.message.info('Скасовано');
  ctx.exit();
}
```

## Примітки

- Після виклику `ctx.exit()` наступний код у поточному JS не виконуватиметься; рекомендується перед викликом пояснити причину користувачеві за допомогою `ctx.message`, `ctx.notification` або модального вікна.
- У бізнес-коді зазвичай немає потреби перехоплювати `FlowExitException`; залиште це рушію потоку подій.
- Якщо вам потрібно завершити всі наступні потоки подій у межах поточної події, використовуйте `ctx.exitAll()`.
