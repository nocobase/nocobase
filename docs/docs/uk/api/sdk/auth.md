# Auth

## Огляд

Клас `Auth` переважно використовується на стороні клієнта для доступу до інформації про користувача та надсилання запитів до API, пов'язаних з автентифікацією користувача.

## Властивості екземпляра

### `locale`

Мова, яку використовує поточний користувач.

### `role`

Роль, яку використовує поточний користувач.

### `token`

`token` для API.

### `authenticator`

Автентифікатор, що використовується для автентифікації поточного користувача. Дивіться [Автентифікація користувача](/auth-verification/auth/).

## Методи класу

### `signIn()`

Вхід користувача.

#### Підпис

- `async signIn(values: any, authenticator?: string): Promise<AxiosResponse<any>>`

#### Деталі

| Назва параметра | Тип      | Опис                                          |
| :--------------- | :------- | :-------------------------------------------- |
| `values`         | `any`    | Параметри запиту для API входу                |
| `authenticator`  | `string` | Ідентифікатор автентифікатора, що використовується для входу |

### `signUp()`

Реєстрація користувача.

#### Підпис

- `async signUp(values: any, authenticator?: string): Promise<AxiosResponse<any>>`

#### Деталі

| Назва параметра | Тип      | Опис                                          |
| :--------------- | :------- | :-------------------------------------------- |
| `values`         | `any`    | Параметри запиту для API реєстрації           |
| `authenticator`  | `string` | Ідентифікатор автентифікатора, що використовується для реєстрації |

### `signOut()`

Вихід із системи.

#### Підпис

- `async signOut(values: any, authenticator?: string): Promise<AxiosResponse<any>>`

#### Деталі

| Назва параметра | Тип      | Опис                                          |
| :--------------- | :------- | :-------------------------------------------- |
| `values`         | `any`    | Параметри запиту для API виходу               |
| `authenticator`  | `string` | Ідентифікатор автентифікатора, що використовується для виходу |
