# Процедури обслуговування

## Перший запуск застосунку

Під час першого запуску застосунку спершу запустіть один вузол. Дочекайтеся, доки плагіни встановляться та активуються, а потім запустіть інші вузли.

## Оновлення версії

Коли потрібно оновити версію NocoBase, дотримуйтесь цієї процедури.

:::warning{title=Увага}
У **виробничому середовищі** кластера такі функції, як керування плагінами та оновлення версії, слід використовувати з обережністю або заборонити.

NocoBase наразі не підтримує онлайн-оновлення для кластерних версій. Щоб забезпечити узгодженість даних, під час процесу оновлення необхідно призупинити зовнішні сервіси.
:::

Кроки:

1.  Зупиніть поточний сервіс

    Зупиніть усі екземпляри застосунку NocoBase та перенаправте трафік балансувальника на сторінку стану 503.

2.  Створіть резервну копію даних

    Перед оновленням наполегливо рекомендується створити резервну копію даних бази даних, щоб запобігти будь-яким проблемам під час процесу оновлення.

3.  Оновіть версію

    Зверніться до розділу [Оновлення Docker](../get-started/upgrading/docker), щоб оновити версію образу застосунку NocoBase.

4.  Запустіть сервіс

    1.  Запустіть один вузол у кластері та дочекайтеся завершення оновлення та успішного запуску.
    2.  Перевірте правильність функціоналу. Якщо виникнуть проблеми, які не вдається вирішити шляхом усунення несправностей, ви можете відкотитися до попередньої версії.
    3.  Запустіть інші вузли.
    4.  Перенаправте трафік балансувальника на кластер застосунків.

## Обслуговування в застосунку

Обслуговування в застосунку означає виконання операцій, пов'язаних з обслуговуванням, поки застосунок працює, включаючи:

*   Керування плагінами (встановлення, активація, деактивація плагінів тощо)
*   Резервне копіювання та відновлення
*   Керування міграцією середовища

Кроки:

1.  Зменшіть кількість вузлів

    Зменшіть кількість вузлів, на яких працює застосунок у кластері, до одного, а інші вузли зупиніть.

2.  Виконайте операції з обслуговування в застосунку, такі як встановлення та активація плагінів, резервне копіювання даних тощо.

3.  Відновіть вузли

    Після завершення операцій з обслуговування та перевірки правильності функціоналу запустіть інші вузли. Після успішного запуску вузлів відновіть робочий стан кластера.
