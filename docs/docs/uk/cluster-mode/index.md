# Кластерний режим

## Вступ

Починаючи з версії v1.6.0, NocoBase підтримує запуск застосунків у кластерному режимі. Коли застосунок працює в кластерному режимі, він може значно покращити свою продуктивність при обробці одночасних запитів, використовуючи кілька екземплярів та багатоядерний режим.

## Архітектура системи

![20251031221530](https://static-docs.nocobase.com/20251031221530.png)

*   **Кластер застосунків**: Кластер, що складається з кількох екземплярів застосунку NocoBase. Його можна розгорнути на кількох серверах або запустити як кілька процесів у багатоядерному режимі на одному сервері.
*   **База даних**: Зберігає дані застосунку. Це може бути однонодова або розподілена база даних.
*   **Спільне сховище**: Використовується для зберігання файлів та даних застосунку, підтримуючи доступ для читання/запису з кількох екземплярів.
*   **Проміжне програмне забезпечення (Middleware)**: Включає такі компоненти, як кеш, сигнали синхронізації, черги повідомлень та розподілені блокування, для підтримки зв'язку та координації всередині кластера застосунків.
*   **Балансувальник навантаження**: Відповідає за розподіл клієнтських запитів між різними екземплярами в кластері застосунків, а також за виконання перевірок стану та відмовостійкості.
