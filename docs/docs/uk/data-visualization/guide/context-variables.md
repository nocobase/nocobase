# Використання змінних контексту

За допомогою змінних контексту ви можете повторно використовувати інформацію з поточної сторінки, дані користувача, час, умови фільтрації тощо, щоб відображати діаграми та забезпечувати їхню взаємодію на основі контексту.

## Сфера застосування
- У запитах даних у режимі Конструктора: вибирайте змінні для умов фільтрації.
![clipboard-image-1761486073](https://static-docs.nocobase.com/clipboard-image-1761486073.png)

- У запитах даних у режимі SQL: при написанні запитів вибирайте змінні та вставляйте вирази (наприклад, `{{ ctx.user.id }}`).
![clipboard-image-1761486145](https://static-docs.nocobase.com/clipboard-image-1761486145.png)

- У параметрах діаграм у режимі Custom (Користувацький): безпосередньо пишіть вирази JavaScript.
![clipboard-image-1761486604](https://static-docs.nocobase.com/clipboard-image-1761486604.png)

- У подіях взаємодії (наприклад, натискання для відкриття вікна деталізації та передачі даних): безпосередньо пишіть вирази JavaScript.
![clipboard-image-1761486683](https://static-docs.nocobase.com/clipboard-image-1761486683.png)

**Примітка:**
- Не обгортайте `{{ ... }}` одинарними або подвійними лапками; система безпечно обробляє прив'язку на основі типу змінної (рядок, число, час, NULL).
- Коли змінна має значення `NULL` або не визначена, явно обробляйте нульові значення в SQL за допомогою `COALESCE(...)` або `IS NULL`.
