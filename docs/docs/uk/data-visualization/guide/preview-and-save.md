# Попередній перегляд та збереження

*   **Попередній перегляд**: Тимчасово відображає зміни з панелі конфігурації на діаграмі сторінки для перевірки результату.
*   **Збереження**: Фактично зберігає зміни з панелі конфігурації в базу даних.

## Точки доступу

![clipboard-image-1761479218](https://static-docs.nocobase.com/clipboard-image-1761479218.png)

-   У візуальному режимі (Basic) усі зміни за замовчуванням автоматично застосовуються до попереднього перегляду.
-   У режимах SQL та Custom, після внесення змін, ви можете натиснути кнопку **Попередній перегляд** праворуч, щоб застосувати їх.
-   У нижній частині всієї панелі конфігурації доступна єдина кнопка **Попередній перегляд**.

## Поведінка попереднього перегляду
-   Тимчасово відображає конфігурацію на сторінці, але не записує її в базу даних. Після оновлення сторінки або скасування операції результат попереднього перегляду не зберігається.
-   Вбудований механізм усунення деренчання (debounce): якщо протягом короткого часу відбувається кілька оновлень, виконується лише останнє, щоб уникнути частих запитів.
-   Повторне натискання кнопки **Попередній перегляд** перезапише попередній результат.

## Повідомлення про помилки
-   Помилки запиту або збої валідації: відображаються в області **Переглянути дані**.
-   Помилки конфігурації діаграми (відсутнє Basic зіставлення, винятки від Custom JS): відображаються в області діаграми або консолі, зберігаючи сторінку працездатною.
-   Щоб ефективно зменшити кількість помилок, спочатку підтвердьте назви стовпців та типи даних у розділі **Переглянути дані**, а потім виконуйте зіставлення полів або пишіть Custom код.

## Збереження та скасування
-   **Зберегти**: Записує поточні зміни з панелі в конфігурацію блоку та негайно застосовує їх до сторінки.
-   **Скасувати**: Відхиляє поточні незбережені зміни та повертає до останнього збереженого стану.
-   **Область збереження**:
    -   **Запит даних**: Параметри конструктора (Builder); у режимі SQL також зберігається текст SQL.
    -   **Опції діаграми**: Тип Basic, зіставлення полів та властивості; текст Custom JS.
    -   **Інтерактивні події**: Текст JS події та логіка прив'язки.
-   Після збереження блок стає доступним для всіх відвідувачів (залежить від налаштувань дозволів сторінки).

## Рекомендований порядок дій
-   Налаштуйте запит даних → Запустіть запит → Перегляньте дані, щоб підтвердити назви стовпців та типи → Налаштуйте опції діаграми для зіставлення основних полів → Виконайте попередній перегляд для перевірки → Збережіть для застосування.
