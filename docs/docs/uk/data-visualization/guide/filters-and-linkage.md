# Фільтри сторінки та взаємодія

Фільтр сторінки (блок фільтра) використовується для уніфікованого введення умов фільтрації на рівні сторінки та об'єднання їх із запитами діаграм, що забезпечує послідовну фільтрацію та взаємодію між кількома діаграмами.

## Огляд функцій
- Додайте блок фільтра на сторінку, щоб забезпечити уніфіковану точку входу для фільтрації для всіх діаграм поточної сторінки.
- Використовуйте кнопки «Фільтрувати», «Скинути» та «Згорнути» для застосування, очищення та згортання.
- Якщо у фільтрі вибрано поля, пов'язані з діаграмою, їхні значення автоматично об'єднуються із запитом діаграми та запускають оновлення діаграми.
- Фільтри також можуть створювати власні поля та реєструвати їх у контекстних змінних, щоб їх можна було посилати в діаграмах, таблицях, формах та інших блоках даних.

## Використання значень фільтра сторінки в запитах діаграм
- Режим Builder (рекомендовано)
  - Автоматичне об'єднання: Якщо джерело даних та колекція збігаються, вам не потрібно додатково писати змінні в запиті діаграми; фільтри сторінки об'єднуються за допомогою `$and`.
  - Ручний вибір: Ви також можете активно вибирати значення «власних полів» фільтра сторінки в умовах фільтрації діаграми.

- Режим SQL (через ін'єкцію змінних)
  - У SQL-запиті використовуйте «Вибрати змінну» для вставки значень «власних полів» фільтра сторінки.
