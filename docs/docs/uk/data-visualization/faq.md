# Часті запитання

## Вибір діаграм
### Як вибрати відповідну діаграму?
Відповідь: Вибирайте, виходячи з ваших даних та цілей:
- Тенденції та зміни: лінійна або площинна діаграма
- Порівняння значень: стовпчаста або гістограма
- Структура частки: кругова або кільцева діаграма
- Кореляція або розподіл: точкова діаграма
- Ієрархія або прогрес етапів: воронкоподібна діаграма

Більше типів діаграм дивіться у [прикладах ECharts](https://echarts.apache.org/examples).

### Які типи діаграм підтримує NocoBase?
Відповідь: Режим візуального налаштування включає поширені діаграми (лінійна, площинна, стовпчаста, гістограма, кругова, кільцева, воронкоподібна, точкова тощо). Режим користувацького налаштування підтримує всі типи діаграм ECharts.

## Запити даних
### Чи взаємодіють режими візуального налаштування та SQL-налаштування?
Відповідь: Ні, вони не взаємодіють. Їхні конфігурації зберігаються окремо. Режим, використаний під час останнього збереження, набуває чинності.

## Параметри діаграм
### Як налаштувати поля діаграми?
Відповідь: У режимі візуального налаштування виберіть поля даних відповідно до типу діаграми. Наприклад, для лінійних або стовпчастих діаграм потрібні поля для осі X та осі Y; для кругових діаграм потрібні поля для категорії та значення.
Рекомендуємо спочатку виконати "Запустити запит", щоб перевірити, чи відповідають дані очікуванням. За замовчуванням поля діаграми зіставляються автоматично.

## Попередній перегляд та збереження
### Чи потрібно вручну переглядати зміни після налаштування?
Відповідь: У режимі візуального налаштування зміни автоматично відображаються в попередньому перегляді. У режимах SQL та користувацького налаштування, щоб уникнути частого оновлення, завершіть редагування та натисніть "Попередній перегляд" вручну.

### Чому попередній перегляд діаграми зникає після закриття спливаючого вікна?
Відповідь: Ефект попереднього перегляду призначений лише для тимчасового перегляду. Після внесення змін, будь ласка, спочатку збережіть їх, а потім закрийте вікно. Незбережені зміни не зберігаються.
