# Колекція виразів

## Створення шаблонної колекції "Вирази"

Перш ніж використовувати вузли операцій з динамічними виразами у робочому процесі, вам потрібно спочатку створити шаблонну колекцію "Вирази" за допомогою інструменту керування колекціями. Ця колекція слугуватиме сховищем для різних виразів:

![Створення колекції виразів](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## Введення даних виразів

Потім ви можете створити табличний блок і додати кілька записів формул до цієї шаблонної колекції. Кожен рядок у шаблонній колекції "Вирази" можна розглядати як правило обчислення, розроблене для певної моделі даних у межах колекції. Ви можете використовувати різні поля з моделей даних різних колекцій як змінні, створюючи унікальні вирази як правила обчислення. Звісно, ви також можете використовувати різні обчислювальні механізми.

![Введення даних виразів](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=Порада}
Після створення формул їх необхідно пов'язати з бізнес-даними. Пряме зв'язування кожного рядка бізнес-даних з даними формул може бути досить виснажливим. Тому зазвичай використовують колекцію метаданих (наприклад, колекцію класифікації) для створення зв'язку "багато до одного" (або "один до одного") з колекцією формул. Потім бізнес-дані пов'язуються з цими класифікованими метаданими у зв'язку "багато до одного". Такий підхід дозволяє вам просто вказати відповідні класифіковані метадані під час створення бізнес-даних, що значно спрощує пошук та використання відповідних даних формул через встановлений шлях зв'язку.
:::

## Завантаження відповідних даних у процес

Наприклад, давайте створимо робочий процес, який запускається подією колекції. Коли створюється замовлення, цей тригер повинен попередньо завантажити пов'язані дані про товар, а також дані виразів, що стосуються цього товару:

![Подія колекції_Конфігурація тригера](https://static-docs.nocobase.com/f181f75b10007afd5de068f3458d2e04.png)
