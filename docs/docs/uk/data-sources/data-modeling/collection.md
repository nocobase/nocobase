# Огляд колекцій

NocoBase пропонує унікальну DSL для опису структури даних, відому як колекція. Вона уніфікує структуру даних з різних джерел, забезпечуючи надійну основу для подальшого керування даними, їх аналізу та використання в застосунках.

![20240512161522](https://static-docs.nocobase.com/20240512161522.png)

Для зручного використання різноманітних моделей даних NocoBase підтримує створення різних типів колекцій:

- [Звичайна колекція](/data-sources/data-source-main/general-collection): Має вбудовані стандартні системні поля;
- [Колекція-спадкоємець](/data-sources/data-source-main/inheritance-collection): Ви можете створити батьківську колекцію, а потім вивести з неї дочірню. Дочірня колекція успадкує структуру батьківської та зможе визначати власні стовпці.
- [Деревоподібна колекція](/data-sources/collection-tree): Колекція з деревоподібною структурою, наразі підтримує лише дизайн списку суміжності;
- [Календарна колекція](/data-sources/calendar/calendar-collection): Використовується для створення колекцій подій, пов'язаних з календарем;
- [Файлова колекція](/data-sources/file-manager/file-collection): Використовується для керування зберіганням файлів;
- []: Використовується для сценаріїв динамічних виразів у робочих процесах;
- [SQL-колекція](/data-sources/collection-sql): Не є фактичною колекцією бази даних, але швидко та структуровано відображає SQL-запити;
- [Колекція-представлення](/data-sources/collection-view): Підключається до існуючих представлень бази даних;
- [Зовнішня колекція](/data-sources/collection-fdw): Дозволяє системі бази даних безпосередньо отримувати доступ та запитувати дані із зовнішніх джерел даних, базуючись на технології FDW.
