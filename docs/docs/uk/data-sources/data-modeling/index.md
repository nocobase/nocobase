# Огляд

Моделювання даних — це ключовий крок при проєктуванні баз даних, що передбачає глибокий аналіз та абстрагування різних типів даних реального світу та їхніх взаємозв'язків. У цьому процесі ми прагнемо виявити внутрішні зв'язки між даними та формалізувати їх у моделі даних, закладаючи основу для структури бази даних інформаційної системи. NocoBase — це платформа, керована моделями даних, яка має такі особливості:

## Підтримка доступу до даних з різних джерел

Джерелами даних NocoBase можуть бути різні поширені бази даних, платформи API (SDK) та файли.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

NocoBase надає [плагін керування джерелами даних](/data-sources/data-source-manager) для керування різними джерелами даних та їхніми колекціями. Плагін керування джерелами даних лише надає інтерфейс для керування всіма джерелами даних, але не забезпечує можливість безпосереднього підключення до них. Він має використовуватися в поєднанні з різними плагінами джерел даних. Наразі підтримуються такі джерела даних:

- [Основна база даних](/data-sources/data-source-main): Основна база даних NocoBase, що підтримує реляційні бази даних, такі як MySQL, PostgreSQL та MariaDB.
- [KingbaseES](/data-sources/data-source-kingbase): Використовує базу даних KingbaseES як джерело даних, може бути як основною, так і зовнішньою базою даних.
- [Зовнішня MySQL](/data-sources/data-source-external-mysql): Використовує зовнішню базу даних MySQL як джерело даних.
- [Зовнішня MariaDB](/data-sources/data-source-external-mariadb): Використовує зовнішню базу даних MariaDB як джерело даних.
- [Зовнішня PostgreSQL](/data-sources/data-source-external-postgres): Використовує зовнішню базу даних PostgreSQL як джерело даних.
- [Зовнішня MSSQL](/data-sources/data-source-external-mssql): Використовує зовнішню базу даних MSSQL (SQL Server) як джерело даних.
- [Зовнішня Oracle](/data-sources/data-source-external-oracle): Використовує зовнішню базу даних Oracle як джерело даних.

## Надає різноманітні інструменти для моделювання даних

**Простий інтерфейс керування колекціями**: Використовується для створення різних моделей (колекцій) або підключення до існуючих.

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**Візуальний інтерфейс у стилі ER-діаграм**: Використовується для вилучення сутностей та їхніх взаємозв'язків з вимог користувачів та бізнесу. Він надає інтуїтивно зрозумілий та легкий для сприйняття спосіб опису моделей даних. Завдяки ER-діаграмам ви можете чіткіше зрозуміти основні сутності даних у системі та їхні зв'язки.

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## Підтримка різних типів колекцій

| Колекція | Опис |
| - | - |
| [Загальна колекція](/data-sources/data-source-main/general-collection) | Містить вбудовані стандартні системні поля |
| [Календарна колекція](/data-sources/calendar/calendar-collection) | Використовується для створення колекцій подій, пов'язаних з календарем |
| Колекція коментарів | Використовується для зберігання коментарів або відгуків про дані |
| [Деревоподібна колекція](/data-sources/collection-tree) | Колекція з деревоподібною структурою, наразі підтримує лише модель списку суміжності |
| [Колекція файлів](/data-sources/file-manager/file-collection) | Використовується для керування файловим сховищем |
| [SQL-колекція](/data-sources/collection-sql) | Не є фактичною таблицею бази даних, а візуалізує SQL-запити у структурованому вигляді |
| [Підключення до подання бази даних](/data-sources/collection-view) | Підключається до існуючих подань бази даних |
| Колекція виразів | Використовується для сценаріїв динамічних виразів у робочих процесах |
| [Підключення до зовнішніх даних](/data-sources/collection-fdw) | Дозволяє системі баз даних безпосередньо отримувати доступ та запитувати дані із зовнішніх джерел даних на основі технології FDW. |
