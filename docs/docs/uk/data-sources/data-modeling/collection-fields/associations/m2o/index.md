# Багато до одного

У базі даних бібліотеки є дві сутності: книги та автори. Один автор може написати багато книг, але кожна книга зазвичай має лише одного автора. У цьому випадку зв'язок між авторами та книгами є багато до одного. Багато книг можуть бути пов'язані з одним і тим же автором, але кожна книга може мати лише одного автора.

## Опис параметрів

### Source collection

Вихідна колекція, тобто колекція, в якій знаходиться поточне поле.

### Target collection

Цільова колекція, тобто колекція, з якою потрібно встановити зв'язок.

### Foreign key

Поле у вихідній колекції, яке використовується для встановлення зв'язку між двома колекціями.

### Target key

Поле в цільовій колекції, на яке посилається зовнішній ключ. Воно повинно бути унікальним.

### ON DELETE

ON DELETE визначає правила, які застосовуються до посилань зовнішнього ключа в пов'язаних дочірніх колекціях при видаленні записів у батьківській колекції. Це опція, яка використовується при визначенні обмеження зовнішнього ключа. Поширені опції ON DELETE включають:

- CASCADE: При видаленні запису в батьківській колекції всі пов'язані записи в дочірній колекції автоматично видаляються.
- SET NULL: При видаленні запису в батьківській колекції значення зовнішнього ключа в пов'язаних записах дочірньої колекції встановлюються на NULL.
- RESTRICT: Опція за замовчуванням, запобігає видаленню запису батьківської колекції, якщо існують пов'язані записи в дочірній колекції.
- NO ACTION: Схоже на RESTRICT, запобігає видаленню запису батьківської колекції, якщо існують пов'язані записи в дочірній колекції.
