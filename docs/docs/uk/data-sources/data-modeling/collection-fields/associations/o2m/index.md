# Зв'язок «один-до-багатьох»

Відносини між класом та його учнями є прикладом зв'язку «один-до-багатьох»: один клас може мати багато учнів, але кожен учень належить лише до одного класу.

## Опис параметрів

### Вихідна колекція

Колекція, в якій знаходиться поточне поле.

### Цільова колекція

Колекція, з якою буде встановлено зв'язок.

### Вихідний ключ

Поле у вихідній колекції, на яке посилається зовнішній ключ. Воно має бути унікальним.

### Зовнішній ключ

Поле в цільовій колекції, яке використовується для встановлення зв'язку між двома колекціями.

### Цільовий ключ

Поле в цільовій колекції, що використовується для перегляду кожного запису рядка в блоці зв'язку, зазвичай це унікальне поле.

### ON DELETE

`ON DELETE` визначає правила, які застосовуються до посилань на зовнішні ключі в пов'язаних дочірніх колекціях при видаленні записів у батьківській колекції. Це опція, що використовується при визначенні обмеження зовнішнього ключа. Поширені опції `ON DELETE` включають:

- **CASCADE**: При видаленні запису в батьківській колекції автоматично видаляються всі пов'язані записи в дочірній колекції.
- **SET NULL**: При видаленні запису в батьківській колекції значення зовнішнього ключа в пов'язаних записах дочірньої колекції встановлюються на `NULL`.
- **RESTRICT**: Опція за замовчуванням, яка запобігає видаленню запису батьківської колекції, якщо існують пов'язані записи в дочірній колекції.
- **NO ACTION**: Подібно до `RESTRICT`, ця опція запобігає видаленню запису батьківської колекції, якщо існують пов'язані записи в дочірній колекції.
