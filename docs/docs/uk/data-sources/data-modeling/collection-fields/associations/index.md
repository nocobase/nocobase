# Поля зв'язків

У NocoBase поля зв'язків не є справжніми полями, а використовуються для встановлення зв'язків між колекціями. Ця концепція еквівалентна зв'язкам у реляційних базах даних.

У реляційних базах даних існують такі основні типи зв'язків:

- [Один-до-одного](./o2o/index.md): Кожна сутність в одній колекції відповідає лише одній сутності в іншій колекції. Цей тип зв'язку зазвичай використовується для зберігання різних аспектів сутності в окремих колекціях, щоб зменшити надмірність і підвищити узгодженість даних.
- [Один-до-багатьох](./o2m/index.md): Кожна сутність в одній колекції може бути пов'язана з кількома сутностями в іншій колекції. Це один із найпоширеніших типів зв'язків. Наприклад, один автор може написати кілька статей, але кожна стаття може мати лише одного автора.
- [Багато-до-одного](./m2o/index.md): Кілька сутностей в одній колекції можуть бути пов'язані з однією сутністю в іншій колекції. Цей тип зв'язку також поширений у моделюванні даних. Наприклад, кілька студентів можуть належати до одного класу.
- [Багато-до-багатьох](./m2m/index.md): Кілька сутностей у двох колекціях можуть бути пов'язані одна з одною. Цей тип зв'язку зазвичай вимагає проміжної колекції для запису зв'язків між сутностями. Наприклад, зв'язок між студентами та курсами: студент може записатися на кілька курсів, а курс може мати кількох студентів.

Ці типи зв'язків відіграють важливу роль у проєктуванні баз даних та моделюванні даних, допомагаючи описувати складні реальні зв'язки та структури даних.
