# Успадкована колекція

## Вступ

:::warning
Ця функція підтримується лише для основних баз даних PostgreSQL.
:::

Ви можете створити батьківську колекцію, а потім вивести з неї дочірню колекцію. Дочірня колекція успадкує структуру батьківської колекції, а також зможе визначати власні стовпці. Цей шаблон проєктування допомагає організувати та керувати даними зі схожими структурами, але з можливими відмінностями.

Ось деякі типові характеристики успадкованих колекцій:

-   **Батьківська колекція:** Батьківська колекція містить спільні стовпці та дані, визначаючи базову структуру всієї ієрархії успадкування.
-   **Дочірня колекція:** Дочірня колекція успадковує структуру батьківської колекції, але також може визначати власні стовпці. Це дозволяє кожній дочірній колекції мати спільні властивості батьківської колекції, водночас містячи атрибути, специфічні для підкласу.
-   **Запити:** Під час виконання запитів ви можете вибрати запит до всієї ієрархії успадкування, лише до батьківської колекції або до конкретної дочірньої колекції. Це дозволяє отримувати та обробляти дані різних рівнів за потреби.
-   **Відношення успадкування:** Між батьківською та дочірньою колекціями встановлюється відношення успадкування. Це означає, що структуру батьківської колекції можна використовувати для визначення узгоджених атрибутів, водночас дозволяючи дочірній колекції розширювати або перевизначати ці атрибути.

Цей шаблон проєктування допомагає зменшити надмірність даних, спростити модель бази даних і полегшити підтримку даних. Однак його слід використовувати з обережністю, оскільки успадковані колекції можуть збільшити складність запитів, особливо при роботі з усією ієрархією успадкування. Системи баз даних, які підтримують успадковані колекції, зазвичай надають специфічний синтаксис та інструменти для керування та запитів до цих структур колекцій.
