# Файловий менеджер

## Вступ

Плагін Файловий менеджер надає колекцію файлів, поле вкладення та рушії зберігання файлів для ефективного керування файлами. Файли — це записи в спеціальному типі колекції, відомому як колекція файлів, яка зберігає метадані файлів і може керуватися через Файловий менеджер. Поля вкладень — це специфічні поля асоціації, пов'язані з колекцією файлів. Плагін підтримує кілька методів зберігання, зокрема рушії зберігання файлів, що вже підтримуються: локальне сховище, Alibaba Cloud OSS, Amazon S3 та Tencent Cloud COS.

## Посібник користувача

### Колекція файлів

Вбудована колекція `attachments` призначена для зберігання всіх файлів, пов'язаних з полями вкладень. Крім того, ви можете створювати нові колекції файлів для зберігання специфічних файлів.

[Дізнайтеся більше в документації про колекції файлів](/data-sources/file-manager/file-collection)

### Поле вкладення

Поля вкладень — це специфічні поля асоціації, пов'язані з колекцією файлів, які можна створити за допомогою типу поля «Вкладення» або налаштувати через поле «Асоціація».

[Дізнайтеся більше в документації про поля вкладень](/data-sources/file-manager/field-attachment)

### Рушій зберігання файлів

Рушій зберігання файлів використовується для збереження файлів у певних службах, включно з локальним сховищем (збереження на жорсткий диск сервера), хмарним сховищем тощо.

[Дізнайтеся більше в документації про рушії зберігання файлів](./storage/index.md)

### HTTP API

Завантаження файлів можна обробляти за допомогою HTTP API, див. [HTTP API](./http-api.md).
