# Керування джерелами даних

## Вступ

NocoBase надає плагін для керування джерелами даних, який використовується для керування джерелами даних та їхніми колекціями. Плагін керування джерелами даних лише надає інтерфейс для керування всіма джерелами даних, але не забезпечує можливість підключення до них. Він має використовуватися разом з різними плагінами джерел даних. Наразі підтримуються такі джерела даних:

- [Основна база даних](/data-sources/data-source-main): Основна база даних NocoBase, що підтримує реляційні бази даних, такі як MySQL, PostgreSQL та MariaDB.
- [Зовнішній MySQL](/data-sources/data-source-external-mysql): Використання зовнішньої бази даних MySQL як джерела даних.
- [Зовнішній MariaDB](/data-sources/data-source-external-mariadb): Використання зовнішньої бази даних MariaDB як джерела даних.
- [Зовнішній PostgreSQL](/data-sources/data-source-external-postgres): Використання зовнішньої бази даних PostgreSQL як джерела даних.
- [Зовнішній MSSQL](/data-sources/data-source-external-mssql): Використання зовнішньої бази даних MSSQL (SQL Server) як джерела даних.
- [Зовнішній Oracle](/data-sources/data-source-external-oracle): Використання зовнішньої бази даних Oracle як джерела даних.

Крім того, за допомогою плагінів можна розширити підтримку інших типів, це можуть бути як поширені типи баз даних, так і платформи, що надають API (SDK).

## Встановлення

Це вбудований плагін, окрема інсталяція не потрібна.

## Інструкції з використання

Під час ініціалізації та встановлення застосунку за замовчуванням надається джерело даних для зберігання даних NocoBase, відоме як основна база даних. Докладніше дивіться в документації [Основна база даних](/data-sources/data-source-main/).

### Зовнішні джерела даних

Зовнішні бази даних підтримуються як джерела даних. Докладніше дивіться в документації [Зовнішня база даних / Вступ](/data-sources/data-source-manager/external-database).
