# Багато-до-багатьох (масив)

## Вступ

Ця функція дозволяє використовувати поля-масиви в колекції даних для зберігання кількох унікальних ключів з цільової таблиці, таким чином створюючи зв'язок багато-до-багатьох між двома таблицями. Наприклад, розгляньмо сутності «Статті» та «Теги». Одна стаття може бути пов'язана з кількома тегами, при цьому таблиця статей зберігає ID відповідних записів з таблиці тегів у полі-масиві.

:::warning{title=Примітка}

- По можливості, рекомендується використовувати проміжну колекцію для встановлення стандартного зв'язку [багато-до-багатьох](../data-modeling/collection-fields/associations/m2m/index.md), уникаючи цього типу зв'язку.
- Наразі лише PostgreSQL підтримує фільтрацію даних вихідної колекції за допомогою полів з цільової таблиці для зв'язків багато-до-багатьох, встановлених за допомогою полів-масивів. Наприклад, у наведеному вище сценарії ви можете фільтрувати статті на основі інших полів у таблиці тегів, таких як заголовок.
  :::

## Опис параметрів

### Вихідна колекція

Вихідна колекція, де знаходиться поточне поле.

### Цільова колекція

Цільова колекція, з якою встановлюється зв'язок.

### Зовнішній ключ

Поле-масив у вихідній колекції, яке зберігає цільовий ключ з цільової таблиці.

Відповідні зв'язки для типів полів-масивів такі:

| NocoBase | PostgreSQL | MySQL  | SQLite |
| -------- | ---------- | ------ | ------ |
| `set`    | `array`    | `JSON` | `JSON` |

### Цільовий ключ

Поле в цільовій колекції, яке відповідає значенням, що зберігаються в полі-масиві вихідної таблиці. Це поле повинно бути унікальним.
