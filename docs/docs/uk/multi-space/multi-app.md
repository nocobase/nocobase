# Мульти-додаток

## Вступ

**Плагін Мульти-додаток (Multi-app)** дозволяє динамічно створювати та керувати кількома незалежними додатками без необхідності окремого розгортання. Кожен дочірній додаток є повністю незалежним екземпляром зі своєю власною базою даних, плагінами та конфігураціями.

#### Сценарії застосування
- **Мультиорендність (Multi-tenancy)**: надання незалежних екземплярів додатків, де кожен клієнт має власні дані, конфігурації плагінів та систему прав доступу.
- **Головна та дочірні системи для різних бізнес-сфер**: велика система, що складається з кількох незалежно розгорнутих малих додатків.

:::warning
Плагін мульти-додатків сам по собі не надає можливості спільного використання користувачів.  
Для налаштування єдиного входу користувачів у різних додатках його можна використовувати разом із **[плагіном автентифікації](/auth-verification)**.
:::

## Інструкція користувача

### Створення дочірнього додатка

У меню налаштувань системи натисніть «Мульти-додаток», щоб перейти на сторінку керування мульти-додатками:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

Натисніть кнопку «Додати», щоб створити новий дочірній додаток:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### Опис полів форми

* **Назва**: ідентифікатор дочірнього додатка, має бути глобально унікальним.
* **Відображувана назва**: назва дочірнього додатка, що відображається в інтерфейсі.
* **Спосіб запуску**:
  * **Запуск при першому доступі**: дочірній додаток запускається лише тоді, коли користувач вперше звертається до нього через URL.
  * **Запуск разом із головним додатком**: дочірній додаток запускається одночасно з головним додатком (це збільшує час запуску головного додатка).
* **Порт**: номер порту, який використовується дочірнім додатком під час роботи.
* **Власний домен**: налаштування незалежного піддомену для дочірнього додатка.
* **Закріпити в меню**: закріплює вхід до дочірнього додатка зліва у верхній панелі навігації.
* **Підключення до бази даних**: використовується для налаштування джерела даних дочірнього додатка, підтримує три способи:
  * **Нова база даних**: повторне використання поточної служби даних для створення незалежної бази даних.
  * **Нове підключення до даних**: налаштування абсолютно нової служби бази даних.
  * **Режим схеми (Schema)**: створення незалежної схеми (Schema) для дочірнього додатка в PostgreSQL.
* **Оновлення**: якщо підключена база даних містить структуру даних NocoBase старішої версії, вона буде автоматично оновлена до поточної версії.

### Запуск та зупинка дочірніх додатків

Натисніть кнопку **Запустити**, щоб запустити дочірній додаток.  
> Якщо під час створення було обрано *«Запуск при першому доступі»*, він запуститься автоматично при першому відвідуванні.  

Натисніть кнопку **Переглянути**, щоб відкрити дочірній додаток у новій вкладці.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)

### Статус роботи та логи дочірнього додатка

У списку можна переглянути обсяг пам'яті та завантаження процесора (CPU) для кожного додатка.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

Натисніть кнопку **Логи**, щоб переглянути журнал роботи дочірнього додатка.  
> Якщо дочірній додаток недоступний після запуску (наприклад, через пошкодження бази даних), ви можете виявити причину за допомогою логів.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)

### Видалення дочірнього додатка

Натисніть кнопку **Видалити**, щоб вилучити дочірній додаток.  
> При видаленні можна обрати, чи видаляти також базу даних. Будьте обережні, оскільки цю дію неможливо скасувати.

### Доступ до дочірніх додатків
За замовчуванням використовуйте `/_app/:appName/admin/` для доступу до дочірніх додатків, наприклад:
```
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
Також можна налаштувати незалежні піддомени для дочірніх додатків. Вам потрібно спрямувати домен на поточну IP-адресу. Якщо використовується Nginx, домен також має бути доданий до конфігурації Nginx.

### Керування дочірніми додатками через CLI

У кореневому каталозі проєкту ви можете використовувати командний рядок для керування екземплярами дочірніх додатків за допомогою **PM2**:

```bash
yarn nocobase pm2 list              # Переглянути список запущених екземплярів
yarn nocobase pm2 stop [appname]    # Зупинити процес конкретного дочірнього додатка
yarn nocobase pm2 delete [appname]  # Видалити процес конкретного дочірнього додатка
yarn nocobase pm2 kill              # Примусово завершити всі запущені процеси (може включати екземпляр головного додатка)
```

### Міграція даних зі старої версії мульти-додатка

Перейдіть на сторінку керування застарілою версією мульти-додатків і натисніть кнопку **Мігрувати дані до нового мульти-додатка**, щоб виконати міграцію.

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)

## Часті запитання (FAQ)

#### 1. Керування плагінами
Дочірні додатки можуть використовувати ті самі плагіни, що й головний додаток (включаючи версії), але плагіни можна налаштовувати та використовувати незалежно.

#### 2. Ізоляція бази даних
Дочірні додатки можуть бути налаштовані з незалежними базами даних. Якщо ви хочете спільно використовувати дані між додатками, це можна реалізувати через зовнішні джерела даних.

#### 3. Резервне копіювання та міграція даних
Наразі резервне копіювання даних у головному додатку не включає дані дочірніх додатків (воно містить лише базову інформацію про них). Резервне копіювання та міграцію потрібно виконувати вручну всередині кожного дочірнього додатка.

#### 4. Розгортання та оновлення
Версії дочірніх додатків будуть автоматично слідувати за оновленнями головного додатка, що гарантує узгодженість версій між головним та дочірніми додатками.

#### 5. Керування ресурсами
Споживання ресурсів кожним дочірнім додатком в цілому таке ж, як і в головного додатка. Наразі споживання пам'яті одним додатком становить приблизно 500-600 МБ.
