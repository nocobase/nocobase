# Історія записів

## Вступ

Плагін **Історія записів** відстежує зміни даних, автоматично зберігаючи знімки та відмінності операцій створення, оновлення та видалення. Він допомагає користувачам швидко переглядати модифікації даних та аудитувати дії.

![](https://static-docs.nocobase.com/202511011338499.png)

## Увімкнення Історії записів

### Додавання колекцій та полів

Спершу перейдіть на сторінку налаштувань плагіна Історія записів, щоб додати колекції та поля, для яких ви хочете відстежувати історію. Для підвищення ефективності запису та уникнення надмірності даних рекомендується відстежувати лише необхідні поля. Такі поля, як **унікальний ID**, **createdAt**, **updatedAt**, **createdBy** та **updatedBy**, зазвичай не потребують запису.

![](https://static-docs.nocobase.com/202511011315010.png)

![](https://static-docs.nocobase.com/202511011316342.png)

### Синхронізація знімків історичних даних

- Для записів, створених до увімкнення відстеження історії, зміни можуть бути зареєстровані лише після того, як перше оновлення згенерує знімок; початкове оновлення або видалення не буде записано.
- Щоб зберегти історію існуючих даних, ви можете виконати одноразову синхронізацію знімків.
- Розмір знімка для кожної колекції розраховується як: кількість записів × кількість відстежуваних полів.
- Для великих наборів даних рекомендується фільтрувати за діапазоном даних та синхронізувати лише важливі записи.

![](https://static-docs.nocobase.com/202511011319386.png)

![](https://static-docs.nocobase.com/202511011319284.png)

Натисніть кнопку **«Синхронізувати історичні знімки»**, налаштуйте поля та діапазон даних і розпочніть синхронізацію.

![](https://static-docs.nocobase.com/202511011320958.png)

Завдання синхронізації буде поставлено в чергу та виконано у фоновому режимі. Ви можете оновити список, щоб перевірити його статус завершення.

## Використання блоку Історія записів

### Додавання блоку

Виберіть **Блок Історія записів** та оберіть колекцію, щоб додати відповідний блок історії на вашу сторінку.

![](https://static-docs.nocobase.com/202511011323410.png)

![](https://static-docs.nocobase.com/202511011331667.png)

Якщо ви додаєте блок історії всередині спливаючого вікна деталей запису, ви можете вибрати **«Поточний запис»**, щоб відобразити історію, специфічну для цього запису.

![](https://static-docs.nocobase.com/202511011338042.png)

![](https://static-docs.nocobase.com/202511011338499.png)

### Редагування шаблонів описів

Натисніть **«Редагувати шаблон»** у налаштуваннях блоку, щоб налаштувати текст опису для записів операцій.

![](https://static-docs.nocobase.com/202511011340406.png)

Ви можете налаштувати окремі шаблони описів для операцій **створення**, **оновлення** та **видалення**. Для операцій оновлення ви також можете налаштувати шаблон опису для змін полів, як єдиний шаблон для всіх полів, так і для окремих полів індивідуально.

![](https://static-docs.nocobase.com/202511011346400.png)

При налаштуванні тексту можна використовувати змінні.

![](https://static-docs.nocobase.com/202511011347163.png)

Після налаштування ви можете вибрати застосування шаблону до **Усіх блоків історії записів поточної колекції** або **Лише цього блоку історії записів**.

![](https://static-docs.nocobase.com/202511011348885.png)
