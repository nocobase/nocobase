# Серверні логи, аудиторські логи та історія записів

## Серверні логи

### Системні логи

> Дивіться [Системні логи](./index.md#системні-логи)

- Фіксують інформацію про роботу системи, відстежують ланцюжки виконання коду та допомагають виявляти помилки виконання та інші аномалії.
- Існує градація логів за рівнями, вони класифікуються за функціональними модулями.
- Виводяться в термінал або зберігаються у вигляді файлів.
- Використовуються переважно для діагностики та усунення несправностей, що виникають під час роботи системи.

### Логи запитів

> Дивіться [Логи запитів](./index.md#логи-запитів)

- Фіксують інформацію про HTTP API запити та відповіді, зосереджуючись на ID запиту, шляху API (Path), заголовках, кодах стану відповіді та тривалості виконання.
- Виводяться в термінал або зберігаються у вигляді файлів.
- Використовуються переважно для відстеження викликів та виконання API.

## Аудиторські логи

> Дивіться [Аудиторські логи](/security/audit-logger/index.md)

- Фіксують дії користувачів (або API) з системними ресурсами, зосереджуючись на типі ресурсу, об'єкті ресурсу, типі операції, інформації про користувача та статусі операції.
- Для кращого відстеження конкретного змісту та результатів дій користувача, параметри запиту та відповіді записуються як метадані. Ця інформація частково збігається з логами запитів, але не є ідентичною; наприклад, у звичайних логах запитів повне тіло запиту зазвичай не зберігається.
- Параметри та відповіді запиту не є еквівалентними знімкам (snapshots) ресурсів. Через параметри та логіку коду можна зрозуміти, які зміни відбулися, але неможливо точно знати вміст запису в таблиці бази даних до його зміни, що обмежує можливості контролю версій або відновлення даних після помилкових операцій.
- Зберігаються у вигляді файлів та таблиць бази даних.

## Історія записів

> Дивіться [Історія записів](/record-history/index.md)

- Фіксує історію змін вмісту даних.
- Основний зміст включає тип ресурсу, об'єкт ресурсу, тип операції, змінені поля, а також значення до та після змін.
- Може використовуватися для порівняння даних.
- Зберігається у вигляді таблиць бази даних.
