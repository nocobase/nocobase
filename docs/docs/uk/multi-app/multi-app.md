# Мультидодаток


## Вступ

Плагін **Мультидодаток** дозволяє динамічно створювати та керувати кількома незалежними додатками без окремого розгортання. Кожен піддодаток є повністю незалежним екземпляром зі своєю базою даних, плагінами та конфігурацією.

#### Сценарії використання
- **Мультиорендність**: Надання незалежних екземплярів додатків, де кожен клієнт має власні дані, конфігурації плагінів та систему дозволів.
- **Основні та дочірні системи для різних бізнес-доменів**: Велика система, що складається з кількох незалежно розгорнутих менших додатків.


:::warning
Плагін Мультидодаток сам по собі не надає можливості спільного використання користувачів.
Якщо вам потрібно об'єднати користувачів між кількома додатками, ви можете використовувати його в поєднанні з **[плагіном Автентифікації](/auth-verification)**.
:::


## Встановлення

У менеджері плагінів знайдіть плагін **Мультидодаток (Multi-app)** та увімкніть його.

![](https://static-docs.nocobase.com/multi-app/Plugin-manager-NocoBase-10-16-2025_03_07_PM.png)


## Посібник з використання


### Створення піддодатка

У меню системних налаштувань натисніть «Мультидодаток», щоб перейти на сторінку керування мультидодатками:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

Натисніть кнопку «Додати новий», щоб створити новий піддодаток:

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### Опис полів форми

* **Назва**: Ідентифікатор піддодатка, глобально унікальний.
* **Відображувана назва**: Назва піддодатка, що відображається в інтерфейсі.
* **Режим запуску**:
  * **Запускати при першому відвідуванні**: Піддодаток запускається лише тоді, коли користувач вперше отримує до нього доступ за URL-адресою;
  * **Запускати разом з основним додатком**: Піддодаток запускається одночасно з основним додатком (це збільшить час запуску основного додатка).
* **Порт**: Номер порту, який використовується піддодатком під час виконання.
* **Власний домен**: Налаштуйте незалежний субдомен для піддодатка.
* **Закріпити в меню**: Закріпити запис піддодатка в лівій частині верхньої панелі навігації.
* **Підключення до бази даних**: Використовується для налаштування джерела даних для піддодатка, підтримує такі три методи:
  * **Нова база даних**: Повторно використовувати поточну службу даних для створення незалежної бази даних;
  * **Нове підключення до даних**: Налаштувати абсолютно нову службу бази даних;
  * **Режим схеми**: Створити незалежну схему для піддодатка в PostgreSQL.
* **Оновлення**: Якщо підключена база даних містить стару версію структури даних NocoBase, вона буде автоматично оновлена до поточної версії.


### Запуск та зупинка піддодатка

Натисніть кнопку **Запустити**, щоб запустити піддодаток;
> Якщо під час створення було обрано *«Запускати при першому відвідуванні»*, він автоматично запуститься при першому відвідуванні.

Натисніть кнопку **Переглянути**, щоб відкрити піддодаток у новій вкладці.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)


### Стан та журнали піддодатка

У списку ви можете переглянути використання пам'яті та ЦП кожним додатком.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

Натисніть кнопку **Журнали**, щоб переглянути журнали виконання піддодатка.
> Якщо піддодаток недоступний після запуску (наприклад, через пошкоджену базу даних), ви можете використовувати журнали для усунення несправностей.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)


### Видалення піддодатка

Натисніть кнопку **Видалити**, щоб видалити піддодаток.
> Під час видалення ви можете вибрати, чи видаляти базу даних також. Будь ласка, дійте обережно, оскільки цю дію неможливо скасувати.


### Доступ до піддодатка
За замовчуванням доступ до піддодатків здійснюється за допомогою `/_app/:appName/admin/`, наприклад:
```bash
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
Ви також можете налаштувати незалежний субдомен для піддодатка. Вам потрібно буде направити домен на поточну IP-адресу, і якщо ви використовуєте Nginx, вам також потрібно додати домен до конфігурації Nginx.


### Керування піддодатками через командний рядок

У кореневому каталозі проєкту ви можете використовувати командний рядок для керування екземплярами піддодатків через **PM2**:

```bash
yarn nocobase pm2 list              # Переглянути список запущених екземплярів
yarn nocobase pm2 stop [appname]    # Зупинити процес певного піддодатка
yarn nocobase pm2 delete [appname]  # Видалити процес певного піддодатка
yarn nocobase pm2 kill              # Примусово завершити всі запущені процеси (може включати екземпляри основного додатка)
```

### Міграція даних зі старого Мультидодатка

Перейдіть на стару сторінку керування мультидодатками та натисніть кнопку **Мігрувати дані до нового мультидодатка** для міграції даних.

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)


## Часті запитання

#### 1. Керування плагінами
Піддодатки можуть використовувати ті ж плагіни, що й основний додаток (включно з версіями), але їх можна налаштовувати та використовувати незалежно.

#### 2. Ізоляція бази даних
Піддодатки можуть бути налаштовані з незалежними базами даних. Якщо ви хочете обмінюватися даними між додатками, ви можете зробити це через зовнішні джерела даних.

#### 3. Резервне копіювання та міграція даних
Наразі резервне копіювання даних в основному додатку не включає дані піддодатків (лише базову інформацію про піддодатки). Вам потрібно вручну створювати резервні копії та мігрувати дані в кожному піддодатку.

#### 4. Розгортання та оновлення
Версія піддодатка буде автоматично оновлюватися разом з основним додатком, забезпечуючи узгодженість версій між основним та піддодатками.

#### 5. Керування ресурсами
Використання ресурсів кожним піддодатком в основному таке ж, як і основним додатком. Наразі один додаток використовує близько 500-600 МБ пам'яті.
