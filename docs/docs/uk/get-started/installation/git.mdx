# Встановлення з вихідного коду Git

:::tip Передумови

- Встановлено Git, Node.js 20+ та Yarn 1.22.x
- Налаштовано та запущено одну з необхідних баз даних: MySQL 8.0.17+, MariaDB 10.9+ або PostgreSQL 10+


:::

## 1. Завантажте NocoBase на ваш локальний комп'ютер

Виберіть версію NocoBase ([Порівняння версій](../quickstart.md)) та виконайте відповідну команду.

<Tabs>
<Tab label="Остання версія">

```bash
git clone https://github.com/nocobase/nocobase.git -b main --depth=1 my-nocobase
```

</Tab>
<Tab label="Версія Beta">

```bash
git clone https://github.com/nocobase/nocobase.git -b next --depth=1 my-nocobase
```

</Tab>
<Tab label="Версія Alpha">

```bash
git clone https://github.com/nocobase/nocobase.git -b develop --depth=1 my-nocobase
```

</Tab>
</Tabs>

## 2. Змініть каталог

```bash
cd my-nocobase
```

## 3. Встановіть залежності

📢 Цей крок може зайняти понад десять хвилин, залежно від вашого мережевого середовища, конфігурації системи та інших факторів.

```bash
yarn install --frozen-lockfile
```

## 4. Налаштуйте змінні середовища

Змінні середовища, необхідні для NocoBase, зберігаються у файлі `.env` в кореневому каталозі. Змініть їх відповідно до вашої фактичної ситуації.

```bash
TZ=UTC
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning Опис змінних середовища

- `TZ` використовується для встановлення часового поясу застосунку, за замовчуванням це часовий пояс операційної системи.
- `APP_KEY` — це секретний ключ застосунку, що використовується для генерації токенів користувачів тощо (якщо `APP_KEY` буде змінено, старі токени стануть недійсними). Це може бути будь-який випадковий рядок. Будь ласка, змініть його на власний секретний ключ і переконайтеся, що він не буде розголошений.
- `DB_*` стосуються бази даних. Якщо ви не використовуєте службу бази даних за замовчуванням з прикладу, будь ласка, змініть їх відповідно до вашої фактичної ситуації.

:::

## 5. Встановіть NocoBase

```bash
yarn nocobase install --lang=uk-UA
```

## 6. Запустіть NocoBase

**Середовище розробки**

```bash
yarn dev
```

**Виробниче середовище (не рекомендовано)**

Не рекомендується розгортати з вихідного коду у виробничому середовищі. Для виробничого розгортання, будь ласка, зверніться до [Розгортання у виробничому середовищі](../deployment/production.md).

```bash
yarn build
yarn start
```

## 7. Увійдіть до NocoBase

Відкрийте [http://localhost:13000](http://localhost:13000) у вашому браузері. Початкові облікові дані: `admin@nocobase.com` та `admin123`.

:::warning Попередження про безпеку облікового запису

Після першого входу, будь ласка, негайно змініть пароль за замовчуванням, щоб забезпечити безпеку системи.

:::
