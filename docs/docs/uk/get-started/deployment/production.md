# Розгортання в робочому середовищі

При розгортанні NocoBase в робочому середовищі, встановлення залежностей може бути складним через відмінності у методах збірки в різних системах та оточеннях. Для повноцінного функціоналу ми рекомендуємо використовувати **Docker** для розгортання. Якщо ваше системне середовище не підтримує Docker, ви також можете розгорнути NocoBase за допомогою **create-nocobase-app**.

:::warning

Не рекомендується розгортати NocoBase безпосередньо з вихідного коду в робочому середовищі. Вихідний код має багато залежностей, великий обсяг, а повна компіляція вимагає значних ресурсів процесора та пам'яті. Якщо вам все ж необхідно розгорнути NocoBase з вихідного коду, радимо спочатку створити власний образ Docker, а потім виконати розгортання.

:::
