# Видалення даних

Використовується для видалення даних з колекції, які відповідають певним умовам.

Базове використання вузла видалення схоже на вузол оновлення, за винятком того, що вузол видалення не потребує присвоєння значень полям. Вам потрібно лише вибрати колекцію та умови фільтрації. Результат вузла видалення повертає кількість успішно видалених рядків, яку можна переглянути лише в історії виконання і не можна використовувати як змінну в наступних вузлах.

:::info{title=Примітка}
Наразі вузол видалення не підтримує порядове видалення; він виконує пакетні видалення. Отже, він не буде запускати інші події для кожного окремого видалення даних.
:::

## Створення вузла

В інтерфейсі конфігурації робочого процесу натисніть кнопку плюса («+») у потоці, щоб додати вузол «Видалити дані»:

![Створення вузла видалення даних](https://static-docs.nocobase.com/e1d6b8728251fcdbed6c7f50e5570da2.png)

## Конфігурація вузла

![Вузол видалення_Конфігурація вузла](https://static-docs.nocobase.com/580600c2b13ef4e01dfa48b23539648e.png)

### Колекція

Виберіть колекцію, з якої потрібно видалити дані.

### Умови фільтрації

Подібно до умов фільтрації для звичайного запиту колекції, ви можете використовувати контекстні змінні робочого процесу.

## Приклад

Наприклад, для періодичного очищення скасованих та недійсних історичних даних замовлень можна використовувати вузол видалення:

![Вузол видалення_Приклад_Конфігурація вузла](https://static-docs.nocobase.com/b94b75077a17252f8523c3f13ce5f320.png)

Робочий процес буде запускатися періодично та виконувати видалення всіх скасованих та недійсних історичних даних замовлень.
