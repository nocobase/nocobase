# Умова

## Вступ

Подібно до оператора `if` у мовах програмування, цей вузол визначає подальший напрямок виконання робочого процесу на основі результату налаштованої умови.

## Створення вузла

Вузол Умова має два режими: «Продовжити, якщо істина» та «Розгалуження за істиною/хибою». Ви повинні обрати один із цих режимів під час створення вузла, і його не можна буде змінити в конфігурації вузла пізніше.

![Condition_Mode_Selection](https://static-docs.nocobase.com/3de27308c1179523d8606c66bf3a5fb4.png)

У режимі «Продовжити, якщо істина», якщо результат умови є «істиною», робочий процес продовжить виконання наступних вузлів. В іншому випадку, робочий процес буде припинено та завершиться достроково зі статусом «невдача».

!["Continue if true" mode](https://static-docs.nocobase.com/0f6ae1afe61d501f8eb1f6dedb3d4ad7.png)

Цей режим підходить для сценаріїв, коли робочий процес не повинен продовжуватися, якщо умова не виконується. Наприклад, кнопка відправки форми для оформлення замовлення прив'язана до «Події перед дією». Якщо запасу товару в замовленні недостатньо, процес створення замовлення не повинен продовжуватися, а має завершитися з помилкою.

У режимі «Розгалуження за істиною/хибою» вузол умови створить дві подальші гілки, які відповідатимуть результатам умови «істина» та «хиба». Кожна гілка може бути налаштована зі своїми власними наступними вузлами. Після завершення виконання будь-якої з гілок, вона автоматично об'єднається з батьківською гілкою вузла умови, щоб продовжити виконання наступних вузлів.

!["Branch on true/false" mode](https://static-docs.nocobase.com/974a1fcd8603629b64ffce6c55d59282.png)

Цей режим підходить для сценаріїв, коли необхідно виконати різні дії залежно від того, чи виконується умова, чи ні. Наприклад, перевірка існування певних даних: якщо їх немає, створити; якщо вони є, оновити.

## Конфігурація вузла

### Рушій обчислень

Наразі підтримуються три рушії:

-   **Базовий**: Отримує логічний результат за допомогою простих бінарних обчислень та групування за операторами «І»/«АБО».
-   **Math.js**: Обчислює вирази, що підтримуються рушієм [Math.js](https://mathjs.org/), для отримання логічного результату.
-   **Formula.js**: Обчислює вирази, що підтримуються рушієм [Formula.js](https://formulajs.info/), для отримання логічного результату.

У всіх трьох типах обчислень змінні з контексту робочого процесу можуть використовуватися як параметри.
