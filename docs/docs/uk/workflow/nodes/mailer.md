# Надсилання електронних листів

## Вступ

Використовується для надсилання електронних листів. Підтримує текстовий та HTML-формат вмісту.

## Створення вузла

В інтерфейсі конфігурації робочого процесу натисніть кнопку плюса («+») у потоці, щоб додати вузол «Надсилання електронних листів»:

![20251031130522](https://static-docs.nocobase.com/20251031130522.png)

## Конфігурація вузла

![20251031131125](https://static-docs.nocobase.com/20251031131125.png)

Кожна опція може використовувати змінні з контексту робочого процесу. Для конфіденційної інформації також можна використовувати глобальні змінні та секрети.

## Поширені запитання

### Обмеження частоти надсилання Gmail

Під час надсилання деяких електронних листів ви можете зіткнутися з такою помилкою:

```json
{
  "code": "ECONNECTION",
  "response": "421-4.7.0 Try again later, closing connection. (EHLO)\n421-4.7.0 For more information, go to\n421 4.7.0 About SMTP error messages - Google Workspace Admin Help 3f1490d57ef6-e7f7352f44csm831688276.30 - gsmtp",
  "responseCode": 421,
  "command": "EHLO"
}
```

Це відбувається тому, що Gmail обмежує частоту надсилання запитів з доменів, які не вказані. Під час розгортання застосунку вам потрібно налаштувати ім'я хоста сервера на домен, який ви налаштували в Gmail. Наприклад, при розгортанні Docker:

```yaml
services:
  app:
    image: nocobase/nocobase
    hostname: <your-custom-hostname> # Встановіть на ваш налаштований домен для надсилання
```

Довідка: [Google SMTP Relay - Intermittent problems](https://forum.nocobase.com/t/google-smtp-relay-intermittent-problems/5483/6)
