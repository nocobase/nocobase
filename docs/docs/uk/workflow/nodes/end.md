# Завершення робочого процесу

Цей вузол, коли він виконується, негайно завершує поточний робочий процес зі статусом, який ви налаштували у вузлі. Зазвичай його використовують для керування потоком на основі певної логіки: якщо виконано певні умови, робочий процес переривається, і подальші кроки не виконуються. Це можна порівняти з інструкцією `return` у мовах програмування, яка дозволяє вийти з поточної функції.

## Додавання вузла

В інтерфейсі конфігурації робочого процесу натисніть кнопку плюса («+») у потоці, щоб додати вузол «Завершення робочого процесу»:

### Статус завершення

Статус завершення впливає на кінцевий статус виконання робочого процесу. Ви можете налаштувати його як «Успішно» або «Невдало». Коли робочий процес досягає цього вузла, він негайно завершується з обраним вами статусом.

:::info{title=Примітка}
Якщо ви використовуєте цей вузол у робочому процесі типу «Подія перед дією», він перехоплюватиме запит, який ініціював цю дію. Детальніше про це ви можете дізнатися у розділі [Використання «Події перед дією»](../triggers/pre-action).

Крім того, конфігурація статусу завершення не лише перехоплює запит, що ініціював дію, але й впливає на статус інформації зворотного зв'язку в «повідомленні-відповіді» для цього типу робочого процесу.
:::
