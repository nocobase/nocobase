# Автентифікація: LDAP

## Вступ

Плагін Автентифікація: LDAP відповідає стандарту протоколу LDAP (Lightweight Directory Access Protocol), дозволяючи користувачам входити в NocoBase, використовуючи облікові дані свого LDAP-сервера.

## Налаштування

### Основні налаштування

<img src="https://static-docs.nocobase.com/202405101605728.png"/>

- Автоматична реєстрація, якщо користувача не існує - Чи слід автоматично створювати нового користувача, якщо не знайдено відповідного наявного користувача.
- LDAP URL - URL-адреса LDAP-сервера
- Bind DN - DN, що використовується для перевірки підключення до сервера та пошуку користувачів
- Bind password - Пароль для Bind DN
- Перевірити підключення - Натисніть кнопку, щоб перевірити підключення до сервера та дійсність Bind DN.

### Налаштування пошуку

<img src="https://static-docs.nocobase.com/202405101609984.png"/>

- Search DN - DN, що використовується для пошуку користувачів
- Фільтр пошуку - Умова фільтрації для пошуку користувачів, де `{{account}}` представляє обліковий запис користувача, що використовується для входу
- Область - `Base`, `One level`, `Subtree`, за замовчуванням `Subtree`
- Обмеження розміру - Розмір сторінки пошуку

### Зіставлення атрибутів

<img src="https://static-docs.nocobase.com/202405101612814.png"/>

- Використовувати це поле для прив'язки користувача - Поле, що використовується для прив'язки до наявних користувачів. Виберіть "ім'я користувача", якщо обліковий запис для входу є ім'ям користувача, або "електронна пошта", якщо це адреса електронної пошти. За замовчуванням - ім'я користувача.
- Зіставлення атрибутів - Зіставлення атрибутів користувача з полями в таблиці користувачів NocoBase.
