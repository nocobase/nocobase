# Автентифікація: DingTalk

## Вступ

Плагін "Автентифікація: DingTalk" дозволяє користувачам входити в NocoBase за допомогою своїх облікових записів DingTalk.

## Активація плагіна

![](https://static-docs.nocobase.com/202406120929356.png)

## Запит дозволів API в консолі розробника DingTalk

Щоб створити застосунок, зверніться до <a href="https://open.dingtalk.com/document/orgapp/tutorial-obtaining-user-personal-information" target="_blank">Відкритої платформи DingTalk - Впровадження входу на сторонні вебсайти</a>.

Перейдіть до консолі керування застосунками та увімкніть "Інформація про особистий номер телефону" та "Дозвіл на читання особистої інформації з адресної книги".

![](https://static-docs.nocobase.com/202406120006620.png)

## Отримання облікових даних з консолі розробника DingTalk

Скопіюйте Client ID та Client Secret.

![](https://static-docs.nocobase.com/202406120000595.png)

## Додавання автентифікації DingTalk в NocoBase

Перейдіть на сторінку керування плагінами автентифікації користувачів.

![](https://static-docs.nocobase.com/202406112348051.png)

Додати - DingTalk

![](https://static-docs.nocobase.com/202406112349664.png)

### Конфігурація

![](https://static-docs.nocobase.com/202406120016896.png)

- **Автоматична реєстрація, якщо користувач не існує** – Чи потрібно автоматично створювати нового користувача, якщо існуючого користувача не знайдено за номером телефону.
- **Client ID та Client Secret** – Введіть інформацію, скопійовану на попередньому кроці.
- **Redirect URL** – URL-адреса зворотного виклику. Скопіюйте її та перейдіть до наступного кроку.

## Налаштування URL-адреси зворотного виклику в консолі розробника DingTalk

Вставте скопійовану URL-адресу зворотного виклику в консоль розробника DingTalk.

![](https://static-docs.nocobase.com/202406120012221.png)

## Вхід

Перейдіть на сторінку входу та натисніть кнопку під формою входу, щоб розпочати вхід через сторонній сервіс.

![](https://static-docs.nocobase.com/202406120014539.png)
