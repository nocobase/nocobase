# Верифікація

:::info{title=Примітка}
Починаючи з `1.6.0-alpha.30`, початкова функція **"код верифікації"** була оновлена до **"Керування верифікацією"**, яка підтримує керування та інтеграцію різних методів верифікації користувачів. Після того, як користувачі прив'яжуть відповідний метод верифікації, вони зможуть виконувати підтвердження особи за потреби. Ця функція планується до стабільної підтримки, починаючи з версії `1.7.0`.
:::

## Вступ

**Центр керування верифікацією підтримує керування та інтеграцію різних методів верифікації користувачів.** Наприклад:

- Код верифікації через SMS – Надається плагіном верифікації за замовчуванням. Дивіться: [Верифікація: SMS](./sms)
- TOTP-автентифікатор – Дивіться: [Верифікація: TOTP-автентифікатор](../verification-totp/)

Розробники також можуть розширювати інші типи верифікації за допомогою плагінів. Дивіться: [Розширення типів верифікації](./dev/type)

**Користувачі можуть виконувати підтвердження особи за потреби після прив'язки відповідного методу верифікації.** Наприклад:

- Вхід за допомогою SMS-коду верифікації – Дивіться: [Автентифікація: SMS](./sms)
- Двофакторна автентифікація (2FA) – Дивіться: [Двофакторна автентифікація (2FA)](../2fa)
- Повторна верифікація для ризикованих операцій – Підтримка в майбутньому

Розробники також можуть інтегрувати підтвердження особи в інші необхідні сценарії, розширюючи плагіни. Дивіться: [Розширення сценаріїв верифікації](./dev/scene)

**Відмінності та зв'язки між модулем верифікації та модулем автентифікації користувачів:**
Модуль автентифікації користувачів в основному відповідає за підтвердження особи в сценаріях входу користувача. При цьому такі процеси, як вхід за допомогою SMS та двофакторна автентифікація, покладаються на верифікатори, надані модулем верифікації. Модуль верифікації, своєю чергою, відповідає за підтвердження особи для різних ризикованих операцій, а вхід користувача є одним із таких сценаріїв.
