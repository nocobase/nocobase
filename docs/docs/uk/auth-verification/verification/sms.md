# Верифікація: SMS

## Вступ

SMS-код верифікації — це вбудований тип верифікації, який використовується для генерації одноразового динамічного пароля (OTP) та його надсилання користувачеві за допомогою SMS.

## Додавання SMS-верифікатора

Перейдіть на сторінку керування верифікацією.

## Конфігурація адміністратора

![](https://static-docs.nocobase.com/202502271727711.png)

Наразі підтримуються такі постачальники SMS-послуг:

- <a href="https://www.aliyun.com/product/sms" target="_blank">Aliyun SMS</a>
- <a href="https://cloud.tencent.com/product/sms" target="_blank">Tencent Cloud SMS</a>

Під час налаштування SMS-шаблону в адміністративній панелі постачальника послуг необхідно зарезервувати параметр для коду верифікації.

- Приклад конфігурації для Aliyun: `Ваш код верифікації: ${code}`

- Приклад конфігурації для Tencent Cloud: `Ваш код верифікації: {1}`

Розробники також можуть розширювати підтримку інших постачальників SMS-послуг у вигляді плагінів. Дивіться: [Розширення постачальників SMS-послуг](./dev/sms-type)

Після додавання верифікатора користувачі можуть прив'язати номер телефону в особистому керуванні верифікацією.

![](https://static-docs.nocobase.com/202502271737016.png)

![](https://static-docs.nocobase.com/202502271737769.png)

![](https://static-docs.nocobase.com/202502271738515.png)

Після успішної прив'язки ідентифікацію можна буде виконувати в будь-якому сценарії, де використовується цей верифікатор.
