# Автентифікатори

## Керування автентифікацією користувачів

Під час встановлення плагіна автентифікації користувачів ініціалізується спосіб автентифікації `паролем`, який базується на імені користувача та електронній пошті.

![](https://static-docs.nocobase.com/66eaa9d5421c9cb713b117366bd8a5d5.png)

## Активація автентифікаторів

![](https://static-docs.nocobase.com/7f1fb8f8ca5de67ffc68eff0a65848f5.png)

Лише активовані типи автентифікації відображатимуться на сторінці входу.

![](https://static-docs.nocobase.com/8375a36ef98417af0f0977f1e07345dd.png)

## Типи автентифікації користувачів

![](https://static-docs.nocobase.com/da4250c0cea343ebe470cbf7be4b12e4.png)

Додаючи різні типи автентифікаторів, ви можете додати відповідні способи автентифікації до системи.

Окрім типів автентифікації, що надаються наявними плагінами, розробники також можуть розширювати типи автентифікації користувачів. Детальніше дивіться у [Посібнику для розробників](./dev/).
