# Висота блоку

## Вступ

Висота блоку підтримує три режими: **Стандартна висота**, **Задана висота** та **Повна висота**. Більшість блоків підтримують налаштування висоти.

![20260211154020](https://static-docs.nocobase.com/20260211154020.png)

## Режими висоти

### Стандартна висота

Стратегія стандартної висоти відрізняється для різних типів блоків. Наприклад, блоки таблиць та форм автоматично адаптують свою висоту відповідно до вмісту, і всередині блоку не з'являються смуги прокручування.

### Задана висота

Ви можете вручну вказати загальну висоту зовнішньої рамки блоку. Блок автоматично розрахує та розподілить доступну висоту всередині.

![20260211154043](https://static-docs.nocobase.com/20260211154043.gif)

### Повна висота

Режим повної висоти подібний до заданої висоти, але висота блоку розраховується на основі поточного **вікна перегляду** (viewport) браузера для досягнення максимальної висоти на весь екран. Смуги прокручування не з'являтимуться на сторінці браузера; вони відображатимуться лише всередині блоку.

Обробка внутрішнього прокручування в режимі повної висоти дещо відрізняється для різних блоків:

- **Таблиця**: внутрішнє прокручування в межах `tbody`;
- **Форма / Деталі**: прокручування всередині сітки (Grid) (прокручування вмісту, за винятком області дій);
- **Список / Сітка карток**: прокручування всередині сітки (Grid) (прокручування вмісту, за винятком області дій та панелі пагінації);
- **Карта / Календар**: загальна адаптивна висота, без смуг прокручування;
- **Iframe / Markdown**: обмеження загальної висоти рамки блоку, смуги прокручування з'являються всередині блоку.
