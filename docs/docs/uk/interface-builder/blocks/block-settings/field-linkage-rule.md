# Правила взаємодії полів

## Вступ

Правила взаємодії полів дозволяють динамічно змінювати стан полів у блоках форм та деталей залежно від дій користувача. Наразі ці правила підтримуються в таких блоках:

- [Блок форми](/interface-builder/blocks/data-blocks/form)
- [Блок деталей](/interface-builder/blocks/data-blocks/details)
- [Підформа](/interface-builder/fields/specific/sub-form)

## Інструкції з використання

### **Блок форми**

У блоці форми правила взаємодії можуть динамічно змінювати поведінку полів на основі певних умов:

- **Керування видимістю поля (показати/приховати)**: Вирішуйте, чи відображати поточне поле, залежно від значень інших полів.
- **Встановлення поля як обов'язкового**: Динамічно встановлюйте поле як обов'язкове або необов'язкове за певних умов.
- **Присвоєння значення**: Автоматично присвоюйте значення полю на основі умов.
- **Виконання вказаного JavaScript**: Пишіть JavaScript відповідно до бізнес-вимог.

### **Блок деталей**

У блоці деталей правила взаємодії в основному використовуються для динамічного керування відображенням та приховуванням полів у цьому блоці.

## Взаємодія властивостей

### Присвоєння значення

Приклад: Коли замовлення позначено як додаткове, статус замовлення автоматично встановлюється на «Очікує перевірки».

![20251029115348](https://static-docs.nocobase.com/20251029115348.png)

### Обов'язкове поле

Приклад: Коли статус замовлення «Оплачено», поле суми замовлення стає обов'язковим.

![20251029115031](https://static-docs.nocobase.com/20251029115031.png)

### Показати/Приховати

Приклад: Платіжний рахунок та загальна сума відображаються лише тоді, коли статус замовлення «Очікує оплати».
