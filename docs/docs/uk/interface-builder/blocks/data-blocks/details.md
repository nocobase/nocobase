# Блок "Деталі"

## Вступ

Блок "Деталі" використовується для відображення значень полів кожного запису даних. Він підтримує гнучке розташування полів та має вбудовані функції для роботи з даними, що робить його зручним для перегляду та керування інформацією.

## Налаштування блоку

![20251029202614](https://static-docs.nocobase.com/20251029202614.png)

### Правила зв'язування блоків

Керуйте поведінкою блоку (наприклад, чи відображати його, чи виконувати JavaScript) за допомогою правил зв'язування.

![20251023195004](https://static-docs.nocobase.com/20251023195004.png)
Детальніше дивіться у розділі [Правила зв'язування](/interface-builder/linkage-rule)

### Встановлення області даних

Приклад: Відображати лише оплачені замовлення.

![20251023195051](https://static-docs.nocobase.com/20251023195051.png)

Детальніше дивіться у розділі [Встановлення області даних](/interface-builder/blocks/block-settings/data-scope)

### Правила зв'язування полів

Правила зв'язування в блоці "Деталі" підтримують динамічне налаштування відображення/приховування полів.

Приклад: Не відображати суму, якщо статус замовлення "Скасовано".

![20251023200739](https://static-docs.nocobase.com/20251023200739.png)

Детальніше дивіться у розділі [Правила зв'язування](/interface-builder/linkage-rule)

## Налаштування полів

### Поля з цієї колекції

> **Примітка**: Поля з успадкованих колекцій (тобто поля батьківської колекції) автоматично об'єднуються та відображаються у поточному списку полів.

![20251023201012](https://static-docs.nocobase.com/20251023201012.png)

### Поля з пов'язаних колекцій

> **Примітка**: Підтримується відображення полів з пов'язаних колекцій (наразі лише для зв'язків "один до одного").

![20251023201258](https://static-docs.nocobase.com/20251023201258.png)

### Інші поля
- JS Field
- JS Item
- Розділювач
- Markdown

![20251023201514](https://static-docs.nocobase.com/20251023201514.png)

> **Порада**: Ви можете написати JavaScript для реалізації власного відображення вмісту, що дозволить вам показувати складнішу інформацію.  
> Наприклад, можна відтворювати різні ефекти відображення залежно від типів даних, умов або логіки.

![20251023202017](https://static-docs.nocobase.com/20251023202017.png)
