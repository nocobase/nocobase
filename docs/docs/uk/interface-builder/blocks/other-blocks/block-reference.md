# Блок посилання

## Вступ
Блок посилання дозволяє відобразити вже налаштований блок безпосередньо на поточній сторінці, просто вказавши UID цільового блоку. Це усуває потребу в повторному налаштуванні.

## Активація плагіна
Цей плагін є вбудованим, але за замовчуванням вимкнений.
Відкрийте «Керування плагінами» → знайдіть «Блок: Посилання» → натисніть «Увімкнути».

![Enable Reference block in Plugin Manager](https://static-docs.nocobase.com/block-reference-enable-20251102.png)

## Як додати блок
1) Додайте блок → група «Інші блоки» → виберіть «Блок посилання».  
2) У «Налаштуваннях посилання» налаштуйте:
   - `Block UID`: UID цільового блоку
   - `Reference mode`: виберіть `Посилання` або `Копіювання`

![Reference block add and configure demo](https://static-docs.nocobase.com/20251102193949_rec_.gif)

### Як отримати UID блоку
- Відкрийте меню налаштувань цільового блоку і натисніть `Копіювати UID`, щоб скопіювати його UID.  

![Copy UID from block settings](https://static-docs.nocobase.com/block-reference-copy-uid-20251102.png)

## Режими та поведінка
- `Посилання` (за замовчуванням)
  - Використовує ту саму конфігурацію, що й оригінальний блок; зміни в оригінальному блоці або в будь-якому місці посилання синхронно оновлюватимуть усі посилання.

- `Копіювання`
  - Створює незалежний блок, ідентичний оригінальному на момент копіювання; подальші зміни в них не впливають одна на одну і не синхронізуються.

## Конфігурація
- Блок посилання:
  - «Налаштування посилання»: використовується для вказівки UID цільового блоку та вибору режиму «Посилання/Копіювання»;
  - Також відображаються повні налаштування блоку, на який посилаються (що еквівалентно прямому налаштуванню оригінального блоку).

![Reference block settings](https://static-docs.nocobase.com/block-reference-settings-20251102.png)

- Результат копіювання:
  - Отриманий після копіювання блок має той самий тип, що й оригінальний блок, і містить лише власні налаштування;
  - Він більше не містить «Налаштувань посилання».

## Стани помилок та заповнювачі
- Якщо ціль відсутня/недійсна: відображається повідомлення про помилку. Ви можете повторно вказати UID блоку в налаштуваннях блоку посилання (Налаштування посилання → Block UID), і після збереження відображення відновиться.  

![Error state when target block is invalid](https://static-docs.nocobase.com/block-reference-error-20251102.png)

## Примітки та обмеження
- Експериментальна функція — використовуйте з обережністю у виробничому середовищі.
- При копіюванні блоку деякі конфігурації, що залежать від UID цільового блоку, можливо, доведеться переналаштувати.
- Усі конфігурації блоку посилання автоматично синхронізуються, включно з такими налаштуваннями, як «діапазон даних». Однак блок посилання може мати власну [конфігурацію потоку подій](/interface-builder/event-flow/), тому за допомогою потоків подій та користувацьких JavaScript-дій ви можете опосередковано досягти різних діапазонів даних або пов'язаних конфігурацій для кожного посилання.
