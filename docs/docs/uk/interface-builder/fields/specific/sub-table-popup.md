# Підтаблиця (редагування у спливаючому вікні)

## Вступ

Підтаблиця (редагування у спливаючому вікні) використовується для керування даними з множинними зв'язками (такими як «один до багатьох» або «багато до багатьох») у межах форми. Таблиця відображає лише поточні пов'язані записи. Додавання або редагування записів здійснюється у спливаючому вікні, а дані зберігаються в базі даних разом із надсиланням основної форми.

## Інструкція з використання

![20260212152204](https://static-docs.nocobase.com/20260212152204.png)

**Відповідні сценарії:**

- Поля зв'язку: o2m / m2m / mbm
- Типове застосування: деталі замовлення, списки підпунктів, пов'язані теги/учасники тощо.

## Налаштування поля

### Дозволити вибір наявних даних (за замовчуванням: Увімкнено)

Підтримує вибір зв'язків із наявних записів.

![20260212152312](https://static-docs.nocobase.com/20260212152312.png)

![20260212152343](https://static-docs.nocobase.com/20260212152343.gif)

### Компонент поля

[Компонент поля](/interface-builder/fields/association-field): перемикання на інші компоненти полів зв'язку, такі як вибір зі списку, селектор колекції тощо.

### Дозволити відв'язування наявних даних (за замовчуванням: Увімкнено)

> Керує тим, чи дозволено відв'язувати наявні пов'язані дані у формі редагування. Щойно додані дані завжди можна видалити.

![20260212165752](https://static-docs.nocobase.com/20260212165752.gif)

### Дозволити додавання (за замовчуванням: Увімкнено)

Керує відображенням кнопки «Додати». Якщо користувач не має прав `create` для цільової колекції, кнопка буде вимкнена з підказкою про відсутність прав.

### Дозволити швидке редагування (за замовчуванням: Вимкнено)

Якщо увімкнено, при наведенні курсору на комірку з'явиться іконка редагування, що дозволяє швидко змінити вміст комірки.

Ви можете увімкнути швидке редагування для всіх полів у налаштуваннях компонента поля зв'язку.

![20260212165102](https://static-docs.nocobase.com/20260212165102.png)

Його також можна увімкнути для окремих полів стовпців.

![20260212165025](https://static-docs.nocobase.com/20260212165025.png)

![20260212165201](https://static-docs.nocobase.com/20260212165201.gif)

### Розмір сторінки (за замовчуванням: 10)

Встановлює кількість записів, що відображаються на одній сторінці підтаблиці.

## Особливості поведінки

- При виборі наявних записів виконується дедуплікація за первинним ключем, щоб уникнути повторного зв'язування одного й того самого запису.
- Щойно додані записи відразу відображаються в підтаблиці, а перегляд автоматично переходить на сторінку, що містить новий запис.
- Рядкове редагування змінює дані лише для поточного рядка.
- Видалення запису лише розриває зв'язок у поточній формі; воно не видаляє вихідні дані з бази даних.
- Дані зберігаються в базі даних лише під час надсилання основної форми.
