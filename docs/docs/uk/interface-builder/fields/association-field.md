# Компоненти полів зв'язків

## Вступ

Компоненти полів зв'язків у NocoBase розроблені, щоб допомогти вам краще відображати та обробляти пов'язані дані. Незалежно від типу зв'язку, ці компоненти є гнучкими та універсальними, дозволяючи вам вибирати та налаштовувати їх відповідно до ваших конкретних потреб.

### Випадаючий список

Для всіх полів зв'язків, окрім випадків, коли цільова **колекція** є файловою, компонентом за замовчуванням у режимі редагування є випадаючий список. Опції випадаючого списку відображають значення поля заголовка, що ідеально підходить для сценаріїв, де пов'язані дані можна швидко вибрати, відобразивши ключову інформацію поля.

![20240429205659](https://static-docs.nocobase.com/20240429205659.png)

Докладніше дивіться у розділі [Випадаючий список](/interface-builder/fields/specific/select)

### Селектор даних

Селектор даних відображає дані у спливаючому вікні. Ви можете налаштувати поля, які потрібно відображати в селекторі даних (включно з полями вкладених зв'язків), що дозволяє точніше вибирати пов'язані дані.

![20240429210824](https://static-docs.nocobase.com/20240429210824.png)

Докладніше дивіться у розділі [Селектор даних](/interface-builder/fields/specific/picker)

### Підформа

При роботі зі складнішими даними зв'язків використання випадаючого списку або селектора даних може бути незручним. У таких випадках вам доведеться часто відкривати спливаючі вікна. Для цих сценаріїв можна використовувати підформу. Вона дозволяє безпосередньо керувати полями пов'язаної **колекції** на поточній сторінці або в поточному блоці спливаючого вікна, не відкриваючи постійно нові спливаючі вікна, що робить робочий процес більш плавним. Багаторівневі зв'язки відображаються як вкладені форми.

![20251029122948](https://static-docs.nocobase.com/20251029122948.png)

Докладніше дивіться у розділі [Підформа](/interface-builder/fields/specific/sub-form)

### Підтаблиця

Підтаблиця відображає записи зв'язків "один-до-багатьох" або "багато-до-багатьох" у табличному форматі. Вона надає чіткий, структурований спосіб відображення та керування пов'язаними даними, а також підтримує масове створення нових даних або вибір існуючих даних для зв'язування.

![20251029123042](https://static-docs.nocobase.com/20251029123042.png)

Докладніше дивіться у розділі [Підтаблиця](/interface-builder/fields/specific/sub-table)

### Піддеталь

Піддеталь є відповідним компонентом підформи в режимі перегляду. Вона підтримує відображення даних із вкладеними багаторівневими зв'язками.

![20251030213050](https://static-docs.nocobase.com/20251030213050.png)

Докладніше дивіться у розділі [Піддеталь](/interface-builder/fields/specific/sub-detail)

### Файловий менеджер

Файловий менеджер — це компонент поля зв'язку, який використовується спеціально тоді, коли цільова **колекція** зв'язку є файловою **колекцією**.

![20240429222753](https://static-docs.nocobase.com/20240429222753.png)

Докладніше дивіться у розділі [Файловий менеджер](/interface-builder/fields/specific/file-manager)

### Заголовок

Компонент поля заголовка — це компонент поля зв'язку, який використовується в режимі перегляду. Налаштувавши поле заголовка, ви можете налаштувати відповідний компонент поля.

![20251030213327](https://static-docs.nocobase.com/20251030213327.png)

Докладніше дивіться у розділі [Заголовок](/interface-builder/fields/specific/title)
