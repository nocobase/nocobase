# Налаштування діапазону даних

## Вступ

Налаштування діапазону даних для поля зв'язку схоже на налаштування діапазону даних для блоку. Воно встановлює умови фільтрації за замовчуванням для пов'язаних даних.

## Інструкції з використання

![20251028211328](https://static-docs.nocobase.com/20251028211328.png)

### Статичне значення

Приклад: Для зв'язування можна вибрати лише не видалені товари.

> Список полів містить поля з цільової колекції поля зв'язку.

![20251028211434](https://static-docs.nocobase.com/20251028211434.png)

### Значення змінної

Приклад: Для зв'язування можна вибрати лише товари, дата обслуговування яких пізніша за дату замовлення.

![20251028211727](https://static-docs.nocobase.com/20251028211727.png)

Докладніше про змінні дивіться у розділі [Змінні](/interface-builder/variables)

### Взаємодія полів зв'язку

Взаємодія між полями зв'язку реалізується шляхом налаштування діапазону даних.

Приклад: Колекція Замовлення має поле зв'язку «Продукт можливості» (один-до-багатьох) та поле зв'язку «Можливість» (багато-до-одного). Колекція Продукт можливості також має поле зв'язку «Можливість» (багато-до-одного). У блоці форми замовлення доступні дані для «Продукту можливості» фільтруються, щоб показати лише ті продукти можливості, які пов'язані з поточною вибраною «Можливістю» у формі.
