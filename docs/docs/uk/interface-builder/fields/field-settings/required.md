# Обов'язково

## Вступ

Обов'язковість — це поширене правило для валідації форм. Ви можете увімкнути його безпосередньо в налаштуваннях поля або динамічно встановити поле як обов'язкове за допомогою правил зв'язування форми.

## Де налаштувати обов'язковість поля

### Налаштування поля колекції

Коли поле колекції встановлено як обов'язкове, це запускає валідацію на стороні сервера, а на стороні клієнта воно також за замовчуванням відображається як обов'язкове (без можливості зміни).
![20251025175418](https://static-docs.nocobase.com/20251025175418.png)

### Налаштування поля

Безпосередньо встановіть поле як обов'язкове. Це підходить для полів, які завжди повинні бути заповнені користувачем, наприклад, ім'я користувача, пароль тощо.

![20251028222818](https://static-docs.nocobase.com/20251028222818.png)

### Правила зв'язування

Встановіть поле як обов'язкове на основі умов за допомогою правил зв'язування полів блоку форми.

Приклад: Номер замовлення є обов'язковим, коли дата замовлення не порожня.

![20251028223004](https://static-docs.nocobase.com/20251028223004.png)
