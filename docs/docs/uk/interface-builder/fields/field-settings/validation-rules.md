# Налаштування правил валідації

## Вступ

Правила валідації використовуються для забезпечення відповідності введених користувачем даних очікуваним вимогам.

## Де можна налаштувати правила валідації полів

### Налаштування правил валідації для полів колекції

Більшість полів підтримують налаштування правил валідації. Після того, як для поля налаштовано правила валідації, при надсиланні даних спрацьовує валідація на стороні сервера. Різні типи полів підтримують різні правила валідації.

- **Поле дати**

  ![20251028225946](https://static-docs.nocobase.com/20251028225946.png)

- **Числове поле**

  ![20251028230418](https://static-docs.nocobase.com/20251028230418.png)

- **Текстове поле**

  Окрім обмеження довжини тексту, текстові поля також підтримують використання власних регулярних виразів для більш точної валідації.

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### Фронтенд-валідація в конфігурації поля

Правила валідації, встановлені в конфігурації поля, запускають фронтенд-перевірку, щоб переконатися, що введені користувачем дані відповідають вимогам.

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

![20251028230255](https://static-docs.nocobase.com/20251028230255.png)

**Текстові поля** також підтримують власну валідацію за допомогою регулярних виразів для відповідності специфічним вимогам до формату.

![20251028230903](https://static-docs.nocobase.com/20251028230903.png)
