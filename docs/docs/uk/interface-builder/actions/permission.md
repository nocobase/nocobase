# Дозволи на дії

## Вступ

У NocoBase 2.0 дозволи на дії наразі головним чином контролюються дозволами на ресурси колекцій:

- **Дозволи на ресурси колекції**: Використовуються для уніфікованого контролю базових дозволів на дії (такі як створення (Create), перегляд (View), оновлення (Update) та видалення (Delete)) для різних ролей щодо колекції. Цей дозвіл застосовується до всієї колекції в межах джерела даних, забезпечуючи, що відповідні дозволи на дії ролі для цієї колекції залишаються узгодженими на різних сторінках, у спливаючих вікнах та блоках.

### Дозволи на ресурси колекції

У системі дозволів NocoBase дозволи на дії для колекцій в основному поділяються за вимірами CRUD, щоб забезпечити узгодженість та стандартизацію в управлінні дозволами. Наприклад:

- **Дозвіл на створення (Create)**: Контролює всі дії, пов'язані зі створенням для цієї колекції, включаючи дії додавання, копіювання тощо. Якщо роль має дозвіл на створення для цієї колекції, то дії додавання, копіювання та інші дії, пов'язані зі створенням, будуть видимими на всіх сторінках та у всіх спливаючих вікнах.
- **Дозвіл на видалення (Delete)**: Контролює дію видалення для цієї колекції. Дозвіл залишається узгодженим, незалежно від того, чи це масове видалення в блоці таблиці, чи видалення окремого запису в блоці деталей.
- **Дозвіл на оновлення (Update)**: Контролює дії типу оновлення для цієї колекції, такі як дії редагування та оновлення записів.
- **Дозвіл на перегляд (View)**: Контролює видимість даних цієї колекції. Відповідні блоки даних (таблиця, список, деталі тощо) видимі лише тоді, коли роль має дозвіл на перегляд для цієї колекції.

Цей універсальний метод управління дозволами підходить для стандартизованого контролю дозволів на дані, забезпечуючи, що для `однієї й тієї ж колекції`, `одна й та сама дія` має `узгоджені` правила дозволів на `різних сторінках, у спливаючих вікнах та блоках`, що забезпечує уніфікованість та зручність обслуговування.

#### Глобальні дозволи

Глобальні дозволи на дії застосовуються до всіх колекцій у межах джерела даних, і класифікуються за типом ресурсу наступним чином:

#### Дозволи на дії для конкретних колекцій

Дозволи на дії для конкретних колекцій переважають загальні дозволи джерела даних, додатково уточнюючи дозволи на дії та дозволяючи налаштовувати дозволи для доступу до ресурсів конкретної колекції. Ці дозволи поділяються на два аспекти:

1. Дозволи на дії: Дозволи на дії включають додавання, перегляд, редагування, видалення, експорт та імпорт. Ці дозволи налаштовуються за виміром обсягу даних:

   - Усі записи: Дозволяє користувачам виконувати дії над усіма записами в колекції.
   - Власні записи: Обмежує користувачів виконувати дії лише над записами даних, які вони створили.

2. Дозволи на поля: Дозволи на поля дозволяють налаштовувати дозволи для кожного поля в різних діях. Наприклад, деякі поля можна налаштувати так, щоб вони були лише для перегляду, без можливості редагування.
