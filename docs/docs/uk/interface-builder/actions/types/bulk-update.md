# Масове оновлення

## Вступ

Дія масового оновлення використовується, коли вам потрібно застосувати одне й те саме оновлення до групи записів. Перш ніж виконати масове оновлення, вам потрібно заздалегідь визначити логіку присвоєння значень полям для оновлення. Ця логіка буде застосована до всіх вибраних записів, коли ви натиснете кнопку оновлення.

![20251029195320](https://static-docs.nocobase.com/20251029195320.png)

## Налаштування дії

![20251029195729](https://static-docs.nocobase.com/20251029195729.png)

### Дані для оновлення

Вибрані/Всі, за замовчуванням — Вибрані.

![20251029200034](https://static-docs.nocobase.com/20251029200034.png)

### Присвоєння значень полям

Налаштуйте поля для масового оновлення. Оновлюватимуться лише ті поля, які ви вкажете.

Наприклад, налаштуйте дію масового оновлення в таблиці замовлень, щоб масово оновити вибрані дані до статусу «Очікує на затвердження».

![20251029200109](https://static-docs.nocobase.com/20251029200109.png)

- [Кнопка редагування](/interface-builder/actions/action-settings/edit-button): Редагуйте заголовок, тип та іконку кнопки;
- [Правило зв'язування](/interface-builder/actions/action-settings/linkage-rule): Динамічно показуйте/приховуйте кнопку;
- [Повторне підтвердження](/interface-builder/actions/action-settings/double-check)
