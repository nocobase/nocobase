# Дублювання

## Вступ

Дія дублювання дозволяє користувачам швидко створювати нові записи на основі наявних даних. Підтримуються два способи дублювання: **Пряме дублювання** та **Дублювання у форму з подальшим заповненням**.

## Встановлення

Вбудований плагін, не потребує окремого встановлення.

## Режими дублювання

![20260209224344](https://static-docs.nocobase.com/20260209224344.png)

### Пряме дублювання

![20260209224506](https://static-docs.nocobase.com/20260209224506.png)

- За замовчуванням виконується як «Пряме дублювання»;
- **Поля шаблону**: вкажіть поля, які потрібно скопіювати. Підтримується функція «Вибрати все». Це обов'язкове налаштування.

![20260209225910](https://static-docs.nocobase.com/20260209225910.gif)

Після завершення налаштування натисніть кнопку, щоб продублювати дані.

### Дублювання у форму з подальшим заповненням

Налаштовані поля шаблону будуть заповнені у формі як **значення за замовчуванням**. Користувачі можуть змінити ці значення перед відправкою для завершення дублювання.

![20260209224704](https://static-docs.nocobase.com/20260209224704.png)

**Налаштування полів шаблону**: лише вибрані поля будуть перенесені та використані як значення за замовчуванням.

![20260209225148](https://static-docs.nocobase.com/20260209225148.png)

#### Синхронізація полів форми

- Автоматично аналізує поля, вже налаштовані в поточному блоці форми, і використовує їх як поля шаблону;
- Якщо згодом поля блоку форми будуть змінені (наприклад, налаштування компонентів полів зв'язку), потрібно знову відкрити конфігурацію шаблону та натиснути **Синхронізувати поля форми**, щоб забезпечити узгодженість.

![20260209225450](https://static-docs.nocobase.com/20260209225450.gif)

Дані шаблону будуть заповнені як значення форми за замовчуванням, і користувачі зможуть надіслати їх після редагування для завершення дублювання.

### Додаткові примітки

#### Дублювання, Посилання, Попереднє завантаження

Різні типи полів (типи зв'язків) мають різну логіку обробки: **Дублювання / Посилання / Попереднє завантаження**. **Компонент поля** зв'язку також впливає на цю логіку:

- Select / Record picker: використовується для **Посилання**
- Sub-form / Sub-table: використовується для **Дублювання**

**Дублювання**

- Звичайні поля дублюються;
- `hasOne` / `hasMany` можуть бути лише продубльовані (для таких зв'язків не слід використовувати компоненти вибору, такі як випадаючий список або вибір запису; замість цього слід використовувати компоненти «Підформа» або «Підтаблиця»);
- Зміна компонента для `hasOne` / `hasMany` **не змінить** логіку обробки (вона залишиться «Дублюванням»);
- Для дубльованих полів зв'язку можна вибрати всі дочірні поля.

**Посилання**

- `belongsTo` / `belongsToMany` розглядаються як Посилання;
- Якщо компонент поля змінити з «Випадаючого списку» на «Підформу», тип зв'язку зміниться з **Посилання на Дублювання** (після того, як він стане «Дублюванням», усі дочірні поля стануть доступними для вибору).

**Попереднє завантаження**

- Поля зв'язку під полем «Посилання» розглядаються як Попереднє завантаження;
- Поля попереднього завантаження можуть стати «Посиланнями» або «Дублюванням» після зміни компонента.

#### Вибрати все

- Вибирає всі **Поля для дублювання** та **Поля для посилання**.

#### Наступні поля будуть відфільтровані із запису, вибраного як шаблон даних:

- Первинні ключі дубльованих даних зв'язку фільтруються; первинні ключі для «Посилання» та «Попереднього завантаження» не фільтруються;
- Зовнішні ключі;
- Поля, що не допускають дублікатів (унікальні);
- Поля сортування;
- Поля автоматичного кодування (Sequence);
- Пароль;
- Ким створено, Дата створення;
- Ким востаннє оновлено, Дата останнього оновлення.

#### Синхронізація полів форми

- Автоматично перетворює поля, налаштовані в поточному блоці форми, у поля шаблону;
- Після зміни полів блоку форми (наприклад, налаштування компонентів полів зв'язку) необхідно виконати синхронізацію знову, щоб забезпечити узгодженість.
