## UID спливаючого вікна

«UID спливаючого вікна» — це унікальний ідентифікатор компонента, який відкриває це вікно. Він також відповідає сегменту `viewUid` у `view/:viewUid` в поточному адресному рядку. Ви можете швидко отримати його, натиснувши «Копіювати UID спливаючого вікна» в меню налаштувань поля або кнопки, що викликає спливаюче вікно. Налаштування UID спливаючого вікна дозволяє повторно використовувати його.

![popup-copy-uid-20251102](https://static-docs.nocobase.com/popup-copy-uid-20251102.png)

### Внутрішнє спливаюче вікно (за замовчуванням)
- «UID спливаючого вікна» дорівнює UID поточної кнопки дії (за замовчуванням використовується UID цієї кнопки).

### Зовнішнє спливаюче вікно (повторне використання)
- Введіть UID іншої кнопки (тобто UID спливаючого вікна) у полі «UID спливаючого вікна», щоб повторно використовувати це вікно в іншому місці.
- Типове використання: кілька сторінок/блоків використовують один і той же інтерфейс та логіку спливаючого вікна, уникаючи дублювання конфігурації.
- При використанні зовнішнього спливаючого вікна деякі параметри стають доступними лише для читання (див. нижче).

## Інші пов'язані параметри

- `Data source / Collection`: Лише для читання. Вказує на джерело даних та **колекцію**, до якої прив'язане спливаюче вікно; за замовчуванням використовується **колекція** поточного блоку. У режимі зовнішнього спливаючого вікна використовуються налаштування цільового вікна, і цей параметр не можна змінити.
- `Association name`: Необов'язково. Використовується для відкриття спливаючого вікна з поля асоціації; відображається лише тоді, коли існує значення за замовчуванням. У режимі зовнішнього спливаючого вікна використовуються налаштування цільового вікна, і цей параметр не можна змінити.
- `Source ID`: З'являється лише тоді, коли встановлено `Association name`; за замовчуванням використовує `sourceId` поточного контексту; може бути змінною або фіксованим значенням за потребою.
- `filterByTk`: Може бути порожнім, змінною або фіксованим значенням, що використовується для обмеження записів даних, які завантажує спливаюче вікно.
