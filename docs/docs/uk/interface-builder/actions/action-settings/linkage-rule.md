# Правила зв'язування дій

## Вступ

Правила зв'язування дій дозволяють користувачам динамічно керувати станом дії (наприклад, відображення, увімкнення, приховування, вимкнення тощо) на основі певних умов. Налаштувавши ці правила, ви можете зв'язати поведінку кнопок дій з поточним записом, роллю користувача або контекстними даними.

![20251029150224](https://static-docs.nocobase.com/20251029150224.png)

## Як використовувати

Коли умова виконана (за відсутності умов, вона вважається виконаною за замовчуванням), це запускає виконання налаштувань властивостей або JavaScript. Константи та змінні підтримуються в умові.

![20251030224601](https://static-docs.nocobase.com/20251030224601.png)

Правило підтримує зміну властивостей кнопок.

![20251029150452](https://static-docs.nocobase.com/20251029150452.png)

## Константи

Приклад: Оплачені замовлення не можна редагувати.

![20251029150638](https://static-docs.nocobase.com/20251029150638.png)

## Змінні

### Системні змінні

![20251029150014](https://static-docs.nocobase.com/20251029150014.png)

Приклад 1: Керування відображенням кнопки на основі поточного типу пристрою.

![20251029151057](https://static-docs.nocobase.com/20251029151057.png)

Приклад 2: Кнопка масового оновлення у заголовку таблиці блоку замовлень доступна лише для ролі адміністратора; інші ролі не можуть виконати цю дію.

### Контекстні змінні

Приклад: Кнопка «Додати» на можливостях замовлення (блок зв'язків) увімкнена лише тоді, коли статус замовлення — «Очікує оплати» або «Чернетка». В інших статусах кнопка буде вимкнена.
