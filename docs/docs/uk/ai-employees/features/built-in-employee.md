# Вбудовані AI-співробітники

NocoBase пропонує кілька вбудованих AI-співробітників, розроблених для конкретних сценаріїв.

Вам потрібно лише налаштувати сервіс LLM та увімкнути відповідного співробітника, щоб розпочати роботу; моделі можна перемикати за потреби безпосередньо в діалозі.


## Огляд

![clipboard-image-1766653060](https://static-docs.nocobase.com/clipboard-image-1766653060.png)

| Ім'я співробітника | Роль | Основні можливості |
| :--- | :--- | :--- |
| **Cole** | Асистент NocoBase | Питання та відповіді по продукту, пошук у документації |
| **Ellis** | Експерт з електронної пошти | Написання листів, створення резюме, пропозиції відповідей |
| **Dex** | Спеціаліст з упорядкування даних | Переклад полів, форматування, вилучення інформації |
| **Viz** | Аналітик інсайтів | Інсайти даних, аналіз трендів, інтерпретація ключових показників |
| **Lexi** | Асистент з перекладу | Багатомовний переклад, допомога в комунікації |
| **Vera** | Аналітик-дослідник | Пошук у мережі, агрегація інформації, глибоке дослідження |
| **Dara** | Експерт з візуалізації даних | Налаштування діаграм, створення візуальних звітів |
| **Orin** | Експерт з моделювання даних | Допомога в проєктуванні структури колекцій, пропозиції щодо полів |
| **Nathan** | Frontend-інженер | Допомога в написанні фрагментів frontend-коду, коригування стилів |


Ви можете натиснути на **плаваючу кнопку AI** у нижньому правому куті інтерфейсу програми та вибрати потрібного співробітника, щоб розпочати співпрацю.


## AI-співробітники для спеціалізованих сценаріїв

Деякі вбудовані AI-співробітники (конструкторського типу) не відображаються у загальному списку в нижньому правому куті; вони мають власні робочі зони, наприклад:

* **Orin** з'являється лише на сторінці налаштування джерела даних;
* **Dara** з'являється лише на сторінці налаштування діаграм;
* **Nathan** з'являється лише в редакторі JS.



---

Нижче наведено кілька типових сценаріїв застосування AI-співробітників, які можуть надихнути вас. Більше можливостей чекає на ваше дослідження в реальних бізнес-процесах.


## Viz: Аналітик інсайтів

### Огляд

> Створюйте діаграми та інсайти в один клік, дозвольте даним говорити самим за себе.

**Viz** — це вбудований **AI-аналітик інсайтів**.
Він вміє зчитувати дані на вашій поточній сторінці (наприклад, Ліди, Можливості, Контрагенти) і автоматично створювати графіки трендів, діаграми порівняння, картки KPI та стислі висновки, роблячи бізнес-аналіз простим та інтуїтивно зрозумілим.

> Хочете знати, «Чому останнім часом впали продажі»?
> Просто запитайте Viz, і він підкаже, на якому етапі стався спад, які можливі причини та якими можуть бути наступні кроки.

### Сценарії використання

Будь то щомісячні бізнес-огляди, ROI каналів або воронки продажів — ви можете доручити Viz аналіз, створення діаграм та інтерпретацію результатів.

| Сценарій | Що ви хочете дізнатися | Результат від Viz |
| -------- | ------------ | ------------------- |
| **Щомісячний огляд** | Чим цей місяць кращий за минулий? | Картка KPI + Графік трендів + Три поради щодо покращення |
| **Аналіз зростання** | Зростання доходу зумовлене обсягом чи ціною? | Діаграма декомпозиції факторів + Таблиця порівняння |
| **Аналіз каналів** | У який канал варто інвестувати далі? | Діаграма ROI + Крива утримання + Рекомендації |
| **Аналіз воронки** | На якому етапі застряє трафік? | Діаграма воронки + Пояснення вузьких місць |
| **Утримання клієнтів** | Які клієнти є найціннішими? | Діаграма сегментації RFM + Крива утримання |
| **Оцінка акцій** | Наскільки ефективною була велика промо-акція? | Діаграма порівняння + Аналіз еластичності цін |

### Як користуватися

**Точки входу на сторінці**

* **Кнопка у верхньому правому куті (рекомендовано)**
  
  На сторінках на кшталт «Ліди», «Можливості» або «Контрагенти» натисніть на **іконку Viz** у верхньому правому куті, щоб вибрати попередньо налаштовані завдання, як-от:

  * Конверсія етапів та тренди
  * Порівняння джерел трафіку
  * Щомісячний аналітичний огляд

  ![clipboard-image-1771913319](https://static-docs.nocobase.com/clipboard-image-1771913319.png)

* **Глобальна панель у нижньому правому куті**
  
  На будь-якій сторінці ви можете викликати глобальну панель AI та звернутися безпосередньо до Viz:

  ```
  Проаналізуй зміни продажів за останні 90 днів
  ```

  Viz автоматично підхопить контекст даних поточної сторінки.

**Взаємодія**

Viz підтримує запити природною мовою та розуміє уточнюючі запитання.
Приклад:

```
Привіт, Viz, створи тренд лідів за цей місяць.
```

```
Покажи результати лише для сторонніх каналів.
```

```
А в якому регіоні зростання найшвидше?
```

Кожне наступне запитання буде поглиблювати аналіз на основі попередніх результатів без необхідності повторно вводити умови даних.

### Поради для спілкування з Viz

| Метод | Ефект |
| ---------- | ------------------- |
| Вказуйте часовий діапазон | «За останні 30 днів» або «Минулий місяць проти поточного» дають точніші результати |
| Визначайте виміри | «Переглянути за регіоном/каналом/продуктом» допомагає структурувати аналіз |
| Зосереджуйтесь на трендах, а не на деталях | Viz найкраще виявляє напрямки змін та ключові причини |
| Використовуйте природну мову | Не потрібен командний синтаксис, просто запитуйте, як у звичайному чаті |

## Dex: Спеціаліст з упорядкування даних

### Огляд

> Швидко вилучайте та заповнюйте форми, перетворюючи неструктуровану інформацію на структуровані дані.

`Dex` — це експерт з упорядкування даних, який вилучає необхідну інформацію з неструктурованих даних або файлів і організовує її в структурований вигляд. Він також може використовувати інструменти для автоматичного заповнення форм цією інформацією.

### Як користуватися

На сторінці форми викличте `Dex`, щоб відкрити вікно діалогу.

![20251022155316](https://static-docs.nocobase.com/20251022155316.png)

Натисніть `Add work context` у полі введення та виберіть `Pick block`; сторінка перейде в режим вибору блоку.

![20251022155414](https://static-docs.nocobase.com/20251022155414.png)

Виберіть блок форми на сторінці.

![20251022155523](https://static-docs.nocobase.com/20251022155523.png)

Введіть у діалоговому вікні дані, які `Dex` має впорядкувати.

![20251022155555](https://static-docs.nocobase.com/20251022155555.png)

Після відправлення `Dex` структурує дані та використає свої навички, щоб оновити інформацію у вибраній формі.

![20251022155733](https://static-docs.nocobase.com/20251022155733.png)


---



## Orin: Експерт з моделювання даних

### Огляд

> Інтелектуальне проєктування колекцій та оптимізація структури бази даних.

`Orin` — експерт з моделювання даних. На сторінці налаштування основного джерела даних ви можете залучити `Orin` для створення або зміни колекцій.

![20251022160628](https://static-docs.nocobase.com/20251022160628.png)

### Як користуватися

Перейдіть до плагіна «Диспетчер джерел даних» та виберіть налаштування основного джерела даних.

![20251022161146](https://static-docs.nocobase.com/20251022161146.png)

Натисніть на аватар `Orin` у верхньому правому куті, щоб відкрити діалогове вікно AI-співробітника.

![20251022161641](https://static-docs.nocobase.com/20251022161641.png)

Опишіть свої вимоги до моделювання для `Orin`, надішліть повідомлення та зачекайте на відповідь. 

Коли `Orin` підтвердить ваші вимоги, він застосує свої навички та надасть попередній перегляд моделі даних.

Після перегляду натисніть кнопку `Finish review and apply`, щоб створити колекції згідно з моделлю, запропонованою `Orin`.

![20251022162142](https://static-docs.nocobase.com/20251022162142.png)

## Nathan: Frontend-інженер

### Огляд

> Допомагає писати та оптимізувати frontend-код для реалізації складної логіки взаємодії.

`Nathan` — експерт з frontend-розробки в NocoBase. У сценаріях, де потрібно писати JavaScript, таких як `JSBlock`, `JSField`, `JSItem`, `JSColumn`, `JSAction`, `Event Flow`, `Linkage`, у верхньому правому куті редактора коду з'явиться аватар `Nathan`. Ви можете попросити його допомогти написати або змінити код у редакторі.

![20251022163405](https://static-docs.nocobase.com/20251022163405.png)

### Як користуватися

У редакторі коду натисніть на `Nathan`, щоб відкрити діалогове вікно. Код із редактора буде автоматично доданий до поля введення як контекст програми та надісланий `Nathan`.

![20251022163935](https://static-docs.nocobase.com/20251022163935.png)

Введіть ваші вимоги до коду, надішліть їх `Nathan` і зачекайте на відповідь.

![20251022164041](https://static-docs.nocobase.com/20251022164041.png)

Натисніть кнопку `Apply to editor` на блоці коду у відповіді `Nathan`, щоб замінити код у редакторі його версією.

![20251022164323](https://static-docs.nocobase.com/20251022164323.png)

Натисніть кнопку `Run` у редакторі коду, щоб миттєво побачити результат.

![20251022164436](https://static-docs.nocobase.com/20251022164436.png)

### Історія коду

Натисніть іконку «Командний рядок» у верхньому правому куті діалогового вікна `Nathan`, щоб переглянути фрагменти коду, які ви надсилали, та відповіді `Nathan` у межах поточної сесії.

![20251022164644](https://static-docs.nocobase.com/20251022164644.png)
