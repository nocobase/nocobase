# Огляд

![clipboard-image-1771905619](https://static-docs.nocobase.com/clipboard-image-1771905619.png)

ШІ-співробітники (`AI Employees`) — це можливості інтелектуальних агентів, глибоко інтегровані в бізнес-системи NocoBase.

Це не просто роботи «лише для чату», а «цифрові колеги», які можуть безпосередньо розуміти контекст і виконувати операції в бізнес-інтерфейсі:

- **Розуміють бізнес-контекст**: сприймають поточну сторінку, блоки, структуру даних та вибраний вміст.
- **Можуть безпосередньо виконувати дії**: можуть викликати навички для виконання таких завдань, як запит, аналіз, заповнення, налаштування, генерація тощо.
- **Рольова співпраця**: налаштовуйте різних співробітників відповідно до посад і перемикайте моделі в розмові для співпраці.

## 5-хвилинний шлях до початку роботи

Спочатку перегляньте [Швидкий старт](/ai-employees/quick-start), виконайте мінімальну конфігурацію в наступному порядку:

1. Налаштуйте принаймні один [LLM сервіс](/ai-employees/features/llm-service).
2. Увімкніть принаймні одного [ШІ-співробітника](/ai-employees/features/enable-ai-employee).
3. Відкрийте чат і почніть [співпрацювати з ШІ-співробітниками](/ai-employees/features/collaborate).
4. Увімкніть [Пошук в інтернеті](/ai-employees/features/web-search) та [Швидкі завдання](/ai-employees/features/task) за потреби.

## Карта функцій

### A. Базова конфігурація (Адміністратор)

- [Налаштування LLM сервісу](/ai-employees/features/llm-service): підключення Provider, налаштування та керування доступними моделями.
- [Увімкнення ШІ-співробітників](/ai-employees/features/enable-ai-employee): запуск/зупинка вбудованих співробітників, контроль сфери доступності.
- [Створення ШІ-співробітника](/ai-employees/features/new-ai-employees): визначення ролі, налаштування особистості, привітання та меж можливостей.
- [Використання навичок](/ai-employees/features/tool): налаштування дозволів навичок (`Ask` / `Allow`), контроль ризиків виконання.

### B. Щоденна співпраця (Бізнес-користувачі)

- [Співпраця з ШІ-співробітниками](/ai-employees/features/collaborate): перемикання співробітників та моделей у чаті для безперервної співпраці.
- [Додавання контексту - Блоки](/ai-employees/features/pick-block): надсилання блоків сторінки як контексту до ШІ.
- [Швидкі завдання](/ai-employees/features/task): попереднє налаштування типових завдань на сторінках/блоках, виконання в один клік.
- [Пошук в інтернеті](/ai-employees/features/web-search): увімкнення пошуку для отримання актуальної інформації, коли це необхідно.

### C. Розширені можливості (Розширення)

- [Вбудовані ШІ-співробітники](/ai-employees/features/built-in-employee): ознайомлення з позиціонуванням та сценаріями застосування попередньо встановлених співробітників.
- [Контроль прав доступу](/ai-employees/permission): контроль доступу до співробітників, навичок та даних відповідно до моделі прав організації.
- [База знань ШІ](/ai-employees/knowledge-base/index): впровадження корпоративних знань для підвищення стабільності та відстежуваності відповідей.
- [Вузол LLM у робочому процесі](/ai-employees/workflow/nodes/llm/chat): інтеграція можливостей ШІ в автоматизовані робочі процеси.

## Основні концепції (рекомендовано узгодити спочатку)

Наступні терміни відповідають глосарію, їх рекомендовано використовувати уніфіковано в команді:

- **ШІ-співробітник (AI Employee)**: виконуваний агент, що складається з налаштування ролі (Role setting) та навичок (Tool / Skill).
- **LLM сервіс (LLM Service)**: одиниця доступу до моделей та конфігурації можливостей, що використовується для керування Provider та списком моделей.
- **Постачальник (Provider)**: постачальник моделей, що стоїть за LLM сервісом.
- **Увімкнені моделі (Enabled Models)**: набір моделей, які поточний LLM сервіс дозволяє вибирати в чаті.
- **Перемикач ШІ-співробітників (AI Employee Switcher)**: перемикання поточного співробітника для співпраці в чаті.
- **Перемикач моделей (Model Switcher)**: перемикання моделей у чаті з запам'ятовуванням уподобань для кожного співробітника.
- **Навичка (Tool / Skill)**: одиниця виконавчої здатності, яку може викликати ШІ.
- **Дозвіл навички (Permission: Ask / Allow)**: чи потрібне підтвердження людиною перед викликом навички.
- **Контекст (Context)**: інформація про бізнес-середовище, така як сторінки, блоки, структури даних тощо.
- **Чат (Chat)**: процес безперервної взаємодії між користувачем та ШІ-співробітником.
- **Пошук в інтернеті (Web Search)**: можливість доповнювати відповіді актуальною інформацією на основі зовнішнього пошуку.
- **База знань (Knowledge Base / RAG)**: впровадження корпоративних знань за допомогою генерації з доповненням пошуку.
- **Векторне сховище (Vector Store)**: векторизоване сховище, що забезпечує можливості семантичного пошуку для бази знань.

## Інструкції з встановлення

ШІ-співробітники — це вбудований плагін NocoBase (`@nocobase/plugin-ai`), готовий до використання відразу, без необхідності окремого встановлення.
