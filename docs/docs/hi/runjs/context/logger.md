# ctx.logger

[pino](https://github.com/pinojs/pino) पर आधारित एक लॉगिंग रैपर (wrapper), जो उच्च-प्रदर्शन वाले स्ट्रक्चर्ड JSON लॉग प्रदान करता है। लॉग संग्रह और विश्लेषण को आसान बनाने के लिए `console` के बजाय `ctx.logger` का उपयोग करने की अनुशंसा की जाती है।

## उपयुक्त परिदृश्य

सभी RunJS परिदृश्यों में डिबगिंग, एरर ट्रैकिंग, प्रदर्शन विश्लेषण आदि के लिए `ctx.logger` का उपयोग किया जा सकता है।

## टाइप परिभाषा

```ts
logger: pino.Logger;
```

`ctx.logger`, `engine.logger.child({ module: 'flow-engine' })` का एक उदाहरण है, जो `module` संदर्भ (context) के साथ एक pino चाइल्ड लॉगर है।

## लॉग स्तर (Log Levels)

pino निम्नलिखित स्तरों का समर्थन करता है (उच्चतम से निम्नतम तक):

| स्तर | विधि | विवरण |
|------|------|------|
| `fatal` | `ctx.logger.fatal()` | घातक त्रुटि, जो आमतौर पर प्रोसेस बंद होने का कारण बनती है |
| `error` | `ctx.logger.error()` | त्रुटि, जो विफल अनुरोध या ऑपरेशन को दर्शाती है |
| `warn` | `ctx.logger.warn()` | चेतावनी, जो संभावित जोखिमों या असामान्य स्थितियों को दर्शाती है |
| `info` | `ctx.logger.info()` | सामान्य रनटाइम जानकारी |
| `debug` | `ctx.logger.debug()` | डिबगिंग जानकारी, जिसका उपयोग विकास के दौरान किया जाता है |
| `trace` | `ctx.logger.trace()` | विस्तृत ट्रेस, जिसका उपयोग गहन निदान के लिए किया जाता है |

## अनुशंसित लेखन शैली

`level(msg, meta)` प्रारूप का उपयोग करने की अनुशंसा की जाती है: संदेश पहले, और वैकल्पिक मेटाडेटा ऑब्जेक्ट बाद में।

```ts
ctx.logger.info('ब्लॉक लोडिंग पूर्ण');
ctx.logger.info('ऑपरेशन सफल', { recordId: 456 });
ctx.logger.warn('प्रदर्शन चेतावनी', { duration: 5000 });
ctx.logger.error('ऑपरेशन विफल', { userId: 123, action: 'create' });
ctx.logger.error('अनुरोध विफल', { err });
```

pino `level(meta, msg)` (ऑब्जेक्ट पहले) या `level({ msg, ...meta })` (एकल ऑब्जेक्ट) का भी समर्थन करता है, जिसे आवश्यकतानुसार उपयोग किया जा सकता है।

## उदाहरण

### बुनियादी उपयोग

```ts
ctx.logger.info('ब्लॉक लोडिंग पूर्ण');
ctx.logger.warn('अनुरोध विफल, कैश का उपयोग कर रहे हैं', { err });
ctx.logger.debug('सहेजा जा रहा है...', { recordId: ctx.record?.id });
```

### child() के साथ चाइल्ड लॉगर बनाना

```ts
// वर्तमान लॉजिक के लिए संदर्भ के साथ एक चाइल्ड लॉगर बनाएँ
const log = ctx.logger.child({ scope: 'myBlock' });
log.info('चरण 1 निष्पादित किया जा रहा है');
log.debug('चरण 2 निष्पादित किया जा रहा है', { step: 2 });
```

### console के साथ संबंध

स्ट्रक्चर्ड JSON लॉग प्राप्त करने के लिए सीधे `ctx.logger` का उपयोग करने की अनुशंसा की जाती है। यदि आप `console` का उपयोग करने के अभ्यस्त हैं, तो आप इस प्रकार मिलान कर सकते हैं: `console.log` → `ctx.logger.info`, `console.error` → `ctx.logger.error`, `console.warn` → `ctx.logger.warn`।

## लॉग प्रारूप

pino स्ट्रक्चर्ड JSON आउटपुट देता है, जहाँ प्रत्येक लॉग प्रविष्टि में शामिल होता है:

- `level`: लॉग स्तर (संख्यात्मक)
- `time`: टाइमस्टैम्प (मिलीसेकंड)
- `msg`: लॉग संदेश
- `module`: `flow-engine` के रूप में स्थिर
- अन्य कस्टम फ़ील्ड (ऑब्जेक्ट के माध्यम से पास किए गए)

## ध्यान देने योग्य बातें

- लॉग स्ट्रक्चर्ड JSON हैं, जिससे उन्हें एकत्र करना, खोजना और विश्लेषण करना आसान हो जाता है।
- `child()` के माध्यम से बनाए गए चाइल्ड लॉगर के लिए भी `level(msg, meta)` शैली की अनुशंसा की जाती है।
- कुछ रनटाइम वातावरण (जैसे वर्कफ़्लो) अलग लॉग आउटपुट विधियों का उपयोग कर सकते हैं।
