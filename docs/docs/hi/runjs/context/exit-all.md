# ctx.exitAll()

वर्तमान इवेंट फ्लो (event flow) और उसी इवेंट डिस्पैच (event dispatch) में ट्रिगर होने वाले सभी बाद के इवेंट फ्लो को समाप्त करता है। इसका उपयोग आमतौर पर तब किया जाता है जब किसी वैश्विक त्रुटि या अनुमति सत्यापन (permission validation) की विफलता के कारण वर्तमान इवेंट के तहत सभी इवेंट फ्लो को तुरंत रोकना आवश्यक हो।

## उपयोग के मामले

`ctx.exitAll()` का उपयोग आम तौर पर उन JS-निष्पादन योग्य संदर्भों (contexts) में किया जाता है, जहाँ **वर्तमान इवेंट फ्लो और उस इवेंट द्वारा ट्रिगर किए गए बाद के इवेंट फ्लो को एक साथ समाप्त करना** आवश्यक हो:

| परिदृश्य | विवरण |
|------|------|
| **इवेंट फ्लो** | मुख्य इवेंट फ्लो सत्यापन विफल हो जाता है (जैसे अपर्याप्त अनुमति), जिससे मुख्य फ्लो और उसी इवेंट के तहत किसी भी बाद के फ्लो को समाप्त करने की आवश्यकता होती है जो अभी तक निष्पादित नहीं हुए हैं। |
| **लिंकेज नियम (Linkage Rules)** | जब लिंकेज सत्यापन विफल हो जाता है, तो वर्तमान लिंकेज और उसी इवेंट द्वारा ट्रिगर किए गए बाद के लिंकेज को समाप्त किया जाना चाहिए। |
| **एक्शन इवेंट** | एक्शन-पूर्व सत्यापन विफल हो जाता है (जैसे हटाने से पहले अनुमति की जाँच), जिससे मुख्य एक्शन और उसके बाद के चरणों को रोकने की आवश्यकता होती है। |

> `ctx.exit()` से अंतर: `ctx.exit()` केवल वर्तमान इवेंट फ्लो को समाप्त करता है; `ctx.exitAll()` वर्तमान इवेंट फ्लो और उसी इवेंट डिस्पैच में **अभी तक निष्पादित नहीं हुए** बाद के सभी इवेंट फ्लो को समाप्त कर देता है।

## टाइप परिभाषा (Type Definition)

```ts
exitAll(): never;
```

`ctx.exitAll()` को कॉल करने पर एक आंतरिक `FlowExitAllException` थ्रो (throw) होता है, जिसे फ्लो इंजन (FlowEngine) द्वारा वर्तमान इवेंट फ्लो इंस्टेंस और उसी इवेंट के तहत बाद के इवेंट फ्लो को रोकने के लिए कैप्चर किया जाता है। एक बार कॉल करने के बाद, वर्तमान JS कोड के शेष स्टेटमेंट निष्पादित नहीं होंगे।

## ctx.exit() के साथ तुलना

| विधि | कार्यक्षेत्र (Scope) |
|------|----------|
| `ctx.exit()` | केवल वर्तमान इवेंट फ्लो को समाप्त करता है; बाद के इवेंट फ्लो प्रभावित नहीं होते हैं। |
| `ctx.exitAll()` | वर्तमान इवेंट फ्लो को समाप्त करता है और उसी इवेंट के तहत **क्रमिक रूप से (sequentially)** निष्पादित होने वाले बाद के इवेंट फ्लो को भी रोक देता है। |

## निष्पादन मोड विवरण (Execution Mode)

- **क्रमिक निष्पादन (Sequential Execution)**: एक ही इवेंट के तहत इवेंट फ्लो क्रम में निष्पादित होते हैं। किसी भी इवेंट फ्लो द्वारा `ctx.exitAll()` कॉल करने के बाद, बाद के इवेंट फ्लो निष्पादित नहीं होंगे।
- **समानांतर निष्पादन (Parallel Execution)**: एक ही इवेंट के तहत इवेंट फ्लो समानांतर में निष्पादित होते हैं। एक इवेंट फ्लो में `ctx.exitAll()` कॉल करने से अन्य समवर्ती (concurrent) इवेंट फ्लो बाधित नहीं होंगे (क्योंकि वे स्वतंत्र होते हैं)।

## उदाहरण

### अनुमति सत्यापन विफल होने पर सभी इवेंट फ्लो को समाप्त करना

```ts
// अनुमति अपर्याप्त होने पर मुख्य इवेंट फ्लो और बाद के इवेंट फ्लो को समाप्त करें
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'ऑपरेशन की अनुमति नहीं है' });
  ctx.exitAll();
}
```

### वैश्विक पूर्व-सत्यापन विफल होने पर समाप्त करना

```ts
// उदाहरण: यदि हटाने से पहले संबंधित डेटा मिलता है जिसे हटाया नहीं जा सकता, तो मुख्य इवेंट फ्लो और बाद के कार्यों को रोकें
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('हटाया नहीं जा सकता: संबंधित डेटा मौजूद है');
  ctx.exitAll();
}
```

### ctx.exit() और ctx.exitAll() के बीच चयन करना

```ts
// केवल वर्तमान इवेंट फ्लो को बाहर निकलने की आवश्यकता है -> ctx.exit() का उपयोग करें
if (!params.valid) {
  ctx.message.error('अमान्य पैरामीटर');
  ctx.exit();  // बाद के इवेंट फ्लो प्रभावित नहीं होंगे
}

// वर्तमान इवेंट के तहत सभी बाद के इवेंट फ्लो को समाप्त करने की आवश्यकता है -> ctx.exitAll() का उपयोग करें
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'अपर्याप्त अनुमति' });
  ctx.exitAll();  // मुख्य इवेंट फ्लो और उसी इवेंट के तहत बाद के सभी इवेंट फ्लो समाप्त हो जाएंगे
}
```

### समाप्त करने से पहले संकेत देना

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('कृपया पहले फॉर्म की त्रुटियों को सुधारें');
  ctx.exitAll();
}
```

## ध्यान देने योग्य बातें

- `ctx.exitAll()` कॉल करने के बाद, वर्तमान JS में बाद का कोड निष्पादित नहीं होगा। यह अनुशंसा की जाती है कि इसे कॉल करने से पहले `ctx.message`, `ctx.notification`, या किसी मोडल (modal) के माध्यम से उपयोगकर्ता को कारण स्पष्ट करें।
- व्यावसायिक कोड (business code) को आमतौर पर `FlowExitAllException` को कैप्चर करने की आवश्यकता नहीं होती है; इसे फ्लो इंजन को संभालने दें।
- यदि आपको बाद के फ्लो को प्रभावित किए बिना केवल वर्तमान इवेंट फ्लो को रोकना है, तो `ctx.exit()` का उपयोग करें।
- समानांतर मोड (parallel mode) में, `ctx.exitAll()` केवल वर्तमान इवेंट फ्लो को समाप्त करता है और अन्य समवर्ती इवेंट फ्लो को बाधित नहीं करता है।
