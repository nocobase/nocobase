# ctx.router

React Router पर आधारित एक राउटर इंस्टेंस, जिसका उपयोग RunJS में कोड के माध्यम से नेविगेशन के लिए किया जाता है। आमतौर पर इसका उपयोग `ctx.route` और `ctx.location` के साथ किया जाता है।

## उपयोग के मामले

| परिदृश्य | विवरण |
|------|------|
| **JSBlock / JSField** | बटन क्लिक करने के बाद विवरण पृष्ठों (detail pages), सूची पृष्ठों (list pages) या बाहरी लिंक पर नेविगेट करें। |
| **लिंकेज नियम / इवेंट फ़्लो** | सफल सबमिशन के बाद सूची या विवरण पृष्ठ पर `navigate` निष्पादित करें, या लक्ष्य पृष्ठ पर `state` पास करें। |
| **JSAction / इवेंट हैंडलिंग** | फ़ॉर्म सबमिशन या लिंक क्लिक जैसे लॉजिक के भीतर रूट नेविगेशन निष्पादित करें। |
| **व्यू नेविगेशन** | आंतरिक व्यू स्टैक स्विचिंग के दौरान `navigate` के माध्यम से URL अपडेट करें। |

> ध्यान दें: `ctx.router` केवल उन RunJS वातावरणों में उपलब्ध है जहाँ रूटिंग संदर्भ (routing context) मौजूद हो (जैसे पेज के भीतर JSBlock, फ़्लो पेज, इवेंट फ़्लो आदि); शुद्ध बैकएंड या बिना रूटिंग वाले संदर्भों (जैसे वर्कफ़्लो) में यह null हो सकता है।

## टाइप परिभाषा

```typescript
router: Router
```

`Router`, `@remix-run/router` से लिया गया है। RunJS में, जंपिंग, पीछे जाने और रिफ्रेश करने जैसे नेविगेशन ऑपरेशन `ctx.router.navigate()` के माध्यम से लागू किए जाते हैं।

## मेथड

### ctx.router.navigate()

लक्ष्य पथ (target path) पर नेविगेट करें, या बैक/रिफ्रेश एक्शन निष्पादित करें।

**सिग्नेचर:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**पैरामीटर:**

- `to`: लक्ष्य पथ (string), सापेक्ष इतिहास स्थिति (number, जैसे पीछे जाने के लिए `-1`), या `null` (वर्तमान पेज को रिफ्रेश करने के लिए)।
- `options`: वैकल्पिक कॉन्फ़िगरेशन।
  - `replace?: boolean`: क्या वर्तमान इतिहास प्रविष्टि (history entry) को बदलना है (डिफ़ॉल्ट `false` है, जो एक नई प्रविष्टि जोड़ता है)।
  - `state?: any`: लक्ष्य रूट को पास किया जाने वाला स्टेट। यह डेटा URL में दिखाई नहीं देता है और इसे लक्ष्य पेज पर `ctx.location.state` के माध्यम से एक्सेस किया जा सकता है। यह संवेदनशील जानकारी, अस्थायी डेटा या ऐसी जानकारी के लिए उपयुक्त है जिसे URL में नहीं रखा जाना चाहिए।

## उदाहरण

### बुनियादी नेविगेशन

```ts
// उपयोगकर्ता सूची पर नेविगेट करें (नई हिस्ट्री प्रविष्टि जोड़ता है, पीछे जाने की अनुमति देता है)
ctx.router.navigate('/admin/users');

// विवरण पेज (detail page) पर नेविगेट करें
ctx.router.navigate(`/admin/users/${recordId}`);
```

### इतिहास बदलना (कोई नई प्रविष्टि नहीं)

```ts
// लॉगिन के बाद होम पेज पर रीडायरेक्ट करें; पीछे जाने पर उपयोगकर्ता लॉगिन पेज पर वापस नहीं आएगा
ctx.router.navigate('/admin', { replace: true });

// फ़ॉर्म सफलतापूर्वक सबमिट होने के बाद वर्तमान पेज को विवरण पेज से बदलें
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### State पास करना

```ts
// नेविगेशन के दौरान डेटा ले जाएं; लक्ष्य पेज इसे ctx.location.state के माध्यम से प्राप्त करता है
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### बैक और रिफ्रेश

```ts
// एक पेज पीछे जाएं
ctx.router.navigate(-1);

// दो पेज पीछे जाएं
ctx.router.navigate(-2);

// वर्तमान पेज को रिफ्रेश करें
ctx.router.navigate(null);
```

## ctx.route और ctx.location के साथ संबंध

| उपयोग | अनुशंसित उपयोग |
|------|----------|
| **नेविगेशन/जंपिंग** | `ctx.router.navigate(path)` |
| **वर्तमान पथ पढ़ें** | `ctx.route.pathname` या `ctx.location.pathname` |
| **नेविगेशन के दौरान पास किया गया state पढ़ें** | `ctx.location.state` |
| **रूट पैरामीटर पढ़ें** | `ctx.route.params` |

`ctx.router` "नेविगेशन क्रियाओं" के लिए ज़िम्मेदार है, जबकि `ctx.route` और `ctx.location` "वर्तमान रूट स्थिति" के लिए ज़िम्मेदार हैं।

## ध्यान दें

- `navigate(path)` डिफ़ॉल्ट रूप से एक नई इतिहास प्रविष्टि जोड़ता है, जिससे उपयोगकर्ता ब्राउज़र के बैक बटन के माध्यम से वापस लौट सकते हैं।
- `replace: true` नई प्रविष्टि जोड़े बिना वर्तमान इतिहास प्रविष्टि को बदल देता है, जो लॉगिन के बाद रीडायरेक्शन या सफल सबमिशन के बाद नेविगेशन जैसे परिदृश्यों के लिए उपयुक्त है।
- **`state` पैरामीटर के बारे में**:
  - `state` के माध्यम से पास किया गया डेटा URL में दिखाई नहीं देता है, जो इसे संवेदनशील या अस्थायी डेटा के लिए उपयुक्त बनाता है।
  - इसे लक्ष्य पेज पर `ctx.location.state` के माध्यम से एक्सेस किया जा सकता है।
  - `state` ब्राउज़र इतिहास में सहेजा जाता है और आगे/पीछे नेविगेशन के दौरान सुलभ रहता है।
  - पेज को हार्ड रिफ्रेश करने के बाद `state` खो जाएगा।

## संबंधित

- [ctx.route](./route.md): वर्तमान रूट मिलान जानकारी (pathname, params आदि)।
- [ctx.location](./location.md): वर्तमान URL स्थान (pathname, search, hash, state); नेविगेशन के बाद `state` यहाँ पढ़ा जाता है।
