# ctx.on()

RunJS में कॉन्टेक्स्ट इवेंट्स (जैसे फ़ील्ड वैल्यू में बदलाव, प्रॉपर्टी में बदलाव, रिसोर्स रिफ्रेश आदि) को सब्सक्राइब करें। इवेंट्स उनके प्रकार के आधार पर `ctx.element` पर कस्टम DOM इवेंट्स या `ctx.resource` के इंटरनल इवेंट बस (event bus) इवेंट्स पर मैप किए जाते हैं।

##适用场景 (उपयोग के मामले)

| परिदृश्य | विवरण |
|------|------|
| **JSField / JSEditableField** | बाहरी स्रोतों (फॉर्म, लिंकेज आदि) से फ़ील्ड वैल्यू में होने वाले बदलावों को सुनें ताकि UI को सिंक्रोनस रूप से अपडेट किया जा सके, जिससे टू-वे बाइंडिंग (two-way binding) प्राप्त हो सके। |
| **JSBlock / JSItem / JSColumn** | डेटा या स्थिति (state) में बदलावों पर प्रतिक्रिया देने के लिए कंटेनर पर कस्टम इवेंट्स को सुनें। |
| **resource संबंधित** | डेटा अपडेट के बाद लॉजिक निष्पादित करने के लिए रिसोर्स लाइफसाइकिल इवेंट्स जैसे रिफ्रेश या सेव को सुनें। |

## 类型定义 (टाइप परिभाषा)

```ts
on(eventName: string, handler: (event?: any) => void): void;
```

## 常见事件 (सामान्य इवेंट्स)

| इवेंट का नाम | विवरण | इवेंट का स्रोत |
|--------|------|----------|
| `js-field:value-change` | फ़ील्ड वैल्यू को बाहरी रूप से संशोधित किया गया (जैसे फॉर्म लिंकेज, डिफ़ॉल्ट वैल्यू अपडेट) | `ctx.element` पर CustomEvent, जहाँ `ev.detail` नई वैल्यू है |
| `resource:refresh` | रिसोर्स डेटा रिफ्रेश हो गया है | `ctx.resource` इवेंट बस |
| `resource:saved` | रिसोर्स सेव करने की प्रक्रिया पूरी हो गई है | `ctx.resource` इवेंट बस |

> इवेंट मैपिंग नियम: `resource:` प्रीफिक्स वाले इवेंट `ctx.resource.on` के माध्यम से जाते हैं, जबकि अन्य आमतौर पर `ctx.element` पर DOM इवेंट्स (यदि मौजूद हों) के माध्यम से जाते हैं।

## 示例 (उदाहरण)

### फ़ील्ड टू-वे बाइंडिंग (React useEffect + क्लीनअप)

```tsx
React.useEffect(() => {
  const handler = (ev) => setValue(ev?.detail ?? '');
  ctx.on?.('js-field:value-change', handler);
  return () => {
    ctx.off?.('js-field:value-change', handler);
  };
}, []);
```

### नेटिव DOM लिसनिंग (जब ctx.on उपलब्ध न हो तो विकल्प)

```ts
// जब ctx.on प्रदान नहीं किया गया हो, तो आप सीधे ctx.element का उपयोग कर सकते हैं
const handler = (ev) => {
  if (selectEl) selectEl.value = String(ev?.detail ?? '');
};
ctx.element?.addEventListener('js-field:value-change', handler);
// क्लीनअप के दौरान: ctx.element?.removeEventListener('js-field:value-change', handler);
```

### रिसोर्स रिफ्रेश के बाद UI अपडेट करना

```ts
ctx.resource?.on('refresh', () => {
  const data = ctx.resource?.getData?.();
  // डेटा के आधार पर रेंडरिंग अपडेट करें
});
```

## 与 ctx.off 的配合 (ctx.off के साथ तालमेल)

- `ctx.on` का उपयोग करके रजिस्टर किए गए लिसनर्स को मेमोरी लीक या डुप्लिकेट ट्रिगर से बचने के लिए [ctx.off](./off.md) के माध्यम से उचित समय पर हटा दिया जाना चाहिए।
- React में, `ctx.off` को आमतौर पर `useEffect` के क्लीनअप फ़ंक्शन के भीतर कॉल किया जाता है।
- हो सकता है कि `ctx.off` मौजूद न हो; इसका उपयोग करते समय ऑप्शनल चेनिंग (optional chaining) का सुझाव दिया जाता है: `ctx.off?.('eventName', handler)`।

## 注意事项 (महत्वपूर्ण बातें)

1. **जोड़ी में रद्दीकरण (Paired Cancellation)**: प्रत्येक `ctx.on(eventName, handler)` के लिए एक संबंधित `ctx.off(eventName, handler)` होना चाहिए, और पास किया गया `handler` रेफरेंस बिल्कुल वही होना चाहिए।
2. **लाइफसाइकिल (Lifecycle)**: मेमोरी लीक को रोकने के लिए कंपोनेंट के अनमाउंट होने या कॉन्टेक्स्ट के नष्ट होने से पहले लिसनर्स को हटा दें।
3. **इवेंट की उपलब्धता (Event Availability)**: अलग-अलग कॉन्टेक्स्ट प्रकार अलग-अलग इवेंट्स का समर्थन करते हैं। विवरण के लिए कृपया विशिष्ट कंपोनेंट दस्तावेज़ देखें।
