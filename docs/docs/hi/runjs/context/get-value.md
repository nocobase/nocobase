# ctx.getValue()

JSField, JSItem जैसे संपादन योग्य (editable) फ़ील्ड परिदृश्यों में, वर्तमान फ़ील्ड का नवीनतम मान (value) प्राप्त करने के लिए इसका उपयोग करें। `ctx.setValue(v)` के साथ मिलकर, यह फ़ॉर्म के साथ टू-वे बाइंडिंग (two-way binding) को सक्षम बनाता है।

##适用场景 (उपयुक्त परिदृश्य)

| परिदृश्य | विवरण |
|------|------|
| **JSField** | संपादन योग्य कस्टम फ़ील्ड में उपयोगकर्ता इनपुट या फ़ॉर्म का वर्तमान मान पढ़ें। |
| **JSItem** | टेबल/सब-टेबल के संपादन योग्य आइटम में वर्तमान सेल मान पढ़ें। |
| **JSColumn** | टेबल कॉलम रेंडरिंग के दौरान संबंधित पंक्ति (row) का फ़ील्ड मान पढ़ें। |

> **ध्यान दें**: `ctx.getValue()` केवल फ़ॉर्म बाइंडिंग वाले RunJS संदर्भों (contexts) में उपलब्ध है; यह वर्कफ़्लो या लिंकेज नियमों जैसे फ़ील्ड बाइंडिंग के बिना वाले परिदृश्यों में मौजूद नहीं होता है।

## 类型定义 (प्रकार परिभाषा)

```ts
getValue<T = any>(): T | undefined;
```

- **रिटर्न वैल्यू**: वर्तमान फ़ील्ड मान, जिसका प्रकार फ़ील्ड के फ़ॉर्म आइटम प्रकार द्वारा निर्धारित किया जाता है; यदि फ़ील्ड पंजीकृत (registered) नहीं है या भरा नहीं गया है, तो यह `undefined` हो सकता है।

## 取值顺序 (प्राप्ति क्रम)

`ctx.getValue()` निम्नलिखित क्रम में मान प्राप्त करता है:

1. **फ़ॉर्म स्थिति (Form State)**: Ant Design Form की वर्तमान स्थिति से पढ़ने को प्राथमिकता देता है।
2. **फ़ॉलबैक मान (Fallback Value)**: यदि फ़ील्ड फ़ॉर्म में नहीं है, तो यह फ़ील्ड के प्रारंभिक मान या प्रॉप्स (props) पर वापस आ जाता है।

> यदि फ़ॉर्म ने रेंडरिंग पूरी नहीं की है या फ़ील्ड पंजीकृत नहीं है, तो यह `undefined` लौटा सकता है।

## 示例 (उदाहरण)

### वर्तमान मान के आधार पर रेंडर करना

```ts
const current = ctx.getValue();
if (current == null || current === '') {
  ctx.render(<span>कृपया पहले सामग्री दर्ज करें</span>);
} else {
  ctx.render(<span>वर्तमान मान: {current}</span>);
}
```

### setValue के साथ टू-वे बाइंडिंग

```tsx
const { Input } = ctx.libs.antd;

// वर्तमान मान को डिफ़ॉल्ट मान के रूप में पढ़ें
const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```
