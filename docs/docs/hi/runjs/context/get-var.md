# ctx.getVar()

वर्तमान रनटाइम संदर्भ (runtime context) से **एसिंक्रोनस रूप से (asynchronously)** वेरिएबल मान पढ़ता है। वेरिएबल रिज़ॉल्यूशन SQL और टेम्प्लेट में `{{ctx.xxx}}` के समान है, जो आमतौर पर वर्तमान उपयोगकर्ता, वर्तमान रिकॉर्ड, व्यू पैरामीटर, पॉपअप संदर्भ आदि से आता है।

## उपयोग के मामले

| परिदृश्य | विवरण |
|------|------|
| **JSBlock / JSField** | रेंडरिंग या लॉजिक के लिए वर्तमान रिकॉर्ड, उपयोगकर्ता, संसाधन आदि के बारे में जानकारी प्राप्त करने के लिए। |
| **लिंकेज नियम / इवेंट फ्लो** | शर्त आधारित लॉजिक (conditional logic) के लिए `ctx.record`, `ctx.formValues` आदि को पढ़ने के लिए। |
| **फॉर्मूला / टेम्प्लेट** | `{{ctx.xxx}}` के समान वेरिएबल रिज़ॉल्यूशन नियमों का उपयोग करता है। |

## टाइप परिभाषा (Type Definition)

```ts
getVar(path: string): Promise<any>;
```

| पैरामीटर | प्रकार | विवरण |
|------|------|------|
| `path` | `string` | वेरिएबल पाथ; **`ctx.` से शुरू होना चाहिए**। यह डॉट नोटेशन और एरे इंडेक्स का समर्थन करता है। |

**रिटर्न वैल्यू**: `Promise<any>`। रिज़ॉल्व किए गए मान को प्राप्त करने के लिए `await` का उपयोग करें; यदि वेरिएबल मौजूद नहीं है तो यह `undefined` लौटाता है।

> यदि ऐसा पाथ पास किया जाता है जो `ctx.` से शुरू नहीं होता है, तो एक एरर थ्रो किया जाएगा: `ctx.getVar(path) expects an expression starting with "ctx.", got: "..."`।

## सामान्य वेरिएबल पाथ

| पाथ | विवरण |
|------|------|
| `ctx.record` | वर्तमान रिकॉर्ड (तब उपलब्ध होता है जब कोई फ़ॉर्म/विवरण ब्लॉक किसी रिकॉर्ड से जुड़ा हो) |
| `ctx.record.id` | वर्तमान रिकॉर्ड की प्राइमरी की (Primary Key) |
| `ctx.formValues` | वर्तमान फ़ॉर्म के मान (आमतौर पर लिंकेज नियमों और इवेंट फ्लो में उपयोग किए जाते हैं; फ़ॉर्म परिदृश्यों में, रीयल-टाइम रीडिंग के लिए `ctx.form.getFieldsValue()` को प्राथमिकता दें) |
| `ctx.user` | वर्तमान में लॉग-इन उपयोगकर्ता |
| `ctx.user.id` | वर्तमान उपयोगकर्ता ID |
| `ctx.user.nickname` | वर्तमान उपयोगकर्ता का निकनेम |
| `ctx.user.roles.name` | वर्तमान उपयोगकर्ता की भूमिकाओं के नाम (एरे) |
| `ctx.popup.record` | पॉपअप के भीतर का रिकॉर्ड |
| `ctx.popup.record.id` | पॉपअप के भीतर के रिकॉर्ड की प्राइमरी की |
| `ctx.urlSearchParams` | URL क्वेरी पैरामीटर (`?key=value` से पार्स किया गया) |
| `ctx.token` | वर्तमान API टोकन |
| `ctx.role` | वर्तमान भूमिका |

## ctx.getVarInfos()

वर्तमान संदर्भ में रिज़ॉल्व होने योग्य वेरिएबल्स की **संरचनात्मक जानकारी** (प्रकार, शीर्षक, उप-गुण आदि) प्राप्त करता है, जिससे उपलब्ध पाथ को खोजना आसान हो जाता है। रिटर्न वैल्यू `meta` पर आधारित एक स्थिर विवरण है और इसमें वास्तविक रनटाइम मान शामिल नहीं होते हैं।

### टाइप परिभाषा (Type Definition)

```ts
getVarInfos(options?: { path?: string | string[]; maxDepth?: number }): Promise<Record<string, any>>;
```

रिटर्न वैल्यू में प्रत्येक की (key) एक वेरिएबल पाथ है, और वैल्यू उस पाथ के लिए संरचनात्मक जानकारी है (जिसमें `type`, `title`, `properties` आदि शामिल हैं)।

### पैरामीटर

| पैरामीटर | प्रकार | विवरण |
|------|------|------|
| `path` | `string \| string[]` | ट्रंकेशन पाथ; केवल इस पाथ के तहत वेरिएबल संरचना एकत्र करता है। `'record'`, `'record.id'`, `'ctx.record'`, `'{{ ctx.record }}'` का समर्थन करता है; एक एरे कई पाथ के विलय (merging) का प्रतिनिधित्व करता है। |
| `maxDepth` | `number` | अधिकतम विस्तार गहराई (expansion depth), डिफ़ॉल्ट `3` है। जब `path` प्रदान नहीं किया जाता है, तो टॉप-लेवल गुणों की गहराई `depth=1` होती है। जब `path` प्रदान किया जाता है, तो पाथ के अनुरूप नोड की गहराई `depth=1` होती है। |

### उदाहरण

```ts
// record के तहत वेरिएबल संरचना प्राप्त करें (अधिकतम 3 स्तरों तक विस्तारित)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });

// popup.record की संरचना प्राप्त करें
const vars = await ctx.getVarInfos({ path: 'popup.record', maxDepth: 3 });

// पूर्ण टॉप-लेवल वेरिएबल संरचना प्राप्त करें (डिफ़ॉल्ट maxDepth=3)
const vars = await ctx.getVarInfos();
```

## ctx.getValue के साथ अंतर

| विधि | परिदृश्य | विवरण |
|------|----------|------|
| `ctx.getValue()` | JSField, JSItem जैसे संपादन योग्य फ़ील्ड | **वर्तमान फ़ील्ड** का मान सिंक्रोनस रूप से प्राप्त करता है; इसके लिए फ़ॉर्म बाइंडिंग की आवश्यकता होती है। |
| `ctx.getVar(path)` | कोई भी RunJS संदर्भ | **किसी भी ctx वेरिएबल** को एसिंक्रोनस रूप से प्राप्त करता है; पाथ `ctx.` से शुरू होना चाहिए। |

JSField में, वर्तमान फ़ील्ड को पढ़ने/लिखने के लिए `getValue`/`setValue` का उपयोग करें; अन्य संदर्भ वेरिएबल्स (जैसे record, user, formValues) तक पहुँचने के लिए `getVar` का उपयोग करें।

## ध्यान देने योग्य बातें

- **पाथ `ctx.` से शुरू होना चाहिए**: जैसे, `ctx.record.id`, अन्यथा एक एरर थ्रो किया जाएगा।
- **एसिंक्रोनस विधि**: परिणाम प्राप्त करने के लिए आपको `await` का उपयोग करना चाहिए, जैसे, `const id = await ctx.getVar('ctx.record.id')`।
- **वेरिएबल मौजूद नहीं है**: `undefined` लौटाता है। आप डिफ़ॉल्ट मान सेट करने के लिए परिणाम के बाद `??` का उपयोग कर सकते हैं: `(await ctx.getVar('ctx.user.nickname')) ?? 'अतिथि'`।
- **फ़ॉर्म मान**: `ctx.formValues` को `await ctx.getVar('ctx.formValues')` के माध्यम से प्राप्त किया जाना चाहिए; यह सीधे `ctx.formValues` के रूप में उपलब्ध नहीं है। फ़ॉर्म संदर्भ में, रीयल-टाइम में नवीनतम मान पढ़ने के लिए `ctx.form.getFieldsValue()` का उपयोग करना बेहतर है।

## उदाहरण

### वर्तमान रिकॉर्ड ID प्राप्त करना

```ts
const recordId = await ctx.getVar('ctx.record.id');
if (recordId) {
  ctx.message.info(`वर्तमान रिकॉर्ड: ${recordId}`);
}
```

### पॉपअप के भीतर रिकॉर्ड प्राप्त करना

```ts
const recordId = await ctx.getVar('ctx.popup.record.id');
if (recordId) {
  ctx.message.info(`वर्तमान पॉपअप रिकॉर्ड: ${recordId}`);
}
```

### एरे फ़ील्ड के सब-आइटम पढ़ना

```ts
const roleNames = await ctx.getVar('ctx.user.roles.name');
// भूमिका नामों का एक एरे लौटाता है, जैसे, ['admin', 'member']
```

### डिफ़ॉल्ट मान सेट करना

```ts
// getVar में defaultValue पैरामीटर नहीं है; परिणाम के बाद ?? का उपयोग करें
const userName = (await ctx.getVar('ctx.user.nickname')) ?? 'अतिथि';
```

### फ़ॉर्म फ़ील्ड मान पढ़ना

```ts
// ctx.formValues और ctx.form दोनों फ़ॉर्म परिदृश्यों के लिए हैं; नेस्टेड फ़ील्ड पढ़ने के लिए getVar का उपयोग करें
const status = await ctx.getVar('ctx.formValues.status');
if (status === 'draft') {
  // ...
}
```

### URL क्वेरी पैरामीटर पढ़ना

```ts
const id = await ctx.getVar('ctx.urlSearchParams.id'); // ?id=xxx के अनुरूप
```

### उपलब्ध वेरिएबल्स खोजना

```ts
// record के तहत वेरिएबल संरचना प्राप्त करें (अधिकतम 3 स्तरों तक विस्तारित)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });
// vars इस तरह दिखता है { 'record.id': { type: 'string', title: 'id' }, ... }
```
