# ctx.exit()

वर्तमान इवेंट फ्लो (event flow) के निष्पादन (execution) को समाप्त करता है; इसके बाद के चरण नहीं चलेंगे। इसका उपयोग आमतौर पर तब किया जाता है जब व्यावसायिक शर्तें पूरी नहीं होती हैं, उपयोगकर्ता रद्द कर देता है, या कोई ऐसी त्रुटि होती है जिसे ठीक नहीं किया जा सकता।

## उपयोग के मामले (Use Cases)

`ctx.exit()` का उपयोग आम तौर पर निम्नलिखित संदर्भों में किया जाता है जहाँ JS निष्पादित किया जा सकता है:

| परिदृश्य | विवरण |
|------|------|
| **इवेंट फ्लो** | फॉर्म सबमिशन, बटन क्लिक आदि द्वारा ट्रिगर किए गए इवेंट फ्लो में, शर्तें पूरी न होने पर बाद के चरणों को समाप्त करता है। |
| **लिंकेज नियम** | फ़ील्ड लिंकेज, फ़िल्टर लिंकेज आदि में, सत्यापन (validation) विफल होने पर या निष्पादन को छोड़ने की आवश्यकता होने पर वर्तमान इवेंट फ्लो को समाप्त करता है। |
| **एक्शन इवेंट** | कस्टम एक्शन (जैसे डिलीट पुष्टिकरण, सेव करने से पहले सत्यापन) में, उपयोगकर्ता द्वारा रद्द करने या सत्यापन विफल होने पर बाहर निकलता है। |

> `ctx.exitAll()` से अंतर: `ctx.exit()` केवल वर्तमान इवेंट फ्लो को समाप्त करता है; उसी इवेंट के तहत अन्य इवेंट फ्लो प्रभावित नहीं होते हैं। `ctx.exitAll()` वर्तमान इवेंट फ्लो के साथ-साथ उसी इवेंट के तहत उन सभी बाद के इवेंट फ्लो को भी समाप्त कर देता है जो अभी तक निष्पादित नहीं हुए हैं।

## टाइप परिभाषा (Type Definition)

```ts
exit(): never;
```

`ctx.exit()` को कॉल करने पर एक आंतरिक `FlowExitException` थ्रो (throw) होता है, जिसे इवेंट फ्लो इंजन द्वारा पकड़ा जाता है और वर्तमान इवेंट फ्लो के निष्पादन को रोक दिया जाता है। एक बार कॉल करने के बाद, वर्तमान JS कोड में शेष स्टेटमेंट निष्पादित नहीं होंगे।

## ctx.exitAll() के साथ तुलना

| विधि | प्रभाव का दायरा |
|------|----------|
| `ctx.exit()` | केवल वर्तमान इवेंट फ्लो को समाप्त करता है; बाद के इवेंट फ्लो अप्रभावित रहते हैं। |
| `ctx.exitAll()` | वर्तमान इवेंट फ्लो को समाप्त करता है और उसी इवेंट के तहत **क्रमानुसार निष्पादित (sequentially execute)** होने वाले बाद के इवेंट फ्लो को भी रोक देता है। |

## उदाहरण

### उपयोगकर्ता द्वारा रद्द करने पर बाहर निकलना

```ts
// पुष्टिकरण मोडल में, यदि उपयोगकर्ता रद्द करें पर क्लिक करता है तो इवेंट फ्लो को समाप्त करें
if (!confirmed) {
  ctx.message.info('ऑपरेशन रद्द कर दिया गया');
  ctx.exit();
}
```

### पैरामीटर सत्यापन विफल होने पर बाहर निकलना

```ts
// सत्यापन विफल होने पर संकेत दें और समाप्त करें
if (!params.value || params.value.length < 3) {
  ctx.message.error('अमान्य पैरामीटर, लंबाई कम से कम 3 होनी चाहिए');
  ctx.exit();
}
```

### व्यावसायिक शर्तें पूरी न होने पर बाहर निकलना

```ts
// शर्तें पूरी न होने पर समाप्त करें; बाद के चरण निष्पादित नहीं होंगे
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'केवल ड्राफ्ट स्थिति ही सबमिट की जा सकती है' });
  ctx.exit();
}
```

### ctx.exit() और ctx.exitAll() के बीच चयन करना

```ts
// केवल वर्तमान इवेंट फ्लो से बाहर निकलने की आवश्यकता है → ctx.exit() का उपयोग करें
if (!params.valid) {
  ctx.message.error('अमान्य पैरामीटर');
  ctx.exit();  // अन्य इवेंट फ्लो अप्रभावित रहते हैं
}

// वर्तमान इवेंट के तहत सभी बाद के इवेंट फ्लो को समाप्त करने की आवश्यकता है → ctx.exitAll() का उपयोग करें
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'अपर्याप्त अनुमतियाँ' });
  ctx.exitAll();  // मुख्य इवेंट फ्लो और उसी इवेंट के तहत बाद के इवेंट फ्लो दोनों समाप्त हो जाते हैं
}
```

### मोडल पुष्टिकरण के बाद उपयोगकर्ता की पसंद के आधार पर बाहर निकलना

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'हटाने की पुष्टि करें',
  content: 'हटाने के बाद इसे वापस नहीं लाया जा सकता। क्या आप जारी रखना चाहते हैं?',
});
if (!ok) {
  ctx.message.info('रद्द कर दिया गया');
  ctx.exit();
}
```

## महत्वपूर्ण बातें

- `ctx.exit()` को कॉल करने के बाद, वर्तमान JS में बाद का कोड निष्पादित नहीं होगा; यह अनुशंसा की जाती है कि इसे कॉल करने से पहले `ctx.message`, `ctx.notification` या मोडल के माध्यम से उपयोगकर्ता को कारण स्पष्ट करें।
- व्यावसायिक कोड (business code) में आमतौर पर `FlowExitException` को पकड़ने (catch) की आवश्यकता नहीं होती है; इसे इवेंट फ्लो इंजन को संभालने दें।
- यदि आपको वर्तमान इवेंट के तहत सभी बाद के इवेंट फ्लो को समाप्त करने की आवश्यकता है, तो `ctx.exitAll()` का उपयोग करें।
