# ctx.setValue()

JSField, JSItem जैसे संपादन योग्य (editable) फ़ील्ड परिदृश्यों में वर्तमान फ़ील्ड का मान सेट करता है। `ctx.getValue()` के साथ मिलकर, यह फ़ॉर्म के साथ टू-वे बाइंडिंग (two-way binding) सक्षम करता है।

##适用场景 (उपयोग के मामले)

| परिदृश्य | विवरण |
|------|------|
| **JSField** | उपयोगकर्ता द्वारा चुने गए या गणना किए गए मानों को संपादन योग्य कस्टम फ़ील्ड में लिखें। |
| **JSItem** | टेबल/सब-टेबल के संपादन योग्य आइटम में वर्तमान सेल मान को अपडेट करें। |
| **JSColumn** | टेबल कॉलम रेंडरिंग के दौरान लॉजिक के आधार पर संबंधित पंक्ति के फ़ील्ड मान को अपडेट करें। |

> **ध्यान दें**: `ctx.setValue(v)` केवल फ़ॉर्म बाइंडिंग वाले RunJS संदर्भों (contexts) में उपलब्ध है। यह वर्कफ़्लो (FlowEngine), लिंकेज नियमों या JSBlock जैसे फ़ील्ड बाइंडिंग के बिना वाले परिदृश्यों में उपलब्ध नहीं है। उपयोग करने से पहले ऑप्शनल चेनिंग (optional chaining) का उपयोग करने की सलाह दी जाती है: `ctx.setValue?.(value)`।

## 类型定义 (टाइप परिभाषा)

```ts
setValue<T = any>(value: T): void;
```

- **पैरामीटर्स**: `value` वह फ़ील्ड मान है जिसे लिखा जाना है। इसका प्रकार फ़ील्ड के फ़ॉर्म आइटम प्रकार द्वारा निर्धारित किया जाता है।

## 行为说明 (व्यवहार)

- `ctx.setValue(v)` Ant Design Form में वर्तमान फ़ील्ड के मान को अपडेट करता है और संबंधित फ़ॉर्म लिंकेज और सत्यापन (validation) लॉजिक को ट्रिगर करता है।
- यदि फ़ॉर्म रेंडरिंग पूरी नहीं हुई है या फ़ील्ड पंजीकृत (registered) नहीं है, तो कॉल प्रभावी नहीं हो सकती है। लिखने के परिणाम की पुष्टि करने के लिए `ctx.getValue()` का उपयोग करने की सलाह दी जाती है।

## 示例 (उदाहरण)

### getValue के साथ टू-वे बाइंडिंग

```tsx
const { Input } = ctx.libs.antd;

const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```

### शर्तों के आधार पर डिफ़ॉल्ट मान सेट करना

```ts
const status = ctx.getValue();
if (status == null || status === '') {
  ctx.setValue?.('draft');
}
```

### अन्य फ़ील्ड से लिंक होने पर वर्तमान फ़ील्ड में वापस लिखना

```ts
// जब कोई अन्य फ़ील्ड बदलता है, तो वर्तमान फ़ील्ड को सिंक्रोनस रूप से अपडेट करें
const otherValue = ctx.form?.getFieldValue('type');
if (otherValue === 'custom') {
  ctx.setValue?.({ label: 'कस्टम', value: 'custom' });
}
```

## 注意事项 (महत्वपूर्ण बातें)

- गैर-संपादन योग्य फ़ील्ड (जैसे JSField विवरण मोड, JSBlock) में, `ctx.setValue` `undefined` हो सकता है। त्रुटियों से बचने के लिए `ctx.setValue?.(value)` का उपयोग करने की सलाह दी जाती है।
- एसोसिएशन फ़ील्ड (m2o, o2m आदि) के लिए मान सेट करते समय, आपको फ़ील्ड प्रकार से मेल खाने वाली संरचना (जैसे `{ id, [titleField]: label }`) पास करनी होगी, जो विशिष्ट फ़ील्ड कॉन्फ़िगरेशन पर निर्भर करती है।
