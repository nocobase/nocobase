# ctx.route

वर्तमान रूट (route) मिलान जानकारी, जो React Router के `route` कॉन्सेप्ट के अनुरूप है। इसका उपयोग वर्तमान मिलान वाले रूट कॉन्फ़िगरेशन, पैरामीटर आदि प्राप्त करने के लिए किया जाता है। आमतौर पर इसका उपयोग `ctx.router` और `ctx.location` के साथ किया जाता है।

##适用场景 (उपयोग के मामले)

| परिदृश्य | विवरण |
|------|------|
| **JSBlock / JSField** | `route.pathname` या `route.params` के आधार पर कंडीशनल रेंडरिंग करना या वर्तमान पेज की पहचान प्रदर्शित करना। |
| **लिंकेज नियम / इवेंट फ्लो** | लॉजिक ब्रांचिंग या चाइल्ड कंपोनेंट्स को पास करने के लिए रूट पैरामीटर (जैसे `params.name`) पढ़ना। |
| **व्यू नेविगेशन** | आंतरिक रूप से `ctx.route.pathname` की तुलना लक्ष्य पथ (target path) से करना ताकि यह तय किया जा सके कि `ctx.router.navigate` को ट्रिगर करना है या नहीं। |

> **ध्यान दें:** `ctx.route` केवल उन RunJS वातावरणों में उपलब्ध है जहाँ रूटिंग संदर्भ (routing context) मौजूद हो (जैसे पेज के भीतर JSBlock, फ्लो पेज आदि); शुद्ध बैकएंड या बिना रूटिंग वाले संदर्भों (जैसे वर्कफ़्लो) में यह रिक्त (null) हो सकता है।

## 类型定义 (प्रकार परिभाषा)

```ts
type RouteOptions = {
  name?: string;   // रूट की विशिष्ट पहचान
  path?: string;   // रूट टेम्पलेट (जैसे /admin/:name)
  params?: Record<string, any>;  // रूट पैरामीटर (जैसे { name: 'users' })
  pathname?: string;  // वर्तमान रूट का पूरा पथ (जैसे /admin/users)
};
```

## 常用字段 (सामान्य फ़ील्ड)

| फ़ील्ड | प्रकार | विवरण |
|------|------|------|
| `pathname` | `string` | वर्तमान रूट का पूरा पथ, जो `ctx.location.pathname` के समान है। |
| `params` | `Record<string, any>` | रूट टेम्पलेट से पार्स किए गए डायनेमिक पैरामीटर, जैसे `{ name: 'users' }`। |
| `path` | `string` | रूट टेम्पलेट, जैसे `/admin/:name`। |
| `name` | `string` | रूट की विशिष्ट पहचान, अक्सर मल्टी-टैब या मल्टी-व्यू परिदृश्यों में उपयोग की जाती है। |

## 与 ctx.router、ctx.location 的关系 (ctx.router और ctx.location के साथ संबंध)

| उपयोग | अनुशंसित उपयोग |
|------|----------|
| **वर्तमान पथ पढ़ना** | `ctx.route.pathname` या `ctx.location.pathname`, मिलान के दौरान दोनों एक समान होते हैं। |
| **रूट पैरामीटर पढ़ना** | `ctx.route.params`, जैसे `params.name` वर्तमान पेज की UID को दर्शाता है। |
| **नेविगेशन जंप** | `ctx.router.navigate(path)` |
| **क्वेरी पैरामीटर, state पढ़ना** | `ctx.location.search`, `ctx.location.state` |

`ctx.route` "मिलान किए गए रूट कॉन्फ़िगरेशन" पर केंद्रित है, जबकि `ctx.location` "वर्तमान URL स्थिति" पर केंद्रित है। दोनों मिलकर वर्तमान रूट स्थिति का पूरा विवरण प्रदान करते हैं।

## 示例 (उदाहरण)

### pathname पढ़ना

```ts
// वर्तमान पथ प्रदर्शित करें
ctx.message.info('वर्तमान पेज: ' + ctx.route.pathname);
```

### params के आधार पर ब्रांचिंग करना

```ts
// params.name आमतौर पर वर्तमान पेज की UID होती है (जैसे फ्लो पेज की पहचान)
if (ctx.route.params?.name === 'users') {
  // उपयोगकर्ता प्रबंधन पेज पर विशिष्ट लॉजिक निष्पादित करें
}
```

### फ्लो (Flow) पेज में प्रदर्शित करना

```tsx
<div>
  <h1>वर्तमान पेज - {ctx.route.pathname}</h1>
  <p>रूट पहचानकर्ता: {ctx.route.params?.name}</p>
</div>
```

- [ctx.location](./location.md): वर्तमान URL स्थिति (pathname, search, hash, state), `ctx.route` के साथ उपयोग किया जाता है।
