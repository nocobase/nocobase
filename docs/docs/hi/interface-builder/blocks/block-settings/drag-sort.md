# ड्रैग-एंड-ड्रॉप सॉर्टिंग

## परिचय

ड्रैग-एंड-ड्रॉप सॉर्टिंग एक सॉर्ट फ़ील्ड (sort field) पर निर्भर करती है, जिसका उपयोग ब्लॉक रिकॉर्ड्स को मैन्युअल रूप से पुनर्व्यवस्थित करने के लिए किया जाता है।


:::info{title=सुझाव}
* जब एक ही सॉर्ट फ़ील्ड का उपयोग कई ब्लॉक्स में ड्रैग-एंड-ड्रॉप सॉर्टिंग के लिए किया जाता है, तो यह मौजूदा क्रम को बिगाड़ सकता है।
* टेबल में ड्रैग-एंड-ड्रॉप सॉर्टिंग का उपयोग करते समय, सॉर्ट फ़ील्ड के लिए ग्रुपिंग नियम (grouping rules) कॉन्फ़िगर नहीं किए जा सकते।
* ट्री टेबल (Tree tables) केवल एक ही स्तर (level) के भीतर नोड्स को सॉर्ट करने का समर्थन करते हैं।

:::


## कॉन्फ़िगरेशन

एक "सॉर्ट" (Sort) प्रकार का फ़ील्ड जोड़ें। संग्रह (collection) बनाते समय सॉर्ट फ़ील्ड अब स्वचालित रूप से जनरेट नहीं होते हैं; उन्हें मैन्युअल रूप से बनाया जाना चाहिए।

![](https://static-docs.nocobase.com/470891c7bb34c506328c1f3824a6cf20.png)

टेबल के लिए ड्रैग-एंड-ड्रॉप सॉर्टिंग सक्षम करते समय, आपको एक सॉर्ट फ़ील्ड चुनना होगा।

## सॉर्टिंग नियमों की व्याख्या

मान लीजिए कि वर्तमान क्रम है:

```
[1,2,3,4,5,6,7,8,9]
```

जब कोई तत्व (जैसे, 5) आगे बढ़कर 3 के स्थान पर ले जाया जाता है, तो केवल 3, 4 और 5 के सॉर्ट मान बदलेंगे: 5 अब 3 का स्थान ले लेगा, और 3 तथा 4 प्रत्येक एक स्थान पीछे खिसक जाएंगे।

```
[1,2,5,3,4,6,7,8,9]
```

यदि आप फिर 6 को पीछे की ओर 8 के स्थान पर ले जाते हैं, तो 6 अब 8 का स्थान ले लेगा, और 7 तथा 8 प्रत्येक एक स्थान आगे बढ़ जाएंगे।

```
[1,2,5,3,4,7,8,6,9]
```
