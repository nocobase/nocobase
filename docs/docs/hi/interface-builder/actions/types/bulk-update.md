# बल्क अपडेट

## परिचय

बल्क अपडेट एक्शन का उपयोग तब किया जाता है जब आपको रिकॉर्ड के एक समूह पर एक ही अपडेट लागू करने की आवश्यकता होती है। बल्क अपडेट एक्शन करने से पहले, उपयोगकर्ता को अपडेट के लिए फ़ील्ड असाइनमेंट लॉजिक को पहले से परिभाषित करना होगा। यह लॉजिक तब सभी चुने हुए रिकॉर्ड पर लागू होगा जब आप अपडेट बटन पर क्लिक करेंगे।

![20251029195320](https://static-docs.nocobase.com/20251029195320.png)

## एक्शन कॉन्फ़िगरेशन

![20251029195729](https://static-docs.nocobase.com/20251029195729.png)

### अपडेट करने के लिए डेटा

चुने हुए/सभी, डिफ़ॉल्ट रूप से 'चुने हुए' होते हैं।

![20251029200034](https://static-docs.nocobase.com/20251029200034.png)

### फ़ील्ड असाइनमेंट

बल्क अपडेट के लिए फ़ील्ड सेट करें। केवल सेट किए गए फ़ील्ड ही अपडेट होंगे।

जैसा कि चित्र में दिखाया गया है, ऑर्डर टेबल में बल्क अपडेट एक्शन को कॉन्फ़िगर करें ताकि चुने हुए डेटा को 'अनुमोदन लंबित' (Pending Approval) पर बल्क अपडेट किया जा सके।

![20251029200109](https://static-docs.nocobase.com/20251029200109.png)

- [लिंकेज नियम](/interface-builder/actions/action-settings/linkage-rule): बटन को गतिशील रूप से दिखाएं/छिपाएं;
