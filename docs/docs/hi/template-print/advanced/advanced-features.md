## उन्नत सुविधाएँ

### पेजिंग

#### 1. पृष्ठ संख्या अपडेट करना

##### सिंटैक्स
इसे अपने Office सॉफ़्टवेयर में आसानी से डालें।

##### उदाहरण
Microsoft Word में:
- "इन्सर्ट (Insert) → पेज नंबर (Page Number)" फ़ंक्शन का उपयोग करें  
LibreOffice में:
- "इन्सर्ट (Insert) → फ़ील्ड (Field) → पेज नंबर (Page Number)" फ़ंक्शन का उपयोग करें

##### परिणाम
तैयार की गई रिपोर्ट में, प्रत्येक पृष्ठ पर पृष्ठ संख्याएँ अपने आप अपडेट हो जाएँगी।


#### 2. विषय-सूची बनाना

##### सिंटैक्स
इसे अपने Office सॉफ़्टवेयर में आसानी से डालें।

##### उदाहरण
Microsoft Word में:
- "इन्सर्ट (Insert) → इंडेक्स और टेबल (Index and Table) → टेबल ऑफ़ कंटेंट्स (Table of Contents)" फ़ंक्शन का उपयोग करें  
LibreOffice में:
- "इन्सर्ट (Insert) → टेबल ऑफ़ कंटेंट्स और इंडेक्स (Table of Contents and Index) → टेबल, इंडेक्स या बिब्लियोग्राफी (Table, Index or Bibliography)" फ़ंक्शन का उपयोग करें

##### परिणाम
रिपोर्ट की विषय-सूची दस्तावेज़ की सामग्री के आधार पर अपने आप अपडेट हो जाएगी।


#### 3. टेबल हेडर दोहराना

##### सिंटैक्स
इसे अपने Office सॉफ़्टवेयर में आसानी से डालें।

##### उदाहरण
Microsoft Word में:
- टेबल हेडर पर राइट-क्लिक करें → टेबल प्रॉपर्टीज़ (Table Properties) → "प्रत्येक पृष्ठ के शीर्ष पर हेडर पंक्ति के रूप में दोहराएँ (Repeat as header row at the top of each page)" को चेक करें  
LibreOffice में:
- टेबल हेडर पर राइट-क्लिक करें → टेबल प्रॉपर्टीज़ (Table Properties) → टेक्स्ट फ़्लो टैब (Text Flow tab) → "हेडिंग दोहराएँ (Repeat heading)" को चेक करें

##### परिणाम
जब कोई टेबल कई पृष्ठों पर फैली होती है, तो हेडर प्रत्येक पृष्ठ के शीर्ष पर अपने आप दोहराया जाएगा।


### अंतर्राष्ट्रीयकरण (i18n)

#### 1. स्टैटिक टेक्स्ट का अनुवाद

##### सिंटैक्स
स्टैटिक टेक्स्ट को अंतर्राष्ट्रीयकृत करने के लिए `{t(text)}` टैग का उपयोग करें:
```
{t(meeting)}
```

##### उदाहरण
टेम्पलेट में:
```
{t(meeting)} {t(apples)}
```
JSON डेटा या एक बाहरी स्थानीयकरण डिक्शनरी (जैसे "fr-fr" के लिए) में संबंधित अनुवाद प्रदान किए जाते हैं, उदाहरण के लिए "meeting" → "rendez-vous" और "apples" → "Pommes"।

##### परिणाम
रिपोर्ट बनाते समय, टेक्स्ट को लक्ष्य भाषा के आधार पर संबंधित अनुवाद से बदल दिया जाएगा।


#### 2. डायनामिक टेक्स्ट का अनुवाद

##### सिंटैक्स
डेटा सामग्री के लिए, आप `:t` फ़ॉर्मेटर का उपयोग कर सकते हैं, उदाहरण के लिए:
```
{d.id:ifEQ(2):show({t(monday)}):elseShow({t(tuesday)})}
```

##### उदाहरण
टेम्पलेट में:
```
{d.id:ifEQ(2):show({t(monday)}):elseShow({t(tuesday)})}
```
JSON डेटा और स्थानीयकरण डिक्शनरी में उचित अनुवाद प्रदान किए जाते हैं।

##### परिणाम
शर्त के आधार पर, आउटपुट "lundi" या "mardi" होगा (लक्ष्य भाषा के उदाहरण के रूप में)।


### कुंजी-मान मैपिंग

#### 1. एनम रूपांतरण (:convEnum)

##### सिंटैक्स
```
{डेटा:convEnum(एनम नाम)}
```
उदाहरण के लिए:
```
0:convEnum('ORDER_STATUS')
```

##### उदाहरण
API विकल्पों के उदाहरण में, निम्नलिखित प्रदान किया गया है:
```json
{
  "enum": {
    "ORDER_STATUS": ["pending", "sent", "delivered"]
  }
}
```
टेम्पलेट में:
```
0:convEnum('ORDER_STATUS')
```

##### परिणाम
आउटपुट "pending" होगा; यदि इंडेक्स एनम सीमा से अधिक है, तो मूल मान आउटपुट होगा।


### डायनामिक इमेज
:::info
वर्तमान में XLSX, DOCX फ़ाइल प्रकारों का समर्थन करता है
:::
आप दस्तावेज़ टेम्पलेट में "डायनामिक इमेज" डाल सकते हैं, जिसका अर्थ है कि टेम्पलेट में प्लेसहोल्डर इमेज रेंडरिंग के दौरान डेटा के आधार पर वास्तविक इमेज से अपने आप बदल जाएंगी। यह प्रक्रिया बहुत सरल है और इसके लिए केवल आवश्यकता है:

1. एक अस्थायी इमेज को प्लेसहोल्डर के रूप में डालें

2. फ़ील्ड लेबल सेट करने के लिए उस इमेज के "ऑल्ट टेक्स्ट (Alt Text)" को एडिट करें

3. दस्तावेज़ को रेंडर करें, और सिस्टम इसे स्वचालित रूप से वास्तविक इमेज से बदल देगा

नीचे हम विशिष्ट उदाहरणों के माध्यम से DOCX और XLSX के लिए ऑपरेशन विधियों की व्याख्या करेंगे।


#### DOCX फ़ाइलों में डायनामिक इमेज डालना
##### एकल इमेज प्रतिस्थापन

1. अपना DOCX टेम्पलेट खोलें और एक अस्थायी इमेज डालें (यह कोई भी प्लेसहोल्डर इमेज हो सकती है, जैसे [ठोस नीली इमेज](https://static-docs.nocobase.com/solid-color-image-2025-04-14-11-00-26.png))

:::info
**इमेज फ़ॉर्मेट निर्देश**

- वर्तमान में, प्लेसहोल्डर इमेज केवल PNG फ़ॉर्मेट का समर्थन करती हैं। हम हमारे द्वारा प्रदान की गई उदाहरण [ठोस नीली इमेज](https://static-docs.nocobase.com/solid-color-image-2025-04-14-11-00-26.png) का उपयोग करने की सलाह देते हैं।
- लक्ष्य रेंडर की गई इमेज केवल PNG, JPG, JPEG फ़ॉर्मेट का समर्थन करती हैं। अन्य इमेज प्रकार रेंडर होने में विफल हो सकते हैं।

**इमेज साइज़ निर्देश**

चाहे DOCX के लिए हो या XLSX के लिए, अंतिम रेंडर की गई इमेज का साइज़ टेम्पलेट में अस्थायी इमेज के आयामों का पालन करेगा। यानी, वास्तविक प्रतिस्थापन इमेज आपके द्वारा डाली गई प्लेसहोल्डर इमेज के साइज़ से स्वचालित रूप से स्केल हो जाएगी। यदि आप चाहते हैं कि रेंडर की गई इमेज का साइज़ 150×150 हो, तो कृपया टेम्पलेट में एक अस्थायी इमेज का उपयोग करें और उसे उस साइज़ में समायोजित करें।
:::

2. इस इमेज पर राइट-क्लिक करें, इसके "ऑल्ट टेक्स्ट (Alt Text)" को एडिट करें, और वह इमेज फ़ील्ड लेबल भरें जिसे आप डालना चाहते हैं, उदाहरण के लिए `{d.imageUrl}`:
   
![20250414211130-2025-04-14-21-11-31](https://static-docs.nocobase.com/20250414211130-2025-04-14-21-11-31.png)

3. रेंडरिंग के लिए निम्नलिखित उदाहरण डेटा का उपयोग करें:
```json
{
  "name": "Apple",
  "imageUrl": "https://images.pexels.com/photos/206959/pexels-photo-206959.jpeg"
}
```

4. रेंडर किए गए परिणाम में, अस्थायी इमेज को वास्तविक इमेज से बदल दिया जाएगा:

![20250414203444-2025-04-14-20-34-46](https://static-docs.nocobase.com/20250414203444-2025-04-14-20-34-46.png)

##### एकाधिक इमेज लूप प्रतिस्थापन

यदि आप टेम्पलेट में इमेज का एक समूह डालना चाहते हैं, जैसे कि उत्पाद सूची, तो आप इसे लूप के माध्यम से भी लागू कर सकते हैं। विशिष्ट चरण इस प्रकार हैं:
1. मान लीजिए आपका डेटा इस प्रकार है:
```json
{
  "products": [
    {
      "name": "Apple",
      "imageUrl": "https://images.pexels.com/photos/206959/pexels-photo-206959.jpeg"
    },
    {
      "name": "Banana",
      "imageUrl": "https://images.pexels.com/photos/61127/pexels-photo-61127.jpeg"
    }
  ]
}
```

2. DOCX टेम्पलेट में एक लूप क्षेत्र सेट करें, और प्रत्येक लूप आइटम में अस्थायी इमेज डालें, जिसमें ऑल्ट टेक्स्ट `{d.products[i].imageUrl}` पर सेट हो, जैसा कि नीचे दिखाया गया है:

![20250414205418-2025-04-14-20-54-19](https://static-docs.nocobase.com/20250414205418-2025-04-14-20-54-19.png)

3. रेंडरिंग के बाद, सभी अस्थायी इमेज को उनके संबंधित डेटा इमेज से बदल दिया जाएगा:
   
![20250414205503-2025-04-14-20-55-05](https://static-docs.nocobase.com/20250414205503-2025-04-14-20-55-05.png)

#### XLSX फ़ाइलों में डायनामिक इमेज डालना

Excel टेम्पलेट (XLSX) में ऑपरेशन विधि मूल रूप से समान है, बस निम्नलिखित बातों पर ध्यान दें:

1. इमेज डालने के बाद, कृपया सुनिश्चित करें कि आपने "सेल के भीतर इमेज (image within cell)" का चयन किया है, न कि इमेज सेल के ऊपर तैर रही हो।

![20250414211643-2025-04-14-21-16-45](https://static-docs.nocobase.com/20250414211643-2025-04-14-21-16-45.png)

2. सेल का चयन करने के बाद, फ़ील्ड लेबल भरने के लिए "ऑल्ट टेक्स्ट (Alt Text)" देखने के लिए क्लिक करें, जैसे `{d.imageUrl}`।

#### बारकोड बनाना (जैसे QR कोड)

बारकोड बनाने का तरीका डायनामिक इमेज के समान है, जिसमें केवल तीन चरण शामिल हैं:

1. टेम्पलेट में बारकोड की स्थिति को चिह्नित करने के लिए एक अस्थायी इमेज डालें

2. इमेज के "ऑल्ट टेक्स्ट (Alt Text)" को एडिट करें और बारकोड फ़ॉर्मेट फ़ील्ड लेबल लिखें, उदाहरण के लिए `{d.code:barcode(qrcode)}`, जहाँ `qrcode` बारकोड का प्रकार है (नीचे समर्थित सूची देखें)
