# स्थानीयकरण प्रबंधन

## परिचय

स्थानीयकरण प्रबंधन प्लगइन का उपयोग NocoBase के स्थानीयकरण संसाधनों को प्रबंधित और लागू करने के लिए किया जाता है, यह सिस्टम के मेनू, संग्रह, फ़ील्ड और सभी प्लगइन का अनुवाद कर सकता है, ताकि विशिष्ट क्षेत्रों की भाषा और संस्कृति के अनुकूल हो सके।

## इंस्टॉलेशन

यह प्लगइन अंतर्निहित है, इसके लिए किसी अतिरिक्त इंस्टॉलेशन की आवश्यकता नहीं है।

### अनुवाद प्रविष्टियों को सिंक करें

<img src="https://static-docs.nocobase.com/202404202134850.png"/>

वर्तमान में निम्नलिखित सामग्री को सिंक किया जा सकता है:

- सिस्टम और प्लगइन के स्थानीय भाषा पैक
- संग्रह शीर्षक, फ़ील्ड शीर्षक और फ़ील्ड विकल्प लेबल
- मेनू शीर्षक

सिंक पूरा होने के बाद, सिस्टम वर्तमान भाषा के लिए सभी अनुवाद योग्य प्रविष्टियों को सूचीबद्ध करेगा।

<img src="https://static-docs.nocobase.com/202404202136567.png"/>

:::info{title=सुझाव}
विभिन्न मॉड्यूल में समान मूल पाठ प्रविष्टियाँ हो सकती हैं, जिन्हें अलग-अलग अनुवाद करने की आवश्यकता है।
:::

### स्वचालित रूप से प्रविष्टियाँ बनाएँ

पेज संपादित करते समय, प्रत्येक ब्लॉक में कस्टम टेक्स्ट स्वचालित रूप से संबंधित प्रविष्टियाँ बनाएगा और वर्तमान भाषा के लिए अनुवाद सामग्री को सिंक करेगा।

![](https://static-docs.nocobase.com/Localization-02-12-2026_08_39_AM.png)

![](https://static-docs.nocobase.com/Localization-NocoBase-02-12-2026_08_39_AM.png)

:::info{title=सुझाव}
कोड में टेक्स्ट परिभाषित करते समय, आपको मैन्युअल रूप से ns (namespace) निर्दिष्ट करने की आवश्यकता होती है, जैसे: `${ctx.i18n.t('My custom js block', { ns: 'lm-flow-engine' })}`
:::
