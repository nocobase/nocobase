# संदर्भ वैरिएबल का उपयोग करें

संदर्भ वैरिएबल का उपयोग करके, आप वर्तमान पेज, उपयोगकर्ता, समय और फ़िल्टर इनपुट जैसी जानकारी का सीधे पुनः उपयोग कर सकते हैं, ताकि संदर्भ के अनुसार चार्ट रेंडर किए जा सकें और उनमें आपसी जुड़ाव स्थापित हो सके।

## लागू होने का दायरा
- बिल्डर मोड में डेटा क्वेरी के लिए फ़िल्टर शर्तों में वैरिएबल चुनें।
![clipboard-image-1761486073](https://static-docs.nocobase.com/clipboard-image-1761486073.png)

- SQL मोड में डेटा क्वेरी के लिए स्टेटमेंट लिखते समय, वैरिएबल चुनें और एक्सप्रेशन डालें (उदाहरण के लिए, `{{ ctx.user.id }}`)।
![clipboard-image-1761486145](https://static-docs.nocobase.com/clipboard-image-1761486145.png)

- कस्टम मोड में चार्ट विकल्पों के लिए सीधे JS एक्सप्रेशन लिखें।
![clipboard-image-1761486604](https://static-docs.nocobase.com/clipboard-image-1761486604.png)

- इंटरैक्शन इवेंट्स के लिए (उदाहरण के लिए, ड्रिल-डाउन डायलॉग खोलने और डेटा पास करने के लिए क्लिक करना), सीधे JS एक्सप्रेशन लिखें।
![clipboard-image-1761486683](https://static-docs.nocobase.com/clipboard-image-1761486683.png)

**ध्यान दें:**
- `{{ ... }}` को सिंगल या डबल कोट्स में न लपेटें; बाइंडिंग के दौरान सिस्टम वैरिएबल के प्रकार (स्ट्रिंग, संख्या, समय, NULL) के आधार पर इसे सुरक्षित रूप से हैंडल करता है।
- जब कोई वैरिएबल `NULL` या अपरिभाषित हो, तो SQL में `COALESCE(...)` या `IS NULL` का उपयोग करके नल मानों को स्पष्ट रूप से हैंडल करें।
