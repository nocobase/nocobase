# अनेक-से-एक (Many-to-One)

एक लाइब्रेरी डेटाबेस में, दो इकाइयाँ होती हैं: किताबें और लेखक। एक लेखक कई किताबें लिख सकता है, लेकिन हर किताब का आमतौर पर एक ही लेखक होता है। इस स्थिति में, लेखक और किताबों के बीच का संबंध अनेक-से-एक होता है। कई किताबें एक ही लेखक से जुड़ी हो सकती हैं, लेकिन हर किताब का केवल एक ही लेखक हो सकता है।

## पैरामीटर विवरण

### Source collection (स्रोत संग्रह)

यह वह स्रोत संग्रह है, जिसमें वर्तमान फ़ील्ड मौजूद है।

### Target collection (लक्ष्य संग्रह)

यह वह लक्ष्य संग्रह है, जिससे इसे संबद्ध किया जाना है।

### Foreign key (फ़ॉरेन की)

यह स्रोत संग्रह का वह फ़ील्ड है, जिसका उपयोग दो संग्रहों के बीच संबंध स्थापित करने के लिए किया जाता है।

### Target key (लक्ष्य की)

यह लक्ष्य संग्रह का वह फ़ील्ड है जिसका संदर्भ फ़ॉरेन की द्वारा दिया जाता है। यह अद्वितीय होना चाहिए।

### ON DELETE

ON DELETE उन नियमों को संदर्भित करता है जो पैरेंट संग्रह में रिकॉर्ड हटाए जाने पर संबंधित चाइल्ड संग्रह में फ़ॉरेन की संदर्भों पर लागू होते हैं। यह फ़ॉरेन की कंस्ट्रेंट परिभाषित करते समय उपयोग किया जाने वाला एक विकल्प है। सामान्य ON DELETE विकल्पों में शामिल हैं:

- **CASCADE**: जब पैरेंट संग्रह में कोई रिकॉर्ड हटाया जाता है, तो चाइल्ड संग्रह में उससे संबंधित सभी रिकॉर्ड स्वचालित रूप से हटा दिए जाते हैं।
- **SET NULL**: जब पैरेंट संग्रह में कोई रिकॉर्ड हटाया जाता है, तो संबंधित चाइल्ड संग्रह रिकॉर्ड में फ़ॉरेन की मान NULL पर सेट कर दिए जाते हैं।
- **RESTRICT**: यह डिफ़ॉल्ट विकल्प है, यह पैरेंट संग्रह रिकॉर्ड को हटाने से रोकता है यदि चाइल्ड संग्रह में उससे संबंधित रिकॉर्ड मौजूद हों।
- **NO ACTION**: RESTRICT के समान, यह पैरेंट संग्रह रिकॉर्ड को हटाने से रोकता है यदि चाइल्ड संग्रह में उससे संबंधित रिकॉर्ड मौजूद हों।
