# अनेक-से-अनेक (ऐरे)

## परिचय

यह सुविधा आपको डेटा संग्रह में ऐरे फ़ील्ड का उपयोग करके लक्ष्य संग्रह के कई अद्वितीय कुंजी (unique keys) को सहेजने की अनुमति देती है, जिससे दोनों संग्रहों के बीच अनेक-से-अनेक संबंध स्थापित किया जा सके। उदाहरण के लिए, लेख (Articles) और टैग (Tags) नामक दो एंटिटीज़ पर विचार करें। एक लेख कई टैग से जुड़ा हो सकता है, जिसमें लेख संग्रह में टैग संग्रह के संबंधित रिकॉर्ड की ID को एक ऐरे फ़ील्ड में सहेजा जाता है।

:::warning{title=ध्यान दें}

- जब भी संभव हो, इस विधि पर निर्भर रहने के बजाय एक मानक [अनेक-से-अनेक](../data-modeling/collection-fields/associations/m2m/index.md) संबंध स्थापित करने के लिए जंक्शन संग्रह का उपयोग करने की सलाह दी जाती है।
- वर्तमान में, ऐरे फ़ील्ड के साथ स्थापित अनेक-से-अनेक संबंधों के लिए, लक्ष्य संग्रह के फ़ील्ड का उपयोग करके स्रोत संग्रह डेटा को फ़िल्टर करने का समर्थन केवल PostgreSQL ही करता है। उदाहरण के लिए, उपरोक्त परिदृश्य में, आप टैग संग्रह में अन्य फ़ील्ड, जैसे शीर्षक, के आधार पर लेखों को फ़िल्टर कर सकते हैं।
  :::

## पैरामीटर विवरण

### स्रोत संग्रह

स्रोत संग्रह, जहाँ वर्तमान फ़ील्ड मौजूद है।

### लक्ष्य संग्रह

लक्ष्य संग्रह, जिसके साथ संबंध स्थापित किया गया है।

### फ़ॉरेन कुंजी (Foreign key)

स्रोत संग्रह में वह ऐरे फ़ील्ड जो लक्ष्य संग्रह से टारगेट कुंजी (Target key) को संग्रहीत करता है।

ऐरे फ़ील्ड प्रकारों के लिए संबंधित संबंध इस प्रकार हैं:

| NocoBase | PostgreSQL | MySQL  | SQLite |
| -------- | ---------- | ------ | ------ |
| `set`    | `array`    | `JSON` | `JSON` |

### टारगेट कुंजी (Target key)

लक्ष्य संग्रह में वह फ़ील्ड जो स्रोत संग्रह के ऐरे फ़ील्ड में संग्रहीत मानों से मेल खाता है। यह फ़ील्ड अद्वितीय (unique) होना चाहिए।
