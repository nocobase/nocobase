# फ़ाइल मैनेजर

## परिचय

फ़ाइल मैनेजर प्लगइन फ़ाइलों को प्रभावी ढंग से प्रबंधित करने के लिए एक फ़ाइल संग्रह, अटैचमेंट फ़ील्ड और फ़ाइल स्टोरेज इंजन प्रदान करता है। फ़ाइलें एक विशेष प्रकार के संग्रह में रिकॉर्ड होती हैं, जिसे फ़ाइल संग्रह कहा जाता है, जो फ़ाइल मेटाडेटा को संग्रहीत करता है और फ़ाइल मैनेजर के माध्यम से प्रबंधित किया जा सकता है। अटैचमेंट फ़ील्ड, फ़ाइल संग्रह से संबंधित विशिष्ट संबंध फ़ील्ड होते हैं। यह प्लगइन कई स्टोरेज विधियों का समर्थन करता है, और वर्तमान में समर्थित फ़ाइल स्टोरेज इंजन में स्थानीय स्टोरेज, अलीबाबा क्लाउड OSS, अमेज़न S3 और Tencent क्लाउड COS शामिल हैं।

## उपयोगकर्ता मैनुअल

### फ़ाइल संग्रह

एक `attachments` संग्रह पहले से मौजूद होता है, जो सभी अटैचमेंट फ़ील्ड से संबंधित फ़ाइलों को संग्रहीत करता है। इसके अलावा, आप विशिष्ट फ़ाइलों को संग्रहीत करने के लिए नए फ़ाइल संग्रह भी बना सकते हैं।

[अधिक जानकारी के लिए फ़ाइल संग्रह दस्तावेज़ देखें](/data-sources/file-manager/file-collection)

### अटैचमेंट फ़ील्ड

अटैचमेंट फ़ील्ड, फ़ाइल संग्रह से संबंधित विशिष्ट संबंध फ़ील्ड होते हैं, जिन्हें 'अटैचमेंट' फ़ील्ड प्रकार के माध्यम से बनाया जा सकता है या 'संबंध' फ़ील्ड के माध्यम से कॉन्फ़िगर किया जा सकता है।

[अधिक जानकारी के लिए अटैचमेंट फ़ील्ड दस्तावेज़ देखें](/data-sources/file-manager/field-attachment)

### फ़ाइल स्टोरेज इंजन

फ़ाइल स्टोरेज इंजन का उपयोग फ़ाइलों को विशिष्ट सेवाओं में सहेजने के लिए किया जाता है, जिसमें स्थानीय स्टोरेज (सर्वर की हार्ड ड्राइव पर सहेजना), क्लाउड स्टोरेज आदि शामिल हैं।

[अधिक जानकारी के लिए फ़ाइल स्टोरेज इंजन दस्तावेज़ देखें](./storage/index.md)

### HTTP API

फ़ाइल अपलोड को HTTP API के माध्यम से संभाला जा सकता है, संदर्भ के लिए [HTTP API](./http-api.md) देखें।
