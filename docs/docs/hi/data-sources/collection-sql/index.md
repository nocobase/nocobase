# SQL संग्रह

## परिचय

SQL संग्रह SQL क्वेरी का उपयोग करके डेटा प्राप्त करने का एक शक्तिशाली तरीका प्रदान करता है। SQL क्वेरी के माध्यम से डेटा फ़ील्ड निकालने और संबंधित फ़ील्ड मेटाडेटा को कॉन्फ़िगर करने के बाद, उपयोगकर्ता इन फ़ील्ड का उपयोग वैसे ही कर सकते हैं जैसे वे एक मानक टेबल के साथ काम कर रहे हों। यह सुविधा विशेष रूप से जटिल जॉइन क्वेरी, सांख्यिकीय विश्लेषण और ऐसे ही अन्य परिदृश्यों के लिए फायदेमंद है।

## उपयोगकर्ता पुस्तिका

### नया SQL संग्रह बनाना

<img src="https://static-docs.nocobase.com/202405191452918.png"/>

1. दिए गए इनपुट बॉक्स में अपनी SQL क्वेरी दर्ज करें और 'एक्सेक्यूट' (Execute) पर क्लिक करें। सिस्टम क्वेरी का विश्लेषण करेगा ताकि यह निर्धारित किया जा सके कि इसमें कौन सी टेबल और फ़ील्ड शामिल हैं, और स्रोत टेबल से संबंधित फ़ील्ड मेटाडेटा को स्वचालित रूप से निकालेगा।

<img src="https://static-docs.nocobase.com/202405191453556.png"/>

2. यदि सिस्टम द्वारा स्रोत टेबल और फ़ील्ड का विश्लेषण गलत है, तो आप सही मेटाडेटा का उपयोग सुनिश्चित करने के लिए मैन्युअल रूप से उचित टेबल और फ़ील्ड का चयन कर सकते हैं। पहले स्रोत टेबल का चयन करें, फिर नीचे दिए गए फ़ील्ड स्रोत अनुभाग में संबंधित फ़ील्ड चुनें।

<img src="https://static-docs.nocobase.com/202405191453579.png"/>

3. जिन फ़ील्ड का कोई सीधा स्रोत नहीं है, उनके लिए सिस्टम डेटा प्रकार के आधार पर फ़ील्ड प्रकार का अनुमान लगाएगा। यदि यह अनुमान गलत है, तो आप मैन्युअल रूप से उचित फ़ील्ड प्रकार का चयन कर सकते हैं।

<img src="https://static-docs.nocobase.com/202405191454703.png"/>

4. जैसे ही आप प्रत्येक फ़ील्ड को कॉन्फ़िगर करते हैं, आप पूर्वावलोकन क्षेत्र में उसका प्रदर्शन देख सकते हैं, जिससे आप अपनी सेटिंग्स का तत्काल प्रभाव देख सकें।

<img src="https://static-docs.nocobase.com/202405191455439.png"/>

5. कॉन्फ़िगरेशन पूरा करने और यह पुष्टि करने के बाद कि सब कुछ सही है, सबमिशन को अंतिम रूप देने के लिए SQL इनपुट बॉक्स के नीचे 'कन्फर्म' (Confirm) बटन पर क्लिक करें।

<img src="https://static-docs.nocobase.com/202405191455302.png"/>

### संपादन

1. यदि आपको SQL क्वेरी को संशोधित करने की आवश्यकता है, तो 'एडिट' (Edit) बटन पर क्लिक करके SQL स्टेटमेंट को सीधे बदलें और आवश्यकतानुसार फ़ील्ड को फिर से कॉन्फ़िगर करें।

2. फ़ील्ड मेटाडेटा को समायोजित करने के लिए, 'कॉन्फ़िगर फ़ील्ड्स' (Configure fields) विकल्प का उपयोग करें, जो आपको फ़ील्ड सेटिंग्स को ठीक वैसे ही अपडेट करने की अनुमति देता है जैसे आप एक नियमित टेबल के लिए करते हैं।

### सिंक्रनाइज़ेशन

यदि SQL क्वेरी अपरिवर्तित रहती है लेकिन अंतर्निहित डेटाबेस टेबल संरचना को संशोधित किया गया है, तो आप 'कॉन्फ़िगर फ़ील्ड्स' (Configure fields) - 'डेटाबेस से सिंक करें' (Sync from database) का चयन करके फ़ील्ड को सिंक्रनाइज़ और फिर से कॉन्फ़िगर कर सकते हैं।

<img src="https://static-docs.nocobase.com/202405191456216.png"/>

:::warning
SQL संग्रह का उपयोग करते समय, सुनिश्चित करें कि आप उन टेबल का चयन करें जिन्हें NocoBase के भीतर प्रबंधित किया जा सकता है। एक ही डेटाबेस से उन टेबल का उपयोग करना जो NocoBase से कनेक्टेड नहीं हैं, SQL क्वेरी पार्सिंग में अशुद्धि का कारण बन सकता है। यदि यह एक चिंता का विषय है, तो एक व्यू बनाने और उसे लिंक करने पर विचार करें।
:::
