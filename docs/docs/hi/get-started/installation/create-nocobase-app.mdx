# create-nocobase-app इंस्टॉलेशन

:::tip आवश्यक शर्तें

- आपके सिस्टम पर Node.js 20+ और Yarn 1.22.x इंस्टॉल होने चाहिए।
- आवश्यक डेटाबेस (MySQL 8.0.17+, MariaDB 10.9+, या PostgreSQL 10+) में से कोई एक कॉन्फ़िगर और चालू होना चाहिए।

:::

:::warning एनवायरनमेंट वेरिएबल्स (Environment Variables) की जानकारी

- `TZ` का उपयोग एप्लिकेशन का टाइम ज़ोन सेट करने के लिए किया जाता है, जो डिफ़ॉल्ट रूप से ऑपरेटिंग सिस्टम का टाइम ज़ोन होता है।
- `APP_KEY` एप्लिकेशन की गुप्त कुंजी है, जिसका उपयोग यूज़र टोकन आदि बनाने के लिए किया जाता है (यदि `APP_KEY` बदल दिया जाता है, तो पुराने टोकन भी अमान्य हो जाएँगे)। यह कोई भी रैंडम स्ट्रिंग हो सकती है। **कृपया इसे अपनी गुप्त कुंजी में बदलें और सुनिश्चित करें कि यह किसी को भी उजागर न हो।**
- `DB_*` डेटाबेस से संबंधित कॉन्फ़िगरेशन हैं। कृपया अपनी वास्तविक सेटअप के अनुसार उन्हें अपने डेटाबेस कनेक्शन जानकारी से बदलें।

:::

## 2. डायरेक्टरी बदलें

```bash
cd my-nocobase-app
```

## 3. डिपेंडेंसीज़ इंस्टॉल करें

📢 आपके नेटवर्क वातावरण, सिस्टम कॉन्फ़िगरेशन और अन्य कारकों के आधार पर, इस चरण में दस मिनट से अधिक का समय लग सकता है।

```bash
yarn install
```

:::tip प्रोडक्शन एनवायरनमेंट (Production Environment) के लिए सुझाव

प्रोडक्शन एनवायरनमेंट में डिप्लॉय करते समय, साइज़ कम करने के लिए, आप केवल आवश्यक डिपेंडेंसीज़ इंस्टॉल कर सकते हैं:

```bash
yarn install --production
```

:::

## 4. NocoBase इंस्टॉल करें

```bash
yarn nocobase install --lang=en-US
```

इंस्टॉलेशन प्रक्रिया स्वचालित रूप से Collection संरचना बनाएगी और डेटा को इनिशियलाइज़ करेगी।

## 5. NocoBase शुरू करें

**डेवलपमेंट एनवायरनमेंट (Development Environment)**

```bash
yarn dev
```

**प्रोडक्शन एनवायरनमेंट (Production Environment)**

```bash
yarn start
```

## 6. NocoBase में लॉग इन करें

अपने ब्राउज़र में [http://localhost:13000](http://localhost:13000) खोलें। प्रारंभिक अकाउंट और पासवर्ड `admin@nocobase.com` और `admin123` हैं।

:::warning अकाउंट सुरक्षा सुझाव

पहली बार लॉग इन करने के बाद, सिस्टम की सुरक्षा सुनिश्चित करने के लिए कृपया डिफ़ॉल्ट पासवर्ड तुरंत बदल दें।

:::
