# NocoBase सुरक्षा गाइड

NocoBase ने अपने कार्यात्मक डिज़ाइन से लेकर सिस्टम कार्यान्वयन तक, डेटा और एप्लिकेशन की सुरक्षा पर विशेष ध्यान दिया है। प्लेटफ़ॉर्म में उपयोगकर्ता प्रमाणीकरण, एक्सेस कंट्रोल और डेटा एन्क्रिप्शन जैसी कई सुरक्षा सुविधाएँ अंतर्निहित हैं, साथ ही यह वास्तविक आवश्यकताओं के अनुसार सुरक्षा नीतियों के लचीले कॉन्फ़िगरेशन की भी अनुमति देता है। चाहे उपयोगकर्ता डेटा की सुरक्षा करनी हो, एक्सेस अनुमतियों का प्रबंधन करना हो, या विकास और उत्पादन वातावरण को अलग करना हो, NocoBase व्यावहारिक उपकरण और समाधान प्रदान करता है। इस गाइड का उद्देश्य NocoBase के सुरक्षित उपयोग के लिए मार्गदर्शन प्रदान करना है, जिससे उपयोगकर्ताओं को डेटा, एप्लिकेशन और वातावरण की सुरक्षा करने में मदद मिल सके, और उपयोगकर्ता सुरक्षा सुनिश्चित करते हुए सिस्टम सुविधाओं का कुशलता से उपयोग किया जा सके।

## उपयोगकर्ता प्रमाणीकरण

उपयोगकर्ता प्रमाणीकरण का उपयोग उपयोगकर्ता की पहचान करने, उपयोगकर्ताओं को अनधिकृत रूप से सिस्टम में प्रवेश करने से रोकने और यह सुनिश्चित करने के लिए किया जाता है कि उपयोगकर्ता की पहचान का दुरुपयोग न हो।

### टोकन कुंजी

डिफ़ॉल्ट रूप से, NocoBase सर्वर-साइड API के प्रमाणीकरण के लिए JWT (JSON Web Token) का उपयोग करता है। उपयोगकर्ता सिस्टम एनवायरनमेंट वेरिएबल `APP_KEY` के माध्यम से टोकन की कुंजी सेट कर सकते हैं। कृपया एप्लिकेशन की टोकन कुंजी का ठीक से प्रबंधन करें, ताकि यह बाहर लीक न हो। ध्यान दें कि यदि `APP_KEY` को संशोधित किया जाता है, तो पुराने टोकन भी अमान्य हो जाएंगे।
