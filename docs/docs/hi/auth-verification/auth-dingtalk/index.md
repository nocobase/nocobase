# प्रमाणीकरण: डिंगटॉक

## परिचय

प्रमाणीकरण: डिंगटॉक प्लगइन उपयोगकर्ताओं को अपने डिंगटॉक खातों का उपयोग करके NocoBase में लॉग इन करने की सुविधा देता है।

## प्लगइन सक्रिय करें

![](https://static-docs.nocobase.com/202406120929356.png)

## डिंगटॉक डेवलपर कंसोल में API अनुमतियों के लिए आवेदन करें

संदर्भ के लिए <a href="https://open.dingtalk.com/document/orgapp/tutorial-obtaining-user-personal-information" target="_blank">डिंगटॉक ओपन प्लेटफ़ॉर्म - थर्ड-पार्टी वेबसाइटों में लॉग इन करें</a> देखें, और एक एप्लिकेशन बनाएँ।

एप्लिकेशन प्रबंधन कंसोल में जाएँ और "व्यक्तिगत फ़ोन नंबर जानकारी" और "एड्रेस बुक व्यक्तिगत जानकारी पढ़ने की अनुमति" को सक्षम करें।

![](https://static-docs.nocobase.com/202406120006620.png)

## डिंगटॉक डेवलपर कंसोल से क्रेडेंशियल प्राप्त करें

क्लाइंट आईडी (Client ID) और क्लाइंट सीक्रेट (Client Secret) कॉपी करें।

![](https://static-docs.nocobase.com/202406120000595.png)

## NocoBase में डिंगटॉक प्रमाणीकरण जोड़ें

उपयोगकर्ता प्रमाणीकरण प्लगइन प्रबंधन पृष्ठ पर जाएँ।

![](https://static-docs.nocobase.com/202406112348051.png)

जोड़ें - डिंगटॉक

![](https://static-docs.nocobase.com/202406112349664.png)

### कॉन्फ़िगरेशन

![](https://static-docs.nocobase.com/202406120016896.png)

- Sign up automatically when the user does not exist - जब फ़ोन नंबर से कोई मौजूदा उपयोगकर्ता नहीं मिलता है, तो क्या स्वचालित रूप से एक नया उपयोगकर्ता बनाना है।
- क्लाइंट आईडी (Client ID) और क्लाइंट सीक्रेट (Client Secret) - पिछले चरण में कॉपी की गई जानकारी भरें।
- रीडायरेक्ट यूआरएल (Redirect URL) - कॉलबैक यूआरएल (Callback URL) है, इसे कॉपी करें और अगले चरण पर आगे बढ़ें।

## डिंगटॉक डेवलपर कंसोल में कॉलबैक यूआरएल (Callback URL) कॉन्फ़िगर करें

कॉपी किए गए कॉलबैक यूआरएल (Callback URL) को डिंगटॉक डेवलपर कंसोल में पेस्ट करें।

![](https://static-docs.nocobase.com/202406120012221.png)

## लॉग इन करें

लॉगिन पेज पर जाएँ और थर्ड-पार्टी लॉगिन शुरू करने के लिए लॉगिन फ़ॉर्म के नीचे दिए गए बटन पर क्लिक करें।

![](https://static-docs.nocobase.com/202406120014539.png)
