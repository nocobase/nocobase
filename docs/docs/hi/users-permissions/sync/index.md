# उपयोगकर्ता डेटा सिंक्रनाइज़ेशन

## परिचय

आप उपयोगकर्ता डेटा सिंक्रनाइज़ेशन के स्रोतों को रजिस्टर और मैनेज कर सकते हैं। डिफ़ॉल्ट रूप से, एक HTTP API उपलब्ध कराया गया है, और प्लगइन के माध्यम से अन्य डेटा स्रोतों को भी जोड़ा जा सकता है। यह डिफ़ॉल्ट रूप से **उपयोगकर्ता** और **विभाग** संग्रह में डेटा सिंक करने का समर्थन करता है, और प्लगइन का उपयोग करके अन्य लक्ष्य संसाधनों तक सिंक्रनाइज़ेशन का विस्तार भी किया जा सकता है।

## डेटा स्रोत प्रबंधन और डेटा सिंक्रनाइज़ेशन

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
यदि उपयोगकर्ता डेटा सिंक्रनाइज़ेशन स्रोत प्रदान करने वाले कोई प्लगइन इंस्टॉल नहीं किए गए हैं, तो उपयोगकर्ता डेटा को HTTP API का उपयोग करके सिंक्रनाइज़ किया जा सकता है। [डेटा स्रोत - HTTP API](./sources/api.md) देखें।
:::

## डेटा स्रोत जोड़ना

उपयोगकर्ता डेटा सिंक्रनाइज़ेशन स्रोत प्रदान करने वाला प्लगइन इंस्टॉल करने के बाद, आप संबंधित डेटा स्रोत जोड़ सकते हैं। केवल सक्षम डेटा स्रोतों पर ही 'सिंक' और 'कार्य' बटन दिखाई देंगे।

> उदाहरण के लिए: वीकॉम (WeCom)

![](https://static-docs.nocobase.com/202412041053785.png)

## डेटा सिंक्रनाइज़ करना

डेटा सिंक्रनाइज़ेशन शुरू करने के लिए 'सिंक' बटन पर क्लिक करें।

![](https://static-docs.nocobase.com/202412041055022.png)

सिंक्रनाइज़ेशन स्थिति देखने के लिए 'कार्य' बटन पर क्लिक करें। सफल सिंक्रनाइज़ेशन के बाद, आप उपयोगकर्ता और विभाग सूचियों में डेटा देख सकते हैं।

![](https://static-docs.nocobase.com/202412041202337.png)

सिंक्रनाइज़ेशन में विफल रहे कार्यों के लिए, आप 'पुनः प्रयास करें' पर क्लिक कर सकते हैं।

![](https://static-docs.nocobase.com/202412041058337.png)

सिंक्रनाइज़ेशन विफल होने की स्थिति में, आप सिस्टम लॉग के माध्यम से समस्या का निवारण कर सकते हैं। इसके अतिरिक्त, मूल सिंक्रनाइज़ेशन रिकॉर्ड एप्लिकेशन लॉग फ़ोल्डर के अंतर्गत `user-data-sync` डायरेक्टरी में सहेजे जाते हैं।

![](https://static-docs.nocobase.com/202412041205655.png)
