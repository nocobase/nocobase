# Bloque de lista

## Introducción

El bloque de lista muestra los datos en formato de lista, lo que lo hace ideal para escenarios como listas de tareas, noticias o información de productos.

## Configuración del bloque

![20251023202835](https://static-docs.nocobase.com/20251023202835.png)

### Establecer el alcance de los datos

Como se muestra: filtre los pedidos con estado 'Cancelado'.

![20251023202927](https://static-docs.nocobase.com/20251023202927.png)

Para más detalles, consulte [Establecer el alcance de los datos](/interface-builder/blocks/block-settings/data-scope)

### Establecer reglas de ordenación

Como se muestra: ordene por el importe del pedido en orden descendente.

![20251023203022](https://static-docs.nocobase.com/20251023203022.png)

Para más detalles, consulte [Establecer reglas de ordenación](/interface-builder/blocks/block-settings/sorting-rule)

## Configurar campos

### Campos de esta colección

> **Nota**: Los campos de las colecciones heredadas (es decir, las colecciones padre) se fusionan y se muestran automáticamente en la lista de campos actual.

![20251023203103](https://static-docs.nocobase.com/20251023203103.png)

### Campos de colecciones relacionadas

> **Nota**: Se pueden mostrar campos de colecciones relacionadas (actualmente solo se admiten relaciones de uno a uno).

![20251023203611](https://static-docs.nocobase.com/20251023203611.png)

Para la configuración de campos de lista, consulte [Campo de detalles](/interface-builder/fields/generic/detail-form-item)
