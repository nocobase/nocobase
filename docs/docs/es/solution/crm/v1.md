# CRM heredado (v1)

> **Solo para vista previa**: Esta versión es funcionalmente completa, pero el diseño general de las tablas de datos no es óptimo, carece de soporte para localización multi-idioma y existe un acoplamiento entre las colecciones de CRM y las de gestión de proyectos. Estos problemas motivaron el rediseño de CRM 2.0. Actualmente se recomienda utilizar la [versión CRM 2.0](./index.md), que cuenta con una estructura de datos más racional, es más fácil de extender y modificar, y ya dispone de localización multi-idioma completa.

## Características de la versión

El CRM heredado incluye los siguientes módulos funcionales:

- **Gestión de clientes**: Información de clientes, contactos y clasificación de clientes.
- **Canal de ventas**: Gestión de leads, seguimiento de oportunidades y embudo de ventas.
- **Gestión de pedidos**: Cotizaciones, pedidos y contratos.
- **Gestión de proyectos** (Aún no incluida en la versión v2).
- **Comunicación y colaboración**: Comunicación por correo electrónico, comentarios y flujos de trabajo de aprobación.
- **Análisis de datos**: Tableros, informes y visualización de datos.

## Cómo instalar

> La versión actual se despliega mediante un proceso de respaldo y restauración.

Antes de comenzar, asegúrese de que:

- Usted ya cuenta con un entorno de ejecución básico de NocoBase. Para la instalación del sistema principal, consulte la [documentación oficial de instalación](https://docs-cn.nocobase.com/welcome/getting-started/installation) detallada.
- La versión de NocoBase es **v2.1.0-beta.2 o superior**.
- Ha descargado los archivos correspondientes de CRM v1:
  - **Archivo de respaldo**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — Aplicable al Método 1.
  - **Archivo SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — Aplicable al Método 2.

**Notas importantes**:
- Esta solución se basa en la base de datos **PostgreSQL 16**; asegúrese de que su entorno utilice PostgreSQL 16.
- **DB_UNDERSCORED no debe ser true**: Verifique su archivo `docker-compose.yml` y asegúrese de que la variable de entorno `DB_UNDERSCORED` no esté establecida en `true`. De lo contrario, entrará en conflicto con el respaldo y la restauración fallará.

## Método 1: Restaurar usando el Administrador de respaldos (Recomendado para usuarios Pro/Enterprise)

Este método utiliza el plugin integrado "**Administrador de respaldos**" (Pro/Enterprise) para realizar una restauración con un solo clic, siendo el enfoque más sencillo.

### Pasos a seguir

**Paso 1: Activar el plugin "Administrador de respaldos"**

1. Inicie sesión en su sistema NocoBase.
2. Diríjase a **`Gestión de plugins`**.
3. Busque y active el plugin **`Administrador de respaldos`**.

**Paso 2: Restaurar desde un archivo de respaldo local**

1. Tras activar el plugin, actualice la página.
2. Acceda al menú de la izquierda en **`Administración del sistema`** -> **`Administrador de respaldos`**.
3. Haga clic en el botón **`Restaurar desde respaldo local`** en la esquina superior derecha.
4. Arrastre el archivo de respaldo descargado al área de carga.
5. Haga clic en **`Enviar`** y espere pacientemente a que el sistema complete la restauración.

### Notas

* **Exclusivo para versiones Pro/Enterprise**: El "Administrador de respaldos" es un plugin de nivel empresarial y solo está disponible para usuarios Pro/Enterprise.
* **Compatibilidad de plugins comerciales**: Asegúrese de haber obtenido y activado los plugins comerciales necesarios para esta solución.

## Método 2: Importar archivo SQL directamente (General)

Este método restaura los datos operando directamente sobre la base de datos y es aplicable a todos los usuarios de NocoBase.

### Pasos a seguir

**Paso 1: Preparar una base de datos limpia**

Prepare una base de datos nueva y vacía para los datos que va a importar.

**Paso 2: Importar el archivo `.sql` a la base de datos**

* **Mediante línea de comandos (Ejemplo con Docker)**:

  ```bash
  # Copiar el archivo sql al contenedor
  docker cp nocobase_crm_v1_sql_260224.sql mi-contenedor-db:/tmp/
  # Entrar al contenedor y ejecutar el comando de importación
  docker exec -it mi-contenedor-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Mediante un cliente de base de datos**: Utilice herramientas como DBeaver, Navicat o pgAdmin para conectarse a la base de datos y ejecutar el archivo SQL.

**Paso 3: Conectar a la base de datos e iniciar la aplicación**

Configure los parámetros de inicio de NocoBase para que apunten a la base de datos donde se importaron los datos y, a continuación, inicie el servicio.
