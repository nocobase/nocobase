# Caddy

Si hoy quieres configurar el proxy de producción para una aplicación NocoBase, lo mejor es empezar por [Reverse proxy en producción](../../../quickstart/production/reverse-proxy/index.md) y después por la página [Caddy](../../../quickstart/production/reverse-proxy/caddy.md).

Esta sección antigua servía principalmente como punto de entrada para el proxy de recursos estáticos. La documentación actual se reorganizó alrededor de `nb proxy caddy`, que cubre de forma unificada la generación de configuración, la ejecución local o con Docker, la entrada HTTPS y las rutas `uploads`, `dist`, `api`, `ws` y SPA.
