# Instalacja z kodu źródłowego Git

:::tip Przede wszystkim

- Zainstalowano Git, Node.js 20+, Yarn 1.22.x
- Skonfigurowano i uruchomiono jedną z wymaganych baz danych: MySQL 8.0.17+, MariaDB 10.9+, PostgreSQL 10+
- Ze względu na środowisko sieciowe, sugerujemy zmianę na krajowe repozytorium lustrzane (opcjonalnie)

```bash
$ yarn config set disable-self-update-check true
$ yarn config set registry https://registry.npmmirror.com/
```

:::

## 1. Pobieranie NocoBase na lokalny komputer

Proszę wybrać wersję NocoBase ([Porównanie wersji](../quickstart.md)) i wykonać odpowiednie polecenie.

<Tabs>
<Tab label="Wersja Latest">

```bash
git clone https://github.com/nocobase/nocobase.git -b main --depth=1 my-nocobase
```

</Tab>
<Tab label="Wersja Beta">

```bash
git clone https://github.com/nocobase/nocobase.git -b next --depth=1 my-nocobase
```

</Tab>
<Tab label="Wersja Alpha">

```bash
git clone https://github.com/nocobase/nocobase.git -b develop --depth=1 my-nocobase
```

</Tab>
</Tabs>

## 2. Przejście do katalogu

```bash
cd my-nocobase
```

## 3. Instalacja zależności

📢 Ze względu na czynniki takie jak środowisko sieciowe i konfiguracja systemu, ten krok może potrwać kilkanaście minut.

```bash
yarn install --frozen-lockfile
```

## 4. Ustawianie zmiennych środowiskowych

Zmienne środowiskowe wymagane przez NocoBase są przechowywane w pliku `.env` w katalogu głównym. Proszę zmodyfikować zmienne środowiskowe zgodnie z Państwa rzeczywistą sytuacją.

```bash
TZ=Europe/Warsaw
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning Opis zmiennych środowiskowych

- `TZ` służy do ustawiania strefy czasowej aplikacji, domyślnie jest to strefa czasowa systemu operacyjnego;
- `APP_KEY` to klucz tajny aplikacji, używany do generowania tokenów użytkowników itp. (jeśli `APP_KEY` zostanie zmieniony, stare tokeny również stracą ważność). Może to być dowolny losowy ciąg znaków. Proszę zmienić go na własny klucz i upewnić się, że nie zostanie on ujawniony publicznie;
- `DB_*` dotyczy bazy danych. Jeśli nie jest to domyślna usługa bazy danych z przykładu, proszę zmodyfikować ją zgodnie z rzeczywistą sytuacją.

:::

## 5. Instalacja NocoBase

```bash
yarn nocobase install --lang=pl-PL
```

## 6. Uruchamianie NocoBase


**Środowisko deweloperskie**

```bash
yarn dev
```

**Środowisko produkcyjne (niezalecane)**

Nie zaleca się wdrażania bezpośrednio z kodu źródłowego w środowisku produkcyjnym. W przypadku środowiska produkcyjnego proszę zapoznać się z sekcją [Wdrożenie produkcyjne](../deployment/production.md).

```bash
yarn build
yarn start
```

## 7. Logowanie do NocoBase

Proszę otworzyć w przeglądarce adres [http://localhost:13000](http://localhost:13000). Inicjalny login i hasło to `admin@nocobase.com` oraz `admin123`.

:::warning Wskazówka dotycząca bezpieczeństwa konta

Po pierwszym zalogowaniu proszę niezwłocznie zmienić domyślne hasło, aby zapewnić bezpieczeństwo systemu.

:::
