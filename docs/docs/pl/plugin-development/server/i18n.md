# I18n

W wtyczkach NocoBase wspieramy internacjonalizację (i18n) dla frontendu i backendu. Dzięki ujednoliconemu mechanizmowi mogą Państwo z łatwością zaimplementować wielojęzyczną zawartość w swoich wtyczkach.

## Zarządzanie plikami wielojęzycznymi

Pliki wielojęzyczne wtyczek są przechowywane w katalogu `src/locale`. Zalecamy nazywanie ich zgodnie z kodem języka, na przykład:

```bash
|- /plugin-hello
  |- /src
    |- /locale
      |- en-US.json   # Język angielski
      |- zh-CN.json   # Język chiński
```

Każdy plik językowy eksportuje obiekt JSON zawierający wszystkie wpisy tłumaczeń dla danego języka, na przykład:

```json
// zh-CN.json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

```json
// en-US.json
{
  "Hello": "Hello",
  "World": "World",
  "Enter your name": "Enter your name",
  "Your name is {{name}}": "Your name is {{name}}"
}
```

> Przy pierwszym dodaniu plików językowych należy ponownie uruchomić aplikację, aby zmiany zostały zastosowane. Mogą Państwo zweryfikować wpisy tłumaczeń za pomocą API:  
> http://localhost:13000/api/app:getLang?locale=zh-CN

## Globalna instancja i18n

`app.i18n` to globalna instancja i18n, odpowiednia dla scenariuszy CLI lub globalnych wtyczek. Można ją połączyć z `inquirer` w celu zaimplementowania interakcji w wierszu poleceń:

```ts
import select from '@inquirer/select';
import input from '@inquirer/input';

export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.command('test-i18n').action(async () => {
      const answer1 = await select({
        message: 'Select a language',
        choices: [
          { name: '中文', value: 'zh-CN' },
          { name: 'English', value: 'en-US' }
        ]
      });

      await this.app.changeLanguage(answer1);

      const answer2 = await input({
        message: app.i18n.t('Enter your name')
      });

      console.log(app.i18n.t('Your name is {{name}}', { name: answer2 }));
    });
  }
}
```

> `app.i18n.t(text, options)` służy do tłumaczenia tekstu i obsługuje zmienne szablonów.

## i18n w kontekście żądania

Każde żądanie posiada `ctx.i18n`, które jest sklonowaną instancją globalnego i18n. Odpowiada ono niezależnie z informacjami wielojęzycznymi, bazując na języku klienta.

Ustawianie języka klienta

- **Parametr zapytania (Query String)**:

```bash
GET /?locale=en-US HTTP/1.1
Host: localhost:13000
```

- **Nagłówek żądania (Request Header) (zalecane)**:

```bash
GET / HTTP/1.1
Host: localhost:13000
X-Locale: en-US
```

Użycie w middleware

```ts
export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.use(async (ctx, next) => {
      if (ctx.path === '/api/test-i18n') {
        ctx.body = ctx.t('Hello', { ns: '@my-project/plugin-hello' });
      }
      await next();
    });
  }
}
```

Dostęp do http://localhost:13000/api/test-i18n?locale=zh-CN zwróci `你好`.

## i18n wewnętrznie w wtyczce

Wtyczki mogą bezpośrednio używać `plugin.t(key, options)` do pobierania tłumaczeń:

```ts
export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.use(async (ctx, next) => {
      if (ctx.path === '/api/plugin-i18n') {
        ctx.body = this.plugin.t('Hello');
      }
      await next();
    });
  }
}
```

> `plugin.t(text)` jest równoważne z `ctx.t(text, { ns })`

## Powiązane API

- app.i18n
- app.t(text, options)
- ctx.i18n
- ctx.t(text, options)
- plugin.t()
- tExpr(text, options)
