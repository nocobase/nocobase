# I18n Internacjonalizacja

W wtyczkach NocoBase wspierana jest wielojęzyczna internacjonalizacja (i18n) zarówno dla frontendu, jak i backendu. Dzięki jednolitemu mechanizmowi mogą Państwo z łatwością zaimplementować wielojęzyczną zawartość w swoich wtyczkach.

## Zarządzanie plikami wielojęzycznymi

Pliki wielojęzyczne wtyczek są przechowywane w jednolity sposób w katalogu `src/locale`. Zaleca się nazywanie ich według języka, na przykład:

```bash
|- /plugin-hello
  |- /src
    |- /locale
      |- en-US.json   # Język angielski
      |- zh-CN.json   # Język chiński
```

Każdy plik językowy eksportuje obiekt JSON zawierający wszystkie wpisy tłumaczeń dla danego języka, na przykład:

```json
// zh-CN.json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

```json
// en-US.json
{
  "Hello": "Hello",
  "World": "World",
  "Enter your name": "Enter your name",
  "Your name is {{name}}": "Your name is {{name}}"
}
```

> Przy pierwszym dodaniu plików językowych należy ponownie uruchomić aplikację, aby zmiany weszły w życie. Mogą Państwo zweryfikować, czy wpisy tłumaczeń zostały zastosowane, za pośrednictwem API:  
> http://localhost:13000/api/app:getLang?locale=zh-CN
