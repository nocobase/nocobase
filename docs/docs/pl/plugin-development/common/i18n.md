# Internacjonalizacja

## Pliki internacjonalizacyjne

W wtyczkach, pliki wielojęzyczne zarówno dla frontendu, jak i backendu, są przechowywane w folderze `src/locale`.

```bash
|- /plugin-i18n
  |- /src
    |- /locale      # Folder z plikami językowymi
      |- en-US.ts   # Język angielski
      |- zh-CN.ts   # Język chiński
```

Wystarczy dodać wpisy tłumaczeń w odpowiednich plikach wielojęzycznych (`/src/locale/${lang}.ts`). Jeśli dodają Państwo plik wielojęzyczny po raz pierwszy, konieczne będzie ponowne uruchomienie aplikacji, aby zmiany weszły w życie. Mogą Państwo sprawdzić API `app:getLang`, aby zweryfikować, czy wpisy tłumaczeń zostały pomyślnie dodane.

Domyślny adres URL: http://localhost:13000/api/app:getLang?locale=zh-CN

## Serwer

### app.i18n <Badge>server</Badge>

app.i18n to globalna instancja i18n, zazwyczaj używana w interfejsie wiersza poleceń (CLI). Na przykład, w połączeniu z `inquirer` do implementacji interakcji w wierszu poleceń.

```ts
import select from '@inquirer/select';
import input from '@inquirer/input';

export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.command('test-i18n').action(async () => {
      const answer1 = await select({
        message: 'Wybierz język',
        choices: [
          {
            name: 'Chiński',
            value: 'zh-CN',
          },
          {
            name: 'Angielski',
            value: 'en-US',
          },
        ],
      });
      await this.app.changeLanguage(answer1);
      const answer2 = await input({
        message: app.i18n.t('Proszę podać swoje imię'),
      });
      console.log(app.i18n.t(`Pańskie imię to {{name}}`, { name: answer2 }));
    });
  }
}
```

### ctx.i18n <Badge>server</Badge>

ctx.i18n to sklonowana instancja globalnego app.i18n. Kontekst (ctx) każdego żądania jest całkowicie niezależny i odpowiada informacjami wielojęzycznymi na podstawie języka klienta.

Parametry żądania klienta można umieścić w ciągu zapytania (query string)

```bash
GET /?locale=en-US HTTP/1.1
Host: localhost:13000
```

Lub w nagłówkach żądania (zalecane)

```bash
GET / HTTP/1.1
Host: localhost:13000
X-Locale: en-US
```

Przykład

```ts
export class PluginSampleI18nServer extends Plugin {
  load() {
    this.app.use(async (ctx, next) => {
      if (ctx.path === '/api/test-i18n') {
        ctx.body = `${ctx.i18n.t('Witaj')} ${ctx.i18n.t('Świecie')}`;
      }
      await next();
    });
  }
}
```

Proszę sprawdzić http://localhost:13000/api/test-i18n?locale=zh-CN
