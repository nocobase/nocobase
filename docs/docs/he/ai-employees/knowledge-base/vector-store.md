# אחסון וקטורי

## מבוא

במאגר ידע, בעת שמירת מסמכים, המסמכים עוברים וקטוריזציה. באופן דומה, בעת שליפת מסמכים, מונחי החיפוש עוברים וקטוריזציה. שני התהליכים הללו דורשים שימוש ב-`Embedding model` כדי לבצע וקטוריזציה לטקסט המקורי.

בתוסף מאגר הידע של AI, אחסון וקטורי הוא הקישור בין `Embedding model` לבין מסד נתונים וקטורי.

## ניהול אחסון וקטורי

עברו לדף התצורה של תוסף עובדי AI, לחצו על לשונית `Vector store`, ובחרו ב-`Vector store` כדי להיכנס לדף ניהול אחסון וקטורי.

![20251023003023](https://static-docs.nocobase.com/20251023003023.png)

לחצו על כפתור `Add new` בפינה הימנית העליונה כדי להוסיף אחסון וקטורי חדש:

- בשדה הקלט `Name`, הזינו את שם האחסון הווקטורי;
- ב-`Vector store`, בחרו מסד נתונים וקטורי שכבר הוגדר. ראו: [מסד נתונים וקטורי](/ai-employees/knowledge-base/vector-database);
- ב-`LLM service`, בחרו שירות LLM שכבר הוגדר. ראו: [ניהול שירותי LLM](/ai-employees/quick-start/llm-service);
- בשדה הקלט `Embedding model`, הזינו את שם מודל ה-`Embedding` לשימוש;
  
לחצו על כפתור `Submit` כדי לשמור את פרטי האחסון הווקטורי.
