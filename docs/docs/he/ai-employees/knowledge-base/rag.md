# אחזור RAG

## מבוא

לאחר שתגדירו את מאגר הידע, תוכלו להפעיל את תכונת ה-RAG בהגדרות עובד ה-AI.

כאשר RAG מופעל, כשמשתמש משוחח עם עובד AI, עובד ה-AI ישתמש באחזור RAG כדי לשלוף מסמכים ממאגר הידע בהתבסס על הודעת המשתמש, וישיב על בסיס המסמכים שאוחזרו.

## הפעלת RAG

עברו לדף הגדרות ה-תוסף של עובד ה-AI, לחצו על לשונית `AI employees` כדי להיכנס לדף ניהול עובדי ה-AI.

![20251023010811](https://static-docs.nocobase.com/20251023010811.png)

בחרו את עובד ה-AI שעבורו תרצו להפעיל RAG, לחצו על כפתור `Edit` כדי להיכנס לדף עריכת עובד ה-AI.

בלשונית `Knowledge base`, הפעילו את מתג `Enable`.

- בשדה `Knowledge Base Prompt`, הזינו את ההנחיה (prompt) להתייחסות למאגר הידע. `{knowledgeBaseData}` הוא מציין מיקום קבוע ואסור לשנותו.
- בשדה `Knowledge Base`, בחרו את מאגר הידע שהוגדר. ראו: [מאגר ידע](/ai-employees/knowledge-base/knowledge-base).
- בתיבת הקלט `Top K`, הזינו את מספר המסמכים לאחזור. ברירת המחדל היא 3.
- בתיבת הקלט `Score`, הזינו את סף הרלוונטיות של המסמכים לאחזור.

לחצו על כפתור `Submit` כדי לשמור את הגדרות עובד ה-AI.
