# סקירה כללית

![clipboard-image-1771905619](https://static-docs.nocobase.com/clipboard-image-1771905619.png)

עובדי AI‏ (`AI Employees`) הם יכולות סוכנים חכמים המשולבים עמוק במערכות העסקיות של NocoBase.

הם אינם רובוטים ש"רק יודעים לצ'וטט", אלא "עמיתים דיגיטליים" שיכולים להבין את ההקשר ישירות בממשק העסקי ולבצע פעולות:

- **הבנת הקשר עסקי**: זיהוי הדף הנוכחי, הבלוקים, מבנה הנתונים והתוכן שנבחר.
- **יכולת ביצוע פעולות ישירה**: קריאה למיומנויות לביצוע שאילתות, ניתוח, מילוי, הגדרה, יצירה ומשימות נוספות.
- **שיתוף פעולה מבוסס תפקידים**: הגדרת עובדים שונים לפי תפקיד, והחלפת מודלים במהלך השיחה לצורך שיתוף פעולה.

## מסלול התחלה ב-5 דקות

תחילה עיינו ב-[תחילת עבודה מהירה](/ai-employees/quick-start), והשלימו את התצורה המינימלית הנדרשת לפי הסדר הבא:

1. הגדרת לפחות [שירות LLM](/ai-employees/features/llm-service) אחד.
2. הפעלת לפחות [עובד AI](/ai-employees/features/enable-ai-employee) אחד.
3. פתיחת שיחה ותחילת [שיתוף פעולה עם עובדי AI](/ai-employees/features/collaborate).
4. הפעלת [חיפוש באינטרנט](/ai-employees/features/web-search) ו-[משימות מהירות](/ai-employees/features/task) לפי הצורך.

## מפת תכונות

### א. תצורה בסיסית (מנהל מערכת)

- [הגדרת שירות LLM](/ai-employees/features/llm-service): חיבור לספקים (Providers), הגדרה וניהול של מודלים זמינים.
- [הפעלת עובדי AI](/ai-employees/features/enable-ai-employee): הפעלה והשבתה של עובדים מובנים, ושליטה בטווח הזמינות.
- [יצירת עובד AI חדש](/ai-employees/features/new-ai-employees): הגדרת תפקיד, אישיות, הודעת פתיחה וגבולות יכולת.
- [שימוש במיומנויות](/ai-employees/features/tool): הגדרת הרשאות מיומנות (`Ask` / `Allow`), ושליטה בסיכוני ביצוע.

### ב. שיתוף פעולה יומיומי (משתמשים עסקיים)

- [שיתוף פעולה עם עובדי AI](/ai-employees/features/collaborate): החלפת עובדים ומודלים בתוך השיחה, ושיתוף פעולה מתמשך.
- [הוספת הקשר - בלוקים](/ai-employees/features/pick-block): שליחת בלוקים מהדף כהקשר ל-AI.
- [משימות מהירות](/ai-employees/features/task): הגדרת משימות נפוצות מראש בדפים/בלוקים, וביצוען בלחיצה אחת.
- [חיפוש באינטרנט](/ai-employees/features/web-search): הפעלת אחזור מידע עדכני כאשר נדרש מידע חדש.

### ג. יכולות מתקדמות (הרחבות)

- [עובדי AI מובנים](/ai-employees/features/built-in-employee): הכרת המיצוי והתרחישים המתאימים לעובדים המוגדרים מראש.
- [בקרת הרשאות](/ai-employees/permission): שליטה בגישה לעובדים, מיומנויות ונתונים לפי מודל ההרשאות הארגוני.
- [מאגר ידע AI](/ai-employees/knowledge-base/index): הכנסת ידע ארגוני לשיפור יציבות התשובות ויכולת המעקב.
- [צומת LLM בתהליך עבודה](/ai-employees/workflow/nodes/llm/chat): שילוב יכולות AI בתוך תהליכי עבודה אוטומטיים.

## מושגי ליבה (מומלץ ליישר קו תחילה)

המונחים הבאים תואמים למילון המונחים, מומלץ להשתמש בהם באופן אחיד בצוות:

- **עובד AI‏ (AI Employee)**: סוכן בר-ביצוע המורכב מהגדרת תפקיד (Role setting) ומיומנויות (Tool / Skill).
- **שירות LLM‏ (LLM Service)**: יחידת חיבור מודלים ותצורת יכולות, המשמשת לניהול ספקים (Providers) ורשימות מודלים.
- **ספק (Provider)**: ספק המודלים שמאחורי שירות ה-LLM.
- **מודלים מופעלים (Enabled Models)**: אוסף המודלים ששירות ה-LLM הנוכחי מאפשר לבחור בשיחה.
- **בורר עובדי AI‏ (AI Employee Switcher)**: החלפת העובד המשתף פעולה בתוך השיחה.
- **בורר מודלים (Model Switcher)**: החלפת מודל בתוך השיחה, עם זיכרון העדפות לפי עובד.
- **מיומנות (Tool / Skill)**: יחידת יכולת ביצוע שה-AI יכול לקרוא לה.
- **הרשאת מיומנות (Permission: Ask / Allow)**: האם נדרש אישור אנושי לפני קריאה למיומנות.
- **הקשר (Context)**: מידע על הסביבה העסקית כגון דפים, בלוקים ומבני נתונים.
- **שיחה (Chat)**: תהליך אינטראקציה רציף אחד בין המשתמש לעובד ה-AI.
- **חיפוש באינטרנט (Web Search)**: יכולת לשיפור תשובות המבוססת על אחזור מידע חיצוני בזמן אמת.
- **מאגר ידע (Knowledge Base / RAG)**: יצירה מוגברת באחזור (RAG) המשלבת ידע ארגוני.
- **אחסון וקטורי (Vector Store)**: אחסון וקטורי המספק יכולות חיפוש סמנטי עבור מאגר הידע.

## הוראות התקנה

עובדי AI הם תוסף מובנה של NocoBase‏ (`@nocobase/plugin-ai`), מוכנים לשימוש ללא צורך בהתקנה נפרדת.
