# ctx.router

מופע של נתב (router) המבוסס על React Router, המשמש לניווט באמצעות קוד בתוך RunJS. בדרך כלל נעשה בו שימוש בשילוב עם `ctx.route` ו-`ctx.location`.

## מקרי שימוש

| תרחיש | תיאור |
|------|------|
| **JSBlock / JSField** | ניווט לדפי פירוט, דפי רשימה או קישורים חיצוניים לאחר לחיצה על כפתור. |
| **כללי קישור (Linkage Rules) / תזרים אירועים** | ביצוע `navigate` לרשימה או לדף פירוט לאחר שליחה מוצלחת, או העברת `state` לדף היעד. |
| **JSAction / טיפול באירועים** | ביצוע ניווט בנתיבים בתוך לוגיקה כגון שליחת טפסים או לחיצה על קישורים. |
| **ניווט בתצוגות** | עדכון ה-URL באמצעות `navigate` בזמן החלפת מחסנית תצוגות (view stack) פנימית. |

> הערה: `ctx.router` זמין רק בסביבות RunJS הכוללות הקשר של ניתוב (כגון JSBlock בתוך דף, דפי Flow, תזרימי אירועים וכו'); הוא עשוי להיות ריק (null) בהקשרים של צד-שרת בלבד או ללא ניתוב (כגון תהליכי עבודה).

## הגדרת סוג (Type Definition)

```typescript
router: Router
```

`Router` נגזר מ-`@remix-run/router`. ב-RunJS, פעולות ניווט כגון מעבר, חזרה אחורה ורענון מיושמות באמצעות `ctx.router.navigate()`.

## מתודות

### ctx.router.navigate()

ניווט לנתיב יעד, או ביצוע פעולת חזרה/רענון.

**חתימה:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**פרמטרים:**

- `to`: נתיב היעד (string), מיקום יחסי בהיסטוריה (number, למשל `-1` כדי לחזור אחורה), או `null` (כדי לרענן את הדף הנוכחי).
- `options`: הגדרות אופציונליות.
  - `replace?: boolean`: האם להחליף את הרשומה הנוכחית בהיסטוריה (ברירת המחדל היא `false`, מה שמוסיף רשומה חדשה - push).
  - `state?: any`: מצב (state) להעברה לנתיב היעד. נתונים אלו אינם מופיעים ב-URL וניתן לגשת אליהם דרך `ctx.location.state` בדף היעד. מתאים למידע רגיש, נתונים זמניים או מידע שאינו אמור להופיע ב-URL.

## דוגמאות

### ניווט בסיסי

```ts
// ניווט לרשימת משתמשים (מוסיף רשומה חדשה להיסטוריה, מאפשר חזרה אחורה)
ctx.router.navigate('/admin/users');

// ניווט לדף פירוט
ctx.router.navigate(`/admin/users/${recordId}`);
```

### החלפת היסטוריה (ללא הוספת רשומה חדשה)

```ts
// הפניה מחדש לדף הבית לאחר התחברות; המשתמש לא יחזור לדף ההתחברות בלחיצה על "חזור"
ctx.router.navigate('/admin', { replace: true });

// החלפת הדף הנוכחי בדף הפירוט לאחר שליחת טופס מוצלחת
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### העברת state

```ts
// העברת נתונים בזמן הניווט; דף היעד שולף אותם דרך ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### חזרה אחורה ורענון

```ts
// חזרה דף אחד אחורה
ctx.router.navigate(-1);

// חזרה שני דפים אחורה
ctx.router.navigate(-2);

// רענון הדף הנוכחי
ctx.router.navigate(null);
```

## הקשר בין ctx.route ל-ctx.location

| מטרה | שימוש מומלץ |
|------|----------|
| **ניווט / מעבר** | `ctx.router.navigate(path)` |
| **קריאת הנתיב הנוכחי** | `ctx.route.pathname` או `ctx.location.pathname` |
| **קריאת state שהועבר בניווט** | `ctx.location.state` |
| **קריאת פרמטרי ניתוב** | `ctx.route.params` |

`ctx.router` אחראי על "פעולות ניווט", בעוד ש-`ctx.route` ו-`ctx.location` אחראים על "מצב הניתוב הנוכחי".

## הערות

- `navigate(path)` מוסיף רשומה חדשה להיסטוריה כברירת מחדל, מה שמאפשר למשתמשים לחזור באמצעות כפתור ה"חזור" של הדפדפן.
- `replace: true` מחליף את הרשומה הנוכחית בהיסטוריה מבלי להוסיף חדשה, דבר המתאים לתרחישים כמו הפניה מחדש לאחר התחברות או ניווט לאחר שליחה מוצלחת.
- **בנוגע לפרמטר `state`**:
  - נתונים המועברים דרך `state` אינם מופיעים ב-URL, מה שהופך אותם למתאימים למידע רגיש או זמני.
  - ניתן לגשת אליהם דרך `ctx.location.state` בדף היעד.
  - ה-`state` נשמר בהיסטוריית הדפדפן ונותר נגיש במהלך ניווט קדימה/אחורה.
  - ה-`state` יאבד לאחר רענון קשיח (hard refresh) של הדף.

## נושאים קשורים

- [ctx.route](./route.md): מידע על התאמת הנתיב הנוכחי (pathname, params וכו').
- [ctx.location](./location.md): מיקום ה-URL הנוכחי (pathname, search, hash, state); ה-`state` נקרא כאן לאחר הניווט.
