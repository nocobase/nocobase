# ctx.exitAll()

מפסיק את זרימת האירועים (event flow) הנוכחית ואת כל זרימות האירועים העוקבות המופעלות באותו שיגור אירוע (event dispatch). משמש בדרך כלל כאשר יש צורך להפסיק מיד את כל זרימות האירועים תחת האירוע הנוכחי עקב שגיאה גלובלית או כשל באימות הרשאות.

## תרחישי שימוש

השימוש ב-`ctx.exitAll()` נעשה בדרך כלל בהקשרים של הרצת קוד JS, כאשר יש צורך **להפסיק בו-זמנית את זרימת האירועים הנוכחית ואת זרימות האירועים העוקבות שהופעלו על ידי אותו אירוע**:

| תרחיש | הסבר |
|------|------|
| **זרימת אירועים** | אימות זרימת האירועים הראשית נכשל (למשל, הרשאות חסרות), מה שמחייב את הפסקת הזרימה הראשית וכל זרימה עוקבת תחת אותו אירוע שטרם בוצעה. |
| **כללי קישור (Linkage)** | כאשר אימות הקישור נכשל, יש להפסיק את הקישור הנוכחי ואת הקישורים העוקבים המופעלים על ידי אותו אירוע. |
| **אירועי פעולה** | אימות טרום-פעולה נכשל (למשל, בדיקת הרשאות לפני מחיקה), מה שמחייב מניעה של הפעולה הראשית ושל הצעדים הבאים. |

> ההבדל מ-`ctx.exit()`: הפונקציה `ctx.exit()` מפסיקה רק את זרימת האירועים הנוכחית; `ctx.exitAll()` מפסיקה את זרימת האירועים הנוכחית ואת כל זרימות האירועים העוקבות **שטרם בוצעו** באותו שיגור אירוע.

## הגדרת סוג (Type Definition)

```ts
exitAll(): never;
```

קריאה ל-`ctx.exitAll()` זורקת חריגה פנימית מסוג `FlowExitAllException`, הנלכדת על ידי מנוע זרימת האירועים כדי לעצור את מופע זרימת האירועים הנוכחי ואת זרימות האירועים העוקבות תחת אותו אירוע. לאחר הקריאה, שאר הפקודות בקוד ה-JS הנוכחי לא יבוצעו.

## השוואה ל-ctx.exit()

| מתודה | טווח השפעה |
|------|----------|
| `ctx.exit()` | מפסיקה רק את זרימת האירועים הנוכחית; זרימות אירועים עוקבות אינן מושפעות. |
| `ctx.exitAll()` | מפסיקה את זרימת האירועים הנוכחית ומבטלת זרימות אירועים עוקבות המבוצעות **באופן סדרתי** תחת אותו אירוע. |

## הסבר על מצבי ביצוע

- **ביצוע סדרתי (sequential)**: זרימות אירועים תחת אותו אירוע מבוצעות לפי הסדר; לאחר שזרימת אירועים כלשהי קוראת ל-`ctx.exitAll()`, זרימות האירועים הבאות לא יבוצעו.
- **ביצוע מקבילי (parallel)**: זרימות אירועים תחת אותו אירוע מבוצעות במקביל; קריאה ל-`ctx.exitAll()` בזרימת אירועים אחת לא תקטע זרימות אירועים מקבילות אחרות (הן עצמאיות).

## דוגמאות

### הפסקת כל זרימות האירועים כאשר אימות ההרשאות נכשל

```ts
// הפסקת זרימת האירועים הראשית והזרימות העוקבות כאשר אין מספיק הרשאות
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'אין הרשאת פעולה' });
  ctx.exitAll();
}
```

### הפסקה כאשר אימות טרום-פעולה גלובלי נכשל

```ts
// דוגמה: אם נמצא לפני מחיקה שנתונים קשורים אינם ניתנים למחיקה, יש למנוע את זרימת האירועים הראשית ופעולות עוקבות
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('לא ניתן למחוק: קיימים נתונים קשורים');
  ctx.exitAll();
}
```

### בחירה בין ctx.exit() ל-ctx.exitAll()

```ts
// רק זרימת האירועים הנוכחית צריכה להיעצר -> השתמשו ב-ctx.exit()
if (!params.valid) {
  ctx.message.error('פרמטרים לא תקינים');
  ctx.exit();  // זרימות אירועים עוקבות לא יושפעו
}

// יש צורך להפסיק את כל זרימות האירועים העוקבות תחת האירוע הנוכחי -> השתמשו ב-ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'הרשאות חסרות' });
  ctx.exitAll();  // גם זרימת האירועים הראשית וגם זרימות עוקבות תחת אותו אירוע יופסקו
}
```

### הצגת הודעה לפני הפסקה

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('אנא תקנו תחילה את השגיאות בטופס');
  ctx.exitAll();
}
```

## הערות

- לאחר הקריאה ל-`ctx.exitAll()`, קוד עוקב ב-JS הנוכחי לא יבוצע; מומלץ להסביר למשתמש את הסיבה באמצעות `ctx.message`, `ctx.notification` או חלון מודאלי לפני הקריאה.
- בדרך כלל אין צורך ללכוד את `FlowExitAllException` בקוד העסקי; השאירו למנוע זרימת האירועים לטפל בכך.
- אם ברצונכם לעצור רק את זרימת האירועים הנוכחית מבלי להשפיע על הבאות, השתמשו ב-`ctx.exit()`.
- במצב מקבילי, `ctx.exitAll()` עוצר רק את זרימת האירועים הנוכחית ואינו קוטע זרימות אירועים מקבילות אחרות.
