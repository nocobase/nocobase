# ctx.exit()

מפסיק את הביצוע של תזרים האירועים (event flow) הנוכחי; הצעדים הבאים לא יופעלו. נעשה בו שימוש נפוץ כאשר תנאים עסקיים אינם מתקיימים, המשתמש מבטל את הפעולה, או כאשר מתרחשת שגיאה בלתי הפיכה.

## תרחישי שימוש

בדרך כלל משתמשים ב-`ctx.exit()` בהקשרים הבאים שבהם ניתן להריץ קוד JS:

| תרחיש | תיאור |
|------|------|
| **תזרים אירועים (Event Flow)** | בתזרימי אירועים המופעלים על ידי שליחת טפסים, לחיצות על כפתורים וכו', מפסיק את הצעדים הבאים כאשר התנאים אינם מתקיימים. |
| **כללי קישוריות (Linkage Rules)** | בקישוריות שדות, קישוריות מסננים וכו', מפסיק את תזרים האירועים הנוכחי כאשר האימות נכשל או כאשר יש צורך לדלג על הביצוע. |
| **אירועי פעולה (Action Events)** | בפעולות מותאמות אישית (למשל, אישור מחיקה, אימות לפני שמירה), יוצא כאשר המשתמש מבטל או כשהאימות נכשל. |

> ההבדל מ-`ctx.exitAll()`: הפונקציה `ctx.exit()` מפסיקה רק את תזרים האירועים הנוכחי; תזרימי אירועים אחרים תחת אותו אירוע אינם מושפעים. `ctx.exitAll()` מפסיקה את תזרים האירועים הנוכחי וגם את כל תזרימי האירועים הבאים תחת אותו אירוע שטרם בוצעו.

## הגדרת סוג (Type Definition)

```ts
exit(): never;
```

קריאה ל-`ctx.exit()` זורקת חריגה פנימית מסוג `FlowExitException`, אשר נתפסת על ידי מנוע התזרים (FlowEngine) כדי לעצור את ביצוע תזרים האירועים הנוכחי. לאחר הקריאה, שאר הפקודות בקוד ה-JS הנוכחי לא יבוצעו.

## השוואה ל-ctx.exitAll()

| מתודה | היקף ההשפעה |
|------|----------|
| `ctx.exit()` | מפסיקה רק את תזרים האירועים הנוכחי; תזרימי אירועים עוקבים אינם מושפעים. |
| `ctx.exitAll()` | מפסיקה את תזרים האירועים הנוכחי ומבטלת תזרימי אירועים עוקבים תחת אותו אירוע המוגדרים ל**ביצוע סדרתי**. |

## דוגמאות

### יציאה בעת ביטול על ידי המשתמש

```ts
// במודאל אישור, אם המשתמש לוחץ על ביטול, תזרים האירועים יופסק
if (!confirmed) {
  ctx.message.info('הפעולה בוטלה');
  ctx.exit();
}
```

### יציאה בעת כישלון באימות פרמטרים

```ts
// הצגת הודעה והפסקת הביצוע כאשר האימות נכשל
if (!params.value || params.value.length < 3) {
  ctx.message.error('פרמטרים לא תקינים, האורך חייב להיות לפחות 3');
  ctx.exit();
}
```

### יציאה כאשר תנאים עסקיים אינם מתקיימים

```ts
// הפסקת הביצוע אם התנאים אינם מתקיימים; הצעדים הבאים לא יבוצעו
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'ניתן להגיש טיוטות בלבד' });
  ctx.exit();
}
```

### בחירה בין ctx.exit() ל-ctx.exitAll()

```ts
// רק תזרים האירועים הנוכחי צריך להיעצר ← השתמשו ב-ctx.exit()
if (!params.valid) {
  ctx.message.error('פרמטרים לא תקינים');
  ctx.exit();  // תזרימי אירועים אחרים אינם מושפעים
}

// יש צורך להפסיק את כל תזרימי האירועים העוקבים תחת האירוע הנוכחי ← השתמשו ב-ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'הרשאות לא מספיקות' });
  ctx.exitAll();  // גם תזרים האירועים הנוכחי וגם תזרימי האירועים העוקבים תחת אותו אירוע יופסקו
}
```

### יציאה על סמך בחירת המשתמש לאחר אישור במודאל

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'אישור מחיקה',
  content: 'פעולה זו אינה ניתנת לביטול. האם ברצונך להמשיך?',
});
if (!ok) {
  ctx.message.info('בוטל');
  ctx.exit();
}
```

## הערות

- לאחר הקריאה ל-`ctx.exit()`, קוד המשך ב-JS הנוכחי לא יבוצע; מומלץ להסביר למשתמש את הסיבה באמצעות `ctx.message`, `ctx.notification` או מודאל לפני הקריאה.
- בדרך כלל אין צורך לתפוס את `FlowExitException` בקוד העסקי; הניחו למנוע התזרים (FlowEngine) לטפל בכך.
- אם עליכם להפסיק את כל תזרימי האירועים העוקבים תחת האירוע הנוכחי, השתמשו ב-`ctx.exitAll()`.
