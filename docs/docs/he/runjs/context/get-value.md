# ctx.getValue()

בתרחישים של שדות ניתנים לעריכה כגון JSField ו-JSItem, השתמשו בפונקציה זו כדי לקבל את הערך העדכני ביותר של השדה הנוכחי. בשילוב עם `ctx.setValue(v)`, היא מאפשרת קישור דו-כיווני (two-way binding) עם הטופס.

## תרחישי שימוש

| תרחיש | הסבר |
|------|------|
| **JSField** | קריאת קלט משתמש או את ערך הטופס הנוכחי בשדות מותאמים אישית ניתנים לעריכה. |
| **JSItem** | קריאת ערך התא הנוכחי בפריטים ניתנים לעריכה בטבלאות/תתי-טבלאות. |
| **JSColumn** | קריאת ערך השדה של השורה המתאימה במהלך רינדור עמודת טבלה. |

> **הערה**: `ctx.getValue()` זמין רק בהקשרי RunJS עם קישור לטופס; הוא אינו קיים בתרחישים ללא קישור לשדות, כגון תהליכי עבודה (workflows) או כללי קישור.

## הגדרת טיפוסים (Type Definition)

```ts
getValue<T = any>(): T | undefined;
```

- **ערך חזרה**: הערך הנוכחי של השדה, שהטיפוס שלו נקבע לפי סוג פריט הטופס של השדה; הוא עשוי להיות `undefined` אם השדה אינו רשום או לא מולא.

## סדר שליפת הערכים

`ctx.getValue()` שולף ערכים לפי הסדר הבא:

1. **מצב הטופס**: עדיפות לקריאה מהמצב הנוכחי של Ant Design Form.
2. **ערך חלופי (Fallback)**: אם השדה אינו קיים בטופס, המערכת חוזרת לערך הראשוני של השדה או ל-props.

> אם הטופס טרם סיים להתרנדר או שהשדה אינו רשום, ייתכן שיוחזר `undefined`.

## דוגמאות

### רינדור לפי הערך הנוכחי

```ts
const current = ctx.getValue();
if (current == null || current === '') {
  ctx.render(<span>אנא הזן תוכן תחילה</span>);
} else {
  ctx.render(<span>ערך נוכחי: {current}</span>);
}
```

### קישור דו-כיווני בשילוב עם setValue

```tsx
const { Input } = ctx.libs.antd;

// קריאת הערך הנוכחי כערך ברירת מחדל
const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```
