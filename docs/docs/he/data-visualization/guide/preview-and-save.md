# תצוגה מקדימה ושמירה

- תצוגה מקדימה: מציגה באופן זמני את השינויים שבוצעו בחלונית התצורה בגרף שבעמוד, לצורך אימות התוצאה.
- שמירה: שומרת את השינויים שבוצעו בחלונית התצורה באופן קבוע במסד הנתונים.

## דרכי הפעלה

![clipboard-image-1761479218](https://static-docs.nocobase.com/clipboard-image-1761479218.png)

- במצב תצורה ויזואלי (Basic), שינויים מיושמים אוטומטית בתצוגה המקדימה כברירת מחדל.
- במצבי SQL ו-Custom, לאחר ביצוע שינויים, ניתן ללחוץ על כפתור ה"תצוגה מקדימה" בצד ימין כדי ליישם את השינויים בתצוגה המקדימה.
- כפתור "תצוגה מקדימה" אחיד זמין בתחתית חלונית התצורה כולה.

## התנהגות התצוגה המקדימה
- מציג את התצורה באופן זמני בעמוד, אך אינו כותב אותה למסד הנתונים. לאחר רענון העמוד או ביטול הפעולה, תוצאת התצוגה המקדימה לא נשמרת.
- מנגנון "Debounce" מובנה: הפעלות רענון מרובות בפרק זמן קצר יבצעו רק את האחרונה, כדי למנוע בקשות תכופות.
- לחיצה חוזרת על "תצוגה מקדימה" תחליף את תוצאת התצוגה המקדימה הקודמת.

## הודעות שגיאה
- שגיאות שאילתה או כשלים באימות: מוצגות באזור "הצג נתונים".
- שגיאות תצורת גרף (מיפוי Basic חסר, שגיאות מ-Custom JS): מוצגות באזור הגרף או בקונסול, תוך שמירה על יכולת תפעול העמוד.
- מומלץ לאשר את שמות העמודות וסוגי הנתונים ב"הצג נתונים" לפני מיפוי שדות או כתיבת קוד Custom, כדי לצמצם שגיאות ביעילות.

## שמירה וביטול
- שמירה: כותבת את השינויים הנוכחיים בחלונית לתצורת הבלוק ומיישמת אותם בעמוד באופן מיידי.
- ביטול: מבטלת את השינויים שלא נשמרו בחלונית הנוכחית ומשחזרת את המצב לשמירה האחרונה.
- היקף השמירה:
  - שאילתת נתונים: פרמטרי ה-Builder; במצב SQL, גם טקסט ה-SQL נשמר.
  - אפשרויות גרף: סוג Basic, מיפוי שדות ומאפיינים; טקסט JS של Custom.
  - אירועי אינטראקציה: טקסט JS של האירוע ולוגיקת הקישור.
- לאחר השמירה, הבלוק ייכנס לתוקף עבור כל המבקרים (בהתאם להגדרות הרשאות העמוד).

## זרימת עבודה מומלצת
- הגדרת שאילתת נתונים ← הפעלת שאילתה ← הצגת נתונים לאישור שמות עמודות וסוגים ← הגדרת אפשרויות גרף למיפוי שדות ליבה ← תצוגה מקדימה לאימות ← שמירה ליישום.
