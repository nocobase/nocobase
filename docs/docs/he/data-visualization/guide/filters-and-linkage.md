# מסנני עמודים וקישוריות

מסנן העמוד (Filter Block) משמש לקבלת קלט אחיד של תנאי סינון ברמת העמוד, ולמיזוגם לתוך שאילתות תרשימים, כדי להבטיח סינון וקישוריות עקביים בין תרשימים מרובים.

## סקירת תכונות
- הוסיפו "בלוק מסנן" לעמוד כדי לספק נקודת כניסה אחידה לסינון עבור כל התרשימים בעמוד הנוכחי.
- השתמשו בלחצני "סינון", "איפוס" ו"כיווץ" כדי להחיל, לנקות ולכווץ את המסננים.
- אם המסנן בוחר שדות המקושרים לתרשים, ערכיהם ימוזגו אוטומטית לבקשת השאילתה של התרשים ויפעילו רענון שלו.
- מסננים יכולים גם להגדיר שדות מותאמים אישית ולרשום אותם במשתני הקשר, כך שניתן יהיה להתייחס אליהם בבלוקי נתונים כמו תרשימים, טבלאות וטפסים.

למידע נוסף על השימוש במסנני עמודים ועל קישוריות עם תרשימים או בלוקי נתונים אחרים, עיינו בתיעוד של [מסנני עמודים](#).

## שימוש בערכי מסנני עמודים בשאילתות תרשימים
- מצב Builder (מומלץ)
  - מיזוג אוטומטי: כאשר מקור הנתונים והאוסף תואמים, אין צורך לכתוב משתנים נוספים בשאילתת התרשים; מסנני העמודים ימוזגו באמצעות `$and`.
  - בחירה ידנית: ניתן גם לבחור באופן ידני ערכים מ"שדות מותאמים אישית" של בלוק המסנן בתנאי הסינון של התרשים.

- מצב SQL (באמצעות הזרקת משתנים)
  - בתוך הצהרת SQL, השתמשו ב"בחירת משתנה" כדי להוסיף ערכים מ"שדות מותאמים אישית" של בלוק המסנן.
