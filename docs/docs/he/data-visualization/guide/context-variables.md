# שימוש במשתני הקשר

באמצעות משתני הקשר, באפשרותכם לעשות שימוש חוזר במידע מהעמוד הנוכחי, המשתמש, הזמן ותנאי הסינון, כדי לרנדר תרשימים ולאפשר אינטראקציה בהתאם להקשר.

## תחום היישום
- בשאילתות נתונים במצב Builder: בחרו משתנים עבור תנאי הסינון.
![clipboard-image-1761486073](https://static-docs.nocobase.com/clipboard-image-1761486073.png)

- בשאילתות נתונים במצב SQL: בחרו משתנים והוסיפו ביטויים (לדוגמה, `{{ ctx.user.id }}`).
![clipboard-image-1761486145](https://static-docs.nocobase.com/clipboard-image-1761486145.png)

- באפשרויות תרשים במצב Custom: כתבו ביטויי JS ישירות.
![clipboard-image-1761486604](https://static-docs.nocobase.com/clipboard-image-1761486604.png)

- באירועי אינטראקציה (לדוגמה, לחיצה לפתיחת חלון קופץ (drill-down) והעברת נתונים): כתבו ביטויי JS ישירות.
![clipboard-image-1761486683](https://static-docs.nocobase.com/clipboard-image-1761486683.png)

**שימו לב:**
- אין לעטוף את `{{ ... }}` במרכאות בודדות או כפולות; הקישור (binding) מטופל באופן בטוח על ידי המערכת בהתאם לסוג המשתנה (מחרוזת, מספר, זמן, NULL).
- כאשר משתנה הוא `NULL` או לא מוגדר, טפלו בערכי NULL באופן מפורש ב-SQL באמצעות `COALESCE(...)` או `IS NULL`.
