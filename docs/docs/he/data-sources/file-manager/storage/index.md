# סקירה כללית

## מנועים מובנים

NocoBase תומכת כרגע בסוגי המנועים המובנים הבאים:

- [אחסון מקומי](./local.md)
- [Alibaba Cloud OSS](./aliyun-oss.md)
- [Amazon S3](./amazon-s3.md)
- [Tencent Cloud COS](./tencent-cos.md)

במהלך התקנת המערכת, מנוע אחסון מקומי מתווסף אוטומטית וניתן להשתמש בו מיד. אתם יכולים גם להוסיף מנועים חדשים או לערוך את הפרמטרים של מנועים קיימים.

## פרמטרים נפוצים למנועים

בנוסף לפרמטרים הספציפיים לכל סוג מנוע, החלקים הבאים הם פרמטרים נפוצים (לדוגמה, נשתמש באחסון מקומי):

![דוגמה לתצורת מנוע אחסון קבצים](https://static-docs.nocobase.com/20240529115151.png)

### כותרת

שם מנוע האחסון, המשמש לזיהוי אנושי.

### שם מערכת

שם המערכת של מנוע האחסון, המשמש לזיהוי על ידי המערכת. הוא חייב להיות ייחודי במערכת. אם תשאירו אותו ריק, המערכת תיצור אותו באופן אקראי.

### כתובת URL בסיסית לגישה

זהו הקידומת של כתובת ה-URL שדרכה ניתן לגשת לקובץ חיצונית. זה יכול להיות כתובת ה-URL הבסיסית של CDN, לדוגמה: "`https://cdn.nocobase.com/app`" (ללא ה-"`/`" בסוף).

### נתיב

הנתיב היחסי המשמש לאחסון קבצים. חלק זה יצורף אוטומטית לכתובת ה-URL הסופית בעת הגישה. לדוגמה: "`user/avatar`" (ללא ה-"`/`" בהתחלה או בסוף).

### מגבלת גודל קובץ

מגבלת הגודל לקבצים המועלים למנוע אחסון זה. קבצים החורגים מגודל זה לא יועלו. מגבלת ברירת המחדל של המערכת היא 20MB, וניתן להתאים אותה למגבלה מקסימלית של 1GB.

### סוג קובץ

ניתן להגביל את סוגי הקבצים שניתן להעלות, באמצעות פורמט תיאור תחביר [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types). לדוגמה, `image/*` מייצג קבצי תמונה. ניתן להפריד בין מספר סוגים באמצעות פסיקים באנגלית, לדוגמה: `image/*, application/pdf` כדי לאפשר קבצי תמונה וקבצי PDF.

### מנוע אחסון ברירת מחדל

כאשר אפשרות זו מסומנת, מנוע זה מוגדר כמנוע האחסון המוגדר כברירת מחדל של המערכת. כאשר שדה קבצים מצורפים או אוסף קבצים אינם מציינים מנוע אחסון, הקבצים המועלים יישמרו במנוע האחסון המוגדר כברירת מחדל. לא ניתן למחוק את מנוע האחסון המוגדר כברירת מחדל.

### שמירת קבצים בעת מחיקת רשומות

כאשר אפשרות זו מסומנת, קבצים שהועלו במנוע האחסון נשמרים גם כאשר רשומות הנתונים באוסף הקבצים המצורפים או באוסף הקבצים נמחקות. כברירת מחדל, אפשרות זו אינה מסומנת, כלומר קבצים במנוע האחסון נמחקים יחד עם הרשומות.

:::info{title=טיפ}
לאחר העלאת קובץ, נתיב הגישה הסופי מורכב מחיבור של מספר חלקים:

```
<כתובת URL בסיסית לגישה>/<נתיב>/<שם קובץ><סיומת קובץ>
```

לדוגמה: `https://cdn.nocobase.com/app/user/avatar/20240529115151.png`.
:::
