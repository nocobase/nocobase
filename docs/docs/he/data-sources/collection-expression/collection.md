# אוסף ביטויים

## יצירת תבנית "אוסף ביטויים"

לפני השימוש בצמתי ביטויים דינמיים בתוך תהליך עבודה, יש ליצור תחילה תבנית "אוסף ביטויים" באמצעות כלי ניהול האוספים. אוסף זה ישמש כמאגר לביטויים שונים:

![יצירת אוסף ביטויים](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## הזנת נתוני ביטויים

לאחר מכן, צרו בלוק טבלה והזינו מספר נוסחאות לאוסף התבנית. כל שורה באוסף תבנית "הביטויים" יכולה להיחשב ככלל חישוב המיועד למודל נתונים ספציפי בתוך האוסף. ניתן להשתמש בשדות שונים ממודלי הנתונים של אוספים שונים כמשתנים, וליצור ביטויים ייחודיים ככללי חישוב. כמובן, ניתן גם להשתמש במנועי חישוב שונים.

![הזנת נתוני ביטויים](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=טיפ}
לאחר יצירת הנוסחאות, יש לקשר אותן לנתונים העסקיים. קישור ישיר של כל שורת נתונים עסקיים לנתוני הנוסחה יכול להיות מייגע, ולכן גישה נפוצה היא להשתמש באוסף מטא-נתונים, בדומה לאוסף סיווג, כדי ליצור קשר של רבים-לאחד (או אחד-לאחד) עם אוסף הנוסחאות. לאחר מכן, הנתונים העסקיים מקושרים למטא-נתונים המסווגים בקשר של רבים-לאחד. גישה זו מאפשרת לכם פשוט לציין את המטא-נתונים המסווגים הרלוונטיים בעת יצירת נתונים עסקיים, מה שמקל על איתור ושימוש בנתוני הנוסחה המתאימים דרך נתיב הקישור שנוצר.
:::

## טעינת נתונים רלוונטיים לתהליך העבודה

לדוגמה, צרו תהליך עבודה המופעל על ידי אירוע אוסף. כאשר נוצרת הזמנה, הטריגר אמור לטעון מראש את נתוני המוצר המשויכים יחד עם נתוני הביטויים הקשורים למוצר:
