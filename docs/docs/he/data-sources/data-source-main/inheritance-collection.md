# אוסף בירושה

## מבוא

:::warning
נתמך רק כאשר מסד הנתונים הראשי הוא PostgreSQL.
:::

ניתן ליצור אוסף אב, ולגזור ממנו אוסף צאצא. אוסף הצאצא יירש את מבנה אוסף האב, ויוכל גם להגדיר עמודות משלו. תבנית עיצוב זו מסייעת לארגן ולנהל נתונים בעלי מבנה דומה אך עם הבדלים אפשריים.

להלן כמה מאפיינים נפוצים של אוספים בירושה:

-   **אוסף אב:** אוסף האב מכיל עמודות ונתונים משותפים, ומגדיר את המבנה הבסיסי של היררכיית הירושה כולה.
-   **אוסף צאצא:** אוסף הצאצא יורש את מבנה אוסף האב, אך יכול גם להגדיר עמודות נוספות משלו. הדבר מאפשר לכל אוסף צאצא לכלול את המאפיינים המשותפים של אוסף האב, ובמקביל להכיל מאפיינים ספציפיים לתת-הקבוצה.
-   **שאילתות:** בעת ביצוע שאילתות, ניתן לבחור לשלוף נתונים מכל היררכיית הירושה, רק מאוסף האב, או מאוסף צאצא ספציפי. הדבר מאפשר לאחזר ולעבד נתונים ברמות שונות, בהתאם לצורך.
-   **קשר ירושה:** קשר ירושה נוצר בין אוסף האב לאוסף הצאצא, מה שאומר שניתן להשתמש במבנה של אוסף האב כדי להגדיר מאפיינים עקביים, תוך מתן אפשרות לאוסף הצאצא להרחיב או לדרוס מאפיינים אלו.

תבנית עיצוב זו מסייעת להפחית כפילות נתונים, לפשט את מודל מסד הנתונים, ולהקל על תחזוקת הנתונים. עם זאת, יש להשתמש בה בזהירות, מכיוון שאוספים בירושה עלולים להגביר את מורכבות השאילתות, במיוחד בעת טיפול בכל היררכיית הירושה. מערכות מסדי נתונים התומכות באוספים בירושה מספקות בדרך כלל תחביר וכלים ספציפיים לניהול ולשאילתות על מבני אוספים אלו.
