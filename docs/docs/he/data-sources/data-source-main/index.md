# מסד נתונים ראשי

## מבוא

מסד הנתונים הראשי של NocoBase משמש לאחסון נתונים עסקיים וגם למטא-נתונים של היישום, כולל נתוני טבלאות מערכת, נתוני טבלאות מותאמות אישית ועוד. מסד הנתונים הראשי תומך במסדי נתונים יחסיים כמו MySQL, PostgreSQL ועוד. בעת התקנת יישום NocoBase, מסד הנתונים הראשי מותקן באופן סינכרוני ואינו ניתן למחיקה.

## התקנה

זהו תוסף (Plugin) מובנה, אין צורך בהתקנה נפרדת.

## ניהול אוספים

מקור הנתונים הראשי מספק פונקציונליות מלאה לניהול אוספים. ניתן ליצור אוספים חדשים דרך NocoBase, וגם לסנכרן מבני טבלאות קיימים ממסד הנתונים.

![20240322230134](https://static-docs.nocobase.com/20240322230134.png)

### סנכרון אוספים קיימים ממסד הנתונים

![nocobase_doc-2025-10-29-19-46-34](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-46-34.png)

תכונה חשובה של מקור הנתונים הראשי היא היכולת לסנכרן אוספים קיימים ממסד הנתונים לתוך NocoBase לצורך ניהול. משמעות הדבר היא:

- **הגנה על השקעות קיימות**: אם יש לכם כבר אוספים עסקיים רבים במסד הנתונים, אין צורך ליצור אותם מחדש – ניתן לסנכרן ולהשתמש בהם ישירות.
- **אינטגרציה גמישה**: אוספים שנוצרו באמצעות כלים אחרים (כמו סקריפטים של SQL, כלי ניהול מסדי נתונים וכדומה) יכולים להיות מנוהלים ב-NocoBase.
- **הגירה הדרגתית**: תמיכה בהעברת מערכות קיימות ל-NocoBase באופן הדרגתי, במקום שחזור חד-פעמי.

באמצעות פונקציית "טעינה ממסד נתונים", תוכלו:
1. לדפדף בכל האוספים במסד הנתונים
2. לבחור את האוספים שברצונכם לסנכרן
3. לזהות אוטומטית את מבני האוספים וסוגי השדות
4. לייבא אותם ל-NocoBase לניהול בלחיצה אחת

### תמיכה בסוגי אוספים מרובים

![nocobase_doc-2025-10-29-19-47-14](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-47-14.png)

NocoBase תומך ביצירה ובניהול של סוגי אוספים שונים:
- **אוסף כללי**: כולל שדות מערכת נפוצים מובנים;
- **אוסף ירושה**: מאפשר יצירת אוסף אב שממנו נגזרים אוספי צאצא. אוספי צאצא יורשים את מבנה אוסף האב ויכולים להגדיר עמודות משלהם.
- **אוסף עץ**: אוסף בעל מבנה עץ, כרגע תומך רק בעיצוב רשימת סמיכות.
- **אוסף לוח שנה**: משמש ליצירת אוספי אירועים הקשורים ללוח שנה.
- **אוסף קבצים**: משמש לניהול אחסון קבצים.
- **אוסף ביטויים**: מיועד לתרחישי ביטויים דינמיים ב-תהליכי עבודה (Workflows).
- **אוסף SQL**: אינו אוסף מסד נתונים ממשי, אלא מציג במהירות שאילתות SQL בצורה מובנית.
- **אוסף תצוגה (View)**: מתחבר לתצוגות מסד נתונים קיימות.
- **אוסף חיצוני (FDW)**: מאפשר למערכת מסד הנתונים לגשת ולשאול נתונים ישירות ממקורות נתונים חיצוניים, מבוסס על טכנולוגיית FDW.

### סוגי שדות עשירים

![nocobase_doc-2025-10-29-19-48-51](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-48-51.png)

#### המרת סוגי שדות גמישה

NocoBase תומך בהמרת סוגי שדות גמישה על בסיס אותו סוג מסד נתונים.

**דוגמה: אפשרויות המרה לשדה מסוג String**

כאשר שדה במסד הנתונים הוא מסוג String, ניתן להמיר אותו ב-NocoBase לכל אחת מהצורות הבאות:

- **סוגים בסיסיים**: טקסט חד-שורי, טקסט רב-שורי, מספר טלפון, דוא"ל, URL, סיסמה, צבע, אייקון
- **סוגי בחירה**: תפריט נפתח (בחירה יחידה), תיבת בחירה (רדיו)
- **סוגי מדיה עשירה**: Markdown, Markdown (Vditor), טקסט עשיר, קובץ מצורף (URL)
- **סוגי תאריך ושעה**: תאריך ושעה (כולל אזור זמן), תאריך ושעה (ללא אזור זמן)
- **סוגים מתקדמים**: קידוד אוטומטי, בורר אוספים, הצפנה

מנגנון המרה גמיש זה אומר ש:
- **אין צורך בשינוי מבנה מסד הנתונים**: סוג האחסון הבסיסי של השדה נשאר ללא שינוי; רק אופן הייצוג שלו ב-NocoBase משתנה.
- **התאמה לשינויים עסקיים**: ככל שצרכים עסקיים מתפתחים, ניתן להתאים במהירות את אופן הצגת השדות והאינטראקציה איתם.
- **אבטחת נתונים**: תהליך ההמרה אינו משפיע על שלמות הנתונים הקיימים.

### סנכרון גמיש ברמת השדה

NocoBase לא רק מסנכרן אוספים שלמים, אלא גם תומך בניהול סנכרון מפורט ברמת השדה:

![nocobase_doc-2025-10-29-19-49-56](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-49-56.png)

#### תכונות סנכרון שדות:

1. **סנכרון בזמן אמת**: כאשר מבנה אוסף מסד הנתונים משתנה, ניתן לסנכרן שדות חדשים בכל עת.
2. **סנכרון סלקטיבי**: ניתן לסנכרן באופן סלקטיבי את השדות הדרושים, ולא את כל השדות.
3. **זיהוי סוג אוטומטי**: מזהה אוטומטית את סוגי שדות מסד הנתונים וממפה אותם לסוגי השדות של NocoBase.
4. **שמירה על שלמות הנתונים**: תהליך הסנכרון אינו משפיע על הנתונים הקיימים.

#### תרחישי שימוש:

- **אבולוציית סכימת מסד הנתונים**: כאשר צרכים עסקיים משתנים ויש צורך להוסיף שדות חדשים למסד הנתונים, ניתן לסנכרן אותם במהירות ל-NocoBase.
- **שיתוף פעולה בצוות**: כאשר חברי צוות אחרים או מנהלי מסדי נתונים (DBA) מוסיפים שדות למסד הנתונים, ניתן לסנכרן אותם באופן מיידי.
- **מצב ניהול היברידי**: חלק מהשדות מנוהלים דרך NocoBase, וחלקם מנוהלים בשיטות מסורתיות – שילוב גמיש.

מנגנון סנכרון גמיש זה מאפשר ל-NocoBase להשתלב היטב בארכיטקטורות טכניות קיימות, מבלי לדרוש שינויים בשיטות ניהול מסד הנתונים הקיימות, ובו בזמן ליהנות מהנוחות של פיתוח Low-Code ש-NocoBase מספקת.

למידע נוסף, עיינו בסעיף 「[שדות אוספים / סקירה כללית](/data-sources/data-modeling/collection-fields)」.
