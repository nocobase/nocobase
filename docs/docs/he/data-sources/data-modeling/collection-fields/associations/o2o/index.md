# אחד לאחד

ביחסים בין עובדים לפרופילים אישיים, כל עובד יכול להיות מקושר לרשומה אחת בלבד של פרופיל אישי, וכל רשומת פרופיל אישי יכולה להיות מקושרת לעובד אחד בלבד. במקרה כזה, היחסים בין העובד לפרופיל האישי הם יחסי אחד לאחד.

מפתח זר ביחסי אחד לאחד יכול להיות ממוקם באוסף המקור או באוסף היעד. אם הוא מייצג "יש לו אחד", עדיף למקם את המפתח הזר באוסף היעד; אם הוא מייצג "שייך ל-", עדיף למקם את המפתח הזר באוסף המקור.

לדוגמה, במקרה שהוזכר לעיל, שבו לעובד יש רק פרופיל אישי אחד והפרופיל האישי שייך לעובד, מתאים למקם את המפתח הזר באוסף הפרופילים האישיים.

## תיאורי פרמטרים

### אוסף מקור

אוסף המקור, שהוא האוסף שבו ממוקם השדה הנוכחי.

### אוסף יעד

אוסף היעד, האוסף שאליו מקושרים הנתונים.

### מפתח זר

משמש ליצירת קשר בין שני אוספים. ביחסי אחד לאחד, המפתח הזר יכול להיות ממוקם באוסף המקור או באוסף היעד. אם הוא מייצג "יש לו אחד", עדיף למקם את המפתח הזר באוסף היעד; אם הוא מייצג "שייך ל-", עדיף למקם את המפתח הזר באוסף המקור.

### מפתח מקור <- מפתח זר (מפתח זר באוסף היעד)

השדה שאליו מפנה אילוץ המפתח הזר חייב להיות ייחודי. כאשר המפתח הזר ממוקם באוסף היעד, הוא מציין "יש לו אחד".

### מפתח יעד <- מפתח זר (מפתח זר באוסף המקור)

השדה שאליו מפנה אילוץ המפתח הזר חייב להיות ייחודי. כאשר המפתח הזר ממוקם באוסף המקור, הוא מציין "שייך ל-".

### ON DELETE

ON DELETE מתייחס לכללי הפעולה עבור הפניית המפתח הזר באוסף הצאצא המקושר, בעת מחיקת רשומות מאוסף האב. זוהי אפשרות המוגדרת בעת יצירת אילוץ מפתח זר. אפשרויות ON DELETE נפוצות כוללות:

- CASCADE: כאשר רשומה באוסף האב נמחקת, כל הרשומות המקושרות באוסף הצאצא נמחקות אוטומטית.
- SET NULL: כאשר רשומה באוסף האב נמחקת, ערך המפתח הזר ברשומות המקושרות באוסף הצאצא מוגדר ל-NULL.
- RESTRICT: אפשרות ברירת המחדל, שבה מחיקת רשומת אוסף אב נדחית אם קיימות רשומות מקושרות באוסף הצאצא.
- NO ACTION: בדומה ל-RESTRICT, מחיקת רשומת אוסף אב נדחית אם קיימות רשומות מקושרות באוסף הצאצא.
