# שדות יחסים

ב-NocoBase, שדות יחסים אינם שדות ממשיים, אלא משמשים ליצירת חיבורים בין אוספים. מושג זה מקביל ליחסים (relationships) במסדי נתונים יחסיים.

במסדי נתונים יחסיים, סוגי היחסים הנפוצים ביותר כוללים את הבאים:

- [אחד לאחד (One-to-one)](./o2o/index.md): כל ישות בשני אוספים מתאימה לישות אחת בלבד באוסף השני. סוג יחס זה משמש בדרך כלל לאחסון היבטים שונים של ישות באוספים נפרדים, כדי להפחית כפילויות ולשפר את עקביות הנתונים.
- [אחד לרבים (One-to-many)](./o2m/index.md): כל ישות באוסף אחד יכולה להיות מקושרת למספר ישויות באוסף אחר. זהו אחד מסוגי היחסים הנפוצים ביותר. לדוגמה, מחבר אחד יכול לכתוב מספר מאמרים, אך לכל מאמר יכול להיות רק מחבר אחד.
- [רבים לאחד (Many-to-one)](./m2o/index.md): מספר ישויות באוסף אחד יכולות להיות מקושרות לישות אחת באוסף אחר. סוג יחס זה נפוץ גם במידול נתונים. לדוגמה, מספר סטודנטים יכולים להשתייך לאותה כיתה.
- [רבים לרבים (Many-to-many)](./m2m/index.md): מספר ישויות בשני אוספים יכולות להיות מקושרות זו לזו. סוג יחס זה דורש בדרך כלל אוסף מתווך כדי לתעד את הקישורים בין הישויות. לדוגמה, היחס בין סטודנטים לקורסים – סטודנט יכול להירשם למספר קורסים, וקורס יכול לכלול מספר סטודנטים.

סוגי יחסים אלו ממלאים תפקיד חשוב בתכנון מסדי נתונים ובמידול נתונים, והם מסייעים לתאר יחסים ומבני נתונים מורכבים בעולם האמיתי.
