# השוואה בין מסדי נתונים ראשיים למסדי נתונים חיצוניים

ההבדלים בין מסדי נתונים ראשיים למסדי נתונים חיצוניים ב-NocoBase באים לידי ביטוי בעיקר בארבעה היבטים: תמיכה בסוגי מסדי נתונים, תמיכה בסוגי אוספים, תמיכה בסוגי שדות, ויכולות גיבוי והעברה (מיגרציה).

לפרטים נוספים, ראו: [ניהול מקורות נתונים](https://docs.nocobase.com/data-sources/data-source-manager)

## 4. השוואת תמיכה בגיבוי והעברה (מיגרציה)

| תכונה | מסד נתונים ראשי | מסד נתונים חיצוני |
|-------|-----------------|-------------------|
| גיבוי ושחזור | ✅ | ❌ (בטיפול המשתמש) |
| ניהול העברה (מיגרציה) | ✅ | ❌ (בטיפול המשתמש) |

:::info
NocoBase מספקת יכולות גיבוי, שחזור והעברת מבנה (מיגרציה) עבור מסדי נתונים ראשיים. עבור מסדי נתונים חיצוניים, פעולות אלו צריכות להתבצע באופן עצמאי על ידי המשתמשים, בהתאם לסביבת מסד הנתונים שלהם. NocoBase אינה מספקת תמיכה מובנית לכך.
:::

## סיכום השוואתי

| פריט השוואה | מסד נתונים ראשי | מסד נתונים חיצוני |
|-------------|-----------------|-------------------|
| סוגי מסדי נתונים | PostgreSQL, MySQL, MariaDB, KingbaseES | PostgreSQL, MySQL, MariaDB, MSSQL, Oracle, KingbaseES |
| תמיכה בסוגי אוספים | כל סוגי האוספים | רק אוספים כלליים ותצוגות |
| תמיכה בסוגי שדות | כל סוגי השדות | כל סוגי השדות למעט שדות קבצים מצורפים |
| גיבוי והעברה (מיגרציה) | תמיכה מובנית | בטיפול המשתמש |

## המלצות

- **אם אתם משתמשים ב-NocoBase כדי לבנות מערכת עסקית חדשה לגמרי**, מומלץ להשתמש ב**מסד הנתונים הראשי**, מה שיאפשר לכם לנצל את מלוא הפונקציונליות של NocoBase.
- **אם אתם משתמשים ב-NocoBase כדי להתחבר למסדי נתונים של מערכות אחרות לצורך פעולות CRUD בסיסיות**, השתמשו ב**מסדי נתונים חיצוניים**.
