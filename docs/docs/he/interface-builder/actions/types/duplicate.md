# שכפול

## מבוא

פעולת השכפול מאפשרת למשתמשים ליצור רשומות חדשות במהירות על בסיס נתונים קיימים. היא תומכת בשני מצבי שכפול: **שכפול ישיר** ו**שכפול לטופס והמשך מילוי**.

## התקנה

תוסף מובנה, אין צורך בהתקנה נוספת.

## מצב שכפול

![20260209224344](https://static-docs.nocobase.com/20260209224344.png)

### שכפול ישיר

![20260209224506](https://static-docs.nocobase.com/20260209224506.png)

- מתבצע כ"שכפול ישיר" כברירת מחדל;
- **שדות תבנית**: ציינו את השדות לשכפול. ניתן לבחור הכל ("Select all"), הגדרה זו היא חובה.

![20260209225910](https://static-docs.nocobase.com/20260209225910.gif)

לאחר ההגדרה, לחצו על הכפתור כדי לשכפל את הנתונים.

### שכפול לטופס והמשך מילוי

שדות התבנית שהוגדרו יאוכלסו בטופס כ**ערכי ברירת מחדל**. משתמשים יכולים לשנות ערכים אלו לפני השליחה כדי להשלים את השכפול.

![20260209224704](https://static-docs.nocobase.com/20260209224704.png)

**הגדרת שדות תבנית**: רק השדות שנבחרו יועברו כערכי ברירת מחדל.

![20260209225148](https://static-docs.nocobase.com/20260209225148.png)

#### סנכרון שדות טופס

- מנתח באופן אוטומטי את השדות שכבר הוגדרו בבלוק הטופס הנוכחי כשדות תבנית;
- אם שדות בלוק הטופס ישתנו בהמשך (למשל, התאמת רכיבי שדות קשר), עליכם לפתוח מחדש את הגדרות התבנית וללחוץ על **סנכרון שדות טופס** כדי להבטיח עקביות.

![20260209225450](https://static-docs.nocobase.com/20260209225450.gif)

נתוני התבנית ימולאו כברירת מחדל בטופס, והמשתמשים יוכלו לשלוח לאחר שינוי כדי להשלים את השכפול.

### הערות נוספות

#### שכפול, התייחסות (Reference), טעינה מראש (Preload)

לסוגי שדות שונים (סוגי קשרים) יש לוגיקת עיבוד שונה: **שכפול / התייחסות / טעינה מראש**. גם **רכיב השדה** של שדה קשר משפיע על לוגיקה זו:

- Select / Record picker: משמש ל**התייחסות**
- Sub-form / Sub-table: משמש ל**שכפול**

**שכפול**

- שדות רגילים משוכפלים;
- `hasOne` / `hasMany` יכולים להיות רק משוכפלים (קשרים אלו לא אמורים להשתמש ברכיבי בחירה כמו בחירה בודדת או בחירת רשומה; במקום זאת, יש להשתמש ברכיבי תת-טופס או תת-טבלה);
- שינוי הרכיב עבור `hasOne` / `hasMany` **לא ישנה** את לוגיקת העיבוד (היא תישאר שכפול);
- עבור שדות קשר משוכפלים, ניתן לבחור את כל שדות המשנה.

**התייחסות (Reference)**

- `belongsTo` / `belongsToMany` מטופלים כהתייחסות;
- אם רכיב השדה משתנה מ"בחירה בודדת" ל"תת-טופס", הקשר משתנה מ**התייחסות לשכפול** (ברגע שהוא הופך לשכפול, כל שדות המשנה הופכים לניתנים לבחירה).

**טעינה מראש (Preload)**

- שדות קשר תחת שדה התייחסות מטופלים כטעינה מראש;
- שדות טעינה מראש עשויים להפוך להתייחסות או לשכפול לאחר שינוי רכיב.

#### בחר הכל

- בוחר את כל **שדות השכפול** ו**שדות ההתייחסות**.

#### השדות הבאים יסוננו מהרשומה שנבחרה כתבנית נתונים:

- מפתחות ראשיים (Primary keys) של נתוני קשר משוכפלים מסוננים; מפתחות ראשיים עבור התייחסות וטעינה מראש אינם מסוננים;
- מפתחות זרים (Foreign keys);
- שדות שאינם מאפשרים כפילויות (Unique);
- שדות מיון;
- שדות מספור אוטומטי (Sequence);
- סיסמה;
- נוצר על ידי, נוצר ב-;
- עודכן לאחרונה על ידי, עודכן ב-.

#### סנכרון שדות טופס

- מנתח באופן אוטומטי את השדות שהוגדרו בבלוק הטופס הנוכחי לשדות תבנית;
- לאחר שינוי שדות בלוק הטופס (למשל, התאמת רכיבי שדות קשר), עליכם לסנכרן שוב כדי להבטיח עקביות.
