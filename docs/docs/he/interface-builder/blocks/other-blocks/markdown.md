# בלוק Markdown

## מבוא

בלוק Markdown אינו דורש קישור למקור נתונים. הוא משתמש בתחביר Markdown כדי להגדיר תוכן טקסט, וניתן להשתמש בו להצגת טקסט מעוצב.

## הוספת בלוק

באפשרותכם להוסיף בלוק Markdown לדף או לחלון קופץ.

![20251026230916](https://static-docs.nocobase.com/20251026230916.png)

כמו כן, ניתן להוסיף בלוק Markdown מוטבע (inline-block) בתוך בלוקי טופס ובלוקי פרטים.

![20251026231002](https://static-docs.nocobase.com/20251026231002.png)

## מנוע תבניות

אנו משתמשים ב**[מנוע התבניות Liquid](https://liquidjs.com/tags/overview.html)**, המספק יכולות רינדור תבניות עוצמתיות וגמישות, ומאפשר יצירה והצגה דינמית ומותאמת אישית של תוכן. באמצעות מנוע התבניות, תוכלו:

- **הצבה דינמית**: השתמשו במצייני מיקום בתבנית כדי להפנות למשתנים, לדוגמה, `{{ ctx.user.userName }}` יוחלף אוטומטית בשם המשתמש המתאים.
- **רינדור מותנה**: תמיכה בהצהרות תנאי (`{% if %}...{% else %}`), המציגות תוכן שונה בהתאם למצבי נתונים שונים.
- **לולאות**: השתמשו ב-`{% for item in list %}...{% endfor %}` כדי לעבור על מערכים או אוספים, וליצור רשימות, טבלאות או מודולים חוזרים.
- **מסננים מובנים**: מציע מגוון עשיר של מסננים (כגון `upcase`, `downcase`, `date`, `truncate` ועוד), המאפשרים לעצב ולעבד נתונים.
- **הרחבה**: תמיכה במשתנים ופונקציות מותאמים אישית, ההופכים את לוגיקת התבנית לניתנת לשימוש חוזר ולתחזוקה קלה.
- **אבטחה ובידוד**: רינדור התבניות מתבצע בסביבת ארגז חול (sandbox), מונע הפעלה ישירה של קוד מסוכן ומשפר את האבטחה.

בעזרת מנוע התבניות Liquid, מפתחים ויוצרי תוכן יכולים **להשיג בקלות הצגת תוכן דינמי, יצירת מסמכים מותאמים אישית ורינדור תבניות למבני נתונים מורכבים**, ובכך לשפר משמעותית את היעילות והגמישות.

## שימוש במשתנים

Markdown בדף תומך במשתני מערכת נפוצים (כגון המשתמש הנוכחי, התפקיד הנוכחי ועוד).

![20251029203252](https://static-docs.nocobase.com/20251029203252.png)

לעומת זאת, Markdown בחלון קופץ של פעולת שורת בלוק (או בדף משנה) תומך במשתני הקשר נתונים נוספים (כגון הרשומה הנוכחית, רשומת החלון הקופץ הנוכחית ועוד).

![20251029203400](https://static-docs.nocobase.com/20251029203400.png)
