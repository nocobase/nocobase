# כללי קישור שדות

## מבוא

כללי קישור שדות מאפשרים להתאים באופן דינמי את מצב השדות בבלוקי טופס/פרטים, בהתבסס על פעולות המשתמש. הבלוקים התומכים כרגע בכללי קישור שדות הם:

- [בלוק טופס](/interface-builder/blocks/data-blocks/form)
- [בלוק פרטים](/interface-builder/blocks/data-blocks/details)
- [תת-טופס](/interface-builder/fields/specific/sub-form)

## הוראות שימוש

### **בלוק טופס**

בבלוק טופס, כללי קישור יכולים להתאים באופן דינמי את התנהגות השדות בהתבסס על תנאים ספציפיים:

- **שליטה על הצגת/הסתרת שדה**: קביעה אם השדה הנוכחי יוצג, בהתבסס על הערכים של שדות אחרים.
- **הגדרת שדה כחובה**: הגדרה דינמית של שדה כחובה או לא חובה בתנאים ספציפיים.
- **הקצאת ערך**: הקצאה אוטומטית של ערך לשדה בהתבסס על תנאים.
- **ביצוע JavaScript מוגדר**: כתיבת JavaScript בהתאם לדרישות העסקיות.

### **בלוק פרטים**

בבלוק פרטים, כללי קישור משמשים בעיקר לשליטה דינמית על הצגתם והסתרתם של שדות בבלוק.

## קישור מאפיינים

### הקצאת ערך

דוגמה: כאשר הזמנה מסומנת כהזמנה משלימה, סטטוס ההזמנה מוקצה אוטומטית ל'ממתין לבדיקה'.

![20251029115348](https://static-docs.nocobase.com/20251029115348.png)

### שדה חובה

דוגמה: כאשר סטטוס ההזמנה הוא 'שולם', שדה סכום ההזמנה הוא שדה חובה.

![20251029115031](https://static-docs.nocobase.com/20251029115031.png)

### הצגה/הסתרה

דוגמה: חשבון התשלום והסכום הכולל מוצגים רק כאשר סטטוס ההזמנה הוא 'ממתין לתשלום'.

![20251030223710](https://static-docs.nocobase.com/20251030223710.png)
