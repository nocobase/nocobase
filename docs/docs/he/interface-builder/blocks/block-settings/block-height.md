# גובה בלוק

## מבוא

גובה הבלוק תומך בשלושה מצבים: **גובה ברירת מחדל**, **גובה מוגדר**, ו**גובה מלא**. רוב הבלוקים תומכים בהגדרות גובה.

![20260211153947](https://static-docs.nocobase.com/20260211153947.png)

![20260211154020](https://static-docs.nocobase.com/20260211154020.png)

## מצבי גובה

### גובה ברירת מחדל

אסטרטגיית גובה ברירת המחדל משתנה עבור סוגי בלוקים שונים. לדוגמה, בלוקים של טבלה וטופס יתאימו את גובהם באופן אוטומטי בהתאם לתוכן, ולא יופיעו פסי גלילה בתוך הבלוק.

### גובה מוגדר

ניתן להגדיר ידנית את הגובה הכולל של המסגרת החיצונית של הבלוק. הבלוק יחשב ויקצה באופן אוטומטי את הגובה הזמין בתוכו.

![20260211154043](https://static-docs.nocobase.com/20260211154043.gif)

### גובה מלא

מצב גובה מלא דומה לגובה מוגדר, אך גובה הבלוק מחושב על סמך **אזור התצוגה** (viewport) הנוכחי של הדפדפן כדי להשיג גובה מקסימלי של מסך מלא. לא יופיעו פסי גלילה בדף הדפדפן; פסי גלילה יופיעו רק בתוך הבלוק.

הטיפול בגלילה פנימית במצב גובה מלא משתנה מעט בין הבלוקים:

- **טבלה**: גלילה פנימית בתוך `tbody`;
- **טופס / פרטים**: גלילה פנימית בתוך ה-Grid (גלילת תוכן למעט אזור הפעולות);
- **רשימה / כרטיס רשת**: גלילה פנימית בתוך ה-Grid (גלילת תוכן למעט אזור הפעולות וסרגל העימוד);
- **מפה / לוח שנה**: גובה אדפטיבי כללי, ללא פסי גלילה;
- **Iframe / Markdown**: מגביל את הגובה הכולל של מסגרת הבלוק, כאשר פסי גלילה מופיעים בתוך הבלוק.
