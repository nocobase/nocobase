# טופס סינון

## מבוא

טופס סינון מאפשר למשתמשים לסנן נתונים על ידי מילוי שדות טופס. ניתן להשתמש בו לסינון בלוקי טבלה, בלוקי תרשים, בלוקי רשימה וכו'.

## כיצד להשתמש

בואו נבין תחילה במהירות כיצד להשתמש בטופס הסינון דרך דוגמה פשוטה. נניח שיש לנו בלוק טבלה המכיל פרטי משתמשים, ואנו רוצים לסנן את הנתונים באמצעות טופס סינון. כך:

![20251031163036_rec_](https://static-docs.nocobase.com/20251031163036_rec_.gif)

שלבי התצורה הם כדלקמן:

1. הפעילו את מצב ההגדרה, הוסיפו לדף בלוק "טופס סינון" ובלוק "טבלה".
![20251031163525_rec_](https://static-docs.nocobase.com/20251031163525_rec_.gif)
2. הוסיפו את השדה "כינוי" בבלוק הטבלה ובבלוק טופס הסינון.
![20251031163932_rec_](https://static-docs.nocobase.com/20251031163932_rec_.gif)
3. כעת ניתן להתחיל להשתמש בו.
![20251031163036_rec_](https://static-docs.nocobase.com/20251031163036_rec_.gif)

## שימוש מתקדם

בלוק טופס הסינון תומך בתצורות מתקדמות נוספות, להלן כמה שימושים נפוצים.

### קישור מספר בלוקים

שדה טופס יחיד יכול לסנן נתונים במספר בלוקים בו-זמנית. הפעולה היא כדלקמן:

1. לחצו על פריט התצורה "Connect fields" של השדה.
![20251031170300](https://static-docs.nocobase.com/20251031170300.png)
2. הוסיפו את בלוקי היעד לקישור, כאן נבחר בבלוק הרשימה שבדף.
![20251031170718](https://static-docs.nocobase.com/20251031170718.png)
3. בחרו שדה אחד או יותר בבלוק הרשימה לקישור. כאן נבחר בשדה "כינוי".
![20251031171039](https://static-docs.nocobase.com/20251031171039.png)
4. לחצו על כפתור השמירה להשלמת התצורה, התוצאה מוצגת להלן:
![20251031171209_rec_](https://static-docs.nocobase.com/20251031171209_rec_.gif)

### קישור בלוקי תרשים

סימוכין: [מסנני דפים וקישוריות](../../../data-visualization/guide/filters-and-linkage.md)

### שדות מותאמים אישית

בנוסף לבחירת שדות מתוך האוסף, ניתן גם ליצור שדות טופס באמצעות "שדות מותאמים אישית". לדוגמה, ניתן ליצור שדה בחירה נפתחת ולהתאים אישית את האפשרויות. הפעולה היא כדלקמן:

1. לחצו על האפשרות "שדות מותאמים אישית", ממשק התצורה יופיע.
![20251031173833](https://static-docs.nocobase.com/20251031173833.png)
2. מלאו את כותרת השדה, בחרו "בחירה" ב"סוג שדה", והגדירו את האפשרויות.
![20251031174857_rec_](https://static-docs.nocobase.com/20251031174857_rec_.gif)
3. שדות מותאמים אישית שנוספו לאחרונה דורשים קישור ידני לשדות בבלוק היעד, שיטת הפעולה היא כדלקמן:
![20251031181957_rec_](https://static-docs.nocobase.com/20251031181957_rec_.gif)
4. התצורה הושלמה, התוצאה מוצגת להלן:
![20251031182235_rec_](https://static-docs.nocobase.com/20251031182235_rec_.gif)

סוגי השדות הנתמכים כיום הם:

- תיבת טקסט
- מספר
- תאריך
- בחירה
- כפתור בחירה (Radio)
- תיבת סימון (Checkbox)
- רשומה קשורה (Association)

#### רשומה קשורה (שדה קשר מותאם אישית)

"רשומה קשורה" מתאימה לתרחישים של "סינון לפי רשומות אוסף קשור". לדוגמה, ברשימת הזמנות, סינון הזמנות לפי "לקוח", או ברשימת משימות סינון משימות לפי "אחראי".

הסבר פריטי תצורה:

- **אוסף יעד**: מציין מאיזה אוסף לטעון רשומות לבחירה.
- **שדה כותרת**: משמש כטקסט התצוגה באפשרויות הנפתחות ובתגיות שנבחרו (כגון שם, כותרת).
- **שדה ערך**: משמש כערך המוגש בעת הסינון בפועל, בדרך כלל נבחר שדה המפתח הראשי (כגון `id`).
- **אפשר בחירה מרובה**: לאחר ההפעלה ניתן לבחור מספר רשומות בו-זמנית.
- **אופרטור**: מגדיר כיצד תנאי הסינון תואמים (ראה הסבר "אופרטור" להלן).

תצורה מומלצת:

1. בחרו בשדה בעל קריאות גבוהה עבור `שדה כותרת` (כגון "שם"), כדי למנוע שימוש ב-ID בלבד שיפגע בשימושיות.
2. העדיפו לבחור בשדה המפתח הראשי עבור `שדה ערך`, כדי להבטיח סינון יציב וייחודי.
3. בתרחישי בחירה יחידה בדרך כלל סוגרים את `אפשר בחירה מרובה`, ובתרחישי בחירה מרובה מפעילים את `אפשר בחירה מרובה` בשילוב עם ה`אופרטור` המתאים.

#### אופרטור

`אופרטור` משמש להגדרת יחסי ההתאמה בין "ערך שדה טופס הסינון" לבין "ערך שדה בלוק היעד".

### כיווץ

הוסיפו כפתור כיווץ, המאפשר לכווץ ולפרוס את תוכן טופס הסינון כדי לחסוך מקום בדף.

![20251031182743](https://static-docs.nocobase.com/20251031182743.png)

התצורות הבאות נתמכות:

![20251031182849](https://static-docs.nocobase.com/20251031182849.png)

- **מספר שורות להצגה בעת כיווץ**: הגדרת מספר שורות שדות הטופס המוצגות במצב מכווץ.
- **כיווץ כברירת מחדל**: לאחר ההפעלה, טופס הסינון יוצג במצב מכווץ כברירת מחדל.
