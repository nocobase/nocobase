# הגדרת כללי אימות

## מבוא
כללי אימות משמשים כדי לוודא שנתונים שהוזנו על ידי המשתמש עומדים בציפיות.

## היכן ניתן להגדיר כללי אימות לשדות

### הגדרת כללי אימות לשדות אוסף
רוב השדות תומכים בהגדרת כללי אימות. לאחר ששדה מוגדר עם כללי אימות, אימות צד-שרת (backend) יופעל בעת שליחת נתונים. סוגי שדות שונים תומכים בכללי אימות שונים.

- **שדה תאריך**

  ![20251028225946](https://static-docs.nocobase.com/20251028225946.png)

- **שדה מספרי**

  ![20251028230418](https://static-docs.nocobase.com/20251028230418.png)

- **שדה טקסט**

  שדות טקסט, בנוסף ליכולת להגביל את אורך הטקסט, תומכים גם בביטויים רגולריים מותאמים אישית (regex) לאימות מדויק יותר.

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### אימות צד-לקוח (Frontend) בהגדרות השדה
כללי אימות המוגדרים בקונפיגורציית השדה יפעילו אימות צד-לקוח (frontend) כדי לוודא שקלט המשתמש עומד בדרישות.

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

![20251028230255](https://static-docs.nocobase.com/20251028230255.png)

**שדות טקסט** תומכים גם באימות באמצעות ביטויים רגולריים מותאמים אישית (regex) כדי לעמוד בדרישות פורמט ספציפיות.

![20251028230903](https://static-docs.nocobase.com/20251028230903.png)
