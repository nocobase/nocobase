# תת-טבלה (עריכה בחלון קופץ)

## מבוא

תת-טבלה (עריכה בחלון קופץ) משמשת לניהול נתוני קשרים מרובים (כגון יחיד-לרבים או רבים-לרבים) בתוך טופס. הטבלה מציגה רק רשומות מקושרות קיימות. הוספה או עריכה של רשומות מתבצעת בתוך חלון קופץ (Popup), והנתונים נשמרים במסד הנתונים באופן מרוכז בעת שליחת הטופס הראשי.

## הוראות שימוש

![20260212152204](https://static-docs.nocobase.com/20260212152204.png)

**תרחישים נפוצים:**

- שדות קשר: o2m / m2m / mbm
- שימושים טיפוסיים: פירוט הזמנות, רשימות פריטי משנה, תגיות/חברים מקושרים וכו'.

## הגדרות שדה

### אפשר בחירת נתונים קיימים (ברירת מחדל: פעיל)

תומך בבחירת קשרים מתוך רשומות קיימות.

![20260212152312](https://static-docs.nocobase.com/20260212152312.png)

![20260212152343](https://static-docs.nocobase.com/20260212152343.gif)

### רכיב שדה

[רכיב שדה](/interface-builder/fields/association-field): מעבר לרכיבי שדה קשר אחרים, כגון בחירה מרשימה, בורר אוספים וכו'.

### אפשר ביטול קישור לנתונים קיימים (ברירת מחדל: פעיל)

> שולט האם ניתן לבטל קישור של נתונים מקושרים קיימים בטופס העריכה. תמיד ניתן להסיר נתונים שנוספו לאחרונה.

![20260212165752](https://static-docs.nocobase.com/20260212165752.gif)

### אפשר הוספה (ברירת מחדל: פעיל)

שולט בהצגת כפתור ההוספה. אם למשתמש אין הרשאות `create` (יצירה) עבור האוסף המיועד, הכפתור יושבת ותופיע הודעה על חוסר בהרשאות.

### אפשר עריכה מהירה (ברירת מחדל: כבוי)

כאשר אפשרות זו פעילה, מעבר עם העכבר מעל תא יציג סמל עריכה, המאפשר שינוי מהיר של תוכן התא.

ניתן להפעיל עריכה מהירה עבור כל השדות דרך הגדרות רכיב שדה הקשר.

![20260212165102](https://static-docs.nocobase.com/20260212165102.png)

ניתן גם להפעיל זאת עבור שדות עמודה בודדים.

![20260212165025](https://static-docs.nocobase.com/20260212165025.png)

![20260212165201](https://static-docs.nocobase.com/20260212165201.gif)

### גודל דף (ברירת מחדל: 10)

קובע את מספר הרשומות המוצגות בכל דף בתת-הטבלה.

## הערות התנהגות

- בעת בחירת רשומות קיימות, מתבצעת מניעת כפילויות על בסיס המפתח הראשי כדי למנוע קישור כפול של אותה רשומה.
- רשומות חדשות שנוספו ממולאות ישירות בתת-הטבלה, והתצוגה עוברת אוטומטית לדף המכיל את הרשומה החדשה.
- עריכה בשורה (Inline) משנה רק את הנתונים של השורה הנוכחית.
- הסרה רק מבטלת את הקישור בתוך הטופס הנוכחי; היא אינה מוחקת את נתוני המקור ממסד הנתונים.
- הנתונים נשמרים במסד הנתונים רק בעת שליחת הטופס הראשי.
