# הליכי תחזוקה

## הפעלה ראשונית של היישום

בעת הפעלה ראשונית של היישום, יש להפעיל תחילה צומת אחד. לאחר שהתוספים (Plugins) יותקנו ויופעלו, ניתן להפעיל את שאר הצמתים.

## שדרוג גרסה

כאשר אתם צריכים לשדרג את גרסת NocoBase, עקבו אחר הליך זה.

:::warning{title=שימו לב}
ב**סביבת ייצור** אשכולית (cluster), יש להשתמש בזהירות או לאסור לחלוטין שימוש בתכונות כמו ניהול תוספים (plugins) ושדרוגי גרסה.

NocoBase אינה תומכת כרגע בשדרוגים מקוונים (online upgrades) עבור גרסאות אשכוליות. כדי להבטיח עקביות נתונים, יש להשעות שירותים חיצוניים במהלך תהליך השדרוג.
:::

שלבי הפעולה:

1.  עצירת השירות הנוכחי

    עצרו את כל מופעי יישום NocoBase והפנו את תעבורת מאזן העומסים (load balancer) לדף סטטוס 503.

2.  גיבוי נתונים

    לפני השדרוג, מומלץ בחום לגבות את נתוני מסד הנתונים כדי למנוע בעיות כלשהן במהלך תהליך השדרוג.

3.  עדכון הגרסה

    עיינו ב-[שדרוג Docker](../get-started/upgrading/docker) כדי לעדכן את גרסת אימג' היישום של NocoBase.

4.  הפעלת השירות

    1. הפעילו צומת אחד באשכול והמתינו לסיום העדכון ולהפעלה מוצלחת של הצומת.
    2. ודאו שהפונקציונליות תקינה. אם קיימות בעיות שלא ניתן לפתור באמצעות איתור תקלות, תוכלו לחזור לגרסה הקודמת.
    3. הפעילו את שאר הצמתים.
    4. הפנו מחדש את תעבורת מאזן העומסים לאשכול היישומים.

## תחזוקה מתוך היישום

תחזוקה מתוך היישום מתייחסת לביצוע פעולות תחזוקה קשורות בזמן שהיישום פועל, וכוללת:

*   ניהול תוספים (Plugins) (התקנה, הפעלה, השבתה של תוספים ועוד)
*   גיבוי ושחזור
*   ניהול העברת סביבות (Environment Migration)

שלבי הפעולה:

1.  צמצום צמתים

    צמצמו את מספר צמתי היישום הפועלים באשכול לצומת אחד, ועצרו את השירות בשאר הצמתים.

2.  בצעו פעולות תחזוקה מתוך היישום, כגון התקנה והפעלה של תוספים (plugins), גיבוי נתונים ועוד.

3.  שחזור צמתים

    לאחר השלמת פעולות התחזוקה ואימות תקינות הפונקציונליות, הפעילו את שאר הצמתים. לאחר שהצמתים יופעלו בהצלחה, שחזרו את מצב הפעולה של האשכול.
