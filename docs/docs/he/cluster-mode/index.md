# מצב אשכול

## מבוא

החל מגרסה v1.6.0, NocoBase תומכת בהרצת יישומים במצב אשכול. כאשר יישום רץ במצב אשכול, הוא יכול לשפר את ביצועיו בטיפול בגישה מקבילה באמצעות שימוש במספר מופעים ובמצב ריבוי ליבות.

## ארכיטקטורת המערכת

![20251031221530](https://static-docs.nocobase.com/20251031221530.png)

*   **אשכול יישומים**: אשכול המורכב ממספר מופעים של יישום NocoBase. ניתן לפרוס אותו על מספר שרתים, או להריץ מספר תהליכים במצב ריבוי ליבות על שרת יחיד.
*   **מסד נתונים**: מאחסן את נתוני היישום. יכול להיות מסד נתונים בעל צומת יחיד (single-node) או מסד נתונים מבוזר (distributed).
*   **אחסון משותף**: משמש לאחסון קבצים ונתונים של היישום, ותומך בגישת קריאה/כתיבה ממספר מופעים.
*   **תווכה (Middleware)**: כוללת רכיבים כמו מטמון (cache), אותות סנכרון, תור הודעות (message queue) ומנעולים מבוזרים (distributed locks) כדי לתמוך בתקשורת ובתיאום בתוך אשכול היישומים.
*   **מאזן עומסים (Load Balancer)**: אחראי על חלוקת בקשות לקוח למופעים שונים באשכול היישומים, וכן על ביצוע בדיקות תקינות (health checks) ומעבר לגיבוי (failover).
