### עיצוב מספרים

#### 1. `:formatN(precision)`

##### הסבר תחבירי
מעצב מספר בהתאם להגדרות הלוקליזציה.
פרמטרים:
- `precision`: מספר הספרות אחרי הנקודה העשרונית.
  עבור פורמטים מסוג ODS/XLSX, מספר הספרות העשרוניות המוצגות נקבע על ידי עורך הטקסט; עבור פורמטים אחרים, פרמטר זה משמש לקביעה.

##### דוגמה
```
// Example environment: API options { "lang": "en-us" }
'10':formatN()         // Outputs "10.000"
'1000.456':formatN()   // Outputs "1,000.456"
```

##### תוצאה
המספר מוצג בפורמט וברמת הדיוק שצוינו, ובהתאם להגדרות הלוקליזציה.


#### 2. `:round(precision)`

##### הסבר תחבירי
מעגל את המספר למספר הספרות העשרוניות שצוין.

##### דוגמה
```
10.05123:round(2)      // Outputs 10.05
1.05:round(1)          // Outputs 1.1
```

##### תוצאה
התוצאה היא המספר מעוגל לרמת הדיוק הנתונה.


#### 3. `:add(value)`

##### הסבר תחבירי
מוסיף את הערך שצוין למספר הנוכחי.
פרמטרים:
- `value`: המספר להוספה.

##### דוגמה
```
1000.4:add(2)         // Outputs 1002.4
'1000.4':add('2')      // Outputs 1002.4
```

##### תוצאה
התוצאה היא סכום המספר הנוכחי והערך שצוין.


#### 4. `:sub(value)`

##### הסבר תחבירי
מחסיר את הערך שצוין מהמספר הנוכחי.
פרמטרים:
- `value`: המספר להחסרה.

##### דוגמה
```
1000.4:sub(2)         // Outputs 998.4
'1000.4':sub('2')      // Outputs 998.4
```

##### תוצאה
התוצאה היא המספר הנוכחי פחות הערך שצוין.


#### 5. `:mul(value)`

##### הסבר תחבירי
מכפיל את המספר הנוכחי בערך שצוין.
פרמטרים:
- `value`: המכפיל.

##### דוגמה
```
1000.4:mul(2)         // Outputs 2000.8
'1000.4':mul('2')      // Outputs 2000.8
```

##### תוצאה
התוצאה היא מכפלת המספר הנוכחי בערך שצוין.


#### 6. `:div(value)`

##### הסבר תחבירי
מחלק את המספר הנוכחי בערך שצוין.
פרמטרים:
- `value`: המחלק.

##### דוגמה
```
1000.4:div(2)         // Outputs 500.2
'1000.4':div('2')      // Outputs 500.2
```

##### תוצאה
התוצאה היא מנת החלוקה.


#### 7. `:mod(value)`

##### הסבר תחבירי
מחשב את השארית (מודולו) של חלוקת המספר הנוכחי בערך שצוין.
פרמטרים:
- `value`: המודולוס (המחלק).

##### דוגמה
```
4:mod(2)              // Outputs 0
3:mod(2)              // Outputs 1
```

##### תוצאה
התוצאה היא שארית פעולת המודולו.


#### 8. `:abs`

##### הסבר תחבירי
מחזיר את הערך המוחלט של המספר.

##### דוגמה
```
-10:abs()             // Outputs 10
-10.54:abs()          // Outputs 10.54
10.54:abs()           // Outputs 10.54
'-200':abs()          // Outputs 200
```

##### תוצאה
התוצאה היא הערך המוחלט של המספר.


#### 9. `:ceil`

##### הסבר תחבירי
מעגל את המספר כלפי מעלה למספר השלם הקטן ביותר שגדול או שווה למספר הנוכחי.

##### דוגמה
```
10.05123:ceil()       // Outputs 11
1.05:ceil()           // Outputs 2
-1.05:ceil()          // Outputs -1
```

##### תוצאה
התוצאה היא המספר מעוגל כלפי מעלה למספר השלם הקרוב ביותר.


#### 10. `:floor`

##### הסבר תחבירי
מעגל את המספר כלפי מטה למספר השלם הגדול ביותר שקטן או שווה למספר הנוכחי.

##### דוגמה
```
10.05123:floor()      // Outputs 10
1.05:floor()          // Outputs 1
-1.05:floor()         // Outputs -2
```

##### תוצאה
התוצאה היא המספר מעוגל כלפי מטה למספר השלם הקרוב ביותר.


#### 11. `:int`

##### הסבר תחבירי
ממיר את המספר למספר שלם (לא מומלץ לשימוש).

##### דוגמה ותוצאה
תלוי במקרה ההמרה הספציפי.


#### 12. `:toEN`

##### הסבר תחבירי
ממיר את המספר לפורמט אנגלי (משתמש בנקודה `.` כמפריד עשרוני). לא מומלץ לשימוש.

##### דוגמה ותוצאה
תלוי במקרה ההמרה הספציפי.


#### 13. `:toFixed`

##### הסבר תחבירי
ממיר את המספר למחרוזת תוך שמירה על מספר הספרות העשרוניות שצוין בלבד. לא מומלץ לשימוש.

##### דוגמה ותוצאה
תלוי במקרה ההמרה הספציפי.


#### 14. `:toFR`

##### הסבר תחבירי
ממיר את המספר לפורמט צרפתי (משתמש בפסיק `,` כמפריד עשרוני). לא מומלץ לשימוש.

##### דוגמה ותוצאה
תלוי במקרה ההמרה הספציפי.
