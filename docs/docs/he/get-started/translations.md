# תרומת תרגום

שפת ברירת המחדל של NocoBase היא אנגלית. נכון לעכשיו, האפליקציה הראשית תומכת באנגלית, איטלקית, הולנדית, סינית מפשוטת ויפנית. אנו מזמינים אתכם בכבוד לתרום תרגומים לשפות נוספות, כדי לאפשר למשתמשים בכל העולם ליהנות מחוויית NocoBase נוחה עוד יותר.

---

#### 1.2 סקירה כללית של תוכן המקומיות
NocoBase משתמשת ב-Git לניהול תוכן המקומיות שלה. המאגר הראשי הוא:
https://github.com/nocobase/nocobase/tree/main/locales

כל שפה מיוצגת על ידי קובץ JSON הקרוי על פי קוד השפה (לדוגמה, de-DE.json, fr-FR.json). מבנה הקובץ מאורגן לפי מודולים של תוספים, תוך שימוש בזוגות מפתח-ערך לאחסון התרגומים. לדוגמה:

```json
{
  // תוסף לקוח
  "@nocobase/client": {
    "(Fields only)": "(Fields only)",
    "12 hour": "12 hour",
    "24 hour": "24 hour"
    // ...זוגות מפתח-ערך נוספים
  },
  "@nocobase/plugin-acl": {
    // זוגות מפתח-ערך עבור תוסף זה
  }
  // ...מודולים של תוספים נוספים
}
```

בעת התרגום, אנא המירו את המבנה בהדרגה למשהו דומה לזה:

```json
{
  // תוסף לקוח
  "@nocobase/client": {
    "(Fields only)": "(שדות בלבד - תורגם)",
    "12 hour": "12 שעות",
    "24 hour": "24 שעות"
    // ...זוגות מפתח-ערך נוספים
  },
  "@nocobase/plugin-acl": {
    // זוגות מפתח-ערך עבור תוסף זה
  }
  // ...מודולים של תוספים נוספים
}
```

## ב. מקומיות התיעוד (NocoBase 2.0)

התיעוד עבור NocoBase 2.0 מנוהל במבנה חדש. קובצי המקור של התיעוד נמצאים במאגר הראשי של NocoBase:

https://github.com/nocobase/nocobase/tree/next/docs

### 2.1 מבנה התיעוד

התיעוד משתמש ב-[Rspress](https://rspress.dev/) כמחולל אתרים סטטיים ותומך ב-22 שפות. המבנה מאורגן באופן הבא:

```

├── docs/
│   ├── en/                    # אנגלית (שפת מקור)
│   ├── cn/                    # סינית מפשוטת
│   ├── ja/                    # יפנית
│   ├── ko/                    # קוריאנית
│   ├── de/                    # גרמנית
│   ├── fr/                    # צרפתית
│   ├── es/                    # ספרדית
│   ├── pt/                    # פורטוגזית
│   ├── ru/                    # רוסית
│   ├── it/                    # איטלקית
│   ├── tr/                    # טורקית
│   ├── uk/                    # אוקראינית
│   ├── vi/                    # וייטנאמית
│   ├── id/                    # אינדונזית
│   ├── th/                    # תאילנדית
│   ├── pl/                    # פולנית
│   ├── nl/                    # הולנדית

│   ├── he/                    # עברית
│   ├── hi/                    # הינדי
│   ├── sv/                    # שוודית
│   └── public/                # נכסים משותפים (תמונות וכו')
├── theme/                     # ערכת נושא מותאמת אישית
├── rspress.config.ts          # הגדרות Rspress
└── package.json
```

### 2.2 תהליך עבודה לתרגום

1. **סנכרון עם מקור אנגלי**: כל התרגומים צריכים להתבסס על התיעוד באנגלית (`docs/en/`). כאשר התיעוד באנגלית מתעדכן, יש לעדכן את התרגומים בהתאם.

2. **אסטרטגיית ענפים (Branches)**:
   - השתמשו בענף `develop` או `next` כייחוס לתוכן האנגלי העדכני ביותר.
   - צרו את ענף התרגום שלכם מתוך ענף היעד.

3. **מבנה קבצים**: כל תיקיית שפה צריכה לשקף את מבנה התיקייה האנגלי. לדוגמה:
   ```
   docs/en/get-started/index.md    →    docs/he/get-started/index.md
   docs/en/api/acl/acl.md          →    docs/he/api/acl/acl.md
   ```

### 2.3 תרומת תרגומים

1. בצעו Fork למאגר: https://github.com/nocobase/nocobase
2. שכפלו (Clone) את ה-fork שלכם ועברו לענף `develop` או `next`.
3. נווטו לתיקיית `docs/docs/`.
4. מצאו את תיקיית השפה שברצונכם לתרום לה (לדוגמה, `he/` עבור עברית).
5. תרגמו את קובצי ה-markdown, תוך שמירה על מבנה קבצים זהה לגרסה האנגלית.
6. בדקו את השינויים שלכם באופן מקומי:
   ```bash
   cd docs
   yarn install
   yarn dev
   ```
7. הגישו Pull Request למאגר הראשי.

### 2.4 הנחיות תרגום

- **שמירה על עקביות בפורמט**: שמרו על מבנה ה-markdown, הכותרות, בלוקי הקוד והקישורים כפי שהם במקור.
- **שימור frontmatter**: השאירו כל YAML frontmatter בראש הקבצים ללא שינוי, אלא אם הוא מכיל תוכן הניתן לתרגום.
- **הפניות לתמונות**: השתמשו באותם נתיבי תמונות מ-`docs/public/` - התמונות משותפות לכל השפות.
- **קישורים פנימיים**: עדכנו קישורים פנימיים כך שיצביעו לנתיב השפה הנכון.
- **דוגמאות קוד**: בדרך כלל, אין לתרגם דוגמאות קוד, אך ניתן לתרגם הערות בתוך הקוד.

### 2.5 הגדרת ניווט

מבנה הניווט לכל שפה מוגדר בקובצי `_nav.json` ו-`_meta.json` בתוך כל תיקיית שפה. בעת הוספת דפים או סעיפים חדשים, הקפידו לעדכן קובצי הגדרה אלו.

### 3.2 מבנה התוכן ושיטת המקומיות

אנו משתמשים בגישת ניהול תוכן מעורבת. תוכן ומשאבים באנגלית, סינית ויפנית מסונכרנים באופן קבוע ממערכת ה-CMS ונדרסים, בעוד ששפות אחרות ניתנות לעריכה ישירה בקבצים מקומיים. תוכן מקומי מאוחסן בתיקיית `content`, המאורגנת כך:

```
/content
  /articles        # מאמרי בלוג
    /article-slug
      index.md     # תוכן באנגלית (ברירת מחדל)
      index.cn.md  # תוכן בסינית
      index.ja.md  # תוכן ביפנית
      metadata.json # מטא-נתונים ומאפייני מקומיות אחרים
  /tutorials       # מדריכים
  /releases        # מידע על גרסאות
  /pages           # דפים סטטיים מסוימים
  /categories      # מידע על קטגוריות
    /article-categories.json  # רשימת קטגוריות מאמרים
    /category-slug            # פרטי קטגוריה בודדת
      /category.json
  /tags            # מידע על תגיות
    /article-tags.json        # רשימת תגיות מאמרים
    /release-tags.json        # רשימת תגיות גרסה
    /tag-slug                 # פרטי תגית בודדת
      /tag.json
  /help-center     # תוכן מרכז העזרה
    /help-center-tree.json    # מבנה ניווט של מרכז העזרה
  ....
```

### 3.3 הנחיות לתרגום תוכן

- לגבי תרגום תוכן Markdown

1. צרו קובץ שפה חדש המבוסס על קובץ ברירת המחדל (לדוגמה, מ-`index.md` ל-`index.he.md`).
2. הוסיפו מאפיינים מקומיים בשדות המתאימים בקובץ ה-JSON.
3. שמרו על עקביות במבנה הקבצים, בקישורים ובהפניות לתמונות.

- תרגום תוכן JSON
מטא-נתונים רבים של תוכן מאוחסנים בקובצי JSON, המכילים בדרך כלל שדות רב-לשוניים:

```json
{
  "id": 123,
  "title": "English Title",       // כותרת באנגלית (ברירת מחדל)
  "title_cn": "中文标题",          // כותרת בסינית
  "title_ja": "日本語タイトル",    // כותרת ביפנית
  "description": "English description",
  "description_cn": "中文描述",
  "description_ja": "日本語の説明",
  "slug": "article-slug",         // נתיב URL (בדרך כלל לא מתורגם)
  "status": "published",
  "publishedAt": "2025-03-19T12:00:00Z"
}
```

**הערות לתרגום:**

1. **מוסכמת שמות שדות**: שדות תרגום משתמשים בדרך כלל בפורמט `{original_field}_{language_code}`.
   - לדוגמה: title_he (כותרת בעברית), description_he (תיאור בעברית).

2. **בעת הוספת שפה חדשה**:
   - הוסיפו גרסת סיומת שפה מתאימה לכל שדה הדורש תרגום.
   - אל תשנו את ערכי השדות המקוריים (כמו title, description וכו'), שכן הם משמשים כתוכן שפת ברירת המחדל (אנגלית).

3. **מנגנון סנכרון CMS**:
   - מערכת ה-CMS מעדכנת מעת לעת תוכן באנגלית, סינית ויפנית.
   - המערכת תעדכן/תדרוס רק תוכן עבור שלוש שפות אלו (מאפיינים מסוימים ב-JSON), ו**לא תמחק** שדות שפה שנוספו על ידי תורמים אחרים.
   - לדוגמה: אם הוספתם תרגום לעברית (title_he), סנכרון ה-CMS לא ישפיע על שדה זה.


### 3.4 הגדרת תמיכה בשפה חדשה

כדי להוסיף תמיכה בשפה חדשה, עליכם לשנות את הגדרת `SUPPORTED_LANGUAGES` בקובץ `src/utils/index.ts`:

```typescript
export const SUPPORTED_LANGUAGES = {
  en: {
    code: 'en',
    locale: 'en-US',
    name: 'English',
    default: true
  },
  cn: {
    code: 'cn',
    locale: 'zh-CN',
    name: 'Chinese'
  },
  ja: {
    code: 'ja',
    locale: 'ja-JP',
    name: 'Japanese'
  },
  // דוגמה להוספת שפה חדשה:
  he: {
    code: 'he',
    locale: 'he-IL',
    name: 'Hebrew'
  }
};
```

### 3.5 קובצי פריסה (Layout) ועיצוב

כל שפה זקוקה לקובצי פריסה מתאימים:

1. צרו קובץ פריסה חדש (לדוגמה, עבור עברית, צרו את `src/layouts/BaseHE.astro`).
2. ניתן להעתיק קובץ פריסה קיים (כגון `BaseEN.astro`) ולתרגם אותו.
3. קובץ הפריסה מכיל תרגומים לאלמנטים גלובליים כמו תפריטי ניווט, כותרות תחתונות (footers) וכו'.
4. הקפידו לעדכן את הגדרת בורר השפות כדי לעבור כראוי לשפה החדשה שנוספה.

### 3.6 יצירת תיקיות דפי שפה

צרו תיקיות דפים עצמאיות עבור השפה החדשה:

1. צרו תיקייה הקרויה על שם קוד השפה בתוך תיקיית `src` (לדוגמה, `src/he/`).
2. העתיקו את מבנה הדפים מתיקיות שפה אחרות (לדוגמה, מ-`src/en/`).
3. עדכנו את תוכן הדפים, תרגמו כותרות, תיאורים וטקסט לשפת היעד.
4. ודאו שהדפים משתמשים ברכיב הפריסה הנכון (לדוגמה, `.layout: '@/layouts/BaseHE.astro'`).

### 3.7 מקומיות רכיבים

חלק מהרכיבים המשותפים זקוקים גם הם לתרגום:

1. בדקו רכיבים בתיקיית `src/components/`.
2. שימו לב במיוחד לרכיבים עם טקסט קבוע (כמו סרגלי ניווט, כותרות תחתונות וכו').
3. רכיבים עשויים להשתמש ברינדור מותנה (conditional rendering) כדי להציג תוכן בשפות שונות:

```astro
{Astro.url.pathname.startsWith('/en') && <p>English content</p>}
{Astro.url.pathname.startsWith('/cn') && <p>中文内容</p>}
{Astro.url.pathname.startsWith('/he') && <p>תוכן בעברית</p>}
```

### 3.8 בדיקה ואימות

לאחר השלמת התרגום, בצעו בדיקה מקיפה:

1. הריצו את האתר באופן מקומי (בדרך כלל באמצעות `yarn dev`).
2. בדקו כיצד כל הדפים מוצגים בשפה החדשה.
3. ודאו שפונקציונליות החלפת השפה פועלת כראוי.
4. ודאו שכל הקישורים מצביעים לדפי גרסת השפה הנכונה.
5. בדקו פריסות רספונסיביות כדי לוודא שהטקסט המתורגם אינו שובר את עיצוב הדף.

## ד. איך להתחיל לתרגם

| רכיב | מאגר (Repository) | ענף (Branch) | הערות |

לאחר השלמת התרגום, אנא הגישו Pull Request ל-NocoBase. השפות החדשות יופיעו בהגדרות המערכת, ויאפשרו לכם לבחור אילו שפות להציג.
