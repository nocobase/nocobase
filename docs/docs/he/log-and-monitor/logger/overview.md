# יומני שרת, יומני ביקורת והיסטוריית רשומות

## יומני שרת

### יומני מערכת

> ראו [יומני מערכת](./index.md#system-logs)

- מתעדים מידע על פעולת מערכת היישום, עוקבים אחר שרשרת הביצוע של לוגיקת הקוד, ומאתרים חריגות או שגיאות זמן ריצה.
- מקוטלגים לפי רמות חומרה ומודולים פונקציונליים.
- פלט דרך מסוף (טרמינל) או נשמרים כקובץ.
- משמשים בעיקר לאבחון ופתרון בעיות מערכת המתרחשות במהלך הפעולה.

### יומני בקשות

> ראו [יומני בקשות](./index.md#request-logs)

- מתעדים פרטי בקשות ותגובות של HTTP API, ומתמקדים בתיעוד מזהה הבקשה (Request ID), נתיב ה-API (API Path), כותרות הבקשה (Request Headers), קוד סטטוס התגובה (Response Status Code), וזמן הביצוע (Duration).
- פלט דרך מסוף (טרמינל) או נשמרים כקובץ.
- משמשים בעיקר למעקב אחר קריאות API וביצועיהן.

## יומני ביקורת

> ראו [יומני ביקורת](/security/audit-logger/index.md)

- מתעדים פעולות משתמשים (או API) על משאבי המערכת, ומתמקדים בתיעוד סוג המשאב, אובייקט המשאב, סוג הפעולה, פרטי המשתמש וסטטוס הפעולה.
- כדי לעקוב טוב יותר אחר התוכן והתוצאות הספציפיות של פעולות משתמשים, פרמטרי הבקשה והתגובות נשמרים כמידע MetaData. מידע זה חופף חלקית ליומני בקשות אך אינו זהה לחלוטין – לדוגמה, יומני בקשות בדרך כלל אינם כוללים את גוף הבקשה המלא.
- פרמטרי בקשה ותגובות **אינם שווי ערך** לתמונות מצב של נתונים (data snapshots). הם יכולים לחשוף איזה סוג של פעולות התרחשו, אך לא את הנתונים המדויקים לפני השינוי, ולכן אינם יכולים לשמש לבקרת גרסאות או לשחזור נתונים לאחר פעולות שגויות.
- נשמרים הן כקובץ והן בטבלאות מסד נתונים.

## היסטוריית רשומות

> ראו [היסטוריית רשומות](/record-history/index.md)

- מתעדת את **היסטוריית השינויים** של תוכן הנתונים.
- התוכן העיקרי המתועד כולל סוג משאב, אובייקט משאב, סוג פעולה, שדות ששונו, וערכים לפני/אחרי השינוי.
- שימושי **להשוואת נתונים וביקורת**.
- נשמרת בטבלאות מסד נתונים.
