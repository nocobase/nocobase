# יומני שרת, יומני ביקורת והיסטוריית רשומות

## יומני שרת

### יומני מערכת

> ראו [יומני מערכת](#)

- מתעדים מידע על פעילות מערכת היישום, עוקבים אחר שרשרת הביצוע של לוגיקת הקוד, ומאתרים חריגות או שגיאות זמן ריצה.
- היומנים מדורגים לפי רמות חומרה ומסווגים לפי מודולים פונקציונליים.
- פלט דרך הטרמינל או נשמרים כקובץ.
- משמשים בעיקר לאבחון ופתרון תקלות מערכת במהלך פעולתה.

### יומני בקשות

> ראו [יומני בקשות](#)

- מתעדים פרטי בקשות ותגובות של HTTP API, תוך התמקדות בתיעוד מזהה בקשה, נתיב API, כותרות, קוד סטטוס תגובה וזמן ביצוע.
- פלט דרך הטרמינל או נשמרים כקובץ.
- משמשים בעיקר למעקב אחר קריאות API וביצועיהן.

## יומני ביקורת

> ראו [יומני ביקורת](../security/audit-logger/index.md)

- מתעדים פעולות משתמשים (או API) על משאבי המערכת, תוך התמקדות בסוג המשאב, אובייקט היעד, סוג הפעולה, פרטי המשתמש וסטטוס הפעולה.
- כדי לעקוב טוב יותר אחר התוכן והתוצאות הספציפיות של פעולות המשתמש, פרמטרי הבקשה והתגובות נשמרים כנתוני MetaData. מידע זה חופף חלקית ליומני בקשות אך אינו זהה לחלוטין – לדוגמה, יומני בקשות בדרך כלל אינם כוללים את גוף הבקשה המלא.
- פרמטרי בקשה ותגובות **אינם שווי ערך** לתמונות מצב של נתונים. הם יכולים לחשוף אילו סוגי פעולות התרחשו, אך לא את הנתונים המדויקים לפני השינוי, ולכן לא ניתן להשתמש בהם לבקרת גרסאות או לשחזור נתונים לאחר פעולות שגויות.
- נשמרים הן כקובץ והן בטבלאות מסד נתונים.

## היסטוריית רשומות

> ראו [היסטוריית רשומות](/record-history/index.md)

- מתעד את **היסטוריית השינויים** של תוכן הנתונים.
- התוכן העיקרי המתועד כולל סוג משאב, אובייקט משאב, סוג פעולה, שדות ששונו, וערכים לפני/אחרי השינוי.
- שימושי ל**השוואת נתונים וביקורת**.
- נשמר בטבלאות מסד נתונים.
