# תצורה מתקדמת

## מצב ביצוע

תהליכי עבודה מבוצעים באופן אסינכרוני או סינכרוני, בהתאם לסוג הטריגר שנבחר בעת יצירתם. במצב אסינכרוני, לאחר הפעלת אירוע ספציפי, תהליך העבודה נכנס לתור ומבוצע אחד אחד על ידי תזמון ברקע. לעומת זאת, במצב סינכרוני, תהליך העבודה אינו נכנס לתור התזמון לאחר הפעלתו; הוא מתחיל להתבצע ישירות ומספק משוב מיידי עם השלמתו.

אירועי אוסף, אירועים לאחר פעולה, אירועי פעולה מותאמים אישית, אירועים מתוזמנים ואירועי אישור מבוצעים אסינכרונית כברירת מחדל. אירועים לפני פעולה מבוצעים סינכרונית כברירת מחדל. גם אירועי אוסף וגם אירועי טופס תומכים בשני המצבים, וניתן לבחור ביניהם בעת יצירת תהליך עבודה:

![מצב סינכרוני_יצירת תהליך עבודה סינכרוני](https://static-docs.nocobase.com/39bc0821f50c1bde4729c531c6236795.png)

:::info{title=הערה}
בשל אופיים, תהליכי עבודה סינכרוניים אינם יכולים להשתמש בצמתים המייצרים מצב "המתנה", כגון "טיפול ידני".
:::

## מחיקה אוטומטית של היסטוריית ביצועים

כאשר תהליך עבודה מופעל לעיתים קרובות, ניתן להגדיר מחיקה אוטומטית של היסטוריית ביצועים כדי להפחית עומס ולהקל על לחץ האחסון במסד הנתונים.

ניתן גם להגדיר אם למחוק אוטומטית את היסטוריית הביצועים של תהליך עבודה, וזאת בחלונות הקופצים ליצירה ועריכה שלו:

![הגדרת מחיקה אוטומטית של היסטוריית ביצועים](https://static-docs.nocobase.com/b2e4c08e7a01e213069912fe04baa7bd.png)

מחיקה אוטומטית ניתנת להגדרה בהתאם למצב תוצאת הביצוע. ברוב המקרים, מומלץ לסמן רק את מצב "הושלם" כדי לשמור תיעוד של ביצועים שנכשלו, לצורך פתרון בעיות עתידי.

מומלץ לא להפעיל מחיקה אוטומטית של היסטוריית ביצועים בעת איתור באגים בתהליך עבודה, כדי שתוכלו להשתמש בהיסטוריה לבדיקה אם לוגיקת הביצוע של תהליך העבודה תואמת את הציפיות.

:::info{title=הערה}
מחיקת היסטוריית תהליך עבודה אינה מפחיתה את ספירת הביצועים שלו.
:::
