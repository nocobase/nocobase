# ניהול גרסאות

לאחר שתהליך עבודה מוגדר הופעל לפחות פעם אחת, אם ברצונכם לשנות את התצורה שלו או את הצמתים שבו, עליכם ליצור גרסה חדשה לפני ביצוע השינויים. פעולה זו מבטיחה גם שבעת סקירת היסטוריית ההפעלות של תהליכי עבודה שהופעלו בעבר, הם לא יושפעו משינויים עתידיים.

בעמוד התצורה של תהליך העבודה, תוכלו לראות את גרסאות תהליך העבודה הקיימות בתפריט הגרסאות בפינה הימנית העליונה:

![הצגת גרסאות תהליך עבודה](https://static-docs.nocobase.com/ad93d2c08166b0e3e643fb148713a63f.png)

בתפריט "פעולות נוספות" ("...") שמימין לו, תוכלו לבחור לשכפל את הגרסה הנצפית כעת לגרסה חדשה:

![שכפול תהליך עבודה לגרסה חדשה](https://static-docs.nocobase.com/2805798e6caca2af004893390a744256.png)

לאחר השכפול לגרסה חדשה, לחצו על מתג "הפעלה"/"השבתה" כדי להעביר את הגרסה המתאימה למצב פעיל, וגרסת תהליך העבודה החדשה תיכנס לתוקף.

אם עליכם לבחור מחדש גרסה ישנה, עברו אליה מתפריט הגרסאות, ולאחר מכן לחצו שוב על מתג "הפעלה"/"השבתה" כדי להעביר אותה למצב פעיל. הגרסה הנצפית כעת תיכנס לתוקף, והפעלות עתידיות יבצעו את התהליך של אותה גרסה.

כאשר עליכם להשבית תהליך עבודה, לחצו על מתג "הפעלה"/"השבתה" כדי להעביר אותו למצב מושבת, ותהליך העבודה לא יופעל יותר.

:::info{title=טיפ}
בניגוד ל"שכפול" תהליך עבודה מרשימת ניהול תהליכי העבודה, תהליך עבודה ש"שוכפל לגרסה חדשה" עדיין מקובץ תחת אותו סט תהליכי עבודה, וניתן להבחין ביניהם רק לפי גרסה. עם זאת, שכפול תהליך עבודה נחשב לתהליך עבודה חדש לחלוטין, שאינו קשור לגרסאות של תהליך העבודה הקודם, ומספר ההפעלות שלו יתאפס.
:::
