# סקירה כללית

הפונקציונליות המובנית של תהליכי עבודה אינה יכולה לכסות את כל התרחישים. לדוגמה, סוגי הצמתים המובנים אינם יכולים למצות כל פעולה אפשרית בכל תרחיש עסקי. לכן, אנו מציעים גם תכנון להרחבת תהליכי העבודה, הכולל הרחבת סוגי טריגרים וצמתים. בתרחישים עסקיים שבהם הפונקציונליות המובנית אינה מספקת, ניתן לפתור זאת באמצעות הרחבות בשיטות low-code.

ההרחבות מתחלקות לשני חלקים עיקריים:

- [הרחבת סוגי טריגרים](./trigger.md)
- [הרחבת סוגי צמתים](./node.md)
