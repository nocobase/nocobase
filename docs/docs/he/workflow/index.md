# סקירה כללית

## מבוא

תוסף תהליך העבודה (Workflow) עוזר לכם לתזמר תהליכים עסקיים אוטומטיים ב-NocoBase, כגון אישורים יומיומיים, סנכרון נתונים, תזכורות ומשימות נוספות. בתהליך עבודה, אתם יכולים ליישם לוגיקה עסקית מורכבת פשוט על ידי הגדרת טריגרים וצמתים קשורים דרך ממשק ויזואלי, ללא צורך בכתיבת קוד כלל.

### דוגמה

כל תהליך עבודה מתזומר באמצעות טריגר אחד ומספר צמתים. הטריגר מייצג אירוע במערכת, וכל צומת מייצג שלב ביצוע. יחד, הם מתארים את הלוגיקה העסקית שיש לעבד לאחר התרחשות האירוע. התמונה הבאה מציגה תהליך טיפוסי של הפחתת מלאי לאחר ביצוע הזמנת מוצר:

![דוגמת תהליך עבודה](https://static-docs.nocobase.com/20251029222146.png)

כאשר משתמש מגיש הזמנה, תהליך העבודה בודק אוטומטית את המלאי. אם המלאי מספיק, הוא מפחית את המלאי וממשיך ביצירת ההזמנה; אחרת, התהליך מסתיים.

### תרחישי שימוש

מנקודת מבט כללית יותר, תהליכי עבודה ביישומי NocoBase יכולים לפתור בעיות במגוון תרחישים:

-   אוטומציה של משימות חוזרות: ביקורות הזמנות, סנכרון מלאי, ניקוי נתונים, חישובי ניקוד ועוד, מבטלים את הצורך בפעולה ידנית.
-   תמיכה בשיתוף פעולה בין אדם למכונה: קביעת אישורים או ביקורות בצמתים מרכזיים, והמשך לשלבים הבאים בהתבסס על התוצאות.
-   חיבור למערכות חיצוניות: שליחת בקשות HTTP, קבלת התראות (pushes) משירותים חיצוניים, והשגת אוטומציה חוצת מערכות.
-   התאמה מהירה לשינויים עסקיים: התאמת מבנה התהליך, תנאים או הגדרות צמתים אחרות, ועלייה לאוויר ללא צורך בגרסה חדשה (release).

## התקנה

תהליך העבודה הוא תוסף מובנה של NocoBase. אין צורך בהתקנה או תצורה נוספת.
