# תגובת HTTP

## מבוא

צומת זה נתמך רק בתהליכי עבודה (Workflow) של Webhook במצב סינכרוני, והוא משמש להחזרת תגובה למערכת צד שלישי. לדוגמה, במהלך עיבוד קריאה חוזרת (callback) של תשלום, אם תהליך עסקי נתקל בתוצאה בלתי צפויה (כגון שגיאה או כשל), ניתן להשתמש בצומת התגובה כדי להחזיר תגובת שגיאה למערכת צד שלישי, כך שמערכות צד שלישי מסוימות יוכלו לנסות שוב מאוחר יותר בהתבסס על הסטטוס.

בנוסף, ביצוע צומת התגובה יסיים את ביצוע תהליך העבודה, וצמתים עוקבים לא יבוצעו. אם לא הוגדר צומת תגובה בכל תהליך העבודה, המערכת תגיב אוטומטית בהתבסס על סטטוס ביצוע התהליך: תחזיר `200` עבור ביצוע מוצלח ו-`500` עבור ביצוע כושל.
