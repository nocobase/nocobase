# סיום תהליך עבודה

כאשר צומת זה מופעל, הוא מסיים באופן מיידי את תהליך העבודה הנוכחי, וקובע את סטטוס הסיום בהתאם להגדרות הצומת. שימוש אופייני בו הוא לבקרת זרימה לוגית: כאשר מתקיימים תנאים לוגיים מסוימים, הוא מאפשר לצאת מתהליך העבודה הנוכחי ולהפסיק את ביצוע השלבים הבאים. ניתן להשוות אותו לפקודת ה-`return` בשפות תכנות, המשמשת ליציאה מהפונקציה הנוכחית.

## הוספת צומת

בממשק הגדרות תהליך העבודה, לחצו על כפתור הפלוס ("+") בזרימה כדי להוסיף צומת "סיום תהליך עבודה":

![סיום תהליך עבודה_הוספה](https://static-docs.nocobase.com/672186ab4c8f7313dd3cf9c880b524b8.png)

### סטטוס סיום

סטטוס הסיום ישפיע על הסטטוס הסופי של ביצוע תהליך העבודה. ניתן להגדיר אותו כ"הצליח" או "נכשל". כאשר תהליך העבודה מגיע לצומת זה, הוא ייצא מיד עם הסטטוס שהוגדר.

:::info{title=הערה}
כאשר משתמשים בו בתהליך עבודה מסוג "אירוע לפני פעולה", הוא יגרום ליירוט הבקשה שיזמה את הפעולה. לפרטים נוספים, עיינו ב[הוראות השימוש ב"אירוע לפני פעולה"](../triggers/pre-action).

בנוסף ליירוט הבקשה שיזמה את הפעולה, הגדרת סטטוס הסיום תשפיע גם על סטטוס המידע שמוחזר ב"הודעת תגובה" בתהליכי עבודה מסוג זה.
:::
