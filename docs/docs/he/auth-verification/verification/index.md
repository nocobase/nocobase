# אימות

:::info{title=הערה}
החל מגרסה `1.6.0-alpha.30`, תכונת "קוד האימות" המקורית שודרגה ל"ניהול אימותים". היא תומכת בניהול ושילוב שיטות שונות לאימות זהות משתמשים. לאחר שמשתמשים מקשרים את שיטת האימות המתאימה, הם יכולים לבצע אימות זהות בעת הצורך. תכונה זו מתוכננת לתמיכה יציבה החל מגרסה `1.7.0`.
:::

## מבוא

**מרכז ניהול האימותים תומך בניהול ושילוב שיטות שונות לאימות זהות משתמשים.** לדוגמה:

- קוד אימות ב-SMS – מסופק כברירת מחדל על ידי תוסף האימות. ראו: [אימות: SMS](./sms)
- מאמת TOTP – ראו: [אימות: מאמת TOTP](../verification-totp/)

מפתחים יכולים גם להרחיב סוגי אימות נוספים באמצעות תוספים. ראו: [הרחבת סוגי אימות](./dev/type)

**משתמשים יכולים לבצע אימות זהות בעת הצורך, לאחר קישור שיטת האימות המתאימה.** לדוגמה:

- התחברות באמצעות קוד אימות ב-SMS – ראו: [אימות: SMS](../auth-sms/index.md)
- אימות דו-שלבי (2FA) – ראו: [אימות דו-שלבי (2FA)](../2fa/)
- אימות משני לפעולות בסיכון – תמיכה עתידית

מפתחים יכולים גם לשלב אימות זהות בתרחישים נחוצים אחרים באמצעות הרחבת תוספים. ראו: [הרחבת תרחישי אימות](./dev/scene)

**הבדלים וקשרים בין מודול האימות למודול אימות המשתמשים:** מודול אימות המשתמשים אחראי בעיקר על אימות זהות בתרחישי התחברות משתמשים, כאשר תהליכים כמו התחברות ב-SMS ואימות דו-שלבי מסתמכים על מאמתים המסופקים על ידי מודול האימות. מודול האימות, לעומת זאת, מטפל באימות זהות עבור פעולות שונות בסיכון גבוה, והתחברות משתמשים היא אחד מהתרחישים הללו.
