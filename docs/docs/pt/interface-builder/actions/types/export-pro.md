# Exportar Pro

## Introdução

O plugin Exportar Pro oferece funcionalidades aprimoradas em relação à função de exportação padrão.

## Instalação

Este plugin depende do plugin de Gerenciamento de Tarefas Assíncronas. Você precisa habilitar o plugin de Gerenciamento de Tarefas Assíncronas antes de usá-lo.

## Melhorias de Funcionalidade

- Suporta operações de exportação assíncronas, executadas em um thread separado, para exportar grandes volumes de dados.
- Suporta exportação de anexos.

## Guia do Usuário

### Configurar Modo de Exportação

![20251029172829](https://static-docs.nocobase.com/20251029172829.png)

![20251029172914](https://static-docs.nocobase.com/20251029172914.png)

No botão de exportação, você pode configurar o modo de exportação. Existem três modos opcionais:

- **Automático**: O modo de exportação é determinado pelo volume de dados. Se o número de registros for inferior a 1000 (ou 100 para exportações de anexos), a exportação síncrona é utilizada. Se for superior a 1000 (ou 100 para exportações de anexos), a exportação assíncrona é utilizada.
- **Síncrono**: Utiliza a exportação síncrona, que é executada no thread principal. É adequado para pequenos volumes de dados. Exportar grandes volumes de dados no modo síncrono pode causar bloqueios, lentidão no sistema e impedir o processamento de outras solicitações de usuários.
- **Assíncrono**: Utiliza a exportação assíncrona, que é executada em um thread de segundo plano separado e não bloqueia a operação atual do sistema.

### Exportação Assíncrona

Após iniciar uma exportação, o processo será executado em um thread de segundo plano separado, sem a necessidade de configuração manual pelo usuário. Na interface do usuário, após iniciar uma operação de exportação, a tarefa de exportação em execução será exibida no canto superior direito, mostrando o progresso em tempo real.

![20251029173028](https://static-docs.nocobase.com/20251029173028.png)

Após a conclusão da exportação, você pode baixar o arquivo exportado nas tarefas de exportação.

#### Exportações Concorrentes
Um grande número de tarefas de exportação concorrentes pode ser afetado pela configuração do servidor, resultando em uma resposta mais lenta do sistema. Por isso, é recomendado que os desenvolvedores do sistema configurem o número máximo de tarefas de exportação concorrentes (o padrão é 3). Quando o número de tarefas concorrentes exceder o limite configurado, as novas tarefas entrarão em estado de fila.

![20250505171706](https://static-docs.nocobase.com/20250505171706.png)

Método de configuração de concorrência: Variável de ambiente `ASYNC_TASK_MAX_CONCURRENCY=numero_de_concorrencias`

Com base em testes abrangentes com diferentes configurações e complexidades de dados, os números de concorrência recomendados são:
- CPU de 2 núcleos, concorrência 3.
- CPU de 4 núcleos, concorrência 5.

#### Sobre o Desempenho
Quando você notar que o processo de exportação está anormalmente lento (veja a referência abaixo), pode ser um problema de desempenho causado pela estrutura da `coleção`.

| Características dos Dados | Tipo de Índice | Volume de Dados | Duração da Exportação |
|---------------------------|------------------------|-----------------|-----------------------|
| Sem Campos de Relacionamento | Chave Primária / Restrição Única | 1 milhão | 3～6 minutos |
| Sem Campos de Relacionamento | Índice Regular | 1 milhão | 6～10 minutos |
| Sem Campos de Relacionamento | Índice Composto (não único) | 1 milhão | 30 minutos |

1. A `coleção` deve atender às seguintes condições:
