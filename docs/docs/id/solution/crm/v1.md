# CRM Versi Lama (v1)

> **Hanya untuk pratinjau**: Versi ini cukup lengkap secara fungsional, tetapi desain tabel data secara keseluruhan tidak ideal, kurang dukungan lokalisasi multi-bahasa, dan tabel CRM memiliki keterkaitan dengan tabel manajemen proyek. Masalah-masalah di atas mendorong perancangan ulang CRM 2.0. Saat ini direkomendasikan menggunakan [versi CRM 2.0](./index.md), dengan struktur tabel data yang lebih baik, mudah diperluas dan dimodifikasi, serta sudah selesai dilokalisasi multi-bahasa.

## Karakteristik Versi

CRM versi lama mencakup modul fungsi berikut:

- **Manajemen Pelanggan**: Informasi Pelanggan, Kontak, klasifikasi Pelanggan
- **Sales Pipeline**: Manajemen Lead, follow-up Peluang, sales funnel
- **Manajemen Pesanan**: Penawaran, Pesanan, kontrak
- **Manajemen Proyek** (versi v2 belum mencakup)
- **Komunikasi & Kolaborasi**: Komunikasi email, komentar, alur persetujuan
- **Analisis Data**: Dashboard, laporan, visualisasi data

## Cara Instalasi

> Versi saat ini menggunakan format **restore backup** untuk deployment.

Sebelum memulai, pastikan:

- Anda sudah memiliki lingkungan NocoBase yang berjalan. Untuk instalasi sistem utama, silakan merujuk ke [dokumentasi instalasi resmi](https://docs-cn.nocobase.com/welcome/getting-started/installation) yang lebih detail.
- Versi NocoBase **v2.1.0-beta.2 atau lebih tinggi**
- Anda telah mengunduh file yang sesuai untuk CRM v1:
  - **File backup**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — untuk metode satu
  - **File SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — untuk metode dua

**Catatan Penting**:
- Solusi ini dibuat berdasarkan database **PostgreSQL 16**, pastikan lingkungan Anda menggunakan PostgreSQL 16.
- **DB_UNDERSCORED tidak boleh true**: Periksa file `docker-compose.yml` Anda, pastikan environment variable `DB_UNDERSCORED` tidak diatur ke `true`, jika tidak akan konflik dengan backup solusi dan menyebabkan kegagalan restore.

## Metode Satu: Restore dengan Backup Manager (Direkomendasikan untuk Pengguna Pro/Enterprise)

Metode ini menggunakan plugin "[Backup Manager](https://docs-cn.nocobase.com/handbook/backups)" (Pro/Enterprise) bawaan NocoBase untuk restore satu klik, paling sederhana untuk dilakukan.

### Langkah-langkah Operasi

**Langkah 1: Aktifkan Plugin "Backup Manager"**

1. Login ke sistem NocoBase Anda.
2. Masuk ke **`Plugin Management`**.
3. Cari dan aktifkan Plugin **`Backup Manager`**.

**Langkah 2: Restore dari File Backup Lokal**

1. Setelah plugin diaktifkan, refresh halaman.
2. Masuk ke menu kiri **`System Management`** -> **`Backup Manager`**.
3. Klik tombol **`Restore from Local Backup`** di pojok kanan atas.
4. Drag file backup yang sudah diunduh ke area upload.
5. Klik **`Submit`**, tunggu dengan sabar hingga sistem menyelesaikan restore.

### Perhatian

* **Khusus Pro/Enterprise**: "Backup Manager" adalah plugin tingkat enterprise, hanya tersedia untuk pengguna Pro/Enterprise.
* **Pencocokan Plugin Komersial**: Pastikan Anda sudah memiliki dan mengaktifkan Plugin komersial yang dibutuhkan oleh solusi.

## Metode Dua: Impor Langsung File SQL (Universal)

Metode ini me-restore data dengan mengoperasikan database secara langsung, berlaku untuk semua pengguna NocoBase.

### Langkah-langkah Operasi

**Langkah 1: Siapkan Database Bersih**

Siapkan database baru yang kosong untuk data yang akan diimpor.

**Langkah 2: Impor File `.sql` ke Database**

* **Melalui command line (contoh Docker)**:

  ```bash
  # Salin file sql ke dalam container
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # Masuk ke container untuk menjalankan perintah impor
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Melalui client database**: Gunakan tool seperti DBeaver, Navicat, pgAdmin untuk terhubung ke database, lalu eksekusi file SQL.

**Langkah 3: Sambungkan Database dan Jalankan Aplikasi**

Konfigurasikan parameter startup NocoBase agar mengarah ke database yang sudah diimpor datanya, lalu jalankan layanan.
