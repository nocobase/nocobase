# CRM Versi Lama (v1)

> **Hanya untuk Pratinjau**: Versi ini memiliki fitur yang cukup lengkap, namun desain struktur data secara keseluruhan kurang optimal, kurangnya dukungan lokalisasi multibahasa, dan terdapat keterkaitan (coupling) antara koleksi CRM dengan koleksi manajemen proyek. Masalah-masalah tersebut mendorong perancangan ulang CRM 2.0. Saat ini direkomendasikan untuk menggunakan [CRM Versi 2.0](./index.md), yang memiliki struktur data lebih rasional, lebih mudah diperluas dan dimodifikasi, serta telah menyelesaikan lokalisasi multibahasa.

## Fitur Versi

CRM versi lama mencakup modul fungsional berikut:

- **Manajemen Pelanggan**: Informasi pelanggan, kontak, dan klasifikasi pelanggan.
- **Pipeline Penjualan**: Manajemen prospek (lead), pelacakan peluang, dan corong penjualan.
- **Manajemen Pesanan**: Penawaran harga, pesanan, dan kontrak.
- **Manajemen Proyek** (Belum disertakan dalam versi v2).
- **Komunikasi & Kolaborasi**: Komunikasi email, komentar, dan alur kerja persetujuan.
- **Analisis Data**: Dasbor, laporan, dan visualisasi data.

## Cara Instalasi

> Versi saat ini dideploy melalui metode pencadangan (backup) dan pemulihan (restore).

Sebelum memulai, pastikan:

- Anda sudah memiliki lingkungan runtime NocoBase dasar. Untuk instalasi sistem utama, silakan merujuk ke [Dokumentasi Instalasi Resmi](https://docs-cn.nocobase.com/welcome/getting-started/installation) yang lebih detail.
- Versi NocoBase **v2.1.0-beta.2 atau lebih tinggi**.
- Anda telah mengunduh berkas CRM v1 yang sesuai:
  - **Berkas cadangan**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — Berlaku untuk Metode 1.
  - **Berkas SQL**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — Berlaku untuk Metode 2.

**Catatan Penting**:
- Solusi ini dibuat berdasarkan database **PostgreSQL 16**. Pastikan lingkungan Anda menggunakan PostgreSQL 16.
- **DB_UNDERSCORED tidak boleh bernilai true**: Silakan periksa berkas `docker-compose.yml` Anda dan pastikan variabel lingkungan `DB_UNDERSCORED` tidak diatur ke `true`. Jika tidak, hal ini akan berkonflik dengan cadangan solusi dan menyebabkan kegagalan pemulihan.

## Metode 1: Pulihkan menggunakan Manajer Cadangan (Direkomendasikan untuk pengguna Pro/Enterprise)

Metode ini menggunakan plugin bawaan NocoBase "**Manajer Cadangan**" (Pro/Enterprise) untuk pemulihan satu klik, yang merupakan cara termudah.

### Langkah-langkah

**Langkah 1: Aktifkan plugin "Manajer Cadangan"**

1. Masuk ke sistem NocoBase Anda.
2. Buka **`Manajer Plugin`**.
3. Temukan dan aktifkan plugin **`Manajer Cadangan`**.

**Langkah 2: Pulihkan dari berkas cadangan lokal**

1. Setelah mengaktifkan plugin, muat ulang halaman.
2. Buka menu sebelah kiri **`Administrasi Sistem`** -> **`Manajer Cadangan`**.
3. Klik tombol **`Pulihkan dari cadangan lokal`** di sudut kanan atas.
4. Seret dan lepas berkas cadangan yang telah diunduh ke area unggah.
5. Klik **`Kirim`** dan tunggu dengan sabar hingga sistem menyelesaikan pemulihan.

### Catatan

* **Khusus Edisi Pro/Enterprise**: "Manajer Cadangan" adalah plugin tingkat perusahaan dan hanya tersedia untuk pengguna Pro/Enterprise.
* **Kesesuaian Plugin Komersial**: Pastikan Anda telah memiliki dan mengaktifkan plugin komersial yang diperlukan untuk solusi ini.

## Metode 2: Impor berkas SQL secara langsung (Umum)

Metode ini memulihkan data dengan mengoperasikan database secara langsung dan berlaku untuk semua pengguna NocoBase.

### Langkah-langkah

**Langkah 1: Siapkan database yang bersih**

Siapkan database baru yang kosong untuk data yang akan Anda impor.

**Langkah 2: Impor berkas `.sql` ke dalam database**

* **Melalui Baris Perintah (Contoh Docker)**:

  ```bash
  # Salin berkas sql ke dalam kontainer
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # Masuk ke kontainer dan jalankan perintah impor
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Melalui Klien Database**: Gunakan alat seperti DBeaver, Navicat, atau pgAdmin untuk terhubung ke database dan mengeksekusi berkas SQL.

**Langkah 3: Hubungkan ke database dan jalankan aplikasi**

Konfigurasikan parameter startup NocoBase agar mengarah ke database tempat data diimpor, lalu jalankan layanan.
