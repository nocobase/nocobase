# Instalasi Sumber Kode Git

:::tip Prasyarat

- Terinstal Git, Node.js 20+, Yarn 1.22.x
- Mengonfigurasi dan menjalankan salah satu database yang diperlukan: MySQL 8.0.17+, MariaDB 10.9+, PostgreSQL 10+
- Karena alasan lingkungan jaringan domestik, disarankan bagi Anda untuk mengganti ke mirror domestik (tidak wajib)

```bash
$ yarn config set disable-self-update-check true
$ yarn config set registry https://registry.npmmirror.com/
```

:::

## 1. Mengunduh NocoBase ke lokal

Pilih versi NocoBase ([Perbandingan versi](../quickstart.md)), jalankan perintah yang sesuai.

<Tabs>
<Tab label="Versi Latest">

```bash
git clone https://github.com/nocobase/nocobase.git -b main --depth=1 my-nocobase
```

</Tab>
<Tab label="Versi Beta">

```bash
git clone https://github.com/nocobase/nocobase.git -b next --depth=1 my-nocobase
```

</Tab>
<Tab label="Versi Alpha">

```bash
git clone https://github.com/nocobase/nocobase.git -b develop --depth=1 my-nocobase
```

</Tab>
</Tabs>

## 2. Berpindah direktori

```bash
cd my-nocobase
```

## 3. Menginstal dependensi

📢 Karena pengaruh faktor lingkungan jaringan, konfigurasi sistem, dll., langkah selanjutnya ini mungkin memerlukan waktu belasan menit.

```bash
yarn install --frozen-lockfile
```

## 4. Mengatur variabel lingkungan

Variabel lingkungan yang diperlukan NocoBase disimpan dalam berkas `.env` di direktori root, ubah variabel lingkungan sesuai dengan situasi aktual.

```bash
TZ=Asia/Jakarta
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning Penjelasan variabel lingkungan

- `TZ` digunakan untuk mengatur zona waktu aplikasi, defaultnya adalah zona waktu sistem operasi;
- `APP_KEY` adalah kunci rahasia aplikasi, digunakan untuk menghasilkan token pengguna, dll. (Jika `APP_KEY` diubah, token lama juga akan menjadi tidak valid). Ini bisa berupa string acak apa pun. Harap ubah ke kunci rahasia Anda sendiri, dan pastikan tidak bocor ke luar;
- `DB_*` adalah hal-hal yang berkaitan dengan database, jika bukan layanan database default dalam contoh, harap ubah sesuai dengan situasi aktual.

:::

## 5. Menginstal NocoBase

```bash
yarn nocobase install --lang=id-ID
```

## 6. Menjalankan NocoBase


**Lingkungan pengembangan**

```bash
yarn dev
```

**Lingkungan produksi (tidak direkomendasikan)**

Tidak disarankan untuk menggunakan penyebaran sumber kode secara langsung di lingkungan produksi, untuk lingkungan produksi silakan merujuk ke [Penyebaran lingkungan produksi](../deployment/production.md).

```bash
yarn build
yarn start
```

## 7. Masuk ke NocoBase

Gunakan browser untuk membuka [http://localhost:13000](http://localhost:13000), akun dan kata sandi inisialisasi adalah `admin@nocobase.com` dan `admin123`.

:::warning Petunjuk keamanan akun

Setelah masuk pertama kali, harap segera ubah kata sandi default untuk memastikan keamanan sistem.

:::
