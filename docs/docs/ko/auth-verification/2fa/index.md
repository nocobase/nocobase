# 이중 인증 (2FA)

## 소개

이중 인증 (2FA)은 애플리케이션 로그인 시 사용되는 추가적인 보안 인증 수단입니다. 애플리케이션에서 2FA를 활성화하면, 사용자는 비밀번호로 로그인할 때 OTP 코드, TOTP 등 다른 인증 방식을 제공해야 합니다.

:::info{title=팁}
현재 2FA 프로세스는 비밀번호 기반 로그인에만 적용됩니다. 애플리케이션에서 SSO와 같은 다른 인증 방식을 활성화한 경우, 해당 IdP(Identity Provider)에서 제공하는 다단계 인증(MFA) 보호 조치를 사용해 주세요.
:::

## 관리자 설정

플러그인을 활성화하면 인증기 관리 페이지에 2FA 설정 페이지가 추가됩니다.

관리자는 "모든 사용자에 대해 이중 인증 (2FA) 활성화" 옵션을 선택해야 합니다. 또한, 사용 가능한 유형의 인증기를 선택하여 연결해야 합니다. 사용 가능한 인증기가 없는 경우, 먼저 인증 관리 페이지로 이동하여 새로운 인증기를 생성해야 합니다. 자세한 내용은 [인증](../verification/index.md)을 참고해 주세요.

![](https://static-docs.nocobase.com/202502282109802.png)

## 사용자 로그인

애플리케이션에서 2FA를 활성화하면, 사용자가 비밀번호로 로그인할 때 2FA 인증 절차를 진행하게 됩니다.

사용자가 지정된 인증기를 아직 연결하지 않은 경우, 인증기를 연결하도록 요청합니다. 연결에 성공하면 애플리케이션에 접속할 수 있습니다.

![](https://static-docs.nocobase.com/202502282110829.png)

사용자가 이미 지정된 인증기를 연결한 경우, 연결된 인증기를 통해 본인 인증을 진행하도록 요청합니다. 인증에 성공하면 애플리케이션에 접속할 수 있습니다.

![](https://static-docs.nocobase.com/202502282110148.png)

로그인에 성공하면, 사용자는 개인 센터의 인증 관리 페이지에서 다른 인증기를 연결할 수 있습니다.

![](https://static-docs.nocobase.com/202502282110024.png)
