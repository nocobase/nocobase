# 인증

:::info{title=참고}
`1.6.0-alpha.30` 버전부터 기존의 **인증 코드** 기능이 **인증 관리**로 업그레이드되었습니다. 이제 다양한 사용자 신원 인증 방식을 관리하고 통합할 수 있으며, 사용자가 해당 인증 방식을 연결하면 필요한 상황에서 신원 인증을 수행할 수 있습니다. 이 기능은 `1.7.0` 버전부터 안정적으로 지원될 예정입니다.
:::



## 소개

**인증 관리 센터는 다양한 사용자 신원 인증 방식을 관리하고 통합할 수 있도록 지원합니다.** 예를 들어:

- SMS 인증 코드 – 인증 플러그인에서 기본으로 제공합니다. 참고: [인증: SMS](./sms)
- TOTP 인증기 – 참고: [인증: TOTP 인증기](../verification-totp/)

개발자는 플러그인 형태로 다른 인증 유형을 확장할 수도 있습니다. 참고: [인증 유형 확장하기](./dev/type)

**사용자는 해당 인증 방식을 연결하여 필요한 상황에서 신원 인증을 수행할 수 있습니다.** 예를 들어:

- SMS 인증 로그인 – 참고: [인증: SMS](./sms)
- 이중 인증 (2FA) – 참고: [이중 인증 (2FA)](../2fa)
- 위험 작업에 대한 2차 인증 – 향후 지원 예정

개발자는 확장 플러그인 형태로 신원 인증을 다른 필요한 시나리오에 연동할 수도 있습니다. 참고: [인증 시나리오 확장하기](./dev/scene)

**인증 모듈과 사용자 인증 모듈의 차이점 및 연관성:** 사용자 인증 모듈은 주로 사용자 로그인 시나리오에서의 신원 인증을 담당하며, 이 중 SMS 로그인, 이중 인증 등은 인증 모듈이 제공하는 인증기에 의존합니다. 반면, 인증 모듈은 다양한 위험 작업에서의 신원 인증을 담당하며, 사용자 로그인은 그러한 위험 작업 시나리오 중 하나입니다.
