# 첨부 파일 필드

## 소개

시스템에 내장된 "첨부 파일" 타입의 필드는 사용자 정의 컬렉션에서 파일 업로드를 지원합니다.

첨부 파일 필드는 기본적으로 다대다(Many-to-Many) 관계 필드이며, 시스템에 내장된 "Attachments" (`attachments`) 컬렉션을 가리킵니다. 어떤 컬렉션에서든 첨부 파일 필드를 생성하면, 다대다 관계를 위한 중간 테이블이 자동으로 생성됩니다. 업로드된 파일의 메타데이터는 "Attachments" 컬렉션에 저장되며, 컬렉션에서 참조되는 파일 정보는 이 중간 테이블을 통해 연결됩니다.

### MIME 타입 제한

업로드할 수 있는 파일 유형을 제한하는 데 사용되며, [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types) 문법 형식을 사용하여 지정합니다. 예를 들어, `image/*`는 이미지 파일을 나타냅니다. 여러 유형은 쉼표(,)로 구분할 수 있습니다. 예시: `image/*,application/pdf`는 이미지 및 PDF 파일 유형을 모두 허용합니다.

### 저장 엔진

업로드된 파일을 저장할 저장 엔진을 선택합니다. 비워두면 시스템의 기본 저장 엔진이 사용됩니다.
