# 스토리지 엔진: 로컬 스토리지

업로드된 파일은 서버의 로컬 하드 드라이브 디렉터리에 저장됩니다. 이 방식은 시스템에서 관리하는 업로드 파일의 총량이 적거나 실험적인 용도로 사용하기에 적합합니다.

## 구성 파라미터

![파일 스토리지 엔진 구성 예시](https://static-docs.nocobase.com/20240529115151.png)

:::info{title=참고}
이 섹션에서는 로컬 스토리지 엔진에 특화된 파라미터만 설명합니다. 일반적인 파라미터는 [엔진 공통 파라미터](./index.md#엔진-공통-파라미터)를 참조해 주세요.
:::

### 경로

이 파라미터는 서버에 파일이 저장되는 상대 경로와 URL 접근 경로를 동시에 나타냅니다. 예를 들어, "`user/avatar`" (시작과 끝에 슬래시(`/`)는 필요 없음)는 다음을 의미합니다.

1. 업로드된 파일이 서버에 저장되는 상대 경로: `/path/to/nocobase-app/storage/uploads/user/avatar`
2. 파일 접근을 위한 URL 접두사: `http://localhost:13000/storage/uploads/user/avatar`
