# 버전 관리

워크플로우가 한 번 이상 트리거된 후, 워크플로우의 설정이나 노드를 수정하고 싶다면 새 버전을 생성한 후 수정해야 합니다. 이렇게 하면 이전에 트리거된 워크플로우의 실행 기록을 검토할 때, 향후 수정 사항의 영향을 받지 않도록 보장할 수 있습니다.

워크플로우 설정 페이지에서 오른쪽 상단의 버전 메뉴를 통해 기존 워크플로우 버전을 확인할 수 있습니다.

![워크플로우 버전 보기](https://static-docs.nocobase.com/ad93d2c08166b0e3e643fb148713a63f.png)

그 오른쪽에 있는 더 보기("...") 메뉴에서 현재 보고 있는 버전을 기반으로 새 버전으로 복사할 수 있습니다.

![워크플로우를 새 버전으로 복사](https://static-docs.nocobase.com/280798e6caca2af004893390a744256.png)

새 버전으로 복사한 후, "활성화"/"비활성화" 스위치를 클릭하여 해당 버전을 활성화 상태로 전환하면 새로운 워크플로우 버전이 적용됩니다.

이전 버전을 다시 선택해야 하는 경우, 버전 메뉴에서 전환한 후 "활성화"/"비활성화" 스위치를 다시 클릭하여 활성화 상태로 전환하면 현재 보고 있는 버전이 적용되며, 이후 트리거는 해당 버전의 프로세스를 실행합니다.

워크플로우를 비활성화해야 할 때, "활성화"/"비활성화" 스위치를 클릭하여 비활성화 상태로 전환하면 해당 워크플로우는 더 이상 트리거되지 않습니다.

:::info{title=팁}
워크플로우 관리 목록의 "복사" 워크플로우와 달리, "새 버전으로 복사"된 워크플로우는 여전히 동일한 워크플로우 그룹에 속하며 버전으로만 구분됩니다. 하지만 워크플로우를 복사하는 것은 완전히 새로운 워크플로우로 간주되며, 이전 워크플로우의 버전과는 무관하고 실행 횟수도 0으로 초기화됩니다.
:::
