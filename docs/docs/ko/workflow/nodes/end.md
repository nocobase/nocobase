# 워크플로우 종료

이 노드가 실행되면 현재 워크플로우가 즉시 종료되며, 노드에 설정된 상태로 마무리됩니다. 이 노드는 특정 로직에 따른 흐름 제어에 주로 사용됩니다. 특정 조건이 충족되면 현재 워크플로우를 벗어나 더 이상 후속 프로세스를 실행하지 않습니다. 이는 프로그래밍 언어에서 현재 실행 중인 함수를 종료하는 데 사용되는 `return` 명령어와 유사합니다.

## 노드 추가

워크플로우 설정 화면에서 흐름 내의 더하기("+"") 버튼을 클릭하여 "워크플로우 종료" 노드를 추가합니다.

### 종료 상태

종료 상태는 해당 워크플로우 실행 계획의 최종 상태에 영향을 미칩니다. "성공" 또는 "실패"로 설정할 수 있습니다. 워크플로우가 이 노드에 도달하면 설정된 상태로 즉시 종료됩니다.

:::info{title=팁}
"작업 전 이벤트" 유형의 워크플로우에서 사용될 경우, 작업을 시작한 요청을 가로채게 됩니다. 자세한 내용은 ["작업 전 이벤트" 사용 설명](../triggers/pre-action)을 참조하십시오.

또한, 작업을 시작한 요청을 가로채는 것 외에도, 종료 상태 설정은 해당 유형의 워크플로우에서 "응답 메시지"를 통해 피드백되는 정보의 상태에도 영향을 미칩니다.
:::
