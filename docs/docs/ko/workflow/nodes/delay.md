# 지연

## 소개

지연 노드는 워크플로우에 지연을 추가할 수 있습니다. 지연이 끝나면 설정에 따라 지연 후의 노드를 계속 실행하거나 워크플로우를 조기에 종료할 수 있습니다.

이 노드는 일반적으로 병렬 분기 노드와 함께 사용됩니다. 병렬 분기 중 한쪽 분기에 지연 노드를 추가하여 타임아웃 발생 시 관련 처리를 수행할 수 있습니다. 예를 들어, 병렬 분기 중 한 분기에 수동 처리가 포함되고 다른 분기에 지연 노드가 포함된 경우를 생각해 봅시다. 수동 처리가 타임아웃될 때, 타임아웃 실패로 설정하면 수동 처리가 제한된 시간 내에 완료되어야 함을 의미합니다. 타임아웃 계속으로 설정하면 시간이 경과한 후 해당 수동 처리를 무시할 수 있습니다.

## 설치

내장 플러그인이므로 별도로 설치할 필요가 없습니다.

## 노드 생성

워크플로우 설정 화면에서 워크플로우 내의 더하기("+"") 버튼을 클릭하여 "지연" 노드를 추가합니다.

## 노드 설정

![지연 노드_노드 설정](https://static-docs.nocobase.com/5fe8a36535f20a087a0148ffa1cd2aea.png)

### 지연 시간

지연 시간은 숫자를 입력하고 시간 단위를 선택할 수 있습니다. 지원되는 시간 단위는 초, 분, 시, 일, 주입니다.

### 타임아웃 시 상태

타임아웃 시 상태는 "통과 후 계속" 또는 "실패 후 종료"를 선택할 수 있습니다. 전자는 지연이 끝난 후 워크플로우가 다음 노드를 계속 실행하는 것을 의미하며, 후자는 지연이 끝난 후 워크플로우가 실패 상태로 조기에 종료되는 것을 의미합니다.

## 예시

작업 요청이 시작된 후 제한 시간 내에 응답해야 하는 시나리오를 예로 들어보겠습니다. 병렬로 실행되는 두 개의 분기 중 하나에는 수동 노드를 추가하고, 다른 분기에는 지연 노드를 추가해야 합니다. 만약 수동 처리가 10분 이내에 응답되지 않으면, 작업 요청 상태는 "타임아웃 및 미처리"로 업데이트됩니다.
