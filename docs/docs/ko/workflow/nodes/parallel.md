# 병렬 브랜치

병렬 브랜치 노드는 워크플로우를 여러 브랜치로 나눌 수 있습니다. 각 브랜치에는 다른 노드를 구성할 수 있으며, 브랜치 모드에 따라 실행 방식이 달라집니다. 여러 작업을 동시에 실행해야 하는 시나리오에서 병렬 브랜치 노드를 사용할 수 있습니다.

## 설치

내장 플러그인이므로 별도로 설치할 필요가 없습니다.

## 노드 생성

워크플로우 설정 화면에서 워크플로우 내의 더하기("+"") 버튼을 클릭하여 "병렬 브랜치" 노드를 추가합니다.

![병렬 브랜치 추가](https://static-docs.nocobase.com/9e0f3faa0b9335270647a30477559eac.png)

워크플로우에 병렬 브랜치 노드를 추가하면 기본적으로 두 개의 하위 브랜치가 추가됩니다. "브랜치 추가" 버튼을 클릭하여 원하는 만큼 브랜치를 더 추가할 수 있습니다. 각 브랜치에는 원하는 수의 노드를 추가할 수 있으며, 필요 없는 브랜치는 브랜치 시작 부분의 삭제 버튼을 클릭하여 제거할 수 있습니다.

![병렬 브랜치 관리](https://static-docs.nocobase.com/36084b8b7970c8a1771eb3ee9bc2a757.png)

## 노드 설정

### 브랜치 모드

병렬 브랜치 노드에는 다음 세 가지 모드가 있습니다.

- **모두 성공**: 모든 브랜치가 성공적으로 실행되어야 워크플로우가 브랜치 종료 후의 노드를 계속 실행합니다. 그렇지 않고 어떤 브랜치라도 실패, 오류 또는 기타 비성공 상태로 조기에 종료되면, 전체 병렬 브랜치 노드는 해당 상태로 조기 종료됩니다. 이를 "모두(All) 모드"라고도 합니다.
- **하나라도 성공**: 어떤 브랜치라도 성공적으로 실행되면 워크플로우는 브랜치 종료 후의 노드를 계속 실행합니다. 모든 브랜치가 실패, 오류 또는 기타 비성공 상태로 조기에 종료되지 않는 한, 전체 병렬 브랜치 노드는 해당 상태로 조기 종료되지 않습니다. 이를 "하나라도(Any) 모드"라고도 합니다.
- **하나라도 성공 또는 실패**: 어떤 브랜치라도 성공적으로 실행되면 워크플로우는 브랜치 종료 후의 노드를 계속 실행합니다. 하지만 어떤 노드라도 실패하면 전체 병렬 브랜치는 해당 상태로 조기 종료됩니다. 이를 "경쟁(Race) 모드"라고도 합니다.

어떤 모드이든 관계없이, 각 브랜치는 왼쪽에서 오른쪽으로 순서대로 실행을 시도하며, 브랜치에 미리 설정된 모드의 관련 조건이 충족될 때까지 이후 노드를 계속 실행하거나 조기에 종료됩니다.
