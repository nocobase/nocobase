# 새 데이터 추가

컬렉션에 새 데이터를 추가할 때 사용합니다.

새로 추가할 데이터 행의 필드 값은 워크플로우 컨텍스트의 변수를 사용할 수 있습니다. 관계 필드에 값을 할당할 때는 컨텍스트에 있는 해당 데이터 변수를 직접 참조할 수 있으며, 이는 객체 또는 외래 키 값일 수 있습니다. 변수를 사용하지 않는 경우, 외래 키 값을 수동으로 입력해야 합니다. 다대다 관계의 여러 외래 키 값은 쉼표로 구분하여 입력해야 합니다.

## 노드 생성

워크플로우 설정 화면에서 워크플로우의 더하기("+"") 버튼을 클릭하여 "새 데이터 추가" 노드를 추가합니다.

![새 데이터 추가 노드](https://static-docs.nocobase.com/386c8c01c89b1eeab848510e77f4841a.png)

## 노드 설정

![새 데이터 추가 노드_예시_노드 설정](https://static-docs.nocobase.com/5f7b97a51b64a1741cf82a4d4455b610.png)

### 컬렉션

새 데이터를 추가할 컬렉션을 선택합니다.

### 필드 값

컬렉션의 필드에 값을 할당합니다. 워크플로우 컨텍스트의 변수를 사용하거나 수동으로 정적 값을 입력할 수 있습니다.

:::info{title="참고"}
워크플로우의 "새 데이터 추가" 노드로 생성된 데이터는 "생성자", "최종 수정자"와 같은 사용자 데이터를 자동으로 처리하지 않습니다. 필요에 따라 이 두 필드의 값을 직접 설정해야 합니다.
:::

### 관계 데이터 미리 로드

새로 추가할 데이터의 필드에 관계 필드가 포함되어 있고, 후속 워크플로우 단계에서 해당 관계 데이터를 사용하려는 경우, 미리 로드 설정에서 해당 관계 필드를 선택할 수 있습니다. 이렇게 하면 새 데이터 추가가 완료된 후, 해당 관계 데이터가 자동으로 로드되어 노드의 결과 데이터에 함께 저장됩니다.

## 예시

예를 들어, "게시물" 컬렉션의 데이터가 새로 추가되거나 업데이트된 후, 게시물의 변경 이력을 기록하기 위해 "게시물 버전" 데이터가 자동으로 추가되어야 할 때, 새 데이터 추가 노드를 사용하여 이를 구현할 수 있습니다.

![새 데이터 추가 노드_예시_노드 설정](https://static-docs.nocobase.com/1a0992e66170be12a068da6503298868.png)

이 설정으로 워크플로우를 활성화하면, "게시물" 컬렉션의 데이터가 변경될 때, "게시물 버전" 데이터가 자동으로 추가되어 게시물의 변경 이력을 기록합니다.
