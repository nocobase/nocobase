# 데이터 삭제

특정 조건을 만족하는 컬렉션의 데이터를 삭제할 때 사용합니다.

삭제 노드의 기본적인 사용법은 업데이트 노드와 유사합니다. 다만, 삭제 노드는 필드 값을 할당할 필요 없이 컬렉션과 필터 조건만 선택하면 됩니다. 삭제 노드의 결과는 성공적으로 삭제된 데이터의 행 수를 반환하며, 이는 실행 기록에서만 확인할 수 있고 이후 노드에서 변수로 사용할 수 없습니다.

:::info{title=참고}
현재 삭제 노드는 개별 데이터 삭제를 지원하지 않으며, 모두 일괄 삭제로 처리됩니다. 따라서 각 데이터 삭제에 대한 다른 이벤트는 트리거되지 않습니다.
:::

## 노드 생성

워크플로우 설정 화면에서 흐름 내의 더하기("+" ) 버튼을 클릭하여 "데이터 삭제" 노드를 추가합니다.

![데이터 삭제 노드 생성](https://static-docs.nocobase.com/e1d6b8728251fcdbed6c7f50e5570da2.png)

## 노드 설정

![삭제 노드_노드 설정](https://static-docs.nocobase.com/580600c2b13ef4e01dfa48b23539648e.png)

### 컬렉션

데이터를 삭제할 컬렉션을 선택합니다.

### 필터 조건

일반적인 컬렉션 조회 시의 필터 조건과 유사하며, 워크플로우의 컨텍스트 변수를 사용할 수 있습니다.

## 예시

예를 들어, 취소된 유효하지 않은 과거 주문 데이터를 정기적으로 정리할 때 삭제 노드를 사용하여 구현할 수 있습니다.

![삭제 노드_예시_노드 설정](https://static-docs.nocobase.com/b94b75077a17252f8523c3f13ce5f320.png)

워크플로우는 정기적으로 트리거되어 취소된 모든 유효하지 않은 과거 주문 데이터를 삭제하는 작업을 실행합니다.
