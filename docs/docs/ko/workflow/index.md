# 개요

## 소개

워크플로우 플러그인은 NocoBase에서 일상적인 승인, 데이터 동기화, 알림 등 자동화된 비즈니스 프로세스를 구성하는 데 도움을 줍니다. 워크플로우에서는 시각적 인터페이스를 통해 트리거와 관련 노드를 설정하기만 하면 코드 작성 없이 복잡한 비즈니스 로직을 구현할 수 있습니다.

### 예시

각 워크플로우는 하나의 트리거와 여러 노드로 구성됩니다. 트리거는 시스템 내 이벤트를 나타내며, 각 노드는 실행 단계를 나타냅니다. 이들은 전체적으로 이벤트 발생 후 처리해야 할 비즈니스 로직을 설명합니다. 다음 이미지는 상품 주문 후 재고 차감의 일반적인 프로세스를 보여줍니다.

![워크플로우 예시](https://static-docs.nocobase.com/20251029222146.png)

사용자가 주문을 제출하면 워크플로우가 자동으로 재고를 확인합니다. 재고가 충분하면 재고를 차감하고 주문 생성을 계속 진행하며, 그렇지 않으면 프로세스가 종료됩니다.

### 활용 사례

더 넓은 관점에서 보면, NocoBase 애플리케이션의 워크플로우는 다양한 시나리오의 문제를 해결할 수 있습니다.

- 반복적인 작업 자동화: 주문 검토, 재고 동기화, 데이터 정제, 점수 계산 등 더 이상 수동 작업이 필요하지 않습니다.
- 사람과 기계의 협업 지원: 주요 노드에서 승인 또는 검토를 배치하고, 처리 결과에 따라 다음 단계를 계속 진행합니다.
- 외부 시스템 연결: HTTP 요청 전송, 외부 서비스 푸시 수신을 통해 시스템 간 자동화를 구현합니다.
- 비즈니스 변화에 빠르게 적응: 프로세스 구조, 조건 또는 기타 노드 구성을 조정하여 새로운 버전 출시 없이 바로 적용할 수 있습니다.

## 설치

워크플로우는 NocoBase의 내장 플러그인으로, 추가 설치나 구성이 필요 없습니다.
