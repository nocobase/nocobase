# 개요

워크플로우의 내장 기능만으로는 모든 시나리오를 커버할 수 없습니다. 예를 들어, 내장된 노드 유형만으로는 모든 비즈니스 시나리오의 모든 종류의 작업을 다룰 수 없습니다. 따라서 저희는 트리거 및 노드 유형 확장 등을 포함하여 워크플로우를 확장할 수 있는 설계를 제공합니다. 내장 기능만으로는 충족할 수 없는 비즈니스 시나리오에서는 로우코드 방식으로 확장하여 문제를 해결할 수 있습니다.

확장은 주로 다음 두 가지 내용으로 나뉩니다:

- [트리거 유형 확장](./trigger.md)
- [노드 유형 확장](./node.md)
