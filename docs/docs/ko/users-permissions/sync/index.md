# 사용자 데이터 동기화

## 소개

이 기능은 사용자 데이터 동기화 소스를 등록하고 관리할 수 있도록 합니다. 기본적으로 HTTP API를 제공하며, 플러그인을 통해 다른 데이터 소스를 확장할 수 있습니다. 기본적으로 **사용자** 및 **부서** 컬렉션으로 데이터를 동기화하는 것을 지원하며, 플러그인을 사용하여 다른 동기화 대상 리소스로 확장할 수도 있습니다.

## 데이터 소스 관리 및 동기화

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
사용자 데이터 동기화 소스를 제공하는 플러그인이 설치되지 않은 경우, HTTP API를 사용하여 사용자 데이터를 동기화할 수 있습니다. 자세한 내용은 [데이터 소스 - HTTP API](./sources/api.md)를 참조하십시오.
:::

## 데이터 소스 추가

사용자 데이터 동기화 소스를 제공하는 플러그인을 설치한 후, 해당 데이터 소스를 추가할 수 있습니다. 활성화된 데이터 소스에만 "동기화" 및 "작업" 버튼이 표시됩니다.

> 예시: 위챗(WeCom)

![](https://static-docs.nocobase.com/202412041053785.png)

## 데이터 동기화

"동기화" 버튼을 클릭하여 데이터 동기화를 시작합니다.

![](https://static-docs.nocobase.com/202412041055022.png)

"작업" 버튼을 클릭하여 동기화 상태를 확인할 수 있습니다. 동기화가 성공적으로 완료되면 사용자 및 부서 목록에서 데이터를 확인할 수 있습니다.

![](https://static-docs.nocobase.com/202412041202337.png)

동기화에 실패한 작업은 "재시도" 버튼을 클릭하여 다시 시도할 수 있습니다.

![](https://static-docs.nocobase.com/202412041058337.png)

동기화 실패 시 시스템 로그를 통해 원인을 파악할 수 있습니다. 또한, 애플리케이션 로그 디렉터리 아래의 `user-data-sync` 디렉터리에 원본 데이터 동기화 기록이 저장됩니다.

![](https://static-docs.nocobase.com/202412041205655.png)
