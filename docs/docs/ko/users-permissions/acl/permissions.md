# 권한 설정

## 일반 권한 설정

![](https://static-docs.nocobase.com/119a9650259f9be71210121d0d3a435d.png)

### 설정 권한

1.  **인터페이스 설정 허용**: 이 권한은 사용자가 인터페이스를 설정할 수 있는지 여부를 제어합니다. 이 권한을 활성화하면 UI 설정 버튼이 나타납니다. "admin" 역할은 기본적으로 이 권한이 활성화되어 있습니다.
2.  **플러그인 설치, 활성화, 비활성화 허용**: 이 권한은 사용자가 **플러그인**을 활성화하거나 비활성화할 수 있는지 여부를 제어합니다. 이 권한이 활성화되면 사용자는 **플러그인** 관리자 인터페이스에 접근할 수 있습니다. "admin" 역할은 기본적으로 이 권한이 활성화되어 있습니다.
3.  **플러그인 설정 허용**: 이 권한은 사용자가 **플러그인** 매개변수를 설정하거나 **플러그인** 백엔드 데이터를 관리할 수 있도록 합니다. "admin" 역할은 기본적으로 이 권한이 활성화되어 있습니다.
4.  **캐시 지우기, 애플리케이션 재시작 허용**: 이 권한은 캐시 지우기 및 애플리케이션 재시작과 같은 시스템 유지보수 작업을 제어합니다. 활성화되면 관련 작업 버튼이 개인 센터에 나타납니다. 이 권한은 기본적으로 비활성화되어 있습니다.
5.  **새 메뉴 항목 기본 접근 허용**: 새로 생성된 메뉴는 기본적으로 접근 가능하며, 이 설정은 기본적으로 활성화되어 있습니다.

### 전역 작업 권한

전역 작업 권한은 모든 **컬렉션**에 전역적으로 적용되며, 작업 유형별로 분류됩니다. 이 권한은 데이터 범위(모든 데이터 또는 사용자 자신의 데이터)에 따라 설정할 수 있습니다. '모든 데이터'는 전체 **컬렉션**에 대한 작업을 허용하며, '자신의 데이터'는 사용자 자신과 관련된 데이터로 작업을 제한합니다.

## 컬렉션 작업 권한

![](https://static-docs.nocobase.com/6a0281391cecdea5b218e6173c5d7.png)

![](https://static-docs.nocobase.com/9814140434ff9e1bf028a6c282a5a165.png)

**컬렉션** 작업 권한은 전역 작업 권한을 더욱 세분화하여, 각 **컬렉션** 내의 리소스 접근에 대한 개별 권한 설정을 가능하게 합니다. 이 권한은 두 가지 측면으로 나뉩니다.

1.  **작업 권한**: 추가, 보기, 편집, 삭제, 내보내기, 가져오기 작업이 포함됩니다. 이 권한은 데이터 범위에 따라 설정됩니다.
    -   **모든 데이터**: 사용자가 **컬렉션** 내의 모든 레코드에 대해 작업을 수행할 수 있도록 허용합니다.
    -   **자신의 데이터**: 사용자가 자신이 생성한 레코드에 대해서만 작업을 수행하도록 제한합니다.

2.  **필드 권한**: 필드 권한을 통해 각 필드에 대해 다양한 작업에서 특정 권한을 설정할 수 있습니다. 예를 들어, 특정 필드는 편집 권한 없이 보기 전용으로 설정할 수 있습니다.

## 메뉴 접근 권한

메뉴 접근 권한은 메뉴 항목을 기준으로 접근을 제어합니다.

![](https://static-docs.nocobase.com/28eddfc843d27641162d9129e3b6e33f.png)

## 플러그인 설정 권한

**플러그인** 설정 권한은 특정 **플러그인** 매개변수를 설정할 수 있는 권한을 제어합니다. 이 권한이 활성화되면 관리 센터에 해당 **플러그인** 관리 인터페이스가 나타납니다.

![](https://static-docs.nocobase.com/5a742ae20a9de93dc2722468b9fd7475.png)
