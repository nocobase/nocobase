# 다중 앱

## 소개

**다중 앱 플러그인**은 여러 독립적인 애플리케이션을 개별적으로 배포할 필요 없이 동적으로 생성하고 관리할 수 있도록 지원합니다. 각 서브 앱은 완전히 독립적인 인스턴스이며, 고유한 데이터베이스, 플러그인, 설정을 가집니다.

#### 활용 사례
- **멀티 테넌시**: 독립적인 애플리케이션 인스턴스를 제공하여, 각 고객이 고유한 데이터, 플러그인 설정, 권한 시스템을 가질 수 있도록 합니다.
- **서로 다른 비즈니스 도메인을 위한 주/서브 시스템**: 하나의 대규모 시스템이 여러 독립적으로 배포된 소규모 애플리케이션으로 구성될 때 유용합니다.

:::warning
다중 앱 플러그인 자체는 사용자 공유 기능을 제공하지 않습니다.
여러 앱 간에 사용자를 연동해야 하는 경우, **[인증 플러그인](/auth-verification)**과 함께 사용할 수 있습니다.
:::

## 설치

플러그인 관리에서 **다중 앱(Multi-app)** 플러그인을 찾아 활성화합니다.

![](https://static-docs.nocobase.com/multi-app/Plugin-manager-NocoBase-10-16-2025_03_07_PM.png)

## 사용 가이드

### 서브 앱 생성

시스템 설정 메뉴에서 「다중 앱」을 클릭하여 다중 앱 관리 페이지로 이동합니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_48_PM.png)

「새로 추가」 버튼을 클릭하여 새로운 서브 앱을 생성합니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_03_50_PM.png)

#### 폼 필드 설명

* **이름**: 서브 앱 식별자로, 전역적으로 고유합니다.
* **표시 이름**: 서브 앱이 인터페이스에 표시되는 이름입니다.
* **시작 방식**:
  * **첫 방문 시 시작**: 사용자가 URL을 통해 해당 서브 앱에 처음 접근할 때만 시작됩니다.
  * **메인 앱과 함께 시작**: 메인 앱이 시작될 때 서브 앱도 동시에 시작됩니다. (이 경우 메인 앱의 시작 시간이 길어질 수 있습니다.)
* **포트**: 서브 앱이 실행될 때 사용되는 포트 번호입니다.
* **사용자 지정 도메인**: 서브 앱에 독립적인 서브 도메인을 설정합니다.
* **메뉴에 고정**: 서브 앱 진입점을 상단 탐색 바 왼쪽에 고정하여 표시합니다.
* **데이터베이스 연결**: 서브 앱의 데이터 소스를 설정하는 데 사용되며, 다음 세 가지 방식을 지원합니다.
  * **새 데이터베이스**: 현재 데이터 서비스를 재사용하여 독립적인 데이터베이스를 생성합니다.
  * **새 데이터 연결**: 완전히 새로운 데이터베이스 서비스를 설정합니다.
  * **스키마 모드**: PostgreSQL에서 서브 앱을 위한 독립적인 스키마를 생성합니다.
* **업그레이드**: 연결된 데이터베이스에 이전 버전의 NocoBase 데이터 구조가 있는 경우, 현재 버전으로 자동 업그레이드됩니다.

### 서브 앱 실행 및 중지

**시작** 버튼을 클릭하여 서브 앱을 시작할 수 있습니다.
> 서브 앱 생성 시 *“첫 방문 시 시작”*을 선택했다면, 첫 방문 시 자동으로 시작됩니다.

**보기** 버튼을 클릭하면 새 탭에서 해당 서브 앱이 열립니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_00_PM.png)

### 서브 앱 실행 상태 및 로그

목록에서 각 애플리케이션이 사용하는 메모리 및 CPU를 확인할 수 있습니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-21-2025_10_31_AM.png)

**로그** 버튼을 클릭하면 서브 앱의 실행 로그를 확인할 수 있습니다.
> 서브 앱 시작 후 접근할 수 없는 경우(예: 데이터베이스 손상 등), 로그를 통해 원인을 파악할 수 있습니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-NocoBase-10-16-2025_04_02_PM.png)

### 서브 앱 삭제

**삭제** 버튼을 클릭하여 서브 앱을 제거할 수 있습니다.
> 삭제 시 데이터베이스도 함께 삭제할지 선택할 수 있습니다. 신중하게 진행해 주십시오. 이 작업은 되돌릴 수 없습니다.

### 서브 앱 접근
기본적으로 `/_app/:appName/admin/` 경로를 사용하여 서브 앱에 접근합니다. 예시:
```
http://localhost:13000/_app/a_7zkxoarusnx/admin/
```
또한, 서브 앱에 독립적인 서브 도메인을 설정할 수도 있습니다. 이 경우 도메인을 현재 IP로 해석해야 하며, Nginx를 사용하는 경우 Nginx 설정에 도메인을 추가해야 합니다.

### 명령줄을 통한 서브 앱 관리

프로젝트 루트 디렉터리에서 명령줄을 통해 **PM2**로 서브 앱 인스턴스를 관리할 수 있습니다.

```bash
yarn nocobase pm2 list              # 현재 실행 중인 인스턴스 목록 보기
yarn nocobase pm2 stop [appname]    # 특정 서브 앱 프로세스 중지
yarn nocobase pm2 delete [appname]  # 특정 서브 앱 프로세스 삭제
yarn nocobase pm2 kill              # 시작된 모든 프로세스 강제 종료 (메인 앱 인스턴스 포함 가능)
```

### 이전 다중 앱 데이터 마이그레이션

이전 다중 앱 관리 페이지로 이동하여 「새 다중 앱으로 데이터 마이그레이션」 버튼을 클릭하여 데이터를 마이그레이션합니다.

![](https://static-docs.nocobase.com/multi-app/Multi-app-manager-deprecated-NocoBase-10-21-2025_10_32_AM.png)

## 자주 묻는 질문

#### 1. 플러그인 관리
서브 앱은 메인 앱과 동일한 플러그인을 사용할 수 있으며(버전 포함), 플러그인을 독립적으로 설정하고 사용할 수 있습니다.

#### 2. 데이터베이스 격리
서브 앱은 독립적인 데이터베이스를 설정할 수 있습니다. 앱 간에 데이터를 공유하려면 외부 데이터 소스를 통해 구현할 수 있습니다.

#### 3. 데이터 백업 및 마이그레이션
현재 메인 앱의 데이터 백업은 서브 앱 데이터를 포함하지 않습니다(서브 앱의 기본 정보만 포함됩니다). 각 서브 앱 내에서 수동으로 데이터를 백업하고 마이그레이션해야 합니다.

#### 4. 배포 및 업데이트
서브 앱의 버전은 메인 앱과 함께 자동으로 업그레이드됩니다. 이를 통해 메인 앱과 서브 앱 간의 버전 일관성이 자동으로 유지됩니다.

#### 5. 리소스 관리
각 서브 앱의 리소스 사용량은 메인 앱과 거의 동일합니다. 현재 단일 애플리케이션의 메모리 사용량은 약 500-600MB 정도입니다.
