# 멀티 애플리케이션 관리

## 개요

NocoBase의 멀티 애플리케이션 관리는 하나 이상의 런타임 환경에서 여러 개의 **물리적으로 격리된** 애플리케이션 인스턴스를 생성/운영할 수 있게 해줍니다. **AppSupervisor**를 통해 단일 진입점에서 통합 관리할 수 있습니다.

## 단일 애플리케이션

프로젝트 초기에는 보통 단일 애플리케이션으로 시작합니다.

규모가 커지면 다음 한계가 나타납니다.

- 기능 누적으로 인한 복잡도 증가
- 도메인 간 격리 어려움
- 운영/확장 비용 상승

## 공유 메모리 멀티 애플리케이션

복잡한 인프라 도입 없이 앱 단위 분리가 필요할 때 사용하는 모드입니다.

여러 앱이 하나의 NocoBase 인스턴스에서 실행되며, 앱별 독립 DB/생성/시작/중지는 가능하지만 프로세스와 메모리는 공유합니다.

![](https://static-docs.nocobase.com/202512231055907.png)

장점:

- 앱 단위 분리
- 설정과 기능 경계가 명확
- 멀티 프로세스/컨테이너 대비 리소스 절감

단점: 한 앱의 장애/고부하가 다른 앱에 영향 가능.

## 멀티 환경 하이브리드 배포

규모와 복잡도가 높아지면 공유 메모리 모델만으로는 한계가 생길 수 있습니다. 이때 진입 앱 + 다수 실행 환경으로 구성된 하이브리드 배포를 사용합니다.

![](https://static-docs.nocobase.com/202512231215186.png)

SaaS, 대규모 데모 환경, 멀티테넌트 시나리오에 적합합니다.
