# UI 템플릿

## 소개

UI 템플릿은 인터페이스 구축 시 설정을 재사용하여 반복적인 작업을 줄이고, 필요한 경우 여러 곳의 설정을 동시에 업데이트하여 동기화 상태를 유지하는 데 사용됩니다.

현재 지원되는 템플릿 유형은 다음과 같습니다:

- 블록 템플릿: 전체 블록 설정을 재사용
- 필드 템플릿: 양식/상세 블록의 '필드 영역' 설정을 재사용
- 팝업 템플릿: 작업/필드에 의해 트리거되는 팝업 설정을 재사용

## 핵심 개념

### 참조 및 복제

템플릿을 사용할 때는 보통 두 가지 방식이 있습니다:

- `참조`: 여러 곳에서 동일한 템플릿 설정을 공유합니다. 템플릿이나 참조된 어느 한 곳을 수정하면 다른 참조 지점들도 동시에 업데이트됩니다.
- `복제`: 독립된 설정으로 복제합니다. 이후 수정 사항은 서로 영향을 주지 않습니다.

### 템플릿으로 저장

블록이나 팝업 설정이 완료되면, 해당 설정 메뉴에서 `템플릿으로 저장`을 사용하여 저장 방식을 선택할 수 있습니다:

- `현재 ...을(를) 템플릿으로 전환`: 저장 후 현재 위치가 해당 템플릿을 참조하는 형식으로 전환됩니다.
- `현재 ...을(를) 템플릿으로 복제`: 템플릿만 생성하며 현재 위치의 설정은 그대로 유지됩니다.

## 블록 템플릿

### 블록을 템플릿으로 저장

1) 대상 블록의 설정 메뉴를 열고 `템플릿으로 저장`을 클릭합니다.  
2) `템플릿 이름` / `템플릿 설명`을 입력하고 저장 모드를 선택합니다:
   - `현재 블록을 템플릿으로 전환`: 저장 후 현재 위치가 `블록 템플릿` 블록으로 교체됩니다(즉, 해당 템플릿을 참조함).
   - `현재 블록을 템플릿으로 복제`: 템플릿만 생성하며 현재 블록은 그대로 유지됩니다.

![save-as-template-block-20251228](https://static-docs.nocobase.com/save-as-template-block-20251228.png)

![save-as-template-block-full-20251228](https://static-docs.nocobase.com/save-as-template-block-full-20251228.png)

### 블록 템플릿 사용

1) 블록 추가 → "기타 블록" → `블록 템플릿`  
2) 설정에서 다음을 선택합니다:
   - `템플릿`: 사용할 템플릿 선택
   - `모드`: `참조` 또는 `복제`

![block-template-menu-20251228](https://static-docs.nocobase.com/block-template-menu-20251228.png)

![select-block-template-20251228](https://static-docs.nocobase.com/select-block-template-20251228.png)

### 참조를 복제로 전환

블록이 템플릿을 참조 중일 때, 블록 설정 메뉴에서 `참조를 복제로 전환`을 사용하여 현재 블록을 일반 블록으로 변경(참조 끊기)할 수 있습니다. 이후 수정 사항은 서로 영향을 주지 않습니다.

![convert-block-template-duplicate-20251228](https://static-docs.nocobase.com/convert-block-template-duplicate-20251228.png)

### 주의 사항

- `복제` 모드는 블록 및 하위 노드의 UID를 새로 생성하므로, UID에 의존하는 일부 설정은 다시 구성해야 할 수도 있습니다.

## 필드 템플릿

필드 템플릿은 **양식 블록** 및 **상세 블록**에서 필드 영역 설정(필드 선택, 레이아웃 및 필드 설정)을 재사용하여 여러 페이지나 블록에서 필드를 반복해서 추가하는 것을 방지합니다.

> 필드 템플릿은 '필드 영역'에만 작용하며 전체 블록을 대체하지 않습니다. 전체 블록을 재사용하려면 위에서 설명한 블록 템플릿을 사용하세요.

### 양식/상세 블록에서 필드 템플릿 사용

1) 설정 모드로 진입하여 양식 블록 또는 상세 블록의 '필드' 메뉴를 엽니다.  
2) `필드 템플릿`을 선택합니다.  
3) 템플릿을 선택하고 모드(`참조` 또는 `복제`)를 선택합니다.

![field-template-menu-20251228](https://static-docs.nocobase.com/field-template-menu-20251228.png)

![use-field-template-config-20251228](https://static-docs.nocobase.com/use-field-template-config-20251228.png)

#### 덮어쓰기 알림

블록에 이미 필드가 존재하는 경우, **참조** 모드를 사용하면 일반적으로 확인 메시지가 표시됩니다(참조 필드가 현재 필드 영역을 대체하기 때문입니다).

### 참조 필드를 복제로 전환

블록이 필드 템플릿을 참조 중일 때, 블록 설정 메뉴에서 `참조 필드를 복제로 전환`을 사용하여 현재 필드 영역을 독립된 설정으로 변경(참조 끊기)할 수 있습니다. 이후 수정 사항은 서로 영향을 주지 않습니다.

![convert-field-template-duplicate-20251228](https://static-docs.nocobase.com/convert-field-template-duplicate-20251228.png)

### 주의 사항

- 필드 템플릿은 **양식 블록** 및 **상세 블록**에만 적용됩니다.
- 템플릿과 현재 블록에 연결된 데이터 표가 일치하지 않으면 선택기에서 템플릿을 사용할 수 없음으로 표시되고 그 이유가 안내됩니다.
- 현재 블록에서 필드에 대해 '개별적인 조정'을 하려면 처음부터 `복제` 모드를 사용하거나, 먼저 '참조 필드를 복제로 전환'을 실행하는 것이 좋습니다.

## 팝업 템플릿

팝업 템플릿은 일련의 팝업 인터페이스와 상호작용 로직을 재사용하는 데 사용됩니다. 팝업 열기 방식, 크기 등 일반적인 설정은 [팝업 편집](/interface-builder/actions/action-settings/edit-popup)을 참고하세요.

### 팝업을 템플릿으로 저장

1) 팝업을 트리거할 수 있는 버튼/필드 설정 메뉴를 열고 `템플릿으로 저장`을 클릭합니다.  
2) 템플릿 이름/설명을 입력하고 저장 모드를 선택합니다:
   - `현재 팝업을 템플릿으로 전환`: 저장 후 현재 팝업이 해당 템플릿을 참조하도록 전환됩니다.
   - `현재 팝업을 템플릿으로 복제`: 템플릿만 생성하며 현재 팝업은 그대로 유지됩니다.

![save-as-template-popup-20251228](https://static-docs.nocobase.com/save-as-template-popup-20251228.png)

### 팝업 설정에서 템플릿 사용

1) 버튼/필드의 팝업 설정을 엽니다.  
2) `팝업 템플릿`에서 템플릿을 선택하여 재사용합니다.

![edit-popup-select-20251228](https://static-docs.nocobase.com/edit-popup-select-20251228.png)

### 사용 조건 (템플릿 사용 가능 범위)

팝업 템플릿은 팝업을 트리거하는 작업 시나리오와 관련이 있습니다. 선택기는 현재 시나리오에 따라 호환되지 않는 템플릿을 자동으로 필터링하거나 비활성화합니다(조건을 충족하지 않을 경우 이유가 표시됩니다).

| 현재 작업 유형 | 사용 가능한 팝업 템플릿 |
| --- | --- |
| **컬렉션 작업** | 동일한 컬렉션의 컬렉션 작업으로 생성된 팝업 템플릿 |
| **비연관 레코드 작업** | 동일한 컬렉션의 컬렉션 작업 또는 비연관 레코드 작업으로 생성된 팝업 템플릿 |
| **연관 레코드 작업** | 동일한 컬렉션의 컬렉션 작업 또는 비연관 레코드 작업으로 생성된 팝업 템플릿, 또는 동일한 연관 필드의 연관 레코드 작업으로 생성된 팝업 템플릿 |

### 관계 데이터 팝업

관계 데이터(연관 필드)에 의해 트리거되는 팝업 템플릿에는 특별한 매칭 규칙이 있습니다:

#### 관계 팝업 템플릿의 엄격한 매칭

팝업 템플릿이 **연관 레코드 작업**에서 생성된 경우(템플릿에 `associationName`이 포함됨), 해당 템플릿은 **완전히 동일한 연관 필드**의 작업/필드에서만 사용할 수 있습니다.

예를 들어: `주문.고객` 연관 필드에서 생성된 팝업 템플릿은 다른 `주문.고객` 연관 필드의 작업에서만 사용할 수 있으며, `주문.추천인` 연관 필드에서는 사용할 수 없습니다(두 필드의 대상 데이터 표가 모두 `고객`이더라도 마찬가지입니다).

이는 관계 팝업 템플릿 내부의 변수와 설정이 특정 연관 관계 컨텍스트에 의존하기 때문입니다.

#### 관계 작업의 대상 데이터 표 템플릿 재사용

연관 필드/작업은 데이터 표가 일치하기만 하면 **대상 데이터 표의 비관계 팝업 템플릿**(컬렉션 작업 또는 비연관 레코드 작업으로 생성된 템플릿)을 재사용할 수 있습니다.

예를 들어: `주문.고객` 연관 필드는 `고객` 데이터 표의 팝업 템플릿을 사용할 수 있습니다. 이 방식은 여러 연관 필드 간에 동일한 팝업 설정(예: 통합된 고객 상세 팝업)을 공유하는 데 적합합니다.

### 참조를 복제로 전환

팝업이 템플릿을 참조 중일 때, 설정 메뉴에서 `참조를 복제로 전환`을 사용하여 현재 팝업을 독립된 설정으로 변경(참조 끊기)할 수 있습니다. 이후 수정 사항은 서로 영향을 주지 않습니다.

![convert-popup-to-duplicate-20251228](https://static-docs.nocobase.com/convert-popup-to-duplicate-20251228.png)


## 템플릿 관리

시스템 설정 → `UI 템플릿`에서 모든 템플릿을 확인하고 관리할 수 있습니다:

- **블록 템플릿 (v2)**: 블록 템플릿 관리
- **팝업 템플릿 (v2)**: 팝업 템플릿 관리

> 필드 템플릿은 블록 템플릿에서 파생되며, 블록 템플릿 내에서 관리됩니다.

![block-template-list-20251228](https://static-docs.nocobase.com/block-template-list-20251228.png)

지원되는 작업: 보기, 필터링, 편집, 삭제.

> **주의**: 템플릿이 참조 중인 경우에는 직접 삭제할 수 없습니다. 먼저 해당 템플릿을 참조하는 위치에서 `참조를 복제로 전환`을 사용하여 참조를 끊은 후 템플릿을 삭제하세요.
