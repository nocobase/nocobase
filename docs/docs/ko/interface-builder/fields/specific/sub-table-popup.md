# 하위 테이블 (팝업 편집)

## 소개

하위 테이블(팝업 편집)은 폼 내에서 일대다(One-to-Many) 또는 다대다(Many-to-Many)와 같은 다중 관계 데이터를 관리하는 데 사용됩니다. 테이블에는 현재 연결된 레코드만 표시됩니다. 레코드 추가 및 편집은 팝업창에서 수행되며, 데이터는 메인 폼이 제출될 때 데이터베이스에 일괄 저장됩니다.

## 사용 방법

![20260212152204](https://static-docs.nocobase.com/20260212152204.png)

적용 시나리오:

- 관계 필드: o2m / m2m / mbm
- 주요 용도: 주문 상세, 하위 항목 리스트, 연결된 태그/멤버 등

## 필드 설정 항목

### 기존 데이터 선택 허용 (기본값: 활성화)

기존 데이터에서 관계를 선택할 수 있도록 지원합니다.

![20260212152312](https://static-docs.nocobase.com/20260212152312.png)

![20260212152343](https://static-docs.nocobase.com/20260212152343.gif)

### 필드 컴포넌트

[필드 컴포넌트](/interface-builder/fields/association-field): 드롭다운 선택, 데이터 선택기 등 다른 관계 필드 컴포넌트로 전환합니다.

### 기존 데이터 연결 해제 허용 (기본값: 활성화)

> 편집 폼에서 기존에 연결된 데이터의 연결 해제 허용 여부를 제어합니다. 새로 추가된 데이터는 언제든지 제거할 수 있습니다.

![20260212165752](https://static-docs.nocobase.com/20260212165752.gif)

### 추가 허용 (기본값: 활성화)

추가 버튼 표시 여부를 제어합니다. 사용자가 대상 컬렉션에 대한 `create` 권한이 없는 경우, 버튼은 비활성화되며 권한 없음 툴팁이 표시됩니다.

### 빠른 편집 허용 (기본값: 비활성화)

활성화하면 셀 위에 마우스를 올렸을 때 편집 아이콘이 나타나며, 셀 내용을 빠르게 수정할 수 있습니다.

관계 필드 컴포넌트 설정을 통해 모든 필드에 대해 빠른 편집을 활성화할 수 있습니다.

![20260212165102](https://static-docs.nocobase.com/20260212165102.png)

또한 개별 열 필드에 대해서도 빠른 편집을 활성화할 수 있습니다.

![20260212165025](https://static-docs.nocobase.com/20260212165025.png)

![20260212165201](https://static-docs.nocobase.com/20260212165201.gif)

### 페이지 크기 (기본값: 10)

하위 테이블의 페이지당 표시할 레코드 수를 설정합니다.

## 동작 설명

- 기존 레코드를 선택할 때 기본 키(Primary Key)를 기준으로 중복을 제거하여 동일한 레코드가 중복 연결되는 것을 방지합니다.
- 새로 추가된 레코드는 하위 테이블에 즉시 반영되며, 기본적으로 해당 레코드가 포함된 페이지로 이동합니다.
- 행 내 편집은 현재 행의 데이터만 수정합니다.
- 제거는 현재 폼에서의 연결만 해제할 뿐, 원본 데이터를 삭제하지 않습니다.
- 데이터는 메인 폼이 제출될 때 데이터베이스에 일괄 저장됩니다.
