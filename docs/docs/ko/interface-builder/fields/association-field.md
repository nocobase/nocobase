# 관계 필드 컴포넌트

## 소개

NocoBase의 관계 필드 컴포넌트는 사용자가 연관 데이터를 더 효과적으로 표시하고 처리할 수 있도록 돕기 위해 설계되었습니다. 관계 유형에 관계없이, 이 컴포넌트들은 유연하고 범용적이어서 사용자의 특정 요구사항에 맞춰 선택하고 구성할 수 있습니다.

### 드롭다운 선택기

대상 컬렉션이 파일 컬렉션인 경우를 제외한 모든 관계 필드에서, 편집 모드의 기본 컴포넌트는 드롭다운 선택기입니다. 드롭다운 옵션에는 제목 필드의 값이 표시되며, 핵심 필드 정보를 통해 연관 데이터를 빠르게 선택해야 하는 상황에 적합합니다.

![20240429205659](https://static-docs.nocobase.com/20240429205659.png)

더 자세한 내용은 [드롭다운 선택기](/interface-builder/fields/specific/select)를 참조하십시오.

### 데이터 선택기

데이터 선택기는 팝업 모달 형태로 데이터를 표시합니다. 사용자는 데이터 선택기에서 표시할 필드(관계의 관계 필드 포함)를 구성하여 연관 데이터를 더 정확하게 선택할 수 있습니다.

![20240429210824](https://static-docs.nocobase.com/20240429210824.png)

더 자세한 내용은 [데이터 선택기](/interface-builder/fields/specific/picker)를 참조하십시오.

### 서브폼

복잡한 관계 데이터를 처리할 때, 드롭다운 선택기나 데이터 선택기를 사용하는 것은 불편할 수 있습니다. 이러한 경우, 사용자는 팝업을 자주 열어야 합니다. 이럴 때 서브폼을 사용하면, 새로운 팝업을 반복해서 열 필요 없이 현재 페이지 또는 현재 팝업 블록에서 관계 컬렉션 필드를 직접 관리할 수 있어 작업 흐름이 훨씬 원활해집니다. 다단계 관계는 중첩된 폼 형태로 표시됩니다.

![20251029122948](https://static-docs.nocobase.com/20251029122948.png)

더 자세한 내용은 [서브폼](/interface-builder/fields/specific/sub-form)을 참조하십시오.

### 서브 테이블

서브 테이블은 일대다 또는 다대다 관계 레코드를 테이블 형식으로 표시합니다. 이는 연관 데이터를 명확하고 구조화된 방식으로 표시하고 관리하는 방법을 제공하며, 데이터를 일괄적으로 새로 생성하거나 기존 데이터를 선택하여 연결하는 것을 지원합니다.

![20251029123042](https://static-docs.nocobase.com/20251029123042.png)

더 자세한 내용은 [서브 테이블](/interface-builder/fields/specific/sub-table)을 참조하십시오.

### 서브 상세

서브 상세는 서브폼의 읽기 모드에 해당하는 컴포넌트이며, 다단계 관계가 중첩된 데이터를 표시하는 것을 지원합니다.

![20251030213050](https://static-docs.nocobase.com/20251030213050.png)

더 자세한 내용은 [서브 상세](/interface-builder/fields/specific/sub-detail)를 참조하십시오.

### 파일 관리자

파일 관리자는 관계의 대상 컬렉션이 파일 컬렉션인 경우에 특별히 사용되는 관계 필드 컴포넌트입니다.

![20240429222753](https://static-docs.nocobase.com/20240429222753.png)

더 자세한 내용은 [파일 관리자](/interface-builder/fields/specific/file-manager)를 참조하십시오.

### 제목

제목 필드 컴포넌트는 읽기 모드에서 사용되는 관계 필드 컴포넌트입니다. 제목 필드를 구성하여 해당 필드 컴포넌트를 설정할 수 있습니다.

![20251030213327](https://static-docs.nocobase.com/20251030213327.png)

더 자세한 내용은 [제목](/interface-builder/fields/specific/title)을 참조하십시오.
