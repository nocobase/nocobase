# 유효성 검사 규칙 설정

## 소개

유효성 검사 규칙은 사용자가 입력하는 데이터가 예상하는 기준을 충족하는지 확인하는 데 사용됩니다.

## 필드 유효성 검사 규칙 설정 위치

### 컬렉션 필드 유효성 검사 규칙 구성

대부분의 필드는 유효성 검사 규칙 구성을 지원합니다. 필드에 유효성 검사 규칙을 설정하면, 데이터 제출 시 백엔드 유효성 검사가 트리거됩니다. 필드 유형에 따라 지원하는 유효성 검사 규칙이 다릅니다.

- **날짜 필드**

  ![20251028225946](https://static-docs.nocobase.com/20251028225946.png)

- **숫자 필드**

  ![20251028230418](https://static-docs.nocobase.com/20251028230418.png)

- **텍스트 필드**

  텍스트 필드는 텍스트 길이를 제한할 수 있을 뿐만 아니라, 더 정교한 유효성 검사를 위해 사용자 정의 정규 표현식도 지원합니다.

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### 필드 구성의 프런트엔드 유효성 검사

필드 구성에서 설정된 유효성 검사 규칙은 프런트엔드 유효성 검사를 트리거하여 사용자의 입력이 규정을 준수하는지 확인합니다.

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

![20251028230255](https://static-docs.nocobase.com/20251028230255.png)

**텍스트 필드**는 특정 형식 요구 사항을 충족하기 위해 사용자 정의 정규식 유효성 검사도 지원합니다.

![20251028230903](https://static-docs.nocobase.com/20251028230903.png)
