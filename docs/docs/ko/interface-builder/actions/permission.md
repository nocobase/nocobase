# 작업 권한

## 소개

NocoBase 2.0에서 작업 권한은 현재 주로 컬렉션 리소스 권한에 의해 제어됩니다.

-   **컬렉션 리소스 권한**: 다양한 역할이 컬렉션에 대해 생성(Create), 조회(View), 수정(Update), 삭제(Delete) 등 기본 작업을 수행할 수 있는 권한을 통합적으로 제어하는 데 사용됩니다. 이 권한은 데이터 소스 내의 전체 컬렉션에 적용되어, 역할이 다양한 페이지, 팝업 또는 블록에서 해당 컬렉션에 대한 관련 작업을 일관된 권한으로 유지하도록 보장합니다.

### 컬렉션 리소스 권한

NocoBase 권한 시스템에서 컬렉션 작업 권한은 기본적으로 CRUD 차원에 따라 분류되어 권한 관리의 일관성과 표준화를 보장합니다. 예를 들어:

-   **생성 권한(Create)**: 해당 컬렉션의 모든 생성 관련 작업을 제어합니다. 여기에는 추가 작업, 복제 작업 등이 포함됩니다. 역할이 해당 컬렉션에 대한 생성 권한을 가지고 있다면, 해당 컬렉션의 추가 작업 및 복제 작업 등 생성 관련 작업은 모든 페이지 또는 팝업에서 표시됩니다.
-   **삭제 권한(Delete)**: 해당 컬렉션의 삭제 작업을 제어합니다. 테이블 블록에서 일괄 삭제 작업이든, 상세 블록에서 단일 레코드 삭제 작업이든 권한은 일관되게 유지됩니다.
-   **수정 권한(Update)**: 해당 컬렉션의 수정 유형 작업을 제어합니다. 예를 들어, 편집 작업, 레코드 업데이트 작업 등이 있습니다.
-   **조회 권한(View)**: 해당 컬렉션의 데이터 가시성을 제어합니다. 역할이 해당 컬렉션에 대한 조회 권한을 가지고 있을 때만 관련 데이터 블록(테이블, 목록, 상세 등)이 표시됩니다.

이러한 범용 권한 관리 방식은 표준화된 데이터 권한 제어에 적합하며, `다른 페이지, 팝업, 블록`에서 `동일한 컬렉션`에 대한 `동일한 작업`에 `일관된` 권한 규칙이 적용되도록 보장하여 통일성과 유지보수성을 제공합니다.

#### 특정 컬렉션 작업 권한

특정 컬렉션 작업 권한은 데이터 소스의 범용 권한보다 우선하며, 작업 권한을 더욱 세분화하여 특정 컬렉션 리소스 접근에 대한 사용자 지정 권한 구성을 허용합니다. 이러한 권한은 두 가지 측면으로 나뉩니다.

1.  작업 권한: 작업 권한에는 추가, 조회, 편집, 삭제, 내보내기 및 가져오기 작업이 포함됩니다. 이러한 권한은 데이터 범위 차원에 따라 구성됩니다.
    -   모든 레코드: 사용자가 컬렉션의 모든 레코드에 대해 작업을 수행하도록 허용합니다.
    -   자신의 레코드: 사용자가 자신이 생성한 데이터 레코드에 대해서만 작업을 수행하도록 제한합니다.
2.  필드 권한: 필드 권한은 각 필드에 대해 다른 작업에서 권한을 구성할 수 있도록 허용합니다. 예를 들어, 일부 필드는 조회만 허용하고 편집은 허용하지 않도록 구성할 수 있습니다.
