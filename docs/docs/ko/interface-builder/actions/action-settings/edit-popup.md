## 모달 UID

"모달 UID"는 해당 모달을 여는 컴포넌트의 UID입니다. 이 값은 현재 주소 표시줄의 `view/:viewUid`에서 `viewUid` 부분과 일치합니다. 모달을 트리거하는 필드나 버튼의 설정 메뉴에서 "모달 UID 복사"를 클릭하여 빠르게 확인할 수 있습니다.

![popup-copy-uid-20251102](https://static-docs.nocobase.com/popup-copy-uid-20251102.png)

모달 UID를 설정하여 모달 재사용 효과를 얻을 수 있습니다.

### 내부 모달 (기본값)
- "모달 UID"는 현재 작업 버튼의 UID와 같습니다 (기본적으로 이 버튼의 UID를 사용합니다).

### 외부 모달 (모달 재사용)
- "모달 UID"에 다른 위치에 있는 트리거 버튼의 UID(즉, 모달 UID)를 입력하여 해당 모달을 재사용합니다.
- 일반적인 용도: 여러 페이지/블록에서 동일한 모달 UI와 로직을 공유하여 중복 설정을 방지합니다.
- 외부 모달을 사용할 경우, 일부 설정은 수정할 수 없습니다 (아래 참조).

## 기타 관련 설정

- ``Data source / Collection``: 읽기 전용으로, 해당 모달이 바인딩된 데이터 소스를 설명합니다. 기본적으로 현재 블록의 컬렉션을 사용합니다. 외부 모달 모드에서는 대상 모달의 설정을 따르므로 이 설정을 수정할 수 없습니다.
- ``Association name``: 선택 사항입니다. "연결 필드"에서 모달을 열 때 사용되며, 기본값이 있을 때만 표시됩니다. 외부 모달 모드에서는 대상 모달의 설정을 따르므로 이 설정을 수정할 수 없습니다.
- ``Source ID``: ``Association name``이 설정된 경우에만 나타납니다. 기본적으로 현재 컨텍스트의 ``sourceId``를 사용하며, 필요에 따라 변수 또는 고정 값으로 변경할 수 있습니다.
- ``filterByTk``: 비워두거나, 변수 또는 고정 값을 사용할 수 있으며, 모달 데이터 레코드를 제한하는 데 사용됩니다.
