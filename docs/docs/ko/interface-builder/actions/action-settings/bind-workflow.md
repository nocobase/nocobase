# 워크플로우 바인딩

## 소개

일부 작업 버튼에서는 워크플로우를 바인딩하도록 설정하여, 해당 제출 작업을 워크플로우와 연결하고 데이터 자동 처리를 구현할 수 있습니다.

## 지원되는 작업 및 워크플로우 유형

현재 바인딩을 지원하는 작업 버튼과 워크플로우 유형은 다음과 같습니다.

| 작업 버튼 \ 워크플로우 유형 | 작업 전 이벤트 | 작업 후 이벤트 | 승인 이벤트 | 사용자 정의 작업 이벤트 |
| --- | --- | --- | --- | --- |
| 폼의 "제출", "저장" 버튼 | ✅ | ✅ | ✅ | ❌ |
| 데이터 행(테이블, 목록 등)의 "데이터 업데이트" 버튼 | ✅ | ✅ | ✅ | ❌ |
| 데이터 행(테이블, 목록 등)의 "삭제" 버튼 | ✅ | ❌ | ❌ | ❌ |
| "워크플로우 트리거" 버튼 | ❌ | ❌ | ❌ | ✅ |

## 여러 워크플로우 바인딩

하나의 작업 버튼에 여러 워크플로우를 바인딩할 수 있습니다. 여러 워크플로우가 바인딩될 때, 워크플로우 실행 순서는 다음 규칙을 따릅니다.

1.  동일한 트리거 유형의 워크플로우에서는 동기 워크플로우가 먼저 실행되고, 비동기 워크플로우가 나중에 실행됩니다.
2.  동일한 트리거 유형의 워크플로우는 설정된 순서대로 실행됩니다.
3.  서로 다른 트리거 유형의 워크플로우 간에는:
    1.  작업 전 이벤트가 항상 작업 후 이벤트 및 승인 이벤트보다 먼저 실행됩니다.
    2.  작업 후 이벤트와 승인 이벤트는 특정 순서가 없으며, 비즈니스 로직은 설정 순서에 의존해서는 안 됩니다.
