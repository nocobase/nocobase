# 복제

## 소개

복제 작업은 사용자가 기존 데이터를 기반으로 새 레코드를 빠르게 생성할 수 있도록 합니다. **직접 복제**와 **양식으로 복제하여 계속 작성**의 두 가지 복제 방식을 지원합니다.

## 설치

내장 플러그인이므로 별도의 설치가 필요하지 않습니다.

## 복제 모드

![20260209224344](https://static-docs.nocobase.com/20260209224344.png)

### 직접 복제

![20260209224506](https://static-docs.nocobase.com/20260209224506.png)

- 기본적으로 '직접 복제' 방식으로 실행됩니다.
- **템플릿 필드**: 복제할 필드를 지정합니다. 전체 선택이 가능하며 필수 설정 항목입니다.

![20260209225910](https://static-docs.nocobase.com/20260209225910.gif)

설정을 완료한 후 버튼을 클릭하면 데이터가 복제됩니다.

### 양식으로 복제하여 계속 작성

설정된 템플릿 필드가 **기본값**으로 양식에 채워지며, 사용자는 템플릿 필드 값을 수정한 후 제출하여 복제를 완료할 수 있습니다.

![20260209224704](https://static-docs.nocobase.com/20260209224704.png)

**템플릿 필드 설정**: 선택한 필드만 기본값으로 가져옵니다.

![20260209225148](https://static-docs.nocobase.com/20260209225148.png)

#### 양식 필드 동기화

- 현재 양식 블록에 이미 설정된 필드를 자동으로 분석하여 템플릿 필드로 사용합니다.
- 이후 양식 블록 필드를 수정(예: 관계 필드 컴포넌트 조정)한 경우, 템플릿 설정을 다시 열고 **양식 필드 동기화**를 클릭하여 일관성을 유지해야 합니다.

![20260209225450](https://static-docs.nocobase.com/20260209225450.gif)

템플릿 데이터가 양식 기본값으로 채워지며, 사용자는 이를 수정한 후 제출하여 복제를 완료할 수 있습니다.


### 추가 설명

#### 복제, 참조, 미리 불러오기

필드 유형(관계 유형)에 따라 **복제 / 참조 / 미리 불러오기**의 처리 로직이 다릅니다. 관계 필드의 **필드 컴포넌트** 또한 처리 로직에 영향을 미칩니다.

- Select / Record picker: **참조**에 사용
- Sub-form / Sub-table: **복제**에 사용

**복제**

- 일반 필드는 복제됩니다.
- hasOne / hasMany는 복제만 가능합니다. (이러한 관계에는 드롭다운 선택, 레코드 선택기 등의 선택형 필드 컴포넌트 대신 하위 양식, 하위 테이블 등의 필드 컴포넌트를 사용해야 합니다.)
- hasOne / hasMany의 컴포넌트를 변경해도 처리 로직은 **변경되지 않습니다** (복제 유지).
- 복제된 관계 필드의 경우 모든 하위 필드를 선택할 수 있습니다.

**참조**

- belongsTo / belongsToMany는 참조로 처리됩니다.
- 필드 컴포넌트를 '드롭다운 선택'에서 '하위 양식'으로 변경하면 관계가 **참조에서 복제로** 변경됩니다 (복제로 변경되면 모든 하위 필드를 선택할 수 있습니다).

**미리 불러오기 (Preload)**

- 참조 필드 하위의 관계 필드는 미리 불러오기로 처리됩니다.
- 미리 불러오기 필드는 컴포넌트 변경 후 참조 또는 복제로 바뀔 수 있습니다.

#### 전체 선택

- 모든 **복제 필드**와 **참조 필드**를 선택합니다.

#### 데이터 템플릿으로 선택된 레코드에서 다음 필드는 제외됩니다.

- 복제된 관계 데이터의 기본 키(Primary Key)는 제외됩니다. 참조 및 미리 불러오기의 기본 키는 제외되지 않습니다.
- 외래 키 (Foreign Key)
- 중복을 허용하지 않는 필드 (Unique)
- 정렬 필드
- 자동 인코딩(Sequence) 필드
- 비밀번호
- 생성자, 생성 일시
- 최종 수정자, 최종 수정 일시

#### 양식 필드 동기화

- 현재 양식 블록에 설정된 필드를 템플릿 필드로 자동 분석합니다.
- 양식 블록 필드를 수정(예: 관계 필드 컴포넌트 조정)한 후에는 일관성을 위해 다시 동기화해야 합니다.
