# 블록 연동 규칙

## 소개

블록 연동 규칙은 사용자가 블록의 표시를 동적으로 제어하고, 블록 수준에서 요소의 표시를 전체적으로 관리할 수 있도록 합니다. 블록은 필드와 작업 버튼의 컨테이너 역할을 하므로, 이러한 규칙을 통해 사용자는 블록 단위로 전체 뷰의 표시를 유연하게 제어할 수 있습니다.

> **참고**: 블록 연동 규칙을 실행하기 전에 블록 표시는 먼저 **ACL 권한 확인**을 거쳐야 합니다. 사용자가 해당 접근 권한을 가지고 있을 때만 블록 연동 규칙의 판단 로직이 적용됩니다. 다시 말해, 블록 연동 규칙은 ACL 보기 권한 요구 사항이 충족된 후에만 적용되며, 블록 연동 규칙이 없으면 블록은 기본적으로 표시됩니다.

### 전역 변수로 블록 제어

**블록 연동 규칙**은 전역 변수를 사용하여 블록에 표시되는 내용을 동적으로 제어할 수 있도록 지원합니다. 이를 통해 다양한 역할과 권한을 가진 사용자가 맞춤형 데이터 뷰를 보고 조작할 수 있습니다. 예를 들어, 주문 관리 시스템에서 관리자, 영업 사원, 재무 담당자 등 여러 역할이 모두 주문을 볼 수 있는 권한을 가지고 있더라도, 각 역할이 봐야 하는 필드와 작업 버튼은 다를 수 있습니다. 전역 변수를 설정하여 사용자의 역할, 권한 또는 기타 조건에 따라 표시되는 필드, 작업 버튼, 심지어 데이터 정렬 및 필터링 규칙까지 유연하게 조정할 수 있습니다.

#### 구체적인 활용 사례:

- **역할 및 권한 제어**: 다양한 역할의 권한에 따라 특정 필드의 가시성 또는 편집 가능 여부를 제어합니다. 예를 들어, 영업 사원은 주문의 기본 정보만 볼 수 있지만 재무 담당자는 결제 세부 정보를 볼 수 있습니다.
- **개인화된 뷰**: 다양한 부서나 팀을 위해 맞춤형 블록 뷰를 제공하여 각 사용자가 자신의 업무와 관련된 내용만 볼 수 있도록 보장하고, 이를 통해 업무 효율성을 높입니다.
- **작업 권한 관리**: 전역 변수를 사용하여 작업 버튼의 표시를 제어합니다. 예를 들어, 일부 역할은 데이터만 볼 수 있지만 다른 역할은 수정, 삭제 등의 작업을 수행할 수 있습니다.

### 컨텍스트 변수로 블록 제어

블록은 컨텍스트 내의 변수를 통해서도 표시를 제어할 수 있습니다. 예를 들어, '현재 레코드', '현재 폼', '현재 팝업 레코드'와 같은 컨텍스트 변수를 사용하여 블록을 동적으로 표시하거나 숨길 수 있습니다.

예시: 주문 상태가 '결제 완료'인 경우에만 '주문 기회 정보' 블록이 표시됩니다.
