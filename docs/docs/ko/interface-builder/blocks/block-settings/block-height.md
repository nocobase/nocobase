# 블록 높이

## 소개

블록 높이는 **기본 높이**, **지정 높이**, **전체 높이**의 세 가지 모드를 지원합니다. 대부분의 블록에서 높이 설정을 지원합니다.

![20260211153947](https://static-docs.nocobase.com/20260211153947.png)

![20260211154020](https://static-docs.nocobase.com/20260211154020.png)

## 높이 모드

### 기본 높이

블록 유형에 따라 기본 높이 정책이 다릅니다. 예를 들어, 테이블 및 폼 블록은 콘텐츠에 따라 높이가 자동으로 조절되며, 블록 내부에 스크롤바가 나타나지 않습니다.

### 지정 높이

블록 외곽 프레임의 전체 높이를 수동으로 지정할 수 있습니다. 블록 내부에서는 사용 가능한 높이를 자동으로 계산하여 할당합니다.

![20260211154043](https://static-docs.nocobase.com/20260211154043.gif)

### 전체 높이

전체 높이 모드는 지정 높이와 유사하지만, 블록 높이가 현재 브라우저의 **가시 영역(Viewport)**을 기준으로 계산되어 화면에 가득 차는 최대 높이로 설정됩니다. 브라우저 페이지에는 스크롤바가 나타나지 않으며, 블록 내부에만 스크롤바가 나타납니다.

전체 높이 모드에서 블록별 내부 스크롤 처리 방식은 다음과 같이 약간의 차이가 있습니다:

- **테이블**: `tbody` 내부 스크롤
- **폼 / 상세**: 그리드(Grid) 내 스크롤 (작업 영역을 제외한 콘텐츠 스크롤)
- **리스트 / 그리드 카드**: 그리드(Grid) 내 스크롤 (작업 영역 및 페이지네이션 바를 제외한 콘텐츠 스크롤)
- **지도 / 캘린더**: 전체 높이 자동 조절, 스크롤바 없음
- **Iframe / Markdown**: 블록 외곽 프레임의 전체 높이를 제한하며, 블록 내부에 스크롤바가 나타남
