# 필드 연동 규칙

## 소개

필드 연동 규칙은 사용자의 행동에 따라 폼/상세 블록의 필드 상태를 동적으로 조절할 수 있도록 합니다. 현재 필드 연동 규칙을 지원하는 블록은 다음과 같습니다:

- [폼 블록](/interface-builder/blocks/data-blocks/form)
- [상세 블록](/interface-builder/blocks/data-blocks/details)
- [하위 폼](/interface-builder/fields/specific/sub-form)

## 사용 방법

### **폼 블록**

폼 블록에서는 연동 규칙을 사용하여 특정 조건에 따라 필드의 동작을 동적으로 조절할 수 있습니다:

- **필드 표시/숨기기 제어**: 다른 필드의 값에 따라 현재 필드를 표시할지 숨길지 결정합니다.
- **필수 입력 설정**: 특정 조건에서 필드를 필수 입력으로 설정하거나 해제할 수 있습니다.
- **값 할당**: 조건에 따라 필드에 값을 자동으로 할당합니다.
- **지정된 JavaScript 실행**: 비즈니스 요구사항에 따라 JavaScript를 작성합니다.

### **상세 블록**

상세 블록에서는 연동 규칙이 주로 상세 블록의 필드 표시/숨기기를 동적으로 제어하는 데 사용됩니다.

## 속성 연동

### 값 할당

예시: 주문이 보충 주문으로 선택되면, 주문 상태는 자동으로 '검토 대기'로 할당됩니다.

![20251029115348](https://static-docs.nocobase.com/20251029115348.png)

### 필수 입력

예시: 주문 상태가 '결제 완료'일 때, 주문 금액은 필수 입력입니다.

![20251029115031](https://static-docs.nocobase.com/20251029115031.png)

### 표시/숨기기

예시: 주문 상태가 '결제 대기'일 때만 결제 계좌와 총 금액이 표시됩니다.
