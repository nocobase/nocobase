# 참조 블록

## 소개
참조 블록은 대상 블록의 UID를 입력하여 이미 구성된 블록을 현재 페이지에 바로 표시합니다. 블록을 다시 구성할 필요가 없습니다.

## 플러그인 활성화
이 플러그인은 내장되어 있지만 기본적으로 비활성화되어 있습니다.
“플러그인 관리”를 열고 → “블록: 참조”를 찾아 → “활성화”를 클릭합니다.

![플러그인 관리자에서 참조 블록 활성화](https://static-docs.nocobase.com/block-reference-enable-20251102.png)

## 블록 추가 방법
1) 블록 추가 → “기타 블록” 그룹 → “참조 블록”을 선택합니다.  
2) “참조 블록 설정”에서 다음을 구성합니다:
   - `블록 UID`: 대상 블록의 UID
   - `참조 모드`: `참조` 또는 `복사`를 선택합니다.

![참조 블록 추가 및 구성 데모](https://static-docs.nocobase.com/20251102193949_rec_.gif)

### 블록 UID 가져오는 방법
- 대상 블록의 설정 메뉴를 열고 `UID 복사`를 클릭하면 해당 블록의 UID를 복사할 수 있습니다.

![블록 설정에서 UID 복사](https://static-docs.nocobase.com/block-reference-copy-uid-20251102.png)

## 모드 및 동작
- `참조` (기본값)
  - 원본 블록과 동일한 구성을 공유합니다. 원본 블록이나 참조된 어느 곳에서든 수정하면 모든 참조가 동기화되어 업데이트됩니다.

- `복사`
  - 원본 블록과 동일한 독립적인 블록을 생성합니다. 이후 각자의 수정 사항은 서로 영향을 주지 않으며 동기화되지 않습니다.

## 구성 설명
- 참조 블록:
  - `참조 블록 설정`: 대상 블록 UID를 지정하고 “참조/복사” 모드를 선택하는 데 사용됩니다.
  - 동시에 “참조된 블록”의 전체 설정이 표시됩니다. (원본 블록에서 직접 구성하는 것과 같습니다.)

![참조 블록 설정 화면](https://static-docs.nocobase.com/block-reference-settings-20251102.png)

- 복사된 블록:
  - 복사 후 생성된 블록은 원본 블록과 유형이 동일하며, 해당 블록 자체의 설정만 포함합니다.
  - 더 이상 `참조 블록 설정`을 포함하지 않습니다.

## 오류 및 대체 상태
- 대상이 없거나 유효하지 않을 때: 오류 상태 메시지를 표시합니다. 참조 블록 설정(참조 블록 설정 → 블록 UID)에서 블록 UID를 다시 지정하고 저장하면 다시 표시됩니다.

![대상 블록이 유효하지 않을 때의 오류 상태](https://static-docs.nocobase.com/block-reference-error-20251102.png)

## 참고 및 제한 사항
- 실험적인 기능이므로 프로덕션 환경에서는 주의하여 사용하십시오.
- 블록을 복사할 때 대상 UID에 의존하는 일부 구성은 다시 구성해야 할 수 있습니다.
- 참조 블록의 모든 구성은 “데이터 범위” 등의 구성을 포함하여 자동으로 동기화됩니다. 하지만 참조 블록은 자체 [이벤트 흐름 구성](/interface-builder/event-flow/)을 가질 수 있으므로, 이벤트 흐름 및 사용자 정의 JavaScript 작업을 통해 간접적으로 다른 데이터 범위 또는 관련 구성을 달성할 수 있습니다.
