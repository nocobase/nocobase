# 블록

블록은 데이터와 콘텐츠를 담는 컨테이너입니다. 페이지(Page), 모달(Modal), 또는 드로어(Drawer) 안에 배치할 수 있으며, 여러 블록을 자유롭게 드래그하여 정렬할 수 있습니다.

## 블록 유형

![블록 유형](https://static-docs.nocobase.com/f71af45b5cd914ea0558f760ddbbba58.png)

- 데이터 블록: 데이터 소스에서 가져온 데이터를 인터페이스에 표시하는 데 사용됩니다.
- 필터 블록: 데이터 소스의 데이터를 필터 조건으로 사용하여 다른 데이터 블록을 필터링하는 데 사용됩니다.
- 기타 블록: 워크플로우 할 일, 감사 로그, 마크다운 등 특정하거나 독립적인 콘텐츠를 담는 데 사용됩니다.

## 블록 추가

블록은 페이지(Page), 모달(Modal), 또는 드로어(Drawer) 안에 배치할 수 있습니다.

### 페이지 내 블록

현재 페이지 내 블록 유형에는 데이터 블록, 필터 블록, 기타 블록이 있습니다.

![20251023222441](https://static-docs.nocobase.com/20251023222441.png)

### 팝업(모달 또는 드로어) 내 블록

팝업에는 모달과 드로어 두 가지 유형이 있습니다. 페이지와 마찬가지로 블록을 추가할 수 있지만, 팝업 내 폼 블록은 일반적으로 단일 레코드의 추가, 편집 또는 조회 등에 사용된다는 차이가 있습니다. 블록 유형에는 데이터 블록과 기타 블록이 포함됩니다.

![20251023222613](https://static-docs.nocobase.com/20251023222613.png)

## 블록 디자이너

각 블록의 오른쪽 상단에는 세 개의 작은 아이콘이 있으며, 왼쪽부터 순서대로 다음과 같습니다.

1. 드래그 앤 드롭 레이아웃
2. [이벤트 흐름](/interface-builder/event-flow)
3. 블록 설정

![20251023224032](https://static-docs.nocobase.com/20251023224032.png)

간단한 블록의 모든 설정 옵션은 '블록 설정'에 집중되어 있으며, JS 블록이 그 예입니다.

![20251023224903](https://static-docs.nocobase.com/20251023224903.png)

복잡한 데이터 유형 블록은 별도로 내장된 '필드 구성'과 '작업 구성'도 제공합니다.

![20251023225141](https://static-docs.nocobase.com/20251023225141.png)

이 외에도 차트 블록처럼 더 많은 중첩 가능성을 자유롭게 활용할 수 있습니다.

![](https://static-docs.nocobase.com/07588190b3f41ae3060e71d8b76b4447.png)

## 블록 레이아웃

여러 블록의 레이아웃은 드래그 앤 드롭으로 조정할 수 있습니다.
