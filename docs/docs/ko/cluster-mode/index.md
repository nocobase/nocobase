# 클러스터 모드

## 소개

NocoBase는 v1.6.0 버전부터 애플리케이션을 클러스터 모드로 실행하는 것을 지원합니다. 애플리케이션이 클러스터 모드로 실행될 때, 여러 인스턴스와 다중 코어 모드를 사용하여 동시 접근 처리 성능을 향상시킬 수 있습니다.

## 시스템 아키텍처

![20251031221530](https://static-docs.nocobase.com/20251031221530.png)

*   **애플리케이션 클러스터**: NocoBase 애플리케이션의 여러 인스턴스로 구성된 클러스터입니다. 여러 서버에 배포하거나 단일 서버에서 다중 코어 모드로 여러 프로세스를 실행할 수 있습니다.
*   **데이터베이스**: 애플리케이션 데이터를 저장합니다. 단일 노드 데이터베이스 또는 분산 데이터베이스일 수 있습니다.
*   **공유 스토리지**: 애플리케이션 파일과 데이터를 저장하는 데 사용되며, 여러 인스턴스에서 읽기/쓰기 접근을 지원합니다.
*   **미들웨어**: 캐시, 동기화 신호, 메시지 큐, 분산 락 등의 컴포넌트를 포함하며, 애플리케이션 클러스터의 통신 및 조정을 지원합니다.
*   **로드 밸런서**: 클라이언트 요청을 애플리케이션 클러스터의 여러 인스턴스로 분배하고, 상태 확인 및 장애 조치를 수행합니다.
