# 서버 로그, 감사 로그 및 기록 내역

## 서버 로그

### 시스템 로그

> [시스템 로그](#)에서 자세히 알아보세요.

- 애플리케이션 시스템의 런타임 정보를 기록하고, 코드 실행 흐름을 추적하며, 코드 실행 오류와 같은 예외 정보를 파악합니다.
- 로그는 심각도 수준 및 기능 모듈별로 분류됩니다.
- 터미널로 출력되거나 파일 형태로 저장됩니다.
- 주로 시스템 운영 중 발생하는 예외 상황을 진단하고 문제 해결하는 데 사용됩니다.

### 요청 로그

> [요청 로그](#)에서 자세히 알아보세요.

- HTTP API 요청 및 응답 정보를 기록하며, 요청 ID, API 경로, 헤더, 응답 상태 코드, 소요 시간 등의 정보에 중점을 둡니다.
- 터미널로 출력되거나 파일 형태로 저장됩니다.
- 주로 API 호출 및 실행 상황을 추적하는 데 사용됩니다.

## 감사 로그

> [감사 로그](../security/audit-logger/index.md)에서 자세히 알아보세요.

- 사용자(또는 API)가 시스템 리소스에 대해 수행한 작업 행위를 기록하며, 리소스 유형, 대상 객체, 작업 유형, 사용자 정보, 작업 상태 등에 중점을 둡니다.
- 사용자 작업의 구체적인 내용과 결과를 더 잘 추적하기 위해 요청 파라미터와 응답은 메타데이터 정보로 기록됩니다. 이 정보는 요청 로그와 부분적으로 중복되지만 완전히 동일하지는 않습니다. 예를 들어, 기존 요청 로그에는 일반적으로 전체 요청 본문이 기록되지 않습니다.
- 요청 파라미터와 응답은 데이터 스냅샷과 **동일하지 않습니다**. 이 정보는 어떤 종류의 작업이 발생했는지는 보여줄 수 있지만, 수정 전 데이터 테이블 기록의 정확한 내용을 알 수는 없으므로 버전 관리나 잘못된 작업 후 데이터를 복원하는 데 사용할 수 없습니다.
- 파일 및 데이터베이스 테이블 형태로 저장됩니다.

## 기록 내역

> [기록 내역](/record-history/index.md)에서 자세히 알아보세요.

- 데이터 내용의 **변경 내역**을 기록합니다.
- 주로 리소스 유형, 리소스 객체, 작업 유형, 변경된 필드, 변경 전후 값을 기록합니다.
- 데이터 비교에 활용할 수 있습니다.
- 데이터베이스 테이블 형태로 저장됩니다.
