# 서버 로그, 감사 로그, 기록 이력

## 서버 로그

### 시스템 실행 로그

> [시스템 로그](./index.md#시스템-로그) 참고

- 애플리케이션 시스템의 실행 정보를 기록하고, 코드 로직의 실행 경로를 추적하며, 코드 실행 오류 등 예외 정보를 추적합니다.
- 로그 레벨이 존재하며, 기능 모듈별로 분류됩니다.
- 터미널 출력 또는 파일 형태로 저장됩니다.
- 주로 시스템 운영 중 발생하는 이상 상황을 파악하고 해결하는 데 사용됩니다.

### 요청 로그

> [요청 로그](./index.md#요청-로그) 참고

- HTTP API 요청 및 응답 정보를 기록하며 요청 ID, API 경로, 요청 헤더, 응답 상태 코드, 소요 시간 등의 정보 기록에 중점을 둡니다.
- 터미널 출력 또는 파일 형태로 저장됩니다.
- 주로 API 호출 및 실행 상태를 추적하는 데 사용됩니다.

## 감사 로그

> [감사 로그](/security/audit-logger/index.md) 참고

- 사용자(API)의 시스템 리소스에 대한 조작 행위를 기록하며 리소스 유형, 리소스 객체, 조작 유형, 사용자 정보, 조작 상태 등에 중점을 둡니다.
- 사용자 조작의 구체적인 내용과 결과를 더 잘 추적하기 위해 요청 파라미터와 응답 정보가 메타데이터(MetaData)로 기록됩니다. 이 정보는 요청 로그와 일부 중복되지만 완전히 일치하지는 않습니다. 예를 들어, 일반적인 요청 로그에는 전체 요청 본문(Request Body)이 기록되지 않는 경우가 많습니다.
- 요청 파라미터와 응답 정보는 리소스의 스냅샷과 동일하지 않습니다. 파라미터와 코드 로직을 통해 어떤 수정이 발생했는지는 알 수 있지만, 데이터 테이블 레코드가 수정되기 전의 정확한 내용을 알 수는 없으므로 버전 관리나 오작동 후의 데이터 복구에는 사용할 수 없습니다.
- 파일 및 데이터 테이블 형태로 저장됩니다.

## 기록 이력

> [기록 이력](/record-history/index.md) 참고

- 데이터 내용의 변경 이력을 기록합니다.
- 주로 리소스 유형, 리소스 객체, 조작 유형, 변경된 필드, 변경 전후의 값을 기록합니다.
- 데이터 비교에 사용할 수 있습니다.
- 데이터 테이블 형태로 저장됩니다.
