# 파일 관리자

## 소개

파일 관리자 플러그인은 파일을 효율적으로 관리하기 위해 파일 컬렉션, 첨부 파일 필드 및 파일 저장 엔진을 제공합니다. 파일은 특정 구조를 가진 데이터 컬렉션의 레코드이며, 이 특정 구조의 데이터 컬렉션을 파일 컬렉션이라고 합니다. 파일 컬렉션은 파일의 메타 정보를 저장하는 데 사용되며 파일 관리자를 통해 관리할 수 있습니다. 첨부 파일 필드는 파일 컬렉션과 연결된 특정 관계 필드입니다. 파일은 다양한 저장 방식을 지원하며, 현재 지원되는 파일 저장 엔진은 로컬 저장소, 알리바바 클라우드 OSS, 아마존 S3 및 텐센트 클라우드 COS를 포함합니다.

## 사용자 매뉴얼

### 파일 컬렉션

모든 첨부 파일 필드와 연결된 파일을 저장하기 위한 `attachments` 컬렉션이 내장되어 있습니다. 이 외에도 새로운 파일 컬렉션을 생성하여 특정 파일을 저장할 수 있습니다.

[더 많은 사용법은 파일 컬렉션 소개 문서를 참조하세요.](/data-sources/file-manager/file-collection)

### 첨부 파일 필드

첨부 파일 필드는 파일 컬렉션과 연결된 특정 관계 필드이며, '첨부 파일 유형 필드'를 통해 생성하거나 '관계 필드'를 통해 구성할 수 있습니다.

[더 많은 사용법은 첨부 파일 필드 소개 문서를 참조하세요.](/data-sources/file-manager/field-attachment)

### 파일 저장 엔진

파일 저장 엔진은 파일을 특정 서비스에 저장하는 데 사용되며, 여기에는 로컬 저장소(서버 하드 드라이브에 저장) 및 클라우드 저장소 등이 포함됩니다.

[더 자세한 내용은 파일 저장 엔진 소개 문서를 참조하세요.](./storage/index.md)

### HTTP API

파일 업로드는 HTTP API를 통해 처리할 수 있습니다. [HTTP API](./http-api.md)를 참조하세요.
