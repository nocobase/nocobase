# 개요

## 내장 엔진

현재 NocoBase에서 지원하는 내장 엔진 유형은 다음과 같습니다.

- [로컬 스토리지](./local.md)
- [알리바바 클라우드 OSS](./aliyun-oss.md)
- [아마존 S3](./amazon-s3.md)
- [텐센트 클라우드 COS](./tencent-cos.md)

시스템 설치 시 로컬 스토리지 엔진이 자동으로 추가되며, 바로 사용할 수 있습니다. 필요에 따라 새로운 엔진을 추가하거나 기존 엔진의 매개변수를 편집할 수도 있습니다.

## 공통 엔진 매개변수

각 엔진 유형별 고유 매개변수 외에, 다음은 공통 매개변수입니다(로컬 스토리지를 예시로 설명합니다).

![파일 스토리지 엔진 설정 예시](https://static-docs.nocobase.com/20240529115151.png)

### 제목

스토리지 엔진의 이름으로, 사용자가 식별하기 위해 사용합니다.

### 시스템 이름

스토리지 엔진의 시스템 이름으로, 시스템이 식별하기 위해 사용합니다. 시스템 내에서 고유해야 하며, 비워두면 시스템이 자동으로 무작위로 생성합니다.

### 접근 기본 URL

해당 파일을 외부에서 접근할 수 있는 URL 주소의 접두사 부분입니다. CDN의 접근 기본 URL이 될 수 있으며, 예시로 "`https://cdn.nocobase.com/app`"과 같습니다(끝에 "`/`"는 필요 없습니다).

### 경로

파일을 저장할 때 사용되는 상대 경로입니다. 이 부분은 파일 접근 시 최종 URL에 자동으로 연결됩니다. 예시로 "`user/avatar`"와 같습니다(시작과 끝에 "`/`"는 필요 없습니다).

### 파일 크기 제한

이 스토리지 엔진에 파일을 업로드할 때의 크기 제한입니다. 설정된 크기를 초과하는 파일은 업로드할 수 없습니다. 시스템 기본 제한은 20MB이며, 최대 1GB까지 조정할 수 있습니다.

### 파일 유형

업로드할 수 있는 파일 유형을 제한할 수 있으며, [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types) 문법 설명 형식을 사용합니다. 예를 들어, `image/*`는 이미지 파일을 나타냅니다. 여러 유형은 쉼표(`,`)로 구분할 수 있으며, `image/*, application/pdf`와 같이 입력하면 이미지 및 PDF 유형 파일을 허용합니다.

### 기본 스토리지 엔진

이 옵션을 선택하면 시스템의 기본 스토리지 엔진으로 설정됩니다. 첨부 파일 필드나 파일 컬렉션에서 스토리지 엔진을 지정하지 않은 경우, 업로드되는 모든 파일은 기본 스토리지 엔진에 저장됩니다. 기본 스토리지 엔진은 삭제할 수 없습니다.

### 레코드 삭제 시 파일 유지

이 옵션을 선택하면 첨부 파일 컬렉션 또는 파일 컬렉션의 데이터 레코드가 삭제되더라도 스토리지 엔진에 업로드된 파일은 유지됩니다. 기본적으로 이 옵션은 선택되어 있지 않으며, 이는 레코드 삭제 시 스토리지 엔진의 파일도 함께 삭제됨을 의미합니다.

:::info{title=팁}
파일이 업로드되면 최종 접근 경로는 여러 부분이 연결되어 구성됩니다.

```
<접근 기본 URL>/<경로>/<파일 이름><확장자>
```

예시: `https://cdn.nocobase.com/app/user/avatar/20240529115151.png`
:::
