# 상속 컬렉션

## 소개

:::warning
메인 데이터베이스가 PostgreSQL일 때만 지원됩니다.
:::

부모 컬렉션을 생성하고, 이 부모 컬렉션에서 자식 컬렉션을 파생시킬 수 있습니다. 자식 컬렉션은 부모 컬렉션의 구조를 상속받으면서 자체 열을 정의할 수도 있습니다. 이러한 설계 패턴은 구조는 비슷하지만 일부 차이가 있을 수 있는 데이터를 구성하고 관리하는 데 도움이 됩니다.

상속 컬렉션의 몇 가지 일반적인 특징은 다음과 같습니다:

-   **부모 컬렉션**: 부모 컬렉션은 공통 열과 데이터를 포함하며, 전체 상속 계층 구조의 기본 구조를 정의합니다.
-   **자식 컬렉션**: 자식 컬렉션은 부모 컬렉션의 구조를 상속받지만, 추가적으로 자체 열을 정의할 수 있습니다. 이를 통해 각 자식 컬렉션은 부모 컬렉션의 공통 속성을 가지면서도 서브클래스에 특정한 속성을 포함할 수 있습니다.
-   **쿼리**: 쿼리 시에는 전체 상속 계층 구조를 쿼리하거나, 부모 컬렉션 또는 특정 자식 컬렉션만 쿼리할 수 있습니다. 이를 통해 필요에 따라 다양한 수준의 데이터를 검색하고 처리할 수 있습니다.
-   **상속 관계**: 부모 컬렉션과 자식 컬렉션 사이에 상속 관계가 설정됩니다. 이는 부모 컬렉션의 구조를 사용하여 일관된 속성을 정의할 수 있음을 의미하며, 동시에 자식 컬렉션이 이러한 속성을 확장하거나 재정의할 수 있도록 합니다.

이러한 설계 패턴은 데이터 중복을 줄이고, 데이터베이스 모델을 단순화하며, 데이터 유지보수를 용이하게 하는 데 도움이 됩니다. 하지만 상속 컬렉션은 쿼리의 복잡성을 증가시킬 수 있으므로, 특히 전체 상속 계층 구조를 다룰 때 신중하게 사용해야 합니다. 상속 컬렉션을 지원하는 데이터베이스 시스템은 일반적으로 이러한 컬렉션 구조를 관리하고 쿼리하기 위한 특정 구문과 도구를 제공합니다.
