# 표현식 컬렉션

## "표현식" 컬렉션 템플릿 생성하기

워크플로우 내에서 동적 표현식 연산 노드를 사용하기 전에, 데이터 컬렉션 관리 도구에서 "표현식" 템플릿 컬렉션을 먼저 생성해야 합니다. 이 컬렉션은 다양한 표현식을 저장하는 데 사용됩니다.

![표현식 컬렉션 템플릿 생성하기](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## 표현식 데이터 입력하기

다음으로, 테이블 블록을 생성하고 해당 템플릿 컬렉션에 몇 가지 수식 데이터를 추가할 수 있습니다. "표현식" 템플릿 컬렉션의 각 행 데이터는 특정 컬렉션 데이터 모델에 대한 계산 규칙으로 이해될 수 있습니다. 각 수식 데이터 행은 다양한 컬렉션의 데이터 모델에 있는 필드 값을 변수로 사용하여 계산 규칙으로 작동하는 고유한 표현식을 작성할 수 있습니다. 물론, 필요에 따라 다른 계산 엔진을 사용할 수도 있습니다.

![표현식 데이터 입력하기](https://static-docs.nocobase.com/761047f8daabacccbc6a6a24a73564093.png)

:::info{title=팁}
수식을 생성한 후에는 비즈니스 데이터와 수식을 연결해야 합니다. 각 비즈니스 데이터 행을 수식 데이터 행에 직접 연결하는 것은 번거로울 수 있습니다. 따라서 일반적으로 분류 컬렉션과 유사한 메타데이터 컬렉션을 사용하여 수식 컬렉션과 다대일(또는 일대일) 관계를 만듭니다. 그런 다음 비즈니스 데이터를 분류된 메타데이터와 다대일 관계로 연결합니다. 이러한 접근 방식을 통해 비즈니스 데이터를 생성할 때 관련 분류된 메타데이터만 지정하면, 설정된 연결 경로를 통해 해당 수식 데이터를 쉽게 찾아 활용할 수 있습니다.
:::

## 프로세스에 관련 데이터 로드하기

예를 들어, 컬렉션 이벤트를 사용하여 워크플로우를 생성해 보겠습니다. 주문이 생성될 때 트리거되며, 주문과 관련된 상품 데이터 및 상품 관련 표현식 데이터를 미리 로드해야 합니다.
