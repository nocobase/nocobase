## 4. 백업 및 마이그레이션 지원 비교

| 기능 | 메인 데이터베이스 | 외부 데이터베이스 |
|-----|---------|-----------|
| 백업 및 복원 | ✅ | ❌ (사용자 직접 관리 필요) |
| 마이그레이션 관리 | ✅ | ❌ (사용자 직접 관리 필요) |

:::info
NocoBase는 메인 데이터베이스의 백업, 복원 및 구조 마이그레이션 기능을 제공합니다. 외부 데이터베이스의 경우, 이러한 작업은 사용자가 자체 데이터베이스 환경에 따라 개별적으로 완료해야 하며, NocoBase는 내장된 지원을 제공하지 않습니다.
:::

## 요약 비교

| 비교 항목 | 메인 데이터베이스 | 외부 데이터베이스 |
|-------|---------|-----------|
| 데이터베이스 유형 | PostgreSQL, MySQL, MariaDB, KingbaseES | PostgreSQL, MySQL, MariaDB, MSSQL, Oracle, KingbaseES |
| 컬렉션 유형 지원 | 모든 컬렉션 유형 | 일반 컬렉션 및 뷰 컬렉션만 지원 |
| 필드 유형 지원 | 모든 필드 유형 | 첨부 파일 필드를 제외한 모든 필드 유형 |
| 백업 및 마이그레이션 | 내장 지원 | 사용자 직접 관리 필요 |

## 권장 사항

- **NocoBase를 사용하여 새로운 비즈니스 시스템을 구축하는 경우**, **메인 데이터베이스**를 사용하세요. NocoBase의 모든 기능을 활용할 수 있습니다.
- **NocoBase를 사용하여 다른 시스템의 데이터베이스에 연결하여 기본적인 CRUD(생성, 읽기, 업데이트, 삭제) 작업을 수행하는 경우**, **외부 데이터베이스**를 사용하세요.
