# 일대다

클래스와 학생의 관계는 일대다 관계의 예시입니다. 하나의 클래스에는 여러 명의 학생이 있을 수 있지만, 각 학생은 하나의 클래스에만 속할 수 있습니다. 이러한 관계를 일대다 관계라고 합니다.

## 파라미터 설명

### Source collection

소스 컬렉션은 현재 필드가 속한 컬렉션입니다.

### Target collection

타겟 컬렉션은 연결할 컬렉션입니다.

### Source key

소스 키는 외래 키 제약 조건이 참조하는 소스 컬렉션의 필드입니다. 고유해야 합니다.

### Foreign key

외래 키는 두 컬렉션 간의 연결을 설정하는 데 사용되는 타겟 컬렉션의 필드입니다.

### Target key

타겟 키는 관계 블록에서 각 행 레코드를 조회하는 데 사용되는 타겟 컬렉션의 필드이며, 일반적으로 고유한 필드입니다.

### ON DELETE

ON DELETE는 부모 컬렉션의 레코드를 삭제할 때 관련 자식 컬렉션의 외래 키 참조에 적용되는 규칙을 의미합니다. 이는 외래 키 제약 조건을 정의할 때 사용되는 옵션입니다. 일반적인 ON DELETE 옵션은 다음과 같습니다:

- **CASCADE**: 부모 컬렉션의 레코드를 삭제할 때, 자식 컬렉션의 모든 관련 레코드를 자동으로 삭제합니다.
- **SET NULL**: 부모 컬렉션의 레코드를 삭제할 때, 자식 컬렉션의 관련 레코드에 있는 외래 키 값을 NULL로 설정합니다.
- **RESTRICT**: 기본 옵션으로, 자식 컬렉션에 관련 레코드가 있는 경우 부모 컬렉션 레코드의 삭제를 방지합니다.
- **NO ACTION**: RESTRICT와 유사하게, 자식 컬렉션에 관련 레코드가 있는 경우 부모 컬렉션 레코드의 삭제를 방지합니다.
