# 개요

## 날짜 및 시간 필드 유형

날짜 및 시간 필드 유형은 다음과 같습니다:

- **날짜 및 시간 (시간대 포함)** - 날짜 및 시간은 UTC(협정 세계시)로 통일되어 변환되며, 필요에 따라 시간대 변환이 적용됩니다.
- **날짜 및 시간 (시간대 미포함)** - 시간대 정보 없이 날짜와 시간을 저장합니다.
- **날짜 (시간 미포함)** - 날짜 정보만 저장하며, 시간 부분은 포함하지 않습니다.
- **시간** - 시간 정보만 저장하며, 날짜 부분은 포함하지 않습니다.
- **Unix 타임스탬프** - Unix 타임스탬프로 저장되며, 일반적으로 1970년 1월 1일 이후 경과된 초를 나타냅니다.

각 날짜 관련 필드 유형의 예시는 다음과 같습니다:

| **필드 유형**               | **예시 값**                  | **설명**                                          |
|------------------------------|-----------------------------|---------------------------------------------------|
| 날짜 및 시간 (시간대 포함)    | 2024-08-24T07:30:00.000Z    | 날짜 및 시간은 UTC(협정 세계시)로 통일되어 변환됩니다. |
| 날짜 및 시간 (시간대 미포함)  | 2024-08-24 15:30:00         | 시간대 정보 없이 날짜와 시간만 기록합니다.           |
| 날짜 (시간 미포함)           | 2024-08-24                  | 날짜 정보만 저장하며, 시간은 포함하지 않습니다.     |
| 시간                         | 15:30:00                    | 시간 정보만 저장하며, 날짜는 포함하지 않습니다.     |
| Unix 타임스탬프              | 1724437800                  | UTC 1970년 1월 1일 00:00:00부터 경과된 초를 나타냅니다. |

## 데이터 소스별 비교

NocoBase, MySQL, PostgreSQL의 비교표는 다음과 같습니다:

| **필드 유형**               | **NocoBase**               | **MySQL**                  | **PostgreSQL**                         |
|------------------------------|----------------------------|----------------------------|----------------------------------------|
| 날짜 및 시간 (시간대 포함)    | Datetime with timezone     | TIMESTAMP<br/> DATETIME    | TIMESTAMP WITH TIME ZONE               |
| 날짜 및 시간 (시간대 미포함)  | Datetime without timezone  | DATETIME                   | TIMESTAMP WITHOUT TIME ZONE            |
| 날짜 (시간 미포함)           | Date                       | DATE                       | DATE                                   |
| 시간                         | Time                       | TIME                       | TIME WITHOUT TIME ZONE                 |
| Unix 타임스탬프              | Unix timestamp             | INTEGER<br/>BIGINT         | INTEGER<br/>BIGINT                     |
| 시간 (시간대 포함)           | -                          | -                          | TIME WITH TIME ZONE                    |

참고:
- MySQL의 `TIMESTAMP` 데이터 범위는 UTC `1970-01-01 00:00:01`부터 `2038-01-19 03:14:07`까지입니다. 이 범위를 벗어나는 경우, `DATETIME` 또는 `BIGINT`를 사용하여 Unix 타임스탬프를 저장하는 것을 권장합니다.

### 시간대 포함

`날짜 및 시간 (시간대 포함)`과 `Unix 타임스탬프`를 포함합니다.

참고:
- 더 넓은 데이터 범위를 지원하기 위해 NocoBase의 날짜 및 시간 (시간대 포함) 필드는 MySQL 데이터베이스에서 `DATETIME` 유형을 사용합니다. 저장되는 날짜 값은 서버의 `TZ` 환경 변수에 따라 변환된 값이며, `TZ` 환경 변수가 변경되면 저장된 날짜 및 시간 값도 변경될 수 있습니다.
- UTC 시간과 로컬 시간 사이에는 시간대 차이가 존재하므로, UTC 원본 값을 직접 표시하면 사용자가 오해할 수 있습니다.

## UTC

UTC(협정 세계시, Coordinated Universal Time)는 전 세계 시간을 조정하고 통일하는 데 사용되는 국제 표준시입니다. 원자시계를 기반으로 하는 고정밀 시간 표준이며, 지구 자전 시간과 동기화됩니다.

UTC 시간과 로컬 시간 사이에는 시간대 차이가 존재하며, UTC 원본 값을 직접 표시하면 사용자가 오해할 수 있습니다. 예를 들어:

| **시간대**       | **날짜 및 시간**                  |
|----------------|----------------------------------|
| UTC            | 2024-08-24T07:30:00.000Z          |
| UTC+8          | 2024-08-24 15:30:00               |
| UTC+5          | 2024-08-24 12:30:00               |
| UTC-5          | 2024-08-24 02:30:00               |
| UTC+0          | 2024-08-24 07:30:00               |
| UTC-6          | 2024-08-23 01:30:00               |

위에 표시된 모든 시간은 동일한 시점을 나타내며, 단지 시간대만 다를 뿐입니다.
