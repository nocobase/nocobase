# Git 소스 코드에서 설치하기

:::tip 전제 조건

- Git, Node.js 20+, Yarn 1.22.x가 설치되어 있어야 합니다.
- 다음 데이터베이스 중 하나를 설정하고 실행해야 합니다: MySQL 8.0.17+, MariaDB 10.9+, PostgreSQL 10+
- 네트워크 환경에 따라, 더 빠른 다운로드를 위해 미러(mirror)를 사용하는 것을 고려해 볼 수 있습니다 (필수 아님).

```bash
$ yarn config set disable-self-update-check true
$ yarn config set registry https://registry.npmmirror.com/
```

:::

## 1. NocoBase를 로컬에 다운로드하기

NocoBase 버전을 선택하고([버전 비교](../quickstart.md)) 해당 명령을 실행하세요.

## 2. 디렉터리 변경하기

```bash
cd my-nocobase
```

## 3. 의존성 설치

📢 네트워크 환경, 시스템 구성 등의 요인에 따라 이 단계는 10분 이상 소요될 수 있습니다.

```bash
yarn install --frozen-lockfile
```

## 4. 환경 변수 설정

NocoBase에 필요한 환경 변수는 루트 디렉터리의 `.env` 파일에 저장되어 있습니다. 실제 상황에 맞게 환경 변수를 수정하세요.

```bash
TZ=Asia/Shanghai
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning 환경 변수 설명

- `TZ`는 애플리케이션의 시간대를 설정하는 데 사용되며, 기본값은 운영 체제의 시간대입니다.
- `APP_KEY`는 사용자 토큰 등을 생성하는 데 사용되는 애플리케이션의 비밀 키입니다. (`APP_KEY`가 변경되면 기존 토큰은 무효화됩니다.) 임의의 문자열이 될 수 있습니다. 자신만의 비밀 키로 변경하고 외부에 노출되지 않도록 주의하세요.
- `DB_*`는 데이터베이스 관련 설정입니다. 예시의 기본 데이터베이스 서비스를 사용하지 않는 경우, 실제 상황에 맞게 수정하세요.

:::

## 5. NocoBase 설치

```bash
yarn nocobase install --lang=ko-KR
```

## 6. NocoBase 시작

**개발 환경**

```bash
yarn dev
```

**운영 환경 (권장하지 않음)**

운영 환경에서는 소스 코드를 직접 배포하는 것을 권장하지 않습니다. 운영 환경 배포에 대해서는 [운영 환경 배포](../deployment/production.md)를 참고하세요.

```bash
yarn build
yarn start
```

## 7. NocoBase 로그인

브라우저에서 [http://localhost:13000](http://localhost:13000)을 엽니다. 초기 계정은 `admin@nocobase.com`이고 비밀번호는 `admin123`입니다.

:::warning 계정 보안 경고

처음 로그인한 후에는 시스템 보안을 위해 기본 비밀번호를 즉시 변경해 주세요.

:::
