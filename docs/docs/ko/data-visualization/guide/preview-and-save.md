# 미리보기 및 저장

*   **미리보기**: 설정 패널에서 변경한 내용을 페이지 차트에 임시로 렌더링하여 효과를 확인할 수 있습니다.
*   **저장**: 설정 패널에서 변경한 내용을 데이터베이스에 실제로 저장합니다.

## 접근 방법

![clipboard-image-1761479218](https://static-docs.nocobase.com/clipboard-image-1761479218.png)

*   시각적(Basic) 모드에서는 모든 변경 사항이 기본적으로 자동으로 미리보기에 적용됩니다.
*   SQL 모드 및 Custom 모드에서는 변경 후 오른쪽에 있는 **미리보기** 버튼을 클릭하여 미리보기에 적용할 수 있습니다.
*   설정 패널 하단에는 통합된 **미리보기** 버튼이 제공됩니다.

## 미리보기 동작 방식
*   설정 내용을 페이지에 임시로 표시하지만 데이터베이스에는 저장하지 않습니다. 페이지를 새로 고치거나 작업을 취소하면 미리보기 결과는 유지되지 않습니다.
*   내장된 디바운스 기능: 짧은 시간 내에 여러 번 새로 고침을 트리거해도 마지막 요청만 실행하여 잦은 요청을 방지합니다.
*   미리보기를 다시 클릭하면 이전 미리보기 결과가 덮어씌워집니다.

## 오류 메시지
*   쿼리 오류 또는 유효성 검사 실패 시: **데이터 보기** 영역에 오류 메시지가 표시됩니다.
*   차트 설정 오류 (예: Basic 매핑 누락, Custom JS 예외 발생) 시: 차트 영역 또는 콘솔에 오류가 표시되며 페이지는 계속 작동 가능한 상태를 유지합니다.
*   필드 매핑 또는 Custom 코드 작성 전에 **데이터 보기**에서 열 이름과 데이터 유형을 먼저 확인하면 오류를 효과적으로 줄일 수 있습니다.

## 저장 및 취소
*   **저장**: 현재 패널의 변경 사항을 블록 설정에 기록하고 즉시 페이지에 적용합니다.
*   **취소**: 현재 패널에서 저장되지 않은 변경 사항을 취소하고 마지막 저장 상태로 되돌립니다.
*   **저장 범위**:
    *   데이터 쿼리: 빌더의 쿼리 매개변수; SQL 모드에서는 SQL 텍스트도 함께 저장됩니다.
    *   차트 옵션: Basic 유형, 필드 매핑 및 속성; Custom JS 텍스트.
    *   상호작용 이벤트: 이벤트의 JS 텍스트 및 바인딩 로직.
*   저장 후, 블록은 모든 방문자에게 적용됩니다 (페이지 권한 설정에 따라 다름).

## 권장 작업 흐름
*   데이터 쿼리 설정 → 쿼리 실행 → **데이터 보기**에서 열 이름 및 유형 확인 → 핵심 필드 매핑을 위한 차트 옵션 설정 → 미리보기로 유효성 검사 → 저장하여 적용
