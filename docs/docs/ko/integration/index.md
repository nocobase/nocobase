# 통합

## 개요

NocoBase는 포괄적인 통합 기능을 제공하여 외부 시스템, 타사 서비스 및 다양한 **데이터 소스**와 원활하게 연결할 수 있습니다. 유연한 통합 방식을 통해 NocoBase의 기능을 확장하여 다양한 비즈니스 요구 사항을 충족할 수 있습니다.

## 통합 방식

### API 통합

NocoBase는 외부 애플리케이션 및 서비스와 통합할 수 있는 강력한 API 기능을 제공합니다.

![20250301003106](https://static-docs.nocobase.com/20250301003106.png)

- **[API 키](/integration/api-keys/)**: API 키를 사용하여 안전하게 인증하고 NocoBase 리소스에 프로그래밍 방식으로 접근합니다.
- **[API 문서](/integration/api-doc/)**: 엔드포인트를 탐색하고 테스트할 수 있는 내장 API 문서를 제공합니다.

### 단일 로그인 (SSO)

기업 ID 시스템과 통합하여 통합 인증을 구현합니다.

![auth_sso-2025-11-24-08-26-46](https://static-docs.nocobase.com/auth_sso-2025-11-24-08-26-46.png)

- **[SSO 통합](/integration/sso/)**: SAML, OIDC, CAS, LDAP 및 타사 플랫폼 인증을 지원합니다.
- 중앙 집중식 사용자 관리 및 접근 제어
- 시스템 전반에 걸친 원활한 인증 경험

### 워크플로우 통합

NocoBase **워크플로우**를 외부 시스템과 연결합니다.

![auth_sso-2025-11-24-12-02-01](https://static-docs.nocobase.com/auth_sso-2025-11-24-12-02-01.png)

- **[워크플로우 웹훅](/integration/workflow-webhook/)**: 외부 시스템에서 이벤트를 수신하여 **워크플로우**를 트리거합니다.
- **[워크플로우 HTTP 요청](/integration/workflow-http-request/)**: **워크플로우**에서 외부 API로 HTTP 요청을 보냅니다.
- 플랫폼 간 비즈니스 프로세스 자동화

### 외부 데이터 소스

외부 데이터베이스 및 데이터 시스템에 연결합니다.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

- **[외부 데이터베이스](/data-sources/)**: MySQL, PostgreSQL, MariaDB, MSSQL, Oracle 및 KingbaseES 데이터베이스에 직접 연결합니다.
- 외부 데이터베이스 테이블 구조를 인식하고 NocoBase 내에서 외부 데이터에 대해 직접 CRUD 작업을 수행합니다.
- 통합 데이터 관리 인터페이스

### 임베디드 콘텐츠

NocoBase 내에 외부 콘텐츠를 임베드합니다.

![20240408220322](https://static-docs.nocobase.com/20240408220322.png)

- **[Iframe 블록](/integration/block-iframe/)**: 외부 웹 페이지 및 애플리케이션을 임베드합니다.
- **JS 블록**: 사용자 지정 JavaScript 코드를 실행하여 고급 통합을 구현합니다.

## 일반적인 통합 시나리오

### 기업 시스템 통합

- NocoBase를 ERP, CRM 또는 기타 기업 시스템과 연결
- 양방향 데이터 동기화
- 시스템 간 **워크플로우** 자동화

### 타사 서비스 통합

- 결제 게이트웨이의 결제 상태를 조회하고, 메시징 서비스 또는 클라우드 플랫폼을 통합합니다.
- 외부 API를 활용하여 기능을 확장
- 웹훅 및 HTTP 요청을 사용하여 사용자 지정 통합 구축

### 데이터 통합

- 여러 **데이터 소스**에 연결
- 서로 다른 시스템의 데이터 집계
- 통합 대시보드 및 보고서 생성

## 보안 고려 사항

NocoBase를 외부 시스템과 통합할 때 다음 보안 모범 사례를 고려하십시오.

1.  **HTTPS 사용**: 데이터 전송 시 항상 암호화된 연결을 사용하십시오.
2.  **API 키 보호**: API 키를 안전하게 저장하고 정기적으로 교체하십시오.
3.  **최소 권한 원칙**: 통합에 필요한 최소한의 권한만 부여하십시오.
4.  **감사 로깅**: 통합 활동을 모니터링하고 기록하십시오.
5.  **데이터 유효성 검사**: 외부 소스에서 들어오는 모든 데이터의 유효성을 검사하십시오.
