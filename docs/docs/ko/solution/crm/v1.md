# 구버전 CRM (v1)

> **미리보기 전용**: 이 버전은 기능적으로는 비교적 완성되어 있으나, 데이터 테이블의 전체적인 설계가 비합리적이고 다국어 로컬라이제이션 지원이 부족하며, CRM 컬렉션과 프로젝트 관리 컬렉션 간의 결합(Coupling)이 존재합니다. 이러한 문제로 인해 CRM 2.0이 새롭게 설계되었습니다. 현재는 데이터 테이블 구조가 더 합리적이고 확장 및 개조가 용이하며 다국어 로컬라이제이션이 완료된 [CRM 2.0 버전](./index.md)을 사용하는 것을 권장합니다.

## 버전 특징

구버전 CRM은 다음과 같은 기능 모듈을 포함하고 있습니다:

- **고객 관리**: 고객 정보, 연락처, 고객 등급
- **영업 파이프라인**: 리드 관리, 기회 추적, 영업 깔대기(Sales Funnel)
- **주문 관리**: 견적서, 주문, 계약
- **프로젝트 관리** (v2 버전에는 아직 포함되지 않음)
- **커뮤니케이션 및 협업**: 이메일 소통, 댓글, 승인 워크플로우
- **데이터 분석**: 대시보드, 보고서, 데이터 시각화

## 설치 방법

> 현재 버전은 백업 및 복구 방식으로 배포됩니다.

시작하기 전에 다음 사항을 확인해 주세요:

- 기초적인 NocoBase 실행 환경이 준비되어 있어야 합니다. 시스템 설치에 대해서는 상세한 [공식 설치 문서](https://docs-cn.nocobase.com/welcome/getting-started/installation)를 참고해 주세요.
- NocoBase 버전 **v2.1.0-beta.2 이상**
- CRM v1 관련 파일을 다운로드해야 합니다:
  - **백업 파일**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — 방법 1 적용 시
  - **SQL 파일**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — 방법 2 적용 시

**중요 설명**:
- 이 솔루션은 **PostgreSQL 16** 데이터베이스를 기반으로 제작되었습니다. 환경에서 PostgreSQL 16을 사용 중인지 확인해 주세요.
- **DB_UNDERSCORED는 true여야 안 됩니다**: `docker-compose.yml` 파일을 확인하여 `DB_UNDERSCORED` 환경 변수가 `true`로 설정되어 있지 않은지 확인해 주세요. 그렇지 않으면 백업 파일과 충돌하여 복구에 실패할 수 있습니다.

## 방법 1: 백업 관리자를 사용하여 복구 (전문가/기업용 버전 사용자 권장)

이 방식은 NocoBase에 내장된 "[백업 관리자](https://docs-cn.nocobase.com/handbook/backups)"(전문가/기업용 버전) 플러그인을 통해 클릭 한 번으로 복구하는 가장 간단한 방법입니다.

### 작업 단계

**1단계: "백업 관리자" 플러그인 활성화**

1. NocoBase 시스템에 로그인합니다.
2. **`플러그인 관리`**로 이동합니다.
3. **`백업 관리자`** 플러그인을 찾아 활성화합니다.

**2단계: 로컬 백업 파일에서 복구**

1. 플러그인을 활성화한 후 페이지를 새로고침합니다.
2. 왼쪽 메뉴에서 **`시스템 관리`** -> **`백업 관리자`**로 이동합니다.
3. 오른쪽 상단의 **`로컬 백업에서 복구`** 버튼을 클릭합니다.
4. 다운로드한 백업 파일을 업로드 영역으로 드래그 앤 드롭합니다.
5. **`제출`**을 클릭하고 시스템 복구가 완료될 때까지 기다립니다.

### 주의 사항

* **전문가/기업용 버전 전용**: "백업 관리자"는 기업용 플러그인으로, 전문가/기업용 버전 사용자만 사용할 수 있습니다.
* **상용 플러그인 일치**: 솔루션에 필요한 상용 플러그인을 보유하고 활성화했는지 확인해 주세요.

## 방법 2: SQL 파일 직접 가져오기 (일반용)

이 방식은 데이터베이스를 직접 조작하여 데이터를 복구하는 방식으로, 모든 NocoBase 사용자에게 적용 가능합니다.

### 작업 단계

**1단계: 깨끗한 데이터베이스 준비**

가져올 데이터를 위해 비어 있는 새로운 데이터베이스를 준비합니다.

**2단계: .sql 파일을 데이터베이스로 가져오기**

* **명령줄을 통한 방법 (Docker 예시)**:

  ```bash
  # sql 파일을 컨테이너 내부로 복사
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # 컨테이너에 접속하여 가져오기 명령 실행
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **데이터베이스 클라이언트를 통한 방법**: DBeaver, Navicat, pgAdmin 등의 도구를 사용하여 데이터베이스에 연결하고 SQL 파일을 실행합니다.

**3단계: 데이터베이스 연결 및 애플리케이션 시작**

NocoBase 시작 매개변수를 설정하여 데이터가 생성된 데이터베이스를 가리키도록 한 후 서비스를 시작합니다.
