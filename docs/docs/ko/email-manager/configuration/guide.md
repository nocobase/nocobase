# 설정 과정

## 개요
이메일 플러그인을 활성화한 후, 관리자는 먼저 관련 설정을 완료해야 일반 사용자가 NocoBase에 이메일 계정을 연결할 수 있습니다. (현재 Outlook 및 Gmail 계정의 인증 로그인만 지원하며, Microsoft 및 Google 계정의 직접 연결은 아직 지원되지 않습니다.)

설정의 핵심은 이메일 서비스 제공업체 API 호출에 대한 인증 설정입니다. 관리자는 플러그인 기능이 정상적으로 작동하도록 다음 단계를 완료해야 합니다.

1.  **서비스 제공업체에서 인증 정보 얻기**
    -   이메일 서비스 제공업체의 개발자 콘솔(예: Google Cloud Console 또는 Microsoft Azure Portal)에 로그인합니다.
    -   새 애플리케이션 또는 프로젝트를 생성하고 Gmail 또는 Outlook 이메일 API 서비스를 활성화합니다.
    -   해당 클라이언트 ID(Client ID)와 클라이언트 시크릿(Client Secret)을 얻습니다.
    -   리디렉션 URI를 NocoBase 플러그인의 콜백 주소와 일치하도록 설정합니다.

2.  **이메일 서비스 제공업체 설정**
    -   이메일 플러그인의 설정 페이지로 이동합니다.
    -   필요한 API 인증 정보(클라이언트 ID(Client ID), 클라이언트 시크릿(Client Secret) 등)를 제공하여 이메일 서비스 제공업체와의 인증 연동이 정상적으로 이루어지도록 합니다.

3.  **인증 로그인**
    -   사용자는 OAuth 프로토콜을 통해 이메일 계정에 로그인합니다.
    -   플러그인은 이후 API 호출 및 이메일 작업을 위해 사용자의 인증 토큰을 자동으로 생성하고 저장합니다.

4.  **이메일 계정 연결**
    -   사용자가 성공적으로 인증하면, 해당 이메일 계정은 NocoBase에 연결됩니다.
    -   플러그인은 사용자의 이메일 데이터를 동기화하고 이메일 관리, 송수신 기능을 제공합니다.

5.  **이메일 기능 사용**
    -   사용자는 플랫폼 내에서 직접 이메일을 확인하고, 관리하며, 보낼 수 있습니다.
    -   모든 작업은 이메일 서비스 제공업체의 API 호출을 통해 완료되어 실시간 동기화 및 효율적인 전송을 보장합니다.

위 과정을 통해 NocoBase의 이메일 플러그인은 사용자에게 효율적이고 안전한 이메일 관리 서비스를 제공합니다. 설정 과정에서 문제가 발생하면 관련 문서를 참조하거나 기술 지원팀에 문의하여 도움을 받으십시오.

## 플러그인 설정

### 이메일 플러그인 활성화

1.  플러그인 관리 페이지로 이동합니다.
2.  "Email manager" 플러그인을 찾아 활성화합니다.

### 이메일 서비스 제공업체 설정

이메일 플러그인을 활성화한 후, 이메일 서비스 제공업체 설정을 진행할 수 있습니다. 현재 Google과 Microsoft 두 가지 이메일 서비스를 지원합니다. 상단 "설정" -> "이메일 설정"을 클릭하여 설정 페이지로 이동합니다.

![](https://static-docs.nocobase.com/mail-1733818617514.png)
