## 2. Структура коллекции лидов

### 2.1 О коллекции лидов

В работе с лидами поле «Статус» играет ключевую роль: оно отражает текущее состояние (не квалифицирован, новый лид, в работе, в подогреве, в сделке, завершено) и при этом управляет отображением и логикой формы. Структура полей коллекции лидов:


| Имя поля       | Отображаемое название | Интерфейс поля   | Описание                                                              |
| -------------- | --------------------- | ---------------- | ------------------------------------------------------------------------ |
| id             | **ID**                | Целое число      | Первичный ключ                                                            |
| account_id     | **ID компании**       | Целое число      | Внешний ключ к коллекции компаний                                           |
| contact_id     | **ID контакта**       | Целое число      | Внешний ключ к коллекции контактов                                          |
| opportunity_id | **ID сделки**         | Целое число      | Внешний ключ к коллекции сделок                                             |
| name           | **Имя лида**          | Однострочный текст | Имя потенциального клиента                                                |
| company        | **Название компании** | Однострочный текст | Название компании потенциального клиента                                  |
| email          | **Email**             | Электронная почта | Email-адрес потенциального клиента                                        |
| phone          | **Телефон**           | Телефон          | Контактный телефон                                                        |
| status         | **Статус**            | Одиночный выбор  | Текущий статус лида (не квалифицирован, новый лид, в работе, в подогреве, в сделке, завершено) |
| Account        | **Компания**          | Многие-к-одному  | Связь с коллекцией компаний                                               |
| Contact        | **Контакт**           | Многие-к-одному  | Связь с коллекцией контактов                                              |
| Opportunity    | **Сделка**            | Многие-к-одному  | Связь с коллекцией сделок                                                 |

## 4. Настройка кнопок действий

### 4.1 Общее описание кнопок

Для нужного функционала создадим всего 10 кнопок. Каждая ведёт себя по-разному (скрыта, активна или заблокирована) в зависимости от статуса записи, направляя пользователя по правильному рабочему процессу.
![20250311083825](https://static-docs.nocobase.com/20250311083825.png)

### 4.2 Подробная конфигурация кнопок


| Кнопка                              | Стиль                                  | Действие                                                          | Правило связывания                                                                                |
| ----------------------------------- | -------------------------------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| Кнопка редактирования               | Операция редактирования                | —                                                                | При статусе «Завершено» автоматически блокируется, чтобы не было лишних правок.                    |
| «Не квалифицирован» (активная)      | «Не квалифицирован >»                   | Обновляет статус записи на «Не квалифицирован».                    | Показывается по умолчанию; при «Завершено» — заблокирована.                                       |
| «Новый лид» (неактивная)            | Операция обновления, «Новый лид >»     | Статус становится «Новый лид», после — сообщение «Новый лид».      | Если статус не «Не квалифицирован», кнопка скрывается (при «Новый лид» и далее — должна быть в активном виде). |
| «Новый лид» (активная)              | Операция обновления, «Новый лид >»     | Обновляет статус на «Новый лид».                                   | При «Не квалифицирован» — скрыта; при «Завершено» — заблокирована.                                |
| «В работе» (неактивная)             | Операция обновления, «В работе >»      | Статус становится «В работе», выводится сообщение «В работе».        | Если статус не «Не квалифицирован» и не «Новый лид» — скрыта.                                     |
| «В работе» (активная)               | Операция обновления, «В работе >»      | Обновляет статус на «В работе».                                    | При «Не квалифицирован» или «Новый лид» — скрыта; при «Завершено» — заблокирована.                |
| «В подогреве» (неактивная)          | Операция обновления, «В подогреве >»   | Статус становится «В подогреве», сообщение «В подогреве».          | Если статус не «Не квалифицирован», «Новый лид» или «В работе» — скрыта.                          |
| «В подогреве» (активная)            | Операция обновления, «В подогреве >»   | Обновляет статус на «В подогреве».                                 | При «Не квалифицирован», «Новый лид» или «В работе» — скрыта; при «Завершено» — заблокирована.    |
| «Конвертировать»                    | Операция редактирования, «Конвертировать», иконка «√» | Открывает форму конвертации; при отправке статус становится «Завершено». | При «Завершено» — скрыта во избежание повторной конвертации.                                      |
| «Конвертировано» (активная)         | Операция просмотра, «Конвертировано», иконка «√»    | Только для просмотра информации после конвертации, без редактирования.   | Показывается только при «Завершено»; иначе скрыта.                                                |

## 6. Правила Markdown и синтаксис Handlebars

### 6.1 Динамический текст

С помощью Handlebars выводим разные подсказки в зависимости от статуса. Примеры:

Когда статус «Не квалифицирован»:

```markdown
{{#if (eq $nRecord.status "Не квалифицирован")}}
**Отслеживайте информацию по неквалифицированным лидам.**  
Если лид не интересуется продуктом или ушёл из своей компании, он может быть неквалифицированным.  
- Фиксируйте уроки на будущее  
- Сохраняйте детали взаимодействия и контакты  
{{/if}}
```

Когда статус «Новый лид»:

```markdown
{{#if (eq $nRecord.status "Новый лид")}}
**Соберите больше информации об этом лиде.**  
- Узнайте потребности и интересы потенциального клиента
- Соберите базовые контактные данные и информацию о компании
- Определите приоритет и способ дальнейшей работы
{{/if}}
```

Когда статус «В работе»:

```markdown
{{#if (eq $nRecord.status "В работе")}}
**Свяжитесь с лидом и предварительно оцените потребности.**  
- Установите контакт по телефону или email
- Узнайте, какие проблемы и вызовы у клиента
- Оцените соответствие потребностей продуктам/услугам компании
{{/if}}
```

Когда статус «В подогреве»:

```markdown
{{#if (eq $nRecord.status "В подогреве")}}
**Глубже изучите потребности и развивайте лид.**  
- Предоставьте материалы по продукту или варианты решений
- Отвечайте на вопросы и снимайте сомнения
- Оцените вероятность конвертации
{{/if}}
```

Когда статус «Завершено»:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Лид успешно сконвертирован в клиента.**  
- Подтвердите, что созданы записи компании и контакта
- Создайте сделку и план дальнейшей работы
- Передайте материалы и историю общения ответственному менеджеру
{{/if}}
```

## 7. Отображение связанных объектов и переходов после конвертации

### 7.1 О связанных объектах

После конвертации мы хотим показывать связанные объекты (компанию, контакт, сделку) и переход на их детальные страницы.
В это время найдите любую страницу деталей, например компанию, и скопируйте ссылку.
![20250311085502](https://static-docs.nocobase.com/20250311085502.png)
Внимание: в других всплывающих окнах или страницах последний фрагмент ссылки (число после `filterbytk`) — это ID текущего объекта. Например:

```text
{Base URL}/admin/w3yyu23uro0/popups/ki0wcnfruj6/filterbytk/{id}
```

### 7.2 Генерация связанных ссылок через Handlebars

Компания:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Компания:**
[{{$nRecord.account.name}}](w3yyu23uro0/popups/ki0wcnfruj6/filterbytk/{{$nRecord.account_id}})
{{/if}}
```

Контакт:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Контакт:**
[{{$nRecord.contact.name}}](1oqybfwrocb/popups/8bbsqy5bbpl/filterbytk/{{$nRecord.contact_id}})
{{/if}}
```

Сделка:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Сделка:**
[{{$nRecord.opportunity.name}}](si0io9rt6q6/popups/yyx8uflsowr/filterbytk/{{$nRecord.opportunity_id}})
{{/if}}
```

## 8. Скрытие связанных объектов с сохранением значения

Чтобы после конвертации связанная информация корректно отображалась, у полей «Компания», «Контакт» и «Сделка» нужно установить состояние «Скрыть (сохранить значение)». Тогда поля не будут показываться в форме, но их значения сохранятся и передадутся.
![](https://static-docs.nocobase.com/20250226093906.png)

## 9. Защита от изменения статуса после конвертации

Чтобы случайно не изменить статус после конвертации, ко всем кнопкам добавляем условие: при статусе «Завершено» все кнопки блокируются.
![](https://static-docs.nocobase.com/20250226094302.png)
