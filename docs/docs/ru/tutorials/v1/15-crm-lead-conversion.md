# Реализация конвертации лидов в CRM

## 1. Введение

В этом руководстве мы шаг за шагом покажем, как в NocoBase реализовать функцию конвертации сделок в CRM. Расскажем, как создать необходимые коллекции, настроить страницы управления данными, спроектировать процесс конвертации и связи между объектами, чтобы Вы смогли успешно построить весь бизнес-процесс.

[Решение NocoBase CRM теперь доступно — приглашаем попробовать!](https://www.nocobase.com/ru/blog/crm-solution)

## 2. Подготовка: создание необходимых коллекции

Перед началом подготовим 4 коллекции и настроим связи между ними.

### 2.1 Коллекция лидов

Хранит информацию о потенциальных клиентах. Поля:


| Имя поля       | Отображаемое название | Интерфейс поля   | Описание                                                              |
| -------------- | --------------------- | ---------------- | ------------------------------------------------------------------------ |
| id             | **ID**                | Целое число      | Первичный ключ                                                            |
| account_id     | **ID компании**       | Целое число      | Внешний ключ к коллекции компаний                                           |
| contact_id     | **ID контакта**       | Целое число      | Внешний ключ к коллекции контактов                                          |
| opportunity_id | **ID сделки**         | Целое число      | Внешний ключ к коллекции сделок                                             |
| name           | **Имя лида**          | Однострочный текст | Имя потенциального клиента                                                |
| company        | **Название компании** | Однострочный текст | Название компании потенциального клиента                                  |
| email          | **Email**             | Электронная почта | Email-адрес потенциального клиента                                        |
| phone          | **Телефон**           | Телефон          | Контактный телефон                                                        |
| status         | **Статус**            | Одиночный выбор  | Текущий статус лида (не квалифицирован, новый лид, в работе, в подогреве, в сделке, завершено) |
| Account        | **Компания**          | Многие-к-одному  | Связь с коллекцией компаний                                               |
| Contact        | **Контакт**           | Многие-к-одному  | Связь с коллекцией контактов                                              |
| Opportunity    | **Сделка**            | Многие-к-одному  | Связь с коллекцией сделок                                                 |

### 2.2 Коллекция компаний

Хранит подробную информацию о компаниях. Поля:


| Имя поля | Отображаемое название | Интерфейс поля   | Описание                  |
| ---------- | --------------------- | ---------------- | ---------------------------- |
| name       | **Название**          | Однострочный текст | Название компании или организации |
| industry   | **Отрасль**           | Одиночный выбор  | Отрасль компании             |
| phone      | **Телефон**           | Телефон          | Контактный телефон компании  |
| website    | **Сайт**              | URL-адрес        | URL официального сайта компании |

### 2.3 Коллекция контактов

Хранит информацию о контактах:


| Имя поля | Отображаемое название | Интерфейс поля   | Описание       |
| ---------- | --------------------- | ---------------- | ----------------- |
| name       | **Имя**               | Однострочный текст | Имя контакта      |
| email      | **Email**             | Электронная почта | Email контакта    |
| phone      | **Телефон**           | Телефон          | Телефон контакта  |

### 2.4 Коллекция сделок

Хранит информацию о сделках. Поля:


| Имя поля | Отображаемое название | Интерфейс поля   | Описание                                                              |
| ---------- | --------------------- | ---------------- | ------------------------------------------------------------------------ |
| name       | **Название**          | Однострочный текст | Название сделки                                                           |
| stage      | **Стадия**            | Одиночный выбор  | Стадия сделки (квалификация, потребности, КП, переговоры, закрытие, успех, провал) |
| amount     | **Сумма**             | Число            | Сумма сделки                                                              |
| close_date | **Дата закрытия**     | Дата и время     | Ожидаемая дата закрытия сделки                                                   |

## 5. Реализация конвертации сделки

В этом разделе мы покажем, как преобразовать лид в записи компании, контакта и сделки и предотвратить повторный запуск конвертации.

### 5.1 Создание операции редактирования «Конвертировать»

В детальной странице лида создайте операцию редактирования с названием «Конвертировать». В открывающемся всплывающем окне настройте следующее:

#### 5.1.1 Отображение базовой информации лида

Покажите данные лида в режиме «только чтение», чтобы пользователь случайно не изменил исходные данные.

#### 5.1.2 Поля связей

Покажите три поля связей и для каждого включите функцию «Быстрое создание», чтобы при отсутствии нужной записи её можно было создать прямо здесь.

![Связные поля](https://static-docs.nocobase.com/20250224234155.png)

#### 5.1.3 Настройка значений по умолчанию для быстрого создания

В настройках всплывающего окна «Быстрое создание» настройте значения по умолчанию для каждого связного поля — так данные лида будут автоматически переноситься в целевые коллекции. Маппинг:

- Лид/Имя лида → Компания/Название
- Лид/Email → Компания/Email
- Лид/Телефон → Компания/Телефон
- Лид/Имя лида → Контакт/Имя
- Лид/Email → Контакт/Email
- Лид/Телефон → Контакт/Телефон
- Лид/Имя лида → Сделка/Название
- Лид/Статус → Сделка/Стадия

## 6. Настройка блоков связанных записей на детальных страницах

Чтобы пользователи могли видеть связанные данные на детальных страницах каждой коллекции, нужно настроить соответствующие блоки списка или блоки деталей.

### 6.1 Детальная страница коллекции компаний

На детальной странице компании (например, во всплывающем окне редактирования/деталей контактов) добавьте следующие блоки списка:

- блок списка контактов
- блок списка сделок
- блок списка лидов

Пример:

![Детали компании](https://static-docs.nocobase.com/20250225085418.png)

### 6.2 Добавление условий фильтрации

Для каждого блока списка добавьте условия фильтрации, чтобы отображались только записи, связанные с текущим ID компании.

Скриншоты:

![Фильтр 1](https://static-docs.nocobase.com/20250225085513.png)
![Фильтр 2](https://static-docs.nocobase.com/20250225085638.png)

### 6.3 Детальные страницы контактов и сделок

Во всплывающем окне деталей контакта добавьте:

- блок списка сделок
- детали компании
- блок списка лидов (с фильтром по ID)
