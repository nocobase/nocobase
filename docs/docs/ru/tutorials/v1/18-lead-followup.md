## 2. Структура коллекции лидов

### 2.1 О коллекции лидов

В работе с лидами поле «Статус» играет ключевую роль: оно отражает текущее состояние лида (не квалифицирован, новый лид, в работе, в подогреве, в сделке, завершено) и при этом управляет отображением и логикой формы. Ниже в табличном блоке представлена структура полей коллекции лидов с подробным описанием:


| Имя поля       | Отображаемое название | Интерфейс поля   | Описание                                                              |
| -------------- | --------------------- | ---------------- | ------------------------------------------------------------------------ |
| id             | **ID**                | Целое число      | Первичный ключ                                                            |
| account_id     | **ID компании**       | Целое число      | Внешний ключ к коллекции компаний                                           |
| contact_id     | **ID контакта**       | Целое число      | Внешний ключ к коллекции контактов                                          |
| opportunity_id | **ID сделки**         | Целое число      | Внешний ключ к коллекции сделок                                             |
| name           | **Имя лида**          | Однострочный текст | Имя потенциального клиента                                                |
| company        | **Название компании** | Однострочный текст | Название компании потенциального клиента                                  |
| email          | **Электронная почта** | Электронная почта | Адрес электронной почты потенциального клиента                            |
| phone          | **Телефон**           | Телефон          | Контактный телефон                                                        |
| status         | **Статус**            | Одиночный выбор  | Текущий статус лида (не квалифицирован, новый лид, в работе, в подогреве, в сделке, завершено) |
| Account        | **Компания**          | Многие-к-одному  | Связь с коллекцией компаний                                               |
| Contact        | **Контакт**           | Многие-к-одному  | Связь с коллекцией контактов                                              |
| Opportunity    | **Сделка**            | Многие-к-одному  | Связь с коллекцией сделок                                                 |

## 4. Настройка кнопок действий

### 4.1 Общее описание кнопок

Чтобы покрыть разные сценарии работы, создадим всего 11 кнопок. Каждая отображается по-разному (скрыта, активна или заблокирована) в зависимости от статуса записи и направляет пользователя по нужному бизнес-процессу.
![20250226173632](https://static-docs.nocobase.com/20250226173632.png)

### 4.2 Подробная настройка функциональных кнопок

#### 4.2.1 Кнопка редактирования

- Правило связывания: когда статус записи равен «Завершено», кнопка автоматически блокируется, чтобы предотвратить лишние правки.

#### 4.2.2 Кнопка «Не квалифицирован» 1 (неактивная)

- Стиль: заголовок «Не квалифицирован >».
- Действие: при клике выполняется обновление, статус записи становится «Не квалифицирован», после успешного обновления — возврат на предыдущий уровень и вывод сообщения «Не квалифицирован».
- Правило связывания: показывается только когда статус пустой; как только у статуса появляется значение, кнопка автоматически скрывается.

#### 4.2.3 Кнопка «Не квалифицирован» 2 (активная)

- Стиль: тоже «Не квалифицирован >».
- Действие: обновляет статус записи на «Не квалифицирован».
- Правило связывания: при пустом статусе кнопка скрыта; при «Завершено» — заблокирована.

#### 4.2.4 Кнопка «Новый лид» 1 (неактивная)

- Стиль: заголовок «Новый лид >».
- Действие: при клике обновляет запись, выставляя статус в «Новый лид», после чего показывается сообщение «Новый лид».
- Правило связывания: если статус уже находится в одном из состояний «Новый лид», «В работе», «В подогреве» или «Завершено», кнопка скрывается.

#### 4.2.5 Кнопка «Новый лид» 2 (активная)

- Стиль: заголовок «Новый лид >».
- Действие: тоже обновляет статус на «Новый лид».
- Правило связывания: при статусе «Не квалифицирован» или пустом — скрыта; при «Завершено» — заблокирована.

#### 4.2.6 Кнопка «В работе» (неактивная)

- Стиль: заголовок «В работе >».
- Действие: при клике статус обновляется на «В работе», выводится сообщение «В работе».
- Правило связывания: если статус уже «В работе», «В подогреве» или «Завершено», кнопка скрыта.

#### 4.2.7 Кнопка «В работе» (активная)

- Стиль: заголовок «В работе >».
- Действие: обновляет статус на «В работе».
- Правило связывания: при статусе «Не квалифицирован», «Новый лид» или пустом — скрыта; при «Завершено» — заблокирована.

#### 4.2.8 Кнопка «В подогреве» (неактивная)

- Стиль: заголовок «В подогреве >».
- Действие: при клике статус обновляется на «В подогреве», выводится сообщение «В подогреве».
- Правило связывания: если статус уже «В подогреве» или «Завершено», кнопка скрыта.

#### 4.2.9 Кнопка «В подогреве» (активная)

- Стиль: заголовок «В подогреве >».
- Действие: обновляет статус на «В подогреве».
- Правило связывания: при статусе «Не квалифицирован», «Новый лид», «В работе» или пустом — скрыта; при «Завершено» — заблокирована.

#### 4.2.10 Кнопка «Конвертировать»

- Стиль: заголовок «Конвертировать», открывает модальное окно.
- Действие: основная функция — выполнить операцию переноса записи. После обновления появляется интерфейс с выдвижной панелью, вкладками и формой для удобной работы.
- Правило связывания: когда статус записи равен «Завершено», кнопка скрыта во избежание повторного переноса.
  ![](https://static-docs.nocobase.com/20250226094223.png)

#### 4.2.11 Кнопка «Конвертировано» (активная)

- Стиль: заголовок «Конвертировано», тоже открывает модальное окно.
- Действие: только для просмотра информации после конвертации, без возможности редактировать.
- Правило связывания: показывается только при статусе «Завершено»; при других статусах («Не квалифицирован», «Новый лид», «В работе», «В подогреве» или пустом) — скрыта.
  ![](https://static-docs.nocobase.com/20250226094203.png)

### 4.3 Итоги настройки кнопок

- Для каждой функции есть отдельные стили кнопок в неактивном и активном состоянии.
- Через правила связывания динамически управляем видимостью (скрытие/блокировка) кнопок по статусу записи, направляя продавца по правильному рабочему процессу.

## 6. Правила Markdown на странице и синтаксис Handlebars

### 6.1 Динамическое отображение текста

На странице с помощью синтаксиса Handlebars выводим разные подсказки в зависимости от статуса. Примеры кода для каждого статуса:

Когда статус «Не квалифицирован»:

```markdown
{{#if (eq $nRecord.status "Не квалифицирован")}}
**Отслеживайте информацию по неквалифицированным лидам.**  
Если лид не интересуется продуктом или ушёл из своей компании, он может быть неквалифицированным.  
- Фиксируйте уроки на будущее  
- Сохраняйте детали взаимодействия и контакты  
{{/if}}
```

Когда статус «Новый лид»:

```markdown
{{#if (eq $nRecord.status "Новый лид")}}
**Определите, какие продукты или услуги нужны для этой сделки.**  
- Соберите примеры из практики, рекомендации или конкурентный анализ  
- Подтвердите ключевых заинтересованных лиц  
- Определите доступные ресурсы  
{{/if}}
```

Когда статус «В работе»:

```markdown
{{#if (eq $nRecord.status "В работе")}}
**Передайте решение заинтересованным лицам.**  
- Донесите ценность решения  
- Согласуйте сроки и бюджет  
- Совместно с клиентом спланируйте, когда и как заключить сделку  
{{/if}}
```

Когда статус «В подогреве»:

```markdown
{{#if (eq $nRecord.status "В подогреве")}}
**Согласуйте план внедрения проекта.**  
- При необходимости заключите договорённости  
- Соблюдайте внутренние правила скидок  
- Получите подписанный контракт  
{{/if}}
```

Когда статус «Завершено»:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Подтвердите план реализации проекта и финальные шаги.**  
- Убедитесь, что все оставшиеся договорённости и подписи на месте  
- Соблюдайте внутреннюю политику скидок  
- Убедитесь, что контракт подписан и реализация идёт по плану  
{{/if}}
```

## 7. Отображение связанных объектов и ссылок для перехода после конвертации

### 7.1 Описание связанных объектов

После конвертации мы хотим отображать связанные объекты (компанию, контакт, сделку) и ссылки для перехода к их детальным страницам. Внимание: в других всплывающих окнах или страницах последняя часть формата ссылки (число после `filterbytk`) — это ID текущего объекта. Например:

```text
http://localhost:13000/apps/tsting/admin/w3yyu23uro0/popups/ki0wcnfruj6/filterbytk/1
```

### 7.2 Генерация связанных ссылок с помощью Handlebars

Для компании:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Компания:**
[{{$nRecord.account.name}}](http://localhost:13000/apps/tsting/admin/w3yyu23uro0/popups/ki0wcnfruj6/filterbytk/{{$nRecord.account_id}})
{{/if}}
```

Для контакта:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Контакт:**
[{{$nRecord.contact.name}}](http://localhost:13000/apps/tsting/admin/1oqybfwrocb/popups/8bbsqy5bbpl/filterbytk/{{$nRecord.contact_id}})
{{/if}}
```

Для сделки:

```markdown
{{#if (eq $nRecord.status "Завершено")}}
**Сделка:**
[{{$nRecord.opportunity.name}}](http://localhost:13000/apps/tsting/admin/si0io9rt6q6/popups/yyx8uflsowr/filterbytk/{{$nRecord.opportunity_id}})
{{/if}}
```

## 8. Скрытие связанных объектов с сохранением значений

Чтобы после конвертации связанные данные корректно отображались, у полей «Компания», «Контакт» и «Сделка» нужно установить состояние «Скрыть (сохранить значение)». Тогда поля не будут отображаться в форме, но их значения будут записаны и переданы.
![](https://static-docs.nocobase.com/20250226093906.png)

## 9. Защита от изменения статуса после конвертации

Чтобы случайно не поменять статус после конвертации, ко всем кнопкам добавляем условие: при статусе «Завершено» все кнопки блокируются.
![](https://static-docs.nocobase.com/20250226094302.png)
