# Глава 4. Формы и детали — ввод и просмотр в одном месте

В прошлой главе мы собрали список заявок и через простую форму внесли тестовые данные. Здесь **усовершенствуем работу с формой** — оптимизируем раскладку полей в [блоке формы](/interface-builder/blocks/data-blocks/form), добавим [блок деталей](/interface-builder/blocks/data-blocks/details), настроим [правила связывания](../../interface-builder/linkage-rule.md) и подключим [историю записей](https://docs.nocobase.com/ru/record-history/) для отслеживания каждого редактирования заявки.

:::tip
Раздел 4.4 «[История записей](https://docs.nocobase.com/ru/record-history/)» доступен в [Pro-версии](https://nocobase.ru/commercial). Пропуск этого раздела не повлияет на остальные главы.
:::

## 4.1 Дорабатываем форму создания заявки

В прошлой главе мы создали быструю работающую форму, теперь её усовершенствуем — поправим порядок полей, зададим значения по умолчанию, оптимизируем раскладку. Если Вы пропустили блок быстрой формы, ничего страшного — здесь начнём с нуля.

### Добавляем кнопку «Добавить новый»

1. Убедитесь, что включён UI Редактор (переключатель в правом верхнем углу).
2. На странице «Список заявок» нажмите **«[Действия](/interface-builder/actions)»** над блоком таблицы.
3. Отметьте **«Добавить новый»**.
4. Над таблицей появится кнопка «Добавить новый», открывающая [всплывающее окно](../../interface-builder/actions/types/pop-up.md).

![04-forms-and-details-2026-03-13-09-43-54](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-09-43-54.png)

### Настраиваем форму во всплывающем окне

1. Нажмите кнопку «Добавить новый», откроется окно.
2. Во всплывающем окне нажмите **«[Добавить блок](/interface-builder/blocks)» → Блок данных → Форма (Добавить новую)**.
3. Выберите **«Текущая [коллекция](/data-sources/data-modeling/collection)»**. Окно уже привязано к нужному контексту коллекции — указывать вручную не нужно.

   ![04-forms-and-details-2026-03-13-09-44-50](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-09-44-50.png)
4. В форме нажмите **«[Поля](/data-sources/data-modeling/collection-fields)»** и отметьте поля:

| Поле | Что важно |
|------|---------|
| Заголовок | Обязательное (наследуется глобально) |
| Описание | Форматированный текст |
| Статус | Выпадающий список (значение по умолчанию настроим через правила связывания) |
| Приоритет | Выпадающий список |
| Категория | Связное поле, отображается как выпадающий список |
| Заявитель | Связное поле (значение по умолчанию через правила связывания) |
| Обработчик | Связное поле |

![04-forms-and-details-2026-03-13-12-44-49](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-12-44-49.png)

Заметьте, что у поля «Заголовок» автоматически появляется звёздочка `*` — потому что в Главе 2 при создании поля мы установили его обязательным, и форма наследует это правило с уровня коллекции. Отдельно настраивать не нужно.

![04-forms-and-details-2026-03-13-12-46-34](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-12-46-34.png)

> **Лайфхак**: если поле на уровне коллекции не помечено обязательным, но в этой форме хочется сделать его обязательным, можно настроить в параметрах поля.
> 
![04-forms-and-details-2026-03-13-12-47-26](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-12-47-26.png)

### Добавляем кнопку «Отправить»

1. Под блоком формы нажмите **«Действия»**.
2. Отметьте **«Отправить»**.

![04-forms-and-details-2026-03-13-16-30-06](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-16-30-06.png)

3. После заполнения пользователь нажимает «Отправить» — создаётся новая заявка.

![04-forms-and-details-2026-03-13-16-32-19](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-16-32-19.gif)

## 4.2 Правила связывания: значения по умолчанию и связки полей

Для одних полей нужно автоматическое заполнение (например, «Статус» по умолчанию — «На рассмотрении»), для других — динамическое поведение (например, обязательное описание для срочных заявок). Сейчас функция значений по умолчанию в 2.0 ещё развивается, поэтому в этих учебных материалах используем **правила связывания** для значений по умолчанию и связок.

1. В правом верхнем углу блока формы нажмите **настройки блока** (иконка трёх линий).
2. Найдите **«Правила связывания»** — справа откроется панель настройки.

![04-forms-and-details-2026-03-13-16-43-35](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-16-43-35.png)

### Значения по умолчанию

Сначала зададим значения по умолчанию для «Статуса» и «Заявителя»:

1. Нажмите **«Добавить правило связывания»**.
2. **Условие не задаём** (оставляем пустым) — правило связывания без условий выполняется при загрузке формы.

![04-forms-and-details-2026-03-13-16-47-34](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-16-47-34.png)

3. Настройте действия:
   - Поле «Статус» → **Установить значение по умолчанию** → На рассмотрении
   - Поле «Заявитель» → **Установить значение по умолчанию** → Текущий пользователь

> **Внимание при выборе значений**: при настройке обязательно сначала выбирайте **«Текущая форма»** в качестве источника. Для полей связей (категория, заявитель, обработчик и т. п. — поля «многие-к-одному») выбирайте сам объект, а не его развёрнутые подполя.
>
> При выборе переменной (например, «Текущий пользователь») сначала **кликом** выделите её, а затем **двойным кликом** перенесите в строку выбора.

![04-forms-and-details-2026-03-13-17-01-06](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-01-06.png)

![04-forms-and-details-2026-03-13-17-02-20](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-02-20.png)


![04-forms-and-details-2026-03-13-17-03-41](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-03-41.png)


Если хотите, чтобы поле нельзя было редактировать (например, статус), в настройках поля установите **«Режим отображения»** в **«Только для чтения»**.

![04-forms-and-details-2026-03-13-17-22-15](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-22-15.png)

> **Три режима отображения**: Редактируемый, Только для чтения (нельзя редактировать, но поле выглядит как обычное), Удобное чтение (только текст).

![04-forms-and-details-2026-03-13-12-54-53](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-12-54-53.png)

### Срочные заявки требуют описание

Добавим правило связывания с условием: при выборе приоритета «Срочный» поле «Описание» становится **обязательным** — пусть заявитель опишет ситуацию подробнее.

1. Нажмите **«Добавить правило связывания»**.

![04-forms-and-details-2026-03-13-17-07-34](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-07-34.png)

2. Настройте правило:
   - **Условие**: Текущая форма / Приоритет **равен** Срочный
   - **Действия**: поле «Описание» → **Обязательное**

![04-forms-and-details-2026-03-13-17-08-46](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-08-46.png)

![04-forms-and-details-2026-03-13-17-18-16](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-18-16.png)

3. Сохраните правило.

Тестируем: выбираем приоритет «Срочный» — у поля «Описание» появляется красная звёздочка `*`. Выбираем другой приоритет — звёздочка пропадает.

![04-forms-and-details-2026-03-13-17-20-18](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-20-18.gif)

Подгоним раскладку, как в предыдущих главах:
![04-forms-and-details-2026-03-13-17-25-55](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-25-55.png)

> **Что ещё умеют правила связывания?** Помимо значений по умолчанию и обязательности, можно управлять видимостью полей и динамически назначать значения. Например: при статусе «Закрыто» скрыть поле «Обработчик». Подробности — в следующих главах.

## 4.3 [Блок деталей](/interface-builder/blocks/data-blocks/details)

В прошлой главе мы добавили в строки кнопку «Просмотр» — клик открывает выезжающую панель. Теперь настроим её содержимое.

1. В таблице нажмите кнопку **«Просмотр»** какой-либо строки — откроется панель.
2. В панели нажмите **«Добавить блок» → Блок данных → Детали**.
3. Выберите **«Текущая коллекция»**.

![04-forms-and-details-2026-03-13-17-27-02](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-27-02.png)

4. В блоке деталей нажмите **«Поля»** и расставьте поля по такой схеме:


| Зона | Поля |
|------|------|
| Шапка | Заголовок, Статус (стиль метки) |
| Тело | Описание (большая текстовая область) |
| Боковая панель | Название категории, Приоритет, Заявитель, Обработчик, Время создания |

Как вставить большой заголовок?
Поля → Markdown → Редактировать Markdown → в редакторе выберите переменную → Текущая запись → Заголовок.
Так заголовок записи динамически вставится в Markdown-блок.
Удалите текст по умолчанию и Markdown-синтаксисом превратите его в заголовок второго уровня (т. е. поставьте `## ` в начало).

![04-forms-and-details-2026-03-13-17-36-26](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-36-26.png)

![04-forms-and-details-2026-03-13-17-39-51](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-39-51.png)

Поле «Заголовок» в самой панели можно убрать — настроим раскладку:

![04-forms-and-details-2026-03-13-17-43-36](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-43-36.png)


> **Лайфхак**: несколько полей можно ставить в одну строку перетаскиванием — раскладка получится плотнее и опрятнее.


1. В блоке деталей в **«Действия»** отметьте **«Редактировать»** — будет удобно из деталей сразу переходить в редактирование.

![04-forms-and-details-2026-03-13-17-45-15](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-45-15.png)

### Настройка формы редактирования

При клике на «Редактировать» откроется новое всплывающее окно — внутри нужна форма редактирования. Поля у неё практически такие же, как у формы создания. Настраивать всё заново?

Не надо. Сохраним форму создания **как шаблон**, а форма редактирования будет на него ссылаться.

**Шаг 1: возвращаемся к форме создания и сохраняем как шаблон**

1. Закройте текущее окно, вернитесь к списку заявок и нажмите «Добавить новый», чтобы открыть форму создания.
2. В правом верхнем углу формы нажмите **настройки блока** (иконка трёх линий) и найдите **«Сохранить как шаблон»**.

![04-forms-and-details-2026-03-13-17-47-21](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-47-21.png)

3. Сохраните; по умолчанию используется **«Ссылка»** — все формы, ссылающиеся на шаблон, разделяют одну конфигурацию: меняешь в одном месте, синхронизируется везде.

![04-forms-and-details-2026-03-13-17-47-44](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-47-44.png)


![04-forms-and-details-2026-03-13-18-39-05](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-39-05.png)

> Наша форма заявки несложная, поэтому удобнее «Ссылка». «Копировать» создаёт независимые копии для каждой формы — изменения у каждой свои.

**Шаг 2: ссылаемся на шаблон в форме редактирования**

1. Возвращаемся к панели деталей или строке таблицы, нажимаем «Редактировать» — открывается окно редактирования.

Может показаться: «Добавить блок → Другие блоки → Шаблоны блоков» — самый быстрый путь? Но Вы получите **форму создания**, и поля не подгрузятся автоматически. Это распространённая ловушка.

![04-forms-and-details-2026-03-13-17-59-36](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-17-59-36.png)

Правильный путь:

2. Нажмите **«Добавить блок» → Блок данных → Форма (Редактировать)** — обычный блок формы редактирования.
3. В нём нажмите **«Поля» → «Шаблоны полей»** и выберите сохранённый шаблон.
4. Поля заполнятся одним кликом, идентично форме создания.
5. Не забудьте добавить кнопку «Отправить», чтобы пользователь мог сохранить изменения.

![04-forms-and-details-2026-03-13-18-05-13](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-05-13.png)

![04-forms-and-details-2026-03-13-18-15-11](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-15-11.gif)

В будущем достаточно изменить шаблон — обе формы обновятся.

### Быстрое редактирование: меняем данные без открытия окна

Помимо редактирования через окно, NocoBase поддерживает **быстрое редактирование** прямо в таблице — без открытия чего-либо, мышью.

Включается в двух местах:

- **Уровень блока таблицы**: настройки блока (иконка трёх линий) → **«Быстрое редактирование»**. Все поля таблицы будут поддерживать быстрое редактирование.
- **Уровень отдельного поля**: настройки колонки → **«Быстрое редактирование»** — управление полем-по-полю.

![04-forms-and-details-2026-03-13-18-20-07](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-20-07.png)

После активации при наведении на ячейку появится иконка карандаша — клик откроет редактор поля, изменения сохраняются автоматически.

![04-forms-and-details-2026-03-13-18-21-09](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-21-09.gif)

> **Когда полезно?** Очень удобно, когда нужно массово менять статус, обработчика и т. д. Например, администратор просматривает список и быстро переводит заявки из «На рассмотрении» в «В обработке», не открывая окна.

## 4.4 Включаем историю записей

:::info Коммерческий плагин
«[История записей](https://docs.nocobase.com/ru/record-history/)» — плагин [Pro-версии](https://nocobase.ru/commercial), требует коммерческой лицензии. В Community-версии раздел можно пропустить, на остальные главы это не влияет.
:::

В системе заявок очень важно: **кто, когда и что изменил — должно быть видно**. Плагин «История записей» автоматически фиксирует каждое изменение.

### Настройка истории

1. В **Настройки → Менеджер плагинов** убедитесь, что плагин «История записей» включён.

![04-forms-and-details-2026-03-13-18-22-44](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-22-44.png)

2. Зайдите в страницу настройки плагина, нажмите **«Добавить коллекцию»** и выберите **«Заявки»**.
3. Отметьте поля для журналирования: **Заголовок, Статус, Приоритет, Обработчик, Описание** и т. д.

![04-forms-and-details-2026-03-13-18-25-11](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-25-11.png)

> **Совет**: не журналируйте всё подряд. Поля вроде идентификатора или даты создания не правят вручную — отслеживать их незачем. Логируйте только то, что важно с точки зрения бизнеса.

4. Теперь в настройках нажмите **«Синхронизировать исторические снимки»** — плагин сделает текущее состояние всех заявок первой записью истории, а далее каждая правка добавит новую запись.

![04-forms-and-details-2026-03-13-18-27-01](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-27-01.png)

![04-forms-and-details-2026-03-13-18-28-50](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-28-50.png)

### История на странице деталей

1. Возвращаемся в панель деталей заявки (клик «Просмотр» на строке).
2. В панели **«Добавить блок» → Блок истории записей**.
3. Выберите **«Текущая коллекция»**, данные — **«Текущая запись»**.
4. Внизу панели появится хронология: видно, кто и когда какое поле и с какого на какое значение изменил.

![04-forms-and-details-2026-03-13-18-31-45](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-31-45.png)

![04-forms-and-details-2026-03-13-18-33-00](https://static-docs.nocobase.com/04-forms-and-details-2026-03-13-18-33-00.gif)

Даже если заявка прошла через нескольких людей, все изменения как на ладони.

## Итоги главы

В этой главе мы прошли полный жизненный цикл данных:

- **Форма** — пользователь создаёт новую заявку, есть значения по умолчанию и валидация
- **Правила связывания** — у срочных заявок описание автоматически становится обязательным
- **Блок деталей** — наглядно показывает всю информацию о заявке
- **История записей** — автоматический аудит каждого изменения (коммерческий плагин, опционально)

От «видеть» к «вводить» и «находить» — система заявок стала по-настоящему пригодной к использованию.
