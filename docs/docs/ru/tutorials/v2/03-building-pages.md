# Глава 3. Сборка страниц — от пустой страницы к работающей

В прошлой главе мы собрали скелет коллекций, но данные существуют только «на серверной части» — пользователь их не видит. В этой главе вытащим данные **на сцену**: создадим [блок таблицы](/interface-builder/blocks/data-blocks/table) для отображения заявок, настроим колонки, сортировку, [фильтрацию](/interface-builder/blocks/filter-blocks/form) и пагинацию, превратив страницу в полноценный список заявок.

## 3.1 Что такое блок

В NocoBase **блок** — это «кубик» страницы. Хотите показать таблицу? Поставьте [блок таблицы](/interface-builder/blocks/data-blocks/table). Хотите форму — поставьте блок формы. На странице можно совмещать сразу несколько блоков и менять их раскладку перетаскиванием.

Распространённые типы блоков:

| Тип | Назначение |
|------|------|
| Таблица | Множество строк/столбцов |
| Форма | Ввод или редактирование данных |
| Детали | Полная информация по одной записи |
| Форма фильтра | Условия фильтрации других блоков |
| Диаграмма | Визуализация: круговая, линейная и т. д. |
| Markdown | Произвольный текст или пояснение |

Метафора: **блок = кубик**. Из них и соберём страницу управления заявками.

## 3.2 Добавляем меню и страницу

Сначала создадим вход в раздел «Управление заявками».

1. Включите переключатель **UI Редактор** в правом верхнем углу — войдёте в [режим UI Редактора](/get-started/how-nocobase-works) (вокруг страницы появятся оранжевые рамки).
2. Подведите курсор к кнопке **«Добавить элемент меню»** в верхней панели и выберите **«Добавить группу»**, назовите её **«Управление заявками»**.

![03-building-pages-2026-03-12-09-38-36](https://static-docs.nocobase.com/03-building-pages-2026-03-12-09-38-36.png)


3. В верхней панели сразу появится «Управление заявками». **Нажмите**, и слева раскроется меню группы.
4. В левом меню нажмите оранжевую кнопку **«Добавить элемент меню»** → **«Страница v2»** и добавьте две дочерние страницы:
   - **Список заявок** — все заявки
   - **Категории заявок** — управление категориями

![03-building-pages-2026-03-12-09-41-26](https://static-docs.nocobase.com/03-building-pages-2026-03-12-09-41-26.png)

> **Внимание**: при добавлении страницы будут опции «Страница v1» и «Страница v2»; в этих учебных материалах используем только **v2**.

## 3.3 Добавляем блок таблицы

Зайдём на страницу «Список заявок» и добавим блок таблицы:

1. На пустой странице нажмите **«Добавить блок»**.
2. Выберите **Блок данных → Таблица**.
3. В появившемся списке коллекций выберите **«Заявки»** (коллекция из прошлой главы).

![03-building-pages-2026-03-13-08-44-07](https://static-docs.nocobase.com/03-building-pages-2026-03-13-08-44-07.png)

Блок таблицы добавлен — на странице появится пустая таблица.

![03-building-pages-2026-03-13-08-44-29](https://static-docs.nocobase.com/03-building-pages-2026-03-13-08-44-29.png)

С пустой таблицей сложно отлаживать — добавим быстро кнопку «Создать», чтобы внести тестовые данные:

1. В правом верхнем углу таблицы нажмите **«Настроить действия»** и отметьте **«Добавить новый»**.

![03-building-pages-2026-03-17-14-58-39](https://static-docs.nocobase.com/03-building-pages-2026-03-17-14-58-39.png)

2. Нажмите кнопку **«Добавить новый»**, во всплывающем окне выберите **Добавить блок → Форма (Добавить новую) → Текущая коллекция**.

![03-building-pages-2026-03-17-14-59-42](https://static-docs.nocobase.com/03-building-pages-2026-03-17-14-59-42.png)

3. Во всплывающем окне нажмите **«Настроить поля»**, отметьте «Заголовок», «Статус», «Приоритет» и т. д.; нажмите **«Настроить действия»** и включите кнопку **«Отправить»**.

![03-building-pages-2026-03-17-15-00-54](https://static-docs.nocobase.com/03-building-pages-2026-03-17-15-00-54.png)

![03-building-pages-2026-03-17-15-01-49](https://static-docs.nocobase.com/03-building-pages-2026-03-17-15-01-49.png)

4. Заполните несколько произвольных заявок — содержимое появится в таблице.

![03-building-pages-2026-03-17-15-03-04](https://static-docs.nocobase.com/03-building-pages-2026-03-17-15-03-04.png)

> Подробную настройку формы (правила связывания, формы редактирования, всплывающие окна с деталями) разберём в [Главе 4](/tutorials/v2/04-forms-and-details). Здесь достаточно ввода данных.

## 3.4 Настраиваем колонки

По умолчанию таблица не показывает все поля — нужно вручную выбрать колонки:

1. В правой части шапки таблицы нажмите **«[Поля](/data-sources/data-modeling/collection-fields)»**.
2. Отметьте нужные поля:
   - **Заголовок** — суть заявки сразу на виду
   - **Статус** — текущий ход работы
   - **Приоритет** — срочность
   - **Категория** — поле связи, показывает имя категории
   - **Заявитель** — кто подал заявку
   - **Обработчик** — кто отвечает
3. Ненужные поля (ID, время создания) не отмечайте — таблица останется чистой.

![03-building-pages-2026-03-13-08-47-18](https://static-docs.nocobase.com/03-building-pages-2026-03-13-08-47-18.png)

> **Лайфхак**: порядок полей меняется перетаскиванием. Поставьте «Заголовок» и «Статус» в начало — ключевая информация будет читаться с одного взгляда.

### Поле связи показывает ID

После выбора «Категория» в таблице будет видно ID (число), а не имя. Это потому, что у поля связи в качестве заголовка по умолчанию используется ID. Два варианта решения:

**Способ 1: изменить в настройках колонки таблицы (только для текущей таблицы)**

В настройках колонки «Категория» найдите **«Заголовок поля»** и поменяйте `id` на **Название**. Это повлияет только на текущий блок.

![03-building-pages-2026-03-13-09-23-03](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-23-03.png)

![03-building-pages-2026-03-13-09-57-40](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-57-40.gif)

**Способ 2: изменить в источнике данных (глобально, рекомендуется)**

Перейдите в **Настройки → [Источники данных](/data-sources) → [Коллекция](/data-sources/data-modeling/collection) → коллекция категорий** и поменяйте **«Заголовок поля»** на **Название**. Тогда везде, где есть ссылка на категории, по умолчанию будет имя — раз настроили и забыли. После изменения на странице нужно заново добавить поле, чтобы изменение применилось.
![03-building-pages-2026-03-13-09-23-41](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-23-41.png)

## 3.5 Добавляем фильтр и сортировку

Когда заявок много, нужно быстро находить нужные. NocoBase предлагает несколько способов; начнём с самого распространённого — **блока формы фильтра**.

### Добавляем форму фильтра

1. На странице списка заявок нажмите **«Добавить блок»** → **Блок фильтра → Форма фильтра**.
2. На странице v2 не надо выбирать таблицу — форма фильтра добавится сразу.
3. В форме фильтра нажмите **«Поля»** — отобразится список всех фильтруемых блоков на странице, например `Таблица: Заявки #c48b` (после имени идёт UID блока для отличия нескольких блоков одной таблицы).

![03-building-pages-2026-03-13-08-48-37](https://static-docs.nocobase.com/03-building-pages-2026-03-13-08-48-37.png)

4. При наведении на имя блока раскроется список фильтруемых полей. Кликните по полю, чтобы добавить его как условие фильтра: **Статус**, **Приоритет**, **Категория**.

![03-building-pages-2026-03-13-09-25-44](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-25-44.png)

5. После добавления, как только пользователь введёт условие в форме фильтра, таблица **обновится автоматически**.

![03-building-pages-2026-03-13-10-00-25](https://static-docs.nocobase.com/03-building-pages-2026-03-13-10-00-25.gif)

### Поиск по нескольким полям

Что, если хочется одним поисковым полем искать по нескольким полям сразу?

В правом верхнем углу поля поиска есть параметр **«Соединить поля»**. Раскрывая его, Вы увидите список доступных для связного поиска полей в каждом блоке — по умолчанию подключён только «Заголовок».
![03-building-pages-2026-03-13-09-30-06](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-30-06.png)

Можно добавить ещё, например **Описание**, тогда при поиске будут проверяться оба поля.

И даже фильтровать по полю связного объекта — выберите «Категория» и в подменю отметьте «Название категории», чтобы поиск шёл и по имени категории.

![03-building-pages-2026-03-13-09-31-35](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-31-35.png)
![03-building-pages-2026-03-13-09-32-20](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-32-20.png)

> **Соединить поля — мощная штука**: работает между блоками и полями. Если на странице несколько блоков данных, поэкспериментируйте!

### Не хочу автоматический фильтр

Если нужно, чтобы фильтр срабатывал по кнопке, в правом нижнем углу формы фильтра нажмите **«[Действия](/interface-builder/actions)»** и отметьте кнопки **«Фильтр»** и **«Сбросить»**. Тогда пользователь введёт условия и нажмёт «Фильтр», чтобы запустить фильтрацию.
![03-building-pages-2026-03-13-09-33-15](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-33-15.png)

### Альтернатива: действие фильтра в самой таблице

Кроме отдельного блока формы фильтра, у блока таблицы тоже есть кнопка **«Фильтр»**. Над таблицей нажмите **«Действия»** и отметьте «Фильтр» — на панели таблицы появится кнопка фильтра. Клик откроет панель условий, где пользователь сможет фильтровать данные по полям.
![03-building-pages-2026-03-13-09-34-25](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-34-25.png)
![03-building-pages-2026-03-13-09-36-09](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-36-09.png)

Чтобы пользователь не выбирал поля каждый раз, в настройках кнопки фильтра можно задать поля по умолчанию.
![03-building-pages-2026-03-13-09-38-37](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-38-37.png)

> **Внимание**: фильтр в самой таблице **не поддерживает поиск по нескольким полям**. Для этого используйте блок формы фильтра вместе с «Соединить поля».

### Сортировка по умолчанию

Хочется, чтобы свежие заявки были сверху:

1. Нажмите настройки блока таблицы (иконка трёх линий) в правом верхнем углу.
2. Найдите **«[Настройка правил сортировки](/interface-builder/blocks/data-blocks/table)»**.
3. Добавьте поле сортировки: **Время создания**, направление — **По убыванию**.

![03-building-pages-2026-03-13-09-40-54](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-40-54.png)

Так новые заявки всегда наверху — обрабатывать удобнее.

## 3.6 Настраиваем действия строк

Просто список — мало; хочется видеть детали и редактировать заявки.

1. Над колонкой действий нажмите вторую «+».
2. Включите действия: **Просмотр**, **[Редактировать](../../interface-builder/actions/types/edit.md)**, **[Удалить](../../interface-builder/actions/types/delete.md)**.
3. В каждой строке появятся кнопки «Просмотр», «Редактировать» и «Удалить».

![03-building-pages-2026-03-13-09-42-42](https://static-docs.nocobase.com/03-building-pages-2026-03-13-09-42-42.png)

При клике «Просмотр» откроется выдвижной блок — в нём разместим блок деталей с полной информацией. Подробно настроим его в следующей главе.
Клик «Удалить» удалит строку.

## 3.7 Раскладка страницы

Сейчас на странице форма фильтра и таблица идут друг под другом — это не всегда удобно. NocoBase позволяет менять расположение и раскладку **перетаскиванием**.

В режиме конфигурации наведите курсор на ручку перетаскивания в левом верхнем углу блока (курсор станет крестообразной стрелкой), удерживайте и тащите.

**Перенесём форму фильтра над таблицу**: схватите блок формы фильтра и тащите к верхней границе таблицы, появится синяя подсказка-линия — отпустите. Форма расположится над таблицей.

![03-building-pages-2026-03-13-11-50-18](https://static-docs.nocobase.com/03-building-pages-2026-03-13-11-50-18.gif)

**Поместим поля фильтра в одну строку**: внутри формы фильтра поля по умолчанию идут вертикально. Перетащите «Приоритет» правее «Статуса», появится вертикальная подсказка-линия — отпустите. Два поля встанут в одну строку и сэкономят место.

![03-building-pages-2026-03-13-11-50-58](https://static-docs.nocobase.com/03-building-pages-2026-03-13-11-50-58.gif)

> Почти все элементы в NocoBase можно перетаскивать — кнопки действий, колонки таблицы, пункты меню и т. д. Экспериментируйте!

## 3.8 Настраиваем страницу категорий заявок

Не забываем: в 3.2 мы создали страницу «Категории заявок». Наполним её содержимым. Логика похожа на список заявок — добавление таблицы, выбор полей, настройка действий — повторяться не будем, а покажем одно ключевое отличие.

Помните, что в Главе 2 мы сделали коллекцию «Категории заявок» **древовидной** (с поддержкой иерархии). Чтобы блок таблицы корректно показывал дерево, нужно включить специальный параметр:

1. На странице «Категории заявок» добавьте блок таблицы и выберите коллекцию «Категории заявок».
2. Откройте настройки блока таблицы (иконка трёх линий) и включите **«Древовидная таблица»**.


После этого таблица будет показывать иерархию с отступами, а не плоский список.

3. Отметьте поля для отображения (название, описание и пр.) и настройте действия строк (добавление, редактирование, удаление).
4. **Совет по раскладке**: поместите «Название» в первую колонку, «Действия» — во вторую. Полей у таблицы немного, такая раскладка компактнее.

![03-building-pages-2026-03-13-18-51-36](https://static-docs.nocobase.com/03-building-pages-2026-03-13-18-51-36.png)

## Итоги главы

Поздравляем! У нашей системы заявок теперь есть полноценный **интерфейс управления**:

- понятная структура меню (Управление заявками → Список заявок / Категории заявок);
- **блок таблицы** для отображения данных заявок;
- **форма фильтра** для быстрой выборки по статусу, приоритету, категории;
- **сортировка** по времени создания (сначала новые);
- кнопки действий для просмотра и редактирования;
- **древовидный блок таблицы** для иерархии категорий.

Проще, чем казалось? Никакого кода — всё через перетаскивание и настройки.

## Анонс следующей главы

Просто видеть данные мало — пользователь должен **подавать новые заявки**. В следующей главе соберём блоки формы, настроим правила связывания и включим историю записей для отслеживания каждого изменения.
