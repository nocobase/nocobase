# Обзор развёртывания производственного окружения

Если NocoBase уже нормально работает на сервере, перед официальным запуском обычно добавляют две возможности:

1. Автоматическое возобновление работы приложения после перезагрузки машины.
2. Подключение входа через обратный прокси для стабильного внешнего доступа.

В NocoBase CLI это в основном две группы команд:

- `nb app autostart`
- `nb proxy`

Документация разделена на две части:

1. **Автозапуск приложения** — возобновление работы после перезагрузки машины.
2. **Обратный прокси** — стабильная точка внешнего доступа к приложению.

Сначала выберите, что нужнее сейчас, и перейдите на соответствующую страницу.

## Какие задачи это решает в производственном окружении

- `nb app autostart` — **как возобновить работу приложений после старта системы**;
- `nb proxy` — **как обеспечить стабильный внешний доступ**.

:::tip Почему не настроить автозапуск напрямую в Docker, PM2 или Supervisor?

`nb app autostart` не обходит эти способы управления процессами, а унифицирует их и сводит автозапуск к одному набору точек входа. Не нужно запоминать отдельные схемы для Docker, PM2 или Supervisor — в том числе для тех, что появятся в будущем.

При старте системы этот уровень обрабатывают `systemd`, `launchd` или скрипт хоста. Они один раз при загрузке машины выполняют:

```bash
nb app autostart run
```

Команда поднимает все приложения с включённым автозапуском.

Два слоя не стоит смешивать:

- Docker, PM2, Supervisor — **как работают приложения и как управлять их процессами**;
- `systemd`, `launchd`, скрипты хоста — **какую команду выполнить при старте системы**.

Если непонятно, зачем нужен `nb app autostart`, см. [Автозапуск приложения](./autostart.md) и [Замысел проектирования `nb app`](../cli-design/nb-app-design-intent.md).

:::

## Перед выходом в производственное окружение

Подтвердите:

- приложение сохранено как CLI-окружение;
- приложение нормально запускается на сервере;
- если подключаете обратный прокси — в окружении сохранён `appPort`;
- для внешнего доступа запланированы доменное имя, порт входа и HTTPS.

Если установка CLI или инициализация окружения не завершены, вернитесь к [Установить с помощью CLI (рекомендуется)](../installation/cli.md).

Если команда сообщает, что в окружении нет `appPort`, сначала выполните [`nb env update`](../../api/cli/env/update.md).
