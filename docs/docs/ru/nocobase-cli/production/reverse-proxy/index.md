# Обратный прокси

Статья относится только к приложениям, установленным через `nb init`.

В NocoBase обратный прокси производственной среды не только перенаправляет запросы в процесс приложения. Он также должен обрабатывать WebSockets, подпути, статические ресурсы интерфейса, каталоги загрузки, маршрут доступа к файлам `/files/` и резервные страницы SPA.

`nb proxy` собирает эти легко упускаемые детали в стабильный набор команд.

## Базовый процесс

Достаточно запомнить три команды:

```bash
nb proxy nginx use docker
nb proxy nginx generate --env test2 --host c.local.nocobase.com
nb proxy nginx reload
```

Для Caddy замените в команде `nginx` на `caddy`.

Если Nginx или Caddy установлены локально, в первой команде используйте `use local` вместо `use docker`.

В большинстве сценариев: `use` → `generate` → `reload`. Подробности — на страницах Nginx и Caddy.

## Когда выбрать Nginx, а когда Caddy

| Сценарий | Рекомендация |
| --- | --- |
| Уже используете Nginx для сайта, сертификатов, кэша или контроля доступа | [Nginx](./nginx.md) |
| Есть доменное имя, нужен HTTPS как можно скорее и меньше возни с TLS | [Caddy](./caddy.md) |
