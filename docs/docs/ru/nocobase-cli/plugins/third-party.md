# Установка и обновление сторонних плагинов

Если Вы получили пакет стороннего плагина, обычно импортируйте его в `storage/plugins` целевого приложения, перезапустите приложение, затем включите плагин или проверьте, что он заработал.

## Сначала подтвердите целевое окружение

Если локально ведётся несколько приложений, сначала переключите целевое окружение:

```bash
nb env use app1
```

## Используйте `nb plugin import` для импорта пакета плагина

`nb plugin import` поддерживает три источника: удалённый архив, локальный архив и имя пакета npm. Команда только импортирует плагин в `storage/plugins` и не включает его автоматически.

Если есть URL пакета, путь к локальному файлу или плагин опубликован в npm:

```bash
# удалённый архив
nb plugin import https://github.com/nocobase/plugin-auth-cas/releases/download/v1.4.0/plugin-auth-cas-1.4.0.tgz

# локальный архив
nb plugin import /your/path/plugin-auth-cas-1.4.0.tgz

# имя пакета или тег npm
nb plugin import @my-scope/plugin-auth-cas@beta
```

Для приватного npm-реестра обычно сначала входят в систему и указывают реестр:

```bash
npm login --registry=https://registry.example.com
nb plugin import @my-scope/plugin-auth-cas@beta --npm-registry=https://registry.example.com
```

## Укажите путь к хранилищу для импорта

Если известен корневой каталог `storage` целевого приложения, можно передать `--storage-path` напрямую, без привязки к текущему окружению:

```bash
nb plugin import /your/path/plugin-auth-cas-1.4.0.tgz --storage-path ./storage
```

CLI запишет плагин в `<storage-path>/plugins`. В этом случае можно не выполнять `nb env use` заранее и не передавать `--env`.

## Перезапуск после импорта

После импорта перезапустите целевое приложение:

```bash
nb app restart
```

Если текущее окружение не переключали, в команде можно явно указать `-e <env>`.

## Включение или проверка после перезапуска

При первой установке после перезапуска включите плагин:

```bash
nb plugin enable @nocobase/plugin-auth-cas
```

При первом включении установка завершится автоматически.

## Что делать при обновлении плагинов

Если плагин уже включён и нужно перейти на новую версию, обычно достаточно двух шагов:

```bash
nb plugin import /your/path/plugin-auth-cas-1.5.0.tgz
nb app restart
```

То же для пакета npm:

```bash
nb plugin import @my-scope/plugin-auth-cas@latest
nb app restart
```

При обновлении дополнительный `nb plugin enable` не нужен — импортируйте новый пакет и перезапустите приложение.

## Когда нет прямого доступа в интернет

Если целевая машина не может скачать плагин по URL, загрузите файл `.tgz` в любой каталог на ней и выполните локальный импорт:

```bash
nb plugin import /your/path/plugin-auth-cas-1.4.0.tgz
nb app restart
```

:::warning Примечание

Вручную распаковывать файлы в `storage/plugins` не нужно — `nb plugin import` поместит плагин в нужный каталог автоматически.

:::
