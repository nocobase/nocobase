# Установка в интрасети

Если сервер не имеет доступа к публичной сети, перед установкой нужно заранее подготовить образы, зависимости и пакеты плагинов для автономного использования. По умолчанию рекомендуется Docker — это самый короткий и воспроизводимый путь.

## Рекомендация по умолчанию: подготовьте образ Docker в автономном режиме.

На машине с доступом к публичной сети сначала загрузите образ приложения и образ базы данных:

```bash
docker pull registry.cn-shanghai.aliyuncs.com/nocobase/nocobase:latest-full
docker pull registry.cn-shanghai.aliyuncs.com/nocobase/postgres:16
```

Затем экспортируйте их в офлайн-файлы:

```bash
docker save -o nocobase-app.tar \
  registry.cn-shanghai.aliyuncs.com/nocobase/nocobase:latest-full

docker save -o nocobase-postgres.tar \
  registry.cn-shanghai.aliyuncs.com/nocobase/postgres:16
```

Если нужны коммерческие плагины, рекомендуется также подготовить пакет плагинов во внешней сети и перенести его в интрасеть вместе с образами.

## Скопируйте файлы на сервер интрасети

Подготовьте как минимум:

- `nocobase-app.tar`
- `nocobase-postgres.tar`
- `docker-compose.yml`
- `.env` или ваши собственные инструкции по развертыванию.

## Импортируйте образы на сервер интрасети

```bash
docker load -i nocobase-app.tar
docker load -i nocobase-postgres.tar
```

## Запустите приложение

После подготовки `docker-compose.yml` запускайте напрямую:

```bash
docker compose up -d
docker compose logs -f app
```

Если файл Compose ещё не готов, сначала прочитайте [Установка через Docker Compose](./docker-compose.md) и сохраните пример локально.

## Что делать, если Docker недоступен

Если в интрасети нельзя использовать Docker, во внешней сети можно создать полный проект через `create-nocobase-app`, установить зависимости, упаковать его и скопировать на сервер интрасети.

Этот путь длиннее, но практичнее в средах без контейнеров. Обычный процесс:

1. Создайте проект во внешней сетевой среде и установите зависимости.
2. Упакуйте каталог проекта.
3. Скопируйте на сервер интрасети.
4. Разархивируйте файл в интрасети, заполните `.env` и запустите приложение.
