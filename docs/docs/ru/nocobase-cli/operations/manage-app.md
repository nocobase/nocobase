# Управление приложением

Если приложение NocoBase сохранено как CLI-окружение, повседневное управление выполняется группой команд `nb app`: запуск, остановка, перезапуск, просмотр логов и обновление.

Обычно не нужно запоминать все параметры. Сначала определите задачу — «запустить приложение», «посмотреть логи для отладки» или «обновить до новой версии» — и выберите соответствующую команду.

Чтобы понять, зачем команды объединены в `nb app` и как это связано с `nb app autostart`, см. [Замысел проектирования `nb app`](../cli-design/nb-app-design-intent.md). На этой странице — только частые повседневные операции.

## Быстрый указатель

| Мне нужно... | Какую команду использовать |
| --- | --- |
| Запустить или возобновить работу приложения | [`nb app start`](../../api/cli/app/start.md) |
| Временно остановить приложение | [`nb app stop`](../../api/cli/app/stop.md) |
| Остановить вместе со встроенной БД, управляемой CLI | [`nb app stop --with-db`](../../api/cli/app/stop.md) |
| Перезапустить приложение после изменения конфигурации | [`nb app restart`](../../api/cli/app/restart.md) |
| Смотреть логи приложения в реальном времени | [`nb app logs`](../../api/cli/app/logs.md) |
| Обновить исходный код или образ до новой версии | [`nb app upgrade`](../../api/cli/app/upgrade.md) |

:::tip Сначала подтвердите текущее окружение

Команда `nb app` по умолчанию действует в текущем окружении. При нескольких окружениях перед запуском, остановкой, просмотром логов или обновлением подтвердите целевое окружение.

При явной передаче другого `--env` CLI обычно запрашивает подтверждение. В скриптах и неинтерактивных сценариях добавьте `--yes`. Переключение, просмотр и удаление нескольких окружений — в [Управление несколькими окружениями](./multi-environment.md).

:::

## Запустить приложение

Поднять приложение по умолчанию:

```bash
nb app start
```

Для окружения, отличного от текущего:

```bash
nb app start --env app1 --yes
```

Другие частые параметры запуска:

- `nb app start` по умолчанию сначала выполняет необходимую подготовку к установке или обновлению, затем запускает службу.

Локальное npm/Git-окружение запускает процесс приложения на машине, Docker-окружение пересобирает контейнер по сохранённой конфигурации. Подробнее — в [`nb app start`](../../api/cli/app/start.md).

## Остановить и перезапустить

Временная остановка:

```bash
nb app stop
```

После изменения конфигурации, зависимостей или кода проще использовать `nb app restart`:

```bash
nb app restart
nb app restart --env app1 --yes
```

`nb app restart` сначала останавливает, затем перезапускает так же, как `start`. Подробнее — в [`nb app stop`](../../api/cli/app/stop.md) и [`nb app restart`](../../api/cli/app/restart.md).

## Просмотр логов

При отладке обычно сначала смотрят логи:

```bash
nb app logs
```

Чтобы увидеть только последние строки или не следить за потоком:

```bash
nb app logs --tail 200
nb app logs --no-follow
nb app logs --env app1 --yes
```

Локальное npm/Git-окружение читает логи PM2, Docker-окружение — логи контейнера. По умолчанию `nb app logs` продолжает выводить новые записи. Подробнее — в [`nb app logs`](../../api/cli/app/logs.md).

## Обновление приложения

Команда обновления — `nb app upgrade`:

```bash
nb app upgrade
```

Это больше, чем «скачать новую версию». Процесс по умолчанию обычно включает:

1. Остановку текущего приложения.
2. Загрузку и замену сохранённого исходного кода или образа.
3. Синхронизацию коммерческих плагинов.
4. Обновление и запуск приложения.
5. Обновление сведений о среде выполнения окружения.

Если исходный код или образ уже обновлены и нужно только продолжить обновление и запуск по текущему содержимому:

```bash
nb app upgrade --skip-download
```

Чтобы явно указать целевую версию:

```bash
nb app upgrade --version beta
```

:::warning Примечание

`nb app upgrade` обычно запрашивает подтверждение перед началом. В скриптах, CI и других неинтерактивных сценариях нужно явно передать `--force`. При работе с разными окружениями обычно также указывают `--yes`.

```bash
nb app upgrade --env app1 --yes --force
```

:::

Подробнее о параметрах — в [`nb app upgrade`](../../api/cli/app/upgrade.md).
