# Обзор

## Введение

Плагин «Рабочий процесс» помогает организовывать автоматизированные бизнес-процессы в NocoBase — например, ежедневные согласования, синхронизацию данных, напоминания и другие задачи. В рабочем процессе вы можете реализовать сложную бизнес-логику, просто настраивая триггеры и связанные узлы через визуальный интерфейс, без написания кода.

### Пример

Каждый рабочий процесс организуется с помощью триггера и нескольких узлов. Триггер представляет событие в системе, а каждый узел — шаг выполнения. Вместе они описывают бизнес-логику, которая будет обработана после наступления события. На следующем изображении показан типичный процесс списания остатков после оформления заказа на продукт:

![Пример рабочего процесса](https://static-docs.nocobase.com/20251029222146.png)

Когда пользователь оформляет заказ, рабочий процесс автоматически проверяет остатки. Если их достаточно, он списывает запасы и переходит к созданию заказа; в противном случае процесс завершается.

### Сценарии использования

В более общем смысле рабочие процессы в приложениях NocoBase помогают решать задачи в разных сценариях:

- Автоматизировать повторяющиеся задачи: проверки заказов, синхронизация остатков, очистка данных, расчёты скоринга и т. д. больше не требуют ручной работы.
- Поддерживать взаимодействие человека и системы: организуйте согласования или проверки в ключевых узлах и продолжайте выполнение последующих шагов по результатам.
- Интегрироваться с внешними системами: отправляйте HTTP-запросы, принимайте push-уведомления от внешних сервисов и добивайтесь сквозной автоматизации.
- Быстро адаптироваться к изменениям бизнеса: изменяйте структуру процесса, условия или другие настройки узлов и вводите в эксплуатацию без нового релиза.

## Установка

«Рабочий процесс» — встроенный плагин NocoBase. Дополнительная установка или настройка не требуется.
