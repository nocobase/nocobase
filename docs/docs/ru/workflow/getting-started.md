# Быстрый старт

## Настройте свой первый рабочий процесс

Из верхнего меню, через меню настройки плагинов, перейдите на страницу управления плагином «Рабочий процесс»:

![Вход на страницу управления плагином «Рабочий процесс»](https://static-docs.nocobase.com/20251027222721.png)

В интерфейсе управления отображаются все созданные рабочие процессы:

![Управление рабочими процессами](https://static-docs.nocobase.com/20251027222900.png)

Нажмите кнопку «Создать», чтобы создать новый рабочий процесс, и выберите событие коллекции:

![Создание рабочего процесса](https://static-docs.nocobase.com/20251027222951.png)

После сохранения нажмите ссылку «Настроить» в списке, чтобы перейти в интерфейс настройки рабочего процесса:

![Пустой рабочий процесс](https://static-docs.nocobase.com/20251027223131.png)

Затем нажмите на карточку триггера, чтобы открыть панель настройки триггера. Выберите ранее созданную коллекцию (например, «Статьи»), для условия срабатывания выберите «После добавления записи» и нажмите кнопку «Сохранить», чтобы завершить настройку триггера:

![Настройка триггера](https://static-docs.nocobase.com/20251027224735.png)

Далее вы можете нажать кнопку с плюсом в рабочем процессе, чтобы добавить узел. Например, выберите узел вычисления, который будет объединять поля «Заголовок» и «ID» из данных триггера:

![Добавление узла вычисления](https://static-docs.nocobase.com/20251027224842.png)

Нажмите на карточку узла, чтобы открыть панель настройки узла. Используйте функцию `CONCATENATE` из Formula.js для объединения полей «Заголовок» и «ID». Оба поля вставляются с помощью селектора переменных:

![Узел вычисления с использованием функций и переменных](https://static-docs.nocobase.com/20251027224939.png)

Затем создайте узел обновления записи, чтобы сохранить результат в поле «Заголовок»:

![Создание узла обновления записи](https://static-docs.nocobase.com/20251027232654.png)

Аналогично, нажмите на карточку, чтобы открыть панель настройки узла обновления записи. Выберите коллекцию «Статьи», для ID обновляемой записи выберите ID данных из триггера, для обновляемого поля выберите «Заголовок», а для значения выберите результат из узла вычисления:

![Настройка узла обновления записи](https://static-docs.nocobase.com/20251027232802.png)

Наконец, нажмите переключатель «Включить»/«Отключить» в панели инструментов в правом верхнем углу, чтобы перевести рабочий процесс в активное состояние. Теперь рабочий процесс готов к запуску.

## Запуск рабочего процесса

Вернитесь в основной интерфейс системы, создайте статью через блок статей и введите заголовок статьи:

![Создание данных статьи](https://static-docs.nocobase.com/20251027233004.png)

После сохранения и обновления блока вы увидите, что заголовок статьи был автоматически обновлен до формата «Заголовок статьи + ID статьи»:

![Заголовок статьи, измененный рабочим процессом](https://static-docs.nocobase.com/20251027233043.png)

:::info{title=Подсказка}
Поскольку рабочие процессы, запускаемые событиями коллекции, выполняются асинхронно, вы не сразу увидите обновление данных в интерфейсе после их сохранения. Однако через некоторое время вы сможете увидеть обновленное содержимое, обновив страницу или блок.
:::

## Просмотр истории выполнения

Рабочий процесс был успешно запущен и выполнен один раз. Вы можете вернуться в интерфейс управления рабочими процессами, чтобы просмотреть соответствующую историю выполнения:

![Просмотр списка рабочих процессов](https://static-docs.nocobase.com/20251027233246.png)

В списке рабочих процессов вы увидите, что этот рабочий процесс сгенерировал одну запись истории выполнения. Нажмите на ссылку в столбце «Количество», чтобы открыть записи истории выполнения для соответствующего рабочего процесса:

![Список истории выполнения для соответствующего рабочего процесса](https://static-docs.nocobase.com/20251027233341.png)

Нажмите ссылку «Просмотреть», чтобы перейти на страницу сведений об этом выполнении, где вы сможете увидеть статус выполнения и данные результатов для каждого узла:

![Детали истории выполнения рабочего процесса](https://static-docs.nocobase.com/20251027233615.png)

Контекстные данные триггера и данные результатов выполнения узла можно просмотреть, нажав кнопку статуса в правом верхнем углу соответствующей карточки. Например, давайте посмотрим данные результатов узла вычисления:

![Результат узла вычисления](https://static-docs.nocobase.com/20251027233635.png)

Вы увидите, что данные результатов узла вычисления содержат вычисленный заголовок. Именно этот заголовок будет использован последующим узлом обновления записи.

## Итоги

Выполнив описанные выше шаги, мы завершили настройку и запуск простого рабочего процесса и познакомились со следующими базовыми понятиями:

- **Рабочий процесс**: Определяет основную информацию о потоке, включая название, тип триггера и статус активности. В нём можно настроить любое количество узлов. Это сущность, которая содержит поток.
- **Триггер**: Каждый рабочий процесс содержит один триггер, который настраивается на определённые условия для запуска рабочего процесса. Это точка входа в поток.
- **Узел**: Узел — это единица инструкций в рабочем процессе, которая выполняет определённое действие. Несколько узлов в рабочем процессе образуют полный поток выполнения через отношения «вышестоящий-нижестоящий».
- **Выполнение**: Выполнение — это конкретный объект, создаваемый после запуска рабочего процесса, также известный как запись выполнения или история выполнения. Оно содержит такую информацию, как статус выполнения и контекстные данные триггера. Для каждого узла также существуют соответствующие результаты выполнения, включающие статус узла и данные результатов.
