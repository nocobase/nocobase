# Завершение рабочего процесса

При выполнении этот узел немедленно завершает текущий рабочий процесс с состоянием, указанным в его настройках. Обычно он используется для управления потоком выполнения на основе определённой логики: когда выполняются определённые условия, рабочий процесс прерывается, и дальнейшая обработка последующих шагов прекращается. Это можно сравнить с инструкцией `return` в языках программирования, которая используется для выхода из текущей функции.

## Добавление узла

В интерфейсе настройки рабочего процесса нажмите кнопку с плюсом («+») в потоке, чтобы добавить узел «Завершение рабочего процесса»:

### Статус завершения

Статус завершения повлияет на итоговый статус выполнения рабочего процесса. Его можно настроить как «Успешно» или «Неуспешно». Когда выполнение рабочего процесса достигает этого узла, он немедленно завершается с указанным статусом.

:::info{title=Примечание}
При использовании в рабочем процессе типа «Событие перед действием» этот узел перехватывает запрос, инициировавший действие. Подробности см. в [Использование «События перед действием»](../triggers/pre-action).

Кроме того, помимо перехвата запроса, инициировавшего действие, настройка статуса завершения также повлияет на статус обратной связи в «сообщении ответа» для этого типа рабочего процесса.
:::
