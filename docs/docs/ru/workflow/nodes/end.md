# Завершить рабочий процесс

При выполнении этого узла текущий рабочий процесс немедленно завершается со статусом, настроенным в узле. Обычно он используется для управления потоком по определенной логике: при выполнении заданных условий выходить из текущего рабочего процесса и останавливать выполнение последующих процессов. Это аналог инструкции возврата в языках программирования, которая завершает текущую функцию.

## Добавление узла

В интерфейсе настройки рабочего процесса нажмите кнопку плюса ("+") в потоке, чтобы добавить узел «Завершить рабочий процесс»:

![Добавление узла завершения рабочего процесса](https://static-docs.nocobase.com/672186ab4c8f7313dd3cf9c880b524b8.png)

## Настройка узла

![Настройка узла завершения рабочего процесса](https://static-docs.nocobase.com/bb6a597f25e9afb72836a14a0fe0683e.png)

### Статус завершения

Статус завершения влияет на итоговый статус выполнения рабочего процесса. Можно настроить «Успешно» или «Ошибка». Когда выполнение рабочего процесса доходит до этого узла, оно немедленно завершится с выбранным статусом.

:::info{title=Примечание}
При использовании в рабочем процессе типа «Событие до действия» он перехватывает запрос, который инициировал действие. Подробности см. в [использовании «События до действия»](../triggers/pre-action).

Кроме перехвата инициирующего запроса, настройка статуса завершения также влияет на статус обратной связи в «Ответном сообщении» для этого типа рабочего процесса.
:::
