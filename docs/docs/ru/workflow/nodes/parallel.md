# Параллельная ветвь

Узел параллельной ветви может разделить рабочий процесс на несколько ветвей. Каждая ветвь может быть настроена с различными узлами, и способ выполнения варьируется в зависимости от режима ветви. Используйте узел параллельной ветви в сценариях, где необходимо одновременно выполнить несколько действий.

## Установка

Встроенный плагин, установка не требуется.

## Создание узла

В интерфейсе настройки рабочего процесса нажмите кнопку плюс («+») в потоке, чтобы добавить узел «Параллельная ветвь»:

![Добавить параллельную ветвь](https://static-docs.nocobase.com/9e0f3faa0b9335270647a3047759eac.png)

После добавления узла параллельной ветви в рабочий процесс по умолчанию добавляются две подветви. Вы также можете добавить любое количество дополнительных ветвей, нажав кнопку добавления ветви. В каждую ветвь можно добавить любое количество узлов. Ненужные ветви можно удалить, нажав кнопку удаления в начале ветви.

![Управление параллельными ветвями](https://static-docs.nocobase.com/36088a8b7970c8a1771eb3ee9bc2a757.png)

## Настройка узла

### Режим ветви

Узел параллельной ветви имеет следующие три режима:

- **Все успешно**: Рабочий процесс продолжит выполнение узлов после завершения ветвей только в том случае, если все ветви будут выполнены успешно. В противном случае, если какая-либо ветвь завершится преждевременно (из-за сбоя, ошибки или любого другого неуспешного состояния), весь узел параллельной ветви завершится преждевременно с этим статусом. Также известен как «режим All».
- **Любой успешно**: Рабочий процесс продолжит выполнение узлов после завершения ветвей, как только любая ветвь будет выполнена успешно. Весь узел параллельной ветви завершится преждевременно только в том случае, если все ветви завершатся преждевременно (из-за сбоя, ошибки или любого другого неуспешного состояния). Также известен как «режим Any».
- **Любой успешно или сбой**: Рабочий процесс продолжит выполнение узлов после завершения ветвей, как только любая ветвь будет выполнена успешно. Однако, если какой-либо узел завершится сбоем, вся параллельная ветвь завершится преждевременно с этим статусом. Также известен как «режим Race».

Независимо от режима, каждая ветвь будет выполняться последовательно слева направо до тех пор, пока не будут выполнены условия предустановленного режима ветви. После этого рабочий процесс либо продолжит выполнение последующих узлов, либо завершится преждевременно.
