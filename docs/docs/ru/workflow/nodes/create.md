# Создать запись

Используется для добавления новой записи в коллекцию.

Значения полей новой записи могут использовать переменные из контекста рабочего процесса. Чтобы назначать значения полям связей, можно напрямую ссылаться на соответствующие переменные данных в контексте — это может быть объект или значение внешнего ключа. Если переменные не используются, нужно вручную ввести значения внешних ключей. Для нескольких значений внешнего ключа в связи «один-ко-многим» они должны быть разделены запятыми.

## Создание узла

В интерфейсе настройки рабочего процесса нажмите кнопку плюса ("+") в потоке, чтобы добавить узел «Создать запись»:

![Добавление узла «Создать запись»](https://static-docs.nocobase.com/386c8c01c89b1eeab848510e77f4841a.png)

## Настройка узла

![Настройка узла «Создать запись»](https://static-docs.nocobase.com/5f7b97a51b64a1741cf82a4d4455b610.png)

### Коллекция

Выберите коллекцию, в которую нужно добавить новую запись.

### Значения полей

Назначьте значения полям коллекции. Можно использовать переменные из контекста рабочего процесса или вручную вводить статические значения.

:::info{title="Примечание"}
Данные, созданные узлом «Создать запись» в рабочем процессе, не заполняют автоматически пользовательские поля вроде «Создано пользователем» и «Последнее обновление пользователем». При необходимости значения этих полей нужно настроить вручную.
:::

### Предзагрузка связанных данных

Если поля новой записи включают поля связей и вы хотите использовать соответствующие данные связей в следующих шагах рабочего процесса, отметьте нужные поля связей в настройке предзагрузки. Тогда после создания новой записи данные связей будут автоматически загружены и сохранены в данных результата узла.

## Пример

Например, когда запись в коллекции «Посты» создается или обновляется, можно автоматически создавать запись «Версии постов» для фиксации истории изменений поста. Для этого можно использовать узел «Создать запись»:

![Пример конфигурации рабочего процесса](https://static-docs.nocobase.com/dfd4820d49c145fa331883fc09c9161f.png)

![Пример конфигурации узла](https://static-docs.nocobase.com/1a0992e66170be12a068da6503298868.png)

После включения рабочего процесса с этой конфигурацией при изменении записи в коллекции «Посты» автоматически будет создаваться запись «Версии постов» для фиксации истории изменений поста.
