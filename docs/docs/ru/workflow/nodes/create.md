# Создание записи

Используется для добавления новой записи в **коллекцию**.

Значения полей для новой записи могут использовать переменные из контекста **рабочего процесса**. Чтобы присвоить значения полям связей, вы можете напрямую ссылаться на соответствующие переменные данных в контексте, которые могут быть как объектом, так и значением внешнего ключа. Если вы не используете переменные, вам потребуется вручную ввести значения внешних ключей. Для нескольких значений внешних ключей в связи "многие ко многим" их необходимо разделять запятыми.

## Создание узла

В интерфейсе настройки **рабочего процесса** нажмите кнопку с плюсом («+») в потоке, чтобы добавить узел «Создание записи»:

![Add 'Create Record' node](https://static-docs.nocobase.com/386c8c01c89b1eeab848510e77f4841a.png)

## Настройка узла

![Create Record Node_Example_Node Configuration](https://static-docs.nocobase.com/5f7b97a51b64a1741cf82a4d4455b610.png)

### Коллекция

Выберите **коллекцию**, в которую вы хотите добавить новую запись.

### Значения полей

Присвойте значения полям **коллекции**. Вы можете использовать переменные из контекста **рабочего процесса** или вручную ввести статические значения.

:::info{title="Примечание"}
Данные, созданные узлом «Создание записи» в **рабочем процессе**, не обрабатывают автоматически пользовательские данные, такие как «Создано» и «Последнее изменение». Вам необходимо самостоятельно настроить значения для этих полей по мере необходимости.
:::

### Предварительная загрузка данных связей

Если поля новой записи включают поля связей и вы хотите использовать соответствующие данные связей на последующих шагах **рабочего процесса**, вы можете отметить соответствующие поля связей в настройках предварительной загрузки. Таким образом, после создания новой записи соответствующие данные связей будут автоматически загружены и сохранены вместе с результирующими данными узла.

## Пример

Например, когда запись в **коллекции** «Статьи» создается или обновляется, необходимо автоматически создать запись «Версии статей», чтобы зафиксировать историю изменений статьи. Для этого можно использовать узел «Создание записи»:

![Create Record Node_Example_Workflow Configuration](https://static-docs.nocobase.com/dfd4820d49c145fa331883fc09c9161f.png)

![Create Record Node_Example_Node Configuration](https://static-docs.nocobase.com/1a0992e6170be12a068da6503298868.png)

После включения **рабочего процесса** с этой конфигурацией, при изменении записи в **коллекции** «Статьи» автоматически будет создана запись «Версии статей» для регистрации истории изменений статьи.
