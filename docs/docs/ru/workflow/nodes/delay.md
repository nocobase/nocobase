# Задержка

## Введение

Узел «Задержка» позволяет добавить задержку в рабочий процесс. После завершения задержки, в зависимости от настроек, рабочий процесс может либо продолжить выполнение последующих узлов, либо завершиться досрочно.

Его часто используют вместе с узлом «Параллельная ветвь». Вы можете добавить узел «Задержка» в одну из ветвей, чтобы обрабатывать события после истечения времени ожидания. Например, в параллельной ветви одна ветвь может содержать ручную обработку, а другая — узел «Задержка». Если ручная обработка превышает лимит времени, то при настройке «Завершить при превышении времени ожидания» ручная обработка должна быть завершена в течение заданного времени. Если же выбрана настройка «Продолжить при превышении времени ожидания», то по истечении времени ручную обработку можно игнорировать.

## Установка

Встроенный плагин, установка не требуется.

## Создание узла

В интерфейсе настройки рабочего процесса нажмите кнопку «плюс» («+») в потоке, чтобы добавить узел «Задержка»:

## Настройка узла

![Узел Задержка_Настройка узла](https://static-docs.nocobase.com/5fe8a36535f20a087a0148ffa1cd2aea.png)

### Время задержки

Для времени задержки вы можете ввести число и выбрать единицу измерения времени. Поддерживаемые единицы измерения: секунды, минуты, часы, дни и недели.

### Статус по истечении времени ожидания

Для статуса по истечении времени ожидания вы можете выбрать «Продолжить успешно» или «Завершить с ошибкой». Первый вариант означает, что после окончания задержки рабочий процесс продолжит выполнение последующих узлов. Второй вариант означает, что после окончания задержки рабочий процесс будет досрочно завершен со статусом ошибки.

## Пример

Рассмотрим сценарий, когда на заявку необходимо ответить в течение ограниченного времени после ее создания. Нам нужно добавить узел ручной обработки в одну из двух параллельных ветвей, а узел «Задержка» — в другую. Если на ручную обработку не поступил ответ в течение 10 минут, статус заявки обновляется на «Время ожидания истекло, не обработано».
