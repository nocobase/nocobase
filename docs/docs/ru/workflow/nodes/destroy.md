# Удаление данных

Используется для удаления данных из **коллекции**, которые соответствуют заданным условиям.

Базовое использование узла удаления похоже на узел обновления, за исключением того, что узлу удаления не требуется присваивать значения полям. Вам нужно только выбрать **коллекцию** и условия фильтрации. Результат работы узла удаления возвращает количество успешно удаленных строк. Его можно просмотреть только в истории выполнения, и он не может быть использован в качестве переменной в последующих узлах.

:::info{title=Примечание}
В настоящее время узел удаления не поддерживает построчное удаление; он выполняет только пакетное удаление. Следовательно, он не будет вызывать другие события для каждой отдельной удаляемой записи.
:::

## Создание узла

В интерфейсе настройки **рабочего процесса** нажмите кнопку с плюсом («+») в потоке, чтобы добавить узел «Удалить данные»:

![Создание узла удаления данных](https://static-docs.nocobase.com/e1d6b8728251fcdbed6c7f50e5570da2.png)

## Настройка узла

![Узел удаления_Настройка узла](https://static-docs.nocobase.com/580600c2b13ef4e01dfa48b23539648e.png)

### Коллекция

Выберите **коллекцию**, из которой нужно удалить данные.

### Условия фильтрации

Аналогично условиям фильтрации при обычном запросе к **коллекции**, вы можете использовать контекстные переменные **рабочего процесса**.

## Пример

Например, для периодической очистки отмененных и недействительных исторических данных заказов можно использовать узел удаления:

![Узел удаления_Пример_Настройка узла](https://static-docs.nocobase.com/b94b75077a17252f8523c3f13ce5f320.png)

**Рабочий процесс** будет запускаться периодически и выполнять удаление всех отмененных и недействительных исторических данных заказов.
