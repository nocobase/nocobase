# HTTP ответ

## Введение

Этот узел поддерживается только в синхронных рабочих процессах с триггером «Вебхук» и используется для возврата ответа сторонней системе. Например, при обработке обратного вызова платежа, если в бизнес-процессе возник неожиданный результат (ошибка или сбой), можно использовать узел «Ответ» для возврата ошибки сторонней системе — тогда некоторые системы смогут повторить запрос позже по статусу.

Кроме того, выполнение узла «Ответ» завершает выполнение рабочего процесса, и последующие узлы не выполняются. Если во всём рабочем процессе не настроен узел «Ответ», система автоматически вернёт ответ по статусу выполнения потока: `200` при успешном выполнении и `500` при неуспешном.

## Создание узла

В интерфейсе настройки рабочего процесса нажмите кнопку плюса («+») в потоке, чтобы добавить узел «Ответ»:

![Добавление узла «Ответ»](https://static-docs.nocobase.com/20241210115120.png)

## Настройка ответа

![Настройка ответа](https://static-docs.nocobase.com/20241210115500.png)

В теле ответа можно использовать переменные из контекста рабочего процесса.
