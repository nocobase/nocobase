# Импорт модулей

В RunJS можно использовать два типа модулей: **встроенные модули** (доступны напрямую через `ctx.libs` без необходимости импорта) и **внешние модули** (загружаются по требованию через `ctx.importAsync()` или `ctx.requireAsync()`).

---

## Встроенные модули — ctx.libs (импорт не требуется)

RunJS включает в себя популярные библиотеки, к которым можно обращаться напрямую через `ctx.libs`. Вам **не нужно** использовать `import` или асинхронную загрузку для них.

| Свойство | Описание |
|------|------|
| **ctx.libs.React** | Ядро React, используется для JSX и хуков |
| **ctx.libs.ReactDOM** | ReactDOM (можно использовать для `createRoot` и т. д.) |
| **ctx.libs.antd** | Библиотека компонентов Ant Design |
| **ctx.libs.antdIcons** | Иконки Ant Design |
| **ctx.libs.math** | [Math.js](https://mathjs.org/): математические выражения, матричные операции и т. д. |
| **ctx.libs.formula** | [Formula.js](https://formulajs.github.io/): формулы в стиле Excel (SUM, AVERAGE и т. д.) |

### Пример: React и antd

```tsx
const { Button } = ctx.libs.antd;

ctx.render(<Button>Нажать</Button>);
```

### Пример: ctx.libs.math

```ts
const result = ctx.libs.math.evaluate('2 + 3 * 4');
// result === 14
```

### Пример: ctx.libs.formula

```ts
const values = [1, 2, 3, 4];
const sum = ctx.libs.formula.SUM(values);
const avg = ctx.libs.formula.AVERAGE(values);
```

## Внешние модули

Если вам нужны сторонние библиотеки, выберите метод загрузки в зависимости от формата модуля:

- **ESM-модули** → используйте `ctx.importAsync()`
- **UMD/AMD-модули** → используйте `ctx.requireAsync()`

---

### Импорт ESM-модулей

Используйте **`ctx.importAsync()`** для динамической загрузки ESM-модулей по URL. Это подходит для таких сценариев, как блоки JS, поля JS и действия JS.

```ts
importAsync<T = any>(url: string): Promise<T>;
```

- **url**: адрес ESM-модуля. Поддерживаются сокращенные форматы, такие как `<имя_пакета>@<версия>` или пути к подфайлам, например `<имя_пакета>@<версия>/<путь_к_файлу>` (например, `vue@3.4.0`, `lodash@4/lodash.js`). К ним будет добавлен префикс настроенного CDN. Также поддерживаются полные URL-адреса.
- **Возвращает**: объект пространства имен разрешенного модуля.

#### По умолчанию https://esm.sh

Если настройки не заданы, для сокращенных форм в качестве префикса CDN будет использоваться **https://esm.sh**. Например:

```ts
const Vue = await ctx.importAsync('vue@3.4.0');
// Эквивалентно загрузке из https://esm.sh/vue@3.4.0
```

#### Собственный сервис esm.sh

Если вам нужно использовать внутреннюю сеть или собственный CDN, вы можете развернуть сервис, совместимый с протоколом esm.sh, и указать его через переменные окружения:

- **ESM_CDN_BASE_URL**: базовый URL для ESM CDN (по умолчанию `https://esm.sh`).
- **ESM_CDN_SUFFIX**: необязательный суффикс (например, `/+esm` для jsDelivr).

Для развертывания собственного сервиса обратитесь к: [https://github.com/nocobase/esm-server](https://github.com/nocobase/esm-server)

### Импорт UMD/AMD-модулей

Используйте **`ctx.requireAsync()`** для асинхронной загрузки модулей UMD/AMD или скриптов, которые прикрепляются к глобальному объекту.

```ts
requireAsync<T = any>(url: string): Promise<T>;
```

- **url**: поддерживает две формы:
  - **Сокращенный путь**: `<имя_пакета>@<версия>/<путь_к_файлу>`, аналогично `ctx.importAsync()`, разрешается в соответствии с текущей конфигурацией ESM CDN. При разрешении добавляется `?raw` для прямого запроса исходного файла (обычно это сборка UMD). Например, `echarts@5/dist/echarts.min.js` фактически запрашивает `https://esm.sh/echarts@5/dist/echarts.min.js?raw` (при использовании esm.sh по умолчанию).
  - **Полный URL**: любой полный адрес CDN (например, `https://cdn.jsdelivr.net/npm/xxx`).
- **Возвращает**: объект загруженной библиотеки (конкретная форма зависит от того, как библиотека экспортирует свое содержимое).

После загрузки многие UMD-библиотеки прикрепляются к глобальному объекту (например, `window.xxx`). Вы можете использовать их в соответствии с документацией библиотеки.

**Пример**

```ts
// Сокращенный путь (разрешается через esm.sh как ...?raw)
const echarts = await ctx.requireAsync('echarts@5/dist/echarts.min.js');

// Полный URL
const dayjs = await ctx.requireAsync('https://cdn.jsdelivr.net/npm/dayjs@1/dayjs.min.js');
```

**Примечание**: если библиотека предоставляет версию ESM, отдавайте предпочтение `ctx.importAsync()` для лучшей семантики модулей и Tree-shaking.
