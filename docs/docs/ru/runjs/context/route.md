# ctx.route

Информация о текущем совпадении маршрута, соответствующая концепции `route` в React Router. Используется для получения конфигурации текущего маршрута, параметров и т. д. Обычно используется совместно с `ctx.router` и `ctx.location`.

## Применимость

| Сценарий | Описание |
|------|------|
| **JSBlock / JSField** | Выполнение условного рендеринга или отображение идентификатора текущей страницы на основе `route.pathname` или `route.params`. |
| **Правила связки / Потоки событий** | Чтение параметров маршрута (например, `params.name`) для логического ветвления или передачи дочерним компонентам. |
| **Навигация в представлениях** | Внутреннее сравнение `ctx.route.pathname` с целевым путем для принятия решения о вызове `ctx.router.navigate`. |

> Примечание: `ctx.route` доступен только в средах RunJS, содержащих контекст маршрутизации (например, JSBlock внутри страницы, страницы рабочих процессов и т. д.). Он может быть пустым в чисто серверных контекстах или контекстах без маршрутизации (например, в рабочих процессах).

## Определение типа

```ts
type RouteOptions = {
  name?: string;   // Уникальный идентификатор маршрута
  path?: string;   // Шаблон маршрута (например, /admin/:name)
  params?: Record<string, any>;  // Параметры маршрута (например, { name: 'users' })
  pathname?: string;  // Полный путь текущего маршрута (например, /admin/users)
};
```

## Основные поля

| Поле | Тип | Описание |
|------|------|------|
| `pathname` | `string` | Полный путь текущего маршрута, совпадает с `ctx.location.pathname`. |
| `params` | `Record<string, any>` | Динамические параметры, извлеченные из шаблона маршрута, например `{ name: 'users' }`. |
| `path` | `string` | Шаблон маршрута, например `/admin/:name`. |
| `name` | `string` | Уникальный идентификатор маршрута, часто используется в сценариях с несколькими вкладками или представлениями. |

## Взаимосвязь с ctx.router и ctx.location

| Назначение | Рекомендуемое использование |
|------|----------|
| **Чтение текущего пути** | `ctx.route.pathname` или `ctx.location.pathname`; оба значения идентичны при совпадении. |
| **Чтение параметров маршрута** | `ctx.route.params`, например `params.name` представляет UID текущей страницы. |
| **Навигация** | `ctx.router.navigate(path)` |
| **Чтение параметров запроса и состояния (state)** | `ctx.location.search`, `ctx.location.state` |

`ctx.route` фокусируется на «конфигурации совпавшего маршрута», в то время как `ctx.location` — на «текущем местоположении URL». Вместе они обеспечивают полное описание состояния маршрутизации.

## Примеры

### Чтение pathname

```ts
// Отображение текущего пути
ctx.message.info('Текущая страница: ' + ctx.route.pathname);
```

### Ветвление на основе params

```ts
// params.name обычно является UID текущей страницы (например, идентификатор страницы рабочего процесса)
if (ctx.route.params?.name === 'users') {
  // Выполнение специфической логики на странице управления пользователями
}
```

### Отображение на странице рабочего процесса

```tsx
<div>
  <h1>Текущая страница - {ctx.route.pathname}</h1>
  <p>Идентификатор маршрута: {ctx.route.params?.name}</p>
</div>
```

## Связанные разделы

- [ctx.router](./router.md): Навигация по маршрутам. После изменения пути через `ctx.router.navigate()` значение `ctx.route` обновится соответствующим образом.
- [ctx.location](./location.md): Текущее местоположение URL (pathname, search, hash, state), используется совместно с `ctx.route`.
