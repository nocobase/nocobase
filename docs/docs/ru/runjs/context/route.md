# ctx.route

Информация о текущем совпадении маршрута, соответствующая маршруту React Router; используется для получения конфигурации совпавшего маршрута, `params` и т. д. Используется вместе с `ctx.router` и `ctx.location`.

## Сценарии использования

| Сценарий | Описание |
|----------|----------|
| **JS-блок / Поле JS** | Условный рендер или отображение по `route.pathname` и `route.params` |
| **Связывание / поток событий** | Чтение параметров маршрута (например, `params.name`) для логики или передачи в дочерние блоки |
| **Навигация по представлениям** | Сравнение `ctx.route.pathname` с целевым путём перед `ctx.router.navigate` |

> `ctx.route` доступен только в контекстах RunJS с маршрутизатором (например, JS-блок на странице, страница потока). В чисто серверных или контекстах без маршрутизации (например, рабочий процесс) может отсутствовать.

## Тип

```ts
type RouteOptions = {
  name?: string;   // Уникальный идентификатор маршрута
  path?: string;   // Шаблон маршрута (например, /admin/:name)
  params?: Record<string, any>;  // Параметры маршрута (например, { name: 'users' })
  pathname?: string;  // Полный путь текущего маршрута (например, /admin/users)
};
```

## Основные поля

| Поле | Тип | Описание |
|------|-----|----------|
| `pathname` | `string` | Полный путь текущего маршрута; совпадает с `ctx.location.pathname` |
| `params` | `Record<string, any>` | Динамические параметры шаблона маршрута, например `{ name: 'users' }` |
| `path` | `string` | Шаблон маршрута, например `/admin/:name` |
| `name` | `string` | Уникальный идентификатор маршрута; часто используется для вкладок и представлений |

## Связь с ctx.router и ctx.location

| Задача | Рекомендуемый API |
|--------|-------------------|
| **Текущий путь** | `ctx.route.pathname` или `ctx.location.pathname`; при совпадении маршрута значения одинаковые |
| **Параметры маршрута** | `ctx.route.params`, например `params.name` для UID текущей страницы |
| **Навигация** | `ctx.router.navigate(path)` |
| **Параметры запроса и состояния** | `ctx.location.search`, `ctx.location.state` |

`ctx.route` — это «конфигурация совпавшего маршрута», а `ctx.location` — «текущий URL»; вместе они описывают состояние маршрута.

## Примеры

### Чтение pathname

```ts
// Отображение текущего пути
ctx.message.info('Текущая страница: ' + ctx.route.pathname);
```

### Ветвление по params

```ts
// params.name обычно является UID текущей страницы (например, идентификатор страницы рабочего процесса)
if (ctx.route.params?.name === 'users') {
  // На странице управления пользователями
}
```

### Использование на странице потока

```tsx
<div>
  <h1>Current page - {ctx.route.pathname}</h1>
  <p>Route id: {ctx.route.params?.name}</p>
</div>
```
