# ctx.openView()

Программное открытие указанного представления (выдвижная панель, диалоговое окно, встроенная страница и т. д.). Предоставляется `FlowModelContext` и используется в таких сценариях, как `JSBlock`, ячейки таблицы, рабочие процессы и др., для открытия настроенных представлений `ChildPage` или `PopupAction`.

## Сценарии использования

| Сценарий | Описание |
|------|------|
| **JSBlock** | Открытие диалогового окна с деталями/редактированием после нажатия кнопки, передача `filterByTk` текущей строки. |
| **Ячейка таблицы** | Отрисовка кнопки в ячейке, при нажатии на которую открывается диалоговое окно с деталями строки. |
| **Рабочий процесс / JSAction** | Открытие следующего представления или диалогового окна после успешного завершения операции. |
| **Поле ассоциации** | Открытие диалогового окна выбора/редактирования через `ctx.runAction('openView', params)`. |

> Примечание: `ctx.openView` доступен в среде RunJS при наличии контекста `FlowModel`. Если модель, соответствующая `uid`, не существует, будет автоматически создана и сохранена модель `PopupActionModel`.

## Сигнатура

```ts
openView(uid: string, options?: OpenViewOptions): Promise<void>
```

## Параметры

### uid

Уникальный идентификатор модели представления. Если он не существует, он будет автоматически создан и сохранен. Рекомендуется использовать стабильный UID, например `${ctx.model.uid}-detail`, чтобы конфигурацию можно было использовать повторно при многократном открытии одного и того же диалогового окна.

### Основные поля options

| Поле | Тип | Описание |
|------|------|------|
| `mode` | `drawer` / `dialog` / `embed` | Способ открытия: выдвижная панель (drawer), диалоговое окно (dialog) или встроенная страница (embed). По умолчанию `drawer`. |
| `size` | `small` / `medium` / `large` | Размер диалогового окна или выдвижной панели. По умолчанию `medium`. |
| `title` | `string` | Заголовок представления. |
| `params` | `Record<string, any>` | Произвольные параметры, передаваемые в представление. |
| `filterByTk` | `any` | Значение первичного ключа, используется для сценариев просмотра деталей или редактирования одной записи. |
| `sourceId` | `string` | ID исходной записи, используется в сценариях ассоциаций. |
| `dataSourceKey` | `string` | Источник данных. |
| `collectionName` | `string` | Имя коллекции. |
| `associationName` | `string` | Имя поля ассоциации. |
| `navigation` | `boolean` | Использовать ли навигацию по маршруту. Если переданы `defineProperties` или `defineMethods`, этот параметр принудительно устанавливается в `false`. |
| `preventClose` | `boolean` | Предотвращение закрытия. |
| `defineProperties` | `Record<string, PropertyOptions>` | Динамическая вставка свойств в модель внутри представления. |
| `defineMethods` | `Record<string, Function>` | Динамическая вставка методов в модель внутри представления. |

## Примеры

### Базовое использование: открытие выдвижной панели

```ts
const popupUid = `${ctx.model.uid}-detail`;
await ctx.openView(popupUid, {
  mode: 'drawer',
  size: 'medium',
  title: ctx.t('Детали'),
});
```

### Передача контекста текущей строки

```ts
const primaryKey = ctx.collection?.primaryKey || 'id';
await ctx.openView(`${ctx.model.uid}-1`, {
  mode: 'dialog',
  title: ctx.t('Детали строки'),
  params: {
    filterByTk: ctx.record?.[primaryKey],
    record: ctx.record,
  },
});
```

### Открытие через runAction

Когда модель настроена с действием `openView` (например, поля ассоциации или кликабельные поля), вы можете вызвать:

```ts
await ctx.runAction('openView', {
  navigation: false,
  mode: 'dialog',
  collectionName: 'users',
  filterByTk: ctx.record?.id,
});
```

### Вставка пользовательского контекста

```ts
await ctx.openView(`${ctx.model.uid}-edit`, {
  mode: 'drawer',
  filterByTk: ctx.record?.id,
  defineProperties: {
    onSaved: {
      get: () => () => ctx.resource?.refresh?.(),
      cache: false,
    },
  },
});
```

## Связь с ctx.viewer и ctx.view

| Назначение | Рекомендуемое использование |
|------|----------|
| **Открыть настроенное представление процесса** | `ctx.openView(uid, options)` |
| **Открыть произвольный контент (без процесса)** | `ctx.viewer.dialog()` / `ctx.viewer.drawer()` |
| **Операции с текущим открытым представлением** | `ctx.view.close()`, `ctx.view.inputArgs` |

`ctx.openView` открывает `FlowPage` (`ChildPageModel`), которая внутри отрисовывает полную страницу процесса; `ctx.viewer` открывает произвольный React-контент.

## Примечания

- Рекомендуется связывать `uid` с `ctx.model.uid` (например, `${ctx.model.uid}-xxx`), чтобы избежать конфликтов между несколькими блоками.
- При передаче `defineProperties` или `defineMethods` параметр `navigation` принудительно устанавливается в `false`, чтобы предотвратить потерю контекста после обновления страницы.
- Внутри диалогового окна `ctx.view` ссылается на текущий экземпляр представления, а `ctx.view.inputArgs` позволяет прочитать параметры, переданные при открытии.
