# ctx.exit()

Завершает выполнение текущего потока событий; последующие шаги не будут запущены. Обычно используется, когда бизнес-условия не соблюдены, пользователь отменил действие или произошла неисправимая ошибка.

## Сценарии использования

`ctx.exit()` обычно используется в следующих контекстах, где возможно выполнение JS:

| Сценарий | Описание |
|------|------|
| **Поток событий** | В потоках событий, инициируемых отправкой форм, нажатием кнопок и т. д., завершает последующие шаги, если условия не соблюдены. |
| **Правила связей** | В связях полей, фильтрах и т. д., завершает текущий поток событий при сбое валидации или необходимости пропустить выполнение. |
| **События действий** | В пользовательских действиях (например, подтверждение удаления, предварительная валидация перед сохранением), выход при отмене пользователем или сбое валидации. |

> Отличие от `ctx.exitAll()`: `ctx.exit()` завершает только текущий поток событий; другие потоки событий в рамках того же события не затрагиваются. `ctx.exitAll()` завершает текущий поток событий, а также любые последующие потоки событий в рамках того же события, которые еще не были выполнены.

## Определение типа

```ts
exit(): never;
```

Вызов `ctx.exit()` выбрасывает внутреннее исключение `FlowExitException`, которое перехватывается движком потока событий для остановки выполнения текущего потока. После вызова оставшиеся инструкции в текущем JS-коде не будут выполнены.

## Сравнение с ctx.exitAll()

| Метод | Область действия |
|------|----------|
| `ctx.exit()` | Завершает только текущий поток событий; последующие потоки событий не затрагиваются. |
| `ctx.exitAll()` | Завершает текущий поток событий и прерывает последующие потоки событий в рамках того же события, настроенные на **последовательное выполнение**. |

## Примеры

### Выход при отмене пользователем

```ts
// В модальном окне подтверждения: если пользователь нажимает «Отмена», поток событий завершается
if (!confirmed) {
  ctx.message.info('Операция отменена');
  ctx.exit();
}
```

### Выход при сбое валидации параметров

```ts
// Вывод сообщения и завершение при сбое валидации
if (!params.value || params.value.length < 3) {
  ctx.message.error('Недопустимые параметры, длина должна быть не менее 3');
  ctx.exit();
}
```

### Выход при несоблюдении бизнес-условий

```ts
// Завершение, если условия не соблюдены; последующие шаги не будут выполнены
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'Можно отправить только черновики' });
  ctx.exit();
}
```

### Выбор между ctx.exit() и ctx.exitAll()

```ts
// Нужно выйти только из текущего потока событий → используйте ctx.exit()
if (!params.valid) {
  ctx.message.error('Недопустимые параметры');
  ctx.exit();  // Другие потоки событий не затрагиваются
}

// Нужно завершить все последующие потоки событий в рамках текущего события → используйте ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'Недостаточно прав' });
  ctx.exitAll();  // И текущий, и последующие потоки событий в рамках этого события будут завершены
}
```

### Выход на основе выбора пользователя после подтверждения в модальном окне

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'Подтвердите удаление',
  content: 'Это действие нельзя отменить. Продолжить?',
});
if (!ok) {
  ctx.message.info('Отменено');
  ctx.exit();
}
```

## Примечания

- После вызова `ctx.exit()` последующий код в текущем JS не будет выполнен; рекомендуется перед вызовом объяснить причину пользователю с помощью `ctx.message`, `ctx.notification` или модального окна.
- В бизнес-коде обычно нет необходимости перехватывать `FlowExitException`; позвольте движку потока событий обработать его.
- Если вам нужно завершить все последующие потоки событий в рамках текущего события, используйте `ctx.exitAll()`.
