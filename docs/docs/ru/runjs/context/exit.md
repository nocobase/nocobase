# ctx.exit()

Останавливает текущий поток событий; последующие шаги в этом потоке не выполняются. Обычно используется, когда бизнес-условия не выполнены, пользователь отменил действие или произошла неисправимая ошибка.

## Сценарии использования

`ctx.exit()` применяется в контекстах, где выполняется JS, например:

| Сценарий | Описание |
|----------|----------|
| **Поток событий** | В потоках, запущенных отправкой формы, кликом кнопки и т. п., остановка при невыполнении условий |
| **Правила связывания** | Связывание полей и фильтров; остановка при ошибке валидации или необходимости пропуска |
| **События действий** | В пользовательских действиях (например, подтверждение удаления, проверка перед сохранением) выход при отмене или ошибке |

> Отличие от `ctx.exitAll()`: `ctx.exit()` останавливает только **текущий** поток событий. Другие потоки того же события продолжают выполняться. `ctx.exitAll()` дополнительно останавливает **последующие** потоки этого события.

## Тип

```ts
exit(): never;
```

Вызов `ctx.exit()` выбрасывает внутреннее исключение `FlowExitException`, которое обрабатывает движок потоков событий для остановки текущего потока. После вызова оставшийся код текущего JS не выполняется.

## Сравнение с ctx.exitAll()

| Метод | Область действия |
|-------|------------------|
| `ctx.exit()` | Останавливает только текущий поток; остальные не затрагиваются |
| `ctx.exitAll()` | Останавливает текущий поток и **все последующие** потоки того же события |

## Примеры

### Выход при отмене пользователем

```ts
// В модальном окне подтверждения: если пользователь нажимает «Отмена», поток событий завершается
if (!confirmed) {
  ctx.message.info('Cancelled');
  ctx.exit();
}
```

### Выход при ошибке валидации

```ts
// Вывод сообщения и завершение при сбое валидации
if (!params.value || params.value.length < 3) {
  ctx.message.error('Invalid: length must be at least 3');
  ctx.exit();
}
```

### Выход при невыполнении бизнес-условия

```ts
// Завершение, если условия не соблюдены; последующие шаги не будут выполнены
const record = ctx.model?.getValue?.();
if (!record || record.status !== 'draft') {
  ctx.notification.warning({ message: 'Only draft can be submitted' });
  ctx.exit();
}
```

### Когда использовать ctx.exit() и ctx.exitAll()

```ts
// Нужно остановить только этот поток -> ctx.exit()
if (!params.valid) {
  ctx.message.error('Invalid params');
  ctx.exit();
}

// Нужно остановить этот и все последующие потоки события -> ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'No permission' });
  ctx.exitAll();
}
```

### Выход после подтверждения в модальном окне

```ts
const ok = await ctx.modal?.confirm?.({
  title: 'Confirm delete',
  content: 'Cannot be recovered. Continue?',
});
if (!ok) {
  ctx.message.info('Cancelled');
  ctx.exit();
}
```

## Примечания

- После `ctx.exit()` остальной код текущего JS не выполняется; перед выходом обычно показывают `ctx.message`, `ctx.notification` или диалог с причиной.
- Обычно не нужно вручную обрабатывать `FlowExitException`; его обрабатывает движок потока.
- Чтобы остановить все последующие потоки этого события, используйте `ctx.exitAll()`.
