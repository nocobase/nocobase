# ctx.getVar()

Асинхронно считывает значения переменных из текущего контекста выполнения. Разрешение переменных совпадает с использованием `{{ctx.xxx}}` в SQL и шаблонах; обычно они поступают от текущего пользователя, текущей записи, параметров представления, контекста всплывающего окна и т. д.

## Области применения

| Сценарий | Описание |
|------|------|
| **JSBlock / JSField** | Получение информации о текущей записи, пользователе, ресурсе и т. д. для рендеринга или логики. |
| **Правила связки / Рабочий процесс** | Чтение `ctx.record`, `ctx.formValues` и т. д. для проверки условий. |
| **Формулы / Шаблоны** | Использование тех же правил разрешения переменных, что и в `{{ctx.xxx}}`. |

## Определение типа

```ts
getVar(path: string): Promise<any>;
```

| Параметр | Тип | Описание |
|------|------|------|
| `path` | `string` | Путь к переменной; **должен начинаться с `ctx.`**. Поддерживает точечную нотацию и индексы массивов. |

**Возвращаемое значение**: `Promise<any>`. Используйте `await` для получения разрешенного значения; возвращает `undefined`, если переменная не существует.

> Если передан путь, который не начинается с `ctx.`, будет выброшена ошибка: `ctx.getVar(path) expects an expression starting with "ctx.", got: "..."`.

## Часто используемые пути переменных

| Путь | Описание |
|------|------|
| `ctx.record` | Текущая запись (доступна, когда блок формы/деталей привязан к записи) |
| `ctx.record.id` | Первичный ключ текущей записи |
| `ctx.formValues` | Значения текущей формы (часто используется в правилах связки и рабочих процессах; в сценариях с формами предпочтительнее использовать `ctx.form.getFieldsValue()` для чтения в реальном времени) |
| `ctx.user` | Текущий вошедший пользователь |
| `ctx.user.id` | ID текущего пользователя |
| `ctx.user.nickname` | Никнейм текущего пользователя |
| `ctx.user.roles.name` | Имена ролей текущего пользователя (массив) |
| `ctx.popup.record` | Запись внутри всплывающего окна |
| `ctx.popup.record.id` | Первичный ключ записи внутри всплывающего окна |
| `ctx.urlSearchParams` | Параметры запроса URL (извлеченные из `?key=value`) |
| `ctx.token` | Текущий API-токен |
| `ctx.role` | Текущая роль |

## ctx.getVarInfos()

Получает **структурную информацию** (тип, заголовок, дочерние свойства и т. д.) разрешаемых переменных в текущем контексте, что упрощает поиск доступных путей. Возвращаемое значение представляет собой статическое описание на основе `meta` и не содержит фактических значений времени выполнения.

### Определение типа

```ts
getVarInfos(options?: { path?: string | string[]; maxDepth?: number }): Promise<Record<string, any>>;
```

В возвращаемом значении каждый ключ является путем к переменной, а значение — структурной информацией для этого пути (включая `type`, `title`, `properties` и т. д.).

### Параметры

| Параметр | Тип | Описание |
|------|------|------|
| `path` | `string \| string[]` | Путь обрезки; собирает структуру переменных только по этому пути. Поддерживает `'record'`, `'record.id'`, `'ctx.record'`, `'{{ ctx.record }}'`; массив означает объединение нескольких путей. |
| `maxDepth` | `number` | Максимальная глубина развертывания, по умолчанию `3`. Если `path` не указан, свойства верхнего уровня имеют `depth=1`. Если `path` указан, узел, соответствующий пути, имеет `depth=1`. |

### Пример

```ts
// Получить структуру переменных для record (развертывание до 3 уровней)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });

// Получить структуру для popup.record
const vars = await ctx.getVarInfos({ path: 'popup.record', maxDepth: 3 });

// Получить полную структуру переменных верхнего уровня (по умолчанию maxDepth=3)
const vars = await ctx.getVarInfos();
```

## Отличие от ctx.getValue

| Метод | Сценарий | Описание |
|------|----------|------|
| `ctx.getValue()` | Редактируемые поля, такие как JSField или JSItem | Синхронно получает значение **текущего поля**; требуется привязка к форме. |
| `ctx.getVar(path)` | Любой контекст RunJS | Асинхронно получает **любую переменную ctx**; путь должен начинаться с `ctx.`. |

В JSField используйте `getValue`/`setValue` для чтения/записи текущего поля; используйте `getVar` для доступа к другим переменным контекста (таким как `record`, `user`, `formValues`).

## Примечания

- **Путь должен начинаться с `ctx.`**: например, `ctx.record.id`, иначе будет выброшена ошибка.
- **Асинхронный метод**: вы должны использовать `await` для получения результата, например, `const id = await ctx.getVar('ctx.record.id')`.
- **Переменная не существует**: возвращает `undefined`. Вы можете использовать оператор `??` после вызова, чтобы установить значение по умолчанию: `(await ctx.getVar('ctx.user.nickname')) ?? 'Гость'`.
- **Значения формы**: `ctx.formValues` необходимо получать через `await ctx.getVar('ctx.formValues')`; они не доступны напрямую как свойство `ctx.formValues`. В контексте формы предпочтительнее использовать `ctx.form.getFieldsValue()` для получения актуальных значений в реальном времени.

## Примеры

### Получение ID текущей записи

```ts
const recordId = await ctx.getVar('ctx.record.id');
if (recordId) {
  ctx.message.info(`Текущая запись: ${recordId}`);
}
```

### Получение записи внутри всплывающего окна

```ts
const recordId = await ctx.getVar('ctx.popup.record.id');
if (recordId) {
  ctx.message.info(`Запись в текущем всплывающем окне: ${recordId}`);
}
```

### Чтение элементов массива

```ts
const roleNames = await ctx.getVar('ctx.user.roles.name');
// Возвращает массив имен ролей, например, ['admin', 'member']
```

### Установка значения по умолчанию

```ts
// У getVar нет параметра defaultValue; используйте ?? после вызова
const userName = (await ctx.getVar('ctx.user.nickname')) ?? 'Гость';
```

### Чтение значений полей формы

```ts
// И ctx.formValues, и ctx.form относятся к сценариям с формами; используйте getVar для чтения вложенных полей
const status = await ctx.getVar('ctx.formValues.status');
if (status === 'draft') {
  // ...
}
```

### Чтение параметров запроса URL

```ts
const id = await ctx.getVar('ctx.urlSearchParams.id'); // Соответствует ?id=xxx
```

### Изучение доступных переменных

```ts
// Получить структуру переменных для record (развертывание до 3 уровней)
const vars = await ctx.getVarInfos({ path: 'record', maxDepth: 3 });
// vars выглядит примерно так: { 'record.id': { type: 'string', title: 'id' }, ... }
```

## Связанные разделы

- [ctx.getValue()](./get-value.md) — синхронное получение значения текущего поля (только для JSField/JSItem и т. д.)
- [ctx.form](./form.md) — экземпляр формы, `ctx.form.getFieldsValue()` позволяет читать значения формы в реальном времени
- [ctx.model](./model.md) — модель, в которой находится текущий контекст выполнения
- [ctx.blockModel](./block-model.md) — родительский блок, в котором находится текущий JS
- [ctx.resource](./resource.md) — экземпляр ресурса в текущем контексте
- `{{ctx.xxx}}` в SQL / шаблонах — использует те же правила разрешения, что и `ctx.getVar('ctx.xxx')`
