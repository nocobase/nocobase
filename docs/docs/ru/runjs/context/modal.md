# ctx.modal

Удобный API на основе Ant Design Modal, используемый для активного открытия модальных окон (информационных сообщений, окон подтверждения и т. д.) в RunJS. Реализован через `ctx.viewer` / систему представлений.

## Сценарии использования

| Сценарий | Описание |
|------|------|
| **JSBlock / JSField** | Отображение результатов операции, сообщений об ошибках или повторных подтверждений после взаимодействия с пользователем. |
| **Рабочий процесс / События действий** | Всплывающее окно подтверждения перед отправкой; прерывание последующих шагов через `ctx.exit()`, если пользователь отменяет действие. |
| **Правила связей** | Всплывающие подсказки для пользователя при сбое валидации. |

> Примечание: `ctx.modal` доступен в средах RunJS с контекстом представления (например, JS-блоки внутри страницы, рабочие процессы и т. д.); он может отсутствовать в серверной части или в контекстах без пользовательского интерфейса. При вызове рекомендуется использовать опциональную цепочку (`ctx.modal?.confirm?.()`).

## Определение типов

```ts
modal: {
  info: (config: ModalConfig) => Promise<void>;
  success: (config: ModalConfig) => Promise<void>;
  error: (config: ModalConfig) => Promise<void>;
  warning: (config: ModalConfig) => Promise<void>;
  confirm: (config: ModalConfig) => Promise<boolean>;  // Возвращает true, если пользователь нажал ОК, и false, если отменил
};
```

`ModalConfig` соответствует конфигурации статических методов `Modal` в Ant Design.

## Основные методы

| Метод | Возвращаемое значение | Описание |
|------|--------|------|
| `info(config)` | `Promise<void>` | Модальное окно с информацией |
| `success(config)` | `Promise<void>` | Модальное окно успеха |
| `error(config)` | `Promise<void>` | Модальное окно ошибки |
| `warning(config)` | `Promise<void>` | Модальное окно предупреждения |
| `confirm(config)` | `Promise<boolean>` | Окно подтверждения; возвращает `true`, если пользователь нажал «ОК», и `false`, если «Отмена» |

## Параметры конфигурации

Аналогично Ant Design `Modal`, основные поля включают:

| Параметр | Тип | Описание |
|------|------|------|
| `title` | `ReactNode` | Заголовок |
| `content` | `ReactNode` | Содержимое |
| `okText` | `string` | Текст кнопки подтверждения |
| `cancelText` | `string` | Текст кнопки отмены (только для `confirm`) |
| `onOk` | `() => void \| Promise<void>` | Выполняется при нажатии «ОК» |
| `onCancel` | `() => void` | Выполняется при нажатии «Отмена» |

## Связь с ctx.message и ctx.openView

| Назначение | Рекомендуемое использование |
|------|----------|
| **Легкое временное уведомление** | `ctx.message`, исчезает автоматически |
| **Инфо/Успех/Ошибка/Предупреждение** | `ctx.modal.info` / `success` / `error` / `warning` |
| **Повторное подтверждение (требуется выбор пользователя)** | `ctx.modal.confirm`, используется вместе с `ctx.exit()` для управления потоком |
| **Сложные взаимодействия (формы, списки и т.д.)** | `ctx.openView` для открытия кастомного представления (страница/боковая панель/модальное окно) |

## Примеры

### Простое информационное окно

```ts
ctx.modal.info({
  title: 'Подсказка',
  content: 'Операция завершена',
});
```

### Окно подтверждения и управление потоком

```ts
const confirmed = await ctx.modal.confirm({
  title: 'Подтверждение удаления',
  content: 'Вы уверены, что хотите удалить эту запись?',
  okText: 'Подтвердить',
  cancelText: 'Отмена',
});
if (!confirmed) {
  ctx.exit();  // Прервать последующие шаги, если пользователь отменил действие
  return;
}
await ctx.runAction('destroy', { filterByTk: ctx.record?.id });
```

### Окно подтверждения с onOk

```ts
await ctx.modal.confirm({
  title: 'Подтверждение отправки',
  content: 'После отправки данные нельзя будет изменить. Продолжить?',
  async onOk() {
    await ctx.form.submit();
  },
});
```

### Сообщение об ошибке

```ts
try {
  await someOperation();
  ctx.modal.success({ title: 'Успех', content: 'Операция завершена' });
} catch (e) {
  ctx.modal.error({ title: 'Ошибка', content: e.message });
}
```

## Связанные разделы

- [ctx.message](./message.md): Легкое временное уведомление, исчезает автоматически.
- [ctx.exit()](./exit.md): Часто используется как `if (!confirmed) ctx.exit()` для прерывания процесса при отмене подтверждения пользователем.
- [ctx.openView()](./open-view.md): Открывает кастомное представление, подходит для сложных взаимодействий.
