# ctx.getModel()

Возвращает экземпляр модели (например, BlockModel, PageModel, ActionModel) из текущего движка или стека представлений по её `uid`. Используется в RunJS для доступа к другим моделям между блоками, страницами и всплывающими окнами.

Если нужна модель или блок только для текущего контекста выполнения, используйте `ctx.model` или `ctx.blockModel` вместо `ctx.getModel`.

## Сценарии использования

| Сценарий | Описание |
|----------|----------|
| **JS-блок / Действие JS** | Получить модель другого блока по известному `uid`; читать и изменять его `resource`, `form`, `setProps` и т. д. |
| **RunJS во всплывающем окне** | Получить модель страницы, открывшей всплывающее окно; передайте `searchInPreviousEngines: true` |
| **Пользовательские действия** | Найти форму или подмодель в панели настроек по `uid` по всему стеку представлений |

## Тип

```ts
getModel<T extends FlowModel = FlowModel>(
  uid: string,
  searchInPreviousEngines?: boolean
): T | undefined
```

## Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `uid` | `string` | Уникальный идентификатор целевой модели из конфигурации или при создании (например, `ctx.model.uid`) |
| `searchInPreviousEngines` | `boolean` | Необязательный, по умолчанию `false`. Если `true`, поиск идёт вверх по стеку представлений от текущего движка, чтобы найти модели в родительских представлениях (например, на странице, открывшей всплывающее окно) |

## Возвращаемое значение

- Если модель найдена, возвращается соответствующий подкласс `FlowModel` (например, `BlockModel`, `FormBlockModel`, `ActionModel`).
- Если модель не найдена, возвращается `undefined`.

## Область поиска

- **По умолчанию (`searchInPreviousEngines: false`)**: поиск только в **текущем движке** по `uid`. Во всплывающих и вложенных представлениях у каждого представления свой движок; по умолчанию ищется только текущее.
- **`searchInPreviousEngines: true`**: поиск от текущего движка по цепочке `previousEngine`; возвращается первое совпадение. Используйте это, когда RunJS во всплывающем окне должен получить модель на странице, которая его открыла.

## Примеры

### Получение другого блока и обновление

```ts
const block = ctx.getModel('list-block-uid');
if (block?.resource) {
  await block.resource.refresh();
}
```

### Доступ к модели страницы из всплывающего окна

```ts
// Доступ к блоку на странице, открывшей текущее всплывающее окно
const pageBlock = ctx.getModel('page-block-uid', true);
if (pageBlock) {
  pageBlock.rerender?.();
}
```

### Обновление другой модели и повторный рендер

```ts
const target = ctx.getModel('other-block-uid');
if (target) {
  target.setProps({ loading: true });
  target.rerender?.();
}
```

### Проверка на отсутствие модели

```ts
const model = ctx.getModel(someUid);
if (!model) {
  ctx.message.warning('Target model not found');
  return;
}
```
