# ctx.getValue()

В контекстах редактируемых полей — **Поле JS**, **Элемент JS** и др. — возвращает текущее значение поля. Используйте совместно с `ctx.setValue(v)` для двусторонней привязки к форме.

## Сценарии использования

| Сценарий | Описание |
|----------|----------|
| **Поле JS** | Чтение пользовательского ввода или текущего значения формы в редактируемых пользовательских полях |
| **Элемент JS** | Чтение текущего значения ячейки в редактируемых элементах таблицы или подтаблицы |
| **JS-столбец таблицы** | Чтение значения поля строки при рендере колонки |

> **Примечание**: `ctx.getValue()` доступен только в контекстах RunJS с привязкой к форме; в потоке событий, при связывании и т. п. (где нет привязки к полю) он отсутствует.

## Тип

```ts
getValue<T = any>(): T | undefined;
```

- **Возвращает**: текущее значение поля; тип зависит от типа элемента формы для поля. Может быть `undefined`, если поле не зарегистрировано или не заполнено.

## Разрешение значения

`ctx.getValue()` разрешается в следующем порядке:

1. **Состояние формы**: предпочтение текущему состоянию формы Ant Design.
2. **Запасной вариант**: если поля нет в форме, используется начальное значение поля или его свойства.

> Если форма ещё не отрендерена или поле не зарегистрировано, результат может быть `undefined`.

## Примеры

### Рендер на основе текущего значения

```ts
const current = ctx.getValue();
if (current == null || current === '') {
  ctx.render(<span>Введите что-нибудь</span>);
} else {
  ctx.render(<span>Текущее: {current}</span>);
}
```

### Двусторонняя привязка с setValue

```tsx
const { Input } = ctx.libs.antd;

// Чтение текущего значения в качестве значения по умолчанию
const defaultValue = ctx.getValue() ?? '';

ctx.render(
  <Input
    defaultValue={defaultValue}
    onChange={(e) => ctx.setValue(e.target.value)}
  />
);
```
