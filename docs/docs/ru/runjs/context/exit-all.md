# ctx.exitAll()

Останавливает текущий поток событий и все **последующие** потоки, которые были запущены в рамках той же операции отправки события. Используется, когда глобальная ошибка или проверка прав должна остановить выполнение всех потоков этого события.

## Сценарии использования

Используйте `ctx.exitAll()` в JS-контекстах, когда нужно **остановить и текущий поток, и все более поздние потоки того же события**:

| Сценарий | Описание |
|----------|----------|
| **Поток событий** | Основной поток завершился ошибкой (например, нет прав); остановка его и последующих потоков этого события |
| **Правила связывания** | При провале валидации связывания, если нужно остановить текущую и последующие цепочки связывания |
| **События действий** | Провал проверки перед действием (например, нет права на удаление); блокировка действия и дальнейших шагов |

> Отличие от `ctx.exit()`: `ctx.exit()` останавливает только текущий поток; `ctx.exitAll()` также останавливает **последующие** потоки этого события.

## Тип

```ts
exitAll(): never;
```

Вызов `ctx.exitAll()` выбрасывает внутреннее исключение `FlowExitAllException`, которое движок использует для остановки текущего потока и последующих потоков того же события. Оставшийся код текущего JS не выполняется.

## Сравнение с ctx.exit()

| Метод | Область действия |
|-------|------------------|
| `ctx.exit()` | Останавливает только текущий поток; последующие потоки выполняются |
| `ctx.exitAll()` | Останавливает текущий поток и **последующие** потоки того же события |

## Режим выполнения

- **Последовательный**: потоки одного события выполняются по очереди; после `ctx.exitAll()` в одном потоке оставшиеся потоки не запускаются.
- **Параллельный**: потоки выполняются параллельно; `ctx.exitAll()` в одном потоке не останавливает другие потоки, уже начавшие выполнение.

## Примеры

### Остановка всех потоков при ошибке прав

```ts
// Прервать основной поток и последующие потоки при недостаточности прав
if (!hasPermission(ctx)) {
  ctx.notification.error({ message: 'No permission' });
  ctx.exitAll();
}
```

### Остановка при провале глобальной предварительной проверки

```ts
// Пример: если перед удалением обнаружено, что связанные данные нельзя удалять,
// необходимо предотвратить основной поток и последующие действия
const canDelete = await checkDeletable(ctx.model?.getValue?.());
if (!canDelete) {
  ctx.message.error('Cannot delete: related data exists');
  ctx.exitAll();
}
```

### Когда использовать ctx.exit() и ctx.exitAll()

```ts
// Останавливаем только этот поток -> ctx.exit()
if (!params.valid) {
  ctx.message.error('Некорректные параметры');
  ctx.exit();
}

// Останавливаем этот и все последующие потоки -> ctx.exitAll()
if (!ctx.model?.context?.getPermission?.()) {
  ctx.notification.warning({ message: 'Нет прав доступа' });
  ctx.exitAll();
}
```

### Сообщение пользователю перед остановкой

```ts
if (!isValidInput(ctx.form?.getValues?.())) {
  ctx.message.warning('Please fix form errors first');
  ctx.exitAll();
}
```

## Примечания

- После `ctx.exitAll()` остальной код текущего JS не выполняется; перед остановкой стоит показать причину через `ctx.message`, `ctx.notification` или диалог.
- Обычно не нужно вручную обрабатывать `FlowExitAllException`; это делает движок.
- Для остановки только текущего потока используйте `ctx.exit()`.
- В параллельном режиме `ctx.exitAll()` останавливает только текущий поток, но не отменяет другие уже запущенные параллельные потоки.
