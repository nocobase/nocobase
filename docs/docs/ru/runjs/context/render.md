# ctx.render()

Рендерит React-элементы, HTML-строки или DOM-узлы в указанный контейнер. Если `container` не передан, по умолчанию рендеринг происходит в `ctx.element`, при этом автоматически наследуются контексты приложения, такие как ConfigProvider, темы и т. д.

## Сценарии использования

| Сценарий | Описание |
|------|------|
| **JSBlock** | Рендеринг пользовательского содержимого блоков (диаграммы, списки, карточки и т. д.) |
| **JSField / JSItem / JSColumn** | Рендеринг пользовательского отображения для редактируемых полей или столбцов таблицы |
| **Блок подробностей** | Настройка формата отображения полей на страницах подробностей |

> Примечание: `ctx.render()` требует наличия контейнера для рендеринга. Если `container` не передан и `ctx.element` не существует (например, в сценариях с чистой логикой без UI), будет выдана ошибка.

## Определение типа

```ts
render(
  vnode: React.ReactElement | Node | DocumentFragment | string,
  container?: Element | DocumentFragment
): ReactDOMClient.Root | null;
```

| Параметр | Тип | Описание |
|------|------|------|
| `vnode` | `React.ReactElement` \| `Node` \| `DocumentFragment` \| `string` | Содержимое для рендеринга |
| `container` | `Element` \| `DocumentFragment` (опционально) | Целевой контейнер для рендеринга, по умолчанию `ctx.element` |

**Возвращаемое значение**:

- При рендеринге **React-элемента**: Возвращает `ReactDOMClient.Root`, что позволяет в дальнейшем вызывать `root.render()` для обновлений.
- При рендеринге **HTML-строки** или **DOM-узла**: Возвращает `null`.

## Описание типов vnode

| Тип | Поведение |
|------|------|
| `React.ReactElement` (JSX) | Рендерится с использованием React `createRoot`, предоставляя полные возможности React и автоматически наследуя контекст приложения. |
| `string` | Устанавливает `innerHTML` контейнера после очистки с помощью DOMPurify; любой существующий корень React будет предварительно размонтирован. |
| `Node` (Element, Text и т. д.) | Добавляется через `appendChild` после очистки контейнера; любой существующий корень React будет предварительно размонтирован. |
| `DocumentFragment` | Дочерние узлы фрагмента добавляются в контейнер; любой существующий корень React будет предварительно размонтирован. |

## Примеры

### Рендеринг React-элементов (JSX)

```tsx
const { Button, Card } = ctx.libs.antd;

ctx.render(
  <Card title={ctx.t('Заголовок')}>
    <Button type="primary" onClick={() => ctx.message.success(ctx.t('Нажато'))}>
      {ctx.t('Кнопка')}
    </Button>
  </Card>
);
```

### Рендеринг HTML-строк

```ts
ctx.render('<h1>Hello World</h1>');

// Использование ctx.t для интернационализации
ctx.render('<div style="padding:16px">' + ctx.t('Содержимое') + '</div>');

// Условный рендеринг
ctx.render(hasData ? `<ul>${items.map(i => `<li>${i}</li>`).join('')}</ul>` : '<span style="color:#999">' + ctx.t('No data') + '</span>');
```

### Рендеринг DOM-узлов

```ts
const div = document.createElement('div');
div.textContent = 'Hello World';
ctx.render(div);

// Сначала рендерим пустой контейнер, затем передаем его сторонней библиотеке (например, ECharts) для инициализации
const container = document.createElement('div');
container.style.width = '100%';
container.style.height = '400px';
ctx.render(container);
const chart = echarts.init(container);
chart.setOption({ ... });
```

### Указание пользовательского контейнера

```ts
// Рендеринг в конкретный DOM-элемент
const customEl = document.getElementById('my-container');
ctx.render(<div>Содержимое</div>, customEl);
```

### Повторные вызовы заменяют содержимое

```ts
// Второй вызов заменит существующее содержимое в контейнере
ctx.render(<div>Первый</div>);
ctx.render(<div>Второй</div>);  // Будет отображено только «Второй»
```

## Важные замечания

- **Многократные вызовы заменяют содержимое**: Каждый вызов `ctx.render()` заменяет существующее содержимое в контейнере, а не добавляет к нему.
- **Безопасность HTML-строк**: Передаваемый HTML очищается с помощью DOMPurify для снижения рисков XSS, однако все равно рекомендуется избегать конкатенации ненадежных пользовательских данных.
- **Не манипулируйте ctx.element напрямую**: `ctx.element.innerHTML` устарел; вместо него следует единообразно использовать `ctx.render()`.
- **Передавайте container, если значение по умолчанию отсутствует**: В сценариях, где `ctx.element` имеет значение `undefined` (например, внутри модулей, загруженных через `ctx.importAsync`), необходимо явно передать `container`.

## Связанные разделы

- [ctx.element](./element.md) — контейнер для рендеринга по умолчанию, используется, когда в `ctx.render()` не передан container.
- [ctx.libs](./libs.md) — встроенные библиотеки, такие как React и Ant Design, используемые для рендеринга JSX.
- [ctx.importAsync()](./import-async.md) — используется в сочетании с `ctx.render()` после загрузки внешних библиотек React или компонентов по требованию.
