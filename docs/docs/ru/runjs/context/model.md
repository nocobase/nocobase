# ctx.model

Экземпляр `FlowModel`, в котором находится текущий контекст выполнения RunJS. Он служит точкой входа по умолчанию для таких сценариев, как JSBlock, JSField и JSAction. Конкретный тип меняется в зависимости от контекста: это может быть подкласс, такой как `BlockModel`, `ActionModel` или `JSEditableFieldModel`.

## Применимые сценарии

| Сценарий | Описание |
|------|------|
| **JSBlock** | `ctx.model` — это модель текущего блока. Вы можете получить доступ к `resource`, `collection`, `setProps` и т. д. |
| **JSField / JSItem / JSColumn** | `ctx.model` — это модель поля. Вы можете получить доступ к `setProps`, `dispatchEvent` и т. д. |
| **События действий / ActionModel** | `ctx.model` — это модель действия. Вы можете читать/записывать параметры шагов, отправлять события и т. д. |

> Подсказка: Если вам нужно получить доступ к **родительскому блоку, содержащему текущий JS** (например, блок формы или таблицы), используйте `ctx.blockModel`. Для доступа к **другим моделям** используйте `ctx.getModel(uid)`.

## Определение типа

```ts
model: FlowModel;
```

`FlowModel` является базовым классом. Во время выполнения это экземпляр различных подклассов (таких как `BlockModel`, `FormBlockModel`, `TableBlockModel`, `JSEditableFieldModel`, `ActionModel` и т. д.). Доступные свойства и методы зависят от конкретного типа.

## Общие свойства

| Свойство | Тип | Описание |
|------|------|------|
| `uid` | `string` | Уникальный идентификатор модели. Может использоваться для `ctx.getModel(uid)` или привязки UID всплывающего окна. |
| `collection` | `Collection` | Коллекция, привязанная к текущей модели (существует, когда блок или поле привязаны к данным). |
| `resource` | `Resource` | Связанный экземпляр ресурса, используемый для обновления, получения выбранных строк и т. д. |
| `props` | `object` | Конфигурация UI/поведения модели. Можно обновить с помощью `setProps`. |
| `subModels` | `Record<string, FlowModel>` | Коллекция дочерних моделей (например, поля внутри формы, колонки внутри таблицы). |
| `parent` | `FlowModel` | Родительская модель (если есть). |

## Общие методы

| Метод | Описание |
|------|------|
| `setProps(partialProps: any): void` | Обновляет конфигурацию модели и инициирует повторный рендеринг (например, `ctx.model.setProps({ loading: true })`). |
| `dispatchEvent(eventName: string, payload?: any, options?: any): Promise<any[]>` | Отправляет событие модели, запуская рабочие процессы, настроенные для этой модели и прослушивающие данное имя события. Необязательный `payload` передается в обработчик рабочего процесса; `options.debounce` включает устранение дребезга. |
| `getStepParams?.(flowKey, stepKey)` | Читает параметры шага потока конфигурации (используется в панелях настроек, пользовательских действиях и т. д.). |
| `setStepParams?.(flowKey, stepKey, params)` | Записывает параметры шага потока конфигурации. |

## Связь с ctx.blockModel и ctx.getModel

| Потребность | Рекомендуемое использование |
|------|----------|
| **Модель текущего контекста выполнения** | `ctx.model` |
| **Родительский блок текущего JS** | `ctx.blockModel`, часто используется для доступа к `resource`, `form` или `collection`. |
| **Получить любую модель по UID** | `ctx.getModel(uid)` или `ctx.getModel(uid, true)` (поиск по всем стекам представлений). |

В JSField `ctx.model` — это модель поля, а `ctx.blockModel` — это блок формы или таблицы, содержащий это поле.

## Примеры

### Обновление статуса блока/действия

```ts
ctx.model.setProps({ loading: true });
await doSomething();
ctx.model.setProps({ loading: false });
```

### Отправка событий модели

```ts
// Отправка события для запуска рабочего процесса, настроенного на этой модели и прослушивающего это имя события
await ctx.model.dispatchEvent('remove');

// Если передан payload, он будет доступен в ctx.inputArgs обработчика рабочего процесса
await ctx.model.dispatchEvent('customEvent', { id: 123 });
```

### Использование UID для привязки всплывающего окна или кросс-модельного доступа

```ts
const myUid = ctx.model.uid;
// В конфигурации всплывающего окна можно передать openerUid: myUid для установления связи
const other = ctx.getModel('other-block-uid');
if (other) other.rerender?.();
```
