# ctx.router

Экземпляр роутера на базе React Router, используемый для программной навигации в RunJS. Обычно используется в сочетании с `ctx.route` и `ctx.location`.

## Сценарии использования

| Сценарий | Описание |
|------|------|
| **JSBlock / JSField** | Переход на страницы подробностей, списков или по внешним ссылкам после нажатия кнопки. |
| **Правила связки / Поток событий** | Выполнение `navigate` для перехода к списку или деталям после успешной отправки формы или передача `state` на целевую страницу. |
| **JSAction / Обработка событий** | Выполнение навигации в логике обработки отправки форм, кликов по ссылкам и т. д. |
| **Навигация по представлениям** | Обновление URL через `navigate` при переключении стека внутренних представлений. |

> Примечание: `ctx.router` доступен только в средах RunJS с контекстом маршрутизации (например, JSBlock на странице, страницы Flow, потоки событий и т. д.); он может иметь значение `null` в чисто серверных контекстах или контекстах без маршрутизации (например, в рабочих процессах).

## Определение типа

```typescript
router: Router
```

`Router` основан на `@remix-run/router`. В RunJS операции навигации, такие как переход, возврат назад и обновление страницы, реализуются через `ctx.router.navigate()`.

## Методы

### ctx.router.navigate()

Выполняет переход по целевому пути или выполняет действие «назад»/«обновить».

**Сигнатура:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**Параметры:**

- `to`: Целевой путь (string), относительная позиция в истории (number, например, `-1` для возврата назад) или `null` (для обновления текущей страницы).
- `options`: Необязательная конфигурация.
  - `replace?: boolean`: Заменять ли текущую запись в истории (по умолчанию `false`, что создает новую запись через push).
  - `state?: any`: Состояние, передаваемое целевому маршруту. Эти данные не отображаются в URL и могут быть получены через `ctx.location.state` на целевой странице. Подходит для конфиденциальной информации, временных данных или информации, которую нежелательно размещать в URL.

## Примеры

### Базовая навигация

```ts
// Переход к списку пользователей (создает новую запись в истории, можно вернуться назад)
ctx.router.navigate('/admin/users');

// Переход на страницу подробностей
ctx.router.navigate(`/admin/users/${recordId}`);
```

### Замена истории (без создания новой записи)

```ts
// Перенаправление на главную страницу после входа; при нажатии «назад» пользователь не вернется на страницу логина
ctx.router.navigate('/admin', { replace: true });

// Замена текущей страницы страницей подробностей после успешной отправки формы
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### Передача state

```ts
// Передача данных при переходе; целевая страница получает их через ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### Назад и обновление

```ts
// На одну страницу назад
ctx.router.navigate(-1);

// На две страницы назад
ctx.router.navigate(-2);

// Обновить текущую страницу
ctx.router.navigate(null);
```

## Взаимосвязь с ctx.route и ctx.location

| Назначение | Рекомендуемое использование |
|------|----------|
| **Навигация / Переход** | `ctx.router.navigate(path)` |
| **Чтение текущего пути** | `ctx.route.pathname` или `ctx.location.pathname` |
| **Чтение state, переданного при переходе** | `ctx.location.state` |
| **Чтение параметров маршрута** | `ctx.route.params` |

`ctx.router` отвечает за «действия навигации», в то время как `ctx.route` и `ctx.location` отвечают за «текущее состояние маршрута».

## Примечания

- `navigate(path)` по умолчанию добавляет новую запись в историю, позволяя пользователю вернуться назад с помощью кнопки браузера.
- `replace: true` заменяет текущую запись в истории без добавления новой, что подходит для таких сценариев, как перенаправление после входа в систему или переход после успешной отправки данных.
- **О параметре `state`**:
  - Данные, переданные через `state`, не отображаются в URL, что делает их подходящими для конфиденциальных или временных данных.
  - Доступ к ним можно получить через `ctx.location.state` на целевой странице.
  - `state` сохраняется в истории браузера и остается доступным при навигации вперед/назад.
  - `state` будет потерян после жесткой перезагрузки страницы.
