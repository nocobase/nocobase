# ctx.router

Экземпляр маршрутизатора на базе React Router; используется для программной навигации в RunJS. Работает вместе с `ctx.route` и `ctx.location`.

## Сценарии использования

| Сценарий | Описание |
|----------|----------|
| **JS-блок / Поле JS** | Переход по кнопке к деталям, списку или внешней ссылке |
| **Связывание / поток событий** | После успешной отправки выполнить `navigate` к списку или деталям или передать `state` |
| **Действие JS / обработчик событий** | Навигация после отправки формы, клика по ссылке и т. д. |
| **Навигация по представлениям** | Обновление URL при переключении представления |

> `ctx.router` доступен только в контекстах RunJS с маршрутизатором (например, JS-блок на странице, страница потока, поток событий). В чисто серверных или контекстах без маршрутизации (например, рабочий процесс) может отсутствовать.

## Тип

```typescript
router: Router
```

`Router` берётся из `@remix-run/router`; для навигации назад/вперёд/обновления используйте `ctx.router.navigate()`.

## Методы

### ctx.router.navigate()

Переход по пути, назад по истории или обновление текущей страницы.

**Сигнатура:**

```typescript
navigate(to: string | number | null, options?: RouterNavigateOptions): Promise<void>
```

**Параметры:**

- `to`: целевой путь (string), относительная позиция в истории (number, например `-1` для "назад"), или `null` (обновить текущую страницу)
- `options`:
  - `replace?: boolean`: заменить текущую запись в истории (по умолчанию `false`, то есть добавление в стек — push)
  - `state?: any`: данные, передаваемые в целевой маршрут. Не попадают в URL; читаются на целевой странице через `ctx.location.state`. Подходит для чувствительных или временных данных.

## Примеры

### Базовая навигация

```ts
// Переход к списку пользователей (создает новую запись в истории, можно вернуться назад)
ctx.router.navigate('/admin/users');

// Переход на страницу подробностей
ctx.router.navigate(`/admin/users/${recordId}`);
```

### Замена записи в истории

```ts
// Перенаправление на главную страницу после входа; при нажатии «назад» пользователь не вернется на страницу логина
ctx.router.navigate('/admin', { replace: true });

// Замена текущей страницы страницей подробностей после успешной отправки формы
ctx.router.navigate(`/admin/users/${newId}`, { replace: true });
```

### Передача state

```ts
// Передача данных при переходе; целевая страница получает их через ctx.location.state
ctx.router.navigate('/admin/users/123', { 
  state: { from: 'dashboard', tab: 'profile' } 
});
```

### Назад и обновление

```ts
// На одну страницу назад
ctx.router.navigate(-1);

// На две страницы назад
ctx.router.navigate(-2);

// Обновить текущую страницу
ctx.router.navigate(null);
```

## Связь с ctx.route и ctx.location

| Задача | Рекомендуемый API |
|--------|-------------------|
| **Навигация** | `ctx.router.navigate(path)` |
| **Текущий путь** | `ctx.route.pathname` или `ctx.location.pathname` |
| **State после навигации** | `ctx.location.state` |
| **Параметры маршрута** | `ctx.route.params` |

`ctx.router` отвечает за "переходы", а `ctx.route` и `ctx.location` описывают "состояние текущего маршрута".

## Примечания

- `navigate(path)` по умолчанию добавляет новую запись в историю; пользователь сможет вернуться кнопкой «Назад».
- `replace: true` заменяет текущую запись; это удобно после перенаправления при входе, после успешной отправки и т. д.
- **О параметре `state`**:
  - Данные, переданные через `state`, не отображаются в URL, что делает их подходящими для конфиденциальных или временных данных.
  - Доступ к ним можно получить через `ctx.location.state` на целевой странице.
  - `state` сохраняется в истории браузера и остается доступным при навигации вперед/назад.
  - `state` будет потерян после жесткой перезагрузки страницы.
