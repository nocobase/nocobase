# ctx.on()

Подписка на события контекста в RunJS (например, изменение значения поля, изменение свойств, обновление ресурса). События сопоставляются с пользовательскими DOM-событиями на `ctx.element` или с внутренней шиной событий на `ctx.resource`.

## Сценарии использования

| Сценарий | Описание |
|----------|----------|
| **Поле JS / JSEditableFieldModel** | Синхронизация UI при внешнем изменении значения поля (форма, связывание); двусторонняя привязка |
| **JS-блок / Элемент JS / JS-столбец таблицы** | Прослушивание пользовательских событий контейнера; реакция на изменения данных и состояния |
| **Ресурс** | Прослушивание обновления, сохранения и т. д.; запуск логики после обновления данных |

## Тип

```ts
on(eventName: string, handler: (event?: any) => void): void;
```

## Частые события

| Событие | Описание | Источник |
|---------|----------|----------|
| `js-field:value-change` | Значение поля изменилось извне (связывание формы, значение по умолчанию) | `CustomEvent` на `ctx.element`; `ev.detail` = новое значение |
| `resource:refresh` | Данные ресурса обновлены | Шина событий `ctx.resource` |
| `resource:saved` | Сохранение ресурса завершено | Шина событий `ctx.resource` |

> События с префиксом `resource:` используют `ctx.resource.on`; остальные обычно используют DOM-события на `ctx.element`, если он доступен.

## Примеры

### Двусторонняя привязка (React useEffect и очистка)

```tsx
React.useEffect(() => {
  const handler = (ev) => setValue(ev?.detail ?? '');
  ctx.on?.('js-field:value-change', handler);
  return () => {
    ctx.off?.('js-field:value-change', handler);
  };
}, []);
```

### Нативный DOM, если ctx.on недоступен

```ts
// Если ctx.on не предоставлен, вы можете использовать ctx.element напрямую
const handler = (ev) => {
  if (selectEl) selectEl.value = String(ev?.detail ?? '');
};
ctx.element?.addEventListener('js-field:value-change', handler);
// Очистка: ctx.element?.removeEventListener('js-field:value-change', handler);
```

### Обновление UI после обновления ресурса

```ts
ctx.resource?.on('refresh', () => {
  const data = ctx.resource?.getData?.();
  // Обновляем рендер на основе данных
});
```

## Вместе с ctx.off

- Отписывайтесь через [ctx.off](./off.md), когда это необходимо, чтобы избежать утечек и дублирующихся обработчиков.
- В React обычно вызывают `ctx.off` в функции очистки `useEffect`.
- `ctx.off` может отсутствовать; используйте опциональную цепочку: `ctx.off?.('eventName', handler)`.

## Примечания

1. **Пара с off**: для каждого `ctx.on(eventName, handler)` должен быть соответствующий `ctx.off(eventName, handler)` с той же ссылкой `handler`.
2. **Жизненный цикл**: удаляйте слушатели перед размонтированием или уничтожением контекста, чтобы избежать утечек.
3. **Поддержка событий**: разные типы контекстов поддерживают разные события; см. документацию компонента.
