# ctx.on()

Подписка на события контекста (такие как изменение значений полей, изменение свойств, обновление ресурсов и т. д.) в RunJS. События сопоставляются с пользовательскими DOM-событиями на `ctx.element` или внутренними событиями шины событий `ctx.resource` в зависимости от их типа.

## Варианты использования

| Сценарий | Описание |
|------|----------|
| **JSField / JSEditableField** | Отслеживание изменений значений полей из внешних источников (формы, связи и т. д.) для синхронного обновления UI, обеспечивая двустороннюю привязку. |
| **JSBlock / JSItem / JSColumn** | Прослушивание пользовательских событий в контейнере для реагирования на изменения данных или состояния. |
| **Связанные с resource** | Отслеживание событий жизненного цикла ресурса, таких как обновление или сохранение, для выполнения логики после обновления данных. |

## Определение типа

```ts
on(eventName: string, handler: (event?: any) => void): void;
```

## Общие события

| Имя события | Описание | Источник события |
|-------------|----------|------------------|
| `js-field:value-change` | Значение поля изменено извне (например, через связи в форме или обновление значения по умолчанию) | `CustomEvent` на `ctx.element`, где `ev.detail` — новое значение |
| `resource:refresh` | Данные ресурса были обновлены | Шина событий `ctx.resource` |
| `resource:saved` | Сохранение ресурса завершено | Шина событий `ctx.resource` |

> Правила сопоставления событий: события с префиксом `resource:` проходят через `ctx.resource.on`, остальные обычно проходят через DOM-события на `ctx.element` (если он существует).

## Примеры

### Двусторонняя привязка поля (React useEffect + очистка)

```tsx
React.useEffect(() => {
  const handler = (ev) => setValue(ev?.detail ?? '');
  ctx.on?.('js-field:value-change', handler);
  return () => {
    ctx.off?.('js-field:value-change', handler);
  };
}, []);
```

### Нативное прослушивание DOM (альтернатива, если ctx.on недоступен)

```ts
// Если ctx.on не предоставлен, вы можете использовать ctx.element напрямую
const handler = (ev) => {
  if (selectEl) selectEl.value = String(ev?.detail ?? '');
};
ctx.element?.addEventListener('js-field:value-change', handler);
// При очистке: ctx.element?.removeEventListener('js-field:value-change', handler);
```

### Обновление UI после обновления ресурса

```ts
ctx.resource?.on('refresh', () => {
  const data = ctx.resource?.getData?.();
  // Обновление рендеринга на основе данных
});
```

## Взаимодействие с ctx.off

- Слушатели, зарегистрированные с помощью `ctx.on`, должны быть удалены в соответствующее время через [ctx.off](./off.md), чтобы избежать утечек памяти или повторных срабатываний.
- В React `ctx.off` обычно вызывается внутри функции очистки `useEffect`.
- `ctx.off` может отсутствовать; рекомендуется использовать опциональную цепочку: `ctx.off?.('eventName', handler)`.

## Примечания

1. **Парная отмена**: Каждому вызову `ctx.on(eventName, handler)` должен соответствовать вызов `ctx.off(eventName, handler)`, при этом ссылка на `handler` должна быть идентичной.
2. **Жизненный цикл**: Удаляйте слушатели до размонтирования компонента или уничтожения контекста, чтобы предотвратить утечки памяти.
3. **Доступность событий**: Разные типы контекста поддерживают разные события. Пожалуйста, обращайтесь к документации конкретных компонентов для получения подробной информации.
