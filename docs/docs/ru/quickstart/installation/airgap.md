#Интранет-установка

Если ваш сервер не может получить доступ к общедоступной сети, метод установки требует, чтобы вы заранее подготовили образы, зависимости и пакеты плагинов, необходимые для автономного использования. По умолчанию рекомендуется сначала использовать Docker, который имеет самый короткий путь и его легче всего воспроизвести.

## Рекомендация по умолчанию: подготовьте образ Docker в автономном режиме.

На машине, которая имеет доступ к общедоступной сети, сначала извлеките образ приложения и образ базы данных:

```bash
docker pull registry.cn-shanghai.aliyuncs.com/nocobase/nocobase:latest-full
docker pull registry.cn-shanghai.aliyuncs.com/nocobase/postgres:16
```

Затем экспортируйте как автономный файл:

```bash
docker save -o nocobase-app.tar \
  registry.cn-shanghai.aliyuncs.com/nocobase/nocobase:latest-full

docker save -o nocobase-postgres.tar \
  registry.cn-shanghai.aliyuncs.com/nocobase/postgres:16
```

Если вам все же нужны коммерческие плагины, рекомендуется также подготовить пакет плагинов во внешней сетевой среде, а затем вместе перенести его во внутреннюю сеть.

## Скопируйте файл на сервер интрасети

Подготовьте как минимум следующие документы:

- `nocobase-app.tar`
- `nocobase-postgres.tar`
- `docker-compose.yml`
- `.env` или ваши собственные инструкции по развертыванию.

## Импортируем изображение на сервер интрасети

```bash
docker load -i nocobase-app.tar
docker load -i nocobase-postgres.tar
```

## Запустить приложение

После подготовки `docker-compose.yml` приступайте непосредственно:

```bash
docker compose up -d
docker compose logs -f app
```

Если вы еще не написали файл Compose, сначала прочитайте [Установка через Docker Compose](./docker-compose.md) и сохраните примеры там локально.

## Что делать, если вы не можете использовать Docker

Если Docker невозможно использовать в вашей интрасети, вы также можете использовать `create-nocobase-app`, чтобы создать полный проект во внешней сетевой среде, установить зависимости и упаковать его, а затем скопировать весь проект на сервер интрасети.

Этот путь будет длиннее, но более практичным в средах без возможностей контейнеров. Общий процесс обычно таков:

1. Создайте проект во внешней сетевой среде и установите зависимости.
2. Упакуйте каталог проекта.
3. Скопируйте на сервер интрасети.
4. Разархивируйте файл в интрасети, заполните `.env` и запустите приложение.
