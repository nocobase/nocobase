# Серверные логи, журналы аудита и история записей

## Серверные логи

### Системные логи

> См. [Системные логи](./index.md#системные-логи)

- Фиксируют информацию о работе прикладной системы, отслеживают цепочки выполнения кода и позволяют выявлять ошибки выполнения и другие исключения.
- Логи разделены по уровням важности и классифицированы по функциональным модулям.
- Выводятся в терминал или сохраняются в виде файлов.
- В основном используются для диагностики и устранения неполадок, возникающих во время работы системы.

### Логи запросов

> См. [Логи запросов](./index.md#логи-запросов)

- Фиксируют информацию о HTTP API запросах и ответах, уделяя основное внимание ID запроса, пути API, заголовкам, кодам состояния ответа и времени выполнения.
- Выводятся в терминал или сохраняются в виде файлов.
- В основном используются для отслеживания вызовов и производительности API.

## Журналы аудита

> См. [Журналы аудита](/security/audit-logger/index.md)

- Фиксируют действия пользователей (или API) с системными ресурсами, уделяя основное внимание типу ресурса, целевому объекту, типу операции, информации о пользователе и статусу операции.
- Для более точного отслеживания действий пользователей и их результатов параметры запросов и ответы сохраняются в виде метаданных. Эта информация частично пересекается с логами запросов, но не идентична им — например, в обычных логах запросов часто не сохраняется полное тело запроса.
- Параметры и ответы запросов **не являются эквивалентом** моментальных снимков (snapshots) ресурсов. Они позволяют понять, какие изменения были внесены логикой кода, но не дают точного представления о том, какими были данные в таблице до изменения. Поэтому их нельзя использовать для контроля версий или восстановления данных после ошибочных операций.
- Хранятся в виде файлов и таблиц базы данных.

## История записей

> См. [История записей](/record-history/index.md)

- Фиксирует историю изменений содержимого данных.
- Основная записываемая информация включает тип ресурса, объект ресурса, тип операции, измененные поля, а также значения до и после изменений.
- Может использоваться для сравнения данных.
- Хранится в таблицах базы данных.
