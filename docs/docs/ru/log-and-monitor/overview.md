# Журналы сервера, журналы аудита и история записей

## Журналы сервера

### Системные журналы

> См. [Системные журналы](#)

- Записывают информацию о работе системы, отслеживают цепочки выполнения кода и помогают выявлять исключения или ошибки во время выполнения.
- Журналы классифицируются по уровням серьезности и функциональным модулям.
- Выводятся в терминал или хранятся в виде файлов.
- В основном используются для диагностики и устранения неполадок системы во время ее работы.

### Журналы запросов

> См. [Журналы запросов](#)

- Записывают детали HTTP API запросов и ответов, уделяя особое внимание идентификатору запроса, пути API, заголовкам, коду состояния ответа и времени выполнения.
- Выводятся в терминал или хранятся в виде файлов.
- В основном используются для отслеживания вызовов API и производительности их выполнения.

## Журналы аудита

> См. [Журналы аудита](/security/audit-logger/index.md)

- Записывают действия пользователей (или API) с системными ресурсами, уделяя особое внимание типу ресурса, целевому объекту, типу операции, информации о пользователе и статусу операции.
- Для более точного отслеживания действий пользователей и полученных результатов, параметры запросов и ответы сохраняются в виде метаданных. Эта информация частично совпадает с журналами запросов, но не идентична — например, журналы запросов обычно не включают полные тела запросов.
- Параметры запросов и ответы **не эквивалентны** снимкам данных. Они могут показать, какие операции были выполнены, но не точное состояние данных до изменения, поэтому их нельзя использовать для контроля версий или восстановления данных после ошибочных операций.
- Хранятся как в файлах, так и в таблицах базы данных.

## История записей

> См. [История записей](/record-history/index.md)

- Записывает **историю изменений** содержимого данных.
- Отслеживает тип ресурса, объект ресурса, тип операции, измененные поля и значения до/после изменения.
- Полезно для **сравнения данных и аудита**.
- Хранится в таблицах базы данных.
