# Правила связывания действий

## Введение

Правила связывания действий позволяют пользователям динамически управлять состоянием действия (например, показывать, включать, скрывать, отключать и т.д.) на основе заданных условий. Настроив эти правила, пользователи могут связать поведение кнопок действий с текущей записью, ролью пользователя или контекстными данными.

![20251029150224](https://static-docs.nocobase.com/20251029150224.png)

## Как использовать

Когда условие выполнено (по умолчанию проходит, если условие не задано), оно запускает выполнение настроек свойств или JavaScript. Поддерживаются константы и переменные в условном выражении.

![20251030224601](https://static-docs.nocobase.com/20251030224601.png)

Правило поддерживает изменение свойств кнопки.

![20251029150452](https://static-docs.nocobase.com/20251029150452.png)

## Константы

Пример: платёжные заказы нельзя редактировать.

![20251029150638](https://static-docs.nocobase.com/20251029150638.png)

## Переменные

### Системные переменные

![20251029150014](https://static-docs.nocobase.com/20251029150014.png)

Пример 1: управляйте видимостью кнопки в зависимости от типа текущего устройства.

![20251029151057](https://static-docs.nocobase.com/20251029151057.png)

Пример 2: кнопка массового обновления в заголовке таблицы блока заказов доступна только роли Admin; другие роли не могут выполнять это действие.

![20251029151209](https://static-docs.nocobase.com/20251029151209.png)

### Контекстные переменные

Пример: кнопка «Добавить» в возможностях заказа включается только тогда, когда статус заказа — «Ожидает платежа» или «Черновик». В остальных статусах кнопка будет отключена.
