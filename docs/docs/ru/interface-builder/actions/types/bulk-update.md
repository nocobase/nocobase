# Массовое обновление

## Введение

Действие Массовое обновление используется, когда необходимо применить одинаковое обновление к группе записей. Перед выполнением массового обновления пользователь должен заранее определить логику присваивания полей для обновления. Эта логика будет применена ко всем выбранным записям, когда пользователь нажмет кнопку обновления.

### Данные для обновления

Выбранные/Все, по умолчанию — Выбранные.

![20251029200034](https://static-docs.nocobase.com/20251029200034.png)

### Присваивание полей

Настройте поля для массового обновления. Будут обновлены только те поля, которые заданы для присваивания.

Как показано на изображении, в таблице заказов настройте действие массового обновления, чтобы применить к выбранным данным статус «Pending Approval».

![20251029200109](https://static-docs.nocobase.com/20251029200109.png)

- [Кнопка редактировани](/interface-builder/actions/action-settings/edit-button): редактировать заголовок, тип и значок кнопки;
- [Правило связывания](/interface-builder/actions/action-settings/linkage-rule): динамически показывать/скрывать кнопку;
- [Подтверждение](/interface-builder/actions/action-settings/double-check)
