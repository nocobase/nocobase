# Компоненты полей связей

## Введение

Компоненты полей связей NocoBase разработаны, чтобы помочь вам более эффективно отображать и обрабатывать связанные данные. Независимо от типа связи, эти компоненты гибки и универсальны, позволяя вам выбирать и настраивать их в соответствии с вашими конкретными потребностями.

### Выпадающий список

Для всех полей связей, за исключением тех случаев, когда целевая коллекция является файловой, компонентом по умолчанию в режиме редактирования является выпадающий список. Опции выпадающего списка отображают значение поля заголовка, что удобно для быстрого выбора связанных данных, показывая ключевую информацию поля.

![20240429205659](https://static-docs.nocobase.com/20240429205659.png)

Подробнее см. [Выпадающий список](/interface-builder/fields/specific/select)

### Селектор данных

Селектор данных отображает информацию во всплывающем окне. Вы можете настроить поля, которые будут отображаться в селекторе (включая поля из вложенных связей), что позволяет более точно выбирать связанные данные.

![20240429210824](https://static-docs.nocobase.com/20240429210824.png)

Подробнее см. [Селектор данных](/interface-builder/fields/specific/picker)

### Вложенная форма

При работе со сложными связанными данными использование выпадающего списка или селектора данных может быть неудобным, так как вам придется часто открывать всплывающие окна. В таких случаях можно использовать вложенную форму. Она позволяет вам напрямую управлять полями связанной коллекции на текущей странице или в текущем блоке всплывающего окна, без необходимости постоянно открывать новые всплывающие окна, что делает рабочий процесс более плавным. Многоуровневые связи отображаются в виде вложенных форм.

![20251029122948](https://static-docs.nocobase.com/20251029122948.png)

Подробнее см. [Вложенная форма](/interface-builder/fields/specific/sub-form)

### Вложенная таблица

Вложенная таблица отображает записи связей «один-ко-многим» или «многие-ко-многим» в табличном формате. Она предоставляет четкий, структурированный способ отображения и управления связанными данными, а также поддерживает массовое создание новых данных или выбор существующих данных для связи.

![20251029123042](https://static-docs.nocobase.com/20251029123042.png)

Подробнее см. [Вложенная таблица](/interface-builder/fields/specific/sub-table)

### Вложенные детали

Вложенные детали — это компонент, соответствующий вложенной форме в режиме просмотра. Он поддерживает отображение данных с многоуровневыми вложенными связями.

![20251030213050](https://static-docs.nocobase.com/2025103050.png)

Подробнее см. [Вложенные детали](/interface-builder/fields/specific/sub-detail)

### Файловый менеджер

Файловый менеджер — это компонент поля связи, специально предназначенный для случаев, когда целевая коллекция отношения является файловой коллекцией.

![20240429222753](https://static-docs.nocobase.com/20240429222753.png)

Подробнее см. [Файловый менеджер](/interface-builder/fields/specific/file-manager)

### Заголовок

Компонент поля заголовка — это компонент поля связи, используемый в режиме просмотра. Настроив поле заголовка, вы можете настроить соответствующий компонент поля.

![20251030213327](https://static-docs.nocobase.com/20251030213327.png)

Подробнее см. [Заголовок](/interface-builder/fields/specific/title)
