# Компоненты поля связи

## Введение

Компоненты поля связи в NocoBase предназначены для того, чтобы помогать пользователям лучше отображать и обрабатывать связанные данные. Независимо от типа связи эти компоненты гибкие и универсальные, позволяя пользователям выбирать и настраивать их в соответствии с конкретными потребностями.

### Выпадающий список

Для всех полей связи, кроме случаев, когда целевая коллекция является файловой коллекцией, по умолчанию в режиме редактирования используется выпадающий список. Варианты выпадающего списка отображают значение поля заголовка, благодаря чему он подходит для сценариев, где связанные данные можно быстро выбрать, показывая ключевую информацию.

![20240429205659](https://static-docs.nocobase.com/20240429205659.png)

Подробнее см. [Выпадающий список](/interface-builder/fields/specific/select)

### Селектор

Селектор отображает данные во всплывающем модальном окне. Пользователи могут настроить поля, которые будут показываться в селекторе (включая поля из вложенных связей), чтобы точнее выбирать связанные данные.

![20240429210824](https://static-docs.nocobase.com/20240429210824.png)

Подробнее см. [Селектор](/interface-builder/fields/specific/picker)

### Подформа

Когда речь идет о более сложных данных связей, использование выпадающего списка или селектора может быть неудобным. В таких случаях пользователям приходится часто открывать всплывающие окна. Для подобных сценариев можно использовать подформу. Она позволяет пользователям напрямую поддерживать поля связанной коллекции на текущей странице или в текущем блоке всплывающего окна, не открывая новые окна снова и снова — рабочий процесс становится плавнее. Связи многоуровневого типа отображаются как вложенные формы.

![20251029122948](https://static-docs.nocobase.com/20251029122948.png)

Подробнее см. [Подформа](/interface-builder/fields/specific/sub-form)

### Подтаблица

Подтаблица отображает записи связей один-ко-многим или многие-ко-многим в табличном виде. Она обеспечивает понятный, структурированный способ отображения и управления связанными данными, а также поддерживает массовое создание новых данных или выбор существующих для установления связи.

![20251029123042](https://static-docs.nocobase.com/20251029123042.png)

Подробнее см. [Подтаблица](/interface-builder/fields/specific/sub-table)

### Подробности

Подробности — это соответствующий компонент подформы в режиме только для чтения. Он поддерживает отображение данных с вложенными многоуровневыми связями.

![20251030213050](https://static-docs.nocobase.com/20251030213050.png)

Подробнее см. [Подробности](/interface-builder/fields/specific/sub-detail)

### Менеджер файлов

Менеджер файлов — компонент поля связи, который используется специально, когда целевая коллекция связи является файловой коллекцией.

![20240429222753](https://static-docs.nocobase.com/20240429222753.png)

Подробнее см. [Менеджер файлов](/interface-builder/fields/specific/file-manager)

### Заголовок

Компонент поля заголовка — это компонент поля связи, используемый в режиме только для чтения. Настроив поле заголовка, вы сможете настроить соответствующий компонент поля.

![20251030213327](https://static-docs.nocobase.com/20251030213327.png)

Подробнее см. [Заголовок](/interface-builder/fields/specific/title)
