# Установка правил валидации

## Введение

Правила валидации используются для того, чтобы убедиться, что введённые пользователем данные соответствуют ожиданиям.

## Где можно настроить правила валидации полей

### Настройка правил валидации для полей коллекции

Большинство полей поддерживают настройку правил валидации. После того как вы настроите правила валидации для поля, при отправке данных будет запускаться серверная проверка. Различные типы полей поддерживают разные правила валидации.

- **Поле даты**

  ![20251028225946](https://static-docs.nocobase.com/20251028225946.png)

- **Числовое поле**

  ![20251028230418](https://static-docs.nocobase.com/20251028230418.png)

- **Текстовое поле**

  Помимо ограничения длины текста, текстовые поля также поддерживают пользовательские регулярные выражения для более точной валидации.

  ![20251028230554](https://static-docs.nocobase.com/20251028230554.png)

### Фронтенд-валидация в настройках поля

Правила валидации, заданные в настройках поля, запускают фронтенд-проверку, чтобы убедиться, что ввод пользователя соответствует требованиям.

![20251028230105](https://static-docs.nocobase.com/20251028230105.png)

**Текстовые поля** также поддерживают пользовательские регулярные выражения для валидации, чтобы соответствовать специфическим требованиям к формату.
