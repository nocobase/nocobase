# Установка правил валидации

## Введение

Правила валидации используются, чтобы гарантировать, что введенные пользователем данные соответствуют ожиданиям.

## Где установить правила валидации поля

### Настройка правил валидации для полей коллекции

Большинство полей поддерживают настройку правил валидации. После того как поле настроено с правилами валидации, серверная валидация срабатывает при отправке данных. Разные типы полей поддерживают разные правила валидации.

- **Текстовое поле**

  Помимо ограничения длины текста, текстовые поля также поддерживают настраиваемые регулярные выражения для более точной валидации.

### Клиентские правила проверки в настройках поля

Правила валидации, заданные в конфигурации поля, запускают проверку на стороне клиента, чтобы убедиться, что ввод пользователя соответствует требованиям.

Если у соответствующего поля коллекции уже есть правила проверки, они отображаются в настройках проверки в разделе **Серверные правила проверки полей**. Они наследуются из конфигурации поля источника данных и доступны здесь только для чтения. Если нужно изменить их, отредактируйте поле коллекции в разделе Источник данных → Настройка коллекции.

Правила, добавленные в разделе **Клиентские правила проверки**, применяются только к текущему компоненту поля. Они не изменяют конфигурацию поля коллекции. Когда существуют обе группы правил, NocoBase применяет унаследованные правила поля и клиентские правила проверки вместе.

**Текстовые поля** также поддерживают настраиваемую проверку по регулярному выражению, чтобы соответствовать требованиям к конкретному формату.
