# Установить область данных

## Введение

Настройка области данных для поля связи аналогична настройке области данных для блока. Она задает условия фильтрации по умолчанию для связанных данных.

## Инструкции по использованию

![20251028211328](https://static-docs.nocobase.com/20251028211328.png)

### Статическое значение

Пример: для связи можно выбрать только неудаленные продукты.

> Список полей содержит поля из целевой коллекции поля связи.

![20251028211434](https://static-docs.nocobase.com/20251028211434.png)

### Значение переменной

Пример: для связи можно выбрать только продукты, у которых дата обслуживания позже даты заказа.

![20251028211727](https://static-docs.nocobase.com/20251028211727.png)

Для дополнительной информации о переменных см. [Переменные](/interface-builder/variables)

### Связывание полей связи

Связывание между полями связи достигается через настройку области данных.

Пример: коллекция «Заказы» имеет поле связи «один-ко-многим» «Продукт возможности» и поле связи «многие-к-одному» «Возможность». Коллекция «Продукт возможности» имеет поле связи «многие-к-одному» «Возможность». В блоке формы заказа выбираемые данные для «Продукт возможности» фильтруются так, чтобы показывались только те продукты возможностей, которые связаны с текущей выбранной «Возможностью» в форме.
