# Установка области данных

## Введение

Установка области данных для поля связи аналогична установке области данных для блока. Она определяет условия фильтрации по умолчанию для связанных данных.

## Инструкции по использованию

![20251028211328](https://static-docs.nocobase.com/20251028211328.png)

### Статическое значение

Пример: Для связи можно выбрать только не удаленные товары.

> Список полей содержит поля из целевой коллекции поля связи.

![20251028211434](https://static-docs.nocobase.com/20251028211434.png)

### Значение переменной

Пример: Для связи можно выбрать только те товары, дата обслуживания которых позже даты заказа.

![20251028211727](https://static-docs.nocobase.com/20251028211727.png)

Дополнительную информацию о переменных см. в разделе [Переменные](/interface-builder/variables)

### Взаимосвязь полей связи

Взаимосвязь между полями связи достигается путем установки области данных.

Пример: В коллекции "Заказы" есть поле связи "Продукт возможности" (один-ко-многим) и поле связи "Возможность" (многие-к-одному). Коллекция "Продукт возможности" также имеет поле связи "Возможность" (многие-к-одному). В блоке формы заказа данные для поля "Продукт возможности" фильтруются так, чтобы отображать только те продукты возможности, которые связаны с выбранной в текущей форме "Возможностью".
