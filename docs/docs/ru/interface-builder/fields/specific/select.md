# Выпадающий список

## Введение

Выпадающий список позволяет связывать данные, выбирая их из существующих записей в целевой коллекции, или добавляя новые данные в коллекцию для последующего связывания. Опции выпадающего списка поддерживают нечёткий поиск.

![20251029205901](https://static-docs.nocobase.com/20251029205901.png)

## Конфигурация поля

### Настройка области данных

Определяет диапазон данных для выпадающего списка.

![20251029210025](https://static-docs.nocobase.com/20251029210025.png)

Более подробную информацию смотрите в разделе [Настройка области данных](/interface-builder/fields/field-settings/data-scope)

### Настройка правил сортировки

Определяет порядок сортировки данных в выпадающем списке.

Пример: Сортировка по дате обслуживания в обратном порядке.

![20251029210105](https://static-docs.nocobase.com/20251029210105.png)

### Разрешить добавление/связывание нескольких записей

Ограничивает отношения «один-ко-многим», позволяя связывать только одну запись.

![20251029210145](https://static-docs.nocobase.com/20251029210145.png)

#### Добавление через выпадающий список

После создания новой записи в целевой коллекции система автоматически выбирает её и связывает при отправке формы.

В примере ниже таблица заказов имеет поле отношения «многие-к-одному» — «Account».

![20251125220447](https://static-docs.nocobase.com/20251125220447.gif)

#### Добавление через модальное окно

Добавление через модальное окно подходит для более сложных сценариев ввода данных и позволяет настроить форму для создания новых записей.
