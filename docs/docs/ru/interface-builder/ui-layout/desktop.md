# Макет для компьютера

В NocoBase **макет для компьютера** — это интерфейс приложения по умолчанию. Он подходит для управления данными, ввода данных через формы, настройки бизнес-процессов и повседневной работы на компьютере, а также может использоваться на мобильных устройствах.

По умолчанию макет для компьютера доступен по адресу `/admin`. Если для приложения настроен отдельный префикс доступа, он автоматически добавляется к фактическому адресу.

## Создание страницы

### Шаг 1. Откройте макет для компьютера

Перейдите по адресу `/admin`. Обычно после успешного входа приложение также открывает этот макет напрямую.

![20260715225049](https://static-docs.nocobase.com/20260715225049.png)

### Шаг 2. Включите UI Editor

Нажмите «UI Editor» в правом верхнем углу страницы, чтобы перейти в режим настройки интерфейса. После этого рядом с меню, страницами, блоками, полями и действиями появятся соответствующие точки настройки.

![20260715225155_rec_](https://static-docs.nocobase.com/20260715225155_rec_.gif)

### Шаг 3. Создайте меню и страницу

В области навигации можно добавить группу, страницу или ссылку, а для страницы можно включить вкладки. После создания откройте страницу и добавьте нужные блоки.

Содержимое настраивается так же, как и в других интерфейсах: сначала добавьте [блоки](../blocks/index.md), затем настройте [поля](../fields/index.md) и [действия](../actions/index.md) в соответствии с рабочим процессом.

![20260715225316_rec_](https://static-docs.nocobase.com/20260715225316_rec_.gif)

### Шаг 4. Настройте содержимое страницы

Добавьте на страницу таблицы, формы, подробные представления, фильтры или другие блоки, затем настройте расположение полей, действий и блоков. Все изменения сразу отображаются на текущей странице.

![20260715225424_rec_](https://static-docs.nocobase.com/20260715225424_rec_.gif)

## Управление маршрутами и меню

После добавления страницы или ссылки в области навигации соответствующий элемент появляется в [менеджере маршрутов](../../routes/index.md). Изменения маршрута в менеджере также синхронно обновляют меню.

Макет для компьютера поддерживает следующие распространённые типы маршрутов:

- **Группа (Group)** — объединяет несколько страниц и ссылок в одну навигационную группу.
- **Страница (Page)** — открывает страницу, на которой можно продолжить добавлять блоки.
- **Ссылка (Link)** — открывает внутренний или внешний адрес.
- **Вкладка (Tab)** — объединяет несколько разделов содержимого на одной странице.

В менеджере маршрутов можно создавать, редактировать, удалять, показывать и скрывать маршруты. Когда нужно привести в порядок всю структуру меню, удобнее работать именно в менеджере маршрутов.

![20260715225711_rec_](https://static-docs.nocobase.com/20260715225711_rec_.gif)

## Адаптация к узким экранам

Макет для компьютера можно использовать на телефоне или в узком окне браузера. В режиме узкого экрана он использует исходные маршруты и страницы для компьютера и не переключается на мобильный макет автоматически.

### Как меняется макет

Навигационное меню сворачивается, а действия в верхней части страницы перемещаются в более компактное меню. Поля и отступы между блоками уменьшаются, а область содержимого подстраивается под видимую высоту мобильного браузера.

На узком экране UI Editor недоступен. Чтобы изменить меню или страницу, необходимо вернуться к браузеру на компьютере.

![20260715224603](https://static-docs.nocobase.com/20260715224603.png)

### Как адаптируется содержимое страницы

Часто используемые компоненты также адаптируют взаимодействие к узкому экрану, чтобы ими было удобнее пользоваться на телефоне. Например, содержимое в несколько колонок перестраивается в одну колонку, таблицы позволяют прокручивать колонки за пределами экрана по горизонтали, а пагинация и точки входа для действий становятся компактнее. Выбор значений, даты и время, фильтры и вложенные страницы также используют более удобные для телефона способы взаимодействия.

:::tip Адаптивный макет для компьютера и мобильный макет

Если приложение открывается с телефона лишь иногда, достаточно адаптации макета для компьютера к узкому экрану. Если нужны отдельные нижняя навигация, мобильные страницы и процессы, дополнительно настройте [мобильный макет](./mobile.md).

:::

## Рекомендации

- Для процессов, которые выполняются в основном на компьютере, используйте макет для компьютера по умолчанию.
- Сначала настройте страницу на широком экране, затем уменьшите окно и проверьте её вид на узком экране.
- Если в таблице много колонок или горизонтально расположенных действий, оставьте только необходимое, чтобы снизить нагрузку на небольшом экране.
- Если процессы на компьютере и мобильном устройстве сильно различаются, удобнее создать отдельные страницы для каждого макета.
