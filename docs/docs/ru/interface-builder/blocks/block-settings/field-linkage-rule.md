# Правила связывания полей

## Введение

Правила связывания полей позволяют динамически менять состояние полей в блоках формы и деталей в зависимости от действий пользователя. В настоящее время блоки, поддерживающие правила связывания полей, включают:

- [Блок формы](/interface-builder/blocks/data-blocks/form)
- [Блок деталей](/interface-builder/blocks/data-blocks/details)
- [Подформа](/interface-builder/fields/specific/sub-form)

## Инструкция по использованию

### Блок формы

В блоке формы правила связывания могут динамически менять поведение полей в зависимости от конкретных условий:

- **Управление видимостью поля (показать/скрыть)**: определить, будет ли отображаться текущее поле, на основе значений других полей.
- **Устанавливать поле обязательным**: динамически задавать, является ли поле обязательным, или нет, в зависимости от условий.
- **Назначать значение**: автоматически назначать значение полю на основе условий.
- **Выполнять указанный JavaScript**: написать JavaScript в соответствии с требованиями бизнеса.

### Блок деталей

В блоке деталей правила связывания главным образом используются, чтобы динамически управлять видимостью (показать/скрыть) полей на блоке.

## Связывание свойств

### Назначить значение

Например: когда заказ отмечается как дополнительный заказ, статус заказа автоматически получает значение `На проверке`.

![20251029115348](https://static-docs.nocobase.com/20251029115348.png)

### Обязательное

Например: когда статус заказа — `Оплачен`, поле суммы заказа становится обязательным.

![20251029115031](https://static-docs.nocobase.com/20251029115031.png)

### Показать/скрыть

Например: счет на оплату и общая сумма отображаются только тогда, когда статус заказа — `Ожидает оплаты`.

![20251030223710](https://static-docs.nocobase.com/20251030223710.png)
