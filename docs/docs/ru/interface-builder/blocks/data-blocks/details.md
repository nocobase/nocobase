# Блок сведений

## Введение

Блок сведений предназначен для отображения значений полей каждой записи данных. Он поддерживает гибкую компоновку полей и имеет встроенные функции для работы с данными, что упрощает просмотр и управление информацией.

## Настройки блока

![20251029202614](https://static-docs.nocobase.com/20251029202614.png)

### Правила связывания блока

С помощью правил связывания вы можете управлять поведением блока (например, его отображением или выполнением JavaScript).

![20251023195004](https://static-docs.nocobase.com/20251023195004.png)

Подробнее см. [Правила связывания](/interface-builder/linkage-rule)

### Настройка области данных

Пример: Отображать только оплаченные заказы

![20251023195051](https://static-docs.nocobase.com/20251023195051.png)

Подробнее см. [Настройка области данных](/interface-builder/blocks/block-settings/data-scope)

### Правила связывания полей

Правила связывания в блоке сведений позволяют динамически настраивать отображение/скрытие полей.

Пример: Не отображать сумму, если статус заказа — «Отменен».

![20251023200739](https://static-docs.nocobase.com/20251023200739.png)

Подробнее см. [Правила связывания](/interface-builder/linkage-rule)

## Настройка полей

### Поля из текущей коллекции

> **Примечание**: Поля из унаследованных коллекций (т.е. поля родительской коллекции) автоматически объединяются и отображаются в текущем списке полей.

![20251023201012](https://static-docs.nocobase.com/20251023201012.png)

### Поля из связанных коллекций

> **Примечание**: Поддерживается отображение полей из связанных коллекций (в настоящее время только для отношений «один к одному»).

![20251023201258](https://static-docs.nocobase.com/20251023201258.png)

### Другие поля
- JS Field
- JS Item
- Разделитель
- Markdown

![20251023201514](https://static-docs.nocobase.com/20251023201514.png)

> **Совет**: Вы можете написать код JavaScript для реализации пользовательского отображения контента, что позволит вам показывать более сложную информацию.
> Например, вы можете отображать различные эффекты в зависимости от типов данных, условий или логики.
