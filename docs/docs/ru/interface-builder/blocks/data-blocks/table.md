 Блок таблицы

## Введение

Блок таблицы — один из основных встроенных блоков данных в **NocoBase**, который в первую очередь используется для отображения и управления структурированными данными в табличном формате. Он предлагает гибкие параметры конфигурации, позволяя пользователям настраивать столбцы таблицы, ширину столбцов, правила сортировки и область данных, чтобы отображаемые данные соответствовали конкретным бизнес-потребностям.

#### Ключевые возможности
- **Гибкая настройка столбцов**: можно настраивать столбцы таблицы и их ширину под разные требования отображения данных.
- **Правила сортировки**: поддерживается сортировка табличных данных. Пользователи могут упорядочивать данные по возрастанию или убыванию на основе разных полей.
- **Настройка области данных**: задавая область данных, пользователи могут контролировать диапазон отображаемых данных, избегая влияния нерелевантных данных.
- **Настройка действий**: блок таблицы имеет различные встроенные варианты действий. Пользователи могут легко настроить действия, такие как «Фильтр», «Добавить», «Редактировать» и «Удалить», для быстрого управления данными.
- **Быстрое редактирование**: поддерживает прямое редактирование данных внутри таблицы, упрощая процесс работы и повышая эффективность.

## Настройки блока

![20251023150819](https://static-docs.nocobase.com/20251023150819.png)

### Правила связывания блока

Управляйте поведением блока (например, отображать ли его или выполнять JavaScript) через правила связывания.

![20251023194550](https://static-docs.nocobase.com/20251023194550.png)

Для подробностей см. [Правила связывания](/interface-builder/linkage-rule)

### Установить область данных

Пример: по умолчанию фильтровать заказы, где «Статус» — «Оплачен».

![20251023150936](https://static-docs.nocobase.com/20251023150936.png)

Для подробностей см. [Установить область данных](/interface-builder/blocks/block-settings/data-scope)

### Установить правила сортировки

Пример: отображать заказы в порядке убывания по дате.

![20251023155114](https://static-docs.nocobase.com/20251023155114.png)

Для подробностей см. [Установить правила сортировки](/interface-builder/blocks/block-settings/sorting-rule)

### Включить быстрое редактирование

Активируйте «Быстрое редактирование» в настройках блока и в настройках столбцов таблицы, чтобы настроить, какие столбцы можно быстро редактировать.

![20251023190149](https://static-docs.nocobase.com/20251023190149.png)

![20251023190519](https://static-docs.nocobase.com/20251023190519.gif)

### Включить древовидную таблицу

Когда таблица данных является иерархической (древовидной), блок таблицы может включить функцию **«Древовидная таблица»**. По умолчанию эта опция отключена. После включения блок будет отображать данные в древовидной структуре и поддерживать соответствующие параметры конфигурации и операции.

![20251125205918](https://static-docs.nocobase.com/20251125205918.png)

### Разворачивать все строки по умолчанию

Когда включена древовидная таблица, блок поддерживает разворачивание всех дочерних строк по умолчанию при загрузке.

### Поля этой коллекции

> **Примечание**: поля из наследуемых коллекций (то есть полей родительской коллекции) автоматически объединяются и отображаются в текущем списке полей.

### Поля связанных коллекций

> **Примечание**: поддерживается отображение полей из связанных коллекций (сейчас поддерживаются только связи «один-к-одному»).
