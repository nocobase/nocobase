# Блок комментариев

## Введение

Блок комментариев добавляет возможность комментирования к бизнес-записям. Его можно добавить на страницы деталей или во всплывающие окна задач, статей, заявок, клиентов и других записей, чтобы пользователи могли просматривать, отвечать и создавать комментарии для текущей записи.

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_02_PM.png)

:::tip Подсказка

Блок комментариев сам не создает коллекцию. Перед использованием подготовьте коллекцию для хранения комментариев и настройте поля для текста комментария, автора комментария, владельца комментария и времени комментария.

:::

## Добавить блок

Блок комментариев обычно добавляется на страницу деталей или во всплывающее окно бизнес-записи.

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_03_PM.png)

1. Откройте страницу деталей или всплывающее окно нужной записи
2. Нажмите "Добавить блок"
3. Выберите "Комментарий"
4. Выберите коллекцию для хранения комментариев
5. Завершите сопоставление полей согласно подсказкам

Если блок комментариев создается из связи, NocoBase попытается автоматически определить поле владельца комментария и значение текущей записи на основе текущей связи. В этом случае "Поле владельца комментария" и "Значение поля владельца комментария" заполняются автоматически и обычно не требуют ручной настройки.

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_04_PM.png)

Если блок создается напрямую из коллекции комментариев, необходимо вручную настроить поле владельца комментария и его значение.

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_03_PM%20(1).png)

## Сопоставление полей

Блок комментариев использует "Сопоставление полей", чтобы определить, как каждый комментарий должен отображаться и сохраняться.

| Настройка | Описание |
| --- | --- |
| Поле содержимого комментария | Выберите поле для хранения текста комментария. |
| Поле автора комментария | Выберите поле многие-к-одному, связанное с коллекцией пользователей. |
| Поле владельца комментария | Выберите поле для хранения идентификатора текущей бизнес-записи. |
| Значение поля владельца комментария | Укажите значение текущей бизнес-записи, например `{{ ctx.popup.record.id }}`. |
| Поле даты комментария | Выберите поле времени комментария, используемое для отображения и сортировки по умолчанию. |

### Поле владельца комментария

"Поле владельца комментария" используется для фильтрации комментариев текущей записи, а также записывается при создании нового комментария.

При ручном выборе выпадающий список показывает только обычные скалярные поля и не показывает поля связей. Частые варианты настройки:

| Бизнес-коллекция | Поле владельца в коллекции комментариев | Значение поля владельца комментария |
| --- | --- | --- |
| Задачи | `taskId` | `{{ ctx.popup.record.id }}` |
| Статьи | `postId` | `{{ ctx.popup.record.id }}` |
| Заявки | `ticketId` | `{{ ctx.popup.record.id }}` |

Если текущая запись использует уникальный идентификатор, отличный от `id`, измените "Значение поля владельца комментария" на соответствующее поле, например `{{ ctx.popup.record.uuid }}`.

### Автоматическое сопоставление из связей

Если блок создается из связи бизнес-записи, блок комментариев в первую очередь использует поле внешнего ключа этой связи как поле владельца комментария, а значение текущей записи — как значение поля владельца комментария.

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_04_PM.png)

Например, если между коллекцией задач и коллекцией комментариев задач есть связь один-ко-многим, а поле внешнего ключа в коллекции комментариев задач — `taskId`, то при добавлении блока комментариев из связи на странице деталей задачи блок автоматически использует:

- Поле владельца комментария: `taskId`
- Значение поля владельца комментария: идентификатор текущей записи задачи

Такой подход подходит для большинства сценариев и снижает риск ошибок при ручной настройке.

## Конфигурация блока

![](https://static-docs.nocobase.com/Comments-07-01-2026_12_07_PM.png)

### Размер страницы

Задайте количество комментариев, отображаемых на одной странице. Доступные значения: `5`, `10`, `20`, `50`, `100` и `200`.

### Область данных

Задайте область фильтрации данных для списка комментариев. Здесь можно добавить дополнительные условия, например показывать только комментарии с определенным статусом или условиями доступа.

Подробнее см. [Установить область данных](../block-settings/data-scope.md).

### Правило сортировки по умолчанию

Задайте сортировку по умолчанию для списка комментариев. Обычно можно сортировать по полю даты комментария по возрастанию или убыванию.

Если правило сортировки по умолчанию не настроено отдельно, блок комментариев в первую очередь использует "Поле даты комментария" как поле сортировки по умолчанию.

Подробнее см. [Установить правило сортировки](../block-settings/sorting-rule.md).

### Автоматический переход на последнюю страницу

По умолчанию отключено. Если отключено, после открытия блок комментариев остается на первой странице.

Если включено, блок комментариев при первой загрузке переходит на последнюю страницу. Это подходит, когда нужно сначала показать пользователям самые новые комментарии.
