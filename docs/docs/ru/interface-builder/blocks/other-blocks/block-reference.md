# Блок-ссылка

## Введение
Блок-ссылка позволяет отобразить уже настроенный блок на текущей странице, указав его UID, без необходимости повторной настройки.

## Активация плагина
Этот плагин встроен, но по умолчанию отключен.
Откройте «Управление плагинами» → найдите «Блок: Ссылка» → нажмите «Включить».

![Включение блока-ссылки (Менеджер плагинов)](https://static-docs.nocobase.com/block-reference-enable-20251102.png)

## Как добавить блок
1) Добавьте блок → группа «Другие блоки» → выберите «Блок-ссылка».  
2) В «Настройках блока-ссылки» укажите:
   - `Block UID`: UID целевого блока
   - `Режим ссылки`: выберите `Ссылка` или `Копия`

![Демонстрация добавления и настройки блока-ссылки](https://static-docs.nocobase.com/20251102193949_rec_.gif)

### Как получить UID блока
- Откройте меню настроек целевого блока и нажмите `Копировать UID`, чтобы скопировать его UID.

![Пример копирования UID блока](https://static-docs.nocobase.com/block-reference-copy-uid-20251102.png)

## Режимы и поведение
- `Ссылка` (по умолчанию)
  - Использует ту же конфигурацию, что и исходный блок; изменения, внесенные в исходный блок или в любое из его ссылочных представлений, будут синхронно обновляться во всех ссылках.

- `Копия`
  - Создает независимый блок, идентичный исходному на момент копирования; последующие изменения не синхронизируются между ними.

## Настройка
- Блок-ссылка:
  - «Настройки блока-ссылки»: используются для указания UID целевого блока и выбора режима «Ссылка»/«Копия»;
  - При этом будут отображаться полные настройки «ссылочного блока» (что эквивалентно прямой настройке исходного блока).

![Интерфейс настроек блока-ссылки](https://static-docs.nocobase.com/block-reference-settings-20251102.png)

- Скопированный блок:
  - Полученный после копирования блок будет того же типа, что и исходный, и будет содержать только собственные настройки;
  - Он больше не будет содержать «Настройки блока-ссылки».

## Ошибки и замещающие состояния
- Если целевой блок отсутствует или недействителен: отображается сообщение об ошибке. Вы можете повторно указать UID блока в настройках блока-ссылки («Настройки блока-ссылки» → `Block UID`), и после сохранения отображение восстановится.  

![Состояние ошибки при недействительном целевом блоке](https://static-docs.nocobase.com/block-reference-error-20251102.png)

## Примечания и ограничения
- Это экспериментальная функция — используйте ее с осторожностью в рабочей среде.
- При копировании некоторые конфигурации, зависящие от целевого UID, могут потребовать повторной настройки.
- Все конфигурации блока-ссылки автоматически синхронизируются, включая такие настройки, как «область данных». Однако блок-ссылка может иметь собственную [конфигурацию потока событий](/interface-builder/event-flow/), поэтому вы можете косвенно достичь различных областей данных или связанных конфигураций для каждой ссылки с помощью потоков событий и пользовательских действий JavaScript.
