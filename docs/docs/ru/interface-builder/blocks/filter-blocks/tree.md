# Фильтр «Дерево»

## Введение

Блок фильтра «Дерево» предоставляет возможности фильтрации данных через древовидную структуру. Подходит для сценариев с иерархическими данными — например, категорий товаров, организационной структуры и т. п.  
Пользователи могут выбирать узлы разных уровней, чтобы выполнять связанную фильтрацию соответствующих блоков данных.

## Как использовать

Блок фильтра «Дерево» работает совместно с блоком данных, обеспечивая для него возможности фильтрации.

Шаги настройки:

1. Включите режим конфигурации, добавьте на страницу блок «Фильтр-дерево» и блок данных (например, «Блок таблицы»).
2. Настройте блок фильтра «Дерево», выбрав древовидную таблицу данных (например, таблицу категорий товаров).
3. Установите связь между блоком фильтра «Дерево» и блоком данных.
4. После настройки нажатие на узлы дерева будет фильтровать блок данных.

## Добавление блока

В режиме конфигурации нажмите кнопку «Добавить блок» на странице и в категории «Блоки фильтрации» выберите «Дерево».

### Привязка блока данных

Блок фильтра «Дерево» работает только при наличии связи с блоком данных.  
С помощью параметра «Привязка блока данных» можно установить связь между фильтром-деревом и таблицей, списком, графиком или другим блоком на странице, чтобы реализовать функцию фильтрации.

![](https://static-docs.nocobase.com/Tree-filter-04-07-2026_03_14_PM.png)

### Поле заголовка

Используется для указания поля, по которому будет отображаться название узла дерева.

### Поиск

После включения поддерживается быстрый поиск и переход к нужным узлам по ключевому слову, что повышает скорость работы.

### Развернуть всё

Управляет тем, будут ли все узлы дерева развёрнуты при первой загрузке страницы.

### Включать дочерние узлы при фильтрации

После включения при выборе узла в фильтрацию также включаются данные всех его дочерних узлов.  
Подходит для случаев, когда нужно посмотреть все дочерние данные по родительской категории.
