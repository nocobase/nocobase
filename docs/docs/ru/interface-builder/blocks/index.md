# Блоки

Блоки являются носителями данных и контента. Их можно размещать на страницах (Page), в модальных окнах (Modal) или выдвижных панелях (Drawer). Несколько блоков можно свободно перетаскивать и располагать.

## Типы блоков

![Типы блоков](https://static-docs.nocobase.com/f71af45b5cd914ea0558f760ddbbba58.png)

- Блок данных: Используется для отображения данных из источника данных на интерфейсе.
- Блок фильтрации: Используется для использования данных из источника данных в качестве условий фильтрации для других блоков данных.
- Другой блок: Используется для отображения специфического или независимого контента, такого как задачи рабочего процесса, журналы аудита, Markdown и т.д.

## Добавление блоков

Блоки можно размещать на страницах (Page), в модальных окнах (Modal) или выдвижных панелях (Drawer).

### Блоки на странице

В настоящее время типы блоков на странице включают: блок данных, блок фильтрации и другой блок.

![20251023222441](https://static-docs.nocobase.com/20251023222441.png)

### Блоки во всплывающих окнах (модальные окна или выдвижные панели)

Всплывающие окна бывают двух типов: модальные окна и выдвижные панели. Как и на страницах, в них также можно добавлять блоки. Отличие заключается в том, что блоки форм во всплывающих окнах обычно используются для добавления, редактирования или просмотра одной записи. Типы блоков включают блок данных и другой блок.

![20251023222613](https://static-docs.nocobase.com/20251023222613.png)

## Дизайнер блоков

В правом верхнем углу каждого блока расположены три небольших значка. Слева направо это:

1. Перетаскивание для изменения макета
2. [Поток событий](/interface-builder/event-flow)
3. Настройки блока

![20251023224032](https://static-docs.nocobase.com/20251023224032.png)

Для простых блоков все параметры конфигурации сосредоточены в разделе «Настройки блока», например, для JS Block.

![20251023224903](https://static-docs.nocobase.com/20251023224903.png)

Сложные блоки данных также предоставляют отдельные встроенные опции «Настройка полей» и «Настройка действий».

![20251023225141](https://static-docs.nocobase.com/20251023225141.png)

Кроме того, существует гибкость для создания дополнительных возможностей вложенности, например, в блоке диаграмм.

![](https://static-docs.nocobase.com/07588190b3f41ae3060e71d8b76b4447.png)
