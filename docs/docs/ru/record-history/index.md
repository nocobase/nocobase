# История записей

## Введение

Плагин **История записей** отслеживает изменения данных, автоматически сохраняя снимки и различия операций **создания**, **обновления** и **удаления**. Он помогает пользователям быстро просматривать изменения данных и проводить аудит операций.

![](https://static-docs.nocobase.com/202511011338499.png)

## Включение истории записей

### Добавление коллекций и полей

Сначала перейдите на страницу настроек плагина «История записей» и добавьте коллекции и поля, для которых нужно отслеживать историю. Для повышения эффективности записи и во избежание избыточности данных рекомендуется отслеживать только ключевые поля. Поля вроде **уникального идентификатора**, **времени создания**, **времени обновления**, **создано пользователем** и **последнее обновление пользователем** обычно записывать не требуется.

![](https://static-docs.nocobase.com/202511011315010.png)

![](https://static-docs.nocobase.com/202511011316342.png)

### Синхронизация исторических снимков данных

- Для записей, созданных до включения отслеживания истории, изменения будут регистрироваться только после первого обновления, которое создаст снимок; первоначальное обновление или удаление не будут записаны.
- Чтобы сохранить историю уже существующих данных, можно выполнить разовую синхронизацию снимков.
- Размер снимков по коллекции считается как: число записей × число отслеживаемых полей.
- Для больших наборов данных рекомендуется фильтровать по области данных и синхронизировать только важные записи.

![](https://static-docs.nocobase.com/202511011319386.png)

![](https://static-docs.nocobase.com/202511011319284.png)

Нажмите **«Синхронизировать исторические снимки»**, настройте поля и область данных, затем запустите синхронизацию.

![](https://static-docs.nocobase.com/202511011320958.png)

Задача синхронизации будет поставлена в очередь и выполнена в фоновом режиме. Обновите список, чтобы проверить статус выполнения.

## Использование блока истории записей

### Добавление блока

Выберите **Блок истории записей** и укажите коллекцию, чтобы добавить на страницу соответствующий блок истории.

![](https://static-docs.nocobase.com/202511011323410.png)

![](https://static-docs.nocobase.com/202511011331667.png)

Если вы добавляете блок истории во всплывающем окне с деталями записи, можно выбрать **«Текущая запись»**, чтобы показывать историю только текущей записи.

![](https://static-docs.nocobase.com/202511011338042.png)

![](https://static-docs.nocobase.com/202511011338499.png)

### Редактирование шаблонов описания

Нажмите **Редактировать шаблон** в настройках блока, чтобы задать текст описания для записей операций.

![](https://static-docs.nocobase.com/202511011340406.png)

Можно отдельно настроить шаблоны описания для операций **создания**, **обновления** и **удаления**. Для операций обновления также можно настроить шаблон описания изменений полей — один общий для всех полей или отдельный для конкретных полей.

![](https://static-docs.nocobase.com/202511011346400.png)

При настройке текста можно использовать переменные.

![](https://static-docs.nocobase.com/202511011347163.png)

После настройки можно применить шаблон ко всем **блокам истории записей текущей коллекции** или только к **этому блоку истории записей**.

![](https://static-docs.nocobase.com/202511011348885.png)
