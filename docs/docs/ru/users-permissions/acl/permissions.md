# Настройка прав

## Общие настройки прав

![](https://static-docs.nocobase.com/119a9650259f9be71210121d0d3a435d.png)

### Права конфигурации

1. **Разрешить настройку интерфейса**: это право определяет, может ли пользователь настраивать интерфейс. При активации добавляется кнопка настройки интерфейса. Для роли «admin» это право включено по умолчанию.
2. **Разрешить установку, активацию и отключение плагинов**: это право определяет, может ли пользователь включать или отключать плагины. Когда оно активно, пользователь получает доступ к интерфейсу менеджера плагинов. Для роли «admin» это право включено по умолчанию.
3. **Разрешить настройку плагинов**: это право позволяет пользователю настраивать параметры плагинов или управлять серверными данными плагинов. Для роли «admin» это право включено по умолчанию.
4. **Разрешить очистку кэша и перезапуск приложения**: это право связано с задачами системного обслуживания, такими как очистка кэша и перезапуск приложения. После активации соответствующие кнопки появляются в личном кабинете. По умолчанию это право отключено.
5. **Новые пункты меню доступны по умолчанию**: вновь созданные меню по умолчанию доступны, и эта настройка включена по умолчанию.

### Глобальные права действий

Глобальные права действий применяются ко всем коллекциям и классифицируются по типу операций. Эти права можно настраивать по области данных: все данные или собственные данные пользователя. Первый вариант разрешает операции со всей коллекцией, второй ограничивает операции данными, относящимися к пользователю.

## Права действий коллекции

![](https://static-docs.nocobase.com/6a6e0281391cecdea5b5218e6173c5d7.png)

![](https://static-docs.nocobase.com/9814140434ff9e1bf028a6c282a5a165.png)

Права действий коллекции позволяют тонко настраивать глобальные права действий через конфигурацию доступа к ресурсам внутри каждой коллекции. Эти права делятся на два аспекта:

1. **Права действий**: включают действия добавления, просмотра, редактирования, удаления, экспорта и импорта. Права задаются по области данных:
   - **Все записи**: даёт пользователю возможность выполнять действия со всеми записями в коллекции.
   - **Собственные записи**: ограничивает пользователя действиями только с записями, которые созданы им самим.

2. **Права полей**: позволяют задавать конкретные права для каждого поля при разных операциях. Например, некоторые поля можно настроить только для просмотра, без права редактирования.

## Права настройки плагинов

Права настройки плагинов управляют возможностью настраивать параметры конкретных плагинов. Когда они включены, соответствующий интерфейс управления плагинами появляется в центре администрирования.
