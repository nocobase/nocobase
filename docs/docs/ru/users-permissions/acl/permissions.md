# Настройка прав доступа

## Общие настройки прав доступа

![](https://static-docs.nocobase.com/119a9650259f9be71210121d0d3a435d.png)

### Права на настройку

1.  **Разрешить настройку интерфейса**: Это право определяет, может ли пользователь настраивать интерфейс. При активации этого права появляется кнопка настройки пользовательского интерфейса. Для роли "admin" это право включено по умолчанию.
2.  **Разрешить установку, активацию, отключение плагинов**: Это право определяет, может ли пользователь включать или отключать плагины. При активации этого права пользователь получает доступ к интерфейсу менеджера плагинов. Для роли "admin" это право включено по умолчанию.
3.  **Разрешить настройку плагинов**: Это право позволяет пользователю настраивать параметры плагинов или управлять данными плагинов на бэкенде. Для роли "admin" это право включено по умолчанию.
4.  **Разрешить очистку кеша, перезапуск приложения**: Это право связано с задачами системного обслуживания, такими как очистка кеша и перезапуск приложения. После активации соответствующие кнопки операций появляются в личном кабинете. По умолчанию это право отключено.
5.  **Новые пункты меню доступны по умолчанию**: Вновь созданные пункты меню доступны по умолчанию, и эта настройка включена по умолчанию.

### Глобальные права на операции

Глобальные права на операции применяются ко всем коллекциям и классифицируются по типу операции. Эти права можно настроить на основе области данных: "все данные" или "собственные данные" пользователя. Первое позволяет выполнять операции со всей коллекцией, тогда как второе ограничивает операции только данными, относящимися к пользователю.

## Права на операции с коллекциями

![](https://static-docs.nocobase.com/6a6e0281713cecdea5b5218e6173c5d7.png)

![](https://static-docs.nocobase.com/9814140434ff9e1bf028a6c282a5a165.png)

Права на операции с коллекциями позволяют более детально настроить глобальные права на операции, конфигурируя индивидуальный доступ к ресурсам в каждой коллекции. Эти права делятся на два аспекта:

1.  **Права на операции**: Сюда входят операции добавления, просмотра, редактирования, удаления, экспорта и импорта. Эти права настраиваются на основе области данных:
    -   **Все записи**: Предоставляет пользователю возможность выполнять операции со всеми записями в коллекции.
    -   **Собственные записи**: Ограничивает пользователя выполнением операций только с записями, которые он создал.

2.  **Права на поля**: Права на поля позволяют настраивать специфические права для каждого поля при выполнении различных операций. Например, некоторые поля можно настроить так, чтобы они были доступны только для просмотра, но не для редактирования.

## Права на настройку плагинов

Права на настройку плагинов контролируют возможность конфигурирования специфических параметров плагинов. При активации этих прав в центре администрирования появляется соответствующий интерфейс управления плагинами.
