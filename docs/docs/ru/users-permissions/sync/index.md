# Синхронизация пользовательских данных

## Введение

Эта функция позволяет регистрировать и управлять источниками синхронизации пользовательских данных. По умолчанию предоставляется HTTP API, но через плагины можно подключать дополнительные источники данных. По умолчанию поддерживается синхронизация данных в коллекции **Пользователи** и **Отделы**, а с помощью плагинов синхронизацию можно расширить на другие целевые ресурсы.

## Управление источниками данных и синхронизация

![](https://static-docs.nocobase.com/202412041043465.png)

:::info
Если не установлены плагины, предоставляющие источники синхронизации пользовательских данных, синхронизацию можно выполнять через HTTP API. См. [Источник данных — HTTP API](./sources/api.md).
:::

## Добавление источника данных

После установки плагина, предоставляющего источник синхронизации пользовательских данных, вы можете добавить соответствующий источник данных. Только для включённых источников данных будут показываться кнопки «Синхронизация» и «Задача».

> Пример: WeCom

![](https://static-docs.nocobase.com/202412041053785.png)

## Синхронизация данных

Нажмите кнопку **Синхронизация**, чтобы запустить синхронизацию данных.

![](https://static-docs.nocobase.com/202412041055022.png)

Нажмите кнопку **Задача**, чтобы просмотреть статус синхронизации. После успешной синхронизации данные можно просмотреть в списках «Пользователи» и «Отделы».

![](https://static-docs.nocobase.com/202412041202337.png)

Для задач синхронизации с ошибкой можно нажать **Повторить**.

![](https://static-docs.nocobase.com/202412041058337.png)

В случае сбоев синхронизации можно выполнить диагностику через системные логи. Кроме того, сырые записи синхронизации сохраняются в директории `user-data-sync` внутри папки логов приложения.

![](https://static-docs.nocobase.com/202412041205655.png)
