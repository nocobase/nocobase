# Настройка ACL

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

## Введение

Навык настройки ACL управляет ролями NocoBase, политиками прав доступа, привязкой пользователей и оценкой рисков ACL на естественном языке — вы описываете бизнес-цель, а он выбирает команды и параметры.


## Возможности

- Создание новых ролей
- Переключение глобальных режимов ролей (независимый режим / режим объединения)
- Пакетная настройка прав действий и областей данных для таблиц данных
- Отвязка пользователей от ролей
- Формирование отчётов об оценке рисков на уровне роли, пользователя и системы

## Примеры запросов

### Сценарий A: пакетная привязка пользователей
:::tip Предварительные требования
В текущем окружении должны быть роль Пользователь и несколько пользователей
:::

```
Помоги привязать роль Пользователь к этим новым пользователям: James, Emma, Michael
```

![Пакетная привязка пользователей](https://static-docs.nocobase.com/20260422202343.png)

### Сценарий B: пакетная настройка прав на страницы
:::tip Предварительные требования
В текущем окружении должны быть роль пользователь и несколько страниц
:::
```
Помоги настроить права доступа роли пользователь на этих страницах: Product, Order, Stock
```

![Пакетная настройка прав на страницы](https://static-docs.nocobase.com/20260422202949.png)

### Сценарий C: пакетная настройка прав на несколько таблиц
:::tip Предварительные требования
В текущем окружении должны быть роль Пользователь и несколько таблиц данных
:::

```
Добавь независимые права только на чтение для роли Пользователь на этих таблицах данных: order, product, stock
```

![Пакетная настройка независимых прав на таблицы](https://static-docs.nocobase.com/20260422205341.png)

![Пакетная настройка независимых прав на таблицы 2](https://static-docs.nocobase.com/20260422205430.png)

### Сценарий D: настройка прав для нескольких ролей и таблиц
:::tip Предварительные требования
В текущем окружении должны быть несколько ролей и несколько таблиц данных
:::

```
Добавь независимые права на чтение и запись для ролей Пользователь и Продажи на этих таблицах данных: order, product, stock
```

![Настройка для нескольких ролей и таблиц](https://static-docs.nocobase.com/20260422213524.png)

### Сценарий E: оценка рисков

```
Оцени риски прав доступа роли Пользователь
```

Будут выведены оценка риска, описание области влияния и рекомендации по улучшению.

## Часто задаваемые вопросы

**Что делать, если права настроены, но не применяются?**

Сначала проверьте, верен ли глобальный режим ролей — если у пользователя одновременно несколько ролей, разница в поведении между режимом объединения и независимым режимом существенна. Можно проверить текущий режим, чтобы подтвердить причину.
