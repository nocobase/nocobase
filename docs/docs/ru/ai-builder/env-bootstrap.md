# Управление окружением

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

## Введение

Навык управления окружением обрабатывает установку приложений NocoBase, обновления, остановку, запуск и управление несколькими окружениями — от разработки и тестирования до производственных окружений: от «NocoBase ещё не установлен» до «готово к входу и использованию».


## Возможности

- Запрос окружения и статуса NocoBase
- Добавление, удаление и переключение окружений экземпляров NocoBase
- Установка, обновление, остановка и запуск экземпляров NocoBase


## Примеры запросов

### Сценарий A: запрос статуса окружения
Режим запроса
```
Какие экземпляры NocoBase у меня есть? В каком окружении я сейчас нахожусь?
```
Режим CLI
```
nb env list
```

### Сценарий B: добавление существующего окружения
:::tip Предварительные требования

Требуется существующий экземпляр NocoBase — локальный или удалённый

:::

Режим запроса
```
Помоги добавить окружение dev по адресу http://localhost:13000
```
Режим CLI
```
nb env add <dev> --base-url http://localhost:13000/api
```
### Сценарий C: установка нового экземпляра NocoBase
:::tip Предварительные требования

Самый быстрый способ установить NocoBase — режим Docker. Перед началом убедитесь, что на машине установлены Node, Docker и Yarn.

:::

Режим запроса
```
Помоги установить NocoBase
```
Режим CLI
```
nb init --ui
```

### Сценарий D: обновление экземпляра

Режим запроса
```
Помоги обновить текущий экземпляр до последней версии
```
Режим CLI
```
nb app upgrade
```

### Сценарий E: остановка экземпляра

Режим запроса
```
Помоги остановить текущий экземпляр
```
Режим CLI
```
nb app stop
```

### Сценарий E: запуск экземпляра

Режим запроса
```
Помоги запустить текущий экземпляр
```
Режим CLI
```
nb app start
```

## Часто задаваемые вопросы

**Что делать, если после установки недоступны возможности Мастера ИИ-разработки?**

Сейчас все возможности Мастера ИИ-разработки находятся в alpha-образе. Проверьте, устанавливали ли вы с помощью этого образа. Если нет, можно обновиться до этого образа.

**Что делать, если Docker при запуске сообщает о конфликте портов?**

Используйте другой порт (например, `port=14000`) или сначала остановите процесс, занимающий порт 13000. На этапе предварительной проверки навык заранее предупредит о конфликте портов.
