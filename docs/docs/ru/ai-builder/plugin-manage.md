# Управление плагинами

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что Вы установили NocoBase CLI и выполнили инициализацию по инструкции [Быстрый старт с Конструктором ИИ](./index.md).

:::

## Введение

Skill управления плагинами служит для просмотра, включения и отключения плагинов NocoBase — он автоматически распознаёт локальное или удалённое окружение, выбирает подходящий backend для выполнения и подтверждает успех операций повторным чтением.


## Возможности

- Просмотр каталога плагинов и их состояния включения.
- Включение плагинов.
- Отключение плагинов.

## Примеры промптов

### Сценарий A: просмотр состояния плагинов

Промпт-режим
```
Какие плагины есть в текущем окружении?
```
Командный режим
```
nb plugin list
```

Будет выведен список всех плагинов с их состоянием включения и информацией о версии.

![Просмотр состояния плагинов](https://static-docs.nocobase.com/20260417150510.png)

### Сценарий B: включение плагина

Промпт-режим
```
Помоги мне включить плагин локализации
```
Командный режим
```
nb plugin enable <локализация>
```

Skill включает плагины по очереди и после каждого включения проверяет, что `enabled=true`.

![Включение плагина](https://static-docs.nocobase.com/20260417153023.png)

### Сценарий C: отключение плагина

Промпт-режим
```
Помоги мне отключить плагин локализации
```
Командный режим
```
nb plugin disable <локализация>
```

![Отключение плагина](https://static-docs.nocobase.com/20260417173442.png)

## Часто задаваемые вопросы

**Плагин включён, но не работает. Что делать?**

Некоторым плагинам после включения требуется перезапуск приложения. В результате выполнения Skill подскажет, нужен ли перезапуск.
