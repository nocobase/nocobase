# Управление плагинами

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

## Введение

Навык управления плагинами просматривает, включает и отключает плагины NocoBase — автоматически определяет локальные или удалённые окружения, выбирает подходящий бэкенд выполнения и проверяет операции через обратное чтение.


## Возможности

- Просмотр каталога плагинов и статуса включения.
- Включение плагинов.
- Отключение плагинов.

## Примеры запросов

### Сценарий A: просмотр статуса плагинов

Режим запроса
```
Какие плагины есть в текущем окружении?
```
Режим CLI
```
nb plugin list
```

Будет выведен список всех плагинов со статусом включения и информацией о версии.

![Просмотр статуса плагинов](https://static-docs.nocobase.com/20260417150510.png)

### Сценарий B: включение плагина

Режим запроса
```
Помоги включить плагин локализации
```
Режим CLI
```
nb plugin pm enable <localization>
```

Навык будет включать плагины по очереди, после каждого включения выполняя обратное чтение для подтверждения `enabled=true`.

![Включение плагина](https://static-docs.nocobase.com/20260417153023.png)

### Сценарий C: отключение плагина

Режим запроса
```
Помоги отключить плагин локализации
```
Режим CLI
```
nb plugin disable <localization>
```

![Отключение плагина](https://static-docs.nocobase.com/20260417173442.png)

## Часто задаваемые вопросы

**Что делать, если плагин не применяется после включения?**

Некоторым плагинам после включения требуется перезапуск приложения. Навык укажет в результатах, нужен ли перезапуск.
