# Решения

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

:::warning Примечание

Функция Решения сейчас на этапе тестирования с ограниченной стабильностью, доступна только для раннего доступа.

:::

## Введение

Навык Решения пакетно создаёт приложения NocoBase из файлов конфигурации YAML — одновременно создаёт таблицы данных, настраивает страницы и генерирует панели и диаграммы.

Подходит для сценариев, где нужно быстро построить целую бизнес-систему, например CRM, управление заявками, управление складом и другие.


## Возможности

Можно:

- Проектировать полные решения приложений на основе описания требований, включая таблицы данных, страницы и панели
- Пакетно создавать таблицы данных и страницы через `structure.yaml`
- Настраивать всплывающие окна и формы через `enhance.yaml`
- Автоматически генерировать панели с KPI-карточками и диаграммами
- Инкрементальные обновления — всегда используется режим `--force` без уничтожения существующих данных

Нельзя:

- Не подходит для тонкой настройки на уровне полей (лучше подходит [навык моделирования данных](./data-modeling))
- Нельзя выполнять миграцию или импорт данных
- Нельзя настраивать права доступа или рабочие процессы (нужны другие Skills)

## Примеры запросов

### Сценарий A: построение целой системы

```
Помоги использовать навык nocobase-dsl-reconciler, чтобы построить систему управления заявками с панелью, списком заявок, управлением пользователями и настройкой SLA
```

Навык сначала выведет план проектирования — со списком всех таблиц данных и структур страниц — затем после подтверждения выполнит сборку по этапам.

![План проектирования](https://static-docs.nocobase.com/20260420100420.png)

![Результат сборки](https://static-docs.nocobase.com/20260420100450.png)

### Сценарий B: изменение существующего модуля

```
Помоги использовать навык nocobase-dsl-reconciler, чтобы добавить в таблицу заявок поле «Приоритет» типа выпадающий список с вариантами от P0 до P3
```

Измените `structure.yaml` и обновите с `--force`.

### Сценарий C: настройка диаграмм

```
Помоги использовать навык nocobase-dsl-reconciler, чтобы на панели заменить «Новые заявки за эту неделю» на «Новые заявки за этот месяц»
```

![Настройка диаграмм](https://static-docs.nocobase.com/20260420100517.png)

Отредактируйте соответствующий SQL-файл, изменив диапазон времени с `'7 days'` на `'1 month'`, затем выполните `--verify-sql` для проверки.

## Часто задаваемые вопросы

**Что делать, если проверка SQL не прошла?**

NocoBase использует PostgreSQL. Имена столбцов должны быть в camelCase и заключены в двойные кавычки (например, `"createdAt"`). Для дат используйте `NOW() - '7 days'::interval` вместо синтаксиса SQLite. Запуск `--verify-sql` помогает выявить такие проблемы до развёртывания.

**Как тонко настроить конкретное поле после сборки?**

Используйте навык Решения для первоначальной полной сборки, а затем [навык моделирования данных](./data-modeling) или [навык настройки интерфейса](./ui-builder) для последующей тонкой настройки — они гибче для отдельных правок.
