# Быстрый старт с Конструктором ИИ

Конструктор ИИ — это возможность NocoBase для построения приложений с помощью ИИ. Вы описываете задачу на естественном языке, а ИИ автоматически выполняет моделирование данных, настройку страниц, конфигурацию прав доступа и другие операции. Это даёт более современный и эффективный опыт построения систем.

## Быстрый старт

Если Вы уже установили [NocoBase CLI](../ai/quick-start.md), пропустите этот шаг.

### Установка одной командой через ИИ

Скопируйте приведённый ниже промпт в Ваш ИИ-помощник (Claude Code, Codex, Cursor, Trae и т. д.), и установка с настройкой выполнятся автоматически:

```
Помоги мне установить NocoBase CLI и выполнить инициализацию: https://docs.nocobase.com/cn/ai/ai-quick-start.md (пожалуйста, перейди по ссылке и используй её содержимое)
```

### Ручная установка

```bash
npm install -g @nocobase/cli@beta
nb init --ui
```

Браузер автоматически откроет страницу визуальной настройки, которая поможет установить NocoBase Skills, настроить базу данных и запустить приложение. Подробные шаги описаны в разделе [Быстрый старт](../ai/quick-start.md).

## Замените ручную настройку диалогом

После установки NocoBase CLI Вы можете управлять NocoBase напрямую через ИИ-помощник на естественном языке. Ниже приведено несколько реальных сценариев — от создания одной таблицы до построения целой системы — чтобы продемонстрировать возможности Конструктора ИИ.

### Опишите задачу — ИИ спроектирует таблицы и связи

Расскажите ИИ, какую систему Вы хотите построить, и он автоматически спроектирует таблицы данных, типы полей и связи между ними — без необходимости рисовать ER-диаграмму вручную.

```
Я создаю CRM, помоги мне спроектировать и построить модель данных
```

![ИИ проектирует модель данных CRM](https://static-docs.nocobase.com/202604162126729.png)

ИИ автоматически создал таблицы клиентов, контактов, сделок, заказов и связи между ними:

![Результат построения модели данных CRM](https://static-docs.nocobase.com/202604162201867.png)

Подробнее о моделировании данных читайте в разделе [Моделирование данных](./data-modeling).

### Опишите страницу бизнес-языком — ИИ её построит

Не нужно изучать правила настройки — просто скажите, какую страницу Вы хотите: поле поиска, таблицу, фильтры. Описали — получили.

```
Создай мне страницу управления клиентами с полем поиска по имени и таблицей клиентов, в таблице должны быть имя, телефон, email и дата создания
```

![Страница управления клиентами](https://static-docs.nocobase.com/20260420100608.png)

Подробнее о настройке интерфейса читайте в разделе [Настройка интерфейса](./ui-builder).

### Оркестрация рабочего процесса одной фразой

Опишите условия запуска и логику обработки бизнес-процесса, и ИИ автоматически создаст триггер и цепочку узлов.

```
Помоги мне настроить рабочий процесс, который автоматически уменьшает остаток товара после создания заказа
```

![Рабочий процесс уменьшения остатков](https://static-docs.nocobase.com/20260419234303.png)

Подробнее о рабочих процессах читайте в разделе [Управление рабочими процессами](./workflow).

### Таблицы, страницы, дашборды — всё за один шаг

:::warning Внимание

Функция готовых решений пока находится в стадии тестирования и имеет ограниченную стабильность — только для предварительного ознакомления.

:::

Опишите бизнес-сценарий одной фразой, и ИИ построит для Вас таблицы данных, страницы управления, дашборды и графики.

```
Помоги мне с помощью skill nocobase-dsl-reconciler построить систему управления заявками: дашборд, список заявок, управление пользователями, настройка SLA
```

ИИ сначала выводит проектное решение, и после Вашего подтверждения выполняет построение за один проход:

![Проект системы заявок](https://static-docs.nocobase.com/20260420100420.png)

![Результат построения системы заявок](https://static-docs.nocobase.com/20260420100450.png)

Подробнее о построении целых систем читайте в разделе [Готовые решения](./dsl-reconciler).

## Безопасность и аудит

Прежде чем разрешать AI Agent выполнять операции в NocoBase, рекомендуется ознакомиться с методами аутентификации, контролем прав доступа и аудитом операций — чтобы ИИ делал только то, что должен, и каждый его шаг фиксировался. См. раздел [Безопасность и аудит](./security).

## NocoBase Skills

[NocoBase Skills](https://github.com/nocobase/skills) — это устанавливаемые в AI Agent пакеты предметных знаний, благодаря которым ИИ понимает систему конфигурации NocoBase. NocoBase предоставляет 8 Skills, охватывающих весь процесс построения:

- [Управление окружениями](./env-bootstrap) — проверка окружения, установка, развёртывание, обновление и диагностика проблем
- [Моделирование данных](./data-modeling) — создание и управление таблицами, полями, связями
- [Настройка интерфейса](./ui-builder) — создание и редактирование страниц, блоков, диалогов, интерактивных связей
- [Управление рабочими процессами](./workflow) — создание, редактирование, включение и диагностика рабочих процессов
- [Настройка прав доступа](./acl) — управление ролями, политиками прав, привязкой пользователей и оценкой рисков
- [Готовые решения](./dsl-reconciler) — массовое построение целых бизнес-систем из YAML
- [Управление плагинами](./plugin-manage) — просмотр, включение и отключение плагинов
- [Управление публикацией](./publish) — публикация между окружениями, резервное копирование, восстановление и миграция

:::tip Подсказка

NocoBase CLI автоматически устанавливает Skills во время инициализации (`nb init`), вручную делать это не нужно.

:::
