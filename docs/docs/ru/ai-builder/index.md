# Быстрый старт Мастера ИИ-разработки

Мастер ИИ-разработки — возможность NocoBase для создания с помощью ИИ: опишите требования на естественном языке — и ИИ автоматически выполнит моделирование данных, настройку страниц, конфигурацию прав доступа и другие операции. Это обеспечивает более современный и эффективный процесс разработки.

## Быстрый старт

Если вы уже установили [NocoBase CLI](../ai/quick-start.md), этот шаг можно пропустить.

### Установка в один клик с помощью ИИ

Скопируйте приведённый ниже запрос в ваш ИИ-ассистент (Claude Code, Codex, Cursor, Trae и т. д.) — он автоматически выполнит установку и настройку:

```
Помоги мне установить NocoBase CLI и завершить инициализацию: https://docs.nocobase.com/ai/ai-quick-start.md (пожалуйста, прочитай содержимое по ссылке напрямую)
```

### Ручная установка

```bash
npm install -g @nocobase/cli
nb init --ui
```

Браузер автоматически откроет страницу визуальной настройки и проведёт вас через установку NocoBase Skills, настройку базы данных и запуск приложения. Подробные шаги см. в [Быстрый старт](../ai/quick-start.md).

## Замените ручную настройку диалогом

После установки вы можете управлять NocoBase прямо из ИИ-ассистента на естественном языке. Ниже — несколько реальных сценариев: от создания одной таблицы до построения целой системы, чтобы вы могли оценить возможности Мастера ИИ-разработки.

### Опишите бизнес-требования — ИИ спроектирует таблицы и связи

Расскажите ИИ, какую систему вы хотите построить — он автоматически спроектирует таблицы данных, типы полей и связи; рисовать ER-диаграммы самому не нужно.

```
Я создаю CRM, помоги спроектировать и построить модель данных
```

![ИИ проектирует модель данных CRM](https://static-docs.nocobase.com/202604162126729.png)

ИИ автоматически сгенерировал таблицы данных для клиентов, контактов, сделок, заказов и их связи:

![Результат модели данных CRM](https://static-docs.nocobase.com/202604162201867.png)

Подробнее о моделировании данных см. [Моделирование данных](./data-modeling).

### Опишите страницы на языке бизнеса — ИИ их создаст

Не нужно изучать правила конфигурации — просто скажите, какая страница вам нужна. Поля поиска, таблицы, фильтры — опишите, и готово.

```
Помоги создать страницу управления клиентами с полем поиска по имени и таблицей клиентов с колонками: имя, телефон, email и время создания
```

![Страница управления клиентами](https://static-docs.nocobase.com/20260420100608.png)

Подробнее о настройке интерфейса см. [Настройка интерфейса](./ui-builder).

### Оркестрируйте автоматизированные рабочие процессы одной фразой

Опишите условия срабатывания и логику обработки бизнес-процесса — ИИ автоматически создаст триггеры и цепочки узлов.

```
Помоги настроить рабочий процесс, который автоматически списывает товар со склада после создания заказа
```

![Рабочий процесс списания товара по заказу](https://static-docs.nocobase.com/20260419234303.png)

Подробнее о рабочих процессах см. [Управление рабочими процессами](./workflow).

### Таблицы данных, страницы и панели — всё сразу

:::warning Примечание

Функция Решения сейчас на этапе тестирования с ограниченной стабильностью, доступна только для раннего доступа.

:::

Опишите бизнес-сценарий одной фразой — и ИИ сразу построит таблицы данных, страницы управления, панели и диаграммы.

```
Помоги использовать навык nocobase-dsl-reconciler, чтобы построить систему управления заявками с панелью, списком заявок, управлением пользователями и настройкой SLA
```

Сначала ИИ выводит план проектирования, затем после подтверждения строит всё сразу:

![План проектирования системы заявок](https://static-docs.nocobase.com/20260420100420.png)

![Результат построения системы заявок](https://static-docs.nocobase.com/20260420100450.png)

Подробнее о построении целых систем см. [Решения](./dsl-reconciler).

### Создайте веху — и ИИ сохранит восстанавливаемую версию

После завершения страницы, набора таблиц данных или рабочего процесса попросите ИИ сохранить текущее состояние как версию — если конфигурация пойдёт не так, всегда можно откатиться к последней понятной вехе.

```
Сохрани текущую сборку как версию: завершена страница управления клиентами, область фильтров и конфигурация формы редактирования
```

![ИИ создаёт версию после построения](https://static-docs.nocobase.com/20260611115804.png)

ИИ не сохраняет версию при каждом изменении поля; версия создаётся только после завершения и проверки понятной вехи — так список версий остаётся читаемым, и проще решить, куда откатываться.

Подробнее о контроле версий см. [Контроль версий](./version-control).

## Безопасность и аудит

Перед тем как позволить ИИ агенту управлять NocoBase, рекомендуем разобраться со способами аутентификации, контролем прав доступа и аудитом операций — чтобы ИИ делал только то, что нужно, а каждый шаг был зафиксирован. См. [Безопасность и аудит](./security).

## NocoBase Skills

[NocoBase Skills](https://github.com/nocobase/skills) — пакеты предметных знаний, устанавливаемые в ИИ агенты, которые позволяют ИИ понимать систему конфигурации NocoBase. NocoBase предоставляет 9 Навыков, охватывающих весь процесс разработки:

- [Управление окружением](./env-bootstrap) — проверка окружения, установка, развёртывание, обновление и устранение неполадок
- [Моделирование данных](./data-modeling) — создание и управление таблицами данных, полями и связями
- [Настройка интерфейса](./ui-builder) — создание и редактирование страниц, блоков, всплывающих окон и интерактивных связок
- [Управление рабочими процессами](./workflow) — создание, редактирование, включение и диагностика рабочих процессов
- [Настройка ACL](./acl) — управление ролями, политиками прав доступа, привязкой пользователей и оценкой рисков
- [Решения](./dsl-reconciler) — пакетное построение целых бизнес-систем из YAML
- [Управление плагинами](./plugin-manage) — просмотр, включение и отключение плагинов
- [Управление релизами](./publish) — кросс-окружное развёртывание, резервное копирование и восстановление, миграция
- [Контроль версий](./version-control) — сохранение восстанавливаемых версий после завершённых вех

:::tip

NocoBase CLI автоматически устанавливает Skills при инициализации (`nb init`), поэтому ручная установка не требуется.

:::
