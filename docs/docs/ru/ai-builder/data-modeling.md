# Моделирование данных

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

## Введение

Навык моделирования данных позволяет создавать и управлять таблицами данных NocoBase на естественном языке — создавать таблицы, добавлять поля, настраивать связи и многое другое.

Перед использованием убедитесь, что целевой источник данных настроен в «Менеджере источников данных».


## Возможности

- Создание, изменение и удаление таблиц данных: общие таблицы, древовидные таблицы, файловые таблицы, календарные таблицы, SQL-таблицы, таблицы представлений и таблицы наследования
- Добавление, изменение и удаление полей, включая все встроенные типы полей NocoBase (в том числе поля связей) и типы полей, расширяемые плагинами

## Примеры запросов

### Сценарий A: создание таблицы данных

```
Создай файловую таблицу для управления договорами
```

Навык подскажет ИИ проанализировать поля, необходимые для таблицы данных, и соответствующие типы полей NocoBase, затем создать файловую таблицу и добавить нужные поля в системе.

![Создание таблицы данных](https://static-docs.nocobase.com/202604162103369.png)

### Сценарий B: добавление полей

```
Добавь в таблицу users поле статуса, показывающее, активен ли пользователь, с тремя значениями: Активен, Увольняется и Уволен
```

Навык подскажет ИИ получить метаданные таблицы users и определить, что поле статуса занятости соответствует типу поля «Выпадающий список (одиночный выбор)» в NocoBase, затем добавить поле в таблицу users и настроить значения перечисления.

![Добавление полей](https://static-docs.nocobase.com/202604162112692.png)

### Сценарий C: инициализация модели данных

```
Я создаю CRM, помоги спроектировать и построить модель данных
```

Навык создаст в системе таблицы данных, добавит поля и настроит связи на основе модели данных, спроектированной ИИ.

![Инициализация модели данных](https://static-docs.nocobase.com/202604162126729.png)


Навык подскажет ИИ получить текущую модель данных системы и на её основе спроектировать моделирование данных для новой функции, затем автоматически создать таблицы данных, добавить поля и настроить связи.

## Часто задаваемые вопросы

**Создаются ли системные поля автоматически при создании таблицы?**

Да. Системные поля вроде `id`, `createdAt`, `createdBy`, `updatedAt` и `updatedBy` автоматически генерируются сервером и не требуют ручного указания.

**Как исправить неправильно настроенную связь?**

Рекомендуется сначала проверить внешний ключ поля связи и обратное поле, затем решить, изменять его или удалить и создать заново. Навык после любых изменений считывает и проверяет состояние связи с обеих сторон.

**Как создать таблицу данных на основе типа таблицы, расширяемого плагином?**

Соответствующий плагин должен быть включён. Если он не включён, ИИ обычно попытается включить плагин. Если ИИ не справится, включите плагин вручную.
