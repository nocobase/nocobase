# Настройка интерфейса

:::tip Предварительные требования

Перед чтением этой страницы убедитесь, что вы установили NocoBase CLI и завершили инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md).

:::

## Введение

Навык настройки интерфейса создаёт и редактирует страницы, блоки, поля и конфигурации действий NocoBase — вы описываете нужную страницу на языке бизнеса, а он берёт на себя генерацию чертежа, макет блоков и интерактивную связку.


## Возможности

Можно:

- Создавать полные страницы: таблицы, формы фильтра и всплывающие окна деталей за один шаг
- Редактировать существующие страницы: добавлять блоки, настраивать поля, конфигурировать всплывающие окна, изменять макеты
- Настраивать интерактивную связку: значения по умолчанию, видимость полей, вычисляемую связку, состояния кнопок действий
- Использовать повторное применение шаблонов: сохранять повторяющиеся всплывающие окна и блоки как шаблоны
- Поддерживать многостраничные задачи: последовательно строить страницы по порядку

Нельзя:

- Нельзя настраивать права доступа ACL (используйте [навык настройки ACL](./acl))
- Нельзя проектировать структуры таблиц данных (используйте [навык моделирования данных](./data-modeling))
- Нельзя оркестрировать рабочие процессы (используйте [навык управления рабочими процессами](./workflow))
- Нельзя работать с навигацией устаревших страниц (v1), поддерживаются только страницы v2.

## Примеры запросов

### Сценарий A: создание страницы управления

```
Помоги создать страницу управления клиентами с полем поиска по имени и таблицей клиентов с колонками: имя, телефон, email и время создания
```

Навык сначала прочитает поля коллекции, затем сгенерирует и запишет чертёж страницы.

![Создание страницы управления](https://static-docs.nocobase.com/20260420100608.png)


### Сценарий B: настройка всплывающих окон

```
При клике по имени клиента в таблице открывай всплывающее окно с деталями, показывающее все поля
```

Он предпочтёт всплывающее окно поля (клик для открытия), а не добавление отдельной кнопки действия.

![Настройка всплывающих окон](https://static-docs.nocobase.com/20260420100641.png)

### Сценарий C: настройка правил связки

```
Добавь правило поля в форму редактирования во всплывающем окне /admin/c0vc2vmkfll/view/cec3e7a69ac/filterbytk/1:
Когда user id равен 1, запрети редактирование username
```

Это реализуется через правила связки без ручного написания конфигурации.

![Настройка правил связки](https://static-docs.nocobase.com/20260420100709.png)

### Сценарий D: многостраничное построение

```
Помоги построить систему управления пользователями с двумя страницами: страницей управления пользователями и страницей управления ролями, сгруппированными в одной группе страниц.
```

Он предложит простой многостраничный дизайн, который можно скорректировать и подтвердить перед построением.

![Многостраничное построение](https://static-docs.nocobase.com/20260420100731.png)

## Часто задаваемые вопросы

**Что делать, если после создания страницы в блоке нет данных?**

Сначала убедитесь, что в соответствующей таблице данных действительно есть записи. Также проверьте, правильно ли у блока привязаны коллекция и источник данных. Можно также использовать [навык моделирования данных](./data-modeling), чтобы напрямую создать тестовые данные.

**Что если мне нужно несколько блоков во всплывающем окне?**

Опишите содержимое всплывающего окна в запросе, например: «Размести форму и связанную таблицу во всплывающем окне редактирования.» Навык сгенерирует пользовательский макет всплывающего окна с несколькими блоками.

**Будут ли мешать друг другу ручная и ИИ-конфигурация?**

Если ручная и ИИ-конфигурация выполняются одновременно, они могут мешать друг другу. Если не одновременно — конфликта не будет.
