# Управление публикацией

:::tip Предварительные требования

- Перед чтением этой страницы убедитесь, что Вы установили NocoBase CLI и выполнили инициализацию по инструкции [Быстрый старт с Конструктором ИИ](./index.md).
- Требуется лицензия Professional Edition и выше [NocoBase Commercial Edition](https://www.nocobase.com/cn/commercial).
- Убедитесь, что плагины управления резервными копиями и управления миграцией активированы и обновлены до последней версии.

:::

:::warning Внимание
Соответствующий CLI для управления публикацией находится в активной разработке и пока не поддерживается для использования.
:::
## Введение

Skill управления публикацией служит для выполнения операций публикации между несколькими окружениями — поддерживает два способа публикации: через резервные копии и через миграцию.


## Возможности

- Восстановление в одном окружении: полное восстановление данных текущей машины из резервной копии.
- Восстановление между окружениями: полное восстановление данных целевого окружения из резервной копии.
- Миграция между окружениями: инкрементальное обновление данных целевого окружения с помощью нового пакета миграции.

## Примеры промптов

### Сценарий A: восстановление в одном окружении
:::tip Предварительные требования

В текущем окружении должна быть резервная копия, либо нужно сначала создать копию, а затем восстановить из неё

:::

Промпт-режим
```
Восстанови из <file-name>
```
Командный режим
```
// Просмотрите доступные резервные копии; если их нет — выполните nb backup <file-name>
nb backup list
nb restore <file-name>
```
![Восстановление из копии](https://static-docs.nocobase.com/20260417150854.png)

### Сценарий B: восстановление между окружениями

:::tip Предварительные требования

Необходимы два окружения: например, локальное окружение dev и удалённое test, либо два локальных окружения.

:::

Промпт-режим
```
Восстанови dev в test
```
Командный режим
```
// Просмотрите доступные резервные копии; если их нет — выполните nb backup <file-name> --env dev
nb backup list --env dev
// Восстановление из резервной копии
nb restore <file-name> --env test
```
![Восстановление из копии](https://static-docs.nocobase.com/20260417150854.png)

### Сценарий C: миграция между окружениями

:::tip Предварительные требования

Аналогично сценарию B, нужно подготовить два окружения: например, локальное dev и удалённое test, либо два локальных.

:::

Промпт-режим
```
Перенеси dev в test
```
Командный режим
```
// Создайте новое правило миграции; будет получен новый ruleId, либо посмотрите историю через nb migration rule list --env dev
nb migration rule add --env dev
// Сгенерируйте пакет миграции по ruleId
nb migration generate <ruleId> --env dev
// Выполните миграцию по пакету
nb migration run <file-name> --env test
```
![Публикация миграции](https://static-docs.nocobase.com/20260417151022.png)

## Часто задаваемые вопросы

**Что выбрать — восстановление или миграцию?**

Если у Вас уже есть готовая резервная копия — выбирайте восстановление. Если нужно по правилам определять, какие данные синхронизировать (например, только структура без данных), выбирайте миграцию.

**Что делать, если нет плагина миграции?**

Плагин управления миграцией требует Professional Edition или выше — подробнее см. [NocoBase Commercial Edition](https://www.nocobase.com/cn/commercial).
