# Управление релизами

:::tip Предварительные требования

- Перед чтением этой страницы установите NocoBase CLI и завершите инициализацию, как описано в [Быстрый старт Мастера ИИ-разработки](./index.md)
- Требуется лицензия редакции Professional или выше. См. [коммерческая редакция NocoBase](https://www.nocobase.com/en/commercial)
- Включите плагины «Управление резервным копированием» и «Менеджер миграций» и обновите их до последней версии

:::

## Введение

Навык управления релизами выполняет операции развёртывания между несколькими окружениями NocoBase. Поддерживаются два подхода: восстановление из резервной копии и миграция.

Если нужно полностью перезаписать одно окружение другим, обычно достаточно восстановления из резервной копии. Если нужно управлять тем, что синхронизируется по правилам — например, синхронизировать только структуру без бизнес-данных — подходит миграция.

## Возможности

- Восстановление из резервной копии в одном окружении: восстановление текущего окружения с использованием существующего пакета резервной копии
- Мгновенное восстановление из резервной копии в одном окружении: сначала создать резервную копию текущего окружения, затем восстановить текущее окружение из неё
- Кросс-окружное восстановление из резервной копии: восстановление пакета резервной копии из исходного окружения в целевое
- Кросс-окружная миграция: инкрементальное обновление целевого окружения пакетом миграции

## Примеры запросов

### Сценарий A: восстановление из резервной копии в одном окружении с указанием файла

:::tip Предварительные требования

В текущем окружении уже должен существовать файл резервной копии с таким же именем.

:::

```text
Восстанови из резервной копии <file-name.nbdata>
```

Навык использует для восстановления файл резервной копии с таким же именем, который уже есть на сервере текущего окружения.

### Сценарий B: восстановление из резервной копии в одном окружении без указания файла

```text
Создай резервную копию и восстанови текущее окружение
```

Навык сначала создаёт резервную копию текущего окружения, затем восстанавливает текущее окружение из этой копии.

### Сценарий C: кросс-окружное восстановление из резервной копии

:::tip Предварительные требования

Подготовьте два окружения, например локальное окружение dev и удалённое тестовое, или два локальных окружения. Можно указать файл резервной копии или не указывать.

:::

```text
Восстанови dev в test
```

Навык создаёт пакет резервной копии в окружении dev, затем восстанавливает этот пакет в окружении test.

### Сценарий D: кросс-окружная миграция

:::tip Предварительные требования

Аналогично сценарию C, подготовьте два окружения. Можно указать файл миграции или не указывать.

:::

```text
Мигрируй dev в test
```

Навык создаёт пакет миграции в окружении dev, затем использует этот пакет для обновления окружения test.

## Часто задаваемые вопросы

**Что выбрать — восстановление из резервной копии или миграцию?**

Восстановление из резервной копии — выбор по умолчанию, особенно если у вас уже есть пригодный пакет резервной копии или нужно полностью перезаписать целевое окружение состоянием исходного. Используйте миграцию только когда нужно управлять областью синхронизации по правилам, например синхронизировать только структуру без данных.

**Что означает отсутствие плагина миграции?**

Плагин «Менеджер миграций» требует лицензии редакции Professional или выше. Подробнее см. [коммерческая редакция NocoBase](https://www.nocobase.com/en/commercial).
