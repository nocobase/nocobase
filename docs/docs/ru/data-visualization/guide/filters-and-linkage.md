# Фильтры страницы и взаимодействие

Фильтр страницы (Filter Block) используется для унифицированного ввода условий фильтрации на уровне страницы и их объединения с запросами диаграмм, что обеспечивает согласованную фильтрацию и взаимодействие между несколькими диаграммами.

## Обзор функций
- Добавьте «блок фильтра» на страницу, чтобы обеспечить единый интерфейс фильтрации для всех диаграмм на текущей странице.
- Используйте кнопки «Фильтровать», «Сбросить» и «Свернуть» для применения, очистки и сворачивания.
- Если в фильтре выбраны поля, связанные с диаграммой, их значения автоматически объединяются с запросом диаграммы, что запускает её обновление.
- Фильтры также могут создавать пользовательские поля, регистрировать их в контекстных переменных и ссылаться на них в диаграммах, таблицах, формах и других блоках данных.

## Использование значений фильтров страницы в запросах диаграмм
- Режим Builder (рекомендуется)
  - Автоматическое объединение: Если источник данных и коллекция совпадают, вам не нужно дополнительно указывать переменные в запросе диаграммы; фильтры страницы будут объединены с помощью `$and`.
  - Ручной выбор: Вы также можете вручную выбрать значения «пользовательских полей» блока фильтра в условиях фильтрации диаграммы.

- Режим SQL (через внедрение переменных)
  - В SQL-запросе используйте «Выбрать переменную», чтобы вставить значения из «пользовательских полей» блока фильтра.
