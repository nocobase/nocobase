# Фильтры страницы и связывание

Фильтр страницы (блок фильтра) предоставляет единый ввод условий фильтрации на уровне страницы и объединяет их в запросы графиков, чтобы поддерживать согласованную фильтрацию и связывание нескольких графиков.

## Обзор возможностей
- Добавьте блок фильтра на страницу, чтобы предоставить единую точку входа фильтра для всех графиков.
- Используйте кнопки «Фильтр», «Сброс» и «Свернуть», чтобы применить фильтр, очистить его и свернуть.
- Если фильтр выбирает поля, связанные с графиком, их значения автоматически объединяются в запрос графика и запускают обновление.
- Фильтры могут определять пользовательские поля и регистрировать их в контекстных переменных, чтобы на них можно было ссылаться в графиках, таблицах, формах и других блоках данных.

## Использование значений фильтра страницы в запросах графиков
- Режим Конструктор (рекомендуется)
  - Автоматическое объединение: когда источник данных и коллекция совпадают, вам не нужно писать переменные в запросе графика; фильтры страницы объединяются через `$and`.
  - Ручной выбор: вы также можете выбирать значения из пользовательских полей блока фильтра в условиях фильтра графика.

- Режим SQL (через подстановку переменных)
  - В SQL используйте вобор переменной, чтобы вставлять значения из пользовательских полей блока фильтра.
