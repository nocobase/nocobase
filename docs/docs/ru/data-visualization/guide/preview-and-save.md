# Предварительный просмотр и сохранение

*   **Предварительный просмотр**: Временно отображает изменения из панели конфигурации на диаграмме страницы для проверки результата.
*   **Сохранение**: Фактически сохраняет изменения из панели конфигурации в базу данных.

## Точки доступа

![clipboard-image-1761479218](https://static-docs.nocobase.com/clipboard-image-1761479218.png)

*   В режиме графической конфигурации (Basic) все изменения по умолчанию автоматически применяются в предварительном просмотре.
*   В режимах SQL и Custom после внесения изменений вы можете нажать кнопку **Предварительный просмотр** справа, чтобы применить их.
*   Унифицированная кнопка **Предварительный просмотр** также доступна в нижней части всей панели конфигурации.

## Поведение предварительного просмотра
*   Временно отображает конфигурацию на странице, но не записывает ее в базу данных. После обновления страницы или отмены операции результат предварительного просмотра не сохраняется.
*   **Встроенная защита от дребезга**: при многократном запуске обновления за короткий промежуток времени выполняется только последнее, чтобы избежать частых запросов.
*   Повторное нажатие кнопки **Предварительный просмотр** перезаписывает предыдущий результат.

## Сообщения об ошибках
*   **Ошибки запроса или сбои проверки**: отображаются в области "Просмотр данных".
*   **Ошибки конфигурации диаграммы** (отсутствие сопоставления Basic, ошибки в Custom JS): отображаются в области диаграммы или в консоли, при этом страница остается работоспособной.
*   Рекомендуется сначала подтвердить имена столбцов и типы данных в разделе "Просмотр данных", а затем выполнять сопоставление полей или писать Custom код, чтобы эффективно уменьшить количество ошибок.

## Сохранение и отмена
*   **Сохранить**: записывает текущие изменения в конфигурацию блока и немедленно применяет их к странице.
*   **Отменить**: отменяет текущие несохраненные изменения на панели и возвращает к последнему сохраненному состоянию.
*   **Область сохранения**:
    *   **Запрос данных**: параметры Builder; в режиме SQL также сохраняется текст SQL-запроса.
    *   **Параметры диаграммы**: тип Basic, сопоставление полей и свойства; текст Custom JS.
    *   **Интерактивные события**: текст JS и логика привязки событий.
*   После сохранения блок становится доступным для всех посетителей (в зависимости от настроек прав доступа к странице).

## Рекомендуемый порядок действий
*   Настройте запрос данных → Выполните запрос → Проверьте данные, чтобы подтвердить имена столбцов и типы → Настройте параметры диаграммы для сопоставления основных полей → Выполните предварительный просмотр для проверки → Сохраните, чтобы применить изменения.
