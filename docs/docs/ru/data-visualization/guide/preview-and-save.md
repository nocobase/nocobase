# Предпросмотр и сохранение

- Предпросмотр: временно отображает изменения из панели конфигурации в графике на странице для проверки результата.
- Сохранить: сохраняет изменения из панели конфигурации в базу данных.

## Точки входа

![clipboard-image-1761479218](https://static-docs.nocobase.com/clipboard-image-1761479218.png)

- В визуальном (базовом) режиме изменения по умолчанию автоматически применяются к предпросмотру.
- В SQL и пользовательском режиме нажмите кнопку «Предпросмотр» справа, чтобы применить изменения к предпросмотру.
- Внизу панели конфигурации также доступна единая кнопка «Предпросмотр».

## Поведение предпросмотра
- Конфигурация временно отображается на странице без записи в базу данных. После обновления страницы или нажатия «Отмена» результат предпросмотра не сохраняется.
- Встроенная задержка выполнения: при множественных триггерах обновления за короткое время выполняется только последний, чтобы избежать частых запросов.
- Повторный клик по «Предпросмотр» перезаписывает предыдущий результат предпросмотра.

## Сообщения об ошибках
- Ошибки запроса или валидации: отображаются в области «Просмотр данных».
- Ошибки конфигурации графика (отсутствует сопоставление в базовом режиме, исключения в пользовательском JS): отображаются в области графика или в консоли, при этом страница остается рабочей.
- Перед сопоставлением полей или написанием пользовательского кода проверьте имена столбцов и типы данных в «Просмотр данных», чтобы сократить число ошибок.

## Сохранить и Отмена
- Сохранить: записывает текущие изменения в конфигурацию блока и сразу применяет их на странице.
- Отмена: отбрасывает текущие несохраненные изменения и возвращает последнее сохраненное состояние.
- Область сохранения:
  - Запрос данных: параметры Конструктора; в SQL-режиме также сохраняется SQL-текст.
  - Параметры графика: тип базового режима, сопоставление полей и свойства; текст пользовательского JS.
  - События взаимодействия: JS-текст и логика привязки.
- После сохранения блок становится активным для всех посетителей (с учетом прав страницы).

## Рекомендуемый порядок работы 
- Настроить «Запрос данных» → «Выполнить запрос» → «Просмотр данных» для проверки имен столбцов и типов → Настроить «Параметры графика» для сопоставления основных полей → «Предпросмотр» для валидации → «Сохранить» для применения.
