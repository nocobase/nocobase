# Использование контекстных переменных

С помощью контекстных переменных вы можете повторно использовать информацию о текущей странице, пользователе, времени, условиях фильтрации и т.д., для рендеринга диаграмм и обеспечения их взаимосвязи в зависимости от контекста.

## Область применения
- В условиях фильтрации в режиме Builder для запросов данных выбирайте переменные.
![clipboard-image-1761486073](https://static-docs.nocobase.com/clipboard-image-1761486073.png)

- При написании SQL-запросов в режиме SQL выбирайте переменные и вставляйте выражения (например, `{{ ctx.user.id }}`).
![clipboard-image-1761486145](https://static-docs.nocobase.com/clipboard-image-1761486145.png)

- В опциях диаграмм в режиме Custom непосредственно пишите JS-выражения.
![clipboard-image-1761486604](https://static-docs.nocobase.com/clipboard-image-1761486604.png)

- В интерактивных событиях (например, при нажатии для детализации, открытия всплывающего окна и передачи данных) непосредственно пишите JS-выражения.
![clipboard-image-1761486683](https://static-docs.nocobase.com/clipboard-image-1761486683.png)

**Обратите внимание:**
- Не заключайте `{{ ... }}` в одинарные или двойные кавычки; система безопасно обрабатывает привязку в зависимости от типа переменной (строка, число, время, NULL).
- Если переменная имеет значение `NULL` или не определена, явно обрабатывайте логику пустых значений в SQL, используя `COALESCE(...)` или `IS NULL`.
