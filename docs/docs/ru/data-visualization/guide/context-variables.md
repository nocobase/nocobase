# Использование контекстных переменных

С помощью контекстных переменных вы можете переиспользовать информацию с текущей страницы, от пользователя, времени и входных данных фильтра, чтобы отрисовывать графики и включать связывание на основе контекста.

## Область применения
- Запрос данных в режиме Конструктор: выбирайте переменные для условий фильтра.
![clipboard-image-1761486073](https://static-docs.nocobase.com/clipboard-image-1761486073.png)

- Запрос данных в SQL-режиме: выбирайте переменные и вставляйте выражения (например, `{{ ctx.user.id }}`).
![clipboard-image-1761486145](https://static-docs.nocobase.com/clipboard-image-1761486145.png)

- Параметры графика в пользовательском режиме: пишите JS-выражения напрямую.
![clipboard-image-1761486604](https://static-docs.nocobase.com/clipboard-image-1761486604.png)

- События взаимодействия (например, клик для открытия деталей и передачи данных): пишите JS-выражения напрямую.
![clipboard-image-1761486683](https://static-docs.nocobase.com/clipboard-image-1761486683.png)

**Примечание:**
- Не оборачивайте `{{ ... }}` в одинарные или двойные кавычки; привязка безопасно обрабатывается на основе типа переменной (string, number, time, NULL).
- Когда переменная равна `NULL` или undefined, явно обрабатывайте null в SQL с помощью `COALESCE(...)` или `IS NULL`.
