# Быстрый старт ИИ-разработки плагинов

ИИ-разработка плагинов — это возможность разработки плагинов с помощью ИИ, предоставляемая NocoBase: вы описываете требования на естественном языке, а ИИ автоматически генерирует полный клиентский и серверный код, включая коллекции, API, клиентские блоки, права доступа и интернационализацию. Это более современный и эффективный опыт разработки плагинов.

Возможность ИИ-разработки плагинов работает на основе навыка [nocobase-plugin-development](https://github.com/nocobase/skills/tree/main/skills/nocobase-plugin-development). Если вы уже выполнили инициализацию через NocoBase CLI (`nb init`), этот навык устанавливается автоматически.

## Быстрый старт

Если вы уже установили [NocoBase CLI](../ai/quick-start.md), этот шаг можно пропустить.

### Установка в один клик с помощью ИИ

Скопируйте следующий запрос в ваш ИИ-ассистент (Claude Code, Codex, Cursor, Trae и т. д.), чтобы автоматически завершить установку и настройку:

```
Помоги мне установить NocoBase CLI и завершить инициализацию: https://docs.nocobase.com/ai/ai-quick-start.md (пожалуйста, прочитай содержимое по ссылке напрямую)
```

### Ручная установка

```bash
npm install -g @nocobase/cli
nb init --ui
```

Браузер автоматически откроет страницу визуальной настройки, которая проведёт вас через установку NocoBase Skills, настройку базы данных и запуск приложения. Подробные шаги см. в разделе [Быстрый старт](../ai/quick-start.md).

:::warning Примечание

- NocoBase переходит с `client` (v1) на `client-v2`, при этом `client-v2` всё ещё находится в разработке. Клиентский код, генерируемый при ИИ-разработке, основан на `client-v2` и может использоваться только по пути `/v/`. Он доступен для раннего доступа и экспериментов, но не рекомендуется для использования в продакшене.
- Сгенерированный ИИ код может быть не на 100% корректным. Рекомендуем проверить его перед включением. Если при выполнении возникают ошибки, отправьте сообщение об ошибке ИИ — он проанализирует и исправит проблему; обычно достаточно нескольких раундов диалога.
- Для разработки рекомендуем использовать модели серии GPT или Claude — они дают лучшие результаты. Другие модели тоже подойдут, но качество генерации может отличаться.

:::

## От одной фразы до готового плагина

После установки вы можете напрямую описать ИИ на естественном языке, какой плагин хотите разработать. Ниже — несколько реальных сценариев, чтобы вы могли оценить возможности ИИ-разработки плагинов.

### Создание плагина водяного знака одной фразой

Одним запросом ИИ поможет сгенерировать полноценный плагин водяного знака — включая логику отрисовки на клиенте, обнаружение вмешательства, API хранения настроек на сервере и страницу настроек плагина.

```
Помоги мне использовать навык nocobase-plugin-development для разработки плагина водяного знака NocoBase.
Требования: наложить полупрозрачный водяной знак на страницу с именем текущего вошедшего пользователя, чтобы предотвратить утечку данных через скриншоты.
Периодически проверять, не удалён ли DOM водяного знака, и при необходимости восстанавливать его.
Поддержать настройку текста водяного знака, прозрачности и размера шрифта на странице настроек плагина.
Имя плагина — @my-project/plugin-watermark
```

### Создание пользовательского компонента поля одной фразой

Нужно отображать целочисленное поле в виде звёздного рейтинга? Опишите ИИ желаемый эффект отображения — и он сгенерирует пользовательский FieldModel для замены стандартного компонента отрисовки поля.

```
Помоги мне использовать навык nocobase-plugin-development для разработки плагина NocoBase с именем @my-scope/plugin-rating.
Создай пользовательский компонент отображения поля (FieldModel), который отображает поля целочисленного типа в виде звёзд,
с поддержкой рейтинга от 1 до 5. Клик по звезде должен напрямую изменять значение рейтинга и сохранять его в базу данных.
```

## Что ИИ может сделать для вас

| Я хочу…                              | ИИ может помочь вам                                                                 |
| ------------------------------------ | ----------------------------------------------------------------------------------- |
| Создать новый плагин                 | Сгенерировать полный каркас, включая структуру каталогов клиентской и серверной части |
| Определить коллекции                 | Сгенерировать определения Collection со всеми типами полей и связями                |
| Создать пользовательский блок        | Сгенерировать BlockModel + панель настроек + зарегистрировать в меню «Добавить блок» |
| Создать пользовательское поле        | Сгенерировать FieldModel + привязать к интерфейсу поля                              |
| Добавить пользовательскую кнопку действия | Сгенерировать ActionModel + всплывающее окно/выдвижной блок/диалог подтверждения |
| Создать страницу настроек плагина    | Сгенерировать форму на клиенте + API на сервере + хранилище                         |
| Написать пользовательский API        | Сгенерировать Resource Action + регистрацию маршрута + конфигурацию ACL             |
| Настроить права доступа              | Сгенерировать правила ACL для управления доступом по ролям                            |
| Поддержать несколько языков          | Автоматически сгенерировать языковые пакеты на китайском и английском               |
| Написать скрипт миграции             | Сгенерировать Migration с поддержкой DDL и миграции данных                            |

Подробные описания и примеры запросов для каждой возможности -> [Поддерживаемые возможности](./capabilities)
