# Товары, предложения и Заказы

> Эта глава охватывает вторую половину процесса продаж: ведение каталога Товаров, создание и согласование Предложений, а также исполнение Заказов и контроль платежей. Предложения также упоминаются в [Управлении Сделками](./guide-opportunities) (с точки зрения Сделок), а здесь акцент сделан на Товарах и Заказах.

![cn_03-products-orders](https://yifan2-500g.oss-cn-hangzhou.aliyuncs.com/crm-guide-mermaid/cn_03-products-orders.png)

## Каталог Товаров

В верхнем меню перейдите на страницу **Товары**. Она содержит две вкладки:

### Список Товаров

Слева — дерево категорий (JS-фильтр), справа — таблица Товаров. Каждый Товар содержит: имя, код, категорию, спецификацию, единицу, прайсовую цену, себестоимость, валюту.

![03-products-orders-2026-04-07-01-18-03](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-18-03.png)

При создании Товара, помимо базовой информации, можно настроить **подтаблицу ступенчатого ценообразования**:

| Поле | Описание |
|------|------|
| Валюта | Валюта цены |
| Минимальное количество | Начальное количество для этого ценового уровня |
| Максимальное количество | Верхний предел количества для этого ценового уровня |
| Цена за единицу | Цена за единицу для этого диапазона количества |
| Скидка | Доля скидки за объём |


![03-products-orders-2026-04-07-01-18-51](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-18-51.png)

При создании Предложения после выбора Товара система автоматически подбирает ступенчатую цену по количеству.

![03-products-orders-2026-04-07-01-19-39](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-19-39.png)

### Управление категориями

Вторая вкладка — древовидная таблица категорий Товаров с поддержкой многоуровневой вложенности. Кликните «Добавить подкатегорию», чтобы создать подкатегорию под текущим узлом.

![03-products-orders-2026-04-07-01-20-19](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-20-19.png)

## Предложения

Предложения обычно создаются из подробностей Сделки (см. раздел «Создание Предложения» в [Сделках и Предложениях](./guide-opportunities)); здесь же дополнены сведения о позициях Товаров и процессе согласования.

### Позиции Товаров

В подтаблице позиций Предложения после выбора Товара несколько полей заполняются автоматически:

| Поле | Описание |
|------|------|
| **Товар** | Выбор из каталога Товаров |
| **Спецификация** | Только для чтения, заполняется автоматически после выбора Товара |
| **Единица** | Только для чтения, заполняется автоматически |
| **Количество** | Заполняется вручную |
| **Прайсовая цена** | Только для чтения, цена из каталога Товаров |
| **Цена за единицу** | Только для чтения, автоматически подбирается по количеству согласно ступенчатому ценообразованию |
| **Скидка** | Только для чтения, скидка из ступенчатого ценообразования |
| **Сумма по строке** | Рассчитывается автоматически |

![03-products-orders-2026-04-07-01-22-22](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-22-22.gif)

Система автоматически выполняет цепочку расчётов: подытог → скидка → налог → доставка → итоговая сумма → эквивалент в USD.

![03-products-orders-2026-04-07-01-23-13](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-23-13.gif)

### Поддержка нескольких валют

Если Клиент рассчитывается не в долларах США, выберите соответствующую валюту. Система при создании **фиксирует текущий обменный курс** и автоматически пересчитывает сумму в эквивалент USD. Управление обменными курсами выполняется на странице **Настройки → Курсы валют**.

### Согласование

Созданное Предложение проходит согласование, после одобрения создаётся Заказ.

## Управление Заказами

В верхнем меню перейдите на страницу **Заказы**. Также можно создать Заказ прямо из подробностей Сделки, нажав «New Order» на одобренном Предложении.

### Список Заказов

В верхней части страницы расположены кнопки фильтрации:

| Кнопка | Значение |
|------|------|
| **Все** | Все Заказы |
| **В обработке** | Заказы, исполняющиеся в данный момент |
| **Ожидают оплаты** | Ожидают оплаты от Клиента |
| **Отгружены** | Отгружены, ждут подтверждения получения |
| **Завершены** | Процесс завершён |

![03-products-orders-2026-04-07-01-25-09](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-25-09.png)

### Столбец прогресса Заказа

В столбце «Прогресс Заказа» текущий статус наглядно отображается визуальным точечно-линейным индикатором:

```
Ожидает подтверждения → Подтверждён → В обработке → Отгружён → Завершён
```

Завершённые шаги подсвечены, ещё не пройденные — затемнены.

### Строка итогов

Итоговая информация в нижней части таблицы:

- **Сумма выбранных / всех Заказов**
- **Распределение статусов оплаты** (в виде меток)
- **Распределение статусов Заказов** (в виде меток)

![03-products-orders-2026-04-07-01-25-51](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-25-51.png)

### Создание Заказа

**Из Предложения в Заказ (рекомендуется)**: в подробностях Сделки на одобренном Предложении (статус Approved) появляется кнопка «New Order» — после клика система автоматически переносит данные о Клиенте, позициях Товаров, суммах, валюте, обменном курсе и т. п.

![03-products-orders-2026-04-07-01-27-16](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-27-16.png)

**Создание вручную**: на странице списка Заказов нажмите «Создать», затем заполните Клиента, позиции Товаров, сумму Заказа, условия оплаты.

### Перевод статусов Заказа

Клик по строке Заказа открывает окно с подробностями. В верхней части — интерактивный поток статусов; кликните следующий узел статуса для перевода. Каждое изменение статуса фиксируется системой.

![03-products-orders-2026-04-07-01-27-50](https://static-docs.nocobase.com/03-products-orders-2026-04-07-01-27-50.png)

### Отслеживание оплаты

Статус Заказа и статус оплаты — две независимые цепочки:

- **Статус Заказа**: Подтверждение → Обработка → Отгрузка → Завершение (процесс исполнения)
- **Статус оплаты**: Ожидает оплаты → Частично оплачен → Оплачен (процесс поступления платежей)

Сейчас мы делаем акцент на фронтенд-процессах CRM и не накладываем условных ограничений на статус Заказа — он используется как информационная отметка. При необходимости можно настроить контроль через правила связывания и события таблиц данных.
