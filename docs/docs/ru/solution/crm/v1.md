# Устаревшая CRM (v1)

> **Только для предварительного просмотра**: Эта версия функционально завершена, однако общая структура таблиц данных не является оптимальной, отсутствует поддержка многоязычности, а коллекции CRM и управления проектами тесно связаны между собой. Эти проблемы привели к переработке CRM 2.0. В настоящее время рекомендуется использовать [CRM 2.0](./index.md), которая имеет более рациональную структуру данных, легче расширяется и модифицируется, а также полностью локализована.

## Особенности версии

Устаревшая версия CRM включает следующие функциональные модули:

- **Управление клиентами**: информация о клиентах, контакты, классификация клиентов.
- **Воронка продаж**: управление лидами, отслеживание сделок, воронка продаж.
- **Управление заказами**: коммерческие предложения, заказы, контракты.
- **Управление проектами** (пока не включено в версию v2).
- **Коммуникация и совместная работа**: электронная почта, комментарии, процессы утверждения.
- **Анализ данных**: дашборды, отчеты, визуализация данных.

## Как установить

> Текущая версия развертывается путем резервного копирования и восстановления.

Перед началом убедитесь, что:

- У вас есть базовая среда выполнения NocoBase. Для установки основной системы обратитесь к подробной [официальной документации по установке](https://docs-cn.nocobase.com/welcome/getting-started/installation).
- Версия NocoBase **v2.1.0-beta.2 или выше**.
- Вы скачали соответствующие файлы CRM v1:
  - **Файл резервной копии**: [nocobase_crm_v1_backup_260224.nbdata](https://static-docs.nocobase.com/nocobase_crm_v1_backup_260224.nbdata) — применимо для Способа 1.
  - **SQL-файл**: [nocobase_crm_v1_sql_260224.zip](https://static-docs.nocobase.com/nocobase_crm_v1_sql_260224.zip) — применимо для Способа 2.

**Важные примечания**:
- Это решение основано на базе данных **PostgreSQL 16**. Убедитесь, что в вашей среде используется PostgreSQL 16.
- **Параметр DB_UNDERSCORED не должен иметь значение true**: проверьте ваш файл `docker-compose.yml` и убедитесь, что переменная окружения `DB_UNDERSCORED` не установлена в значение `true`. В противном случае возникнет конфликт с резервной копией, что приведет к ошибке восстановления.

## Способ 1: Восстановление с помощью Менеджера резервного копирования (рекомендуется для пользователей версий Pro/Enterprise)

Этот метод использует встроенный плагин «[Менеджер резервного копирования](https://docs-cn.nocobase.com/handbook/backups)» (Pro/Enterprise) для восстановления в один клик. Это самый простой способ.

### Инструкция

**Шаг 1: Включите плагин «Менеджер резервного копирования»**

1. Войдите в свою систему NocoBase.
2. Перейдите в **`Управление плагинами`**.
3. Найдите и включите плагин **`Менеджер резервного копирования`**.

**Шаг 2: Восстановление из локального файла резервной копии**

1. После включения плагина обновите страницу.
2. Перейдите в раздел **`Администрирование системы`** -> **`Менеджер резервного копирования`** в левом меню.
3. Нажмите кнопку **`Восстановить из локальной копии`** в правом верхнем углу.
4. Перетащите скачанный файл резервной копии в область загрузки.
5. Нажмите **`Отправить`** и дождитесь завершения процесса восстановления.

### Примечания

* **Только для версий Pro/Enterprise**: «Менеджер резервного копирования» является плагином корпоративного уровня и доступен только пользователям версий Pro/Enterprise.
* **Совместимость коммерческих плагинов**: Убедитесь, что у вас есть и включены коммерческие плагины, необходимые для этого решения.

## Способ 2: Прямой импорт SQL-файла (универсальный)

Этот метод восстанавливает данные путем прямого взаимодействия с базой данных и подходит для всех пользователей NocoBase.

### Инструкция

**Шаг 1: Подготовьте чистую базу данных**

Подготовьте новую пустую базу данных для импортируемых данных.

**Шаг 2: Импортируйте файл `.sql` в базу данных**

* **Через командную строку (пример для Docker)**:

  ```bash
  # Копирование sql-файла в контейнер
  docker cp nocobase_crm_v1_sql_260224.sql my-nocobase-db:/tmp/
  # Вход в контейнер и выполнение команды импорта
  docker exec -it my-nocobase-db psql -U nocobase -d nocobase -f /tmp/nocobase_crm_v1_sql_260224.sql
  ```

* **Через клиент базы данных**: Используйте такие инструменты, как DBeaver, Navicat или pgAdmin, чтобы подключиться к базе данных и выполнить SQL-файл.

**Шаг 3: Подключитесь к базе данных и запустите приложение**

Настройте параметры запуска NocoBase так, чтобы они указывали на базу данных, в которую были импортированы данные, а затем запустите службу.
