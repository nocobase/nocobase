# Основная база данных

## Введение

База данных, настроенная при [развёртывании NocoBase](/ai/install-nocobase-app), используется для хранения данных системных таблиц NocoBase, а также поддерживает хранение бизнес-данных пользователей.

Поддерживаемые версии баз данных и коммерческие редакции основной базы данных:

| База данных | Поддерживаемая версия | Community | Standard | Professional | Enterprise |
| --- | --- | --- | --- | --- | --- |
| MySQL | >= 8.0.17 | ✅ | ✅ | ✅ | ✅ |
| PostgreSQL | >= 10 | ✅ | ✅ | ✅ | ✅ |
| MariaDB | >= 10.9 | ✅ | ✅ | ✅ | ✅ |
| KingbaseES | >=V9 | ❌ | ❌ | ✅ | ✅ |
| OceanBase | >=4.3 | ❌ | ❌ | ❌ | ✅ |

:::tip Примечание

KingbaseES поддерживает только режим совместимости с PostgreSQL, а OceanBase — только режим совместимости с MySQL.

:::

## Установка плагинов

| База данных | Соответствующий плагин | Способ установки |
| --- | --- | --- |
| MySQL | Нет | Встроенный плагин, отдельная установка не требуется. |
| PostgreSQL | Нет | Встроенный плагин, отдельная установка не требуется. |
| MariaDB | Нет | Встроенный плагин, отдельная установка не требуется. |
| KingbaseES | `@nocobase/plugin-data-source-kingbase` | Требуется коммерческая лицензия; после установки плагин включён по умолчанию. |
| OceanBase | `@nocobase/plugin-data-source-oceanbase` | Требуется коммерческая лицензия; после установки плагин включён по умолчанию. |

## Доступ к источнику основной базы данных

1. Нажмите меню источников данных в системных функциях, чтобы открыть главную страницу источников данных.
2. Выберите источник данных **Main** в списке источников данных и нажмите действие **Настроить**, чтобы открыть основную базу данных для управления.

![configure_main_datasource](https://static-docs.nocobase.com/configure_main_datasource.png)

## Управление источником основной базы данных

Основная база данных предоставляет функции управления таблицами данных: поиск, создание, изменение и удаление таблиц данных, а также синхронизацию полей существующих таблиц из базы данных; поддерживается создание, изменение и удаление полей таблиц данных.
- **Фильтр**: поиск таблиц данных, управляемых в основной базе данных NocoBase
- **Создать таблицу данных**: добавление новой таблицы бизнес-данных
- **Редактировать**: изменение таблицы бизнес-данных
- **Удалить**: удаление таблицы бизнес-данных
- **Синхронизировать из базы данных**: синхронизация структуры существующих таблиц базы данных
- **Настроить поля**: создание, изменение и удаление полей таблицы данных
-  **+** : **+** на вкладке используется для управления категориями таблиц данных, включая создание, изменение и удаление категорий
![main_datasource_management](https://static-docs.nocobase.com/main_datasource_management.png)

### Синхронизация существующих таблиц из базы данных

![nocobase_doc-2025-10-29-19-46-34](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-46-34.png)

Важная особенность источника основной базы данных — возможность синхронизировать уже существующие в базе данных таблицы с NocoBase для дальнейшего управления ими. Это означает:

- **Сохранение существующих инвестиций**: если в вашей базе данных уже есть большое количество бизнес-таблиц, их не нужно создавать заново — можно сразу синхронизировать и использовать
- **Гибкая интеграция**: таблицы, созданные с помощью других инструментов (например, SQL-скриптов, инструментов управления базами данных и т. д.), можно включить в управление NocoBase
- **Постепенная миграция**: существующую систему можно постепенно перенести в NocoBase вместо её полной перестройки за один раз

С помощью функции «Загрузить из базы данных» вы можете:
1. просматривать все таблицы в базе данных
2. выбирать таблицы для синхронизации
3. автоматически распознавать структуру таблиц и типы полей
4. импортировать таблицы в NocoBase одним нажатием для дальнейшего управления

### Поддержка различных типов структур таблиц

![nocobase_doc-2025-10-29-19-47-14](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-47-14.png)

NocoBase поддерживает создание и управление таблицами данных различных типов:
- **Обычная таблица**: содержит встроенные часто используемые системные поля;
- **Таблица-наследник**: можно создать родительскую таблицу, а затем вывести из неё дочерние таблицы. Дочерняя таблица наследует структуру родительской, при этом в ней можно определить собственные столбцы.
- **Древовидная таблица**: таблица с древовидной структурой; в настоящее время поддерживается только модель смежных списков;
- **Календарная таблица**: используется для создания таблиц событий, связанных с календарём;
- **Файловая таблица**: используется для управления хранением файлов;
- **SQL-таблица**: не является фактической таблицей базы данных, а позволяет быстро представить результаты SQL-запроса в структурированном виде;
- **Таблица представления**: подключение к существующему представлению базы данных;

#### Гибкое преобразование типов полей

NocoBase поддерживает гибкое преобразование типов полей в рамках одного и того же типа базы данных.

**Пример: варианты преобразования поля типа String**

Если поле в базе данных имеет тип String, в NocoBase его можно преобразовать в любой из следующих типов:

- **Базовые типы**: однострочный текст, многострочный текст, номер телефона, адрес электронной почты, URL, пароль, цвет, значок
- **Типы выбора**: выпадающий список (одиночный выбор), переключатель
- **Типы мультимедиа**: Markdown, Markdown (Vditor), форматированный текст, вложение (URL)
- **Типы даты и времени**: дата и время (с часовым поясом), дата и время (без часового пояса)
- **Расширенные типы**: автонумерация, селектор таблицы данных, шифрование

Этот механизм гибкого преобразования означает:
- **Изменение структуры базы данных не требуется**: базовый тип хранения поля остаётся неизменным, меняется только его представление в NocoBase
- **Адаптация к изменениям бизнеса**: по мере изменения бизнес-требований можно быстро корректировать способ отображения и взаимодействия с полем
- **Безопасность данных**: процесс преобразования не влияет на целостность существующих данных

### Гибкая синхронизация на уровне полей

NocoBase поддерживает не только синхронизацию всей таблицы, но и детальное управление синхронизацией на уровне отдельных полей:

![nocobase_doc-2025-10-29-19-49-56](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-49-56.png)

#### Особенности синхронизации полей:

1. **Синхронизация в реальном времени**: при изменении структуры таблицы базы данных можно в любой момент синхронизировать новые поля
2. **Выборочная синхронизация**: можно синхронизировать только необходимые поля, а не все поля целиком
3. **Автоматическое распознавание типов**: автоматическое распознавание типов полей базы данных и их сопоставление с типами полей NocoBase
4. **Сохранение целостности данных**: процесс синхронизации не влияет на существующие данные

#### Сценарии использования:

- **Развитие структуры базы данных**: когда бизнес-требования меняются и в базу данных необходимо добавить новые поля, их можно быстро синхронизировать с NocoBase
- **Командная работа**: если другие участники команды или DBA добавили поля в базу данных, их можно своевременно синхронизировать
- **Гибридная модель управления**: часть полей управляется через NocoBase, а часть — традиционными способами, что позволяет гибко комбинировать подходы

Благодаря этому гибкому механизму синхронизации NocoBase легко интегрируется в существующую техническую архитектуру: не требуется менять исходный способ управления базой данных, при этом сохраняются преимущества low-code-разработки NocoBase.
