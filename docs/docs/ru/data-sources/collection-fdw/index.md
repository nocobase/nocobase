# Подключение внешних таблиц данных (FDW)

## Введение

Плагин для подключения удалённых таблиц данных на основе технологии foreign data wrapper базы данных. В настоящее время поддерживаются базы данных MySQL и PostgreSQL.

:::info{title="Подключение источника данных и подключение внешних таблиц данных"}
- **Подключение источника данных** означает установление соединения с определённой базой данных или API-сервисом, что позволяет использовать все возможности базы данных или сервисы, предоставляемые API;
- **Подключение внешних таблиц данных** означает получение данных из внешнего источника и их отображение для использования локально. В базах данных это называется FDW（Foreign Data Wrapper）— технология баз данных, предназначенная для использования удалённых таблиц как локальных. Подключение выполняется только по одной таблице за раз. Поскольку доступ осуществляется удалённо, при использовании существуют различные ограничения и особенности.

Эти два подхода также можно использовать совместно: первый — для установления соединения с источником данных, второй — для доступа к данным из разных источников. Например, можно подключиться к источнику данных PostgreSQL, в котором определённая таблица создана на основе FDW и является внешней таблицей данных.
:::

### MySQL

MySQL использует движок `federated`, который необходимо активировать. Он поддерживает подключение к удалённым MySQL и совместимым с его протоколом базам данных, например MariaDB. Подробнее см. в документации [Federated Storage Engine](https://dev.mysql.com/doc/refman/8.0/en/federated-storage-engine.html).

### PostgreSQL

В PostgreSQL для поддержки различных типов удалённых данных можно использовать расширения `fdw` разных типов. В настоящее время поддерживаются следующие расширения:

- [postgres_fdw](https://www.postgresql.org/docs/current/postgres-fdw.html)：подключение в PostgreSQL к удалённой базе данных PostgreSQL.
- [mysql_fdw（в разработке）](https://github.com/EnterpriseDB/mysql_fdw)：подключение в PostgreSQL к удалённой базе данных MySQL.
- Для расширений fdw других типов см. [PostgreSQL Foreign Data Wrappers](https://wiki.postgresql.org/wiki/Foreign_data_wrappers). Для подключения к NocoBase необходимо реализовать соответствующий адаптационный интерфейс в коде.

## Установка

Предварительные условия

- Если основной базой данных NocoBase является MySQL, необходимо активировать `federated`. См. [Как включить движок federated в MySQL](./enable-federated.md).

Затем установите и активируйте плагин через менеджер плагинов.

## Руководство пользователя

В раскрывающемся списке «Управление таблицами данных > Создать таблицу данных» выберите «Подключить внешние данные».

![Подключение внешних данных](https://static-docs.nocobase.com/029d946a6d067d1c35a39755219d623c.png)

В раскрывающемся списке «Служба базы данных» выберите существующую службу базы данных или «Создать службу базы данных».

![Служба базы данных](https://static-docs.nocobase.com/766271708a911950a5599d60d6be4a4d.png)

Создание службы базы данных

![Создание службы базы данных](https://static-docs.nocobase.com/1e357216e04cc4f200bd6212827281c8.png)

После выбора службы базы данных в раскрывающемся списке «Удалённая таблица» выберите таблицу данных, которую необходимо подключить.

![Выбор таблицы данных для подключения](https://static-docs.nocobase.com/e91fd6152b52b4fc01b3808053cc8dc4.png)

Настройка информации о полях

![Синхронизация с удалённой таблицей](https://static-docs.nocobase.com/3751a9a39f933889fb3fcc4d85a6f4ad.png)
