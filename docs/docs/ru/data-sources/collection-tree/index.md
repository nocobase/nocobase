# Дерево

## Введение

Дерево подходит для хранения данных с иерархическими связями, например организационной структуры, категорий товаров, уровней регионов, каталогов отделов и каталогов базы знаний. Дерево использует структуру смежных списков для хранения родительско-дочерних связей, и каждая запись может указывать на свой родительский узел.

Дерево можно создать только на странице основной базы данных. Внешние базы данных, источники данных REST API и внешние источники данных NocoBase не поддерживают создание деревьев.

## Подходящие сценарии

Дерево подходит для следующих бизнес-сценариев:

- Организационная структура компании, иерархия отделов
- Категории товаров, каталоги базы знаний и документов
- Иерархия провинций, городов и районов, зоны продаж, уровни сервисных центров
- Категории BOM, оборудования и активов

## Создание и настройка

В основной базе данных нажмите «Create collection» и выберите «Tree collection», чтобы создать дерево.

![20240324143228](https://static-docs.nocobase.com/20240324143228.png)

Настройки создания дерева в основном совпадают с настройками обычной таблицы.

| Настройка | Описание |
| --- | --- |
| Collection display name | Название таблицы, отображаемое в интерфейсе, например «Организационная структура», «Категории товаров» или «Уровни регионов». |
| Collection name | Идентификатор таблицы, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. |
| Inherits | Выберите родительскую таблицу, от которой нужно наследовать настройки. Отображается только при использовании PostgreSQL в качестве основной базы данных. |
| Categories | Категории таблицы. Категории влияют только на способ организации интерфейса управления таблицами и не изменяют структуру таблицы. |
| Description | Описание таблицы. Здесь можно указать, какие иерархические данные хранит дерево, кто их поддерживает и для каких фильтров страниц они используются. |
| Preset fields | Предустановленные поля. При создании дерева рекомендуется сохранить системные поля и встроенные поля дерева. |

### Встроенные поля

После создания дерево обычно содержит следующие встроенные поля. `parentId`, `parent` и `children` используются для хранения иерархических связей.

| Поле | Имя поля | Описание |
| --- | --- | --- |
| ID | `id` | Поле первичного ключа по умолчанию, используется для уникальной идентификации записи. |
| Время создания | `createdAt` | Автоматически сохраняет время создания записи. |
| Создатель | `createdBy` | Автоматически сохраняет пользователя, создавшего запись. |
| Время обновления | `updatedAt` | Автоматически сохраняет время последнего обновления записи. |
| Обновил | `updatedBy` | Автоматически сохраняет пользователя, последним обновившего запись. |
| Parent ID | `parentId` | Сохраняет ID родительского узла. Для корневого узла обычно имеет пустое значение. |
| Parent | `parent` | Поле связи «многие к одному», указывающее на родительский узел в текущей таблице. |
| Children | `children` | Поле связи «один ко многим», представляющее дочерние узлы текущего узла. |
| Пространство | `space` | Доступно после включения [плагина нескольких пространств](../../multi-app/multi-space/index.md) и используется для изоляции данных по пространствам. Не отображается, если функция нескольких пространств не включена. |

![20240324143555](https://static-docs.nocobase.com/20240324143555.png)

:::warning Внимание

Не допускайте образования циклических связей в данных дерева: например, если родительским узлом A является B, а родительским узлом B снова является A. Циклические связи могут привести к некорректному отображению дерева и результатам фильтрации.

:::

### Поле первичного ключа

Как и обычная таблица, дерево должно иметь поле первичного ключа. Поля иерархических связей связываются с записями первичного ключа в той же таблице через ID родительского узла.

Если у дерева нет первичного ключа, при редактировании таблицы необходимо задать «Record unique key». В противном случае блоки страниц могут неправильно отображать или редактировать записи, а иерархическое представление может нестабильно определять узлы.

## Использование в настройках страниц

Для дерева можно использовать большинство блоков данных обычной [таблицы](../data-source-main/general-collection.md) для создания, удаления, изменения и просмотра записей. Кроме того, его можно использовать вместе с функциями иерархического отображения:

| Блок | Назначение |
| --- | --- |
| [Блок таблицы](../../interface-builder/blocks/data-blocks/table.md#启用树表) | Отображение иерархических записей для просмотра и управления родительско-дочерней структурой. |
| [Блок формы](../../interface-builder/blocks/data-blocks/form.md) | Добавление или редактирование отдельной записи узла дерева. |
| [Блок подробностей](../../interface-builder/blocks/data-blocks/details.md) | Просмотр подробностей отдельного узла дерева. |
| [Блок иерархической фильтрации](../../interface-builder/blocks/filter-blocks/tree.md) | Фильтрация других блоков данных с помощью иерархической структуры, часто используется для фильтрации по категориям, организациям, регионам и другим уровням. |

## Редактирование настроек

В списке таблиц нажмите «Edit» справа от дерева, чтобы изменить отображаемое имя таблицы, категории, описание, режим простой пагинации, «Record unique key» и другие настройки.

Обычно не рекомендуется без необходимости удалять поля родительско-дочерних связей дерева или изменять их назначение. Если требуется изменить иерархическую структуру, сначала измените связи с родительскими узлами в данных записей.

## Удаление таблицы

В списке таблиц нажмите «Delete» справа от дерева, чтобы удалить его.

При удалении дерева удаляются метаданные Collection, физическая таблица данных и данные иерархических связей. Перед удалением убедитесь, что блоки страниц, блоки иерархической фильтрации, поля связей, права доступа, рабочие процессы и API больше не зависят от него.

:::danger Предупреждение

Дерево часто используется как условие фильтрации для других блоков. После удаления дерева связанные блоки иерархической фильтрации и настройки страниц, зависящие от этой иерархии категорий, могут перестать работать.

:::
