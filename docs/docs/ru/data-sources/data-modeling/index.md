# Обзор

Моделирование данных — ключевой этап проектирования базы данных. Он включает глубокий анализ и абстрагирование различных типов данных реального мира и взаимосвязей между ними. В ходе этого процесса мы стремимся выявить внутренние связи между данными и формализованно описать их в виде модели данных, заложив основу для структуры базы данных информационной системы. NocoBase — платформа, управляемая моделями данных, со следующими особенностями:

## Поддержка подключения данных из различных источников

Источниками данных NocoBase могут быть распространённые базы данных, платформы API (SDK) и файлы.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

NocoBase предоставляет [плагин управления источниками данных](/data-sources/data-source-manager) для управления различными источниками данных и их таблицами. Плагин управления источниками данных предоставляет только интерфейс управления всеми источниками данных и не поддерживает подключение к ним. Для этого его необходимо использовать совместно с плагинами соответствующих источников данных. В настоящее время поддерживаются следующие источники данных:

- [Основная база данных](/data-sources/data-source-main): основная база данных NocoBase, поддерживающая реляционные базы данных MySQL, PostgreSQL, MariaDB и другие.
- [KingbaseES](/data-sources/data-source-kingbase): база данных KingbaseES компании Renmin Jincang, которую можно использовать как основную или внешнюю базу данных.
- [Внешняя MySQL](/data-sources/data-source-external-mysql): внешняя база данных MySQL в качестве источника данных.
- [Внешняя MariaDB](/data-sources/data-source-external-mariadb): внешняя база данных MariaDB в качестве источника данных.
- [Внешняя PostgreSQL](/data-sources/data-source-external-postgres): внешняя база данных PostgreSQL в качестве источника данных.
- [Внешняя MSSQL](/data-sources/data-source-external-mssql): внешняя база данных MSSQL (SQL Server) в качестве источника данных.
- [Внешняя Oracle](/data-sources/data-source-external-oracle): внешняя база данных Oracle в качестве источника данных.

![20240512083651](https://static-docs.nocobase.com/20240512083651.png)

## Предоставление разнообразных инструментов моделирования данных

**Простой интерфейс управления таблицами данных**: используется для создания различных моделей (таблиц данных) или подключения существующих моделей (таблиц данных).

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**Интерфейс визуализации в виде ER-диаграммы**: используется для извлечения сущностей и связей между ними из требований пользователей и бизнеса. Он предоставляет наглядный и понятный способ описания модели данных. ER-диаграмма позволяет лучше понять основные сущности данных в системе и связи между ними.

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## Поддержка создания различных таблиц данных

| Таблица данных | Описание |
| - | - |
| [Обычная таблица данных](/data-sources/data-source-main/general-collection) | Содержит распространённые системные поля |
| [Таблица календаря](/data-sources/calendar/calendar-collection) | Используется для создания таблиц событий, связанных с календарём |
| Таблица комментариев | Используется для хранения комментариев или отзывов о данных |
| [Таблица древовидной структуры](/data-sources/collection-tree) | Таблица древовидной структуры, в настоящее время поддерживается только дизайн списка смежности |
| [Таблица файловых данных](/data-sources/file-manager/file-collection) | Используется для управления хранением файлов |
| [Таблица данных SQL](/data-sources/collection-sql) | Не является фактической таблицей базы данных, а предназначена для быстрого структурированного отображения SQL-запросов |
| [Подключение к представлению базы данных](/data-sources/collection-view) | Подключение к существующему представлению базы данных |
| Таблица выражений | Используется для сценариев динамических выражений в рабочих процессах |
| [Подключение к внешним данным](/data-sources/collection-fdw) | Подключение к удалённым таблицам данных на основе технологии FDW базы данных |
