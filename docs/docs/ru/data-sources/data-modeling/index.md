# Обзор

Моделирование данных — это ключевой этап при проектировании баз данных, включающий глубокий анализ и абстрагирование различных типов данных реального мира и их взаимосвязей. В ходе этого процесса мы стремимся выявить внутренние связи между данными и формализовать их в модели данных, закладывая основу для структуры базы данных информационной системы. NocoBase — это платформа, ориентированная на модели данных, и обладающая следующими особенностями:

## Поддержка подключения к данным из различных источников

В качестве источников данных NocoBase могут выступать различные распространенные базы данных, платформы API (SDK) и файлы.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

NocoBase предоставляет [плагин управления источниками данных](/data-sources/data-source-manager) для управления различными источниками данных и их коллекциями. Плагин управления источниками данных предоставляет только интерфейс для управления всеми источниками данных и не обеспечивает прямого доступа к ним. Он должен использоваться в сочетании с различными плагинами источников данных. В настоящее время поддерживаются следующие источники данных:

- [Основная база данных](/data-sources/data-source-main): Основная база данных NocoBase, поддерживающая реляционные базы данных, такие как MySQL, PostgreSQL и MariaDB.
- [KingbaseES](/data-sources/data-source-kingbase): Использование базы данных KingbaseES в качестве источника данных; может использоваться как основная, так и как внешняя база данных.
- [Внешняя MySQL](/data-sources/data-source-external-mysql): Использование внешней базы данных MySQL в качестве источника данных.
- [Внешняя MariaDB](/data-sources/data-source-external-mariadb): Использование внешней базы данных MariaDB в качестве источника данных.
- [Внешняя PostgreSQL](/data-sources/data-source-external-postgres): Использование внешней базы данных PostgreSQL в качестве источника данных.
- [Внешняя MSSQL](/data-sources/data-source-external-mssql): Использование внешней базы данных MSSQL (SQL Server) в качестве источника данных.
- [Внешняя Oracle](/data-sources/data-source-external-oracle): Использование внешней базы данных Oracle в качестве источника данных.

![20240512083651](https://static-docs.nocobase.com/20240512083651.png)

## Разнообразные инструменты для моделирования данных

**Простой интерфейс управления коллекциями**: Используется для создания различных моделей (коллекций) или подключения к существующим.

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**Визуальный интерфейс в стиле ER-диаграмм**: Используется для извлечения сущностей и их взаимосвязей из пользовательских и бизнес-требований. Он предоставляет интуитивно понятный и простой для восприятия способ описания моделей данных. С помощью ER-диаграмм вы сможете более четко понять основные сущности данных в системе и их связи.

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## Поддержка различных типов коллекций

| Коллекция | Описание |
| - | - |
| [Общая коллекция](/data-sources/data-source-main/general-collection) | Включает встроенные системные поля. |
| [Календарная коллекция](/data-sources/calendar/calendar-collection) | Используется для создания коллекций событий, связанных с календарем. |
| Коллекция комментариев | Используется для хранения комментариев или отзывов о данных. |
| [Древовидная коллекция](/data-sources/collection-tree) | Древовидная коллекция, в настоящее время поддерживается только модель списка смежности. |
| [Файловая коллекция](/data-sources/file-manager/file-collection) | Используется для управления хранением файлов. |
| [SQL-коллекция](/data-sources/collection-sql) | Не является фактической коллекцией базы данных, но структурированно визуализирует SQL-запросы. |
| [Подключение к представлению базы данных](/data-sources/collection-view) | Подключается к существующим представлениям базы данных. |
| Коллекция выражений | Используется для сценариев динамических выражений в рабочих процессах. |
| [Подключение к внешним данным](/data-sources/collection-fdw) | Позволяет системе базы данных напрямую получать доступ и запрашивать данные во внешних источниках данных на основе технологии FDW. |
