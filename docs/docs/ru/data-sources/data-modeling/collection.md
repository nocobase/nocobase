# Обзор коллекций

NocoBase предлагает собственный DSL (Domain-Specific Language) для описания структуры данных, называемый коллекцией. Он объединяет структуры данных из различных источников, обеспечивая надежную основу для дальнейшего управления данными, их анализа и использования в приложениях.

![20240512161522](https://static-docs.nocobase.com/20240512161522.png)

Для удобного использования различных моделей данных поддерживается создание различных типов коллекций:

- [Обычная коллекция](/data-sources/data-source-main/general-collection): Включает встроенные общие системные поля;
- [Наследуемая коллекция](/data-sources/data-source-main/inheritance-collection): Позволяет создать родительскую коллекцию, а затем вывести из нее дочернюю. Дочерняя коллекция наследует структуру родительской и может определять собственные столбцы.
- [Древовидная коллекция](/data-sources/collection-tree): Коллекция с древовидной структурой, в настоящее время поддерживает только модель списка смежности;
- [Календарная коллекция](/data-sources/calendar/calendar-collection): Используется для создания коллекций событий, связанных с календарем;
- [Файловая коллекция](/data-sources/file-manager/file-collection): Используется для управления хранением файлов;
- : Используется для сценариев динамических выражений в рабочих процессах;
- [SQL-коллекция](/data-sources/collection-sql): Не является фактической коллекцией базы данных, но позволяет быстро представить SQL-запросы в структурированном виде;
- [Коллекция-представление](/data-sources/collection-view): Подключается к существующим представлениям базы данных;
- [Внешняя коллекция](/data-sources/collection-fdw): Позволяет системе базы данных напрямую получать доступ и запрашивать данные из внешних источников данных, используя технологию FDW.
