# Обзор коллекций

NocoBase предоставляет уникальный DSL для описания структуры данных, известного как Коллекция. Он объединяет структуру данных из разных источников, создавая надежную основу для управления данными, анализа и приложений.

![20240512161522](https://static-docs.nocobase.com/20240512161522.png)

Чтобы удобно использовать различные модели данных, поддерживаются разные типы коллекций:

- [Общая коллекция](/data-sources/data-source-main/general-collection): встроенные общие системные поля;
- [Наследуемая коллекция](/data-sources/data-source-main/inheritance-collection): вы можете создать родительскую коллекцию, а затем вывести дочернюю коллекцию из родительской. Дочерняя коллекция унаследует структуру родительской коллекции и также сможет определить свои собственные колонки.
- [Древовидная коллекция](/data-sources/collection-tree): коллекция с древовидной структурой; сейчас поддерживается только список смежности;
- [Коллекция календаря](/data-sources/calendar/calendar-collection): используется для создания коллекций событий, связанных с календарем;
- [Файловая коллекция](/data-sources/file-manager/file-collection): используется для управления хранением файлов;
- Коллекция выражений (Expression collection): используется для сценариев динамических выражений в рабочих процессах;
- [SQL-коллекция](/data-sources/collection-sql): это не физическая коллекция в базе данных, но быстро представляет SQL-запросы в структурированном виде;
- [Подключение к представлению БД (View collection)](/data-sources/collection-view): подключение к существующим представлениям БД;
- [Внешняя коллекция](/data-sources/collection-fdw): позволяет системе БД напрямую обращаться к данным и выполнять запросы к ним во внешних источниках на базе технологии FDW.
