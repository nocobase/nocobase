# Поля

## Введение

В NocoBase **Field (поле)** — это бизнес-атрибут, используемый в [Collection (таблице данных)](../collection.md) для описания данных. Поле определяет, какую информацию можно хранить в записи, а также как эту информацию вводить, отображать, фильтровать и использовать в бизнес-логике на странице.

| Определение | Описание |
| --- | --- |
| Какие данные хранить | Например, текст, числа, даты, файлы, статусы, связи, JSON. |
| Как использовать на странице | Например, вводить и отображать с помощью текстового поля, выбора даты, раскрывающегося списка, загрузки вложений и селектора связей. |
| Как участвовать в бизнес-функциях | Поля используются блоками страниц, фильтрами, сортировкой, правами доступа, рабочими процессами, API, импортом и экспортом данных и другими функциями. |

Поле может соответствовать:
- реальному столбцу базы данных в основной базе данных
- существующему столбцу базы данных во внешней базе данных
- полю в представлении базы данных
- полю в результате SQL-запроса
- полю в ответе REST API
- полю связи, системному или виртуальному полю таблицы данных

Его можно понимать как «атрибут бизнес-объекта». Например:

| Бизнес-объект | Соответствующие поля |
| --- | --- |
| Клиент | Имя клиента, номер телефона, уровень клиента, ответственный |
| Заказ | Номер заказа, сумма заказа, статус заказа, клиент |
| Договор | Название договора, дата подписания, вложение договора, статус согласования |
| Задача | Заголовок задачи, срок выполнения, приоритет, исполнитель |
| Файл | Имя файла, размер, MIME-тип, URL |

## Сценарии использования

Ниже распространённые сценарии использования сгруппированы по категориям полей. Здесь вы можете определить, какую категорию поля выбрать; подробные сведения о настройке, сопоставлении типов и важных нюансах доступны в документации соответствующей категории.

| Категория поля | Сценарии использования |
| --- | --- |
| [Текстовые поля](./basic/input.md) | Подходят для хранения названий, номеров, описаний, контактных данных, URL-адресов и другой подобной информации. |
| [Поля форматированного текста](./media/rich-text.md) | Подходят для хранения основного текста, документов с описанием, решений по обработке, фрагментов кода и другой сложной информации. |
| [Числовые поля](./basic/number.md) | Подходят для хранения количества, сумм, оценок, процентных значений и других чисел. |
| [Поля даты и времени](./datetime/index.md) | Подходят для хранения моментов времени, дат, времени, временных меток внешних систем и других подобных данных. |
| [Поля статуса и вариантов](./choices/select.md) | Подходят для хранения значений из фиксированного набора, например статуса включения, статуса заказа, уровня клиента, метки клиента. |
| [Поля вложений](./media/field-attachment.md) | Подходят для загрузки файлов или хранения адресов внешних файлов. |
| [Поля связей](./associations/index.md) | Подходят для выражения связей между разными таблицами данных, например заказ принадлежит клиенту, клиент имеет заказы, пользователь связан с ролью. |
| [Поля идентификаторов и кодов](./advanced/uuid.md) | Подходят для хранения внутренних первичных ключей, внешних идентификаторов синхронизации, идентификаторов публичного доступа и бизнес-номеров. |
| [Поля геометрии](./geometric/point.md) | Подходят для хранения пространственной или географической информации, например местоположения магазина, маршрута доставки, зоны обслуживания. |
| [Системные поля](./system-info/created-at.md) | Подходят для хранения системной информации, поддерживаемой NocoBase или базой данных, например идентификатора, времени создания, создателя и времени обновления. |
| [Другие поля](./advanced/json.md) | Подходят для полей, связанных с сортировкой, формулами, JSON и другими задачами, которые не относятся напрямую к остальным категориям. |

## Типы данных полей

Каждый Field Interface имеет тип данных по умолчанию. Например, для поля с Interface «число» (Number) тип данных по умолчанию — double, но также могут использоваться float, decimal и другие типы. В настоящее время поддерживаются следующие типы данных:

## Сопоставление типов полей

Процесс добавления поля в основную базу данных:

1. Выберите тип Interface
2. Настройте доступные типы данных для текущего Interface

![20240512172416](https://static-docs.nocobase.com/20240512172416.png)

Процесс сопоставления полей внешнего источника данных:

1. Автоматически сопоставьте тип данных (Field type) и тип UI (Field Interface) на основе типа поля во внешней базе данных.
2. При необходимости измените их на более подходящие тип данных и тип Interface
