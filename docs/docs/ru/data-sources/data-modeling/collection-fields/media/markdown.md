# Markdown

## Введение

В NocoBase **Markdown（Markdown）** используется для хранения содержимого в формате Markdown.

Поле Markdown подходит для документации, планов решения, основного содержимого базы знаний, журналов изменений и других материалов. Оно хранит текст, который при отображении на странице рендерится в формате Markdown.

Если нужен редактор с возможностью визуального редактирования, можно выбрать [форматированный текст](./rich-text.md) или [Markdown Vditor](../../../field-markdown-vditor/index.md)。

## Сценарии использования

Markdown подходит для следующих бизнес-сценариев:

- Основное содержимое базы знаний и документация
- Планы решения и записи об устранении неполадок
- Примечания к выпуску и журналы изменений
- Длинное текстовое содержимое, для которого требуется лёгкое форматирование

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Markdown», чтобы создать поле Markdown.

![20240512181311](https://static-docs.nocobase.com/20240512181311.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейс поля. Для Markdown соответствует `markdown` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Документация», «План решения» или «Основное содержимое». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для сохранения содержимого поле Markdown обычно использует `text`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно ограничить длину или сделать поле обязательным. |
| Description | Описание поля. Подходит для указания назначения поля, требований к заполнению, источника данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в будущем.

:::

## Особенности поля

Поле Markdown по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `markdown`. |
| Тип Field type по умолчанию | `text`. |
| Доступные типы Field type | `text`、`string` — в зависимости от фактической конфигурации поля. |
| Компонент страницы | В режиме редактирования используется компонент редактора Markdown. |
| Фильтрация | Поддерживаются фильтры для текстовых данных, например «содержит», «пусто» и «не пусто». |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются текстовые проверки, например ограничение длины и обязательность заполнения. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля Markdown. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле получено из уже синхронизированной таблицы основной базы данных, редактирование обычно выполняет роль сопоставления поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет имя поля, отображаемое в интерфейсе, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | В зависимости от условий | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. Если существующих данных много, сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле Markdown. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля Markdown обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
