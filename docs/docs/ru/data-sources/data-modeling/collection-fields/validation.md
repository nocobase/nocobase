# Валидация полей
Чтобы обеспечить точность данных, безопасность и согласованность в коллекциях, NocoBase предоставляет функциональность проверки полей. Эта функция состоит из двух основных частей: конфигурации правил и применения правил.

## Конфигурация правил

![20250819181342](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819181342.png)

Системные поля NocoBase интегрируют правила [Joi](https://joi.dev/api/), поддержка реализована следующим образом:

### Строка
Типы строк Joi соответствуют следующим типам полей NocoBase: Однострочный текст, Длинный текст, Телефон, Электронная почта, URL-адрес, Пароль и UUID.
#### Общие правила
- Минимальная длина
- Максимальная длина
- Длина
- Шаблон
- Обязательное

#### Электронная почта

[Посмотреть дополнительные параметры](https://joi.dev/api/?v=17.13.3#stringemailoptions)

#### URL-адрес

![20250819192409](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819192409.png)

[Посмотреть дополнительные параметры](https://joi.dev/api/?v=17.13.3#stringurioptions)

#### UUID

![20250819192731](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819192731.png)

[Посмотреть дополнительные параметры](https://joi.dev/api/?v=17.13.3#stringguid---aliases-uuid)

### Число
Типы чисел Joi соответствуют следующим типам полей NocoBase: Целое число (Integer), Число (Number) и Процент (Percentage).
#### Общие правила
- Больше чем
- Меньше чем
- Максимальное значение
- Минимальное значение
- Кратность

#### Целое число (Integer)
Помимо общих правил, целочисленные поля дополнительно поддерживают [проверку целых чисел](https://joi.dev/api/?v=17.13.3#numberinteger) и [проверку небезопасных целых чисел](https://joi.dev/api/?v=17.13.3#numberunsafeenabled).

![20250819193758](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819193758.png)

#### Число и процент (Number & Percentage)
Помимо общих правил, поля с числами и процентами дополнительно поддерживают [проверку точности](https://joi.dev/api/?v=17.13.3#numberinteger).

![20250819193954](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819193954.png)

### Дата (Date)
Типы дат Joi соответствуют следующим типам полей NocoBase: Дата (с часовым поясом), Дата (без часового пояса), Дата(без времени) и Unix-временная метка (Unix timestamp).

Поддерживаемые правила проверки:
- Больше чем
- Меньше чем
- Максимальное значение
- Минимальное значение
- Метка времени
- Обязательное

### Поля связей
Поля связей поддерживают только обязательную проверку. Обратите внимание: обязательная проверка для полей связей в настоящее время не поддерживается в сценариях под-формы или под-таблицы.

## Применение правил проверки
После настройки правил полей соответствующие правила проверки будут срабатывать при добавлении или изменении данных.

![20250819201027](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819201027.png)

Когда поле используется в форме, правила проверки поля также отображаются в настройках проверки этого поля. Эти правила находятся в разделе **Серверные правила проверки полей** и доступны там только для чтения. Если нужно изменить их, отредактируйте поле в разделе Источник данных → Настройка коллекции.

Вы по-прежнему можете добавить дополнительные правила для текущего поля формы в разделе **Клиентские правила проверки**. Эти правила применяются только к текущему компоненту поля. Итоговая проверка объединяет **Серверные правила проверки полей** и **Клиентские правила проверки**.

Правила проверки также применяются к компонентам вложенных таблиц и вложенных форм:

![20250819202357](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819202357.png)

Обратите внимание: в сценариях под-формы или под-таблицы обязательная проверка для полей связей не применяется.

## Отличия серверных правил проверки полей от клиентских правил проверки
Серверные правила проверки полей и клиентские правила проверки настраиваются в разных местах и имеют разную область действия.

### Различия в способах настройки
- **Серверные правила проверки полей**: Правила для полей задаются в разделе Источник данных → Настройка коллекции. Это базовые правила поля.
- **Клиентские правила проверки**: Дополнительные правила настраиваются в параметрах поля формы. Они влияют только на текущий компонент поля.

![20250819203836](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819203836.png)

![20250819203845](https://nocobase-docs.oss-cn-beijing.aliyuncs.com/20250819203845.png)

### Различия в моменте срабатывания проверки
- **Серверные правила проверки полей**: Запускают фронтенд-проверку, когда поле используется в форме, а также выполняют проверку перед записью данных. Они применяются и в сценариях создания или обновления данных, таких как рабочие процессы и импорт данных.
- **Клиентские правила проверки**: Запускают фронтенд-проверку только в текущем поле формы.
- **Отображение правил**: Серверные правила проверки полей отображаются как унаследованные правила только для чтения. Клиентские правила проверки отображаются отдельно, и их можно редактировать там.
- **Сообщения об ошибках**: Клиентские правила проверки поддерживают настраиваемые сообщения об ошибках, тогда как серверные правила проверки полей в настоящее время не поддерживают их.
