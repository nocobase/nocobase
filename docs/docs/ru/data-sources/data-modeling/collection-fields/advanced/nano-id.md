# Nano ID

## Введение

В NocoBase **Nano ID (Nano ID)** используется для генерации коротких случайных уникальных идентификаторов.

Nano ID подходит для сценариев, требующих коротких строковых идентификаторов, например коротких ссылок, общедоступных номеров и идентификаторов внешних ссылок.

Если требуется использовать идентификатор в качестве внутреннего первичного ключа по умолчанию, Snowflake ID обычно будет более прямым решением. Если нужен читаемый бизнес-номер, выберите [последовательность](../../../field-sequence/index.md).

## Сценарии использования

Nano ID подходит для следующих бизнес-сценариев:

- Идентификаторы коротких ссылок
- Общедоступные ID для обмена
- Номера внешних ссылок
- Короткие случайные уникальные строки

## Создание конфигурации

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Nano ID», чтобы создать поле Nano ID.

![20240512173225](https://static-docs.nocobase.com/20240512173225.png)

| Конфигурация | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для Nano ID соответствует `nanoId` и определяет способ ввода и отображения в интерфейсе. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «ID для обмена», «Общедоступный ID» или «Короткий идентификатор». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для Nano ID по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При добавлении записи, если пользователь не указал значение, оно может подставляться автоматически. |
| Validation rules | Обычно создаются системой автоматически и не требуют ручной проверки. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственное лицо. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле Nano ID по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `nanoId`. |
| Тип Field type по умолчанию | `string`. |
| Доступный тип Field type | `string`. |
| Компонент страницы | Обычно создаётся автоматически и не требует ручного ввода. |
| Фильтрация | Поддерживается точный поиск по Nano ID. |
| Сортировка | Обычно Nano ID не используется для бизнес-сортировки. |
| Проверка | Обычно создаётся автоматически и обеспечивает уникальность. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля Nano ID. Редактирование поля в основном используется для настройки способа его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Конфигурация | Доступно редактирование | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого имени поля в интерфейсе без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля после создания обычно нельзя изменить в форме редактирования. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей можно изменить интерфейс при сопоставлении. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей можно изменить тип при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение назначения поля, требований к заполнению, источника данных или ответственного лица. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле Nano ID. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля Nano ID, созданного в основной базе данных, обычно одновременно удаляются соответствующий реальный столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
