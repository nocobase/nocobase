# Административные регионы Китая

<PluginInfo name="field-china-region"></PluginInfo>

## Введение

В NocoBase **административные регионы Китая (China region)** используются для сохранения информации о провинциях, городах, районах и уездах Китая.

Поле административных регионов Китая основано на встроенной таблице данных `chinaRegions` административных единиц, а для ввода на странице используется каскадный селектор. Пользователь может последовательно выбрать провинцию, город и район по уровням, а при отображении они объединяются в полный путь.

Если необходимо сохранять подробный адрес, например улицу и номер дома, поле можно использовать вместе с полем [однострочного текста](../basic/input.md) или [многострочного текста](../basic/textarea.md).

## Сценарии использования

Поле административных регионов Китая подходит для следующих бизнес-сценариев:

- Местонахождение клиентов, контактных лиц, магазинов и проектов
- Основная информация об адресе: место регистрации, родной край, регион доставки и т. п.
- Зоны обслуживания, продаж и реализации проектов
- Данные, которые необходимо фильтровать или агрегировать по провинциям, городам и районам

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Административные регионы Китая», чтобы создать соответствующее поле.

![](https://static-docs.nocobase.com/data-source-manager-main-NocoBase-04-29-2026_04_52_PM.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для административных регионов Китая используется `chinaRegion`, который определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Регион нахождения», «Зона обслуживания» или «Регион доставки». Рекомендуется использовать понятное сотрудникам название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Административные регионы Китая обычно сохраняются как связанные записи или структурированное значение; конкретный вариант зависит от настройки поля. |
| Выбор уровня | Определяет самый глубокий доступный для выбора уровень. В настоящее время поддерживаются «Провинция», «Город» и «Район»; по умолчанию используется «Район». |
| Обязательно выбрать последний уровень | Если включено, пользователь должен выбрать самый глубокий настроенный уровень, чтобы отправить форму; если выключено, выбор можно завершить на промежуточном уровне. |
| Validation rules | Правила проверки. Обычно настраиваются обязательность заполнения и уровень выбора. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в будущем.

:::

## Особенности поля

Поле административных регионов Китая по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Field interface по умолчанию | `chinaRegion`. |
| Источник данных | Встроенная таблица данных административных единиц `chinaRegions`. |
| Компонент страницы | В режиме редактирования используется каскадный селектор. |
| Уровни выбора | В настоящее время поддерживаются три уровня: провинция, город и район. |
| Способ отображения | В режиме просмотра данные отображаются по уровням как `省 / 市 / 区`. |
| Фильтрация | Поддерживается фильтрация по сохранённому значению региона; конкретные возможности зависят от настройки поля и блока страницы. |
| Множественный выбор | Множественный выбор не поддерживается. |

## Редактирование настройки

После создания нажмите «Edit» справа от поля, чтобы изменить его настройки. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени, описания, правил проверки, уровня выбора или обязательности выбора последнего уровня.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления полей — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не меняя его идентификатор. |
| Field name | Нет | Идентификатор поля после создания обычно нельзя изменить в форме редактирования таблицы. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей можно изменить сопоставление. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей можно изменить сопоставление. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Выбор уровня | Да | Позволяет изменить доступный для выбора уровень: провинция, город или район. |
| Обязательно выбрать последний уровень | Да | Определяет, обязательно ли при отправке выбирать самый глубокий настроенный уровень. |
| Validation rules | Да | Позволяет изменить правила проверки, например обязательность заполнения. |
| Description | Да | Позволяет дополнить смысл поля, требования к заполнению, источник данных или сведения об ответственном. |

:::warning Внимание

Поле административных регионов Китая зависит от таблицы данных `chinaRegions`, предоставляемой плагином. Перед использованием убедитесь, что плагин поля «Административные единицы Китая» включён.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле административных регионов Китая. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля административных регионов Китая обычно одновременно удаляются реальный столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в настройках страницы

Поле административных регионов Китая подходит для работы с адресами, регионами и статистикой.

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Выбор провинции, города и района с помощью каскадного селектора. |
| Блок подробной информации | Отображение пути административных единиц. |
| Блок таблицы | Отображение региона, к которому относится запись. |
| Блок фильтрации | Фильтрация записей по региону. |
| Блок диаграммы | Агрегация бизнес-данных по провинциям, городам и районам. |

### Режим редактирования

В режиме редактирования поле административных регионов Китая отображается как каскадный селектор.

![](https://static-docs.nocobase.com/%E7%9C%81%E5%B8%82%E5%8C%BA-04-29-2026_04_54_PM.png)

### Режим просмотра

В режиме просмотра поле административных регионов Китая отображается как текстовый путь, например:

```text
北京市 / 市辖区 / 东城区
```
