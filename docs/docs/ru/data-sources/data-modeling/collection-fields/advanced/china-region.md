# Административные регионы Китая

<PluginInfo name="field-china-region"></PluginInfo>

## Введение

Поле «Административные регионы Китая» используется для хранения в таблице данных информации об административно-территориальном делении Китая: провинциях, городах, округах и районах. Поле основано на встроенной таблице `chinaRegions` и предоставляет каскадный селектор: в форме пользователь может последовательно выбрать провинцию, город и район по уровням иерархии.

Применимые сценарии:

- Местонахождение клиентов, контактов, торговых точек, проектов и подобных записей
- Базовая адресная информация: место регистрации, родина, регион доставки
- Данные, которые нужно фильтровать или агрегировать по провинции, городу и району

Значение поля сохраняется как ссылка на запись и по умолчанию связано с таблицей `chinaRegions`, отображаясь в порядке административной иерархии. Например, после выбора «Пекин / Городской округ / Дунчэн» в режиме просмотра значение склеивается в полный путь.

## Конфигурация поля

![](https://static-docs.nocobase.com/data-source-manager-main-NocoBase-04-29-2026_04_52_PM.png)

При создании поля выберите тип «Административные регионы Китая». Доступны следующие настройки:

| Параметр | Описание |
| --- | --- |
| Уровень выбора | Управляет максимальным уровнем, до которого пользователь может выбирать. Сейчас поддерживаются «Провинция», «Город» и «Район». По умолчанию — «Район». «Улица» и «Деревня» в интерфейсе недоступны. |
| Обязательно выбирать до последнего уровня | Если флажок установлен, пользователь должен выбрать значение до настроенного максимального уровня, иначе форму нельзя отправить. Если флажок снят, выбор можно завершить на промежуточном уровне. |

## Использование в интерфейсе

В форме поле отображается как каскадный селектор:

1. При открытии выпадающего списка загружаются данные на уровне провинций.
2. При раскрытии конкретной провинции по запросу подгружаются её города.
3. При раскрытии города по запросу подгружаются районы.
4. После сохранения в режимах просмотра — например, в подробностях или в таблице — значение отображается по уровням как `Провинция/Город/Район`.

Поле поддерживает стандартные настройки формы: заголовок, описание, обязательность, значение по умолчанию, режим только для чтения и т. п. В режиме только для чтения поле отображается как текстовый путь, например:

```text
Пекин / Городской округ / Дунчэн
```

## Замечания

- Поле «Административные регионы Китая» в настоящее время поддерживает выбор только одного пути и не поддерживает множественный выбор.
- Сейчас встроены и активны три уровня: провинция, город и район. Уровни «Улица» и «Деревня» пока недоступны.
- При импорте необходимо указывать названия в точном соответствии со встроенными данными административного деления и разделять уровни символом `/`.
- Поле зависит от таблицы `chinaRegions`, предоставляемой плагином, поэтому убедитесь, что плагин «Поле административного деления Китая» включён.
