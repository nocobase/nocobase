# UUID

## Введение

В NocoBase **UUID (UUID)** используется для генерации уникального идентификатора UUID.

UUID подходит для синхронизации между системами, идентификации в публичных API, импорта и экспорта. По сравнению с автоинкрементным ID он в меньшей степени раскрывает объём данных.

Если нужен только первичный ключ по умолчанию для внутренних задач NocoBase, обычно достаточно Snowflake ID. Если нужен короткий идентификатор, выберите [Nano ID](./nano-id.md) или[последовательность](../../../field-sequence/index.md).

## Сценарии использования

UUID подходит для следующих бизнес-сценариев:

- Идентификатор синхронизации с внешней системой
- Идентификатор публичного API
- Идентификатор записи при миграции между базами данных
- Идентификатор записи, не раскрывающий закономерность инкрементирования

## Создание конфигурации

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «UUID», чтобы создать поле UUID.

![20240512173354](https://static-docs.nocobase.com/20240512173354.png)

| Конфигурация | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для UUID соответствует `uuid` и определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «UUID», «Внешний идентификатор» или «Публичный ID». Рекомендуется использовать название, понятное сотрудникам без дополнительных пояснений. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля UUID по умолчанию используется `uuid`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь не ввёл значение. |
| Validation rules | Обычно генерируются системой автоматически и не требуют ручной проверки. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в будущем.

:::

## Свойства поля

Поле UUID по умолчанию ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Field interface по умолчанию | `uuid`. |
| Field type по умолчанию | `uuid`. |
| Доступный Field type | `uuid`. |
| Компонент страницы | Обычно генерируется автоматически и не требует ручного ввода. |
| Фильтрация | Поддерживается точный поиск по UUID. |
| Сортировка | Сортировка поддерживается, но обычно UUID не используется для бизнес-сортировки. |
| Проверка | Обычно генерируется автоматически и обеспечивает уникальность. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля UUID. Редактирование поля используется главным образом для настройки его отображения и применения в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает сопоставление поля — преобразование поля базы данных в Field type и Field interface NocoBase.

| Конфигурация | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не меняет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Изменение повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом количестве существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле UUID. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля UUID обычно одновременно удаляются настоящий столбец в базе данных и уже имеющиеся в этом столбце данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
