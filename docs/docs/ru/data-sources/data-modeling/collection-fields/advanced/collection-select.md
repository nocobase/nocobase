# Селектор таблиц данных

## Введение

В NocoBase **селектор таблиц данных (Collection select)** используется для выбора одной или нескольких таблиц данных.

Селектор таблиц данных подходит для настройки плагинов, правил, управления метаданными и других подобных сценариев. Он сохраняет идентификаторы таблиц данных, а не бизнес-записи.

Если нужно выбрать записи из определённой таблицы, следует использовать поле связи, а не селектор таблиц данных.

## Сценарии использования

Селектор таблиц данных подходит для следующих бизнес-сценариев:

- выбор целевых таблиц данных в настройках плагина
- указание диапазона таблиц данных в настройках правил
- управление метаданными и настройка шаблонов
- настройка функций, которым требуется ссылка на идентификатор Collection

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Селектор таблиц данных», чтобы создать поле селектора таблиц данных.

![20240512174047](https://static-docs.nocobase.com/20240512174047.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Селектору таблиц данных соответствует `collectionSelect`, определяющий способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Целевая таблица данных», «Таблица назначения» или «Диапазон данных». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связи, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Селектор таблиц данных обычно сохраняет идентификатор таблицы данных, а тип хранения определяется фактической конфигурацией. |
| Default value | Значение по умолчанию. При создании записи значение может быть подставлено автоматически, если пользователь его не указал. |
| Validation rules | Правила проверки. Обычно задаются обязательность заполнения или диапазон доступных вариантов. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поля селектора таблиц данных по умолчанию имеют следующие свойства:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `collectionSelect`. |
| Тип Field type по умолчанию | `string`. |
| Доступные типы Field type | `string`, `json` — в зависимости от фактической конфигурации. |
| Компонент страницы | В режиме редактирования используется компонент выбора таблиц данных. |
| Фильтрация | Обычно не используется как поле бизнес-фильтрации. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживает базовые проверки, например обязательность заполнения. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля селектора таблиц данных. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом количестве существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле селектора таблиц данных. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля селектора таблиц данных обычно одновременно удаляются реальный столбец в базе данных и имеющиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в конфигурации страниц

Селектор таблиц данных подходит для использования в формах конфигурации.

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Выбор одной или нескольких таблиц данных. |
| Блок подробной информации | Отображение выбранных таблиц данных. |
| Конфигурация плагина | Указание диапазона таблиц данных, к которым применяется функция. |
| Рабочий процесс или правило | Участие в логике в качестве конфигурации метаданных. |
