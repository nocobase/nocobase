# Создатель

## Введение

В NocoBase поле **«Создатель» (Created by)** автоматически записывает пользователя, создавшего запись.

Создатель обычно создаётся как предустановленное поле. Оно подходит для управления правами доступа, отслеживания ответственности, фильтрации и аудита.

Если нужно указать ответственного за бизнес-процесс, исполнителя или утверждающего, рекомендуется отдельно создать поле связи с пользователем и не использовать для этого поле «Создатель».

## Сценарии использования

Поле «Создатель» подходит для следующих бизнес-сценариев:

- Просматривать только созданные мной данные
- Фильтровать записи по создателю
- Проводить аудит ответственности за создание записей
- Определять создателя записи в рабочем процессе

## Настройка создания

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Создатель», чтобы создать поле «Создатель».

![index-2025-11-01-00-50-59](https://static-docs.nocobase.com/index-2025-11-01-00-50-59.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для поля «Создатель» соответствует `createdBy` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Создатель». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Поле «Создатель» обычно является полем связи `belongsTo` с таблицей пользователей. |
| Default value | Значение по умолчанию. При создании записи, если пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Поддерживается системой автоматически и обычно не требует ручной проверки. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле «Создатель» по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `createdBy`. |
| Тип Field type по умолчанию | `belongsTo`. |
| Доступный тип Field type | `belongsTo`. |
| Компонент страницы | Заполняется системой автоматически; на странице обычно отображается с помощью компонента выбора или отображения пользователя. |
| Фильтрация | Поддерживается фильтрация по пользователю. |
| Сортировка | Обычно сортировка по создателю не используется. |
| Проверка | Выполняется системой автоматически. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля «Создатель». Редактирование поля в основном используется для настройки его отображения и способа использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления полей — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. После изменения будут затронуты способы ввода, отображения и проверки на странице. |
| Field type | В зависимости от условий | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. Если существующих данных много, сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле «Создатель». В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля «Создатель», созданного в основной базе данных, обычно одновременно удаляются физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
