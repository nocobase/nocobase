# Дата создания

## Введение

В NocoBase поле **Дата создания (Created at)** автоматически записывает время создания записи.

Дата создания обычно создаётся предустановленным полем. Она подходит для сортировки, фильтрации, аудита, условий рабочих процессов и статистики данных.

Если нужно вручную указать бизнес-дату, например дату заключения договора или дату окончания срока действия, используйте [дату](../datetime/date.md) или [дату и время](../datetime/datetime.md).

## Подходящие сценарии

Дата создания подходит для следующих бизнес-сценариев:

- Сортировка по времени создания
- Фильтрация данных, созданных за определённый период
- Аудит времени создания записи
- Проверка времени создания новой записи в рабочем процессе

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Дата создания», чтобы создать поле даты создания.

![20240512174347](https://static-docs.nocobase.com/20240512174347.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для даты создания соответствует `createdAt` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Дата создания» или «Время создания». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для даты создания обычно используется `date`. |
| Default value | Значение по умолчанию. При добавлении записи оно может быть подставлено автоматически, если пользователь не указал значение. |
| Validation rules | Поддерживается системой автоматически, обычно не требует ручной проверки. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

Имя поля после создания будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле даты создания по умолчанию работает следующим образом:

| Характеристика | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `createdAt`. |
| Тип Field type по умолчанию | `date`. |
| Доступный тип Field type | `date`. |
| Компонент страницы | Заполняется системой автоматически; обычно отображается на странице только для чтения. |
| Фильтрация | Поддерживается фильтрация по времени и диапазонам. |
| Сортировка | Поддерживается сортировка по времени. |
| Проверка | Выполняется системой автоматически. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля даты создания. Редактирование поля предназначено главным образом для настройки способа его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет его идентификатор. |
| Field name | Нет | Идентификатор поля после создания обычно нельзя изменить в форме редактирования. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных рабочих процессов. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле даты создания. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля даты создания, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурациях.

:::
