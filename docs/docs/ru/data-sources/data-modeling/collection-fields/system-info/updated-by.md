# Обновивший пользователь

## Введение

В NocoBase поле **«Обновивший пользователь» (Updated by)** автоматически записывает пользователя, который последним изменил запись.

Обычно поле «Обновивший пользователь» создаётся предустановленным полем. Оно подходит для аудита, отслеживания ответственности, фильтрации и использования в условиях рабочих процессов.

Если нужно указать ответственного за бизнес-процесс, исполнителя или согласующего, рекомендуется отдельно создать поле связи с пользователем.

## Сценарии использования

Поле «Обновивший пользователь» подходит для следующих бизнес-сценариев:

- Просмотр пользователя, последним обслуживавшего запись
- Фильтрация записей по обновившему пользователю
- Аудит изменений данных
- Уведомление обновившего пользователя в рабочем процессе

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Обновивший пользователь», чтобы создать соответствующее поле.

![index-2025-11-01-00-51-12](https://static-docs.nocobase.com/index-2025-11-01-00-51-12.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для поля «Обновивший пользователь» соответствует `updatedBy` и определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Обновивший пользователь» или «Последний изменивший». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Поле «Обновивший пользователь» обычно является полем связи `belongsTo` с таблицей пользователей. |
| Default value | Значение по умолчанию. При добавлении записи, если пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Поддерживается системой автоматически, обычно не требует ручной проверки. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле «Обновивший пользователь» по умолчанию обладает следующими свойствами:

| Характеристика | Описание |
| --- | --- |
| Default Field interface | `updatedBy`. |
| Default Field type | `belongsTo`. |
| Доступный Field type | `belongsTo`. |
| Компонент страницы | Значение записывается системой автоматически; обычно на странице оно отображается с помощью компонента пользователя. |
| Фильтрация | Поддерживается фильтрация по пользователю. |
| Сортировка | Обычно сортировка по обновившему пользователю не используется. |
| Проверка | Значение записывается системой автоматически. |

## Редактирование настройки

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля «Обновивший пользователь». Редактирование поля предназначено главным образом для настройки его отображения и использования в NocoBase, например изменения отображаемого имени, описания, значения по умолчанию, правил проверки или параметров, специфичных для поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления полей — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, но не изменяет его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле «Обновивший пользователь». В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля «Обновивший пользователь» обычно одновременно удаляются реальный столбец в базе данных и хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
