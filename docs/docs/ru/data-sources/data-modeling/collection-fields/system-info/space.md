# Пространство

## Введение

В NocoBase **пространство (Space)** используется для хранения информации о том, к какому пространству относятся данные.

Поле пространства обычно появляется после включения плагина поддержки нескольких пространств и используется для изоляции данных по пространствам. Его не следует произвольно изменять как обычное бизнес-поле.

Если речь идет только о таких бизнес-измерениях, как отдел, регион или проект, рекомендуется создать обычное поле связи или поле вариантов.

## Сценарии использования

Пространства подходят для следующих бизнес-сценариев:

- Изоляция данных в нескольких пространствах
- Фильтрация данных по пространству
- Управление правами на уровне пространства
- Принадлежность данных в сценариях с несколькими арендаторами

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Пространство», чтобы создать поле пространства.

![index-2025-11-01-00-50-45](https://static-docs.nocobase.com/index-2025-11-01-00-50-45.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для пространства соответствует `space` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Пространство». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связи, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчеркивания, причем имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Поле пространства обычно является полем связи, указывающим на таблицу пространств. |
| Default value | Значение по умолчанию. Если при создании записи пользователь ничего не указал, это значение может быть подставлено автоматически. |
| Validation rules | Обычно поддерживаются системой или контекстом пространства. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственное лицо. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле пространства по умолчанию обладает следующими свойствами:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `space`. |
| Тип Field type по умолчанию | `belongsTo`. |
| Допустимый Field type | `belongsTo`. |
| Компонент страницы | Поддерживается системой или функциональностью нескольких пространств; обычно на странице доступен только просмотр или используется контекст пространства. |
| Фильтрация | Поддерживается фильтрация по пространству; конкретное поведение зависит от конфигурации нескольких пространств. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживается функциональностью нескольких пространств. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля пространства. Редактирование поля в основном используется для настройки его отображения и применения в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из уже синхронизированной таблицы основной базы данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определенных условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определенных условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение назначения поля, требований к заполнению, источника данных или ответственного лица. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле пространства. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля пространства обычно одновременно удаляются реальный столбец в базе данных и имеющиеся в нем данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::

## Использование в конфигурации страниц

Поле пространства подходит для сценариев изоляции данных и управления правами в нескольких пространствах.

| Сценарий | Назначение |
| --- | --- |
| Блок таблицы | Отображение пространства, к которому относится запись. |
| Блок фильтрации | Фильтрация записей по пространству. |
| Права доступа | Изоляция доступа к данным по пространствам. |
| Рабочий процесс | Чтение пространства, к которому относится запись, в качестве контекста. |
