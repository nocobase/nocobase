# Дата обновления

## Введение

В NocoBase **дата обновления (Updated at)** автоматически записывает время последнего обновления записи.

Дата обновления обычно создаётся предустановленным полем. Она подходит для аудита, определения необходимости синхронизации, сортировки, фильтрации и условий рабочих процессов.

Если необходимо сохранить время завершения операции, согласования и т. п., используйте [дату и время](../datetime/datetime.md).

## Подходящие сценарии

Дата обновления подходит для следующих бизнес-сценариев:

- Просмотр времени последнего обновления
- Фильтрация недавно обновлённых данных
- Определение данных, которые давно не поддерживались
- Сравнение времени обновления при синхронизации с внешними системами

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Дата обновления», чтобы создать поле даты обновления.

![20240512174826](https://static-docs.nocobase.com/20240512174826.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для даты обновления соответствует `updatedAt` и определяет способ ввода и отображения на странице. |
| Field display name | Имя, отображаемое для поля в интерфейсе, например «Дата обновления» или «Время последнего обновления». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для даты обновления обычно используется `date`. |
| Default value | Значение по умолчанию. При создании записи, если пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Поддерживаются системой автоматически; обычно не требуют ручной проверки. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле даты обновления по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `updatedAt`. |
| Тип Field type по умолчанию | `date`. |
| Доступный тип Field type | `date`. |
| Компонент страницы | Система записывает значение автоматически; на странице оно обычно отображается только для чтения. |
| Фильтрация | Поддерживается фильтрация по времени и диапазону. |
| Сортировка | Поддерживается сортировка по времени. |
| Проверка | Значение записывается системой автоматически. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля даты обновления. Редактирование поля предназначено главным образом для настройки способа его отображения и использования в NocoBase, например изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени, отображаемого для поля в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс можно изменить при сопоставлении полей основной базы данных или синхронизированных полей. Это повлияет на способы ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Тип можно изменить при сопоставлении полей основной базы данных или синхронизированных полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле даты обновления. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля даты обновления обычно одновременно удаляются фактический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурациях.

:::

| Сценарий | Назначение |

| Блок таблицы | Отображение и сортировка по времени последнего обновления. |
| Блок фильтрации | Фильтрация недавно обновлённых или давно не обновлявшихся записей. |
| Блок подробной информации | Просмотр времени последнего обновления. |
| Рабочий процесс | Использование в качестве временного условия. |
