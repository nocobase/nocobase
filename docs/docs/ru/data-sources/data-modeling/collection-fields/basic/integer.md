# Целое число

## Введение

В NocoBase **целое число (Integer)** используется для хранения значений без дробной части.

Поле целого числа подходит для хранения количества, числа повторений, количества людей, порядковых номеров и других бизнес-данных. Оно может использоваться в фильтрах, сортировке, статистике, правах доступа и условиях рабочих процессов.

Если необходимо хранить дробные значения, денежные суммы, вес или соотношения, лучше выбрать [число](./number.md) или [процент](./percent.md).

## Сценарии использования

Целые числа подходят для следующих бизнес-сценариев:

- Количество товаров, складские запасы, объём закупок
- Количество участников, оставшиеся места, статистика повторений
- Продолжительность работ в днях, количество дней задержки, срок оплаты в днях
- Целочисленные коды во внешних системах

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Целое число», чтобы создать поле целого числа.

![20240512175723](https://static-docs.nocobase.com/20240512175723.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для целого числа используется `integer`, который определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Количество», «Число участников», «Дни задержки». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля целого числа по умолчанию используется `integer`; для целых чисел большого диапазона можно выбрать `bigInt`. |
| Default value | Значение по умолчанию. При создании записи оно подставляется автоматически, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно ограничить минимальное и максимальное значение или указать, является ли поле обязательным. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

По умолчанию поле целого числа имеет следующие параметры:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `integer`. |
| Тип Field type по умолчанию | `integer`. |
| Доступные типы Field type | `integer`, `bigInt`. |
| Компонент страницы | В режиме редактирования используется поле ввода числового значения. |
| Фильтрация | Поддерживается фильтрация по числовым условиям: равно, не равно, больше, меньше, диапазон, пусто, не пусто и т. д. |
| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Проверка | Поддерживается проверка минимального и максимального значения, обязательности поля и других числовых условий. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля целого числа. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс основного или синхронизированного поля можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Тип основного или синхронизированного поля можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле целого числа. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля целого числа, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного из внешнего источника данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурациях.

:::

Поля целого числа подходят для использования в таблицах, формах, статистике и рабочих процессах.

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод количества, числа повторений, количества дней и других значений без дробной части. |
| Блок таблицы | Отображение, сортировка и фильтрация целых чисел. |
| Блок диаграммы | Построение статистики по таким полям, как количество и число повторений. |
| Рабочие процессы и права доступа | Использование в качестве поля условия, например для проверки, больше ли количество 0. |
