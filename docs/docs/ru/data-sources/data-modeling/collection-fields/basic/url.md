# URL

## Введение

В NocoBase поле **URL (URL)** используется для хранения веб-адресов и ссылок.

Поле URL подходит для адресов внешних систем, ссылок на документы, официальные сайты, адресов обратного вызова и других подобных данных. По сравнению с обычным текстом оно точнее отражает семантику ссылки.

Если нужно загрузить файл, выберите [вложение](../media/field-attachment.md). Если это обычный пояснительный текст, выберите [однострочный текст](./input.md) или [многострочный текст](./textarea.md).

## Сценарии использования

URL подходит для следующих бизнес-сценариев:

- Официальный сайт клиента или поставщика
- Ссылка на страницу сведений во внешней системе
- Ссылка на договор, документ или страницу базы знаний
- Адрес Webhook или адрес обратного вызова

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «URL», чтобы создать поле URL.

![20240512175641](https://static-docs.nocobase.com/20240512175641.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для URL соответствует `url` и определяет способ ввода и отображения в интерфейсе. |
| Field display name | Имя, под которым поле отображается в интерфейсе, например «Официальный сайт», «Ссылка на документ» или «Внешний адрес». Рекомендуется использовать понятное сотрудникам название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, разрешениях, рабочих процессах и других компонентах. Обычно после создания не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля URL по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При создании записи оно будет подставлено автоматически, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно настроить проверку формата URL, длины или обязательности заполнения. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля используется блоками страниц, разрешениями, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле URL по умолчанию ведёт себя следующим образом:

| Характеристика | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `url`. |
| Тип Field type по умолчанию | `string`. |
| Доступный тип Field type | `string`. |
| Компонент страницы | В режиме редактирования используется поле ввода, а в режиме просмотра значение обычно отображается как ссылка. |
| Фильтрация | Поддерживаются текстовые фильтры, например «содержит», «равно», «пусто» и «не пусто». |
| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Проверка | Поддерживается проверка формата URL, длины, обязательности и других параметров. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля URL. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле поступает из таблицы, синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления полей — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе имени поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Интерфейс поля основной базы данных или синхронизированного поля можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | В зависимости от условий | Тип поля основной базы данных или синхронизированного поля можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение информации о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле URL. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля URL обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, разрешения, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в конфигурации страниц

Поле URL подходит для использования в блоках сведений, таблиц и переходов на внешние страницы.
![20260709224747](https://static-docs.nocobase.com/20260709224747.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод адреса ссылки. |
| Блок сведений | Отображение и открытие ссылки. |
| Блок таблицы | Отображение краткой информации о ссылке или кликабельной ссылки. |
| Рабочий процесс | Использование ссылки в содержимом уведомления или в качестве параметра внешнего запроса. |
