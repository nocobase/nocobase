# Однострочный текст

## Введение

В NocoBase **однострочный текст (Single line text)** — наиболее часто используемое текстовое поле. Оно подходит для хранения кратких текстовых данных длиной не более одной строки, например названия клиента, номера заказа, контактного лица, краткого адреса или идентификатора во внешней системе.

Поле однострочного текста по умолчанию использует `Input` поле ввода, а его Field type по умолчанию — `string`. Его можно использовать как поле заголовка, а также в фильтрах, сортировке, настройках доступа, условиях рабочих процессов и API-запросах.

Если содержимое может содержать переносы строк или быть достаточно длинным, по умолчанию лучше выбрать [многострочный текст](./textarea.md). Если содержимое имеет фиксированный формат, например адрес электронной почты, номер телефона или URL, предпочтительно выбрать соответствующее специализированное поле.

## Сценарии использования

Однострочный текст подходит для следующих бизнес-сценариев:

- Название клиента, название компании, имя контактного лица
- Номер заказа, номер договора, номер проекта
- Заголовок задачи, заголовок обращения, заголовок статьи
- ID внешней системы, номер синхронизации, бизнес-код
- Город, краткий адрес, название магазина и другие краткие текстовые данные

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Single line text», чтобы создать поле однострочного текста.

![20240512163555](https://static-docs.nocobase.com/20240512163555.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для однострочного текста соответствует `input`; на странице по умолчанию используется поле ввода для ввода и отображения данных. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Название клиента», «Номер заказа» или «Заголовок задачи». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, настройках доступа, рабочих процессах и т. д. После создания обычно не изменяется; поддерживаются только буквы, цифры и символы подчёркивания, при этом название должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для однострочного текста по умолчанию используется `string`; также можно выбрать `uid`. Для обычного краткого текста обычно достаточно `string`. |
| Automatically remove heading and tailing spaces | Автоматически удалять пробелы в начале и конце. Подходит для названий клиентов, номеров, заголовков и других данных, в которых не следует сохранять начальные и конечные пробелы. |
| Default value | Значение по умолчанию. При создании записи, если пользователь ничего не ввёл, это значение будет подставлено автоматически. |
| Primary | Использовать поле в качестве первичного ключа. Доступно только при создании поля в основной базе данных; обычные бизнес-тексты не рекомендуется использовать как первичный ключ. |
| Unique | Ограничение уникальности. Подходит для номеров заказов, договоров, ID внешних систем и других текстовых данных, которые не должны повторяться. |
| Validation rules | Правила проверки. Можно задать минимальную, максимальную или фиксированную длину, а также регулярное выражение. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за сопровождение. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, настройками доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле однострочного текста по умолчанию ведёт себя следующим образом:

| Характеристика | Описание |
| --- | --- |
| Field interface по умолчанию | `input`. |
| Field type по умолчанию | `string`. |
| Доступные Field type | `string`、`uid`. |
| Компонент страницы | В режиме редактирования используется поле ввода `Input`. |
| Поле заголовка | Поддерживается использование в качестве поля заголовка таблицы данных. Подходит для назначения полями заголовка значений «Название клиента», «Номер заказа» и «Заголовок задачи». |
| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Фильтрация | Поддерживаются фильтры для текста, например «содержит», «не содержит», «равно», «не равно», «пусто», «не пусто» и другие. |
| Проверка | Поддерживаются проверки минимальной, максимальной и фиксированной длины, регулярных выражений и другие проверки. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля однострочного текста. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например отображаемого названия, описания, значения по умолчанию, правил проверки или настройки автоматического удаления начальных и конечных пробелов.

Если поле поступает из уже синхронизированной таблицы основной базы данных, при редактировании обычно выполняется сопоставление полей — поле базы данных сопоставляется с Field type и Field interface NocoBase. Например, короткие текстовые столбцы `varchar`、`char` и другие в базе данных можно сопоставить с полем однострочного текста.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Для полей основной базы данных или синхронизированных полей можно изменить сопоставление. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | В зависимости от условий | Для полей основной базы данных или синхронизированных полей можно изменить сопоставление. Перед изменением необходимо проверить, смогут ли существующие данные использоваться с новым типом. |
| Automatically remove heading and tailing spaces | Да | Определяет, будут ли при сохранении удаляться начальные и конечные пробелы. |
| Default value | Да | Изменяет текст, подставляемый по умолчанию при создании новых записей. |
| Unique | В зависимости от условий | Можно настроить для нового поля основной базы данных. Если в существующих данных уже есть повторяющиеся значения, добавление ограничения уникальности может завершиться ошибкой. |
| Validation rules | Да | Изменяет проверки длины, формата или регулярного выражения. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за сопровождение. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле однострочного текста. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля однострочного текста, созданного в основной базе данных, обычно одновременно удаляются реальный столбец в базе данных и все данные, уже содержащиеся в этом столбце. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, настройки доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в настройках страниц

Поле однострочного текста можно использовать в большинстве сценариев с блоками данных и формами.

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод или редактирование кратких текстовых данных, например названия клиента, номера заказа или заголовка задачи. |
| Блок таблицы | Отображение, сортировка и фильтрация кратких текстовых данных. Если содержимое длинное, в таблице оно будет сокращено в соответствии с настройками интерфейса. |
| Блок подробной информации | Отображение текстовой информации отдельной записи. |
| Блок фильтрации | Использование в качестве условия запроса для фильтрации записей, например по названию клиента, номеру или заголовку. |
| Отображение поля связи | Если поле однострочного текста задано как поле заголовка, при выборе записи в поле связи этот текст будет отображаться в первую очередь. |
| Рабочие процессы и настройки доступа | Использование в качестве поля условия, например для проверки, пуст ли номер заказа или содержит ли название клиента определённое ключевое слово. |
