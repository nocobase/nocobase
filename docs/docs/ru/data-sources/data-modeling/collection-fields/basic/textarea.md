# Многострочный текст

## Введение

В NocoBase **многострочный текст (Multi-line text)** используется для хранения текста, который требует переносов строк или имеет большой объём.

По умолчанию для многострочного текста используется многострочное поле ввода, подходящее для заметок, пояснений, комментариев по обработке и других подобных данных. Это поле можно использовать в фильтрах, правах доступа, условиях рабочих процессов и API-запросах.

Если содержимое обычно состоит только из одной строки, например названия, номера или заголовка, лучше выбрать [однострочный текст](./input.md). Если требуется форматирование, изображения или возможности форматированного текста, выберите поле форматированного текста или Markdown.

## Сценарии использования

Многострочный текст подходит для следующих бизнес-сценариев:

- Заметки клиента, заметки к заказу, комментарии по обработке заявки
- Подробный адрес, описание проблемы, описание требований
- Краткое содержание условий договора, описание контекста проекта
- Текст, который необходимо вводить с переносами строк

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Многострочный текст», чтобы создать поле многострочного текста.

![20240512165017](https://static-docs.nocobase.com/20240512165017.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для многострочного текста соответствует `textarea` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Заметки клиента», «Комментарий по обработке», «Подробный адрес». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. Обычно после создания не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для многострочного текста по умолчанию используется `text`; в зависимости от исходного поля также может быть сопоставлен с `string` или `json`. |
| Default value | Значение по умолчанию. При создании записи оно подставляется автоматически, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно ограничить минимальную, максимальную или фиксированную длину, регистр или задать регулярное выражение. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Многострочное текстовое поле по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `textarea`. |
| Тип Field type по умолчанию | `text`. |
| Доступные типы Field type | `text`, `json`, `string`. |
| Компонент страницы | В режиме редактирования используется многострочное поле ввода. |
| Фильтрация | Поддерживаются текстовые фильтры, например «содержит», «не содержит», «пусто», «не пусто» и другие. |
| Сортировка | Обычно не рекомендуется использовать для сортировки. Возможность сортировки зависит от конкретной базы данных и типа поля. |
| Проверка | Поддерживаются проверки минимальной, максимальной и фиксированной длины, регистра, регулярных выражений и другие. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить настройки многострочного текстового поля. Редактирование поля в основном используется для настройки его отображения и способов использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, не меняя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить многострочное текстовое поле. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных многострочного текстового поля обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в конфигурации страниц

Многострочное текстовое поле подходит для отображения длинного содержимого в формах и подробных представлениях.

![20260709224428](https://static-docs.nocobase.com/20260709224428.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод или редактирование заметок, пояснений, комментариев по обработке и других данных. |
| Блок подробного представления | Отображение длинного текста. |
| Блок таблицы | Отображение краткого содержания; длинный текст обычно сокращается. |
| Рабочие процессы и права доступа | Использование в качестве условного поля, например для проверки, является ли заметка пустой. |
