# Процент

## Обзор

В NocoBase **процент (Percent)** используется для хранения и отображения пропорциональных данных.

Процентное поле подходит для хранения таких бизнес-данных, как процент выполнения, скидка, коэффициент конверсии и доля. По сути, это числовое поле, но отображение и ввод на странице больше соответствуют семантике процентов.

Если речь идет об обычной сумме, количестве или оценке, лучше выбрать [число](./number.md).

## Сценарии использования

Процент подходит для следующих бизнес-сценариев:

- Процент выполнения проекта, прогресс выполнения задачи
- Скидка, налоговая ставка, размер комиссии
- Коэффициент конверсии, процент достижения, доля
- Вес оценки, пропорция распределения

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Процент», чтобы создать процентное поле.

![20240512175847](https://static-docs.nocobase.com/20240512175847.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для процента используется `percent`, который определяет способ ввода и отображения на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Процент выполнения», «Скидка» или «Коэффициент конверсии». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. Обычно после создания не изменяется; допускаются только буквы, цифры и символы подчеркивания, причем имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для процентного поля обычно используется `double`, но в зависимости от требований к точности можно выбрать `decimal`. |
| Default value | Значение по умолчанию. При создании записи оно подставляется автоматически, если пользователь не ввел значение. |
| Validation rules | Правила валидации. Позволяют ограничить минимальное и максимальное значения или сделать поле обязательным. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Процентное поле по умолчанию ведет себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `percent`. |
| Тип Field type по умолчанию | `double`. |
| Доступные типы Field type | `float`, `double`, `decimal`. |
| Компонент страницы | В режиме редактирования используется компонент ввода процентов. |
| Фильтрация | Поддерживается фильтрация числовых значений: больше, меньше, диапазон, пусто и не пусто. |
| Сортировка | Поддерживается сортировка в блоке таблицы. |
| Валидация | Поддерживается проверка диапазона значений, обязательности поля и другие правила. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки процентного поля. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил валидации или специальных параметров поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет имя поля, отображаемое в интерфейсе, но не меняет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определенных условиях | Для полей основной базы данных или синхронизированных полей его можно изменить во время сопоставления. Изменение влияет на способ ввода, отображения и валидации на странице. |
| Field type | При определенных условиях | Для полей основной базы данных или синхронизированных полей его можно изменить во время сопоставления. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила валидации поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила валидации, условия фильтрации и использование переменных в рабочих процессах. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить процентное поле. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении процентного поля, созданного в основной базе данных, обычно одновременно удаляются реальный столбец в базе данных и все данные, которые в нем хранились. При удалении поля, синхронизированного с базой данных или сопоставленного из внешнего источника данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурации.

:::

## Использование в настройках страниц

Процентные поля подходят для отображения пропорций в бизнес-формах, на информационных панелях, графиках и отчетах.
![20260709225150](https://static-docs.nocobase.com/20260709225150.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод процента выполнения, скидки, налоговой ставки и других пропорций. |
| Блок таблицы | Отображение, сортировка и фильтрация пропорциональных данных. |
| Блок графика | Отображение таких показателей, как доля и коэффициент конверсии. |
| Рабочие процессы и права доступа | Использование в качестве поля условия, например для проверки, достиг ли процент выполнения 100%. |
