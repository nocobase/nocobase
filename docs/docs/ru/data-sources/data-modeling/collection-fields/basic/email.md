# Электронная почта

## Описание

В NocoBase поле **электронной почты (Email)** используется для хранения адресов электронной почты.

Поле электронной почты подходит для хранения адресов клиентов, сотрудников, поставщиков и других контактных данных. По сравнению с обычным однострочным текстом оно имеет более явную семантику адреса электронной почты и поддерживает проверку формата.

Если значение не является адресом электронной почты, а представляет собой обычную контактную информацию, лучше выбрать [однострочный текст](./input.md).

## Сценарии использования

Поле электронной почты подходит для следующих бизнес-сценариев:

- Адреса клиентов и контактных лиц
- Адреса сотрудников и контактные адреса для входа
- Адреса поставщиков и служебные адреса
- Адреса получателей уведомлений

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Электронная почта», чтобы создать поле электронной почты.

![20240512175609](https://static-docs.nocobase.com/20240512175609.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для электронной почты используется `email`, который определяет способ ввода и отображения значения на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Электронная почта клиента», «Электронная почта контакта» или «Адрес получателя». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля электронной почты по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При добавлении записи оно может быть подставлено автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Обычно следует включить проверку формата электронной почты; также можно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за его сопровождение. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поле электронной почты по умолчанию ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `email`. |
| Тип Field type по умолчанию | `string`. |
| Доступный тип Field type | `string`. |
| Компонент страницы | В режиме редактирования используется поле ввода с проверкой формата электронной почты. |
| Фильтрация | Поддерживаются текстовые условия фильтрации, например «содержит», «равно», «пусто» и «не пусто». |
| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Проверка | Поддерживаются проверка формата электронной почты, обязательность заполнения и другие правила. |

## Редактирование конфигурации

После создания поля нажмите «Edit» справа от него, чтобы изменить его настройки. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не меняет его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Это повлияет на ввод, отображение и проверку на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за сопровождение. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом количестве существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле электронной почты. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля электронной почты, созданного в основной базе данных, обычно одновременно удаляются фактический столбец в базе данных и уже имеющиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
