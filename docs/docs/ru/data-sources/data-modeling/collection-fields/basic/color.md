# Цвет

## Введение

В NocoBase **цвет (Color)** используется для хранения значений цвета.

Поле цвета подходит для хранения цветов категорий, тегов, статусов, диаграмм или настроек отображения. Обычно в нём хранятся шестнадцатеричные значения цвета или форматы цветов, поддерживаемые компонентом.

Если цвет является лишь частью поля вариантов, его можно настроить непосредственно в поле вариантов — создавать отдельное поле цвета не обязательно.

## Сценарии использования

Цвет подходит для следующих бизнес-сценариев:

- Цвет уровня клиента, цвет статуса
- Цвет тега, цвет категории
- Цвет серии диаграммы
- Настройки отображения страницы или карточки

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Цвет», чтобы создать поле цвета.

![20240512175956](https://static-docs.nocobase.com/20240512175956.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для цвета соответствует `color` и определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Цвет статуса», «Цвет тега» или «Цвет диаграммы». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля цвета по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При создании записи оно будет подставлено автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за поддержку. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поле цвета по умолчанию ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Field interface по умолчанию | `color`. |
| Field type по умолчанию | `string`. |
| Доступный Field type | `string`. |
| Компонент страницы | В режиме редактирования используется компонент выбора цвета. |
| Фильтрация | Можно фильтровать по значению цвета, но обычно это не основной критерий поиска. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, например обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля цвета. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например названия, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления поля — отображение поля базы данных в Field type и Field interface NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, но не изменяет его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле цвета. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля цвета обычно одновременно удаляются фактический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
