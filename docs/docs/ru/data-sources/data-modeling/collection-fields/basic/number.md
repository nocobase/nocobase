# Число

## Введение

В NocoBase **число (Number)** используется для хранения числовых значений, которые могут содержать десятичные дроби.

Числовые поля подходят для хранения сумм, веса, площади, оценок, цен за единицу и других бизнес-данных. Их можно использовать для фильтрации, сортировки, статистики, формул и условий рабочих процессов.

Если значение должно быть целым числом, выберите[целое число](./integer.md). Если нужно отображать значение как долю или процент, выберите[процент](./percent.md).

## Сценарии использования

Числа подходят для следующих бизнес-сценариев:

- Сумма заказа, сумма договора, цена за единицу
- Вес, площадь, объём, расстояние
- Оценка, коэффициент, значение до применения скидки
- Десятичные значения, участвующие в статистике или вычислениях по формулам

## Настройка при создании

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Число», чтобы создать числовое поле.

![20240512175752](https://static-docs.nocobase.com/20240512175752.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для числового поля соответствует `number` и определяет способ ввода и отображения значения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Сумма заказа», «Оценка» или «Вес». Рекомендуется использовать название, понятное сотрудникам бизнеса. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. По умолчанию числовое поле имеет тип `double`; для сценариев, требующих точных десятичных значений, например денежных сумм, можно выбрать `decimal`. |
| Default value | Значение по умолчанию. При добавлении записи оно автоматически подставляется, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно ограничить минимальное и максимальное значения, точность или обязательность заполнения. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за его ведение. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

По умолчанию числовое поле ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `number`. |
| Тип Field type по умолчанию | `double`. |
| Доступные типы Field type | `float`, `double`, `decimal`. |
| Компонент страницы | В режиме редактирования используется поле ввода числа. |
| Фильтрация | Поддерживается фильтрация по числовым значениям: равно, не равно, больше, меньше, диапазон, пусто, не пусто и т. д. |
| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Проверка | Поддерживается проверка диапазона значений, обязательности заполнения и т. д. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки числового поля. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого названия, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления полей — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс поля основной базы данных или синхронизированного поля можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Тип поля основной базы данных или синхронизированного поля можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию при добавлении новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за его ведение. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить числовое поле. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении числового поля, созданного в основной базе данных, обычно одновременно удаляется соответствующий физический столбец в базе данных и все данные, которые в нём хранятся. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::

## Использование в настройках страниц

Числовые поля подходят для ввода данных, статистики, построения графиков и принятия решений в рабочих процессах.
![20260709225103](https://static-docs.nocobase.com/20260709225103.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод сумм, оценок, веса и других числовых значений. |
| Блок таблицы | Отображение, сортировка и фильтрация числовых значений. |
| Блок графика | Группировка, суммирование или вычисление среднего по числовому полю. |
| Поле формулы | Использование в качестве входного поля для вычислений по формуле. |
