# Номер телефона

## Введение

В NocoBase поле **номера телефона (Phone)** используется для хранения номеров мобильных телефонов и контактных телефонов.

Поле номера телефона подходит для хранения телефонов клиентов, контактных телефонов, мобильных номеров сотрудников и другой контактной информации. Оно лучше обычного текста подходит для представления данных телефонного типа.

Если необходимо сохранить несколько телефонных номеров или сложную контактную информацию, можно использовать [многострочный текст](./textarea.md) или создать отдельную таблицу контактов.

## Сценарии использования

Поле номера телефона подходит для следующих бизнес-сценариев:

- Мобильные номера клиентов, телефоны контактных лиц
- Мобильные номера сотрудников, резервные телефоны
- Контактные телефоны магазинов, горячие линии
- Номера для SMS-уведомлений

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Номер телефона», чтобы создать поле номера телефона.

![20240512175526](https://static-docs.nocobase.com/20240512175526.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для номера телефона используется `phone`, который определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Номер телефона», «Контактный телефон» или «Горячая линия». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём название должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля номера телефона по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При добавлении записи оно может быть подставлено автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно настроить длину, регулярное выражение или обязательность заполнения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

| Характеристика | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `phone`. |
| Тип Field type по умолчанию | `string`. |

| Сортировка | Поддерживается сортировка в блоках таблиц. |
| Проверка | Поддерживается проверка длины, регулярного выражения, обязательности и другие проверки. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля номера телефона. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле номера телефона. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля номера телефона обычно одновременно удаляются настоящий столбец в базе данных и имеющиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
