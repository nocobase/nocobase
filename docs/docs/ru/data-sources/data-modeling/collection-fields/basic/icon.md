# Иконка

## Введение

В NocoBase **иконка (Icon)** используется для хранения идентификатора иконки.

Поле иконки подходит для добавления визуальной идентификации категориям, меню, статусам и точкам входа. Оно хранит значение иконки, а при отображении на странице визуализируется компонентом иконки.

Если нужно загрузить настоящее изображение, выберите [вложение](../media/field-attachment.md). Если нужно только сохранить описание иконки, выберите [однострочный текст](./input.md).

## Сценарии использования

Иконки подходят для следующих бизнес-сценариев:

- Иконки меню и функциональных точек входа
- Иконки категорий и тегов
- Иконки статусов и уровней
- Визуальная идентификация на дашбордах или карточках

## Создание конфигурации

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Иконка», чтобы создать поле иконки.

![20240512180027](https://static-docs.nocobase.com/20240512180027.png)

| Конфигурация | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для иконки соответствует `icon` и определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Иконка меню», «Иконка категории» или «Иконка статуса». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля иконки по умолчанию используется `string`. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь удобно указать назначение поля, требования к заполнению, источник данных или ответственного за его поддержку. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поле иконки по умолчанию ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `icon`. |
| Тип Field type по умолчанию | `string`. |
| Доступный тип Field type | `string`. |
| Компонент страницы | В режиме редактирования используется компонент выбора иконки. |
| Фильтрация | Обычно не используется как основное условие фильтрации. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, включая обязательность заполнения. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля иконки. Редактирование поля предназначено главным образом для настройки его отображения и использования в NocoBase, например изменения отображаемого названия, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Конфигурация | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе названия поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. После изменения способ ввода, отображения и проверки на страницах будет изменён. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение назначения поля, требований к заполнению, источника данных или сведений об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле иконки. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля иконки, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::

## Использование в конфигурации страниц

Поле иконки подходит для визуальной индикации в списках, карточках и подробной информации.
![20260709225630](https://static-docs.nocobase.com/20260709225630.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Выбор иконки. |
| Блок подробной информации | Отображение иконки. |
| Список или карточка | Визуальная идентификация категории, статуса или точки входа. |
| Права доступа и рабочие процессы | Обычно не используется как основное условное поле. |
