# Пароль

## Введение

В NocoBase поле **«Пароль» (Password)** используется для ввода паролей и других подобных значений.

Поле «Пароль» подходит для хранения данных, ввод которых необходимо скрыть, например паролей внешних сервисов и временных кодов доступа. Его основное назначение — скрывать ввод и отображение данных; оно не является полноценным решением для управления ключами.

Для хранения высокочувствительных ключей или долгосрочных учётных данных рекомендуется в первую очередь рассмотреть специализированные решения для шифрования, управления ключами или использования переменных окружения.

## Сценарии использования

Поле «Пароль» подходит для следующих бизнес-сценариев:

- Временные пароли внешних систем
- Коды доступа в конфигурациях подключений
- Чувствительный текст, который требуется вводить только в замаскированном виде
- Временные коды подтверждения или пароли во внутренних процессах

## Создание конфигурации

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Пароль», чтобы создать поле «Пароль».

![20240512175917](https://static-docs.nocobase.com/20240512175917.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для поля «Пароль» используется `password`, который определяет способ ввода и отображения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Пароль доступа», «Код доступа для подключения» или «Временный пароль». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля «Пароль» обычно используются `password` или `string`. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно настроить длину, регулярное выражение или обязательность заполнения. |
| Description | Описание поля. Здесь удобно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля «Пароль». Редактирование поля используется главным образом для настройки его отображения и способов использования в NocoBase, например для изменения отображаемого названия, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле поступает из уже синхронизированной таблицы основной базы данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе название поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Для полей основной базы данных или синхронизированных полей интерфейс можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | В зависимости от условий | Для полей основной базы данных или синхронизированных полей тип можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. При наличии большого объёма данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле «Пароль». В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля «Пароль», созданного в основной базе данных, обычно одновременно удаляются фактический столбец в базе данных и хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного из внешнего источника данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
