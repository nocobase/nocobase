# Полигон

## Введение

В NocoBase поле **«Полигон (Polygon)»** используется для хранения площадных пространственных областей.

Поле «Полигон» подходит для хранения данных о территориях административных районов, зонах доставки и продаж, закрытых и других областях. В сочетании с блоком карты оно позволяет отображать границы областей.

Если область имеет форму простого круга, выберите [Круг](./circle.md). Если нужно сохранить только одну позицию, выберите [Точку](./point.md).

## Сценарии использования

Полигон подходит для следующих бизнес-сценариев:

- Зоны продаж и доставки
- Зоны обслуживания и управления
- Закрытые и опасные зоны
- Границы бизнес-территорий на карте

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Полигон», чтобы создать поле «Полигон».

![20240512181547](https://static-docs.nocobase.com/20240512181547.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейс поля. Для полигона соответствует `polygon` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Зона продаж», «Зона доставки» или «Опасная зона». Рекомендуется использовать понятное сотрудникам название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других функциях. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля «Полигон» по умолчанию используется `polygon`. |
| Default value | Значение по умолчанию. При добавлении записи оно автоматически подставляется, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле «Полигон» по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `polygon`. |
| Тип Field type по умолчанию | `polygon`. |
| Доступный тип Field type | `polygon`. |
| Компонент страницы | В режиме редактирования используется компонент для рисования на карте. |
| Фильтрация | Возможность пространственной фильтрации зависит от возможностей плагина карты и источника данных. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, включая обязательность заполнения. |

## Редактирование настройки

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля «Полигон». Редактирование поля в основном используется для настройки его отображения и поведения в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение назначения поля, требований к заполнению, источника данных или сведений об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле «Полигон». В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля «Полигон», созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурации.

:::
