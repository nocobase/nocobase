# Линия

## Введение

В NocoBase **линия (LineString)** используется для хранения линейных пространственных данных.

Поле линии подходит для таких бизнес-данных, как маршруты, траектории, трубопроводы и пути инспекций. В сочетании с блоком карты оно позволяет отображать пути.

Если требуется указать только одно местоположение, выберите[точку](./point.md). Если требуется указать область, выберите[полигон](./polygon.md).

## Сценарии использования

Линии подходят для следующих бизнес-сценариев:

- Маршруты доставки и инспекций
- Траектории движения транспортных средств и людей
- Трубопроводы, линии и границы
- Результаты планирования маршрутов на карте

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Линия», чтобы создать поле линии.

![20240512181454](https://static-docs.nocobase.com/20240512181454.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для линии соответствует `lineString` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Маршрут доставки», «Траектория инспекции» или «Трубопровод». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля линии по умолчанию используется `lineString`. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Заранее проверьте название, чтобы избежать дополнительных затрат на настройку при последующих изменениях.

:::

## Особенности поля

Поле линии по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `lineString`. |
| Тип Field type по умолчанию | `lineString`. |
| Доступный тип Field type | `lineString`. |
| Компонент страницы | В режиме редактирования используется компонент рисования на карте. |
| Фильтрация | Возможность пространственной фильтрации зависит от возможностей плагина карты и источника данных. |
| Сортировка | Обычно для сортировки не используется. |
| Проверка | Поддерживаются базовые проверки, включая обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля линии. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки данных на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. Если существующих данных много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле линии. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля линии, созданного в основной базе данных, обычно одновременно удаляется соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
