# Круг

## Введение

В NocoBase **круг (Circle)** используется для хранения круглой области.

Поле «Круг» подходит для хранения таких бизнес-данных, как радиус обслуживания, зона доставки и зона покрытия магазина.

Если область не является круглой, выберите[многоугольник](./polygon.md). Если нужна только центральная точка, выберите[точку](./point.md).

## Сценарии использования

Круг подходит для следующих бизнес-сценариев:

- Радиус обслуживания магазина
- Зона покрытия доставки
- Зона влияния устройства
- Область поиска вокруг определённой точки

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Круг», чтобы создать поле «Круг».

![20240512181522](https://static-docs.nocobase.com/20240512181522.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейс поля. Для круга используется `circle`, определяющий способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Радиус обслуживания», «Зона покрытия» или «Зона влияния». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом название должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля «Круг» по умолчанию используется `circle`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за его сопровождение. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

| Проверка | Поддерживаются базовые проверки, такие как обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля «Круг». Редактирование поля в основном используется для настройки способа его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе имени поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Для полей основной базы данных или синхронизированных полей интерфейс можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | В зависимости от условий | Для полей основной базы данных или синхронизированных полей тип можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о назначении поля, требованиях к заполнению, источнике данных или ответственном за сопровождение. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле «Круг». В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля «Круг», созданного в основной базе данных, обычно одновременно удаляются соответствующий столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурациях.

:::
