# Точка

## Введение

В NocoBase **точка (Point)** используется для сохранения координат одного местоположения.

Поле точки подходит для хранения пространственных данных о местоположении магазинов, клиентов, устройств и других объектов. В сочетании с блоком карты записи можно отображать на карте.

Если нужно сохранить маршрут, выберите [линию](./line.md). Если нужно сохранить область, выберите [полигон](./polygon.md) или [окружность](./circle.md).

## Подходящие сценарии

Точка подходит для следующих бизнес-сценариев:

- Местоположение магазина или склада
- Координаты адреса клиента
- Местоположение установки устройства
- Место отметки при проведении инспекции

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Точка», чтобы создать поле точки.

![20240512181420](https://static-docs.nocobase.com/20240512181420.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для точки соответствует `point` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Местоположение магазина», «Координаты устройства» или «Местоположение клиента». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля точки по умолчанию используется `point`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно достаточно настроить обязательность заполнения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственное лицо. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Заранее проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле точки по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `point`. |
| Тип Field type по умолчанию | `point`. |
| Допустимый тип Field type | `point`. |
| Компонент страницы | В режиме редактирования используется компонент выбора карты или координат. |
| Фильтрация | Возможность пространственной фильтрации зависит от возможностей плагина карты и источника данных. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, например обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля точки. Редактирование поля в основном используется для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей интерфейс можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей тип можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном лице. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных рабочих процессов. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле точки. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля точки, созданного в основной базе данных, обычно одновременно удаляются соответствующий реальный столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::

## Использование в настройках страницы

Поле точки подходит для сценариев работы с картами и управления местоположениями.
![20260710144034](https://static-docs.nocobase.com/20260710144034.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Выбор или ввод одного местоположения. |
| Блок сведений | Отображение координат местоположения или точки на карте. |
| Блок карты | Отображение точек на карте. |
| Рабочий процесс | Использование в качестве входных данных для условий, связанных с местоположением. |
