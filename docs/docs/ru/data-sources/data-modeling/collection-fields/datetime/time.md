# Время

## Введение

В NocoBase **время (Time)** используется для хранения времени в течение дня.

Поле времени подходит для бизнес-данных, не привязанных к конкретной дате: часов работы, времени напоминаний, временных интервалов смен и т. п.

Если необходимо одновременно сохранять дату и время, выберите[дату и время](./datetime.md).

## Подходящие сценарии

Время подходит для следующих бизнес-сценариев:

- Время начала и окончания работы
- Ежедневное время напоминания
- Время начала и окончания смены
- Настройка фиксированных моментов времени

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Время», чтобы создать поле времени.

![20240512181216](https://static-docs.nocobase.com/20240512181216.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для времени соответствует `time` и определяет способ ввода и отображения на странице. |
| Field display name | Название, под которым поле отображается в интерфейсе, например «Время начала», «Время напоминания», «Часы работы». Рекомендуется использовать название, понятное сотрудникам. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля времени по умолчанию используется `time`. |
| Default value | Значение по умолчанию. При добавлении записи оно автоматически подставляется, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно настроить обязательность заполнения, диапазон времени и другие параметры. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

Имя поля после создания будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать последующих затрат на изменение конфигурации.

:::

## Особенности поля

Поле времени по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `time`. |
| Тип Field type по умолчанию | `time`. |
| Доступный тип Field type | `time`. |
| Компонент страницы | В режиме редактирования используется выбор времени. |
| Фильтрация | Поддерживается фильтрация по времени, диапазону, пустым и непустым значениям. |
| Сортировка | Поддерживается сортировка по времени. |
| Проверка | Поддерживается проверка обязательности заполнения и диапазона времени. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля времени. Редактирование поля в основном используется для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из уже синхронизированной таблицы основной базы данных, его редактирование обычно означает сопоставление поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое имя поля в интерфейсе, не изменяя идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. После изменения меняются способы ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить смысл поля, требования к заполнению, источник данных или сведения об ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. Если существующих данных много, сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле времени. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля времени обычно одновременно удаляются фактический столбец в базе данных и хранящиеся в этом столбце данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтрацию, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
