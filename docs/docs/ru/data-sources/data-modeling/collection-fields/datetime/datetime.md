# Дата и время (с часовым поясом)

## Введение

В NocoBase поле **даты и времени (с часовым поясом) (Date time with timezone)** используется для хранения даты и времени с обработкой с учётом часового пояса.

Дата и время (с часовым поясом) подходят для совместной работы в разных часовых поясах, интернационализированных бизнес-процессов и сценариев, требующих однозначной временной точки, например для создания бронирований, указания сроков и времени выполнения.

Если бизнес-логика учитывает только текст местного времени и преобразование часовых поясов не требуется, можно выбрать [дату и время (без часового пояса)](./datetime-without-tz.md). Если требуется только дата, выберите [дату](./date.md).

## Сценарии использования

Дата и время (с часовым поясом) подходят для следующих бизнес-сценариев:

- время начала встречи, время бронирования
- срок выполнения задачи, время выполнения
- временные точки в процессах между часовыми поясами
- время, связанное с условиями планирования рабочих процессов

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Дата и время (с часовым поясом)», чтобы создать поле даты и времени (с часовым поясом).

![20240512181142](https://static-docs.nocobase.com/20240512181142.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Дате и времени (с часовым поясом) соответствует `datetime`, определяющий способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Время начала», «Срок выполнения» или «Время выполнения». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом название должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для даты и времени (с часовым поясом) обычно используется `date`. |
| Default value | Значение по умолчанию. При добавлении записи оно может быть подставлено автоматически, если пользователь не указал значение. |
| Validation rules | Правила проверки. Можно настроить обязательность заполнения, диапазон времени и другие параметры. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать последующих затрат на изменение конфигурации.

:::

## Особенности поля

Поле даты и времени (с часовым поясом) по умолчанию имеет следующие характеристики:

| Характеристика | Описание |
| --- | --- |
| Default Field interface | `datetime`. |
| Default Field type | `date`. |
| Optional Field type | `date`. |
| Компонент страницы | В режиме редактирования используется селектор даты и времени. |
| Фильтрация | Поддерживается фильтрация по временной точке, диапазону, пустым и непустым значениям. |
| Сортировка | Поддерживается сортировка по времени. |
| Проверка | Поддерживается проверка обязательности заполнения, диапазона времени и других условий. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля даты и времени (с часовым поясом). Редактирование поля в основном используется для изменения способа его отображения и использования в NocoBase, например названия, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы основной базы данных, которая уже синхронизирована, редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет название поля, отображаемое в интерфейсе, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Поле основной базы данных или синхронизированное поле можно изменить при сопоставлении. После изменения изменятся способы ввода, отображения и проверки на странице. |
| Field type | В зависимости от условий | Поле основной базы данных или синхронизированное поле можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно будет использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При наличии большого объёма данных сначала убедитесь, что формат данных совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле даты и времени (с часовым поясом). В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля даты и времени (с часовым поясом), созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
