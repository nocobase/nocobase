# Дата и время (без часового пояса)

## Введение

В NocoBase **дата и время (без часового пояса) (Date time without timezone)** используются для сохранения даты и времени без преобразования часового пояса.

Дата и время (без часового пояса) подходят для таких сценариев, как составление расписания, часы работы и время занятий, где важнее локальное отображаемое значение.

Если нужно выразить единый для всего мира фактический момент времени, лучше выбрать [дату и время (с часовым поясом)](./datetime.md).

## Подходящие сценарии

Дата и время (без часового пояса) подходят для следующих бизнес-сценариев:

- Локальное время расписания
- Время начала занятия, время экзамена
- Время открытия или закрытия магазина
- Бизнес-время, которое не должно преобразовываться при смене часового пояса

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Дата и время (без часового пояса)», чтобы создать поле даты и времени (без часового пояса).

![20260709232834](https://static-docs.nocobase.com/20260709232834.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Дате и времени (без часового пояса) соответствует `datetimeNoTz`, который определяет способ ввода и отображения на странице. |
| Field display name | Имя, под которым поле отображается в интерфейсе, например «Время расписания», «Время занятия» или «Часы работы». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для даты и времени (без часового пояса) обычно используется `datetimeNoTz`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно настроить обязательность заполнения, диапазон времени и другие параметры. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за поддержку. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле даты и времени (без часового пояса) по умолчанию ведёт себя следующим образом:

| Характеристика | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `datetimeNoTz`. |
| Тип Field type по умолчанию | `datetimeNoTz`. |
| Допустимый тип Field type | `datetimeNoTz`. |
| Компонент страницы | В режиме редактирования используется выбор даты и времени. |
| Фильтрация | Поддерживается фильтрация по моменту времени, диапазону, пустым и непустым значениям. |
| Сортировка | Поддерживается сортировка по времени. |
| Проверка | Поддерживается проверка обязательности заполнения, диапазона времени и других условий. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля даты и времени (без часового пояса). Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из уже синхронизированной таблицы основной базы данных, при редактировании обычно выполняется сопоставление полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о назначении поля, требованиях к заполнению, источнике данных или ответственном за поддержку. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле даты и времени (без часового пояса). В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля даты и времени (без часового пояса) обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурации.

:::
