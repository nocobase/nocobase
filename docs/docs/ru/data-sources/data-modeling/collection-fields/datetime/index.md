# Обзор

## Типы полей даты и времени

Типы полей даты и времени включают следующие варианты:

- **Дата и время (с часовым поясом)** — дата и время преобразуются в единое время UTC (всемирное координированное время), а при необходимости выполняется преобразование часового пояса;
- **Дата и время (без часового пояса)** — хранит дату и время без информации о часовом поясе;
- **Дата (без времени)** — хранит только дату, без компонента времени;
- **Время** — хранит только время, без компонента даты;
- **Временная метка Unix** — хранится как временная метка Unix, обычно представляющая количество секунд с 1 января 1970 года.

Примеры типов полей, связанных с датой:

| **Тип поля**                    | **Пример значения**           | **Описание**                                      |
|-------------------------------|-------------------------------|---------------------------------------------------|
| Дата и время (с часовым поясом) | 2024-08-24T07:30:00.000Z      | Дата и время преобразуются в единое время UTC (всемирное координированное время) |
| Дата и время (без часового пояса) | 2024-08-24 15:30:00         | Дата и время без часового пояса, записываются только дата и время |
| Дата (без времени)             | 2024-08-24                    | Хранится только информация о дате, без времени |
| Время                         | 15:30:00                      | Хранится только информация о времени, без даты |
| Временная метка Unix           | 1724437800                    | Количество секунд, прошедших с 00:00:00 1 января 1970 года по времени UTC |

## Сопоставление источников данных

Таблица сопоставления NocoBase, MySQL и PostgreSQL:

| **Тип поля**                  | **NocoBase**                  | **MySQL**          | **PostgreSQL**                |
|------------------------------|-------------------------------|--------------------|-------------------------------|
| Дата и время (с часовым поясом) | Datetime with timezone       | TIMESTAMP<br/> DATETIME | TIMESTAMP WITH TIME ZONE      |
| Дата и время (без часового пояса) | Datetime without timezone | DATETIME           | TIMESTAMP WITHOUT TIME ZONE   |
| Дата (без времени)            | Date                          | DATE               | DATE                          |
| Время                        | Time                          | TIME               | TIME WITHOUT TIME ZONE        |
| Временная метка Unix           | Unix timestamp                | INTEGER<br/>BIGINT | INTEGER<br/>BIGINT             |
| Время (с часовым поясом)       | -                             | -                  | TIME WITH TIME ZONE           |

Примечание:
- Диапазон данных MySQL TIMESTAMP находится между временем UTC `1970-01-01 00:00:01 ~ 2038-01-19 03:14:07`, при выходе за пределы этого диапазона рекомендуется использовать DATETIME или BIGINT для хранения временной метки Unix.

## UTC

UTC (всемирное координированное время, Coordinated Universal Time) — мировой стандарт времени, используемый для координации и унификации времени в разных частях мира. Это высокоточный стандарт времени, основанный на атомных часах и синхронизированный с вращением Земли.

Между временем UTC и местным временем существует разница в часовых поясах, поэтому прямое отображение исходного значения UTC может ввести пользователя в заблуждение, например:

| **Часовой пояс**       | **Дата и время**                 |
|------------------------|----------------------------------|
| UTC                    | 2024-08-24T07:30:00.000Z         |
| Восточная зона (UTC+8) | 2024-08-24 15:30:00             |
| Восточная зона (UTC+5) | 2024-08-24 12:30:00             |
| Западная зона (UTC-5) | 2024-08-24 02:30:00              |
| Британское время (UTC+0) | 2024-08-24 07:30:00           |
| Центральное время (UTC-6) | 2024-08-23 01:30:00           |

Все приведённые значения обозначают один и тот же момент времени и отличаются только часовыми поясами.
