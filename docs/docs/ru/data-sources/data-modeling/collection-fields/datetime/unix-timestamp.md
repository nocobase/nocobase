# Unix timestamp

## Введение

В NocoBase **Unix timestamp (Unix timestamp)** используется для хранения временных меток Unix.

Unix timestamp часто используется при интеграции с внешними системами, работе с данными журналов или миграции исторических данных. В поле хранится числовое значение, но его бизнес-смысл — время.

Если внешняя система не требует использования временных меток, для понимания и обслуживания лучше использовать [дату и время](./datetime.md).

## Сценарии использования

Unix timestamp подходит для следующих бизнес-сценариев:

- Синхронизация временных меток с внешними системами
- Время возникновения записи в журнале
- Unix timestamp, возвращаемый API
- Поле времени при миграции исторических данных

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Unix timestamp», чтобы создать поле Unix timestamp.

![20240512180432](https://static-docs.nocobase.com/20240512180432.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для Unix timestamp соответствует `unixTimestamp` и определяет способ ввода и отображения значения на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Временная метка синхронизации», «Время записи журнала» или «Внешнее время обновления». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для хранения Unix timestamp обычно используется целое число или большое целое число. |
| Default value | Значение по умолчанию. Если при создании записи пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Правила валидации. Можно настроить обязательность и допустимый диапазон числовых значений. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поля Unix timestamp по умолчанию ведут себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `unixTimestamp`. |
| Тип Field type по умолчанию | `bigInt`. |
| Доступные типы Field type | `integer`、`bigInt`. |
| Компонент страницы | В режиме редактирования обрабатывается компонентом поля временной метки. |
| Фильтрация | Поддерживается фильтрация по числовому значению временной метки или по диапазону времени после преобразования. |
| Сортировка | Поддерживается сортировка. |
| Валидация | Поддерживается проверка обязательности и допустимого диапазона числовых значений. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля Unix timestamp. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени, описания, значения по умолчанию, правил валидации или настроек, относящихся только к этому полю.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | В зависимости от условий | Для полей основной базы данных или синхронизированных полей интерфейс можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и валидации на страницах. |
| Field type | В зависимости от условий | Для полей основной базы данных или синхронизированных полей тип можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила валидации поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила валидации, условия фильтрации и способ использования переменной в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле Unix timestamp. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля Unix timestamp обычно одновременно удаляются физический столбец в базе данных и уже хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
