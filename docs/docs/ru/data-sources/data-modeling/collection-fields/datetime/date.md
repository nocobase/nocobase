# Дата

## Введение

В NocoBase **дата (Date)** используется для хранения дат без указания конкретного времени.

Поле даты подходит для хранения дней рождения, дат подписания договоров, дат истечения срока действия, плановых дат и других данных, где важны только год, месяц и день.

Если необходимо сохранять конкретные часы, минуты и секунды, выберите [дату и время](./datetime.md). Если нужно сохранять только время в течение дня, выберите [время](./time.md).

## Сценарии использования

Дата подходит для следующих бизнес-сценариев:

- дни рождения клиентов, даты приёма сотрудников на работу
- даты подписания договоров, даты истечения срока действия
- плановые даты, даты поставки
- бизнес-даты, для которых не требуется конкретное время

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Дата», чтобы создать поле даты.

![20260709232951](https://static-docs.nocobase.com/20260709232951.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для даты используется `date`, который определяет способ ввода и отображения значения на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Дата подписания», «Дата истечения срока действия» или «День рождения». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для API, полей связей, прав доступа, рабочих процессов и других внутренних ссылок. Обычно после создания не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля даты по умолчанию используется `dateonly`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Можно настроить обязательность заполнения, диапазон дат и другие ограничения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Характеристики поля

Поле даты по умолчанию работает следующим образом:

| Характеристика | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `date`. |
| Тип Field type по умолчанию | `dateonly`. |
| Доступный тип Field type | `dateonly`. |
| Компонент страницы | В режиме редактирования используется средство выбора даты. |
| Фильтрация | Поддерживается фильтрация по дате, диапазону, пустым и непустым значениям. |
| Сортировка | Поддерживается сортировка по дате. |
| Проверка | Поддерживается проверка обязательности и диапазона дат. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля даты. Редактирование поля используется главным образом для настройки способа отображения и использования поля в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет название поля, отображаемое в интерфейсе, но не меняет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. После изменения меняются способы ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле даты. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля даты, созданного в основной базе данных, обычно одновременно удаляются реальный столбец в базе данных и все данные, которые в нём хранились. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
