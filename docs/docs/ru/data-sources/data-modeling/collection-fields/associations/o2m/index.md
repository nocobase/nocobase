# Один ко многим

Связь между классом и учениками: в одном классе может быть несколько учеников, но каждый ученик может принадлежать только одному классу. В этом случае между классом и учениками существует связь «один ко многим».

## Описание параметров

### Исходная коллекция

Исходная таблица, то есть таблица, в которой находится текущее поле.

### Целевая коллекция

Целевая таблица, с которой устанавливается связь.

### Ключ исходной таблицы

Поле, на которое ссылается ограничение внешнего ключа; оно должно быть уникальным.

### Внешний ключ

Поле целевой таблицы, используемое для установления связи между двумя таблицами.

### Ключ целевой таблицы

Поле целевой таблицы, используемое для просмотра каждой записи в блоке связей; обычно это уникальное поле.

### ON DELETE

ON DELETE — это правило обработки ссылок внешнего ключа в дочерней таблице при удалении записи из родительской таблицы. Это один из параметров, используемых при определении ограничения внешнего ключа. К распространённым параметрам ON DELETE относятся:

- CASCADE: при удалении записи из родительской таблицы автоматически удаляются все связанные с ней записи из дочерней таблицы.
- SET NULL: при удалении записи из родительской таблицы значение внешнего ключа в связанных записях дочерней таблицы устанавливается в NULL.
- RESTRICT: параметр по умолчанию; если при попытке удалить запись из родительской таблицы существуют связанные записи в дочерней таблице, удаление записи из родительской таблицы отклоняется.
- NO ACTION: аналогично RESTRICT: если существуют связанные записи в дочерней таблице, удаление записи из родительской таблицы отклоняется.
