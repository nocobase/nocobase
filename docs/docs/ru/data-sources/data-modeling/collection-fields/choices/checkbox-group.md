# Группа флажков

## Введение

В NocoBase **группа флажков (Checkbox group)** используется для выбора нескольких значений из набора вариантов с их непосредственным отображением в форме.

Группа флажков подходит для сценариев с небольшим количеством вариантов, когда требуется множественный выбор. От выпадающего списка с множественным выбором она отличается главным образом способом взаимодействия с интерфейсом.

Если вариантов много, выберите [выпадающий список с множественным выбором](./multiple-select.md), чтобы сэкономить место. Если можно выбрать только один вариант, выберите [группу переключателей](./radio-group.md).

## Подходящие сценарии

Группа флажков подходит для следующих бизнес-сценариев:

- сфера обслуживания, доступные каналы
- пункты выбора функциональных разрешений
- теги потребностей клиентов
- небольшое количество фиксированных вариантов для множественного выбора

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Группа флажков», чтобы создать поле группы флажков.
![20260709231244](https://static-docs.nocobase.com/20260709231244.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для группы флажков соответствует `checkboxGroup` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Сфера обслуживания», «Доступные каналы», «Теги потребностей». Рекомендуется использовать понятное бизнес-пользователям имя. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Группа флажков обычно сохраняется как массив или JSON; конкретный вариант зависит от настройки поля и возможностей источника данных. |
| Default value | Значение по умолчанию. При добавлении записи оно может подставляться автоматически, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно настраиваются обязательность заполнения и допустимый набор вариантов. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

По умолчанию поле группы флажков ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `checkboxGroup`. |
| Тип Field type по умолчанию | `array`. |
| Доступные типы Field type | `array`、`json`, в соответствии с фактическим сопоставлением поля. |
| Компонент страницы | В режиме редактирования используется группа флажков. |
| Фильтрация | Поддерживается фильтрация по наличию определённого варианта. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются обязательность заполнения и ограничения набора вариантов. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля группы флажков. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из уже синхронизированной таблицы основной базы данных, редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не меняя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей интерфейс можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей тип можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле группы флажков. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля группы флажков обычно одновременно удаляются соответствующий физический столбец в базе данных и уже сохранённые в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного из внешнего источника данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::

## Использование в настройках страницы

Группа флажков подходит для отображения небольшого количества вариантов множественного выбора непосредственно в форме.
![20260709230421](https://static-docs.nocobase.com/20260709230421.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Непосредственно отображает все варианты и позволяет выбрать несколько. |
| Блок подробностей | Отображает несколько выбранных вариантов в виде тегов или текста. |
| Блок фильтрации | Фильтрует записи по наличию определённых вариантов. |
| Рабочие процессы и права доступа | Участвует в проверке условий, таких как «содержит» и «не содержит». |
