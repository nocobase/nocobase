# Множественный выбор из выпадающего списка

## Введение

В NocoBase **множественный выбор из выпадающего списка (Multiple select)** используется для выбора нескольких значений из набора вариантов.

Множественный выбор из выпадающего списка подходит для таких полей, как теги, навыки, предпочтения и область применения. В нем сохраняются значения нескольких вариантов, которые на странице обычно отображаются в виде тегов.

Если можно выбрать только одно значение, выберите [одиночный выбор из выпадающего списка](./select.md) или [группу переключателей](./radio-group.md).

## Сценарии использования

Множественный выбор из выпадающего списка подходит для следующих бизнес-сценариев:

- Теги клиентов, предпочтения пользователей
- Область применения продукта, возможности оборудования
- Риски проекта, категории проблем
- Поля, в которых можно выбрать несколько фиксированных значений

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Множественный выбор из выпадающего списка», чтобы создать соответствующее поле.

![20240512180236](https://static-docs.nocobase.com/20240512180236.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для множественного выбора из выпадающего списка используется `multipleSelect`, который определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Теги клиентов», «Сценарии использования» или «Категория проблем». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для API, полей связей, разрешений, рабочих процессов и других внутренних ссылок. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчеркивания, при этом название должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Множественный выбор из выпадающего списка обычно сохраняется как массив или JSON; конкретный вариант зависит от настройки поля и возможностей источника данных. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно настраиваются обязательность заполнения и допустимый набор вариантов. |
| Description | Описание поля. Здесь можно указать его назначение, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, разрешениями, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле множественного выбора из выпадающего списка по умолчанию ведет себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `multipleSelect`. |
| Тип Field type по умолчанию | `array`. |
| Доступные типы Field type | `array`, `json` — в зависимости от фактического соответствия поля. |
| Компонент страницы | В режиме редактирования используется селектор множественного выбора из выпадающего списка. |
| Фильтрация | Поддерживается фильтрация по наличию определенного варианта. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются обязательность заполнения и ограничение набора вариантов. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля множественного выбора из выпадающего списка. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например названия, описания, значения по умолчанию, правил проверки или настроек, специфичных для поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления поля — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе названия поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определенных условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определенных условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно будет использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объеме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле множественного выбора из выпадающего списка. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля множественного выбора из выпадающего списка, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и хранящиеся в нем данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, разрешения, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
