# Флажок

## Введение

В NocoBase **флажок (Checkbox)** используется для сохранения двоичного булевого значения.

Поле флажка подходит для простых проверок: включен ли статус, является ли значение заданным по умолчанию, завершена ли операция, требуется ли согласование и т. п. Его смысл понятнее, чем при сохранении «да/нет» в виде текста.

Если у состояния больше двух значений, например «черновик», «в процессе», «завершено», лучше выбрать [выпадающий список с одиночным выбором](./select.md).

## Подходящие сценарии

Флажок подходит для следующих бизнес-сценариев:

- Включено ли значение, используется ли по умолчанию
- Завершено ли, прочитано ли
- Требуется ли согласование, выставлен ли счет
- Является ли общедоступным, архивировано ли

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Флажок», чтобы создать поле флажка.

![20240512180122](https://static-docs.nocobase.com/20240512180122.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для флажка соответствует `checkbox` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Включено», «Завершено», «Счет выставлен». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, разрешениях, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчеркивания, причем имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля флажка по умолчанию используется `boolean`. |
| Default value | Значение по умолчанию. При добавлении записи, если пользователь не указал значение, оно подставляется автоматически. |
| Validation rules | Правила проверки. Обычно настраиваются обязательность заполнения или значение по умолчанию. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за сопровождение. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, разрешениями, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поле флажка по умолчанию обладает следующими свойствами:

| Свойство | Описание |
| --- | --- |
| Field interface по умолчанию | `checkbox`. |
| Field type по умолчанию | `boolean`. |
| Доступный Field type | `boolean`. |
| Компонент страницы | В режиме редактирования используется флажок. |
| Фильтрация | Поддерживается фильтрация по значениям «да», «нет» и по пустым значениям. |
| Сортировка | Поддерживается сортировка по булевым значениям. |
| Проверка | Поддерживаются базовые настройки, такие как обязательность заполнения и значение по умолчанию. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля флажка. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например названия, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле поступает из таблицы, уже синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе названия поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определенных условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на страницах. |
| Field type | При определенных условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о смысле поля, требованиях к заполнению, источнике данных или ответственном за сопровождение. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого названия. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных рабочих процессов. Если существующих данных много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле флажка. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля флажка обычно одновременно удаляются соответствующий реальный столбец базы данных и уже имеющиеся в нем данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, разрешения, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурации.

:::

## Использование в настройках страниц

Поле флажка подходит для использования в формах, таблицах, фильтрах и условиях рабочих процессов.
![20260709225738](https://static-docs.nocobase.com/20260709225738.png)

| Сценарий | Назначение |
| --- | --- |
| Блок формы | Ввод значения «да» или «нет». |
| Блок таблицы | Отображение состояния флажка и поддержка фильтрации. |
| Блок фильтрации | Фильтрация по условиям, например включено ли значение или завершена ли операция. |
| Рабочие процессы и разрешения | Использование в качестве булевого условия. |
