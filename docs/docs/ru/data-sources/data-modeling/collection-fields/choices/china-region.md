# Административно-территориальное деление Китая（устарело）

## Введение

:::warning Внимание

Поле административно-территориального деления Китая устарело. Рекомендуется использовать поле связи для связывания с древовидной таблицей.

:::

В NocoBase **административно-территориальное деление Китая (China region)** используется для выбора провинций, городов, районов и уездов Китая.

Поле административно-территориального деления Китая подходит для сценариев, где требуется структурированный выбор региона: адреса клиентов, адреса магазинов, зоны обслуживания и другие подобные случаи. По сравнению с ручным вводом адреса оно упрощает фильтрацию и статистический анализ.

Если необходимо сохранять полный подробный адрес, можно использовать [однострочный текст](../basic/input.md) или [многострочный текст](../basic/textarea.md) для хранения улицы и номера дома.

## Сценарии использования

Поле административно-территориального деления Китая подходит для следующих бизнес-сценариев:

- Провинция, город и район проживания клиента
- Зона обслуживания магазина
- Регион реализации проекта
- Административно-территориальная единица в адресе доставки

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Административно-территориальное деление Китая», чтобы создать соответствующее поле.

![20240512180305](https://static-docs.nocobase.com/20240512180305.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для административно-территориального деления Китая используется `chinaRegion`, определяющий способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Регион проживания», «Зона обслуживания» или «Регион доставки». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связи, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Значение поля административно-территориального деления обычно хранится в структурированном виде; конкретный Field type зависит от настроек поля. |
| Default value | Значение по умолчанию. При создании записи оно подставляется автоматически, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Обычно настраиваются обязательность заполнения и уровни выбора. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле административно-территориального деления Китая по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Field interface по умолчанию | `chinaRegion`. |
| Field type по умолчанию | `json`. |
| Доступные Field type | `json` и `string`, в зависимости от фактической конфигурации поля. |
| Компонент страницы | В режиме редактирования используется компонент выбора административно-территориальных единиц. |
| Фильтрация | Поддерживается фильтрация по региональному значению; конкретные возможности зависят от конфигурации поля. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, например обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля административно-территориального деления Китая. Редактирование поля используется главным образом для настройки способа его отображения и использования в NocoBase: можно изменить отображаемое имя, описание, значение по умолчанию, правила проверки и специальные настройки поля.

Если поле относится к таблице, уже синхронизированной с основной базой данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не изменяя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | Поля основной базы данных или синхронизированные поля можно изменить при сопоставлении. Перед изменением необходимо проверить, можно ли использовать существующие данные с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет информацию о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. Если существующих данных много, сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле административно-территориального деления Китая. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля административно-территориального деления Китая, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и имеющиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
