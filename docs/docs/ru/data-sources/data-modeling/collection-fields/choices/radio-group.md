# Группа радиокнопок

## Введение

В NocoBase **группа радиокнопок (Radio group)** используется для выбора одного значения из набора вариантов с их непосредственным отображением в форме.

Группа радиокнопок подходит для случаев, когда вариантов немного и пользователю нужно сразу видеть их все. От одиночного выбора в раскрывающемся списке она отличается главным образом способом взаимодействия на странице.

Если вариантов много, выберите [одиночный выбор в раскрывающемся списке](./select.md), чтобы сэкономить место. Если нужно выбрать несколько вариантов, используйте [группу флажков](./checkbox-group.md).

## Сценарии использования

Группа радиокнопок подходит для следующих бизнес-сценариев:

- Приоритет: низкий, средний, высокий
- Пол, тип и другие варианты категорий «да/нет»
- Результат согласования: одобрено, отклонено
- Быстрый выбор из небольшого числа фиксированных вариантов

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Группа радиокнопок», чтобы создать поле группы радиокнопок.
![20260709231205](https://static-docs.nocobase.com/20260709231205.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для группы радиокнопок соответствует `radioGroup` и определяет способ ввода и отображения на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Приоритет», «Результат согласования» или «Тип». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. По умолчанию для группы радиокнопок используется `string`, в котором сохраняется выбранное значение. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь не ввёл значение. |
| Validation rules | Правила проверки. Обычно здесь задают обязательность заполнения и поддерживаемые значения. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Заранее проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

По умолчанию поле группы радиокнопок работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `radioGroup`. |
| Тип Field type по умолчанию | `string`. |
| Доступный Field type | `string`. |
| Компонент страницы | В режиме редактирования используется группа радиокнопок. |
| Фильтрация | Поддерживается фильтрация по вариантам. |
| Сортировка | Поддерживается сортировка по значениям вариантов. |
| Проверка | Поддерживаются обязательное заполнение и ограничение набора вариантов. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля группы радиокнопок. Редактирование поля в основном используется для изменения способа его отображения и использования в NocoBase, например имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, его редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на странице. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения поля, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. Если существующих данных много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле группы радиокнопок. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля группы радиокнопок обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::

| Блок подробной информации | Отображение выбранного варианта. |
| Блок фильтрации | Фильтрация записей по вариантам. |
| Рабочие процессы и права доступа | Использование в качестве условия для проверки статуса, типа и других значений. |
