# Одиночный выбор из выпадающего списка

## Введение

В NocoBase **одиночный выбор из выпадающего списка (Select)** используется для выбора одного значения из набора вариантов.

Одиночный выбор из выпадающего списка подходит для бизнес-полей с фиксированным набором значений, таких как статус, уровень, тип, источник и т. д. Для вариантов можно настроить отображаемое имя, значение и цвет.

Если необходимо выбрать несколько значений, выберите [множественный выбор из выпадающего списка](./multiple-select.md). Если доступны только варианты «да» или «нет», выберите [флажок](./checkbox.md).

## Сценарии использования

Одиночный выбор из выпадающего списка подходит для следующих бизнес-сценариев:

- Статус заказа, статус задачи, статус согласования
- Уровень клиента, источник лида, приоритет
- Тип проекта, категория актива, тип договора
- Поля, для которых в фиксированном наборе можно выбрать только одно значение

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Одиночный выбор из выпадающего списка», чтобы создать поле одиночного выбора из выпадающего списка.

![20240512180203](https://static-docs.nocobase.com/20240512180203.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейс поля. Для одиночного выбора из выпадающего списка соответствует `select` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Статус заказа», «Уровень клиента», «Приоритет». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. Обычно после создания не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для одиночного выбора из выпадающего списка по умолчанию используется `string`, в котором сохраняется значение выбранного варианта. |
| Default value | Значение по умолчанию. При создании записи оно автоматически подставляется, если пользователь не указал значение. |
| Validation rules | Правила проверки. Обычно используются для настройки обязательности заполнения и управления допустимыми значениями. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственное лицо. |

:::warning Внимание

После создания имя поля используется блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле одиночного выбора из выпадающего списка по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `select`. |
| Тип Field type по умолчанию | `string`. |
| Доступный тип Field type | `string`. |
| Компонент страницы | В режиме редактирования используется селектор выпадающего списка. |
| Фильтрация | Поддерживается фильтрация по вариантам. |
| Сортировка | Поддерживается сортировка по значениям вариантов. |
| Проверка | Поддерживаются обязательное заполнение и ограничение допустимых вариантов. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля одиночного выбора из выпадающего списка. Редактирование поля используется главным образом для настройки способа его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специфических параметров поля.

Если поле поступает из таблицы основной базы данных, которая уже синхронизирована, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение отображаемого в интерфейсе имени поля без изменения его идентификатора. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | При сопоставлении полей основной базы данных или синхронизированных полей интерфейс можно изменить. Это повлияет на способ ввода, отображения и проверки данных на странице. |
| Field type | При определённых условиях | При сопоставлении полей основной базы данных или синхронизированных полей тип можно изменить. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение информации о назначении поля, требованиях к заполнению, источнике данных или ответственном лице. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле одиночного выбора из выпадающего списка. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля одиночного выбора из выпадающего списка обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
