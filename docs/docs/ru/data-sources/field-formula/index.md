# Расчетная формула

## Введение

В NocoBase **формула (Formula)** используется для вычисления значения поля на основе выражения.

Поле формулы подходит для вычисления сумм, оценок, объединения текста, условных вычислений и других сценариев. Его значение обычно генерируется выражением, поэтому оно не предназначено для непосредственного ввода вручную.

Если результат должен вводиться вручную, выберите соответствующее базовое поле. Если логика вычислений слишком сложная, рассмотрите возможность использовать рабочий процесс или представление базы данных.

## Сценарии использования

Формула подходит для следующих бизнес-сценариев:

- Промежуточная сумма заказа, сумма с налогом
- Оценка, весовой балл, балл эффективности
- Отображаемое имя после объединения текста
- Бизнес-результат, вычисляемый на основе условий

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Формула», чтобы создать поле формулы.

![20240512173541](https://static-docs.nocobase.com/20240512173541.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для формулы соответствует `formula` и определяет способ ввода и отображения значения на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Промежуточная сумма заказа», «Итоговая оценка», «Отображаемое имя». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для поля формулы используется `formula`, а тип результата зависит от настроек формулы. |
| Default value | Значение по умолчанию. При создании записи, если пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Правила проверки. Важно убедиться, что выражение формулы указано полностью, а используемые поля существуют. |
| Description | Описание поля. Здесь удобно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте имя, чтобы избежать затрат на последующую корректировку конфигурации.

:::

## Свойства поля

Поведение поля формулы по умолчанию:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `formula`. |
| Тип Field type по умолчанию | `formula`. |
| Доступные типы Field type | `formula`. |
| Компонент страницы | В режиме редактирования обычно настраивается выражение формулы, а в режиме просмотра отображается вычисленный результат. |
| Фильтрация | Возможность фильтрации зависит от результата формулы и способа её выполнения. |
| Сортировка | Возможность сортировки зависит от результата формулы и способа её выполнения. |
| Проверка | Зависит от выражения формулы и типа результата. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля формулы. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специфичных настроек поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, при редактировании обычно выполняется сопоставление полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменение имени поля, отображаемого в интерфейсе, без изменения идентификатора поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс основного поля базы данных или синхронизированного поля можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Тип основного поля базы данных или синхронизированного поля можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменение значения по умолчанию для новых записей. |
| Validation rules | Да | Изменение правил проверки поля. |
| Description | Да | Дополнение сведений о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных рабочих процессов. Если данных уже много, сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле формулы. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении поля формулы, созданного в основной базе данных, обычно одновременно удаляются реальный столбец в базе данных и все данные, которые в нём хранились. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурациях.

:::
