# Коллекция комментариев

## Введение

Коллекция комментариев — это специализированный шаблон таблицы данных для хранения пользовательских комментариев и обратной связи. С помощью функции комментариев можно добавить возможность комментирования к любой таблице данных, чтобы пользователи могли обсуждать записи, оставлять обратную связь или добавлять заметки к конкретным данным. Коллекция комментариев поддерживает редактирование форматированного текста и дает гибкие возможности создания контента.

![comment-collection-2025-11-01-00-39-01](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-39-01.png)

## Возможности

- **Редактирование форматированного текста**: по умолчанию включает редактор Markdown (vditor), поддерживает создание форматированного контента.
- **Связь с любой таблицей данных**: позволяет связывать комментарии с записями любой таблицы данных через поля связей.
- **Многоуровневые комментарии**: поддерживает ответы на комментарии и построение древовидной структуры обсуждения.
- **Отслеживание пользователя**: автоматически фиксирует автора комментария и время создания.

## Руководство пользователя

### Создание коллекции комментариев

![comment-collection-2025-11-01-00-37-10](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-37-10.png)

1. Перейдите на страницу управления таблицами данных.
2. Нажмите кнопку "Создать коллекцию".
3. Выберите шаблон "Коллекция комментариев".
4. Введите имя таблицы (например, "Комментарии к задачам", "Комментарии к статьям" и т.д.).
5. Система автоматически создаст таблицу комментариев со следующими полями по умолчанию:
   - Содержимое комментария (тип Markdown vditor)
   - Создано пользователем (Создано пользователем, связь с таблицей пользователей)
   - Создано в (Время создания, формат даты и времени)

### Настройка связей

Чтобы связать комментарии с целевой таблицей данных, нужно настроить поля связей:

![](https://static-docs.nocobase.com/Solution/demoE3v1-19N.gif)

1. Добавьте поле связи "Многие-к-одному" в таблицу комментариев.
2. Выберите целевую таблицу данных для связи (например, таблицу задач, таблицу статей и т.д.).
3. Задайте имя поля (например, "Относится к задаче", "Относится к статье" и т.д.).

### Использование блока комментариев на страницах

![Включение коллекции комментариев](https://static-docs.nocobase.com/Solution/demoE3v1-20.gif)

1. Перейдите на страницу, где нужно добавить функциональность комментариев.
2. Добавьте блок в деталях записи или во всплывающем окне целевой записи.
3. Выберите тип блока "Комментарии".
4. Выберите созданную ранее коллекцию комментариев.

### Типовые сценарии использования

- **Системы управления задачами**: участники команды обсуждают задачи и оставляют обратную связь.
- **Системы управления контентом**: читатели комментируют статьи и взаимодействуют с контентом.
- **Рабочие процессы согласования**: согласующие добавляют пометки и комментарии к формам заявок.
- **Обратная связь от клиентов**: сбор отзывов клиентов о продуктах или услугах.

## Примечания

- Коллекция комментариев — это функция коммерческого плагина и требует включенного плагина Комментарии.
- Рекомендуется настроить корректные права доступа к таблице комментариев, чтобы контролировать просмотр, создание и удаление комментариев.
- Для сценариев с большим количеством комментариев рекомендуется включить пагинацию для лучшей производительности.
