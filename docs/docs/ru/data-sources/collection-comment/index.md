# Таблица комментариев

## Введение

Таблица комментариев предназначена для хранения обсуждений, отзывов и примечаний, связанных с бизнес-записями. Например, с её помощью можно хранить комментарии к задачам, мнения по заявкам на согласование, комментарии к статьям и отзывы клиентов.

Таблица комментариев обычно не используется отдельно в качестве основной бизнес-таблицы. Чаще всего сначала создают таблицу комментариев, затем настраивают поля связей в бизнес-таблице и, наконец, добавляют блок комментариев на страницу сведений или во всплывающее окно бизнес-записи.

![таблица комментариев 2025-11-01-00-39-01](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-39-01.png)

## Сценарии использования

Таблица комментариев подходит для следующих бизнес-сценариев:

- Совместное обсуждение задач, требований и дефектов
- Мнения по заявкам на согласование, обращениям и договорам
- Комментарии к статьям, базе знаний и объявлениям
- Отзывы клиентов, сопровождение после продажи и внутренние примечания

## Процесс использования

Таблица комментариев обычно используется вместе с бизнес-таблицей и блоком комментариев:

1. Создайте таблицу комментариев для хранения содержимого комментариев, связей ответов, создателей, времени создания и другой информации.
2. Создайте в бизнес-таблице поле связи с таблицей комментариев. Например, свяжите таблицу «Задачи» с таблицей «Комментарии к задачам».
3. Добавьте блок комментариев на страницу сведений или во всплывающее окно бизнес-таблицы.
4. Пользователи оставляют комментарии или отвечают на них в блоке комментариев. Данные комментариев записываются в таблицу комментариев и связываются с текущей бизнес-записью.
5. Настройте права доступа к таблице комментариев в соответствии с требованиями бизнеса, чтобы контролировать, кто может просматривать, создавать или удалять комментарии.

## Создание и настройка

В основной базе данных нажмите «Create collection» и выберите «Comment collection», чтобы создать таблицу комментариев.

![таблица комментариев 2025-11-01-00-37-10](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-37-10.png)

| Настройка | Описание |
| --- | --- |
| Collection display name | Имя таблицы, отображаемое в интерфейсе, например «Комментарии к задачам», «Мнения по согласованию» или «Комментарии к статьям». |
| Collection name | Идентификационное имя таблицы, используемое для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. |
| Inherits | Выбор родительской таблицы, от которой выполняется наследование. Отображается только при использовании PostgreSQL в качестве основной базы данных. |
| Categories | Категории таблицы. Категории влияют только на организацию интерфейса управления таблицами и не изменяют структуру таблицы. |
| Description | Описание таблицы. Здесь можно указать, для какого бизнес-объекта предназначена таблица комментариев, кто отвечает за её обслуживание и как спроектированы права доступа к комментариям. |
| Preset fields | Предустановленные поля. При создании таблицы комментариев рекомендуется сохранить системные поля и встроенные поля таблицы комментариев. |

### Встроенные поля

После создания таблица комментариев обычно содержит следующие встроенные поля. Блок комментариев в основном использует `content`, `createdBy` и `createdAt` для отображения содержимого комментария, автора и времени комментария.

| Поле | Имя поля | Описание |
| --- | --- | --- |
| ID | `id` | Поле первичного ключа по умолчанию, используемое для уникальной идентификации записи комментария. |
| Содержимое комментария | `content` | Хранит текст комментария, введённый пользователем; по умолчанию используется компонент Markdown Vditor. |
| Время создания | `createdAt` | Автоматически записывает время создания комментария; блок комментариев использует его для отображения времени комментария. |
| Создатель | `createdBy` | Автоматически записывает пользователя, оставившего комментарий; блок комментариев использует его для отображения автора комментария. |
| Время обновления | `updatedAt` | Автоматически записывает время последнего обновления комментария. |
| Обновил | `updatedBy` | Автоматически записывает пользователя, последним обновившего комментарий. |
| Пространство | `space` | Доступно после включения [плагина многопространственности](../../multi-app/multi-space/index.md) и используется для изоляции данных по пространствам. При отключённой многопространственности это поле не отображается. |

:::warning Внимание

Встроенные поля таблицы комментариев обычно поддерживаются блоком комментариев. Не рекомендуется без необходимости удалять их или изменять их бизнес-смысл. Если нужно хранить категорию комментария, статус обработки и другую информацию, можно добавить соответствующие бизнес-поля.

:::

### Поле первичного ключа

Как и обычная таблица, таблица комментариев должна иметь поле первичного ключа. Блок комментариев использует первичный ключ для поиска записей комментариев и связей ответов.

Если в таблице комментариев отсутствует первичный ключ, при редактировании таблицы данных необходимо указать «Record unique key». В противном случае блок комментариев может неправильно отображать, отвечать на сообщения или удалять комментарии.

| Блок комментариев | Отображение списка комментариев, добавление комментариев и ответов на них. |

## Редактирование настроек

В списке таблиц данных нажмите «Edit» справа от таблицы комментариев, чтобы изменить отображаемое имя таблицы, категории, описание, простой режим разбиения на страницы, «Record unique key» и другие настройки.

После ввода таблицы комментариев в эксплуатацию не рекомендуется без необходимости изменять поле содержимого комментария и поле связи ответов. Эти поля могут использоваться блоком комментариев, правами доступа, рабочими процессами и API.

## Удаление таблицы данных

В списке таблиц данных нажмите «Delete» справа от таблицы комментариев, чтобы удалить таблицу комментариев.

Удаление таблицы комментариев приведёт к удалению записей комментариев, связей ответов и связанных метаданных Collection. Перед удалением убедитесь, что поля связей в бизнес-таблицах, блоки комментариев, права доступа, рабочие процессы и API больше не зависят от неё.

:::danger Предупреждение

Удаление таблицы комментариев приведёт к потере данных комментариев для существующих бизнес-записей. Комментарии часто содержат сведения о совместной работе и мнения по обработке. Перед выполнением операции убедитесь, что резервное копирование или архивирование не требуется.

:::
