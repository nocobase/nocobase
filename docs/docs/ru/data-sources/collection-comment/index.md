# Коллекция комментариев

## Введение

Коллекция комментариев — это специализированный шаблон таблицы данных, предназначенный для хранения пользовательских комментариев и отзывов. С помощью функции комментариев вы можете добавить возможность комментирования к любой таблице данных, позволяя пользователям обсуждать, оставлять отзывы или помечать определенные записи. Коллекция комментариев поддерживает редактирование форматированного текста, предоставляя гибкие возможности для создания контента.

![comment-collection-2025-11-01-00-39-01](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-39-01.png)

## Возможности

- **Редактирование форматированного текста**: По умолчанию включает редактор Markdown (vditor), поддерживающий создание контента с форматированием.
- **Связывание с любой таблицей данных**: Вы можете связать комментарии с записями в любой таблице данных с помощью полей отношений.
- **Многоуровневые комментарии**: Поддерживается возможность отвечать на комментарии, создавая древовидную структуру обсуждений.
- **Отслеживание пользователей**: Автоматически записывает автора комментария и время его создания.

## Руководство пользователя

### Создание коллекции комментариев

![comment-collection-2025-11-01-00-37-10](https://static-docs.nocobase.com/comment-collection-2025-11-01-00-37-10.png)

1. Перейдите на страницу управления таблицами данных.
2. Нажмите кнопку «Создать коллекцию».
3. Выберите шаблон «Коллекция комментариев».
4. Введите название таблицы (например, «Комментарии к задачам», «Комментарии к статьям» и т.д.).
5. Система автоматически создаст коллекцию комментариев со следующими полями по умолчанию:
   - Содержимое комментария (тип Markdown vditor)
   - Автор (связано с таблицей пользователей)
   - Время создания (тип «Дата и время»)

### Настройка связей

Чтобы комментарии могли быть связаны с целевой таблицей данных, вам необходимо настроить поля отношений:

![](https://static-docs.nocobase.com/Solution/demoE3v1-19N.gif)

1. Добавьте поле отношения «Многие-к-одному» в коллекцию комментариев.
2. Выберите целевую таблицу данных для связывания (например, таблицу задач, таблицу статей и т.д.).
3. Установите имя поля (например, «Принадлежит задаче», «Принадлежит статье» и т.д.).

### Использование блока комментариев на страницах

![Enable Comments Collection](https://static-docs.nocobase.com/Solution/demoE3v1-20.gif)

1. Перейдите на страницу, куда вы хотите добавить функцию комментирования.
2. Добавьте блок в детали или всплывающее окно целевой записи.
3. Выберите тип блока «Комментарии».
4. Выберите коллекцию комментариев, которую вы только что создали.

### Типичные сценарии использования

- **Системы управления задачами**: Члены команды обсуждают задачи и оставляют по ним отзывы.
- **Системы управления контентом**: Читатели комментируют статьи и взаимодействуют с ними.
- **Рабочие процессы утверждения**: Утверждающие лица оставляют примечания и мнения по заявкам.
- **Обратная связь с клиентами**: Сбор отзывов клиентов о продуктах или услугах.

## Примечания

- Коллекция комментариев является функцией коммерческого плагина и требует включения плагина комментариев для использования.
- Рекомендуется установить соответствующие разрешения для коллекции комментариев, чтобы контролировать, кто может просматривать, создавать и удалять комментарии.
- В сценариях с большим количеством комментариев рекомендуется включить постраничную загрузку для повышения производительности.
