# Обзор

Моделирование данных — ключевой этап проектирования баз данных, включающий глубокий анализ и абстракцию различных типов данных и их взаимосвязей в реальном мире. В этом процессе мы стремимся выявить внутренние связи между данными и формализовать их в виде моделей данных, закладывая основу структуры базы данных информационной системы. NocoBase — платформа, управляемая моделями данных, со следующими возможностями:

## Поддержка доступа к данным из разных источников

NocoBase поддерживает источники данных различного происхождения, включая распространенные базы данных, платформы API/SDK и файлы.

![20240512085558](https://static-docs.nocobase.com/20240512085558.png)

NocoBase предоставляет [менеджер источников данных](/data-sources/data-source-manager) для управления различными источниками данных и их коллекциями. Плагин менеджера источников данных предоставляет только интерфейс управления всеми источниками данных и не дает возможности напрямую обращаться к источникам данных. Его нужно использовать совместно с различными плагинами источников данных. Сейчас поддерживаются следующие источники данных:

- [Основная база данных](/data-sources/data-source-main): основная база данных NocoBase, поддерживающая реляционные СУБД, такие как MySQL, PostgreSQL и MariaDB.
- [KingbaseES](/data-sources/data-source-kingbase): использование базы данных KingbaseES как источника данных; может использоваться как основная и как внешняя база данных.
- [Внешняя MySQL](/data-sources/data-source-external-mysql): использование внешней базы данных MySQL как источника данных.
- [Внешняя MariaDB](/data-sources/data-source-external-mariadb): использование внешней базы данных MariaDB как источника данных.
- [Внешняя PostgreSQL](/data-sources/data-source-external-postgres): использование внешней базы данных PostgreSQL как источника данных.
- [Внешняя MSSQL](/data-sources/data-source-external-mssql): использование внешней базы данных MSSQL (SQL Server) как источника данных.
- [Внешняя Oracle](/data-sources/data-source-external-oracle): использование внешней базы данных Oracle как источника данных.

![20240512083651](https://static-docs.nocobase.com/20240512083651.png)

## Разнообразные инструменты моделирования данных

**Простой интерфейс управления коллекциями**: используется для создания различных моделей (коллекций) или подключения к уже существующим.

![20240512090751](https://static-docs.nocobase.com/20240512090751.png)

**Визуальный интерфейс в виде ER-диаграмм**: используется для выделения сущностей и их связей из пользовательских и бизнес-требований. Он дает интуитивный и понятный способ описания моделей данных. С помощью ER-диаграмм можно яснее понять основные сущности данных системы и связи между ними.

![20240512091042](https://static-docs.nocobase.com/20240410075906.png)

## Поддерживает различные типы коллекций

| Коллекция | Описание |
| - | - |
| [Общая коллекция](/data-sources/data-source-main/general-collection) | встроенные общие системные поля |
| [Коллекция календаря](/data-sources/calendar/calendar-collection) | используется для создания коллекций событий, связанных с календарем |
| Коллекция комментариев | используется для хранения комментариев или обратной связи по данным |
| [Древовидная коллекция](/data-sources/collection-tree) | коллекция с древовидной структурой, сейчас поддерживается только модель Список смежности |
| [Файловая коллекция](/data-sources/file-manager/file-collection) | используется для управления хранением файлов |
| [SQL-коллекция](/data-sources/collection-sql) | не является физической коллекцией в базе данных, а визуализирует SQL-запросы в структурированном виде |
| [Подключение к представлению БД](/data-sources/collection-view) | подключение к существующим представлениям БД |
| Коллекция выражений | используется для сценариев с динамическими выражениями в рабочих процессах |
| [Подключение к внешним данным (Connect to foreign data)](/data-sources/collection-fdw) | позволяет системе БД напрямую обращаться и выполнять запросы к данным во внешних источниках на базе технологии FDW |
