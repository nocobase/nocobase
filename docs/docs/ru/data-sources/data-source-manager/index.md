# Менеджер источников данных

## Введение

NocoBase предоставляет плагин управления источниками данных для управления источниками данных и их коллекциями. Плагин управления источниками данных предоставляет только интерфейс управления всеми источниками данных и не дает возможности обращаться к источникам данных напрямую. Его необходимо использовать совместно с различными плагинами источников данных. Сейчас поддерживаются следующие источники данных:

- [Основная база данных](/data-sources/data-source-main): основная база данных NocoBase, поддерживающая реляционные СУБД, такие как MySQL, PostgreSQL и MariaDB.
- [Вннешняя MySQL](/data-sources/data-source-external-mysql): использование внешней базы MySQL как источника данных.
- [Вннешняя MariaDB](/data-sources/data-source-external-mariadb): использование внешней базы MariaDB как источника данных.
- [Вннешняя PostgreSQL](/data-sources/data-source-external-postgres): использование внешней базы PostgreSQL как источника данных.
- [Вннешняя MSSQL](/data-sources/data-source-external-mssql): использование внешней базы MSSQL (SQL Server) как источника данных.
- [Вннешняя Oracle](/data-sources/data-source-external-oracle): использование внешней базы Oracle как источника данных.
- [Внешний NocoBase](/data-sources/data-source-external-nocobase/): Использование другого приложения NocoBase как внешнего источника данных через удаленный API NocoBase.

Кроме того, через плагины можно расширить поддержку другими типами источников, включая распространенные типы БД и платформы, предоставляющие API (SDK).

## Установка

Встроенный плагин, отдельная установка не требуется.

## Инструкция по использованию

При инициализации и установке приложения по умолчанию создается источник данных для хранения данных NocoBase — основная база данных. Подробнее см. документацию [Основная база данных](/data-sources/data-source-main/).

### Внешние источники данных (External Data Sources)

Внешние базы данных поддерживаются как источники данных. Подробнее см. документацию [Внешняя база данных / Введение ](/data-sources/data-source-manager/external-database).
