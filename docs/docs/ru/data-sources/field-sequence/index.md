# Последовательность

## Введение

В NocoBase **последовательность (Sequence)** используется для генерации бизнес-номеров.

Поле последовательности подходит для номеров заказов, договоров, заявок на обслуживание, заявлений и других номеров с понятными правилами. В отличие от первичного ключа, оно больше ориентировано на отображение в бизнес-процессах и идентификацию пользователями.

Если требуется только внутренний уникальный идентификатор, используйте Snowflake ID, UUID или Nano ID.

## Сценарии использования

Последовательность подходит для следующих бизнес-сценариев:

- Номера заказов и договоров
- Номера заявок на обслуживание и заявлений
- Номера активов и оборудования
- Номера с префиксом, датой или правилом последовательного увеличения

## Создание конфигурации

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Последовательность», чтобы создать поле последовательности.

![20240512173752](https://static-docs.nocobase.com/20240512173752.png)

| Конфигурация | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для последовательности соответствует `sequence` и определяет способ ввода и отображения данных на странице. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Номер заказа», «Номер договора» или «Номер заявки». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, причём имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Тип хранения поля последовательности зависит от правила последовательности; обычно это `string`. |
| Default value | Значение по умолчанию. При добавлении записи, если пользователь не указал значение, оно может быть подставлено автоматически. |
| Validation rules | Обычно генерируются системой согласно правилам и не требуют ручной проверки. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Заранее проверьте название, чтобы избежать затрат на изменение конфигурации в дальнейшем.

:::

## Свойства поля

Поле последовательности по умолчанию ведёт себя следующим образом:

| Свойство | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `sequence`. |
| Тип Field type по умолчанию | `string`. |
| Доступные типы Field type | `string`, `integer` — в зависимости от фактической конфигурации последовательности. |
| Компонент страницы | Обычно создаётся автоматически и используется после настройки правила нумерации. |
| Фильтрация | Поддерживается точный поиск по номеру или фильтрация по тексту. |
| Сортировка | Пригодность для сортировки зависит от правила нумерации. |
| Проверка | Зависит от правила последовательности; обычно обеспечивается уникальность. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля последовательности. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или настроек, относящихся только к этому полю.

Если поле получено из таблицы основной базы данных, которая уже была синхронизирована, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Конфигурация | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки данных на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно будет использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных рабочих процессов. Если существующих данных много, сначала убедитесь, что их формат совместим.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле последовательности. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля последовательности, созданного в основной базе данных, обычно одновременно удаляются физический столбец в базе данных и все имеющиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область последствий зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурации.

:::
