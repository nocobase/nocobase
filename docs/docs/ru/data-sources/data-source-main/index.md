# Основная база данных

## Введение

База данных, настроенная при [развёртывании NocoBase](/ai/install-nocobase-app), используется для хранения данных системных таблиц NocoBase, а также поддерживает хранение бизнес-данных пользователей.

Поддерживаемые версии баз данных и коммерческие редакции основной базы данных:

| База данных | Поддерживаемая версия | Community | Standard | Professional | Enterprise |
| --- | --- | --- | --- | --- | --- |
| MySQL | >= 8.0.17 | ✅ | ✅ | ✅ | ✅ |
| PostgreSQL | >= 10 | ✅ | ✅ | ✅ | ✅ |
| MariaDB | >= 10.9 | ✅ | ✅ | ✅ | ✅ |
| KingbaseES | >=V9 | ❌ | ❌ | ✅ | ✅ |
| OceanBase | >=4.3 | ❌ | ❌ | ❌ | ✅ |

:::tip Примечание

KingbaseES поддерживает только режим совместимости с PostgreSQL, а OceanBase — только режим совместимости с MySQL.

:::

## Установка плагинов

| База данных | Соответствующий плагин | Способ установки |
| --- | --- | --- |
| MySQL | Нет | Встроенный плагин, отдельная установка не требуется. |
| PostgreSQL | Нет | Встроенный плагин, отдельная установка не требуется. |
| MariaDB | Нет | Встроенный плагин, отдельная установка не требуется. |
| KingbaseES | `@nocobase/plugin-data-source-kingbase` | Требуется коммерческая лицензия; после установки плагин включён по умолчанию. |
| OceanBase | `@nocobase/plugin-data-source-oceanbase` | Требуется коммерческая лицензия; после установки плагин включён по умолчанию. |

## Доступ к источнику основных данных

1. Нажмите пункт «Источники данных» в системных функциях, чтобы открыть главную страницу источников данных.
2. Выберите источник данных **Main** в списке источников данных и нажмите**Настроить**, чтобы открыть основную базу данных и управлять ею.

![configure_main_datasource](https://static-docs.nocobase.com/configure_main_datasource.png)

## Управление источником основных данных

Основная база данных предоставляет функции управления таблицами данных: поиск, создание, изменение и удаление таблиц данных, а также синхронизацию полей уже существующих в базе данных таблиц; кроме того, доступны создание, изменение и удаление полей таблиц данных.
- **Фильтр**: поиск таблиц данных, управляемых в основной базе данных NocoBase
- **Создать таблицу данных**: добавление новой таблицы бизнес-данных
- **Редактировать**: изменение таблицы бизнес-данных
- **Удалить**: удаление таблицы бизнес-данных
- **Синхронизировать из базы данных**: синхронизация структуры уже существующих в базе данных таблиц
- **Настроить поля**: создание, изменение и удаление полей таблицы данных
-  **+** : **+** на вкладке используется для управления категориями таблиц данных, включая создание, изменение и удаление категорий
![main_datasource_management](https://static-docs.nocobase.com/main_datasource_management.png)

### Синхронизация существующих таблиц из базы данных

![nocobase_doc-2025-10-29-19-46-34](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-46-34.png)

Важная особенность источника основных данных — возможность синхронизировать уже существующие в базе данных таблицы с NocoBase для последующего управления ими. Это означает:

- **Сохранение существующих инвестиций**: если в базе данных уже есть большое количество бизнес-таблиц, их не нужно создавать заново — их можно сразу синхронизировать и использовать
- **Гибкая интеграция**: таблицы, созданные с помощью других инструментов (например, SQL-скриптов или средств управления базами данных), можно передать под управление NocoBase
- **Постепенная миграция**: поддерживается постепенный перенос существующей системы в NocoBase вместо её полной перестройки за один раз

С помощью функции «Загрузить из базы данных» можно:
1. просмотреть все таблицы в базе данных
2. выбрать таблицы для синхронизации
3. автоматически определить структуру таблиц и типы полей
4. одним нажатием импортировать таблицы в NocoBase для управления

### Поддержка различных типов структур таблиц

![nocobase_doc-2025-10-29-19-47-14](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-47-14.png)

NocoBase поддерживает создание и управление таблицами данных различных типов:
- **Обычная таблица**: содержит распространённые системные поля;
- **Таблица наследования**: можно создать родительскую таблицу, а затем производные дочерние таблицы. Дочерние таблицы наследуют структуру родительской таблицы, при этом для них можно определить собственные столбцы.
- **Древовидная таблица**: таблица древовидной структуры; в настоящее время поддерживается только модель смежных списков;
- **Таблица календаря**: используется для создания таблиц событий, связанных с календарём;
- **Таблица файлов**: используется для управления хранением файлов;
- **SQL-таблица**: не является фактической таблицей базы данных, а служит для быстрого структурированного отображения результатов SQL-запросов;
- **Таблица представления**: подключение к существующему представлению базы данных;

#### Гибкое преобразование типов полей

NocoBase поддерживает гибкое преобразование типов полей в рамках одного и того же типа базы данных.

**Варианты преобразования поля типа String**

Если поле в базе данных имеет тип String, в NocoBase его можно преобразовать в любую из следующих форм:

- **Базовые типы**: однострочный текст, многострочный текст, номер телефона, адрес электронной почты, URL, пароль, цвет, значок
- **Типы выбора**: выпадающий список (одиночный выбор), переключатель
- **Мультимедийные типы**: Markdown, Markdown (Vditor), форматированный текст, вложение (URL)
- **Типы даты и времени**: дата и время (с часовым поясом), дата и время (без часового пояса)
- **Расширенные типы**: автоматическая нумерация, селектор таблицы данных, шифрование

Такой гибкий механизм преобразования означает:
- **Не требуется изменять структуру базы данных**: базовый тип хранения поля остаётся неизменным, изменяется только его представление в NocoBase
- **Адаптация к изменениям бизнеса**: по мере изменения бизнес-требований можно быстро корректировать способ отображения и взаимодействия с полем
- **Безопасность данных**: процесс преобразования не влияет на целостность существующих данных

### Гибкая синхронизация на уровне полей

NocoBase поддерживает не только синхронизацию таблиц целиком, но и детальное управление синхронизацией на уровне отдельных полей:

![nocobase_doc-2025-10-29-19-49-56](https://static-docs.nocobase.com/nocobase_doc-2025-10-29-19-49-56.png)

#### Особенности синхронизации полей:

1. **Синхронизация в реальном времени**: при изменении структуры таблицы базы данных новые поля можно синхронизировать в любое время
2. **Выборочная синхронизация**: можно синхронизировать только необходимые поля, а не все поля
3. **Автоматическое распознавание типов**: типы полей базы данных автоматически определяются и сопоставляются с типами полей NocoBase
4. **Сохранение целостности данных**: процесс синхронизации не влияет на существующие данные

#### Сценарии использования:

- **Развитие структуры базы данных**: если из-за изменения бизнес-требований в базе данных добавляются новые поля, их можно быстро синхронизировать с NocoBase
- **Командная работа**: если другие участники команды или DBA добавили поля в базе данных, их можно своевременно синхронизировать
- **Гибридная модель управления**: часть полей управляется через NocoBase, а часть — традиционными способами, что позволяет гибко комбинировать подходы

Такой гибкий механизм синхронизации позволяет NocoBase легко интегрироваться в существующую техническую архитектуру без изменения исходного способа управления базой данных, одновременно предоставляя преимущества low-code разработки NocoBase.

Подробнее см. в разделе «[Поля таблицы данных / Обзор](../data-modeling/collection-fields/index.md)».
