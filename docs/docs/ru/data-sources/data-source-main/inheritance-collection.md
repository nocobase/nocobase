# Наследуемые коллекции

## Введение

:::warning
Эта функция поддерживается только в том случае, если основная база данных — PostgreSQL.
:::

Вы можете создать родительскую коллекцию, а затем создать дочерние коллекции на её основе. Дочерняя коллекция наследует структуру родительской и может определять собственные столбцы. Этот шаблон проектирования помогает организовать данные со схожей структурой, но возможными различиями, и управлять ими.

Ниже приведены некоторые характерные особенности наследования коллекций:

- **Родительская коллекция:** Содержит общие столбцы и данные, определяя базовую структуру всей иерархии наследования.
- **Дочерняя коллекция:** Наследует структуру родительской коллекции, но может также определять собственные столбцы. Это позволяет каждой дочерней коллекции иметь общие свойства родительской коллекции и при этом содержать атрибуты, специфичные для подкласса.
- **Запросы:** При выполнении запросов вы можете выбрать запрос ко всей иерархии наследования, либо только к родительской, либо к конкретной дочерней коллекции. Это позволяет получать и обрабатывать данные различных уровней по мере необходимости.
- **Отношение наследования:** Между родительской и дочерней коллекциями устанавливается отношение наследования, что означает, что структура родительской коллекции может быть использована для определения согласованных атрибутов, при этом дочерняя коллекция может расширять или переопределять эти атрибуты.

Этот шаблон проектирования помогает уменьшить избыточность данных, упростить модель базы данных и облегчить обслуживание данных. Однако использовать его следует с осторожностью, поскольку наследуемые коллекции могут увеличить сложность запросов, особенно при работе со всей иерархией наследования. Базы данных, поддерживающие наследуемые коллекции, обычно предоставляют специальный синтаксис и инструменты для управления и запроса таких структур коллекций.
