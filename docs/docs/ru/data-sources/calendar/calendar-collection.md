# Таблица календаря

## Введение

Таблица календаря подходит для хранения данных с временными диапазонами, например бронирований переговорных комнат, графиков проектов, расписаний занятий, планов дежурств и календарей мероприятий. По сути, это по-прежнему таблица данных, но с предустановленными полями, связанными с событиями календаря, что упрощает последующее использование блока календаря.

Таблицу календаря можно создать только на странице основной базы данных. Внешние базы данных, источники данных REST API и внешние источники данных NocoBase не поддерживают создание таблиц календаря.

## Сценарии использования

Таблица календаря подходит для следующих бизнес-сценариев:

- Бронирование переговорных комнат, автомобилей и оборудования
- Графики проектов, планы задач и расписания контрольных точек
- Расписания занятий, планы обучения и календари мероприятий
- Планы дежурств, графики смен и планы инспекций
- Записи событий, которые нужно просматривать по дням, неделям или месяцам

## Создание и настройка

В основной базе данных нажмите «Create collection» и выберите «Calendar collection», чтобы создать таблицу календаря.

Настройка создания таблицы календаря в основном совпадает с настройкой обычной таблицы. `Preset fields` используется для управления часто используемыми системными полями, а в таблице календаря также будут предустановлены поля для хранения повторяющихся событий.

| Настройка | Описание |
| --- | --- |
| Collection display name | Название таблицы данных, отображаемое в интерфейсе, например «Бронирование переговорных комнат», «Расписание занятий» или «План дежурств». |
| Collection name | Идентификационное имя таблицы данных, используемое для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. |
| Inherits | Выбор родительской таблицы, от которой выполняется наследование. Видно только при использовании PostgreSQL в качестве основной базы данных. |
| Categories | Категории таблицы данных. Категории влияют только на способ организации интерфейса управления таблицами данных и не изменяют структуру таблицы. |
| Description | Описание таблицы данных. Здесь можно указать, какие события хранит эта таблица календаря, кто её обслуживает и с какими бизнес-процессами она связана. |
| Preset fields | Предустановленные поля. При создании таблицы календаря рекомендуется сохранить системные поля и встроенные поля таблицы календаря. |

### Встроенные поля

После создания таблица календаря обычно содержит следующие встроенные поля. `cron` и `exclude` используются для хранения правил повторения событий и исключённых дат.

| Поле | Имя поля | Описание |
| --- | --- | --- |
| ID | `id` | Поле первичного ключа по умолчанию, используемое для уникальной идентификации записи события. |
| Время создания | `createdAt` | Автоматически записывает время создания записи события. |
| Создатель | `createdBy` | Пользователь, создавший запись события, записывается автоматически. |
| Время обновления | `updatedAt` | Автоматически записывает время последнего обновления записи события. |
| Обновил | `updatedBy` | Пользователь, последним обновивший запись события, записывается автоматически. |
| Сортировка | `sort` | Хранит значение сортировки записи события и поддерживает такие возможности, как сортировка перетаскиванием. |
| Repeats | `cron` | Хранит правила повторения событий, например повторение каждый день, неделю, месяц или год. |
| Exclude | `exclude` | Хранит даты, исключённые из повторяющихся событий; обычно поддерживается автоматически при взаимодействии с календарём. |
| Пространство | `space` | Доступно после включения [плагина мультипространств](../../multi-app/multi-space/index.md) и используется для изоляции данных по пространствам. При отключённом мультипространстве не отображается. |

При использовании таблицы календаря в блоке календаря также необходимо указать бизнес-поля, используемые для отображения событий:

| Настройка | Описание |
| --- | --- |
| Поле заголовка | Определяет заголовок события в календаре, например «Тема встречи» или «Название занятия». |
| Поле даты начала | Определяет время начала события. Обычно используется поле даты и времени. |
| Поле даты окончания | Определяет время окончания события. Обычно используется поле даты и времени. |

:::warning Внимание

`cron` и `exclude` обычно поддерживаются функциями календаря, поэтому не рекомендуется редактировать их напрямую как обычные бизнес-поля. Поля заголовка, даты начала и даты окончания необходимо создать и настроить в соответствии с бизнес-требованиями, иначе блок календаря не сможет корректно отображать события.

:::

### Поле первичного ключа

Как и обычная таблица, таблица календаря должна иметь поле первичного ключа. При создании таблицы рекомендуется сохранить предустановленное поле ID; тип первичного ключа по умолчанию — `Snowflake ID (53-bit)`.

Если в таблице календаря нет первичного ключа, при редактировании таблицы необходимо задать «Record unique key», иначе блок календаря может некорректно открывать, редактировать или находить записи событий.

## Редактирование настроек

В списке таблиц данных нажмите «Edit» справа от таблицы календаря, чтобы изменить отображаемое имя таблицы данных, категории, описание, режим простой пагинации, «Record unique key» и другие настройки.

Встроенные поля таблицы календаря `cron`, `exclude` и другие обычно используются функциями календаря, поэтому не рекомендуется придавать им другое бизнес-значение. Если нужно расширить информацию о событиях, можно добавить обычные бизнес-поля, например место проведения, участников, переговорную комнату, статус и другие.

## Удаление таблицы данных

В списке таблиц данных нажмите «Delete» справа от таблицы календаря, чтобы удалить её.

Удаление таблицы календаря удалит записи событий, данные встроенных полей календаря и связанные метаданные Collection. Перед удалением убедитесь, что блоки календаря и таблиц, права доступа, рабочие процессы и API больше не зависят от этой таблицы.

:::danger Предупреждение

Таблицы календаря обычно хранят данные о расписаниях, бронированиях и дежурствах. После удаления исторические события и правила повторения будут утрачены. Перед выполнением операции убедитесь, что данные сохранены в резервной копии или больше не нужны.

:::
