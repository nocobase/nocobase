# Поле сортировки

## Введение

В NocoBase **поле сортировки (Sort)** используется для хранения значения порядка записей в таблице данных. Оно часто применяется для сортировки записей перетаскиванием в таких блоках, как таблица и канбан.

Поле сортировки поддерживает как сортировку без группировки, так и сортировку после группировки. Сортировка по группам подходит для сценариев, в которых записи сортируются независимо внутри каждой группы, например для распределения студентов по классам или задач по статусам на канбан-доске.

:::warning Внимание

Поскольку поле сортировки является полем той же таблицы, при сортировке по группам одна и та же запись не может одновременно находиться в нескольких группах.

:::

## Установка

Поле сортировки предоставляется встроенным плагином и не требует отдельной установки.

## Создание поля сортировки

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Сортировка», чтобы создать поле сортировки.

![20240409091123_rec_](https://static-docs.nocobase.com/20240409091123_rec_.gif)

При создании поля сортировки NocoBase инициализирует значения сортировки:

- Если сортировка по группам не выбрана, значения инициализируются на основе поля первичного ключа и поля даты создания
- Если выбрана сортировка по группам, данные сначала группируются, а затем значения инициализируются на основе поля первичного ключа и поля даты создания

:::warning Внимание

Если при создании поля не удаётся инициализировать значения сортировки, поле сортировки создано не будет. Если в определённом диапазоне одна запись перемещается с позиции A на позицию B, значения сортировки всех записей в диапазоне AB изменяются; если для одной из них операция завершается ошибкой, перемещение не выполняется, и значения сортировки связанных записей не изменяются.

:::

## Сортировка перетаскиванием

Поле сортировки в основном используется для сортировки записей различных блоков перетаскиванием. В настоящее время сортировка перетаскиванием поддерживается в блоках таблицы и канбан-доски.

:::warning Внимание

- Использование одного поля сортировки в нескольких блоках может нарушить существующий порядок
- Для сортировки строк таблицы перетаскиванием нельзя выбрать поле сортировки с правилом группировки
- В блоке таблицы связи «один-ко-многим» внешний ключ можно использовать для группировки
- В настоящее время только блок канбан-доски поддерживает сортировку перетаскиванием по группам

:::

### Сортировка строк таблицы перетаскиванием

В блоке таблицы можно использовать поле сортировки для изменения порядка записей перетаскиванием.

![20240409104621_rec_](https://static-docs.nocobase.com/20240409104621_rec_.gif)

В блоке таблицы связей также можно сортировать записи перетаскиванием с помощью поля сортировки.

<video controls width="100%" src="https://static-docs.nocobase.com/20240409111903_rec_.mp4" title="Сортировка блока таблицы связей перетаскиванием"></video>

:::warning Внимание

В блоке связи «один-ко-многим», если выбрать поле сортировки без группировки, в сортировке могут участвовать все записи; если сначала сгруппировать записи по внешнему ключу, а затем отсортировать, правило сортировки повлияет только на данные текущей группы. Итоговый результат может выглядеть одинаково, но диапазон записей, участвующих в сортировке, различается.

:::

### Сортировка карточек канбан-доски перетаскиванием

В блоке канбан-доски можно использовать поле сортировки для изменения порядка карточек перетаскиванием.

## Описание правил сортировки

### Перемещение между записями без группировки

Предположим, имеется набор данных:

```text
[1,2,3,4,5,6,7,8,9]
```

Когда 5 перемещается вперёд на позицию 3, изменяются только номера 3, 4 и 5. Запись 5 занимает позицию 3, а записи 3 и 4 сдвигаются на одну позицию назад.

```text
[1,2,5,3,4,6,7,8,9]
```

Затем, если переместить 6 назад на позицию 8, запись 6 занимает позицию 8, а записи 7 и 8 сдвигаются на одну позицию вперёд.

```text
[1,2,5,3,4,7,8,6,9]
```

### Перемещение между разными группами

При сортировке по группам, когда запись перемещается в другую группу, изменяется и группа, к которой она относится. Предположим, имеются две группы данных:

```text
A: [1,2,3,4]
B: [5,6,7,8]
```

Когда 1 перемещается после 6, её группа также изменяется с A на B.

```text
A: [2,3,4]
B: [5,6,1,7,8]
```

### Изменение порядка не зависит от данных, отображаемых в интерфейсе

Предположим, имеется набор данных:

```text
[1,2,3,4,5,6,7,8,9]
```

В интерфейсе отображаются только:

```text
[1,5,9]
```

Когда 1 перемещается на позицию 9, позиции промежуточных записей 2, 3, 4, 5, 6, 7 и 8 также изменяются.

```text
[2,3,4,5,6,7,8,9,1]
```

В итоге интерфейс отображает:

```text
[5,9,1]
```
