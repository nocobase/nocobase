# Markdown Vditor

## Введение

В NocoBase **Markdown Vditor (Markdown Vditor)** используется для редактирования содержимого Markdown в редакторе Vditor.

Markdown Vditor подходит для материалов, которым требуется более полноценный опыт редактирования Markdown, например для текста комментариев, содержимого базы знаний, описаний решений и т. д.

Если нужно простое редактирование Markdown, можно выбрать [Markdown](../data-modeling/collection-fields/media/markdown.md). Если требуется WYSIWYG-редактор для работы с форматированным текстом, выберите [форматированный текст](../data-modeling/collection-fields/media/rich-text.md).

## Подходящие сценарии

Markdown Vditor подходит для следующих бизнес-сценариев:

- Содержимое комментариев и обсуждений
- Содержимое базы знаний и описания решений
- Длинные тексты с разметкой Markdown
- Материалы, для которых нужны возможности предварительного просмотра и редактирования

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Markdown Vditor», чтобы создать поле Markdown Vditor.

![20240512180647](https://static-docs.nocobase.com/20240512180647.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для Markdown Vditor соответствует `vditor` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя, под которым поле отображается в интерфейсе, например «Содержимое комментария», «Содержимое базы знаний» или «Описание решения». Рекомендуется использовать название, понятное бизнес-пользователям. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и т. д. После создания обычно не изменяется; может содержать только буквы, цифры и символы подчёркивания и должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для сохранения содержимого поля Markdown Vditor обычно используется `text`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно ограничить длину или сделать поле обязательным. |
| Description | Описание поля. Здесь можно указать смысл поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле Markdown Vditor по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `vditor`. |
| Тип Field type по умолчанию | `text`. |
| Доступный тип Field type | `text`. |
| Компонент страницы | В режиме редактирования используется редактор MarkdownVditor. |
| Фильтрация | Поддерживаются текстовые фильтры, например «содержит», «пусто» и «не пусто». |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются текстовые проверки, такие как ограничение длины и обязательность заполнения. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля Markdown Vditor. Редактирование поля используется главным образом для изменения способа его отображения и использования в NocoBase, например имени отображения, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, синхронизированной с основной базой данных, редактирование обычно выполняется в форме сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет имя поля, отображаемое в интерфейсе, но не изменяет идентификатор поля. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Интерфейс полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Тип полей основной базы данных или синхронизированных полей можно изменить при сопоставлении полей. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о смысле поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных в рабочих процессах. При большом объёме существующих данных сначала убедитесь, что формат данных соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле Markdown Vditor. В основной базе данных также можно выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля Markdown Vditor обычно одновременно удаляются соответствующий физический столбец в базе данных и уже хранящиеся в нём данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
