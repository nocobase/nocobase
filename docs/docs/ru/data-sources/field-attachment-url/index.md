# URL вложения

## Введение

В NocoBase **URL вложения (Attachment URL)** используется для хранения адреса внешнего файла.

Поле URL вложения подходит для сценариев, когда файл уже хранится во внешней системе, объектном хранилище или CDN, а в NocoBase нужно сохранить только адрес доступа к нему.

Если необходимо загружать и управлять файлами через NocoBase, выберите [Вложение](../file-manager/field-attachment.md). Если это обычный веб-адрес, выберите [URL](../data-modeling/collection-fields/basic/url.md).

## Сценарии использования

URL вложения подходит для следующих бизнес-сценариев:

- Адрес файла во внешнем объектном хранилище
- Адрес изображения в CDN
- Адрес документа в сторонней системе
- Ссылки на файлы после миграции исторических данных

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «URL вложения», чтобы создать поле URL вложения.

![20241017092323](https://static-docs.nocobase.com/20241017092323.png)

![20241017092456](https://static-docs.nocobase.com/20241017092456.png)

![20241017092759](https://static-docs.nocobase.com/20241017092759.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Интерфейсный тип поля. Для URL вложения соответствует `attachmentUrl` и определяет способ ввода и отображения в интерфейсе. |
| Field display name | Название поля, отображаемое в интерфейсе, например «Адрес файла», «URL изображения» или «Внешнее вложение». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для API, полей связей, разрешений, рабочих процессов и других внутренних ссылок. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Для хранения адреса URL вложения обычно используется `string` или `text`. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь ничего не указал. |
| Validation rules | Правила проверки. Можно настроить формат URL, длину или обязательность заполнения. |
| Description | Описание поля. Здесь удобно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

После создания имя поля будет использоваться блоками страниц, разрешениями, рабочими процессами и API. Перед созданием проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в дальнейшем.

:::

## Особенности поля

Поле URL вложения по умолчанию работает следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `attachmentUrl`. |
| Тип Field type по умолчанию | `string`. |
| Доступные типы Field type | `string`, `text` — в зависимости от фактической конфигурации поля. |
| Компонент страницы | В режиме редактирования используется компонент ввода URL или адреса вложения. |
| Фильтрация | Поддерживаются текстовые фильтры и проверка на пустое значение. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются проверки формата URL, длины, обязательности и другие. |

## Редактирование конфигурации

После создания нажмите «Edit» справа от поля, чтобы изменить конфигурацию поля URL вложения. Редактирование поля используется главным образом для настройки его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных параметров поля.

Если поле получено из уже синхронизированной таблицы основной базы данных, редактирование обычно выполняется в рамках сопоставления полей — поле базы данных сопоставляется с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, не меняя его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Дополняет сведения о назначении поля, требованиях к заполнению, источнике данных или ответственном за обслуживание. |

:::warning Внимание

Переключение Field type или Field interface — это не простое изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и использование переменных рабочих процессов. При большом объёме существующих данных сначала убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле URL вложения. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении созданного в основной базе данных поля URL вложения обычно одновременно удаляются соответствующий физический столбец в базе данных и содержащиеся в нём данные. При удалении поля, синхронизированного из базы данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника данных и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, разрешения, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что поле больше не используется в бизнес-конфигурациях.

:::
