## Добавление коллекции

RESTful-ресурс является коллекцией NocoBase, например ресурс Users.

```bash
GET /users
POST /users
GET /users/1
PUT /users/1
DELETE /users/1
```

Настройка сопоставления с API NocoBase выглядит следующим образом:

```bash
GET /users:list
POST /users:create
POST /users:get?filterByTk=1
POST /users:update?filterByTk=1
POST /users:destroy?filterByTk=1
```

Полную спецификацию проектирования API NocoBase см. в документации по API.

### Преобразование информации об ошибках

При возникновении ошибки в стороннем API формат информации об ошибке в ответе может отличаться от стандартного формата NocoBase. Для корректного отображения в интерфейсе его необходимо преобразовать.

![20251201170545](https://static-docs.nocobase.com/20251201170545.png)

Если преобразование информации об ошибках не настроено, по умолчанию она преобразуется в сообщение об ошибке, содержащее код состояния HTTP.

![20251201170732](https://static-docs.nocobase.com/20251201170732.png)

После настройки преобразования информации об ошибках она приводится к стандарту вывода NocoBase, и интерфейс корректно отображает информацию об ошибках стороннего API.

## Переменные

Источник данных REST API предоставляет три типа переменных для подключения интерфейсов:

- Пользовательские переменные источника данных
- Запрос NocoBase
- Ответ стороннего API

### Пользовательские переменные источника данных

![20240716221937](https://static-docs.nocobase.com/20240716221937.png)

![20240716221858](https://static-docs.nocobase.com/20240716221858.png)

### Запрос NocoBase

- Params: параметры запроса URL (Search Params); набор Params может различаться для разных интерфейсов;
- Headers: заголовки запроса, главным образом содержащие пользовательские X-параметры NocoBase;
- Body: тело запроса;
- Token: API-токен текущего запроса NocoBase.

![20251201164833](https://static-docs.nocobase.com/20251201164833.png)

### Ответ стороннего API

В настоящее время предоставляется только Body ответа.

![20251201164915](https://static-docs.nocobase.com/20251201164915.png)

При подключении каждого интерфейса доступны следующие переменные:

### List

| Параметр                 | Описание                                                     |
| ------------------------ | ------------------------------------------------------------ |
| request.params.page     | Текущая страница                                             |
| request.params.pageSize | Количество элементов на странице                             |
| request.params.filter   | Условия фильтрации (должны соответствовать формату Filter NocoBase) |
| request.params.sort     | Правила сортировки (должны соответствовать формату Sort NocoBase) |
| request.params.appends  | Поля, загружаемые по запросу; обычно используются для загрузки полей отношений по запросу |
| request.params.fields   | Какие поля должен выводить интерфейс (белый список)          |
| request.params.except   | Какие поля следует исключить (черный список)                |

### Get

| Параметр                   | Описание                                                     |
| -------------------------- | ------------------------------------------------------------ |
| request.params.filterByTk | Обязательно; обычно это ID текущих данных                    |
| request.params.filter     | Условия фильтрации (должны соответствовать формату Filter NocoBase) |
| request.params.appends    | Поля, загружаемые по запросу; обычно используются для загрузки полей отношений по запросу |
| request.params.fields     | Какие поля должен выводить интерфейс (белый список)          |
| request.params.except     | Какие поля следует исключить (черный список)                |

### Create

| Параметр                  | Описание              |
| ------------------------- | --------------------- |
| request.params.whiteList | Белый список           |
| request.params.blacklist | Черный список          |
| request.body             | Исходные данные для создания |

### Update

| Параметр                   | Описание                                                     |
| -------------------------- | ------------------------------------------------------------ |
| request.params.filterByTk | Обязательно; обычно это ID текущих данных                    |
| request.params.filter     | Условия фильтрации (должны соответствовать формату Filter NocoBase) |
| request.params.whiteList  | Белый список                                                 |
| request.params.blacklist  | Черный список                                                |
| request.body              | Обновляемые данные                                           |

### Destroy

| Параметр                   | Описание                                                     |
| -------------------------- | ------------------------------------------------------------ |
| request.params.filterByTk | Обязательно; обычно это ID текущих данных                    |
| request.params.filter     | Условия фильтрации (должны соответствовать формату Filter NocoBase) |
