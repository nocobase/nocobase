# Файловый менеджер

## Введение

Плагин Файловый менеджер предоставляет коллекцию файлов, поле вложений и движки хранения файлов для эффективного управления файлами. Файлы — это записи в специально структурированной коллекции, известной как коллекция файлов, которая хранит метаданные файлов и может управляться через Файловый менеджер. Поля вложений — это специальные поля связей, относящиеся к коллекции файлов. Плагин поддерживает несколько способов хранения, включая локальное хранилище, Alibaba Cloud OSS, Amazon S3 и Tencent Cloud COS.

## Руководство пользователя

### Файловая коллекция

Встроенная коллекция вложений предназначена для хранения всех файлов, связанных с полями вложений. Также можно создавать новые коллекции файлов для хранения определенных файлов.

[Подробнее в документации по файловой коллекции](/data-sources/file-manager/file-collection)

### Поле вложений

Поля вложений — это специальные поля связей, относящиеся к коллекции файлов. Их можно создать через тип поля «Вложение» или настроить с помощью поля «Ассоциация».

[Подробнее в документации по полю вложений](/data-sources/file-manager/field-attachment)

### Движок хранения файлов

Движок хранения файлов используется, чтобы сохранять файлы в конкретные сервисы, включая локальное хранилище (сохранение на жесткий диск сервера), облачное хранилище и т.д.

[Подробнее в документации по движку хранения файлов](./storage/index.md)

### HTTP API

Загрузка файлов может быть обработана через HTTP API, см. [HTTP API](./http-api.md).
