# Файловый менеджер

## Введение

Плагин Файловый менеджер предоставляет файловую коллекцию, поле вложения и движки хранения файлов для эффективного управления файлами. Файлы — это записи в специальном типе коллекции, известной как файловая коллекция, которая хранит метаданные файлов и может управляться через Файловый менеджер. Поля вложения — это специфические поля связи, ассоциированные с файловой коллекцией. Плагин поддерживает несколько методов хранения, включая локальное хранилище, Alibaba Cloud OSS, Amazon S3 и Tencent Cloud COS.

## Руководство пользователя

### Файловая коллекция

Встроена коллекция `attachments` для хранения всех файлов, связанных с полями вложения. Кроме того, вы можете создавать новые файловые коллекции для хранения определённых файлов.

[Узнайте больше в документации по файловой коллекции](/data-sources/file-manager/file-collection)

### Поле вложения

Поля вложения — это специфические поля связи, относящиеся к файловой коллекции. Их можно создать с помощью типа поля «Вложение» или настроить через поле «Связь».

[Узнайте больше в документации по полю вложения](/data-sources/file-manager/field-attachment)

### Движок хранения файлов

Движок хранения файлов используется для сохранения файлов в определённые сервисы, включая локальное хранилище (сохранение на жёсткий диск сервера), облачное хранилище и т.д.

[Узнайте больше в документации по движку хранения файлов](./storage/index.md)

### HTTP API

Загрузка файлов может быть обработана через HTTP API. См. [HTTP API](./http-api.md).
