# Менеджер файлов

<PluginInfo name="file-manager"></PluginInfo>

## Введение

Плагин менеджера файлов предоставляет таблицы файлов, поля вложений и движки хранения файлов для эффективного управления файлами. Файлы представляют собой записи таблиц данных определённой структуры. Такая таблица данных называется таблицей файлов и используется для хранения метаданных файлов; управлять ею можно через менеджер файлов. Поле вложений — это специальное поле связи, связанное с таблицей файлов. Файлы поддерживают несколько способов хранения. В настоящее время доступны следующие движки хранения: локальное хранилище, Alibaba Cloud OSS, Amazon S3 и Tencent Cloud COS.

## Руководство пользователя

### Таблица файлов

В систему встроена таблица attachments, предназначенная для хранения файлов, связанных со всеми полями вложений. Кроме того, можно создавать новые таблицы файлов для хранения определённых файлов.

[Подробнее об использовании см. в документации о таблицах файлов](/data-sources/file-manager/file-collection)

### Поле вложений

Поле вложений — это специальное поле связи, связанное с таблицей файлов. Его можно создать с помощью «поля типа вложений» или настроить через «поле связи».

[Подробнее об использовании см. в документации о полях вложений](/data-sources/file-manager/field-attachment)

### Движок хранения файлов

Движок хранения файлов используется для сохранения файлов в определённых сервисах, включая локальное хранилище (сохранение на жёстком диске сервера), облачное хранилище и другие варианты.

[Подробнее см. в документации о движках хранения файлов](./storage/index.md)

### HTTP API

Загрузку файлов можно выполнять через HTTP API. См. [HTTP API](./http-api.md).
