# Хранилище файлов: S3 (Pro)

## Введение

Опираясь на плагин управления файлами, эта версия добавляет поддержку типов хранения файлов, совместимых с протоколом S3. Любой сервис объектного хранения, поддерживающий протокол S3, можно легко интегрировать, например Amazon S3, Alibaba Cloud OSS, Tencent Cloud COS, MinIO, Cloudflare R2 и т.д., повышая совместимость и гибкость сервисов хранения.

## Возможности

1. **Загрузка на стороне клиента:** Файлы загружаются напрямую в сервис хранения без прохождения через сервер NocoBase, что обеспечивает более эффективную и быструю загрузку.

2. **Приватный доступ:** Все URL файлов являются подписанными временными адресами авторизации, что обеспечивает безопасный доступ к файлам и ограничение по времени.

## Сценарии использования

1. **Управление таблицей файлов:** Централизованно управляйте и храните все загруженные файлы, поддерживая различные типы файлов и методы хранения для удобной классификации и поиска.

2. **Хранение вложений в поле вложений:** Храните вложения, загруженные через формы или записи, и связывайте их с конкретными записями данных.

## Настройка плагина

1. Включите плагин `plugin-file-storage-s3-pro`.

2. Перейдите в "Настройки -> Файловый менеджер", чтобы открыть настройки управления файлами.

3. Нажмите кнопку "Добавить новое" и выберите "S3 Pro".

![](https://static-docs.nocobase.com/20250102160704938.png)

4. В появившемся окне вы увидите подробную форму для заполнения. Обратитесь к документации ниже, чтобы получить нужные параметры для вашего сервиса хранения и корректно ввести их в форму.

![](https://static-docs.nocobase.com/20250413190828536.png)

## Конфигурация провайдера сервиса

### Amazon S3

#### Создание бакета

1. Откройте [консоль Amazon S3 (Amazon S3 Console)](https://ap-southeast-1.console.aws.amazon.com/s3/home).

2. Нажмите кнопку "Create bucket (Создать бакет)" справа.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355969452.png)

3. Укажите `Bucket Name`, оставьте остальные поля значениями по умолчанию, прокрутите вниз и нажмите кнопку **"Create"**, чтобы завершить процесс.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355969622.png)

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355969811.png)

#### Настройка CORS

1. В списке бакетов найдите созданный бакет и нажмите на него, чтобы открыть сведения о нем.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355969980.png)

2. Перейдите на вкладку `Permission` и прокрутите вниз до раздела настройки CORS.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970155.png)

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970303.png)

3. Введите следующую конфигурацию (настройте по необходимости) и сохраните.

```json
[
    {
        "AllowedHeaders": [
            "*"
        ],
        "AllowedMethods": [
            "POST",
            "PUT"
        ],
        "AllowedOrigins": [
            "*"
        ],
        "ExposeHeaders": [
            "ETag"
        ],
        "MaxAgeSeconds": 3000
    }
]
```

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970494.png)

#### Получение AccessKey и SecretAccessKey

1. Нажмите кнопку `Security credentials` в правом верхнем углу.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970651.png)

2. Прокрутите до раздела `Access Keys` и нажмите `Create Access Key`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970832.png)

3. Подтвердите условия (рекомендуется использовать IAM в производственной среде).

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355970996.png)

4. Сохраните отображаемые Access Key и Secret Access Key.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355971168.png)

#### Получение параметров и настройка

1. Используйте полученные `AccessKey ID` и `AccessKey Secret`.

2. Откройте панель свойств бакета, чтобы найти `Bucket Name` и `Region`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355971345.png)

#### Публичный доступ (опционально)

Это необязательная настройка. Используйте ее, когда нужно сделать загруженные файлы полностью публичными.

1. В панели `Permissions` прокрутите до `Object Ownership`, нажмите `Edit` и включите ACLs.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355971508.png)

2. Прокрутите до `Block public access`, нажмите `Edit` и установите, чтобы разрешить контроль ACL.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355971668.png)

3. В NocoBase включите `Публичный доступ`.

#### Настройка превью миниатюр (опционально)

Эта настройка необязательна и должна использоваться, когда нужно оптимизировать размер или эффект предпросмотра изображений. **Обратите внимание: это развертывание может повлечь дополнительные расходы. Подробнее см. условия и прайс-лист AWS.**

1. Откройте [Dynamic Image Transformation for Amazon CloudFront](https://aws.amazon.com/solutions/implementations/dynamic-image-transformation-for-amazon-cloudfront/?nc1=h_ls).

2. Нажмите кнопку `Launch in the AWS Console` в нижней части страницы, чтобы начать развертывание.

![](https://static-docs.nocobase.com/20250221164214117.png)

3. Следуйте подсказкам, чтобы завершить конфигурацию. На следующие параметры нужно обратить особое внимание:
   1. При создании стека необходимо указать имя Amazon S3 бакета, в котором находятся исходные изображения. Введите имя бакета, которое вы указали ранее.
   2. Если вы выбрали развертывание демо-интерфейса (demo UI), после развертывания можно использовать интерфейс для тестирования функциональности обработки изображений. В консоли AWS CloudFormation выберите ваш стек, перейдите на вкладку `Outputs`, найдите значение, соответствующее ключу `DemoUrl`, и откройте демо-интерфейс по ссылке.
   3. Это решение использует библиотеку `sharp` Node.js для эффективной обработки изображений. Исходный код можно скачать из репозитория GitHub и при необходимости настроить.

![](https://static-docs.nocobase.com/20250221164315472.png)

![](https://static-docs.nocobase.com/20250221164404755.png)

4. Когда конфигурация завершена, дождитесь, пока статус развертывания изменится на `CREATE_COMPLETE`.

5. В конфигурации NocoBase обратите внимание на следующее:
   1. `Правило для миниатюр`: Укажите параметры обработки изображений, например `?width=100`. Подробнее см. в [документации AWS](https://docs.aws.amazon.com/solutions/latest/serverless-image-handler/use-supported-query-param-edits.html).
   2. `Конечная точка доступа`: После развертывания введите значение из Outputs -> ApiEndpoint.
   3. `Стиль URL для доступа`: Выберите **Игнорировать** (так как имя бакета уже указано в конфигурации, оно не требуется для доступа).

![](https://static-docs.nocobase.com/20250414152135514.png)

### Alibaba Cloud OSS

#### Создание бакета

1. Откройте [консоль OSS](https://oss.console.aliyun.com/overview).

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355972149.png)

2. В левом меню выберите `Buckets` и нажмите `Create Bucket`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355972413.png)

3. Укажите данные бакета и нажмите `Create`.

   - `Bucket Name`: Выберите в соответствии с вашими бизнес-задачами.

   - `Region`: Выберите ближайший регион для ваших пользователей.

   - Остальные параметры можно оставить по умолчанию или настроить при необходимости.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355972730.png)

#### Настройка CORS

1. Перейдите на страницу сведений о бакете, который вы только что создали.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355973018.png)

2. В центральном меню нажмите `Content Security -> CORS`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355973319.png)

3. Нажмите `Create Rule`, заполните поля, прокрутите вниз и нажмите `OK`. Вы можете ориентироваться на приведенный ниже скриншот или настроить более детальные параметры.

![](https://static-docs.nocobase.com/20250219171042784.png)

#### Получение AccessKey и SecretAccessKey 

1. Нажмите `AccessKey` под аватаром аккаунта в правом верхнем углу.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355973884.png)

2. В демонстрационных целях мы создадим AccessKey с помощью основной учетной записи. В производственной среде рекомендуется использовать RAM для создания AccessKey. Инструкции см. в [документации Alibaba Cloud](https://www.alibabacloud.com/help/en/ram/user-guide/create-an-accesskey-pair).
    
3. Нажмите кнопку `Create AccessKey`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355974171.png)

4. Завершите проверку учетной записи.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355974509.png)

5. Сохраните отображаемые Access Key и Secret Access Key.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355974781.png)

#### Получение параметров и настройка

1. Используйте `AccessKey ID` и `AccessKey Secret`, полученные на предыдущем шаге.

2. Перейдите на страницу сведений о бакете, чтобы получить имя `Bucket`.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355975063.png)

3. Прокрутите вниз, чтобы получить `Region` (конечная строка ".aliyuncs.com" не нужна).

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355975437.png)

4. Получите адрес endpoint и добавьте префикс `https://` при вводе в NocoBase.

![](https://static-docs.nocobase.com/file-storage-s3-pro-1735355975715.png)

#### Настройка превью миниатюр (опционально)

Эта настройка необязательна и применяется только при оптимизации размера или эффекта предпросмотра изображений.

1. Укажите соответствующие параметры для `Правило для миниатюр`. Для конкретных настроек параметров см. документацию Alibaba Cloud по [обработке изображений](https://www.alibabacloud.com/help/en/oss/user-guide/image-processing).

2. Параметры `Стиль URL для загрузки` и `Стиль URL для доступа` оставьте без изменений (то же значение).

### MinIO

#### Создание бакета

1. Нажмите в левом меню **Buckets**, затем нажмите **Create Bucket**, чтобы открыть страницу создания.
2. Введите имя бакета, затем нажмите кнопку **Save**.

#### Получение AccessKey и SecretAccessKey

1. Перейдите в **Access Keys**, затем нажмите кнопку **Create access key**, чтобы открыть страницу создания.

![](https://static-docs.nocobase.com/20250106111922957.png)

2. Нажмите кнопку **Save**.

![](https://static-docs.nocobase.com/20250106111850639.png)

3. Сохраните **Access Key** и **Secret Key** во всплывающем окне для дальнейшей конфигурации.

![](https://static-docs.nocobase.com/20250106112831483.png)

#### Конфигурация параметров

1. Перейдите на страницу **Файловый менеджер** в NocoBase.

2. Нажмите кнопку **Добавить новый** и выберите **S3 Pro**.

3. Заполните форму:
   - **ID ключа доступа** и **Секрет ключа доступа**: используйте значения, сохраненные на предыдущем шаге.
   - **Регион**: приватно развернутый MinIO не имеет понятия региона; можно задать значение `"auto"`.
   - **Точка доступа**: введите доменное имя или IP-адрес развернутого сервиса.
   - Установите **Стиль URL для доступа** на **Стиль пути**.
