# Обзор

## Встроенные движки

В настоящее время NocoBase поддерживает следующие типы встроенных движков:

- [Локальное хранилище](./local.md)
- [Alibaba Cloud OSS](./aliyun-oss.md)
- [Amazon S3](./amazon-s3.md)
- [Tencent Cloud COS](./tencent-cos.md)

При установке системы автоматически добавляется движок локального хранилища, который можно использовать сразу. Также можно добавить новый движок или изменить параметры существующего.

## Общие параметры движка

Помимо параметров, характерных для разных типов движков, существуют следующие общие параметры (на примере локального хранилища):

![Пример настройки движка хранения файлов](https://static-docs.nocobase.com/20240529115151.png)

### Заголовок

Название движка хранения, используемое для его идентификации пользователем.

### Системное имя

Системное имя движка хранения, используемое для его идентификации системой. Должно быть уникальным в системе. Если оставить поле пустым, система автоматически сгенерирует случайное имя.

### Базовый URL доступа

Префикс URL-адреса, по которому файл доступен извне. Это может быть базовый URL доступа к CDN, например: “`https://cdn.nocobase.com/app`” (без завершающего “`/`”).

### Путь

Относительный путь, используемый при сохранении файлов. При доступе к файлу эта часть также автоматически добавляется к конечному URL. Например: “`user/avatar`” (без начального и завершающего “`/`”).

### Ограничение размера файла

Ограничение размера файлов, загружаемых в этот движок хранения. Файлы, размер которых превышает заданное значение, загрузить невозможно. По умолчанию система ограничивает размер 20 МБ; максимальное значение можно увеличить до 1 ГБ.

### Тип файла

Можно ограничить тип загружаемых файлов, используя формат описания [MIME](https://developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types). Например: `image/*` обозначает файлы изображений. Несколько типов можно разделить английскими запятыми, например: `image/*, application/pdf` означает, что разрешены файлы изображений и PDF.

### Движок хранения по умолчанию

После установки этого флажка движок становится системным хранилищем по умолчанию. Если в поле вложений или таблице файлов не указан движок хранения, загружаемые файлы будут сохраняться в движке хранения по умолчанию. Движок хранения по умолчанию нельзя удалить.

### Сохранять файлы при удалении записей

Если установлен этот флажок, при удалении записей из таблицы вложений или таблицы файлов загруженные в движок хранения файлы сохраняются. По умолчанию флажок снят: при удалении записи файлы в движке хранения также удаляются.

:::info{title=Подсказка}
После загрузки файла конечный путь доступа формируется объединением нескольких частей:

```
<访问 URL 基础>/<路径>/<文件名><后缀名>
```

Например: `https://cdn.nocobase.com/app/user/avatar/20240529115151.png`.
:::
