# Вложения (устарело)

## Введение

:::warning Внимание

Поле вложений устарело. Рекомендуется использовать поле [таблицы файлов](./file-collection.md) или [URL вложения](../field-attachment-url/index.md).

:::

В NocoBase **вложения (Attachment)** используются для загрузки файлов и связывания записей о файлах с текущей бизнес-записью.

Поле вложений обычно связано с таблицей файлов. Сам файл сохраняется механизмом хранения файлов, а такие метаданные, как имя файла, размер, URL и MIME-тип, сохраняются в таблице файлов.

Если нужно сохранить только ссылку на внешний файл, выберите [URL вложения](../field-attachment-url/index.md) или [URL](../data-modeling/collection-fields/basic/url.md).

## Сценарии использования

Вложения подходят для следующих бизнес-сценариев:

- Вложения к договорам, счета-фактуры, документы для возмещения расходов
- Изображения товаров, удостоверения сотрудников, документы проектов
- Скриншоты заявок, вложения к проблемам
- Несколько файлов, связанных с бизнес-записью

## Создание и настройка

На странице «Configure fields» таблицы данных нажмите «Add field» и выберите «Вложения», чтобы создать поле вложений.

![20240512180916](https://static-docs.nocobase.com/20251031000729.png)

| Настройка | Описание |
| --- | --- |
| Field interface | Тип интерфейса поля. Для вложений соответствует `attachment` и определяет способ ввода и отображения данных на странице. |
| Field display name | Имя поля, отображаемое в интерфейсе, например «Вложения к договору», «Счета-фактуры», «Изображения товаров». Рекомендуется использовать понятное бизнес-пользователям название. |
| Field name | Идентификатор поля, используемый для внутренних ссылок в API, полях связей, правах доступа, рабочих процессах и других компонентах. После создания обычно не изменяется; допускаются только буквы, цифры и символы подчёркивания, при этом имя должно начинаться с буквы. |
| Field type | Тип поля на уровне данных. Поле вложений обычно является полем связи, связанным с записями файлов в таблице файлов. |
| Default value | Значение по умолчанию. При создании записи оно может подставляться автоматически, если пользователь не указал значение. |
| Validation rules | Правила проверки. Можно ограничить обязательность заполнения; количество, размер и тип файлов обычно контролируются компонентом загрузки или настройками хранилища файлов. |
| Description | Описание поля. Здесь можно указать назначение поля, требования к заполнению, источник данных или ответственного за обслуживание. |

:::warning Внимание

Имя поля после создания будет использоваться блоками страниц, правами доступа, рабочими процессами и API. Заранее проверьте название, чтобы избежать дополнительных затрат на изменение конфигурации в будущем.

:::

## Особенности поля

Поле вложений по умолчанию ведёт себя следующим образом:

| Особенность | Описание |
| --- | --- |
| Интерфейс Field interface по умолчанию | `attachment`. |
| Тип Field type по умолчанию | `belongsToMany`. |
| Доступный Field type | `belongsToMany` и другие типы связей; конкретный вариант зависит от настроек поля файлов. |
| Компонент страницы | В режиме редактирования используется компонент загрузки вложений. |
| Фильтрация | Обычно выполняется по наличию значения или связанных файлов. |
| Сортировка | Обычно не используется для сортировки. |
| Проверка | Поддерживаются базовые проверки, например обязательность заполнения; ограничения загрузки зависят от настроек компонента. |

## Редактирование настроек

После создания нажмите «Edit» справа от поля, чтобы изменить настройки поля вложений. Редактирование поля в основном используется для изменения способа его отображения и использования в NocoBase, например для изменения отображаемого имени, описания, значения по умолчанию, правил проверки или специальных настроек поля.

Если поле получено из таблицы, уже синхронизированной с основной базой данных, редактирование обычно означает настройку сопоставления поля — сопоставление поля базы данных с Field type и Field interface в NocoBase.

| Настройка | Можно редактировать | Описание |
| --- | --- | --- |
| Field display name | Да | Изменяет отображаемое в интерфейсе имя поля, но не изменяет его идентификатор. |
| Field name | Нет | Идентификатор поля обычно нельзя изменить в форме редактирования после создания. |
| Field interface | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Это повлияет на способ ввода, отображения и проверки данных на страницах. |
| Field type | При определённых условиях | Для полей основной базы данных или синхронизированных полей его можно изменить при сопоставлении. Перед изменением необходимо убедиться, что существующие данные можно использовать с новым типом. |
| Default value | Да | Изменяет значение по умолчанию для новых записей. |
| Validation rules | Да | Изменяет правила проверки поля. |
| Description | Да | Позволяет дополнить назначение поля, требования к заполнению, источник данных или сведения об ответственном. |

:::warning Внимание

Переключение Field type или Field interface — это не просто изменение отображаемого имени. Оно влияет на способ хранения данных, компонент ввода, правила проверки, условия фильтрации и способ использования переменных в рабочих процессах. Если существующих данных много, заранее убедитесь, что их формат соответствует новому типу.

:::

## Удаление поля

Нажмите «Delete» справа от поля, чтобы удалить поле вложений. В основной базе данных можно также выбрать несколько полей и удалить их одновременно.

При удалении поля вложений, созданного в основной базе данных, обычно одновременно удаляются соответствующий физический столбец в базе данных и все имеющиеся в этом столбце данные. При удалении поля, синхронизированного с базой данных или сопоставленного с внешним источником данных, область воздействия зависит от соответствующего источника и происхождения поля.

:::danger Предупреждение

Удаление поля может повлиять на блоки страниц, формы, фильтры, права доступа, рабочие процессы, API, импорт и экспорт, а также существующие данные. Перед удалением убедитесь, что это поле больше не используется в бизнес-конфигурации.

:::
