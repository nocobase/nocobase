# Коллекция выражений

## Создание шаблонной коллекции «Выражения»

Прежде чем использовать узлы динамических выражений в рабочем процессе, вам необходимо сначала создать шаблонную коллекцию «Выражения» с помощью инструмента управления коллекциями. Эта коллекция будет служить хранилищем для различных выражений:

![Создание шаблонной коллекции для выражений](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## Ввод данных выражений

Затем вы можете создать табличный блок и ввести несколько записей формул в шаблонную коллекцию. Каждая строка в шаблонной коллекции «Выражения» может рассматриваться как правило расчета, разработанное для определенной модели данных в коллекции. Вы можете использовать различные поля из моделей данных разных коллекций в качестве переменных, создавая уникальные выражения как правила расчета. Кроме того, при необходимости вы можете использовать различные движки расчетов.

![Ввод данных выражений](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=Совет}
После создания формул их необходимо связать с бизнес-данными. Прямое связывание каждой строки бизнес-данных с данными формул может быть утомительным. Поэтому часто используется коллекция метаданных, похожая на коллекцию классификации, для создания связи «многие к одному» (или «один к одному») с коллекцией формул. Затем бизнес-данные связываются с классифицированными метаданными по принципу «многие к одному». Такой подход позволяет вам просто указать соответствующие классифицированные метаданные при создании бизнес-данных, что упрощает поиск и использование соответствующих данных формул через установленный путь связи.
:::
