# Таблица выражений

## Создание шаблонной таблицы «Выражения»

Перед использованием узла вычисления динамических выражений в рабочем процессе необходимо сначала создать шаблонную таблицу «Выражения» в инструменте управления таблицами данных для хранения различных выражений:

![Создание шаблонной таблицы выражений](https://static-docs.nocobase.com/33afe3369a1ea7943f12a04d9d4443ce.png)

## Ввод данных выражений

Затем создайте блок таблицы и добавьте в шаблонную таблицу несколько записей с формулами. Каждую строку данных в шаблонной таблице «Выражения» можно рассматривать как отдельное правило вычисления для конкретной модели данных таблицы. В каждой строке с формулой можно использовать значения полей из моделей данных различных таблиц в качестве переменных и записывать разные выражения как правила вычисления. Разумеется, можно также использовать различные вычислительные движки.

![Ввод данных выражений](https://static-docs.nocobase.com/761047f8daabacccbc6a924a73564093.png)

:::info{title=Подсказка}
После создания формул необходимо связать бизнес-данные с формулами. Поскольку напрямую связывать каждую строку бизнес-данных со строкой данных формулы довольно сложно, обычно используют таблицу метаданных, например классификатор, и связывают её с таблицей формул отношением «многие к одному» (или «один к одному»), а затем связывают бизнес-данные с метаданными классификатора отношением «многие к одному». Тогда при создании бизнес-данных достаточно указать конкретные метаданные классификатора, чтобы впоследствии через этот путь связи найти соответствующие данные формулы и использовать их.
:::

## Загрузка соответствующих данных в рабочем процессе

В качестве примера возьмём событие таблицы данных и создадим рабочий процесс, который запускается при создании заказа и предварительно загружает связанные с заказом данные о товарах и связанные с товарами данные выражений:

![Настройка триггера события таблицы данных](https://static-docs.nocobase.com/f181f75b10007afd5de068f3458d2e04.png)
