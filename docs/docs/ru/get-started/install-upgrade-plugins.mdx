# Установка и обновление плагинов

## Встроенные плагины

Встроенные плагины NocoBase автоматически обновляются вместе с версией ядра и не требуют ручных действий.

## Коммерческие плагины

Подробные способы активации и обновления см. в разделе: [Активация и обновление коммерческих плагинов](../api/cli/license/index.md)

## Сторонние плагины

### Установка плагинов через командную строку

```bash
yarn pm pull https://github.com/nocobase/plugin-auth-cas/releases/download/v1.4.0/plugin-auth-cas-1.4.0.tgz
yarn pm pull /your/path/plugin-auth-cas-1.4.0.tgz
```

Если этот плагин уже установлен в приложении и его нужно обновить, выполните:

```bash
yarn nocobase upgrade --skip-code-update
```

### Загрузка и распаковка вручную

Сначала скачайте пакет плагина локально, затем вручную загрузите и распакуйте его в каталог `./storage/plugins`. Если плагин уже включён, после загрузки и распаковки выполните следующую команду, чтобы завершить обновление плагина.

> ⚠️ **Примечание**: Чтобы избежать проблем с кэшем, перед выполнением убедитесь, что приложение NocoBase остановлено.

```bash
yarn nocobase upgrade --skip-code-update
```

### Правильный способ распаковки плагина

В следующем примере показано, как правильно распаковать пакет плагина в указанный каталог:

```bash
mkdir -p /my-nocobase/storage/plugins/@nocobase/plugin-auth-cas && \
  tar -xvzf /downloads/plugin-auth-cas-1.4.0.tgz \
  -C /my-nocobase/storage/plugins/@nocobase/plugin-auth-cas \
  --strip-components=1
```

Эта команда распакует плагин в указанный каталог и не создаст лишний уровень каталога `package`.

```bash
/my-nocobase/storage/plugins/@nocobase/plugin-auth-cas
```
