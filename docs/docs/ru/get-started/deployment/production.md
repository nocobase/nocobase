# Развертывание в производственной среде

При развертывании NocoBase в производственной среде установка зависимостей может быть довольно сложной из-за различий в методах сборки для разных систем и окружений. Для получения полного функционала мы рекомендуем использовать **Docker** для развертывания. Если ваша системная среда не позволяет использовать Docker, вы также можете развернуть NocoBase с помощью **create-nocobase-app**.

:::warning

Не рекомендуется развертывать NocoBase напрямую из исходного кода в производственной среде. Исходный код имеет множество зависимостей, он объемен, а полная компиляция требует значительных ресурсов CPU и памяти. Если вам все же необходимо развернуть NocoBase из исходного кода, рекомендуется сначала собрать собственный образ Docker, а затем выполнить развертывание.

:::

В производственной среде рекомендуется управлять статическими ресурсами с помощью прокси-сервера, например:
