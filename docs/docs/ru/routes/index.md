# Менеджер маршрутов

## Введение

Менеджер маршрутов — это инструмент управления маршрутами основной страницы системы с поддержкой конечных точек для настольной и мобильной версий интерфейса. Маршруты, созданные через менеджер маршрутов, синхронизируются с меню (можно настроить, чтобы они не отображались в меню). В обратную сторону это тоже работает: меню, добавленное в меню страницы, синхронизируется в список менеджера маршрутов.

## Руководство пользователя

### Типы маршрутов

Система поддерживает четыре типа маршрутов:

- **Группа**: используется для группового управления маршрутами и может содержать подмаршруты.
- **Страница**: внутренняя страница системы.
- **Вкладка**: используется для переключения вкладок внутри страницы.
- **Ссылка**: внутренняя или внешняя ссылка с прямым переходом на заданный адрес.

### Добавление маршрута

Нажмите кнопку **«Добавить новый»** в правом верхнем углу, чтобы создать маршрут:

1. Выберите тип маршрута
2. Заполните название маршрута
3. Выберите иконку маршрута
4. Укажите, отображать ли маршрут в меню
5. Укажите, включать ли вкладки страницы
6. Для типа «Страница» система автоматически сгенерирует уникальный путь маршрута

![20250124131803](https://static-docs.nocobase.com/20250124131803.png)

### Действия с маршрутом

Для каждого маршрута доступны действия:

- **Добавить дочерний элемент**: добавить дочерний маршрут
- **Редактировать**: редактировать конфигурацию маршрута
- **Просмотр**: открыть страницу маршрута
- **Удалить**: удалить маршрут

### Массовые действия

Верхняя панель инструментов поддерживает:

- **Обновить**: обновить список маршрутов
- **Удалить**: удалить выбранный маршрут
- **Скрыть в меню**: скрыть выбранный маршрут в меню
- **Показать в меню**: показать выбранный маршрут в меню

### Фильтрация маршрутов

Используйте функцию **«Фильтр»** в верхней части страницы для фильтрации списка маршрутов.

:::info{title=Примечание}
Изменение конфигурации маршрутов напрямую влияет на структуру навигационного меню системы. Выполняйте изменения осторожно и проверяйте корректность настроек.
:::
