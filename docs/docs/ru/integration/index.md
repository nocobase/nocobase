# Интеграции

## Обзор

NocoBase предоставляет широкие возможности интеграции, позволяя бесшовно подключаться к внешним системам, сторонним сервисам и различным источникам данных. Благодаря гибким методам интеграции вы можете расширять функциональность NocoBase под разные бизнес-задачи.

## Методы интеграции

### Интеграция через API

NocoBase предоставляет мощные возможности API для интеграции с внешними приложениями и сервисами:

![20250301003106](https://static-docs.nocobase.com/20250301003106.png)

- **[Ключи API](/integration/api-keys/)**: безопасная аутентификация с использованием ключей API для программного доступа к ресурсам NocoBase
- **[Документация API](/integration/api-doc/)**: встроенная документация API для просмотра и тестирования эндпоинтов

### Единый вход (SSO)

Интеграция с корпоративными системами идентификации для единой аутентификации:

![auth_sso-2025-11-24-08-26-46](https://static-docs.nocobase.com/auth_sso-2025-11-24-08-26-46.png)

- **[Интеграция SSO](/integration/sso/)**: поддержка SAML, OIDC, CAS, LDAP и аутентификации через сторонние платформы
- Централизованное управление пользователями и контроль доступа
- Бесшовная аутентификация между системами

### Интеграция рабочих процессов

Подключайте рабочие процессы NocoBase к внешним системам:

![auth_sso-2025-11-24-12-02-01](https://static-docs.nocobase.com/auth_sso-2025-11-24-12-02-01.png)

- **[HTTP запрос для рабочих процессов](/integration/workflow-http-request/)**: отправлять HTTP-запросы во внешние API из рабочих процессов
- Автоматизировать бизнес-процессы между платформами

### Внешние источники данных

## Типовые сценарии интеграции

### Интеграция с корпоративными системами

- Подключение NocoBase к ERP, CRM или другим корпоративным системам
- Двусторонняя синхронизация данных
- Автоматизация межсистемных рабочих процессов

### Интеграция со сторонними сервисами

- Запрос статуса оплаты у платёжных шлюзов, интеграция сервисов обмена сообщениями или облачных платформ
- Использование внешних API для расширения функциональности
- Создание пользовательских интеграций с помощью вебхуков и HTTP-запросов

### Интеграция данных

- Подключение нескольких источников данных
- Агрегация данных из разных систем
- Создание единых дашбордов и отчётов

## Соображения безопасности

При интеграции NocoBase с внешними системами учитывайте следующие рекомендации по безопасности:

1. **Используйте HTTPS**: всегда применяйте шифрованные соединения для передачи данных
2. **Защищайте ключи API**: храните ключи API безопасно и регулярно выполняйте ротацию
3. **Принцип наименьших привилегий**: выдавайте только необходимые права для интеграций
4. **Логирование аудита**: отслеживайте и записывайте действия интеграций
5. **Валидация данных**: проверяйте все данные, поступающие из внешних источников
