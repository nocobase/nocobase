# Настройка локального окружения разработки

Эта страница поможет подготовить локальное окружение для NocoBase CLI и приложений NocoBase. Она предназначена для локальной разработки, оценки возможностей и для установки или управления NocoBase ИИ агентами на вашем компьютере.

Для развёртывания, ориентированного на реальных пользователей, см. [Системные требования](../get-started/system-requirements.md).

## Windows: используйте WSL

Для локальной настройки на Windows рекомендуем держать основное окружение разработки внутри WSL 2: установите Node.js, Yarn и NocoBase CLI в дистрибутив Linux в WSL и выполняйте связанные команды из терминала WSL.

WSL ближе к Linux-окружениям, где NocoBase обычно развёртывают. Это даёт несколько преимуществ:

- Установка зависимостей, сборка, запуск и устранение неполадок по логам ближе к реальному серверному рабочему процессу
- После включения интеграции WSL в Docker Desktop можно выполнять команды `docker` прямо внутри WSL
- Можно избежать лишних проблем из-за нативных путей Windows, прав на файлы, символических ссылок и сборки нативных зависимостей
- Удобнее для сценариев с ИИ агентами. Когда агент выполняет команды вроде `nb`, `yarn` или `docker`, он использует единые Linux-пути к файлам, синтаксис оболочки и среду выполнения, что упрощает диагностику

Если локальное окружение разработки на базе WSL ещё не готово, см. [Настройка локального окружения разработки на Windows с WSL](./windows-wsl.md).

Рекомендуемая конфигурация:

- Windows 10 / 11
- WSL 2
- Ubuntu LTS
- Node.js >= 22
- Yarn 1.x
- Docker Desktop, если планируете установку NocoBase через Docker

Обычно Node.js, Yarn и NocoBase CLI следует устанавливать внутри WSL. Если вы используете Docker Desktop, включите интеграцию WSL в Docker Desktop, чтобы WSL имел доступ к Docker.

Проверьте окружение:

```bash
node -v
yarn -v
docker version
```

:::tip Примечание

NocoBase также можно установить на Windows Server. WSL здесь рекомендуется для локальной разработки и настройки ИИ агентов на персональных компьютерах. Это не означает, что Windows Server нельзя использовать для развёртывания.

:::

## macOS

На macOS можно работать напрямую в локальном терминале.

Подготовьте:

- Node.js >= 22
- Yarn 1.x
- Docker Desktop, OrbStack или Colima, если планируете установку NocoBase через Docker

Проверьте окружение:

```bash
node -v
yarn -v
docker version
```

Если Docker не используется, команду `docker version` можно пропустить.

## Linux

Linux можно использовать напрямую как локальное окружение разработки. Рекомендуются Ubuntu, Debian или другие распространённые дистрибутивы.

Подготовьте:

- Node.js >= 22
- Yarn 1.x
- Docker Engine, если планируете установку NocoBase через Docker

Проверьте окружение:

```bash
node -v
yarn -v
docker version
```

Если Docker не используется, команду `docker version` можно пропустить.
