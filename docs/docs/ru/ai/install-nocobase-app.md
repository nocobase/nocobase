# Установка приложения NocoBase

## Предварительные требования

- Node.js >= 22
- Yarn 1.x
- Если планируете установку через Docker, убедитесь, что Docker уже запущен

## Шаг 1: Установка CLI

Сначала установите NocoBase CLI глобально:

```bash
npm install -g @nocobase/cli
nb --version
```

Если вы часто работаете в нескольких терминалах или хотите параллельно взаимодействовать с ИИ агентами, также рекомендуем один раз выполнить `nb session setup`. Это позволит каждой сессии хранить собственное `current env`, чтобы они реже мешали друг другу.

## Шаг 2: Инициализация приложения

Рекомендуем установку через мастер в интерфейсе:

```bash
nb init --ui
```

Если предпочитаете работу в терминале, можно также выполнить:

```bash
nb init
```

Если нужна инициализация в скриптах или CI, используйте неинтерактивный режим:

```bash
nb init --yes --env app1
```

:::tip Установка на удалённом сервере

Если вы выполняете `nb init --ui` на сервере, рекомендуем сначала изменить хост по умолчанию на IP этого сервера. Тогда мастер можно будет открыть из локального браузера.

```bash
nb config set default-ui-host <server-ip>
nb config set default-api-host <server-ip>
```

:::

## Шаг 3: Убедитесь, что приложение готово

После установки обычно стоит сначала проверить три вещи:

- Окружение успешно сохранено
- Приложение нормально запустилось
- Вы можете войти с учётной записью администратора

Распространённые команды:

```bash
nb env list
nb env info
nb app logs
```

При локальной установке по умолчанию обычно можно сразу открыть `http://localhost:13000` в браузере. После входа запустите новую сессию ИИ агента или перезапустите текущую, и ИИ сможет начать работу с этим приложением NocoBase.

Конфигурация CLI по умолчанию хранится в `~/.nocobase/`, поэтому ИИ агенты обычно могут получить к ней доступ из любого рабочего каталога.

Если это приложение позже будет доступно реальным пользователям, не рекомендуем долго использовать `IP + порт`. Следующий шаг обычно — разместить его за обратным прокси и включить HTTPS.
