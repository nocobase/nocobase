# Установить приложение NocoBase

## Предварительные требования

- Node.js >= 22
- Yarn 1.x
- Если Вы планируете установку через Docker, сначала убедитесь, что Docker уже запущен

## Шаг 1. Установить CLI

Сначала установите NocoBase CLI глобально:

```bash
npm install -g @nocobase/cli
nb --version
```

Если Вы часто работаете сразу в нескольких терминалах или хотите действовать параллельно с AI Agents, мы также рекомендуем один раз выполнить `nb session setup`. Тогда каждая сессия будет хранить собственный `current env`, и они будут меньше мешать друг другу.

## Шаг 2. Инициализировать приложение

По умолчанию мы рекомендуем сразу открыть визуальный мастер:

```bash
nb init --ui
```

![2026-06-11-20-14-32](https://static-docs.nocobase.com/2026-06-11-20-14-32.png)

В зависимости от выбранного пути setup шаги могут немного отличаться. Если идти по стандартному пути `Install a new app`, обычно Вы увидите эти шесть шагов:

1. `Getting started` - задать идентификатор `--env` и выбрать `Install a new app`
2. `App environment` - задать основные параметры приложения, место хранения и runtime-порт
3. `App source and version` - выбрать способ получения приложения, а также source и version
4. `Configure the database` - выбрать встроенную базу данных или пользовательскую базу данных
5. `Create an admin account` - настроить первую учётную запись администратора
6. `Connection & authentication` - указать URL доступа к приложению и выбрать способ аутентификации

Если Вам привычнее работать в терминале, можно также выполнить:

```bash
nb init
```

Если инициализация нужна в скриптах или CI, используйте неинтерактивный режим:

```bash
nb init --yes --env app1
```

:::tip Установка на удалённом сервере

Если Вы запускаете `nb init --ui` на сервере, мы рекомендуем сначала изменить стандартный host на IP-адрес этого сервера. Тогда Вы сможете открыть мастер из локального браузера.

```bash
nb config set default-ui-host <server-ip>
nb config set default-api-host <server-ip>
```

:::

## Шаг 3. Проверить, что приложение готово

После установки обычно стоит сначала проверить три вещи:

- Env успешно сохранён
- Приложение запустилось без ошибок
- Вы можете войти под учётной записью администратора

Чаще всего используются такие команды:

```bash
nb env list
nb env info
nb app logs
```

Для локальной установки с настройками по умолчанию обычно достаточно открыть `http://localhost:13000` в браузере. После входа откройте новую сессию AI Agent или перезапустите текущую — и ИИ сможет начать работу с этим приложением NocoBase.

Конфигурация CLI по умолчанию хранится в `~/.nocobase/`, поэтому AI Agents обычно могут обращаться к ней из любого рабочего каталога.

Если это приложение позже будет доступно реальным пользователям, мы не рекомендуем надолго оставлять прямой доступ через `IP + port`. Обычно следующим шагом становится настройка reverse proxy и включение HTTPS.
