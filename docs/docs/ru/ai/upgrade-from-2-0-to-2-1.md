# Как обновить NocoBase с 2.0 до 2.1

Обновление с NocoBase 2.0 до NocoBase 2.1 для самого приложения проходит гладко. Более заметные изменения касаются NocoBase CLI.

Где:

- В 2.0 и более ранних версиях команды CLI обычно начинаются с `yarn nocobase`
- В 2.1 и более поздних версиях команды CLI используют глобально установленный `nb`

Старые приложения не обязаны сразу переходить на `nb`. Если вам нужно только обновить уже стабильное приложение NocoBase 2.0 до 2.1, по умолчанию продолжайте использовать прежний способ установки и обновления. Для новых приложений рекомендуем новый CLI `nb`.

## Как мигрировать на CLI `nb`

Если в будущем вы хотите единообразно управлять приложениями через `nb`, на данный момент более надёжный подход — создать новое приложение и перенести в него данные старого.

Шаги миграции:

1. Создайте новое приложение CLI с помощью `nb init`
2. Перенесите базу данных старого приложения, `storage` и необходимые переменные окружения
3. После проверки корректной работы нового приложения переключите на него производственное окружение

Можно также подождать некоторое время. Возможность для `nb` взять на себя управление существующими локальными приложениями всё ещё в разработке.
