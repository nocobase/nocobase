# Обновление приложения NocoBase

:::tip Область применения

Данное руководство относится к приложениям, установленным с помощью `nb init`. Если ваше приложение было установлено прежним способом, сначала прочитайте [Как обновить NocoBase с 2.0 до 2.1](./upgrade-from-2-0-to-2-1.md).

:::

## Шаг 1: Подтвердите текущее окружение

```bash
nb env current
```

Если вы не уверены, какие окружения доступны, сначала выведите их список:

```bash
nb env list
```

Если текущее окружение — не то приложение, которое нужно обновить, переключитесь на целевое окружение:

```bash
nb env use <env-name>
```

## Шаг 2: Выполните обновление

:::warning Примечание

По умолчанию при обновлении исходный код приложения или образ Docker загружается заново.

Для окружений npm / Git каталог `source/` удаляется и загружается снова. Не размещайте в `source/` файлы, которые нужно сохранить.

Если вы уже вручную подготовили исходный код или образ Docker и не хотите, чтобы CLI загружал его снова, добавьте к команде `--skip-download`.

:::

Команда обновления по умолчанию:

```bash
nb app upgrade
```

Эта команда обычно выполняет следующие шаги:

1. Остановить текущее приложение
2. Загрузить и заменить сохранённый исходный код или образ
3. Синхронизировать коммерческие плагины
4. Обновить и запустить приложение
5. Обновить информацию о выполнении окружения

В скриптах, CI или сессиях ИИ агента явно указывайте `--force`:

```bash
nb app upgrade --force
```

Если обновляемое приложение — не текущее окружение, укажите окружение:

```bash
nb app upgrade --env app1 --yes --force
```

### Обновление до конкретной версии

Используйте `--version`, чтобы обновить до определённого канала версий:

```bash
nb app upgrade --version beta
```

Можно также указать точную версию:

```bash
nb app upgrade --version 2.1.0-beta.24
```

После успешного обновления CLI записывает целевую версию обратно в конфигурацию окружения, чтобы последующие обновления или процедуры восстановления могли её повторно использовать.

### Пропуск загрузки

Если вы уже обновили исходный код или образ Docker и хотите только выполнить обновление и запуск на основе текущего содержимого, добавьте `--skip-download`:

```bash
nb app upgrade --skip-download
```

Этот параметр пропускает загрузку исходного кода или образа, а также синхронизацию коммерческих плагинов. Используйте только когда целевая версия уже подготовлена вручную.

## Шаг 3: Проверьте результат

После обновления проверьте выполнение окружения и логи приложения:

```bash
nb env info
nb app logs
```

Затем откройте приложение и убедитесь, что учётная запись администратора может войти в систему. Если вы хотите, чтобы ИИ агент продолжил работу с этим приложением, запустите новую сессию ИИ агента или перезапустите текущую, чтобы он мог прочитать актуальную информацию об окружении.
