# OpenCode + NocoBase: открытый исходный код, бесплатно, без привязки к вендору

[OpenCode](https://github.com/opencode-ai/opencode) — терминальный ИИ агент с открытым исходным кодом: поддерживает более 75 моделей (Claude, GPT, Gemini, DeepSeek и др.), не привязан к одному вендору и позволяет свободно выбирать лучшую модель под ваши задачи. После подключения к NocoBase вы можете на естественном языке создавать таблицы данных, собирать страницы, настраивать рабочие процессы, сохраняя полный контроль над выбором моделей и затратами.

## Что такое OpenCode

OpenCode разработан Anomaly Innovations (более 140 тыс. звёзд на GitHub), позиционируется как «терминальный ИИ агент без привязки к вендору». Написан на Go, предоставляет удобный интерфейс TUI. Основные возможности:

- **Поддержка более 75 моделей** — Claude, GPT, Gemini, DeepSeek, локальные модели и другие — свободное переключение между ними
- **Нулевая привязка к вендору** — используйте свой API-ключ, платите по фактическому использованию, без дополнительных подписок
- **Мультиагентная архитектура** — встроенные роли агентов Build, Plan, Review, Debug и Docs
- **Приоритет конфиденциальности** — не хранит код и контекст, все данные остаются локально

OpenCode также поддерживает интеграцию с VS Code, JetBrains, Zed, Neovim и другими редакторами, а также отдельное настольное приложение.

## Почему выбрать OpenCode

Если вы выбираете, какой ИИ агент использовать для управления NocoBase, вот сценарии, где OpenCode подходит лучше всего:

- **Не хотите быть привязаны к одной модели** — сегодня Claude, завтра GPT, послезавтра DeepSeek — один инструмент для всего
- **Важен контроль затрат** — свой API-ключ и оплата по использованию; поддерживается использование существующих подписок ChatGPT Plus
- **Есть требования к конфиденциальности** — код и контекст не проходят через сторонние сервисы; поддерживаются локальные модели
- **Нравится высокая настраиваемость** — конфигурация YAML для пользовательского поведения агента под потребности команды

## Как происходит подключение

OpenCode работает с NocoBase следующим образом:

```
Вы (Terminal / VS Code / JetBrains / ...)
  |
  └─→ OpenCode
        |
        ├── NocoBase Skills (позволяет агенту понимать систему конфигурации NocoBase)
        |
        └── NocoBase CLI (выполняет создание, изменение, развёртывание и другие операции)
              |
              └─→ NocoBase Service (ваша бизнес-система)
```

- **NocoBase Skills** — пакеты предметных знаний, которые объясняют OpenCode, как управлять NocoBase
- **NocoBase CLI** — инструмент командной строки, который фактически выполняет моделирование данных, создание страниц и другие операции
- **NocoBase Service** — ваш работающий экземпляр NocoBase

## Предварительные требования

Перед началом убедитесь, что подготовлено следующее окружение:

- Установлен OpenCode ([руководство по установке](https://opencode.ai/docs/))
- Node.js >= 22 (для работы NocoBase CLI и Skills)
- Если у Вас уже есть экземпляр NocoBase — **из-за быстрого развития ИИ-возможностей минимальная требуемая версия — NocoBase 2.1.0. Рекомендуется обновиться до последней версии.**

## Быстрый старт

### Установка в один клик с помощью ИИ

Скопируйте приведённый ниже запрос в OpenCode — он автоматически выполнит установку NocoBase CLI, инициализацию и настройку окружения:

```
Помоги мне установить NocoBase CLI и завершить инициализацию: https://docs.nocobase.com/ai/ai-quick-start.md (пожалуйста, прочитай содержимое по ссылке напрямую)
```

### Ручная установка

```bash
npm install -g @nocobase/cli
nb init --ui
```

Браузер автоматически откроет страницу визуальной настройки и проведёт вас через установку NocoBase Skills, настройку базы данных и запуск приложения. Подробные шаги см. в [Быстрый старт](../quick-start.md).

После завершения установки выполните `nb env list`, чтобы подтвердить состояние окружения:

```bash
nb env list
```

Убедитесь, что в выводе отображаются настроенное окружение и его статус выполнения.
