# Руководство по подключению AI Agent

NocoBase полностью поддерживает совместную работу с AI Agent — Вы можете на естественном языке моделировать данные, строить страницы, проектировать workflow, настраивать права доступа и даже разрабатывать собственные плагины. Независимо от того, используете ли Вы Claude Code, Codex, Cursor или другой ИИ-инструмент, Вы можете работать с NocoBase через два базовых компонента:

1. **[NocoBase CLI](../api/cli/index.md)** — инструмент командной строки для инициализации, подключения и управления приложениями NocoBase в локальной рабочей области. Он помогает подготовить приложение NocoBase, сохраняет конфигурацию env CLI и предоставляет команды для повседневного управления — запуск, остановка, просмотр логов, обновление и очистка, — позволяя AI Agent подключаться и работать с NocoBase.

CLI поддерживает два распространённых способа инициализации:

- Подключение к уже существующему приложению NocoBase, чтобы AI Agent мог работать с ним напрямую.
- Установка нового приложения NocoBase через Docker, npm или Git с сохранением как env в CLI.

2. **[NocoBase Skills](https://github.com/nocobase/skills)** — пакеты доменных знаний, благодаря которым AI Agent понимает систему конфигурации и стандарты разработки NocoBase. Они охватывают моделирование данных, настройку интерфейса, управление workflow, настройку прав доступа, разработку плагинов и другие области. NocoBase CLI автоматически устанавливает Skills в процессе инициализации (`nb init`), отдельно ставить ничего не нужно.

После установки CLI и завершения инициализации AI Agent может сразу начать работу с Вашим приложением NocoBase.

## Предварительные требования

Перед началом убедитесь, что соблюдены следующие требования к окружению:

- Node.js >= 22
- Yarn 1.x
- Рекомендуется использовать AI Agent на базе ведущих больших языковых моделей, таких как Claude или GPT. Качество результата напрямую зависит от используемой модели — для лучшего опыта рекомендуется выбирать самые свежие флагманские модели.

## Установка нового NocoBase и подключение AI Agent

> Если у Вас уже есть запущенный экземпляр NocoBase, можно сразу перейти к следующему разделу — [Подключение AI Agent к существующему NocoBase](#подключение-ai-agent-к-существующему-nocobase).

Если у Вас ещё нет приложения NocoBase, его можно установить и инициализировать через CLI, а затем подключить к нему AI Agent.

<Tabs>
<Tab label="Ручная установка и подключение" value="manual-install">

#### Шаг 1. Установите CLI

```bash
npm install -g @nocobase/cli@beta
nb --version // проверка успешной установки
```

#### Шаг 2. Инициализация

```bash
nb init --ui
```

:::tip Расположение файлов конфигурации

Файлы конфигурации и данные приложения, создаваемые при инициализации, по умолчанию сохраняются в домашнем каталоге пользователя. Подробности см. в разделе [переменная окружения NB_CLI_ROOT](../api/cli/index.md#environment-variables).

:::

В браузере автоматически откроется визуальная страница настройки, которая в 5 шагов проведёт Вас через всю конфигурацию:

1. **Начало настройки** — задайте имя приложения (оно же — имя env в CLI), выберите «Полная установка»
2. **Создание приложения** — выберите способ установки (Docker / npm / Git), укажите порт и базовые параметры
3. **Загрузка файлов приложения** — выберите источник и версию, загрузите исходный код или образ NocoBase
4. **Настройка базы данных** — используйте встроенную базу или подключите внешнюю (PostgreSQL, MySQL, MariaDB)
5. **Создание учётной записи администратора** — задайте email и пароль администратора

![визуальный мастер nocobase cli](https://static-docs.nocobase.com/2026-04-29-15-55-19.png)

После установки откройте в браузере `http://localhost:13000` (порт по умолчанию) и войдите под только что созданной учётной записью администратора.

(Если Вам привычнее работать в командной строке, можно сразу запустить `nb init` для интерактивного мастера или выполнить `nb init --env=app -y`, чтобы установить с настройками по умолчанию.)

#### Шаг 3. Передайте управление AI Agent

Откройте новую сессию AI Agent или перезапустите существующую — после этого AI Agent сможет начать работу с NocoBase. Конфигурация CLI хранится в глобальном каталоге (по умолчанию `~/.nocobase/`), и AI Agent может обращаться к ней из любого каталога.

</Tab>
<Tab label="Установка и подключение через AI Agent" value="ai-install">

Скопируйте промпт ниже и отправьте его Вашему ИИ-помощнику (Claude Code, Codex, Cursor, Trae и т. д.). Он установит CLI и проведёт Вас через инициализацию нового NocoBase:

```
Помоги мне установить NocoBase CLI и выполнить инициализацию: https://docs.nocobase.com/cn/ai/ai-quick-start.md (открой ссылку и следуй её содержимому)
```

После завершения настройки перезапустите сессию Вашего AI Agent — и можно начинать работу с NocoBase.

</Tab>
</Tabs>

## Подключение AI Agent к существующему NocoBase

Если у Вас уже есть запущенный экземпляр NocoBase, можно сразу подключить к нему AI Agent. **Из-за быстрого развития ИИ-возможностей полноценный опыт сейчас доступен только в последней beta-версии. Минимальная требуемая версия >= 2.1.0-beta.20, настоятельно рекомендуется обновиться до самой свежей версии.**

<Tabs>
<Tab label="Ручное подключение" value="manual-connect">

#### Шаг 1. Установите CLI

```bash
npm install -g @nocobase/cli@beta
nb --version // проверка успешной установки
```

#### Шаг 2. Инициализация и подключение

```bash
nb init --ui
```

:::tip Расположение файлов конфигурации

Файлы конфигурации, создаваемые при инициализации, по умолчанию сохраняются в домашнем каталоге пользователя. Подробности см. в разделе [переменная окружения NB_CLI_ROOT](../api/cli/index.md#environment-variables).

:::

В браузере автоматически откроется визуальная страница настройки. На первом шаге «Начало настройки» выберите «Подключить существующее приложение», затем:

1. **Укажите адрес API** — введите адрес API Вашего существующего экземпляра NocoBase (например: `http://localhost:13000/api`)
2. **Выберите способ аутентификации** — выберите [способ аутентификации](../ai-builder/security) (для быстрого старта рекомендуется OAuth)

![визуальный мастер nocobase cli](https://static-docs.nocobase.com/2026-04-29-15-55-55.png)

Способ установки, базу данных и учётную запись администратора настраивать не нужно — мастер пропустит эти шаги.

#### Шаг 3. Передайте управление AI Agent

Откройте новую сессию AI Agent или перезапустите существующую — после этого AI Agent сможет начать работу с NocoBase. Конфигурация CLI хранится в глобальном каталоге (по умолчанию `~/.nocobase/`), и AI Agent может обращаться к ней из любого каталога.

</Tab>
<Tab label="Подключение через AI Agent" value="ai-connect">

Скопируйте промпт ниже и отправьте его Вашему ИИ-помощнику. Он установит CLI и проведёт Вас через настройку подключения к существующему экземпляру NocoBase. На первом шаге визуального мастера выберите «Подключить существующее приложение»:

```
Помоги мне установить NocoBase CLI и выполнить инициализацию: https://docs.nocobase.com/cn/ai/ai-quick-start.md (открой ссылку и следуй её содержимому)
```

После завершения настройки перезапустите сессию Вашего AI Agent — и можно начинать работу с NocoBase.

## Что дальше

### Управление окружениями приложений

Если Вам нужно управлять существующими экземплярами NocoBase (например, тестовым или продуктивным окружением) или переключаться между несколькими окружениями, см. [Управление окружениями](../ai-builder/env-bootstrap) — там описаны проверка окружения, добавление окружений, обновление и диагностика проблем.

### Начните создавать с ИИ

Опишите задачу на естественном языке, и ИИ выполнит моделирование данных, построение страниц и проектирование workflow — диалог вместо ручной настройки. См. [Быстрый старт конструктора ИИ](../ai-builder/index.md#заменим-ручную-настройку-диалогом).

### Продолжайте разрабатывать плагины с ИИ

Если Вы разрабатываете собственные плагины, ИИ тоже Вам поможет — от генерации каркаса до кода компонентов и бизнес-логики, охватывая весь процесс разработки. См. [Разработка плагинов с ИИ](../ai-dev/index.md#от-одной-фразы-до-готового-плагина).

### Использование AI-сотрудников

Работайте с AI-сотрудниками прямо в интерфейсе NocoBase — анализ данных, генерация отчётов, переводы, поддержка решений, не покидая бизнес-системы. См. [AI-сотрудники](../ai-employees/index.md).

### Справочник команд NocoBase CLI

Если у Вас уже есть запущенное приложение NocoBase или Вы успешно прошли инициализацию и хотите продолжить разработку, отладку или управление приложением, полное описание параметров всех команд `nb` см. в [Справочнике команд NocoBase CLI](../api/cli/index.md).
