# Кластерный режим

## Введение

Начиная с версии v1.6.0, NocoBase поддерживает запуск приложений в кластерном режиме. При работе приложения в кластерном режиме вы можете повысить его производительность при обработке параллельных запросов, используя несколько экземпляров и многоядерный режим.

## Архитектура системы

![20251031221530](https://static-docs.nocobase.com/20251031221530.png)

*   **Кластер приложений**: Кластер, состоящий из нескольких экземпляров приложения NocoBase. Его можно развернуть на нескольких серверах или запустить как несколько процессов в многоядерном режиме на одном сервере.
*   **База данных**: Хранит данные приложения. Это может быть одноузловая или распределенная база данных.
*   **Общее хранилище**: Используется для хранения файлов и данных приложения, поддерживая доступ для чтения и записи из нескольких экземпляров.
*   **Промежуточное ПО (Middleware)**: Включает такие компоненты, как кэш, сигналы синхронизации, очередь сообщений и распределенные блокировки, для поддержки связи и координации внутри кластера приложений.
*   **Балансировщик нагрузки**: Отвечает за распределение клиентских запросов между различными экземплярами в кластере приложений, а также за выполнение проверок работоспособности и аварийное переключение.
