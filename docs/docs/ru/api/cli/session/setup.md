# nb session setup

Устанавливает интеграцию сессии для `NB_SESSION_ID`.

Эта команда определяет текущую оболочку или использует оболочку, переданную через `--shell`, и записывает соответствующий файл инициализации, чтобы новые сессии оболочки автоматически получали `NB_SESSION_ID`.

Если на машине обнаружена конфигурация opencode, команда также записывает соответствующую интеграцию плагина, чтобы среда выполнения агента могла внедрять собственный `NB_SESSION_ID`.

## Использование

```bash
nb session setup [flags]
```

## Параметры

| Параметр | Тип | Описание |
| --- | --- | --- |
| `--shell` | string | Целевая оболочка. Поддерживаемые значения: `bash`, `zsh`, `fish`, `powershell`, `cmd` |

## Примечания

В большинстве случаев эту команду нужно выполнить только один раз.

После этого откройте новую сессию оболочки или перезагрузите профиль, чтобы `NB_SESSION_ID` инициализировался автоматически.

В средах выполнения агентов, таких как Codex, если уже есть переменная контекста вроде `CODEX_THREAD_ID`, CLI сначала использует именно её значение.

## Примеры

```bash
nb session setup
nb session setup --shell zsh
nb session setup --shell powershell
```
