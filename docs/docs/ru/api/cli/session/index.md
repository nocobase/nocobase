# nb session

Управление режимом сессии для `NB_SESSION_ID`.

После включения режима сессии команды `nb env use` и `nb env current` сначала используют контекст текущей оболочки или среды выполнения агента, вместо того чтобы напрямую делить одно глобальное текущее окружение.

## Использование

```bash
nb session <command>
```

## Подкоманды

| Команда | Описание |
| --- | --- |
| [`nb session setup`](./setup.md) | Установить интеграцию оболочки или среды выполнения для `NB_SESSION_ID` |
| [`nb session id`](./id.md) | Показать текущий действующий идентификатор сессии |
| [`nb session remove`](./remove.md) | Удалить интеграцию оболочки или среды выполнения для `NB_SESSION_ID` |

## Когда это нужно

По умолчанию рекомендуется один раз выполнить `nb session setup`, когда вы начинаете пользоваться CLI. Тогда:

- терминал 1 может использовать `env1`
- терминал 2 может одновременно использовать `env2`
- среда выполнения агента тоже может хранить своё текущее окружение

Без режима сессии разные сессии в итоге используют один и тот же глобальный `last env` как запасной вариант, поэтому параллельная работа чаще влияет друг на друга.
