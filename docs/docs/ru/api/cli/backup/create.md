# nb backup create

Создаёт резервную копию через выбранный env и скачивает файл резервной копии локально. Если `--output` опущен, CLI сохранит файл в текущем рабочем каталоге и использует имя файла резервной копии, возвращённое удалённой стороной — обычно это `backup_*.nbdata`.

## Использование

```bash
nb backup create [flags]
```

## Параметры

| Параметр              | Тип     | Описание                                                                                                                                                                                      |
| --------------------- | ------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--env`, `-e`         | string  | Имя CLI env, для которого нужно создать резервную копию; если не указано, используется текущий env                                                                                            |
| `--yes`, `-y`         | boolean | Пропустить интерактивное подтверждение, если env, явно указанный через `--env`, отличается от текущего env                                                                                    |
| `--output`, `-o`      | string  | Целевой путь для загрузки. Если не указан, сохраняется в текущий каталог; если указывает на существующий каталог, имя файла резервной копии с удалённой стороны будет добавлено автоматически |
| `--json-output`, `-j` | boolean | Вывести итоговый результат в формате JSON, включая поля `env`, `name` и `output`                                                                                                              |

## Примеры

```bash
nb backup create
nb backup create --output ./backups
nb backup create --output ./backups/base.nbdata
nb backup create --env e2e --output ./backups --yes
nb backup create --env e2e --json-output
```

## Описание

CLI проверяет, совпадает ли `--env` с текущим env, только если вы явно передали `--env`. Если явно указан другой env, в интерактивном терминале сначала будет запрошено подтверждение; в неинтерактивном терминале или в сценарии AI agent вам нужно самостоятельно явно добавить `--yes` либо сначала выполнить `nb env use <name>`, а затем повторить попытку.

Процесс создания делится на два этапа: сначала вызывается backup API целевого env для создания удалённой резервной копии, затем каждые 2 секунды выполняется опрос статуса; после завершения резервного копирования файл скачивается локально. Если спустя 600 секунд удалённая сторона всё ещё возвращает `inProgress: true`, команда завершится по тайм-ауту.

`--output` может быть как путём к файлу, так и путём к существующему каталогу. CLI распознаёт как каталог только уже существующий путь; если путь не существует, он будет обработан как путь к целевому файлу.

После передачи `--json-output` CLI больше не выводит текст прогресса, а сразу печатает итоговый JSON-результат. Этот режим лучше подходит для дальнейшего использования в скриптах и цепочках agent.
