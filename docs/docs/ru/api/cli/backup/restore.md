# nb backup restore

Восстанавливает локальный файл резервной копии в целевой env. Обычно здесь используется файл `*.nbdata`. Восстановление перезапишет данные целевого приложения, поэтому CLI по умолчанию запрашивает дополнительное подтверждение.

## Использование

```bash
nb backup restore --file <path> [flags]
```

## Параметры

| Параметр       | Тип     | Описание                                                                                                           |
| -------------- | ------- | ------------------------------------------------------------------------------------------------------------------ |
| `--env`, `-e`  | string  | Имя CLI env, в который нужно восстановить данные; если не указано, используется текущий env                        |
| `--yes`, `-y`  | boolean | Пропускает интерактивное подтверждение, если env, явно указанный через `--env`, отличается от текущего env         |
| `--file`, `-f` | string  | Путь к локальному файлу резервной копии; обязательно                                                               |
| `--force`      | boolean | Подтверждает перезапись данных приложения; должен быть явно указан в неинтерактивных терминалах и сессиях AI agent |

## Примеры

```bash
nb backup restore --file ./backups/base.nbdata --force
nb backup restore --env e2e --file ./backups/base.nbdata --yes --force
```

## Описание

CLI проверяет, совпадает ли `--env` с текущим env, только если вы явно передали `--env`. Если явно указан другой env, в интерактивном терминале сначала будет запрошено подтверждение; в неинтерактивных терминалах или в сценариях AI agent вам нужно явно добавить `--yes` самостоятельно либо сначала выполнить `nb env use <name>`, а затем повторить попытку.

Перед выполнением CLI сначала проверяет, существует ли путь, указанный в `--file`, и подтверждает, что это обычный файл. Если путь не существует или указывает на каталог, команда немедленно завершится ошибкой.

Если `--force` не передан, в интерактивном терминале появится еще одно подтверждение с явным указанием, что это восстановление перезапишет данные приложения. В неинтерактивных терминалах и сессиях AI agent при отсутствии `--force` CLI сразу откажется выполнять команду и выдаст подсказку для повторного запуска, которую можно сразу скопировать. Если при этом операция выполняется между разными env, обычно нужно передать и `--yes`, и `--force`.

После успешной загрузки CLI продолжит ждать, пока целевое приложение снова не пройдет `__health_check`. Иными словами, к моменту успешного завершения команды приложение обычно уже восстановлено до доступного состояния.
