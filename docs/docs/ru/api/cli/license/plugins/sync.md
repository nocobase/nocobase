# nb license plugins sync

Синхронизирует коммерческие плагины, разрешённые текущей лицензией.

## Использование

```bash
nb license plugins sync [flags]
```

## Параметры

| Параметр | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI-окружения; если не указано, используется текущее окружение |
| `--yes`, `-y` | boolean | Если явно переданный `--env` указывает на окружение, отличное от текущего, пропускает интерактивное подтверждение |
| `--dry-run` | boolean | Предварительный просмотр изменений без установки, обновления или удаления плагинов |
| `--version` | string | Версия реестра или тег дистрибуции для синхронизации; по умолчанию используется текущая версия приложения |
| `--skip-if-no-license` | boolean | Пропустить без ошибки, если у текущего окружения ещё нет сохранённого ключа лицензии |
| `--verbose` | boolean | Показывать подробные логи синхронизации по каждому плагину |
| `--json` | boolean | Вывод JSON |

## Примеры

```bash
nb license plugins sync
nb license plugins sync --env app1
nb license plugins sync --env app1 --yes
nb license plugins sync --env app1 --dry-run
nb license plugins sync --env app1 --skip-if-no-license
nb license plugins sync --env app1 --json
```

## Примечания

Если `--version` не указан, CLI автоматически определяет текущую версию приложения и использует её, чтобы решить, какую версию реестра коммерческих плагинов нужно загрузить.

`--skip-if-no-license` игнорирует только один случай: у текущего окружения ещё нет сохранённого ключа лицензии. Другие ошибки, например отсутствие учётных данных реестра в ключе, сбой входа в реестр или ошибка загрузки плагинов, по-прежнему выводятся как обычно.

Если вы явно передаёте `--env`, и оно отличается от текущего окружения, CLI сначала запросит подтверждение. В неинтерактивных терминалах или сессиях ИИ агента добавьте `--yes` самостоятельно либо сначала выполните `nb env use <name>`, а затем повторите попытку.
