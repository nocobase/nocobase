# NocoBase CLI

## Описание

NocoBase CLI (`nb`) — это входная точка командной строки NocoBase, предназначенная для инициализации, подключения и управления приложениями NocoBase в локальной рабочей области.

Поддерживаются два распространённых пути инициализации:

- Подключение к существующему приложению NocoBase и сохранение его как CLI env
- Установка нового приложения NocoBase через Docker, npm или Git и последующее сохранение его как CLI env

При создании нового локального приложения [`nb init`](./init.md) также может установить или обновить NocoBase AI coding skills. Если этот шаг нужно пропустить, используйте `--skip-skills`.

## Использование

```bash
nb [command]
```

Сама корневая команда в основном предназначена для отображения справки и передачи вызовов в группы команд или отдельные команды.

## Группы команд (Topics)

В выводе `nb --help` отображаются следующие группы команд:

| Группа команд | Описание |
| --- | --- |
| [`nb api`](./api/index.md) | Вызов API NocoBase через CLI. |
| [`nb app`](./app/index.md) | Управление состоянием приложения: запуск, остановка, перезапуск, логи и обновление. |
| [`nb db`](./db/index.md) | Управление встроенной базой данных выбранного env. |
| [`nb env`](./env/index.md) | Управление окружениями проектов NocoBase, состоянием, деталями и runtime-командами. |
| [`nb plugin`](./plugin/index.md) | Управление плагинами выбранного env NocoBase. |
| [`nb scaffold`](./scaffold/index.md) | Генерация шаблона разработки плагинов NocoBase. |
| [`nb self`](./self/index.md) | Проверка или обновление самого NocoBase CLI. |
| [`nb skills`](./skills/index.md) | Проверка или синхронизация NocoBase AI coding skills в текущей рабочей области. |
| [`nb source`](./source/index.md) | Управление локальными исходниками: загрузка, разработка, сборка и тестирование. |

## Команды (Commands)

Отдельные команды, доступные напрямую через корневую команду:

| Команда | Описание |
| --- | --- |
| [`nb init`](./init.md) | Инициализация NocoBase, чтобы coding agent мог подключаться и работать. |

## Просмотр справки

Просмотр справки по корневой команде:

```bash
nb --help
```

Просмотр справки по конкретной команде или группе команд:

```bash
nb init --help
nb app --help
nb api resource --help
```

## Примеры

Интерактивная инициализация:

```bash
nb init
```

Инициализация через форму в браузере:

```bash
nb init --ui
```

Неинтерактивное создание приложения Docker:

```bash
nb init --env app1 --yes --source docker --version alpha
```

Подключение к существующему приложению:

```bash
nb env add app1 --api-base-url http://localhost:13000/api
```

Запуск приложения и обновление runtime-команд:

```bash
nb app start -e app1
nb env update app1
```

Вызов API:

```bash
nb api resource list --resource users -e app1
```

## Переменные окружения

Следующие переменные окружения влияют на поведение CLI:

| Переменная | Описание |
| --- | --- |
| `NB_CLI_ROOT` | Корневой каталог, в котором CLI хранит конфигурацию `.nocobase` и файлы локального приложения. По умолчанию — домашний каталог текущего пользователя. |
| `NB_LOCALE` | Язык подсказок CLI и локального UI инициализации, поддерживаются `en-US` и `zh-CN`. |

Пример:

```bash
export NB_CLI_ROOT=/your/workspace
export NB_LOCALE=zh-CN
```

## Файл конфигурации

Файл конфигурации по умолчанию:

```text
~/.nocobase/config.json
```

После установки `NB_CLI_ROOT=/your/workspace` путь к файлу конфигурации станет:

```text
/your/workspace/.nocobase/config.json
```

CLI также совместим с чтением старой конфигурации project в текущем рабочем каталоге.

Кэш runtime-команд сохраняется в:

```text
.nocobase/versions/<hash>/commands.json
```

Этот файл создаётся или обновляется командой [`nb env update`](./env/update.md) и используется для кэширования runtime-команд, синхронизированных с целевым приложением.
