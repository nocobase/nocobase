# nb app destroy

Уничтожает выбранный env, удаляя управляемые runtime-ресурсы, данные storage и сохранённую конфигурацию CLI env.

Для локальных и Docker env команда сначала удаляет на этой машине управляемые runtime-ресурсы приложения, при наличии также удаляет runtime встроенной базы данных, затем удаляет данные storage и сохранённую конфигурацию CLI env. Для HTTP и SSH env она удаляет только сохранённую конфигурацию CLI env и не затрагивает внешние сервисы.

Для скачанных локальных npm/Git env команда также удаляет локальные файлы приложения, которыми управляет CLI. Для пользовательских локальных путей app локальные исходные файлы сохраняются; удаляются только управляемые runtime-ресурсы, данные storage и сохранённая конфигурация env.

По умолчанию команда запрашивает подтверждение. В неинтерактивном режиме нужно явно передать `--env` и `--force`.

## Использование

```bash
nb app destroy [flags]
```

## Параметры

| Параметр | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI env, который нужно уничтожить; в интерактивном режиме при отсутствии параметра по умолчанию используется текущий env |
| `--force`, `-f` | boolean | Пропустить подтверждение и немедленно уничтожить выбранный env; обязателен в неинтерактивном режиме |
| `--verbose` | boolean | Показать сырой вывод команд уничтожения |

## Примеры

```bash
nb app destroy --env app1
nb app destroy --env app1 --force
```
