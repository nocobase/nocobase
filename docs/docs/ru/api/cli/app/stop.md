# nb app stop

Останавливает приложение NocoBase для указанного окружения. При установке через npm/Git команда останавливает локальный процесс приложения, а при установке через Docker очищает сохранённый контейнер приложения.

Если вы передадите `--with-db` и это окружение использует встроенную базу данных, управляемую CLI, команда также очистит контейнер базы данных. Если это окружение использует внешнюю базу данных, ресурсы базы данных затронуты не будут.

## Использование

```bash
nb app stop [flags]
```

## Параметры

| Параметр      | Тип     | Описание                                                                                                    |
| ------------- | ------- | ----------------------------------------------------------------------------------------------------------- |
| `--env`, `-e` | string  | Имя CLI-окружения, которое нужно остановить; если не указано, используется текущее окружение                |
| `--yes`, `-y` | boolean | Пропустить интерактивное подтверждение, когда окружение, явно указанное через `--env`, отличается от текущего |
| `--with-db`   | boolean | Также очищает контейнер базы данных, если существует встроенная база данных, управляемая CLI                |
| `--verbose`   | boolean | Показывает вывод низкоуровневых локальных или Docker-команд                                                 |

## Примеры

```bash
nb app stop
nb app stop --env local
nb app stop --env local --with-db
nb app stop --env local --verbose
nb app stop --env local-docker
```

## Примечания

CLI проверяет, совпадает ли указанное окружение с текущим, только если вы явно передаёте `--env`. Если явно указано другое окружение, в интерактивном терминале сначала будет запрошено подтверждение; в неинтерактивном терминале или в сессии ИИ-агента вам нужно самостоятельно явно добавить `--yes` либо сначала выполнить `nb env use <name>`, а затем повторить попытку.

`--with-db` влияет только на контейнеры встроенной базы данных, управляемые CLI. Обычно, если вы хотите просто остановить само приложение, этот параметр не нужен; добавляйте его только если вы также хотите остановить среду выполнения встроенной базы данных на текущей машине.

Эта команда может работать только с локальной или Docker-средой выполнения на текущей машине. Если окружение представляет собой только подключение к HTTP API или является зарезервированным SSH-окружением, `nb app stop` не сможет удалённо остановить его за вас.
