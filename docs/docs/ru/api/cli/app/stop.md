# nb app stop

Останавливает приложение NocoBase для указанного env. При установке через npm/Git команда останавливает локальный процесс приложения, а при установке через Docker очищает сохраненный контейнер приложения.

Если вы передадите `--with-db` и этот env использует встроенную базу данных, управляемую CLI, команда также очистит контейнер базы данных. Если этот env использует внешнюю базу данных, ресурсы базы данных затронуты не будут.

## Использование

```bash
nb app stop [flags]
```

## Параметры

| Параметр      | Тип     | Описание                                                                                                    |
| ------------- | ------- | ----------------------------------------------------------------------------------------------------------- |
| `--env`, `-e` | string  | Имя CLI env, который нужно остановить; если не указано, используется текущий env                            |
| `--yes`, `-y` | boolean | Пропустить интерактивное подтверждение, когда env, явно указанный через `--env`, отличается от текущего env |
| `--with-db`   | boolean | Также очищает контейнер базы данных, если существует встроенная база данных, управляемая CLI                |
| `--verbose`   | boolean | Показывает вывод базовых локальных или Docker-команд                                                        |

## Примеры

```bash
nb app stop
nb app stop --env local
nb app stop --env local --with-db
nb app stop --env local --verbose
nb app stop --env local-docker
```

## Описание

CLI проверяет, совпадает ли указанный env с текущим env, только если вы явно передаете `--env`. Если явно указан другой env, в интерактивном терминале сначала будет запрошено подтверждение; в неинтерактивном терминале или в сценарии AI-агента вам нужно самостоятельно явно добавить `--yes` либо сначала выполнить `nb env use <name>`, а затем повторить попытку.

`--with-db` влияет только на контейнеры встроенной базы данных, управляемые CLI. Обычно, если вы хотите просто остановить само приложение, этот параметр не нужен; добавляйте его только если вы также хотите остановить runtime встроенной базы данных на текущей машине.

Эта команда может работать только с локальным или Docker runtime на текущей машине. Если env представляет собой только подключение к HTTP API или является зарезервированным SSH env, `nb app stop` не сможет удаленно остановить его за вас.
