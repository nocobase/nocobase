# nb app down

Остановка и очистка локальных runtime-ресурсов указанного env. По умолчанию данные storage и конфигурация env сохраняются; для удаления всего содержимого необходимо явно передать `--all --force`.

## Использование

```bash
nb app down [flags]
```

## Параметры

| Параметр | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI env для очистки; если не указано, используется текущий env |
| `--all` | boolean | Удалить всё содержимое этого env, включая данные storage и сохранённую конфигурацию env |
| `--yes`, `-y` | boolean | Если явно переданный `--env` указывает на env, отличающуюся от текущей env, пропускает интерактивное подтверждение |
| `--force`, `-f` | boolean | Принудительно выполняет разрушительную очистку, например `--all` или другую высокорискованную очистку в неинтерактивном режиме |
| `--verbose` | boolean | Показать вывод низкоуровневых команд остановки и очистки |

## Примеры

```bash
nb app down --env app1
nb app down --env app1 --all --force
nb app down --env app1 --force
```

`--yes` лишь пропускает интерактивное подтверждение, когда явно переданный `--env` указывает на env, отличающуюся от текущей env. `--force` нужен именно для принудительной разрушительной очистки, например с `--all` или при другой высокорискованной очистке в неинтерактивном режиме.
