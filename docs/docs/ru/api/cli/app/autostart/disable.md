# nb app autostart disable

Отключает флаг автозапуска приложения для одного окружения.

После отключения это окружение больше не будет участвовать в `nb app autostart run`. Эта команда сама по себе не останавливает уже работающее приложение. Если вы также хотите остановить текущую среду выполнения, отдельно используйте `nb app stop`.

## Использование

```bash
nb app autostart disable [flags]
```

## Флаги

| Флаг | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI-окружения, которое будет удалено из автозапуска; если флаг не указан, используется текущее окружение |
| `--yes`, `-y` | boolean | Пропускает интерактивное подтверждение, когда явный `--env` указывает на окружение, отличное от текущего |

## Примеры

```bash
nb app autostart disable
nb app autostart disable --env app1
nb app autostart disable --env app1 --yes
```

## Примечания

Эта команда меняет только сохранённый флаг автозапуска. Она не останавливает приложение напрямую. Если для окружения автозапуск и так не был включён, команда просто оставит его в отключённом состоянии.

Как и в случае с `enable`, CLI запрашивает подтверждение операции с другим окружением только при явной передаче `--env`. В неинтерактивных терминалах или сессиях ИИ-агента при необходимости добавляйте `--yes` самостоятельно.
