# nb app autostart run

Запускает все окружения, для которых включён автозапуск приложения.

Эта команда обычно вызывается после загрузки хост-системы через ваш собственный механизм запуска. CLI читает все сохранённые окружения, отбирает те, у которых включён автозапуск, и затем пытается запустить их по одному.

## Использование

```bash
nb app autostart run [flags]
```

## Флаги

| Флаг | Тип | Описание |
| --- | --- | --- |
| `--verbose` | boolean | Показывает исходный вывод запуска от нижележащих локальных или Docker-команд |

## Примеры

```bash
nb app autostart run
nb app autostart run --verbose
```

## Примечания

Если ни у одного окружения не включён автозапуск, команда выведет `No environments have app autostart enabled.`.

Во время выполнения CLI обрабатывает каждое включённое окружение по очереди:

- окружения, которые можно запустить, отображаются как `started`
- окружения, которые не должны автоматически запускаться на текущей машине, отображаются как `skipped`
- окружения, которые завершаются ошибкой при запуске, отображаются как `failed`

Внутри команда вызывает `nb app start --env <name> --yes`. Если вы передали `--verbose`, этот флаг также будет передан в нижележащий процесс запуска.

Если хотя бы один результат имеет статус `failed`, команда завершится с ошибкой и выведет `Some app autostart envs failed to start.`. Это делает сбои заметными в `systemd`, CI и других механизмах запуска хоста.
