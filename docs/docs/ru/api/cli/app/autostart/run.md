# nb app autostart run

Запускает все env, для которых включен автозапуск приложения.

Эта команда обычно вызывается после загрузки хост-системы через ваш собственный механизм запуска. CLI читает все сохраненные env, отбирает те, у которых включен автозапуск, и затем пытается запустить их по одному.

## Использование

```bash
nb app autostart run [flags]
```

## Флаги

| Флаг | Тип | Описание |
| --- | --- | --- |
| `--verbose` | boolean | Показывает необработанный вывод запуска от нижележащих local или Docker-команд |

## Примеры

```bash
nb app autostart run
nb app autostart run --verbose
```

## Примечания

Если ни у одного env не включен автозапуск, команда выведет `No environments have app autostart enabled.`.

Во время выполнения CLI обрабатывает каждый включенный env по очереди:

- env, которые можно запустить, отображаются как `started`
- env, которые не должны автоматически запускаться на текущей машине, отображаются как `skipped`
- env, которые завершаются ошибкой при запуске, отображаются как `failed`

Внутри команда вызывает `nb app start --env <name> --yes`. Если вы передали `--verbose`, этот флаг также будет передан в нижележащий процесс запуска.

Если хотя бы один результат имеет статус `failed`, команда завершится с ошибкой и выведет `Some app autostart envs failed to start.`. Это делает сбои заметными в `systemd`, CI и других механизмах запуска хоста.
