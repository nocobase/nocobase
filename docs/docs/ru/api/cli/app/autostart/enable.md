# nb app autostart enable

Включает флаг автозапуска приложения для одного окружения.

Этот флаг применяется только к окружениям `local` или `docker`, чья среда выполнения управляется CLI на текущей машине. Он не запускает приложение немедленно. Вместо этого окружение добавляется в набор, который позже может быть запущен через `nb app autostart run`.

## Использование

```bash
nb app autostart enable [flags]
```

## Флаги

| Флаг | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI-окружения, которое будет добавлено в автозапуск; если флаг не указан, используется текущее окружение |
| `--yes`, `-y` | boolean | Пропускает интерактивное подтверждение, когда явный `--env` указывает на окружение, отличное от текущего |

## Примеры

```bash
nb app autostart enable
nb app autostart enable --env app1
nb app autostart enable --env app1 --yes
```

## Примечания

CLI проверяет, отличается ли целевое окружение от текущего, только если вы явно передали `--env`. В интерактивных терминалах сначала будет подтверждение. В неинтерактивных терминалах или сессиях ИИ-агента вам нужно самостоятельно добавить `--yes` или сначала переключиться на нужное окружение через `nb env use <name>`.

Если целевое окружение не является управляемой CLI средой выполнения `local` или `docker` на текущей машине, команда завершится ошибкой и не сохранит флаг автозапуска.
