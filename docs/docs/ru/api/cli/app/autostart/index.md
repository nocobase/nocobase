# nb app autostart

Управление настройками автозапуска приложения.

Эта группа команд охватывает два типа задач:

- включение или отключение флага автозапуска для одного окружения
- запуск всех окружений, для которых автозапуск уже включён

`nb app autostart` применяется только к окружениям со средой выполнения, управляемой CLI на текущей машине, то есть `local` и `docker`. Если окружение представляет собой только удалённое API-подключение или это не управляемая CLI среда выполнения приложения, которую можно запустить на этой машине, оно не может участвовать в этом процессе автозапуска.

## Использование

```bash
nb app autostart <command>
```

## Подкоманды

| Команда | Описание |
| --- | --- |
| [`nb app autostart enable`](./enable.md) | Включает флаг автозапуска для одного окружения |
| [`nb app autostart disable`](./disable.md) | Отключает флаг автозапуска для одного окружения |
| [`nb app autostart list`](./list.md) | Показывает состояние автозапуска всех окружений |
| [`nb app autostart run`](./run.md) | Запускает все окружения с включённым автозапуском |

## Примечания

`nb app autostart enable` только помечает окружение как разрешённое для автоматического запуска. Сам по себе этот флаг не встраивает окружение в системный процесс загрузки. В реальной производственной конфигурации обычно всё равно нужно вызывать `nb app autostart run` из собственного механизма запуска хоста, например через `systemd`, стартовый скрипт контейнерной платформы или другой уже используемый процесс загрузки.

Кроме того, `nb app autostart run` проверяет каждое включённое окружение по очереди. Окружения, которые можно запустить, продолжают работу через `nb app start --env <name> --yes`. Окружения, которые не должны автоматически запускаться на текущей машине, появятся в таблице результатов как `skipped` или `failed`.

## Примеры

```bash
nb app autostart enable
nb app autostart enable --env app1 --yes
nb app autostart list
nb app autostart run
nb app autostart run --verbose
```
