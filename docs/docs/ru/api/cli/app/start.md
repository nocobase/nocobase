# nb app start

Запуск приложения NocoBase выбранного окружения. Если это применимо, CLI сначала синхронизирует коммерческие плагины, разрешённые текущей лицензией. Затем установки npm/Git автоматически выполняют необходимую подготовку установки или обновления перед запуском локальных команд приложения, а установки Docker пересоздают контейнер приложения из сохранённой конфигурации окружения.

## Использование

```bash
nb app start [flags]
```

## Параметры

| Параметр | Тип | Описание |
| --- | --- | --- |
| `--env`, `-e` | string | Имя CLI-окружения для запуска; если не указано, используется текущее окружение |
| `--yes`, `-y` | boolean | Если явно переданный `--env` указывает на окружение, отличное от текущего, пропускает интерактивное подтверждение |
| `--verbose` | boolean | Показать вывод низкоуровневых локальных или Docker-команд |

## Примеры

```bash
nb app start
nb app start --env local
nb app start --env local --verbose
nb app start --env local-docker
```

Если вы явно передаёте `--env`, и оно отличается от текущего окружения, CLI сначала запросит подтверждение. В неинтерактивных терминалах или сессиях ИИ-агента добавьте `--yes` самостоятельно либо сначала выполните `nb env use <name>`, а затем повторите попытку.

По умолчанию, если это применимо, CLI сначала выполняет `nb license plugins sync --skip-if-no-license`, чтобы синхронизировать коммерческие плагины, разрешённые текущей лицензией. Затем локальные окружения автоматически выполняют необходимую подготовку установки или обновления перед запуском, а Docker-окружения пересоздают контейнер приложения из сохранённой конфигурации окружения. Каждый раз, когда CLI нужно дождаться готовности приложения, она проверяет `__health_check`: сначала выводит одну строку ожидания, а затем одну строку прогресса каждые 10 секунд, пока приложение не станет доступно или не истечёт время ожидания.
