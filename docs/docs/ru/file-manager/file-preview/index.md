# Предпросмотр файлов

В интерфейсах, содержащих поля файлов (включая поля вложений), вы можете предварительно просматривать файлы, щёлкнув по миниатюре файла или значку. Встроенная функция предпросмотра поддерживает различные типы файлов, включая изображения, PDF и большинство типов файлов, которые браузеры поддерживают нативно.

![20251129232307](https://static-docs.nocobase.com/20251129232307.png)

Для типов файлов, которые не поддерживают нативный предпросмотр, вы можете включить функциональность предпросмотра, установив или расширив соответствующие плагины предпросмотра файлов. Например, после установки плагина предпросмотра Office-файлов вы сможете просматривать файлы Word, Excel и PowerPoint.

В настоящее время NocoBase предоставляет следующие плагины предпросмотра файлов:

* [Плагин предпросмотра Office-файлов](../file-preview/ms-office.md)

## Предпросмотр PDF при использовании внешнего хранилища

Предпросмотр PDF использует PDF.js для отрисовки файлов в браузере. Браузер должен сначала прочитать содержимое PDF-файла, а затем передать его в PDF.js для отрисовки. Поэтому, если файлы хранятся во внешнем хранилище, таком как OSS, S3, COS или CDN, и домен доступа к файлу отличается от домена сайта NocoBase, внешнее хранилище должно разрешать сайту NocoBase чтение файлов между источниками.

Если CORS не настроен, скачивание PDF может продолжать работать нормально, но предпросмотр может завершиться ошибкой загрузки файла.

Конфигурация CORS для внешнего хранилища или CDN должна включать:

```http
Access-Control-Allow-Origin: https://your-nocobase-domain
Access-Control-Allow-Methods: GET, HEAD
Access-Control-Allow-Headers: *
Access-Control-Expose-Headers: Content-Length, Content-Range, Accept-Ranges, Content-Disposition, Content-Type
```

`Access-Control-Allow-Origin` должен быть настроен на фактический домен, используемый для доступа к NocoBase. Не рекомендуется долго использовать `*` для непубличных файлов, так как это расширяет круг сайтов, которые могут читать файлы.
