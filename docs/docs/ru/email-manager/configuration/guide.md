# Процесс настройки

## Обзор
После включения плагина Email администраторам необходимо сначала выполнить соответствующую настройку, прежде чем обычные пользователи смогут подключить свои почтовые аккаунты к NocoBase. (В настоящее время поддерживается только авторизованный вход для аккаунтов Outlook и Gmail; прямое подключение аккаунтов Microsoft и Google пока не доступно).

Ключевым моментом настройки являются параметры аутентификации для вызовов API поставщика почтовых услуг. Администраторам необходимо выполнить следующие шаги, чтобы обеспечить корректную работу плагина:

1.  **Получение данных для аутентификации от поставщика услуг**
    -   Войдите в консоль разработчика поставщика почтовых услуг (например, Google Cloud Console или Microsoft Azure Portal).
    -   Создайте новое приложение или проект и включите службу API Gmail или Outlook.
    -   Получите соответствующие Client ID и Client Secret.
    -   Настройте URI перенаправления (Redirect URI), убедившись, что он совпадает с адресом обратного вызова плагина NocoBase.

2.  **Настройка поставщика почтовых услуг**
    -   Перейдите на страницу настройки плагина Email.
    -   Предоставьте необходимую информацию для аутентификации API, включая Client ID, Client Secret и другие данные, чтобы обеспечить корректную авторизацию с поставщиком почтовых услуг.

3.  **Авторизованный вход**
    -   Пользователи входят в свои почтовые аккаунты через протокол OAuth.
    -   Плагин автоматически сгенерирует и сохранит токен авторизации пользователя для последующих вызовов API и операций с почтой.

4.  **Подключение почтовых аккаунтов**
    -   После успешной авторизации почтовый аккаунт пользователя будет подключен к NocoBase.
    -   Плагин синхронизирует почтовые данные пользователя и предоставит функции для управления, отправки и получения писем.

5.  **Использование почтовых функций**
    -   Пользователи смогут просматривать, управлять и отправлять электронные письма непосредственно на платформе.
    -   Все операции выполняются через вызовы API поставщика почтовых услуг, что обеспечивает синхронизацию в реальном времени и эффективную передачу данных.

Благодаря описанному выше процессу, плагин Email в NocoBase предоставляет пользователям эффективные и безопасные услуги по управлению почтой. Если вы столкнетесь с какими-либо проблемами во время настройки, пожалуйста, обратитесь к соответствующей документации или свяжитесь с командой технической поддержки для получения помощи.

## Настройка плагина

### Включение плагина Email

1.  Перейдите на страницу управления плагинами.
2.  Найдите плагин "Email manager" и включите его.

### Настройка поставщиков почтовых услуг

После включения плагина Email вы можете настроить поставщиков почтовых услуг. В настоящее время поддерживаются почтовые сервисы Google и Microsoft. Нажмите "Настройки" -> "Настройки почты" в верхней панели, чтобы перейти на страницу настроек.
