# Узел ИИ-сотрудника

## Введение

Узел ИИ-сотрудника используется в рабочем процессе для назначения ИИ-сотруднику выполнения определённой задачи с последующим выводом структурированной информации.

После создания рабочего процесса при добавлении узлов рабочего процесса можно выбрать узел ИИ-сотрудника.

![20260420142250](https://static-docs.nocobase.com/20260420142250.png)

## Настройка узла
### Предварительная подготовка

Перед настройкой узла ИИ-сотрудника необходимо разобраться, как создавать рабочие процессы, как настраивать службу языковой модели, а также узнать о назначении встроенных ИИ-сотрудников и о том, как создавать ИИ-сотрудников.

Можно ознакомиться со следующими документами:

### Задача
#### Выбор ИИ-сотрудника

Выберите ИИ-сотрудника, который будет отвечать за выполнение задачи в этом узле. В выпадающем списке выберите включённого в системе встроенного ИИ-сотрудника или ИИ-сотрудника, созданного вами самостоятельно.

![20260420143554](https://static-docs.nocobase.com/20260420143554.png)

#### Выбор модели

Выберите языковую модель, которая будет управлять ИИ-сотрудником. В выпадающем списке выберите модель, предоставляемую настроенной в системе службой языковой модели.

![20260420145057](https://static-docs.nocobase.com/20260420145057.png)

#### Выбор оператора

Выберите пользователя в системе, который будет предоставлять ИИ-сотруднику права доступа к данным. При запросе данных ИИ-сотрудник будет ограничен правами этого пользователя.

Если триггер предоставляет оператора (например, `Пользовательское событие действия`), то приоритетно используются права этого оператора.

![20260420145244](https://static-docs.nocobase.com/20260420145244.png)

#### Промпты и описание задачи

`Описание` отправляется ИИ как системный промпт и обычно используется для описания задачи и ограничений.

`Сообщение пользователя по умолчанию` — это пользовательский промпт, отправляемый ИИ; обычно описывает содержание задачи и сообщает ИИ, что нужно сделать.

![20260420174515](https://static-docs.nocobase.com/20260420174515.png)

#### Вложения

`Вложения` отправляются ИИ вместе с `Сообщение пользователя по умолчанию`. Обычно это документы или изображения, которые необходимы для выполнения задачи.

Вложения поддерживают два типа:

1. `Файл (загрузка через коллекцию файлов)` — использование первичного ключа для получения данных из указанной коллекции файлов и использования их в качестве вложения, отправляемого ИИ.

![20260420150933](https://static-docs.nocobase.com/20260420150933.png)

2. `Файл (загрузка по URL)` — получение файла по указанному URL и использование его в качестве вложения, отправляемого ИИ.

![20260420151702](https://static-docs.nocobase.com/20260420151702.png)

#### Навыки и инструменты

Обычно к ИИ-сотруднику привязано несколько навыков и инструментов; здесь можно ограничить использование лишь некоторых из них в текущей задаче.

По умолчанию установлен режим `Предустановленный`, при котором используются предустановленные навыки и инструменты ИИ-сотрудника. При установке `Пользовательский` можно выбрать только некоторые навыки или инструменты ИИ-сотрудника.

![20260426231701](https://static-docs.nocobase.com/20260426231701.png)

#### Веб-поиск

Переключатель `Веб-поиск` управляет тем, использует ли ИИ в текущем узле возможность веб-поиска. О веб-поиске для ИИ-сотрудника см.: [Веб-поиск](/ai-employees/features/web-search)

![20260426231945](https://static-docs.nocobase.com/20260426231945.png)

### Вывод и уведомления
#### Структурированный вывод

Пользователь может в соответствии со спецификацией [Схема JSON](https://json-schema.org/) определить структуру данных, которую в итоге выводит узел ИИ-сотрудника.

![20260426232117](https://static-docs.nocobase.com/20260426232117.png)

При получении данных узла ИИ-сотрудника другими узлами рабочего процесса варианты также будут формироваться в соответствии с этой схемой JSON.

![20260426232509](https://static-docs.nocobase.com/20260426232509.png)

##### Значение по умолчанию

По умолчанию предоставляется следующее определение схемы JSON, которое описывает объект, содержащий свойство с именем result строкового типа. Свойству также задан заголовок: Result.

```json
{
  "type": "object",
  "properties": {
    "result": {
      "title": "Result",
      "type": "string",
      "description": "The text message sent to the user"
    }
  }
}
```

Согласно этому определению, узел ИИ-сотрудника выводит JSON-данные, соответствующие описанной структуре.

```json
{
  result: "Some text generated from LLM "
}
```

#### Настройки согласования

Узел поддерживает три режима согласования:

- `Не требуется` — выводимое ИИ содержимое не требует ручной проверки. После завершения вывода ИИ рабочий процесс автоматически продолжается.
- `Решение человека` — выводимое ИИ содержимое обязательно отправляется на ручную проверку проверяющему; рабочий процесс продолжается только после ручной проверки.
- `Решение ИИ` — ИИ сам решает, отправлять ли выводимое содержимое на ручную проверку проверяющему.

![20260426232232](https://static-docs.nocobase.com/20260426232232.png)

Если режим согласования отличается от `Не требуется`, для узла обязательно нужно настроить одного или нескольких проверяющих.

После того как ИИ в узле ИИ-сотрудника завершит вывод всего содержимого, всем настроенным для узла проверяющим будет отправлено уведомление. Достаточно, чтобы кто-то один из уведомлённых проверяющих выполнил действие согласования, и рабочий процесс сможет продолжиться.

![20260426232319](https://static-docs.nocobase.com/20260426232319.png)
