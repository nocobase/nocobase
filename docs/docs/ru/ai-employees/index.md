# Обзор

![clipboard-image-1771905619](https://static-docs.nocobase.com/clipboard-image-1771905619.png)

AI-сотрудники (`AI Employees`) — это возможности интеллектуальных агентов, глубоко интегрированные в бизнес-системы NocoBase.

Это не просто «умеющие болтать» роботы, а «цифровые коллеги», которые могут напрямую понимать контекст в бизнес-интерфейсе и выполнять действия:

- **Понимают бизнес-контекст**: воспринимают текущую страницу, блоки, структуру данных и выбранный контент.
- **Могут напрямую выполнять действия**: могут вызывать навыки для выполнения таких задач, как запрос, анализ, заполнение, настройка, генерация и т. д.
- **Ролевое взаимодействие**: настраивайте разных сотрудников в соответствии с должностями и переключайте модели для совместной работы в диалоге.

## 5-минутный путь к началу работы

Сначала ознакомьтесь с [Быстрым стартом](/ai-employees/quick-start) и выполните минимально необходимую конфигурацию в следующем порядке:

1. Настройте как минимум один [LLM-сервис](/ai-employees/features/llm-service).
2. Включите как минимум одного [AI-сотрудника](/ai-employees/features/enable-ai-employee).
3. Откройте чат и начните [взаимодействие с AI-сотрудниками](/ai-employees/features/collaborate).
4. При необходимости включите [Поиск в сети](/ai-employees/features/web-search) и [Быстрые задачи](/ai-employees/features/task).

## Карта функций

### A. Базовая конфигурация (Администратор)

- [Настройка LLM-сервиса](/ai-employees/features/llm-service): подключение провайдеров (Provider), настройка и управление доступными моделями.
- [Включение AI-сотрудников](/ai-employees/features/enable-ai-employee): включение и выключение встроенных сотрудников, управление областью доступа.
- [Создание AI-сотрудника](/ai-employees/features/new-ai-employees): определение роли, личности, приветствия и границ возможностей.
- [Использование навыков](/ai-employees/features/tool): настройка прав доступа к навыкам (`Ask` / `Allow`), контроль рисков выполнения.

### B. Ежедневное взаимодействие (Бизнес-пользователь)

- [Взаимодействие с AI-сотрудниками](/ai-employees/features/collaborate): переключение сотрудников и моделей внутри чата для непрерывной совместной работы.
- [Добавление контекста — Блоки](/ai-employees/features/pick-block): отправка блоков страницы AI в качестве контекста.
- [Быстрые задачи](/ai-employees/features/task): предварительная настройка часто используемых задач на страницах/блоках для выполнения в один клик.
- [Поиск в сети](/ai-employees/features/web-search): включение поиска для дополнения ответов актуальной информацией при необходимости.

### C. Продвинутые возможности (Расширения)

- [Встроенные AI-сотрудники](/ai-employees/features/built-in-employee): понимание позиционирования и сценариев применения предустановленных сотрудников.
- [Управление правами доступа](/ai-employees/permission): управление доступом к сотрудникам, навыкам и данным в соответствии с моделью прав организации.
- [База знаний AI](/ai-employees/knowledge-base/index): внедрение корпоративных знаний для повышения стабильности и прослеживаемости ответов.
- [Узел LLM в рабочем процессе](/ai-employees/workflow/nodes/llm/chat): встраивание возможностей AI в автоматизированные рабочие процессы.

## Основные концепции (рекомендуется согласовать заранее)

Следующие термины соответствуют глоссарию, рекомендуется использовать их единообразно внутри команды:

- **AI-сотрудник (AI Employee)**: исполняемый агент, состоящий из настроек роли (Role setting) и навыков (Tool / Skill).
- **LLM-сервис (LLM Service)**: единица подключения моделей и настройки возможностей, используемая для управления провайдерами (Provider) и списком моделей.
- **Провайдер (Provider)**: поставщик моделей, стоящий за LLM-сервисом.
- **Включенные модели (Enabled Models)**: набор моделей, которые текущий LLM-сервис позволяет выбирать в чате.
- **Переключатель AI-сотрудников (AI Employee Switcher)**: переключение текущего взаимодействующего сотрудника внутри чата.
- **Переключатель моделей (Model Switcher)**: переключение моделей внутри чата с запоминанием предпочтений для каждого сотрудника.
- **Навык (Tool / Skill)**: единица исполнительной способности, которую может вызывать AI.
- **Права доступа к навыку (Permission: Ask / Allow)**: требуется ли подтверждение человека перед вызовом навыка.
- **Контекст (Context)**: информация о бизнес-среде, такая как страницы, блоки, структуры данных и т. д.
- **Чат (Chat)**: процесс непрерывного взаимодействия между пользователем и AI-сотрудником.
- **Поиск в сети (Web Search)**: возможность дополнения ответов информацией в реальном времени на основе внешнего поиска.
- **База знаний (Knowledge Base / RAG)**: внедрение корпоративных знаний через генерацию с расширенным поиском.
- **Векторное хранилище (Vector Store)**: векторизованное хранилище, обеспечивающее возможность семантического поиска для базы знаний.

## Инструкция по установке

AI-сотрудники — это встроенный плагин NocoBase (`@nocobase/plugin-ai`), он готов к использованию сразу и не требует отдельной установки.
