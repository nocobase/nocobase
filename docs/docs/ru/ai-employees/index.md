# Обзор

![clipboard-image-1771905619](https://static-docs.nocobase.com/clipboard-image-1771905619.png)

`ИИ-сотрудники` - это возможности ИИ-агентов, глубоко интегрированные в бизнес-системы NocoBase.

Они не просто чат-боты, а «цифровые коллеги», которые могут понимать бизнес-контекст в интерфейсе и выполнять действия напрямую:

- **Понимать бизнес-контекст**: воспринимать текущие страницы, блоки, структуры данных и выбранный контент.
- **Выполнять действия напрямую**: вызывать инструменты и навыки для запросов, анализа, заполнения, настройки и генерации.
- **Сотрудничать с опорой на роли**: настраивайте разных сотрудников по ролям и переключайте модели в ходе диалога.

## Запуск за 5 минут

Начните с [Быстрого старта](/ai-employees/quick-start) и выполните минимальную настройку в следующем порядке:

1. Настройте как минимум одну [службу языковой модели](/ai-employees/features/llm-service).
2. Включите как минимум одного [ИИ-сотрудника](/ai-employees/features/enable-ai-employee).
3. Откройте сессию и начните [работать с ИИ-сотрудниками](/ai-employees/features/collaborate).
4. При необходимости включите [Веб-поиск](/ai-employees/features/web-search) и [Быстрые задачи](/ai-employees/features/task).

## Карта функций

### A. Базовая настройка (Admin)

- [Настроить службу языковой модели](/ai-employees/features/llm-service): подключите провайдеров и управляйте доступными моделями.
- [Включить ИИ-сотрудников](/ai-employees/features/enable-ai-employee): включайте/отключайте встроенных сотрудников и управляйте областью применения.
- [Создать нового ИИ-сотрудника](/ai-employees/features/new-ai-employees): задайте роль, личность, приветственное сообщение и границы возможностей.
- [Использование навыков](/ai-employees/features/tools): настройте разрешения навыков и контролируйте выполнение.

### B. Ежедневная работа (Бизнес-пользователи)

- [Работа с ИИ-сотрудниками](/ai-employees/features/collaborate): переключайте сотрудника и модель в чате для непрерывного рабочего процесса.
- [Добавить контекст - блоки](/ai-employees/features/pick-block): отправляйте блоки страницы в качестве контекста для ИИ.
- [Быстрые задачи](/ai-employees/features/task): заранее настройте типовые задачи на страницах/блоках и запускайте одним щелчком.
- [Веб-поиск](/ai-employees/features/web-search): включите получение актуальной информации извне, когда это необходимо.

### C. Расширенные возможности

- [Встроенные ИИ-сотрудники](/ai-employees/features/built-in-employee): встроенные роли и подходящие сценарии.
- [Контроль прав доступа](/ai-employees/permission): управляйте доступом сотрудников, навыков и данных с помощью прав организации.
- [База знаний ИИ](/ai-employees/knowledge-base/index): импортируйте корпоративные знания, чтобы ответы были стабильными и отслеживаемыми.
- [Узлы языковой модели для рабочего процесса](/ai-employees/workflow/nodes/llm/chat): оркестрируйте возможности ИИ в автоматизированные рабочие процессы.

## Основные концепции (рекомендуется согласовать в первую очередь)

Согласуйте следующие термины с глоссарием:

- **ИИ-сотрудник**: исполняемый агент, состоящий из настройки роли и инструмента/навыка.
- **Служба языковой модели**: доступ к моделям и настройка возможностей — единица для управления провайдером и списком моделей.
- **Провайдер**: поставщик моделей для службы языковой модели.
- **Включенные модели**: набор моделей, который текущая служба языковой модели разрешает в чате.
- **Переключатель ИИ-сотрудника**: переключает текущего сотрудника для работы в чате.
- **Переключатель модели**: переключает модель в чате и запоминает предпочтения для каждого сотрудника.
- **Инструмент / навык**: исполняемая единица возможностей, вызываемая ИИ.
- **Права доступа навыка**: нужно ли требовать подтверждение человека перед вызовом навыков.
- **Контекст**: информация о бизнес-среде, такая как страницы, блоки и структуры данных.
- **Чат**: одно непрерывное взаимодействие между пользователем и ИИ-сотрудником.
- **Веб-поиск**: возможность, которая улучшает ответы за счет внешнего поиска.
- **База знаний / Генерация с расширенными возможностями поиска (RAG)**: генерация, дополненная поиском, с использованием корпоративных знаний.
- **Векторное хранилище**: векторизованное хранилище, которое обеспечивает семантический поиск для базы знаний.

## Установка

ИИ-сотрудники встроены в NocoBase (`@nocobase/plugin-ai`) и готовы к использованию без отдельной установки.
