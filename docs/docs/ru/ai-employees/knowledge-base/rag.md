# Генерация с расширенными возможностями поиска

## Введение

После настройки базы знаний вы можете включить функцию Генерации с расширенными возможностями поиска (RAG) в настройках ИИ-сотрудника.

При включенной Генерации с расширенными возможностями поиска (RAG), когда пользователь общается с ИИ-сотрудником, ИИ-сотрудник будет использовать извлечение RAG, чтобы получать документы из базы знаний на основе сообщения пользователя, и отвечать на основе извлеченных документов.

## Включить Генерацию с расширенными возможностями поиска (RAG)

Перейдите на страницу настроек плагина ИИ-сотрудника, нажмите вкладку `ИИ-сотрудники`, чтобы открыть страницу управления ИИ-сотрудниками.

![20251023010811](https://static-docs.nocobase.com/20251023010811.png)

Выберите ИИ-сотрудника, для которого вы хотите включить Генерацию с расширенными возможностями поиска (RAG), нажмите кнопку `Редактировать`, чтобы перейти на страницу редактирования ИИ-сотрудника.

На вкладке `База знаний` включите переключатель `Включить`.

- В `Запрос базы знаний` введите запрос для обращения к базе знаний. `{knowledgeBaseData}` это фиксированный заполнитель, и его изменять нельзя.
- В `База знаний` выберите настроенную базу знаний. См.: [База знаний](./knowledge-base/).
- В поле ввода `Количество документов для извлечения` введите количество документов для извлечения; по умолчанию это 3.
- В поле ввода `Порог релевантности` укажите порог релевантности документа для извлечения.

Нажмите кнопку `Отправить`, чтобы сохранить настройки ИИ-сотрудника.
