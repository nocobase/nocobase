# Подключение MCP

AI-сотрудники могут подключаться к MCP-сервисам, реализующим протокол Model Context Protocol (MCP). После подключения MCP-сервиса AI-сотрудник сможет использовать предоставляемые им инструменты для выполнения задач.


## Конфигурация MCP

В модуле конфигурации MCP можно добавлять новые MCP-сервисы и управлять уже подключёнными.

![20260323095943](https://static-docs.nocobase.com/20260323095943.png)


## Добавление MCP-сервиса

Нажмите кнопку `Добавить` в правом верхнем углу списка MCP-сервисов и в открывшемся окне введите данные подключения, чтобы завершить добавление MCP-сервиса.

Поддерживаются два транспортных протокола MCP-сервисов: Stdio и HTTP (Streamable / SSE).

![20260323100904](https://static-docs.nocobase.com/20260323100904.png)

При добавлении MCP-сервиса необходимо указать `Имя`, `Заголовок` и `Описание`. `Имя` является уникальным идентификатором MCP-сервиса; `Заголовок` — это название сервиса, отображаемое в системе; `Описание` указывается по желанию и кратко описывает функциональность MCP-сервиса.

![20260323101635](https://static-docs.nocobase.com/20260323101635.png)

### Stdio

При добавлении MCP-сервиса с поддержкой транспортного протокола stdio необходимо указать `команду` запуска MCP-сервиса и `аргументы команды`, при необходимости можно добавить `переменные окружения`, нужные для работы команды.

:::warning
Команды запуска MCP-сервиса, такие как node, npx, uvx, go и подобные, требуют поддержки в серверной среде, где развёрнут NocoBase.

Docker-образ NocoBase поддерживает только команды среды Node.js, такие как node и npx.
:::

![20260323103511](https://static-docs.nocobase.com/20260323103511.png)

### HTTP

При добавлении MCP-сервиса с поддержкой транспортного протокола http необходимо указать `URL` MCP-сервиса, при необходимости можно добавить `заголовки запроса`.

Транспортный протокол http поддерживает два способа передачи: Streamable и SSE. Streamable — это новый способ, добавленный в стандарт MCP, а SSE-передача в скором времени будет признана устаревшей. Выбирайте конкретный способ передачи в соответствии с документацией используемого MCP-сервиса.

![20260323103906](https://static-docs.nocobase.com/20260323103906.png)

### Проверка доступности

При добавлении и редактировании MCP-сервиса после ввода информации о конфигурации можно запустить проверку доступности MCP-сервиса. Если данные конфигурации полные и корректные, а MCP-сервис доступен, будет возвращено сообщение об успешной проверке.

![20260323105608](https://static-docs.nocobase.com/20260323105608.png)

## Просмотр MCP-сервиса

Нажмите кнопку `Просмотр` в списке MCP-сервисов, чтобы увидеть список инструментов, предоставляемых MCP-сервисом.

В списке инструментов MCP-сервиса также можно настроить права использования инструмента AI-сотрудником: при значении `Ask` перед вызовом будет запрашиваться подтверждение; при значении `Allow` инструмент будет вызываться напрямую при необходимости.

![20260323111106](https://static-docs.nocobase.com/20260323111106.png)

## Использование MCP-сервиса

После включения нужного MCP-сервиса в модуле конфигурации MCP, при общении с AI-сотрудником, он будет автоматически использовать инструменты, предоставляемые MCP-сервисом, для выполнения задач.

![20260323110535](https://static-docs.nocobase.com/20260323110535.png)
