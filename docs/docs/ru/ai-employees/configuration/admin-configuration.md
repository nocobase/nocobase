# ИИ-сотрудник · Руководство по настройке для администратора

> Этот документ поможет быстро разобраться, как настраивать и управлять ИИ-сотрудниками (AI Employees), ведя вас пошагово через весь процесс от служб моделей до назначения задач.


## I. Перед началом

### 1. Системные требования

Перед настройкой убедитесь, что ваша среда соответствует следующим условиям:

* Установлен **NocoBase 2.0 или выше**
* Включен **плагин ИИ-сотрудников**
* Доступна как минимум одна **служба большой языковой модели** (например, OpenAI, Claude, DeepSeek, GLM, GigaChat и т.д.)


### 2. Понимание двухуровневой архитектуры ИИ-сотрудников

ИИ-сотрудники разделены на два уровня: **«Определение роли»** и **«Настройка задач»**.

| Уровень | Описание | Характеристики | Назначение |
|---|---|---|---|
| **Определение роли** | Базовая личность сотрудника и его ключевые возможности | Стабильные и неизменные, как «резюме» | Обеспечивает согласованность роли |
| **Настройка задач** | Конфигурация для различных бизнес-сценариев | Гибкая и настраиваемая | Адаптируется под конкретные задачи |

**Проще говоря:**

> «Определение роли» определяет, кто этот сотрудник,
> «Настройка задач» определяет, что он делает прямо сейчас.

Преимущества такой конструкции:

* Роль остается неизменной, но может обрабатывать разные сценарии
* Обновление или замена задач не влияет на самого сотрудника
* Фон и задачи независимы, благодаря чему обслуживание проще


## II. Процесс настройки (в 5 шагов)

### Шаг 1: Настройте службу модели

Служба модели — это как «мозг» ИИ-сотрудника, и ее нужно настроить в первую очередь.

> 💡 Подробные инструкции по настройке см.: [Настроить службу языковой модели](/ai-employees/features/llm-service)

**Путь:**
`Настройки системы → ИИ-сотрудники → Служба языковой модели`

![Enter configuration page](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-15-40-47.png)

Нажмите **Добавить** и заполните следующую информацию:

| Пункт | Описание | Примечания |
|---|---|---|
| Провайдер | например, OpenAI, Claude, Gemini, GigaChat и т.д. | Совместим с сервисами, использующими ту же спецификацию |
| Ключ API | ключ, предоставленный поставщиком сервиса | Держите его в тайне и регулярно меняйте |
| Базовый URL | API Endpoint (необязательно) | Требует изменения при использовании прокси |
| Включенные модели | Рекомендуемые модели / Выбрать модели / Ручной ввод | Определяет, какие модели доступны в чате |

![Create a large model service](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-15-45-27.png)

После настройки используйте `Тест запуска (Test flight)`, чтобы **проверить соединение**.
Если проверка не проходит, проверьте сеть, ключ API или имя модели.

![Test connection](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-18-25.png)


### Шаг 2: Создайте ИИ-сотрудника

> 💡 Подробные инструкции см.: [Создать ИИ-сотрудника](/ai-employees/features/new-ai-employees)

Путь: `Управление ИИ-сотрудниками → Создать сотрудника`

Заполните базовую информацию:

| Поле | Обязательно | Пример |
|---|---|---|
| Имя | ✓ | viz, dex, cole |
| Никнейм | ✓ | Viz, Dex, Cole |
| Статус включения | ✓ | On |
| Описание | - | "Data Analysis Expert" |
| Основная задача | ✓ | см. Руководство по проектированию запросов |
| Приветственное сообщение | - | "Привет, я Viz…" |

![Basic information configuration](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-21-09.png)

На этапе создания сотрудника сосредоточьтесь на настройке роли и навыков. Фактическую модель можно выбирать в чате через `Переключатель модели`.

**Рекомендации по составлению описания:**

* Четко укажите роль сотрудника, тон и ответственность
* Используйте слова вроде «должен» и «никогда», чтобы подчеркнуть правила
* По возможности добавляйте примеры, чтобы избежать абстрактных описаний
* Держите описание в диапазоне 500–1000 символов

> Чем яснее описание, тем стабильнее работа ИИ.
> См. [Руководство по проектированию запросов](./prompt-engineering-guide.md).


### Шаг 3: Настройте навыки

Навыки определяют, что сотрудник «может делать».

> 💡 Подробные инструкции см.: [Навыки](/ai-employees/features/tools)

| Тип | Область возможностей | Пример | Уровень риска |
|---|---|---|---|
| Фронтенд | Взаимодействие со страницей | Читать данные блоков, заполнять формы | Низкий |
| Модель данных | Запрос и анализ данных | Агрегирующая статистика | Средний |
| Рабочий процесс | Выполнение бизнес-процессов | Пользовательские инструменты | Зависит от рабочего процесса |
| Другое | Внешние расширения | Веб-поиск и операции с файлами | Различается |

**Рекомендации по настройке:**

* 3–5 навыков на сотрудника — самый подходящий вариант
* Не рекомендуется выбирать все навыки, так как это может вызвать путаницу
* Для важных операций предпочитайте `Спросить` вместо `Разрешить`

![Configure skills](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-26-06.png)


### Шаг 4: Настройте базу знаний (необязательно)

Если вашему ИИ-сотруднику нужно запоминать или ссылаться на большой объем материалов (например, руководства по продукту, FAQ и т.д.), вы можете настроить базу знаний.

> 💡 Подробные инструкции см.:
> - [Обзор базы знаний ИИ](/ai-employees/knowledge-base/index)
> - [Векторная база данных](/ai-employees/knowledge-base/vector-database)
> - [Настройка базы знаний](/ai-employees/knowledge-base/knowledge-base)
> - [RAG (Генерация с расширенными возможностями поиска)](/ai-employees/knowledge-base/rag)

Для этого нужно установить плагин векторной базы данных.

![Настройка базы знаний](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-32-54.png)

**Подходящие сценарии:**

* Чтобы ИИ понимал корпоративные знания
* Чтобы поддерживать вопросы и ответы по документам
* Чтобы обучать ассистентов по предметной области


### Шаг 5: Проверьте эффект

После завершения вы увидите аватар нового сотрудника в правом нижнем углу страницы.

![Verify configuration](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-36-54.png)

Проверьте каждый пункт:

* ✅ Отображается ли значок корректно?
* ✅ Может ли он вести базовый диалог?
* ✅ Корректно ли вызываются навыки?

Если все пункты пройдены, настройка успешна 🎉


## III. Настройка задач: заставляем ИИ работать

То, что мы сделали до этого, это «создание сотрудника».
Далее нужно «заставить его работать».

Задачи ИИ определяют поведение сотрудника на конкретной странице или блоке.

> 💡 Подробные инструкции см.: [Задачи](/ai-employees/features/task)


### 1. Задачи на уровне страницы

Применяется ко всей области страницы, например «Проанализировать данные на этой странице».

**Параметр конфигурации:**
`Настройки страницы  → ИИ-сотрудник  → Добавить задачу `

| Поле | Описание | Пример |
|---|---|---|
| Название | Имя задачи | Анализ конверсии этапов |
| Контекст | Контекст текущей страницы | Страница списка лидов |
| Сообщение по умолчанию | Готовый старт диалога | "Проанализируй тренды в этом месяце" |
| Блок по умолчанию | Автоматически связывать с коллекцией | leads_table |
| Навыки | Доступные инструменты | Запрос данных, генерация диаграмм |

![Настройка задач на уровне страницы](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-40-34.png)

**Поддержка нескольких задач:**
ИИ-сотрудник можно настроить на несколько задач, которые отображаются пользователю как варианты для выбора:

![Поддержка нескольких задач](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-46-00.png)

Рекомендации:

* Одна задача должна ориентироваться на одну цель
* Название должно быть четким и понятным
* Количество задач держите в пределах 5–7


### 2. Задачи на уровне блока

Подходит для работы с конкретным блоком, например «Перевести текущую форму».

**Метод настройки:**

1. Откройте конфигурацию действий блока
2. Добавьте «ИИ-сотрудник»

![Add AI Employee button](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-51-06.png)

3. Привяжите нужного сотрудника

![Select AI Employee](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-52-26.png)

![Block-level task configuration](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-53-35.png)

| Сравнение | На уровне страницы | На уровне блока |
|---|---|---|
| Область данных | Вся страница | Текущий блок |
| Гранулярность | Глобальный анализ | Детальная обработка |
| Типичное использование | Анализ трендов | Перевод формы, извлечение полей |


## IV. Лучшие практики

### 1. Рекомендации по настройке

| Параметр | Рекомендация | Причина |
|---|---|---|
| Количество навыков | 3–5 | Высокая точность и быстрый ответ |
| Режим прав (Спросить / Разрешить) | Для изменений данных выбирайте Спросить | Предотвращает случайные операции |
| Длина подсказки | 500–1000 символов | Баланс между скоростью и качеством |
| Цель задачи | Одна и четкая | Не дает ИИ запутаться |
| Рабочий процесс | Используйте после инкапсуляции сложных задач | Более высокий процент успешных результатов |


### 2. Практические рекомендации

**Начните с малого и постепенно оптимизируйте:**

1. Сначала создайте базовых сотрудников (например, Viz, Dex)
2. Включите 1–2 ключевых навыка для тестирования
3. Убедитесь, что задачи выполняются корректно
4. Затем постепенно расширяйте набор навыков и задач

**Процесс непрерывной оптимизации:**

1. Запустите начальную версию
2. Соберите обратную связь от пользователей
3. Оптимизируйте подсказки и конфигурации задач
4. Тестируйте и повторяйте итерации


## V. Часто задаваемые вопросы

### 1. Этап настройки

**Вопрос: Что делать, если сохранение не удаётся?**
Ответ: Проверьте, что заполнены все обязательные поля, особенно служба модели и подсказка.

**Вопрос: Какую модель выбрать?**

* Для кода → Claude, GPT-4
* Для анализа → Claude, DeepSeek
* Если важны затраты → Qwen, GLM
* Для длинных текстов → Gemini, Claude


### 2. Этап использования

**Вопрос: Ответ ИИ слишком медленный?**

* Уменьшите число навыков
* Оптимизируйте подсказку
* Проверьте задержку службы модели
* Рассмотрите возможность смены модели

**Вопрос: Почему выполнение задачи неточное?**

* Подсказка недостаточно ясная
* Слишком много навыков приводит к путанице
* Разбейте задачу на более мелкие части и добавьте примеры

**Вопрос: Когда выбирать  вместо Allow?**

* `Разрешить` можно использовать для задач только на запрос
* `Спросить` рекомендуется для задач, изменяющих данные

**В: Как заставить ИИ обрабатывать конкретную форму?**

О: Для конфигураций на уровне страницы вам нужно вручную выбрать блок.

![Ручной выбор блока](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-17-02-22.png)

Для конфигураций задач на уровне блока контекст данных привязывается автоматически.

## Заключение

Самое важное при настройке ИИ-сотрудников: **сначала добейтесь работоспособности, затем оптимизируйте**.
Сначала успешно запустите первого сотрудника, затем постепенно расширяйте возможности и тонко настраивайте.

Если нужно устранить проблему, действуйте в следующем порядке:

1. Подключена ли служба модели?
2. Не слишком ли много навыков?
3. Ясна ли подсказка?
4. Хорошо ли определена цель задачи?

Пошагово двигаясь вперед, вы сможете сформировать действительно эффективную команду ИИ-сотрудников.
