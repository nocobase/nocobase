# AI-сотрудник · Руководство по настройке для администратора

> Этот документ поможет вам быстро разобраться, как настраивать и управлять AI-сотрудниками: от сервиса модели до ввода в эксплуатацию, шаг за шагом проводя вас через весь процесс.


## I. Перед началом работы

### 1. Системные требования

Перед настройкой убедитесь, что ваша среда соответствует следующим условиям:

* Установлена версия **NocoBase 2.0 или выше**
* Включен **плагин AI-сотрудник**
* Доступен как минимум один **сервис большой языковой модели** (например, OpenAI, Claude, DeepSeek, GLM и т. д.)


### 2. Двухуровневый дизайн AI-сотрудников

AI-сотрудники разделены на два уровня: **«Определение роли»** и **«Настройка задачи»**.

| Уровень | Описание | Характеристики | Роль |
| -------- | ------------ | ---------- | ------- |
| **Определение роли** | Базовая личность и ключевые способности сотрудника | Стабильно и неизменно, как «резюме» | Обеспечивает согласованность роли |
| **Настройка задачи** | Конфигурация для различных бизнес-сценариев | Гибкая настройка | Адаптация к конкретным задачам |

**Простое понимание:**

> «Определение роли» определяет, кем является этот сотрудник,
> «Настройка задачи» определяет, что он должен делать в данный момент.

Преимущества такого дизайна:

* Роль остается неизменной, но сотрудник может справляться с различными сценариями
* Обновление или замена задач не влияет на самого сотрудника
* Контекст и задачи независимы друг от друга, что упрощает обслуживание


## II. Процесс настройки (5 шагов)

### Шаг 1: Настройка сервиса модели

Сервис модели эквивалентен мозгу AI-сотрудника, его необходимо настроить в первую очередь.

> 💡 Подробные инструкции по настройке см. в: [Настройка сервиса LLM](/ai-employees/features/llm-service)

**Путь:**
`Настройки системы → AI-сотрудник → LLM service`

![Вход на страницу конфигурации](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-15-40-47.png)

Нажмите **Добавить** и заполните следующую информацию:

| Пункт | Описание | Примечания |
| ------ | -------------------------- | --------- |
| Provider | Например, OpenAI, Claude, Gemini, Kimi и т. д. | Совместим с сервисами тех же стандартов |
| API-ключ | Ключ, предоставленный поставщиком услуг | Храните в секрете и регулярно меняйте |
| Base URL | API Endpoint (опционально) | Необходимо изменить при использовании прокси |
| Enabled Models | Рекомендуемые модели / Выбор моделей / Ручной ввод | Определяет диапазон моделей, доступных в чате |

![Создание сервиса большой модели](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-15-45-27.png)

После настройки используйте `Test flight` для **тестирования соединения**.
Если проверка не удалась, проверьте сеть, ключ или название модели.

![Тестирование соединения](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-18-25.png)


### Шаг 2: Создание AI-сотрудника

> 💡 Подробные инструкции см. в: [Создание AI-сотрудника](/ai-employees/features/new-ai-employees)

Путь: `Управление AI-сотрудниками → Создать сотрудника`

Заполните основную информацию:

| Поле | Обязательно | Пример |
| ----- | -- | -------------- |
| Имя | ✓ | viz, dex, cole |
| Псевдоним | ✓ | Viz, Dex, Cole |
| Статус | ✓ | Включено |
| Биография | - | «Эксперт по анализу данных» |
| Основной промпт | ✓ | См. руководство по промпт-инжинирингу |
| Приветствие | - | «Здравствуйте, я Viz…» |

![Настройка основной информации](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-21-09.png)

На этапе создания сотрудника в основном настраиваются роль и навыки. Конкретную модель можно выбрать во время диалога через `Model Switcher`.

**Рекомендации по написанию промптов:**

* Четко опишите роль, тон и обязанности сотрудника
* Используйте слова «должен», «ни в коем случае» для акцента на правилах
* Старайтесь включать примеры, избегайте абстрактных описаний
* Ограничьте объем в пределах 500–1000 символов

> Чем четче промпт, тем стабильнее ведет себя AI.
> Вы можете обратиться к [Руководству по промпт-инжинирингу](./prompt-engineering-guide.md).


### Шаг 3: Настройка навыков

Навыки определяют, что сотрудник «может делать».

> 💡 Подробные инструкции см. в: [Навыки](/ai-employees/features/tool)

| Тип | Область способностей | Пример | Уровень риска |
| ---- | ------- | --------- | ------ |
| Фронтенд | Взаимодействие со страницей | Чтение данных блока, заполнение формы | Низкий |
| Модель данных | Запрос и анализ данных | Агрегированная статистика | Средний |
| Рабочий процесс | Выполнение бизнес-процессов | Пользовательские инструменты | Зависит от рабочего процесса |
| Другое | Внешние расширения | Поиск в интернете, операции с файлами | По ситуации |

**Рекомендации по настройке:**

* 3–5 навыков на одного сотрудника — оптимальный вариант
* Не рекомендуется выбирать все навыки сразу, это может вызвать путаницу
* Для важных операций рекомендуется использовать разрешение `Ask` вместо `Allow`

![Настройка навыков](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-26-06.png)


### Шаг 4: Настройка базы знаний (опционально)

Если вашему AI-сотруднику необходимо помнить или ссылаться на большой объем материалов, таких как инструкции к продуктам, FAQ и т. д., вы можете настроить базу знаний.

> 💡 Подробные инструкции см. в:
> - [Обзор базы знаний AI](/ai-employees/knowledge-base/index)
> - [Векторная база данных](/ai-employees/knowledge-base/vector-database)
> - [Настройка базы знаний](/ai-employees/knowledge-base/knowledge-base)
> - [RAG (Retrieval-Augmented Generation)](/ai-employees/knowledge-base/rag)

Для этого требуется дополнительная установка плагина векторной базы данных.

![Настройка базы знаний](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-32-54.png)

**Сценарии применения:**

* Позволить AI понимать корпоративные знания
* Поддержка ответов на вопросы по документам и поиск
* Обучение специализированных помощников в конкретных областях


### Шаг 5: Верификация результата

После завершения вы увидите аватар нового сотрудника в правом нижнем углу страницы.

![Верификация конфигурации](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-36-54.png)

Пожалуйста, проверьте по пунктам:

* ✅ Правильно ли отображается иконка
* ✅ Возможен ли базовый диалог
* ✅ Корректно ли вызываются навыки

Если все проверки пройдены, настройка завершена успешно 🎉


## III. Настройка задач: ввод AI в эксплуатацию

Ранее мы завершили этап «создания сотрудника»,
теперь необходимо отправить их «работать».

Задачи AI определяют поведение сотрудника на конкретной странице или в блоке.

> 💡 Подробные инструкции см. в: [Задачи](/ai-employees/features/task)


### 1. Задачи уровня страницы

Подходят для всей области страницы, например, «Проанализировать данные на этой странице».

**Вход в настройки:**
`Настройки страницы → AI-сотрудник → Добавить задачу`

| Поле | Описание | Пример |
| ---- | -------- | --------- |
| Заголовок | Название задачи | Анализ конверсии этапов |
| Контекст | Контекст текущей страницы | Страница списка лидов (Leads) |
| Сообщение по умолчанию | Предустановленный диалог | «Проанализируйте тенденции этого месяца» |
| Блок по умолчанию | Автоматическая привязка к коллекции | Таблица leads |
| Навыки | Доступные инструменты | Запрос данных, создание графиков |

![Настройка задачи уровня страницы](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-40-34.png)

**Поддержка нескольких задач:**
Для одного AI-сотрудника можно настроить несколько задач, которые будут предлагаться пользователю на выбор:

![Поддержка нескольких задач](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-46-00.png)

Советы:

* Одна задача должна быть сфокусирована на одной цели
* Названия должны быть четкими и понятными
* Количество задач рекомендуется ограничивать в пределах 5–7


### 2. Задачи уровня блока

Подходят для работы с конкретным блоком, например, «Перевести текущую форму».

**Способ настройки:**

1. Откройте конфигурацию действий блока
2. Добавьте «AI-сотрудника»

![Кнопка добавления AI-сотрудника](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-51-06.png)

3. Привяжите нужного сотрудника

![Выбор AI-сотрудника](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-52-26.png)

![Настройка задачи уровня блока](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-53-35.png)

| Сравнение | Уровень страницы | Уровень блока |
| ---- | ---- | --------- |
| Область данных | Вся страница | Текущий блок |
| Детализация | Глобальный анализ | Детальная обработка |
| Типичное использование | Анализ трендов | Перевод форм, извлечение полей |


## IV. Лучшие практики

### 1. Рекомендации по настройке

| Пункт | Рекомендация | Обоснование |
| ---------- | ----------- | -------- |
| Количество навыков | 3–5 штук | Высокая точность, быстрый отклик |
| Режим прав (Ask / Allow) | Для изменения данных — Ask | Предотвращение ошибочных действий |
| Длина промпта | 500–1000 символов | Баланс скорости и качества |
| Цель задачи | Единственная и четкая | Чтобы AI не путался |
| Рабочий процесс | Использовать после инкапсуляции сложных задач | Выше вероятность успеха |


### 2. Практические советы

**От малого к большему, постепенная оптимизация:**

1. Сначала создайте базовых сотрудников (например, Viz, Dex)
2. Включите 1–2 основных навыка для тестирования
3. Убедитесь, что задачи выполняются нормально
4. Затем постепенно расширяйте количество навыков и задач

**Процесс непрерывного улучшения:**

1. Запустите первую версию
2. Соберите отзывы об использовании
3. Оптимизируйте промпты и настройки задач
4. Тестируйте и повторяйте цикл улучшений


## V. Часто задаваемые вопросы

### 1. Этап настройки

**В: Что делать, если сохранение не удалось?**
О: Проверьте, заполнены ли все обязательные поля, особенно сервис модели и промпт.

**В: Какую модель выбрать?**

* Код → Claude, GPT-4
* Анализ → Claude, DeepSeek
* Чувствительность к затратам → Qwen, GLM
* Длинный текст → Gemini, Claude


### 2. Этап использования

**В: AI отвечает слишком медленно?**

* Уменьшите количество навыков
* Оптимизируйте промпт
* Проверьте задержку сервиса модели
* Рассмотрите возможность смены модели

**В: Задача выполняется неточно?**

* Промпт недостаточно четкий
* Слишком много навыков вызывает путаницу
* Разбейте на мелкие задачи, добавьте примеры

**В: Когда выбирать Ask, а когда Allow?**

* Для задач запроса данных можно использовать `Allow`
* Для задач изменения данных рекомендуется использовать `Ask`

**В: Как заставить AI обрабатывать конкретную форму?**

О: Если это настройка уровня страницы, необходимо вручную выбрать блок.

![Ручной выбор блока](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-17-02-22.png)

Если это настройка задачи уровня блока, контекст данных привязывается автоматически.

## Заключение

Самое важное в настройке AI-сотрудников: **сначала запустить, затем оптимизировать**.
Сначала добейтесь успешного ввода в эксплуатацию первого сотрудника, а затем постепенно расширяйте и дорабатывайте систему.

Направления для поиска неисправностей могут быть в следующем порядке:

1. Подключен ли сервис модели
2. Не слишком ли много навыков
3. Достаточно ли четкий промпт
4. Ясна ли цель задачи

Двигаясь последовательно, вы сможете создать по-настоящему эффективную AI-команду.
