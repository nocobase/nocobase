# Аутентификаторы

## Управление аутентификацией пользователя

После установки плагина аутентификации пользователя он инициализирует метод аутентификации `password` на основе имени пользователя и email пользователя.

![](https://static-docs.nocobase.com/66eaa9d5421c9cb713b117366bd8a5d5.png)

## Активировать аутентификаторы

![](https://static-docs.nocobase.com/7f1fb8f8ca5de67ffc68eff0a65848f5.png)

На странице входа будут отображаться только активированные типы аутентификации.

![](https://static-docs.nocobase.com/8375a36ef98417af0f0977f1e07345dd.png)

## Типы аутентификации пользователя

![](https://static-docs.nocobase.com/da4250c0cea343ebe470cbf7be4b12e4.png)

Добавив разные типы аутентификаторов, вы сможете включить соответствующие методы аутентификации в системе.

Кроме типов аутентификации, предоставляемых существующими плагинами, разработчики также могут расширять типы пользовательской аутентификации. Подробнее см. [Руководство для разработчиков](./dev/).
