# Аутентификация: DingTalk

## Введение

Плагин Аутентификация: DingTalk позволяет пользователям входить в NocoBase, используя свои аккаунты DingTalk.

## Активация плагина

![](https://static-docs.nocobase.com/202406120929356.png)

## Получение разрешений API в консоли разработчика DingTalk

См. <a href="https://open.dingtalk.com/document/orgapp/tutorial-obtaining-user-personal-information" target="_blank">Открытая платформа DingTalk - Реализация входа на сторонние веб-сайты</a>, чтобы создать приложение.

Перейдите в консоль управления приложением и включите "Информация о личном номере телефона" и "Разрешение на чтение личной информации из адресной книги".

![](https://static-docs.nocobase.com/202406120006620.png)

## Получение учетных данных из консоли разработчика DingTalk

Скопируйте Client ID и Client Secret.

![](https://static-docs.nocobase.com/202406120000595.png)

## Добавление аутентификации DingTalk в NocoBase

Перейдите на страницу управления плагинами пользовательской аутентификации.

![](https://static-docs.nocobase.com/202406112348051.png)

Добавить - DingTalk

![](https://static-docs.nocobase.com/202406112349664.png)

### Настройка

![](https://static-docs.nocobase.com/202406120016896.png)

- Автоматическая регистрация, если пользователь не существует – будет ли автоматически создаваться новый пользователь, если существующий пользователь не найден по номеру телефона.
- Client ID и Client Secret – заполните информацию, скопированную на предыдущем шаге.
- Redirect URL – URL обратного вызова, скопируйте его и перейдите к следующему шагу.

## Настройка URL обратного вызова в консоли разработчика DingTalk

Вставьте скопированный URL обратного вызова в консоль разработчика DingTalk.

![](https://static-docs.nocobase.com/202406120012221.png)

## Вход

Перейдите на страницу входа и нажмите кнопку под формой входа, чтобы инициировать вход через сторонний сервис.

![](https://static-docs.nocobase.com/202406120014539.png)
