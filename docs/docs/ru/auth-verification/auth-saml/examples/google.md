# Google Workspace

## Настройка Google в качестве поставщика удостоверений (IdP)

[Консоль администратора Google](https://admin.google.com/) - Приложения - Веб- и мобильные приложения

![](https://static-docs.nocobase.com/0812780b990a97a63c14ea8991959827.png)

После настройки приложения скопируйте **URL-адрес SSO**, **Идентификатор сущности** и **Сертификат**.

![](https://static-docs.nocobase.com/aafd20a74730e85411c0c8f368637e0.png)

## Добавление нового аутентификатора в NocoBase

Настройки плагина - Аутентификация пользователей - Добавить - SAML

![](https://static-docs.nocobase.com/5bc18c7952b8f15828e26bb07251a335.png)

Последовательно введите скопированную информацию:

- SSO URL: URL-адрес SSO
- Public Certificate: Открытый сертификат
- idP Issuer: Идентификатор сущности
- http: Отметьте, если вы тестируете локально по HTTP.

Затем скопируйте SP Issuer/EntityID и ACS URL из раздела Usage.

## Заполнение информации о поставщике услуг (SP) в Google

Вернитесь в консоль Google, на странице **Сведения о поставщике услуг** введите ранее скопированные URL-адрес ACS и Идентификатор сущности, а также установите флажок **Подписанный ответ**.

![](https://static-docs.nocobase.com/1536268bf8df4a5ebc72384317172191.png)

![](https://static-docs.nocobase.com/c7de1f8b84c1335de110e5a7c96255c4.png)

В разделе **Сопоставление атрибутов** добавьте сопоставления для соответствующих атрибутов.

![](https://static-docs.nocobase.com/27180f2f46480c3fee3016df86d6fdb8.png)
