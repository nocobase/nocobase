# Верификация: SMS

## Введение

SMS-код верификации — это встроенный тип верификации, который используется для генерации одноразового динамического пароля (OTP) и его отправки пользователю по SMS.

## Добавление SMS-верификатора

Перейдите на страницу управления верификацией.

![](https://static-docs.nocobase.com/202502271726791.png)

Добавить - SMS OTP

![](https://static-docs.nocobase.com/202502271726056.png)

## Конфигурация администратора

![](https://static-docs.nocobase.com/202502271727711.png)

В настоящее время поддерживаются следующие SMS-провайдеры:

- <a href="https://www.aliyun.com/product/sms" target="_blank">Aliyun SMS</a>
- <a href="https://cloud.tencent.com/product/sms" target="_blank">Tencent Cloud SMS</a>

При настройке SMS-шаблона в панели администратора провайдера услуг вам необходимо зарезервировать параметр для кода верификации.

- Пример конфигурации Aliyun: `Ваш код верификации: ${code}`

- Пример конфигурации Tencent Cloud: `Ваш код верификации: {1}`

Разработчики также могут расширить поддержку других SMS-провайдеров в виде плагинов. См.: [Расширение поддержки SMS-провайдеров](./dev/sms-type)

## Привязка пользователя

После добавления верификатора пользователи могут привязать номер телефона в личном кабинете управления верификацией.

![](https://static-docs.nocobase.com/202502271737016.png)

![](https://static-docs.nocobase.com/202502271737769.png)

![](https://static-docs.nocobase.com/202502271738515.png)

После успешной привязки можно выполнить верификацию личности в любом сценарии верификации, к которому привязан данный верификатор.

![](https://static-docs.nocobase.com/202502271739607.png)

## Отвязка пользователя

Для отвязки номера телефона требуется верификация с использованием уже привязанного метода верификации.

![](https://static-docs.nocobase.com/202502282103205.png)
