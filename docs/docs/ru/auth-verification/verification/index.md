# Верификация

:::info{title=Примечание}
Начиная с версии `1.6.0-alpha.30` исходная функция «Код верификации» была обновлена до «Управления верификацией», которая позволяет управлять и интегрировать различные методы верификации пользователей. Когда пользователи привязывают соответствующий метод верификации, они могут выполнять верификацию личности по мере необходимости. Функция планируется к стабильной поддержке, начиная с версии `1.7.0`.
:::



## Введение

**Центр управления верификацией поддерживает управление и интеграцию различных методов верификации пользователей.** Например:

- Код верификации по SMS — предоставляется плагином верификации по умолчанию. См.: [Верификация: SMS](./sms)
- TOTP-аутентификатор — см.: [Верификация: TOTP-аутентификатор](../verification-totp/)

Разработчики также могут расширять другие типы верификации через плагины. См.: [Расширение типов верификации](./dev/type)

**Пользователи могут выполнять верификацию личности по мере необходимости после привязки соответствующего метода верификации.** Например:

- Вход по SMS с верификацией — см.: [Аутентификация: SMS](../auth-sms/index.md)
- Двухфакторная аутентификация (2FA) — см.: [Двухфакторная аутентификация (2FA)](../2fa/)
- Вторичная верификация для операций, связанных с риском — в будущем

Разработчики также могут интегрировать верификацию личности в другие необходимые сценарии, расширяя плагины. См.: [Расширение сценариев верификации](./dev/scene)

**Отличия и связи между модулем верификации и модулем аутентификации пользователей:** Модуль аутентификации пользователей в первую очередь отвечает за аутентификацию личности при входе пользователя, при этом процессы вроде SMS-входа и двухфакторной аутентификации используют верификаторы, предоставляемые модулем верификации; в то же время модуль верификации обрабатывает верификацию личности для различных операций повышенного риска, и вход пользователя является одним из таких сценариев.
