# Верификация

:::info{title=Совет}
Начиная с версии `1.6.0-alpha.30`, исходная функция **кода верификации** была обновлена до **Управления верификацией**. Теперь она поддерживает управление и интеграцию различных методов верификации пользователя. После привязки соответствующего метода верификации пользователи смогут подтверждать свою личность в необходимых сценариях. Стабильная поддержка этой функции планируется начиная с версии `1.7.0`.
:::



## Введение

**Центр управления верификацией поддерживает управление и интеграцию различных методов верификации пользователя.** Например:

- SMS-код верификации – Предоставляется плагином верификации по умолчанию. См. также: [Верификация: SMS](./sms)
- TOTP-аутентификатор – См. также: [Верификация: TOTP-аутентификатор](../verification-totp/)

Разработчики также могут расширять другие типы верификации с помощью плагинов. См. также: [Расширение типов верификации](./dev/type)

**Пользователи могут выполнять проверку личности в необходимых сценариях после привязки соответствующего метода верификации.** Например:

- Вход по SMS-коду верификации – См. также: [Аутентификация: SMS](./sms)
- Двухфакторная аутентификация (2FA) – См. также: [Двухфакторная аутентификация (2FA)](../2fa)
- Вторичная верификация для рискованных операций – Поддержка в будущем.

Разработчики также могут интегрировать проверку личности в другие необходимые сценарии, расширяя плагины. См. также: [Расширение сценариев верификации](./dev/scene)

**Различия и взаимосвязь между модулем верификации и модулем аутентификации пользователя:** Модуль аутентификации пользователя в основном отвечает за проверку личности при входе пользователя в систему, при этом такие процессы, как вход по SMS и двухфакторная аутентификация, зависят от верификаторов, предоставляемых модулем верификации. В свою очередь, модуль верификации занимается проверкой личности для различных рискованных операций, и вход пользователя в систему является одним из таких сценариев.
