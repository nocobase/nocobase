# Аутентификация: LDAP

## Введение

Плагин Аутентификация: LDAP следует стандарту протокола LDAP (Lightweight Directory Access Protocol), позволяя пользователям входить в NocoBase, используя учетные данные своего LDAP-сервера.

## Конфигурация

### Базовая конфигурация

<img src="https://static-docs.nocobase.com/202405101605728.png"/>

- Автоматически регистрировать пользователя, если он не существует — будет ли автоматически создаваться новый пользователь, если не найден подходящий существующий пользователь.
- LDAP URL — URL LDAP-сервера
- Bind DN — DN, используемый для проверки подключения к серверу и поиска пользователей
- Bind password — пароль для DN подключения
- Тест подключения — нажмите кнопку, чтобы проверить подключение к серверу и валидировать подключениие DN.

### Конфигурация поиска

<img src="https://static-docs.nocobase.com/202405101609984.png"/>

- Точка начала поиска — DN, используемый для поиска пользователей
- Фильтр поиска — условие фильтрации при поиске пользователей; используйте `{{account}}` для обозначения учетной записи пользователя, используемой при входе
- Область поиска — `Базовый объект`, `Один уровень`, `Поддерево`, по умолчанию `Поддерево`
- Максимальное число результатов — размер страницы поиска

### Сопоставление атрибутов

<img src="https://static-docs.nocobase.com/202405101612814.png"/>

- Использовать это поле для привязки пользователя — поле для привязки к существующим пользователям. Выберите `Имя пользователя`, если вход в учетную запись осуществляется по имени пользователя, или `email`, если по email-адресу. По умолчанию используется Имя пользователя.
- Сопоставление атрибутов — сопоставление атрибутов пользователя с полями в таблице пользователей NocoBase.
