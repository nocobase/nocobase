# Аутентификация: LDAP

## Введение

Плагин Аутентификация: LDAP соответствует стандарту протокола LDAP (Lightweight Directory Access Protocol), позволяя пользователям входить в NocoBase, используя свои учетные данные LDAP-сервера.

## Настройка

### Основные настройки

<img src="https://static-docs.nocobase.com/202405101605728.png"/>

- Автоматическая регистрация, если пользователь не существует – Автоматически создавать нового пользователя, если не найден соответствующий существующий пользователь.
- LDAP URL – Адрес LDAP-сервера.
- Bind DN – DN, используемый для проверки подключения к серверу и поиска пользователей.
- Bind password – Пароль для Bind DN.
- Проверить подключение – Нажмите кнопку, чтобы проверить подключение к серверу и валидность Bind DN.

### Настройки поиска

<img src="https://static-docs.nocobase.com/202405101609984.png"/>

- Search DN – DN, используемый для поиска пользователей.
- Search filter – Условие фильтрации для поиска пользователей, где `{{account}}` представляет учетную запись пользователя, используемую при входе.
- Область поиска (Scope) – `Base`, `One level`, `Subtree`, по умолчанию `Subtree`.
- Ограничение размера (Size limit) – Размер страницы поиска.

### Сопоставление атрибутов

<img src="https://static-docs.nocobase.com/202405101612814.png"/>

- Использовать это поле для привязки пользователя – Поле, используемое для привязки к существующим пользователям. Выберите «имя пользователя», если учетная запись для входа является именем пользователя, или «электронная почта», если это адрес электронной почты. По умолчанию используется имя пользователя.
- Карта атрибутов (Attribute map) – Сопоставление атрибутов пользователя с полями в таблице пользователей NocoBase.
