# SSO приложений

SSO приложений упрощает вход пользователей в дочерние приложения в режиме Multi-app.

После включения, когда пользователь входит в дочернее приложение из основного приложения или переключается между дочерними приложениями, система пытается автоматически войти в целевое приложение как текущий пользователь. Пользователю не нужно вводить логин и пароль в каждом приложении.

## Сценарии использования

SSO приложений подходит, когда:

- Основное приложение используется как единая точка входа в разные бизнес-приложения
- Система разделена на несколько бизнес-приложений, но вход должен оставаться непрерывным
- Пользователи часто переключаются между дочерними приложениями
- Учетные записи сопоставляются между приложениями по одинаковому имени пользователя

## Включение SSO

Откройте "App Supervisor", создайте или отредактируйте дочернее приложение и включите "App SSO" в "Authentication configuration".

После этого дочернее приложение сможет запускать автоматический вход из основного приложения или переключателя.

> После изменения настроек аутентификации обычно нужно перезапустить дочернее приложение.

## Автоматическая регистрация

Если соответствующего пользователя нет в целевом дочернем приложении, можно включить "Automatically sign up when user does not exist".

При первом входе через SSO система создаст базового пользователя в дочернем приложении на основе данных пользователя из основного приложения.

Сопоставление пользователей в основном выполняется по имени пользователя:

- Если имя совпадает в основном и дочернем приложении, выполняется вход в соответствующего пользователя
- Если имени нет в дочернем приложении, пользователь создается только при включенной автоматической регистрации
- Если автоматическая регистрация отключена, администратор должен заранее создать пользователя

Роли и права после создания определяются настройками самого дочернего приложения.

## Что запускает автоматический вход

SSO запускается в основном через:

- Вход в дочернее приложение из основного приложения
- Переключатель приложений в левом верхнем углу
- Переход из одного дочернего приложения в другое

Прямой доступ к странице входа или собственному адресу дочернего приложения не принуждает использовать сессию основного приложения. Так сохраняются собственные способы входа дочернего приложения.

## Частые вопросы

### SSO включен, но автоматического входа нет?

Проверьте:

- Включен ли App SSO для дочернего приложения
- Было ли дочернее приложение перезапущено
- Вошел ли пользователь из основного приложения или переключателя
- Есть ли пользователь с тем же именем в дочернем приложении
- Если пользователя нет, включена ли автоматическая регистрация

### Почему прямой доступ не выполняет автоматический вход?

Это ожидаемое поведение. При прямом доступе дочернее приложение может использовать собственный способ входа.
