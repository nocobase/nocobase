# Мультипортал

## Что такое портал

Портал используется для предоставления нескольких точек входа в рамках одного приложения.

Каждый портал может независимо иметь:

- Страницы
- Меню
- Структуру навигации
- Макет
- Настройки прав доступа

Плагин Мультипортал предоставляет следующие возможности:

- Управление порталами
- Переключение порталов
- Управление правами доступа к порталам

Благодаря этим возможностям можно предоставлять разный пользовательский опыт для разных ролей, при этом используя одни и те же данные и бизнес-возможности.

## Зачем нужны порталы

В реальных бизнес-сценариях разные роли часто требуют разных интерфейсов.

Например, в системе управления розничной торговлей:

```text
Система управления розничной торговлей

├─ Портал головного офиса
├─ Портал магазина
├─ Портал дистрибьютора
└─ Мобильный портал
```

Сотрудники головного офиса сосредоточены на:

- Управлении товарами
- Управлении запасами
- Аналитике данных

Сотрудники магазина сосредоточены на:

- Кассе
- Инвентаризации
- Обработке заказов

Дистрибьюторы сосредоточены на:

- Закупках
- Сверке
- Статусе отгрузки

Хотя все используют одну и ту же систему, разным ролям не нужно видеть одни и те же меню и страницы.

Именно эту задачу и решают порталы.

## Связь между порталами и меню

У каждого портала есть собственное дерево меню.

Меню в разных порталах не влияют друг на друга.

Например:

```text
Портал головного офиса
├─ Управление товарами
├─ Управление цепочкой поставок
└─ Аналитика данных

Портал магазина
├─ Касса
├─ Управление заказами
└─ Инвентаризация
```

## Связь между порталами и страницами

Страницы принадлежат соответствующим порталам.

Одна и та же страница также может отображаться только в определенных порталах.

Это позволяет проектировать совершенно разные рабочие процессы для разных ролей.

## Связь между порталами и правами доступа

Для самих порталов можно настраивать права доступа.

Только авторизованные пользователи могут получить доступ к соответствующему порталу.

Неавторизованные порталы:

- Не появляются в списке переключателя
- Недоступны для прямого доступа

## Управление порталами

После включения плагина Мультипортал система по умолчанию предоставляет два встроенных портала:

| Портал | Путь | Назначение |
|----------|----------|----------|
| Desktop | `/v/admin` | Вход для настольной версии |
| Mobile | `/v/mobile` | Вход для мобильной версии |

## Создание портала

Помимо встроенных порталов, вы можете создавать дополнительные порталы в соответствии с потребностями бизнеса.

Например:

- Портал магазина
- Портал дистрибьютора
- Портал службы поддержки
- Портал аналитики данных

После создания можно настроить:

- Страницы
- Меню
- Права доступа
- Навигацию

## Права доступа к порталам

Вы можете управлять тем, к каким порталам пользователь может получить доступ, с помощью ролевых прав.

Неавторизованные порталы не отображаются в списке переключателя порталов, и пользователи не могут напрямую открыть эти входы.
