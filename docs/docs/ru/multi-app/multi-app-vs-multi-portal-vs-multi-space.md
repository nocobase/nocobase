# Мультипортал, Мульти-приложения и Мультипространство

NocoBase предоставляет три возможности: Мультипортал, Мульти-приложения и Мультипространство.

Они решают задачи на разных уровнях и могут использоваться как по отдельности, так и вместе.

## Ключевые различия

| Возможность | Мультипортал | Мульти-приложения | Мультипространство |
|------|------|------|------|
| Какую задачу решает | Предоставляет несколько точек входа | Разделяет бизнес на несколько систем | Изолирует бизнес-данные |
| Основной фокус | Откуда пользователь входит | Как разделена система | Кому принадлежат данные |
| Данные | Общие | Независимые по умолчанию | Изолированные |
| Страницы и меню | Независимые | Независимые | Общие |
| Конфигурация плагинов | Общая | Независимая | Общая |
| Пользовательская система | Общая | Может быть общей через SSO | Общая |
| Типичные сценарии | Разным ролям нужны разные входы | Разным бизнес-направлениям нужна независимая работа | Несколько организаций, магазинов или арендаторов |
| Можно комбинировать | Да | Да | Да |

## Мультипортал

Мультипортал предоставляет несколько точек входа в рамках одного приложения.

Например:

```text
ERP-приложение

├─ Админ-портал (/v/admin)
├─ Портал магазина (/v/store)
├─ Портал дистрибьютора (/v/dealer)
└─ Мобильный портал (/v/mobile)
```

Особенности:

- Используется одно и то же приложение
- Используются общие данные
- Используется общая конфигурация плагинов
- Страницы и меню можно настраивать независимо

Подходит для сценариев, где разным ролям нужны разные точки входа, например:

- Администраторы
- Сотрудники
- Клиенты
- Дистрибьюторы

## Мульти-приложения

Мульти-приложения позволяют разделить бизнес на несколько независимых приложений.

Например:

```text
Групповая система

├─ CRM
├─ ERP
├─ OA
└─ Аналитика
```

Особенности:

- Каждое приложение управляется независимо
- Независимая конфигурация плагинов
- Независимое подключение к базе данных
- Независимые обновления и сопровождение

Подходит для:

- Разделения крупных бизнес-систем
- Совместной разработки несколькими командами
- Массового создания приложений для SaaS-платформ
- Независимых приложений для разных клиентов

## Мультипространство

Мультипространство изолирует бизнес-данные в рамках одного приложения.

Например:

```text
Приложение для управления магазинами

Пространства
├─ Магазин Пекин
├─ Магазин Шанхай
└─ Магазин Шэньчжэнь
```

Особенности:

- Общие страницы
- Общие меню
- Общие рабочие процессы
- Общая конфигурация
- Изолированные данные

Для таблиц, в которых включено поле пространства, система автоматически фильтрует данные в соответствии с текущим пространством.

С точки зрения пользователя:

- Магазин Пекин видит только свои данные
- Магазин Шанхай видит только свои данные
- Магазин Шэньчжэнь видит только свои данные

Но все магазины по-прежнему используют одну и ту же систему.

## Связь между тремя возможностями

Эти три возможности не конфликтуют друг с другом. Они работают на разных уровнях.

Их можно использовать вместе:

```text
Групповая система

CRM-приложение
├─ Админ-портал
├─ Портал продаж
└─ Клиентский портал

Пространства
├─ Филиал Пекин
├─ Филиал Шанхай
└─ Филиал Шэньчжэнь
```

Концептуально:

```text
Портал
    ↓
Откуда пользователь входит в систему

Приложение
    ↓
Как разделена система

Пространство
    ↓
Кому принадлежат данные
```

## Как выбрать

Если вам нужно только предоставить разные входы для разных ролей, выберите **Мультипортал**.

Если вы хотите разделить бизнес на несколько независимых систем, выберите **Мульти-приложения**.

Если вы хотите изолировать данные разных организаций или арендаторов в рамках одной системы, выберите **Мультипространство**.

В реальных проектах эти три возможности обычно используются вместе, а не как взаимозаменяемые решения.

Кратко:

> Мультипортал решает задачу входов, Мульти-приложения решают задачу разделения системы, а Мультипространство решает задачу изоляции данных.
