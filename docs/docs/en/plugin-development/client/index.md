# Overview

NocoBase client plugin development provides a variety of features and capabilities to help developers customize and extend the front-end functionality of NocoBase. The following are the main capabilities of NocoBase client plugin development and their related chapters:

| Feature Module | Description | Related Chapters |
|---|---|---|
| **Plugin Class** | Create and manage client plugins to extend front-end functionality | [plugin.md](plugin.md) |
| **Router Management** | Customize front-end routes for page navigation and redirection | [router.md](router.md) |
| **Resource Operations** | Manage front-end resources, handle data retrieval and operations | [resource.md](resource.md) |
| **Request Handling** | Customize HTTP requests to handle API calls and data transfer | [request.md](request.md) |
| **Context Management** | Get and use application context to access global state and services | [context.md](context.md) |
| **Access Control** | Implement front-end access control for pages and features | [acl.md](acl.md) |
| **Data Source Management** | Manage and use multiple data sources, enabling switching and access | [data-source-manager.md](data-source-manager.md) |
| **Styles & Themes** | Customize styles and themes for UI customization and enhancement | [styles-themes.md](styles-themes.md) |
| **Internationalization** | Integrate multi-language support for internationalization and localization | [i18n.md](i18n.md) |
| **Logging** | Customize log formats and output to improve debugging and monitoring | [logger.md](logger.md) |
| **Writing Test Cases** | Write and run test cases to ensure plugin stability and functionality | [test.md](test.md) |
