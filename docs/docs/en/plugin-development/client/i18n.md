# I18n Internationalization

NocoBase plugins support both frontend and backend multilingual internationalization (i18n). Through a unified mechanism, you can easily implement multilingual content in plugins.

## Multilingual File Management

The multilingual files for a plugin are stored in the `src/locale` directory. It is recommended to name them by language file, for example:

```bash
|- /plugin-hello
  |- /src
    |- /locale
      |- en-US.json   # English language
      |- zh-CN.json   # Chinese language
```

Each language file exports a JSON object containing all the translation entries for that language, for example:

```json
// zh-CN.json
{
  "Hello": "你好",
  "World": "世界",
  "Enter your name": "请输入你的名字",
  "Your name is {{name}}": "你的名字是 {{name}}"
}
```

```json
// en-US.json
{
  "Hello": "Hello",
  "World": "World",
  "Enter your name": "Enter your name",
  "Your name is {{name}}": "Your name is {{name}}"
}
```

> When adding a language file for the first time, you need to restart the application for it to take effect. You can verify if the translation entries are effective through the API:
> http://localhost:13000/api/app:getLang?locale=zh-CN
