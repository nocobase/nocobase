# 从钉钉同步用户数据

## 介绍

**钉钉**插件提供用户数据同步来源类型 **dingtalk**，支持：

- **全量同步**：按部门树拉取部门与成员（基于钉钉 `oapi.dingtalk.com` 通讯录相关接口）。
- **增量同步**（可选）：在钉钉开放平台配置**事件订阅** HTTP 回调后，根据用户/部门变更事件更新 NocoBase。

## 准备钉钉应用

在钉钉开放平台创建企业内部应用（或符合你组织类型的应用），获取：

- **Client ID**（AppKey）
- **Client Secret**（AppSecret）

并为应用申请通讯录只读等相关权限（以钉钉控制台为准）。

## 在 NocoBase 添加同步数据源

![](https://static-docs.nocobase.com/20260513231840.png)

路径：**用户和权限 → 同步 → 添加**，同步来源类型选择 **钉钉**，填写：

- **Client ID**、**Client Secret**：与上述应用一致。
- **Token**、**EncodingAESKey**（可选，用于事件回调）：与钉钉事件订阅中配置一致。

保存后，可复制页面上的 **事件订阅回调地址**，填入钉钉开放平台事件订阅配置，并订阅通讯录相关事件（如用户新增/变更/离职、部门新增/变更/删除等）。

## 执行同步

在数据源列表点击「同步」进行全量拉取。成功后可在用户、部门列表中查看数据。

## 同步字段

钉钉同步会拉取**部门**和**用户**两类数据，并转换为用户数据同步插件的标准字段后写入 NocoBase。同步记录使用钉钉侧 ID 作为来源唯一标识，用于后续增量更新、删除和关联关系维护。

### 部门字段

| 钉钉字段 | NocoBase 同步字段 | 说明 |
| --- | --- | --- |
| 部门 ID | `uid` | 来源唯一标识，用于识别同一个钉钉部门，不会直接作为 NocoBase 部门表的主键。 |
| 部门名称 | `title` | 写入 NocoBase 部门名称。 |
| 父级部门 ID | `parentUid` | 用于维护 NocoBase 部门的上下级关系。 |
| 删除状态 | `isDeleted` | 增量同步中用于删除或解除对应部门记录。 |

### 用户字段

| 钉钉字段 | NocoBase 同步字段 | 说明 |
| --- | --- | --- |
| 用户 ID | `uid` | 来源唯一标识，用于识别同一个钉钉用户，不会直接作为 NocoBase 用户表的主键。 |
| 姓名 | `nickname` | 写入 NocoBase 用户昵称。 |
| 手机号 | `phone` | 写入 NocoBase 用户手机号，可用于匹配已有用户。 |
| 邮箱 | `email` | 写入 NocoBase 用户邮箱，可用于匹配已有用户。 |
| 所属部门 ID 列表 | `departments` | 维护用户所属部门关系。同步时需先存在对应部门记录。 |
| 部门负责人信息 | `departments[].isOwner` | 标记用户是否为对应部门负责人。 |
| 主属部门信息 | `departments[].isMain` | 标记用户的主属部门。 |
| 删除/离职状态 | `isDeleted` | 增量同步中用于删除或解除对应用户记录；拥有 root 角色的用户不会被删除。 |

## 增量说明

启用事件订阅并正确配置 Token、EncodingAESKey 后，钉钉推送的加密事件会在服务端解密并调用用户数据同步资源写入变更。若未配置回调，仍可通过定时或手动全量同步保持一致。

## 与认证器的关系

同步来源使用的 Client ID/Secret 可与钉钉**认证器**相同（同一应用），也可单独使用具备通讯录权限的应用；按安全与权限最小化原则配置即可。
