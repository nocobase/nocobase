# 从钉钉同步用户数据

## 介绍

**钉钉**插件提供用户数据同步来源类型 **dingtalk**，支持：

- **全量同步**：按部门树拉取部门与成员（基于钉钉 `oapi.dingtalk.com` 通讯录相关接口）。
- **增量同步**（可选）：在钉钉开放平台配置**事件订阅** HTTP 回调后，根据用户/部门变更事件更新 NocoBase。

## 准备钉钉应用

在钉钉开放平台创建企业内部应用（或符合你组织类型的应用），获取：

- **Client ID**（AppKey）
- **Client Secret**（AppSecret）

并为应用申请通讯录只读等相关权限（以钉钉控制台为准）。

## 在 NocoBase 添加同步数据源

![](https://static-docs.nocobase.com/20260513231840.png)

路径：**用户和权限 → 同步 → 添加**，同步来源类型选择 **钉钉**，填写：

- **Client ID**、**Client Secret**：与上述应用一致。
- **Token**、**EncodingAESKey**（可选，用于事件回调）：与钉钉事件订阅中配置一致。

保存后，可复制页面上的 **事件订阅回调地址**，填入钉钉开放平台事件订阅配置，并订阅通讯录相关事件（如用户新增/变更/离职、部门新增/变更/删除等）。

## 执行同步

在数据源列表点击「同步」进行全量拉取。成功后可在用户、部门列表中查看数据。

## 增量说明

启用事件订阅并正确配置 Token、EncodingAESKey 后，钉钉推送的加密事件会在服务端解密并调用用户数据同步资源写入变更。若未配置回调，仍可通过定时或手动全量同步保持一致。

## 与认证器的关系

同步来源使用的 Client ID/Secret 可与钉钉**认证器**相同（同一应用），也可单独使用具备通讯录权限的应用；按安全与权限最小化原则配置即可。
