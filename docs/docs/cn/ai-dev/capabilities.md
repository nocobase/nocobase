# 支持的能力

这里列出了 AI 目前能帮你做的所有事情。每个能力都附有提示词示例，你可以直接复制、改改需求描述就能用。

## 快速索引

| 我想要…… | AI 能帮你做 |
|----------|-----------|
| 创建一个新插件 | 生成完整脚手架，包括前后端目录结构 |
| 定义数据表 | 生成 Collection 定义，支持所有字段类型和关联关系 |
| 做一个自定义区块 | 生成 BlockModel + 配置面板 + 注册到「添加区块」菜单 |
| 做一个自定义字段 | 生成 FieldModel + 绑定到字段接口 |
| 添加自定义操作按钮 | 生成 ActionModel + 弹窗/抽屉/确认框 |
| 做一个插件设置页 | 生成前端表单 + 后端 API + 存储 |
| 写自定义 API | 生成 Resource Action + 路由注册 + ACL 配置 |
| 配置权限 | 生成 ACL 规则，按角色控制访问 |
| 多语言支持 | 自动生成中英文语言包 |
| 写升级脚本 | 生成 Migration，支持 DDL 和数据迁移 |

## 插件脚手架

AI 可以根据你的需求描述，生成一个完整的 NocoBase 插件目录结构——包括前后端入口文件、类型定义和基础配置。

提示词示例：

```
帮我创建一个 NocoBase 插件，插件名叫 @my-scope/plugin-todo
```

AI 会执行 `yarn pm create @my-scope/plugin-todo` 并生成标准目录：

```
packages/plugins/@my-scope/plugin-todo/
├── src/
│   ├── server/
│   │   └── plugin.ts
│   ├── client-v2/
│   │   └── plugin.tsx
│   └── locale/
│       ├── zh-CN.json
│       └── en-US.json
├── package.json
└── ...
```

## 数据表定义

AI 支持生成所有 NocoBase 字段类型的 Collection 定义，包括关联关系（一对多、多对多等）。

提示词示例：

```
帮我在插件里定义一张"订单"表，字段包括：订单编号（自增）、客户名称（字符串）、
金额（小数）、状态（单选：待处理/处理中/已完成）、创建时间。
订单和客户是多对一关系。
```

AI 会生成 `defineCollection` 定义，包含字段类型、默认值、关联配置等。

## 自定义区块

区块是 NocoBase 前端最核心的扩展方式。AI 可以帮你生成区块模型、配置面板和菜单注册。

提示词示例：

```
帮我做一个"数据统计"区块，在区块里用 Ant Design 的 Statistic 组件展示
订单总数、总金额和本月新增订单数。数据从后端 API 获取。
```

AI 会生成：

- `BlockModel` 或 `CollectionBlockModel`（取决于是否关联数据表）
- `registerFlow` 配置面板
- 注册到「添加区块」菜单

## 自定义字段

如果 NocoBase 内置的字段组件不满足需求，AI 可以帮你做一个自定义的。

提示词示例：

```
帮我做一个"评分"字段，用星星图标展示，支持 1-5 分，点击可以修改评分值。
```

AI 会生成 `FieldModel`，通过 `bindModelToInterface` 绑定到 NocoBase 的字段接口上。

## 自定义操作

操作按钮可以触发弹窗、抽屉、确认框，也可以直接调用 API。

提示词示例：

```
帮我在订单表格上加一个"批量导出"按钮，点击后调用后端 API 导出选中的订单为 CSV 文件。
```

AI 会生成 `ActionModel`、弹窗/确认逻辑和后端导出 API。

## 插件设置页

很多插件需要一个设置页面让用户配置参数——比如第三方服务的 API Key、Webhook 地址等。

提示词示例：

```
帮我做一个插件设置页，让用户配置以下参数：
- API Key（字符串，必填）
- Webhook URL（字符串，选填）
- 是否启用通知（开关）
配置保存到数据库，前端用 Ant Design 表单。
```

AI 会生成前端表单组件、后端存储 API 和 ACL 配置。

## 自定义 API

如果内置的 CRUD 接口不够用，AI 可以帮你写自定义的 REST API。

提示词示例：

```
帮我写一个 API，路径是 myPlugin:statistics，返回订单的统计数据
（总数、总金额、按状态分组的数量）。只有登录用户可以访问。
```

AI 会生成 Resource Action 处理函数、路由注册和 ACL 配置。

## 权限配置

AI 会自动为生成的 API 和资源配置合理的 ACL 规则。你也可以明确指定权限需求：

提示词示例：

```
这个插件的权限要求：普通用户只能看自己创建的订单，管理员可以看所有订单，
只有管理员可以删除订单。
```

## 国际化

AI 默认会生成中英文两个语言包（`zh-CN.json` 和 `en-US.json`），你不需要额外提。

如果有其他语言需求：

```
帮我额外生成日文语言包
```

## 升级脚本

当插件需要更新数据库结构或迁移数据时，AI 可以帮你生成 Migration 脚本。

提示词示例：

```
我的插件要升级，需要给"订单"表新增一个"备注"字段（长文本，选填），
并且把现有订单的备注字段默认填上"无"。
```

AI 会生成带版本号的 Migration 文件，包含 DDL 操作和数据迁移逻辑。
