# 多环境管理

## 简介

多环境管理 skill 负责 NocoBase 的环境检查、安装部署、升级和故障诊断——把 NocoBase 从「还没装」带到「可以登录使用」，并维护 CLI 运行环境。

## 安装

```bash
npx skills add nocobase/skills --skill nocobase-env-bootstrap -y
```

## 能力范围

可以做：

- 环境预检：自动检测依赖、端口、网络和路径
- 安装部署：支持 Docker、create-nocobase-app、Git 三种方式
- 单实例升级：带备份确认和版本校验的安全升级流程
- MCP 连接引导：安装后自动引导 MCP 连通性验证
- 故障诊断：采集诊断信息，按高频故障给最小修复方案
- CLI 环境管理：新增、切换、查看应用环境配置

不能做：

- 不能处理跨环境发布（用发布管理 skill）
- 不能自动做破坏性删除（比如删库、删卷），必须你明确确认
- 不能替代手动操作所有场景，遇到未知错误会如实展示

## 提示词示例

### 场景 A：快速安装

```
帮我用 Docker 安装 NocoBase，端口用 13000
```

skill 会先做环境预检，然后按 Docker 流程安装并启动，最后给出登录地址和默认账号。

### 场景 B：升级实例

```
把当前实例升级到最新版本
```

升级前会要求确认备份状态，确认后才执行。升级完成后自动做可用性检查。

### 场景 C：故障诊断

```
应用启动失败了，帮我诊断一下
```

会采集日志和环境信息，定位常见问题（端口冲突、权限不足、依赖缺失等）并给出修复方案。

## 常见问题

**安装完成后默认账号是什么？**

默认账号 `admin@nocobase.com`，密码 `admin123`。首次登录后建议立即修改密码。

**Docker 启动报端口冲突怎么办？**

换一个端口（比如 `port=14000`），或者先停掉占用 13000 端口的进程。skill 的预检阶段会主动提示端口冲突。
