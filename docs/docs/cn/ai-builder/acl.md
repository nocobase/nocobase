# 权限配置

## 简介

权限配置 skill 用于通过自然语言管理 NocoBase 的角色、权限策略、用户绑定和 ACL 风险评估——你描述业务目标，它来选择命令和参数。

使用前需要确保 NocoBase CLI 环境已就绪。

## 安装

```bash
npx skills add nocobase/skills --skill nocobase-acl-manage -y
```

## 能力范围

可以做：

- 创建角色并设置初始权限基线
- 切换全局角色模式（独立模式 / 联合模式）
- 给数据表配置动作权限和数据范围
- 批量绑定或解绑用户与角色的关系
- 输出角色级、用户级、系统级风险评估报告

不能做：

- 不能绕过 CLI 直接调用 REST API 或操作数据库
- 不能处理非 ACL 相关的系统配置（比如数据建模、工作流）

## 提示词示例

### 场景 A：创建角色

```
帮我创建一个 sales 角色
```

创建后会自动带上默认的只读基线权限，然后引导你做后续权限配置。

### 场景 B：配置数据表权限

```
给 sales 角色配置客户表的查看和编辑权限，数据范围限制为自己创建的
```

skill 会自动解析「客户表」对应的真实集合名，确认数据源、动作和范围后再写入。

### 场景 C：风险评估

```
评估一下 admin 角色的权限风险
```

会输出风险评分、影响范围说明和改进建议。

## 常见问题

**配置了权限但不生效怎么办？**

先确认全局角色模式是否正确——如果用户同时拥有多个角色，联合模式和独立模式的行为差异很大。可以用 `global.role-mode.get` 查看当前模式。

**想一次给多个表配权限，能批量操作吗？**

目前每次 `permission.data-source.resource.set` 针对一个集合。不过你可以连续下达多条指令，skill 会按顺序逐个执行并校验。
