# 数据建模

## 简介

数据建模 skill 用于通过自然语言创建和管理 NocoBase 的数据表结构——建表、加字段、设关联关系等。

使用前需要确保目标数据源已在「数据源管理」中配置好。

## 安装

```bash
npx skills add nocobase/skills --skill nocobase-data-modeling -y
```

## 能力范围

可以做：

- 根据描述创建数据表，自动推断字段类型
- 为已有表添加、修改、删除字段
- 设置表间关联关系（一对多、多对多等）
- 支持多种表类型：普通表、树表、文件表、日历表、SQL 表、视图表、继承表
- 批量建表（比如「帮我建一套 CRM 需要的表」）

不能做：

- 不能操作外部数据源（比如已连接的 MySQL）的表结构
- 不能做数据迁移（改字段类型不会自动迁移已有数据）
- 不能直接操作 ACL 权限或页面配置

## 提示词示例

### 场景 A：创建数据表

```
帮我创建一张客户表，包含名称、联系电话、邮箱、所属行业
```

skill 会自动推断字段类型——名称用单行文本，电话用手机号，邮箱用邮箱字段，行业用下拉单选。

### 场景 B：添加关联关系

```
在订单表上添加一个「所属客户」字段，多对一关联到客户表
```

skill 会处理外键和反向字段的创建。

### 场景 C：使用公式字段

```
在订单表上添加一个总金额字段，用公式自动计算单价乘以数量
```

skill 会先确认公式引擎和语法，再写入字段配置。

## 常见问题

**建表时能自动创建系统字段吗？**

会的。`id`、`createdAt`、`createdBy`、`updatedAt`、`updatedBy` 这些系统字段由服务端自动生成，不需要手动指定。

**关联关系建错了怎么修改？**

建议先检查当前关联字段的外键和反向字段，再决定是修改还是删除重建。skill 会在变更后读回校验两端的关联状态。
