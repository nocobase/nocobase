# 工作流管理

## 简介

工作流管理 skill 用于创建、编辑、启用和诊断 NocoBase 工作流——从触发器选择到节点链搭建，再到执行结果排查，覆盖工作流使用的完整生命周期。

## 安装

```bash
npx skills add nocobase/skills --skill nocobase-workflow-manage -y
```

## 能力范围

可以做：

- 创建工作流：选择触发器类型，逐个添加处理节点
- 编辑工作流：修改触发器配置、增删改节点、移动和复制节点
- 版本管理：已执行过的版本会自动创建新修订，不影响历史记录
- 启用和手动执行工作流
- 诊断失败的执行：定位失败节点和错误信息

不能做：

- 不能设计数据模型（用数据建模 skill）
- 不能安装 MCP 或处理环境问题（用多环境管理 skill）
- 不能删除整个工作流（需要单独确认的高风险操作）
- 不能凭空编造节点类型或触发器类型

## 提示词示例

### 场景 A：创建新工作流

```
帮我编排一个订单创建之后自动扣减商品库存的工作流
```

Skill 会先确认触发器类型和节点链设计，确认后逐步创建。

### 场景 B：编辑已有工作流

```
在发货通知工作流里，审批通过后再加一个更新库存的节点
```

如果当前版本已执行过，会先创建新修订再编辑，不影响历史执行记录。

### 场景 C：诊断执行失败

```
发货通知工作流最近一次执行失败了，帮我看看哪里出了问题
```

会找到最近一次失败的执行，定位失败节点和错误信息，给出修复建议。

### 场景 D：分析或总结工作流逻辑

```
帮我分析一下"订单创建扣减商品库存"这个工作流的处理逻辑
```

## 常见问题

**工作流创建后为什么没有触发？**

新创建的工作流默认是禁用状态（`enabled: false`）。需要确认触发器配置正确后再手动启用。

**修改工作流后历史执行记录会受影响吗？**

不会。如果当前版本已有执行记录，skill 会自动创建新修订（revision），历史执行记录绑定在旧版本上，不受影响。
