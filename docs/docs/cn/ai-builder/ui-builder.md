# 界面配置

:::tip 前置条件

阅读本页前，请确保你已按照 [AI 搭建快速开始](./index.md) 安装了 NocoBase CLI 并完成了初始化。

:::

## 简介

界面配置 Skill 用于创建和编辑 NocoBase 的页面、区块、字段和操作配置——你用业务语言描述想要的页面，它来处理蓝图生成、区块布局和交互联动。


## 能力范围

可以做：

- 创建完整页面：表格、筛选表单、详情弹窗一步到位
- 编辑已有页面：添加区块、调整字段、配置弹窗、调整布局
- 设置交互联动：默认值、字段显隐、计算联动、操作按钮状态
- 使用模板复用：重复的弹窗和区块可以保存为模板
- 支持多页面任务：按顺序逐页搭建

不能做：

- 不能配置 ACL 权限（用[权限配置 Skill](./acl)）
- 不能设计数据表结构（用[数据建模 Skill](./data-modeling)）
- 不能编排工作流（用[工作流管理 Skill](./workflow)）
- 不能处理非现代页面（v1）的导航，只支持处理 v2 的页面。

## 提示词示例

### 场景 A：创建管理页面

```
帮我创建一个客户管理页面，包含姓名搜索框和客户表格，表格显示名称、电话、邮箱、创建时间
```

Skill 会先读取数据表字段，生成页面蓝图并写入。

![创建管理页面](https://static-docs.nocobase.com/20260420100608.png)


### 场景 B：配置弹窗

```
点击表格中的客户名称时弹出详情页，显示所有字段
```

会优先用字段弹窗（点击即弹出），而不是额外添加操作按钮。

![配置弹窗](https://static-docs.nocobase.com/20260420100641.png)

### 场景 C：设置联动规则

```
为弹窗 /admin/c0vc2vmkfll/view/cec3e7a69ac/filterbytk/1 里的编辑表单增加字段规则：
当用户id为1时，禁止编辑username
```

会通过配置联动规则实现，且不需要手动编写配置。

![设置联动规则](https://static-docs.nocobase.com/20260420100709.png)

### 场景 D：多页面搭建

```
帮我搭建一个用户管理系统，该系统有两个页面：用户管理页面和角色管理页面，它们在一个页面分组下。
```

会给出多页面的简单设计，人工调整和确认后即可进行搭建。

![多页面搭建](https://static-docs.nocobase.com/20260420100731.png)

## 常见问题

**页面创建后区块里没有数据怎么办？**

先确认对应的数据表里确实有记录。另外检查区块绑定的集合和数据源是否正确。也可以直接使用[数据建模 Skill](./data-modeling) 来创建模拟数据。

**弹窗里想放多个区块怎么办？**

可以在提示词中描述弹窗内容，比如「编辑弹窗里放一个表单和一个关联表格」。Skill 会生成包含多个区块的自定义弹窗布局。

**手动配置页面和 AI 配置页面会互相影响吗？**

如果手动配置和 AI 配置是同时进行的会互相影响，如果不是同一时间配置的则不会有影响。
