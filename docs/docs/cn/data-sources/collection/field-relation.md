# 关系字段

## 适用场景

关系字段用来表达不同数据表之间的连接。它不是简单的数据库文本列，而是一组关系配置，用来告诉 NocoBase 两张表通过哪些字段关联、页面应该如何选择和展示关联记录。

| 你要表达 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 当前表的一条记录属于另一张表的一条记录 | 多对一 | 订单属于客户、任务属于项目、联系人属于公司。 |
| 当前表的一条记录拥有多条子记录 | 一对多 | 客户拥有多张订单、项目拥有多个任务。 |
| 一条记录只关联另一条记录 | 一对一 | 员工和员工档案、用户和扩展资料。 |
| 两边都可以关联多条记录 | 多对多 | 用户和角色、学生和课程、产品和标签。 |
| 多对多关系保存为数组 | 多对多（数组） | 外部数据或特殊结构中已经用数组保存关联 ID。 |

## 从 NocoBase 创建

通过**[主数据库配置字段](../main/database.md#新增字段)**在页面新增关系字段。创建时重点确认关系类型、目标数据表、关联键和外键字段。

![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 关系字段的界面类型。可以选择「多对一」「一对多」「一对一」「多对多」「多对多（数组）」等。它决定当前表和目标表之间的关系结构。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「所属客户」「关联订单」「参与人员」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Target collection | 关联到哪张目标数据表。比如订单的「客户」字段通常关联客户表。 |
| Target key | 目标表中用于关联的字段，通常是主键或唯一字段。 |
| Foreign key | 当前表或目标表中保存关联值的字段。多对一通常在当前表保存外键，一对多通常在目标表保存外键。 |
| Through collection | 多对多关系使用的中间表。用于保存两边记录之间的多对多连接。 |
| Source key / Other key | 多对多关系中用来连接两边记录的字段。 |
| Description | 字段说明。适合写清楚关系含义、维护规则和是否允许为空。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库结构 | 默认长度 |
| --- | --- | --- | --- |
| 多对一 | `belongsTo` | 当前表外键字段 |  |
| 一对多 | `hasMany` | 目标表外键字段 |  |
| 一对一 | `hasOne` / `belongsTo` | 单侧外键字段和唯一约束 |  |
| 多对多 | `belongsToMany` | 中间表 |  |
| 多对多（数组） | `belongsToArray` | 数组或 JSON 字段 |  |

关系字段的真实数据库结构取决于关系类型。多对一通常会创建或使用外键字段，多对多通常会创建或使用中间表。

### 校验规则

关系字段常用校验规则包括必填、唯一和关联范围。关系字段会影响页面区块、权限、工作流和联动筛选，创建后不要随意修改关系类型和关联键。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | 多对一、一对一。 | 提交记录时必须选择关联记录。适合订单必须选择客户、任务必须选择项目等场景。 | 字段为空时，提交表单会提示必填错误。 |
| Unique | 一对一。 | 保证一条记录只被另一条记录关联。适合员工和员工档案、用户和扩展资料等场景。 | 选择已经被关联的记录时，保存可能失败或提示唯一性错误。 |
| Target filter | 多对一、多对多。 | 限制可选择的目标记录范围。适合只允许选择启用状态的客户、当前项目成员等场景。 | 下拉选择列表只显示符合条件的记录。 |

:::warning 注意

关系字段创建后不要随意修改关系类型、目标表和关联键。已有数据、页面区块、权限和工作流可能都依赖这些配置。

:::

## 从数据源映射

外部数据库中的关系通常通过外键字段、唯一约束、中间表或数组字段表达。NocoBase 同步字段后，需要先识别真实数据库结构，再配置为合适的关系字段。

同步后，需要进入数据表的「Configure fields」检查关系字段是否识别正确。重点确认关系类型、目标数据表、Target key、Foreign key、Through collection 和页面选择体验。

编辑字段配置不会修改外部数据库里的真实字段名、字段类型、外键约束或中间表结构。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步关系字段时，映射顺序通常是先读取数据库表结构、主键、唯一键和外键，再判断关系类型，最后映射为 NocoBase 的关系 Field type 和 Field interface。

| 数据库结构 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| 当前表有外键指向目标表主键或唯一键 | `belongsTo` | 多对一。 | 适合订单表中有 `customer_id`，指向客户表 `id` 的场景。 |
| 目标表有外键指向当前表主键或唯一键 | `hasMany` | 一对多。 | 适合客户表查看自己的订单列表。 |
| 外键字段带唯一约束 | `hasOne` / `belongsTo` | 一对一。 | 适合用户和用户扩展资料这类一对一关系。 |
| 独立中间表保存两边外键 | `belongsToMany` | 多对多。 | 适合用户和角色、产品和标签等关系。 |
| 数组或 JSON 保存多个目标 ID | `belongsToArray` | 多对多（数组）。 | 适合外部系统已经用数组保存关联 ID 的场景。 |

:::warning 注意

外部数据库是否存在外键约束，会影响 NocoBase 能否自动识别关系。即使数据库没有显式外键，也可以根据字段命名和业务含义手动配置关系字段，不过需要确认关联值确实能对应到目标表的主键或唯一字段。

:::
