# 关系字段

## 介绍

关系字段用来表达不同数据表之间的连接。它不是简单的数据库文本列，而是一组关系配置。

## 适用场景

| 你要表达 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 当前表的一条记录属于另一张表的一条记录 | 多对一 | 订单属于客户、任务属于项目、联系人属于公司。 |
| 当前表的一条记录拥有多条子记录 | 一对多 | 客户拥有多张订单、项目拥有多个任务。 |
| 一条记录只关联另一条记录 | 一对一 | 员工和员工档案、用户和扩展资料。 |
| 两边都可以关联多条记录 | 多对多 | 用户和角色、学生和课程、产品和标签。 |
| 多对多关系保存为数组 | 多对多（数组） | 外部数据或特殊结构中已经用数组保存关联 ID。 |

## 使用建议

关系字段适合表达业务对象之间的连接。比如订单里不要重复保存客户名称、客户电话、客户地址的所有信息；通常做法是用关系字段关联客户表，再按需要展示客户字段。

## 常见配置

关系字段常见配置包括：

| 配置 | 说明 |
| --- | --- |
| Target collection | 关联到哪张目标数据表。 |
| Target key | 目标表中用于关联的字段，通常是主键或唯一字段。 |
| Foreign key | 当前表或目标表中保存关联值的字段。 |
| Through collection | 多对多关系使用的中间表。 |
| Source key / Other key | 多对多关系中用来连接两边记录的字段。 |

:::warning 注意

关系字段创建后不要随意修改关系类型、目标表和关联键。已有数据、页面区块、权限和工作流可能都依赖这些配置。

:::
