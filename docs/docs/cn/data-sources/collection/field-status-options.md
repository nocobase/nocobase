# 状态和选项字段

## 适用场景

状态和选项字段适合保存固定范围内的值。创建前先确认这个值是简单的是 / 否、单个选项，还是可以同时选择多个选项。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 是 / 否 | 勾选、开关 | 是否启用、是否归档、是否已付款、是否需要回访。 |
| 单个固定选项 | 下拉菜单、单选框 | 订单状态、客户等级、工单优先级、合同类型。 |
| 多个固定选项 | 多选框、复选框组 | 客户标签、适用渠道、技能标签、产品特性。 |

## 从 NocoBase 创建

通过**[主数据库配置字段](../main/database.md#新增字段)**在页面新增状态和选项字段。创建时重点确认 Field interface、选项列表、默认值、是否必填，以及选项值是否会被 API、权限或工作流引用。

![add_field](https://static-docs.nocobase.com/add_field.png)
![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 状态和选项字段的界面类型。可以选择「勾选」「开关」「下拉菜单」「单选框」「多选框」「复选框组」等。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「订单状态」「客户等级」「是否启用」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。是 / 否通常使用 `boolean`，单选通常使用 `string`，多选通常使用数组或 JSON 结构。 |
| Options | 选项列表。每个选项通常包含显示标题、实际值和颜色。实际值会被 API、筛选、权限和工作流使用，创建后不建议随意修改。 |
| Default value | 默认值。适合给状态字段设置初始状态，比如「待处理」「未启用」。 |
| Required | 必填。适合订单状态、客户等级、审批状态等必须明确选择的字段。 |
| Description | 字段说明。适合写清楚每个状态的业务含义和流转规则。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库字段类型 | 默认长度 |
| --- | --- | --- | --- |
| 勾选 | `boolean` | `boolean` |  |
| 开关 | `boolean` | `boolean` |  |
| 下拉菜单 | `string` | `varchar` | 255 |
| 单选框 | `string` | `varchar` | 255 |
| 多选框 | `array` 或 `json` | `jsonb` |  |
| 复选框组 | `array` 或 `json` | `jsonb` |  |

这里的默认数据库字段类型以 PostgreSQL 为例。不同数据库最终生成的字段类型显示可能略有差异。

### 校验规则

状态和选项字段常用校验规则包括必填、选项范围和默认值。选项值会进入数据、筛选、权限和工作流，调整前需要确认影响范围。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | 下拉菜单、单选框、多选框、复选框组。 | 提交记录时必须选择。适合订单状态、客户等级、审批结果等核心字段。 | 字段为空时，提交表单会提示必填错误。 |
| Options | 下拉菜单、单选框、多选框、复选框组。 | 限制只能选择预设选项。适合值集合稳定的字段。 | 输入或导入不在选项范围内的值时，可能无法正确展示或保存。 |
| Default value | 勾选、开关、下拉菜单、单选框。 | 创建记录时自动带出初始值。适合「待处理」「未启用」「否」等初始状态。 | 新建表单会自动显示默认状态。 |

## 从数据源映射

外部数据库中的状态和选项字段通常保存为布尔值、字符串、数字编码或 JSON 数组。NocoBase 同步字段后，需要把数据库字段类型映射为 Field type，再根据业务含义选择 Field interface。

同步后，需要进入数据表的「Configure fields」检查选项字段是否识别正确。重点确认选项标题、实际值、颜色、默认值，以及页面展示是否符合业务语义。

编辑字段配置不会修改外部数据库里的真实字段名、字段类型、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步字段时，映射顺序通常是先读取数据库字段类型，再映射为 NocoBase 的 Field type，最后根据字段名称、已有值和业务含义选择 Field interface。

| 数据库字段类型 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| `boolean` / `tinyint(1)` | `boolean` | 勾选、开关。 | 适合是否启用、是否归档、是否已付款等是 / 否字段。 |
| `varchar` / `char` / `enum` | `string` | 下拉菜单、单选框。 | 适合订单状态、客户等级、优先级等单选字段。需要根据已有值补齐选项列表。 |
| `integer` / `smallint` | `integer` 或 `string` | 下拉菜单、单选框。 | 适合已有系统用数字编码保存状态的字段。建议在选项标题中写业务含义。 |
| `json` / `jsonb` / 字符串数组 | `json` 或 `array` | 多选框、复选框组。 | 适合客户标签、适用渠道、技能标签等多选字段。 |

:::warning 注意

字段映射只改变 NocoBase 如何理解和展示字段，不会改变外部数据库中的真实字段结构。已有数据库如果用 `1`、`2`、`3` 表示状态，建议先确认每个值的含义，再配置选项标题和颜色。

:::
