# 其他字段

## 介绍

其他字段适合处理不直接归入文本、数字、日期、选项、附件、关系等分类的配置类需求。

## 章节目录

- [适用场景](#适用场景)

- [相关链接](#相关链接)

## 适用场景

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 手动排序值 | 排序 | 流程节点顺序、菜单顺序、列表顺序、看板列顺序。 |
| 根据其他字段计算出的结果 | 公式 | 小计金额、利润、年龄、剩余天数、风险等级。 |
| 不固定结构的数据 | JSON | 第三方 API 原始响应、扩展配置、动态表单结果。 |

## 使用建议

排序字段适合保存人为调整后的展示顺序。如果只是按创建时间、更新时间、编号或金额排序，不需要额外创建排序字段。

公式字段适合展示派生值，不适合保存需要人工反复修改的业务字段。JSON 适合扩展性很强的数据，不过如果某个属性经常用于筛选、排序或权限判断，建议拆成独立字段。
