# 其他字段

## 适用场景

其他字段适合处理不直接归入文本、数字、日期、选项、附件、关系等分类的配置类需求。创建前先确认字段是用于人工排序、自动计算，还是保存不固定结构的数据。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 手动排序值 | 排序 | 流程节点顺序、菜单顺序、列表顺序、看板列顺序。 |
| 根据其他字段计算出的结果 | 公式 | 小计金额、利润、年龄、剩余天数、风险等级。 |
| 不固定结构的数据 | JSON | 第三方 API 原始响应、扩展配置、动态表单结果。 |

## 从 NocoBase 创建

通过**[主数据库配置字段](../main/database.md#新增字段)**在页面新增其他字段。创建时重点确认字段用途、Field interface、是否由系统计算或维护，以及是否需要参与筛选、排序和权限判断。

![add_field](https://static-docs.nocobase.com/add_field.png)
![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 其他字段的界面类型。常见包括「排序」「公式」「JSON」等。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「显示顺序」「小计金额」「接口响应」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。排序通常是数字，公式根据计算结果决定类型，JSON 使用结构化数据类型。 |
| Formula | 公式字段的计算规则。适合根据已有字段自动计算结果，比如金额小计、剩余天数、风险等级。 |
| Sort scope | 排序字段的分组范围。适合按项目、栏目、看板列分别排序。 |
| Default value | 默认值。适合 JSON 字段设置默认结构，或排序字段设置初始值。 |
| Description | 字段说明。适合写清楚计算口径、排序规则或 JSON 结构。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库字段类型 | 默认长度 |
| --- | --- | --- | --- |
| 排序 | `sort` 或 `integer` | `integer` |  |
| 公式 | `formula` | 根据结果类型决定 |  |
| JSON | `json` | `jsonb` |  |

这里的默认数据库字段类型以 PostgreSQL 为例。不同数据库最终生成的字段类型显示可能略有差异。

### 校验规则

其他字段的校验规则取决于字段用途。排序字段通常关注是否为数字，JSON 字段关注结构是否稳定，公式字段更关注计算表达式是否符合业务规则。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | JSON、排序。 | 提交记录时必须填写。适合必须保存扩展配置或排序值的字段。 | 字段为空时，提交表单会提示必填错误。 |
| Number range | 排序。 | 限制排序值范围。适合需要控制排序区间的场景。 | 输入值超出范围时，会提示范围错误。 |
| JSON structure | JSON。 | 限制 JSON 数据结构符合业务约定。适合第三方 API 原始响应、扩展配置等场景。 | JSON 格式不正确时，可能无法保存或无法被后续流程解析。 |
| Formula expression | 公式。 | 确认公式表达式能正确引用字段并返回预期结果。 | 公式配置错误时，页面可能无法得到正确计算结果。 |

## 从数据源映射

外部数据库中的其他字段可能保存为排序数字、JSON / JSONB、计算列或数据库视图中的派生字段。NocoBase 同步字段后，需要根据真实字段类型和业务用途选择 Field interface。

同步后，需要进入数据表的「Configure fields」检查字段是否识别正确。重点确认排序字段是否只是普通数字，JSON 是否需要结构化展示，计算结果是否来自数据库侧还是 NocoBase 公式字段。

编辑字段配置不会修改外部数据库里的真实字段名、字段类型、计算列、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步字段时，映射顺序通常是先读取数据库字段类型，再映射为 NocoBase 的 Field type，最后根据字段用途选择 Field interface。

| 数据库字段类型或结构 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| `integer` / `bigint` 排序列 | `integer` | 排序、整数。 | 如果字段由业务人员拖拽排序维护，可以映射为「排序」；如果只是普通数字，映射为「整数」更直观。 |
| `json` / `jsonb` | `json` | JSON。 | 适合第三方 API 原始响应、扩展配置、动态表单结果等不固定结构数据。 |
| `text` 保存的 JSON 字符串 | `text` 或 `json` | JSON、多行文本。 | 如果内容始终是合法 JSON，可以映射为 JSON；否则保留为多行文本更稳。 |
| 数据库计算列 | 按结果类型映射 | 数字、文本、日期、公式。 | 如果计算由数据库维护，NocoBase 通常按结果类型展示；只有需要在 NocoBase 中配置计算规则时才使用公式字段。 |
| 数据库视图中的派生字段 | 按结果类型映射 | 数字、文本、日期、JSON。 | 视图字段通常由 SQL 决定结果，NocoBase 负责展示和筛选。 |

:::warning 注意

JSON 适合扩展性很强的数据，不过如果某个属性经常用于筛选、排序、权限判断或报表统计，建议拆成独立字段。公式字段适合展示派生值，不适合保存需要人工反复修改的业务字段。

:::
