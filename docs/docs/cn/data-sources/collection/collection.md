# 数据表

在 NocoBase 中，**数据表（Collection）** 用来描述一类业务数据。数据表下面包含字段，字段决定每条记录可以保存哪些信息。页面区块、权限、工作流和 API 都会围绕数据表和字段工作。

如果你正在从零搭建业务，默认从主数据库创建普通表就够了。只有当数据本身有特殊结构，比如树形结构、日历事件、文件元信息、SQL 查询结果或数据库视图时，才需要选择其他数据表类型。

## 数据表类型

创建数据表时，可以选择不同模板。模板决定数据表的默认字段、结构来源和后续可操作能力。

| 数据表类型 | 适合场景 | 数据来源 | 增删改支持 |
| --- | --- | --- | --- |
| 普通表 | 客户、订单、合同、工单、报销单等常规业务数据。 | NocoBase 创建并维护真实数据库表。 | 支持。 |
| 继承表 | 多类业务对象共享一组公共字段，同时每一类又有自己的专属字段。 | NocoBase 创建并维护真实数据库表。 | 支持。 |
| 树表 | 组织架构、商品分类、地区层级、知识库目录等上下级数据。 | NocoBase 创建并维护真实数据库表。 | 支持。 |
| 日历表 | 会议预约、项目排期、课程安排、值班计划等带时间范围的数据。 | NocoBase 创建并维护真实数据库表。 | 支持。 |
| 评论表 | 任务评论、文章评论、审批意见、客户反馈等记录讨论内容的数据。 | NocoBase 创建并维护真实数据库表。 | 支持。 |
| 文件表 | 合同附件、发票文件、产品图片、员工证件等文件元信息。 | NocoBase 创建并维护真实数据库表，文件本体由存储引擎保存。 | 支持。 |
| SQL 表 | 销售汇总、库存预警、跨表统计报表等 SQL 查询结果。 | SQL 查询结果，不创建新的真实数据库表。 | 通常只读。 |
| 数据库视图 | DBA 已经维护好的 view，比如财务报表视图、过滤后的客户视图。 | 数据库中已有 view。 | 取决于数据库视图本身是否可写。 |

## 常见选择

- **保存普通业务记录**：选择普通表，比如客户、订单、项目、合同
- **多个数据表有公共字段**：选择继承表，比如资产父表派生电脑资产、车辆资产、办公家具
- **需要上下级结构**：选择树表，比如组织架构、商品分类、地区层级
- **需要在日历中展示**：选择日历表，比如会议预约、课程安排、值班计划
- **需要围绕记录讨论**：选择评论表，比如任务评论、审批意见、客户反馈
- **需要上传和管理文件**：选择文件表，或在业务表中使用附件字段
- **需要展示 SQL 查询结果**：选择 SQL 表
- **数据库里已经有 view**：选择连接数据库视图

## 创建数据表

在主数据库的数据表列表中，点击「Create collection」，选择合适的数据表类型后填写基础配置。

| 配置 | 说明 |
| --- | --- |
| Collection display name | 数据表在界面中显示的名称，比如「客户」「订单」「合同附件」。建议使用业务人员能直接理解的名称。 |
| Collection name | 数据表的内部名称，用于 API、关系字段、权限、工作流等内部引用。它会自动生成，也可以手动修改；只支持字母、数字和下划线，并且必须以字母开头。 |
| Inherits | 选择要继承的父表。当前数据表会继承父表的字段结构，同时还可以继续定义自己的字段。仅主数据库使用 PostgreSQL 时可见。 |
| Categories | 数据表分类。只影响数据表管理界面的组织方式，不改变数据表结构。 |
| Description | 数据表说明。可以写数据表保存什么数据、由谁维护、和哪些业务流程有关。 |
| Use simple pagination mode | 简单分页模式。启用后，分页时会跳过总记录数统计，适合数据量很大的表。 |
| Preset fields | 预设字段。创建表时可以选择是否自动添加 ID、创建时间、创建人、更新时间、更新人等常用字段。 |
| Primary key | 数据库层面的主键字段，用于唯一标识一条记录。普通业务表通常保留默认 ID 字段即可。 |
| Record unique key | NocoBase 用来定位记录的唯一字段。SQL 表、数据库视图或无主键表需要重点确认。 |

:::tip 提示

`Primary key` 和 `Record unique key` 不是同一个概念。`Primary key` 是数据库表的主键，通常在创建真实数据表时确定；`Record unique key` 是 NocoBase 用来定位记录的字段，SQL 表、数据库视图或没有标准主键的数据表也可以通过它指定唯一字段。

:::

## 编辑数据表

在数据表列表中，点击「Edit」可以修改数据表的基础配置。编辑数据表主要用于维护数据表元信息和部分运行配置，不用于批量调整字段结构。

常见可编辑内容包括显示名称、分类、描述、简单分页模式和记录唯一标识。`Collection name` 创建后通常只读，因为它可能已经被页面区块、关系字段、权限、工作流或 API 引用。

如果要新增、删除或调整字段，请使用数据表右侧的「Configure fields」。

## 删除数据表

在数据表列表中，点击「Delete」可以删除数据表。主数据库中的数据表还支持批量选择后统一删除。

删除前建议检查页面区块、关系字段、权限配置、工作流和外部 API 是否仍然依赖这个数据表。

:::danger 警告

删除数据表是高风险操作，可能导致数据、页面配置、关系字段、权限和工作流不可用。只有确认不再使用，或已经完成备份和迁移后，才建议删除数据表。

:::

## 配置字段

数据表创建后，点击「Configure fields」可以进入字段配置页面。字段决定一条记录里可以保存哪些信息，也决定这些信息在页面里如何输入和展示。

字段配置通常包括显示名称、字段名称、Field type、Field interface、标题字段和字段描述。更多内容见 [字段](./field.md)。
