# 字段

## 介绍

在 NocoBase 中，**Field（字段）** 是 [Collection（数据表）](./collection.md) 中用来描述数据的一项业务属性，用来描述一条记录里可以保存什么信息，以及这些信息在页面中如何输入、展示、筛选和参与业务逻辑。

| 定义 | 说明 |
| --- | --- |
| 保存什么数据 | 比如文本、数字、日期、文件、状态、关系、JSON。 |
| 页面怎么使用 | 比如用输入框、日期选择器、下拉菜单、附件上传、关系选择器来录入和展示。 |
| 如何参与业务能力 | 字段会被页面区块、筛选、排序、权限、工作流、API、数据导入导出等功能使用。 |

它可以对应：
- 主数据库里的真实数据库列
- 外部数据库里的已有数据库列
- 数据库视图中的字段
- SQL 查询结果中的字段
- REST API 响应中的字段
- 数据表中的关系字段、系统字段或虚拟字段

把它理解为“业务对象的一个属性”。比如：

| 业务对象 | 对应 Field |
| --- | --- |
| 客户 | 客户名称、手机号、客户等级、负责人 |
| 订单 | 订单编号、订单金额、订单状态、客户 |
| 合同 | 合同名称、签约日期、合同附件、审批状态 |
| 任务 | 任务标题、截止时间、优先级、执行人 |
| 文件 | 文件名、大小、MIME 类型、URL |

## 创建管理字段

- **主数据库** — 在 NocoBase 页面中创建管理文本、数字、日期、选项、关系字段等。操作步骤见[主数据库](../main/database.md#配置字段)、[创建 SQL 数据表](../main/collection-sql.md)、[连接数据库视图](../main/collection-view.md)
- **外部数据库** — 从外部数据库已有表或 view同步字段信息。操作步骤见[外部数据库](../external/database.md#配置字段)
- **REST API 数据源** — 字段来自接口响应或手动配置的字段映射。操作步骤见[REST API 数据源](../external/rest-api.md)
- **外部 NocoBase** — 字段来自另一个 NocoBase 应用中的数据表。字段结构通常在远端应用维护，当前应用不支持配置字段。更多说明见[外部 NocoBase](../external/nocobase.md)

## 使用场景选择字段

如果不知道要选哪个类型的字段，而是先知道业务上要保存什么。可以先按场景判断。

### 文本

文本字段适合保存名称、编号、说明、联系方式、链接地址等内容。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 简短文本 | 单行文本 | 客户名称、项目名称、合同编号、职位。 |
| 较长说明 | 多行文本 | 备注、需求描述、处理意见、问题说明。 |
| 邮箱地址 | 邮箱 | 客户邮箱、员工邮箱、供应商联系邮箱。 |
| 手机号码 | 手机号 | 客户手机号、联系人电话、员工联系方式。 |
| 链接地址 | URL | 官网、合同链接、外部系统地址、图片或文件链接。 |
| 密码或敏感输入 | 密码 | 初始密码、密钥、临时访问凭证。 |
| 颜色值 | 颜色 | 状态颜色、分类颜色、标签颜色、看板分组颜色。 |
| 图标标识 | 图标 | 菜单图标、业务类型图标、资产分类图标、状态图标。 |

### 富文本

富文本字段适合保存正文、说明文档、处理方案、代码片段等较复杂内容。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| Markdown 内容 | Markdown | 知识库正文、需求说明、技术文档、处理方案。 |
| 所见即所得 Markdown | Markdown(Vditor) | 需要实时预览、所见即所得编辑、上传图片或附件到对象存储的说明文档、文章正文。 |
| 富文本内容 | 富文本 | 公告、产品说明、带格式的处理记录。 |
| 代码片段 | Code | 脚本、SQL 片段、配置片段、技术备注。 |

### 数字

数字字段适合保存数量、金额、评分、比例等数值。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 整数 | 整数 | 数量、人数、库存、次数。 |
| 小数 | 数字 | 重量、面积、评分、用量、单价。 |
| 百分比 | 百分比 | 完成率、折扣率、毛利率、进度。 |
| 金额 | 金额 | 合同金额、订单金额、预算、费用。 |

### 日期和时间

日期和时间字段适合保存时间点、日期、时间、外部系统时间戳等。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 跨时区时间点 | 日期时间（带时区） | 跨地区会议时间、全球团队任务截止时间、跨时区工单 SLA。 |
| 本地业务时间点 | 日期时间（不带时区） | 没有跨时区业务的本地预约开始时间、门店活动时间、项目排期。 |
| 只有日期 | 日期 | 生日、入职日期、发票日期、项目开始日期。 |
| 只有时间 | 时间 | 每日打卡时间、营业开始时间、提醒时间。 |
| 时间戳 | Unix Timestamp | 接入外部系统、日志数据、已有数据库中使用时间戳的字段。 |

### 状态和选项

状态和选项字段适合保存固定范围内的值。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 是 / 否 | 勾选、开关 | 是否启用、是否归档、是否已付款、是否需要回访。 |
| 单个固定选项 | 下拉菜单、单选框 | 订单状态、客户等级、工单优先级、合同类型。 |
| 多个固定选项 | 多选框 | 客户标签、适用渠道、技能标签、产品特性。 |

如果选项会频繁增删，或者每个选项还需要保存更多属性，建议单独建一张数据表，再用关系字段关联。

### 附件

附件字段适合上传文件，或者保存外部文件地址。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 上传到 NocoBase 管理的文件 | 附件 | 合同附件、发票图片、员工证件、项目资料。 |
| 已经在外部系统中的文件地址 | 附件 URL | 对象存储、网盘、第三方系统中的文件链接。 |

附件字段依赖文件表和文件存储引擎。用户上传文件后，文件本体由文件存储引擎保存，文件名、大小、MIME 类型、URL、存储位置等元信息会保存到文件表中。

### 关系字段

关系字段用来表达不同数据表之间的连接。它不是简单的数据库文本列，而是一组关系配置。

| 你要表达 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 当前表的一条记录属于另一张表的一条记录 | 多对一 | 订单属于客户、任务属于项目、联系人属于公司。 |
| 当前表的一条记录拥有多条子记录 | 一对多 | 客户拥有多张订单、项目拥有多个任务。 |
| 一条记录只关联另一条记录 | 一对一 | 员工和员工档案、用户和扩展资料。 |
| 两边都可以关联多条记录 | 多对多 | 用户和角色、学生和课程、产品和标签。 |
| 多对多关系保存为数组 | 多对多（数组） | 外部数据或特殊结构中已经用数组保存关联 ID。 |

### 标识和编码

自动标识和编码字段适合保存内部主键、外部同步 ID、业务编号等。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 默认主键 | ID、Snowflake ID (53-bit) | NocoBase 默认主键、系统内部唯一标识。 |
| 全局唯一字符串 | UUID、Nano ID | 对外同步、跨系统引用、公开链接标识。 |
| 业务编号 | 自动编码（Sequence） | 订单编号、合同编号、工单编号、资产编号。 |

如果字段只是数据库内部主键，默认 ID 通常够用。如果这个编号会展示给业务人员或外部系统使用，建议用自动编码这类更有业务含义的字段。

自动编码通常需要配置编码规则，比如固定前缀、日期片段、流水号位数等。比如合同编号可以配置成 `HT-{YYYYMMDD}-{0001}` 这类格式。

### 几何图形

几何字段适合保存空间或地理信息。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 点 | 点 | 门店位置、用户位置。 |
| 线 | 线 | 配送路线、巡检路线。 |
| 面 | 面 | 配送范围、地理围栏。 |
| 圆 | 圆 | 覆盖范围、影响范围。 |

### 系统信息

系统信息字段由 NocoBase 或数据库维护，通常不作为普通业务字段手动创建。

| 字段 | 说明 | 常见用途 |
| --- | --- | --- |
| ID | 默认主键字段，用于唯一标识一条记录。 | 关系字段、API、权限、工作流。 |
| 创建时间 | 自动记录记录创建时间。 | 排序、筛选、审计。 |
| 创建人 | 自动记录创建记录的用户。 | 责任追踪、权限控制。 |
| 更新时间 | 自动记录最后更新时间。 | 审计、同步判断。 |
| 更新人 | 自动记录最后更新记录的用户。 | 协作追踪、审计。 |
| Table OID | PostgreSQL 相关系统字段。 | 继承表、底层表识别等特殊场景。 |

普通业务表通常建议保留 ID、创建时间、创建人、更新时间、更新人这些预设字段。它们对后续权限、审计、工作流和数据排查都有帮助。

### 其他

其他字段适合处理不直接归入文本、数字、日期、选项、附件、关系等分类的配置类需求。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 手动排序值 | 排序 | 流程节点顺序、菜单顺序、列表顺序、看板列顺序。 |
| 根据其他字段计算出的结果 | 公式 | 小计金额、利润、年龄、剩余天数、风险等级。 |
| 不固定结构的数据 | JSON | 第三方 API 原始响应、扩展配置、动态表单结果。 |

公式字段适合展示派生值，不适合保存需要人工反复修改的业务字段。JSON 适合扩展性很强的数据，不过如果某个属性经常用于筛选、排序或权限判断，建议拆成独立字段。

## 从页面创建字段

主数据库中的普通表、树表、日历表、评论表、文件表等数据表，可以在 NocoBase 页面中创建字段。入口通常在数据表的「Configure fields」或字段配置页面。

从页面创建字段时，通常先选择 Field interface，再填写字段属性。不同字段类型会显示不同的专属配置。

### 基本属性

| 属性 | 说明 | 示例 |
| --- | --- | --- |
| Field display name | 字段在界面中显示的名称，建议使用业务人员能直接理解的名称。 | 「客户名称」「联系电话」「订单金额」。 |
| Field name | 字段内部名称，用于 API、权限、工作流等内部引用。创建后不建议修改。 | `customerName`、`phone`、`amount`。 |
| Field interface | 字段的界面类型，决定页面怎么录入和展示。 | 单行文本、日期时间、下拉菜单、附件。 |
| Field type | 字段的数据类型。通常由 Field interface 自动带出，部分字段可以手动选择。 | `string`、`integer`、`date`、`json`。 |
| Description | 字段说明。适合写字段含义、录入要求、数据来源或维护人。 | 「客户合同中的签约主体名称」。 |

字段名称建议在创建时一次想清楚。显示名称可以随着业务语言调整，内部名称如果已经被 API、工作流、权限或页面引用，修改成本会更高。

### 常用属性

| 属性 | 说明 | 示例 |
| --- | --- | --- |
| Default value | 默认值。创建记录时，如果用户没有填写该字段，可以自动填入默认值。 | 状态默认「待处理」，负责人默认当前用户。 |
| Required | 必填。提交记录时必须填写。 | 客户名称、订单金额。 |
| Unique | 唯一约束。保证字段中非空值不重复。 | 手机号、邮箱、工号、订单编号。 |
| Pattern / Format | 格式校验。限制输入内容符合指定格式。 | 邮箱、手机号、URL。 |
| Title field | 标题字段。决定记录在关系选择、详情标题、关联数据展示等位置如何显示。 | 客户表使用「客户名称」，订单表使用「订单编号」。 |

默认值只在创建记录时生效，通常不会自动修改已有记录。如果字段有唯一约束，不建议再配置固定默认值，否则容易产生重复值。

唯一约束适合真正不能重复的业务标识。启用前需要确认已有数据已经满足唯一性，否则新增约束可能失败。

### 特殊字段属性

不同字段类型会有自己的专属配置：

| 字段类型 | 常见特殊属性 | 配置说明 |
| --- | --- | --- |
| 选项字段 | 选项值、颜色、默认选项、是否允许多选。 | 比如「订单状态」可以配置「待付款」「已付款」「已取消」。 |
| 数字字段 | 精度、最小值、最大值、步进、百分比格式。 | 比如「折扣率」限制在 0 到 100 之间。 |
| 日期时间字段 | 是否显示时间、是否带时区、默认当前时间。 | 比如「预约开始时间」使用日期时间，「生日」只使用日期。 |
| 附件字段 | 文件表、文件存储、文件数量、文件类型限制。 | 比如「合同附件」只允许上传 PDF 和图片。 |
| 关系字段 | 目标数据表、关系类型、关联键、中间表、标题字段。 | 比如「订单」通过多对一关系关联「客户」。 |
| 自动编码 | 编码规则、前缀、日期片段、流水号位数。 | 比如订单编号 `SO-{YYYYMMDD}-{0001}`。 |
| 公式字段 | 表达式、返回值类型、可引用字段。 | 比如「小计金额 = 单价 * 数量」。 |
| JSON 字段 | 结构化数据输入方式。 | 比如保存第三方 API 原始响应或扩展配置。 |

:::warning 注意

切换 Field interface 不等于修改数据库字段类型。它主要影响页面控件、展示方式、校验规则和后续可用配置。切换前先确认已有数据是否能被新的界面类型正确识别。

:::

### 标题字段、主键和记录唯一标识

标题字段、主键和记录唯一标识经常会被混在一起理解，实际用途不同。

| 概念 | 作用 | 常见配置 |
| --- | --- | --- |
| Primary key | 数据库层面的主键，用来唯一标识一条记录。 | 默认 ID 字段。 |
| Record unique key | NocoBase 定位记录时使用的唯一字段。 | SQL 表、数据库视图、外部数据库表需要特别关注。 |
| Title field | 记录在界面中的默认显示标题。 | 客户名称、订单编号、项目名称。 |

标题字段只影响界面显示，不等同于主键，也不等同于记录唯一标识。没有可用的记录唯一标识时，页面区块可能无法正确查看、编辑或删除记录。

## 映射字段

字段映射用于告诉 NocoBase：一个数据源里的字段在 NocoBase 中应该用什么 Field type 理解、用什么 Field interface 展示和输入。

它主要出现在这些场景：

- 主数据库中同步已有真实表
- 外部数据库中接入已有表或 view
- SQL 表把查询结果映射成数据表
- 数据库视图连接已有 view
- REST API 数据源把响应字段映射成数据表字段

### 映射逻辑

字段映射通常按这个顺序发生：

1. NocoBase 读取数据源中的字段结构或返回结果。
2. 根据数据库字段类型、接口响应值或已有元数据推断 Field type。
3. 根据 Field type 给出可用的 Field interface。
4. 你检查字段业务含义，必要时调整 Field display name、Field interface、标题字段、描述和校验配置。

比如数据库字段是 `varchar`，NocoBase 可以识别它是字符串类型。不过它在业务上可能是普通文本、邮箱、手机号、URL、颜色或图标，所以需要你在「Configure fields」里确认 Field interface 是否符合业务含义。

### 主数据源中的字段映射

主数据源有两类情况：

| 场景 | 字段映射方式 |
| --- | --- |
| 从页面新建字段 | 通常先选择 Field interface，NocoBase 自动生成对应 Field type 和数据库字段。 |
| 从主数据库同步已有表 | NocoBase 读取已有数据库字段，再根据字段类型推断 Field type 和 Field interface。 |

从页面新建字段时，用户通常不需要直接处理数据库字段类型。比如选择「手机号」，底层通常仍然是字符串；选择「百分比」，底层通常是数值类型。

从主数据库同步已有表时，需要检查推断结果。比如 `tinyint(1)` 可能适合映射为勾选，也可能是普通整数；`varchar` 可能是普通文本，也可能是邮箱或 URL。

### 外部数据源中的字段映射

外部数据源通常不由 NocoBase 创建数据库字段，而是读取已有表结构或接口结构，再在 NocoBase 中保存字段元数据。

| 场景 | 字段映射重点 |
| --- | --- |
| 外部数据库 | 根据数据库列类型推断 Field type，再选择合适的 Field interface。 |
| SQL 表 | 根据 SQL 查询结果字段和预览值推断字段类型，再手动确认 Field interface。 |
| 数据库视图 | 根据 view 字段类型和来源表字段辅助识别字段。 |
| REST API 数据源 | 根据响应 JSON 字段、示例值和转换配置映射字段。 |

外部数据源字段映射不会自动改变外部系统里的真实字段结构。它改变的是 NocoBase 如何理解、展示和使用这些字段。

### 常见映射参考

| 数据源字段类型 | 常见 Field type | 常见 Field interface |
| --- | --- | --- |
| `varchar` / `char` | `string` | 单行文本、邮箱、手机号、URL、颜色、图标、下拉菜单。 |
| `text` | `text` | 长文本、Markdown、富文本、URL、JSON。 |
| `int` / `bigint` | `integer` / `bigInt` | 整数、排序、Unix Timestamp、下拉菜单、单选框。 |
| `decimal` / `numeric` / `float` / `double` | `decimal` / `float` / `double` | 数字、百分比、金额。 |
| `boolean` / `tinyint(1)` | `boolean` | 勾选、开关。 |
| `date` / `datetime` / `timestamp` / `time` | `date` / `dateOnly` / `time` | 日期时间、日期、时间、创建时间、更新时间。 |
| `json` / `jsonb` | `json` | JSON。 |
| `enum` / `set` | `enum` / `set` | 下拉菜单、单选框、多选框。 |
| 外键或关联字段 | 关系类型 | 多对一、一对多、一对一、多对多。 |

:::warning 注意

数据库字段和 NocoBase 关系字段不是同一个概念。数据库字段是数据库表里的列；NocoBase 关系字段是一组关系元数据，用来描述当前表和目标表如何关联。外部数据库里即使有外键，也需要在 NocoBase 中确认关系字段配置后，页面区块、权限和工作流才能按关系使用。

:::
