# 附件字段

## 适用场景

附件字段适合上传文件，或者保存外部文件地址。创建前先确认文件本体是否由 NocoBase 管理，还是只在 NocoBase 中保存外部文件地址。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 上传到 NocoBase 管理的文件 | 附件 | 合同附件、发票图片、员工证件、项目资料。 |
| 已经在外部系统中的文件地址 | 附件 URL、URL | 对象存储、网盘、第三方系统中的文件链接。 |

## 从 NocoBase 创建

通过**[主数据库配置字段](../main/database.md#新增字段)**在页面新增附件字段。创建时重点确认 Field interface、文件存储、是否允许多文件、文件大小和文件类型限制。

![add_field](https://static-docs.nocobase.com/add_field.png)
![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 附件字段的界面类型。常用于上传文件、展示文件列表、预览图片或下载附件。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「合同附件」「发票图片」「项目资料」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。附件字段通常通过关系保存文件元信息，不只是一个普通文本列。 |
| Storage | 文件存储位置。用于决定上传文件保存到哪个文件存储引擎。 |
| Multiple | 是否允许上传多个文件。合同附件、项目资料通常允许多个；头像、身份证正面通常只需要一个。 |
| Required | 必填。适合必须上传材料的场景，比如合同附件、报销发票。 |
| File size / File type | 文件大小和文件类型限制。适合限制只能上传图片、PDF、Office 文档等。 |
| Description | 字段说明。适合写清楚允许上传的文件类型、大小、命名要求和维护人。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库字段类型 | 默认长度 |
| --- | --- | --- | --- |
| 附件 | `belongsToMany` 或附件关系 | 关系字段 / 中间表 |  |
| 附件 URL | `string` 或 `text` | `varchar` / `text` | 255 或无固定长度 |

附件字段会把文件本体保存到文件存储引擎，把文件名、大小、MIME 类型、URL、存储位置等元信息保存到文件表。它通常不是单个数据库列能完整表达的结构。

### 校验规则

附件字段常用校验规则包括必填、文件数量、文件大小和文件类型。文件越大，越要关注上传体验、存储成本和备份恢复成本。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | 附件、附件 URL。 | 提交记录时必须上传或填写文件地址。适合合同附件、报销凭证等核心材料。 | 字段为空时，提交表单会提示必填错误。 |
| File count | 附件。 | 限制最多上传几个文件。适合只允许一个头像、一个证件照，或最多上传若干附件的场景。 | 文件数量超过限制时，会提示数量限制。 |
| File size | 附件。 | 限制单个文件大小。适合控制上传性能和存储成本。 | 文件过大时，会提示大小限制。 |
| File type | 附件。 | 限制允许上传的文件类型。适合只允许图片、PDF、Office 文档等场景。 | 文件类型不符合要求时，会提示类型限制。 |

## 从数据源映射

外部数据库中的附件可能保存为文件 URL、文件 ID、JSON 元数据，或者由另一张文件表维护。NocoBase 同步字段后，需要先判断外部数据的保存方式，再决定映射为附件 URL、URL、JSON，还是配置关系字段关联文件表。

同步后，需要进入数据表的「Configure fields」检查附件相关字段是否识别正确。重点确认页面是否应该上传新文件，还是只展示已有文件地址。

编辑字段配置不会修改外部数据库里的真实字段名、字段类型、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步字段时，映射顺序通常是先读取数据库字段类型，再映射为 NocoBase 的 Field type，最后根据文件保存方式选择 Field interface。

| 数据库字段类型 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| `varchar` / `text` | `string` 或 `text` | 附件 URL、URL。 | 适合外部系统已经保存文件地址的字段。NocoBase 只展示或跳转，不接管文件本体。 |
| `json` / `jsonb` | `json` | JSON、附件 URL。 | 适合保存文件名、URL、大小等元数据的字段。需要根据 JSON 结构决定页面展示方式。 |
| 外部文件 ID | `string` 或 `integer` | 单行文本、关系字段。 | 适合已有系统用文件 ID 关联文件表的场景。通常需要额外配置关系字段或展示逻辑。 |
| 文件表关联 | 关系字段 | 附件或关系字段。 | 如果已有独立文件表，可以通过关系字段表达业务记录和文件记录的关联。 |

:::warning 注意

如果文件本体不由 NocoBase 管理，不要把外部 URL 字段直接当作 NocoBase 附件上传字段使用。否则用户可能误以为文件会被 NocoBase 存储、权限和备份机制完整接管。

:::
