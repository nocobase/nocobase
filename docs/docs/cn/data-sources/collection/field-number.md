# 数字字段

## 适用场景

数字字段适合保存数量、金额、评分、比例等数值。创建前先确认这个字段是否允许小数、需要多少精度、是否会参与计算或排序。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 整数 | 整数 | 数量、人数、库存、次数。 |
| 小数 | 数字 | 重量、面积、评分、用量、单价。 |
| 百分比 | 百分比 | 完成率、折扣率、毛利率、进度。 |
| 金额 | 金额 | 合同金额、订单金额、预算、费用。 |

## 从 NocoBase 创建

通过**[主数据库配置字段](../main/database.md#新增字段)**在页面新增数字字段。创建时重点确认 Field interface、Field type、默认值、是否必填、是否唯一、精度和取值范围。

![add_field](https://static-docs.nocobase.com/add_field.png)
![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 数字字段的界面类型。可以选择「整数」「数字」「百分比」「金额」等。它决定页面中使用普通数字输入、百分比显示还是金额格式化。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「合同金额」「库存数量」「完成率」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。整数通常使用 `integer`，小数、百分比和金额通常使用数值型字段。 |
| Default value | 默认值。适合给进度、数量、排序值设置初始值，比如 `0`。 |
| Required | 必填。适合金额、数量、比例等业务流程必须录入的字段。 |
| Unique | 唯一约束。只适合编号类数字字段。金额、数量、比例通常不建议设置唯一。 |
| Validation rules | 数字字段的校验规则。常用于限制最小值、最大值、是否必填、是否为整数等。 |
| Description | 字段说明。适合写清楚单位、精度和口径，比如「单位：万元」「百分比输入 0 到 100」。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库字段类型 | 默认长度 |
| --- | --- | --- | --- |
| 整数 | `integer` | `integer` |  |
| 数字 | `double` | `double precision` |  |
| 百分比 | `double` | `double precision` |  |
| 金额 | `decimal` | `decimal` |  |

这里的默认数据库字段类型以 PostgreSQL 为例。不同数据库最终生成的字段类型显示可能略有差异。

### 校验规则

数字字段常用校验规则包括必填、唯一、最小值、最大值和整数限制。金额和比例还要额外关注精度和显示格式。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | 整数、数字、百分比、金额。 | 提交记录时必须填写。适合合同金额、库存数量、完成率等核心字段。 | 字段为空时，提交表单会提示必填错误。 |
| Unique | 整数。 | 保证非空值不重复。只适合数字编号、外部数字 ID 等字段。 | 输入重复值并提交时，会提示唯一性校验失败。 |
| Min | 整数、数字、百分比、金额。 | 限制最小值。适合库存不能小于 0、折扣率不能小于 0 等场景。 | 输入值小于最小值时，会提示取值范围错误。 |
| Max | 整数、数字、百分比、金额。 | 限制最大值。适合进度不能超过 100、评分不能超过 5 等场景。 | 输入值大于最大值时，会提示取值范围错误。 |
| Integer | 整数。 | 限制只能输入整数。适合人数、次数、库存等不能有小数的字段。 | 输入小数时，会提示整数校验失败。 |

## 从数据源映射

外部数据库中的数字字段通常已经定义了真实字段类型、精度、默认值和约束。NocoBase 同步字段后，需要把数据库字段类型映射为 Field type，再根据业务含义选择 Field interface。

同步后，需要进入数据表的「Configure fields」检查数字字段是否识别正确。重点确认 Field interface、单位说明、精度、取值范围，以及是否需要在页面中隐藏某些技术字段。

编辑字段配置不会修改外部数据库里的真实字段名、字段类型、精度、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步字段时，映射顺序通常是先读取数据库字段类型，再映射为 NocoBase 的 Field type，最后根据字段名称和业务含义选择 Field interface。

| 数据库字段类型 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| `integer` / `int` / `smallint` / `bigint` | `integer` 或 `bigInt` | 整数。 | 适合数量、次数、外部数字 ID 等不需要小数的字段。 |
| `decimal` / `numeric` | `decimal` | 金额、数字、百分比。 | 适合金额、单价、比例等需要固定精度的字段。 |
| `float` / `double` / `real` | `double` | 数字、百分比。 | 适合评分、面积、重量、计算结果等允许浮点误差的字段。 |
| 数字形式的比例 | `decimal` 或 `double` | 百分比。 | 需要确认数据库中保存的是 `0.8` 还是 `80`，再决定页面展示和录入说明。 |
| 数字形式的金额 | `decimal` | 金额。 | 适合合同金额、订单金额、费用等字段。建议确认币种和精度。 |

:::warning 注意

字段映射只改变 NocoBase 如何理解和展示字段，不会改变外部数据库中的真实字段结构。如果数据库字段精度不足，页面中即使能输入更多小数位，保存时也可能被数据库四舍五入或拒绝。

:::
