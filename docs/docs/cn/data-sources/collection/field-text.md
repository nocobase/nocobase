# 文本字段

## 适用场景

文本字段适合保存名称、编号、说明、联系方式、链接地址等内容。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 简短文本 | 单行文本 | 客户名称、项目名称、合同编号、职位。 |
| 较长说明 | 多行文本 | 备注、需求描述、处理意见、问题说明。 |
| 邮箱地址 | 邮箱 | 客户邮箱、员工邮箱、供应商联系邮箱。 |
| 手机号码 | 手机号 | 客户手机号、联系人电话、员工联系方式。 |
| 链接地址 | URL | 官网、合同链接、外部系统地址、图片或文件链接。 |
| 密码或敏感输入 | 密码 | 初始密码、密钥、临时访问凭证。 |
| 颜色值 | 颜色 | 状态颜色、分类颜色、标签颜色、看板分组颜色。 |
| 图标标识 | 图标 | 菜单图标、业务类型图标、资产分类图标、状态图标。 |

![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

### 字段配置

| 配置 | 说明 |
| --- | --- |
| Field interface | 文本字段的界面类型。可以选择「单行文本」「多行文本」「邮箱」「手机号」「URL」「密码」「颜色」「图标」等。它决定字段在页面中使用什么输入控件和展示方式。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「客户名称」「联系电话」「官网地址」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。文本字段通常是 `string`、`text` 或 `password`，会根据 Field interface 自动带出，也可以在编辑字段时按需要调整。 |
| Default value | 默认值。适合给文本字段设置固定初始值，比如 URL 前缀、默认说明模板等。唯一字段不建议配置固定默认值。 |
| Unique | 唯一约束。适合邮箱、手机号、客户编号、合同编号等不能重复的字段。启用前需要确认已有数据没有重复值。 |
| Validation rules | 文本字段的校验规则。常用于限制长度、固定格式或输入模式，比如邮箱格式、URL 格式、手机号格式等。 |
| Description | 字段说明。适合写字段含义、录入要求、数据来源或维护人，比如「填写客户主联系人手机号」。 |

### 字段类型映射

| Field interface | 默认 Field type | 默认数据库字段类型 | 默认长度 |
| --- | --- | --- | --- |
| 单行文本 | `string` | `varchar` | 255 |
| 多行文本 | `text` | `text` |  |
| 邮箱 | `string` | `varchar` | 255 |
| 手机号 | `string` | `varchar` | 255 |
| URL | `text` | `text` |  |
| 密码 | `password` | `varchar` | 255 |
| 颜色 | `string` | `varchar` | 255 |
| 图标 | `string` | `varchar` | 255 |

这里的默认数据库字段类型、默认长度以 PostgreSQL 为例。不同数据库最终生成的字段类型显示可能略有差异。

### 校验规则

文本字段常用的校验规则包括必填、唯一、长度和格式校验。校验规则会同时影响表单提交和页面反馈：如果输入不符合规则，字段下方会显示错误提示，记录不会提交。

| 校验规则 | 适合字段 | 说明 | 页面反馈 |
| --- | --- | --- | --- |
| Required | 单行文本、多行文本、邮箱、手机号、URL、密码。 | 提交记录时必须填写。适合客户名称、手机号、登录账号等核心字段。 | 字段为空时，提交表单会提示必填错误。 |
| Unique | 单行文本、邮箱、手机号、URL。 | 保证非空值不重复。适合邮箱、手机号、客户编号、外部系统 ID 等业务唯一标识。 | 输入重复值并提交时，会提示唯一性校验失败。 |
| Min length | 单行文本、多行文本、密码。 | 限制最少字符数。适合密码、编号、说明内容等需要最低长度的字段。 | 输入字符数不足时，会提示最小长度要求。 |
| Max length | 单行文本、多行文本、邮箱、手机号、URL。 | 限制最多字符数。适合防止输入过长，尤其是会参与列表展示、筛选或第三方系统同步的字段。 | 输入字符数过多时，会提示最大长度要求。 |
| Length | 单行文本、手机号、固定编号。 | 限制固定字符数。适合固定长度编号、手机号、邮编等格式稳定的数据。 | 输入长度不等于规则要求时，会提示长度不符合要求。 |
| Pattern / Format | 邮箱、手机号、URL、单行文本。 | 限制输入符合指定格式。邮箱、手机号、URL 通常建议配置或保留对应格式校验。 | 格式不正确时，会提示格式校验失败。 |

## 从数据源映射

外部数据库中的文本字段通常已经由原业务系统、数据库迁移脚本或 DBA 创建。NocoBase 不直接创建这些真实字段，而是通过「Sync from database」读取外部表结构，并在 NocoBase 中生成对应的数据表和字段元数据。

同步后，需要进入数据表的「Configure fields」检查文本字段是否识别正确。重点确认字段显示名称、Field interface、标题字段、描述、校验规则，以及是否需要在页面中隐藏某些技术字段。

编辑外部数据库字段配置不会修改外部数据库里的真实字段名、字段类型、长度、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段类型映射

从数据源同步字段时，映射顺序通常是先读取数据库字段类型，再映射为 NocoBase 的 Field type，最后根据字段名称和业务含义选择 Field interface。Field type 描述字段在 NocoBase 数据层中的类型；Field interface 决定页面里的录入和展示方式。

| 数据库字段类型 | 映射后的 Field type | 常用 Field interface | 说明 |
| --- | --- | --- | --- |
| `varchar` / `char` / `nvarchar` / `nchar` | `string` | 单行文本、邮箱、手机号、URL、颜色、图标。 | 适合短文本和有固定格式的字符串。字段名或业务含义是 `email`、`phone`、`homepage` 时，可以分别映射为「邮箱」「手机号」「URL」。 |
| `text` / `mediumtext` / `longtext` / `clob` | `text` | 多行文本、URL。 | 适合较长说明、备注、日志文本等内容。如果保存的是长链接，也可以映射为「URL」。 |
| 字符串形式的密码或密钥 | `password` 或 `string` | 密码。 | 适合需要隐藏输入内容的字段。同步字段不会自动判断是否为密码，通常需要根据字段名和业务含义手动调整。 |
| 字符串形式的颜色值 | `string` | 颜色。 | 适合保存 `#1677ff`、`rgb(22, 119, 255)` 这类颜色值。 |
| 字符串形式的图标标识 | `string` | 图标。 | 适合保存已有系统里的图标名称、图标 key 或图标类名。 |

数据库只能提供真实字段类型，不能完整表达业务含义。同步后如果 NocoBase 自动识别不符合业务含义，可以在字段配置中调整 Field interface。比如同样是 `varchar`，客户名称更适合「单行文本」，邮箱地址更适合「邮箱」，官网地址更适合「URL」。

:::warning 注意

外部数据库字段映射只改变 NocoBase 如何理解和展示字段，不会改变外部数据库中的真实字段结构。如果外部数据库字段长度不足，NocoBase 页面里即使能输入更长内容，保存时也可能被数据库拒绝。

:::
