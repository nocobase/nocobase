# 文本字段

## 适用场景

文本字段适合保存名称、编号、说明、联系方式、链接地址等内容。

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 简短文本 | 单行文本 | 客户名称、项目名称、合同编号、职位。 |
| 较长说明 | 多行文本 | 备注、需求描述、处理意见、问题说明。 |
| 邮箱地址 | 邮箱 | 客户邮箱、员工邮箱、供应商联系邮箱。 |
| 手机号码 | 手机号 | 客户手机号、联系人电话、员工联系方式。 |
| 链接地址 | URL | 官网、合同链接、外部系统地址、图片或文件链接。 |
| 密码或敏感输入 | 密码 | 初始密码、密钥、临时访问凭证。 |
| 颜色值 | 颜色 | 状态颜色、分类颜色、标签颜色、看板分组颜色。 |
| 图标标识 | 图标 | 菜单图标、业务类型图标、资产分类图标、状态图标。 |

## 主数据库创建的字段

### 配置字段

通过**[主数据库配置字段](../main/database.md#配置字段)**在页面新增字段、编辑字段来配置字段的显示名称、标识名称、字段类型、界面类型、校验规则、描述等。

![add_field](https://static-docs.nocobase.com/add_field.png)
![add_field_configure](https://static-docs.nocobase.com/add_field_configure.png)

| 配置 | 说明 |
| --- | --- |
| Field interface | 文本字段的界面类型。可以选择「单行文本」「多行文本」「邮箱」「手机号」「URL」「密码」「颜色」「图标」等。它决定字段在页面中使用什么输入控件和展示方式。 |
| Field display name | 字段在界面中显示的名称。建议使用业务人员能直接理解的名称，比如「客户名称」「联系电话」「官网地址」。 |
| Field name | 字段标识名称，用于 API、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在 NocoBase 数据层的类型。文本字段通常是 `string`、`text` 或 `password`，会根据 Field interface 自动带出，也可以在编辑字段时按需要调整。 |
| 默认数据库类型和长度 | 页面新增字段时，NocoBase 会根据 Field type 创建默认数据库字段。比如 `string` 通常对应 `varchar(255)`，`text` 对应 `text`。 |
| Default value | 默认值。适合给文本字段设置固定初始值，比如 URL 前缀、默认说明模板等。唯一字段不建议配置固定默认值。 |
| Required | 必填。适合客户名称、手机号、邮箱等创建记录时必须录入的字段。 |
| Unique | 唯一约束。适合邮箱、手机号、客户编号、合同编号等不能重复的字段。启用前需要确认已有数据没有重复值。 |
| Validation rules | 文本字段的校验规则。常用于限制长度、固定格式或输入模式，比如邮箱格式、URL 格式、手机号格式等。 |
| Description | 字段说明。适合写字段含义、录入要求、数据来源或维护人，比如「填写客户主联系人手机号」。 |

### 校验规则

文本字段常用的校验规则包括必填、唯一、长度和格式校验。不同 Field interface 会带出不同的默认输入体验，不过是否需要更严格的校验，还要看业务规则。

| 校验规则 | 适合字段 | 说明 |
| --- | --- | --- |
| Required | 单行文本、多行文本、邮箱、手机号、URL、密码。 | 提交记录时必须填写。适合客户名称、手机号、登录账号等核心字段。 |
| Unique | 单行文本、邮箱、手机号、URL。 | 保证非空值不重复。适合邮箱、手机号、客户编号、外部系统 ID 等业务唯一标识。 |
| Min length | 单行文本、多行文本、密码。 | 限制最少字符数。适合密码、编号、说明内容等需要最低长度的字段。 |
| Max length | 单行文本、多行文本、邮箱、手机号、URL。 | 限制最多字符数。适合防止输入过长，尤其是会参与列表展示、筛选或第三方系统同步的字段。 |
| Length | 单行文本、手机号、固定编号。 | 限制固定字符数。适合固定长度编号、手机号、邮编等格式稳定的数据。 |
| Pattern / Format | 邮箱、手机号、URL、单行文本。 | 限制输入符合指定格式。邮箱、手机号、URL 通常建议配置或保留对应格式校验。 |

如果字段会作为登录账号、外部系统编号、关系字段关联值或数据同步标识，建议同时关注唯一性和格式校验。如果只是普通备注或说明，多行文本通常只需要按业务需要设置最大长度。

### 类型映射

| Field interface | 默认 Field type | 默认数据库类型 | 默认长度 |
| --- | --- | --- | --- |
| 单行文本 | `string` | `varchar` | 255 |
| 多行文本 | `text` | `text` |  |
| 邮箱 | `string` | `varchar` | 255 |
| 手机号 | `string` | `varchar` | 255 |
| URL | `text` | `text` |  |
| 密码 | `password` | `varchar` | 255 |
| 颜色 | `string` | `varchar` | 255 |
| 图标 | `string` | `varchar` | 255 |

这里的默认长度指 NocoBase 创建字段时的常见默认结构。不同数据库最终生成的字段类型显示可能略有差异，比如 PostgreSQL、MySQL、MariaDB、SQLite 对字符串和文本类型的显示方式不完全相同。

## 同步映射文本字段

外部数据库中的文本字段通常已经由原业务系统、数据库迁移脚本或 DBA 创建。NocoBase 不直接创建这些真实字段，而是通过「Sync from database」读取外部表结构，并在 NocoBase 中生成对应的数据表和字段元数据。

同步后，需要进入数据表的「Configure fields」检查文本字段是否识别正确。重点确认字段显示名称、Field interface、标题字段、描述、校验规则，以及是否需要在页面中隐藏某些技术字段。

编辑外部数据库字段配置不会修改外部数据库里的真实字段名、字段类型、长度、默认值或索引。如果需要调整这些结构，请先在数据库侧修改，再重新同步到 NocoBase。

### 字段映射

外部数据库字段映射通常按数据库列类型先推断 Field type，再选择合适的 Field interface。

| 外部数据库字段类型 | 常见 Field type | 可选择的 Field interface | 说明 |
| --- | --- | --- | --- |
| `varchar` / `char` | `string` | 单行文本、邮箱、手机号、URL、颜色、图标。 | 适合短文本和有固定格式的字符串。 |
| `text` / `mediumtext` / `longtext` | `text` | 多行文本、URL。 | 适合较长说明、备注、日志文本等内容。 |
| 字符串形式的颜色值 | `string` | 颜色。 | 比如 `#1677ff` 这类颜色值。 |
| 字符串形式的图标标识 | `string` | 图标。 | 比如已有系统保存的图标名称或图标 key。 |

外部数据库里同样是 `varchar` 的字段，在业务上可能完全不同。比如 `email` 更适合映射为「邮箱」，`phone` 更适合映射为「手机号」，`homepage` 更适合映射为「URL」。同步后如果 NocoBase 自动识别不符合业务含义，可以在字段配置中调整 Field interface。

:::warning 注意

外部数据库字段映射只改变 NocoBase 如何理解和展示字段，不会改变外部数据库中的真实字段结构。如果外部数据库字段长度不足，NocoBase 页面里即使能输入更长内容，保存时也可能被数据库拒绝。

:::
