# 标识和编码字段

## 介绍

标识和编码字段适合保存内部主键、外部同步 ID、业务编号等。

## 章节目录

- [适用场景](#适用场景)
- [使用建议](#使用建议)
- [相关链接](#相关链接)

## 适用场景

| 你要保存 | 建议字段 | 适合场景 |
| --- | --- | --- |
| 默认主键 | ID、Snowflake ID (53-bit) | NocoBase 默认主键、系统内部唯一标识。 |
| 全局唯一字符串 | UUID、Nano ID | 对外同步、跨系统引用、公开链接标识。 |
| 业务编号 | 自动编码（Sequence） | 订单编号、合同编号、工单编号、资产编号。 |

## 使用建议

如果字段只是数据库内部主键，默认 ID 通常够用。如果这个编号会展示给业务人员或外部系统使用，建议用自动编码这类更有业务含义的字段。

自动编码通常需要配置编码规则，比如固定前缀、日期片段、流水号位数等。比如合同编号可以配置成 `HT-{YYYYMMDD}-{0001}` 这类格式。
