# SQL 表

<PluginInfo name="collection-sql"></PluginInfo>

## 介绍

SQL 表用于把一段 SQL 查询结果包装成 NocoBase 可使用的数据表。它不会创建新的真实数据库表，而是读取 SQL 查询结果，再配置字段元数据，让查询结果可以在表格、详情、图表和工作流中使用。

它适合这些场景：

- 销售汇总、库存预警、跨表统计报表
- 查询结果来自多个数据表的 join 或聚合
- 只需要在 NocoBase 中展示查询结果
- 不方便在数据库侧维护 view，但希望快速验证一段查询

:::warning 注意

SQL 表通常按只读结果使用，不适合承担普通业务表的新增、编辑和删除。如果查询逻辑需要长期维护，或者数据库侧已经有稳定 view，优先考虑 [连接数据库视图](./collection-view.md)。

:::

## 安装

SQL 表由 `collection-sql` 插件提供。启用插件后，可以在主数据库的数据表列表中选择「SQL collection」。

## 创建 SQL 表

创建 SQL 表时，需要先输入 SQL statement，再根据查询结果配置字段元数据。

| 属性 | 新增 / 编辑 | 说明 |
| --- | --- | --- |
| Collection display name | 新增、编辑 | SQL 表在界面中显示的名称，比如「销售汇总」「库存预警」。建议使用能说明查询结果含义的名称。 |
| Collection name | 新增、编辑时只读 | SQL 表的内部名称，用于 API、区块和权限等内部引用。新增时可以修改，编辑时不可修改。 |
| Categories | 新增、编辑 | 数据表分类。只影响数据表管理界面的组织方式，不改变 SQL 查询。 |
| Description | 新增、编辑 | 数据表说明。建议写清楚这段 SQL 查询了哪些数据、用于哪个报表或业务页面。 |
| Record unique key | 新增、编辑 | 记录唯一标识。SQL 查询结果没有真实主键时，需要选择一个能唯一定位记录的字段，否则可能无法在区块中正确查看记录。 |
| SQL statement | 新增、编辑 | SQL 表使用的查询语句。NocoBase 会基于查询结果配置字段元数据，再把查询结果作为数据表使用。 |
| Sources | 新增、编辑 | SQL 查询结果的字段来源。用于把查询结果中的字段和已有数据表字段关联起来，帮助 NocoBase 识别字段类型和界面类型。 |
| Fields | 新增，编辑时可重新配置 | 字段映射配置。用于确认每个字段的名称、标题、数据类型和界面类型。 |
| Preview | 新增、编辑 | 预览 SQL 查询结果。提交前可以先确认字段映射和展示效果是否符合预期。 |

## 字段映射

SQL 查询结果本身只告诉 NocoBase 返回了哪些列。为了让这些列像普通字段一样在界面中使用，还需要配置字段映射。

字段映射主要解决三件事：

| 配置 | 说明 |
| --- | --- |
| 字段来源 | 选择查询结果字段来自哪个已有数据表和字段。选中来源后，NocoBase 可以复用原字段的 Field type 和 Field interface。 |
| 字段类型 | 如果查询结果字段没有明确来源，需要手动确认字段的数据类型。 |
| 界面类型 | 确认字段在页面中如何展示和输入，比如单行文本、数字、日期、下拉选项。 |

比如 SQL 查询里返回 `users.email`，可以把它映射到用户表的邮箱字段。这样 NocoBase 就能按邮箱字段的界面类型展示，而不是只把它当作普通字符串。

如果字段来自计算结果，比如 `count(*) as total`，通常需要手动选择数字类型和合适的 Field interface。

## 记录唯一标识

SQL 表需要配置 Record unique key。它用于定位查询结果中的某一行。

适合作为 Record unique key 的字段通常满足这些条件：

- 查询结果中每一行都唯一
- 字段值稳定，不会因为分页、排序或统计口径变化而变化
- 字段不为空
- 字段在查询结果中始终返回

如果查询结果来自多个表，可以在 SQL 中返回一个稳定的业务 ID，或通过组合字段生成一个唯一值。不要使用 `row_number()` 这类会随排序变化的值作为长期稳定的记录唯一标识。

## 预览查询结果

提交前先使用 Preview 确认查询结果、字段映射和展示效果。预览时重点看：

- SQL 是否能正常执行
- 返回字段是否完整
- 字段类型是否符合预期
- Record unique key 是否存在并且唯一
- 查询结果是否适合页面展示

## 编辑 SQL 表

SQL 查询发生变化时，点击「Edit」可以修改 SQL statement，并重新确认字段来源、字段映射和预览结果。

如果只是底层表结构发生变化，SQL 语句本身没有变化，可以进入「Configure fields」，使用「Sync from database」同步字段，再检查字段映射。

:::warning 注意

修改 SQL statement 可能导致字段名称、字段类型或记录唯一标识变化。修改后请重新检查页面区块、图表、权限和工作流是否仍然可用。

:::

## 配置字段

SQL 表创建后，也可以通过「Configure fields」查看和维护字段元数据。

跟普通表不同，SQL 表的字段来自查询结果。通常不建议像普通表一样新增业务字段，而是调整 SQL 查询、字段来源和字段映射。

## SQL 表和数据库视图

| 对比项 | SQL 表 | 连接数据库视图 |
| --- | --- | --- |
| 查询逻辑位置 | 保存在 NocoBase 的 SQL 表配置中。 | 保存在数据库 view 中。 |
| 适合场景 | 快速验证查询、轻量报表、临时统计。 | 长期维护的复杂查询、DBA 管理的报表视图。 |
| 数据库对象 | 不创建新的数据库表或 view。 | 依赖数据库中已有 view。 |
| 字段同步 | 根据 SQL 查询结果同步。 | 根据数据库 view 结构同步。 |
| 增删改支持 | 通常不支持。 | 取决于数据库视图本身是否可写。 |

默认建议：临时查询和轻量报表用 SQL 表；长期稳定、复杂或需要数据库侧治理的查询，用数据库视图。

## 删除 SQL 表

删除 SQL 表会删除 NocoBase 中的 SQL 表配置和字段元数据，不会删除底层来源表。

删除前仍然需要检查页面区块、图表、权限、工作流和外部 API 是否仍在使用这个 SQL 表。
