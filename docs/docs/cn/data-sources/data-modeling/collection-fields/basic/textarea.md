# 多行文本

## 介绍

在 NocoBase 中，**多行文本（Multi-line text）** 用于保存需要换行或篇幅较长的文本内容。

多行文本默认使用多行输入框，适合备注、说明、处理意见等内容。它可以参与筛选、权限、工作流条件和 API 查询。

如果内容通常只有一行，比如名称、编号、标题，默认选择[单行文本](./input.md)更合适。如果内容需要排版、图片或富文本能力，选择富文本或 Markdown 字段。

## 适用场景

多行文本适合这些业务场景：

- 客户备注、订单备注、工单处理意见
- 详细地址、问题描述、需求说明
- 合同条款摘要、项目背景说明
- 需要换行录入的文本内容

## 创建配置

在数据表的「Configure fields」页面中，点击「Add field」，选择「多行文本」可以创建多行文本字段。

![20240512165017](https://static-docs.nocobase.com/20240512165017.png)

| 配置 | 说明 |
| --- | --- |
| Field interface | 字段的界面类型。多行文本对应 `textarea`，决定页面中如何录入和展示。 |
| Field display name | 字段在界面中显示的名称，比如「客户备注」「处理意见」「详细地址」。建议使用业务人员能直接理解的名称。 |
| Field name | 字段标识名称，用于 API、关系字段、权限、工作流等内部引用。创建后通常不再修改，只支持字母、数字和下划线，并且必须以字母开头。 |
| Field type | 字段在数据层的类型。多行文本默认是 `text`，也可以根据来源字段映射为 `string` 或 `json`。 |
| Default value | 默认值。新增记录时，如果用户没有填写，可以自动带出默认值。 |
| Validation rules | 校验规则。可以限制最小长度、最大长度、固定长度、大小写或正则表达式。 |
| Description | 字段说明。适合写字段含义、填写要求、数据来源或维护人。 |

:::warning 注意

字段名创建后会被页面区块、权限、工作流和 API 引用。创建前先确认命名，避免后续修改带来配置调整成本。

:::

## 字段特性

多行文本字段的默认行为如下：

| 特性 | 说明 |
| --- | --- |
| 默认 Field interface | `textarea`。 |
| 默认 Field type | `text`。 |
| 可选 Field type | `text`、`json`、`string`。 |
| 页面组件 | 编辑模式使用多行输入框。 |
| 筛选 | 支持文本类筛选，比如包含、不包含、为空、不为空等。 |
| 排序 | 通常不建议用于排序。是否可排序取决于具体数据库和字段类型。 |
| 校验 | 支持最小长度、最大长度、固定长度、大小写、正则表达式等校验。 |

## 编辑配置

创建后，点击字段右侧的「Edit」可以编辑多行文本字段配置。编辑字段主要用于调整字段在 NocoBase 中的展示和使用方式，比如修改显示名称、说明、默认值、校验规则或字段专属配置。

如果字段来自主数据库中已经同步的表，编辑时通常是在做字段映射——把数据库字段映射为 NocoBase 的 Field type 和 Field interface。

| 配置 | 允许编辑 | 说明 |
| --- | --- | --- |
| Field display name | 是 | 修改字段在界面中的显示名称，不改变字段标识名称。 |
| Field name | 否 | 字段标识名称创建后通常不能在编辑表单中修改。 |
| Field interface | 条件支持 | 主数据库字段或同步字段在字段映射时可以调整。调整后会影响页面输入、展示和校验方式。 |
| Field type | 条件支持 | 主数据库字段或同步字段在字段映射时可以调整。调整前需要确认已有数据能否按新类型使用。 |
| Default value | 是 | 调整新增记录时的默认值。 |
| Validation rules | 是 | 调整字段校验规则。 |
| Description | 是 | 补充字段含义、填写要求、数据来源或维护人。 |

:::warning 注意

切换 Field type 或 Field interface 不等于简单改一个显示名称。它会影响字段的存储方式、输入组件、校验规则、筛选条件和工作流变量使用方式。已有数据较多时，先确认数据格式是否匹配。

:::

## 删除字段

点击字段右侧的「Delete」可以删除多行文本字段。主数据库中还可以勾选多个字段后批量删除。

删除主数据库中新建的多行文本字段时，通常会同时删除数据库中的真实列及该列已有数据。删除从数据库同步或外部数据源映射出的字段时，影响范围取决于对应数据源和字段来源。

:::danger 警告

删除字段可能影响页面区块、表单、筛选、权限、工作流、API、导入导出和已有数据。删除前先确认字段是否仍被业务配置引用。

:::

## 页面配置使用

多行文本字段适合在表单和详情中展示较长内容。

![20260709224428](https://static-docs.nocobase.com/20260709224428.png)

| 场景 | 用途 |
| --- | --- |
| 表单区块 | 录入或编辑备注、说明、处理意见等内容。 |
| 详情区块 | 展示较长文本内容。 |
| 表格区块 | 展示摘要内容，较长内容通常会省略。 |
| 工作流和权限 | 作为条件字段参与判断，比如备注是否为空。 |
