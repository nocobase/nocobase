# 中国行政区

<PluginInfo name="field-china-region"></PluginInfo>

## 介绍

中国行政区字段用于在数据表中保存省、市、区县等中国行政区划信息。字段基于内置的 `chinaRegions` 行政区划数据表，提供级联选择器，用户在表单中可以按层级依次选择省、市、区。

适用场景包括：

- 客户、联系人、门店、项目等记录的所在地
- 户籍地、籍贯、收货地区等地址基础信息
- 需要按省市区筛选或统计的数据

字段值以关联记录的方式保存，默认关联 `chinaRegions` 数据表，并按行政区划层级排序展示。例如选择“北京市 / 市辖区 / 东城区”后，展示态会按层级拼接为完整路径。

## 字段配置

![](https://static-docs.nocobase.com/data-source-manager-main-NocoBase-04-29-2026_04_52_PM.png)

创建字段时，选择字段类型为“中国行政区”，可配置以下选项：

| 配置项 | 说明 |
| --- | --- |
| 选择层级 | 控制可选择到的最深层级。当前支持“省”“市”“区”，默认为“区”。“街道”“村”在界面中为禁用状态。 |
| 必须选择到最后一级 | 勾选后，用户必须选择到配置的最深层级才能提交；不勾选时，可以在中间层级完成选择。 |

## 界面使用

在表单中，中国行政区字段显示为级联选择器：

1. 打开下拉框时加载省级数据。
2. 展开某个省份时，按需加载其下级城市。
3. 继续展开城市时，按需加载区县。
4. 保存后，在详情、表格等展示场景中按层级显示为 `省/市/区`。

字段支持常见表单配置，例如字段标题、描述、必填、默认值、阅读模式等。阅读模式下，字段会显示为文本路径，例如：

```text
北京市 / 市辖区 / 东城区
```

![](https://static-docs.nocobase.com/%E7%9C%81%E5%B8%82%E5%8C%BA-04-29-2026_04_54_PM.png)

## 注意事项

- 中国行政区字段目前仅支持单条路径选择，不支持多选。
- 当前内置并启用的是省、市、区三级数据，街道和村级选项暂不可选。
- 导入时需要填写与内置行政区划数据一致的名称，并按层级用 `/` 分隔。
- 该字段依赖插件提供的 `chinaRegions` 数据表，请确保“中国行政区划”字段插件已启用。
