# 数据表

## 介绍

在 NocoBase 中，**Collection（数据表）** 是用来描述一类业务数据的数据模型。它不是单纯的数据库表名，而是 NocoBase 对一类数据的统一描述。

一个 Collection 通常定义三件事：

| 定义 | 说明 |
| --- | --- |
| 数据保存在哪里 | 数据可以来自主数据库表、外部数据库表、SQL 查询结果、数据库视图、REST API 资源或外部 NocoBase 应用。 |
| 有哪些字段 | 字段描述每条记录有哪些信息，比如客户名称、手机号、订单金额、创建时间、负责人。 |
| 如何被 NocoBase 使用 | 页面区块、权限、工作流、API 和关系字段都会基于 Collection 工作。 |

你可以把 Collection 理解为“业务对象的数据结构”。比如「客户」「订单」「合同」「任务」都可以是一个 Collection。

数据表创建或接入后，通常还需要继续做三件事：

- 配置字段，让数据表能保存业务需要的信息
- 在页面中[添加区块](../../interface-builder/blocks/index.md#添加区块)，让用户可以查看、录入和处理数据
- 配置权限、工作流和 API，让数据可以按业务规则被访问和流转

## 表结构类型

- **普通表** — 适合保存客户、订单、合同、工单、报销单、项目、任务等常规业务数据
- **树表** — 适合保存组织架构、商品分类、地区层级、部门目录、知识库目录等上下级数据
- **日历表** — 适合保存会议室预约、项目排期、课程安排、值班计划、活动日程等带时间范围的数据
- **评论表** — 适合保存任务评论、文章评论、审批意见、客户反馈等围绕业务记录产生的讨论内容；在业务表（普通表、树表、日历表）中创建[关系字段](./collection-fields/associations/index.md)关联评论表，使用业务表弹窗页面创建[评论区块](../../plugins/@nocobase/plugin-comments/index.md)，对业务数据进行评论
- **文件表** — 适合保存合同附件、发票文件、产品图片、员工证件等文件元信息，文件实际由文件存储引擎保存；在业务表（普通表、树表、日历表）中创建[关系字段](./collection-fields/associations/index.md)关联文件表，使用业务表创建区块配置关系字段上传文件，文件元信息会自动保存到文件表中
- **数据库视图** — 数据库里已有的 view，比如财务报表视图、过滤后的客户视图、跨系统同步后的聚合视图
- **SQL 表** — 适合把销售汇总、库存预警、跨表统计报表、运营看板等 SQL 查询结果作为数据表使用
- **继承表** — 多类业务对象共享一组公共字段，同时每一类又有自己的专属字段，比如资产父表派生电脑资产、车辆资产、办公家具
