# REST API 和外部 NocoBase 对比

REST API 数据源和外部 NocoBase 都用于接入外部系统的数据。区别在于：REST API 面向任意第三方 HTTP API，需要你手动配置接口映射；外部 NocoBase 面向另一个 NocoBase 应用，可以直接复用远端应用的数据表和字段元数据。

默认判断很简单：

- 外部系统是 NocoBase 应用，用 **外部 NocoBase**
- 外部系统只是提供 REST API，用 **REST API 数据源**

## 核心差异

| 对比项 | REST API 数据源 | 外部 NocoBase |
| --- | --- | --- |
| 接入对象 | 任意 RESTful API 或近似 REST 的 HTTP API。 | 另一个 NocoBase 应用。 |
| 配置方式 | 手动配置 List、Get、Create、Update、Destroy 接口映射。 | 填写远端 NocoBase API 地址、Origin、API key 等信息后加载数据表。 |
| 字段来源 | 从接口响应中提取字段，再手动确认 Field type 和 Field interface。 | 直接复用远端 NocoBase 的数据表字段元数据。 |
| 关系字段 | 需要根据接口能力手动设计和适配。 | 通常可以复用远端应用已有关系字段。 |
| 文件字段 | 需要接口自行支持上传、下载和预览。 | 文件上传和预览会代理到远端 NocoBase。 |
| 权限 | 当前应用权限 + 第三方 API 自身权限。 | 当前应用权限 + 远端 NocoBase API key 权限。 |
| 工作流 | 当前应用通过接口读写外部数据。 | 当前应用可操作远端记录，但远端数据表事件在远端应用触发。 |
| 维护成本 | 接口字段、参数、响应结构变化后，需要更新映射。 | 远端 NocoBase 结构变化后，需要重新加载数据源。 |

## 什么时候用 REST API 数据源

REST API 数据源适合接入非 NocoBase 系统。比如 CRM、ERP、工单系统、支付系统或自研服务只提供 HTTP API 时，可以把这些接口包装成 NocoBase 数据表。

它更适合这些场景：

- 第三方系统没有开放数据库访问
- 第三方 API 已经封装好业务逻辑
- 需要实时调用外部服务
- 接口结构比较稳定，可以长期维护映射
- 只需要接入少量资源，比如客户、订单、工单

配置时，建议先打通 List 和 Get，再配置 Create、Update、Destroy。这样可以先确认读取、分页、字段映射和记录唯一标识都是正确的。

## 什么时候用外部 NocoBase

外部 NocoBase 适合连接另一个已经搭好的 NocoBase 应用。它可以复用远端应用的数据表、字段界面、标题字段、关系字段和部分数据能力。

它更适合这些场景：

- 远端系统本身就是 NocoBase
- 希望复用远端应用已有数据模型
- 不想重新配置字段界面和关系字段
- 需要在当前应用中查看、编辑远端数据
- 多个 NocoBase 应用之间需要共享数据

外部 NocoBase 的字段结构以远端应用为准。需要调整字段时，请在远端应用修改，再回到当前应用重新加载数据源。

## 配置复杂度

| 配置内容 | REST API 数据源 | 外部 NocoBase |
| --- | --- | --- |
| 数据源连接 | 配置 Base URL、请求头、变量、超时时间。 | 配置 API 地址、Origin、API key、请求头、超时时间。 |
| 数据表 | 每个资源都要创建 Collection。 | 自动加载远端应用已有数据表。 |
| 接口操作 | 需要逐个配置 List、Get、Create、Update、Destroy。 | 使用远端 NocoBase 已有 API 能力。 |
| 字段 | 从响应中提取并映射字段。 | 复用远端字段元数据。 |
| 调试 | 需要调试参数映射、响应转换和异常转换。 | 重点检查远端地址、API key 和权限。 |

## 字段和记录唯一标识

REST API 数据源需要从接口响应中提取字段。你需要确认字段名称、字段类型、界面类型和记录唯一标识。没有正确的记录唯一标识时，详情、编辑和删除通常无法稳定工作。

外部 NocoBase 会使用远端应用的数据表元数据。通常来说，字段界面、标题字段和关系字段会跟随远端应用，不需要在当前应用重新配置。

## 权限差异

两种数据源都会同时受当前应用和外部系统影响。

| 权限问题 | REST API 数据源 | 外部 NocoBase |
| --- | --- | --- |
| 当前应用用户权限 | 由当前应用控制。 | 由当前应用控制。 |
| 外部系统权限 | 由第三方 API 的认证方式控制。 | 由远端 NocoBase API key 控制。 |
| 排查方向 | 检查请求头、token、第三方 API 返回错误。 | 检查当前应用权限和远端 API key 权限。 |

如果页面上能看到按钮但操作失败，通常说明外部系统拒绝了请求，或者当前应用服务端权限校验没有通过。

## 文件和附件

REST API 数据源是否支持文件，取决于第三方 API 是否提供上传、下载和预览接口。你需要按接口要求配置请求和响应转换。

外部 NocoBase 的文件会保存到远端应用使用的存储中。当前应用只负责代理上传、预览和下载请求。使用本地文件存储时，需要正确配置 `Origin`，否则文件可能上传成功但无法预览。

## 工作流差异

| 能力 | REST API 数据源 | 外部 NocoBase |
| --- | --- | --- |
| 当前应用查询记录 | 支持。 | 支持。 |
| 当前应用创建、更新、删除记录 | 取决于是否配置对应接口。 | 支持，前提是远端 API key 有权限。 |
| 数据表事件 | 当前应用只能感知本地请求链路中的操作。 | 远端数据表内部事件只在远端应用触发。 |
| SQL 节点和聚合节点 | 通常不适用。 | 通常不适用。 |

需要围绕外部数据编排流程时，建议优先使用请求前事件、请求后事件、自定义操作事件，或在远端系统中配置自己的自动化逻辑。

## 怎么选

| 你的情况 | 建议 |
| --- | --- |
| 对方系统是 NocoBase | 使用外部 NocoBase。 |
| 对方系统不是 NocoBase，只提供 HTTP API | 使用 REST API 数据源。 |
| 需要复用远端数据表、字段和关系配置 | 使用外部 NocoBase。 |
| 需要接入第三方系统的少量接口 | 使用 REST API 数据源。 |
| 第三方 API 响应结构经常变化 | 谨慎使用 REST API 数据源，需要维护映射。 |
| 文件和附件能力很重要，且远端是 NocoBase | 使用外部 NocoBase。 |
