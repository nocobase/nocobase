# 应用区块和应用切换

多应用功能除了在后台统一管理子应用，也可以在前台为用户提供应用入口。常见方式包括：

- 在页面中添加「应用」区块，集中展示可访问的子应用
- 在左上角启用应用切换列表，方便用户在主应用和子应用之间切换

## 应用区块

![](https://static-docs.nocobase.com/202605271350840.png)

「应用」区块用于在前台页面中展示子应用列表，适合做成一个简单的应用门户页。用户可以在主应用中添加该区块，让最终用户从一个页面进入不同业务应用。

区块中的每个应用会显示：

- 应用图标
- 应用名称
- 访问入口

点击应用后，会打开对应子应用。

### 配置应用图标

在「应用监管器」中创建或编辑应用时，可以在「展示配置」中上传应用图标。

如果没有上传图标，系统会使用应用名称的首字母生成一个默认图标，便于在列表中区分不同应用。

![](https://static-docs.nocobase.com/202605271402603.png)

### 隐藏应用

如果某个应用不希望出现在前台「应用」区块中，可以在应用配置中勾选「在应用区块中隐藏」。

隐藏后：

- 应用仍然可以在后台管理
- 应用仍然可以通过直接访问地址进入
- 只是不再显示在前台「应用」区块中

![](https://static-docs.nocobase.com/202605271403980.png)

## 应用切换列表

![](https://static-docs.nocobase.com/202605271403304.png)

应用切换列表显示在页面左上角，用于快速切换到其他应用。

如果希望某个应用显示在应用切换列表中，可以在应用配置中开启「显示在应用切换列表中」。

开启后，用户可以在主应用或子应用的左上角看到应用切换入口，并从列表中进入其他应用。

![](https://static-docs.nocobase.com/202605271404322.png)

### 打开方式

应用切换时的打开方式如下：

- 从主应用进入子应用：在新标签页打开
- 从一个子应用切换到另一个子应用：在当前标签页打开

这样可以避免从主应用进入子应用时打断主应用操作，同时也让子应用之间切换更加自然。
