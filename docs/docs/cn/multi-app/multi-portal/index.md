# 多工作区

## 什么是工作区

工作区（Portal）用于在同一个应用内提供多个访问入口。

每个工作区都可以拥有独立的：

- 页面
- 菜单
- 导航结构
- 布局
- 权限配置

多工作区插件提供以下能力：

- 工作区管理
- 工作区切换
- 工作区权限控制

借助这些能力，可以在共享同一套数据和业务能力的前提下，为不同角色提供差异化的使用体验。

## 为什么需要工作区

在实际业务场景中，不同角色通常需要不同的操作界面。

例如一个零售管理系统：

```text
零售管理系统

├─ 总部工作区
├─ 门店工作区
├─ 经销商工作区
└─ 移动工作区
```

总部员工关注：

- 商品管理
- 库存管理
- 数据分析

门店员工关注：

- 收银
- 库存盘点
- 订单处理

经销商关注：

- 采购
- 对账
- 发货状态

虽然使用的是同一个系统，但不同角色并不需要看到相同的菜单和页面。

工作区正是用来解决这一问题的。

## 工作区与菜单的关系

每个工作区都有独立的菜单树。

不同工作区之间的菜单互不影响。

例如：

```text
总部工作区
├─ 商品管理
├─ 供应链管理
└─ 数据分析

门店工作区
├─ 收银
├─ 订单管理
└─ 库存盘点
```

## 工作区与页面的关系

页面归属于各自的工作区。

同一个页面也可以只在特定工作区中显示。

因此，可以为不同角色设计完全不同的操作流程。

## 工作区与权限的关系

工作区本身可以配置访问权限。

只有获得授权的用户，才能访问对应的工作区。

未授权的工作区：

- 不会出现在切换列表中
- 无法直接访问对应入口

## 工作区管理

启用多工作区插件后，系统会默认提供两个内置工作区：

| 工作区 | 路径 | 用途 |
|----------|----------|----------|
| Desktop | `/v/admin` | 桌面端入口 |
| Mobile | `/v/mobile` | 移动端入口 |

## 创建工作区

除了系统内置工作区外，还可以根据业务需求创建更多工作区。

例如：

- 门店工作区
- 经销商工作区
- 客户服务工作区
- 数据分析工作区

创建后即可为其配置：

- 页面
- 菜单
- 权限
- 导航

## 工作区权限

可以通过角色权限控制用户能够访问哪些工作区。

未被授权的工作区不会显示在工作区切换列表中，用户也无法直接访问对应入口。
