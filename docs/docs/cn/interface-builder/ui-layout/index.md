# UI 布局

NocoBase 提供桌面端布局和移动端布局。两种布局都可以使用界面搭建能力创建页面，并在页面中配置区块、字段和操作。

桌面端布局是默认选择，适合在电脑中完成日常管理和数据处理。如果需要独立的移动导航和移动页面，可以另外搭建移动端布局。

## 桌面端布局

[桌面端布局](./desktop.md) 默认通过 `/admin` 访问。它由顶部导航、侧边导航和页面内容区组成，适合表格管理、表单录入、数据查看等常见业务场景。

桌面端布局也支持窄屏响应式。当页面在较窄的屏幕中显示时，导航、间距和常用组件会调整为更适合小屏幕的形式，仍然使用原有的桌面端菜单和页面。

## 移动端布局

[移动端布局](./mobile.md) 默认通过 `/mobile` 访问。它使用底部标签栏组织一级导航，并提供独立的移动端页面、链接和页面标签页。

移动端布局适合需要在手机上频繁操作的场景，比如现场录入、移动审批、任务处理和数据查询。你可以在电脑浏览器中搭建和预览页面，再通过二维码在真机中检查效果。

## 如何选择

默认使用桌面端布局即可。

| 我想要…… | 推荐使用 |
| --- | --- |
| 主要在电脑中使用，偶尔需要通过手机访问 | [桌面端布局](./desktop.md) |
| 为手机单独设计导航、页面和操作流程 | [移动端布局](./mobile.md) |
| 同时提供完整的电脑端和移动端体验 | 分别搭建桌面端布局和移动端布局 |

## 配置之间的关系

桌面端布局和移动端布局使用相同的数据源、集合和业务数据。你可以在两种布局中基于同一个集合分别搭建适合不同设备的页面。

菜单、路由和页面配置分别维护。修改桌面端页面不会自动更新移动端页面，移动端导航的变化也不会影响桌面端导航。两种布局的[路由访问权限](../../users-permissions/acl/permissions.md)同样需要分别配置。
