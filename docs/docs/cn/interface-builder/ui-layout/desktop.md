# 桌面端布局

在 NocoBase 中，**桌面端布局**是默认的应用界面。它适合在电脑中完成数据管理、表单录入、业务配置和日常操作，也能在窄屏设备中调整导航与页面内容。

桌面端布局默认通过 `/admin` 访问。如果应用配置了独立的访问前缀，实际地址会自动带上对应前缀。

## 布局特点

桌面端布局主要由以下区域组成：

- **顶部导航** — 显示应用切换和全局操作入口。
- **侧边导航** — 显示当前分组下的页面和链接。
- **页面内容区** — 展示页面标签页、区块、字段和操作。
- **界面配置** — 用于进入界面搭建状态，调整菜单和页面内容。

顶部导航和侧边导航会根据当前路由保持选中状态。切换页面时，页面内容显示在右侧内容区，已经打开的页面状态通常会被保留。

## 搭建页面

### 第一步：进入桌面端布局

访问 `/admin`，进入桌面端布局。通常来说，应用登录成功后也会直接进入这里。

<!-- 需要一张包含 /admin 地址和登录后默认落地页的截图 -->

### 第二步：打开界面配置

点击页面右上角的「界面配置」，进入界面搭建状态。进入后，菜单、页面、区块、字段和操作附近会显示对应的配置入口。

<!-- 需要一张「界面配置」按钮位置和开启后页面效果的截图 -->

### 第三步：创建菜单和页面

你可以在导航区域添加分组、页面或链接，也可以为页面启用标签页。创建页面后，进入页面并添加需要的区块。

页面内容的搭建方式和其他界面一致：先添加[区块](../blocks/index.md)，再根据业务需要配置[字段](../fields/index.md)和[操作](../actions/index.md)。

<!-- 需要一个展示添加菜单、创建页面并进入页面的视频 -->

### 第四步：配置页面内容

在页面中添加表格、表单、详情、筛选等区块，然后调整字段、操作和区块之间的排列。所有修改都会直接反映在当前页面中。

## 管理路由和菜单

桌面端菜单和路由使用同一份配置。在导航区域添加页面或链接后，相应内容也会出现在[路由管理器](../../routes/index.md)中；在路由管理器中调整路由，也会同步更新菜单。

桌面端支持以下常用路由：

- **分组（Group）** — 将多个页面和链接组织到同一个导航分组中。
- **页面（Page）** — 打开一个可以继续搭建区块的页面。
- **链接（Link）** — 打开站内地址或外部地址。
- **标签页** — 在一个页面中组织多个内容标签。

你可以在路由管理器中新增、编辑、删除、显示或隐藏路由。需要集中整理菜单结构时，使用路由管理器会更方便。

## 窄屏响应式

桌面端布局可以直接在手机或较窄的浏览器窗口中使用。进入窄屏显示状态后，布局仍然使用原来的桌面端路由和页面，不会自动切换到移动端布局。

### 布局变化

导航菜单会进入折叠状态，顶部操作会收进更紧凑的入口。页面的边距和区块间距也会缩小，内容区域会根据移动浏览器的可视高度调整。

窄屏中不提供界面配置。需要修改菜单或页面时，建议回到电脑浏览器中完成搭建。

<!-- 需要一个展示同一桌面端页面从宽屏切换到窄屏效果的视频 -->

### 页面内容的适配

页面中的常用布局和组件也会针对窄屏调整。比如多列区块会更适合纵向浏览，表格可以横向查看超出屏幕的列，分页和操作入口会更加紧凑；选择、日期时间、筛选和子页面等交互也会使用更适合小屏幕的方式。

其他区块是否具有额外的窄屏行为，取决于对应区块自身的支持情况。表格在窄屏中仍然保持表格形态，不会自动转换成卡片。

:::tip 桌面端响应式和移动端布局

如果只是偶尔通过手机访问，使用桌面端布局的窄屏响应式即可。如果需要独立的底部导航、移动页面和操作流程，可以另外搭建[移动端布局](./mobile.md)。

:::

## 使用建议

- 主要在电脑中完成的业务，默认使用桌面端布局。
- 先在宽屏中完成页面搭建，再缩小窗口检查窄屏效果。
- 页面包含较多表格列或横向操作时，尽量保留必要内容，减少小屏幕中的操作负担。
- 如果电脑端和移动端的使用流程差异较大，分别搭建两套页面会更清晰。
