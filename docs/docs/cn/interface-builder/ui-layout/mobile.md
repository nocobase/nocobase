# 移动端布局

在 NocoBase 中，**移动端布局**用于搭建独立的移动导航和页面。它默认通过 `/mobile` 访问，使用底部标签栏作为一级导航，更适合在手机上完成数据录入、查询、审批和任务处理。

移动端布局和桌面端布局使用相同的数据源和业务数据，不过菜单、路由和页面内容分别配置。这样可以根据移动端的使用方式重新组织页面，而不必受到桌面端页面结构的限制。

## 进入和预览移动端布局

启用 UI 布局插件后，可以在设置中心点击「移动端」进入，也可以直接访问 `/mobile`。

建议在电脑浏览器中完成页面搭建。电脑中会显示移动端预览区域和顶部工具栏，其中：

- 「UI 配置」用于打开或关闭界面搭建状态。
- 「平板预览」用于检查较宽移动设备中的显示效果。
- 「移动端预览」用于恢复手机尺寸的预览区域。
- 「二维码」用于在手机上打开当前移动端地址。

<!-- 需要一张桌面浏览器中的移动端预览截图，标出 UI 配置、平板预览、移动端预览和二维码 -->

在电脑中搭建完成后，再使用二维码进入真机检查。需要重点确认导航、滚动、表单输入、弹出页面和安全区是否符合预期。

## 搭建移动端导航

移动端使用底部标签栏作为一级导航。一级导航目前主要支持页面和链接。

### 添加页面

1. 打开「UI 配置」。
2. 点击底部标签栏右侧的添加按钮。
3. 选择「页面」。
4. 填写页面标题并选择图标。
5. 提交后进入新页面，继续添加页面内容。

<!-- 需要一个展示在底部标签栏添加移动端页面的视频 -->

### 添加链接

如果需要跳转到站内地址或外部地址，可以选择「链接」，然后配置标题、图标和 URL。

链接可以在当前窗口打开，也可以在新窗口打开。具体行为取决于链接的配置。

<!-- 需要一张添加移动端链接的配置截图 -->

### 调整导航

在界面搭建状态下，可以拖拽底部标签调整顺序。每个标签还可以编辑标题和图标、配置联动规则、复制 UID 或删除。

如果需要集中查看、显示、隐藏或删除移动端路由，可以打开「设置中心 / 路由 / 移动端路由」。

## 搭建移动端页面

先创建并进入一个移动端页面，再在页面中添加区块。页面内容的搭建思路和桌面端基本一致，都可以使用[区块](../blocks/index.md)、[字段](../fields/index.md)和[操作](../actions/index.md)组织业务内容。不过移动端的导航和部分组件交互会针对小屏幕调整。

### 添加页面内容

1. 进入需要搭建的移动端页面。
2. 确认「UI 配置」已经打开。
3. 点击页面中的「添加区块」。
4. 选择表格、表单、详情、筛选或其他区块。
5. 继续配置字段、操作和区块设置。

<!-- 需要一个展示进入移动端页面并添加区块的视频 -->

### 使用页面标签页

一个移动端页面也可以启用标签页。适合放在同一导航入口下、但内容相对独立的页面，可以分别放入不同的标签页中。

1. 打开页面设置，启用「启用标签页」。也可以在「设置中心 / 路由 / 移动端路由」中编辑页面并勾选「启用页面选项卡」。
2. 打开「UI 配置」。
3. 点击页面标签栏右侧的「添加标签页」。
4. 添加标签页，并继续配置名称和页面内容。

如果移动端页面的内容较少，直接使用一个页面即可，不需要额外启用标签页。

### 常用组件的移动交互

页面中的常用组件会根据移动端布局调整排列和交互方式。比如多列内容会更适合纵向浏览，选择和日期时间字段会使用移动选择方式，筛选、关联记录和子页面也会使用更适合触摸操作的界面。

表格在移动端仍然保持表格形态，可以横向查看超出屏幕的列。其他区块是否具有额外的移动端行为，取决于对应区块自身的支持情况。

## 页面和子页面

从查看、编辑、关联选择等操作打开的内容，会以移动端子页面的形式显示。子页面提供返回按钮，返回后会回到之前的页面。

进入较深的子页面时，底部标签栏会暂时隐藏，为当前内容留出更多空间。关闭子页面或返回上一级后，底部标签栏会重新显示。

在底部标签之间切换时，已经打开的页面状态通常会被保留，方便在多个移动任务之间切换。

## 管理路由和权限

移动端路由可以在[路由管理器](../../routes/index.md)中集中维护。打开「设置中心 / 路由 / 移动端路由」，可以新增、编辑、删除、显示或隐藏页面和链接，也可以为页面配置标签页。

移动端路由的访问权限和桌面端分别配置。在角色权限中打开「移动端路由」，选择当前角色可以访问的页面。详细说明见[权限配置](../../users-permissions/acl/permissions.md)。

## 和桌面端布局的关系

桌面端布局和移动端布局可以基于同一个集合分别搭建页面。比如桌面端使用包含较多字段的表格处理数据，移动端则使用更简洁的列表或表单完成现场录入。

两种布局的页面不会自动同步。修改桌面端页面、菜单或路由时，移动端配置不会跟着变化；移动端的修改同样不会影响桌面端。

:::tip 使用建议

如果移动端只需要偶尔查看桌面页面，可以先使用[桌面端布局](./desktop.md)的窄屏响应式。只有需要独立移动导航和页面流程时，才需要单独搭建移动端布局。

:::
