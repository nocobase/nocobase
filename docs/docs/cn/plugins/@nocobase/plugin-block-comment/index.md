# 区块：评论

## 介绍

在 NocoBase 中，**评论区块** 用于给业务记录添加评论能力。你可以在任务、文章、工单、客户等记录的详情页或弹窗里添加评论区块，让用户围绕当前记录查看、回复和新增评论。

评论区块本身不创建数据表。使用前需要先准备一张用于保存评论的数据表，并配置好评论内容、评论人、评论归属和评论时间等字段。

:::tip 提示

如果你使用的是「评论表」模板，可以先参考 [评论表](../../../data-sources/collection-comment/) 文档了解评论数据表的创建方式。

:::

## 准备评论数据表

评论区块需要一张保存评论记录的数据表。通常来说，这张表至少包含这些字段：

| 字段用途 | 推荐字段类型 | 说明 |
| --- | --- | --- |
| 评论内容 | 长文本、Markdown、富文本等 | 用于保存评论正文。 |
| 评论人 | 多对一关系，关联用户表 | 用于显示评论创建人。 |
| 评论归属 | 普通标量字段，比如 `taskId`、`postId` | 用于把评论记录关联到某条业务记录。 |
| 评论时间 | 日期时间字段，或创建时间字段 | 用于显示和排序评论。 |

如果评论表通过关系字段关联业务表，也可以保留这层关系。不过在手动配置「评论归属字段」时，推荐选择实际保存外键值的标量字段，比如 `taskId`，而不是选择关系字段本身。

## 添加评论区块

评论区块通常添加在某条业务记录的详情页或弹窗中。

1. 打开目标记录的详情页或弹窗
2. 点击「添加区块」
3. 选择「评论」
4. 选择用于保存评论的数据表
5. 按提示完成字段映射

如果评论区块是从关联关系创建的，NocoBase 会尽量根据当前关系自动识别评论归属字段和当前记录值。这种情况下，「评论归属字段」和「评论归属字段值」会被自动填充，通常不需要手动配置。

如果是直接从评论数据表创建区块，则需要手动配置评论归属字段和值。

## 字段映射

评论区块通过「字段映射」知道每条评论应该怎么显示、怎么保存。字段映射包含以下配置：

| 配置项 | 作用 |
| --- | --- |
| 评论内容字段 | 选择保存评论正文的字段。 |
| 评论人字段 | 选择关联用户表的多对一字段。 |
| 评论归属字段 | 选择保存当前业务记录标识的字段。 |
| 评论归属字段值 | 指定当前业务记录的值，比如 `ctx.record.id`。 |
| 评论日期字段 | 选择评论时间字段，用于显示和默认排序。 |

### 评论归属字段

「评论归属字段」用于筛选当前记录的评论，也会在新增评论时写入对应的归属值。

手动选择时，下拉列表只显示普通标量字段，不显示关系字段。常见配置是：

| 业务表 | 评论表中的归属字段 | 评论归属字段值 |
| --- | --- | --- |
| 任务表 | `taskId` | `ctx.record.id` |
| 文章表 | `postId` | `ctx.record.id` |
| 工单表 | `ticketId` | `ctx.record.id` |

如果当前记录使用的唯一标识不是 `id`，可以把「评论归属字段值」改成对应字段，比如 `ctx.record.uuid`。

### 从关联关系创建时的自动映射

如果区块是从业务记录的关联关系中创建的，评论区块会优先使用该关联关系的外键字段作为评论归属字段，并使用当前记录值作为评论归属字段值。

比如任务表和任务评论表之间有一对多关系，任务评论表中保存外键的字段是 `taskId`。从任务详情页的关联关系添加评论区块时，区块会自动使用：

- 评论归属字段：`taskId`
- 评论归属字段值：当前任务记录的标识

这种方式更适合大多数场景，也能减少手动配置出错。

## 区块配置项

评论区块支持以下配置项：

### 字段映射

配置评论内容、评论人、归属关系和评论时间。第一次添加区块后，建议先检查字段映射是否完整。如果映射不完整，区块会显示提示信息，不会加载评论列表。

### 分页大小

设置每页显示的评论数量。可选值包括 `5`、`10`、`20`、`50`、`100`、`200`。

### 数据范围

设置评论列表的数据筛选范围。你可以在这里继续增加条件，比如只显示符合某些状态或权限条件的评论。

### 默认排序规则

设置评论列表的默认排序。通常可以按评论日期字段升序或降序排序。

如果没有单独配置默认排序，评论区块会优先使用「评论日期字段」作为默认排序字段。

### 默认跳转到最后一页

默认不开启。关闭时，评论区块打开后停留在第一页。

如果开启，评论区块会在首次加载时跳转到最后一页，适合希望优先看到最新评论的场景。

## 注意事项

- 评论区块需要启用 `@nocobase/plugin-block-comment` 插件
- 老的 `@nocobase/plugin-comments` 区块会继续兼容已有配置，不过不建议再新增老评论区块
- 手动配置「评论归属字段」时，优先选择外键标量字段，比如 `taskId`
- 如果「评论归属字段值」为空，评论区块无法判断当前记录对应哪些评论，会显示提示信息
- 如果归属字段和值的数据类型不匹配，区块会提示重新配置字段映射
