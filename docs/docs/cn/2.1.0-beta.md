# NocoBase 2.1.0-beta.23 发布日志

NocoBase 2.1.0-beta.23 是一次围绕 AI 能力全面升级的 Beta 版本。

这一版本的重点，不只是新增几个功能点，而是把 NocoBase 从“可配置的低代码平台”进一步推进为“可与 AI Agent 协同工作的业务系统平台”。从 CLI 接入、AI 搭建，到 AI 员工、MCP 和 Skills，2.1.0-beta.23 初步打通了 AI 参与系统搭建、业务执行与环境管理的完整链路，形成了从环境接入、系统搭建到业务协作的完整能力框架。

如果你正在关注以下方向，这一版本尤其值得体验：

- 希望让 AI Agent 直接参与 NocoBase 的搭建和配置
- 希望通过自然语言完成数据建模、页面搭建、工作流编排和权限配置
- 希望在业务系统中引入可执行任务的 AI 员工
- 希望通过 CLI 和标准化能力包提升团队协作与自动化效率

## 版本亮点

### NocoBase CLI 正式成为 AI Agent 接入入口

在 2.1.0-beta 中，NocoBase CLI（`nb`）成为连接 AI Agent 与 NocoBase 应用的核心入口。

CLI 用于在本地工作区初始化、连接和管理 NocoBase 应用，支持两类常见场景：

- 连接已有的 NocoBase 应用，并保存为 CLI env
- 通过 Docker、npm 或 Git 安装新的 NocoBase 应用，再保存为 CLI env

这意味着，无论你是要把 AI 接入现有业务系统，还是从零开始创建一个新的 NocoBase 应用，都可以通过统一的 CLI 方式完成初始化和后续管理。

对于团队协作来说，CLI 的意义不只是“安装工具”，更重要的是提供了一个适合 AI Agent 理解和操作的标准入口，让环境初始化、连接配置、运行管理和后续维护都更加一致。

相关文档：

- [AI Agent 接入指南](https://docs.nocobase.com/cn/ai/quick-start)
- [NocoBase CLI 命令参考](https://docs.nocobase.com/cn/api/cli)

### AI 搭建能力成型，从一句话到一整套系统

2.1.0-beta 将 AI 搭建能力作为核心体验之一。用户可以直接用自然语言描述业务需求，由 AI 协助完成数据建模、页面配置、权限设置和工作流编排。

相比传统低代码搭建方式，这种模式的价值在于：

- 降低上手门槛，不必先熟悉全部配置概念
- 缩短从需求描述到原型落地的路径
- 让数据、界面和流程配置可以由 AI 连续完成

无论是“帮我设计一个 CRM 数据模型”，还是“帮我创建一个客户管理页面”，甚至是“帮我编排一个订单创建后自动扣减库存的工作流”，AI 都可以在 NocoBase 的能力边界内协助完成。

相关文档：

- [AI 搭建快速开始](https://docs.nocobase.com/cn/ai-builder)

### NocoBase Skills 覆盖 AI 搭建全流程

为了让 AI 真正理解 NocoBase 的配置体系，2.1.0-beta 提供了一组可安装到 AI Agent 中的领域知识包：NocoBase Skills。

这些 Skills 不是简单的提示词集合，而是围绕 NocoBase 关键能力域进行组织的标准化知识与操作能力封装，帮助 AI 更准确地理解对象模型、配置结构和执行边界。

当前已提供 8 个 Skills，覆盖搭建全流程：

- 环境管理：环境检查、安装部署、升级和故障诊断
- 数据建模：创建和管理数据表、字段、关联关系
- 界面配置：创建和编辑页面、区块、弹窗、交互联动
- 工作流管理：创建、编辑、启用和诊断工作流
- 权限配置：管理角色、权限策略、用户绑定和风险评估
- 解决方案：从 YAML 批量搭建整套业务系统
- 插件管理：查看、启用和停用插件
- 发布管理：跨环境发布、备份恢复和迁移

通过 Skills，AI 不再只是“知道你在说什么”，而是更接近“知道该如何在 NocoBase 中把这件事搭出来”。

相关文档：

- [NocoBase Skills](https://github.com/nocobase/skills)

### AI 插件开发

2.1.0-beta 进一步补齐了 AI 插件开发所需的基础能力，让 AI 不只是参与应用搭建，也能更深入地参与 NocoBase 自定义插件开发。

这一部分的重点主要体现在三个方面：

- 统一使用 `rsbuild/rspack` 构建，进一步收敛插件开发和前端构建体系
- 提供面向后续 AI 开发的 `client-v2` 能力和 `/v2/` 路由体系，为新一代客户端插件开发做准备
- 提供 AI 插件开发相关 Skill，让 AI 能更好地理解插件结构、代码组织和实现方式

围绕 `client-v2` 的这轮准备，当前已经包括：

- `@nocobase/app` 提供 `client-v2`
- 内核提供 `@nocobase/client-v2`
- 各插件提供 `/src/client-v2` 目录
- 路由新增 `/v2/`
- 内核逐步迁移到 V2
- 插件逐步迁移到 V2

这意味着，NocoBase 正在为 AI 辅助插件开发建立更统一、更清晰的技术底座。一方面，统一的构建链路有助于降低前端插件开发和调试成本；另一方面，`client-v2` 的逐步落地，也是在为 AI 生成和维护插件代码提供更稳定的目标结构。

对于开发者来说，这一能力的价值在于：你可以直接用自然语言描述一个插件需求，由 AI 协助生成前后端代码、数据表、API、权限配置和国际化内容，把 AI 从“代码建议工具”进一步推进到“插件开发协作工具”。

相关文档：

- [AI 开发插件快速开始](https://docs.nocobase.com/cn/ai-dev/)
- [插件开发](https://docs.nocobase.com/cn/plugin-development/)

### AI 员工能力持续增强，内核能力进一步完善

如果说 AI 搭建解决的是“如何用 AI 搭系统”，那么 AI 员工解决的就是“如何让 AI 进入系统里工作”。

AI 员工并不是 2.1.0-beta 才首次提供的能力。在这一版本中，NocoBase 进一步增强了 AI 员工相关能力，并对 AI 相关内核能力做了完善，包括：

- 内核提供 AI 扩展接口
- 提供 MCP Server
- 支持接入 MCP
- 支持 AI 员工子代理调度
- 提供 AI 员工节点
- 提供 AI 聚合工具
- 提供基于 LLM 的联网搜索工具
- 支持生成报告

这些增强和内核改进，进一步提升了 AI 员工在业务系统中的可扩展性、可编排性和执行能力，也让 AI 不再局限于对话辅助，而是可以逐步成为业务系统中的“数字同事”：

- 理解当前业务上下文
- 调用技能执行具体任务
- 参与自动化流程
- 结合外部信息完成分析与输出

对于需要在业务系统中引入智能协作、自动分析和任务执行能力的团队来说，这会是 2.1.0-beta 最值得关注的方向之一。

相关文档：

- [AI 员工](https://docs.nocobase.com/cn/ai-employees/)

### 面向 2.0 生态继续补齐关键能力

在 AI 主线之外，2.1.0-beta 也继续补充了面向实际业务场景的配置能力，包括：

- 签名字段
- JS Item 操作
- 自定义请求
- 中国行政区
- 树筛选区块
- 日历区块
- 看板区块

这些能力一部分面向数据和交互扩展，一部分面向中国本地化业务场景和常见业务界面建设，能够帮助团队更快完成具体系统落地。

## 这一版本适合谁

NocoBase 2.1.0-beta 特别适合以下几类用户优先体验：

- 正在探索 AI Agent 如何参与业务系统搭建的团队
- 希望提升实施效率、减少重复配置工作的交付团队
- 希望把 AI 引入业务流程、报表生成和任务执行环节的企业用户
- 正在评估 NocoBase 下一阶段 AI 能力路线的技术负责人和产品负责人

如果你已经在使用 NocoBase 2.0，并且希望进一步引入 AI 协作能力，那么 2.1.0-beta 提供了一个非常明确的升级方向：从“人工配置系统”逐步走向“人与 AI 一起搭建和运行系统”。
