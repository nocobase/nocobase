# 继承表

## 介绍

继承表是在普通表基础上的扩展，适合多个数据表共享一组公共字段，同时每个子表又有自己的专属字段。

比如先创建「资产」父表，保存资产编号、资产名称、购入日期、负责人等公共字段，再派生「电脑资产」「车辆资产」「办公家具」等子表。子表会继承父表字段结构，同时可以继续定义自己的字段。

:::warning 注意

继承表只能在主数据库为 PostgreSQL 时创建。其他主数据库、外部数据库、REST API 数据源和外部 NocoBase 数据源不支持继承表。

:::

## 适用场景

继承表适合这些业务场景：

- 资产父表派生电脑资产、车辆资产、办公家具
- 人员父表派生员工、外包人员、访客
- 事项父表派生任务、缺陷、需求
- 合同父表派生采购合同、销售合同、服务合同

适合使用继承表的前提是：这些对象有稳定的公共字段，而且子表之间的差异主要体现在少量专属字段上。

## 创建配置

在主数据库中点击「Create collection」，选择普通表或支持继承的创建入口时，可以通过 `Inherits` 选择父表。

![20240324085907](https://static-docs.nocobase.com/20240324085907.png)

| 配置 | 说明 |
| --- | --- |
| Inherits | 选择要继承的父表。当前数据表会继承父表字段结构，同时可以继续定义自己的字段。 |

继承表可以使用[普通表](./general.md)的区块和字段配置方式。对页面区块来说，它仍然是一个可以增删改查的数据表。

:::warning 注意

继承表适合结构高度相似的业务对象。如果不同对象的业务流程、权限和页面差异很大，拆成普通表再用关系字段连接通常更清晰。

:::
