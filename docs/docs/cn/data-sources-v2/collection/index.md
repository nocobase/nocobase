# 数据表

## 介绍

在 NocoBase 中，**Collection（数据表）** 是用来描述一类业务数据的数据模型。它不是单纯的数据库表名，而是 NocoBase 对一类数据的统一描述。

一个 Collection 通常定义三件事：

| 定义 | 说明 |
| --- | --- |
| 数据保存在哪里 | 数据可以来自主数据库表、外部数据库表、SQL 查询结果、数据库视图、REST API 资源或外部 NocoBase 应用。 |
| 有哪些字段 | 字段描述每条记录有哪些信息，比如客户名称、手机号、订单金额、创建时间、负责人。 |
| 如何被 NocoBase 使用 | 页面区块、权限、工作流、API 和关系字段都会基于 Collection 工作。 |

你可以把 Collection 理解为“业务对象的数据结构”。比如「客户」「订单」「合同」「任务」都可以是一个 Collection。

数据表创建或接入后，通常还需要继续做三件事：

- 配置字段，让数据表能保存业务需要的信息
- 在页面中[添加区块](../../interface-builder/blocks/index.md#添加区块)，让用户可以查看、录入和处理数据
- 配置权限、工作流和 API，让数据可以按业务规则被访问和流转

## 表结构类型

不同数据表类型适合不同的数据来源和业务结构。常见选择如下：

| 我想要…… | 选择 | 说明 |
| --- | --- | --- |
| 保存客户、订单、合同、工单、报销单、项目、任务等常规业务数据 | [普通表](./general.md) | 普通表是默认选择，适合大多数业务对象。 |
| 多个业务对象共享一组公共字段，又各自有专属字段 | [继承表](./inheritance.md) | 继承表只在主数据库为 PostgreSQL 时可用。 |
| 保存组织架构、商品分类、地区层级、部门目录等上下级数据 | [树表](./tree.md) | 树表使用邻接表结构保存父子关系。 |
| 保存合同附件、发票文件、产品图片、员工证件等文件元信息 | [文件表](./file.md) | 文件表保存文件元数据，文件本体由文件存储引擎保存。 |
| 把数据库里已有的 view 接入 NocoBase | [视图表](./view.md) | 视图表复用数据库侧已经定义好的查询逻辑。 |
| 把 SQL 查询结果作为数据表展示 | [SQL 表](./sql.md) | SQL 表不创建真实表，通常用于统计、汇总和报表。 |

## 主数据库与外部数据表的区别

主数据库和外部数据源都可以在 NocoBase 中作为 Collection 使用，不过能力边界不同。

| 能力 | 主数据库 | 外部数据库 |
| --- | --- | --- |
| 创建普通表 | 支持。 | 不支持。外部数据库只能接入已有表。 |
| 同步已有表 | 支持从主数据库同步已有表。 | 支持从外部数据库读取已有表和视图。 |
| 修改真实表结构 | 支持在 NocoBase 中创建、编辑和删除字段。 | 不支持。真实表结构由外部数据库侧维护。 |
| 创建继承表 | 仅 PostgreSQL 主数据库支持。 | 不支持。 |
| 创建树表 | 支持。 | 不支持。 |
| 创建文件表 | 支持。 | 不支持。 |
| 创建 SQL 表 | 支持。 | 不支持。 |
| 连接数据库视图 | 支持连接主数据库中的 view。 | 支持读取外部数据库中的 view。 |
