# Visningsläge

## Introduktion

Till skillnad från block har fältkomponenter tre visningslägen (endast fält inom ett formulär stöder detta). När ni växlar mellan olika visningslägen, motsvarar det olika fältkonfigurationsalternativ.

- Redigerbart;
- Skrivskyddat (ej redigerbart);
- Skrivskyddat (läsläge);

### Visningslägen för relationsfält

**Visningsläget för ett relationsfält** bestämmer hur fältet visas i gränssnittet och vilka fältkomponenttyper som är tillgängliga.

I **redigerbart tillstånd** stöder relationsfält flera komponenttyper, vilket gör att användare kan välja olika relationsfältskomponenter för att visa eller välja associerad data baserat på affärsbehov.

#### Relationsfältskomponenter i redigerbart tillstånd

![20251028220447](https://static-docs.nocobase.com/20251028220447.png)

I detta tillstånd kan användare flexibelt välja den lämpligaste visningsmetoden för att hantera data mer effektivt.

#### Relationsfältskomponenter i läsläge

När ni växlar till **läsläge** använder systemet automatiskt **titel-fältkomponenten** som standard för att visa den associerade datan. Detta passar bra för scenarier där ni bara behöver visa data utan att göra ändringar.
