# create-nocobase-app installation

:::tip Förutsättningar

- Node.js 20+, Yarn 1.22.x installerade
- Konfigurerat och startat en av de nödvändiga databaserna MySQL 8.0.17+, MariaDB 10.9+, PostgreSQL 10+
- På grund av den lokala nätverksmiljön rekommenderas ni att byta till en inhemsk spegel (ej obligatoriskt)

```bash
$ yarn config set disable-self-update-check true
$ yarn config set registry https://registry.npmmirror.com/
```

:::

:::warning Förklaring av miljövariabler

- `TZ` används för att ställa in applikationens tidszon, standard är operativsystemets tidszon
- `APP_KEY` är applikationens hemliga nyckel, som används för att generera användartokens m.m. (Om `APP_KEY` ändras blir gamla tokens ogiltiga). Det kan vara vilken slumpmässig sträng som helst. **Vänligen ändra till er egen hemliga nyckel och se till att den inte exponeras för utomstående**
- `DB_*` är databasrelaterade konfigurationer, vänligen ändra till er databasanslutningsinformation baserat på den faktiska situationen

:::

## 2. Byt katalog

```bash
cd my-nocobase-app
```

## 3. Installera beroenden

📢 På grund av nätverksmiljö, systemkonfiguration och andra faktorer kan detta steg ta ett tiotal minuter.

```bash
yarn install
```

:::tip Tips för produktionsmiljö

Vid driftsättning i produktionsmiljö kan ni installera endast nödvändiga beroenden för att minska storleken:

```bash
yarn install --production
```

:::

## 4. Installera NocoBase

```bash
yarn nocobase install --lang=zh-CN
```

Installationsprocessen skapar automatiskt databastabellstruktur och initialiserar data.

## 5. Starta NocoBase

**Utvecklingsmiljö**

```bash
yarn dev
```

**Produktionsmiljö**

```bash
yarn start
```

## 6. Logga in på NocoBase

Använd webbläsaren för att öppna [http://localhost:13000](http://localhost:13000), initialt konto och lösenord är `admin@nocobase.com` och `admin123`.

:::warning Tips för kontosäkerhet

Efter er första inloggning, vänligen ändra standardlösenordet omedelbart för att säkerställa systemsäkerheten.

:::
