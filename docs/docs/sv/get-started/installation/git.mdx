# Git-källkodsinstallation

:::tip Förutsättningar

- Installerat Git, Node.js 20+, Yarn 1.22.x
- Konfigurerat och startat nödvändig databas MySQL 8.0.17+, MariaDB 10.9+, PostgreSQL 10+ (välj en av dem)
- På grund av den inhemska nätverksmiljön föreslås att ni byter till en inhemsk spegel (ej obligatoriskt)

```bash
$ yarn config set disable-self-update-check true
$ yarn config set registry https://registry.npmmirror.com/
```

:::

## 1. Ladda ner NocoBase lokalt

Välj NocoBase-version ([Versionsjämförelse](../quickstart.md)), utför motsvarande kommando.

<Tabs>
<Tab label="Latest-version">

```bash
git clone https://github.com/nocobase/nocobase.git -b main --depth=1 my-nocobase
```

</Tab>
<Tab label="Beta-version">

```bash
git clone https://github.com/nocobase/nocobase.git -b next --depth=1 my-nocobase
```

</Tab>
<Tab label="Alpha-version">

```bash
git clone https://github.com/nocobase/nocobase.git -b develop --depth=1 my-nocobase
```

</Tab>
</Tabs>

## 2. Byt katalog

```bash
cd my-nocobase
```

## 3. Installera beroenden

📢 På grund av faktorer som nätverksmiljö, systemkonfiguration etc., kan detta steg ta ett tiotal minuter.

```bash
yarn install --frozen-lockfile
```

## 4. Ställ in miljövariabler

Miljövariablerna som krävs för NocoBase lagras i filen `.env` i rotkatalogen, ändra miljövariablerna efter den faktiska situationen.

```bash
TZ=Asia/Shanghai
APP_KEY=your-secret-key
DB_HOST=localhost
DB_PORT=5432
DB_DATABASE=postgres
DB_USER=nocobase
DB_PASSWORD=nocobase
```

:::warning Beskrivning av miljövariabler

- `TZ` används för att ställa in applikationens tidszon, standard är operativsystemets tidszon;
- `APP_KEY` är applikationens hemliga nyckel, som används för att generera användartokens etc. (om `APP_KEY` ändras kommer även gamla tokens att bli ogiltiga). Det kan vara vilken slumpmässig sträng som helst. Vänligen ändra till er egen hemliga nyckel och se till att den inte läcks ut externt;
- `DB_*` är databasrelaterat, om det inte är exempel-databastjänsten, vänligen ändra efter den faktiska situationen.

:::

## 5. Installera NocoBase

```bash
yarn nocobase install --lang=sv-SE
```

## 6. Starta NocoBase


**Utvecklingsmiljö**

```bash
yarn dev
```

**Produktionsmiljö (rekommenderas ej)**

Det rekommenderas inte att använda källkodsdistribution direkt i produktionsmiljö, för produktionsmiljö se [Produktionsdistribution](../deployment/production.md).

```bash
yarn build
yarn start
```

## 7. Logga in på NocoBase

Använd webbläsaren för att öppna [http://localhost:13000](http://localhost:13000), initialt konto och lösenord är `admin@nocobase.com` och `admin123`.

:::warning Säkerhetsmeddelande för konto

Efter första inloggningen, vänligen ändra standardlösenordet i tid för att säkerställa systemets säkerhet.

:::
