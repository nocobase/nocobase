# 人工处理

## 节点类型

`manual`
请使用以上 `type` 值创建节点，不要使用文档文件名作为 type。

## 节点描述
生成待办任务，等待指定人员处理并提交表单后继续流程（仅异步流程可用）。

## 业务场景举例
人工核对资料后再继续流程。

## 配置项列表
| 字段 | 类型 | 默认值 | 必填 | 说明 |
| --- | --- | --- | --- | --- |
| assignees | array | [] | 是 | 处理人列表。元素可以是用户 ID，或用户筛选条件对象（filter 结构）。 |
| mode | number | 1 | 否 | 多人处理模式：`0` 协作同一任务；`1` 全部通过；`-1` 任一通过；`0<mode<1` 表示投票阈值（如 0.6 表示通过率 >60% 才通过）。 |
| title | string | 节点标题 | 否 | 任务标题，支持变量模板。 |
| schema | object | 无 | 否 | 任务界面 UI Schema（由配置界面生成）。 |
| forms | object | {} | 否 | 表单配置（由配置界面生成），包括自定义/新增/更新表单等定义。 |

## 分支说明
不支持分支。

## 示例配置
```json
{
  "assignees": ["{{ $context.data.ownerId }}"],
  "mode": 0,
  "title": "{{ $context.data.title }} - 待处理"
}
```
