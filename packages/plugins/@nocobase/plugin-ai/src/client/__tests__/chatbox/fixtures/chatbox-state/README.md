# AI employee chatbox state fixtures

This directory contains stable fixtures for the AI employee chatbox state regression tests.

The `stable/` fixtures are derived from captured case records and, for a small number of cases, from source-level state contracts where the runtime condition was not reliably reproducible. They are intentionally smaller than the original records: timestamps, request identifiers, DOM details, long response bodies, and other volatile capture noise are removed.

The retained fields focus on state behavior that must remain stable across the client state migration:

- chatbox window flags, current employee, model, readonly state, and draft sender state
- conversation list state, current conversation, segmented mode, search keyword, and web search flag
- per-session message state, including message role, loading flag, content length, tool calls, attachments, context items, workflow state, and sub-agent conversations
- session-level loading, abort, background work, resume failure, attachment, context, system message, and skill setting state
- tool-call decisions and chat-tools indexes
- workflow task list, filters, current task, readonly state, and unread counts
- selection and dialog hide/resume state
