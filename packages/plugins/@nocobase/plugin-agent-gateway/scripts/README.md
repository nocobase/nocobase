# Agent Gateway release gate

The release gate has one orchestrator, `agent-gateway-release-gate.ts`, plus the retained terminal smoke daemon used by focused tests. The old seed, denial, browser-action, and REST-route scripts were removed because they duplicated setup and called contracts that are no longer authoritative.

## Prepare a fresh execution

From the plugin directory:

```bash
yarn acceptance:prepare \
  --evidence-dir /tmp/agent-gateway-release \
  --required-providers codex,claude-code,opencode
```

Preparation refuses to reuse an existing `context.json`. It creates:

- `context.json`: the fresh execution ID, start time, provider release matrix, and current clean Git branch/commit.
- `api-paths.json`: canonical resource-action paths generated by the shared API contract helpers.
- `evidence-template.json`: a non-passing template for live evidence collection.

Preparation refuses a dirty worktree. Verification must run on the same branch and commit with a clean tree.

## Collect real outputs

Do not hand-edit the template into a passing manifest. The consolidated collector converts raw Agent Browser, real-provider, WebSocket-denial, and multi-instance outputs into anchored evidence envelopes:

```bash
yarn acceptance:collect \
  --evidence-dir /tmp/agent-gateway-release \
  --source-dir /tmp/agent-gateway-live-source
```

The source directory must contain:

```text
context.json (when transferred as the agent-gateway-live-source artifact)
anchors.json
browser/<admin|union|restricted>/snapshot.json
browser/<admin|union|restricted>/action.json
browser/<admin|union|restricted>/console-errors.json
browser/<admin|union|restricted>/network.har
browser/<admin|union|restricted>/*.(png|jpg|webp|mp4|webm)
providers/<codex|claude-code|opencode>.json
websocket/<missingTicket|forgedTicket|replayedTicket|crossRun|crossOrigin>.json
multi-instance/file.json
multi-instance/terminal.json
findings.json (optional)
```

Prepare `context.json` before starting Agent Browser/provider/WS collection. Every raw JSON/HAR source must contain that context's `executionId` and a `capturedAt` timestamp at or after `context.startedAt`; transferred source artifacts must include the same `context.json`. The collector rejects stale or cross-execution raw files instead of stamping them into the current execution.

Browser `action.json` files must explicitly declare `status: "passed"` and contain every current anchor. Browser snapshots must contain the current `runId`, and each HAR must contain that run ID in its parsed JSON. `console-errors.json` uses an `errors` array inside its provenance envelope. Provider files must explicitly declare `status: "passed"`, `source: "real"`, a provider, run ID, and provider session ID. WebSocket files must explicitly declare a passing result and the exact expected code or HTTP status. Multi-instance files must declare `status: "passed"`, all current anchors, and distinct `sourceInstanceId` and `targetInstanceId` values. Optional `findings.json` uses a `findings` array inside the same provenance envelope.

The collector rejects fixture markers and fixture-backed paths, refuses to overwrite an existing `evidence.json`, copies media/HAR files into the evidence directory, and writes current execution/anchor envelopes only after validating the raw outputs.

## Required live evidence

`evidence.json` must contain:

- non-empty current `nodeId`, `runId`, `skillVersionId`, provider, and provider session anchors;
- admin, union-role, and restricted-role browser results with matching anchors, snapshots, media, and explained console errors;
- explicit provider results for Codex, Claude Code, and OpenCode;
- missing, forged, replayed, cross-run, and cross-origin WebSocket denial results;
- multi-instance file and terminal results, or an explicit tested single-instance fail-fast declaration;
- all Critical and Important findings marked resolved.

A provider marked `passed` must have `source: "real"`, a run ID, a provider session ID, and evidence files. Adapter fixtures are separate automated checks and never satisfy a real-provider requirement. A provider that is not part of the release matrix must be marked `skipped` with a reason.

Every declared evidence path is resolved beneath the current evidence directory. Symlinks, directories, empty files, escaping paths, fixture paths, and malformed JSON/HAR fail the gate. Snapshot/action envelopes must contain all current anchors; provider, WebSocket, and multi-instance envelopes must contain their asserted run/session/code/status values.

Ticket denial cases must report `TERMINAL_STREAM_TICKET_SCOPE_MISMATCH`. Cross-origin upgrade denial must report HTTP 403. Generic socket close or JSON parse failures do not pass the gate.

## Verify

Run the source-wide canonical alias scan independently while Task 11 boundary changes are in progress:

```bash
yarn acceptance:scan
```

Then run the full gate:

```bash
yarn acceptance:verify \
  --evidence-dir /tmp/agent-gateway-release \
  --scope full
```

Scopes:

- `full`: repository binding, source-wide canonical alias scan, canonical contracts, packaging/bootstrap, Task 12/14/15 feature suites, API logging/security/uploads, focused daemon/client/server suites, multi-instance file and `terminalStreamCluster` suites, stress/backpressure, and provider adapter fixtures. The alias scan rejects DTO fallback expressions and raw resource-action paths across application code and path-sensitive tests; only raw provider-log normalizers are excluded.
- `static`: ESLint, diff check, build, and package inventory only. Other required checks are recorded as skipped, so the summary remains incomplete.
- `none`: evaluate supplied evidence without running commands. This is useful only when importing trusted automated results into a higher-level orchestrator; missing checks remain incomplete.

The command always writes:

- `summary.json`: machine-readable results, warnings, skips, package contents, and release-gate status.
- `report.md`: human-readable report.
- `logs/*.log`: command output for every executed automated check.

`status` becomes `complete` only when every required automated check and live evidence gate passes. Use `--allow-incomplete` only for evidence-producing CI runs; it does not change the summary status.

Vitest's internal skipped-test counts are parsed from command output. A required suite that exits successfully but contains skipped tests remains incomplete and records the skip details in `summary.json` and `report.md`.
