## Was ist NocoBase

NocoBase ist eine Open-Source-Plattform fuer KI + No-Code, mit der sich Geschaeftssysteme schneller aufbauen lassen. Anstatt alles von Grund auf neu zu erzeugen, arbeitet die KI auf einer in der Praxis bewaehrten Infrastruktur und einer visuellen WYSIWYG-Oberflaeche auf. So verbinden Sie Geschwindigkeit mit Zuverlaessigkeit.

## Besondere Merkmale

### 1. Kollaborativ: KI und Menschen bauen gemeinsam

Coding-Agents arbeiten mit vollstaendiger CLI und Skills, waehrend Menschen eine visuelle WYSIWYG-Oberflaeche nutzen. So entsteht effiziente Zusammenarbeit auf derselben Basis.

#### Arbeiten Sie mit den KI-Coding-Agents, die Sie bereits kennen

Von der Bereitstellung bis zum lauffaehigen System in wenigen Minuten, mit gaengigen Agents aus dem Alltag.

- Funktioniert mit gaengigen Agents wie Claude Code, Cursor, Codex, OpenCode und TRAE
- Agents koennen Setup, Entwicklung, Migration und Release von Anfang bis Ende uebernehmen

![coding-agent](https://static-docs.nocobase.com/coding-agent.png)

#### Bauen Sie manuell in einer WYSIWYG-No-Code-Oberflaeche

Auch ohne KI koennen Teams Systeme visuell aufbauen und direkt weiterentwickeln.

- Wechseln Sie mit einem Klick zwischen Nutzungsmodus und Konfigurationsmodus
- Pruefen und konfigurieren Sie Datenmodelle, Seiten, Workflows und Berechtigungen visuell
- Entwickelt fuer Fachanwender, nicht nur fuer Entwickler

![wysiwyg](https://static-docs.nocobase.com/wysiwyg.gif)

#### Kombinieren Sie KI-Entwicklung und manuelles Bauen ganz nach Bedarf

Teilen Sie die Arbeit flexibel auf: Menschen verfeinern, was KI erstellt, und KI setzt auf menschlichen Konfigurationen auf.

- KI kann Datenmodelle, Seiten und Workflows schnell erzeugen
- Menschen koennen UI und Interaktionen schnell verfeinern
- Arbeiten Sie nach Bedarf zusammen und entwickeln Sie das System laufend weiter

![ai-no-coding](https://static-docs.nocobase.com/ai-no-coding.png)

### 2. Intelligent: KI hilft beim Betrieb, nicht nur beim Aufbau des Systems

NocoBase bringt KI-Mitarbeiter mit, damit KI direkt innerhalb des Systems arbeiten kann.

#### KI-Mitarbeiter in Geschaeftsablaeufe integriert

KI-Mitarbeiter erhalten den Geschaeftskontext automatisch und fuehren Aufgaben direkt im System aus.

- Im Frontend helfen sie bei Analysen, Frage-und-Antwort, Formularen und mehr
- Im Backend uebernehmen sie automatisch Dokumentenerkennung, Risikoueberwachung und Aufgabenrouting
- In Workflows integriert, koennen KI-Mitarbeiter Entscheidungen und Ausfuehrung mit uebernehmen

![AI-employee](https://static-docs.nocobase.com/ai-employee-home.png)

#### Offene Schnittstellen fuer das Agenten-Oekosystem

MCP, HTTP-APIs, CLI und umfangreiche Skills ermoeglichen eine sichere Anbindung externer Agents.

- Plattformen wie OpenClaw, Hermes, Dify, Coze und n8n verbinden sich ueber Standardprotokolle
- Verbunden mit Telegram, WhatsApp, Slack und Gmail koennen Daten abgefragt, Aktionen ausgeloest und Geschaefts-Workflows ausgefuehrt werden
- Ein gemeinsames Schnittstellenmodell haelt interne und externe Agents innerhalb derselben Grenzen

![agents](https://static-docs.nocobase.com/f-agents-logos.jpeg)

#### Rechtekontrollen halten das Verhalten der KI unter Kontrolle

Jede KI-Aktion folgt denselben fein abgestuften Berechtigungen wie menschliche Nutzer.

- Jeder KI-Mitarbeiter hat eine eigene Rolle mit Lese- und Schreibrechten bis auf Feldebene
- Audit-Logs machen jede Datenaenderung und jeden Workflow-Trigger nachvollziehbar
- Admins koennen KI-Berechtigungen jederzeit anpassen, damit die Grenzen klar bleiben

![permission](https://static-docs.nocobase.com/f-permission.png)

### 3. Zuverlaessig: fertige Infrastruktur fuer das reale Geschaeft

Datenmodelle, Rechte und Workflows sind komplex und fehleranfaellig.  
NocoBase liefert sie als integrierte Infrastruktur, in der Praxis getestet und in Produktion bewaehrt.

#### Vollstaendige Infrastruktur, ohne bei null zu beginnen

Dutzende integrierte Module decken die haeufigsten Anforderungen an Geschaeftssysteme bereits ab.

- Data models, permissions, workflows, and audit logs work out of the box
- Proven in production, instead of regenerated as black-box code each time
- Built-in guardrails keep AI output aligned with the system architecture

![core](https://static-docs.nocobase.com/f-core.png)

#### Datenmodellgetrieben, mit klarer Trennung von Daten und UI

Geschaeftsdaten bleiben in standardisierten relationalen Strukturen und sind sauber von der Oberflaeche getrennt.

- Use the main database, external databases, and third-party APIs as data sources
- AI and people work on the same data model, so results stay transparent
- Your data always stays in your own database, without platform lock-in

![model](https://static-docs.nocobase.com/model.png)

#### Plugin-Architektur fuer nachhaltige Weiterentwicklung

Mit einem Microkernel-Design laesst sich das System ueber Plugins erweitern, ohne an Kontrolle zu verlieren.

- New features are added through composable plugins with shared conventions
- Mix custom and official plugins to fit your business
- The same architecture applies to both AI-built and manually built plugins

![plugins](https://static-docs.nocobase.com/plugins.png)

## AI-Agent-Anbindung

Wenn ein AI Agent direkt an Aufbau und Bedienung von NocoBase mitarbeiten soll, ist der einfachste Weg: NocoBase CLI installieren, die Initialisierung abschliessen und danach die Agent-Session in diesem Arbeitsverzeichnis starten oder neu starten.

- Das NocoBase CLI ist fuer Installation, Verbindung und Verwaltung von NocoBase-Anwendungen zustaendig
- Bei der Initialisierung installiert das CLI automatisch NocoBase Skills, damit der Agent Datenmodelle, Seiten, Workflows, Berechtigungen und Plugins versteht
- Nach der Initialisierung kann der AI Agent direkt arbeiten, solange sein Workspace auf dieses Verzeichnis zeigt

```bash
npm install -g @nocobase/cli@beta
mkdir my-nocobase && cd my-nocobase
nb init --ui
cd my-nocobase && codex
```

Mehr dazu:  
https://docs.nocobase.com/de/ai/quick-start

## Installation

NocoBase unterstuetzt drei Installationsmethoden:

- <a target="_blank" href="https://docs.nocobase.com/de/welcome/getting-started/installation/docker-compose">Mit Docker installieren (empfohlen)</a>

  Diese Methode eignet sich am besten fuer No-Code-Szenarien und erfordert kein Schreiben von Code. Fuer Upgrades muessen Sie nur das neueste Image laden und neu starten.

- <a target="_blank" href="https://docs.nocobase.com/de/welcome/getting-started/installation/create-nocobase-app">Mit create-nocobase-app installieren</a>

  Der Business-Code Ihres Projekts bleibt vollstaendig unabhaengig und eignet sich gut fuer Low-Code-Entwicklung.

- <a target="_blank" href="https://docs.nocobase.com/de/welcome/getting-started/installation/git-clone">Aus dem Git-Quellcode installieren</a>

  Wenn Sie die neueste noch unveroeffentlichte Version ausprobieren oder direkt am Quellcode mitarbeiten, aendern und debuggen moechten, ist diese Methode empfehlenswert. Sie setzt staerkere Entwicklungskenntnisse voraus, und Sie koennen Aktualisierungen spaeter per Git abrufen.
